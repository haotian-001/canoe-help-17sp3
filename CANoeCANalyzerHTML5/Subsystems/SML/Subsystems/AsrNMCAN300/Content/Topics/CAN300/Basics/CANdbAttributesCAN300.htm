CANdb Attributes

 

Introduction

The communication matrix of a CAN network is usually described by a CANdb database. It defines the nodes present on the network, and allows specification of their behavior by attributes.

Network attributes are assigned to the database as a whole, whereas node attributes define one particulate network node.

CANdb attributes are used to define NM parameters that are belonging to a certain CANoe configuration. Alternatively, a specific INI file may be used. Defining attributes in the database or in such a specific INI file allows multi-bus scenarios with different parameters.

Caution!

The mandatory (non-optional) attributes have to be defined and set correctly for the NM to work properly! The optional attributes may be absent, if the default values (from the INI file or computed by the DLL/SO) should be used.

Required Attributes

The following attributes are required by the node layer modeling library and not included in the INI file by default. They must be defined either in the database or in of the INI files, or the measurement cannot be started!

For the meaning of the global parameters and node specific parameters, see the corresponding pages.

Note

The CAN ID of the NM message a node sends is computed as
CAN ID := NmAsrBaseAddress + NmAsrNodeIdentifier

Only CAN messages with
NmAsrBaseAddress <= CAN ID < NmAsrBaseAddress + NmAsrMessageCount
are considered to be NM messages.

Optional Attributes

The following parameters are optional both in the database and in the INI files. Missing optional parameters will be initialized with computed default values. The INI file overwrites the values defined in the database.

Defining these attributes in the database allows multi-bus scenarios with different parameters.

For the meaning of the parameters, see the descriptions for INI file.
