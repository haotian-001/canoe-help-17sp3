CAPL Functions

Additional Functions

These functions are implemented in the modeling library and can be called in CAPL programs.
Note that all functions start with the prefix Nm_.

Return Values

All functions returning long values indicate the success by returning
1 (OK), or a positive value indicating the requested information.
0 indicates that the action could not be executed; exceptions are described at the functions.

CAPL Callback Functions

Introduction

The CAPL program has to define these optional functions (marked with the postfix Ind or Notification) to receive indications about NM events like the reception of a PDU, or a state transition.
