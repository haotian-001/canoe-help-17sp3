Introduction

Since in FlexRay, a schedule has to be defined before the network can start to operate, it is necessary to “reserve” the slots used by the transport protocol during the initialization phase, i.e. in the on prestart handler.

Alternatively, it is possible to define TPL or DIAGNOSTICS PDUs in the FIBEX database. These will be recognized and “reserved” automatically during the start of the measurement.

The CAPL program has to create the connection it intends to use, receiving a connection handle for each one. This handle must be given to every function that performs any change to the configuration of the connection or starts the transfer of data. In addition, CAPL callback functions will receive the handle to indicate the connection creating events like the reception of data.

CAPL Functions

Additional Functions

These functions are implemented in the DLL and can be called in CAPL programs.
Note that all functions start with the prefix FrTP_.

Return values

All functions returning long values indicate the success by returning 0 (no error), or a positive value indicating the requested information, e.g. the connection handle. Exceptions are indicated at the functions.

Functions

Short Description

FrTP_Set2ndAddresses

Sets optionally 2nd ECU addresses that are used for incoming or outgoing connection data.

FrTP_SetBufferSize

Configures the way a sender should send data to this node.

FrTP_SetConnMode

This function allows to change the mode of a connection, where connMode may have different values.

FrTP_SetAddresses

Set addresses (C_AI) used in the C-PDUs to the values given.

FrTP_SetBandwidthControl

Configures the way a sender should send data to this node.

FrTP_SetFillByte

Defines the fill byte for padding un-used bytes in the C-PDUs. If the function is not called, then 0 is used.

FrTP_SetMaxPDULength

The transport protocol will send data in slots with the specified maximum C-PDU length filling undefined bytes with the fill byte specified by FrTP_SetFillByte( ).

FrTP_SetTimingParams

Defines some timing parameters that handle the distance between two C-PDUs for transmission or reception (timeout).

CAPL Callback Functions

Introduction

The CAPL program has to define these optional functions (marked with the postfix Ind) to receive indications about TP events like the reception of data or a protocol error. It is highly recommended to implement at least the error indication function.
