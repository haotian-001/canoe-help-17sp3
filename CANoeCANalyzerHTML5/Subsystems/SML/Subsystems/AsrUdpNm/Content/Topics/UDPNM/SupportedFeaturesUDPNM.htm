Supported Features

 

Feature

Short Description

Feature selection from INI file

Global settings for timeouts, counters, PDU format, and advanced features, additional node configuration can be specified in an INI file for every network. Additional INI files can be created for specific CANoe configurations.

Communication control

It is possible to enable and disable the sending of NM messages, as necessary for the ISO14229 Communication Control service.

Passive mode

A simulation node can be put into passive mode, i.e. though it will not send any NM messages, the NM state is still available.

Immediate restart

If activated in the INI file, requesting the network during Prepare-Bus-Sleep will send a NM message immediately.

Active wake-up bit

A particular bit in the NM message is set to 1 for the node whose request leads to a network start.

Partial networking

Nodes can be configured to compose partial networks or interact with them, by receiving and sending partial network requests within the NM messages. These partial networks can be woken and shut down individually.

Car wake-up

It is possible to define a car wake-up bit which triggers a special indication.

Spontaneous transmissions

The sending of NM messages can be triggered spontaneously, e.g. to transmit user data without delay.

Immediate NM messages

If activated, nodes will send multiple NM messages in short intervals when in certain situations.

PDU format

It is possible to configure the length of the NM PDU and the positions of the NM node identifier and the control bit vector in the payload. Also, the position of car wake-up and partial networking bits can be defined if these features are applied.

User data

The non-protocol bytes of the message can be used for the transfer of user data.
