INI Files - Global Parameters

 

The following parameters are applied to the whole network. They can be defined in each of the possible INI files.

Note

Parameters without a default value must be specified, unless they are defined in a DBC database.

Section

Parameter

Description

Default value

[FEATURES]

NM_PN_ENABLED

NM_PN_INFO_LENGTH

Length of the PN information in bytes (max. 8).

NM_PN_REQUEST_MASK

—

[FEATURES]

NM_PN_VALID_MASK

Binary filter mask for partial networks, where 0 suppresses the generation of system variables for the specific partial network and thus suppresses the display of the according partial network in the NM panel.

(all PNs active)

[FEATURES]

NM_TX_RX_MSG_ID_COMPUTATION_METHOD
  • 0: Tx NM message ID = NM base address + NM node identifier; Rx NM messages defined by range.
  • 1: Tx NM message and properties (FDF, BRS, DLC) derived from data base; Rx NM messages defined by range.
  • 2: Tx NM message ID = NM base address + NM node identifier; Rx NM messages derived from database.
  • 3: Tx NM message and properties (FDF, BRS, DLC) derived from data base; Rx NM messages derived from database.

0

Reduced NM message period for the immediate transmissions feature.

[TIMER]

NM_PN_RESET_TIME

Defines how long a PN is considered to be still requested after an NM message with bit set for the according PN (only relevant for indications and EIRA calculation).

2000 ms

[TIMER]

NM_REMOTE_SLEEP_IND_TIME

If a NM node has not received a NM message from another node for this long, it will signal Remote Sleep to the application.

1350 ms

[TIMER]

NM_REPEAT_MESSAGE_TIME

Defines how long the node stays in state Repeat Message.

3200 ms

[TIMER]

NM_TIMEOUT_TIME

In state Ready Sleep, the NM will wait this long before entering Prepare Bus-Sleep.

2000 ms

[TIMER]

NM_WAIT_BUS_SLEEP_TIME

In state Prepare Bus-Sleep, wait this long before entering Bus-Sleep.

1500 ms

[TIMER]

T_SLEEPTIME

Sleep timer for coupling between NM and interaction layer.

4000 ms

Note

The user data will start directly after the node identifier or the Control Bit Vector (whichever is last) in the message and run until its end!

Caution!

The node identifier and the control bit vector cannot occupy the same byte in the message! If their value is <0 or >= the message length, the respective parameter will NOT be sent!

The same applies to car wake-up bit and PN info range. Additionally, both of them must be located within the user data.

If no car wake-up bit position respectively PN info range is specific, the particular feature is completely deactivated!
