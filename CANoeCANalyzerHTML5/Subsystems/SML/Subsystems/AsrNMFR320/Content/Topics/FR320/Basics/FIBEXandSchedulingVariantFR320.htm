FIBEX Definition / Scheduling Variant

 

FIBEX Definition

On FlexRay, the communication matrix is defined by a FIBEX database. In it the schedule of the slots in the FlexRay cycle are defined, and also send/receive relations to network nodes.

For every node that should participate in the NM, including nodes that are not simulated by CANoe, the NM PDUs it sends have to be defined with type NM (= 1).

If a node does not send NM PDUs, it will be considered passive, i.e. it cannot participate in the NM itself.

Scheduling Variant

The AUTOSAR FlexRay Network Management specification (chapter 7.9 Schedule details) describes 7 scheduling variants that can be implemented on a network. Two of them are not recommended.

The CANoe node layer modeling library can follow any of the 5 relevant variants, where the variant is recognized automatically from the configuration:
