INI Files - Introduction

 

Main INI File

On measurement start, the modeling library will read a number of parameters from the file AutosarFlexRayNM32.INI (when using Windows runtime kernel) or libAutosarFlexRayNM32.INI (when using Linux runtime kernel) which is distributed along with this modeling library.

Note

The file is searched for in the same directory from which the AutosarFlexRayNM32.dll (when using Windows runtime kernel) or libAutosarFlexRayNM32.so (when using Linux runtime kernel) is loaded!

Specific INI Files

Besides the main INI file, a specific INI file may be created for each FlexRay network in a CANoe configuration.The usage of such files is recommended, because the node layer modeling library needs to read parameters which depend on the nodes in the CANoe configuration. Also, they may be used to override global settings, thus allowing multi-bus scenarios with different parameters.

Note

The file is searched for in the same directory as the CANoe configuration CFG file and must obey the naming rule:
[Network]_AUTOSARFlexRayNM32.INI, where [Network] denotes the name of the target FlexRay network in the configuration.

The naming of the specific INI file is identical for Windows as well as Linux runtime kernel.

INI file Loading Sequence

Any of the two INI files is sufficient to run a configuration using AUTOSAR FlexRay NM, as long as it holds all necessary parameters. However, if both are present for the loaded configuration, definitions in the specific INI file prevail.

Note

On measurement start, have a look at the Write Window to see which INI files were loaded for your configuration.
