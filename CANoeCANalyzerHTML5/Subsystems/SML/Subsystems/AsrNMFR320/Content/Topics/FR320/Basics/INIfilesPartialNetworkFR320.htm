INI Files - Usage of Partial Networking

 

Introduction

This section provides more information about configuration and usage of the partial networking (PN) feature.

Configuration Rules

To use partial networking in your configuration, these rules must be followed, by editing the INI files accordingly:

  • A filter mask and a request mask must be available for every node that shall use PN (either through global default or in local scope). If one or both of them are missing, NM_PN_ENABLED will have no effect on this node.
  • The length of filter and request masks must always match NM_PN_INFO_LENGTH, i.e. (NM_PN_INFO_LENGTH * 8) bits must be supplied. Binary representation must be used, e.g.:
    NM_PN_FILTER_MASK=10001010
  • The position of the partial networking info range (denoted here as the request bits) must be correctly configured using the respective parameters.
  • If NM_PN_ENABLED is set, then usage of the CBV is mandatory, i.e. a valid value must be supplied for NM_PDU_CBV_POSITION.

Note

A node may belong to multiple partial networks, and also request multiple partial networks. As for the latter, the parameter NM_PN_REQUEST_MASK only denotes the behaviour upon measurement start. The request bits may then be changed by CAPL code (Modification of User Data). PN membership, however, as defined by NM_PN_FILTER_MASK, is considered a constant.

Notes for PN Usage

  • Though FlexRay clusters cannot be partially deactivated, the moment when a partial network is not requested any more is indicated with FrNm_PnInactiveInd() (see).
  • As the PN request bits reside in user data, the CAPL function Nm_SetUserData should be used with care as it can disturb the PN request bits.
