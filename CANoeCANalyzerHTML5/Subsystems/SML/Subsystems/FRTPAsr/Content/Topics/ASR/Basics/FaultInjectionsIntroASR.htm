Fault Injections - Introduction

 

Introduction

The following functions (all beginning with the prefix FrTPFI_) allow, before a transmission or reception is starting, to define some miss-behavior for the TP DLL per connection. Thus, failures can be injected for the C-PDUs that are to be sent by the DLL in order to test the peer.

 

Categories

The following categories of failures exist:

  • Delay a Tx C-PDU
    The sending of any C-PDU will additionally be delayed.
  • Double a Tx C-PDU
    Any C-PDU will be transmitted twice, whereas the doubled C-PDU can optionally be delayed.
  • Drop a Tx C-PDU
    Any C-PDU will not be sent on the bus, although the DLL assumes that it was sent.
  • Miss-interpret a Rx FC or ACK-RETRY
    Any received Flow Control or Acknowledge Frame can be miss-interpreted.

Thereby Data Frames (DF: STF, CF and LF) are distinguished versus Flow Control (FC) and Acknowledge (ACK_RET) C-PDUs.

 

Restrictions

Each of the functions can define exactly one failure. Thus, it is not possible to define e.g. the dropping of the FF and the CF simultaneously.

But when the failure was injected (e.g. the FF was dropped), a new one (dropping the CF) can immediately be defined.

But, any two different functions can be used in order to define simultaneously different fault injections (e.g. to define dropping a Data Frame n and doubling any other Data Frame m; n ≠ m).
