Error Codes

 

Error Codes returned for CAPL Calls and their Meanings

Code

Short Description

-19

 No such connection, i.e. creation of a connection failed or the given handle is unknown.
-18 The connection is known, but has been closed in the meantime.
-17 The function is not yet implemented.
-16 CAPL function received incorrect argument.
-15 CAPL function may only be called from the appropriate callback function.
-14 Function cannot be called for the addressing mode it is applied on.

-13

This is not possible from within a callback function.

-12

Fault Injection is not active.

0

OK, no error

Text

Code

Short Description

1

 Timeout waiting for Consecutive Frame.
2 Timeout waiting for Flow Control Frame.
3 (not reported to CAPL)
4 Busy, only one transmission possible at the same time.
5 Received unexpected PDU.
6 Timeout while trying to send a CAN message.

7

Too many Flow Control Wait Frames sent.

8

Receiver buffer overflow, i.e. a Flow Control Overflow Frame was received.
9 Wrong parameter, e.g. it was tried to send 0 bytes.
10 An invalid flow status was received in an expected Flow Control Frame.

11

The data transfer has been aborted explicitly from the CAPL program.

12

No data has been provided for transmission in time, cf. Small Buffer Handling.
