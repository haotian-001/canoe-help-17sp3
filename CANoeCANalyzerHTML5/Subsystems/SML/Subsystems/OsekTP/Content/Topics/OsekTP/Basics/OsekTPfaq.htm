Frequently Asked Questions

 

This section collects some of the most commonly encountered problems and suggests solutions.

FAQ

Question
Can a normal mode connection use extended CAN message IDs?

Answer
Yes. When setting the Rx and Tx ID values, set the most significant bit of the ID to 1. Refer to CAN Message IDs for details.

FAQ

Question
In the Write Window the following message is printed: Optional callback function … not found or invalid argument list!

Answer
The callback function is not defined in the node, or its signature is not as defined in this help. Please refer the list of callback functions and their signatures.

FAQ

Question
How do I access more than one CAN network in a gateway node?

Answer
In a gateway, the bus context controls the CAN network messages are sent and received on. The CAPL program can change the bus context by calling the appropriate functions; please refer to the CANoe CAPL Help for details.

FAQ

Question
How can I set the flow status a receiver node sends in Flow Control messages?

Answer
Please refer to section Connection Level Configuration for details.

FAQ

Question
Event though the receiver indicates that CANoe as sender may send with a STmin value < 1 ms, Consecutive Frames are sent with 1 ms delay. For example, the receiver may indicate a STmin value of 0xF1 for 100µs.

Answer
In order to limit the process load on the CPU, CANoe uses timers with a resolution of 1 ms. It is possible to change the resolution of CANoe timers in the CAN.INI file:

[PCBOARD]
TimerRate=100 ; Resolution of time management in 10 us

Set this value to 25 for a timer resolution of 250 µs.

Note: This will cause higher process load by CANoe!
