CanTpGet/SetAckMode/Result

 

Note

ISO TP specifies an unacknowledged data transfer, i.e. the sender does not get any confirmation that the receiver has received all of the data correctly. An extension to this standard behavior can be activated with these functions – please note that all communication partners have to implement this extension!

Function Syntax

long CanTpGetAckMode(long connHandle);

long CanTpSetAckMode(long connHandle, dword mode);

long CanTpGetAckResult(long connHandle);

long CanTpSetAckResult(long connHandle, dword result);

Description

A receiver will send an acknowledgment Flow Control Frame (FC.Ack, new type) to the transmitter after it has correctly received the last data frame (Single Frame or Consecutive Frame) expected. If an error occurred, the receiver will NOT send this frame, i.e. the transmitter will run into the Flow Control timeout.

The transmitter node waits for the FC.Ack after it has sent the last data frame. If no such frame is received within the N_Bs timeout, the transmission error is reported with a call to the CAPL function CanTp_ErrorInd(2).

Parameters

Format of FC.Ack
  • PCI byte #1 FlowStatus 0xF
  • PCI byte #2 AckRes 0 (default, can be overwritten)
  • PCI byte #3 etc.: (reserved) 0

Activate or deactivate (default) the acknowledged data transfer, or retrieve the current setting. When acknowledged transfers are active, the receiver can set the value of AckRes to sent, while the transmitter can retrieve the last AckRes value received.

  • 0: Do not use acknowledged transfers (default)
  • 1: Use acknowledged transfers as defined above
  • Other: Reserved

Return Values

—

Example

Example 1

On the press of the <Space> key, activate acknowledged transfers for the database connection and send data to the receiver. Print the result returned by the receiver for successful data transfers, or report an error if no acknowledgment was received.

On key ' ' {
  BYTE data[20];
  CanTpSetAckMode( CanTpGetDBConnection(), 1);
  CanTpSendData(CanTpGetDBConnection(), data, elcount(data));
}

CanTp_SendCon( long handle, dword count) {
  write( "Successful transfer of %d byte on connection %d: %d"
  , count, handle, CanTpGetAckResult( handle));
}

CanTp_ErrorInd( long handle, long error) {
  if(CanTpGetAckMode( handle) == 1 && error == 2)
    write( "No ACK for data on connection %d!"
    , handle);
}

Example 2

Transfer of a single frame (normal addressing) with acknowledgment.

Id Data bytes
441 [05] 53 68 6f 72 74 00 00 // Single Frame, 5 byte
442 [3f 00 00] 00 00 00 00 00 // FlowControl.Ack, AckRes = 0
