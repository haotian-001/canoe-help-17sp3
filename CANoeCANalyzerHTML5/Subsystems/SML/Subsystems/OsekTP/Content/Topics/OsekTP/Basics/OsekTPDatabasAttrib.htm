Database Attributes

 

Configuration from Database

The same mechanisms as for the OSEKTL are used, but only activated if the database connection is retrieved (cf. CanTpGetDBConnection).

In the CAN database (CANdb) assigned to the network, node attributes are evaluated for the database node assigned to the simulation node. All attributes are of numeric types.

If the attribute definition is not consistent, e.g. some mandatory attributes are not defined, the CanTp API will not return a database connection handle.

Attributes evaluated depending on Addressing Mode

Depending on the addressing mode (attribute TpAddressingMode), these attributes are evaluated.

Database Attributes

Please refer to the table above to determine the attributes necessary for a particular addressing mode!

Name

Values

Description

TpAddressingMode
  • 0: Normal
  • 1: Extended (based)
  • 2: NormalFixed
  • 3: Mixed
  • 4: Mixed (11 bit)
  • 5: Extended (free)

This attribute determines the addressing mode used by the database connection, and which other attributes will be considered. See CanTpCreateConnection.

TpRxIdentifier

 Valid CAN ID

CAN ID the node listens to for TP frames. See CAN Message IDs.

TpTxIdentifier

 Valid CAN ID

CAN ID of the TP frames the node sends. See CAN Message IDs.

TpTargetAddress

 0 — 255

Value of the target address byte sent by the node. See ECU and Target Addresses.

DiagStationAddress
(Diagnose- Stationsadresse)

 0 — 255

Address of the node. Note that the second attribute is only evaluated if the first one is not defined! See ECU and Target Addresses.

TpAddressExtension

 0 — 255

Value of the address extension byte. See Address Extension.

TpNodeBaseAddress

 Valid CAN ID

First CAN ID used for TP. See Extended (Based) Addressing Mode.

TpTxAdrMode
  • 0: Physical
  • not 0: Functional

Select protocol field value. See Physical / Functional Addressing.
