INI Files

 

OSEKNM.ini

[LINEMODE]
Chips = 0; ; 0 - both, 1 - chip 1 only, 2 - chip 2 only

[WAKEUP]
WakeUpOnAllMsg = 1      ; 0 - Wakeup from NMBusSleep only by NM messages
                        ; 1 - wakeup from NMBusSleep by any message

[MESSAGE]
DlcLen = 8              ; length of NM message in bytes (minimum 2, maximum 8 (CAN) or 64 (CAN FD))
RingDataStartIndex = 2  ; minimum 2
RingDataLength = 6      ; length of ring data in bytes
RingDataFill = 0

M_MESSAGE_FDF = 0       ; 0 - NM message is sent as classic CAN message
                        ; 1 - in case of a CAN FD network, the NM message is sent with FDF = 1

NM_MESSAGE_BRS = 0      ; 0 - NM message is sent with BRS = 0
                        ; 1 - in case of a CAN FD network and in case that FDF = 1, the NM message is sent with BRS = 1

[WRITE]
VERBOSE = 1   ; verbose level
              ; 0 - no messages
              ; 1 - some messages
              ; 2 - more messages
              ; 3 - even more messages

[NAMES]
CALLBACKFunc1 = apBusSleep
CALLBACKFunc2 = apCanSleep
CALLBACKFunc3 = apCanNormal
CALLBACKFunc4 = apCanWakeUp
CALLBACKFunc5 = apCanOff
CALLBACKFunc6 = apCanOn
CALLBACKFunc7 = apIndRingData

[TIMER]
; all timers are given in milliseconds
T_RING_TYP    = 100   ; Typical time interval between two ring messages
T_RING_MAX    = 250   ; max. time interval between two ring
T_NOTLAUF     = 1000  ; time interval between two ring messages with NMLimpHome identification
T_BUSSLEEP    = 1500  ; Time the NM waits before transition into the state NMBusSleep
T_LIMPHOME    = 4000  ; Time the NM stays in LimpHome before transition into the ; state NMBusSleep

Note

The layout of the NM message can be configured. This configuration of the message layout is done in the section [Message]. First of all, the DLC of the message is configured by the attribute DlcLen . The position and length of the ring data is configured with RingDataStartIndex and RingDataLength The value RingDataStartIndex indicates the start of the ring data. First byte is indicated with 0, but 0 and 1 shall not be chosen, because this byte contains the destination address resp. the NMPDU information. RingDataLength is the size of the ring data. The example above configures 6 bytes of ring data, starting from byte 2 to 7. In the [TIMER] section the DirectNMParams could be configured. This is done statically here in the INI file instead of using the function InitDirectNMParams.
