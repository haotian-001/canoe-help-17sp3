CANdb Attributes

 

Note

The following information is only relevant for DBC databases.

The modeling library obtains its data from the associated database. The data is contained in the corresponding attributes. In order for OSEK network management to function properly, the database needs to include the attributes described below. The attribute names should be entered exactly as shown.

Attributes Related to the Database

  • NmBaseAddress
  • NmMessageCount

A specific number of ECUs can participate in a ring’s network management. The identifiers of the network management messages are placed in a fixed area, so that N consecutive identifiers are reserved. By default, the maximum number of nodes N is set to 64, but this can be changed as needed using the NmMessageCount database attribute. If this attribute is used, the number of ECUs should always equal 2m, where m is an integer (giving rise to e.g. N=16, 32,...).

A general prerequisite is that the base address always represents an integer multiple of the number of nodes. The base address is specified using the NmBaseAddress attribute.

Example

Identifiers 400h — 43fh are reserved for network management. The CANdb database attributes are then:

  • NmBaseAddress= 400h
  • NmMessageCount = 64
  • NMIdType (enumeration)
    This attribute defines if standard or extended IDs are used. Valid values are:
  • 0: standard (11 bit, default)
  • 1: extended (29 bit)
  • NmCAN (integer)
    Using this attribute it is possible to specify the CAN channel on which the NM should send and receive. Note that this attribute is only taken into consideration in older versions of CANoe or if the compatible mode of a newer version is used.

Attributes Related to the Node

  • NmStationAddress
  • NodeLayerModules
  • NmCAN

The NM number of the ECU is specified in NmStationAddress. This address is derived directly from the relevant identifier of the corresponding network management message. The attribute is a numerical field.

NmStationAddress = NMNODE(CAN identifier) — NmBaseAddress

Example

NMNODE(CAN identifier) = 412h, NmBaseAddress = 400h:

NmStationAddress = 18

Note

The node layer modules attribute should be defined as a string field. It contains the name of the module library, i.e. OSEKNM01.vmodule.
