Overview

Distinctions

Regarding the interface functions, a distinction is made between API functions (CAPL Functions), which are made available by the NM modeling library and can be called from within CAPL, and CAPL Callbacks, which are defined in CAPL and called by the NM modeling library.

CAPL Functions

Overview [▲ back]

The network management modeling library has a number of API functions. Both OSEK names and names from earlier modeling library versions are listed on the page New and Old Names of Functions. Please use the functions with their new names because the old names may not be supported in future.

NM Configuration [▲ back]

Some functions are deprecated and should not be used any more. See: New and Old Names of Functions.

Functions

Short Description

CmpConfig

Checks whether all ECUs in the target configuration are participating.

GetConfig

The current configuration shows which ECUs are actively participating.

NMGetAwakeConfig

This function is used to query the number of awake nodes.

NMGetConfig

The elements in the field flags are set to 1, if the node corresponding to the element index participates in the NM, to 0 otherwise.

NMGetStatus

This function returns the network management status.

NMSetAutoStartParam

This function defines the behavior of NM at measurement start.

NMSetConfigMask

Edits all ECUs that are relevant to the target configuration.

NMSetTargetConfig

Specifies the desired configuration.

NMSetTimerBusSleep

This function sets the value for the time the NW waits.

NMSetTimerError

This function sets the value for the time interval.

NMSetTimerRingMax

This function sets the value for the maximum time interval.

NMSetTimerRingTyp

This function sets the value for the typical time interval.

NMSetVerbosity

This function sets the modeling library’s verbosity.

NMStartOffline

No NM messages are sent at measurement start.

NM Control [▲ back]

Functions

Short Description

GotoMode_Awake

This function can be used while network management is running to reverse the ready-to-sleep status of the application.

GotoMode_BusSleep

This function indicates to network management that the application is ready to sleep.

NMCtrlSimulationOff

This function deactivates the NM simulation completely.

NMCtrlSimulationOn

This function reactivates the NM simulation.

ReadRingData

This function writes the received ring data to the field.

SilentNM

This call results in network management no longer participating in the ring.

TalkNM

Activates network management, enabling participation in the ring.

TransmitRingData

This function enters the transmitted bytes into the node’s NM message as ring data.

CAPL Callback Functions

Overview [▲ back]

The following functions can be made available inside a CAPL node. These are called by the network management modeling library during specific state changes. The function names can be assigned freely. The names are specified in the [NAMES] section of the OSEKNM.ini file. The CAPL functions are described below.

Functions

Short Description

apIndLastAwake

This function is called when the node is the last active node on the bus that does not signal sleep readiness.

apIndPreSend

Directly before sending a NM message on the bus, this function is called to allow the CAPL program to change the actual message sent.

apBusSleep

The function is called during the transition to sleep mode directly after the apCanSleepfunction call.

apCanNormal

This switches the CAN transceiver to active state.

apCanOff

This function is called during the transition into sleep mode.

apCanOn

Network management is reactivated and CAN output is released.

apCanSleep

The function is called during the transition to sleep mode directly before the apBusSleepfunction call.

apCanWakeUp

This function is called when the network management modeling library receives a message in sleep mode.

apIndRingData

This function is called when the node receives the token and the ring is stable.
