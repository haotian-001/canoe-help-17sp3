Write Window Noifications (Selection)

 

Verbose Level

Description

0

OSEK NM: Configuration error! No node layer was generated for node — check database, attribute NodeLayerModules!

A DLL function was called in an invalid context.

<node>: Base addresses in CANdb: not found or invalid!

The NMBaseAddress… attribute was not found in the database.

Duplicated ID detected : id= …

Two or more nodes appear to have the same NM station address, which is not allowed. This may result from missing or incorrect attribute values. Please make sure, that each node on a bus has a unique NM station address.

Node address <NmStationAddress> in CANdb: not found or invalid!

There is a problem with the NM definitions in the database, i.e. no NM station address was found because the attribute was not defined or not set correctly. Refer to CANdb Attributes for details.

<node>: Callback function... not found:

The callback function (possibly specified in the INI file) is not defined in CAPL.

<node>: Callback function... has invalid signature:

The signature of the callback function (possibly specified in the INI file) in CAPL does not match the expected signature.

1

<node>: WakeUp by msg: …

A message was received in sleep mode (not a network management message). The callback function apCanWakeUp is called.

<node>: WakeUp by NM message, state=…

An NM message was received in sleep mode. The callback function apCanWakeUp is called.

<node>: Abort TWaitBusSleep by NM message, state=…

An NM message was received while waiting for the sleep timer to elapse. The callback function apCanOn is called.

<node>: Appl: Bus Awake

Network management was in sleep mode and was woken by the function call GotoMode_Awake.

OSEK NM: ERROR Project INI-File OSEKNM.INI doesn’t exist!

The INI file was not found. Default values will be used instead.

2

<node>: NM enter Sleep Mode ECU-No.

Network management goes to sleep mode.

<node>: apCanSleep ECU-No.

The callback function apCanSleep is called. This message appears also in case that the callback is not implemented.

<node>: apBusSleep ECU-No.

The callback function apBusSleep is called. This message appears also in case that the callback is not implemented.

<node>: apCanNormal ECU-No.

The callback function apCanNormal is called. This message appears also in case that the callback is not implemented.

<node>: Appl: NM Silent

The function SilentNM was called.

<node>: Appl: NM Talk

The function TalkNMwas called.

<node>: Sent Alive

An alive message will be sent.

3

<node>: NM enters LimpHome state via start phase in ECU-No.

Network management is entering limphome.

<node>: SetSleep Ack Bit ECU-No.

This ECU is sending SleepAck.

OSEK NM: Projectfile is OSEKNM.INI

The INI file was found and read. After this, the values found in the TIMER section are printed to the Write Window.

4

<node>: unknown NM-Command received in ECU-No.

An invalid network management message was received.

Please refer to NMSetVerbosity for setting the verbosity level.
