Structure of Network Management Messages

 

The network management message that is sent out by the modeling library has the following layout:

  • Data byte 1: Receiver address.

For the alive and limphome messages the node‘s own station address is found here. This station address is also entered here for the first ring message, provided that no other NM message could have been received between the alive message and this ring message. Otherwise the address of the logical successor within the ring will be located here.

  • Data byte 2: Identifies the type of NM message and contains sleep information.

  • Bit 1: alive message
  • Bit 2: ring message
  • Bit 3: limphome message
  • Bit 4: Reserved
  • Bit 5: Sleep readiness (SleepIndicator)
  • Bit 6: Sleep acknowledgment (SleepAcknowledge)
  • Bit 7: Reserved
  • Bit 8: Reserved
  • Data byte 3: Contains cause for wake-up of network management.

    At the measurement start the wake-up cause 01 (PowerOn) is entered for each node in the CANoe modeling library. If the bus is awakened by a CAN message after it has completely reached sleep mode, the value 02 is automatically entered as the wake-up cause.

    If a CAPL node calls the GotoMode_Awake function from the sleep state, the wake-up cause 04 (internal event) is set for this node.
  • 00 : no entry
  • 01 : power ON
  • 02 : wake-up via CAN bus
  • 03 : wake-up caused by external event
  • 04 : wake-up caused by internal event


  • Data byte 4: Reserved for expansions. Default value is 0xFF (hex).
