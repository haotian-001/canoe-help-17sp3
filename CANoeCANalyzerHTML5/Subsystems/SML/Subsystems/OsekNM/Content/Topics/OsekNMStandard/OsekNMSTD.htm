OSEK Network Management

 

When a CAPL node is used to simulate an ECU, it needs to participate in network management. To avoid having to generate this functionality for each individual node, network management functions are implemented as a CANoe node layer modeling library. This serves to make the API functions of the network management module available in almost exactly the same way as in the ECU itself. It also ensures that relevant CAN messages are sent in the same way as they would have been sent from the ECU.

This documentation describes the steps required to fully integrate the OSEK network management modeling library into the CANoe environment.

Used Files

The network management consists of the following files:

  • OSEKNM01.vmodule
  • DMOSEKNM.vmodule

The module description file (*.vmodule) references the required DLL/SO and INI file. The modeling library is a node layer modeling library, which means it is loaded only once, but a different data set is available for each node.

These files are normally located in the global add-on installation directory („%PROGRAMDATA%\Vector\AddOn Packages\Vector AddOn\vModules“). For more, see the links in the sections below.

Sample Configuration

CANoe is provided with a sample configuration of OSEK NM vModule for free:

  • DMOSEKNM.vmodule

The DMOSEKNM.vmodule file references the required DMOSEKNM.dll/libdmoseknm.so and dmoseknm.ini file

The sample configuration has the following limitations:

  • A copyright message is always printed in the Write Window.
  • The timer T_RING_TYP is fixed at 300 ms.
  • The timer T_RING_MAX is fixed at 500 ms.
  • Only the station addresses 26, 27, 28 and 29 may be used.
