AddWindowCapture
Function creates a screen capture of a Graphics, Statistics, Data, Trace or Panel window. The test case verdict is not affected. Window capture is not allowed in standalone mode. Errors on creating the window capture are reported as a warning in the test report.
FileName
Function sets the name of the report file using the user interface. To be effective, the function must be called in a test module/test unit before every other CAPL function that relates to the report and executed before every test case. If the report file name is modified, it is not be possible to open it via the CANoe user interface (test run dialog). Note: To be effective this function must be called before any other reporting or test function.
SetEngineerInfo
