Vector CANoe .NET API for test modules

Vector.CANoe.TFS Namespace
Contains the API for the CANoe Test Feature Set for test modules
Classes
Class Description
AbsoluteCycleTimeCheckT
The AbsoluteCycleTimeCheck class allows to verify that the message cycle time is in an expected range. The range is specified in absolute terms. The check can be used with global scope in a test module/test unit or it can be used in test cases. Checks cannot be used as static variable.
Assert
Internal class. You should never use this type.
Check
Check class. Can be used as Condition, Constraint or Observation in combination with the Criterion classes Checks can be used with global scope in a test module/test unit or they can be used in test cases. Checks cannot be used as static variable.
Diagnostics
Diagnostics functionality of a test module/test unit.
DlcCheckT
The DlcCheck class allows to monitor if the DLC of a message is as defined in the network database. The DlcCheck is not available for user defined frames but the UserCheck class can be used to implement such a check. The check can be used with global scope in a test module or it can be used in test cases. Checks cannot be used as static variable.
ExpectedAttribute
Expected attribute for test commands. The Expected attribute is used to check simple value conditions within test commands.
Initialize
Test command Initialize
InputAttribute
Input attribute for test commands. The Input attribute is used to set values within test commands.
OccurenceTimeCheckT
The OccurenceTimeCheck class allows to monitor if the message cycle time is in an expected range. The range boundaries are calculated from the message delay and cycle times. This check can be used for cyclic messages that can asynchronously restart their send cycle. The OccurenceTimeCheck is not available for user defined frames. The check can be used with global scope in a test module/test unit or it can be used in test cases. Checks cannot be used as static variable.
OccurrenceCountCheckT
The OccurrenceCountCheck class allows to monitor if the message occurrence is in an expected range. The range boundaries are calculated from the message delay and cycle times. This check can be used for cyclic messages that can asynchronously restart their send cycle. The OccurrenceCountCheck is not available for user defined frames. The check can be used with global scope in a test module/test unit or it can be used in test cases. Checks cannot be used as static variable.
RelativeCycleTimeCheckT
The RelativeCycleTimeCheck class allows to monitor if the message cycle time is in an expected range. The range is specified relative to the message cycle time. The RelativeCycleTimeCheck is not available for user defined frames. The check can be used with global scope in a test module/test unit or it can be used in test cases. Checks cannot be used as static variable.
Report
Reporting functionality of a test module.
ReportMiscInfoList
A list of key value pairs, used by AddMiscInfoBlock. Contains the information of the information block.
StateChange
Test command StateChange
StructuredTestModule
If you'd like to define a test module that predefines a test module structure, derive from this class instead of TestModule. The structure information defined in the StructuredTestModule is shown in the CANoe execution dialog and allows to enable test cases and test groups individually. The test execution dialog will either display the test case method name or the test case title provided with the TestcaseAttribute. The dialog will not show the test case title when it is provided within the test case.
TestCaseAttribute
Mark a method with this attribute, and it will be handled as a test case.
TestFunctionAttribute
Mark a method with this attribute, and it will be handled as a test function.
TestGroupAttribute
Mark a method with this attribute, and it will be handled as a test group. In structured test modules test cases called in such a test groups are not individually disabled. A test group like this should be used when the test case execution sequence is dynamic, e.g. it depends on user interactions, signals states or test results and thus the structure before running the test group is unknown.
TestModule
If you'd like to define a test module, derive from this class instead of from Script or MeasurementScript.
TestModuleBase
Internal class. You should never use this type.
TestResetNamespaceDistObjValues
Allows the user to reset an object, namespace or member
UserCheck
UserCheck class. A user check can be used as Condition, Constraint or Observation in combination with user defined check functions. Checks can be used with global scope in a test module/test unit or they can be used in test cases. Checks cannot be used as static variable.
Validation
Validation functionality of a test module/test unit.
ValueCheckT
Represents a simple check for a value of a signal, environment variable or system variable. Checks can be used with global scope in a test module/test unit or they can be used in test cases. Checks cannot be used as static variable.

Interface Description
ICheck
Interface of a check. A check observes a criterion while a test sequence will be executed. The interface can be used for a faster notation of value checks. Checks can be used with global scope in a test module/test unit or they can be used in test cases.
