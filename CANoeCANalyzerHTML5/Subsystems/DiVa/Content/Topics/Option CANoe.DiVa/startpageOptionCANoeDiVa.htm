Option DiVa

 

Note

The Option DiVa is not installed!

The Option DiVa (Diagnostic Integration and Validation Assistant) provides you an automated test of the implementation and integration of the diagnostic protocol in ECUs.

Properties and Advantages

DiVa enables the automatic generation of test cases and their specification with comprehensive test coverage. The test cases generated in this way are based on ECU diagnostic descriptions in ODX or CANdela format and can be implemented as automated test cases in CANoe. You can choose the test scope yourself and generate an automatic test report. Various diagnostics standards (UDS, KWP2000, GMW3110, OBD) as well as manufacturer-specific interpretations are supported.

DiVa expands CANoe into a tool for automatic generation and execution of reproducible test cases for the implementation and integration of the diagnostic protocol in the ECU and thus saves users time and money while increasing the quality of the ECU diagnostic software.

Application Areas

Automotive suppliers require comprehensive tests during development as well as numerous regression and release tests. Tests for integration and release are also required by automotive manufacturers.

DiVa is thus designed for use by automotive manufacturers and suppliers in the following areas:

  • Function and software development
  • Test lab and vehicle integration

On the automotive supplier side, DiVa supports the implementation of the diagnostics in the ECU. The developer tests during the development phase and performs complete regression tests at the end. These can also be used within the framework of a release procedure of the diagnostic part in the ECU.

Automotive manufacturers use DiVa for automated incoming tests of delivered ECU samples. Integration tests on the development installation or in the vehicle are supported. Automotive manufacturers simply create test specifications for quality requirements or for test documentation. Automotive manufacturers and suppliers can expand the test scope to include new test patterns or modify existing test patterns to meet their particular requirements.
