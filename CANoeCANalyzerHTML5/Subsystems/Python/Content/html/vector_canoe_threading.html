set ( ) ¶

Signals the user event. Consequently resolves a possibly-active wait condition on this event. If such a wait condition is active, the corresponding code will be executed immediately. The current function is therefore interrupted and the code will be executed at a later point in time after the set method call.

This method cannot be called on background thread without wait() on the same UserEvent object. Will also signal all events with the same id in the current test module.

This method signals all user events with the same ID. :returns: a SignalingErrorCode with the following values:

  • SUCCESS, Event was signaled successfully

  • FAILURE, Event could not be signaled

Return type

SignalingErrorCode

wait ( time_out_ms : int ) ¶

Waits for the set method to be called on the event instance. Should this event not occur before the expiration of the time time_out_ms, the wait condition is resolved nevertheless.

Parameters

time_out_ms ( int ) – The timeout in milliseconds after which the wait ends. A value of 0 disables the timeout for this wait.

Returns

A WaitingErrorCode with the following values:

  • CONSTRAINT_VIOLATION, Resume due to constraint violation

  • GENERAL_ERROR, General error, for example, functionality is not available

  • TIMEOUT, Resume due to timeout

  • EVENT_OCCURRED, Resume due to event occurred (value has been reached)

Return type

WaitingErrorCode

class WaitingErrorCode ( value ) ¶

Bases: enum.IntEnum

Return codes of waiting functions of this module.

The following values are possible:

  • JOIN_ERROR, Error joining events to the event stack

  • CONSTRAINT_VIOLATION, Resume due to constraint violation

  • GENERAL_ERROR, General error, for example, functionality is not available

  • TIMEOUT, Resume due to timeout

  • EVENT_OCCURRED, Resume due to event occurred (value has been reached)

wait_for_all_joined_events ( joined_events : list , timeout : int ) → int ¶

Waits for a list of “joined events.” Only accepts events as returned from the join functions in this module. The wait condition is resolved if all of the joined events were signaled. Should not all events occur before the expiration of the time aTimeout, the wait condition is resolved nevertheless.

Parameters

  • joined_events ( list [ Callable [ [ ] , int ] ] ) – The events to wait on.

  • timeout ( int ) – Maximum time that should be waited [ms]. A timeout of 0 waits indefinitely.

Returns

a WaitingErrorCode return code if the wait failed, the index of the final resolving event otherwise. Indices start at 1.

Return type

int

Waits for the answer to a method call. The call must have been created through call_async on a consumed method.

wait_for_any_joined_event ( joined_events : list , timeout : int ) → int ¶

Waits for a list of “joined events.” Only accepts events as returned from the join functions in this module. Each individual of these events can resolve the wait state. Should none of the events occur before the expiration of the time aTimeout, the wait condition is resolved nevertheless.

Parameters

  • joined_events ( list [ Callable [ [ ] , int ] ] ) – The events to wait on.

  • timeout ( int ) – Maximum time that should be waited [ms]. A timeout of 0 waits indefinitely.

Returns

a WaitingErrorCode return code if the wait failed, the index of the resolving event otherwise. Indices start at 1.

Return type

int

wait_for_change ( value_entity , timeout : int ) → vector.canoe.threading.WaitingErrorCode ¶

Waits for the next change of a value entity. This is equivalent to calling wait_for_change_count_greater(value, value_entity.get_change_count(), timeoutMs).

Note that only changes which occur while the function waits are considered. If you want to wait for changes which may occur earlier, or if you want to wait for several changes, use wait_for_change_flag() or wait_for_change_count_greater() .

Parameters

  • value_entity – Value entity of a communication object or distributed object.

  • timeout ( int ) – Timeout in milliseconds.

Returns

a WaitingErrorCode return code

Return type

WaitingErrorCode

wait_for_change_flag ( value_entity , timeout : int ) → vector.canoe.threading.WaitingErrorCode ¶

Waits for the change flag of a value entity to be set. Each value entity has a change flag which is set when the value changes and reset through an explicit call to value_entity.clear_change_flag().

Use this function instead of wait_for_change() if an undetermined number of changes may occur between a point where you reset the flag and a point where at least one change must have occurred.
