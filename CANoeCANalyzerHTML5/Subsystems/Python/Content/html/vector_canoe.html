vector.canoe ¶

The package vector.canoe contains the general parts of the Python API.

class InOutParameter ( value = None ) ¶

Bases: vector.canoe.OutParameter [ vector.canoe.T ], Generic [ vector.canoe.T ]

This class holds a value for an inout-parameter of an application layer object’s method. It inherits the “value” property from the OutParameter-class and additionally has a constructor that accepts an initial value of the parameter. E.g.: 

//vcdl
namespace myNamespace {
  interface myInterface {
    internal method void myMethod(inout uint32 inout_parameter);
  }
  myInterface myObject;
}

#Python
inout_parameter = vector.canoe.InOutParameter(1)
application_layer.myNamespace.myObject.myMethod.call(inout_parameter)
vector.canoe.write(f"new value: {inout_parameter}")

See also the properties and methods of the base class OutParameter

class ValueStateEnum ( value ) ¶

Bases: enum.IntEnum

Indicates whether a value has been measured. It is returned by the value_state property of an application_layer’s value entity. E.g.: 

//vcdl
namespace myNamespace {
  interface myInterface {
    internal data uint32 myDataMember;
  }
  myInterface myObject;
}

#Python
if application_layer.myNamespace.myObject.myDataMember.value_state == vector.canoe.ValueStateEnum.OfflineValue:
   vector.canoe.write("myDataMember not yet measured")

measurement_script ( class_ ) ¶

This decorator registers the class that should be instantiated during initialitation of the CANoe measurement. Their “initialize”, “start”, “stop”, “shutdown” methods will be called if they are implemented.

on_any_called ( if_provided_or_internal_method_member ) ¶

This decorator registers the method as on_any_called-handler for the passed interface method member. It must be used for methods of the class decorated with measurement_script() only. The decorated method should have the following parameters (and no return type):

  • self

  • the DO whose method is called, of the interface type the decorator parameter belongs to

  • the call context of the called method, for access to parameters and return values

E.g. (for the interface type application_layer.myNamespace.myInterface): 

@vector.canoe.measurement_script
class MyModel:

   @vector.canoe.on_any_called(application_layer.myNamespace.myInterface_objects.myMethod)
   def on_myMethodMember_any_called(self, my_if_obj: application_layer.myNamespace.myInterface, my_method_member):
       vector.canoe.write(f"the called DO was {str(my_if_obj.path)} with param {str(my_method_member.param.copy())} and result {str(my_method_member.result.copy())}")
Parameters

if_provided_or_internal_method_member – a provided or internal interface method member. E.g. application_layer.myNamespace.myInterface.myInterface_objects.myMethod

on_call ( do_provided_or_internal_method_member ) ¶

This decorator registers the method as on_call-handler for the passed provided or internal method member. It will be called whenever the function given in the decorator parameter should be simulated on the provider side. It must be used for methods of the class decorated with measurement_script() only. The decorated method should take self and the same parameters as the function prototype and have the same return type. E.g.: 

@vector.canoe.measurement_script
class MyModel:

   @vector.canoe.on_call(application_layer.myNamespace.myObj.myMethod)
   def on_myMethod_call(self, param, *args):
       vector.canoe.write(f"received param {param}, returning {param - 1} {str(*args)}")
       return param - 1
Parameters

do_provided_or_internal_method_member – a provided or internal method member of a DO. E.g. application_layer.myNamespace.myObj.myMethod
