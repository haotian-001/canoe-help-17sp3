Vector.Diagnostics Namespace
 
Classes
Class Description
Application
The Application is the starting point for each script. The Application class has access to the loaded ECUs of the scripting host via the application 'environment' (i.e. CANape, CANoe, etc.; e.g. Write window, CANape device or CANoe target, ...).
ClearDtcResult
Result of clearing the Dtcs of an Ecu.
DiagnosticException
The exception that is thrown when the conversion of a parameter value fails.
Dtc
Represents a DTC in an Ecu.
DtcCollection
Strongly typed collection for Dtcs.
DtcInformation
Additional information about a Dtc.
DtcInformationCollection
Strongly typed collection for DtcInformations.
Ecu
The ECU represents a diagnostics server. Each ECU references a diagnostics description that defines the diagnostics services. Within the application the communication channel is configured to allow access to the ECU via a network.
Parameter
Represents a parameter of a message (request/response).
ParameterCollection
Provide a strongly typed collection for Parameters
ReadDtcResult
Result of reading the Dtcs of an Ecu.
ReadEnvironmentDataResult
Result of reading the environment data of a Dtc.
Request
Provides a diagnostic request message.
RequestCollection
Provide a strongly typed collection for Requests.
Response
Provides a diagnostic response message.
SendResult
The result of the service communication transaction. Contains the transaction status (e.g. error state) and the response object if available.
Value
To access a parameter's value.
