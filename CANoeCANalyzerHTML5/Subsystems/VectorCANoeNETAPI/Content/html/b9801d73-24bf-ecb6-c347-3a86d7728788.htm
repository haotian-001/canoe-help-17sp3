Remarks
The criterion has a "Signaled" state which can be set from the background thread. Signaled is on creation of the criterion. It will automatically be reset to whenever a call to Execution.Wait evaluates the Signaled state.

You can join the criterion with other criteria in an instance of the Criterion class.

A UserEventCriterion instance can't be used simultaneously in several wait calls.
Examples
Copy
 1private static void UserEventCriterion()
 2{
 3  // create a user event (initially not signaled)
 4  var userEvent = new UserEventCriterion("Event 1");
 5  // create and start a BG thread
 6  var bgThread = new System.Threading.Thread(() =>
 7  {
 8    // simulate work by sleeping, then signal the event
 9    System.Threading.Thread.Sleep(200);
10    userEvent.Signaled = true;
11    System.Threading.Thread.Sleep(200);
12    userEvent.Signaled = true;
13  });
14  bgThread.Start();
15  // wait for the event to be signaled
16  Execution.Wait(userEvent);
17  // Combine the event with another criterion
18  var joinedCriterion = new Criterion();
19  joinedCriterion.AddMandatory(userEvent);
20  joinedCriterion.AddMandatory(new ValueCriterion<NetworkDB.MotorControl.OnOff>(1));
21  // wait for both sub-criteria to be true
22  Execution.Wait(joinedCriterion);
23}
