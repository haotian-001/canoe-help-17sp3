Use Case 2

 

Suppose you want to merge two ARXML extracts (SYS-E) named sys-e_1.arxml and sys-e_2.arxml into one system description (SYS-D) named sys-d.arxml. Various clusters with different protocols are modeled in the two SYS-E files. For example, one CAN cluster (CANClusterName) and three Ethernet clusters (FirstEthernetClusterName, SecondEthernetClusterName, ThirdEthernetClusterName). Of these, you want to keep only the first and third Ethernet clusters named FirstEthernetClusterName and ThirdEthernetClusterName. Furthermore, the third Ethernet cluster is to be renamed to NewSecondEthernetClusterName.

Note

This more complex use case is only possible with a configuration file (Config), because filters and patches cannot be specified via the command line. It is not possible, for example, to specify the input-group and the output-file via the command line and also to create a configuration file containing filters and patches.

First, the SYS-E are merged into one overall model.

Then all defined Built-In filters are executed and lastly the defined Built-In patches.

version: 1.0
input-group:
- 'C:\Username\Desktop\sys-e1.arxml'
- 'C:\Username\Desktop\sys-e2.arxml'
output-file: 'C:\Users\username\desktop\result'
builtin-filters:
- filter-type: include-cluster
  protocol: Ethernet
  names:
  - FirstEthernetClusterName
  - ThirdEthernetClusterName
builtin-patches:
- patch-type: rename-cluster
  current-name: ThirdEthernetClusterName
  new-name: NewSecondEthernetClusterName

The input-group parameter contains the individual file paths to the SYS-E.

The output-file parameter contains the file path to the result ARXML (SYS-D).

All file paths must be enclosed in single quotes.

This is followed by the builtin-filters list, where only one filter of type include-cluster is specified.

This filter is parameterized with the protocol Ethernet and the names FirstEthernetClusterName and ThirdEthernetClusterName. This means that all clusters with the protocol Ethernet and the two names are kept. Related to the use case, the CAN cluster and the Ethernet cluster are filtered with the name SecondEthernetClusterName.

Then the defined patch of type rename-cluster is executed. This renames the cluster named ThirdEthernetClusterName to NewSecondEthernetClusterName, regardless of the protocol.
