Use Case 1

 

Suppose you want to merge two ARXML extracts (SYS-E) named sys-e_1.arxml and sys-e_2.arxml into one system description (SYS-D) named sys-d.arxml. Nothing is to be filtered or patched in the process.

There are two ways you can do this:

Example 1

You can parameterize the AUTOSAR Preprocessor directly from the command line, as shown in the following code block:

.\AUTOSAR-preprocessor.exe --input-group "C:\Users\Username\Desktop\sys-e1.arxml"
"C:\Users_username\desktop\sys-e2.arxml" --output-file "C:\Users_username\desktop\result"

The --input-group parameter specifies all file paths to the SYS-E files that are to be merged. The individual file paths must be enclosed in quotation marks and separated by a space.

The --output-file parameter specifies the file path to the result ARXML (SYS-D). The file path must be specified in quotation marks. If you specify the file path without a filename extension, the *.arxml filename extension is automatically added.

Example 2

The second possibility is the call with a configuration file. The following code block shows this call:

.\AUTOSAR-preprocessor.exe --config-file "C:\Users\Username\desktop\config.yaml"

In the command line, the AUTOSAR Preprocessor is called with the --config-file parameter and the file path in quotes to the valid configuration file.

The specified configuration must be in YAML file format. The content of the configuration file is shown in the following code block:

version: 1.0
input-group:
- 'C:\Username\Desktop\sys-e1.arxml'
- 'C:\Username\Desktop\sys-e2.arxml'
output-file: 'C:\Users\username\desktop\result'

The configuration file must specify the version of the file format. The version number in the example above is 1.0.

The input-group parameter is a list, so accordingly the individual file paths are noted as a YAML list. The file paths must be specified in single quotes.

Finally, the output-file parameter describes exactly the file path of the output file. Again, the file path must be enclosed in single quotes.
