Filter

 

You can use the filters in AUTOSAR Preprocessor to filter out specific elements from the ARXML files. The so-called Built-In filters are fixed predefined filters, which you can specify only via the configuration file. The following table contains an overview of all existing Built-In filters:

Name

Identifier

Parameter

Description

Cluster-Filter

exclude-cluster

protocol names

Filters out all clusters with the parameterized protocol and/or name.

Cluster-Filter

include-cluster

protocol names

Filters out all clusters that do not match the parameterized protocol and/or name.

Channel-Filter

exclude-channel

protocol names

Filters out all channels with the parameterized protocol and/or name.

Channel-Filter

include-channel

protocol names

Filters out all channels that do not match the parameterized protocol and/or name.

You can specify multiple Built-In filters, applying them sequentially from top to bottom.

The cluster filters and channel filters must be specified with at least one of the parameters.

The protocol parameter corresponds to the bus protocol and is an enumeration.

Note

The following values are allowed:

  • CAN
  • Ethernet
  • LIN
  • FlexRay

The filter is not applied for other values.

The names parameter corresponds to a list of cluster names or channel names, depending on the filter type.

See the Use Case 2 page for an example of how to specify the filter list as well as a concrete filter in the configuration file in YAML syntax.
