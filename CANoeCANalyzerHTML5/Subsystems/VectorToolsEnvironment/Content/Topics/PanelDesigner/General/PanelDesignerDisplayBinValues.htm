Displaying and Entering Binary Values

 

Binary display is supported in the Input/Output Box and the Combo Box in the Panel Designer. The Input/Output Box also supports the entry of binary values.

Symbols can be displayed and entered in binary format if their raw or physical value is an integer.

The Combo Box can only display raw values.

Displaying Raw Values in Binary Format

Depending on the symbol definition, integer values are displayed binary. Leading zeros are not displayed in this case.

Displaying Physical Values in Binary Format

  • Depending on the symbol definition, integer values are displayed binary. Leading zeros are not displayed in this case.
  • Symbols with a Signed Integer or Unsigned Integer data type can be displayed binary.
  • The physical value range of integer symbols with Factor = 1 and Offset = 0 corresponds exactly to the value range of the underlying data type. In this case the physical value is identical to the raw value.
  • However, if you define Factor and Offset values, this results in a physical value range that differs from the value range of the raw value. When physical values are displayed/entered binary, the number of significant places in the element is based on the physical value range.
    In the case of an unsigned integer symbol, this can result in an offset into a negative value range (see example).

Tooltips

Tooltips help users interpret the data on the panel. For detailed information see CANoe help page Panel Configuration: Settings.

Limitations

  • Symbols with a Double or Float data type and Integer symbols with a non-integer Offset or Factor cannot be displayed as binary values. The display (data interpretation) is automatically set to decimal in the Panel Designer Properties Window.
  • Physical values larger than integer 64 bit maximum or smaller than integer 64 bit minimum cannot be displayed. A yellow warning triangle appears after the element.
