Method Call Control

 

Use Case

The Method Call Control is used to call a method. You can set parameters and receive the corresponding return value.

CAPL Access

The control can be activated with the following CAPL functions:

  • With the function setControlVisibility, you can set, if the control is displayed during runtime or not.

Configuration

Assign a symbol to the control in the Panel Designer.

A method is always sent from the consumer to the provider. This means that the consumer sends the method and the provider responds. Accordingly, you can only connect the methods of the consumers side. To do this, drag the desired method to the Method Call Control.

You can assign a symbol to the control

  • by dragging and dropping it from the Symbol Explorer
  • by opening the corresponding Symbol Selection dialog:
  • click the links at the bottom of the Properties Window
  • under Symbol (once you have selected the symbol filter)
  • via the context menu

To enable you to adjust the size of the control optimally, the parameters are displayed immediately after linking to a method. The return values are only displayed at runtime of the panel.
It is possible that the parameters may take up more space at runtime than the Panel Designer displays. Therefore, always check the size at runtime.

You can configure the control via the ribbon or the Properties Window.

Display and Control Element

Request

At runtime of CANoe, you can set and send the parameters of the linked method on the Request page.

Click on the corresponding line and enter a value for the parameter. The [Send] button calls the method.

If the parameter is an array, you can restrict the display to certain elements. To do this, open the list of possible items for the Items element in the Value column and select the required items. All items in the array are always sent.

If a method contains optional parameters, the Send column is displayed in addition to the Parameter Name and Value column. By activating/deactivating the check box you determine whether the parameter is sent or not.
If the parameters of the method are mandatory, the Send column remains empty.

The last column shows you whether the value was sent or whether the value was edited and has not been sent yet .

The call of the method and the wait for the response are indicated the symbol on the [Send] button.

Response

At runtime of CANoe, the Response page displays the received return value (if available) and the values of the Out parameters (if available).

The Response page title shows you whether the contents of both pages corresponds to each other. If the answer has been received and the parameters on the first page have not been changed, will be displayed. As soon as the parameters are changed after sending, will be shown.

The control does not have a history. Only the last response to the method call sent by this control is displayed.
