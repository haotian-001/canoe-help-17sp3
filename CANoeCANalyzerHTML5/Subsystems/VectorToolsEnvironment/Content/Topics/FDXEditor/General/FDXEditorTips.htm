Tips and Tricks

 

Automatic Group Size

The FDX Editor can calculate the size of FDX groups automatically. This can be activated and deactivated either globally using the Automatic Group Size menu command (Home ribbon tab) or by setting the size of a group explicitly to the value 0.

The global setting can be recognized visually from the state of the icon in the ribbon or in the menu. The setting is changed by selecting the icon once and takes effect immediately on all groups. In so doing, the manually entered sizes are not overwritten. Rather, the editor shows the groups continuously with their calculated size. When the option is deactivated again, the previous sizes are thus assumed again. As soon as an FDX description is saved, however, the FDX description file then contains the calculated value so that the original value is lost when the FDX description is closed and re-opened.

To automatically calculate the size for only certain groups, you can set their size to 0, which causes the calculated value to always be displayed for these groups. Subsequent overwriting with a specific value greater than 0 turns off this behavior again. Even in this case, the calculated value is transferred to the description file when the FDX description is saved, in which case no other automatic calculations are made when the FDX description is re-opened.

The use of the global option or the group-specific setting depends on the respective application:

  • If you have specified a datagram layout from the perspective of the HIL system and this layout contains group sizes that do not match the automatic calculation, you should deactivate the global option.
  • If you create a new FDX description without limitations, the groups are typically initialized by drag and drop of symbols. The adding of new items then leads to errors because the group size is not automatically adapted. In this case, it is recommended that the global option be activated until you complete a first version of the protocol.
  • If you yourself take responsibility for the definition of the protocol, you can have the global option generally activated and always use the most compact groups sizes possible.
  • If you add new groups to a previously published definition or a definition that can no longer be modified for other reasons, it is recommended that the option be deactivated and the new groups be set locally to size 0. Alternatively, you can also use a separate FDX description for adding the new groups, because CANoe can manage multiple description files simultaneously.

The automatic calculation of the group size leaves all offsets of items untouched. Only the physical end of the last item is sought and this boundary is defined as the group size. To clean up gaps between items, an automatic layout (see below) can be applied to one or more groups.

Automatic Layout

With the Automatic Layout menu command (Home ribbon tab), you can completely reorganize the arrangement and extent of the contained items of selected groups.

The order of items in the editor or in the description file serves as the basis for the automatic layout of a group. The items are arranged in this order without gaps and the group size is set to the matching size (thus it is thereafter not automatically recalculated following further changes to the group if the global Automatic Group Size option (see above) is deactivated).

To achieve a specific layout, you can sort the items of a group in the Item Table as they should appear in the layout and then have the correct offsets and group size calculated by the automatic layout.
