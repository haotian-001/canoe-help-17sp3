Concepts

 

File Management

The FDX Editor works exclusively offline on FDX descriptions in the form of XML files. It thus cannot manipulate FDX descriptions that are already loaded in CANoe . However, it is possible to add files undergoing editing directly from the editor to an open CANoe configuration. When new files are saved, the editor automatically prompts the user to indicate whether the files are to be added to the current configuration.

The FDX description files are edited following the principle of the "Multiple Document Interface (MDI)". FDX descriptions are loaded into the editor, and a window opens for each file in which its data are edited.Through explicit saving, the manipulated contents are then written back to the file. Each window indicates the editing status with a * in the file name.

The editor can be used with or without a connection to a running CANoe application. For this reason, the editor treats the FDX descriptions as pure XML files without direct reference to the symbols defined in CANoe – these serve more as an aid for creating and manipulating FDX items.

Focus and Selection

Different areas of the FDX Editor display context-dependent data in order to reduce the amount of data having the focus and thus to strike a good compromise between ease of viewing and detailed information. It is important in this regard to distinguish between focus and selection, especially in the main window of the editor.

The focus can be changed in each table via a mouse click or keyboard navigation . A maximum of one cell (and thus implicitly one row) has the focus at any given time in any table. This focus is the deciding factor for which hierarchical data are displayed in the subordinate tools. For example, if a particular group (a row in the Group Table) has the focus, the Item Table displays the content of this group. If in this table, in turn, an item has the focus, the Symbol Properties tool displays the detailed information for the symbol of this item. The three graphic elements – Group Table, Item Table, and Symbol Properties – thus form a hierarchy, whereby the subordinate element having the focus in each case determines the content of the next subordinate element.

In addition to the focus, multiple lines can be selected in a table (using the mouse or keyboard by holding down the <Shift> key during selection or navigation). The set of selected elements primarily supports edit and drag and drop operations that can affect a set of groups or items. When groups are selected, the set of groups displayed in the graphic layout is also controlled. The focus does not play a role in the Symbol Explorer or in the Identifier List. Only the selected elements are affected by edit and drag and drop operations.

In the following example, the first two rows in the Group Table are selected and the focus is on the FDX data group of the second row. The items of the EasyDataWrite group are thus displayed in the Item Table on the right. The first three rows in this table are selected and the first row has the focus. The different colors in the display indicate that the Item Table currently has the input focus. Thus, for example, a "Cut" command would remove the first three items from the group and place them in the clipboard.
