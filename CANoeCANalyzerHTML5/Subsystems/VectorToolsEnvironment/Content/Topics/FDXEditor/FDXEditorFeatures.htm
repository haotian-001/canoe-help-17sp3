Function Overview

 

Creating and Editing FDX Descriptions

The FDX plug-in FDX Editor allows FDX descriptions to be graphically created and edited based on XML. The integration with the symbol database of CANoe makes it easy to consistently define and display the payload data of FDX datagrams.

The description is implicitly checked for layout errors, which can be displayed and corrected both in the editor and in the graphic layout.

The MDI design of the user interface of the editor allows multiple FDX descriptions to be edited in parallel.

Creating Groups and Items

The contents of an FDX description can be generated by creating new groups and inserting new items:

  • via drag and drop from the Symbol Explorer,
  • via drag and drop from the Identifier List,
  • via the ribbon or the context menu,
  • via a keyboard input in the editor.

New groups are automatically assigned a unique FDX data group. If they are created by drag and drop, their identifier is take automatically from the source. The size of the group is preset in such a way that it adapts automatically to its content; a fixed size is only inserted into the XML attribute when the FDX description file is explicitly overwritten or saved.

Items also the definition of each drag and drop source, especially the details of a CANoe symbol. Thus, new items are automatically assigned meaningful names and their offset and size are initialized consistently. When symbol names are manually entered, an auto-completion list is created from the current database and offers filtered symbols for selection.

Editing Groups and Items

All parameters of the groups and items can be edited in the tables of the editor or in the supporting Symbol Properties tool. Only the size of an item or group is blocked from being edited when the data type of an item explicitly specifies the size or when the editor is run in Automatic Group Size mode.

The order of selected groups or items or the offset of items can be changed with a button press (or a key combination).

Using drag and drop within the editor table, it is possible to change the order of groups or items and create copies of groups and items.

Attributes of symbols (e.g. the namespace of a system variable) can be edited in the peripheral Symbol Properties tool. Because they normally originate from the CANoe database, however, it is relatively rare to have to adapt them.

The assignment of identifier and symbol for an existing item or group can be changed via drag and drop.

Editing the Physical Layout

The Offset and Size parameters of an item define its physical location in the FDX datagram, and this does not have to be consistent with the arrangement of items within the group. In particular, the physical layout can have gaps. The FDX Editorsupports the understanding of these various perspectives of an FDX description through the graphical representation of the physical layout. By a single button press, the offsets of items can also be recalculated from the order and size of the items in the FDX description.
