Symbol Properties

 

In the Symbol Properties Window, you edit the details of the item that currently has the focus.

The details of an item consist of the data that allow unique identification of database symbols and how the data are coded during transmission.

Normally, all symbols that you link to items from the Symbol Explorer using drag and drop can be correctly identified. If, however, no database is available or the items are created in a different way, the properties in the Symbol Properties Window enable explicit linking with the help of additional information.

Depending on the linked symbol type, the window allows different information for more detailed identification of the symbol:

  • Signals allow the input of message, database, network, and channel.
  • Messages and PDUs allow the input of database, network, channel and network node.
  • System variables allow the input of a namespace.

To configure the transmission of data, other symbol type-dependent additional attributes are available:

  • Signals allow the specification of a direction (Auto / TxRq) and whether data are to be transmitted as a raw value or physically scaled.
  • Messages and PDUs do not allow any special additional specifications.
  • System variables allow the specification of whether data are to be transmitted as a raw value or physically scaled.

The attributes as a whole correspond exactly to the attributes that are permitted as an XML element in the definition of an FDX item (see user manual for the FDX protocol).

If you set the coding to Phys for a system variable or a signal, the data type is changed automatically to Double. This setting can be manually overwritten because physical values can also be converted to other data types.

For items with an Array data type, the Symbol Properties Tool contains an additional entry in which the number of array elements can be input, or this can be calculated from the item size. If the number of array elements is input, the item size is adapted accordingly (taking the required 4 bytes for the length of the array into account).

Note

In the case of system variables of Generic Array type, it must be noted that information about the structure of the elements of the array cannot be saved in the FDX description. For this reason, the BYTEARRAY data type allows only an input of the number of bytes minus the 4 bytes for the length of the array. However, if you select such a system variable directly via drag and drop or as a symbol name, the actual size is calculated from the symbol definition.
