Graphic Layout

 

The Graphic Layout graphically displays the physical layout of the selected groups.

This functionality is especially useful when the order of the items in a group do not correspond to the physical layout. In this case, it is time consuming to calculate this layout from the offsets and sizes of the items. The physical layout is always the deciding factor for communication with the HIL system, which has to act on the assumption of the same layout of the payload data of a datagram.

For each selected group, the layout displays a row that is labeled on the left side with the name of the group (the identifier if specified; otherwise the FDX data group). After this label zone, the physical layout follows in the resolution of bytes. The entire extent of the group is marked by a dark-gray background. A color block is marked over this background for each item of the group corresponding to its offset and size. The color of the block corresponds to the symbol type. In addition, the block is labeled with a symbol type-specific icon and the name of the item (the name of the linked symbol if specified; otherwise the identifier).

For items with an array or string data type, the FDX protocol expects a length specification within the data to be transmitted itself (see user manual for the FDX protocol). The first 4 bytes of the item must be reserved for this, which is indicated in the graphic layout by a gray hatching of the corresponding items

Below the Graphic Layout, the window also displays a scale that indicates the respective byte offsets. Depending on the current scaling factor, each byte or just each eight bytes is labeled.

Item Selection and Group Focus

The Graphic Layout Window always follows the focus of the current editor and vice versa. The selected items of the group that currently has the focus are marked with an additional white frame. This marking can also be changed directly in the Graphic Layout by clicking on a corresponding item with the left mouse button. If you hold down the <Ctrl> or <Shift> key, you can also select multiple items of a group. The active editor follows the changes made to the selection and focus.

Within the Graphic Layout Window, the selected items can be moved with respect to their offset and thus with respect to their physical position. This corresponds to globally reducing or increasing the offset of the selected items using the Reduce Offset and Increase Offset context menu command. In particular, you can correct the layout with the key combinations <Shift>+<Left Arrow> and <Shift>+<Right Arrow>.

Scaling Factor

The display of the layout uses scaling in the horizontal direction to visualize the amount of physical memory of an item or group consumed. This scaling is set to 50 pixels for one byte by default. You can manipulate the scaling within certain limits in different ways:

  • If the window has the input focus, you can rotate the mouse wheel upwards to zoom in on the scaling and downwards to zoom out from the scaling.
  • Using the Zoom In and Zoom Out context menu command.
  • Using the Fit to Window Size context menu command, the scaling is selected in such a way that all currently selected groups are displayed in their entirety.
