Table Editor

 

The Table Editor is the central element for creating and editing FDX files. It is used to create, edit and delete FDX data groups and their elements as well as FDX functions. You can switch between Data Groups and Functions using the pages at the bottom of the Table Editor.

The Data Groups page is displayed in a split view. The table on the left lists all defined groups and their properties and the table on the right lists all items from the selected group.

Also the result of the validation, which is executed with each change of the FDX file, is displayed with corresponding icons in the affected cells. Moving the cursor over the icon displays the corresponding message.

In addition, the Table Editor links the available windows of the FDX Editor.

Opening/Creating FDX Files

You create an empty FDX description with the New FDX Description menu command (File ribbon tab). You load an existing FDX description using the Open FDX Description menu command (File ribbon tab) or via the Project Documents window.

Multiple FDX descriptions can be edited simultaneously and the arrangement of the editor windows can be configured as desired.

Creating Data Groups, Items and Functions

You can add new data groups, items and functions to the active FDX description in different ways:

  • Via drag and drop or copy and paste from the Symbol Explorer (groups, items)
  • Via drag and drop or copy and paste from the Identifier List (groups, items, functions)
  • Via the New menu command (Home ribbon tab),

  • By entering in the input line at the end of the table

Generally, the position where the new group is inserted can be influenced by the position at which the mouse button is released:

  • Release the mouse button when the pointer is on the table header in order to place the element at the beginning.
  • Release the mouse button on the separating line between two rows in order to insert the new element at this location.
  • Release the mouse button below the last group in order to append the new element at the end.
  • Release the mouse button middle of an existing group in order to modify its definition
    For example, you either add a signal to a group or replace a signal in a group.

The table gives visual feedback here to show which action is occurring.

If you hold down the <Shift> key when you release the mouse button, the dragged data are deleted from their source (except for symbols in the Symbol Explorer whose database cannot be edited).

Editing Data Groups, Items and Functions

Existing entries can be edited as needed in the cells of the table. Depending on the content of the cell, there is either a preselection, e.g. for the symbol type, or it can be edited freely. Editing can be completed by clicking outside the cell or by pressing the Enter key, or alternatively it can be aborted by pressing the <Escape> key.

The group size cannot be changed when the Automatic Group Size mode is switched on in the Home ribbon tab. If this mode is switched off, you can modify the group size individually. Set the size of a group explicitly to 0 in order for its size to be automatically adapted.

With the Automatic Layout menu command (Home ribbon tab), you can completely reorganize the arrangement and extent of the contained items of selected groups.

If a change is incorrect or does not achieve the desired effect, the Undo and Redo functions are available on the Home ribbon tab.

Modifying the Physical Layout

Edit the Offset, Data Type and Size columns of an Item Table in order to influence the physical extent and position of the item in the datagram.

You can also move the physical position of the selected items byte-wise using the Reduce Offset or Increase Offset menu command (Home ribbon tab) or their corresponding key combinations. This modification can also be applied from the Group Layout window.

Note

The physical layout of a group is independent of the order of the items in the XML file (and thus in the Item Table).

Configuring Tables

  • Column width
    In the header line of the table, the boundary line between two columns can be moved by clicking and holding down. The column width can be adjusted to the table content via the context menu (right click) of the header line.
  • Column order
    A column can be moved as a whole by clicking and holding it.
  • Sorting
    The rows can be sorted based on a specific column. To do this, simply click on the column header, a second time to reverse the sorting. The sorting can be canceled via the context menu (right click) of the column header.

Note

The sorting of the table has no effect on the order in the FDX description file – it is solely for visualization purposes. As long as sorting is active, an explicit change of the order does not affect the visualization. Without active sorting, the order corresponds to the rows of the table.

By clicking the header of the table with the right mouse button, you can adapt the configuration by displaying or hiding columns or adjusting the column widths. The latter can also be done manually by moving the boundary line between columns. By clicking the header of the table with the left mouse button, sorting can be performed based on the respective column. This sorting can be removed again via the context menu.
