FAQ

 

Working With Multiple Monitors and Different Scaling in Panel Designer

You should consider the following points when creating panels:

  • Do not switch screens when designing a panel if they have different scales.
  • You will have the most accurate alignment and display if the scaling is 100%. There, no rounding inaccuracies occur during the calculation.
  • Check the display of your panels after saving.

FAQ: Why is a text not displayed completely or with dots, or why is a text cut off in the height?

With different screen scaling, rounding inaccuracies can occur when calculating font and control size. Basically, the control size and positioning on a panel are not automatically adjusted to avoid changing your wanted layout. This can result in a font size that is minimally too large for a scale greater than 100% and does not fit the control size.

For the following controls, the text can be displayed with dots and cut off in the height:

  • Check Box
  • Radio Button
  • Combo Box
  • Input/Output Box

For the following controls, the text automatically can be wrapped to the next line, which is not displayed:

  • Static Text
  • Button

FAQ: Why are scroll bars, buttons, and drag elements displayed unnaturally large, small, or wide in the Panel Designer?

If you are working with multiple open panels in the Panel Designer Windows may use the old size specifications from the previous monitor when scaling for the new monitor. This can lead to incorrect scaling in the following controls:

  • Complex Data Control (scroll bar)
  • Method Call Control (scroll bar)
  • NM Control (scroll bar)
  • Track Bar (gripper)
  • Numeric Up/Down (buttons)

This refers only to the display in the Panel Designer, not to the finished panel at runtime. Close the panel and open it again, then the display will be correct again. You can also save it. It is an incorrect display and has no effect on saving

FAQ: Update of the window sizes

When changing the screen, some windows are not scaled correctly and are displayed too small, for example. Change the size of the main window minimally and these windows will be displayed correctly again.

FAQ: Font sizes with Windows 10 (Version 1803 and 2004)

Font sizes are scaled twice at runtime and during editing and are therefore displayed incorrectly at resolutions not equal to 100%.
