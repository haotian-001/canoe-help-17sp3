Create a C Library

 

To create a CANoe C library, you must translate your C code and the CCL.cpp file and link it to a shared library (a DLL on Windows or a SO on Linux). The architecture of the shared library must match the architecture of the used runtime kernel of CANoe.

The typical steps (independent of the development environment) for creating a new shared library are:

  1. Create a new project.
  2. Copy the directory CCL together with the contained file to the new project.
  3. Add the file CCL.cpp to the list of source files, that the project should compile.
  4. Add your own C file to the project, include the header file CCL.h and implement the function cclOnDllLoad.
  5. When targeting the 32-bit platform Windows_x86, you need a compiler specific Module Definition File for the linker. The newer 64-bit platforms Windows_x64 and Linux_x64 don't need such a file.
  6. Compile and link your project to receive your C library as a shared library.
  7. Write a module description file that contains a relative path to the shared library. The module description file can than be added to the CANoeconfiguration.

Details for some of the above steps when using Visual Studio 2017

  • (1.) For creating a project, use the menu entry File|New|Project and select the template Visual C++|Windows Desktop|Dynamic-Link Library (DLL). If the Project Wizard created a source file with the DLLMain function, delete this function. Implementation of the function is already provided by the C Library framework.
  • (3.) After adding the file CCL.cpp to your Visual Studio project, deactivate the use of the Precompiled Header for this file in the compiler settings of Visual Studio.
  • (5.) When targeting the 32-bit platform Windows_x86, add the file CCL_VC_x86.def from the CCL directory to the linker settings of the Visual Studio project (Configuration Properties|Linker|Input|Module Definition File).

Details for some of the above steps when using CMake with Mingw-w64

  • (1.) At the CMakeLists.txt file use the CMake command add_library with option SHARED to create a shared library.
  • (5.) When targeting the 32-bit platform Windows_x86, add the module definition file CCL_Mingw_x86.def to the list of source files.
