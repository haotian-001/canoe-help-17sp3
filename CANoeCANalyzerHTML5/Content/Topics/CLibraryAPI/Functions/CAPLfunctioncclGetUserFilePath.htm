cclGetUserFilePath

 

Note

The main application case for this function is to forward the path of a registered user file to a CAPL DLL in case of a distributed environment. The function is, however, also available in a single PC setup. For more information, see the CAPL function GetUserFilePath.

Function Syntax

int32_t cclGetUserFilePath(const char* filename, char* pathBuffer, int32_t pathBufferLength);

Description

Gets the absolute path of a user file.

In case of execution in a distributed environment the absolute user file path (including filename) on the remote device (e.g. VN8900) is returned if the user file was predefined. If the file was not predefined an error code is returned.

In case of a single PC environment the registered absolute file path (including filename) of the user file is returned if the user file was predefined. If the file was not predefined the function converts a path relative to the configuration directory to an absolute path.

Returns the same result as getAbsFilePath (converting a path relative to the configuration directory to an absolute path).

Parameters

filename

A filename potentially containing a path. If the filename is predefined the path is ignored. Otherwise the path is interpreted as relative to the current configuration path in a single PC environment.

pathBuffer

Buffer to which the full path name should be copied.

pathBufferLength

 

Return Values

CCL_SUCCESS

The function call was successful.

CCL_USERFILENOTDEFINED

The file was not predefined.

CCL_INVALIDNAME

Parameter filename is a null pointer or does not contain any character.

CCL_NOTIMPLEMENTED

CANoe/CANalyzer version 7.5 SP2 or newer is required for this function.

CCL_WRONGSTATE

The function may only be called in the PreStartHandler of the measurement and during a running measurement.

CCL_BUFFERTOSMALL

The specified buffer is too small for the absolute path of the user file.

CCL_PARAMETERINVALID

Parameter pathBuffer is a null pointer or parameter pathBufferLength is less or equal zero.

Example

See example GlobalEventHandler.

Availability

The C Library is executed inside the RT Kernel of CANoe. So the architecture of the C Library (Windows 32-bit, Windows 64-bit or Linux 64-bit) must fit the architecture of the used RT Kernel.
