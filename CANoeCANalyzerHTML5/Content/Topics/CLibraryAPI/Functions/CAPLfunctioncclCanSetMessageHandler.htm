cclCanSetMessageHandler

 

Function Syntax

struct cclCanMessage
{
  int64_t time;
  int32_t channel;
  uint32_t id;
  uint32_t flags;
  uint8_t dir;
  uint8_t dataLength;
  uint8_t data[64];
};

int32_t cclCanSetMessageHandler(int32_t channel, uint32_t identifier, void (*function) (struct cclCanMessage* message) );

Struct

cclCanMessage

The struct that is transferred to the event routine, contains the following fields:

time

Time stamp (end-of-frame) of the event in nanoseconds.

channel

The CAN channel on which the message was received or sent.

id

CAN identifier of the message.

The most significant bit of this parameter determines whether it is a 29-bit extended identifier or an 11-bit standard identifier. Using the functions cclCanIsExtendedIdentifier and cclCanIsStandardIdentifier, you can determine whether or not it is a standard identifier or an extended identifier. The function cclCanValueOfIdentifier extracts the value of the identifier regardless of its type.

flags

Using the flags, some additional properties of the CAN message can be determined.

CCL_CANFLAGS_RTR

Protocol Bit RTR (Remote Transmission Request, for classic CAN only).

If this bit is set (1), the frame is a remote transmission request.

CCL_CANFLAGS_WAKEUP

The message was sent in High Voltage Mode.

CCL_CANFLAGS_TE

The NERR flag of the CAN transceiver was active upon reception of the message (system is in single-wire mode).

CCL_CANFLAGS_FDF

Protocol Bit FDF (FD Frame Indicator).

If the bit is set, the CAN frame uses the CAN FD format, otherwise the classic CAN format is used.

CCL_CANFLAGS_BRS

Protocol Bit BRS (Bitrate Switch, for CAN FD only).

If this flag is not set (0), then there is no change of the bitrate for the data phase.

If this flag is set (1), there is a change to a higher bitrate for the data phase.

CCL_CANFLAGS_ESI

Protocol Bit ESI (Error State Indicator, for CAN FD only).

If this flag is not set (0), then the sending CAN FD node was in error active mode.

If this flag is set (1), then the sending CAN FD node was in error passive mode.

dir

The dir field can accept one of the following three values:

CCL_DIR_RX

The message was received over the bus or sent by another component in CANoe.

CCL_DIR_TX

The message was sent by this or another C library.

CCL_DIR_TXRQ

The message was stopped in the Send register of the CAN controller, and the CAN controller begins the arbitration. This event only occurs if the TXRQ feature was activated in CANoe beforehand.

dataLength

Data length of the CAN message in bytes.

data

Data bytes of the CAN message.

Description

Registration of an event routine for the reception of CAN messages.

Registration of the handler may only be carried out in the PreStartHandler. The registered event routines are unregistered automatically at the end of measurement.

Parameters

channel

The CAN channel for which the event routine is to be active.

identifier

The CAN identifier of the message for which the event routine is to be registered. The most significant bit of this parameter determines whether it is a 29-bit extended identifier or an 11-bit standard identifier.

Use the functions cclCanMakeExtendedIdentifier and cclCanMakeStandardIdentifier to create this parameter. The special value CCL_CAN_ALLMESSAGES registers the event routine for all messages of the specified CAN channel.

function

The event routine to be registered.
The event routine is called as soon as an appropriate CAN message has been received or sent.

Return Values

CCL_SUCCESS

The event routine was registered successfully.

CCL_WRONGSTATE

The function may only be called up in the PreStartHandler of the measurement.

CCL_INVALIDCHANNEL

The specified CAN channel is invalid.

CCL_INVALIDFUNCTIONPOINTER

The parameter function is a ZERO pointer.

Example

See example CAN.

Availability

The C Library is executed inside the RT Kernel of CANoe. So the architecture of the C Library (Windows 32-bit, Windows 64-bit or Linux 64-bit) must fit the architecture of the used RT Kernel.
