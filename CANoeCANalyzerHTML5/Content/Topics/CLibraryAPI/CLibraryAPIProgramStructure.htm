Program Structure

 

Every CANoe C library must first implement the cclOnDllLoad function.

This function is called once by CANoe when loading the DLL. Additional event routines are registered in the body of this function, usually for the initialization and start of a measurement (see cclSetMeasurementPreStartHandler and cclSetMeasurementStartHandler).

The receiving handlers for CAN and LIN frames are then registered in the event routine for initialization of the measurement (MeasurementPreStartHandler). Timers are created (see cclTimerCreate), and the identifier for accessing system variables (see cclSysVarGetID) are determined.

The first timers are started in the event routine for the measurement start (MeasurementStartHandler). If necessary, the first messages are output over the bus.

While the measurement is running, the various event routines are now called up by CANoe:

  • message was received/sent over the bus,
  • timer has expired,
  • system variable has changed,
  • communication object value has changed,
  • network function was called or got a response,
  • service consumer or provider detected.

The event routines send out additional messages over the bus, change the contents of system variables or communication object values, invoke network function calls or set other timers.
