Breaking Changes

  • All built-in attributes have UpperCamelCase names, e.g. [ByteOrder=Motorola].
  • Enum like built-in attributes are real vCDL enumerations, meaning values are written directly, not as string literal surrounded by double quotes, e.g. [CommunicationPattern=PublishSubscribe].
  • Locations where attributes may be specified have been restricted, but they are always allowed in front of the artefact.
  • Minimum and initial size of list data types are written within the angled brackets.
  • Encodings must have a base type assigned.
  • Communication Objects and all related artefacts have been removed.
  • The IP stack cannot be configured (readded in upcoming versions).
  • Inline DO interfaces cannot inherit from other interfaces anymore.
  • Clear distinction between optionality and nullability
  • New syntax for custom attribute definitions
  • Improved syntax for defining discriminator values of union members including a new attribute DeriveDiscriminatorValues to explicitly advice CANoe to generate discriminator values.
  • A module import is mandatory to use a specific binding or some predefined attributes.
  • DO reference members do not have a direction and thus are declared the same way as embedded members.
  • The previous attributes setterHasReturn and setterIsOneWay have been combined into the SetterMode attribute with NoReturn as default.
  • Comments are not transferred into CANoe (readded in upcoming versions).
  • File encoding is limited to UTF-8 (other encodings will be readded in upcoming versions).

New Features

  • Flag attributes can be negated using the !-operator.
  • Constants are imported into CANoe.
  • System Variables can be specified using vCDL.
  • Struct data types can be embedded into other struct data types.
  • Enumeration types can be derived from union data types.
  • Enumeration values can be used like any other numeric value.
  • Single DOs or DO references in a container can be referenced using an index style syntax.
  • DO instances, (reference-)containers, and their elements can be refined at their definition without a separate refinement statement.
  • Improved value assignment for union data types.
  • More intuitive naming of implicitly generated artefacts such as data types.
  • Redesigned compiler for improved performance on large files.
