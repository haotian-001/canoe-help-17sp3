Preprocessor

Navigation: « Refinements

vCDL comes with an integrated preprocessor like those used in C/C++ and C#, but with a reduced feature set consisting of conditional parsing, regions, and pragmas. It does not support macros and include directives. However, the latter are supported through the import statements in vCDL. Every preprocessor directive starts with the #-symbol followed by the name of the directive. Any whitespace may precede the directive, e.g. for indentation purposes. As a result, a single line can contain only a single directive. As its name suggests the preprocessor is executed prior to the vCDL parser. Thus, preprocessor directives are not restricted to any syntactic vCDL structures like scopes and statements. However, preprocessor constructs themselves must be properly nested, e.g. a region starting inside a conditional scope may not expand outside this scope. The following example shows some practical use cases for preprocessor directives by tailoring a vCDL to the active scenario configured in the CI/CD toolchain.

version 2.0;
#ifdef SCENARIO_MQTT
import module "MQTT";
const string BindingName = "MQTT";
#else
import module "Abstract";
const string BindingName = "Abstract";
#endif

namespace Content
{
  // Regions can be collapsed in the editor
  #region Data Types

  typedef IntType   = int16;
  typedef FloatType = float;

  #endregion

  [Binding=BindingName]
  interface DOInterface
  {
    provided data IntType Member;
  }
}

Conditional Parsing

Conditional parsing can be used to activate or deactivate portions of the code based on previously defined constants. Those constants must be defined within the same file as they are used. They can then be used directly or as part of more complex expressions. If constants are not valid throughout the whole file, they can be undefined again. In contrast to the actual vCDL language where several objects like data types can be defined anywhere to become usable, the preprocessor works sequentially on the file starting at the first line . Thus, constants need to be defined prior to their first usage. The following table gives an overview of the preprocessor directives that can be used to control parsing.

Directive

Example

Description

#define

#define CANOE_VERSION 15

#define DEBUG

#define CONST2 (CONST1 + 1)

Defines a new constant and sets it to a specific value or 0 if no value is given.

#undef

#undef CANOE_VERSION

Removes a constant’s definition if it has been defined earlier, otherwise it has no effect.

#if

#if CANOE_VERSION > 13
#if DEBUG || RELEASE

Opens a new conditional scope. If the condition evaluates to true, the following code is passed to the vCDL parser. Otherwise subsequent conditions are evaluated (see below).

#ifdef

#ifdef DEBUG

Shortcut for #if defined(DEBUG)

#ifndef

#ifndef RELEASE

Shortcut for #if !defined(RELEASE)

#elif

#elif DEBUG != 0

Like #if but gets only evaluated if all previous conditions evaluated to false. If the condition evaluates to true, the following code is passed to the vCDL parser.

#elifdef

#elifdef CANOE_VERSION

Shortcut for #elif defined(CANOE_VERSION)

#elifndef

#elifndef RELEASE

Shortcut for #elif !defined(RELEASE)

#else

#else

Opens the else-block and is entered if none of the previous conditions evaluated to true.

#endif

#endif

Closes a conditional block and is always required at the end.

The preprocessor supports boolean and integer arithmetic where integer and boolean values can be used interchangeably. The preprocessor uses following conversion rules implicitly: An integer is true if it is not 0, false otherwise. Having the opposite direction true always evaluates to 1 and false to 0. It is therefore possible, though not recommended, to apply arithmetic operators to logical expressions and logical operators to arithmetic expressions. Because of these implicit conversions there is a difference between testing a constant’s value, e.g. using #if, and just testing whether a constant is defined, e.g. using #ifdef. If a constant is defined and its value has been implicitly or explicitly set to 0, #ifdef will nevertheless evaluate to true whereas #if will be false. Using an undefined constant in any expression will be equivalent to using 0 instead. The following table lists all operators in descending precedence.

Operators

Association

(x), defined(c), defined c

inner to outer

+x, -x, !x

right to left

x*y, x/y, x%y

left to right

x+y, x-y

x<y, x<=y, x>=y, x>y

x==y, x!=y

x&&y

x||y

#define and #undef directives may appear within conditional scopes and are then only evaluated if the surrounding conditions hold true. As this construct impairs readability it should be avoided whenever possible.

Regions

Regions can be used to further structure the code as they can be collapsed in the vCDL editor to hide portions of the code that are not currently in focus. They can also be nested. Each region always consists of the following two directives.

Directive

Example

Description

#region

#region Data Types

#region

Starts a new region, optionally assigning it a name. There are no limitations for the name, it is always the remaining line.

#endregion

#endregion

Closes a region.
