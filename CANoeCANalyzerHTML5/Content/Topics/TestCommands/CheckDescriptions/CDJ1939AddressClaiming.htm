J1939 Address Claiming

 

Description

This check observes the address claiming.

The following validations are done during this check:

  • For a global Request for Address Claim all configured nodes must respond with an Address Claimed message.

  • For a specific Request for Address Claimed, only the addressed node must respond with an Address Claimed message.

  • The Address Claimed message must always be sent globally.

  • The DLC of the Address Claimed message must be 8, the DLC of the request must be 3.

  • The default time-out is 250 ms (within this time a Request for Address Claim has to be responded).

  • Reserved bit must be 0.

  • Device name (NAME) in the Address Claimed message is validated against device name of the node.

  • Source Address of the Address Claimed message is validated against address of the node.

This check is available for network nodes with the Address Claimed (AC) Parameter Group (PGN EE00h) in the Tx list.

Note

In a test configuration you can activate Report only first background check violation to reduce the number of error messages in the report.

Error messages are then reported once for every combination of the values marked bold in the table above.

If, for example, there is the following sequence of error messages...

Reserved bit is 0x01 (1) in Address Claim message from address 1

Reserved bit is 0x01 (1) in Address Claim message from address 1

Reserved bit is 0x01 (1) in Address Claim message from address 1

Reserved bit is 0x01 (1) in Address Claim message from address 2

Reserved bit is 0x01 (1) in Address Claim message from address 2

Reserved bit is 0x01 (1) in Address Claim message from address 1

...only these messages are listed in the test report:

Reserved bit is 0x01 (1) in Address Claim message from address 1

Reserved bit is 0x01 (1) in Address Claim message from address 2
