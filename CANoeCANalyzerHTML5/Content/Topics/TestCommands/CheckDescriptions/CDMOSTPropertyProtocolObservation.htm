MOST Property Protocol Observation

 

Name

MOST Property Protocol Observation

Description

This check observes the compliance to the MOST protocol with regard to message sequences and response times for a given property function.

The behavior of the check at runtime may be described as follows:

  • The check uses spy messages to observe the properties. For this reason, it is important to make sure that the node or channel in which the check is implemented is receiving the relevant spy messages before the check is used.
  • Command OpType messages (Get, SetGet, GetInterface, Set, Increment, Decrement) — so-called requests — are saved upon receipt along with their time stamp.
  • Incoming report OpType messages are compared with the saved messages to establish whether the response is valid. If this is the case and the difference between the receipt time stamps is smaller than the maximum response time, the request message is deleted again.
  • The saved request messages are compared with the current time stamp at regular intervals. If the maximum response time is exceeded, a check violation is triggered for those request messages that require a response (Get, SetGet and GetInterface).
  • If a sequence consists of multiple identical command messages that follow one another sequentially, only the last one received is saved and used for resolution with an appropriate report message. Conversely, the first appropriate report message is used to resolve an open request. Any identical messages that follow are ignored if they can no longer be assigned to an open request.
  • When AMS messages are being observed, the response time is determined using the time stamp of the last telegram in the command message and the time stamp of the first telegram in the report message.

Note

Error OpType report messages are correct response messages in terms of protocol compliance and are used to resolve requests in the same way as Status messages.

In the case of AMS messages that are segmented over several telegrams, the time stamp of the last telegram in the request message and the time stamp of the first telegram in the response message are used to determine the response time.

Check Functions

  • ChkCreate_MostPropertyProtocolError, ChkStart_MostPropertyProtocolError
    When implementing a PropertyProtocolError check, you can use a callback function to obtain information about the last request message that triggered the current protocol violation.
    Functions for statistical analysis are also available.
