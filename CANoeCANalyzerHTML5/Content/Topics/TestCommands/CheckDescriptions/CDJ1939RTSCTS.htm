J1939 RTS/CTS

 

Description

This observes the RTS/CTS (Ready to Send/Clear to Send) transport protocol.

It is possible to observe all RTS/CTS transmissions or only the transmission of specific nodes.

Following validations are done by the RTS/CTS check:

  • The transmission must begin with a RTS.

  • Number of data bytes and number of packets must be consistent.

  • Sequence of TP.CM (RTC, CTS, Abort, EndOfMsgACK) and TP.DT messages must be valid.

  • Timings (T1, T2, T3 and T4) must not be violated.

  • Sequence number must start with 1 and must be increasing.

  • Unused bytes of the last data packet (TP.DT) must be set to FFh.

Note

In a test configuration you can activate Report only first background check violation to reduce the number of error messages in the report.

Error messages are then reported once for every combination of the values marked bold in the table above.

If, for example, there is the following sequence of error messages...

TP.DT message with source address 1 and destination address 2 has invalid DLC 5

TP.DT message with source address 1 and destination address 2 has invalid DLC 6

TP.DT message with source address 1 and destination address 2 has invalid DLC 7

TP.DT message with source address 6 and destination address 4 has invalid DLC 5

TP.DT message with source address 5 and destination address 4 has invalid DLC 5

...only these messages are listed in the test report:

TP.DT message with source address 1 and destination address 2 has invalid DLC 5

TP.DT message with source address 6 and destination address 4 has invalid DLC 7

TP.DT message with source address 5 and destination address 4 has invalid DLC 7
