Occurrence Of Messages

 

Description

This check is used to monitor the minimum and maximum distance of messages. This check is well-suited even for mixed, cyclic and spontaneous transmission modes. When setting up the check the times to be monitored are calculated from the message's cycle time (minimum of GenMsgCycleTime and GenSigCycleTime), a minimum send delay (GenMsgDelayTime) and relative time entries.

  • The GenSigCycleTime attribute is not absolutely necessary.
  • The GenMsgDelayTime attribute must be defined if a relative minimum time is entered.

The check can be set up for one message or for all Tx messages of a node.

When setting up the check for a node, only those Tx messages for which periodic transmission is defined are considered. This involves looking at the attributes GenMsgSendType (Enum values: 0, 4, 5) and GenSigSendType (Enum value 0). If the message attribute or the signal attribute of a signal of this message is set to one of the named Enum values, this message is monitored by the check; otherwise it is ignored.

The cycle period of the message is calculated according to the absolutely scheduled timing of the FlexRay frame.

For FlexRay only valid data frames and PDUs are recognized as communication, Null Frames and Erroneous Frames are ignored.
