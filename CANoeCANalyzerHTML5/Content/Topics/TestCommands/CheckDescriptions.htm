Check Descriptions

 

General Description

Event Diagrams for Checks are Drawn Using the Following Legend

  • Each function specification ChkStart_ implies a specification of a ChkCreate_ function.

    This ChkStart_ function creates and additionally starts the check.

  • The names of checks are chosen always in the following manner:

    We check for an error instead of presuming the correctness.

    E.g.: We name ChkCreate_CycleTimeViolation instead of ChkCreate_CycleTimeOK (see also Checks in CAPL).
