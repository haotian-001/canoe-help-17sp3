System Under Test

 

Name

System Under Test

Description

With this element, System Under Test information can be taken up into the report. Therefore add the element sut directly to the test module using information pairs of name and description.

A corresponding CAPL function exists to this XML report element.

Syntax

<sut variants="A">
  <info>
    <name>Name of the name/value pair in the SUT section</name>
    <description>Value specification for this name/value pair</description>
  </info>

  //additional info elements
  <info>...</info>
</sut>

Attributes
  • variants (optional)
    …defines the test module variants where the information block is considered

See Also

XML file structure, Notes on writing XML files, XML sample file

Example: Display System Under Test information in the test report

<testmodule title="Central locking system test" version="1.0" xmlns="http://www.vector-informatik.de/CANoe/TestModule/1.14">

  <sut>
    <info>
      <name>CLS</name>
      <description>Central Locking System - Version 1.1</description>
    </info>
    <info>
      <name>SUT-Driver</name>
      <description>Signal-Driver and CAPL-Driver (Diagnosis) for the SUT - Version 1.0</description>
    </info>
  </sut>

...
</testmodule>
