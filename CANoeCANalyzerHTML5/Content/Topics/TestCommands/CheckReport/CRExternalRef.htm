External Reference

 

Name

External Reference

Description

With this element, an external reference is added to the report (URL or DOORS link) that is shown as a link in the test report.

The element can be used within the following XML elements: test module/ test group/ test case/ test case template/ CAPL test case/ .NET test case.

A corresponding CAPL function exists to this XML report element.

Note

  • The content of the externalref element is truncated at the beginning and at the end – blanks and line breaks are removed – but must not contain line breaks within the URL!
  • Details and tips for the usage, see External Reference.

Syntax

<externalref type="url|doors" title="Title of external Reference" owner="Owner specification">
   URL of the external resource
</externalref>

Attributes
  • type (mandatory)
    url - for any kind of URLs
    doors - for links to DOORS
    easee – for links to eASEE
  • title (optional)
    ...text shown with the link in the test report.
    The content of the element is used if nothing is specified.
  • owner (optional)
    This attribute is intended to be used by applications which maintain links to external resources automatically so they can identify their external reference elements. E.g. the Test Automation Editor DOORS Extension marks its DOORS references with a specific value of this attribute. It's recommended not to use it individually.

Example: Display external references in the test report

<testcase ident="tc-1.1" title="Test Case 1.1">
   <externalref type="url" title="Requirement">
      http://intranet.mycompany.com/myproject/wiki/req-tc1
   </externalref>
   <externalref type="doors" title="Requirement zu tc-1.1">
      doors://doorssrv:36677/?version=1,prodID=0,dbid=42d2481361dc551c,container=00004600,object=19
   </externalref>
...
<testcase>
