Check Report

 

General

For some checks statistic information is available in the automatically generated test report (test unit/test module).

On the configuration level of the check the "Statistic" of the check consists of the check parameters and the measured statistic data. The statistic data is check-specific. Information about the check-specific statistic data can be found in each check description.

Additional on each execution level of the check, in which the check fails, a "Mini-Statistic" of the check is available. It consists of the check parameters and of runtime-limited, not check-specific statistic data, which is composed of the number of failures and the number of samples.

Checks with statistic information within the automatically generated report

  • Cycle Time Absolute, Cycle Time Relative
  • DLC
  • Error Frame Count
  • Fallback
  • Message Distance
  • Messages Known
  • Node Active
  • Node Inactive
  • No Value Change
  • Occurrence Count
  • Occurrence of Messages
  • Request Response
  • Timeout
  • Value Dependency
  • Value Valid

Check report elements

The following XML elements can be used in XML test modules to report information in the test report:

Test Report Element

Description

Elements for Test Sequence

Comment (Report)

Output general information in the test report

Value Comment

Output runtime specific values in the test report

Elements for detailed information to specific structuring elements of the test module

Description

Output the description of a specific structuring element in the test report

External Reference

Output an external reference (link) in the test report

Engineer

Output test engineer information in the test report

Test Setup

Output test setup information in the test report

System Under Test

Output System Under Test (SUT) details in the test report

Miscinfo

Output miscellaneous information in the test report

Extended Info

Output HTML strings or other text information in the test report
