TLS CAPL Functions

 

TLS

The TLS API provides functions for encrypting a socket connection (see TCP/IP CAPL Functions).

To use the CAPL Transport Layer Security (TLS) functions, you must add a profile in the Vector Security Manager. This TLS profile is used in the tlsAuthenticateAsClientWithConfiguration and tlsAuthenticateAsServerWithConfiguration functions.

Please also use the sample configuration TCP Chat (CAPL) with TLS encryption for reference.

Short Description

HasEarlyDataBeenAccepted

To check if the early data has been accpeted by the TLS 1.3 server, this function should be called, but only after a succesfull TLS handshake.

Starts the (D)TLS authentication handshake as client.

tlsClose

Returns the TLS error code of the last operation that failed on a specified TLS socket.

Retrieves the error message of the last operation that failed on a specified TLS socket.

tlsOpen

Callbacks

Short Description

OnDtlsServerConnect

This callback is called when a new DTLS client connects to the DTLS server.

OnObservedAndDecryptedTlsApplicationData

This callback is called when observed and decrypted TLS application data is available.

OnTlsClose

This callback is called when the peer closes the tls connection while a receive call is pending.

OnTlsEarlyDataReceived

To handle early data from the TLS 1.3 client, the CAPL server program must implement this callback.

OnTlsHandshakeComplete

This callback is called when a TLS socket completes its handshake.
