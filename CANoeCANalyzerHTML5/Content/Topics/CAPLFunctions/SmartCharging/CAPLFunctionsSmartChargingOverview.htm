SmartCharging CAPL Functions

 

SmartCharging

  • Only available with Option SmartCharging.

  • The available CAPL functions depend on the protocol(s) used and thus the underlying technology.

  • CCS (Communication Setup)

  • CCS is used for the following protocol: ISO15118-20.

  • Only available if the CCS-Binding is activated.

  • To use these CAPL-Functions, CCS-Binding needs to be activated in the Communication Setup window.

  • CHAdeMO (Communication Setup)

  • CHAdeMO is used for the following protocol: CHAdeMO.

  • To use these CAPL-Functions, CHAdeMO needs to be activated in the Communication Setup window.

  • SCC Modeling Libraries (Simulation Setup)

  • The modeling libraries are used for the following protocols: DIN70121, ISO 15118‑2.

  • To use the CAPL functions, the modeling library SCC_Vehicle.vmodule and/or SCC_ChargePoint.vmodule must be included. You can include the libraries by using the configuration dialog of the node on page Components.

  • All response callbacks with ResponseCode parameter contain a Boolean parameter in their signatures that specifies the type of response code.

  • GB/T 27930 Modeling Libraries (Simulation Setup)

  • The modeling libraries are used for the following protocol: GB/T 27930.

  • To use the CAPL functions, the modeling library GBT27930_IL.vmodule must be included. Refer to the help page GB/T 27930 Interaction Layer.

  • OCPP (Communication Setup)

  • OCPP is used for the following protocol: OCPP.

  • To use these CAPL-Functions, OCPP-Binding needs to be activated in the Communication Setup window.

Note

You cannot mix CCS Binding CAPL functions with SCC Modeling Library CAPL functions as they address different protocols.

Embedded Members

These members encapsulate all messages of a special (sub-)schema. This makes it clear to which schema a message belongs. This is especially useful if there are messages with similar or the same names in several schemes.

Member

Description

SLAC

Contains all SLAC messages.

MME

Contains all MME messages, that do not belong directly to SLAC.

SDP

Contains all SECC Discovery messages.

SAP

Contains all Supported App Protocol messages.

ISO20

Contains all currently supported ISO20 messages.

ScheduleRenegotiation

Contains all ScheduleRenegotiation messages.

Reporting

Contains all messages that are output by the StateMachine as error information. E.g. in case of errors in the communication or when decoding or encoding EXI.

They are used to pass internal states to the outside (consumed) or signals and measurement data to the inside (provided). Further information can be found on the page Distributed Objects.

Embedded Members

These members encapsulate all messages of a special (sub-)schema. This makes it clear to which schema a message belongs. This is especially useful if there are messages with similar or the same names in several schemes.

Member

Description

OCPP16

Contains all OCPP 1.6 messages.

OCPPGeneric

Contains messages for sending generic OCPP messages. Can be useful for fault injection.

Contains:

  • GenericOCPPMessage

  • The compelete message to be sent must be set in content, including '[' ']'

  • GenericOCPPMessageParts

  • The action type can be omitted for responses

  • The complete payload must be set, including '{' '}'

Charging Station and CSM contain data members that are either used by both sides or are only relevant for one side of the communication. They are used to pass internal states to the outside (consumed) or signals and measurement data to the inside (provided). Further information can be the page Distributed Objects.
