Basic Connector (CV2x)

 

Note

Click on the blue link to load the sample configuration in CANoe.

Basic Connector (CV2x) - CV2xConnector.cfg

Note: CV2x stands for Cellular Vehicle to X.

This configuration shows how to create a UDP connection with a CV2x modem and use it to transmit and receive CV2x packets based on Chinese standards.

This configuration shows how CANoe can establish a connection to a CV2x modem based on the IP address and the Tx/Rx ports that are set in the panel UDP Connection Settings. You have to implement the counterpart on the CV2x modem. By means of a CV2x database with Chinese standards CV2x frames are created from the UDP communication that can be displayed in the CANoe Trace Window.

Desktops

This configuration contains several desktops.

Setup

This desktop contains the database configuration and the measurement setup. The database CSAE0053_MsgSet is integrated in the database configuration. It contains the ASNV module for Chinese application messages. Furthermore, two nodes are contained in the Simulation Setup. The node UdpCommunicationHandler starts the UDP communication (or ends it at measurement stop) and provides API functions. The node Node uses these API functions to transmit frames via UDP and shows an example of how CV2x is filled. This way, the node Node represents a communication participant that can be copied as often as desired. The node UdpCommunicationHandler should remain unique.

Furthermore, you can see the panel UDP Connection Settings. Here you can enter the IP address and the Tx/Rx ports for the connection. If a valid UDP communication can be established, the number of sent and received frames are displayed in the UDP status. If errors should occur, an error count is incremented.

In the panel Metadata you can set that up to 20 bytes are send before the actual CV2x payload (e.g. for specific implementations).

Measurement

This desktop contains a Map Window to display ITS stations on one of the supported maps. You can integrate online and local map data.

Furthermore, the desktop also contains two Trace Windows for the analysis. The Trace Window CV2x displays CV2x frames created from the UDP data. A detailed interpretation of the different CV2x protocol layers (Adaption Layer / DSMP) as well as further information about the CV2x paket can be found in the detailed view. Additionally, the used data with the ASN.1 coded application messages like e.g., BasicSafetyMessage are decoded and interpreted.

In the Trace Window UDP you can observe the incoming and outgoing UDP data stream.

The also included panels UDP Connection Settings and Metadata are already explained in the desktop Setup.
