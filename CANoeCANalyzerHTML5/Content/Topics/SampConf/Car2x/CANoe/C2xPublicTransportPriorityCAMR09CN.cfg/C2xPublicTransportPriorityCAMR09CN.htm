Public Transport Priority – CAM-R09 (Car2x)

 

Note

Click on the blue link to load the sample configuration in CANoe.

Public Transport Priority – CAM-R09 (Car2x) - Car2xPublicTransportPriorityCAMR09.cfg

Demonstrates public transport priority with CAM-R09 telegrams.

Introduction

The sample configuration provides an aid for a quick introduction to the coding and decoding of CAM R-09 telegrams.

For this the parameters for the CAM R-09 telegram and for the PriorStates in the SPAT are simulated in a scenario. The reporting point number of the CAM R-09 telegram is visualized in the Map Window.

Desktops

The sample configuration contains several desktops.

Setup

The Simulation Setup and Measurement Setup are displayed on this desktop.In addition, the Scenario Manager is displayed, in which a scenario for demonstrating public transport priority with CAM R-09 telegrams is loaded.

The nodes Sender_CAM_R09, R09_Decoder and TL are present. The node Sender_CAM_R09 sends the CAM messages with R-09 telegram for the priority at the traffic light. The node TL (Traffic Light) sends the MAP and SPAT messages. The parameters for the R-09 telegram and the PriorState of the SPAT are defined in the scenario CAM_R09.scn.

The node R09_Decoder draws the reporting point number of the R-09 telegram next to the sending station.

Measurement

This desktop displays a Map Window, a Data Window, a panel and the Trace Window.

The Map Window displays the MAP, SPAT and CAM messages. In addition, the reporting point numbers of the CAM R-09 telegram are drawn in the Map Window. In the Data Window the parameters from the scenario are displayed as system variables. In the panel the PriorState can be overwritten in the SPAT. It is also displayed here when the green phase of the traffic light is extended due to the priority. This happens if the PriorState of the SPAT is granted.
