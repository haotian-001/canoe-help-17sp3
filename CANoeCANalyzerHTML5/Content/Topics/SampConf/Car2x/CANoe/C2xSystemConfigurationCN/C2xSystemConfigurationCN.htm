System Configuration (Car2x)

 

Note

Click on the blue link to load the sample configuration in CANoe.

System Configuration (Car2x) - Car2xSystem.cfg

Simulation of a WLAN network with several ITS stations and display in the Map Window, based on the European GeoNetworking standard ETSI EN 302 636-4-1.

This sample configuration demonstrates the simulation of a WLAN network with CANoe. The GeoNetworking standard ETSI EN 302 636-4-1 is used for the simulated C2X messages.

The simulation contains several »ITS stations:

  • one Device Under Test DuT
  • two »RSUs: a traffic light (SimRsuTL) and a sender of a roadworks warning (SimRsuRWW)
  • a simulated vehicle SimCarEEBL to simulate an Emergency Electronic Brake Light Event
  • a simulated vehicle SimCarEVA to simulate an Emergency Vehicle Approaching Event

The vehicles drive a defined route based on the scenario Car2xSystemScenario. The RSUs are positioned on this route, too. As soon as a vehicle reaches the communication range of another vehicle or one of the RSUs, the received messages are passed to the application via CAPL and evaluated.

Introduction

On measurement start, there is a switch to desktop Overview. The Device Under Test starts to drive and sends »CAMs based on the stack timing. The Trafficlight RSU sends a MAP message each 5 seconds and a »SpaT message each second. The Roadworks Warning RSU sends a »DENM each second.

Desktops

This configuration contains several desktops.

Setup

This desktop displays the Simulation and Measurement Setup, the Scenario Manager as well as a Write Window.

The Simulation Setup contains the simulated ECUs, the System View of the Simulation Setup contains the Car2x database EU-ApplMsg already. The Measurement Setup contains blocks with which the data traffic in the network can be analyzed. The Write Window shows general information, warnings and error messages. The scenario is loaded in the Scenario Manager.

Note

Open the Car2x Network Explorer via the button Car2x Network Explorer in the Tools ribbon tab. There you get an overview of all application messages that are available in the database. Each application message is displayed as tree view with all elements (mandatory and optional elements).

Overview

This desktop displays various windows to display and analyze the network traffic:

  • The panels SimRSU Traffic Light and Head Unit Device Under Test for graphical display of state values
  • The panel SimRSU Road Works Warning to configure the roadworks warning DENM to be sent
  • The Map Window to display ITS stations in one of several supported maps. You can use online map data or integrate local map data.
  • The Trace Window to analyze the WLAN packets. A detailed interpretation of the different Car2x protocol layers (GeoNetworking and Basic Transport Protocol BTP) as well as further information about the WLAN packet can be found in the detailed view. Additionally, the payload is decoded and interpreted with the ASN.1 coded application messages CAM, DENM, MAP, and SPaT.
  • The Station Manager to change node assignments, and to influence the color and names of ITS stations.

Note

  • Change the value Relevance Distance in panel SimRSU Road Works Warning and watch what happens in the Map Window.

Traffic Light

This desktop displays several windows to display and analyze the traffic light situation and the reaction of the vehicle simulation:

  • Panel SimRSU Traffic Light for graphic display of the states of the traffic light simulation (simulated node SimRsuTL)
  • State Tracker to analyze states and state changes by using system variables
  • Data and Graphics Window to display important values of the simulation
    Besides system variables, signals from application messages can be directly displayed. The access to signals of application messages is provided by the ASN.1 Interpreter.

The simulated node SimRsuTL sends a MAP message each 5 seconds and a simple SPat messages each second. These messages contains information about the intersection topology, the currently displayed traffic light signal as well as the time until the next signal change of the traffic light.

The simulated node DuT of the vehicle simulation receives messages of other ITS stations in the range of approx. 200 ms. If the DuT reaches the communication area of the traffic light, the vehicle simulation analyzes the transmitted information and outputs the derived information as system variables (e.g. DistToTrafficLight and DisplayState).

Note

Add more signals from the Symbol Explorer to the Data and Graphics Window. You can show the Symbol Explorer with View | Symbol Explorer.

Emergency Electronic Brake Light

On this desktop various windows for the simulation and display of the Emergency Electronic Brake Light Warning (EEBL) are displayed.

  • Data and Graphic Windows for displaying signals sent in the DENM.
  • Trace Window for analysis of EEBL DENM.

The simulated node SimCarEEBL sends CAMs and DENMs. These messages contain information about the position, extent and type (Cause and Sub-Cause) of the EEBL event as well as waypoints leading to the EEBL event.

Road Works Warning

This desktop displays several windows to simulate and display the road works warning (RWW):

  • PanelSimRSU Road Works Warning to control the road works warning and to change any values that are sent in the DENM
  • Data Window to display signals that are sent in the DENM
  • Trace Window to analyze the DENM

The simulated node SimRsuRWW sends a DENM each second. This message contains information about the position, dimension and type of the road works as well as way points leading to the road works. Some of this information can be changed in the panel.

Emergency Vehicle Approaching

On this desktop, various windows for the simulation and display of the Emergency Vehicle Approaching Warning (EVA) are displayed.

  • Data Window for displaying signals sent in the DENM.
  • Trace Window for analysis of EVA DENM.

The simulated node SimCarEVA sends CAMs and DENMs. These messages contain information about the position, extent and type (Cause and Sub-Cause) of the EVA event as well as waypoints leading to the EVA event.

Certificates

On this desktop, the Certificate Explorer is displayed. It provides a quick overview of used certificates and their content.

Protocol-Analyzer

This desktop contains a Car2x Protocol Analyzer, a Trace Window and a Panel. The panel can be used to create errors and warnings that are displayed in the Protocol Analyzer and marked in the Trace Window.
