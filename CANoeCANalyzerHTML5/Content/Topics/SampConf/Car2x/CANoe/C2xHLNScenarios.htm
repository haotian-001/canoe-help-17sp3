Car2x Application Testing - HLN Scenarios

 

Note

Click on the blue link to load the sample configuration in CANoe.

Car2x Application Testing - HLN Scenarios (EU) - Car2xHLNScenarios.cfg

Shows the simulation of Car2x application and test scenarios for HLN EU.

Introduction

At the start of the measurement, the system switches to the desktop Measurement and simulates an Ego vehicle driving on a predefined sample route and its Car2x communication. The desired scenario can be selected and started in the panel. Scenarios can be started relatively or on a loaded route. The scenarios are based on the C-Roads HLN scenarios for Road Side Units.

Desktops

This configuration contains several desktops.

Setup

Contains the Simulation Setup and Measurement Setup.

A Trace Window and an inactive Logging Block are included in the Measurement Setup.

The database is included in the Simulation Setup. This contains the ASNV modules for the EU application messages. Nodes are included in the Simulation Setup according to the application scenario.

In addition, there are two other nodes in the Simulation Setup:

  • DuT_Simulator: disable this node if the measurement is done on the real bus and the tested vehicle and its Car2x communication behavior are real. The node can be used to simulate both the Car2x communication and the driving behavior of the ego vehicle. In a CAPL program, a driving Ego vehicle is simulated on the loaded route.

  • ScenarioControl: In a CAPL program the loading and starting of the scenarios is controlled.

The Document Window contains a short description of the configuration and the contained scenarios.

Measurement

This desktop displays a Map Window, a Trace Window, a Write Window and panels.

  • The Map Window visualizes the ITS stations and DENM events.

  • The Trace Window shows the data traffic.

  • The Write Window displays information, warnings and errors.

  • The panels can be used to influence the simulation.

In the Scenario Control Panel there are two sections to configure the scenario.

In the Scenario area, the specific scenario is selected and controlled. The scenario is selected in the Scenario list and the start mode in the Start Mode list. There are two start modes for the scenario:

  • Start Scenario Relative: the scenario is started relatively based on the current GNSS position and orientation.

  • Start On Route: The scenario is started based on the current GNSS position and orientation adjusted to the route specified in the Route section.

The Start Scenario and Stop Scenario buttons control the scenario.

In the Route area, a route can be defined using the file path (format: kml, gpx or nmea) and the name of the route. The name of the route in the file depends on the format:

  • .nmea: the filename

  • .gpx: the name of the trk object

  • .kml: the name of the placemark object

Additionally, the route can be configured as a closed loop in the Load As Loop drop-down list to let stations continuously travel the route. To apply the configuration correctly in real mode, the ego vehicle must be on the defined route.

The panel HLN Control Panel offers the possibility to customize certain values of the DENM messages individually and dynamically. This mainly concerns the configuration of the event area.

  • The event can be sent Linear or Circular by selecting the appropriate event area type in the Event Type list.

  • The different Relevance Distance values can be selected for circular events.

  • The Information Quality can be set.

If an EventZone is to be created, the length of the EventZone can be dynamically customized.

In the Ego Vehicle Simulation panel, an ego vehicle can be simulated on the specified route. By pressing the Simulate GNSS Simulation button, the ego vehicle travels the route at the selected speed and sends its position as GNSS data. A Car2x communication can be generated from the sent GNSS data by activating the check box Simulate Car2x Communication. If the DuT_Simulator node is deactivated, this simulation is not possible.
