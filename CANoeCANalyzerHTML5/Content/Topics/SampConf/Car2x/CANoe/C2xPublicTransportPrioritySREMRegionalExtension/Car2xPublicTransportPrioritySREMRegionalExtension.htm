Public Transport Priority – SREM with Regional Extension (Car2x)

 

Note

Click on the blue link to load the sample configuration in CANoe.

Public Transport Priority – SREM with Regional Extension (Car2x) - Car2xPublicTransportPrioritySREMRegionalExtension.cfg

Demonstrates public transport priority using SREMs (Signal Request Extended Messages) with regional extensions.

Introduction

The sample configuration gives an assistance for a quick start into the simulation of SREM and SSEM messages.

For this purpose, the parameters for the SREM and the SSEM are simulated in a scenario. The sending of the SREM and SSEM is also displayed in the Map Window. More information about the visualization can be found here. The difference of this configuration to the Car2xPublicTransportPrioritySREM.cfg is that the parameters are defined in a regional extension of the SREM and the actual request is empty.

Desktops

The sample configuration contains several desktops.

Setup

The Simulation setup and Measurement Setup is displayed on this desktop. In addition, the Scenario Manager is displayed with a scenario loaded to demonstrate public transport priority using SREM messages with regional extensions.

The nodes Sender_SREM, TSC, SREM_Visualizer and TL are present. The node TL (Traffic Light) sends the MAP and SPAT messages. The node Sender_SREM sends the SREM messages for the priority at the traffic light. The TSC (Traffic Signal Controller) answers this message with a SSEM. The parameters for both messages are defined in the scenario SREM_RegionalExtension.scn.

The SREM_Visualizer node draws the parameters of the regional extension in the Map Window next to the sending station. The TL node simulates the traffic light and sends MAP and SPAT messages.

Visualization

This desktop displays a Map Window, two Data Windows, a panel and the Trace Window.

The Map Window displays the MAP, SPAT, SREM and SSEM messages. In addition, the contents of the regional extension of the SREM are drawn as text in the Map Window.

In the two Data Windows, the parameters from the scenario are displayed as system variables. In the panel the status in the SSEM can be overwritten. It is also displayed here when the green phase of the traffic light is extended due to the priority. This happens if the status of the SSEM is granted.
