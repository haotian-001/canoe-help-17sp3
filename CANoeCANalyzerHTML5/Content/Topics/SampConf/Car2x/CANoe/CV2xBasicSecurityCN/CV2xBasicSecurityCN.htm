Basic Security (CV2x)

 

Note

Click on the blue link to open the directory of the sample configuration for copy procedure.

Basic Security (CV2x) - CV2xBasicSecurity.cfg

Note: CV2x stands for Cellular Vehicle to X

Simple example of using the Security CAPL API and the Certificate Manager to create valid security headers in Car2x messages according to GB/T 31024.3.

The sample configuration provides a guide for a quick introduction to the Security CAPL API and the Car2x Certificate Manager. The simulation contains two nodes, a sending node and a receiving node. Valid and invalid certificates are stored in the certificate manager.

Node

Description

Send node

Using the Sender panel, you can set for a BSM which certificate from the certificate manager should be used to send a message with security header should be used. A valid certificate and an invalid certificate are available, in addition you can select the transfer type select. Then click [Send] to send a package.

In the associated CAPL program, all necessary parameters of the security header are set (e.g. timestamp, BSM specific parameters, ...), so that you can use this sample code as a template for your own simulations. The functionality is implemented with the Security CAPL API.

Reception node

The Receive node checks whether the signature of the packet is valid and displays the validity on the Receiver panel. The functions of the Security CAPL API are used to read and evaluate information from the received messages.

Desktops

This configuration contains several desktops.

Setup

On this desktop the simulation and measurement setup as well as a Write window are displayed.

The Simulation Setup contains the simulated transmitters and receivers, the system view of the Simulation Setup already contains the Car2x database CSAE0053_MsgSet. The measurement setup contains blocks that you can use to analyze the data traffic in the network. The Write window displays general information, warnings and error messages.

Note

  • The sample configuration runs in simulated mode. Communication takes place only between the simulated nodes in CANoe. If you switch to real mode, communication will take place via the configured WLAN adapter.
  • If you double-click on a node in the Simulation Setup, the CAPL Browser opens. Use the Handler Overview to display security-related functions.
  • Use the Car2x Certificate Manager (Options dialog) to browse the certificates and keys used.

Measurement

On this desktop, the Sender and Receiver panels, the Trace Window and a Write Window are displayed.

The Sender Panel controls the sending node and the Receiver Panel carries out the evaluation. In the Trace Window the messages can be analyzed and compared with the result of the Receiver Node.

Note

In the details view of the trace window, expand the China V2X Security|Summary entry to get an overview of the transmitted message preserved. Under China V2X Security|Message Tree you can see all sent fields of the Security Header.

CertificateExplorer

The Car2x certificate explorer is displayed on this desktop.

On the left side you will find a list of certificates that are already available in the Car2x Certificate Explorer. The signature hierarchy of the certificates is displayed as a tree. Please also note the content of the Type column. On the right side of the Car2x Certificate Explorer you will find the interpreted content of the selected certificate.

Note

The Car2x certificate explorer is not assigned to a specific WLAN channel. Always open it from the View menu.
