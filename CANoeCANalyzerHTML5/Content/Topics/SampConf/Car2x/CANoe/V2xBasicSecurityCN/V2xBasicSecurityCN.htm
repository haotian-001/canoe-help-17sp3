Basic Security (V2x)

 

Note

Click on the blue link to load the sample configuration in CANoe.

Basic Security (V2x) - V2xBasicSecurity.cfg

Basic configuration how to use the Security CAPL API and the Certificate Manager to create valid WAVE Security Services in V2x messages according to IEEE 1609.2.

This sample configuration is intended to help the user quickly learn how to use the Security CAPL API and the Certificate Manager.

The simulation contains two nodes, one sender node and one receiving node. Valid and invalid certificates are stored in the Certificate Manager.

Node

Description

Sender Node

You can use the Sender panel to set, for a BSM as well as for a TIM, which certificate from the Certificate Manager should be used to send a message with a security header. A valid certificate and an invalid certificate are available to you; in addition, you can select the transmission type.

Then you can use [Send] to send out a packet.

All necessary parameters of the WAVE Security Service are set in the associated CAPL program (e.g. PSID, time stamp, etc.), so that you can use this sample code as a template for your own simulations. Its functionality is implemented with the Security CAPL API.

Receiver Node

The receiving node checks whether the packet signature is valid, and it represents the validity on the Receiver panel. The functions of the Security CAPL API are used here to read and evaluate information from the received messages.

Notes about the used 1609.2 standard

The sample configuration can be used with document versions IEEE 1609.2/D9.3 2011-09, IEEE 1609.2-2013 and IEEE 1609.2-2016 of the standard. Certificates for all versions are entered in the Certificate Manager, and the certificates that do not match the active version are inactive. The Certificate Explorer displays only the certificates that match the active version.

Additional information about the document version can be found in the help on WAVE Security Services (WSS) protocol.

Desktops

This configuration contains several desktops.

Setup

This desktop displays the Simulation Setup and Measurement Setup, and a Write Window.

The Simulation Setup contains the simulated senders and receivers. The System View of the Simulation Setup already contains the V2x database C2XNetwork. The Measurement Setup contains blocks you can use to analyze the data traffic in the network. General information, warnings, and error messages are output in the Write Window.

Notes

  • The sample configuration runs in simulated mode. Communication takes place only between the simulated nodes in CANoe. If you switch to real mode, the communication takes place via the configured WLAN adapter.
  • If you double click a node in the Simulation Setup, the CAPL Browser opens. Use the handler overview to display security-related functions.
  • Open the Car2x Certificate Manager (Options dialog) to browse the used certificates and keys.

Measurement

This desktop displays the sender panel and the receiver panel as well as the Trace Window.

The sender panel controls the send node, and the receive node performs the evaluation. The messages can be analyzed and compared with the result of the receive node in the Trace Window.

Note

In the Detail View of the Trace Window expand the entry WAVE Security Services | Summary to get an overview of all transmitted messages. Under entry WAVE Security Services | Message Tree you see all sent fields of the Security Header.

Certificate Explorer

This desktop displays the Car2x Certificate Explorer.

On the left side you find a list of certificates that are already available in the Car2x Certificate Explorer. The signature hierarchy of the certificates is shown as a tree. Note also the content of the Type column. On the right side of the Car2x Certificate Explorer you find the interpreted content of the selected certificate.

Note

The Car2x Certificate Explorer is not assigned to a specific WLAN channel. Always open it from the View menu.
