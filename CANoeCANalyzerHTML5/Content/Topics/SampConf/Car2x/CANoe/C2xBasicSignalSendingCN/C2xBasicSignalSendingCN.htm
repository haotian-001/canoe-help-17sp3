Basic Signal Sending (Car2x)

 

Note

Click on the blue link to load the sample configuration in CANoe.

Basic Signal Sending (Car2x) - Car2xBasicSignalSending.cfg

Basic configuration how to use the signal sending concept for the automatic sending based on Car2x messages according to ETSI EN 302 636-4-1 / ETSI EN 302 637-2/3

The sample configuration shows the signal sending concept with the help of a database. The simulation contains two ITS stations that demonstrate the setting of signals in the CAPL code.

The configuration takes place in the Car2x Network Explorer. For this, a message assigned to a network node must be navigated to in the Car2x Network Explorer: Network Node | SimCar1 | Tx Messages | CAM

The Attributes Window is located here at the bottom left of the Car2x Network Explorer. Various attributes can be configured here.

Starting the Configuration

At the start of the measurement, the vehicles start sending CAMs and DENMs and the vehicles approach each other.

Desktops

The sample configuration contains several desktops.

Setup

This desktop displays the Simulation Setup and Measurement Setup and a Write Window.

The Simulation Setup contains two network nodes that demonstrate the setting of signals. The Measurement Setup contains blocks you can use to analyze the data traffic in the network. General information, warnings, and error messages are output in the Write Window.

Note

  • The configuration runs in simulated mode. Communication takes place only between the simulated nodes in CANoe. If you switch to real mode, the communication takes place via the configured WLAN adapter.
  • If you double click a node in the Simulation Setup, the CAPL Browser opens. Use the Handler Overview to view the CAPL program.

Measurement

This desktop displays various windows for the display and analysis of the network traffic:

  • Map Window for display of ITS Stations in one of the supported maps. You can use online map data or integrate local map data.
  • Graphics Window for analysis of the SimCar2 heading.
  • Trace Window for analysis of the packets.
  • Data Window for display of latitude / longitude / heading for SimCar1 and SimCar2.
