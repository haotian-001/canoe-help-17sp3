System Configuration (V2x)

 

Note

Click on the blue link to load the sample configuration in CANoe.

System Configuration (V2x)- V2xSystem.cfg

Simulation of a C-V2X network with multiple ITS stations and display in the Map Window, based on the U.S. WAVE standard IEEE 1609.3 and the DSRC message sets SAE J2735/C-V2X SAE J3161/1.

Introduction

at the start of measurement, the system switches to Desktop Measurement and simulates an Ego vehicle driving on a predefined example route and its CV2x communication. The desired scenario can be selected in the panel and adapted to the route or started relatively.

Desktops

This configuration contains several desktops.

Setup

This desktop contains the Simulation setup and the Measurement Setups.

The Measurement Setup contains blocks with which the data traffic in the network can be analyzed. In the Write Window general information, warnings and error messages are displayed.

The Simulation Setup contains the database. This contains the ASNV modules for the US application messages. The Simulation Setup contains nodes according to the application scenario.

Additionally there are two more nodes in the Simulation Setup:

  • DuT_Simulator: Deactivate this node if the measurement is done on the real bus and the tested vehicle and its CV2x communication behavior are real. The node can be used to simulate both the CV2x communication and the driving behavior of the ego vehicle. In a CAPL program, a driving Ego vehicle is simulated on the loaded route.

  • ScenarioControl: In a CAPL program the loading and starting of the scenarios is controlled.

The document window contains a short description of the configuration and the contained scenarios.

Measurement

This desktop displays a Map Window, a Trace Window and panels.

  • The panels Scenario Control and Ego Vehicle Simulation to control the scenarios.

  • The panel Head Unit to visualize the use cases.

  • The Map Window to display the ITS stations.

  • The Data Window to display current values of signals and system variables.

In the Scenario Control Panel, there are two sections to configure the scenario.

In the Scenario section, the specific scenario is selected and controlled. There is an accordingly named drop-down list for selecting the scenario and the start mode. There are two start modes for the scenario:

  • Relative Scenario Start: the scenario is started relatively based on the current GNSS position and orientation.

  • Start On Route: the scenario is started based on the current GNSS position and orientation adjusted to the route specified in the Route section.

The Start Scenario and Stop Scenario buttons control the scenario.

In the Route section, a route can be defined using the file path (format: kml, gpx or nmea) and the name of the route. The name of the route in the file depends on the format:

  • .nmea: the filename

  • .gpx: the name of the trk object

  • .kml: the name of the Placemark object.

Additionally, the route can be configured as a closed loop in the Load As Loop drop-down list to let stations continuously travel the route. To apply the configuration correctly in real mode, the ego vehicle must be on the defined route.

In the Ego Vehicle Simulation panel, an ego vehicle can be simulated on the specified route. By pressing the Simulate GNSS Simulation button, the ego vehicle travels the route at the selected speed and sends its position as GNSS data. If desired, a CV2x communication can be generated from the sent GNSS data (real input or simulated) by checking Simulate CV2x Communication. If the DuT_Simulator node is disabled, this action is not possible.

Emergency Electronic Brake Light

This desktop displays various windows for simulating and displaying the Emergency Electronic Brake Light Warning (EEBL).

  • Data and Graphics Windows for displaying signals sent by the BSM.

  • Trace Window for analyzing the BSM.

  • Panel Head Unit for visual display of state values.

The simulated node Sender_EEBL sends BSMs.

Stationary Vehicle

This desktop displays various windows for simulating and displaying the Stationary Vehicle Warning (StatV).

  • Data and Graphics Windows for displaying signals sent by the BSM.

  • Trace Window for analyzing the BSM.

  • Panel Head Unit for visual display of state values.

The simulated node Sender_StatVehicle sends BSMs.
