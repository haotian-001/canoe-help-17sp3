Application and Test Scenarios (CV2x)

  • GLOSA – Green Light Optimal Speed Advisory (CV2x) - CV2xScenarioGLOSA.cfg
  • RLVW – Red Light Violation Warning (CV2x) - CV2xScenarioRLVW.cfg

Note: CV2x stands for Cellular Vehicle to X.

Simulation of CV2x application and test scenarios based on the Chinese standard T/CSAE 53-2017.

Introduction

After starting the CANoe measurement select the desired scenario on the CANoe Desktop Scenario in Panel Scenario. Press the green Start button in the Scenario panel to start the selected scenario.

Desktops

This configuration contains several desktops.

Setup

Contains the windows Simulation Setup and Measurement Setup. The database CSAE0053_MsgSet is integrated in the Simulation Setup. It contains the ASNV modules for the Chinese application messages. The Simulation Setup contains nodes according to the CV2x application scenario. The node HV exists in each scenario and transmits the geographical position of the tested vehicle simulated in scenario. Deactivate this node if the measurement is performed on the real bus and the tested vehicle and its CV2x communication behavior are real.

Measurement

Contains Trace Window, Write Window and Bus Statistics Window for analyzing CV2x data traffic during CANoe measurement.

Scenario

Contains a Map Window to display ITS stations in one of the supported maps. You can include online and local map data.

Document Window

The document window contains the description of one or more scenarios of the same use case and a reference to the definition in the Chinese Standard T/CSAE 53-2017.

Scenario Panel

Click the green Start button in the Scenario panel to start the selected scenario.

In this panel you can select a concrete scenario of the use case and control it with buttons Start or Stop.
