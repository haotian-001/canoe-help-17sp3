SIL Adapter Benchmark

 

Click on the blue link to load the sample configuration in CANoe.

  • SIL Adapter Benchmark – SilAdapterBenchmark.cfg
    Enables performance measurements (latency and throughput) of the SIL Adapter / SIL Kit for variable message sizes.

This sample configuration enables the performance of the SIL Adapter / SIL Kit to be measured by blocking message exchange (ping-pong) between two participants. The participants are a C++ application (SUT) and CANoe. The message exchange is realized via distributed objects (PingPong, MsgSize, Result).

Introduction

To start the message exchange as well as the performance measurement, proceed as follows:

  1. Open the Application_Build folder via the Open SUT Folder panel on the Setup desktop.

  2. Start the C++ application contained in this folder.

  3. In the in the Home ribbon tab, click  Start to start the measurement.

The message exchange (ping-pong) between the C++ application (SUT) and CANoe as well as the performance measurement have started. The measured latencies and throughputs are visualized immediately on the desktop Analysis. You can vary the message size while the measurement is running using the Message Size [B] slider on the desktop Analysis.

Desktops

This configuration contains the following desktops:

Setup

Analysis

 

Setup [▲ back]

The Communication Setup contains all configuration settings. Among others, the interfaces between the C++ application and CANoe are described here. This is done in the form of distributed objects defined in the DataSource.vcdl file.

The Communication Setup also contains the application model ApplicationModel.can. In this application model, the ping-pong mechanism on the CANoe side is implemented in CAPL.

Analysis [▲ back]

After you have started the C++ application (SUT) and CANoe as described above, you can view the performance of the message exchange in the desktop Analysis as follows:

  • In the Graphics Window Graphics Latency, both the current latency (individual points) and the time-averaged latency (solid line) are visualized. Note that the current latency corresponds to the mean value of the latencies of a sample of 50 Round Trips. The time-averaged latency is calculated using exponential smoothing.

  • In the Graphics Window Graphics Throughput, the current and averaged throughputs are visualized in the same way as the latencies.

  • In the Panel, you can read the current latency and the current throughput from two analog gauges. These values correspond exactly to the points in the Graphic Windows above and are additionally displayed here for intuitive understanding. In addition, you can use the Message Size [B] slider to vary the size of the messages between 10 B and 10 kB, even while the measurement is running.

  • The Trace Window displays events and value transfers between the SUT and CANoe. Here you see the exact values for latency and throughput during the measurement.
