SIL Adapter Basic (Python)

 

Click on the blue link to load the sample configuration in CANoe.

  • SIL Adapter Basic (Python) – SilAdapterBasicPython.cfg
    Shows how to interactively and automatically test a temperature control implemented in Python.

Introduction

The software to be tested is available as Python application. The software receives three temperature sensor values and determines whether heating or cooling should be activated. The functional system interfaces (Sensor1, Sensor2, Sensor3 and Heating) are modeled as Distributed Objects and serve for data exchange with the software.

To execute the Python application, you must install the SIL adapter runtime, which is provided as a Python package. Then, you can also open the folder of the Python software to be tested using the Open SUT Folder panel on the Destokp Setup. The application is located at Application_Python\application.py.

First start the measurement in CANoe and then one of the applications via the file system.

You can stimulate the temperature values either via a panel, Signal Generators, automated tests or via an environment model. You can observe the response of the tested application (heating or cooling active) both in the panel and in numerous analysis windows. Further details can be found in the descriptions of the individual desktops.

Setup [▲ back]

The Communication Setup contains all configuration settings. Among other things, the functional system interfaces of the software under test are described here. The description is in the form of distributed objects in a vCDL file.

In addition, the following application models are configured:

  • PanelHelper to calculate the average temperature value for the panel and to control the display for heating and cooling

  • EnvironmentSimulation to implement the environment simulation for the automatic setting of the temperature values

Exploratory Testing [▲ back]

The panel Room Temperature Control enables the interactive stimulation of the transmitted temperature values for the application. Deactivate Environment Simulation by Model to change the temperature values manually via the sliders. The panel also displays the heating and cooling values received from the application under test. The listed threshold values for heating and cooling define the expected behavior of the software.

The Signal Generators window can be used to replay temperature value curves. The signal generators can be controlled via Signal Generators (Home ribbon tab).

In addition, the desktop contains some analysis windows for observing the temperature and heating values: Graphics Window, Trace Window and State Tracker.

  • The Graphics Window displays the value progressions.

  • The Trace Window shows events and value transfers between the simulated environment in CANoe and the software under test.

  • The State Tracker allows you to visualize states and state transitions.

Automated Tests [▲ back]

The Desktop Automated Tests consists of a test configuration and a Test Trace Window.

The test configuration Heating Cooling Control contains the test unit Temperature Regulation, which was created with vTESTstudio. The test cases are displayed in the execution dialog. Implementation details can be viewed using vTESTstudio or vTESTstudio Viewer as part of the CANoe installation. Open the test design via the Open Test Design button.

Start the execution of the automatic tests by pressing the Start button in the Heating Cooling Control test configuration.

During test execution you can observe all actions of the test in the Test Trace Window.

When the tests are finished, all test results and a detailed report are available via the Test Report Viewer. Open it by clicking the Open Test Report button in the Test Configuration.
