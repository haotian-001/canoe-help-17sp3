SIL Kit (CAN)

 

Click on the blue link to load the sample configuration in CANoe.

  • SIL Kit (CAN) – SilKitCAN.cfg
    Shows the interaction of SIL Kit and CANoe, based on the Easy (CAN) sample configuration.

This sample configuration shows the interaction between CANoe and the Vector SIL Kit. The Vector SIL Kit is an open source library for distributed simulations and is made available via a publicly accessible repository on github.com.

Introduction

This configuration is based on the Easy (CAN) sample configuration. However, the CAN frames of the engine and the alternator are processed and sent by separate SIL Kit participants, which are also supplied with this configuration in executable form and source code. The sample configuration supports Real Bus mode and Simulated Bus mode with animation factor 1.

Analysis [▲ back]

To start the sample configuration correctly, you must execute the following actions in any order:

  • Start the measurement in CANoe.
  • Start the SUTs for the engine and the alternator via the button in the SIL Kit Setup panel.
    The SUTs are implemented as SIL Kit simulation participants.

After you have started the measurement and the SUTs in CANoe, you can send CAN frames to the engine and the alternator via the Panel Control. The three buttons start and stop the engine, or activate and deactivate the headlights / hazard lights.

The engine, the headlights and the hazard warning lights send their current status via CAN frames back to CANoe on a regular basis. Various visualizations of the received and sent data are presented:

  • The Graphics Window displays the status reported by the engine, headlights and hazard lights, i.e. at what speed the engine is currently running or whether a light is currently on. E.g. you can see in which cycle the warning flasher is flashing.
  • The State Tracker also displays the status reported by the engine, headlights and hazard lights.
  • In the Data Window, the last reported numerical values of the engine, headlights and hazard lights can be read.
  • The Display panel shows a visualization of the vehicle status.

Setup [▲ back]

In the Simulation Setup, you can view and change the settings and scripts assigned to each node and network.

The Control node translates the outgoing ignition signal or the outgoing light control signals from system variables into CAN frames.

The Display node reads the incoming CAN frames for motor and light data and stores the read values in system variables that are displayed in the Desktop Analysis.

Test [▲ back]

The Desktop Test consists of the test configuration Test Configuration and the panels Display and SIL Kit Setup.

The Test Configuration contains the test unit SILKitCANTest, which was created with vTESTstudio. You can view the implementation details with vTESTstudio or vTESTstudio Viewer, which is part of the CANoe installation. To do this, open the test design via  Open Test Design.

The test unit SILKitCANTest contains test cases for engine tests and headlight tests, which are displayed in the execution dialog of the Test Configuration. The tests send CAN frames to activate and deactivate the engine or headlight.

To start the tests, proceed as follows:

  1. In the Home ribbon tab, click  Start to start a measurement.

  2. In the SIL Kit Setup panel, click to start the SIL Kit environment.

  3. In the Test Configuration window, click  Start to start the tests.

The tests are started and the results of the tests and their runtime are displayed in the Test Configuration window.
