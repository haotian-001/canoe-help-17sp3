AI SIL Testing and Analysis

 

Click on the blue link to load the sample configuration in CANoe.

  • AI SIL Testing and Analysis – AISilTestingAnalysis.cfg
    Demonstrates testing and analyzing a System Under Test with an Artificial Intelligence (AI) Deep Learning approach using CANoe.

This sample configuration requires an external System Under Test with an AI implemented in the AI frameworks TensorFlow or PyTorch. Examples of this are provided in a repository at https://github.com/vectorgrp/AI-SUT-Demo. To set up the AI SUTs, please follow the instructions in the readme file.

Introduction

This sample configuration shows how you can address a system with an AI component via CANoe/CANoe4SW. With the help of Explainable AI methods (xAI), you can also make the results of the AI/Deep Learning networks comprehensible and thus explainable.

In the AI Demo-GitHub-Repository, you can find several sample applications with standard Deep Learning classifiers and object detectors. The applications incorporate networks pre-trained by TensorFlow or PyTorch in a System Under Test (SUT) that can communicate with CANoe/CANoe4SW. Communication between SUT and CANoe/CANoe4SW takes place via the SIL Adapter.

Please make sure that after starting the measurement of the sample configuration, the SUT is also started.

Setup [▲ back]

The data sources and help scripts of the sample configuration are managed here. With the help of ObjectDetection.vcdl, the interfaces to the AI Visualizer and the AI in the SUT are provided. Data such as images or annotations can be exchanged between CANoe/CANoe4SW and SUT via the DOs defined in vCDL. The PanelHelper_DataMediator.cs script contains the functions for communication between SUT and AI Visualizer.

Automatic Testing [▲ back]

The SUT and thus the AI can be tested using vTESTstudio.

In the Configuration panel, the path to a test folder is specified. By default, the folder is expected in the Testdata directory of the sample configuration. By starting the measurement and the test configuration, the specified folder is automatically searched for images and a ground truth file in COCO format named testset.json.

Sequentially, the images are sent to the external SUT containing the AI. The AI is supposed to recognize objects in the image and return the category_ids associated with the objects. If there is a response and the output of the SUT is within a timeout window, the result is checked against the ground truth. Each test result including input image is documented in the test report.

The test progress can be followed in the Trace Window.

Analysis [▲ back]

A more detailed analysis of the results of the AI from the SUTs is possible with the Analysis Desktop.

For this purpose, you can load any images in the Image Transmission panel by specifying the image path and send them to the SUT for evaluation by the AI components. The default directory is analogous to the Automatic Testing Desktop the folder Testdata in the folder of the sample configuration.

Two modes are allowed in the sample configuration: DetectionAndClassification evaluates the output of the AI network and displays them in the AI Visualizer and in the Top3 Label panel.

In the AI Visualizer the input image of the SUT is displayed. If the AI in the SUT is an object detector, the detected objects are displayed with their positions in the image as bounding boxes. If it is a classifier, the most likely results of the network are displayed in the Top3 label panel.

In addition to displaying annotations, xAI methods can also be visualized. In this example configuration, the relevance of each pixel of the input image for each AI decision can be displayed in so-called saliency maps in the AI Visualizer. By visualizing the pixel relevance, you can check which image sections and features the deep learning network in the SUT uses for decision making. This helps understand AI network decisions in both good and bad cases. To generate saliency maps, you must select the explainAI mode in the Image Transmission panel.

The Image Meta Information panel provides information about meta information of the images sent.

Example

For this example configuration, a SUT with a Deep Learning approach is expected. Currently, AI or deep learning networks are expected that use TensorFlow or PyTorch. You can see how these can be driven using the sample applications in our GitHub Repository.

There, image classifiers from TensorFlow or object detectors from PyTorch can be loaded. These are accessed via Python classes. Communication between SUT and CANoe takes place via DOs and the SIL Adapter.

Data from CANoe is converted to the format required for the network via a pre-processing function. The native result of the deep learning network is then filled back into DOs via another method and sent toward CANoe.

To understand how a Deep Learning network determines an annotation, an xAI method is available in the repository. This quantifies the relevance of each pixel of the input image for a particular annotation. In this way, so-called saliency maps are computed and then displayed in the AI Visualizer.

If you want to test your own object detectors or classifiers, you can replace models from the sample applications with your own. If necessary, the pre- and post-processing steps have to be adapted.
