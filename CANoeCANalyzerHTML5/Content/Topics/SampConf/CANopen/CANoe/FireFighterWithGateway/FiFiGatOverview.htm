Fire Fighter with Gateway

 

Click on the blue link to load the sample configuration in CANoe.

The simulation can only be started if both the option CANoe .CANopen and the option CANoe .J1939 are installed for this configuration.

  • Fire Fighter with Gateway – CANopenGateway.cfg
    Simulation of a CANopen system in a fire escape car with a gateway to J1939.

This configuration should simulate the possibility of the information exchange between two protocols (CANopen and J1939). The vehicle can be accelerated or braked using the panels (drive train, J1939). If the vehicle is stationary, the fire escape (structure, CANopen) can be extended.

Introduction

After the start of the measurement, the vehicle can be started by right clicking the switch Ignition (switch must be in position 3) and then left clicking on the Starter button. The vehicle acceleration is set using the Accelerator position slider. In the Gear input field, the current gear can be set by actuating the [+] or [-] buttons. The current acceleration and the engine speed can be read in the Combined display.

If the fire escape (the CANopen superstructure) is to be used, the vehicle must be stopped (put in gear 0) and the switch Park Brake must be actuated. Then activate the PTO switch. The supports of the fire escape should be locked by clicking the [Down] button. The ladder can be extended by actuating the [Up] button.

Note

The engine speed (see Combined display) rises as soon as the ladder of the superstructure is extended or retracted. This illustrates that there is an exchange of information between the two protocols (J1939 and CANopen). This exchange of information is realized via the Truck Trailer Gateway (TTGateway).

Simulation Setup[▲ back]

This desktop contains the Simulation Setup with the network nodes to be simulated. The networks CANopen and J1939 are contained in the system view. The connection between the two systems is made using the TTGateway node.

Measurement Setup [▲ back]

This desktop contains the Measurement Setup. An additional Trace Window supplements the Measurement Setup for displaying J1939 messages.

Certain CANopen messages are filtered out via the CAN Filter Block before the CANopen Trace Window.
