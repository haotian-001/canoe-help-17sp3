System Configuration (CANopen)

 

Click on the blue link to load the sample configuration in CANoe.

  • System Configuration (CANopen) – CANopenSystem.cfg
    Simulation of a CANopen system with a fire escape using the application profile CiA402.

This configuration represents a greatly simplified application for using a fire escape. The possibilities for using bitmaps in panels and CAPL programming should be revealed. A CANopen Master and four CANopen Slaves are used for simulating the fire escape.

Note

The devices LadderDrive, SupportFront and SupportRear contained in the sample configuration were implemented based on the device profile CiA402.

Introduction

All CANopen devices register on the bus with their boot-up message and go to Pre-Operational status after measurement start. This status is indicated by yellow LEDs on the Network panel. The NMTMaster device has been set up so that the other devices (Slaves) are configured and started automatically after the boot-up (CMT functionality).

As soon as all devices have been configured and started, the LEDs change from yellow to green on the Network panel.

Note

The status of the devices (Operational, Pre-Operational etc.) is also displayed in the CANopen scanner (CO Scanner window).

The supports for the fire escape can be extended with the [Down]button (Group: Support). If the supports are engaged, the Locked display in the Ladder group disappears. The fire escape can now be extended using the [Up] button which leads to blocking the supports. This is indicated with the Locked display for the Support group. To retract the fire escape and the supports again, execute these procedures in reverse sequence.

Emergency messages from the slaves can be sent with the toggle switch on the Network panel. The appearance of an emergency message is displayed in the CANopen scanner accordingly.
