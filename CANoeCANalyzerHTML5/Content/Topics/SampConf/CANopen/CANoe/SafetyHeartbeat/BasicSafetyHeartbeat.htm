Basic Safety Heartbeat (CANopen)

 

Click on the blue link to load the sample configuration in CANoe.

  • Basic Safety Heartbeat – CANopenBasicSafetyHeartbeat.cfg
    Simulation of a simple test bench with pressure sensors using CANopen Safety for data exchange. Error monitoring is carried out by heartbeat.

The configuration demonstrates a simple test bench having a CANopen Master and three CANopen Slaves (Sensors). The sensors transmit their current values (pressure) to the master, which displays them on a display element (Panel Display). The transfer of data is done via SRDOs. Heartbeat is configured in the nodes for error monitoring.

Introduction

The CANopen devices register themselves on the bus with the boot-up frame and are then started by the Master. The Operational status is indicated by the green LED on the panel.

Pressure Change
A pressure change on a sensor can be generated with the Pressure slider for panels Sensor_1, Sensor_2 and Sensor_3. As CANopen Safety is concerned, the value is transmitted bit-inverted. There is a second slider for this. If you turn off the Synchronize Values check box, you can move the sliders independently of each other. If the values do not match, an error in the CANopen Safety communication is triggered by the receiver. To do this, it sends a Global Fail Command Frame, which is displayed in red in the Trace Window.

Switch off Devices
The entire network can be switched off and back on again with the main switch (Main Switch). After switching on, the master starts the sensors again automatically. This procedure can be traced in the Trace Window.

A device-failure can be simulated by actuating the plug symbols on the panels. If the master determines a sensor failure during operation, this is indicated with the red warning light ERR on the panel Display.
