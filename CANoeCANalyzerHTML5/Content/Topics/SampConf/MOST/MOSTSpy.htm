Spy

 

Click on the blue link to load the sample configuration in CANoe.

  • Spy – MOSTSpyDemo.cfg
    CAPL analysis nodes in the Measurement Setup, MOST Graphic and Data Window configuration as well as MOST Filter.

This example runs with a MOST network interface (e.g., VN2600) in spy mode and any timing master in the MOST ring.

The following functions are shown in this sample configuration:

  • Connection of CANoe to a MOST network interface (e.g., VN2600) in spy mode
  • CAPL analysis node in the Measurement Setup (examples: Busload and Response Time)
  • Filter options in the Measurement Setup using a MOST Application Filter
  • Configurable Graphics and Data Window
  • Online and offline analysis

Example  - Busload

In the Measurement Setup (Setup desktop), the BusLoad CAPL node is attached before the Graphics - Busload Graphics Window. It measures the busload, that is generated by the nodes (up to 16) in the ring. The results are illustrated in the Graphics Window (Busload desktop) over time.

The following values are measured/calculated (in messages/s):

  • Current value
  • Moving average (default setting: over 15 s)
  • Minimum
  • Maximum

These four values are also provided as percentages of the maximum total busload. The results are transmitted in eight messages, each of which contains one of these values for all 16 possible nodes. The message structures are defined in a brief XML catalog, which is attached to the configuration.

Configuration options

  • Some interesting values have already been configured in the Graphics Window (Busload desktop). They can simply be activated or deactivated in order to obtain a different view of the system. The value ranges displayed [msgs/s] and [%] are coordinated for a bus frequency of 44.1 kHz so that the signals are located precisely on top of one another.
  • Any other signals can be inserted in or removed from the window.
  • The time period, over which the moving average is mapped, can be set in the BusLoad CAPL program in the Init_ConfigurationVariables user function.
  • The bus frequency can also be set in the BusLoad CAPL program in the Init_ConfigurationVariables user function. This setting only affects the percentages.

Example - Response Time

In the Measurement Setup (Setup desktop), the ResponseTime CAPL node is located before the Data - Response Time Data Window. It measures the response times of Command-Report message pairs on the control channel, which are set in a MOST Application Filter, located before the CAPL node. The results are illustrated in the Data Window (Response Time desktop) in the form of a histogram.

The histogram consists of 10 ranges. Every range has the same width (in ms), which is specified in Histogram range width. The first range starts from a Histogram offset value (in ms). An offset of 0 ms and a range width of 20 ms are set, giving the histogram a total range of 0 ms to 200 ms. Two additional ranges, Response times below Range 1 and Response times above Range 10, collect all response times that fall below or above the range of the histogram itself. The number of monitored Command-Report message pairs is also displayed, along with the minimum and maximum response times.

The analysis node distinguishes between the different Command-Report message pairs using the source and destination addresses of the messages. Any Command-OpType (with the exception of Set) will be dissolved by any Report-OpType, provided that the addresses match one another. Therefore, it is essential that the Command-Report message pairs to be considered are precisely defined.

Configuration options

  • The Command-Report message pairs to be considered can be set in the Measurement Setup (Setup desktop) in the MOST Application Filter. The most precise results are obtained if exactly two lines are inserted in the filter, which define the FBlock, Function and OpType of the command and the report. Of course, you can also make any other settings.

    • In the default setting, a line is inserted here, which only defines the AudioAmplifier FBlock. This means that all commands and reports of all the FBlock functions are considered.

  • The offset and the width of the histogram ranges can be set in the ResponseTime CAPL node in the Measurement Setup (Setup desktop), Init_ConfigurationVariables user function.

Bus-statistic signals

Bus statistics signals are illustrated in the Graphics – Bus Statistics Graphics Window (Bus Statistics desktop) over time. These signals relate to the entire bus. Other, predefined MOST bus statistics signals can be added via the Graphics Window context menu command Insert Signal….

Offline mode

If the configuration is operated in real mode, a logging file is created. In offline mode, this logging file is configured as a source file and the analyses described above can be applied to it. Of course, any other logging files can also be specified and analyzed as an offline source.
