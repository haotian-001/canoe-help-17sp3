MOST System Demo - Simulation

 

Simulation Setup

The Simulation Setup of the sample configuration shows:

  • the three simulated devices Head Unit, CDC (CD changer), and Amplifier,
  • the assignment of the applications (CAPL node) to the devices,
  • the connection of the devices with the simulation channels (MOST 1, MOST 2, MOST 3), and
  • the assignment of the network description database (MOSTRing.DBC) and the message description (MOST function catalog).

Controller applications

The CAPL nodes HeadUnit.CAN (in the head unit device) and AudioConsole.CAN (in the amplifier device) are typical controller applications. They are used to demonstrate the following:

  • Reception and analysis as well as combining and sending AMS messages while using the symbolic definitions from the MOST function catalog or CANdb database.
  • Sending messages via the address handler
  • Connection of the MOSTHigh modeling library for the reception of transmissions of the MOST High Protocol (see callback function MHP_IndRxBufferReq in the CAPL node HeadUnit.CAN). (See further Desktop High Protocol).
  • Configuration of shadow function blocks by using the shadow service and notification shadow service of CANoe
  • Integration of panels with control and display elements (see User Interfaces desktop)

Function block applications

The majority of simulated applications is classic MOST function blocks that receive command messages and send report messages. The following is shown in the CAPL programs of the simulation:

  • Sending and receiving AMS messages
  • Sending via the MOSTHigh modeling library (see the MHP_StartFileTransmission() function in Navigation.CAN)
  • Configuration of a CAPL node as MOST function block. For this purpose, observe the assignment to the nodes in the MostRing.DBC network description database.
  • Use of the notification service and function service of CANoe for efficiently creating MOST function blocks.
  • Use of CANoe standard implementations of the function blocks NetworkMaster and PowerMaster

The function blocks ConnectionMaster and AudioDiskPlayer are sufficiently modeled by the corresponding CAPL programs (ConnectionMaster.CAN and AudioDiskPlayer.CAN) so that they can be used in other simulations directly or with minor adaptations.

The remaining function blocks (AudioAmplifier, AmFmTuner, Navigation, VectorSystemManager) contain only the functions required for the simulation, which are partially proprietary.

Adaptation of the simulation

The simulation concept of CANoe allows for a flexible adaptation of the modeled system.

  • The devices can be removed from the simulation, e.g. to replace them with real devices by disabling them in the system view of the simulation design. Individual applications can also be removed in the same way by disabling the corresponding CAPL nodes.
  • Simulated devices can be combined by assigning the same simulation channel to them. This allows for a rest bus simulation on the application layer with only a few MOST network interfaces.
  • Applications can be moved into other devices using drag and drop.
