MOST High Protocol - Multi Connection Use Cases

 

  • In the Simulation Setup there are two devices: the CAPL node DSO (data source) is contained in the first device and the CAPL node DSI (data sink) in the second.
  • The DSO CAPL node implements two data sources that are controlled via the Data Source 1 and Data Source 2 panels. The DSI node, on the other hand, implements two data sinks controlled via the Data Sink 1 (DSI 1) and Data Sink 2 (DSI 2) panels.
  • After the start of measurement, the function address that is using the relevant data source or data sink to send or receive data via the high protocol is displayed in the top line of all panels. It is evident that DSO 1 sends to DSI 1 and DSO 2 sends to DSI 2.
  • In the panel of a data source there is a two-column hex editor control that can be used to enter the payload data for transmission (and it is also possible to paste data from the clipboard). The transmission is started by clicking the Send button.
  • Alternatively a data transmission can be started using the Fill&Send button and the edit box located to the right. When you click on the button the byte array is automatically filled with the number of bytes indicated in the edit box and is then transmitted immediately.
  • The panel of a data source also contains a send cyclic check box. If this check box is activated the given payload data is sent repeatedly via the existing connection at intervals of several seconds. In this manner it is relatively simple to create a situation in which both data sources are active simultaneoulsly.
  • In all the panels there is a Stop button that can be used to end the transmission in a protocol compliant manner.
