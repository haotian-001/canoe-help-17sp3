MOST High Protocol - Use Cases

 

  • Each of the two nodes A and B are assigned a sender and receiver panel; the button at the top right in each panel allows switching between them.
  • The target address of the transmission (Target Address) can be set in the sender panel, the own receiver address (My Address) in the receiver panel.
  • In the sender panel, the useful data to be transmitted can be provided either as byte array with a maximum size of 65536 bytes (Single packet option, represented in the two-column hex editor control) or as a file (File option, file path in the file selection control). The transmission of the data is started via the Send button at the bottom left.
  • The data transfer can also be started by using the Fill&Send button and the input field located to the left of it. When clicking on the button, the byte array is first automatically filled with the number of bytes specified in the input field and then immediately transmitted.
  • Analogous to the sender as receiver buffer, a file or the byte array displayed in the hex editor can be selected in the receiver panel. If the byte array is selected for reception, and it is determined during the transmission that the next block does no longer fit into the remaining free space of array, it is written again from the start.
  • Each panel contains a Settings button which opens an additional panel for setting the transfer parameters of the respective node. This panel is divided into the settings for send mode (Tx Options) and receive mode (Rx Options).
  • For the send mode, you can set the transport mode (Control or Packet), the number of bytes per frame (Bytes/Frame), the acknowledge mode (Block or Frame), and the priority. The changes are adopted as soon as the respective control has been set.
  • For the receive mode, you can set the number of frames (Frames/Block), the size of the frames (Bytes/Frame), the average interrupt rate (AIR) in microseconds, and the priority. Setting the receiver properties requires activating the Set button after the entries since it must always be checked whether the specified frame size and number does not exceed the maximum block size. If no setting of the receiver properties is performed after starting the measurement, the number of frames and frame size automatically adjust to the parameters suggested by the sender when establishing the connection.
  • It is possible to hold an on-going transfer on the sender and receiver panels by enabling the check box at the bottom left (Hold). Depending on the panel, a HOLD CONNECTION TX or HOLD CONNECTION RX message is cyclically sent from the node.  The LED illuminates as soon as the connection is maintained, i.e. even if the HOLD messages are sent by the other side.

      

Fault Injection: Desktop MHP Fault Injection

  • The Fault Injection option can be activated on sender and receiver panel A in order to generate protocol errors during a transmission in a targeted way. The Pre Send Settings and Pre Receive Settings buttons can be used to manipulate outgoing and incoming messages.
  • The PreSend panels can be used to manipulate messages generated by the MOST High DLL before they are transmitted via MOST. Depending on the message type, the message fields or data bytes can be overwritten starting from a selected position. The Send message n time(s) setting can be used to send a message 0..n times.
  • The PreReceive panels can be used to manipulate messages before they are processed by the MOST High DLL. Depending on the message type, the message fields or data bytes can be overwritten from a selected position. Receipt of the message can also be suppressed.
  • The Overwrite AIR setting in the Fault Injection sender panel can be used to change the minimum interval between DATA FRAMES specified by the receiver in order to provoke faster transmission.
  • The Use Burst Mode setting switches measurement of the AIR interval from MOST message transmission responses to transmission requests. This enables faster transmission because it eliminates waiting for the respective response. In case of transmission delays, however, this can lead to violation of the minimum AIR interval
