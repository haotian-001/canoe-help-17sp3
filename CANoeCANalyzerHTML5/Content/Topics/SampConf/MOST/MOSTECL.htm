Electrical Control Line

 

Click on the blue link to load the sample configuration in CANoe.

  • Electrical Control Line – MOST_ECL.cfg
    Contains an implementation of the protocol for the MOST150 Electrical Control Line (ECL).

Configuration

The API ECL.CIN used for the ECL protocol is realized in CAPL and can be found in the following folder: Open Folder

Here, the available API interfaces and the necessary callbacks are defined (see comments in source code).

This example is designed so that the ECL system test and the electrical wake-up (EWU) pulse can be simulated.
Two MOST devices are configured for this. The ECL system test initiator is incorporated in Device1. Two ECL system test participants that can react individually to an ECL system test are incorporated in Device2.
In addition, the Measurement Setup also includes an ECL observer that passively monitors the events on the ECL line and writes them to the Write Window.

In order to use the sample configuration, it must be operated in real bus mode with two VN2640 network interfaces.

Desktops

This configuration contains the following desktops:

Simulation

Trace

Setup

 

Simulation [▲ back]

The running simulation is displayed and controlled with panels on this desktop. It provides a separate panel each for the initiator and the two participants. In addition, the level characteristic of the ECL signal is displayed in the ECL Level Graphics Window.

The Initiator Panel

This panel controls the ECL system test initiator.

Here, the parameters for the ECL system test are specified, the test is started, and the results of the test and any error messages are displayed. In addition, the electrical wake-up pulse can be sent using this panel.

The Participant Panels

The two panels for the participants each control one ECL system test participant.

Here, the device class of the participant can be set, and the results that this participant is to deliver in response to a system test can be specified. In addition the currently received test parameter is displayed.

Global Options Panel

This panel can be used to make global settings, such as activating debug outputs and setting the ECL termination resistance for the respective MOST channel.

The "ECL Level" Graphics Window

The Graphics Window shows the ECL signal level for participant and initiator.

Trace [▲ back]

A Trace Window that outputs MOST and ECL messages and events is incorporated on this desktop. Also visible is a Write Window that displays text outputs of the CAPL nodes.

Setup [▲ back]

The ECL configuration with two devices is represented here in the Simulation Setup: Device1 contains the ECL initiator as CAPL node (source code: Nodes\AppECL_STI.CAN) and Device2 contains the ECL participants in another CAPL node (source code: Nodes\AppECL_STP.CAN). These *.CAN files use the ECL CAPL-API as is provided by ECL.cin. The Measurement Setup contains the ECL observer as CAPL program before the Data Window (source code: Nodes\AppECLObserver.CAN).
