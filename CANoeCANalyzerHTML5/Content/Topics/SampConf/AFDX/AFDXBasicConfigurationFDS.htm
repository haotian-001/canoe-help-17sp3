Basic Configuration FDS (AFDX)

 

Note

Click on the blue link to load the sample configuration in CANoe.

Basic Configuration FDS (AFDX) - AFDXBasicFDS.cfg

Set GPS coordinates in FDS and send them.

This sample configuration shows the sending functionality of Functional Data Sets (FDS). Starting the measurement triggers an exemplary flight route in the area of Stuttgart.
The modified GPS values are automatically transferred and added to the FDS singal by means of a background CAPL program. Only then the complete FDS will be sent.

Start the Configuration

On measurement start, there is a switch to desktop Trace. It remains open after you ended the measurement.

Desktops

This configuration contains several desktops with different windows.

Configuration

This desktop contains the Simulation Setup, the Measurement Setup and a Write Window. The Simulation Setup contains an AFDX network (AFDX1)and the simulated node IOM_APP1.

The node IOM_APP1 works as a send node. It retrieves the GPS values, sets the FDS signal and sends the complete FDS.

Trace

This desktop contains the FDS Demo Panel, the Map Window, the Trace Window and the AFDX VL Statistics Window.

In the FDS Demo Panel, the current GPS coordinates are displayed under [Current Value].

In the Map Window, you can see the current position as well as the already covered flight route on a map. You do not need internet access to display the map.

Analysis

This desktop contains a Data Window, a Graphics Window and a Bus Statistics Window.

The Data Window shows the transferred signal values as well as some additional bus statistic values. You can find the FDS-values under the names GPS1.Latitude and GPS1.Longitude, the FDS values under the names Demo_LAT_float and Demo_Long_float.

The Graphics Window only shows the FDS values under the names Demo_LAT_float and Demo_Long_float.

The Bus Statistics Window shows bus specific statistic values.
