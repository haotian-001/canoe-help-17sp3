Man In The Middle (AFDX)

 

Note

Click on the blue link to load the sample configuration in CANoe.

Man In The Middle (AFDX) - AFDXManInTheMiddle.cfg

Forward AFDX messages between two channels with modifying the signal content in mode Real bus (hardware required).

Man In The Middle Simulated (AFDX) - AFDXSimManInTheMiddle.cfg

Forward AFDX messages between two channels with modifying the signal content in mode Simulated bus (no hardware required).

In a Man In The Middle (MiM) scenario messages are forwarded between two channels including modifications of the signal contents.

In the sample configuration (real mode) a message DEMOMSG_INT3 is sent via CAPL on the first channel (AFDX1) and forwarded via loopback cable to the second channel (AFDX2). Some signal values of the message can be changed via slider. The forwarding is done via a gateway node that is realized in CAPL: incoming messages are analyzed, relevant signals modified and sent back to AFDX1.

Via CAPL the signals IOM_DEMO_VFG_OIL_TEMP_A_31 and IOM_DEMO_VFG_OIL_TEMP_A_32 of message DEMOMSG_INT3 are modified, the signal IOM_DEMO_VFG_OIL_TEMP_A_33 is sent unmodified.

Note

To use the sample configuration four network interfaces (e.g. two VN5610) are necessary and the outputs must be connected directly.

If no hardware is available you can use the sample configuration ManInTheMiddleSimulated.cfg to simulate the scenario. Due the cable connection cannot be simulated, the manipulation of the signals is done on the first channel (AFDX1) and the modified message is sent to AFDX2. The programming of the CAPL gateway is almost identical, only the node assignments are inverted.

Introduction

On measurement start, there is a switch to desktop Trace.

With [Shot] you send a message on port AFDX1. If you activate the check box Send Cyclic the message is sent cyclically. If you activate the check box Signal Temp A 33 the value of signal Temp A 33 is changed cyclically.

Desktops

This configuration contains several desktops.

Configuration

This desktop contains the Simulation Setup with two AFDX networks (AFDX1 and AFDX2) and a gateway between AFDX1 and AFDX2, as well as the Measurement Setup.

Trace

This desktop contains the panel Desktop Setup and two Trace Windows to show the sent AFDX frames (AFDX1) and the manipulated AFDX frames (AFDX2).

IOM_APP1

The left area of the panel provides the controlling of the simulated AFDX network (AFDX1) and the display of the signal values that are sent to the second AFDX network via port AFDX2.

With the slider Signal Temp A 31 the signal value is changed and directly sent with message DEMOMSG_INT3. Expand the entry DEMOMSG_INT3 and the corresponding signal state in the Trace Window to display the signal value.

With the slider Signal Temp A 32 the temperature value is changed. The signal value is sent two seconds at the earliest after the last value change.

The signal Signal Temp A 33 is cyclically sent each 500 ms. The signal value is automatically changed, the slider only displays the current signal value. The field Tx displays the number of messages that are sent from node IOM_APP1.

IOM_APP2

The transmitted signal values are displayed in the appropriate fields. The field Rx displays the number of messages that are received via port AFDX2.

Graphic DEMOMSG_INT3

This desktop contains a Graphic Window to show the transmitted signal values.

Analysis

This desktop contains an AFDX VL Statistic Window, a Data Window, and a Write Window. The AFDX VL Statistic Window shows the statistic values on Virtual Link level. The Data Window shows the transmitted signal values and some additional statistic values of message DEMOMSG_INT3.
