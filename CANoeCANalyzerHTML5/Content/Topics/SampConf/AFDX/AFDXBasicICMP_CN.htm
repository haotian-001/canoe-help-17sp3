Basic ICMP (AFDX)

 

Note

Click on the blue link to load the sample configuration in CANoe.

Basic ICMP (AFDX) – AFDXBasicICMP.cfg

Send and receive ICMP messages with CAPL.

This sample configuration shows the possibility to transmit and receive ICMP messages via CAPL. For the sample configuration real hardware with two interconnected channels is necessary.

The sample configuration contains two AFDX nodes of which the first one creates and transmits the ICMP message in the network AFDX1 and the second one receives and responds to the message on the network AFDX2. The content of the ICMP message can be changed or error scenarios in the second node can be activated by means of a panel. Failure to answer or resending the wrong payload are possible error scenarios.

Starting the Configuration

At the start of measurement the desktop Trace is displayed and ICMP messages can be send via the panel.

Composition of the Sample Configuration

The Sample Configuration contains various desktops.

Configuration

This Desktop contains the Simulation and Measurement Setup. In the Simulation Setup, the node IOM_APP1 is located in the network AFDX1 and the node IOM_APP2 is located in the network AFDX2.

Node IOM_APP1

This node is intended as ICMP send node and transmits ICMP messages. When transmitting, the current simulation time is inserted as ICMP payload to compare it to the payload of the ICMP respond. Via the panel a defective ICMP message with a false Virtual Link can be sent.

Node IOM_APP2

This node checks the corresponding network for ICMP messages and responds to them. Via the panel error cases can be inserted. An error case can be that responses of the node fail. As a consequence, the node IOM_APP1 runs into an ICMP Timeout and displays this. Also, via the panel the ICMP payload can be changed leading to an ICMP error in the node IOM_APP1.

Trace

This desktop contains the panel DemoPanel, AFDX VL Statistics, Bus Statistics and the Trace Window showing all AFDX frames. In the Statistics Windows, statistic values concerning bus or virtual link are displayed.

When pausing the update of the packets (button [Pause]), the details of the selected frame are displayed in the detail view of the Trace Window.

Panel DemoPanel

In the upper area the behavior of the response node IO_APP2 can be controlled. Responses can fail or contain a modified ICMP payload.

In the lower area, valid and invalid ICMP messages can be transmitted and the size of the ICMP payload can be modified. If an ICMP response arrives at node IOM_APP1, the runtime can be calculated by means of the ICMP payload containing the simulation time.

The panel displays this runtime and if the ICMP payload is correct.

Analysis

This desktop contains a Graphics Window and a Data Window displaying the statistic values of both AFDX buses.
