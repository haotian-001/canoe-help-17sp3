Basic Configuration (AFDX)

 

Note

Click on the blue link to load the sample configuration in CANoe.

Basic Configuration (AFDX) - AFDXBasic.cfg

Send AFDX messages with CAPL.

This sample configuration demonstrates how to send AFDX messages with CAPL. Furthermore the visualization of transmitted signal values in different analysis windows is shown.

The sample configuration contains two simulated AFDX nodes and a Replay Block. The Replay Block cyclically sends a recorded message sequence. The simulated node IOM_APP1 can be controlled with the panel Desktop Setup. Three temperature values are available which can be changed, except the value Signal Temp A 33 that is cyclically changed.

Start the Configuration

On measurement start, there is a switch to desktop Trace. The Replay Block starts to send the recorded message sequence (message DEMOFLOAT_MSG_1). Furthermore the signal value Signal Temp A 33 is cyclically changed so that the message DEMOMSG_INT3 is transmitted by the simulated node IOM_APP1.

Desktops

This configuration contains several desktops.

Configuration

This desktop contains the Simulation Setup and the Measurement Setup. The Simulation Setup contains the Replay Block and two simulated nodes IOM_APP1 and IOM_APP2.

Node IOM_APP1

This node is the send node.

The signal value Signal Temp A 33 is cyclically transmitted. The signal values Signal Temp A 31 and Signal Temp A 32 can be controlled with panel Desktop Setup. The CAPL file of the node provides setting the signal values with function AfdxSetSignalInt that is part of the external CAPL DLL AFDX_IL.dll. This DLL is part of the option AFDX.

Node IOM_APP2

This node is the receive node.

The received signal values are displayed on panel Desktop Setup.

Replay Block

The Replay Block cyclically sends a recorded message sequence.

Trace

This desktop contains the panel Desktop Setup and a Trace Window that shows all AFDX frames.

If the update of the packets is paused (button [Pause]) the detail view of the Trace Window shows the details of the selected frame.

Panel Desktop Setup

IOM_APP1

The right area allows to control the simulated node IOM_APP1 and shows the signal values that are transmitted to node IOM_APP2.

With the slider Signal Temp A 31 the signal value is changed and is transmitted with message DEMOMSG_INT3. Expand the entry DEMOMSG_INT3 and the signal state to watch the signal value in the Trace Window.

With the slider Signal Temp A 32 the temperature value is changed. The signal value is transmitted after at least two seconds of the last value change.

The signal Signal Temp A 33 is cyclically transmitted with a frequency of 500 ms. The signal value is changed automatically, the slider visualizes the current signal value only. The field Tx shows the number of messages that are sent from node IOM_APP1.

IOM_APP2

The transmitted signal values are displayed in the appropriate text fields. The field Rx shows the number of messages that are received by node IOM_APP2.

Graphic DEMOMSG_INT3

This desktop contains the panel Desktop Setup to control the simulated nodes and a Graphics Window that displays the transmitted temperature values.

Graphic DEMOFLOAT_MSG_1

This desktop contains a Graphics Window that displays the signal values of message DEMOFLOAT_MSG_1. This message is cyclically sent by the Replay Block.

Analysis

This desktop is activated after measurement stop. The AFDX Statistics Window shows statistics values on Virtual-Link layer.

The Data Window displays the signal values that are transmitted by node IOM_APP1. Statistics values of message DEMOMSG_INT3 are displayed too. Additionally the BAG value of these frames and of frames DEMOFLOAT_MSG_1 are displayed.

Notes

  • The BAG values of frames DEMOMSG_INT3 are not calculated in mode Simulated bus because the message is sent by the CAPL file. If you have an AFDX hardware you can switch in mode Real bus and the BAG values can be calculated.
  • The BAG values of frames DEMOFLOAT_MSG_1 can be calculated in mode Simulated bus because these values are recorded with real hardware and stored in the replay file.
