ADAS System (Scenario Editor)

 

Note

Click on the blue link to load the sample configuration in CANoe.

ADAS System (Scenario Editor)- ADASSystemSE.cfg

Shows the use of all ADAS features based on the test of an ADAS algorithm. A scenario created with the Scenario Editor is used for stimulation.

Introduction

At measurement start, the system switches to the desktop Measurement. A scenario created with the Scenario Editor is loaded and started with the measurement. This scenario plays an emergency braking of a vehicle in front before an intersection. You can observe the scenario in the Scene Window and in the Map Window. The ADAS algorithm under test is included as an Application Model and is stimulated by the scenario. A test unit tests the ADAS algorithm and generates a test report.

Desktops

This configuration contains several desktops.

Setup

The measurement setup, the Communication Setup as well as the Scenario Manager are displayed on this desktop.

A Trace Window, a Scene Window and a Map Window are integrated in the measurement setup, as well as an inactive logging block.

In the Communication Setup, static ADAS objects were created via the model editor. In addition to the objects that appear in the scenario (EgoVehicle, BrakingVehicle, EmergencyVehicle and FrontRadar), an OutputSimpleAlgorithm object was also defined. This contains the output of the ADAS algorithm to be tested.

In the Scenario Manager, the scenario EmergencyBrakeScenariowas loaded, which was previously created via the Scenario Editor. This scenario starts simultaneously with the measurement.

In addition, several Application Models are included in the Communication Setup:

  • AlgorithmUnderTest
    The ADAS algorithm to be tested as C# code. In the example, there is a very simplified emergency braking assistant that warns the driver and brakes independently if the Time To Impact is less than 3s.
  • DashboardControl
    Controls the display of the Dashboard panel with CAPL code.
  • MapObjectsVisualizer
    Uses the Map API to display the objects of the scenario in the Map Window.

Measurement

This desktop displays the panel Dashboard, a Trace Window, the Scene Window, the Map Window, and the Test Configuration.

Panel Dashboard

Displays the current speed and RPM of the Ego vehicle at all times. If a warning is active from the ADAS algorithm for Emergency Brake Assist, a warning is displayed centrally in the Information screen.

Trace Window

Displays the data received from the scenario. The FrontRadar.sensor_info object contains information such as the Sensor View (Sensor ID, Sensor Mounting Position) or the number of currently detected objects. The currently detected objects in turn are displayed as independent objects in the Trace Window (Detected_id_xxx_by_FrontRadar). These contain all known information about the objects such as dimension, position, speed or classification.

Additionally the object OutputSimpleAlgorithm can be seen. In this the output of the AlgorithmUnderTest Application Model is stored.

Scene Window

Visualizes the data received from the scenario. Based on the information in the detected objects, the position and dimension of the objects are displayed.

Map Window

Visualizes the scenario on a map. The Map API was used to display the objects.

Test Configuration

An included test unit checks the ADAS algorithm included in the Application Model AlgorithmUnderTest. Different tests check e.g. if a radar object was detected at all, if the probability rate exceeds a certain threshold, if the radar object is in the same lane. If all this is the case, the outputs of the ADAS algorithm written to the OutputSimpleAlgorithm object are used to check whether (correctly) no warning is issued if no collision is to be feared or whether a warning is issued if a collision is to be feared. Afterwards a test report is generated. The tests are based on the ADAS Test Feature Set which is used in a C# test script and included in the test unit.
