ADAS Sensor Analysis

 

Note

Click on the blue link to load the sample configuration in CANoe.

ADAS Sensor Analysis- ADASSensorAnalysis.cfg

This sample configuration shows how data of a bus system (CAN) can be written programmatically to objects of the ADAS data model.

Introduction

The ADAS DO API can be used to programmatically transfer sensor data to the ADAS data model. In this sample configuration, a CAN database was used to replay sensor data as messages and signals via a logging file in a replay block and read out as CAN frame events in a C# measurement script.

Desktops

This configuration contains several desktops.

Setup

In the Simulation setup, a CAN network with a CAN database is available, which contains CAN messages for sensors. The recorded data (logging file) for a sensor is replayed via a replay block.

In the Communication Setup, a C# application model (measurement script) has been added, in which event handlers for the sensor CAN messages are implemented. In these handlers the sensor data are read out as signals and written to an ISensor object of the ADAS data model via the ADAS API. In this process, the sensor data for a sensor and its detected objects are aggregated and are then visible as ADAS objects in CANoe in the analysis windows (e.g. Trace Window and Scene Windows).

Measurement

In the Trace Window the sensor data are displayed as CAN messages and as ADAS objects (ISensor, IDetectedMovingObject).

The analysis window Scene Window displays the sensor cone and the objects detected by the sensor.

Detected objects can be selected from the list in the left pane. Their properties are then visible in the lower left area and the respective object is framed in the main area (Scene View).
