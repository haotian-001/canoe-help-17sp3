ADAS System (DYNA4)

 

Note

Click on the blue link to load the sample configuration in CANoe.

ADAS System (DYNA4) — ADASSystem.cfg

Shows the use of all ADAS features by testing an ADAS algorithm. A DYNA4 scenario is used for stimulation.

Preconditions

At measurement start the configuration tries to start the 3D environment DYNAanimation. If you have installed DYNAanimation, DYNAanimation opens and you see the DYNA4 scenario modeled in a 3D environment. If you have not installed DYNAanimation, a message will appear in the trace window. The measurement can still be started and you will only see the objects in the scene window. If you want to install DYNAanimation, you can download the installation package for free on this website.

This sample configuration is intended for demonstration purposes and does not require any further software licenses. However, please note the following license requirements for a productive use with your own models:

  • Executing closed-loop simulations with DYNA4 models in CANoe, like in this sample configuration, requires a DYNA4 Run license, possibly with options depending on the used models.
  • Preparing DYNA4 models and exporting them for the execution in CANoe requires a DYNA4 Studio license, again possibly with options depending on the used models.
  • DYNAanimation is used to visualize simulations - live or in replay - and requires either a DYNA4 Run or DYNA4 Studio license. If no activated license is available, DYNAanimation will show a watermark at all times.

For further information on the possibilities for closed-loop simulation with DYNA4 and its licensing, please contact dyna4@vector.com or visit https://www.vector.com/dyna4.

Introduction

At the start of the measurement, the system switches to the desktop Measurement. A DYNA4 scenario is loaded. This scenario plays an emergency braking of a vehicle in front in a construction site. If DYNAanimation is installed (installation package here ) it will open. If not, you can follow the scenario in the scene window. To play the scenario, click on the Apply and Run button in the DYNA4-Control panel. The scenario is then played several times with different parameters (speed of the ego-vehicle) and stimulates an ADAS algorithm integrated as an application model. A test unit tests the ADAS algorithm and generates a test report.

Desktops

This configuration contains several desktops.

Configuration

The measurement setup as well as the Communication Setup are displayed on this desktop.

A trace window and a scene window are integrated in the measurement setup, as well as an inactive logging block.

The ADASSystem_Setup vCDL is integrated as database in the Communication Setup. In addition to the static objects LRRadar77GHz and Camera defined here, that represent the sensors of the DYNA4 scenario, an OutputSimpleAlgorithm object was defined. This contains the output of the ADAS algorithm to be tested.

In addition, several application models are integrated in the system and communication structure:

  • AlgorithmUnderTest: The ADAS algorithm to be tested as C# code. In the example, a very simplified emergency braking assistant that warns the driver and brakes automatically if the Time To Collision is less than 3s.
  • ClosedLoop: A short C# program reacts to the output of the ADAS algorithm. If the ADAS algorithm writes a warning to the output object, a braking signal is sent to the DYNA4 scenario.
  • DashboardControl: Controls with CAPL code the display of the Dashboard panel.
  • SingleTrack_DriverAssistanceCANoe: Modeling library which contains the compiled DYNA4 scenario.

Measurement

On this desktop the panel Scenario Control DYNA4, the panel Dashboard, a Trace Eindow, the scene window and the test configuration are displayed.

Panel DYNA4Control

After starting the measurement, you have to click on Apply and Run before the scenario is played. Thus, a possibly installed DYNAanimation has time to start and initialize itself. After clicking on Apply and Run the scenario will be played. The DYNAanimation version to be used can also be selected in this panel.

Panel Dashboard

Shows the current speed and rpm of the ego vehicle at any time. If a warning is active from the ADAS algorithm for the emergency brake assistant, a warning is displayed centrally in the Information screen.

Trace Window

Displays the data received from the DYNA4 scenario. The objects LRRadar77GHz.sensor_info and Camera.sensor_info contains information such as the sensor view (sensor ID, sensor mounting position) or the number of currently detected objects. The currently detected objects in turn are displayed as independent objects in the trace window (e.g. Detected_id_xxx_by_LRRadar77GHz). These contain all known information about the objects such as dimension, position, speed or classification.

Additionally the object OutputSimpleAlgorithm can be seen. In this object the output of the AlgorithmUnderTest application model is stored.

Scene Window

Visualizes the data received from the DYNA4 scenario. Based on the information in the detected objects, the position and dimension of the objects is displayed.

Test Configuration

An included test unit checks the ADAS algorithm that is included in the Application Model AlgorithmUnderTest. Various tests check, for example, whether the OutputSimpleAlgorithm reacts correctly if the Time To Collision is less than 2s. For this purpose, the outputs of the ADAS algorithm, which are written to the OutputSimpleAlgorithm object, are checked. In addition, the distance and the Time To Collision are checked. The tests are based on the ADAS Test Feature Set and the ADAS Test Service Library Checks, which are used in a CAPL test script and included in the test unit.

The parameters of the test scenarios can be configured in the Test Unit. For this purpose, a C# script modifies an OpenSCENARIO file in which the parameters are defined. By default, 6 scenarios are run with different combinations of speed of the ego vehicle and speed of the braking vehicle. These parameters are defined in the Roadworks_ParameterSet.vparam file. A test report is generated afterwards.
