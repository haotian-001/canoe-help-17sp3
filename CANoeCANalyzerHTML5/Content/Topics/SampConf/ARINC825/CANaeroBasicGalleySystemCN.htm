CANaero Basic Galley System

 

Note

Click on the blue link to load the sample configuration in CANoe.

CANaero Basic Galley System – CANaeroBasicGalleySystem.cfg

Simulation of an airplane galley.

This more complex configuration consists of several desktops with a whole series of panels. They show the communication within an airplane galley (galley insert; ARINC-812-1; 2006-12-29). The various electrical consumers each request electrical energy, and power is distributed to them according to a certain power budget.

Desktops

Configuration

The Simulation Setup contains several CAPL nodes:

  • GalleyMaster: This node reacts to a Power Budget Request (PBR) with a suitable response. This node does not operate panels.
  • DisplayPanel: This node controls the Display Panel on the Diagnostics desktop.
  • CoffeeMaker: This node simulates a coffee-maker, and it controls the associated Coffee Maker panel on the Measurement and Heating Cycles desktops.
  • SteamOven: This node simulates an oven.
  • TrashCompactor: This node simulates a trash compactor.

The configuration also includes several databases.

Measurement

Several panels can be seen here. The buttons on the panels can each be used to request electrical power. The current value of the power budget is shown in the Graphics Window.

Diagnostics

The current communication states of the connected electrical consumers are shown on the Display Panel. When you shut off a consumer on the relevant panel, its state changes to Offline on the Display Panel.

A test environment and related execution dialog are also shown on this desktop. The test itself is described in an XML file, and it checks the cycle times of messages, which are sent out by the simulated devices.

Heating Cycles

This desktop shows the mutual interaction in distributing the power budget between the Coffee Maker and Steam Oven devices. Energy can be requested via the panels, and in the Graphics Windows you see the temperature response or requested power budget.
