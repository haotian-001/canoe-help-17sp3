Diagnostic Tester (LIN)

 

Click on the blue link to load the sample configuration in CANoe.

  • Diagnostic Tester (LIN) – LINDiagnosticsTester.cfg
    Demonstrates the usage of the diagnostic consoles with LIN.

Introduction

In this sample configuration, you use the Diagnostic Window with Diagnostic Console and ECU Control, the Fault Memory Window as well as a CAPL test module for work with diagnostic services, Diagnostic Trouble Codes (DTC) and functional group requests.

Desktops

This configuration contains the following desktops:

LIN Diagnostic Console

ECU Panel

Automated Test

Functional addressing

Setup

 

LIN Diagnostic Console [▲ back]

Diagnostic Window 1 / Diagnostic Console

In the Diagnostic Console , all diagnostic services that are defined in the diagnostic description file (CDD) with the ECU qualifier Door are displayed. With the help of the Diagnostic Console, you can select, parameterize and send services. The response from the ECU is displayed in the Diagnostic Trace of the same Diagnostic Window.

Door - Fault Memory

With the help of the Fault Memory Window, you can access the fault memory of the Door ECU, i.e., read out or delete DTCs.

ECU Panel [▲ back]

Located on this desktop is an ECU simulation, the status of which is displayed in the ECU Panel:

ECU Status

  • Active session
    The vehicle icon indicates which session the ECU is currently in:
  • Closed engine hood: Default session
  • Open engine hood: Extended session (possible to unlock the ECU)
  • Computer icon: Programming session (unlocked)

Please note that the ECU returns to the locked default session if no requests are received within time P3. If the sending of Tester Present was manually deactivated, this must be reactivated in order to keep the ECU in a non-default session.

  • ECU unlocked
    The ECU can be set to the safe mode so that certain services cannot be performed and the ECU denies the security access. The LED illuminates if the ECU is unlocked. In this state, the programming session can be started.
  • Suppressed Response
    LED illuminates for a short time if a request is received for which the positive response is suppressed (UDS feature).
  • Negative Response
    LED illuminates if the ECU sends a negative response.

Values for Querying in the Diagnostic Console

You can change and query the values from the Serial Number, Device Control - Door Contact and Variant Coding Data areas with the help of the Diagnostic Console (LIN Diagnostic Console desktop).

Fault Memory

The ECU maintains a memory for Diagnostic Trouble Codes (DTC) which can be read out via the Diagnostic Console or the Fault Memory Window.

You can also add other fault memory entires by selecting the desired DTC with the help of the slider and then clicking on the Add Snapshot Record button. This is only possible if at least one of the check boxes in the Fault Memory areas is activated.

Expected Security Key

So that you can manually unlock the ECU via the Diagnostic Console in this sample configuration, the required key is displayed in this area.

A simple and flexible possibility for generating a diagnostic security access key is made available with SeednKey.dll. In this example, the SecuritySeed is sent by the simulated ECU to SeednKey.dll, which returns the calculated SecuritySeed. SeednKey.dll is located in the Exec32 directory of the CANoe installation.

AutomatedTest [▲ back]

Displayed here is the test unit DiagTest, which performs a number of diagnostic tests. The serial number of the ECU is queried and a check is performed to determine whether the delivered value is valid. If the serial number was changed manually, the test fails.

The second test case can be activated by the corresponding check box and checks whether the fault memory is empty. This test case will fail as long as the fault memory has not been cleared after measurement start e.g. by using the Fault Memory Window.

Functional Addressing [▲ back]

Located on this desktop are tools for sending functional group requests.

In Diagnostic Window 2 / Diagnostic Console, all diagnostic services that are defined in the diagnostic description file (CDD) with the ECU qualifier Functional are displayed. You can select, parameterize and send services, such as deleting the fault memory, with the help of the Diagnostic Console.

In the FunctionalTesterPresent panel, you can use the Tester Present button to send the functional Tester Present command. The Send function is implemented in the CAPL program of the LIN Master.

Setup [▲ back]

Located on this desktop are the Measurement Setup, the Simulation Setup, the Test Setup for test modules and the Write Window.
