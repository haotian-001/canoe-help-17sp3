Transport Protocol (LIN)

 

Click on the blue link to load the sample configuration in CANoe.

  • Transport Protocol (LIN) – LINBasicTP.cfg
    Demonstrates how a diagnostics request from a CAN backbone network is routed to a LIN network.

Introduction

This sample configuration shows the routing for the following cases:

  • Diagnostic request from one CAN network to a LIN network
  • Diagnostic request from the LIN network to a CAN network

The LIN Master node functions here as a transport protocol gateway.

The CAN transport protocol is implemented with osek_tp.dll.

The LIN transport protocol is implemented with LINtp.dll.

Both DLL files are assigned to the LINMaster node in the Simulation Setup. (Right click on LINMaster|Configuration|Components).

Desktops

This configuration contains the following desktops:

TP Gateway

Trace

Basic Diagnostics

Setup

 

TP Gateway [▲ back]

Located on this desktop is the CANBackbone - Diagnostic Console, via that you can send diagnostic requests on the CAN backbone. The request is routed from the gateway (LIN Master) to the LIN bus. The diagnostic response on the LIN network is also routed from the gateway to the CAN bus.

In the NAD panel, you can set the node address that is used by the Diagnostic Console.

You can view all message traffic of the CAN LIN network in the Trace All Trace Window.

Trace [▲ back]

There are three Trace Windows on this desktop.

  • Trace All
    You can observe all traffic of the CAN LIN network here.
  • CAN Backbone
    Here, you only see the frames of the CAN network.
  • LINDoor
    Here, you only see the frames of the LIN network.

The frames are selected in the Measurement Setup by means of appropriate Channel Filters before the Trace Windows.

Basic Diagnostics [▲ back]

Located on this desktop is the Basic Diagnostic Editor, which you can use to describe simple diagnostic services for an ECU. The diagnostic services defined here can be performed with the help of the Diagnostic Console.

Setup [▲ back]

Located on this desktop are the Measurement Setup, the Simulation Setup and the Write Window.
