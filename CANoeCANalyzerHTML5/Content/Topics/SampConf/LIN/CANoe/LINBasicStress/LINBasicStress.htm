Stress (LIN)

 

Click on the blue link to load the sample configuration in CANoe.

  • Stress (LIN) – LINBasicStress.cfg
    Demonstrates the usage of the LIN Stress Feature Set functions.

Introduction

Note

This sample configuration cannot be operated in the Simulated Bus mode of CANoe. The default setting is therefore Real Bus mode. Furthermore, a network interface that supports LIN must be connected.

This sample configuration shows how you can simulate various protocol errors with the help of the Stress Feature Set for LIN.

Desktops

This configuration contains the following desktops:

Stress via LIN Stress IG

Stress via LIN Disturbance Block

Stress via CAPL

Analysis

Setup

 

Stress via LIN Stress IG [▲ back]

You can trigger the following stress functions with the help of the LIN Stress IG and the buttons in the three lines:

  • Send Header Error
    Send header without the dominant part in the Break Field (Header Error).
  • Send Frame Error
    Send frame with invalid checksum (Frame Error).
  • Send Frame
    Send valid frame that is not defined in the LDF (frame with ID=2).

You can observe the result directly in the Trace Window.

The LIN Stress IG and the LIN Disturbance Block are configured in the Test Setup (Setup desktop).

Stress via LIN Disturbance Block [▲ back]

With the help of the LIN Disturbance Block, you can trigger the following stress functions via the Activate button in the respective line:

  • Line 0
    Invert the first data bits in the Sync Byte of the LIN Header (header disturbance).
  • Line 1
    Invert the stop bit in the first data byte of the frame with ID 0x33 (response disturbance).

You can observe the result directly in the Trace Window.

The LIN Disturbance Block is configured in the Test Setup (Setup desktop).

Stress via CAPL [▲ back]

Located on this desktop are the Control Panel and Invert Header Bits panels, the Write Window as well as the Trace Window.

You can trigger the stress functions with the help of the panels. The actual stress function is implemented in a CAPL node, which is configured in the Test Setup (see Setup desktop). If a stress function is triggered via the panel, the name of the CAPL function and the used parameters are displayed in the Write Window.

Each of the following buttons in the Control Panel open their own panels, via which you can trigger the stress function:

  • Invert Header Bits
    Invert individual bits in the LIN Header.
  • Invert Response Bits
    Invert individual bits in the LIN response.
  • Set Header Error
    Transmission of LIN Headers with invalid parameters.
  • Set Response Length/Count
    Transmission of a LIN response with an invalid number of bytes.
  • Set Response Length/Count
    Transmission of a LIN response with a limited number of transmissions.
  • Set Checksum Error
    Transmission of a LIN response with an invalid checksum.
  • Set Bus Disturbance
    Dominant or recessive fault on the LIN bus.
  • Send Frame as Bitstream
    Manual transmission of an assembled LIN Frame as bitstream.
  • Send Bytes as Bitstream
    Transmission of an arbitrary byte value.

All named panels can also be opened via the Panels button ( Home Ribbon Tab).

Analysis [▲ back]

Located on this desktop are the LIN Statistics Window, the LIN Network Management Window, the Trace Window and the Write Window. During the stress situations, you can analyze the LIN bus here.

Setup [▲ back]

Located on this desktop are the Measurement Setup, the Simulation Setup and the Write Window. Three simulated LIN nodes are configured on LIN channel 1 in the Simulation Setup.
