Basic Configuration AUTOSAR (LIN - ISO17987)

 

Click on the blue link to load the sample configuration in CANoe.

  • Basic Configuration AUTOSAR (LIN - ISO17987) – LINBasicAUTOSAR.cfg
    Demonstrates all essential LIN ISO17987 features based on an AUTOSAR 4 database.

Introduction

This sample configuration shows the LIN-ISO17987 functions ETF, Reconfiguration (LIN ISC), Sporadic Frame as well as the LIN diagnostic functions.

An AUTOSAR 4 database is used in this sample configuration to describe the LIN ISO17987 network.

Desktops

This configuration contains the following desktops:

ETF

Reconfiguration

Sporadic Frame

ReadByIdentifier

Setup

 

ETF [▲ back]

Displayed on this desktop is the support of Event Triggered Frames (EFT). The simulation of an individual response occurs automatically if one of the assigned signals is updated via the Event Triggered Frame Panel.

Use the Make Collision button to simulate a collision of ETF responses. This function is implemented with the help of system variables and CAPL (see the CAPL program of the master node). The collision is resolved automatically by changing the schedule table of the master.

Reconfiguration [▲ back]

The commands for reconfiguration are demonstrated on this desktop. With the CAPL program of the master node, the program switches to the InitTable on every measurement start and after every wake-up event. This schedule table contains a number of commands for reconfiguring the slave node.

Sporadic Frame [▲ back]

This desktop illustrates the use of sporadic frames. With the Sporadic Frame Panel, you can update the values of two system variables. By updating, the sending of a sporadic frame for the corresponding signal is triggered in the CAPL program of the master node.

ReadByIdentifier [▲ back]

The LIN diagnostic functions are demonstrated on this desktop. In the Basic Diagnostic Editor, two LIN diagnostic services of type ReadDataByIdentifierLIN are defined for ECU Motor1. These can be sent to the simulated ECU during the measurement via the Diagnostic Console.

Note

The service for reading out the LDF revision is undefined in the case of an AUTOSAR database. Therefore, corresponding diagnostic requests are answered with a negative response by the simulated ECU.

Setup [▲ back]

Located on this desktop are the Measurement Setup, the Simulation Setup and the Write Window.
