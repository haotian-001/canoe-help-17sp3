Gateway (LIN)

 

Click on the blue link to load the sample configuration in CANoe.

  • Gateway (LIN) – LINGateway.cfg
    Demonstrates a CAN-LIN-Gateway.

The simulated CAN LIN Gateway transmits the frames of the LockingSystem node (CAN) to the LIN network. The seat settings are controlled there. You can perform function tests with the help of the CAN\LIN Gateway Tester. The gateway is stimulated by certain bus activities. Afterwards, the result is checked. Logging is started automatically with the start of the test and stopped as soon as all tests are ended.

Introduction

  • Start the measurement.
  • Click on the Open Car button on the key to start the initial communication.
  • Click on Power to start the ignition.
  • Click on button 1 or 2 on the key to activate a previously stored seat position.
  • You control the individual seat functions via the LIN Information panel.
  • The Seat Display panel displays the current seat positions.

LIN/CAN [▲ back]

This desktop contains various panels, which you can use to control and observe the functions of the gateway.

Using the CAN Information panel, you can unlock the vehicle and start the ignition. Furthermore, you can select two previously stored seat positions.

With the LIN Information panel, you can control the individual functions of the seat by clicking on the gray controls.

The Seat Display panel is used to display the current seat position and the active seat functions.

The two Trace Windows, Trace LIN and Trace CAN, show the current bus traffic on the respective networks.

Analysis [▲ back]

Located on this desktop are various analysis windows for displaying the current bus traffic: Two Statistics Windows for CAN and LIN, the Data Window as well as two Trace Windows for CAN and LIN.

Test [▲ back]

This desktop offers tools for checking the gateway functions. You can start the CAN\LIN Gateway Tester during the running measurement via . This causes the various function tests of the CAN LIN gateway to be performed.

You can simulate various error scenarios via the Error Simulation panel, which can trigger the failure of one or more test cases.

The Trace All Trace Window logs all bus traffic and displays CAN Frames as well as LIN Frames.

Setup [▲ back]

Located on this desktop are the Measurement Setup, the Simulation Setup and the Write Window.
