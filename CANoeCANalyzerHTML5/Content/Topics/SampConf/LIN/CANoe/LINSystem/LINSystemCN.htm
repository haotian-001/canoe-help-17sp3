System Configuration (LIN)

 

Click on the blue link to load the sample configuration in CANoe.

  • System Configuration (LIN) – LINSystem.cfg
    Demonstrates CANoe LIN features.

The sample configurations illustrates the simulation of several LIN buses in a vehicle. Furthermore, they introduce the basic and current CANoe.LIN/DENoe.LIN functions.

Introduction

After the measurement starts, the RemoteKey macro starts the simulation automatically.

If the macro is deactivated, the vehicle must be opened via the Unlock button on the Remote Key panel to commission the LIN buses. All simulated functions on the panels can then be executed.

Additional macros are available on the Simulation ribbon tab|Automation and via the Automation Sequences toolbar.

The RemoteKey macro simulates opening of the vehicle.

Note

The baudrate in the sample configuration is set at 19200 kbit/s. You can change the baudrate for each channel via the Hardware ribbon tab|Network Hardware under Setup.

An overview and detailed description of the functions of this sample configuration can be found on the functional overview page.

Remote Control [▲ back]

The Remote Control desktop contains the remote control for centralized locking of the vehicle. This remote control is controlled via the RemoteKey macro.

If you would like to operate the sample manually, deactivate the Start replay on measurement start option in the macro configuration dialog.

You can execute the following actions on the Remote Control panel:

  • Open/close vehicle
  • Seat position memory 1 or 2

After the vehicle is opened and closed, the LIN buses are woken up or set to sleep mode. Opening and closing are signaled by simultaneous brief activation of the hazard lights.

The panel elements are mapped to system variables and signals. If you make changes to the panels, this information must be transferred. The modified variable value is transferred to the CAPL program. Modified signal values are passed on to the CANoe Interaction Layer, which is responsible for transfer.

Exterior View [▲ back]

You can see the Exterior Console operating panel on the Exterior View desktop.

You can execute the following actions on the panel:

  • Windshield wiper on/off
  • Low-beam light on/off
  • Hazard light on/off
  • Left or right blinker on/off

Operate the windshield wiper, the low-beam light, the hazard light and the blinker. The Exterior View panel shows the current status.

The panel elements are mapped to the system variables and signals. If you make changes to the panels, this information must be transferred. The modified variable value is transferred to the CAPL program. Modified signal values are passed on to the CANoe Interaction Layer, which is responsible for transfer.

Door [▲ back]

The bus traffic is illustrated on the Door desktop using the Trace Window and with panels.

Manipulating the simulation model

You can manipulate the simulation using the Door Opener panel. You can open and close the vehicle door using the Door Opener switch. The Exterior View panel is used for display.

Electric Window [▲ back]

You can operate the electric window controls from the Interior Console operating panel.
The windows of the driver- and passenger-side doors can be moved up and down by pressing the left and right mouse buttons, respectively.
The current window positions of the four windows are shown in the display.

Setup [▲ back]

The Simulation Setup and the Measurement Setup of CANoe are presented on the Setup desktop.

The complete system including LIN buses and all network nodes are illustrated graphically in the Simulation Setup.

It must be defined whether the network nodes are real or simulated in the Simulation Setup. The behavior of the simulated network nodes can be reproduced using CAPL programs, which are stored in the network nodes.

The Door, RoofUnit and RCInterface buses are linked together via gateway nodes for information exchange.

In the Measurement Setup, you can see all function blocks used. Additional function blocks can be added.

Statistics [▲ back]

The bus statistics are displayed in the Bus Statistics, Graphic and Data Windows on the Statistics desktop.

The Bus Statistics Window displays bus characteristics such as peak load, busload, sent messages, sent Error Frames etc.

Bus statistics can also be displayed in the Graphics Window. It shows the percent load of all LIN buses.

Statistical analyses of messages/second are displayed as bar graphs in the Data Window.

NM [▲ back]

In the sample configuration, the status of the LIN buses is displayed via the Network Management Window, the Trace Window and the Bus Statistics Window.

Individual LIN buses can be switched to the asleep or the awake state using the buttons in the Network Management Window.

Visual Sequencer [▲ back]

Contains two Visual Sequencer Windows and two Trace Windows.
