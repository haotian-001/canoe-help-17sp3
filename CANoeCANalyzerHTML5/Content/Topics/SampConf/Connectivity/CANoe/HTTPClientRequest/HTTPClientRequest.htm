HTTP Client Requests

 

Click on the blue link to load the sample configuration in CANoe.

  • HTTP Client Requests – HTTPClientRequests.cfg
    Shows, how CANoe can be used to send client requests using the HTTP protocol.

Introduction

When the configuration is loaded, you can send HTTP requests to an API during the measurement to get the corresponding sunrise and sunset times for latitude and longitude coordinates as response.

Note

In this sample configuration you are using the Sunset and sunrise times API. Its instructions for use must be observed. The functions described here are depending on the functioning of the API.

Desktops

This configuration contains the following desktops:

HTTP Requests

On this desktop you will find a panel to control the HTTP requests to be sent, two Trace Windows and the Write Window.

The Sunrise API Control panel can be used to send HTTP requests to the Sunset and sunrise times API. In the panel, enter the longitude and latitude coordinates for which you want to get sunrise and sunset times. Select whether the HTTP request should be sent in High Level or Low Level mode. To start, click the Execute Request button. The Trace Window Trace High Level or the Trace Window Trace Low Level will then display the corresponding method call and the return triggered by it.

In High Level mode, the client Application::HighLevel::SunriseApiClient modeled in vCDL is used.

In Low Level mode the provided client _HTTP::Client is used.

Modeling

On this desktop you will find the Measurement Setup, the System- and Communication Setup and a Write Window. Here you can see the communication setup. For the modeling the language vCDL was used as data source. To control the simulation, scripts in the C# language are used.
