Basic MQTT Connectivity

 

Click on the blue link to load the sample configuration in CANoe.

  • Basic MQTT Connectivity – MQTTBasicConnectivity.cfg
    Shows how CANoe can be used in connection with the IoT protocol MQTT.

The sample configuration reuses the example from Help and expands it by a climate control unit and three additional temperature sensors. A speed for the ventilator is determined and set with this.

Introduction

When the configuration is loaded, you will see a schematically represented living room with four temperature sensors, a humidity sensor and a fan. All of this together represents a control circuit which is to keep the temperature at a constant 20 °C.

You can activate or deactivate Automatic mode for the fan. To simulate a rise in temperature manually, you can use the controller for the individual temperature sensors accordingly. Ventilator operation is not initiated here – only the speed is increased.

Note

To use the configuration a MQTT broker is needed. The internal broker CANoe Hosted is used, but if desired a local MQTT broker can also be used. A download is e.g. available under the following address: https://mosquitto.org/download/

Under Options|Extensions|Connectivity you can adjust the TCP port of the internal MQTT broker in the CANoe Hosted area. The usual TCP port for MQTT is 1883.

MQTT Living Room [▲ back]

Three Trace Windows and the Write Window can also be seen on this desktop. The living room with its four temperature sensors, ventilator and humidity sensor are shown on the right-hand side.

The ClimateControl panel can be used to change the temperature in the room for each individual sensor. This change results in the ventilator beginning to rotate. If the temperature continues rising, this causes the ventilator to rotate faster.

If Automatic mode is deactivated, none of the values are being controlled.

If Automatic mode is activated, the temperature for the sensors is randomly adjusted in a range of 0 °C to 10 °C. If a temperature is over 20 °C, the ventilator begins cooling. Attempts are repeatedly made to control the temperature down to 20 °C for each temperature sensor here.

The sensor which is sending its values can be seen in the MQTTLivingRoom panel. The connecting line is colored red here if the sensor is sending a value to the ventilator. The illustration shown is simplified, as the value is not transferred directly to the ventilator, but first to the MQTT broker and from there to the ventilator.

Modeling [▲ back]

The Communication Setup, Measurement Setup and a Write Window are found on this desktop. The Communication Setup can be seen here. The vCDL language is used as the data source for modeling. A script in the C# language serves to operate the simulation.

FanTest [▲ back]

This desktop shows the use of tests in combination with vTESTstudio and MQTT.

Using the ThresholdSetting panel, you can enter the average temperature at which the fan starts to move. The test is executed at the beginning of the measurement. You can restart the test Fan Test by clicking on .

The test checks whether:

  • at an average temperature > 30 °C the fan rotates

  • at an average temperature < 30 °C the fan speed is 0

The Trace Window Fan Test Trace shows the individual steps of the test execution.
