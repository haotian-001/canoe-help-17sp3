Introduction

In the Room panel you can see a room whose lighting and shades are controlled in this sample configuration. After the start of the measurement, you will see the switching on and off of the lamp and the closing and opening of the blinds as part of the execution of the test in the Test desktop.

Two shades (east and west) can be closed and a lamp can be switched on. By clicking on the Start / Stop Day Simulation button in the Time panel, you can start a simulated day sequence so that shades and lamp are controlled depending on the position of the sun. Additionally, you can control them by clicking on the buttons in the Manual Control panel. In the figure in the Flowchart panel you can see the different communication participants and the protocols that are used between them. For more information on the protocols used and how they are integrated in CANoe, please refer to the Connectivity Feature Set overview page.

Light Simulation [▲ back]

On this desktop the room is simulated. In the Time panel, the simulated time is displayed, which is used to use a faster daily schedule. At the start of measurement the current time is used. Click on the Start / Stop Day Simulation button to let the time run on. 100 ms in reality corresponds to 10 minutes of the day simulation. Click the button again to stop the time again. As long as the time is running, the room is controlled automatically. However, you can close and open the shades and turn the lamp on and off at any time in the Manual Control panel by pressing a button. Then click the AUTO button to have the respective shade or lamp controlled automatically again.

Control Depending on Sun Position

At the start of measurement and at each new day (shortly after midnight) in the simulated time, the data for sunrise, sunset and sun noon are requested via HTTP from https://sunrise-sunset.org/api and displayed in the Sunset Sunrise panel. Depending on this data, the room is controlled to have - in a very simplified way - good lighting conditions in the room within certain times, i.e. to always see something, but to prevent glare from the sun.

The lamp is on before sunrise and after sunset within the period from 6 am to 10 pm. The shade in the east is closed from sunrise to sun noon, and from sun noon to sunset the shade in the west is closed.

Modeling [▲ back]

In the Communication Setup you will find five application models:

  • In DaySimulation.cs the daily schedule is simulated. The time simulated here is available to each communication participant.

  • In HTTPSunriseData.cs the sunrise data is requested via HTTP and then passed to a MQTT client. Together with the objects sunriseSunsetobtainer and httptomqtt (see DataModel.vcdl) this script forms the translator between HTTP and MQTT. Each distributed object (DO) can only belong to one binding. Therefore, to simulate a communication participant using multiple protocols, it is necessary to use a single DO per protocol used (HTTP, MQTT, DDS) and connect them via an application model.

  • In MainControl.cs, the sun position data received via HTTP is used to determine the room control data from it and send it via MQTT. Manual inputs from the Manual Control panel are also processed here. The manual input always has priority. The automated control of the shades and the lamp is only used if the respective AUTO button has been clicked.

  • The script MQTTtoDDS.cs is used exclusively for passing on the room control data received via MQTT to a DDS client. Together with the two objects mqtttodds_mqtt and mqtttodds_dds, this script forms the translator between MQTT and DDS.

  • For the animation of the shades in the Room panel there is also the script ShadesSimulation.cs, which triggers the nine shades positions one after the other at intervals of 50 ms each when the shades are opened or closed. The representation of the shades are switches (Switch/Indicator) with 9 different states, each of which is assigned an individual graphic.

Test [▲ back]

On this desktop you can see the Test Configuration Test Light and Shades. (The test configuration was created using the Test Ribbon Tab, the Test Configuration button, and the Add Test Configuration command.) Test Light and Shades contains a test unit that was built using vTESTstudio. The six test cases test the switching on and off of the lamp and the closing and opening of the shades. Here the functionality of the panel Manual Control is used, the object manualControl is assigned values for the states of the buttons. The test starts at the start of measurement and can also be started manually. If you make changes during the test using the Manual Control panel, the test may fail even though the shades and lamp are working. In this case, simply restart the test.
