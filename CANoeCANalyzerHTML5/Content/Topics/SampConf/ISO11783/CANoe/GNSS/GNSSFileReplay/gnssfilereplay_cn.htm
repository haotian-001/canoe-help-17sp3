GNSS File Replay

 

Click on the blue link to load the sample configuration in CANoe.

  • GNSS File Replay – GNSSFileReplay.cfg
    Replay of store position data.

This sample configuration demonstrates the usage of the GNSS node layer and the GNSS Monitor by using a GNSS logging file.

Introduction

After measurement start the desktop Simulation is activated. Then start the simulation with [Start] in the panel GNSS Receiver. The node loads the coordinates from the logging file and transmits them on the CAN bus. You can see the speed and the current position data in the Data Window and the GNSS Monitor.

With the slider you can control the speed. If you don't want to use the speed set in the panel but the speed stored in the logging file, then active the Use time information check box before starting the simulation by pressing [Start].

If you want to use a different logging file, the stop the simulation with [Stop] and open the dialog to select the file with the Folder button. Select the desired logging file, close the dialog with [Open] and start the simulation with [Start] again.

Note

You can create GNSS logging files with the GNSS Monitor. To do this enter the filename of the logging file in the options dialog of the GNSS Monitor and then activate the logging.

Desktops

This configuration contains the following desktops:

Setup

Desktop Setup contains the Simulation Setup with a node simulating the GNSS receiver, and the Measurement Setup with the GNSS Monitor recording the sent position data, a Trace Window and a Data Window. Furthermore a Write Window is contained.

Simulation

Desktop Simulation contains a panel for the GNSS receiver, the GNSS Monitor, a Trace Window, and a Data Window.
