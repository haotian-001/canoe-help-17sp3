System Configuration (ISO11783)

 

Click on the blue link to load the sample configuration in CANoe.

  • System Configuration (ISO11783) – ISO11783SystemDemo.cfg
    Simulation of a ISO11783 total system.

The sample configuration demonstrates the simulation of a ISO11783 network. You can see a tractor with several components as well as an implement.

Introduction

On measurement start the Simulation desktop is activated. The tractor is started. The object pool of the sprayer is sent to the Virtual Terminal. The device description is sent to the Task Controller.

The application rate (in g / m²), you can increase or decrease as follows.:

  • Virtual Terminal: VT window, via the input box in the middle.
  • Joystick buttons: Aux Input 1 panel , button A or B.
  • Task Controller: ITC window, Devices|Sprayer|Section X|Setpoint App Rate, enter value in Value column and click the Set button.

The current application rate (in l/min) is computed dependent of the tractor speed.

Auto Play

After measurement start the Auto Play mode is activated automatically. In this mode all user inputs are generated automatically by the simulation:

  • put in gear
  • control speed via the accelerator pedal
  • turn rear PTO
  • start sprayer
  • At the end of each straight section the tractor is braked, the right or the left turn signal turned on and the sprayer is switched off.
  • At the beginning of each straight section the signal is switched off, the tractor is accelerated and the sprayer is turned on again.

Manual Control

You can turn off the Auto Play mode by clicking the toggle switch in the panel. This switch to manual control. In manual mode you have to do the following steps to run the simulation:

Put the gear to drive the tractor:
Tractor Control panel , under Gear click the D button.

To activate the sprayer, turn the rear PTO:
Tractor Control panel, under Rear click the bottom button for activation.

Then start the sprayer via the Virtual Terminal:
VT panel, Data Mask page, click top right the on button.

Setup [▲ back]

This desktop shows the Simulation Setup and the Measurement Setup. The Measurement Setup contains blocks to analyze data traffic on the CAN bus. The Simulation Setup contains the simulated devices.

Simulation [▲ back]

This desktop displays the following panels:

Panel

Description

Sprayer

The sprayer is coupled to the tractor. In the panel the status of the track width and the application rate is shown. For more information see model Sprayer.

Virtual Terminal

The Virtual Terminal displays the Object Pool of the sprayer. So you can control the sprayer.

Tractor Control

With this panel you can control the tractor. For more information see model Tractor ECU.

Aux Control

In this panel, a joystick is displayed, which allows you to control the sprayer: change track width and application rate.

Control

In this panel you can switch between the Auto Play mode and manual control.

Map

In the Map Window the position data of the tractor are displayed.

Task Controller [▲ back]

This desktop contains the panels of the Interactive Task Controller and the Sprayer, as well as a Trace Window that displays the relevant process data messages.

After measurement start the sprayer implement transfers its device description to the Interactive Task Controller. You can edit the values with the key <F2> and send the new value with the [Set] button to the sprayer.

displays important simulation values

Diagnostics [▲ back]

On this desktop the diagnostic messages are evaluated. The parameter groups and DTC of diagnostic messages are defined in the ISO11783-12 specification.

With the panel Tractor Diagnostics you can simulate error states of the Tractor Engine Speed by using the slider. In neutral position no error state is reported. If you move the slider to the left, the value will be too small. If you move it to the right, the value will be too big. Both states signals an error state.

The DTC monitor evaluates the diagnostic messages and provides a clear overview for each ECU. Due the DTC monitor is used for J1939 networks too, some functions (e.g. deletion of active DTC, the page Freeze Frame) are not working for ISO11783.

The Trace Window protocols the diagnostic parameter groups.

Note

The identification messages specified in ISO11783-12 can be requested with Request ECU information in the J1939 scanner on desktop Analyze.
