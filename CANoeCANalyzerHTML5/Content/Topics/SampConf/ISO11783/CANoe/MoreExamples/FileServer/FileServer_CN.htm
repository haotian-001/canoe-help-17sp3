File Server Client and Browser

 

Click on the blue link to load the sample configuration in CANoe.

  • File Server Client and Browser – ISO11783FileServer.cfg
    Access to a File Server by a client and a browser.

Introduction

This sample configuration demonstrates the use of an ISO11783 file server, which allows other network nodes to access directories and files. In this example, there are two clients and one file server. The clients use the ISO11783 Interaction Layer, which handles communication according to ISO11783. The File Server is implemented with the File Server Window.

Desktops

This configuration contains the following desktops:

Setup

This desktop contains the Simulation Setup with the network nodes File Server, FileEditor and Browser. The File Server network node uses the File Server Window which is directly linked to the Interaction Layer.

The Measurement Setup is located on the right side. To the left of one of the two Trace Windows the CAPL program FileServerFilter is inserted, which filters the maintanance messages and status messages of the File Server.

Measurement

This desktop contains the Browser and File Editor panels of the two clients, a File Server Window, and the Trace Window with the filter.

File Editor

With this panel files can be read and written.

In the File input box you can specify the filename. The path can be absolute or relative to the default directory.

With the button [Read] you read the file, which is then displayed in the Hex&Text editor.

The input boxes Offset and Length define the position and length of the data to be read within the file. In the Hex&Text editor you can also change the file.

The [Write] button writes the data to the file. If the file does not exist yet, a new file is created.

The [Get Attributes] button requests the file attributes, which are then displayed with check boxes. To set the attributes, activate the check boxes and click the [Set Attributes] button.

Browser

This panel allows you to view the contents of the folders on the file server.

In the Directory input box, specify the folder.

With [Get Directory] the content of the folder is read out and displayed.

With [..] you change to the parent folder.

The [New Directory] button creates a new folder with the name specified in the Directory field.

The [Open] button next to the filename allows you to switch to a folder.

[Delete] deletes the specified file.

With [Rename] you rename a file. Enter the new name in the Filename column.

File Server Window

The File Server Window sets the configuration of the File Server and displays the states of the clients as well as opened and read files.

Trace Window

The Trace Window Trace provides a compact representation of bus traffic by filtering out TP.CM, TP.DT, FS-Maintenance and FS-Status messages.

Trace All

The desktop Trace All contains a Trace Window that shows all parameter groups.
