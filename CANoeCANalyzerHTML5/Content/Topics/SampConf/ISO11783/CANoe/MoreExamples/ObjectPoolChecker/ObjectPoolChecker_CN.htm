Object Pool Checker

 

Click on the blue link to load the sample configuration in CANoe.

  • Object Pool Checker – ISO11783ObjectPoolChecker.cfg
    Configuration to test object pools.

With this configuration an object pool - available as binary file (*.iop) or as XML file (*.iopx) - can be tested with the Virtual Terminal. The IOP file must conform to the ISO11783 Part 6 format.

Introduction

On measurement start the desktop Panel and ECU page of the Object Pool Checker panel is activated automatically.

To transfer an object pool to the Virtual Terminal execute the following steps:

  1. Set the switch to Online.
  2. Select an object pool file (*.iop or *.iopx).
  3. Click the Load and Transmit button.

To modify the object pool you can use commands on the tabs VT Commands 1, VT Commands 2, VT Graphic Context Commands and Version 5 of the Object Pool Checker panel.

If the Transmit End Of Object Pool message check box is activated, the End Of Object Pool message is sent to the Virtual Terminal after the object pool is transmitted to complete the transmission. To transmit the object pool in several parts, you can deactivate the check box, send several files to the Virtual Terminal using the Load and Transmit button, and only reactivate the check box when the last file is transmitted.

With the activation of the Suppress parsing of Object Pool check box you can transmit faulty files to the Virtual Terminal. In this case, however, not all commands of the panel pages can then be used, because without the parsing of the object pool many CAPL functions of the ISO11783 interaction layer no longer work.

Use the Export button to export a loaded object pool to a previously selected file (*.iop or *.iopx).

Clicking Delete Object Pool and Deactivate Working Set sends a command to delete the object pool to the Virtual Terminal and the Object Pool Checker stops sending the Working Set Maintenance message. Clicking the Activate Working Set and Transmit Object Pool button sends the Working Set Maintenance message again and the object pool is sent to the Virtual Terminal.

The [Store] button stores the object pool in a file by using a version label (the file is located in the folder VTData). You can load it again into the Virtual Terminal with the Load button. To request the version label you can use the Get or Get Extended button. The available versions can be read in the Trace Window. With the Delete button you can delete a stored object pool. If you enter a version label which has more than 7 characters then Extended Store Version, Extended Load Version or the Extended Delete Version commands are used.

Desktops

This configuration contains the following desktops:

Setup

Desktop Setup contains the Simulation Setup with a node for the Virtual Terminal and a node for the Object Pool Checker. Furthermore, it contains the Measurement Setup and a Write Window. To the left of the Trace Window Trace VT is the CAPL program VTFilter.can. This filter allows only VT-relevant messages to pass, which are not sent cyclically.

Panel

Desktop Panel contains a panel for the Object Pool Checker, the Virtual Terminal, and a Trace Window that displays VT relevant messages only.

Measurement

Desktop Measurement contains a Trace Window that displays all messages, a Trace Window that displays VT relevant messages only, a CAN Statistics Window, and a Write Window.
