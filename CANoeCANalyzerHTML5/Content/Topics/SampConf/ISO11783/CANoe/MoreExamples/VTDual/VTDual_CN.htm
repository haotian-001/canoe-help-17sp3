Move To Another VT

 

Click on the blue link to load the sample configuration in CANoe.

  • Move To Another VT – ISO11783MoveToAnotherVT.cfg
    Working Set Master moves the display of the object pool from one Virtual Terminal (VT) to another VT.

This example demonstrates the Move to another VT feature, in which a Working Set Master can change over the display of its object pool from one Virtual Terminal (VT) to another. Three Virtual Terminals as well as a simulated Implement are used for this.

Introduction

At the start of measurement, the Implement sends its object pool to the first Virtual Terminal. The data mask contains the Next VT button. If you click this button, the Implement deletes the object pool from the current VT and starts a transfer to the next VT. After another start of the measurement, the Implement sends the object pool to the last used VT.

Desktops

This configuration contains the following desktops:

Setup

The Setup desktop contains the Simulation Setup with three nodes for the Virtual Terminals and one node for the Implement. The Measurement Setup and the Write Window are also contained. In the Measurement Setup you find on the left of the Trace Window Trace VT the CAPL node VTFilter. This filter only lets through cyclic VT-relevant frames when they exhibit a change.

Simulation

The Simulation desktop shows the three Virtual Terminals VT1, VT2, and VT3 and the Trace Window Trace VT.

Network

The Network desktop contains the J1939 Scanner, in which the addresses and device names of the devices involved can be seen, and the Trace Window Trace All. The transport protocol frames are suppressed in the Trace Window due to the activated analysis filters.
