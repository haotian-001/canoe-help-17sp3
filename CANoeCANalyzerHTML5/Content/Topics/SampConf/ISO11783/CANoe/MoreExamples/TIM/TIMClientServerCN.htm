TIM Client and Server

 

Click on the blue link to load the sample configuration in CANoe.

  • TIM Client and Server – ISO11783TIMClientAndServer.cfg
    Demonstrates the communication between a TIM (Tractor Implement Management) server and one or several TIM clients.

Introduction

The configuration contains one TIM Server and three TIM Clients. After the measurement start, you can configure the Facilities supported by the TIM Server and the Facilities required by the TIM Client. Then you can start the TIM Server and the required TIM Clients via the Start TIM Server or Start TIM Client button in the respective Panel. After authentication, TIM Clients perform the action Assign for each of the required functions. After you confirm automation (corresponding button Operator ACK, is located in the TIM Server panel and is colored green as soon as authentication is successfully completed) you can control each function of the TIM Server via the corresponding TIM Client. If acknowledgment for start of motion is required (Require acknowledgment for start of motion check box is activated) button ACK start of motion is colored green as soon as a TIM Client wants to start motion. Then click the button to allow this procedure.

TIM Server and TIM Clients work in demonstration mode, i.e. there is no verification of transferred certificates and other data. The demonstration mode can be deactivated via the check box in the Panel Demo Mode. For successful verification, however, the Authentication Library is required, which must be copied to the Exec32 directory of CANoe under the name AuthLib.dll.

Setup [▲ back]

The Setup desktop contains the Simulation Setup with six nodes: The TIMClient, TIMClient2, TIMClient3 and TIMServer nodes use the ISO11783 Interaction Layer DLL to simulate three TIM clients and a TIM server. The TIMStatus node supplies the TIM Status panel with the current client/server status and the TIMAuthDetail node controls the Authentication Details panel, which is displayed on the Authentication Details desktop.

Located before the Trace Filtered Trace Window in the Measurement Setup is the TraceFilter CAPL node, which only allows cyclical TIM messages to pass if their content has changed. This reduces the number of displayed messages. The filter can be controlled via the TIM Trace Filter Settings panel on the TIM Client and Server and TIM Clients desktops.

TIM Client and Server [▲ back]

This desktop is used to simulate communication between a TIM Server and a TIM Client. After the measurement starts, click on the Start TIM Server button in the TIM Server panel to start the server. Then click on the Start TIM Client button in the TIM Client panel to establish communication by the client with the server

After the initial start, the TIM client and TIM server perform a Fullweight Authentication (FwA) by exchanging their certificates and then negotiating a shared key, which can be used in the next step for a Lightweight Authentication (LwA). Following a FwA, you can decide whether FwA or LwA should be used during the next authentication. To use a Lightweight Authentication(LwA), click on the Store LwA table button following successful authentication. This is then used during the next authentication for the LwA. If this key is deleted again with Clear LwA table, the TIM elements again use Fullweight Authentication (FwA).

Note

Lightweight authentication is not supported in demonstration mode.

The TIM Server panel also includes the Operator ACK button which allows you to accept control of TIM functions by the client.

The TIM Client panel, on the other hand, has the Operator Enable button which allows you control functions.

Panel TIM Client also has the following buttons:

  • Reconnect
    Reestablish a broken connection to the server.
  • Restart Auth.
    Repeat a failed authentication.

Both panels also contain four TIM Server pages or five TIM Client pages for further control of the TIM client and TIM server:

  • On the Settings page, you can configure the properties of the client/server (Compliance Information group) and specify the certificates, CRL lists and private keys to be used for authentication (Certificates group):
  • For the TIM Server you can also configure Supported Facilities - the facilities that TIM Server lists in its Functions Support Response.
  • For the TIM Client, you can instead define facilities that the TIM Client absolutely needs (Required Facilities group). The TIM Client will then cancel its authentication if the TIM Server does not support these facilities. If the TIM Server supports all Required Facilities and authentication is successfully completed by both sides, the TIM Client performs the Assign action for each configured facility.
  • On the Behavior page, you can influence the behavior of the TIM Server or the TIM Client:
  • TIM Server/Client Status message
    State of TIM Server / Client.
  • Connection to selected TIM Client/Server
    Pause the internal status machine of the client/server at a certain state or skip some states.
  • Current Auth. State
    State of authentication.
  • If you select under Freeze State a state not equal to Offline, the client or server stops at this state. With Continue the authentication is continued. Continue and Freeze continues authentication until a new state is selected.
  • Sequence of requested certificates
    Define order in which certificates are requested.
  • Number of Certificate Requests to ignore
    Define how many certificate requests are ignored.
  • Deactivated Checks
    Here you can deactivate the following checks:
  • NAME
    The ISOBUS NAME transmitted on the bus matches the corresponding certificates.
  • CCID
    Conformity of the contents of the ISOBUS Compliance Certification message (CCID) with the certificates.
  • Capabilities
    Checking the certification of the TIM functions when they are assigned.
  • CRL
    Verification of the received certificates against the Certificate Revocation list.
  • Require acknowledgment for start of motion
    If this check box is activated you have to click the ACK start of motion button if a client wants to start motion.
  • Freeze Client/Server Heartbeat Counter
    Via this check box you can stop counting up the Heartbeat Counter.
  • On the Functions pages, you can control (TIM Client) or monitor (TIM Server) the functions used in automation.
  • For the TIM Client, in addition to a slider, there are also dedicated buttons available with which you can activate individual facilities. You can also use the buttons Unassign and Assign to end and resume control of the function.

The status of the TIM status messages are signaled in the TIM Status Messages panel with a colored LED. If a message is sent cyclically without interruptions, the LED turns green. Following successful authentication, if the AUTH server status message and AUTH client status message are no longer sent, the corresponding indicator turns gray with a green border. If the cyclical sending of one of the status messages is interrupted, the color of the LED changes to red.

Use the TIM Trace Filter Settings panel to activate or deactivate the filter conditions for the Trace Filtered Trace Window (with the help of the TraceFilter CAPL node).

In the Trace Window Trace Filtered, you can monitor communication between the TIM Server and all connected TIM clients.

3 TIM Clients [▲ back]

This desktop is used to simulate up to three separate TIM Clients. For each simulated TIM Client you have to configure the required facilities and then start the respective TIM Client with the button Start TIM Client.

Authentication Details [▲ back]

Located on this desktop is a panel by the same name that depicts the individual steps of the authentication. Once a step has been successfully completed, the corresponding LED turns green. Otherwise, it remains red. Errors can thereby be more easily found during authentication.

NMT [▲ back]

The J1939 Network Management analysis filter is activated in the Trace NMT Trace Window so that only messages relevant for network management are displayed. All active devices are displayed in the J1939 Scanner.

Trace All [▲ back]

You can use this desktop to view all messages unfiltered.

Trace Filtered [▲ back]

This desktop contains the Trace Window Trace Filtered, which is also used in the TIM Client and Server and 3 TIM Clients desktops.
