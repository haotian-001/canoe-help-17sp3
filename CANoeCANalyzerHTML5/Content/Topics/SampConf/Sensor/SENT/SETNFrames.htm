SENT Frames (Sensor)

 

Click on the blue link to load the sample configuration in CANoe.

  • SENT Frames – SENTFrames.cfg
    Demonstrates the frame based communication between a sensor and an ECU by means of SENT.

The SENT interface is a unidirectional, asynchronous interface, which has been standardized under the designation SAE J2716.

Introduction

The .Sensor option must be activated for CANoe in order to use the PSI5 sensor protocol. You also need to establish a physical Ethernet connection between CANoe and the VT System hardware. After that, open the VT System configuration dialog via the ribbon Hardware | VT System | Configuration. In this dialog, first choose the network adapter your VT System is connected to via the menu command Network Adapter. Finally use Configuration | Adapt to Connected Modules to map your configuration to the current hardware.

Desktops

This configuration contains the following desktops:

Trace_SENT

SENTFrames

In this panel, you can influence the communication between the SENT sensor and the ECU. Activate the Ramp Signal check box to automatically update the data content of the SENT frame; otherwise, the value entered under DataBits is used.

If you set eSentAutoCRC as CRC mode, the CRC value is calculated automatically and the value entered under CRCBits is ignored; otherwise, the CRCBits value is sent as CRC. Under certain circumstances, this can cause protocol errors.

You can modify the SENT status bits in the StatusBits input box.

Test Setup for Test Modules

The CAPL program that is integrated in the test environment uses the settings from the SENTFrames panel as parameters for the sensorQueueSentFrame CAPL function.

Trace

The SENT communication between the simulated elements is shown in the Trace Window.

Analysis

The Graphics Window and the Data Window display information relevant to the SENT communication. While the Data Window shows the last current value in each case, the Graphics Window displays the time sequences of the information.
