SPC Rolling Counter (Sensor)

 

Click on the blue link to load the sample configuration in CANoe.

  • SPC Rolling Counter – SPCRollingCounter.cfg
    Shows the use of Short PWM Code (SPC) on a VT2710 module.

Short PWM Code (SPC) is a synchronized data transmission based on the SENT protocol. In contrast to SENT sensors, which send data continuously, with SPC the transmission process is initiated by the ECU by sending a trigger pulse. By varying the trigger pulse length it is possible to address several sensors on one bus line.

The sample configuration demonstrates the use of SPC using four differently configured sensors sharing the same bus line.

Introduction

To use this sample configuration, you need a VT System with the VT2710 module equipped with at least two PSI5SENTpiggyA.

The SPC ECU is simulated on Piggyback channel 1. The four SPC sensors are simulated on Piggyback channel 2. For the example to function correctly, the two hardware channels must be connected as follows:

Desktops

This configuration contains the following desktops:

SPC VT System

This desktop contains the Sensor Configuration panel, which allows you to manipulate the SPC send behavior as well as the sent sensor signals.

The left side of the panel represents the ECU view, through which you control the generation of the trigger pulses. The triggering of each sensor can be done cyclically as well as in form of a single pulse. The output fields each contain the last received sensor signals and counter values.

The right side of the Sensor Configuration panel represents the sensor view. It allows the manipulation of the sent sensor signals and shows the last sent counter values.

The Graphics Window shows the counter values received from the ECU. In continuous transmission mode, the values show a sawtooth progression.

The Test Setup for test modules contains two CAPL nodes: The logic for generating the trigger pulses is implemented in the SPC_Scheduler node. The Sensor_Frame_Generator node handles the generation of the counter values and the filling of the sensor send buffers.

Sensor Traces

This desktop contains four Trace Windows, by means of which you can analyze the communication of each sensor.
