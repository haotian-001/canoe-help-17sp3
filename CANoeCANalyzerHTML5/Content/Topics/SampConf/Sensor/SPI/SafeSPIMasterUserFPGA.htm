SafeSPI Master User FPGA

 

Click on the blue link to load the sample configuration in CANoe.

  • SafeSPI Master User FPGA – SafeSPIMasterUserFPGA.cfg
    Demonstrates communication of a simulated SafeSPI Master with a simulated SafeSPI Slave using the VT2710 User FPGA.

Introduction

To use this sample configuration, you need a VT System with the VT2710 module.

The simulation of the SafeSPI Master is done by the User FPGA of the VT2710 on hardware channel 1. The SafeSPI Slave simulation is done on hardware channel 2 by usage of the standard SPI implementation. For a correct function of the example the two hardware channels must be connected as follows:

The sample configuration also assumes correct programming of the VT2710 User FPGA. Proceed therefor as follows:

Step by Step Procedure

  1. Start the VT System FPGA Manager.
  2. In the ribbon click on Import Projects.
  3. Open the file VT2710_SafeSPI_Master_Release.zip from the subfolder FPGA of the sample configuration.
  4. Open the imported FPGA project by double clicking VT2710_SafeSPI_Master_Release in the Project Explorer.
  5. Click Persist or Test, in the ribbon to load the FPGA project permanently or temporarily onto the VT System module.
  6. Select the network adapter that will be used to connect to the VT System. Note: Make sure that no other application (e.g. CANoe) is using the connection to the VT System during this step.
  7. Under Target Module, select the VT2710 module whose FPGA is to be programmed.
  8. Clickf Start, to program the FPGA

Before you start the measurement, make sure that the configured VT2710 (VT2710_SafeSPI) is assigned to the real hardware in the configuration dialog of the VT Sytem.

As soon as the measurement in CANoe has been started successfully, the communication between master and slave starts automatically.

Desktops

This configuration contains the following desktops:

Trace_SPI [▲ back]

The SPI communication between the two SPI nodes is displayed in two Trace Windows.

The Trace SPI Slave window displays the frame data sent by the simulated Slave node and the Chip-Select signal triggered by the SafeSPI master.

THe Trace SPI Master window displays the SafeSPI raw data and the response data interpretation of the three configured SafeSPI frames.

Analysis [▲ back]

The Data Windows Data SPI Slave and Data SPI Master as well as the Graphics Window display each the values of the transmitted signals.

If the hardware configuration is correct and no CRC error simulation is active for FastFrame1, the FastFrame1.RollingCounter signal in the Graphics Window points a sawtooth course.

SPI_VT_System [▲ back]

This desktop contains the Safe SPI Communication, which allows manipulation and monitoring of data for SafeSPI.

A total of three different SafeSPI frames are configured:

  • Fast Frame 1: Signal update frequency 100 Hz, 2x UInt8 signal
  • Fast Frame 2: Signal update frequency 100 Hz, 1x Int16 signal
  • Slow Frame: Signal update frequency 1 Hz, 1x UInt16 signal

The left side of the Safe SPI Communication represents the view of the SafeSPI Master node. The signal data received by the slave is displayed here. Frame CRC errors, if any, are signaled by a corresponding notification.

The right side of the Safe SPI Communication panel represents the view of the SafeSPI Slave node, which allows a manipulation of the sent signal data. Furthermore a CRC error can be generated for each of the configured frames via the Send Invalid CRC check box.

The Test Setup for test modules contains two CAPL nodes. The Slave_Logic node contains the logic for generating the slave data. In addition, the node ensures that the send buffer of the slave is always sufficiently filled and that the data is enqueued in the correct order.

The Master_Logic node takes over the initial configuration of the SafeSPI User FPGA Master. For the use of the User FPGA interface a special access library is necessary (SafeSPIMasterFPGA.vmodule), which is assigned to the network node. After the initial configuration, the SafeSPI master runs autonomously and requires no further attention from CANoe.
