PSI5 - Rolling Counter

 

Click on the blue link to load the sample configuration in CANoe.

  • PSI5 - Rolling Counter – PSI5RollingCounter.cfg
    Shows the usage of the Option .Sensor and the VT2710 module.
    Channel 1 simulates a PSI5 ECU, Channel 2 simulates a PSI5 sensor.

Between sensor and ECU, the RollingCounter signal is transferred by PSI5 protocol. During every transfer, the value of this signal is increased by one from 0 to 255, and then starts at 0 again. The example permits the application of VT2710 to be tested and to explore the data exchange by PSI5 protocol. The change of the signal value is ensured by a CAPL program. By means of the CAPL program, the change of the sensor values can be comprehended by CAPL functions.

Setting up the Hardware

In the Setting up the hardware description of the VT System there is a description of how a VT rack can be put into operation. In the PSI5RollingCounterExample the data are transmitted between channel 1 and channel 2 by means of the PSI5 protocol. For this purpose, the following signal cables have to be connected so that the data can be transferred:

  • ConnectPin 7 Sensor+1 of PSI5SENTpiggy Connector 1 to
    Pin 9 ECU+2 of PSI5SENTpiggy Connector 2
  • ConnectPin 6 Sensor-1 of PSI5SENTpiggy Connector 1 to
    Pin 8 ECU-2 of PSI5SENTpiggy Connector 2

Introduction

During the execution of the example, the changing values are displayed in the Graphics Window in the form of a saw tooth. The data SENSORSIM::Sensor1::TimeSlot1::Signals::RollingCounter are sent by the simulated sensor and the data ECUSIM::Sensor1::TimeSlot1::Signals::RollingCounter are received by the simulated ECU.

In the data window, the data are displayed as current numerical value.

The PSI5 protocol is displayed and recorded in the Trace window. After stopping the program execution, you can analyze the PSI5 protocol in the Trace window.

In the Write window, some information on the execution of the program is displayed. When a fault occurs when starting the example, they will be displayed in this window.

Significance of LEDs in the VT2710

When executing the example, the following LEDs are activated at VT2710:

PSI5/SENT 1

  • LED PSI5 flashes: Data are being transmitted by PS15 protocol.
  • LED ECU is lit: An ECU is simulated.

PSI5/SENT 2

  • LED PSI5 flashes: Data are being transmitted by PS15 protocol.
  • LED Sens. is lit: A sensor is simulated.

Description of the CAPL Program

By defining the RollingCounter signal for the simulated sensor, a RollingCounter_Stim system variable is set up. With the CAPL function QueueValues this system variable can receive data which are then transferred with the PSI5 protocol. In the process, no individual value is transferred but an array with values. The data buffer of the RollingCounter_Stim system variable is filled with data when the program is started. A timer is called up cyclically and fills the data storage of the system variables up again. Thus, the data storage of the system variable is always filled with data, and the data are transferred continuously.
