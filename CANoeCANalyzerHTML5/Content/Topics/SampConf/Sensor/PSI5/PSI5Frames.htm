PSI5 Frames (Sensor)

 

Click on the blue link to load the sample configuration in CANoe.

  • PSI5 Frames – PSI5Frames.cfg
    Demonstrates the communication between an ECU and a sensor via a PSI5 bus by means of a VT2710.

In this example, the sensor is completely modeled by the sending of PSI5 frames in CAPL with the use of the sensorQueuePsi5Frame function, in contrast to the signal-based communication of the other sensor sample configurations.

Introduction

For this sample configuration, a VT2710 with two PSI5Sent Piggybacks is needed. Connect two VT2710 ports to each other:

  • A VT2710 port has two PSI5 connections, which must be connected to the PSI5 connections of the other VT2710 ports:
  • Connect ECU+ and ECU- (are used by the simulated ECU) to
  • Sensor+ and Sensor- (used by the simulated sensor)

The .Sensor option must be activated for CANoe in order to use the PSI5 sensor protocol. You also need to establish a physical Ethernet connection between CANoe and the VT System hardware. After that, open the VT System configuration dialog via the ribbon Hardware | VT System | Configuration. In this dialog, first choose the network adapter your VT System is connected to via the menu command Network Adapter. Finally use Configuration | Adapt to Connected Modules to map your configuration to the current hardware.

In this sample configuration, use the Start button to activate communication between the simulated ECU and the simulated PSI5 sensor.

In this example, the channels are operated in asynchronous mode.

Desktops

This configuration contains the following desktops:

Trace_PSI5 [▲ back]

PSI5_Sensor

With this panel, you can change various aspects of PSI5 communication before and during the measurement. An exception is the Manchester coding on bit level, modification of which is not currently supported.

In the upper Initialization area, you can change the data for the PSI5 initialization. Please observe the PSI5 base specification for a more detailed explanation of these data. In this example, the transfer of the initialization data is configured in the Data Region A area The initialization data are queued for sending prior to the start of communication (in CAPL on prestart) by means of sensorQueuePsi5Frame. Deactivate the Send when starting check box to prevent this. You can send the initialization data again at any time with the Send Now button.

In the lower area, Communication, you can change the communication parameters of the PSI5 protocol during the measurement. These are:

  • Cycle time
  • Start bits
  • Size of Data Region A as well as it content.
  • CRC

Furthermore, you can use the Running check box to switch communication on and off or use the Interrupt button to skip a to-be-defined number of cycles.

ECU View

This Trace Window displays the data as they are received and interpreted on the ECU side. The PSI5 communication between the ECU and the PSI5 sensor is represented in the Trace Window in the form of frames. Each frame transports the contents of one signal in the Data Region A area.

Trace All

This Trace Window displays all events, including the frames sent via the sensor channel.

Analysis [▲ back]

The Data Window and the Graphics Window display information relevant to the PSI5. While the Data Window shows the last current value in each case, the Graphics Window displays the time sequences.
