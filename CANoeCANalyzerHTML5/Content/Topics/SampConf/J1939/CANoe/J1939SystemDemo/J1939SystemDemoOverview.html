Overview of Functions

An overview and detailed description of the functions of this sample configuration can be found on the Overview of functions page.

Introduction

A macro is automatically started when the measurement is started. The macro controls the gas pedal, the brake, and the gearshift.

If you would like to start the simulation manually (panel operation), deactivate the macro replay mode at the start of measurement via the Simulation ribbon tab|Automation menu.

System Demo [▲ back]

This desktop displays the panels of the ECUs. Before you can manually control the panels, you must deactivate the macro.

The following panels are displayed:

  • Truck
This panel displays various engine data and the gear engaged. The tire pressures of the individual tires are also displayed. You can change these with a slider. If the tire pressure leaves a set range, this is indicated by a change of the tire color. The TPMS ECU will then send DM1 messages.
 
  • IC
This panel displays the vehicle instrument cluster. The velocity and RPM can be read off here. The error display lights are located between these displays.
The address of the respective ECU is also shown at the bottom. You can disconnect the ECU from the network by clicking the connector icon.
 
  • TruckControl
You can control the engine with this panel. You start the engine with the ignition switch and shift to a different gear using the [Shift Up] and [Shift Down] buttons. You change the speed with the gas and brake pedals.

 

This desktop also displays a Graphics Window containing important values of the simulation. Changes on the panels affect these values.

Setup [▲ back]

This desktop displays the Simulation Setup and Measurement Setup. The Simulation Setup contains the simulated ECUs. The Measurement Setup contains blocks you can use to analyze the data traffic on the CAN bus.

Panels [▲ back]

This desktop displays the panels of the ECUs. Before you can manually control the panels, you must deactivate the macro.

All panels of the configuration display the address of the respective ECU at the bottom. You can disconnect the ECU from the network by clicking the connector icon.

The following panels are displayed:

  • EMS: The motor can be controlled with this panel. The position of the gas pedal can be influenced with the slider. This has an immediate effect on the number of revolutions If no gear is engaged.
  • TECU: This panel can be used to shift to a higher or lower gear. The vehicle has 5 gears and neutral gear. At the start of the measurement, neutral gear is engaged.
  • EBS: The brake pedal can be influenced using the slider on this panel.
  • IC: This panel displays the vehicle instrument cluster. The velocity and RPM can be read off here. The error display lights are located between these displays.

This desktop also displays a Graphics Window containing important values of the simulation. Changes on the panels affect these values.

  • Graphics Window: displays important values of the simulation
  • Data Window: displays other important values of the simulation

NMT [▲ back]

You can monitor the network management on this desktop. Each ECU must send an address claim Parameter Group when switched on. The J1939 scanner in the Measurement Setup receives this Parameter Group and can thus display the available ECUs in the network.

The Trace Window then displays only the Parameter Groups relevant for the network management.

Diagnostics [▲ back]

This desktop displays the windows for the diagnostics. The DTC monitor receives messages that contain error codes (DM1, DM2, etc. ). The Trace Window then displays only the Parameter Groups relevant for the diagnostics.

You can use the panels to activate a fault condition on the respective ECUs. Click the OK icon in the Faults section for this. A lightning icon is then displayed. The ECU now cyclically sends a DM1 Parameter Group containing an error code (DTC).

The DTC monitor evaluates the diagnostic messages. The list at the top provides an overview of the available nodes and their warning statuses. With the help of this list, the output of the data in the lower part (diagnostic trouble code) can be focused on one or more nodes. The prepared data are presented in different ways on different pages. If the macro of the sample configuration is active, the following information can be read for the TPMS node:

  • DTC History: At 65 s, a tire pressure warning at tire 11 (front rear wheel on right) is displayed. At 72 s, a tire pressure alarm is displayed for the same tire.
  • Active DTC: Nothing is displayed here initially. The tire pressure warning is displayed approximately 65 s after the start of the measurement and finally the tire pressure alarm after another 7 s.
  • Freeze Frame: Both the tire pressure warning and the tire pressure alarm support the extended freeze frame. Such freeze frames are contained in the DM25 message, which can be requested using the button on the right. If, after the request, an entry is selected on the left, the associated data are output on the right. The [sym] button of the CANoe tool list toggles between SPN and the signal name from the database.
    The various tire positions are differentiated by signal multiplexers and SPNs from the manufacturer-specific number range.
    Attempt to generate DTCs at other tire positions, as well. You can use, for example, the TPMS desktop for this.

FMS [▲ back]

The FMS desktop contains a Trace Window and a Data Window in which the messages of the FMS bus are displayed.

The FMS_Gateway (VGW node) is connected to both buses. It forwards selected Parameter Groups from the powertrain bus to the FMS bus. The parameters that the gateway must forward are specified in the FMS standard.

TPMS [▲ back]

This desktop displays the use of signal multiplexers using a tire pressure monitoring system as an example.

The tire-specific signals are transmitted within the same Parameter Groups (e.g. PGN FEF4h), namely the signals for one tire position per Parameter Group. The signals of different tire positions are transmitted in successive Parameter Groups. The TireLocation signal is used as the multiplexer.

The signals for the various tire pressures dependent on the multiplexer are visualized in the Graphics Window and Data Window. Changes in the tire pressures at various tire positions can be directly tracked in this way (e.g., using the slider in the Truck) panel. Note also the effect on the output in the DTC monitor on the Diagnostics desktop.

OBD-I/M [▲ back]

This desktop shows the J1939 OBD-I/M monitor (J1939 on-board diagnostic inspection and maintenance monitor). You can use this to configure and send DM7 test messages and receive the corresponding test results (DM8/DM30).

The Trace Window shows all Parameter Groups that are relevant for the On-Board Diagnostics.

Using the J1939 OBD-I/M monitor

Step by Step Procedure

  1. Select one or more nodes in the node overview.
  2. Select one or more predefined test commands in the bottom area and press the Send button.
  3. The test results are displayed in the result area.

The EMS and EBS nodes can evaluate multiple test results. The test results represent typical test scenarios: Pass, Fail, Not Complete, and Can Not Be Performed.

You can use the [sym] button of the CANoe toolbar to change between the display of the SPN and the signal name (as defined in the database).

TSC1 [▲ back]

This desktop displays the TSC1 Parameter Group. This PG is sent from two different nodes to the EMS engine management system in this sample. At gear shifting operations, the TECU transmission electronic control unit first requests a speed limiting and then a torque limiting. During a breaking operation, on the other hand, the EBS electronic braking system requests a negative (braking) torque. When the macro runs, the braking operation begins after approximately 74 s.

Various windows are displayed in which the differentiation of the TSC1 PG and its signals by source address is very important to enable analysis of the control requests of the two send nodes.

  • It can be easily seen in the Trace Window how the cycle time of the two TSC1 Parameter Groups changes as soon as there is a control request.
  • The signal sequences are compared in the Graphics Window. The control requests of the TECU during the gear shifting operations and the control request of the EBS during the braking operation can be easily seen.
  • The current signal values are compared in the Data Window. An active control request for speed or torque limiting is signaled with green on the J1939 indicator. The other signal values are easy to understand through symbolic interpretation.
