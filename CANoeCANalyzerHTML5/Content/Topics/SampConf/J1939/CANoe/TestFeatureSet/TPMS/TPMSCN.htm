Tire Pressure Monitoring System

 

Click on the blue link to load the sample configuration in CANoe.

  • Tire Pressure Monitoring System – J1939TPMS.cfg
    This configuration shows the use of the J1939 Test Feature Set using tire pressure monitoring as an example.

The ECU is simulated in CANoe . A test module performs the tests with the simulated ECU.

Test Implementation

When the measurement has been started, the test can be started with the TPMS Test panel. The [] button must be actuated for this. During the test execution at the bottom right Running is displayed and the Verdict column indicates the status of the test cases.

The test module changes the tire pressures during the test. This is done using system variables that are influenced by the CAPL node TirePressurecontroller.can. The simulated ECU can react to this and sends diagnostic messages (DM1) when the set reference pressure is fallen below or exceeded. This can be observed in the DTC monitor.

When the test is complete, Passed or Failed is displayed in the panel. After the measurement is stopped, the test report can be opened in the Vector Test Report Viewer via the Open Test Report button in the toolbar.

Desktops

This configuration contains the following desktops:

Setup

The desktop Setup contains the windows of the Simulation Setup, Measurement Setup and Test Setup. The Simulation Setup contains the node TirePressureController as it can also be found in the database J1939.dbc, which simulates the ECU. The Test Setup contains the test configuration TPMS Test. This test configuration calls successively individual test cases, which are implemented in a CAPL program in the folder vTESTstudioProject/TPMS_Test in the file TPMS_TestLib.can.

Test

At the start of measurement, the system automatically switches to the desktop Test. On this desktop there are various Windows for controlling the test. The TPMS Test panel controls the test module. Using the symbolic buttons for Start, Pause and Stop, you can initiate or interrupt the test execution. With the panel TPMS Chassis you can influence the simulated ECU TirePressureController. Using the controls on each tire, you can customize the tire pressure. The Recalibrate button can be used to set all selected tire pressures as a reference.

The DTC Monitor points to active and no longer active Diagnostic Messages (DM). DM1 is sent cyclically every second and contains the current states of the MIL, RSL, AWL and Protect lamps as well as active Diagnostic Trouble Codes (DTC).

Trace

This desktop contains the Trace Window and the Write Window.
