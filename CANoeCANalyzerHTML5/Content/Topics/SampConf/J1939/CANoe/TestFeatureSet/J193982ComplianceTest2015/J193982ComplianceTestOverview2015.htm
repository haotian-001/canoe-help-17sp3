Introduction

Note

This sample configuration is NOT intended for use in productive systems. To create a new test for your productive system, go to File ribbon tab, click on New and select under the J1939 Category J1939-82 Compliance Tests (2015). Further information you find on page J1939-82 Compliance Test (2015).

The sample configuration shows an example of how you can use the J1939.CT_Configurator panel to configure a J1939-82 conformance test and what the test procedure looks like.

In addition, you can activate individual protocol violations of the DUT to see how they are detected in the test and displayed in the Test Report and Trace Window.

Except for one difference, the sample configuration corresponds exactly to a configuration created from the J1939-82 Compliance Test (2015) template:

It contains a simulated DUT that is simulated by the DUT node in the Measurement Setup and controlled by the panels DUT Control and DUT: activate wrong Behavior.

Desktops

This configuration contains the following desktops:

Setup

CFG

Trace

Write

 

Setup [▲ back]

This desktop shows the Simulation Setup and the Measurement Setup.

The following nodes can be found in the Simulation Setup:

  • DUT: DUT as CAPL node in Simulated Bus mode. This node has to be deactivated in the Real Bus mode.
  • TOOL: This node represents the testing tool and should always remain active.
  • CFG: This node is an auxiliary tool for configuring the test. This mainly examines the prerequisites for the configuration and processes the auto start of the test.

CFG [▲ back]

On this desktop, you can configure the test and monitor the test procedure.

You can use the Trace Window and the Write Window track the test along with its bus traffic.

J1939.CT_Configurator

This panel is used to configure PGNs and other properties that describe the behavior of the simulated DUT.

DUT Control

In Table 10 you are prompted to switch the DUT off and on again or initialize it in different ways. On this panel there is a button for each type.

DUT: activate wrong Behavior

In this panel you can influence the behavior of the simulated DUT by activating individual errors and analyzing the effects in the test report and Trace Window.

Help

You can open the description of the help via this panel.

Trace [▲ back]

The Trace Window is emphasized on this desktop. Additionally, the test unit is displayed.

Write [▲ back]

The Write Window is emphasized on this desktop. Additionally, the test unit is displayed.
