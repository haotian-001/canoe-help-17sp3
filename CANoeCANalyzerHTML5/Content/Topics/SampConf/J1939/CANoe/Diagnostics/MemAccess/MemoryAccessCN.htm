J1939 Diagnostic Memory Access

    Memory access via CAN with J1939-73 diagnostic messages.

This sample demonstrates the functionality of the Diagnostic Memory Access Window. Data are read from and written to a simulated ECU using diagnostic messages (SAE J1939-73). The ECU requires authentication with the seed/key procedure.

Memory Access

Objects of the ECU can be read or written with the two buttons on the Object Access property page. After a request (DM14) the ECU answers with a numerical code (seed) to the Memory Access Window (DM15), which calculates the matching response code (key) using the CAPL program MemAccUtil.CAN and sends it back to the ECU (DM14). The ECU confirms the successful authentication (DM15). Only afterwards are the actual user data transmitted (DM16), and the ECU signals the completion of the complete transmission (DM15). The Memory Access Window then sends another request (DM14) or ends the memory access (DM14).

Note

For better monitoring of the message traffic in the Trace Window, you can deactivate the entries on the Cyclic Access page of the Memory Access Window.

The following objects are supported by the ECU, the value xy specifies the tire position (xy = {00, 01, 10, 11, 20, 21}):

Object

Access

TirePress_xy

Read-only

TireTemp_xy

 Read-only

ReferenceTirePress_xy

 Read and write

Accesses to other objects and write accesses to write-protected objects are canceled with an error message. After a write access to the ReferenceTirePress_xy object, a warning is output, depending on the written value, for the tire involved on the TPMSChassis panel.

Objects are accessed cyclically on the Cyclic access page while any memory area of the ECU can be read or written on the Direct Access page. In the case of reading, the ECU supplies data with ascending values. In the case of successful writing, only a message occurs in the Write Window.

User-Defined SPACE

The TireTemp_00 object can also be referenced in the user-defined SPACE 128 (0x80) using Index 7. So that the Diagnostic Memory Access Window can determine the correct unit, offset, and factor of the object from the database TPMS_Bus.DBC, the following attributes have been created:

The J1939SpaceSigAttr attribute specifies which user-defined SPACE the ECU supports. The object IDs in SPACE 128 (0x80) can be specified in this example using the TPMS_Space signal attribute.

Broadcast Messages

You can stop and restart the broadcast messages on all three access pages using a button (DM13). You can thus influence the sending of the DM1 message in this sample configuration.

Desktops

This configuration contains the following desktops:

Setup

The Setup desktop contains the Simulation Setup with the simulated ECU, the Memory Access Window, and a MemAccUtil node that calculates the key for the authentication. The Trace Window is inserted in the Measurement Setup. A filter is located before the Trace Window in order to prevent the display of transport protocol messages.

Diagnostics

At the start of the measurement, a changeover to the Diagnostics desktop occurs automatically. This contains the Memory Access Window, the panel of the simulated ECU, the Trace Window, and the Write Window.
