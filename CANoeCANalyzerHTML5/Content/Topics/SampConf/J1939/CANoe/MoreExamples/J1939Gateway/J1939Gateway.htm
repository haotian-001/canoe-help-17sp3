J1939 Gateway

 

Click on the blue link to load the sample configuration in CANoe.

  • J1939 Gateway – J1939Gateway.cfg
    Demonstrates two ways to implement a J1939 gateway.

Introduction

The configuration contains three CAN networks with two J1939 ECUs each. The CAN1 network is connected to the CAN2 network via a gateway and to the CAN3 network via a second gateway. A panel is assigned to each network, with which parameter groups are sent to the ECUs of another network.

Setup [▲ back]

The Simulation Setup contains three networks:

  • CAN1 contains the ECU_10 and ECU_11 as well as the two gateways CAN1/CAN2 and CAN1/CAN3.
  • CAN2 contains ECUs ECU_20 and ECU_21 as well as the gateway CAN1/CAN2.
  • CAN3 contains the ECU_30 and ECU_31 and the gateway CAN1/CAN3 .

The ECUs use the J1939 Interaction Layer.

The Measurement Setup contains three Trace Windows. There is a Channel Filter on the left before each Trace Window, so that only messages from one channel are displayed. There is also a J1939 Scanner.

CAN1/CAN2/CAN3 [▲ back]

A Trace Window and a Panel are displayed for each network. In the panels, click one of the Send buttons to send a parameter group (PG). You can send a PDU1 parameter group to a specific ECU (via the Destination list) or a PDU2 parameter group to global address 255.

The CAN1/CAN2 and CAN1/CAN3 gateways are programmed so that PDU1 PGs from CAN1 can be sent to CAN2 and CAN3 and that PDU1 PGs from CAN2 or CAN3 are only sent to CAN1.

The two gateways use two different methods:

  • The CAN1/CAN2 gateway first receives the entire parameter group on one channel and then sends it on the other channel. For PGs transmitted with the transport protocol CMDT, the gateway handles the transport protocol in the CAPL function HandleCMDTMessages. Only the composite message is then forwarded in the event procedure on pg. The gateway is more complex to program. However, before the PG is forwarded, its content can be analyzed, e.g. to enable filtering of the PGs or to modify the PGs before forwarding. The disadvantage is the longer transmission time for PGs that are longer than 8 bytes, since the transport protocol must be handled on both channels.
  • The CAN1/CAN3 gateway, on the other hand, only forwards PGs with the maximum length of 8 bytes to the other channel. The gateway is easier to program and the messages arrive faster at their destination.

In the Analysis panel, the duration from sending the parameter group to its reception by the ECU in the other network is displayed.

With the Gateway CAN1/CAN2 panel, you can influence the CAN1/CAN2 gateway in such a way that the PGN is changed when a message is forwarded to the other network.
