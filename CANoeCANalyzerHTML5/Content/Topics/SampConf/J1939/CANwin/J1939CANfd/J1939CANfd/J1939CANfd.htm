J1939 CAN FD (J1939-22)

 

Click on the blue link to load the sample configuration in CANoe.

  • J1939 CAN FD (J1939-22) – J1939CANFD.cfg
    Shows the support of J1939 in a CAN FD network according to J1939-22.

The transmission of J1939 parameter groups with CAN with flexible data rate (CAN FD) is specified in document 1939-22. Parameter groups are sent there either as Contained-PGs (C-PGs) of a Multi-PG or with the FD transport protocol (FD.TP). This sample configuration shows how easy you can simulate a J1939 ECU sending C-PGs with the help of a Multi-PG.

Introduction

The sample configuration contains three nodes which can send J1939 parameter groups. After the measurement start, the node Node1 begins with the cyclic sending of two parameter groups, which have a different cycle time.

Desktops

This configuration contains the following desktops:

Setup

In the Simulation Setup, the J1939.dbc database is assigned to . In this, the protocol type CAN FD is selected for the network J1939, which activates the usage of CAN FD in the configuration. There are also two simulation nodes (Node1 and Node2), which all use the J1939 Interaction Layer (which is also defined in the database), and a simulation node Node3, which is implemented exclusively with CAPL. The Measurement Setup includes a Trace Window, a Bus Statistics Window and a Graphics Window. In addition, the Write Window can also be seen.

Measurement

Each of the three simulation nodes has a panel that can be used to configure the node and send messages.

Since the Node1 and Node2 nodes are simulated with J1939 interaction layer, their send behavior is predefined via the J1939.dbc database and can be customized at runtime via CAPL.

In contrast, the Node3 is simulated with a CAPL program, therefore its transmission behavior can only be influenced via the CAPL program.

All CAPL calls are triggered by panels.

Node 1

Under Node Properties you can set whether Multi-PGs sent to global address 255 are sent with FBFF (Flexible Data Rate Base Frame Format) or with FEFF (Flexible Data Rate Extended Frame Format). You can also set at which fill level the Multi-PG is to be sent. The Multi-PG is filled with C-PGs until the fill level is exceeded. Only then is the Multi-PG placed on the bus.

In the Diagnostics area you can activate the diagnostic support via the Activate Diagnostic Support check box. If this is done, the node additionally sends the diagnostic message DM1 cyclically. If you activate the Activate Diagnostic Trouble Code check box, the DM1 transmits a preset Diagnostic Trouble Code (DTC) in the DM1.

You can activate or deactivate the two cyclic messages CPgCylic1000 (cycle time 1000ms) and CpgCyclic2000 (cycle time 2000ms) in the Cyclic C-PGs area and change the values of the transmitted signals.

You can also request the parameter group CPg1 of node Node2 and the address-claimed messages of all nodes via a J1939 request. This is done with the buttons in the Request area.

Node 2

Under Node Properties you can specify the maximum time to wait until the node sends a Multi-PG after a C-PG has been triggered, i.e. released for sending. As with Node1, the maximum fill level of the Multi-PG can also be specified. If one of the two conditions is exceeded, the Multi-PG with the contained C-PGs is placed on the bus.

The area CPg1 allows you to configure properties of the parameter group CPg1. For this message you can also define the maximum time to wait until the Multi-PG is placed on the bus after CPg1 has been triggered. If -1 is entered here, the time defined for the node is used.

With the list Appearance on Bus you can define whether the message CPg1 is transmitted as Contained-PG by means of a Mutli-PG, or whether it is transmitted as a single CAN frame or CAN FD frame.

You can use the Type of Service and Trailer Format Value input boxes to specify whether the C-PG should also contain safety data or security data (possible combinations of values are listed on the Send and Receive Parameter Groups with CAPL page). In the CAPL program of the node, the so-called Assurance Data is then filled with fixed dummy values in the callback J1939ILOnTxPrepare before the C-PG is inserted into the Multi-PG. Their length is displayed in the Assurance Data Length field.

The Trigger C-PG button can then be used to add the CPg1 once or several times to the Multi-PG, which is then placed on the bus, depending on the settings described. Triggering of the PG is triggered by the J1939ILSetMsgEvent function of the J1939 interaction layer.

Node 3

In this panel you can send a Multi-PG containing two Contained-PGs. In the same way as in the Node2 panel, you can specify in the Type of Service and Trailer Format input boxes whether the two PGs should contain CPg2 and CPg3, safety data or security data.

However, the sending of the parameter groups is done differently than with Node2: When clicking the button Transmit both C-PGs, the functions J1939InitializeMultiPG and J1939AddContainedPG are called in the CAPL program of the node. These are used to initialize and fill a Multi-PG. The Multi-PG is then placed on the bus with the command output.
