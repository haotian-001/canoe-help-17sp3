Graphics Window and Data Window (J1939)

 

Click on the blue link to load the sample configuration in CANoe.

  • Graphics Window and Data Window (J1939) – J1939GraphicAndData.cfg
    Demonstration of the J1939-specific particularities of the Graphics and Data Windows.

The following signal values are used in the sample configuration:

  • Signal ActualEngPercentTorque (1 byte):
  • Maximally used valid value: 80%
  • No values for "error"
  • Equivalent physical value when "not available": 130%
  • Signal EngSpeed (2 bytes):
  • Maximally used valid value: 2400 rpm
  • Equivalent physical values used for "error": 8128 rpm and 8130 rpm
  • Equivalent physical value when "not available": 8160 rpm
  • Signal ACCDistanceAlertSignal (discrete)

Setup [▲ back]

The Setup desktop contains the Simulation Setup and Measurement Setup. The Simulation Setup contains an Interactive Generator you can use to generate messages for the measurement as well as a CAPL node that sends a Parameter Group with an ASCII signal. The signal is displayed by two Graphics Windows and Data Windows each located in the Measurement Setup.

Generator [▲ back]

The Generator desktop contains the Interactive Generator. At the start of the measurement, a changeover to the desktop with Graphics and Data Window occurs automatically.

Graphic & Data with different senders [▲ back]

The Graphic & Data with different senders desktop contains the Graphics Window Graphic Diff Sender and the Data Window Data Diff Sender. Here you can see representation of the signal, that is sent simultaneously by two ECUs.

Multiple Senders Send the Same Signal

If two senders send the same Parameter Group and one signal from this Parameter Group is to be displayed in the Graphics Window or Data Window, the source address must be set in order to receive meaningful values. The Graphic & Data with different senders desktop shows this behavior.

The same signal is entered three times in the Graphics Window. Two nodes in the network send this signal. In the case of the upper signal sequence, a source address is not set. Consequently, the signal ‘jumps’ between the values of the different senders. In the case of each of the two lower signal sequences, the address of the send node is entered. A clean signal sequence is therefore displayed here.

The Data Window, on contrary, automatically recognizes that the defined signals come from two different addresses and creates a separate line for each direction. Furthermore, the Data Window shows the ASCII signal generated by the CAPL node.

Graphic & Data w. Error Signals a. Not Available Signals [▲ back]

The Graphic & Data with Error Signals and Not Available Signals desktop contains the Graphics Window Graphic Error Signals and the Data Window Data Eror Signals. Here you can see representation of errorneous or an unavailable signal values.

Signals with Error and ‘Not available’

J1939 has defined signal values that identify a signal error or an unavailable signal. The Graphic & Data with Error Signals and Not Available Signals desktop shows how these signal values are displayed in CANalyzer.

The Graphics Window and Data Window show three different signals.

In the Graphics Window, the output without J1939 indicator is compared to the output with J1939 indicator. The Graphics Window ignores the unavailable signal values when the indicator is activated in the signal settings.

The Data Window always shows the current value - with one exception: if the value corresponds to the not available state. In this case, the last available value is displayed. The current state of the signal can always be recognized by the background color (green = valid, gray=not available, yellow=reserved, red=error).
