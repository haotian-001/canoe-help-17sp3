Truck Overview

 

Click on the blue link to load the sample configuration in CANoe.

  • Truck Overview – J1939TruckOverview.cfg
    Sample with display panels.

The purpose of this sample configuration is to provide an overview of the signals present in a truck. It can be used with a real bus of a truck. It is advantageous when the SAE J1939 standard is supported.

No Parameter Groups are transmitted. There is only a graphical display of the typically present bus signals.

Panels

The panels display signals that are present in a standard J1939 network. Note that only the signals that are also present in the vehicle can be displayed. Signals not supported by the vehicle are not displayed.

Note

The J1939 database J1939.DBC from Vector is used for evaluating the messages and signals. This database is contained in every standard installation of the J1939 option. Messages and signals are displayed in the following format:

<ECU>.<Message>.<Signal>

A few signals are accepted independent of the sender, and thus the <ECU> information is omitted in the display.

This panel displays signals that can be present in a J1939 network. These are signals from the engine, transmission, brake system, passenger compartment controller, instrument cluster, faults that have occurred, etc.

Aftertreatment System [▲ back]

This desktop contains the Aftertreatment System panel. This panel displays signals that are used during exhaust gas aftertreatment. The diagram shown is based on Figure SPN1761 – Example Aftertreatment System Schematic of the SAE Specification J1939-71 Jan 2009.

Battery Information [▲ back]

This desktop contains the BatteryInformationPanel panel. This panel displays signals that are used in a battery management system. The diagram shown is based on Figure PGN65104_A – Battery Information and Figure PGN65104_B – Battery Information Legend of the SAE Specification J1939-71 Jan 2009.
