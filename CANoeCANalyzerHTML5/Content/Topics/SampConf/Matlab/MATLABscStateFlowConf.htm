MATLAB Integration: State Flow System

 

This sample configuration is only available after you have installed the MATLAB Integration Package: Open Setup Folder

See further: MATLAB Integration and MATLAB Integration Setup Introduction

Click on the blue link to load the sample configuration in CANoe.

State Flow System – MATLAB StateFlowSystemDemo.cfg
The generated code of the Simulink sample model sf_car.mld has been added to the CANSystemDemo. (MATLAB Integration Package installation required).

This configuration is derived from the CAN System Demo sample configuration that is supplied with CANoe. In addition to the original configuration the generated code of the Simulink sample model sf_car.mdl has been added. The original System Demo models the behavior of a car engine with CAPL. This part was replaced by the Automatic Transmission Control sample configuration that comes with Simulink. The model also shows the usage of a state flow subsystem.

You will find this model in: Open Folder
...\StateFlowSystemDemo\Models\sf_car.mdl

It contains seven CANoe specific signal input/output blocks:

  • BreakForce
  • EngineRunning
  • PedalPosition
  • EngSpeed
  • Gear
  • CarSpeed
  • Simulation step

The Simulation step block is only relevant to the Simulink Simulation.

When you build the model a folder (sf_car_cn_rtw) containing several files generated during the code building process is made in the current MATLAB working directory. To specify a destination for the DLL file generated you must first specify the compiler's output directory. Open the menu path Tools| Code Generation Options item and click the [Options…] button. In the Output Directory input box specify your directory: Open Folder
...\StateFlowSystemDemo\Models\Exec32

Note

If the directory path contains spaces (e.g.: C:\Program files\CANoe) you must specify the MS-DOS name (usually: C:\Progra~1\CANoe). Use the Explorer to find the MS-DOS name.

If nothing is specified the DLL will be generated in the directory
..\sf_car_cn_rtw\release\
and must be copied manually to Open Folder
...\StateFlowSystemDemo\Models\Exec32.

Additional settings in the Simulation Parameters dialog:

  • Export CAPL function:
    With this option activated two special CAPL functions are exported by the generated DLL. Refer to MATLAB CAPL Function Overview for more details.
  • Enable Parameterization from CANoe:
    If this option is activated values of all model parameters are accessible from CANoe. Refer to Sine Parameterization Configuration for an example.
  • Enable Simulink Signal Analysis:
    Enables analysis of all named Simulink signals. With CANoe 7.0 or newer system variables are created for each signal below the namespace MODEL::Signals. This option requires the activation of the C-API for signals.
  • Generate Visual C++ project file:
    The build process first builds a makefile. If this option is selected Visual Studio is opened. The compilation process now must be finished within Visual Studio. This is useful to generate a DLL with debug info. If this option is not selected, the DLL is built directly by a command line call.
  • Perform Simulation Step at t(0):
    If the model does not contain blocks with states (e.g. the model contains only algebraic blocks) this option can be selected to get a value for the simulation step at t=0.

After all settings are done…

  1. Close all dialogs and save your changes.
  2. Open the menu path Tools|TRW Build.
  3. The DLL will be linked into your Exec32 directory.
  4. Start CANoe and open the configuration StateFlowSystemDemo.cfg (see blue link on top).
  5. Open the Environment ribbon tab|Compile All Nodes.
  6. To verify that CANoe has detected the DLL open the Simulation Setup (Simulation ribbon tab).
  7. The Engine node should display a Simulink Coder string in its second description line although this depends on the order the modeling libraries are defined in the CANdb.
  8. Start the measurement to view the models in action
  9. There is a Replay Block that controls the measurement. 20 seconds after measurement start, the pedal position moves to 100%. Now you can see the model’s behavior.
