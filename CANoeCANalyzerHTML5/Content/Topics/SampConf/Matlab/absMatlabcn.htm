MATLAB ABS Brake Control

 

Click on the blue link to load the sample configuration in CANoe.

  • ABS Brake Control – ABS_Brake.cfg
    Sample of model based development with CANoe.

This sample configuration* illustrates the use of the CANoe MATLAB® interface for model-based development with CANoe.

The MATLAB model depicts a brake control - once with and once without ABS. In the MATLAB model, these two controls are implemented as subsystems that are calculated in the Control Panel once the corresponding pedal is engaged.

MATLAB's Simulink Coder is used to generate a modeling library that is assigned to the Chassis node in the CANoe configuration as a nodelayer module.

At the start of measurement a test module is automatically started that performs the following checks:

  • Check the braking distance compared to a nominal value that must not be exceeded – with and without ABS.

This test is performed with a Simulink model containing a block for dynamic monitoring of a maximum value. The maximum value can be specified in the TestManipulation panel (Test desktop). The test results are returned in the system variable BrakeDistVerdict, which is monitored in the test module via a StateChange pattern.

  • Check the braking time compared to a nominal value that must not be exceeded – with and without ABS

This test is also performed using a Simulink model. The model monitors the slope of the braking distance and measures the corresponding time. If the slope is null, this means that the braking process is completed. The braking time is returned in the system variable BrakeTimeVerdict which is monitored in the test module via a StateChange pattern. This setting can also be modified in the TestManipulation panel.

  • Illustration of changed rule behavior through specification of different nominal slip values.

This test case changes the internal values of the simulation model. Three braking procedures are performed using different nominal slip values – all with ABS. The changed rule behavior can be observed in the Graphics Window.

 

In each case, the test module sets the speed to 100 km/h. The Graphics Window shows the following values:

  • Braking distance (xF)
  • Vehicle speed (vF)
  • Wheel speed (vR)
  • Wheel slip (λ) defined as λ = (vF-vR)/vF.

In the Graphics Window, you can see that the braking distance is much shorter when braking with ABS than when braking without it.

The wheel slip value without ABS λ = 1, due to the blocking of the wheel (vR = 0).

In the Parameterization desktop it is possible to change the model behavior without having to recompile the model. The vehicle mass, for example, which is a model-internal constant, can be changed. Several model-internal values are also displayed in the Graphics Window.

* The MATLAB example is taken from the following book: H.E. Scherf — Modellbildung und Simulation dynamischer Systeme (Modeling and Simulation of Dynamic Systems), Oldenbourg Verlag München Wien 2004.
