ASAM XIL API - Server

 

Click on the blue link to load the sample configuration in CANoe.

This sample consists of several parts:

  • ASAM XIL API – Server – XILAPIServer.cfg
    Shows how CANoe can be accessed by third party test automation tools using its XIL API interface.
  • XIL API Client – Open Folder
    Contains the application (C#/.NET) in that serves as the test automation tool in this example.
  • XIL API Configuration – Open Folder
    Contains the test bench and port configuration files for the CANoe XIL API interface.

This sample configuration shows how CANoe can be accessed by third party test automation tools using its XIL API interface. Therefore a simple .NET application is included in this sample configuration, which takes the role of the test automation tool.

Running the Example

Step by Step Procedure

  1. Before running this example you need to install the ASAM XIL API on your computer. The EXE file for setup is part of this CANoe installation: Start Setup
  2. After that you need to start CANoe and load the sample configuration. Load Configuration
  3. 4) Now start the XIL API client application from the bin folder in this example and connect to CANoe. Start Application

CANoe Sample Configuration

The CANoe sample configuration is very simple. It contains a single system variable called Variable1. This variable is stimulated by a Signal Generator with a sine wave. This variable is exposed to the XIL API via the FDX description.

FDX description (Fdx.XML) see: Open Folder

Client Application

The client application takes the role of the third party test automation tool. It demonstrates how you can instantiate and work with the CANoe XIL API test bench. It starts the CANoe measurement and displays the received variable data.

Note

  • This application is written in C# and .NET 4.0. A precompiled executable is available in the bin folder: Open Folder
  • The Visual Studio project and all sources are also part of this sample and can be found in the Source folder: Open Folder
