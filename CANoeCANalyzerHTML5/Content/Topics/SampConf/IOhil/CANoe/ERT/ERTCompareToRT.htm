Extended Real Time - Compare To RT

 

Click on the blue link to load the sample configuration in CANoe.

  • Extended Real Time - Compare To RT – ERTComparisonWithRT.cfg
    Direct comparison between a MATLAB model execution in distributed mode and with Extended Real Time.

Setting up the Hardware

This sample configuration requires an ERT-enabled VTP Device. ERT must be activated on the VTP Device.

To execute this sample configuration, the ERT-enabled network interface must be connected via USB or Ethernet and CANoe must be configured in distributed mode.

Note

ERT can only be executed in Real Bus mode. If no CAN hardware is available, CAN1 can also be assigned to a virtual CAN bus.

Introduction

This CANoe configuration executes a MATLAB model simultaneously as an ECU node in the RT kernel and as an ERT node with the Extended Real Time feature. To compare the two execution modes, different metrics are displayed in a panel, the Graphics Window as well as the Data Window.

Note

When the ERT feature is activated, not all system resources are available for RT execution. Therefore, the performance shown in this sample configuration is worse for the ECU nodes compared to an execution completely without ERT. However, if the ERT nodes are deactivated in the Simulation Setup, this sample configuration can be executed without ERT.

Simulation Setup

The Simulation Setup contains two nodes with a MATLAB model (ERT Model and RT Model). Each node is assigned a simple MATLAB model that outputs a sinusoid. The models are controlled by a timer through a CAPL program. Execution using CAPL allows the cycle time of model execution to be changed and execution to be paused as needed. The ERT Model node is executed as an ERT node with the ERT feature. The RT Model node is executed as an ECU node directly from the RT kernel without the ERT feature.

In addition, there are two CAPL programs in the Simulation Setup for generating generic load (ERT Load and RT Load). Again, one node is executed as an ERT node and the other as an ECU node.

Measurement Setup

Another CAPL program is hooked into the Measurement Setup, which performs some of the necessary performance calculations for MATLAB step time as well as CPU utilization.

Demo [▲ back]

When you start the measurement, the desktop Demo is automatically activated.

ComparePanel

In the upper part of this panel, you can explicitly disable execution in RT or in ERT with the corresponding Enable Execution check boxes. Also, several metrics about the execution in RT and in ERT are output:

  • Step Time Deviation
  • This value indicates the absolute deviation from the set cycle time of the model.
  • Example: The cycle time set by default is 5000 μs. If 5100 μs elapse between the previous execution of the model and the current execution, this results in a step time deviation of 100 μs.
  • You can reset the Step Time Deviation values using the Reset Step Time Deviation button.
  • Step Execution Time
  • This value specifies the execution time of a single model execution, i.e. a MATLAB step.
  • CPU Load
  • This value contains the average CPU usage during the last 10 seconds.

Advanced Settings

Model Cycle Time

With this slider you can change the cycle time of the model. For very small cycle times < 1 ms, execution without ERT no longer makes sense, therefore model execution in RT is automatically stopped when the set cycle time falls below 1 ms. However, you can activate execution again manually if required.

Additional Load

Use this slider to activate the additional load.

Base Load

Use this list to set the base load. This should match the VTP Device in usage:

  • Low = VN8911, Medium = VN8914 / VT6060
  • High = RT Rack.

In the following, you can use the Additional Load slider to set the load more finely granular in various stages.

Graphics Window

Here you can see the Step Times of the RT and ERT models and have a direct comparison of the absolute values. If required, you can show the calculated sine curves.

Data Window

Here you can see more raw values that can be interesting on a second look.
