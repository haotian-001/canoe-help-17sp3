System Configuration (Extended Real Time)

 

Click on the blue link to load the sample configuration in CANoe.

  • System Configuration (Extended Real Time) – ERTSystem.cfg
    Demonstrates the execution of network nodes in a real-time environment using the ERT feature.

Introduction

Note

The example requires one of the following ERT-enabled CAN network interfaces:

  • VN8911 with VN8970
  • VN8914 with VN8970/VN8972

ERT must be activated on the network interface.

To execute this sample configuration, the ERT-enabled network interface must be connected via USB or Ethernet and CANoe must be configured in distributed mode. In addition, the channels CAN1 and CAN2 must be connected with a CANcable1.

Note

This sample configuration, like all configurations in which ERT is used, can only be executed in Real Bus mode.

When the measurement starts, the MATLAB models and CAPL scripts of the network nodes are executed in ERT. On the Demo desktop some calculations of the network nodes are displayed graphically. In addition, this desktop shows the performance in terms of latencies and execution times of the tasks.

Configuration [▲ back]

This desktop shows the Simulation Setup and the Measurement Setup. There are two CAN channels in the Simulation Setup.

On the first channel several ERT nodes are inserted. Each network node runs in ERT on its own thread. This means that the different nodes are executed in parallel.

The ErtNode node contains an interaction layer and a CAPL script, which controls the interaction layer and reacts to a changed signal with a CAN frame.

Two MATLAB models and a CAPL script are inserted in the TwoModels node. The MATLAB models are controlled by the CAPL script and the step functions of the two models are executed consecutively.

In the SineModel node, a MATLAB model is executed that writes a sine signal. This node is configured so that the thread that executes the model in ERT always has the highest priority.

In the last node GenericLoad, a CAPL script for demostration purposes creates additional load in ERT.

The analysis windows used are shown in the Measurement Setup. Also included here is a CAPL script that calculates a performance variable.

Analysis [▲ back]

This desktop shows the CAN Statistics and in the Data Window the signals and system variables relevant for the measurement.

The system variables SysvarSine, SysvarSineModified, and the SignalSine signal are described by the MATLAB models. The MATLAB model that writes the sine signal calculates two variables as performance indicators: StepTime is the cyclic time used to calculate this MATLAB model and maxSampleTimeis the maximum time required by the MATLAB model for the calculation.

In addition, the Data Window displays the percentage utilization of the two CPUs of the realtime operating system in which ERT is running.

You can monitor the bus traffic in the Trace Window.

Demo [▲ back]

This desktop demonstrates the various features of ERT and the specifics of executing tasks in ERT.

Five system variables and signals are displayed in the Graphics Window Graphics. The top two system variables are each written by a MATLAB model. Since the two MATLAB models are mounted on the same ERT node, they are executed on a thread in ERT and can be executed sequentially using a CAPL script. The result of a calculation step of the first model SysvarSine is multiplied by a factor and output using the second MATLAB model SysvarSineModified.

In the MATLAB Model Control panel, you can start and stop the execution of both models using the check boxes through the CAPL script. When execution of Model 1 is stopped, Model 2 is no longer executed because its input depends on the output of Model 1. The execution of Model 2 can be stopped separately. In addition, the sine factor can be varied between 0 and 10 using the slider or the input box. The amplitude of SysvarSineModified in the Graphics Window is then modified accordingly.

In the Graphics Window the third signal (SignalSine) is described by another MATLAB model. The model has a periodic execution time of 500 µs for each calculation step. The actual measured execution time is displayed in the Graphics Window by the system variable SignalStepTime.

In the ERT Performance Variables panel, the performance of the execution of this MATLAB model should be displayed. The maximum deviation of the actual execution time is displayed with Max Deviation from Step Time. Max Sample Time describes the maximum time the MATLAB model takes to calculate a step. This panel also displays the utilization (in %) of the CPUs used by the realtime operating system.

If you activate the Calculate harmonic series check box in the CAPL script execution control panel, this cyclic and computationally intensive operation increases the CPU load in the realtime operating system. However, since the nodes in ERT are executed in parallel, the performance of the sine signal calculation (Max Deviation from Step Time and Max Sample Time) is hardly affected.

The lowest signal in the Graphics Window demonstrates a simple use case of CAPL in ERT. If the sine signal makes a zero crossing, a frame with the signal Signal Zero is sent in ERT. In the Write Window, the notification Hello from CAPL in ERT! is displayed at the start of a measurement from a CAPL script running in ERT.

In the InteractionLayer Control panel, CAPL can be used to control the interaction layer in ERT. If a frame is sent from the interaction layer, a counter (Message Counter) is incremented. If you deactivate the check box InteractionLayer: Send Signals, you can see in the Graphics Window that the two signals SignalSine and SignalZero are no longer sent. Also the Message Counter then stops at the current value.
