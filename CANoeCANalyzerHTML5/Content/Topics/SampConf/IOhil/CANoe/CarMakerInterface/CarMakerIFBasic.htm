CarMaker Interface Basic

 

This sample configuration is only available after you have installed the CarMaker Add-on Package: Open Setup Folder

See further CarMaker Interface.

Click on the blue link to load the sample configuration in CANoe.

  • CarMaker Interface Basic – CarMakerIFBasic.cfg
    Shows the basic functionality of the CANoe/CarMaker interface.

The sample configuration shows connection control, subscribing to different groups of CarMaker quantities, and retrieving test results.

Introduction

Before starting the measurement, CarMaker must be started on the local computer and set to the Idle state. During the measurement, CANoe establishes a connection to CarMaker and starts a CarMaker test series.

Note

This sample configuration uses the CarMaker sample project Examples/DriverAssistance/BrakingAssist.

This sample configuration demonstrates the use of the CarMaker CAPL functions using a CarMaker test series. When the measurements starts in CANoe, a connection to the locally running CarMaker instance is established. A test series is loaded based on a sample project installed with CarMaker. The test series consists of twelve test runs. For each test run, CANoe creates an ASCII log file in which the quantities subscribed to from CarMaker are logged as system variables. At the end of each test run, its result is queried by CANoe and also stored in the log file via a system variable. For the second half of the test runs, the group of subscribed CarMaker quantities is switched.

Desktops

Analysis

In the Data Window and the Graphics Window, the quantities subscribed to from CarMaker are displayed. The Write Window displays status information about the current test run.

Configuration

The NodeCarMakerControl network node handles communication with the CarMaker. The CarMakerControl component and the CAPL program for controlling the CarMaker are configured here.
