TCP/UDP Client and Server

 

This page describes several sample configurations.

Click on the blue link to load the sample configuration in CANoe.

  • TCP/UDP Client – Tcp_Udp_Client.cfg
    Shows the usage of the TCP/IP CAPL API Client in CANoe.
  • TCP/UDP Server – Tcp_Udp_Server.cfg
    Shows the usage of the TCP/IP CAPL API Server in CANoe.

The example demonstrates the use of TCP/IP CAPL API for sending or receiving TCP or UDP data in CANoe.

Note

  • To enable sending of data via TCP and UDP, the computer requires a network card with a configured IP address.
  • If client and server configurations are opened on a computer with two CANoe instances, the client and server ports must be configured differently.

Start two CANoe instances and load the client configuration and the server configuration.

At the start of the measurement, the IP address of the computer is displayed automatically in the IP Address field for the server in the respective Client/Server Panel. If the field does not contain an IP address, the IP address of the computer could not be determined. In this case, the sending of data via TCP or UDP cannot be demonstrated.

Sending Data via UDP

If you want to send data via UDP, you have to enter a port number for the server and for the client.

For UDP communication, there is no difference between client and server configuration: both can be used for opening the port and sending/receiving data.

The following procedure is used in the use case below:

  • The port is opened on the server.
  • The port is opened on the client.
  • The first data are sent from the client to the server.

To start, enter a port number, e.g., 5050, in the Port field for the server in the UDP area of the Server Panel, and open the port with Open Port.

Enter the following data in the Client Panel:

  • Enter a port number, e.g., 5000, in the Port field in the UDP area, and open the port with Open Port.
  • Enter the IP address of the server in the Server IP Address field.
  • Enter the port number of the server in the Server Port field.
  • In the Send data field, enter the data that are to be sent to the server.

The data can now be sent to the server with Send data to server.

Besides the received data, the IP address and the port number of the client also appear in the server.

You can now send/receive data from/to the client or server as required.

Note

The port numbers of the server and client can be changed.

Sending Data via TCP

If you want to send data via TCP, a connection between the client and server must first be established.

Enter the port number for the server in the server configuration in the Server Panel. When you activate Start listening, the server waits for incoming connections.

Enter the following data in the Client Panel:

  • Enter the IP address of the server in the Server IP Address field.
  • Enter the port number of the server in the Server Port field.

Connect the client to the server by activating Connect to server in the Client Panel. The client port can be configured (change port number), optionally. Otherwise, the client port is assigned automatically.

In the Send data field, enter the data that are to be sent to the server.

The data can now be sent to the server with Send data to server.

As long as a client is connected, data can be received from and sent to the client.

Note

Brief information and possible errors are output in the form of a message in the Write Window.
