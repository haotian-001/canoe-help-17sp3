VT System - Offline

 

Click on the blue link to load the sample configuration in CANoe.

  • VT System - Offline – VTDemoOffline.cfg
    Use different VT System modules to test ECUs with CANoe.

This sample configuration shows you how you can use different VT System modules to test ECUs with CANoe. This is demonstrated using a concrete example, in which a temperature ECU is simulated and evaluated using VT modules.

The fictional ECU used in this configuration has two inputs and three outputs, as shown in the image below:

The ECU regulates the heating and cooling depending on the temperature, which is measured by a temperature sensor. If the temperature exceeds a critical value, a warning LED is also switched on.

The following VT modules are used in this example to simulate the ECU and monitor its outputs:

The individual VT modules execute the following tasks:

  • The VT7001 module generates the power to run the ECU.
  • The VT2004 module simulates a temperature sensor via its internal decade resistor.
  • The VT1004 module measures the output voltages that the ECU uses to control the heating and cooling. The first two channels of the VT1004 are used for this purpose. An internal load at channel 1 and an internal short circuit relay at channel 2 are also available. These allow you to simulate changes in the cooling unit's internal resistance and generate short circuits in the heating module.
  • The VT2516 module measures the digital output of the ECU that switches the warning LED on and off.

Introduction

You can use a number of the VT System module functions in this simulated test setup. For instance, it is possible to simulate different temperature values for the ECU by changing the resistance generated by the VT2004. You can evaluate the ECU's reactions to these changes with the VT1004 and/or VT2516. The following sections explain how to use this configuration for the test setup and how to use the simulated VT modules.

Overview [▲ back]

This desktop provides access to the VT System Demo Panel. You can use this panel to operate the four VT modules and monitor the ECU. The panel supports the following elements:

Operating the Simulated VT2004 Module

The VT2004 module simulates the temperature sensor. Its internal decade resistor is used for this purpose. The temperature transmitted to the ECU depends on the value of the generated resistance.

By moving the slider you can modify the current resistance value. Permissible values for this sample configuration range from 10 Ohm (minimum measurable temperature) to 6000 Ohm (maximum measurable temperature).

You can also use the Short circuit switch to simulate a short circuit in the sensor.

Operating the Simulated VT7001 Module

This module generates the power supply for the simulated ECU. The ECU requires a 12V power supply. Using the slider on the VT7001 panel it is possible to generate other voltages. Voltage values > 13V generate excess voltage, whereas values < 11V generate low voltage. The ECU switches itself off in both cases.

The VT7001 measures the currently flowing voltage and displays it in the panel. This is independent of the specified output voltage.

Operating the Simulated VT1004 Module

The simulated VT1004 module's first two channels measure the ECU's output voltages, which control the heating and cooling. The measured voltages are displayed both in text form and as a status bar. These control voltages range from 0V (device is deactivated) to 5V (device is running at maximum capacity).

You can change the electronic load at channel 1 using the Internal load slider. The ECU expects an output current proportional to a load of 500 Ohm here. If the load (and thus the current) changes too much, the ECU reacts by switching off the cooling.

You can use the Short circuit switch to simulate a short circuit at channel 2, i.e. the heating. In this case the ECU reacts by switching off the heating.

Operating the Simulated VT2516 Module

This module measures the voltage at the ECU's digital output. The voltage corresponds to the state of the warning LED (0V: LED is off, 5V: LED is on) and is displayed in the panel.

Operating the Simulated ECU

The simulated temperature ECU displays data about some of its internal states in the panel.
The first data item is the state of the power supply. This is also shown via a LED (off: low voltage, red: excess voltage, green: voltage in permissible range).
The second data item is the currently measured temperature, which the ECU calculates from the temperature sensor's resistance value.
The third data item is the state of the two analog outputs. This shows whether the heating or the cooling is activated, and at what capacity they are running.
The fourth data item is the state of the digital output, which controls the warning LED.

You can use the Faulty LED control switch to influence the ECU's behavior. If you activate the switch, the warning LED is no longer switched on when a short circuit occurs in the temperature sensor.

Analysis [▲ back]

This desktop provides two analysis windows:

The Data Window lets you monitor the internal states of the sample configuration.

The Graphics Window provides a visual display of the most important simulation values. These are the current resistance of the temperature sensor, the heating and cooling control voltage and the state of the warning LED.

You can influence the simulation by using the VT2004 Mini Panel to change the VT2004's current resistance.

Diagnostics [▲ back]

This desktop enables you to access a number of CANoe diagnostic functions.

You can use the ECU Fault Memory Window to read out the fault memory of the simulated ECU. It contains entries such as warnings in response to low/excess voltage or short circuits.

You can also send diagnostic commands to the ECU via the ECU Diagnostic Console. The ECU will react, for example, when its serial number is queried or its fault memory is deleted. The Trace Window shows communications between the ECU and the tester at the CAN message level.

Testing [▲ back]

This desktop contains four sample test cases, which allow you to check part of the ECU's functionality:

CAPL: Power Supply Test

This test module is implemented in CAPL. It contains four test cases.

In the first case, the cooling is switched on by generating a high temperature value. Then the power supply is configured in such a way that the ECU has excess voltage. A check is then carried out to see whether the ECU and the cooling are switched off as would be expected.

The second test case is set up in a similar way, but in this case a low voltage is generated. Once again a check is carried out to see whether the ECU and cooling are switched off as would be expected.

The third and fourth test cases check the ECU's fault memory. A low voltage is generated in one case and an excess voltage in the other, and the ECU's fault memory is read out after each.

CAPL: Heating and Cooling Tests

This test module is implemented in CAPL. The first test group checks the heating by generating different temperatures and evaluating the ECU's reactions.
The second test group repeats these tests for the cooling unit.
The third test group contains tests for a set of error cases. A short circuit is generated in the sensor, another short circuit is generated in the heating, and the behavior of the ECU is monitored. In addition, various values are generated for the cooling unit's internal load, and the behavior of the ECU is monitored.

XML: Warning LED Test

This test module is implemented in XML. A template is used to generate various temperature values and check whether the ECU correctly activates/deactivates the warning LED.
The last test case checks whether the ECU activates the warning LED when a short circuit occurs in the temperature sensor. You can influence the behavior of the ECU in this case via the Faulty LED control check box.

The last test case checks the ECU's fault memory. A short circuit is generated in the temperature sensor, then the ECU's fault memory is read out.

XML: Abstract Test

This test module is implemented in XML and contains three test cases. Here abstract system variables are used (set and read out) instead of the system variables of the individual VT modules. These are connected to the system variables of the simulated VT modules by mapping.

In the first test case, a check is made to determine whether the cooling runs at 100% at a high temperature setting. By contrast, the second test case checks to determine whether the heating runs at 100% at a low ambient temperature.

The third test case sets an error in the temperature sensor and checks to determine whether the corresponding LED on the ECU is activated.
