VT System Inter Board Communication

 

Click on the blue link to load the sample configuration in CANoe.

  • VT System Inter Board Communication – VTSInterBoardCommunication.cfg
    Shows the direct connection of several VT System User FPGAs via ring bus.

Configuration

  • To set up this sample configuration, you will need at least two different VT System modules with User FPGA which are supported by the configuration, as well as a suitable connecting cable for each module used.
  • Not all modules integrated in the configuration need to be present. Only two modules are required, so the remaining modules which are not present are set to Offline at the start of measurement.
  • One module must be present as the master. In the basic setting, this is the VT2816. Should this module not be present, another module must be specified as the master in FPGA Manager. The project must then be recompiled for this module and written to the module.
  • The sample configuration is copied to the workbench automatically when the manager is first set up and can be restored later using the Tools|Option|Restore Demo Projects menu path.
  • The code is precompiled and can be written directly to the FPGA or the EEPROM of the module.
  • You must select the connection to the VT System module in CANoe:
  • Click VT System on the Hardware Ribbon Tab and then click Configuration.
  • Click the Hardware command.
  • Select the network card that the VT System is connected to in the Network Interface list.
  • You can see how the module is to be interconnected via the Connection desktop.

Introduction

  • Connect all modules to be used and plug them in with the corresponding connecting cables.
  • Start the measurement.
  • The transfer of data between the FPGAs is carried out manually using a panel in CANoe.

Desktops

This configuration contains the following desktops:

Control

Connection

VT System

 

Control [▲ back]

The IBCDemoControl panel is used to specify the data to be sent between the individual user FPGAs. The date of a module can be received by any other module in the configuration. Modules which are not present are partially hidden after the start of measurement. The presence or absence of a VT2816 is also indicated, as this is specified as the master.

Connection [▲ back]

The IBCDemoConnection panel shows the interconnection of the modules for this configuration.

VT System [▲ back]

The VT System Control panel shows the individual modules. Indication of whether or not a module is offline is also provided for each module.
