Easy (FMI)

 

This page describes several sample configurations.

Click on the blue link to load the sample configuration in CANoe.

  • Easy (FMI) – EasyFMI.cfg
    Provides the CANoe Configuration as FMU Functional Mock-up Unit via FMI Functional Mock-up Interface.
  • FMU Consumer – FMUConsumer.cfg
    Uses the FMU provided from the sample configuration Easy (FMI) and serves their inputs and outputs.

The communication between the two sample configurations takes place via the Functional Mock-up Interface (FMI).

The example is based on the Easy (CAN) sample configuration. The lamp control is calculated in the Easy (FMI) sample configuration and was made available as EasyFMI.fmu via the FMI Export. The Control panel can only be found in the FMU Consumer sample configuration. The measurement in Easy (FMI) is automatically started at the start of the measurement in FMU Consumer. All inputs using the Control panel are transferred to the lamp control. The outputs of Easy (FMI) are displayed both in Easy (FMI) and in FMU Consumer in the Display panel as well as the other analysis windows.

Introduction

Load the two configurations in their own respective instance of CANoe.

Note

  • You can run the two configurations on different computers.
  • In the FMU Consumer you must specify the IP address of the computer that runs the Easy (FMI) (default: 127.0.0.1).
  • When a RT system is used, specify the IP address of the RT server computer.
  • When a VN8900 is used, specify the IP address of this device.
  • To specify the IP address, you must adapt the following system variables in the Start Values Window:
  • FMI::EasyFMI::Parameter::CANoeInAddr_B1 (default: 127)
  • FMI::EasyFMI::Parameter::CANoeInAddr_B2 (default: 0)
  • FMI::EasyFMI::Parameter::CANoeInAddr_B3 (default: 0)
  • FMI::EasyFMI::Parameter::CANoeInAddr_B4 (default: 1)
  • Start the measurement in the FMU Consumer sample configuration.
  • Activate the desired functions in the Control panel in the FMU Consumer sample configuration.
