COM .NET

 

Click on the blue link to load the sample configuration in CANoe.

  • COM .NET – COM.Net.cfg
    Shows the usage of VB.Net and C# programs to control CANoe via COM Interface.

The COM Interface can be used to control CANoe from external programs. In addition to applications, external programs also involve scripts or programming languages. Here are the best known script or programming languages that are currently available: VBScript, JScript, Perl, VBA, VB.NET, C#, Delphi and C/C++.

Sources

In the C# and VB sub directories you will find the corresponding Visual Studio solutions. In order to use the sample, you have to build the solution first. This should be possible without any further adaptions.

Open Folder

Using the Sample Application

The sample application consists of a simple form with which you can load a configuration and start/stop the measurement. Furthermore you can use CAPL functions and also read/write system variables and signals.

Cross Reference

The application note CANoe/CANalyzer as COM Server offers further information on how to set up a solution to use the COM Interface. It also shows code snippets of this sample application and gives further explanations (e.g. on how the application uses CAPL functions).
