vFlash Automation

 

Click on the blue link to load the sample configuration in CANoe.

  • vFlash Automation – vFlashNodeLayer.cfg
    The sample configuration shows how you can flash an ECU with CANoe.

Targets

The sample configuration shows how you can flash a control unit from CANoe. The control unit can be flashed from CANoe test modules. For this, the reusable include file vFlash\Utilities.cin will be read into a test module, as can be seen in /test/simple-use-case.CAN.

The vFlash program will thus be remote controlled via its automation interface. You can then use vFlash to flash previously generated vFlashPacks to a control unit. With just a few adjustments you can also flash via LIN and FlexRay as well as via CAN.

Using the Sample Configuration Locally (CAN)

  1. Install CANoe and vFlash starting with 2.7 SP3.
    If you have installed vFlash the first time, you have to restart Windows.

  2. Start CANoe.

  3. Make the following settings in the Network Hardware Config dialog (call via Control Panel or Hardware ribbon tab|Network Hardware|[Drivers...]):

    • connect vFlash CAN1 to one channel
    • connect CANoe CAN1 to another channel
    • when not using a real ECU, connect both channels with a wire

  4. Load the configuration vFlashNodeLayer.cfg.

  5. Start the measurement.

  6. On the Flash ECU desktop start the test case.

Using the Sample Configuration on a VN8900 or VT System (CAN)

You can use the sample configuration on a VN8900 or VT System realtime mode in standalone mode.

vFlash Installation on VN8900/VT System (if not already available):

  1. Open the Vector Tool Platform dialog of CANoe via the Home ribbon tab and connect to the VN8900/VT System via USB or Ethernet.

  2. Go to the CANoe/CANalyzer | vFlash page and click the Update button.

  3. Select the vFlash VN8900 Setup Package (e.g. vFlash2.7SP3.vn89 from the vFlash Setup CD).

  4. Confirm installation.

vFlash Configuration on the VN8900/VT System:

In order to use vFlash a flash package (vFlashPack) needs to be available on the VN8900/VT System and vFlash needs to be assigned to a communication channel. Download the sample vFlash project file.

  1. Open the Vector Tool Platform dialog of CANoe via the Home ribbon tab and connect to the VN8900/VT System via USB or Ethernet.

  2. Go to the CANoe/CANalyzer | vFlash page and click the Add... button.

  3. Select the demo.vflashpack file from the sample configuration folder: Open Folder

Assign vFlash Communication Channel on a VT System:

  1. Connect to the VT System with a Remote Desktop Connection.

  2. Open the Vector Hardware Config dialog.

  3. Assign vFlash and CANoe to different CAN hardware channels that are physically interconnected with a CAN loop cable.

Assign vFlash Communication Channel on a VN8900:

  1. Connect the VN8900 via USB to your computer.

  2. Open the Vector Hardware Config dialog and assign vFlash and CANoe to different CAN hardware channels that are physically interconnected with a CAN loop cable.

Notes for Customizing the Sample Configuration

With CAN you are able to set the flash baudrate in the CAPL test with the variable gFlashingBaudRate. With LIN and FlexRay the baudrate cannot be changed during measurement.

You can define the flash package setting via the variable gFlashpack in the CAPL test. The path can be absolute.

With LIN and FlexRay you can use the flash packages from the sample configuration of the vFlash installation.

The Blocking Use Case (test modules/units only)

In a test module or test unit, you can use TestWaitForvFlash… CAPL functions to perform the necessary steps. The program flow will stop in the function until the task is completed, or an error has been detected.

The function TestWaitForvFlashPackReprogrammed from the reusable include file vFlash\Utilities.cin will take care of all necessary steps in a single call. This is demonstrated in the CAPL program /test/simple-use-case.can. It includes the starting of vFlash, the loading of the flashpack, the flashing, the unloading, and the closing of vFlash. The necessary steps are performed automatically one after the other in this order.

The Event-driven Use Case

In simulation nodes, the CAPL program flow must not be stopped, therefore TestWaitFor… functions are not available. It is still possible to use vFlash for flashing, but a different set of functions has to be used.

The function vFlashStartPackReprogramming from the reusable include file vFlash\Utilities.cin will start the sequence of CAPL function calls that indicate completion by the call of a CAPL callback function also implemented in the include file.

The call in the CAPL test module is: vFlashStartPackReprogramming(flashpack);
with flashpack as the vFlashPack to be flashed, e.g. Open Folder

Error Codes/Return Values

The return values of TestWaitForvFlashPackReprogrammed are:

  • 1: if flashing is successful.
  • 0: if a timeout occurs. Is the timeout (in milliseconds) too short?
  • <0: an error occurred

Baudrate

The baudrate is often increased during flashing via CAN compared to normal operation. The flash baudrate is specified in vFlash and saved in the flash pack. The current baudrate can be read out, checked, and if necessary set in the CAPL test module prior to flashing. Likewise it can also be reset after flashing. The commands for this are:

  • canGetConfiguration(channel, settings);
  • canSetConfiguration(channel, settings);

A example is contained in the test module of the sample configuration; see
/test/Baudrate.cin in SetBaudRate.
