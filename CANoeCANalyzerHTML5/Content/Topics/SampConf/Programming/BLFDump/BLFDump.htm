BLF Dump

 
  • BLF Dump
    Demonstrates how to use the binlog library under Windows and Linux to read and output the contents of a BLF logging file on the command line.

The example contains C++ code (C++11) and uses the build tool CMake. The example only supports systems with 64-bit architecture and to build the example the following tools must be installed on the system used:

  • CMake (starting with version 3.6)
  • Visual Studio starting with version 2013 (for Windows systems only)
  • gcc (starting with version 7.4.0) or clang (starting with version 6.0.0) Compiler (for Linux systems only)

Note

The example only supports the processing of CAN frames or CAN FD frames and system variables. For all other objects contained in a BLF logging file the notification BlfDump: Unhandled type: &lt;binlog-Object-ID&gt; is issued.

The following format descriptions of the objects in BLF format for different bus systems can be used to use or output these additional object types:

  • CAN_and_General_BLF_Format.pdf
  • FlexRay_BLF_Format.pdf
  • LIN_BLF_Format.pdf
  • MOST_BLF_Format.pdf

Open Folder

The supplied example contains both the required binlog header files in the Include folder and the binlog library for Windows (binlog.dll and binlog.lib) and Linux (binlog.so) in the subdirectories of the Libs folder. The sample source code for demonstrating the use of the binlog library is located in the Source folder. You can use the build directory to store the files generated by the build tool CMake (see the instructions for executing the build process with CMake below). When the build process is complete, the executable files created during this process are installed in the appropriate subdirectories (Win64_Debug, Win64_Release, Linux_Debug and Linux_Release) of the bin folder.

Instructions for Executing the Build Process with CMake

To create the BlfDump.exe (Windows) or BlfDump (Linux) debug or release variant executable file, follow these steps:

1. Open a command line and navigate to the BLF_Dump folder.

2. Change to the directory build: cd build

Note

Before performing the next steps, make sure that the build directory is empty and does not contain any files or folders from previous builds. This can prevent problems during the further build process.

3. Executing the build process:

a) To create the debug variant of BLF_Dump, execute the following commands depending on the platform you are using:

On Windows with Visual Studio (starting with version 2013) as CMake generator:

cmake .. -A x64

cmake --build . --target install

On Linux with Unix Makefiles as CMake generator:

cmake ..

cmake --build . --target install

b) To create the release variant of BLF_Dump, execute the following commands one after the other, depending on the platform used:

On Windows with Visual Studio (starting with version 2013) as CMake generator:

cmake .. -A x64 -DCMAKE_BUILD_TYPE:STRING=Release

cmake --build . --config Release --target install

On Linux with Unix Makefiles as CMake generator:

cmake .. -DCMAKE_BUILD_TYPE:STRING=Release

cmake --build . --config Release --target install

After executing these steps, the selected variant (Debug or Release) of BLF_Dump is located in the corresponding subfolder of the bin directory (Win64_Debug, Win64_Release, Linux_Debug or Linux_Release), depending on the platform. The required version of the binlog library (binlog.dll for Windows or libbinlog.so for Linux) is automatically copied to this directory. It is necessary for the execution of the example application that these files are located in the same directory.

Using the Created BlfDump Application

1. Use a command line to navigate to the corresponding target directory of the created application.

Note

Make sure that the corresponding version of the binlog library (binog.dll for Windows or binlog.so for Linux) is in the same directory. If this is not the case, the application will not work correctly!

2. Run the application using one of the following commands:

On Windows:

BlfDump.exe <Path to the BLF logging file>

On Linux:

./BlfDump <Path to the BLF logging file>
