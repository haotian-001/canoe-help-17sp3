Basic Configuration FDX (A429)

Basic Configuration FDX (A429) - A429BasicFDX.cfg

This sample configuration shows the usage of the CANoe FDX protocol in connection with the System Configuration (A429). It consists of a CANoe configuration and the additional FDXClient.exe program.

Overview

The sample configuration shows the usage of the CANoe FDX protocol in connection with the System Configuration (A429 Sample Configuration). It consists of the CANoe configuration and the additional FDXClient.exe program. This is provided in variants C# (Start program) and C++ (Start program) in the directory of the sample configurations. Both variants behave identically.

The interplay of CANoe and the FDXClient is carried out via the FDX protocol (CANoe FDX Protocol).

The sample configuration supports only online mode.

Description

The CANoe FDX sample configuration is based on the A429 FDX System sample configuration. The control of the overhead panel buttons and the stimulation of the tank sensors is carried out using the FDXClient instead of via the Interactive Generator in the case of the tank sensors, or manually in the case of the overhead panel buttons.

Errors, changes in the measurement status, or the current gross weight are output in the console window, if a change occurred in the last 250 ms.

Sequence of the FDX sample configuration

  1. Start CANoe and load the configuration A429BasicFDX.cfg.

  2. In the command line interpreter of Windows (command shell, cmd.exe), start the external program FDXClient.exe and specify the IP address and the port of the CANoe computer as parameters.

    If no IP address and port is entered, the FDXClient is connected to the local computerusage.
    Example: FDXClient.exe 10.49.52.79 2809

    Note

    • When a VN8900 device is used, a network connection to the device is required and the IP address of the VN8900 must be specified.
    • When an RT system is used, the IP address of the RT server computer must be specified. The measurement must already be started before the FDXClient is connected.
  3. The FDXClient starts the measurement and stimulates CANoe until it is closed with <Ctrl>+<C>. In so doing, the measurement in CANoe is also ended in an orderly manner.
  4. A message is output in the console window when the measurement is started or ended.
