Bitmask Analysis (FlexRay - Scope)

 

Click on the blue link to load the sample configuration in CANoe.

  • Bitmask Analysis (FlexRay - Scope) – FlexRayBitMaskAnalysis.cfg
    Demonstrates how you can execute a bitmask analysis for FlexRay with the Scope option.

This sample configuration is an expansion of the FlexRaySystemDemo sample configuration. The additionally added Scope desktop illustrates how you can execute a bitmask analysis for FlexRay with the Scope option.

The configuration can be carried out in CANoe exclusively in the online mode with the Real bus setting.

Requirements

  • Scope option
  • A vector Vector VN interface with at least one FlexRay channel
  • The channel connection must be terminated with a terminating resistor
  • A PicoScope supported by CANoe (PDF manual, Scope option)

Procedure

  • Start the measurement (Home ribbon tab or <F9> function key).
  • Start the test module in the BitMaskTest window with the Start button.
  • A remaining bus simulation is carried out. The frames sent by an ECU are recorded and analyzed by the Scope. See the Scope Window and the other analysis windows for additional information.

BitMaskTest

Test Cases

The entire test is divided into two steps on the Test Cases page in the BitMaskTest window. These can be activated or deactivated separately:

  • Analysis ECU
    Analysis of all send frames of an ECU.
  • Provoke errors
    Analysis of all send frames of an ECU.
    If a bitmask is defined too large, bitmask errors will b provoked.

Edit Test Module

The definition of the bitmasks, the analysis control and the error evaluation are carried out with functions of the BitAnalysis.CAN library, which can be viewed or edited with the Edit Test Module button.

Report

The results of the analysis are written to the test report, which can be called up in the BitMaskTest window with the Report button.

System Under Test

In the System Under Test panel you will find the ECU under test input field. The name of the ECU to be tested is entered there.

The Frames under test list is then used to specify which frames are to be analyzed:

  • All (all frames)
  • KeySlots (only frames in key slots)
  • StaticSlots (only frames in the static segment)
  • DynamicSlots (only frames in the dynamic segment)
