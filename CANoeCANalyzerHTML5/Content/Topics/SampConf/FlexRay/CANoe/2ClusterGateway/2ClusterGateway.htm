Introduction

Real Mode Usage

Following requirements must met for Real-Mode usage:

  1. Two VN interfaces have to be connected to CANoe and must be mapped to channel FlexRay 1 and FlexRay 2.

  2. Do NOT connect any nodes to the interfaces! All nodes are currently simulated.

  3. Do NOT interconnect the FlexRay channels between the interfaces!

  4. You must connect the hardware synchronization line between both interfaces by a Sync cable!

  5. You must enable the hardware synchronization in the hardware configuration settings of CANoe.

  6. Pay attention to the Write Window that the hardware time synchronization works properly during runtime.

Gateway Functionality

  1. The gateway is able to control the forwarding and to manipulate the payload of the forwarded frames.

  2. Additionally the gateway is able to synchronize the cluster time difference of the start of cycles to a requested value.

  3. The gateway also measures the routing delay of particular frames of both directions. The routing delay depends on the cluster offset to each other.

FRFR Delay [▲ back]

This desktop displays the delay measurement.

Gateway Sync Control

This panel allows you to add a random offset when sending using the Create Random Offset button:

  • PowerTrain_FlexRay (FR1) -> PowerTrain2_FlexRay (FR2)
  • PowerTrain2_FlexRay (FR2) -> PowerTrain_FlexRay (FR1)

This leads to a delay of the current signal values. Subsequently, the gateway node tries to synchronize the offset again to a value of 495 µs.

Use the following buttons to start the attempt to set the delay:

  • Request Equal Min Delay
  • for both directions to approx. 5 ms
  • Req. Min Delay FR2 -> FR1
  • FR2 -> FR1 to < 2 ms
  • FR1 -> FR2 to > 8 ms
  • Req. Min Delay FR1 -> FR2
  • FR1 -> FR2 to < 2 ms
  • FR2 -> FR1 to > 8 ms

Gateway Routing Delay

This panel shows the results of the delay measurements:

  • (A) PowerTrain_FlexRay -> PowerTrain2_FlexRay (for EngineData)
  • (B) PowerTrain2_FlexRay -> PowerTrain_FlexRay (for GearBoxInfo)

FRFR Synchronization [▲ back]

This desktop shows the effect of a different cycle start of the two FlexRay clusters.

The FRFR Sync Control panel allows you to control the cycle offset of the two clusters. By activating the Automatic Adjust check box, the automatic customization of the cycle start of the PowerTrain2_FlexRay cluster to the first cluster is activated. The synchronization process takes a few seconds. You can monitor the current state of the offset via the panel.

You can use the Info 4 button to open another panel that explains the options that can be set.
