FlexRay AUTOSAR NM - Node Panel

 

The configuration contains three nodes, A, B and C for which a panel is displayed.

As default, node C starts in passive mode while all other nodes are active. This behavior can be changed during Bus-Sleep mode.

In the upper part of the panel, the state of the node is displayed.

Control

Description

Status line

The line in the upper left corner indicates the current state of the node.

Partial Network

Here the membership to a partial network is denoted.
Note that on a FlexRay bus, the PN membership has no influence on when a node goes to sleep.

Car Wake-Up

When the car wake-up bit is received, this lamp will flash for a short time.

Repeat Message

A lamp that indicates if the node is in Repeat Message state.

 

In the middle and left part of the panel, you can influence the behavior of the node.

Control

Description

Network Request

Indicates whether the node requests the bus to be active (green, pressed), or if it allows the network to go to sleep (red, released).
The number indicates which partial network(s) will be requested when the button is pressed.
Node B will request both networks, while the other nodes will only request the network they belong to.

Active Mode

Controls the passive mode. If pressed (green), the node is active, i.e. not in the passive mode. If released (gray) the node is in passive mode. The button is inactive (gray text) if the node is not in Bus-Sleep.

Communication

When in Normal Operation state, you can disable communication with this button (gray, released), or re-enable it later (green, pressed). In other NM states the button is inactive (gray text).

Partial Networking

Enables or disables the handling of partial networks, which corresponds to the parameter NM_PN_ENABLED. If pressed (green), PN information is processed and the node sends messages with the PN information bit set. If released (gray), PN information is ignored. In that case, the PN information bit in its messages is 0, else it is set to 1.

Goto Repeat Message / Detected Nodes

If pressed, the node will enter Repeat Message state, if possible, and set the Repeat Message bit. The LEDs are updated once the node leaves Repeat Message state again, and indicate which nodes have sent NM messages.

Note

Due to the mechanics of partial networking, only the nodes in the same network can be detected as long as partial networking is enabled.

 

In the lower part of the panel, the NM message can be influenced.

Control

Description

Set User Data

You can enter a byte sequence here that will be put into the user data part of the NM message this node sends. If the message length is exceeded, spare bits will be cut. Press [Enter] to apply the input.

Set Control Bit Vector

Allows setting the CBV byte value sent by this node. Press [Enter] to apply.

User Data Rx

If Update is pressed, the last user data received by this node will be displayed.

FAQ

Only every 25th NM data PDU a node receives is indicated in the Write Window. This reduces the amount of output.
