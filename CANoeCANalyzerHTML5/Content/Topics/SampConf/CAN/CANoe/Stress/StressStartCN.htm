Stress [▲ back]

Bus traffic is displayed on the Stress desktop using the Graphics Window, the Trace Window and panels.

The Display panel displays the current cycle times of individual messages. In addition, the HiPrStress1 (txRq) time is displayed for the HiPreStress1 message, to which the transmission request was allocated to the controller. Difference measurements between the transmission request and the transmission time are possible.

You can start and stop logging to the CAN bus 1 and CAN bus 2 with pre- an post-triggering with the Logging control panel to record all bus traffic on the respective bus.

The Graphics Window displays the message cycle times exactly like the Display panel.

The Trace Window records the bus traffic.

You will find diverse system information on the various tabs in the Write Window.

Jitter [▲ back]

The Jitter desktop displays the manipulation of a message's cycle time.

After the measurement starts, the CycleAndResponse CAPL node sends the CyclicMessage cyclical message using timers.

Using the slide control on the Jitter panel, you can manipulate the time response of the CyclicSending network node at the runtime, which means you can manipulate the network node cycle time fluctuations.

An Interactive Generator (IG) is opened from the Jitter desktop. To increase the busload, activate the check boxes of the HiPrStress2, HiPrStress3, Stress1, Stress3 disabled messages. You can view the cycle times changes very well in the Display panel and the Graphics Window.

Find above (key combinations) additional information about stimulating the bus.

The minimum, maximum and current cycle times of the CyclicMessage message are displayed in the Cycle Time Graphics Window and the Jitter data for cyclic message Data Window.

Latency [▲ back]

CycleAndResponse CAPL node latency time is illustrated in the Latency desktop.

For this purpose, the CAPL node receives the CyclicMessage message and immediately sends the ResponseMessage message. The time difference (latency) is determined using both time stamps.

The message time stamp and the delay is displayed in the Latency time measurement panel. The display can be shown in various time units.

Bus Statistics [▲ back]

The bus statistics are displayed in the Graphics, Data and Statistics windows in the Bus Statistics desktop. The bus properties displayed are peak load, busload, sent messages, sent Error Frames, etc.

Statistical analyses of messages are displayed in the Statistics Window as bar graphs.

Setup [▲ back]

The Measurement and Simulation Setup are opened from the Setup desktop.

The Measurement Setup includes CAPL nodes. The AnlyzCycles CAPL nodes analyze the message cycle times. The actual cycle time is calculated from the time stamp difference between the currently sent message at time t and the message at time (t-1).

The calculated cycle times are displayed in the Display panel. They are forwarded to system variables in the Panel CAPL node.

The calculated cycle time of the AnlyzCycles CAPL node are also displayed in the Graphics Window.

Channel and Message Filters are used to filter message information.

CAPL nodes and an Interactive Generator (IG) are included in the Simulation Setup.

The IG can apply messages to the bus by activating the check box for unsent messages. The busload can be greatly increased this way.

The Stress CAPL nodes generate automatic bus traffic on both buses after the measurement starts. The bus can also be stimulated by using the available key combinations.

The CyclicSending CAPL nodes calculate the time difference (latency) between an incoming message and a message sent immediately thereafter, see Latency desktop.
