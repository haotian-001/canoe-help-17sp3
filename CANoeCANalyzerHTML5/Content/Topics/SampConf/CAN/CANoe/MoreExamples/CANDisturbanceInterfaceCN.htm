Disturbance Interface

 

Click on the blue link to load the sample configuration in CANoe.

  • Disturbance Interface – CANDisturbance.cfg
    Demonstrates the CAPL functionality of a CAN Disturbance Interface for various triggers and output sequences.

Introduction

The sample configuration contains a desktop for each basic functionality of a CAN Disturbance Interface. In order to be able to use the sample configuration, you need Vector hardware that provides this functionality. On the various desktops, you can configure various triggers and output sequences during measurement and observe the corresponding effect on the CAN bus.

Configuration

 

Configuration Panel [▲ back]

This desktop serves to demonstrate a quick introduction to the topic CAN Disturbance Interface. Settings you make during the measurement are retained for the time being after the measurement has been stopped, but they cannot be saved in the configuration. When you close the configuration, the settings are reset. You cannot generate a CAPL program and save settings permanently with this desktop.

You make the settings via the MainConfigPanel panel. The panel only works when the measurement is started, since a CAPL program processes the user input in the background. Using the Interactive Generator Frame Sender you can quickly test the trigger condition that has been configured.

MainConfigPanel

Trigger
In this area you configure a frame trigger and a raw sequence. As soon as a frame is transmitted on the bus which corresponds to the trigger condition, the raw sequence is output. The output is at the defined trigger position. The Trigger Configuration button opens the TriggerConfigurationPanel where you can configure the trigger.
In the Repetition Configuration area, you can configure the repetitions using the input fields. With these repetitions it is possible to trigger the trigger several times. The default value is 1 for a repetition.

Sequence
The Sequence Configuration button opens the SequencePanel panel in which you can configure the raw sequence. The Configured Sequence field displays the result of these settings.

Enable On Device
Use this button to activate the trigger on the connected device.

Configuration Status
This field indicates whether the settings made represent a valid configuration for the CAN Disturbance Interface. The color green means valid configuration.

Disturbance Interface Channel Information
This area displays the current channel information. You can also adjust the Device ID there if several devices are used.

Disturbance Interface State
In this area on top the three possible disturbance states Idle, Active, Triggerd are displayed. Below the repetitions (Repetition, Cycle) and the Source are displayed.

Frame Trigger[▲ back]

Frame Trigger

With this panel you configure a frame trigger, which outputs a raw sequence or an entire frame as soon as a frame that meets the trigger condition is sent on the bus. It is output at a specified trigger position.

Output: Raw Sequence

Disturb Ack Slot Once
You can use this button to configure a frame trigger for the CAN frame with ID 0x100. The specified trigger position is the CRC delimiter of the frame. The raw sequence that is output is one bit long and disturbs the Ack Slot by sending a recessive level instead of the dominant level.

Disturb Ack Delimiter 32 Times
You can use this button to configure a frame trigger for the CAN frame with extended ID 0x100. The specified trigger position is the Ack Slot of the frame. The length of the output raw sequence is one bit long and disturbs the Ack delimiter by sending a dominant level instead of the recessive level.

Output: Frame Sequence

Start Sending Frame in 3. IFS Once
You can use this button to configure a frame trigger for the CAN frame with ID 0x100. The specified trigger position is the second bit of the Inter Frame Space (IFS). The frame sequence that is output corresponds to a CAN frame with ID 0x200 and a DLC of 0. Within the Statistics Window, the Bursts counter increases, since the SOF of the frame sequence is in the third IFS.

Start Sending Frame in 3. IFS Multiple
You can use this button to configure a frame trigger for the CAN frame with extended ID 0x100. The specified trigger position is the second bit of the IFS. The frame sequence that is output corresponds to a CAN frame with extended ID 0x200 and a DLC of 0. Within the Statistics Window, the Bursts counter increases, since the SOF of the frame sequence is in the third IFS.

Trigger enable configuration

Via the TriggerEnable list you specify what level the DIN0 or DIN1 of the CAN Disturbance Interface must have for the frame trigger to be activated. In order to be able to use the TriggerEnable function, you must connect the DIN0 or DIN1 externally to a power source.

Disturb Ack Delimiter 32 Times
You can use this button to configure a frame trigger for the CAN frame with extended ID 0x100. The specified trigger position is the Ack Slot of the frame. The length of the output raw sequence is one bit long and disturbs the Ack delimiter by sending a dominant level instead of the recessive level.

Start Sending Frame in 3. IFS Multiple
You can use this button to configure a frame trigger for the CAN frame with extended ID 0x100. The specified trigger position is the second bit of the IFS. The frame sequence that is output corresponds to a CAN frame with extended ID 0x200 and a DLC of 0. Within the Statistics Window, the Bursts counter increases, since the SOF of the frame sequence is in the third IFS.

CAN disturbance interface state

Indicates the current state of the trigger:

  • Idle: no trigger configured
  • Active: Trigger active
  • Triggered: Trigger has been executed

Furthermore, the number of repetitions (Count), the number of cycles, (Cycle) and the trigger source (Source) are displayed.

CAN Bus Control

You can use the Reset CAN Bus to revert the settings for the CAN bus.

Combined Frame Trigger[▲ back]

Combined Frame Trigger

Via this panel you configure combined frame triggers, which consist of multiple frames that can cause the trigger to activate. As soon as one of the frames on the bus meets the trigger condition, a raw sequence or entire frame is output. It is output at a specified trigger position.

Output: Raw Sequence

Disturb Data Byte 1 Bit 4 Once
Via this button you configure a frame trigger that applies to the CAN frame with ID 0x100 and 0x200. If one of these frames is transmitted on the bus, the trigger is activated. The specified trigger position is the fifth bit in data byte 1. The raw sequence that is output is one bit long and disturbs the fourth bit by sending a recessive level instead of the dominant level. If a CAN FD frame is sent, the trigger is not activated, since this trigger is only configured for CAN frames.

Disturb Data Byte 1 Bit 4 Multiple
Via this button you configure a frame trigger that applies to the CAN frame with extended ID 0x100 and 0x200. If one of these frames is transmitted on the bus, the trigger is activated. The specified trigger position is the fifth bit in data byte 1. The raw sequence that is output is one bit long and disturbs the fourth bit by sending a recessive level instead of the dominant level. If a CAN FD frame is sent, the trigger is not activated, since this trigger is only configured for CAN frames.

Output: Frame Sequence

Start Sending Frame in 3. IFS Once
You can use this button to configure a frame trigger that applies to the CAN frame with ID 0x100 and 0x200. If one of these frames is transmitted on the bus, the trigger is activated. The specified trigger position is the second bit of the IFS. The frame sequence that is output corresponds to a CAN frame with ID 0x300 and a DLC of 0. Within the Statistics Window, the Bursts counter increases, since the SOF of the frame sequence is in the third IFS.

Start Sending Frame in 3. IFS Multiple
You can use this button to configure a frame trigger that applies to the CAN frame with extended ID 0x100 and the CAN FD frame with extended ID 0x200. If one of these frames is transmitted on the bus, the trigger is activated. The specified trigger position is the second bit of the IFS. The frame sequence that is output corresponds to a CAN frame with Extend ID 0x300 and a DLC of 0. Within the Statistics Window, the Bursts counter increases, since the SOF of the frame sequence is in the third IFS.

External Trigger[▲ back]

External Trigger

Via this panel you configure an external trigger. As soon as the condition is met on DIN0 or DIN1, the raw sequence or a complete frame is output. To use the external trigger, an external voltage source is required, which must be connected to DIN0 or DIN1 correspondingly.

External Trigger Configuration

DIN Channel
Via this list you configure which digital input (DIN channel) the trigger should be configured for.

DIN State
Via this list you configure which state the DIN channel must have in order to be fulfilled.

Output: Raw Sequence

Send One Dominant Bit Once
Via this button you configure an external trigger corresponding to the settings within the External Trigger Configuration list. When the trigger condition is met, a dominant bit is output with a length of 2 µs. This leads to a Stuff Bit Error, since a CAN interfaces recognizes this as an SOF, but nothing further is sent.

Send One Dominant Bit Multiple
Via this button you configure an external trigger corresponding to the settings within the External Trigger Configuration list. When the trigger condition is met, a dominant bit is output with a length of 2 µs. Two Stuff-Bit Error Frames are displayed in the trace, since a CAN interface recognizes this as an SOF, but nothing further is sent, and the repetition is set to 2.

Send Frame Once
Via this button you configure an external trigger corresponding to the settings within the External Trigger Configuration list. If this trigger condition is met, the frame sequence is output that represents a CAN frame with ID 0x100 with DLC = 0.

Send Frame Multiple
Via this button you configure an external trigger corresponding to the settings within the External Trigger Configuration list. If the trigger condition is met, the frame sequence is output 10 times. Here the frame sequence represents a CAN frame with ID 0x100 with DLC = 0.

CAN disturbance interface state

Indicates the current state of the trigger:

Direction Trigger

This desktop contains the Direction Trigger panel. This panel allows you to configure a direction dependent frame trigger on a CAN Disturbance Interface.

The configured frame trigger disturbed a frame sent on the bus, if the transmitter direction corresponds to the configured one and the trigger condition is full-filled.

In this case the output of the configured raw sequence starts at the defined trigger position.

Output: Raw Sequence

TriggerDirection

Via this list you configure the send direction that has to be considered.
Possible values are Both, Tx and Rx.

Disturb Ack Delimiter 32 Times
You can use this button to configure a frame trigger for the CAN frame with extended ID 0x100. The specified trigger position is the Ack Slot of the frame. The length of the output raw sequence is one bit long and disturbs the Ack delimiter by sending a dominant level instead of the recessive level.

Inverted Trigger

This desktop contains the Inverted Trigger panel. This panel configures an inverted frame trigger on a CAN Disturbance Interface.

The configured frame trigger is fulfilled if a frame on the bus does not meet with the trigger condition.

In this case the output of the configured raw sequence starts at the defined trigger position.

Output: Raw Sequence

InvertedTrigger

Via this list you configure, if the trigger is inverted or not.
Possible values are NotInverted and Inverted.

Disturb Ack Delimiter 32 Times
You can use this button to configure a frame trigger for the CAN frame with extended ID 0x100. The specified trigger position is the Ack Slot of the frame. The length of the output raw sequence is one bit long and disturbs the Ack delimiter by sending a dominant level instead of the recessive level.

If the InvertedTrigger list is set to Inverted, the frame with the extended ID 0x100 will not be disturbed, but all other frames.

Missing Bit Trigger [▲ back]

Missing Bit Trigger

This desktop contains the Missing Bit Trigger panel. This panel allows you to configure a frame trigger on a CAN Disturbance Interface.

The configured frame trigger disturbed a frame sent on the bus, if:

  • the trigger condition is fulfilled up to the defined trigger position
  • the trigger condition is fulfilled, but the trigger-position isn’t reached

In this case the output of the configured raw sequence starts at the defined trigger position, if the trigger position is reached. Otherwise the trigger position isn‘t reached, the output of the raw sequence starts directly or with the next bit.

Output: Raw Sequence

MissingBitTrigger

Via this list you configure, if the trigger should also trigger if the trigger position is not in the frame. Possible values are Disabled and Enabled.

Disturb CRC Delimiter 32 Times

You can use this button to configure a frame trigger for the CAN Frame with the extended ID 0x100. The specified trigger position is the CRC of the frame. The raw sequence that is output is one bit long and disturbs the CRC Delimiter by sending a dominant level instead of the recessive level.

If the MissingBitTrigger list is set to Enabled, the frame with the extended ID 0x100 will be disturbed, otherwise not. This is because the offset within the CRC field has been set to 21. This position is not reached in a CAN frame. However, if triggering is switched on even though the trigger position has not been reached, the output occurs after the last bit that was before the trigger position.

CAN Bus Control

(E.g. if the CAN bus has switched to the Bus-Off stage).
