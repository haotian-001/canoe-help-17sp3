CANstress

 

Click on the blue link to load the sample configuration in CANoe.

  • CANstress – CANstress.cfg
    Demonstrates how CANstress can be used together with CANoe to test a control unit.

To execute the configuration, a CANstress hardware (CANstress D or CANstress DR) must be connected to the computer and the CANstress software (version 2.1.48 or later) must be installed.

Using an example, an actual ECU will be simulated using the Engine ECU nodes.

The CANstress hardware is controlled by the CANstress test node with the help of CAPL calls. To do this, a CANstress COM server is first created in CAPL, via which further diverse CANstress CAPL commands are called up such as to link with the hardware or to load a configuration.

In the actual test, the test node CANstress requests the (simulated) ECU to send a data frame through a remote frame.

The data frame sent from the (simulated) ECU is subsequently interrupted by CANstress. In the following attempt to correctly send the interrupted data frame, CANstress interrupts the ECU again.

This repeats until the (simulated) ECU bus-off is initiated.

The number of Error Frames generated is recorded by the CANoe bus statistic. This will be used at the end of the test to check whether the test ran successfully or not.

Note

The configuration can be also implemented in combination with a VN8900 or a realtime module (VT6000 family).

In this case the CANstress hardware must be connected with a VN8900 or with a realtime module (VT6000 family).
