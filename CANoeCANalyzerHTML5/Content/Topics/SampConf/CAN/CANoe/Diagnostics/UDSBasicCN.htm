UDS Diagnostic Basic

 

Click on the blue link to load the sample configuration in CANoe.

  • UDS Diagnostic Basic – UDSBasic.cfg
    UDS based simple ECU simulation and diagnostic test.

Introduction

This sample configuration shows the following use cases:

  • Writing a diagnostic parameter (physical & symbolical representation).
  • Writing a raw/large diagnostic parameter.
  • Reading a diagnostic parameter (physical, numerical, coded and symbolical representation).
  • Reading a raw/large diagnostic parameter.
  • Executing a service which gets a negative response.
  • Performing a simple diagnostic test module.
  • Simulating simple diagnostic functionality of an ECU.

The sample configuration consists of a simulated ECU implementing simple diagnostic functionality (node SimDiagEcu), a simulation node implementing simple interactive test functionality based on keys pressed by the user (node Tester) and a simple automated test within a CAPL test module (TestModule).

After starting the measurement, the possible actions are shown in the Write Window. You may press the keys 1 to 5 to send the corresponding diagnostic request to the simulated ECU. You can see the diagnostic request(s) in the Trace Window, as well as the diagnostic response(s) of the simulated ECU. In the Write Window the results of the respective action and the read parameter values are shown.

Desktops

Trace [▲ back]

In this desktop you find the Write-Window, the Test Module and the Trace-Window.

In the Trace Window, you can see two types of events:

  • ISO TP Observer events interpreting the segmented diagnostic frames on CAN (indicated by <OTP>).
  • Diagnostic Observer events interpreting the payload of the OSEK TP transport protocol (event types req, pos and neg). The requests are colored black, while the positive and negative responses are colored green and red, respectively.

In this desktop you find the Simulation Setup with the nodes Tester, TestModule and SimDiagECU. Further there is the Measurement Setup and also a Write Window.
