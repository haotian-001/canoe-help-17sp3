K-Line

 

Click on the blue link to load the sample configuration in CANoe.

  • K-Line – KLineDemo.cfg
    Demonstrates how a K-Line tester and ECU can be simulated.

Requirements

  • CANoe 8.0 SP4 or higher.
  • K-Line capable VN interface or interfaces.
  • Two K-Line channels looped with a CANcable0.
  • Actual drivers for VN interfaces.

General

  • The sample configuration shows how a K-Line tester and ECU can be simulated.
  • The K-Line tester is configured and uses the interactive diagnostic console to send a service.
  • The ECU simulation is programmed in CAPL. See the CAPL program of the node K-LineECU in the Simulation Setup.
  • A K-Line ECU simulation requires the ECU configured in the diagnostics configuration dialog.
  • On a K-Line channel you can simulate either a tester or an ECU, but not both together.

Introduction

Start the measurement and send a diagnostics service e.g. ReadEcuIdentification via the diagnostic console. Watch the answers from the ECU in the diagnostic console or analyze the K-Line communication in the Trace Window.

K-Line Parameters [▲ back]

In this desktop you find several panels:

K-Line ECU Settings

Header Format
In this list you select between a OneByteHeader to a FourByteHeader.
DefaultHeader is the header setting that is negotiated with the StartCommunciation service.

Baudrate
You can enter the following baudrates in the input box: 10400, 19200, 38400, 57600.

After a stopped communication (StopCommunication service) the baudrate is reset to the K-Line default of 10400 which is required for a K-Line fast initialization.

P1_Time
The inter byte spaces of a K-line ECU response you can set between 0 and 15 ms. Check the value in the Trace Window (column: interbyte space).

Response Behavior

There are two possibilities to influence a simulated response:

  1. Patched Response
    Activates the CAPL Pre_Send handler. Before a response is transmitted it can be changed in the Pre_Send handler.
    In this example the checksum byte is changed (see CAPL program KLineECU.CAN).
  2. Raw Response
    Activates sending a K-Line message as a byte stream. A response can be defined as a pure byte stream that can be sent instead of the default response.
    In this example the response is changed to a response containing more bytes than the default response (see CAPL program KLineECU.CAN).
  3. Default
    Resets the response behavior.

Status Indication Communication Layer ECU

Shows the notification of:

  • Fast Init Pattern Indicated
  • Byte Notification: received and transmitted bytes
  • Frame Notification: received and transmitted frames
