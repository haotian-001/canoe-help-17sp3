CAN XL Basic (CAN)

 

Click on the blue link to load the sample configuration in CANoe.

    Shows the use of the CAN-XL protocol by means of an audio stream.

Introduction

The sample configuration can be used in simulated mode as well as in real mode. To use the sample configuration in real mode, a VN1670 with current CAN-XL driver is required. The MediaClient panel is used to start the audio stream. Here the audio file is played back on the sound card.

Desktops

This configuration contains the following desktops:

MediaServer

This panel displays the information determined by the server for the audio file. For the transmission, after starting the audio stream by the MediaClient, the period of the frame as well as the user data length is displayed. The server can be configured for CAN XL and CAN FD. If CAN FD is selected, the audio stream is not output correctly on the sound card. This shows that CAN FD is not sufficient in this case.

MediaClient

This panel allows you to start and stop the audio stream. The time indicator shows how much of the audio file has been played. When the audio stream is stopped and restarted, the transmission starts again.

Data Window

Here you can see the current transmission rate in bytes per second and the transmitted data volume in Mbyte.
