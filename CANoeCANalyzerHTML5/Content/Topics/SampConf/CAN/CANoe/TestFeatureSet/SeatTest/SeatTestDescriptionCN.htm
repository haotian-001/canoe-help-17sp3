Seat Test

 

Click on the blue link to load the sample configuration in CANoe.

  • Seat Test – SeatTest.cfg
    Demonstrates how the functionality of a seat and the communication of a seat ECU can be tested. The test is realized with test tables and CAPL test functions in vTESTstudio.

There are buttons at the top of the execution dialog for operating the test configuration:

After starting the test configuration, you should enter your personal information in the displayed dialog. Please fill out the Value and Comments fields. These entries apply to all subsequent test runs during the actual measurement and are therefore only requested at the very start of the test configuration.

Various test runs are then started depending on the seat variant set. You can select the variant of the seat to be tested at the bottom of the execution dialog, apply it and then start the test configuration. The test run is displayed in the execution dialog and in the Test Trace Window during execution.

1. Check Functions

In the enclosing Seat Tests test fixture, check functions are started at the outset. These functions are responsible for monitoring the specified cyclical messages during execution, making it possible to identify errors at any point.

2. Functionality test

Depending on the seat variant, various test cases are performed. You can set the seat variant at the bottom of the execution dialog.

Selection boxes can be used to affect the functionality test:

  • Increasing the resistance of an axis, making the seat adjustment functions run slower.
  • The cyclical message of the seat can also be changed. If the error is set, every second message is sent only. The check functions provide the result in the protocol.

Both test runs lead to different results depending on the seat parameter setting on the panel.

The results are written to a test report that can be opened via the respective button in the execution dialog.
