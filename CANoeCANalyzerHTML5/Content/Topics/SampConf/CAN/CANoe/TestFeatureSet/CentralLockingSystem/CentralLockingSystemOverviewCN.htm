Central Locking System

 

Click on the blue link to load the sample configuration in CANoe.

  • Central Locking System – CentralLockingSystem.cfg
    Test a vehicle's central locking system and the power window in vTESTstudio. All available test design editors are used (Test Table Editor, CAPL Editor, C# Editor, Test Diagram Editor and State Diagram Editor).

The sample configuration Central Locking System demonstrates following features of CANoe TestFeatureSet:

  • Overview of all test design editors in vTESTstudio
  • Test setup
  • Test execution dialog
  • Test report

System under test (SUT) implements the functions of:

  • The central locking system
  • The window lift

All of the connected sensors and actors are abstracted and communicate via the bus with the SUT.
In real, they are implemented by other nodes, and also communicate with the SUT via the bus.

Note

  • There is a COM sample (TestSetup.vbs) to describe the access to the Test Setup (TestSetup (COM Object) object) using external programs.
  • The integration of an additional style sheet enables all images used in the test report to be embedded in the HTML page itself. More information and instructions on the procedure are available on the following page: Test Report Styles - Embedded Images.
  • The Test Report Output Format page provides an overview of different output formats.
    The Test Report Styles - Embedded Images page provides further information and instructions on embedding images contained in the test report.

Testing [▲ back]

The Testing desktop consists of a Panel, a test configuration and a Test Trace Window.

Sensors and actors can be operated on the Panel System under Test. Additionally it displays state changes.

Following actions are possible:

  • Central locking system (CLS):
  • Lock, unlock, comfort-close (remotely by the key-remote-control), where the "comfort-close" locks the car and closes the windows.
  • If the velocity of 50 kmph is reached, then the CLS locks automatically.
  • If a crash occurs (while the engine is running, or while the vehicle moves, then the CLS unlocks the car.
  • The window lift supports opening and closing functions.
  • Start/stop engine
  • Accelerate/brake the vehicle (Before start engine).
  • It is possible to inject several typical ECU errors, that have happened either on architectural-, on design- or on implementation-  level.  If you operate the error-injection-switches, then some test case will fail. The errors do not effect on panel operations.

The test configuration Door Control Unit contains multiple test units. The detailed test cases for the central locking station and the window lift can be seen within the test units in the execution dialog.

Tests can be started as follows:

  1. Start the test configuration. This can be done best by using the Start button in the Door Control Unit execution dialog.
  2. Inject some errors, then execute the test configuration, or only some of the test units or test cases by activating or deactivating single elements within the execution dialog.
  3. Operate the SUT manually using the System under Test Panel. The test configuration is not used in that case.

After the tests are done, then you can have a look onto the generated report by clicking onto the corresponding Report button within the test configuration execution dialog.

During test execution all actions of the executed test configuration are shown in the Test Trace Window.

Simulation [▲ back]

On the Simulation desktop the Simulation Setup is opened and contains following nodes:

The node SUT contains the functions of the central locking station and the window lift. Additionally the node SUT implements a few diagnostic services that will be used during the test run.

The node SUT Driver implements a minimal dynamic model where the vehicle can be accelerated and decelerated. Additionally it is able to invoke a few diagnostic services that are controlled by the test units.

The Trace Window reports the bus traffic.
