Test Report Styles - Embedded Images

 

CANoe Test Report Viewer Format

If you selected the recommended Test Report Viewer Format in the global CANoe options, always all images of the test report will be embedded in the report file. This includes all images dynamically generated via WindowCapture (XML function or CAPL function) and explicitly referenced in tzhe test module, as long as they are available as local file during test execution. Images referenced via HTTP URL will not be embedded.

XML/HTML Format

If you selected the XML/HTML test report format in the global CANoe options, you can use the extensions in this directory to generate HTML report with embedded images instead of separate image files.

Open folder

The image files generated, if necessary, by CANoe with WindowCapture are kept but their content is embedded in the HTML file so that this file can be forwarded directly without having to copy the image files too.

All images linked in the test report are embedded, thus not only images generated dynamically via the WindowCapture (XML test pattern or CAPL function) but also images explicitly referenced in the test module.

All images that exist as a local file at the time of the transformation are embedded. Images referenced via http URL are not embedded. Also excluded from the embedding are images in bitmap format (BMP) because these are generally too large to be processed during the transformation to HTML. Use a different format, e.g., PNG, instead.

The embedding can be activated in the same way for each style sheet. See description below. This is possible for test reports of test modules as well as test reports of test units.

Procedure

Copy the following files to directory exec32\report (for test module reports) or exec32\testunit_report (for test unit test reports) of the CANoe installation:

  • embed-images.bat
  • restore-original.bat
  • InlineReportImages.exe

Copy the following lines into the desired style sheet in the same directory:

<!-- EXECUTE PRE embed-images.bat -->
<!-- EXECUTE POST restore-original.bat -->

Insert the lines directly after the existing copyright block.

Note: These lines must not have leading blank spaces!

In the case of multipage.XSLT, a batch file is already called with EXECUTE PRE. In this case, the line with EXECUTE PRE must be kept as is. Instead, insert the following line at the end in the batch file specified there (e.g. multipage.BAT):

call embed-images.bat

If an error occurs when embedding images, the test run stops in status Transforming test report... and a window with an error message appears and must be acknowledged.

If you want to archive the XML test report with embedded images, you can omit the linking of the restore-original.BAT in the style sheet. In this case, the XML test reports are kept with and without images. The test report without pictures is renamed as *_original.XML.
