Sample Configurations Diagnostics

 

Introduction

Most ECUs provide diagnostic services, e.g. for reading fault memory, queryings and changing settings. For this purpose, diagnostic data is exchanged between a Tester (Client)and the ECU (Server) according to the Keyword Protocol 2000 (KWP2000; ISO 15765-3, ISO 14230-3) or Unified Diagnostic Services (UDS; ISO 14229-1).

This example demonstrates the main diagnostic procedures for:

  • querying the ECU serial number and identification
  • querying detected errors (Diagnostic Trouble Codes, DTC)
  • deleting of the fault memory
  • querying dynamic data such as battery voltage
  • querying a continually changing application value
  • querying changes in the controller state
  • controlling of ECU session (session management)
  • unlocking of ECU (security access)

Targets of the Sample Configurations

  • To show how CANdelaStudio descriptions (CDD files) can be used to analyze diagnostic data in the Trace Window, but also how to parameterize the standard consoles and how to access diagnostic parameters in CAPL programs.
  • To give an exemplary implementation of a Tester and a ECU with the aid of extended diagnostic features in CAPL.
  • To give an exemplary implementation of a test module.
