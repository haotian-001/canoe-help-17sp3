System Configuration (SOA)

 

Click on the blue link to load the sample configuration in CANoe.

  • System Configuration (SOA) – SOASystem.cfg
    Shows the simulation and testing of a service-oriented architecture (SOA) for vehicles.

The considered vehicle architecture consists of the following services:

  • Object_Recognition
    This service supplies information on objects detected in the vehicle's surroundings. Service providers are the LIDAR_Front and LIDAR_Rear communication participants. The service consumer is the communication participant Drive_Master.
  • Navigation
    This service supplies navigation data at a given point in time. The service provider is the Backend communication participant. The service consumer is Drive_Master.
  • Passenger_Information
    This service displays information on the vehicle's surroundings and navigation for the vehicle passengers. The service provider is the Dashboard communication participant; the service consumer is the Drive_Master communication participant.
  • Mirror
    This service influences the position of the exterior mirror and can switch the mirror heating on or off. Service providers are the communication participants Mirror_Driver and Mirror_Passenger. Service Consumer is the communication participant Console. In the sample configuration only the communication between the participant Mirror_Driver and the participant Console is shown.

All communication participants are simulated in CANoe. The simulation occurs directly on the service level; underlying network layers are not simulated.

Introduction

The measurement can be started directly. The Simulation desktop contains panels for the visualization and interactive stimulation of the communication participants as well as an automated test module.

The Analysis desktop contains the Trace Window, which displays the communication on the service level. The Setup contains the Communication Setup.

Simulation [▲ back]

The Dashboard panel visualizes the values received from the Dashboard communication participant for the vehicle surroundings and the navigation data.

The LIDAR panel enables the interactive stimulation of the detected objects in the front and rear vehicle surroundings.

The Backend panel enables the interactive stimulation of the navigation data transmitted by the Backend.

The Tester test module performs an automated test. Stimulation takes place here on the LIDAR_Front, LIDAR_Rear and Backend communication participants with test data. A check is performed to determine whether this test data is correctly received by the Dashboard communication participant.

The Mirror panel allows you to set and send the values of the AdjustMirror function. The changed mirror position and the mirror heating are displayed in the same panel. Valid values for the function are also indicated on the panel.

Analysis [▲ back]

The Trace Window records the communication on the service level. Here, you can follow the method calls as well as transmitted events between the communication participants.

Setup [▲ back]

The Communion Setup contains all configuration settings for the given configuration. Configured there are, in particular, the used services and application models.
