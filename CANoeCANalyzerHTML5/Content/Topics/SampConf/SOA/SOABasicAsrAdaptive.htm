Basic AUTOSAR Adaptive (SOA)

 

Click on the blue link to load the sample configuration in CANoe.

  • Basic AUTOSAR Adaptive (SOA) – SOABasicAsrAdaptive.cfg
    Shows the basic functions for the simulation and analysis of service-oriented architecture (SOA).

The sample configuration is based on an AUTOSAR Adaptive database. This describes a Calculator service interface with the communication participants Consumer and Provider.

The Calculator service interface contains four methods, that correspond to the four basic arithmetic operations. In addition, it contains an event that transfers the number of calls made for each of the four methods.

The Consumer communication participant uses the Calculator service interface. The Consumer calls the methods and receives the event.

The Provider communication participant provides the Calculator service interface. The Provider implements the four methods and sends the event.

All communication participants are simulated in CANoe. The simulation takes place at the service and SOME/IP protocol level.

Introduction

You can start the measurement directly.

On the Simulation you can stimulate the Consumer communication participant interactively. You can watch the communication on the Analysis desktop. On the Setup desktop you can modify the configuration.

Desktops

This configuration contains the following desktops:

Simulation

Analysis

Setup

 

Simulation [▲ back]

Via the Consumer panel you can stimulate the Consumer communication participant.

Via the separated controls you can call the four methods of the Calculator service interface. Via the Request page you can set the In-Paramenters. The method call you can start with the Send button. The result is shown on the Response page.

Via the Trace Window you can watch the communication on service level. The method calls and transmitted events between Consumer and Provider are displayed there.

Analysis [▲ back]

The individual elements of the State event are visualized in a Graphics Window and a Data Window.

Setup [▲ back]

The Communication Setup contains the central configuration settings for this configuration.

The AUTOSAR Adaptive database is configured in the Data Sources area.

In the Available Elements area are further configured the communication participants Consumer and Provider with the state Simulated and the corresponding application models.

In the application models, the call of methods and the implementation of methods are demonstrated. Additionally the sending and receiving of events is shown. The application models are implemented in C#. Equivalent CAPL models are also included as inactive application models.
