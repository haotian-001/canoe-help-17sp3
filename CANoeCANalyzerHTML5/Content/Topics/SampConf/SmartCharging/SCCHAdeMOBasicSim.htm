CHAdeMO Basic Simulation

 

Click on the blue link to load the sample configuration in CANoe.

  • CHAdeMO Basic Simulation - CHAdeMOBasicDO.cfg
    Demonstrates the communication between an electric vehicle (EV) and a charging station (EVSE) according to the CHAdeMO protocol.

Note

Some functions of the protocol may not be supported yet. For an overview of the feature set, refer to the help page CHAdeMO Overview.

Introduction

The configuration presents the main features of the Option SmartCharging for the CHAdeMO protocol by combining a vehicle simulation with a charging station simulation. In the communication setup you will find the following application models:

  • EVSEsimDO.can: Provides data for the charging station simulation.

  • EVsimDO.can: Provides data for the vehicle simulation.

  • Monitor.can: Matches the set protocol versions to ensure compatibility.

  • VT_HW_Access_EV.can: Hardware access according to CHAdeMO 2.0 for the vehicle simulation.

  • VT_HW_Access_EVSE.can: Hardware access according to CHAdeMO 2.0 for the charging station simulation.

In these application models and the panels of the Simulation Desktop, it is also shown how easy the manipulation of messages can be realized using DOs.

Desktops

This configuration contains the following desktops:

Simulation Power Monitor Configuration VT System

Simulation

On the desktop Simulation you can observe the communication of a single vehicle with the charging station. The behavior of the participants can be influenced via the corresponding panels. To start the communication, simply click on the Connect vehicle button. The simulation runs automatically until the end of the communication.

In the Trace Window, the SmartCharging messages (CAN messages) are displayed in decoded form with data interpretation.

EV Panel

This panel serves as an operator control and display element for the vehicle simulation module. The following operator controls are available:

Operator Control

Description

DC tab

In the DC tab, the charging voltage requested by the vehicle can be set.

Limits tab

In the Limits tab, the maximum and minimum battery charging voltage supported by the vehicle, as well as the current for trickle charging can be specified.

Battery

Displays the current state of charge of the traction battery. It can be reset at any time with [Clear].

Reset buttom

Resets the status machine of the vehicle. The charging process is not terminated in the regular way.

Stop button

Ends the charging process in a regular way by the vehicle (normal stop).

Connector (plug) button

Connects or disconnects the vehicle from the charging station. As soon as the button is pressed, charging communication starts according to the set CHAdeMO protocol. If the button is released during charging, the simulation stops and is reset.

Status/Error

Via this field it is possible to observe and change live status signals during the charging process.

Protocol Version

Set here the protocol to be used for charging communication. If you select version 3.0, this will also be used for the charging station.

EVSE Panel

This Panel is used to control and configure the charging station. The following controls are available:

Operator Control

Description

Return FAILED

Causes the charging station to send a ChargerError.

Abort Conn.

Simulates a disconnection of the CAN bus in Real Bus mode. Has no function in simulated mode.

Status/Error

Via this field it is possible to observe and change live status signals during the charging process.

Reset button

Resets the status machine of the charging station. The charging process is not terminated in a regular way.

Stop button

Ends the charging process in a regular way by the charging station (normal stop). The switch is only active during the charging process (State E).

Power Output tab

Shows the current and voltage at the output of the charging station.

Limits tab

Here you can set the maximum charging current and the maximum charging voltage of the charging station.

Protocol Ver. tab

Set the protocol to be used for charging communication here. If you select version 3.0, this will also be used for the vehicle.

Protocol Panel

In this Panel, the progress in the communication protocol is displayed via a red bar.

Power Monitor

In the Power Monitor, the current values of the charging voltage and charging current of the vehicle and charging station are displayed graphically over time.

Configuration

This desktop contains the basic configuration windows. Here you have access to the application models.

VT System

Contains panels for measurement and control of the VT2816 and VT2004 modules. The control of the switches as well as the setting of the resistors is done in the individual CAPL nodes. The hardware and controls support communication for CHAdeMO 3.0, CHAdeMO 2.0 and lower versions. The PE line must be connected to the GND of the VT2816 so that the same potential is used for all measurements. Otherwise, all inputs marked with PE labels must be wired individually. For more information, refer to the manual of the respective VT2816 and VT2004 modules.

Note

The backplane of the VT2004 cards must be wired to the backplane of the VT system housing to use the 12 V and PE.
