GB/T 27930 Traffic Analysis

 

Click on the blue link to load the sample configuration in CANoe.

  • GB/T 27930 Traffic Analysis – GBT27930TrafficAnalysis.cfg
    Demonstrates the possibilities that CANoe offers for the analysis of GB/T 27930 traffic.

Introduction

Using the J1939 Interpretation column and intelligent filtering allows the entire loading process to be displayed briefly and clearly in the Trace Window.

Desktops

This configuration contains the following desktops:

Setup

Analysis of a logging file:

  1. Activate the Simulated Bus mode (default).
  2. Select the Configuration entry in the context menu of the Replay Block to configure the logging file to be analyzed.
  3. Start the measurement.

Analysis of the network traffic on the real bus:

  1. Activate the Real Bus mode.
  2. Deactivate the Replay Block.
  3. Start the measurement.

Measurement

GB/T 27930 Traffic is displayed in two Trace Windows one below the other. For clarity, both windows hide all transport protocol messages TPCMxx (PGN 0xEC00) and TPDT (PGN 0xEB00). In each of the Trace Windows, the column J1939 Interpretation is configured, which displays the content of each message briefly and clearly.

The typical loading process takes several minutes and contains several thousand messages. All the messages can be seen in the Trace Window Trace full. However, this large amount of data makes it very difficult to analyze the process and identify possible problems. For a better overview, an additional filter node SmartFilter.can is activated before the Trace Window Trace filtered (see Measurement Setup). This node only lets through messages that transmit new information and blocks all messages with constant content. This reduces the number of messages by a factor of 100 or more. In the specific example, the entire loading process is compressed to only 35 messages - without the information content having to be affected.

The state transitions of the Charger and BMS are shown in the StateDiagram panel. After initialization all state fields are gray, the achieved state is colored green. If a state was not reached or reached too late, it is colored red and the transition is marked. Depending on which node finishes the loading process, the diagram is extended by the BST/CST and BSD/CSD messages. The states are reset with each new loading process and measurement start. During the measurement the diagram can also be reset using the Reset button.
