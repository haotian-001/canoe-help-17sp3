OCPP Basic Simulation

 

Click on the blue link to load the sample configuration in CANoe.

  • OCPP Basic Simulation - OCPPBasicDO.cfg
    Demonstrates the communication between a charging station and a charging station management system (CSMS) according to the OCPP protocol.

Introduction

The configuration presents the main features of the Option SmartCharging for the OCPP protocol by combining a charging station simulation with a charging station management system (CSMS) simulation. In the Communication Setup you will find the following application models:

  • ChargingStationSim.can: Provides data for the charging station simulation.

  • CSMSSim.can: Provides data for CSMS simulation.

In these application models and the panels of the Simulation Desktop, it is also shown how the manipulation of messages can be realized using DOs.

Desktops

This configuration contains the following desktops:

Simulation Configuration

Simulation

On the desktop Simulation you can observe the communication of a single charging station with the CSMS. The behavior of the participants can be influenced via the corresponding panels. At the start of measurement, the charging station and the CSMS establish a connection and exchange sample messages.

In the Trace Window, the OCPP messages (websocket messages) are displayed in decoded form with data interpretation.

The Application Panels responsible for the respective participant can be used to send messages.

ChargingStation Panel

This panel serves as an operator control and display element for the charging station simulation module. The following operator controls are available:

Operator Control

Description

Schema

Only OCPP 1.6 can be selected here at the moment.

Start button

Starts the establishment of a websocket connection from the charging station to the CSMS.

Stop button

Closes the websocket connection to the CSMS.

CSMS Panel

This Panel is used to control and configure the CSMS. The following controls are available:

Operator Control

Description

Schema

Only OCPP 1.6 can be selected here at the moment.

Start button

Switches the CSMS into an active state, in which it is ready to accept incoming websocket connections.

Stop button

Closes the WebSocket connection to the charging station. The CSMS can accept further connections afterwards.

Configuration

This desktop contains the basic configuration windows. Here you have access to the application models.
