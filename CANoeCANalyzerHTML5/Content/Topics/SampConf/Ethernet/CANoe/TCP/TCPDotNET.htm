Basic TCP Server and Client (C#/.NET)

 

Click on the blue link to load the sample configuration in CANoe.

  • Basic TCP Server and Client (C#/.NET) – TCPBasicDotNet.cfg
    Shows how TCP based communication can be handled with .NET nodes.

The configuration consists of a TCP server and a TCP client whose communication is represented in a simple way. Once you have connected the server to the client, send a string (request) with upper and lower case letters from the client to the server. When completed successfully, the server automatically returns the converted string in upper case letters (response).

Introduction

  1. Start the measurement.
  2. The Measurement desktop is displayed automatically.
  3. Click the Connect button in the TcpClientPanel to connect the client to the server.
  4. Following a successful connection, the WELCOME message appears in the ServerResponse field.
  5. Click the Send button. This sends the request (text in the DataString field) to the server.
  6. When completed successfully, the response of the server then occurs. The text appears in the ServerResponse field in upper case letters.

Protocol

When the connection is established, the server accepts the client and welcomes it with WELCOME. The client awaits this exact connection establishment. The server then waits for a request from the client and answers it with a response. If the server receives a character string that it is to convert, the server answers this with the response OK, followed by the converted character string as the second response. If the server could not convert the characters, the response NOT CONVERTED is returned.

Semantics of the Server

The server accepts character strings and converts these to a corresponding character string consisting of upper case letters. If the server was unable to convert the character string because the result is identical to the input, the server returns the response NOT CONVERTED.

Error Protection

The following cases are also covered:

  • Sending of the client although the client is not yet connected to the server.
  • Disconnection of the client although the connection was not yet established.
  • Client establishes a connection, and server is terminated.
  • Server is not started, and client attempts to establish a connection.

Note for Implementation

If a socket is closed by a communication partner (network stream close), the partner realizes this in these ways: BeginRead throws an exception if the closing happened before (half-closed socket), or if the connection has taken place after a BeginRead, in the callback of the BeginRead in parameter result via flag result.IsCompleted (false).

Because the implementation under C# is based on base classes of the framework, one should put aside the classic socket view and instead think of a network stream. Both reading and writing are possible using this stream as are implicit inferences about the connection status.

Desktops

This configuration contains the following desktops:

Configuration

Measurement

 

Configuration [▲ back]

This desktop contains the Simulation Setup, the Measurement Setup, the Write Window, and the Start Values Window.

You can use the Start Values Window to set the key data for the server that is to apply at the start of the measurement. The server starts at the start of the measurement as soon as the system variable NSys::TCPServer::AutoStart is set to 1. This is the standard setting.

Measurement [▲ back]

This desktop contains the Trace Window and the Write Window as well as the following two panels.

TcpServerPanel

Start/Stop
You can use this button to start or stop the service.

IP
Shows the currently used IP of the listen socket of the server.

IP
Shows the currently used port of the listen socket of the server.

Started
This LED shows the status of the server:

  • Green = started
  • Red = stopped

TcpClientPanel

Connect
The client is not automatically connected. To establish the connection between the client and server, click the Connect button after the start of the measurement.

Disconnect
Click this button to disconnect the connection between client and server.

Connected
Status of connection:

  • 1: Connected
  • 0: Disconnected

DataString
In this input box you enter the characters that you send to the server with the Send button in order for these to then be processed.

Send
Click on this button to send the characters in the DataString input box to the server.

ServerResponse
This field displays the response of the server:

  • Message WELCOME after successful new connection.
  • Converted text in upper case letters after successful send and response.
  • Message NOT CONVERTED if the text in the DataString field was already entered in upper case letters.
