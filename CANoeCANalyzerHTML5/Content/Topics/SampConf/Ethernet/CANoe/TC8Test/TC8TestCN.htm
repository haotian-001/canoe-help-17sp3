TC8 Test

 

Click on the blue link to load the sample configuration in CANoe.

  • TC8 Test (Ethernet) – EthernetTC8Test.cfg
    Demonstrates multiple TC8 test cases with CANoe and vTESTstudio.

Introduction

TC8 is a test specification created by the OPEN alliance (http://www.opensig.org) to test Automotive Ethernet ECUs. This configuration contains several TC8 test cases, which can be executed with CANoe. The tests have been defined using vTESTstudio. The vTESTstudio project is in the TestSuite directory. The project contains the test case definition and parameterization values for the tests.

Desktops

This configuration contains the following desktops:

Setup

The configuration includes a DUT node, which is a golden device implementation in CANoe for demonstration purposes. The DUT passes most of the TC8 tests, but not all.

Also included is a Logging Block, which logs the bus communication of all failed test cases. The logging files are grouped into one directory and each failed test case generates its own logging file. The logging file can be easily matched to a test case. The filename contains the test case name.

Test

Here you can select the test cases that should be executed and check the results of the test case execution. You can use the Trace-Window to inspect the network communication during the test case execution.
