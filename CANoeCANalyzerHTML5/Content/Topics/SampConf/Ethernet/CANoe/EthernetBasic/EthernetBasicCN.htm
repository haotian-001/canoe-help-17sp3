Basic Analysis (Ethernet)

 

Click on the blue link to load the sample configuration in CANoe.

  • Basic Analysis (Ethernet) – EthernetBasic.cfg
    A logging file is played back in offline mode. You can trace and evaluate the communication in the Trace Window. The Data Window and Graphics Window show the signal evaluation.

Introduction

You can use the context menu of the Trace Window to synchronize it with the Graphics Window. The Graphics Window then shows a measurement cursor at the time of the selected message in the Trace Window. You can select other messages or move the measurement cursor through the Graphics Window using the mouse pointer.

Desktops

This configuration contains the following desktops:

Setup

Analysis

Trace

 

Setup [▲ back]

This desktop shows the Measurement Setup and the Simulation Setup. In the Simulation Setup, only the ARXML database for evaluating the PDUs and signals have been configured. The Measurement Setup has been switched to the offline mode and reads a logging file.

An IP filter containing a filter rule for AUTOSAR PDUs of the VGW node is inserted before the Logging Block. Thus only Ethernet packets that contain AUTOSAR PDUs and are sent or received by the VGW are forwarded and written to the logging file.

Analysis [▲ back]

Some of the transmitted values are displayed here in the Graphics Window and Data Window. The Map Window shows the distance traveled.

After the end of the measurement, you can activate the Window Synchronization (Home ribbon tab) button. This sets the measurement cursors in the analysis windows to the same time. If, for example, you move the measurement marker in the Graphics Window, the displays of the Map Window and the Trace Window are also adjusted to this position. For this to work, the measurement cursor of the Graphics Window must be switched on, or the Trace Window must be in chronological mode.

Trace [▲ back]

The Trace Window takes up most of the space on this desktop. To restrict the view in the trace, you can drag and drop PDUs and SOME/IP events from the Trace Window or the Symbol Explorers into the analysis filter of the Trace Window.
