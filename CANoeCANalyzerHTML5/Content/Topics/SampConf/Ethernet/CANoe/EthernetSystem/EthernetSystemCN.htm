System Configuration (Ethernet)

 

Click on the blue link to load the sample configuration in CANoe.

  • System Configuration (Ethernet) – EthernetSystem.cfg
    Simulates an Ethernet network with CANoe. The network communication described by ARXML is simulated with the various interaction node layers. AUTOSAR PDU, SOME/IP, UDP-NM, AVB-TP, and PTP are used.

The ARXML file contains the description for the following Ethernet ECUs:

Node

Short description

VGW

Simulates a gateway to the vehicle network.

The values for engine speed and driving speed are calculated by this node on the basis of the position Accelerator Position slider. The values are then output to the bus. Moreover, the lighting and the steering motions are controlled from this node in accordance with the waypoints.

IC

The instrument cluster displays the vehicle signals such as speed, detected traffic signs and battery charge status.

The Instrument Cluster panel displays the data of this ECU.

ADAS

This ECU evaluates the information provided by the CAMF and LRR and processes the data in such a way that it can be displayed on the instrument cluster.

CAMF

The Camera Front is responsible for detecting lane marking and traffic signs. The data is sent to the ADAS ECU as SOME/IP events.

The data is displayed on the CAMF panel. The data can be influenced by the sliders.

LRR

The Long Range Radar detects obstacles. The sensor data is sent to the ADAS.

The data is displayed on the LRR panel.

HU

The Head Unit controls the radio.

An audio stream is sent to the AMP by means of AVTP.

The Head Unit can be used to operate the HU.

AMP

The Amplifier receives the audio stream of the HU and outputs it on the loudspeakers.

The Powertrain & Chassis, ADAS, Infotainment and Network Management domains are divided into different VLANs. The ECUs are allocated to these VLANs accordingly. The following protocols are used in the VLANs.

  • VLAN 1 - AUTOSAR UDP NM and PTP
  • VLAN 2 - AUTOSAR I-PDUs and Service Discovery
  • VLAN 3 - SOME/IP and Service Discovery
  • VLAN 4 - AVB

Introduction

At the start of a measurement, the vehicle gets started and accelerated.

In the Visual Sequencer an automation sequence has been configured that starts the vehicle. At the start of measurement, the Drive Cycle sequence is started (bottom left on the Setup desktop). First the vehicle is started, and then the speed is adjusted as a function of the distance covered.

The route is defined in the CAPL program via waypoints.

Notes

The vehicle can be controlled via the Console Control panel. However, as long as the Visual Sequencer is active, the inputs will be overwritten.

Simulation [▲ back]

This desktop shows an overview of the panels and of the most important vehicle data. The distance covered can be monitored in the Card Window.

Setup [▲ back]

This desktop shows the Simulation Setup, including all simulated ECUs, and the Measurement Setup. In the lower area, you can select two automation sequences and the Start Values Window on the corresponding pages. Here you can also start the automation sequence.

Network Management [▲ back]

On this desktop, you can observe and influence the network management implemented via UDP-NM. After the start of a measurement, all ECUs are still deactivated. The ECUs can be activated by clicking the OnOff button on the Head Unit panel.

If the Visual Sequencer is activated, it sets OnOff automatically. To control the network management via the panels, you can deactivate the Start Sequence on Measurement Start option () in the Visual Sequencer.

Via UDP-NM, the AMP ECU is also woken up. This process can be observed in the Network Management panel and in the State Tracker. All ECUs can be woken up clicking the Start button on the Console Control panel.

ADAS [▲ back]

On this desktop, you can look at the sensors for the Advanced Driver Assistance System in more detail. The LRR and CAMF panels show the information collected by the sensors. In the ADAS – Data Data Window, the data sent via the network are displayed. The SOME/IP events sent by LRR and CAMF are evaluated by the ADAS ECU and processed for display in the instrument cluster. The ADAS – Graphics Graphics Window shows the current and allowed speed over the distance covered. To do so, the Graphics Window has been configured for X/Y operation. The detected traffic signs are displayed in the State Tracker.

Powertrain & Chassis [▲ back]

Here the drive data sent via the network is displayed. The signals sent by the VGW as I-PDU have been configured here in the Graphics Window and Data Window.

Infotainment [▲ back]

In the Graphics Window, a system variable has been configured to which the AMP ECU will write the amplitude of the received audio signal. The audio data is sent by the HUD by means of AVBTP. If you activate the Head Unit by clicking the Power button, you can transmit a short test tone by clicking the Tuning button. This test tone is then visible in the Graphics Window.

Trace [▲ back]

This desktop contains a Trace Window in which the various filter groups have been configured. To show the relevant event in the Trace, you can activate or deactivate the filter groups.
