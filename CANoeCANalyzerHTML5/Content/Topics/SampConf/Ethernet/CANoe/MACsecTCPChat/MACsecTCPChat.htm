MACsec TCP Chat (Ethernet)

 

Click on the blue link to load the sample configuration in CANoe.

  • MACsec TCP Chat (Ethernet) – MACsecTCPChat.cfg
    Demonstrates how to secure TCP communication using MACsec.

Introduction

This sample configuration shows how data integrity and optionally encryption of the data traffic between two measuring ports of CANoe can be ensured using the MACsec standard. It uses the TCPSimChat sample configuration as a basis and only adds MACsec.

MACsec is only available for physical ports. So you need a VN5000 network interface with at least two ports and a loopback cable connecting these two ports. It is not possible to use MACsec in simulation mode.

MACsec requires a suitable security profile, which you add in the Vector Security Manager and select for the desired measuring port. The configuration for the measurement ports named Port1 and Port2 is already part of the CANoe settings of this sample configuration.

You still need to perform the following configuration steps:

  • Import the attached MACsec Security profile (MACsec.vsme, only once):

  • In CANoe, navigate to the Simulation ribbon and click the Security Configuration button. In the dialog of the same name, click the Open Security Manager button.
  • In Vector Security Manager, on the File tab, select the Import command and then open the MACsec.vsme file that came with the sample configuration and is located in the SecurityProfiles subfolder.
  • Open the Security Configuration dialog again. On the MACsec page, make sure that the MACsec port configuration displays one port as Supplicant and one as Authenticator.

  • Connect a VN5000 network interface with a loopback cable between two measuring ports and configure it accordingly:

  • A VN5000 network interface with at least two physical ports is required.
  • Connect it to the computer running CANoe.
  • Start the Vector Hardware Manager and make the following settings:
  • Network: Ethernet.
  • Two switch segments named Switch_1 and Switch_2.
  • Add a physical port to each segment and name them Port1 and Port2.
  • Configure the two ports so that they can communicate with each other (e.g. correct master/slave settings when using automotive Ethernet).
  • For this purpose, please refer to the scheme of the setup, which can be seen on the desktop Measurement.

Setup [▲ back]

The Simulation Setup contains several nodes:

Nodes

Description

ChatServer

Waits for incoming TCP connections. Received Ethernet packets are then sent to all logged in clients.

ChatClient1

Establishes a connection to the Chat Server. Ethernet packets are then sent to the server. Packets coming back from the server are displayed on the desktop Measurement in the Chat Client 1 panel.

ChatClient2

Like ChatClient1, but with a different address.

PacketBuilder

Contains an Ethernet packet from a previous run. CANoe will identify this Ethernet packet as invalid.

Measurement [▲ back]

In the three panels of the chat clients you can enter text messages and send them to the chat server using the Send button. The Chat Server distributes the incoming Ethernet packets to all logged in clients.

The Trace Window on the right side is configured to display only outgoing Ethernet packets from one of the two measurement ports Port1 or Port2. The left Trace Window Simulation Ports displays only the Ethernet packets in the simulation. The authentication by the MACsec protocol is only visible in the right Trace Window.

The schematic drawing on the top right shows the necessary hardware configuration.

Fault Injection [▲ back]

This desktop allows you to inject a wrong MACsec packet into the communication. To do this, select the packet in the Packet Builder and click the Send button there. The MACsec protocol recognizes this packet as invalid, which is then logged in the Trace Window.
