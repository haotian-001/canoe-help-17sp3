Basic UDP Server and Client (C#/.NET)

 

Click on the blue link to load the sample configuration in CANoe.

  • Basic UDP Server and Client (C#/.NET) – UDPBasicDotNet.cfg
    Shows how UDP based communication can be handled with .NET nodes.

The configuration consists of a UDP server and a UDP client, which operates or controls the server. The server starts at the start of the measurement. The UDP client has six functions that you can use to display the communication between server and client.

Introduction

  1. Start the measurement.
  2. The Measurement desktop is displayed automatically.
  3. Click on the buttons 1-6 in the UDPClientPanel or press the corresponding number keys on your keyboard to send a frame to the server: e.g. sending a string or controlling the server LED.
  4. When completed successfully, the server reacts without sending a response. You can monitor the reaction of the server in the UdpServerPanel or in the Write Window.

Technical Backgrounds

  • The UDP client controls the UDP server without waiting for a response.
  • The UDP server strings the actions it is to perform in a queue, which is processed using a timer. Thus, multiple instructions can be sent from a client to the server, which the server than implements one after the other (unwaveringly).
  • With UDP, a server socket is not fundamentally different from a client socket. A UDP client object is used in both cases. The UDP client object has a network stream on which reading and writing is possible. In this sample configuration, the UDP socket of the server is used exclusively for reading from the network stream, while the client, on the other hand, is used for writing to it.
  • An action consists of a byte stream. The first byte specifies the action number. The following bytes in the stream are optional and dependent on the corresponding action. Depending on the action, the server knows whether it is to interpret the following bytes as parameters or not.
  • The server IP and the utilized port can only be specified with system variables.

UdpServerPanel

IP
Shows the currently used IP of the listen socket of the server.

IP
Shows the currently used port of the listen socket of the server.

Status
You can use this list to set the status of the server: Off/On/Invalid

Switch-Button
You can use this switch to put the server in Off and On status.

LEDValue
A byte value that controls the LED Strip (bit pattern).

LED Strip
Eight LEDs that represent the LEDValue (bit pattern).

Receiving Text
Text that was sent from the client to the server.

UdpClientPanel

Actions
You can use the buttons 1-6 to have the client perform the corresponding action on the server. Alternatively, you can initiate the functions with the number keys on your keyboard.

  1. Send a string to the server
    Sends a randomly generated string to the server. In the UdpServerPanel the string is displayed in the Receiving Text field.
  2. Set LED Value to a random value
    The value is displayed in the UdpServerPanel. By setting this value the LEDs will show up the bit based pattern.
  3. Trigger LED error indication on server
    Demonstrates a visible server error response. All LEDs blinking at the same time.
  4. Trigger LED test pattern on server
    Example how to test the LEDs on a real server panel. One LED by another.
  5. Send invalid command to server
    Shows what happens, if an invalid command is sent from the client to the server. An invalid command the server does not understand due it has no implementation for it.
  6. Trigger action the client does not recognize
    Shows what happens if you tell the client to send a command, the client does not recognize.
