TLS Chat (CAPL)

 

Click on the blue link to load the sample configuration in CANoe.

  • TLS Chat (CAPL) – TLSSimChat.cfg
    Demonstrates the usage of TLS sockets in CAPL.

The sample configuration shows how you can use TLS sockets of the TCP/IP API.

Desktops

This configuration contains the following desktops:

Setup

The configuration includes several nodes:

Node

Description

ChatServer

The chat server waits for incoming TLS connections. Received messages are sent to all registered clients.

ChatClient1

The chat client establishes a connection to the chat server to send the text messages to the server. Messages, that are sent back from the server, are displayed in the panel.

ChatClient2

see ChatClient1, but with another address

The nodes use the Security Configuration profile called Demo Ethernet TLS Chat. The certificates are stored in the Certificates directory next to the TLSSimChat configuration.

Measurement

You can insert text messages in the panels of the chat clients. Then you can send the messages with the [Send] button to the chat server. The server distributes the incoming message to all registered clients.
