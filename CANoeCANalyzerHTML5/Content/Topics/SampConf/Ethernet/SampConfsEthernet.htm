General

System Configuration (Ethernet) – EthernetSystem.cfg
Simulates an Ethernet network with CANoe. The network communication described by ARXML is simulated with the various interaction node layers. AUTOSAR PDU, SOME/IP, UDP-NM, AVB-TP, and PTP are used.

Analysis of Ethernet Networks with CANoe

Basic Analysis (Ethernet) – EthernetBasic.cfg
A logging file is played back in offline mode. You can trace and evaluate the communication in the Trace Window. The Data Window and Graphics Window show the signal evaluation.

View Description

Proprietary IP Protocol – EthernetBasicProprietaryProtocol.cfg
CAPL is used to evaluate a proprietary IP protocol and store the data in system variables. The system variables can be processed further in CANoe.

Diagnostics

System Configuration (DoIP) – DoIPSystem.cfg
Shows the setup of a Diagnostics over IP (DoIP) test system consisting of DoIP Tester and DoIP ECU with dynamic IP address assignment by a simulated DHCP server. The usage of TLS encryption is also demonstrated.

View Description

Ethernet Diagnostics (HSFZ / DoIP) – HSFZSystem.cfg
Shows the use of Diagnostics over High-Speed-Fahrzeugzugang (HSFZ) in CANoe and can be switched to the DoIP protocol. A simple gateway between HSFZ or DoIP and CAN is implemented.
