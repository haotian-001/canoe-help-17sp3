XCP Tester CAN

 

The principle of the CANoe XCP option is explained in this sample configuration. Here, CANoe represents the tester (XCP master) and the XCP Simulator the control unit to be tested (XCP slave).

Load Configuration

XCP Simulator Configuration

The XCP Simulator must be configured in the Vector hardware configuration dialog on the same channel as CANoe (e.g. XCPsim CAN1, CANoe CAN1 etc.).

Note

If an XcpSim is not shown in the dialog, you must start the XCP Simulator (XCPsim.EXE) from the following directory: Open Folder

An XCP device is defined in the XCP/CCP Window.

The two XCP parameters ampl and channel3 are configured for the Polling read mode. This means that upload commands are sent to the XCP slave cyclically as specified. The responses of the XCP slave are written directly to the system variables.

The XCP signal channel2 is configured for the DAQ measurement. There, the XCP slave automatically sends measurement values cyclically as specified. The measurement values are also written to a system variable. The remaining parameters are only used for the download.

System variables are automatically created for all configured XCP parameters.

The test setup contains a test unit that writes values to the control unit, reads out values and monitors certain values via XCP.

You can change values interactively in the panel via XCP during the measurement. You can monitor the changes in the Graphics Window on the Trace Graphic Display desktop.
