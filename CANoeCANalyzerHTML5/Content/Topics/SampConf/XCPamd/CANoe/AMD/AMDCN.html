AUTOSAR Measurement (AMD)

Sample Configurations » AMD/XCP » AUTOSAR Measurement (AMD)

 

The AMD sample configuration demonstrates the measurement of an AUTOSAR ECU using the AMD (AUTOSAR Measurement and Debugging) option (Load Configuration).

Note

This sample configuration requires the CANoe option .AMD/XCP in order to function properly.

Simulated AUTOSAR ECU

This measures an AUTOSAR ECU simulated in CANoe. The simulated ECU contains a Vector MICROSAR Classic stack, an AUTOSAR sample application with software components and a MICROSAR Classic RTE. These are integrated into CANoe as a modeling library.

The sample application implements a simple interior light control.

Measurement and Configuration

The AMD option reads values from the ECU via XCP on CAN. The XCP configuration (XCP/CCP Window) references the A2L file Result.a2l. This file was generated by the RTE (AUTOSAR Runtime Environment) and BSW (AUTOSAR Basic Software) generators during integration of the MICROSAR Classic stack. The measurement is carried out via DAQ lists.

Analysis [▲ back]

The AMD Car Model panel is used to control the model.
The door contacts are the sensors; the front and rear interior lights are the actuators.
Click on the doors to open and close them. When all the doors are shut, both lights go out and the bus goes to sleep after a short interval. If a door is opened, the bus wakes up and both lights go on.

XCP communication is terminated before the network management goes to sleep and is re-established on waking. This function is implemented in CAPL in the RearECU network node.

Internal ECU variables are shown as system variables in CANoe. These variables are displayed in the Trace window and the State Tracker. This enables you to measure and display state changes in the ECU.
