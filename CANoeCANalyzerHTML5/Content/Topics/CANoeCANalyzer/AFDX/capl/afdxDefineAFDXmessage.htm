Declaration of AFDX Messages

 

AFDX is accessible via two different interfaces:

  • a separate DLL (AFDX Interaction Layer) or
  • natively via AFDX message objects.

AFDX messages may be controlled by means of a CAPL program. An AFDX message is created with the keyword a664Message. You need a message for the following scenarios:

  • create AFDX traffic and control the payload
  • store received AFDX messages for later evaluation/comparison

The complete declaration includes the message identifier or message name when working with symbolic databases. The message identifier is entered as a number (integer) in decimal or hexadecimal notation to identify messages. The entry of “*” signifies that at first this variable does not have any message identifier. If the message is created with a message name from a database (example (1)), the corresponding setup information (e.g. VL, IP addresses and UDP ports) from the database attributes is used for initialization.

A message object created based on a number or “*” has no valid address information. If you would like to transmit a message you need the following steps:

Step by Step Procedure

  1. Create the message object (see example statements 2 – 6)
  2. A664InitMessage() : initialize addressing information
  3. A664VLConfig() : configure the virtual link for the message
  4. A664MsgConfig() : configure the Tx behavior (IP fragmentation, output line, sub VL ID)
  5. A664TriggerMessage(): schedule the AFDX message

Note: If the message is created based on a database name the steps 2 - 4 may be omitted.

For more convenience in the programming environment a CAPL include file is provided. The file gives the programmer a number of symbolic constants and some helper routines as a supplement for the CAPL API. Use this statement in the include section of your CAPL code:

#include "AFDX/utils.cin".

Example: Create AFDX message object

// variables section
variables {
  a664Message DEMOMSG20_INT myAfdxMsg2; // (1)
  a664Message * myAfdxMsg3; // (2)
  a664Message * myAfdxMsg4 = {msgChannel = 1}; // (3)
  a664Message * myAfdxMsg5[2] = {
  {msgChannel = 1},
  {msgChannel = 1}}; // (4)
  a664Message * myAfdxMsg6 = {AFDXLength = 800}; // (5)
  a664Message 0x0x1A182B7A myAfdxMsg1; // (6)

A newly created message object has no valid address information and cannot be sent. The address information may be added with the function A664InitMessage.

Example: Initialize address header

// define message and control parameters completely from scratch
a664Message * myMsg2 = {msgChannel = 1};

…

a664InitMessage(myMsg2,
   IpGetAddressAsNumber("10.1.2.1"), // src IP
   IpGetAddressAsNumber("224.224.2.2"), // dst IP
   1024, // src UDP
   1024, // dst UDP
   123, // VL ID
   4095); // payload size
myMsg2.TxCycle = 256;
a664VLConfig(myMsg2, // apply this to VL = 123
   3, // activate redundancy / integrity checking
   7, // transmit on A and B / allow fragmentation on VL
 128, // set the BAG to 128 ms
 512, // set the maximum frame size to 512 bytes
   1) // allow a skew max of 1 us
a664MsgConfig(myMsg2,
   1, // set fragmentation
   3, // output on A and B
   1);// use sub VL = 1

a664TriggerMessage(myMsg2, 2); // start cyclic transmission

The members of the AFDX message are accessed via selectors. Note, that the byte ordering for Ethernet is big endian. Nevertheless, all selectors which are not arrays are presented in the host byte order.

AFDX message specific event procedures

If there is a valid AFDX message event received, an event procedure is called. Note that AFDX message events may be triggered from a connected hardware channel or even internally from a simulated node or an Interactive Generator. The event procedures

  • on a664Frame and
  • on a664Message

are available.

Note

This description refers to the so called frame mode (bus oriented view). In the so called FDS-mode a664Messages and a664Frames are still available, but with limited functionality, because of the different data model (e.g. signals are part of FDS in this case).

However in this case the generic callback on PDU is also available.
