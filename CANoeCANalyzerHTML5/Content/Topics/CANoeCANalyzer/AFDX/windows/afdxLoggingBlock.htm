Logging

 

You can log received Ethernet packets in logging files in ASCII, BLF (Binary Logging format) or MDF4 format. You can then play back these files in Offline mode or using a Replay Block.

Note

ASCII is not recommended at high data rates due to its poorer performance. BLF format is much better suited for this.You can also reduce the memory requirements by using the trigger block with trigger conditions for symbolic messages, signals, system variables or for AFDX raw frames.

On the other hand, ASCII format is advisable if specific subareas of the Trace window need to be exported as a template with the intention of making specific modifications to the file and replaying the revised file later.

Libpcap and LibpcapNG File Format

The libpcap file format is also supported for replay. This format does not contain any statistical, channel and direction information.

The following special conditions should be observed here:

  • Replay Block (libpcap format): In the channel mapping, only the AFDX 1 channel exists, and it can be replayed on any other desired channels. If the selected replay channel has two Ethernet port allocations (Line A and Line B), then a frame is allocated to a port based on the MAC source address. If the Interface_ID has the value 2, the frame is sent on Line B. If there is only one port for the replay channel, all Ethernet packets are sent over this port.
  • PCAPNG files logged with Wireshark do not contain information about the assignments of the channels. The assignment of channels is applied as described above.
  • PCAPNG files with AFDX messages logged with CANoe contain information about the assigned channels. When replaying with the Replay Block, the logged channel assignments are replayed.
  • Offline mode (libpcap format): In Offline mode, the fields in the Bus Statistics Window remain empty. In this case, however, the key statistical values can be taken from the cumulative entries of the AFDX VL statistics.

Internally, CANoe works with bus events; of course, only AFDX events are evaluated for option AFDX. The internal processing routine for log files takes the bus system for the events from the log file. That is not a problem, provided that only one option based on Ethernet is installed (e.g. option AFDX). If option Ethernet is also installed, you must decide whether AFDX events or Ethernet events should be generated when a file in libpcap format is read in. This setting is made in the Options dialog (Bus Systems/Protocols|AFDX). Changes take effect after a restart of CANoe.
