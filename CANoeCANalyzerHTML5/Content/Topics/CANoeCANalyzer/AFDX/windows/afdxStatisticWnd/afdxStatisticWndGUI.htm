User Interface

 

General

Note

You can use several AFDX VL Statistics Windows, e.g. to have one window available for each AFDX channel.

Toolbar

The following commands are available to you on the toolbar.

Command

Description

AFDX Channel

Select one of the following layout options.
AFDX1

Select the AFDX channel from the list for which the statistic quantities should be displayed.

After the current AFDX channel the name of the AFDX network in the Simulation Setup is displayed too.
AFDX All

Here you can view the Current/ Last values of the VL-Statistic for all active channels.

You can set system variables by selecting one (or more) cells as it is done in the channel specific view.

AFDX All supports the different available views.

Note

System variables cannot be set in an empty cell during measurement .

Configuration...

Here you open the configuration dialog where you can select the statistic quantities that should be displayed.

Collapse All

Here you can collapse all entries of the tree view. Alternatively press <Shift><->.

Expand All

Here you can expand all entries of the tree view. Alternatively press <Shift><+>.

Show only items connected with system variables

Here you can restrict the display to elements that are connected with system variables.

  • activated: only connected elements are displayed
  • deactivated: all elements are displayed

AFDX VL Statistics System Variables Manager

Here you open the AFDX VL Statistics System Variables Manager for managing AFDX specific system variables.

View

Here you can change the grouping of the statistic quantities.

Reset All Statistics

Here you can reset the statistics during measurement. Alternatively press <DEL>.

Pause Statistics

Here you can pause the statistics during measurement.

Statistic View

Most statistic values are based on counters and are recalculated with every arriving frame.
Furthermore there are also time based statistic values, especially the frame rate and the throughput rate integrated over the duration of an epoch and the MinBAG describing the minimal BAG within an epoch.
Therefore the statistic values for these two groups have a different signification:

  • Current/Last shows the average value for rates , the smallest value over the period of time of the last epoch for MinBAG and the current sum for counters.
  • Min and Max show the minimum and maximum values since measurement start or since the last reset for rates and MinBAG. For counters they show the minimum or maximum increase per second since measurement start.
  • Avg indicates the average value integrated over the epoch since measurement start or last reset for rates and MinBAG, for counters it indicates the average increase per second.
  • Std.Dev shows the mean square deviation corresponding to the average value.

The values are calculated and the rates are averaged over configurable epoch periods.

Note

Statistic sub-elements which contain statistic values per VL but have no relevant statistics info (value 0 or -) are not displayed, depending on the selected view. However those trees are automatically extended at runtime as soon as significant statistic values are received.
