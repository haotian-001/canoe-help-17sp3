Columns in the Message List

 

The following columns are available to you.

Column

Description

Standard

Editable

#

unique identifier of the message

Sequential numbering; a number is automatically assigned when the message is inserted in the list.

shown

—

Message

display of the MessageID from the database

If symbolic display is selected in CANoe, the name of the message is shown in this column.

shown

—

Description

In this field you can add a description to the message.

 shown X

Line

Indicates the line on which the message should be sent.

Possible values are:

  • Auto: The information about the specific line on which the message should be sent is taken from the database. This is the default setting.
  • A: The message is sent on Line A.
  • B: The message is sent on line B.
  • AB: The message is sent redundantly on lines A and B.
 shown X

Channel

AFDX channel on which the message should be sent

All channels existing in the configuration are offered for selection in the list.

Note

The AFDX IG is inserted in the Simulation Setup, and a fixed channel is allocated to it. Nonetheless, the channel number can still be modified at this point. So, with a generator it is possible to send messages on multiple channels.
shown X

Tx Rate

cycle time of the message

shown

 —

Send

Note

The information on whether a message is transmitted periodically or aperiodically (event driven) is taken from the database.

Periodic messages

Two buttons are provided, one to start and one to stop the periodic transmission.

By means of the third button with a flash icon you may trigger a single transmission of the selected message, corresponding to the SignleShot mode in CAPL.

If a message occurs more than once in the Message List (same identifier in the database), periodic transmission can only be started for one of these messages. That is, periodic transmission of one message is automatically stopped when periodic transmission of another message with the same identifier is started.

Asynchronous messages

A button [Send] is displayed. When the button is pressed, the associated message is sent once.

 shown X

Signal Update

This check box is only shown for periodic messages.

If the check box is active, the signal values are immediately passed to the scheduler whenever a signal value is changed in the generator.

If the check box is inactive, all signal values belonging to the message can be changed without passing them to the scheduler. The signal values are not transferred until the check box is activated.
