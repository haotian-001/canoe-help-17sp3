Signal List

 

In the Signal List the signals of all signal states (Functional Status) and messages selected in the FS View are shown. You can set specific signal values or signal states in this list. When the signal values are transmitted depends on the send type of the message and the specific settings in the Message List.

Erroneous signals are marked with an error icon. Messages with such signals cannot be sent.

Select multiple lines

Multiple lines are selected with one of the following procedures:

  • Hold down the <Ctrl> key and select the desired lines with the left mouse button.
  • Select the first line of the desired range with the left mouse button. Hold down the <Shift> key and select the last line of the range.
  • Select all lines with <Ctrl>+<A>.

Change signal states

Signal states can be modified in the Signal List as follows:

  • Select the desired value for the signal state in the list in the Status column.
  • Select one or more signals and set the signal state from the context menu by the commands Status | ND, Status | NO, Status | FT or Status | NCD. You can modify multiple signal states simultaneously using this method.

Change signal values

Signal values can be modified in the Signal List as follows:

  • Enter the desired signal value in the Value column.
  • Select the desired value from the list (lists are offered for signals with the data types Bool and Enum).
  • In the context menu of the Signal List, use the Copy Signal Value command to copy an existing signal value. Select all signals to which you wish to assign the copied value. Then choose the Paste Signal Value command in the context menu. If the copied value cannot be assigned to all of the selected signals, an error message is output.

For changing multiple signal values at once, select the corresponding lines in the signal list first.

  • Now move the input focus with the keys <Right Arrow> or <Left Arrow>to the column value.
  • Insert the desired value into the field and leave it with <Enter>, <Up Arrow> or <Down Arrow>.

Note

If no signal value is given, the value 0 is used for signals with the No Data (ND), Functional Test (FT) or No Computed Data (NCD) signal state when sending the message.

For signals with the Normal Operation (NO) signal state, it is mandatory to enter a signal value before sending. An exception are string signals, empty signal values are valid for these signals.

Invalid signal values

Invalid signal values are color coded in the Signal List.

Cell Color

Description

red

The signal is in the Normal Operation (NO) state, and no signal value was given. A signal value must be entered before the message belonging to the signal can be sent.

Messages for which at least one signal value must be entered for sending are identified by an icon in the Message List.

Filter signals

In the Signal List, you can create column-specific filters for reducing the number of signals displayed in the list. When you move the mouse pointer to the column header, a filter icon is shown. Activate the filter icon with a left mouse click.

To create a user-specific filter, choose the command Custom. In the Custom AutoFilter dialog you can now set up a customized filter.

Example of a filter in the Name column

[like] *show_me*

Only those signals are shown which contain the character string show_me in the signal name.
