Frame Attributes

 

Packets can also be filtered according to their frame attributes. The Frame Status and Error Type tables provide three states in the Activated column for each bit switch: Ignored (bit is ignored), Activated (packet is filtered if bit is set), and Deactivated (packet is filtered if bit is not set).

Frame Status

Flag

Description

Interface is B

The frame was received on the assigned B interface.

Frame is redundant

The frame is the later frame in the case of redundant frames (received simultaneously on line A/B).

Frame is fragmented

The frame is an IP fragment and not yet a complete or valid message.

Packet is reassembled

The message was composed from multiple fragments.

Packet is SAP

The message is an SAP message (only identifiable with database support).

Error Type

Flag

Description

Wrong interface

The interface byte of the MAC address does not match the physical line on which the frame was received.

Invalid frame

The frame is not a valid AFDX frame.

Integrity error

The frame does not meet the requirements of a consecutive sequential number according to ARINC-664p7 Ch. 3.2.6.2.1.

This check must be activated in the hardware dialog.

Redundancy error

The frame violates the redundancy requirements according to ARINC-664p7 Ch. 3.2.6.2.2.

This check must be activated in the hardware dialog.

Fragmentation error

An error occur when reassembling this frame in a message.

Protocol error

The frame contains error on a higher protocol level.

BAG

If a minimum or a maximum value is specified in the BAG area, the packets are filtered according to their measured BAG too.
