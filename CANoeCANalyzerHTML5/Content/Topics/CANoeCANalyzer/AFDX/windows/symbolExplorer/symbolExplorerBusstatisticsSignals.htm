Bus Statistics System Variables for AFDX

 

In the Symbol Explorer, the following system-defined bus statistics system variables are automatically available for AFDX under the namespace _Statistics::AFDX[n][A/B] (n is the channel number):

Bus Statistics Value

System Variable

Description

Link speed [Mbit/s]

LinkSpeed

communication speed of the connected links from hardware polling

Link status

LinkStatus

current link state - status from hardware polling

  • Up (link is active)
  • Down (link is inactive)

Link down total

LinkDownEvents

number of link interruptions (change of link status from active to inactive) that occurred during the measurement

Rx frames raw [total]

RxFramesRaw

number of Ethernet packets received by the hardware adapter, including corrupt frames

Note

Corrupt frames may be recorded differently depending on the type of adapter used.

Rx frames [total]

RxFrames

number of Ethernet packets actually received in CANoe

This value may be smaller than the value Rx frames raw due to hardware filters or queue overruns.

Rx frames [fr/s]

RxFrameRate

rate of frames received in CANoe per second

Tx frames [total]

TxFrames

total number of Ethernet packets that have been sent

Tx frames [fr/s]

TxFrameRate

rate of frames sent by CANoe per second

Dropped frames [fr/s]

RxDroppedFrameRate

rate of frames received by CANoe but actively rejected due to the two configuration settings Ignore Dropped Packets or Ignore Unknown Packets

Invalid frames [fr/s]

RxInvalidFrameRate

rate of frames received by CANoe that are detected as invalid

Invalid frames [fr]

RxInvalidFrames

total number of frames received by CANoe that are detected as invalid

Rx error frames hw [fr/s]

RxFrameErrorRate

rate of Rx frames detected by the hardware adapter as corrupt

Note

The error types which are detected will depend on the adapter used.

Rx error frames hw [total]

RxFrameErrors

number of Rx frames detected by the hardware adapter as corrupt

Tx error frames hw [fr/s]

TxFrameErrorRate

rate of Tx frames detected by the hardware adapter as corrupt

Note

The error types which are detected will depend on the adapter used.

Tx error frames hw [total]

TxFrameErrors

number of Tx frames detected by the hardware adapter as corrupt

Rx bitrate [Mbit/s]

RxBitRate

Rx data rate per second determined by the adapter including Preamble and IFG (interframe gap)

This value is an indicator of the load of the specific link in the receiving direction and indicates how many bits are being transmitted per second.

Tx bitrate [Mbit/s]

TxBitRate

Tx data rate per second determined by the adapter including Preamble and IFG (interframe gap)

This value is an indicator of the load of the specific link in the transmitting direction and indicates how many bits are being transmitted per second.

You can use familiar drag and drop operations to drag the bus statistics system variables from the Symbol Explorer to a Graphics or Data Window.

Note

You cannot delete or rename statistics system variables.

If statistical calculation is disabled in the Options dialog of CANoe (Measurement|Performance), the system variables are not recalculated either (in contrast to earlier statistics signals).

Statistics signals that have been defined with a CANoe version smaller than 8.2 are no longer supported.
