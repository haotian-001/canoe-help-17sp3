Network Hardware Configuration - Filters

Ribbon: Hardware Tab|Network Hardware|AFDX|Filters

AFDX

Filters

Signal Protocol

 

On this page, you can define low-level filters on the driver level so that you only see the relevant packets and improve performance. Various input options are available for this.

Source and VL based filters

This special dialog lets you conveniently define the filters based on the selection of senders and receivers. For a usage of the dialog the senders and receivers should be defined in the database but not necessarily.

You can use the [Selectâ¦] button to open the Hardware Filter dialog, in which you can define new filters and add them to the list of active filters. The input methods are the same as for Source and VL filters of the AFDX Filter.

The filters generated from this are integrated in the MAC and IP-based filters. Only active filters are considered. This means that the two filter types may be combined.

MAC or IP based filters

You can define filters for the IEEE 802.3 specified fields Destination MAC address, Source MAC address, Ethertype, and Source IP address.

Pressing the [Addâ¦] button opens the Add Filter dialog, in which you can define a new filter and add them to the list of active filters. A new filter requires entry of the following values:

  • Field: Selection of the filter type from the selection box
  • Code: Acceptance code with which the relevant bits of the packet (specified by the acceptance mask) are compared
  • Mask: Acceptance mask indicating which bits that are compared to the acceptance code

Note

Defined filters are used later during the measurement in the sequence in which they appear in the filter list.

You can use the [Edit...] button to modify the currently selected filter in the list.

If the filter list is empty, indicating that no filters have been defined, then all packets are blocked on this channel, and no data can be received from the network.

You can remove individual filters or all filters from the list with the [Remove] or [Remove all] button.

When the [Accept all packets] option is selected, you can deactivate all filters in the filter list.

Accepting or rejecting an AFDX packet

Note

Acceptance filtering only affects received packets (Rx).

The acceptance mask determines which bits of the received packet should be compared to the acceptance code. If the bit in the mask is set to zero, then this bit is irrelevant for the comparison. On the other hand, if it is set to one, then the bit of the received packet is compared to the bit of the acceptance code. If they are identical, the packet is received. Otherwise, it is blocked.

Example

Filter definition

(1) Filter type Destination-MAC address, Code 03:00:00:00:11:C6, mask 00:00:00:00:FF:FF

(2) Filter type Destination-MAC address, Code 03:00:00:00:11:C6, mask FF:FF:FF:FF:FF:FF

Effect

(1) Only packets with a Virtual Link-ID of 0x11C6 (referred to the last two byte of the address) are accepted, whatever the other bytes in the address. So it is possible that erroneous AFDX frames are accepted as long as the Virtual Link-ID fits.

(2) The MAC address must exactly fit and therefore only valid AFDX frames with the Virtual Link-ID are accepted.

Example

Filter definition

Filter type Source-MAC address, Code 02:00:00:0A:01:20, mask 00:00:00:FF:FF:00

Effect

All packets of an end system with the User-ID 0x0A01 are accepted.

The bit comparison is performed for all filters in the list. If a filter accepts the packet, it is passed. If none accept it, the packet is blocked and rejected.
