Network Hardware Configuration - AFDX

Ribbon: Hardware Tab|Network Hardware|AFDX

AFDX

Filters

Signal Protocol

 

On this page you can assign the relevant network interfaces to all AFDX channels or exclude those interfaces that are not to be used. Please consider the information on page Exclusive use of an Ethernet interface.

To configure your hardware, follow the next steps:

Step by Step Procedure

  1. Open the dialog from the Hardware ribbon tab|Network Hardware.
  2. Switch to the page of the AFDX channel AFDX<channel number> to be configured.
  3. For Line A and/or Line B, select the physically connected AFDX or Ethernet adapter under consideration of redundancy requirements, i.e. adapters connected to Line-A should be entered under Line-A and vice versa. Otherwise an error message is displayed. For redundancy, both lines (A and B) must be entered.
  4. Configure the AFDX channel, depending on the physically connected network hardware considering the limitations.
  5. Repeat steps 3 and 4 for each AFDX channel.
  6. Select Unused for lines that you don't need.
  7. Close the dialog with [OK].

Configuration of default network interfaces

Note

The Vector network driver must be installed so that the local Ethernet adapter of the computer can be selected.

If necessary you have to do some settings for the Ethernet adapter in the WindowsÂ® network settings.

To do your configuration, perform the following steps:

Step by Step Procedure

  1. For Line A and/or Line B, select the physically connected AFDX or Ethernet adapter.
  2. Configure the AFDX channel.
  3. Repeat steps 1 and 2 for each AFDX channel.
  4. Select Unused for lines that you don't need.
  5. Close the dialog with [OK].

Configuration of VN5610/ VN5610A modules

Note

To use VN5610 modules you have to assign channels and parameterize the interfaces in the Vector Hardware Config dialog first.

When using Vector VN-hardware, e.g. VN5610 or VN5610A), select the entry VN Ethernet Hardware as hardware type. The exact assignment or device configuration must be carried out in the Vector Hardware Configuration dialog (open this dialog with [Driver...]).
By selecting an Ethernet adapter, this selection may be undone.

For AFDX you should use Auto Negotiation (restricted to 100 Mbit â RJ45) for standard parameterization. The setting of parameter Bypass depends on the intended use:

  • Off: normal use as network adapter
  • PHY: use as passive TAP (between switch and end system)
  • MAC: use as TAP with active send possibility

Configuring hardware from third party vendors

You can configure approved network hardware from third party vendors by the following steps.

Step by Step Procedure

  1. Select the connected hardware for Line A and/or Line B.
  2. Configure the bitrate:
    • 10 MBit/s
    • 100 MBit/s
    • 1 GBit/s
  3. Repeat steps 1 and 2 for each AFDX channel.
  4. Set channels that are not needed to Unused.
  5. Close the dialog with [OK].

Limitations on allocating real hardware to AFDX channels

The following limitations apply to allocating real hardware to AFDX channels:

  • If two Ethernet ports are used for an AFDX channel (redundancy management), these ports must have the same type of hardware.

Example

You want to use two AFDX channels, each with Line A and Line B. Your computer contains a 2-channel network card (NIC), and you have connected a VN5610 interface. In the hardware configuration, you allocate AFDX 1 to the network card and AFDX 2 to the VN5610.
  • If hardware from a third party vendor can process exactly one AFDX channel, then separate hardware is needed for each channel.

Example

You want to use two AFDX channels, each with Line A. When using third party vendor hardware, you need two cards.

Further Settings

In addition, you can make the following settings:

  1. Switch the behavior of this channel between regular AFDX (Full AFDX) as per ARINC-664p7 and a lightweight variant (Light AFDX).
    When Light AFDX is selected, only one network interface can be configured and the redundancy check is disabled (no redundancy support).
  2. Configure the following checks:

Control Field

Description

Default

Enable Integrity Checks
  • active: integrity checks are executed for each VL according to ICD or DBC
  • inactive: integrity checks are not executed at all

active

Enable Redundancy Checks
  • active: redundancy checks are executed for each VL according to ICD or DBC
  • inactive: redundancy checks are not executed at all

active

Ignore Unknown Packets
  • active: packets that are not defined in the database, are skipped
  • inactive: packets that are not defined in the database, are passed up to the Trace Window for analysis

inactive

Ignore Dropped Packets
  • active: for redundant packets the faster one is used, the second one is skipped not displayed in the Trace Window
  • inactive: redundant packets are passed up to the Trace Window for analysis
