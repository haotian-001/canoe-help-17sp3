Stimulation

 

Stimulation using messages requires control over the time a message is sent (scheduling) and the associated message contents (payload).

Triggering mechanisms

  • [A] Replay block: When a recording that was made using the Logging block is replayed, the message is triggered at the time of the recorded time stamp.
  • [B] AFDX Interactive Generator: The message is triggered using the Sending column of the message list.
  • [C] CAPL: The message is triggered using the AfdxOutputPacket and AfdxOutputPacketRaw functions.
  • [D] CAPL: The message is triggered using the AfdxSchedule function.
  • [E] CAPL: The message is triggered using the A664TriggerMessage or A664TriggerFrame function.
  • [F] The message is triggered via a change of the payload using mechanisms [3], [4], and [6].

Mechanisms for specifying the payload

  • [1] Replay block: The entire message, and thus also the payload, is replayed as it was recorded.
  • [2] AFDX Interactive Generator: Modification of the values in the signal list
  • [3] Panels: Modification of signals using panels
  • [4] Mapping: Modification of signals using mapping to system variables
  • [5] CAPL: Access using AFDX CAPL functions
  • [6] CAPL: direct access using signals (Version 8.2 SP2 or higher)
