Use The Right Operating Mode

 

CANoe provides various operating modes for the AFDX option that you can select and configure as required. You will find information on the different conceptual approaches and the mode of functioning of the operating modes on the page Operating Modes for AFDX.

Note

The display in the Trace Window is different for each mode.

Frame Mode

CANoe with the AFDX option is in frame mode by default. If necessary, ensure that the FDS mode in the AFDX FDS Mode area (Options dialog) is deactivated. The bus-oriented frame mode is especially well-suited for the following:

  • Receiving and evaluating all messages on the bus
  • Receiving and evaluating all signals on the bus
  • Checking and evaluating time response
  • Checking and evaluating protocol behavior

FDS Mode

You activate the FDS mode in the AFDX FDS Mode area (Options dialog).
The application-oriented FDS mode is especially well-suited for the following:

  • Processing large signal quantities and high data throughput
  • Stimulating a few signals and monitoring many signals
  • Monitoring signal value changes individually (fading), especially with periodic sending of messages and signals
  • Performing tests of application behavior, e.g., integration tests
  • For other test purposes
  • Significantly increasing the performance in lab environments or high computing time

Changeover Between Frame Mode and FDS Mode

If you want to change back and forth between frame mode and FDS mode, you must observe the following:

  • You can use the same databases (DBCs) for both modes
  • All databases must be reloaded at each changeover.

Frame mode and FDS mode in mixed operation

While the bus-oriented view and the application-oriented view are normally used separately for their respective purposes, you can also configure a mixed operation. In this case only frames whose content has changed are processed in the frame mode, while all FDS events are generated and messages are displayed in the Trace Window in the FDS mode. However, this can affect performance.

Note

  • In the FDS mode, the signals are generally no longer directly connected to messages but rather to the FDS (PDU). The FDS are in turn connected to the messages. For this reason the signal names are not compatible between the two modes.
  • In the FDS mode, the complete signal names contain the PDU name. In the frame mode, the message name is used directly. Signals in the Data Window or Graphics Window or in CAPL must therefore be redefined if necessary after a changeover of the mode.
