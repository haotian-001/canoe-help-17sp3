Monitoring at the AFDX Switch Monitoring Port

 

If the measurement system is to be operated at an AFDX switch's monitoring port, the following aspects must be considered:

  • No send functionality is available, because the monitoring port can only output packets.
  • The data rate at the monitoring port may be very high, depending on the configuration of the switch, especially when all AFDX channels are being operated and are output by monitoring. In such cases, it is urgently advisable to use driver-based packet filters to filter out all non-relevant packets as early as possible.
  • If the entire data traffic should still be analyzed with as few losses as possible, it is advisable to proceed as follows:
  • Deactivate all analysis windows such as Trace Windows, data Windows, etc.
  • Activate logging by the BLF format and record the data traffic.
  • Then analyze the data without load effects in offline mode or with the Replay Block.
  • Many ICDs or many DBCs or large DBCs may be necessary for complete signal decoding, since the packet multicast can result in messages accumulating from many end systems via the switches.
