Protocol Information

Before each Ethernet packet the preamble with 7 bytes (AAh) and the Start Frame Delimiter (D5h) are sent.

The data of an Ethernet packet (MAC-SDU) can contain further protocols and the payload. The data must have a minimum size of 46 bytes and a maximum size of 1500 bytes. If less data are available a padding is necessary, more data have to be fragmented.

Following the payload the checksum (CRC) is sent.

Note

Preamble, Start Frame Delimiter and checksum are not part of an Ethernet packet in CANoe and are not evaluated.

Received packets with an incorrect checksum are discarded and not displayed in the Trace Window.

The field type informs about the protocol of the next higher layer within the payload. The values are greater than 0600h, otherwise it is an Ethernet-I-Frame with a length field at this position.

  • 0800h: IP Internet Protocol (IPv4)
  • 0806h: Address Resolution Protocol (ARP)
  • 22EAh: Multiple Stream Registration Protocol (MSRP)
  • 22F0h: Audio Video Transport Protocol (AVTP)
  • 8035h: Reverse Address Resolution Protocol (RARP)
  • 8100h: VLAN Tag (VLAN)
  • 86DDh: Internet Protocol, Version 6 (IPv6)
  • 88F5h: Multiple VLAN Registration Protocol (MVRP)
  • 88F6h: Multiple MAC Registration Protocol (MMRP)
  • 88F7h: Precision Time Protocol (PTP)

Note

If no packet type designator is specified by the Packet API, the header length is 14 byte. If the packet type designator "vlan" is used, a VLAN packet (IEEE 802.1Q) is created with a header length of 18 byte. The Ethernet type is set to 8100h.
