AFDX Protocol

 

On the frame level, the AFDX protocol does not contain its own header, rather its only visible element is the sequence number at the end of the AFDX frames. However, the MAC addresses are specifically interpreted, and certain limitations apply to valid AFDX packets and their Ethernet headers.

An AFDX frame has the following layout of content (without Preamble or Interframe GAP).

  • The Domain_ID designates a functional group to which the Equipment belongs.
  • The Side_ID indicates the position within the domain.
  • The Location_ID indicates the position of the Equipment relative to the Side_ID.
  • The Interface_ID defines which of the two redundant Ethernet channels is the source of the frame (Interface_ID=1: Line-A or Interface_ID=2: Line-B).

The IP addresses are set up according to a scheme similar to that of the MAC addresses. This results in a fixed relationship between IP and MAC addresses (for details, see ARINC 664 Part 4 and Part 7).

Validity of an AFDX frame

An Ethernet packet is accepted as a valid AFDX frame if the following conditions are met:

  • Source MAC address begins with 0x0200000xxxxx
  • Destination MAC address contains the bit pattern xxx xx11 in the leading byte
  • Packet size lies within the allowable range (60 to 1514 bytes)
  • Ethernet type = 0x800 (IPv4)
  • if VN5610 is used: no frame errors that are detected by the hardware
