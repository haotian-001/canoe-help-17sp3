Log Status and Bus Statistic Events

 

Status Events

Now, status events are also saved in BLF and ASC logging files. They contain detailed information on all state changes of the Ethernet adapters being used. This makes it possible to detect any link problems or changes in bitrate, etc., such as when replaying in Offline mode.

Cross Reference

For details see file CAN_and_General_BLF_format.pdf in theCANoe installation directory.

Bus Statistic Events

Until now, bus statistics events were written to logging files, but only in BLF format. So, no statistical values, or just rudimentary values, were available when replaying ASC files in Offline mode.

Since version 8.2 of CANoe the bus statistics events are now also saved in ASC format.

Note

This continues to apply to the replay of PCAP files as well.

There have been changes to the bus statistics information in logging files; now the values are saved as lines, and some new statistical values were added, while others were eliminated. Old logging files can still be played back, but with loss of information; the old event attributes are assigned to new event attributes as follows:
