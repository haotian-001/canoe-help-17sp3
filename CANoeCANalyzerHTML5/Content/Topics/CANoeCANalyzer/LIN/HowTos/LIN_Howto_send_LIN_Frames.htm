How to Send a LIN Frame

 

Sending of Reconfiguration Commands

Normally, reconfiguration commands are defined in a separate table. Headers defined in this table are only sent once after a power-on or a reset. This behavior can be simulated in CANoe. By using the CAPL-function LINChangeSchedTable in on preStart, it is possible to start with any table. In order to change to another table after having sent all the headers of the first one, the CAPL-function LINChangeSchedTable should be called again in on preStart. This time with the ID of the table to be used after reconfiguration and with the third parameter set to –1. This third parameter simply means that the schedule table change should take place on completion of the currently configured table.

Note

When sending reconfiguration commands, the transmission data for the frame 0x3C (MasterReq) will be changed permanently.

Sending of Diagnostic Frames

If diagnostic frames are defined in Free Format, then it is possible to specify the transmission data in the LDF file. In this case, the transmission data are automatically configured during sending of the table.

Note

When sending diagnostic frames in Free Format, the transmission data for the frame 0x3C (MasterReq) will be changed permanently.

Sending of Event Triggered Frames

If a collision occurs while sending an Event Triggered Frame (e.g. receive error with invalid ID or checksum error), then the internal scheduler will attempt to resolve the collision as follows:

  • During subsequent passes of the table, headers for the associated frames are sent successively instead of the event-triggered frame.
    The order in which the headers for the associated frames are sent is determined by their order in the LDF-file.

With each pass of the table only one associated frame header is sent.
The cycle time of the other headers defined in the table remains unchanged.

  • After headers have been sent for all associated frames, the Event Triggered Frame is sent again.

If the resolution of a collision is interrupted by a schedule table change or a sleep mode command, then the scheduler sends on reactivation the Event Triggered Frame. The collision information is therefore lost.

Sending of Sporadic Frames

In a slot containing a Sporadic Frame, the highest priority associated command, with its transmission flag set, is sent. A frame's transmission flag can be set with the CAPL-function LINSendAsSporadic. If none of the transmission flags for the associated frames are set, then the Sporadic Frame slot remains empty. After sending an associated frame as a Sporadic Frame, its transmission flag is reset.

Sending of Dynamic Frames (LIN 2.0 only)

For LIN 2.0 it is possible to declare an ID dynamic and to assign the ID to different frames. Which frame is sent will be controled with the reconfiguration commands AssignFrameID and UnassignFrameID. With an LDF using dynamic IDs and defining a stable to switch the frames, it is possible to send dynamic frames.
