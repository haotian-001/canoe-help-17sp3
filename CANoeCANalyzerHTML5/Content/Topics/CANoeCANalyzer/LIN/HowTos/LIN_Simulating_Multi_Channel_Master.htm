How to Simulate a Multi-Channel Master (LIN)

 

Note

The sample configuration System Configuration (LIN) contains GWI and GWE as multi-channel master nodes.

To create a configuration with a multi-channel master node, please do as follows:

Define the multi-channel master in the Simulation Setup

  • Create all LIN buses in the Simulation Setup.
  • Create a CAPL node on one of the LIN buses. Add the name of the master node from the LDF file in the CANdb Name field of the network node's configuration dialog.
  • Assign all needed LIN buses to the network node using the Buses tab of the configuration dialog.

The Bus Context - Working with multiple LIN channels in CAPL

Modeling a LIN master in CAPL the used network interface (channel) has to be unique. To facilitate this distinction, a bus context is placed in the CAPL program by the runtime environment. This context identifies the bus, on which an action has to be done. CAPL functions are used to set and access the bus context.

Define the Bus Context in CAPL

In following the steps are described to constitute the bus context.

  • Step 1: Define Variables

Variables are used to realize the bus context in CAPL. For every LIN bus a variable has to be defined to represent the corresponding LIN bus.

Example

variables
{
dword gDoorLeft_context     = 0;
dword gDoorRight_context    = 0;
dword gRoofUnit_context     = 0;
char gDoorLeft[32]    = "DoorLeft" ;
char gDoorRight[32]   = "DoorRight";
char gRoofUnit[32]    = "RoofUnit";
}

Note

The name of the char variable has to be identical with the bus name in the Simulation Setup. Small differences in writing, e.g. a blank, will lead to an error.
  • Step 2: Assign the bus to the corresponding variable

The bus context for the individual LIN buses is assigned in the event procedure On preStart, that is the bus from then Simulation Setup is assigned to the corresponding bus context variable using CAPL function GetBusNameContext.

Example

on preStart
{
gDoorLeft_context     = GetBusNameContext(gDoorLeft);
gDoorRight_context    = GetBusNameContext(gDoorRight);
gRoofUnit_context     = GetBusNameContext(gRoofUnit);

   if(0 == gDoorLeft_context)
writeex( 0, 3, "%NODE_NAME% Error: Cannot determine context for bus: %s", gDoorLeft);

   if(0 == gDoorRight_context)
writeex( 0, 3, "%NODE_NAME% Error: Cannot determine context for bus: %s", gDoorRight);

   if(0 == gRoofUnit_context)
writeex( 0, 3, "%NODE_NAME% Error: Cannot determine context for bus: %s", gRoofUnit);
}

Setting the Bus Context

If an action in CAPL has to take place for a specific LIN bus, the corresponding bus context has to be set. To set the bus context function SetBusContext is used.

  • Example 1: Buses have to be set into sleep mode.

Example

SendLINClustersInSleepMode ()
{
SetBusContext(gDoorLeft_context);
LINSendSleepModFrm(0, 0 , 0xFF);

 SetBusContext(gDoorRight_context);
LINSendSleepModFrm(0, 0 , 0xFF);

 SetBusContext(gRCInterface_context);
LINSendSleepModFrm(0, 0 , 0xFF);
}
  • Example 2: Set the schedule table on the event procedure on sysVar.

Example

on sysVar SysVarScheduleTableL1
{
   SetBusContext(gDoorLeft_context);
   LINChangeSchedtable(@this);
}
  • Example 3: Awake all buses when a wake-up frame is detected on a specific channel.

Example

on linWakeupFrame
{
if(this.MsgChannel == 7)
{
SetBusContext(gDoorLeft_context);
linSendWakeup();
LINChangeSchedtable(1);

     SetBusContext(gDoorRight_context);
linSendWakeup();
LINChangeSchedtable(1);

     SetBusContext(gRCInterface_context);
linSendWakeup();
LINChangeSchedtable(1);
}
}

React on LIN Events  - Working with this.MsgChannel

In CAPL the execution of program blocks is controlled by events. These program blocks are named as event procedures. Within an event procedure there may be reactions to LIN events. The keyword this designates the data structure of the object within the event procedure. The keyword this and the selectors can be used to access the data of the event that has just been received.

If a LIN event occurs, the selector MsgChannel can be used to find out on which channel the event has been received.

  • Example 1: Detect on which channel (bus) the SleepModeEvent has been sent

Example

on linSleepModeEvent
{
if(this.MsgChannel == 1  && this.lin_IsAwake == 0)
{
Write("LIN 1 is asleep!");
//For more actions on that bus, the bus context has to be set
SetBusContext(gDoorLeft_context);
}
if(this.MsgChannel == 2  && this.lin_IsAwake == 0)
{
Write("LIN 2 is asleep!");
//For more actions on that bus, the bus context has to be set
SetBusContext(gDoorRight _context);
}
}
  • Example 2: There are messages with identical names on different buses. Detect on which channel (bus) the message has been sent.

Example

on linFrame RCR_01
{
if(this.MsgChannel == 1)
{
write("RCR_01 Message received on LIN 1");
}

  if(this.MsgChannel == 2)
{
write("RCR_01 Message received on LIN 2");
}
}
