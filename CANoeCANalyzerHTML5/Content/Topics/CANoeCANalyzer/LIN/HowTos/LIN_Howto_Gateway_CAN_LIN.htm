How to simulate a CAN-LIN Gateway

 

A CAN-LIN gateway works on the basis of a common network node (gateway node) on a CAN and a LIN network. The network node shall have the same name in both databases. The network node in the Simulation Setup shall reference this node. The node contains a CAPL program for both networks, which routs messages between these two buses.

To configure a CAN-LIN gateway, please do following.

Create a configuration

Create a configuration containing a CAN and a LIN network. Assign a CAN database for the CAN network and a LIN database (LDF file) for the LIN network.

Configuration of the CAN network

  • Add a node to the CAN database, e.g. with the name Gateway.
  • Add a node to the CAN network in the Simulation etup. The database and the network node in the Simulation Setup has to be connected. Please assign the Gateway node from the database to the network node in the Simulation Setup using node context menu Configuration…
  • Add an empty CAPL program to the gateway node.

Configuration of the LIN network

  • Add a master node to the LIN database named as Gateway.

Notes

  • In a LIN network the master node has to be the Gateway node.
  • While the gateway node contains the same name in both networks the node is automatically identified as gateway node.
  • Copy the gateway node from the CAN network and add it to the LIN network using context menu Paste Shortcut. With this action the node is automatically created as a gateway node in the LIN network, visible on the sign on the bottom left corner, see figure. Also the gateway node is automatically assigned to the LIN network, which can be checked using context menu path Configuration|Buses.

Configuration of the CAPL program

  • Open the CAPL program and describe, which messages shall be routed from the CAN network to the LIN network and reverse.

Example

A sample CAPL program can be found in the CAN-LIN gateway sample configuration.

Note

If you create the gateway configuration using the configuration wizard, the gateway node is identified on both buses automatically.

Therefore you have to create both databases before containing a gateway node in both of them.
