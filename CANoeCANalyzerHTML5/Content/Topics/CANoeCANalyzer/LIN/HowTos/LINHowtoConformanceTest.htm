How to Test Conformance of my LIN Node

 

LIN Slave Conformance Tester

  • Add a Test Environment to your configuration.
  • Select Insert LIN Slave Conformance Tester in the context menu of the Test Environment.
  • Open the Conformance Tester's configuration dialog.
  • Select the bus on which you want to test a slave node.
  • Select the slave node to be tested.
  • Select the data source (LDF or NCF) which should be used to parameterize the test specification.
  • Perform further settings if wanted (e.g filename for report or test specification)
  • Leave the dialog via the button Generate XML. A XML specification will be generated which is parameterized with the selected data source.
  • If you test a real slave node, answer the popup message if all nodes on the bus should be deactivated with yes.
  • If you want to simulate the test deactivate all nodes except the tested one.
  • Open the Test Module Execution dialog (if not already opened) with a double click on the Conformance Tester in the Test Environment.
  • Start the measurement; Start the test within the Test Module Execution dialog.
  • When the tests are finished the test report can be opened with the Test Module Execution dialog.
  • Conformance Tests can be performed for the following protocol versions: LIN1.3, LIN 2.0 and SAE J2602.
  • There are also tests to perform a LIN 2.0 Conformance Test without Reconfiguration. These tests are a development of Vector Informatik corporation. These tests can be activated by deselecting the reconfiguration services in the IUT Settings dialog.
