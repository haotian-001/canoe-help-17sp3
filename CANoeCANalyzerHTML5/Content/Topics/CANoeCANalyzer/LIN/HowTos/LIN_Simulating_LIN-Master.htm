How to Simulate a LIN Master

 

General

On a LIN network, a bus node (network node) has to act as the LIN master. The LIN master is responsible for transmitting LIN headers on the LIN network which are responded to by the LIN slaves. The cyclical, time-controlled transmission of LIN headers is called scheduling. Accordingly, the hardware or software component which executes scheduling is called a scheduler.

In CANoe, an internal scheduler is set up through the assignment of a network node as an LDF master node. This scheduler supports all LIN frame types. For LIN 2.0 and LIN 2.1, reconfiguration commands, diagnostic messages and headers for event triggered frames and sporadic frames are also transmitted.

The assignment of a node as an LDF master is done in the node configuration dialog of the Simulation Setup.

Methods

The following methods for the simulation of a LIN Master are available:

1. Using schedule tables

  • Use a LIN database to configure a Master node and its schedule tables
  • Assign the LIN database to a LIN channel
  • Configure the master nodes as follows:
  • Create a network node on the suitable channel and assign the master node from the database to it in the configuration dialog.

Note

This method automatically activates the Master mode of the LIN hardware.
  • The LIN Interactive Master can also be used; it can be added in the Simulation Setup. You can switch between tables during the measurement with this.

Note

The new LIN Interactive Scheduler (LIN ISC) component is available with CANoe 11.0 and higher. This replaces the LIN Interactive Master and makes additional functions available.

2. Using the LIN Interactive Scheduler (LIN ISC)

  • Create an LIN Interactive Scheduler (LIN ISC) in the Simulation Setup.
  • Configure a LIN frame and select in its Frame column the option Header.

Notes

  • This method will only work if the Master mode of the LIN hardware is activated.
  • This method may be used simultaneously with schedule tables defined by LIN database or by CAPL APIs.

3. Using CAPL function linTransmitHeader()

Using this function, message headers can be put on the bus from CAPL.

Notes

  • This method will only work if the Master mode of the LIN hardware is activated.
  • Transmission via linTransmitHeader() can be executed at the same time as schedule tables from the LIN database or theLIN ISC.
  • The linTransmitHeader() function replaces older methods for transmitting LIN headers using output() with linFrame.RTR = 1.

4. Using LIN Replay Block

  • Define a Replay block LIN in the Simulation Setup.
  • Using the LIN Options page of the Replay Block configuration dialog, set the Replay Mode to Send header
  • To simulate a LIN scheduler, you can use the Repeated transmission option in the Replay configuration dialog. To create a defined space between the final header of the log file and the repeated first header, you can use the Delayed by … ms option in the Time condition for start section of the Replay configuration dialog.

Notes

  • This method will only work if the Master mode of the LIN hardware is activated.

  • This method may be used simultaneously with schedule tables defined by LIN database or by CAPL APIs.

5. Use of a modeling library

Some Vector products such as vVIRTUALtarget use co-called modeling libraries which run as a component of a network node. Such modeling libraries can use internal functions of CANoe for transmitting LIN headers and also represent a LIN master node accordingly.

Switching between master nodes and restricting the output of LIN headers

With CANoe 11.0 and higher, it is possible to configure multiple different master nodes in the Simulation Setup of a channel and to only simulate precisely one of these master nodes. Only the active master node transmits LIN headers with such a configuration. Each master node can be simulated with one of the methods listed above.

Switching between the master nodes is carried out using the linSetMasterNode() function. The master node to be activated can be referenced either using the name or title of the node or using an internal node ID. The internal node ID of the currently active master can be queried with linGetMasterNode(). The linSetMasterNode() function returns the internal node ID of the previously active master node.

The first call of linSetMasterNode() restricts the output of LIN headers to the active master node; the output of headers by other nodes is suppressed. This restriction can be lifted using linResetMasterNode, all nodes can transmit LIN headers again after the call.

By selecting a dedicated master node using linSetMasterNode(), it becomes easy to switch between different methods which output the sequences of LIN frames during the simulation.

Example

For example, a Simulation Setup could contain an LDF master node, a Replay Block and a CAPL test node which are to output a defined sequence of LIN frames. To replay the LIN frames of the Replay Block without chronologically mixing them with frames output by the LDF master, a simple CAPL program would call linSetMasterNode() with the name of the Replay Block as the parameter, followed by replayStart(). When the Replay Block is finished, linSetMasterNode() can be called again with the node ID returned from the first call in order to switch back to the original master. The CAPL test node, on the other hand, can then designate itself the master node (by calling linSetMasterNode() with no parameters) also in order to switch back to the LDF master following completion of the test.

When switching occurs from an LDF master node to another node, the LIN scheduler is stopped and the current status of the scheduler (active or inactive) is recorded. When switching back to the LDF master node, this status is restored.

The following blocks in the Simulation Setup can be activated with linSetMasterNode() as the LIN master:

  • ECU node in the Simulation Setup
  • Test node in the Simulation Setup
  • Replay Blocks
  • Interactive Scheduler (ISC)
