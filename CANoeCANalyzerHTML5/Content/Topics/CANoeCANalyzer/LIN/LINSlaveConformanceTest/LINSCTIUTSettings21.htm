IUT Settings LIN 2.1

Test Setup for Test Modules|Slave Conformance Test Module|Configuration...|IUT Settings...

IUT settings not specified in the database can be configured here.

Note

Depending on the settings configured in this dialog the test cases will be automatically activated or deactivated.

Common Settings

  • Supports interbyte >= 1 bit
    This check box should only be deactivated, if the IUT's interbyte space is not always larger than or equal to 1 bit. All test cases requiring interbyte space will be deactivated, in order to prevent inconsistent test results.
  • Sends wake-up signal
    This check box should only be deactivated, if the IUT is unable to send a wake-up sequence. All test cases for wake-up sequences will be deactivated.
  • Deviant initial node address (NAD)
    Initial node address value is a mandatory parameter for performing conformance tests.
    Here it is possible to set it or to override the value defined in the assigned database.
  • Deviant variant ID
    Here it is possible to override the Variant ID value defined in the assigned database with a manually entered value.
  • Deviant Supplier ID
    Here it is possible to override the Supplier ID value defined in the assigned database with a manually entered value.
  • Deviant Function ID
    Here it is possible to override the Function ID value defined in the assigned database with a manually entered value.

Configuration Services

  • Reconfigure before each test case
    The IUT initialization (reconfiguration) is required before each test case.
    This is normally only needed for unconfigured slaves, i.e. for slaves which have no valid configuration after a hardware reset.
  • Supports AssignNAD (0xB0) service
    This check box should be deactivated, if the IUT does not support this optional LIN2.x service.
  • Supports ConditionalChangeNAD (0xB3) service
    This check box should be deactivated, if the IUT does not support this optional LIN2.x service.

  • Supports SaveConfiguration (0xB6) service
    This check box should be deactivated, if the IUT does not support this optional LIN2.x service.

Persistent Configuration Memory

This setting describes how the IUT remembers its configuration (Configured NAD and frame PIDs). It is relevant for several ISO17987 test cases.

  • None
    The IUT has no valid configuration after it is powered up.
    Corresponds to the Unconfigured slave node in ISO17987 -3, section 6.3.2.
  • Readonly
    The IUT has a fixed, valid configuration after it is powered up.
    Corresponds to the Preconfigured slave node in ISO17987-3, section 6.3.2.
  • Writable
    The IUT stores its configuration persistently, either automatically or using the SaveConfiguration (0xB6) service. If the service is supported, it will be used by the Tester.
    Corresponds to the Full configured slave node in ISO17987-3, section 6.3.2.

Device Class

The integration level of the IUT can be specified here. Some test cases have a different test procedure depending on these settings

  • Class B
    Microcontroller-based devices with RxD and TxD connectors.
  • Class C
    Integrated devices with microcontroller, transceiver and analog LIN bus connector.

Transport Layer

All transport layer tests for Slaves can be performed only if IUT supports Functional Requests, Segmented Requests and Segmented Responses. The test system has to be aware of services to be used and data to expect in responses. The services and their data have to be defined in a separate text file, which can be opened by clicking the [Edit] button.
