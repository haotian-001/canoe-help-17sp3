LIN Slave Conformance Test

Test Environment Context Menu: Insert LIN Slave Conformance Tester

General

This special test module allows the conformance of a Slave node (IUT) to be easily tested using a CANoe configuration. The slave conformance test module can be inserted in the Test Setup using the context menu. The test module generates a XML test specification for the selected slave node. The generic test cases are implemented in CAPL libraries, which are automatically assigned to the test module. The test cases can be controlled via the test module's execution window. It is currently possible to perform slave conformance tests for the LIN protocol versions 1.x, 2.x, ISO17987, SAE J2602:2012, and SAE J2602:2021. The Vector hardware VN1611 is currently not supported.

Notes for LIN 2.1

With the VN modules (VN8900, VN1600) all tests (including the sampling point test 3.17) can be performed strictly according to the LIN2.1 conformance test specification.

Notes on Test Coverage

  • The slave conformance test module supports tests for layer 2 and 3.
  • Physical tests and EMC tests are not supported.

Configuration Dialog

Select Configuration... in the context menu of the LIN Slave Conformance test module to open this dialog.

Test Generation page

The conformance test module is automatically assigned to the appropriate LIN network. The XML test specification can be either generated from an LDF file or from an NCF file. Per default the LDF file is used, which is currently assigned to the selected LIN network. If NCF is selected, a file selection control for the NCF file will appear. The assigned NCF file can be viewed or edited by pressing the button for the LDF Explorer utility. The IUT can be selected via the drop down control for Slave nodes. The protocol version of the selected node is also shown.

  • IUT settings LIN 1.x
  • IUT settings LIN 2.0
  • IUT settings LIN 2.1 / ISO17987
  • IUT settings SAE J2602:2012
  • IUT settings SAE J2602:2021
  • IUT initialization settings
  • Test system settings

Report page

The report generation is activated per default. Here it is possible to set the name and the location of the resulting report.

Report Filter page

The resume reasons are deactivated per default, since this information is rarely required.

Libraries page

The following CAPL libraries are automatically assigned:

LIN2.0_Slave_Conformance_Test_Lib.can,
LIN2.0_Slave_Conformance_Test_NoReconf_Lib.can,
LIN1.3_Slave_Conformance_Test_NoReconf_Lib.can,
J2602_Slave_Conformance_Test_Lib.can,
LIN2.1_Slave_Conformance_Test_Lib.can, and
J2602_2021_Slave_Conformance_Test_Lib.can

The libraries contain the generic implementations of the test cases.

XML Specification page

The XML specification file can be selected here.

Notes

  • Generation of the XML specification: On leaving the configuration dialog with OK the XML specification is generated and the test module is updated. Since active simulated nodes can lead to incorrect test results it is recommended to deactivate all simulated nodes when testing a real slave node.

  • To perform all LIN 2.1 tests, VN modules (VN8900, VN1600) are required and recommended. The hardware has extended stress functionality and higher accuracy.

  • LIN 2.x tests can be automated using Vector's VH1100 test hardware or the VT7001 module (part of the VT System family). Both can automatically reset the IUT and detect sleep/awake mode by measuring the IUT's current consumption.
  • On using NCF only, i.e. when no LDF is assigned to the LIN network, the LIN protocol version in the Network Hardware Configuration dialog has to be adjusted manually to match the protocol version in the NCF.
