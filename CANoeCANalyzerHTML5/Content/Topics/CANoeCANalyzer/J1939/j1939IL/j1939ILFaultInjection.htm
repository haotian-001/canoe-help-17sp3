Fault Injection

 

With test modules you can influence the behavior of the J1939 Interaction Layer to inject errors. So you can produce communication errors within a simulation of the remaining bus. The SUT (System Under Test) must react on that errors.

The J1939 Interaction Layer must be assigned to one or more nodes in the Simulation Setup and sends the corresponding Parameter Groups. The signal values can be changed within test modules.

Within a test module the following CAPL functions are available to influence the behavior of the J1939 Interaction Layer:

Function

Short Description

TestDisableMsg

disables the transmission of a Parameter Group

TestEnableMsg

enables the transmission of a Parameter Group

TestEnableMsgAllTx

enables all Tx Parameter Groups

TestDisableMsgAllTx

disables all Tx Parameter Groups

TestSetMsgEvent

stimulates the transmission of a non-cyclic Parameter Group

TestSetMsgCycleTime

changes the cycle time and enables the cyclic transmission of a Parameter Group

TestResetMsgCycleTime

resets the cycle time and the send type to the values defined in the database

TestSetMsgDlc

changes the DLC of a Parameter Group

TestResetMsgDlc

resets the DLC of a Parameter Group to the value defined in the database

TestResetAllFaultInjections

resets all values to the initial values defined in the database

Note

The CAPL functions uses Parameter Group names from the database as parameters. These Parameter Group names must be assigned to a node (sender) explicitly. If the Parameter Group is assigned to more than one node, the unique assignment is not possible. In this case the Parameter Groups must be copied in the database and assigned to the corresponding senders.

Note

The J1939 Interaction Layer does not support further CAPL functions that are available for the Vector CAN Interaction Layer.
