States

 

Depending on the state of the J1939 IL different actions are performed.

State

Description

Initialized

At the start of measurement the J1939 IL is in this state. It switches automatically to state Claiming, if the function J1939ILControlInit does not suppress this behavior.

Claiming

During Address Claiming phase the J1939 IL is in this state. No messages are sent and the J1939 IL reacts only on Address Claiming messages.

After successfully claiming an address, the state is changed to Active.

Active

The J1939 IL is active. Messages are sent according the configured send types.

Stopped

The J1939 IL is deactivated. No messages are sent and signals cannot be set. On change to this state a Cannot Claim Address message is sent.

The J1939 IL changes to this state if, for example, a conflict occurs during address claiming, the advanced address claiming support is not activated and the J1939 IL loses address arbitration.

How the behavior of the simulated node can be influenced after a lost address claim depends on the node attribute NmJ1939AAC and is described here.

Suspended

No messages are sent in this state.

State Transition

Description

1

At the start of measurement or when the function J1939ILControlStart is called, the J1939 IL is activated and Address Claiming is started.

If NMT is not activated, this state is skipped.

2

If Address Claiming is completed successful, the J1939 IL switches to state Active.

  • cyclic messages are sent
  • setting of signals is possible

If Address Claiming was not successful the J1939 IL switches to state Initialized.

3

With the function J1939ILControlStop the J1939 IL switches to state Stopped.

  • cyclic messages are not sent
  • setting signals is not possible
  • a Cannot Claim Address is sent

4

With the function J1939ILControlStart the J1939 IL is started again.

  • Address Claim is performed
  • cyclic message are sent
  • setting of signal is possible

5

The function J1939ILControlWait shifts the J1939 IL to state Suspended.

  • cyclic messages are not sent
  • setting of signals is not possible

6

With the function J1939ILControlResume the J1939 IL can be started again.

  • cyclic message are sent again
  • setting of signals is possible

7

If J1939ILControlStop is called during state Suspended, the J1939 IL switches to state Stopped.

  • a Cannot Claim Address message is sent

The state of the J1939 IL can be get with the function J1939ILGetState. On a state transition the callback function J1939ILOnChangedState is called.

With system variables it is possible too to request and change the state and address of the J1939 IL.

It is also possible to retrieve the state of the J1939 IL with the environment variable Ev<node name>_ILState (but it is not possible to change the state with this environment variable). With the environment variable Ev<node name>_ILEnable> the J1939 IL can be activated or deactivated.

To control the state of the J1939 IL following CAPL functions are available:

  • J1939ILControlInit
  • J1939ILControlStart
  • J1939ILControlStop
  • J1939ILControlWait
  • J1939ILControlResume
