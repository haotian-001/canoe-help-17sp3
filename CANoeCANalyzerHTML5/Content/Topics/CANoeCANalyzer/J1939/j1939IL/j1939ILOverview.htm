Interaction Layer

 

The J1939 Interaction Layer (J1939 IL) provides signal based access to the J1939 bus. The mapping of the signals on the Tx messages is done accordingly to the configuration in the used database. The J1939 IL manages the Tx messages, Rx messages are handled directly by CANoe and can be processed in CAPL.

The J1939 IL must be assigned to a node in the Simulation Setup. The J1939 IL transmits the Tx messages which are assigned to the node in the database. With attribute the transmission properties of the Tx message can be configured.

The signal values can be modified via CAPL, Panels or environment variables. The J1939 IL send the message of the signal according the configured send type. It is also possible to modify the signal value of any node in the Simulation Setup (which is using the J1939 IL) from a test module (J1939ILSetSignal). A test module can control several nodes. A device under test (DUT) can be supplied with messages/signals by this mechanism.

The J1939 Interaction Layer provides the following features:

  • sending of cyclic messages
  • sending of messages on change/write of a mapped signal
  • signals can be connected to environment variables
  • sending of messages on request (SAE J1939-21)
  • support of Address Claiming (SAE J1939-81)
  • support of transport protocols (SAE J1939-21, BAM, RTS/CTS)
  • support of J1939 diagnostics (SAE J1939-73)
  • support of Functional Safety (SAE J1939-76)

Remote Control of J1939 IL Nodes by Test Node or Test Units

To address a simulated node that uses the J1939 IL from a test node, the IL offers one or more additional forms for most CAPL functions. With these function forms, the simulated node to which the function is to be applied is specified as the first parameter. Such forms can also be recognized by the fact that they are only available for Test nodes in the Availability table row:

If you want to use the functions of the J1939 IL in vTESTstudio, you have to enter the library J1939_IL.dll as a modeling library. Open therefor the configuration of the test unit. On the Modeling Libraries page you can add the file J1939_IL.dll. It is a good practice to first copy the J1939_IL.dll from the CANoe folder Exec32 and store it in a subfolder of the vTESTstudio project and then add this copy to the vTESTstudion project as described above. This copy will be used during the development of the test unit. When the test unit is executed in CANoe, the J1939_IL.dll from the CANoe installation is used.
