Configure Database

 

The J1939 IL is configured through the assigned database. The node must be defined in the database.

The J1939 IL must be assigned to the node. For this the node attribute NodeLayerModules must have the value J1939_IL.DLL. The J1939 IL is implemented as DLL, which is located in the exec32 directory of the CANoe installation.

Furthermore the address must be defined, which the J1939 IL should use as source address. The address must be defined with the node attribute NmStationAddress.

Note

The input field Address in the node configuration dialog of the CANdb++ Editor contains the value of the node attribute NmStationAddress.

Each message which should be sent by a node with the J1939 IL, must be assigned to the node as Tx message. Furthermore the send type must be configured and depending on the send type more attributes must be defined.

In principle the transmission behavior of all messages is influenced by the send types of the message and all contained signals. The results of all send types are prioritized in the following order beginning with the highest priority:

  • spontaneous transmission
  • fast cyclic transmission
  • cyclic transmission
  • no transmission

GenSigILSupport

Signal

Enum

signal is handled by the J1939 IL

  • No: J1939 IL ignores this signal
  • Yes: J1939 IL handles this signal

GenSigSendType

Signal

Enum

send type of signal

  • Cyclic: message of this signal is sent cyclic with GenMsgCycleTime
  • OnWrite: message of this signal is sent spontaneously once, if signal is set
  • OnWriteWithRepetition: same as OnWrite but with GenMsgNrOfRepetition repetitions
  • OnChange: message of this signal is sent spontaneously once, if the signal is changed
  • OnChangeWithRepetition: same as OnChange but with GenMsgNrOfRepetition repetitions
  • IfActive: Message of this signal is sent cyclic with GenMsgCycleTimeFast if this signal is active
  • IfActiveWithRepetition: same as IfActive but with GenMsgNrOfRepetition repetitions on transition to signal inactive state
  • NoSigSendType: signal does not define the send type of the message

GenSigInactiveValue

Signal

Integer

raw value of the signal at which it is in an inactive state

This attribute corresponds with send type IfActive and IfActiveWithRepetition.

GenSigStartValue
