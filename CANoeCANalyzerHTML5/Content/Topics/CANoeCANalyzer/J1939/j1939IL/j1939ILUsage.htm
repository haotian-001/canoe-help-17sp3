Setup a Node

 

This section shows the steps, which are necessary to use the J1939 Interaction Layer with a node in the Simulation Setup. The J1939 IL is implemented as node layer and extends the features of a node. There a more CAPL functions available, but it is also possible to use the J1939 IL without CAPL programming.
  • First there must be setup a node in the CANdb++ Editor. The easiest way is to use the database J1939.dbc or J1939base.dbc, which is delivered with CANoe, as base. These databases contains all necessary attributes.
    The databases are located in the directory Database of the J1939 examples.
  • The J1939 IL must be assigned to the node. For this the node attribute NodeLayerModules must be set to the value J1939_IL.vmodule. The attribute can be edited in the node configuration dialog of the CANdb++ Editor on the Attribute page.
  • The address of the node must be defined. The address must be entered in the Address field in the node configuration dialog of the CANdb++ Editor. As alternative, the node attribute NmStationAddress can be edited.
  • Additionally the 64 bit device name of the node should be defined for the Address Claiming. For this several NmJ1939-Attribute must be defined.
    These attributes are available on the Attribute page of the node configuration dialog of the CANdb++ Editor.
  • The node can now be inserted into the Simulation Setup of CANoe. To create a new configuration the Model Generation Wizard can be used. It is available in the start menu under CANoe/Tools. In the generation parameters J1939 Modeling must be selected.

After these steps a base configuration for use with J1939 IL is completed. As next step Tx messages must be assigned to the node.
