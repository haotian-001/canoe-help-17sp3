J1939-82 Compliance Tests Overview (2015) - A10

Precondition DUT

Strategy

Constriction / Limitation / Typical Errors / Resolution Notes

1

CA NAME
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).
  1. Request for Address Claim.
  1. Request AC from DUT and await AC response within Tr (200 ms).
  2. Check AC responses payload equals to the configured NAME.
  1. Fails, if DUT does not react to the request with proper AC within T3 (1250 ms).
  2. Fails, if payload of DUT's AC response does not fit configured NAME.

2

System Initialization
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).
  1. Address Claim on Bootup.
  1. Tool opens a dialog and instructs user to reboot the DUT within 5 minutes.
  2. Tool detects DUT's bootup AC and checks SA against configured DUT address.
  1. Fails, if DUT does not send an AC within the waiting time of max. 5 minutes.
  2. Fails, if user cancels the dialog.
  3. DUT sends AC from an unexpected source address (different from the configured one).
  4. Fails, if DUT does not reset / reboot with proper AC when instructed to reset it.

3

Non-Configurable Address CA
  1. DUT address.
  2. NAME (with AAC).
  3. Tool address (defaults to 0xF9).
  1. Non-Configurable Address Device or Service-Configurable Address Device (AAC unset).
  1. If device has AAC bit set in it's device name, the Tool is inconclusive.
  2. Tool tries to use the DUT's address with high prio AC.
  3. Check that DUT sends Cannot Claim Address message within 250 ms.
  4. Check the Cannot Claim message's content fits the configured NAME.
  5. Check DUT doesn't send any messages and doesn't answer any requests for at least 30 seconds.
  6. Tool opens a dialog and instructs user to reset and restart the DUT within 5 minutes.
  1. Inconclusive, if AAC bit is set, due Tool is not suitable for this type of devices.
  2. Fails, if user cancels the reset dialog.
  3. Fails, if DUT does not send the Cannot Claim message within 250 ms.
  4. Fails, if DUT sends the Cannot Claim message from unexpected SA. (0xFE is expected).
  5. Fails, if DUT sends the Cannot Claim message with a wrong payload (NAME).
  6. Fails, if DUT does transmits any messages in listen only mode within at least 30 seconds.
  7. Fails, if DUT reacts on a request when in listen only mode.
  8. Fails, if DUT does not reset with proper AC when instructed to reset it.

4

Commanded Addresses
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).
  4. Supports Commanded Address.
  1. DUT has to support Commanded Address (0x00FED8).
  1. Check if DUT is configured to support Commanded Address, else Tool is inconclusive.
  2. Send Commanded Address to the DUT and await AC from DUT within Tr (200 ms).
  3. Check payload from DUT's AC to be the configured NAME.
  4. Check DUT uses the new address or it's current address in it's AC.
  5. Tool opens a dialog and instructs user to reset and restart the DUT within 5 minutes.
  1. Inconclusive, if DUT is configured not to support Commanded Address (within J1939-CT_Configurator).
  2. Fails, if user cancels the reset dialog.
  3. Fails, if DUT does not reset with proper AC when instructed to reset it.
  4. Fails, if DUT does AC but with unexpected payload (NAME).
  5. Fails, if DUT sends AC from an unexpected address. (response timeout).

5

Self-Configurable Address CA
  1. DUT address.
  2. NAME (without AAC).
  3. Tool address (defaults to 0xF9).
  1. Self-Configurable Address Device (AAC set).
  1. Tool tries to use DUT's address with an AC under NAME of higher prio.
  2. Check for DUT's response for a recalculated address within 250 ms.
  3. Tool tries to use DUT's current address again.
  4. Check for DUT's response for a recalculated address within 250 ms.
  5. Tools opens a dialog and instructs user to reset and restart the DUT within 5 minutes.
  1. Inconclusive, if AAC bit is not set, due Tool is not suitable for this type of devices.
  2. Fails, if DUT does not react on the AC within 250 ms.
  3. Fails, if DUT reacts with Cannot Claim instead of an AC under a new address.
  4. Fails, if DUT uses the same address again instead of calculating a new one.
  5. Fail if DUT attempts to use an address outside the range of 128 to 247.

7

Request for Address Claimed
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).
  1. DUT has to be online with an claimed address.
  1. Request AC globally and check DUT's response within Tr (200 ms).
  2. DUT has to respond with either AC or Cannot Claim.
  3. If DUT answered with Cannot Claim address message, tool opens a dialog and instructs user to reset and restart the DUT within 5 minutes.
  1. Fails if DUT does not react on request within Tr (200 ms).
  2. Fails, if DUT responds with an AC message but with unexpected device NAME.
  3. Fails, if DUT responds with an AC but from an unexpected SA with its NAME.

8

Request for Address Claimed
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).
  1. DUT has to be online with an claimed address.
  1. Request AC directly and check DUT's response within Tr (200 ms).
  2. DUT has to respond with either AC or Cannot Claim.
  3. If DUT answered with Cannot Claim address message, Tool opens a dialog and instructs user to reset and restart the DUT within 5 minutes.
  1. Fails if DUT does not react on request within T3 (200 ms).
  2. Fails, if DUT responds with an AC message but with unexpected device NAME.
  3. Fails, if DUT responds with an AC but from an unexpected SA with its NAME.

9

Address Claimed Cannot Claim
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).

—

Test passes for devices within the predefined address range. For all devices out of the predefined address range:

  1. Check for predefined address range.
  2. Tool opens a dialog and instructs user to reset and restart the DUT within 5 minutes.
  3. Check DUT doesn't send any messages within 250 ms after AC on bootup.
  1. Passes, if DUT's address is within the predefined address range [0;127] or [248;253].
  2. Fails, if DUT does not boot up with AC after reboot.
  3. Fails, if user canceled the dialog.
  4. Fails, if DUT sent an AC from unexpected SA.
  5. Fails, if DUT sent an AC with unexpected device NAME.
  6. Fails, if DUT sends non AC-Messages within 250 ms after it started with AC.

10

Address Claimed Cannot Claim
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).

—
  1. Tool tries to use DUT's address with an AC under NAME of lower prio…
  2. Check that DUT sends AC message with the same address within 250 ms.
  3. If DUT does not send AC message, Tool opens a dialog and instructs user to reset and restart the DUT within 5 minutes.
  1. Fails, if DUT does not respond with correct AC message with same address within 250 ms.

11

Address Claimed Cannot Claim
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).

—
  1. Tool tries to use DUT's address with an AC under NAME of higher prio…
  2. Check that DUT sends AC or Cannot Claim Address message within 250 ms.
  3. If DUT answered with Cannot Claim address message, Tool opens a dialog and instructs user to restart the DUT within 5 minutes.
  1. Fails, if DUT does not respond with correct AC or Cannot Claim within 250 ms.

12

Address Not Claimed
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).

—

Test passes for devices with AAC=1. For devices with AAC=0:

  1. Tool tries to use DUT's address with an AC under NAME of higher prio…
  2. Check that DUT sends Cannot Claim Address message within 250 ms.
  3. Request AC and check DUT responds with Cannot Claim within Tr (200 ms).
  4. Check that no messages other than Cannot Claim have been sent in listen only mode.
  5. Tool opens a dialog and instructs user to reset and restart the DUT within 5 minutes. B) If device has AAC bit set in it's device name. Test passes.
  1. Passes for devices with AAC bit set.
  2. Fails, if DUT does not react with correct Cannot Claim after Tool tries to use its address.
  3. Fails, if DUT does not respond with correct Cannot Claim to AC requests in listen only mode.
  4. Fails, if DUT does sent any other messages than Cannot Claim within listen only mode.

13

Power Interruption
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).

—

PART 1: Power loss <2 ms.

  1. Show a Yes/No dialog to check if the user is able to simulate a power loss < 2 ms.
  2. In the case the user is NOT able, he shall answer with 'NO'. This toolcase then is inconclusive.
  3. In the case the user is able, the dialog also advices him to execute it and answer the dialog with 'Yes'.
  4. After the user clicked 'YES' the buffer is checked that the DUT did not send an initial bootup AC during the this power loss.

PART 2: Power loss >1 s.

  1. Tell the user to simulate a power loss >1 second within a given time and check for an AC from DUT.

Part 1:

  1. Inconclusive if power loss <2 ms is answered to be not possible by the user.
  2. Inconclusive if user does not react on dialog within waiting time.
  3. Fails, if DUT sent unexpectedly AC after user didin't react on the dialog.
  4. Fails, if DUT sent unexpectedly AC after user told not to be able to do a power loss <2 ms.

Part 2:

  1. Inconclusive, if user didn't react and DUT didn't sent any AC within waiting time.
  2. Fails, if DUT sends AC from unexpected SA.
  3. Fails, if DUT sends AC from unexpected NAME.
  4. Fails, If DUT didn't send AC after power loss >1s and user clicked cancel.

14

Network Disruption
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).

—
  1. Tool asks user if DUT is connected and working. Await user's answer.
  2. Tool prompt the user to disconnect and reconnect the DUT some times. Await user's confirmation.
  3. Tool prompt the user to turn the DUT on and off a couple of times. Await user's confirmation.
  4. Tool checks if the Tool communication worked properly during the steps before.
  5. Tool opens a dialog and instructs user to reset and restart the DUT within 5 minutes.
  1. Inconclusive on any dialog timeout.
  2. Fails, if user answers any of the question dialog with 'No'.
  3. Fails, on any Error Frame on the bus.

15

Address Continuity
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).

—

Test successful for devices that neither support Commanded Address nor are "Arbitrary Address Capable" (Bit "AAC" = 0). For all other devices:

  1. Tool prompts user to reset and reboot the DUT within 5 minutes.
  2. Await and check DUT's Address Claiming message.
  3. If DUT supports Commanded Address: then Tool reconfigures the DUT's address using Commanded Address message.
  4. Else If DUT has AAC bit set: then Tool claims DUT's current address and checks if the DUT has changed its address.
  5. Tool prompts user to restart the DUT within 5 minutes.
  6. Check if the DUT answers with a correct address claiming message using the address it obtained before the reboot.
  7. Tool prompts user to reset and restart the DUT within 5 minutes.
  1. Inconclusive, on any dialog timeout.
  2. Fails, if DUT does not send an Address Claiming message after its restart.
  3. Fails, if DUT sends an Address Claiming message from unexpected Source Address.
  4. Fails, if DUT sends an Address Claiming message with unexpected NAME.
  5. Fails, if user cancels any of the dialogs.
  6. Fails if DUT sends Cannot Claim.
  7. Fails, if "Arbitrary Address Capable" DUT (Bit "AAC" = 1) does not recalculate its Source Address (Step 4).

16

Address Violation response
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).

—
  1. Tool prompts user to reset the DUT within 5 minutes.
  2. Wait 2 s.
  3. Tool sends messages from the DUT's SA every 500 ms.
  4. Wait for first AC (reclaim) from DUT within 10 s.
  5. Wait for second AC (reclaim) from DUT within 10 s.
  6. Check AC's time distance >=5 s.
  1. Inconclusive, on reset dialog timeout.
  2. Fails, if DUT does not sends 2 AC's from its current SA to the global address.
  3. Fails, if the time distance of the AC's is <5 s.
  4. Fails, if user cancels reset dialog.

17

System Notification of continued address violation
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).
  4. Addr. Violation DM.

—
  1. Tool prompts user to reset and restart the DUT within 5 minutes.
  2. Wait 2 s.
  3. Tool sends messages from the DUT's SA every 500 ms.
  4. Wait for DM01 from DUT within 30 s, check if there is a DTC (SPN 2000+SA, FMI 31).
  1. Inconclusive, if "DM1 indicates Add.Viol" is not set.
  2. Inconclusive on reset dialog timeout.
  3. Fails, if user cancels reset dialog.
  4. Fails, if DUT does not send a DM01.
  5. Fails, if DUT sends DM01 but without DTC SPN 2000+SA, FMI 31.

18

NAME Management
  1. DUT address.
  2. NAME.
  3. Tool address (defaults to 0xF9).
  4. Name Management.

—
  1. Tool prompts user to reset the DUT within 5 minutes.

For each of 8 qualifier flags:

  1. Check pending:
  • Send NM "Set Pending NAME" with the corresponding qualifier flag set to 0 and the desired NAME.
  • Await NM message from DUT within Tr (200 ms).
  • If DUT answered with "NM NACK", step is passed and next field is checked.
  1. Check adoption:
  • If DUT answers with "NM ACK": validate whether the pending NAME and SA are correct.
  • Send "NM Adopt" for the pending NAME and await AC within 250 ms.
  • Validate whether the NAME and SA in AC Response are correct.
  • Wait 200 ms and continue with the next qualifier flag. (restart with step 2).
  1. Soft reset the DUT's original NAME: (set ALL fields)
  • Send NM Set Pending for change using the all qualifier flags.
  • Await NM message from DUT within Tr (200 ms).
  • If the DUT answered with "NM NACK",the step is recognized as "PASSED".
  • If the DUT answers with "NM ACK": validate whether the pending NAME and SA are correct.
  • Send "NM Adopt" for the pending NAME and await AC within 250 ms.
  • Validate whether the NAME and SA in AC Response are correct.
  1. Final step after all fields are set:
  • Reset the DUT's original NAME over NM command. If this works, test is done.
  • Tool opens a dialog and instructs user to reset the DUT within 5 minutes, if the NM soft reset didn't work.
  1. Inconclusive, if DUT is not configured to support NM.
  2. Fails, if DUT does not react within Tr (200 ms) to any request.
  3. Fails, if DUT sends an unexpected NAME content.
  4. Fails, if user cancels reset.
  5. Fails, if DUT sends AC or NM from unexpected SA.
  6. Fails, if DUT sends unexpected NM response (only NACK and ACK are expected).
  7. Fails, if DUT does not send AC after NM Adopt.
