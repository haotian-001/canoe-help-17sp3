J1939-82 Compliance Tests Overview (2015) - A6

 

Reference Table in the SAE Specification: A6 — DUT RESPONDS TO BAM

Precondition DUT

Strategy

Constriction / Limitation / Typical Errors / Resolution Notes

1

Verify DUT inspects the PGN (including DP and EDP) in the TP.CM_BAM message
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Tool commands the DUT to change its address using CA.
  2. Check, if the DUT reacts with an Address Claimed using the new address.
  3. Tool uses a wrong PGN for the commanded address message by setting the DP/EDP bits (0x01FED8 and 0x02FED8). (These are no CAs!).
  4. DUT shall not react on the wrong CAs within T3 (1250 ms).
  5. Reset the DUT address using a final CA.
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Timeout if DUT does not react within Tr (200 ms) to the first CA.
  3. Fails, if DUT reacts to a wrong CA.
  4. Fails, if DUT sends wrong content within the ACL.

2

Ensure DUT validates TP.CM_BAM message parameter values (content and format)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Tool tries to command the DUT to change its addres using an invalid BAM.
  2. DUT shall neither react on the wrong CAs within T3 (1250 ms), nor send any TP.Conn_Aborts.
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails, if DUT reacts or sends any TP.Conn_Aborts.

3

Ensure DUT discards BAM transport for an unsupported PGN (content and format)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Fullmode.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Tool sends a bunch of PG's using BAM. (Bruteforce).
  2. Check DUT does neither react, nor send any aborts.
  1. Fails if DUT reacts in any way to the BAM sending of the tool.

Note

If Fullmode is set, then the complete range of available PGN's are used. Else every 10th PGN is used.

4

Ensure the time between BAM transport messages meetsthe timing requirements (DUT as Responder)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  4. DUT has to support Acknowledgments, especially ACK or NACK.
  1. Tool sends correct timed Commanded Address (CA) message.
  2. Check, that DUT reacts with ACL on valid CA.
  3. Reset the DUT address using another CA.
  4. Tool sends incorrect timed Commanded Address (CA) message.
  5. Check, that DUT does ignore the invalid CA and does not send NACKs, ABORTS or ACLs.
  6. Check, that DUT ignores TP.DT's that are sent beyond T1 (750 ms)
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails if DUT reacts in any way to an invalid BAM from the tool.

5

Ensure DUT receives TP.DT data packets for BAM transport when received within timing requirements (DUT as Responder)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  4. DUT has to support Acknowledgments, especially ACK or NACK.
  1. Tool sends correct timed Commanded Address (CA) message.
  2. Check, that DUT reacts with ACL on valid CA.
  3. Reset the DUT address using another CA.
  4. Tool sends incorrect timed Commanded Address (CA) message.
  5. Check, that DUT does ignore the invalid CA and does not send NACKs, ABORTS or ACLs.
  6. Check, that DUT ignores TP.DT's that are sent beyond T1 (750 ms)
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails if DUT reacts in any way to an invalid BAM from the tool.

6

Receiving BAM transport data packets in sequential order (DUT as Responder)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Tool commands the DUT to change its address using CA.
  2. Check DUT reacts with ACL on valid BAM.
  3. Reset the DUT address using another CA.
  4. Tool sends CA within invalid sequenced TP.DTs.
  5. Check DUT does not react on invalid BAM.
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails if DUT reacts in any way to an invalid BAM from the tool.

7

Ensure DUT discards BAM transport when TP.DT data packets are not correct size (DUT as Responder)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Tool tries to command the DUT to change its address using a CA with invalid DLC.
  2. Check, if the DUT does not react to it.
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails if DUT reacts in any way to an invalid BAM from the tool.

8

Ensure DUT accepts BAM transport when Priority of TP.DT data packets are not correct value (DUT as Responder)
  1. DUT address.
  2. Tool address (defaults to 0xF9)
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Tool commands the DUT to change its address using CA containing TP.DT's using all different possible priorites.
  2. Check, if the DUT reacts with ACL to the CA.
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails if DUT ignores CA.

9

Ensure DUT properly reassembles received TP.DT data packets(DUT as Responder)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Tool commands the DUT to change its address using CA containing TP.DT's using the unused (padding) bytes in the last TP.DT.
  2. Check, if the DUT reacts with ACL to the CA.
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails if DUT ignores CA.

10

Ensure DUT doesn’t transmit an End of Message Acknowledge following the reception of the final Data Transfer data packet (DUT as Responder)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Transport Protocol RTS/CTS.
  4. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Tool commands the DUT to change its address using CA.
  2. Check, if the DUT reacts with ACL to the CA.
  3. Check, if the DUT sends any TP.CM_EndOfMsgACKs as a trial to fianalize the transport.
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails if DUT ignores CA.
  3. Fails if DUT sends any TP.CM_EndOfMsgACK after last TP.DT of the BAM.

11

Ensure DUT never transmits a Connection Abort

executes / repeats A6 Rows 1,6,7,8,9.10,12,13 in sequence

executes / repeats A6 Rows 1,6,7,8,9.10,12,13 in sequence

executes / repeats A6 Rows 1,6,7,8,9.10,12,13 in sequence
  1. Executes / repeats A6 Rows 1,6,7,8,9.10,12,13 in sequence.
  2. Fails, if any abort was detected.

12

Ensure DUT supports only one BAM transport from a Controller Application at any given time
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Tool starts sending a long BAM message interrupted by a CA and vice versa. Both variants are done to be simultanous-like (Transports mixed up).
  2. Check DUT does not react to the CA, due simultaneous BAM from one source node lead every BAM to be invalid.
  3. Tool starts sending a long BAM message and interrupts it by sending a CA, but stopping the prevously sending BAM, so the CA can be seen as a complete valid BAM-Transport-Block.
  4. Check that the DUT does not react to the CA: CA transport is to be ignored because it interrupted another uncompleted BAM transmission.
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails if the DUT reacts to at least one CA.

Important Note

Interruption of a BAM is not specified how to do this within the specification and should not be a planned strategy to do something fancy.

A BAM that stops in the middle of the transmission before it is completed is a valid strategy to invalidate a started BAM, but this shall not be used to force such interruption like behaviors.

It is recommended to wait some time between BAM messages that are sent from one node. A good recommended value is T1 (750 ms) between them to avoid mix ups and have the receivers give a chance to detect a timeout and discard the BAM before receiving any other BAM messages from the same sender. T1 implies a Tr (200 ms) and a good value that may be needed for transport of the frame, especially when passing gateways.

To be safe and stable, try to send as polite as possible.

13

Ensure DUT supports multipleBAM transports from different Controller Applications
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. Supports Commanded Address.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Tool sends a long BAM message from an address different of the Tool's and DUT's address.
  2. During the long BAM transport of the third node the tool commands the DUT to change its address. (a BAM and the CA are now sent simultanously from different source addresses and therefore can be accepted).
  3. Check that DU changed the address with ACL.
  4. Reset the DUT's address using another CA.
  5. Repeat the steps by trying to interrupt the CA by another node's BAM.
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. Fails if the DUT does not react on any of the CA with ACL. All CA's are valid, due there are no BAM-Interruptions.

Note

Each node can send one BAM at a time, and this could be done from all nodes simultanously. This means each node has one public communication channel that can be used independantly of the others. Within one channel only one transport at a time can be done. In consequence this leads to the conclusion, that a BAM only can be "interrupted" by the originator itself, but not by others.

Important Note

  • Trying to interrupt a BAM is a bad practice and may lead to unpredictable results.

  • Stopping a started BAM is valid and should be done politely by waiting e.g. T1 (750 ms) before starting another BAM.
