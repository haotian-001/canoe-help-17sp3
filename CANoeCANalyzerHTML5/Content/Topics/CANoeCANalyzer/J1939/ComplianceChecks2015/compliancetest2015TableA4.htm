1

Standard Frame Message Tolerance
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable PGN.
  4. At least one cyclic PGN.
  5. Msg Tolerance Abs [ms].
  6. Msg Tolerance Rel [%].
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to produce cyclic messages.
  1. Generate continuously high frequent low prio standard CAN Frames (11bit) during the test randomly to namingly increase the busload.
  2. Request all requestable messages from the DUT. They have to be answered correctly within response timeout Tr (200 ms).
  3. Wait 4 times the max. cycle time of all cyclic messages.
  4. The cyclic messages have to be sent within the configured abs. and rel. message tolerances to identify they are not "disturbed".
  1. Inconclusive, if there are no requestable or cyclic PGNs configured.
  2. Fails, if any error frames are detected.

3

PGN received independant of priority bits
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non requestable PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT supports Acknowledgments, especially NACK.
  1. Request the configured non requestable PGNs with all possible priorities (0-7).
  2. Await NACK from DUT within response timeout Tr (200 ms).
  1. Inconclusive, if there are no non-requestable PGNs configured.
  2. Fails, if any error frames are detected.

4

Use of EDP Bit to identify PGN
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non requestable PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT supports Acknowledgments, especially NACK.
  1. Request the configured non requestable PGNs.
  2. Await NACK from DUT within response timeout Tr (200 ms).
  3. Do a request "lookalike" request with modified EDP bit (0x02EA00).
  4. DUT shall not react to this "wrong" request. Else it seems it ignores the EDP bit.
  1. Inconclusive, if there are no non-requestable PGNs configured.
  2. DUT reacts with a NACK to a request with a modified EDP bit. The DUT seems not to check the EDP bit within the Frame Header. (0x020000).

5

Use of DP Bit to identify PGN
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non requestable PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to supports Acknowledgments, especially NACK.
  1. Request the configured non requestable PGNs.
  2. Await NACK from DUT within response timeout Tr (200 ms).
  3. Do a request "lookalike" request with modified DP bit (0x01EA00).
  4. DUT shall not react to this "wrong" request. Else it seems it ignores the DP bit.
  1. Inconclusive, if there are no non-requestable PGNs configured.
  2. DUT reacts with a NACK to a request with a modified EDP bit. The DUT seems not to check the DP bit within the Frame Header. (0x0´10000).

6

Filtering on destination address
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non requestable PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to supports Acknowledgments, especially NACK.
  1. Request the configured non requestable PGNs.
  2. Await NACK from DUT within response timeout Tr (200 ms).
  3. Repeat the request but to another destination address different (bitwise NOT) from the DUT.
  1. Inconclusive, if there are no non-requestable PGNs configured.
  2. Fails, if DUT reacts to a request to a foreign node.

7

Supports Receive of Global Destination Address
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. DUT supports Commanded Address..
  4. Device NAME.
  1. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Use an address different from the DUT's and Tool's address and command the DUT to change its address to it.
  2. Await ACL from DUT within response timeout Tr (200 ms) on the new commanded address.
  3. Check if the DUT reacts on the new address with proper payload (Device NAME).
  1. Inconclusive, if DUT does not support Commanded Address (CA).
  2. DUT does not react to the CA with ACL within response timeout Tr (200 ms).
  3. Fails, if the CA for resetting the DUT's address fails.
  4. Fails, if the DUT reacts with ACL but sent a wrong payload not containing the Device NAME. ( new address and padbytes 0xFF within the payload are not checked in this test).

Note

Tests following up may fail if DUT's address could not be reset.

8

PDU Processing Capabilities
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT supports Acknowledgments, especially NACK.
  1. Request confugred non requestable PGNs from the DUT.
  2. Create a P2P burst for about 10 ms.
  3. Check DUT answers with NACK within Tr (200 ms).
  4. Request configured non requestable PGNs from the DUT.
  5. Check DUT answers with NACK within Tr (200 ms).
  6. Request confugred non requestable PGNs from the DUT.
  7. Create a global burst for about 10 ms
  8. Check DUT answers with NACK within Tr (200 ms).
  9. Request configured non requestable PGNs from the DUT.
  10. Check DUT answers with NACK within Tr (200 ms).
  1. Inconclusive, if there are no non requestable PGNs configured.
  2. Fails, if the DUT does not answer the request with NACK within Tr (200 ms).

12

Correct Interpretation of 'Requested PGN' in Request (PGN 59904 - 0x00EA00)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non cyclic PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  1. Request the configured PGNs.
  2. Await the answer and check for proper response that can be a PG or an ACK. If the PG is additionally sent after an ACK, this is ignored, the ACK is seen as a suitable reaction to the request. Timeout after response timeout Tr (200 ms).
  1. Inconclusive, if there are no non cyclic PGNs configured. Accepts any non cyclic PGN.

Proper Response to Destination SpecificRequest for Single Packet Destination Specific (PDU1) PGN and DLC8
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable PDU1-PGN with DLC=8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  1. Request the configured PDU1 PGNs with DLC=8destination specific.
  2. Await the response within response timeout Tr (200 ms).
  3. Check, if the response has a DLC=8 bytes.
  1. Inconclusive, if there are no requestable PGNs of type PDU1 with DLC=8 configured.
  2. Fails if DUT does not respond or the response has a wrong DLC.

22

Proper NACK Response for Destination Specific Request sent to DUT’s Address for a PGN that is not supported by DUT
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non requestable PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to supports Acknowledgments, especially NACK.
  1. Request the configured PGNs destination specific.
  2. Await NACK response within response timeout Tr (200 ms).
  3. Check, if the response is a NACK for the requested PGN, sent to the global address.
  1. Inconclusive, if there are no non-requestable PGNs configured.
  2. Fails if DUT does not respond or does not send the NACK to the global address (0xFF).

23

No Response for Globally Addressed Request for a PGN not supported by DUT
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non requestable PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to supports Acknowledgments, especially NACK.
  1. Request the configured PGNs globally.
  2. Await the NACK response within response timeout T3 (1200 ms).
  3. Check, if the response is a NACK for the requested PGN, sent to the global address.
  1. Inconclusive, if there are no non-requestable PGNs configured.
  2. Fails if DUT does not respond or does not send the NACK to the global address (0xFF).

24

Proper use of ACK Response when Applicable
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support DM3 (clear active).
  3. DUT has to support DM11 (clear).
  4. DUT has to support Acknowledgments, especially NACK.
  1. Request DM3 and DM11
  2. Await the responses within the response timeout Tr (200 ms).
  3. Check the answer to be an ACK and if ACK was sent to the global address (0xFF) with expected payload.
  1. Fails on no reaction after response timeout Tr (200 ms).
  2. Fails, if DUT sends ACK with unexpected control byte.
  3. Fails, if DUT sends ACK with wrong PGN in payload.
  4. Fails, if DUT does not send the ACK to the global address (0xFF).

25

Proper use of Access Denied Response for Destination Specific Request sent to DUT Address
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non cyclic DENY producing requestable PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Acknowledgments, especially DENY.
  1. Request the configured DENY producing PGNs.
  2. Await response within the response timeout Tr (200 ms).
  3. Check the response for ACK-DENY (control byte = 2) that was sent to the global address (0xFF).
  1. Inconclusive, if there are no DENY producing, non cyclic PGNs configured.
  2. Fails on no reaction after response timeout Tr (200 ms).
  3. Fails, if DUT sends ACK with wrong control byte for DENY (2).
  4. Fails, if DUT sends ACK with with wrong PGN in payload.
  5. Fails, if DUT does not send the ACK to the global address (0xFF).

  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non cyclic BUSY producing requestable PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Acknowledgments, especially BUSY.
  1. Request the configured BUSY producing PGNs.
  2. Await response within the response timeout Tr (200 ms).
  3. Check the response for ACK-BUSY (control byte = 3) that was sent to the global address (0xFF).
  1. Inconclusive, if there are no DENY producing, non cyclic PGNs configured.
  2. Fails on no reaction after response timeout Tr (200 ms).
  3. Fails, if DUT sends ACK with wrong control byte for BUSY (3).
  4. Fails, if DUT sends ACK with with wrong PGN in payload.
  5. Fails, if DUT does not send the ACK to the global address (0xFF).

27

Support of the Address Acknowledge data byte for each Acknowledge message type
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one ACK producing or non requestable PGN.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Acknowledgments, especially ACK or NACK.
  1. Request the configured ACK producing and NACK producing PGNs.
  2. Await response within the response timeout Tr (200 ms).
  3. Check the response for ACK that was sent to the global address (0xFF).

*) NACK producing PGNs are the non reqeustable ones. (Req. column unchecked).
  1. Inconclusive, if there are no ACK or non requestable PNGs configured.
  2. Fails on no reaction after response timeout Tr (200 ms).
  3. Fails, if DUT sends ACK with wrong control byte.
  4. Fails, if DUT sends ACK with with wrong PGN in payload.
  5. Fails, if DUT does not send the ACK to the global address (0xFF).
  6. Fails, if DUT sends ACK with wrong destination address in payload.
  7. Despite the spec wants to test all ack-types (control words), we decided to check only the most commonly used, like positive or negative ACK.

28

Correct Response to Request2 with 'yes' for the "Use Transfer PGN" Option
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable PGN.
  1. DUT has to support and answer a J1939 Request2 (0x00C900).
  2. DUT has to support Transfer PG (0x00CA00).
  1. Request the configured PGN using Request2, Transfermode='yes'.
  2. Await response within the response timeout Tr (200 ms).
  3. Check the DUT uses the Transfer PG to respond with.
  1. Inconclusive, if there are no requestable PGNs configured.
  2. Fails on no reaction after response timeout Tr (200 ms).
  3. Fails if the response does not use the Transfer PG or detects wrong payload.

  1. DUT address.

  3. At least one requestable PGN.
  1. DUT has to support and answer a J1939 Request2 (0x00C900).
  1. Request the configured PGN using Request2, Transfermode='no'.
  2. Await response within the response timeout Tr (200 ms).
  3. Check the DUT answers with PG.
  1. Inconclusive, if there are no requestable PGNs configured.
  2. Fails on no reaction after response timeout Tr (200 ms).

30

Proper NACK Response for Request2 for Unsupported PGN
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one non requestable non cyclic PGN.
  1. DUT has to support and answer a J1939 Request2 (0x00C900).
  1. Request the configured PGN using Request2, Transfermode='yes' (!).
  2. Await NACK response within the response timeout Tr (200 ms).
  1. Inconclusive, if there are no PGNs configured producing an NACK.
  2. This test uses the transfer mode = 'yes' to check that the Transfer PG is not used for the NACK.

31

Proper AcknowledgmentBehavior for Destination Specific Request (PGN 59904 - 0x00EA00) sent to a different Address
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  1. Start a huge comminication with other nodes. (DUT foreign target addresses).
  2. Wait T3 (1200 ms).
  3. Check that the DUT does not respond in no way to any foreign address.
  1. Fails if DUT reacts in any way to the foreign communication.
