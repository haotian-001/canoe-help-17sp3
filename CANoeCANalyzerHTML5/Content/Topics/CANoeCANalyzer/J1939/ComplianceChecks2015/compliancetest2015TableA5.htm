J1939-82 Compliance Tests Overview (2015) - A5

Precondition DUT

Strategy

Constriction / Limitation / Typical Errors / Resolution Notes

1

BAM Transport:Ensure the Broadcast Announce message and its resulting Data Transfer data packets are valid (content and format)
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Request the configured requestable PGNs globally (BAM).
  2. Await DUT response for the requested PGN within response timeout Tr (200 ms).
  3. Check for valid BAM response
  4. Check the response's TP.DT's are beyond BAM termination.
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  4. Fails, if response is not sent to global address (0xFF).
  5. Fails, if DUT sends extra TP.DT's within T3 (1250 ms) after the BAM completed.

2

BAM Transport:Ensure the Broadcast Announce Message is sent prior to Data Transfer data packets
  1. DUT address
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Request the configured requestable PGNs globally (BAM).
  2. Await DUT response for the requested PGN within response timeout Tr (200 ms).
  3. Check for valid BAM response.
  4. Check the response's first TP.DT's is within a valid time range after TP.CM_BAM. [50 ms ; Tr=200 ms].
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  4. Fails, if the first TP.DT is prior to the TP.CM_BAM.
  5. Fails, if the first TP.DT's time distance is not within 50 ms and Tr (200 ms).

3

BAM Transport:Ensure the timing between the Data Transfer data packets for a BAM transport is within defined bounds
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Request the configured requestable PGNs globally (BAM).
  2. Await DUT response for the requested PGN within response timeout Tr (200 ms).
  3. Check for valid BAM response.
  4. Check the response's subsequent TP.DT's are within a valid time range after TP.CM_BAM. [50 ms ; Tr=200 ms].
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  4. Fails, if any subsequent TP.DT's time distance is not within 50 ms and Tr (200 ms).

4

BAM Transport:Ensure DUT does not send a Connection Abort for the BAM Transport
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support TP Connection Management (0x00EC00).
  1. Request the configured requestable PGNs globally (BAM).
  2. Await DUT response for the requested PGN within response timeout Tr (200 ms).
  3. Check for valid BAM response.
  4. Check, that there are no TP.CM_ABORTs from DUT.
  5. Check, if DUT may try something like a "retransmit" due to an TP.CM_ABORT.
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  4. Fails if DUT sends or reacts on any TP.CM_ABORTs during BAM.

5

BAM Transport:Ensure the Data Transfer data packets for a BAM transport contain the correct content
  1. DUT address.
  2. Tool address.
  3. At least one requestable non cyclic PGN with DLC>8.
  4. PG with known data (DLC>8).
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  3. DUT has to support Commanded Address (CA) (0x00FED8).
  1. Request the configured requestable PGNs globally (BAM).
  2. Await DUT response for the requested PGN within response timeout Tr (200 ms).
  3. Check for valid BAM response.
  4. Check the TP.DT sequence numbers are between [1;255].
  5. Request the PG with known content globally.
  6. Check for valid content.
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Inconclusive, if no or invalid PG with known content is configured.
  3. Fails, if the response PG has a DLC<=8.
  4. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  5. Fails, if invalid content is detected.

6

BAM Transport:Ensure the Data Transfer data packets for a BAM transport are sent in sequential order
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Request the configured requestable PGNs globally (BAM).
  2. Await DUT response for the requested PGN within response timeout Tr (200 ms).
  3. Check for valid BAM response.
  4. Check the TP.DT sequence numbers are in a sequential order.
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails, if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  4. Fails, if TP.DT's sequence numbers are not in a sequential order.

7

BAM Transport:Ensure every Data Transfer data packet contains 8 data bytes
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Request the configured requestable PGNs globally (BAM).
  2. Await DUT response for the requested PGN within response timeout Tr (200 ms).
  3. Check for valid BAM response.
  4. Check, that all the sent TP.DT frames has a DLC=8.
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails, if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  4. Fails, if any TP.DT has a DLC unequal to 8.

8

BAM Transport:Ensure extra (unused) bytes of last Data Transfer data packet is/are filled-in correctly
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Request the configured requestable PGNs globally (BAM).
  2. Await DUT response for the requested PGN within response timeout Tr (200 ms).
  3. Check for valid BAM response.
  4. Check, that all the sent TP.DT frames has a DLC=8.
  5. For the last TP.DT check the unused bytes to be 0xFF (padding bytes).
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails, if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  4. Fails, if any TP.DT has a DLC unequal to 8.
  5. Fails, if last TP.DT's unused bytes (padding bytes) are not filled with 0xFF.

Note

Unused bytes (padding bytes) are only available if the length of the data does not fit into the transport frames completely (a multiple of 7 bytes per TP.DT). So naturally, If there are no padding bytes, then the test may not fail in the padding byte check.

9

  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Request the configured requestable PGNs globally (BAM).
  2. Await DUT response for the requested PGN within response timeout Tr (200 ms).
  3. Check for valid BAM response.
  4. Check, that all the sent TP.DT frames have a prio of 7.
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails, if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.

10

BAM Transport:Ensure only one active BAM transport per Originator at a time
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Request the configured requestable PGNs globally (BAM) in a special manner.
  2. Do a request (A) from tool node.
  3. Await TPCM.BAM.
  4. Request (B) PGN from another node during BAM.
  5. Await both requests to be answered in sequence.
  6. Check if there are overlapping TPCM.BAM's within the transport to know that the responses are sent sequentially and not in parallel.
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails, if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  4. Fails, if the TP.CM_BAM messages are sent within another running BAM transport.

11

BAM Transport:Ensure simultaneous BAM transports with different Originators
  1. DUT address.
  2. Tool address (defaults to 0xF9).
  3. At least one requestable non cyclic PGN with DLC>8.
  1. DUT has to support and answer a J1939 Request (0x00EA00).
  2. DUT has to support Transport Protocol BAM.
  1. Tool starts BAM and requests the configured non cyclic requestable PGN's with DLC>8 globally in a special manner.
  2. Tool starts sending the PG for the configured PGN using BAM.
  3. Immediately request the PGN from the DUT globally (BAM).
  4. Await both BAM transports (from Tool and DUT) to be finished.
  5. Check the TP.CM_BAM messages are "overlapping". That indicates that the BAM messages from different sources can be sent simulatneously.
  1. Inconclusive, if no requestable non cyclic PGNs with DLC>8 are configured.
  2. Fails, if the response PG has a DLC<=8.
  3. Timeout, if DUT does not send an expected response to the request within response timeout Tr (200 ms). Invalid BAM responses are not detected and lead to a timeout.
  4. Fails if the TP.CM_BAM messages are not "overlapping" each other.
