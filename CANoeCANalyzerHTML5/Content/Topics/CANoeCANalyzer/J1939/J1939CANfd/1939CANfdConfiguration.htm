Set J1939 Configuration for CAN FD [▲ back]

The following steps are necessary to set your CANoe configuration to J1939 CAN FD:

  1. Change Database to J1939 CAN FD
  2. Set Network Hardware Configuration to J1939 CAN FD

The two steps are explained below.

Change Database to J1939 CAN FD [▲ back]

You can create a J1939 CAN FD database from scratch. However, it is much easier to convert an existing J1939 Classical CAN database to J1939 CAN FD. For example, you can use one of the databases supplied with the CANoe sample configurations for J1939. To do this, follow the steps below:

Step by Step Procedure

  1. Via the Tools ribbon tab open the CANdb++ Editor.
  2. Via the File|Open Database menu path open the database, that you would like to convert to J1939 CAN FD . Alternatively, you can select the J1939Template.dbc template via File|Create Database and create the database from scratch.
  3. In the Overall View on the left side select Networks.
  4. Your network is displayed in the detail view on the right side. Double click on the network to open the network dialog.
  5. There you change the Protocol from CAN to CAN FD.

Note

The message attribute J1939PgAppearanceOnBus is created when the protocol is changed.

Set Network Hardware Configuration to J1939 CAN FD [▲ back]

To set the network hardware configuration to J1939 CAN FD, perform the following steps:

Step by Step Procedure

  • Switch to the Hardware ribbon tab and click Network Hardware.
  • The Network Hardware Configuration dialog opens.
  • Repeat the following steps for all channels you want to assign to J1939 CAN FD:
  • Select the CAN channel in the tree view on the left.
  • In the CAN Controller area select in the Mode list the ISO CAN FD entry.
  • In the tree view, switch to the J1939 Configuration entry.
  • Activate the CAN FD J1939 option.
  • Close the Dialog with OK.

Analysis in the Trace Window [▲ back]

The Trace Window supports you in analyzing J1939 CAN FD messages. During the measurement, the Multi-PG messages are displayed with the familiar letter symbol . All C-PGs appear with the icon of a clock . Since the C-PGs are contained in a Multi-PG, the time stamps are identical. If you expand the C-PGs, the contained signals appear as you know it from J1939 PGs.

FBFF messages always appear with Destination Address 0xFF.

Simulation and Analysis with CAPL [▲ back]

For sending J1939 parameter groups (PG) you may use the CAPL function output. You can use the TM selector to control whether a PG is to be sent with a Multi-PG or via the FD transport protocol or whether xx determines the transmission type itself depending on the length. In addition, you can also define assurance data with the selectors ADT and ADS.

However, sending with the CAPL function output has a limitation: with this it is not possible to send several Contained-PGs with only one Multi-PG. For this purpose, other CAPL functions are available to fill a Multi-PG with several Contained-PGs:

  • J1939InitializeMultiPG
  • J1939AddContainedPG

More detailed information about sending and receiving in CAPL is described on the page Send and Receive Parameter Groups with CAPL.

Simulation with J1939 Interaction Layer [▲ back]

The J1939 Interaction Layer (J1939 IL) allows to simulate an ECU communicating according to J1939-22 in a very simple way. The J1939 IL takes over the transmission of the Tx messages that are assigned to the respective node in the database. Such messages are transmitted as Contained-PGs (C-PGs) using the multi-PGs. It is not necessary to manually assign individual C-PGs to the Multi-PGs, because the assignment is done automatically.

More detailed information about sending and receiving with J1939 IL according to J1939-22 is described on the page Send Parameter Groups With J1939 IL according to J1939-22.
