J1939 CAN FD

 

J1939 CAN FD

CANoe starting with version 15.3 supports J1939 based on CAN FD according to the specifications SAE J1939-17 and SAE J1939-22. CAN FD enables transmission rates of the user data of up to 2000 kBaud with a length of up to 64 bytes. This increases the throughput compared to classical CAN (250 or 500 kBaud, 8 bytes), so that either the busload can be reduced or the number of participants can be increased.

Technical Background

CAN FD is backward compatible to classical CAN, because the arbitration phase still takes place at 500 kBaud and indicates whether the data is to be transmitted using CAN FD or classical CAN. To signal the usage of CAN FD to the receiver, the FDF bit (FD Format Indicator) is set in the check box of the CAN frame.

A CAN FD frame can transport 0 - 64 bytes of user data. How long a frame is, is specified by the Data Length Code (DLC), which is located in the check box of the CAN FD frame. The DLC can have values from 0-15. The length is not proportional to the DLC and can be taken from the table below:

    CAN FD Frame 11 Bit Identifier

In J1939 CAN FD three Protocol Data Units (PDU) are distinguished:

  • D_PDU1

    FEFF, Destination Specific

  • D_PDU2

    FEFF, Broadcast

  • D_PDU3

    FBFF, Broadcast

In a J1939 Classical CAN network, a CEFF frame is used to transmit a single parameter group (PG). In contrast, in a J1939 CAN FD network more than one PG can be transported using both an FBFF frame and an FEFF frame. These frames are therefore also called Multi-PG Data Frame. The Multi-PG messages can be sent either to a specific target address or as a broadcast.

The data field of a Multi-PG can contain none, one or more parameter groups (PG), so-called Contained-PGs (C-PGs). The Contained-PGs correspond to the PGs from previous J1939 Classical CAN networks as defined e.g. in J1939-71. Since C-PGs do not always use exactly the specified length of the CAN FD message, a special Padding Service C-PG is placed after the last C-PG, which fills the message to the end.
