Functional Safety J1939-76

 

Introduction

The SAE J1939-76 standard (SAE J1939 Functional Safety Communications Protocol) describes a J1939 extension for the transmission of safety-critical data. This is achieved by sending an additional message (Safety Header Message, SHM) before the message containing the critical data (Safety Data Message, SDM). SHM and SDM then form a Safety Data Group (SDG).

THe SHM contains the following information:

  • a sequence counter (per SDG, 5 Bit)

  • the CRC over the data of the SDM (32 Bit)

  • the inverted CAN ID of the SDM (ohne Priorität, 26 Bit)

The sender (Producer) generates the SHM and SDM and sends them in the correct order and in compliance with the time constraints. The receiver (Consumer) is responsible for checking the sequence and time constraints as well as the integrity of the transmitted data.

The following times must be observed when transferring the SDG:

SRVT

Safety-Relevant Validation Time

Time between a SHM and a SDM

SCT

Safety Cycle Time

Time between continuous SDM instances

The time base for the calculation of SRVTMaximum and SCTMaximum is derived from the cycle time (Transmission Rate) in the SAE J1939 PG, which is to be transmitted as SDM.

Trace Window Support

During a measurement in CANoe, the following checks are made in the Trace Window:

  • Consecutive sequence numbers of the SHM.

  • Validation of the CRC of the SHM on receipt of the associated SDM.

  • Missing or too late SDM after a SHM.

  • Missing SHM before an SDM.

  • Compliance with SRVTMaximum (calculation as described above).

A SHM is always recognized by the unique PGN 3584 (0E00h). After a SHM has been received, the check recognizes the associated SDM by means of the CAN ID (without priority) contained in the payload of the SHM and can then perform the validation.

The database attribute FsJ1939NeedsSHM is used to check for a missing SHM. For each PG where the attribute has the value True, a SHM must have preceded.

With the help of the context search you can jump to the corresponding SHM or SDM.

Test Feature Set Support

With the J1939-76 Functional Safety Check you can monitor the communication according to J1939-76. The same checks are performed as in the Trace Window.

J1939 Interaction Layer Support

With the help of the J1939 Interaction Layer (J1939 IL) you can easily simulate an ECU. Functional Safety is also supported by the J1939 IL. If a PG is defined as SDM in the database by the FsJ1939NeedsSHM attribute, the J1939 IL takes care of generating and sending the associated SHM. The delay between SHM and SDM is determined by the database attribute FsJ1939Delay.

For more information on J1939-76 support in the J1939 IL, see the following page: Functional Safety (J1939).

CAPL Support

To verify messages transmitted with J1939-76, CANoe provides special CAPL functions. With these you can read the data of the SDM contained in a SHM and calculate the checksum over a SDM.

In addition, there is a function that allows you to fill a SHM with the data from the SDM.

A more detailed description of the functions can be found on the following page: J1939 CAPL Functions
