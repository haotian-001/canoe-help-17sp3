Parameter Groups and PGNs

 

J1939 Parameter Group (PG)

A J1939 Parameter Group (PG) is a set of data parameters which are always transferred together as a logical unit (equivalent to a CAN frame, which contains a set of CAN signals). It represents an atomic data burst with which electronic ECUs exchange information. Most parameter groups contain one protocol data unit (PDU) each from the application layer – e.g. motor/gearbox data or diagnostics data – e.g. active Diagnostic Trouble Codes. However, there are also parameter groups which support the basic service of the J1939 protocol, such as PG Request, Transport Protocol and Address Claim.

Each Parameter Group is uniquely identified by a so-called Parameter Group number (PGN). Parameter groups with the same PGN are interpreted in the same manner and, in fact, independently of their sender and receiver (that is, from their source and destination address). The definition of the parameter groups and the parameters contained within them can be found in the J1939 Digital Annex or in one of the corresponding J1939 subdocuments, e.g. J1939-21, J1939-71, J1939-73, J1939-75, J1939-76 and J1939-81.

The length of a Parameter Group is not limited to the length of a CAN frame. A Parameter Group typically has a length of between 8 bytes and 1,785 bytes. Parameter groups with more than 8 bytes require a transport protocol for transfer.

As mentioned above, each Parameter Group is identified by a unique number – the Parameter Group number (PGN).

The PGN is an 18-bit value, comprised of the fields Extended Data Page (1 bit), Data Page (1 bit), PDU Format (8 bits) and PDU Specific (8 bits) as follows:

Extended Data Page (EDP)

Data Page (DP)

PDU Format (PF)

PDU Specific (PS)

1 bit

1 bit

8 bits

8 bits

Careful attention must be paid here, however. While the Extended Data Page, Data Page and PDU Format of a PGN are always identical to the corresponding values from the CAN ID, it may be different with PDU Specific. In fact, it depends on whether the corresponding CAN ID corresponds to the PDU1 Format or the PDU2 Format.

  • PDU1 Format: PDU Format < 240 (F0h)
    Corresponding Parameter Groups are characterized by the fact that they can be sent to a specific target. The PDU Specific part of the PGN is always 0, whereas the PDU Specific part of the CAN ID is interpreted as a destination address (DA). If such a Parameter Group is not to be sent to just one node, but rather all the nodes on the network (broadcast), the destination address (the PDU Specific part of the CAN ID) is to be defined as 255 (FFh). This address is designated the global DA.
  • PDU2 Format: PDU Format >= 240 (F0h)
    Corresponding Parameter Groups are characterized by the fact that they can always be sent on all nodes of the network (broadcast). Therefore, you do not need a destination address and the PDU Specific part of the CAN ID is interpreted as the PDU Specific part of the PGN.
