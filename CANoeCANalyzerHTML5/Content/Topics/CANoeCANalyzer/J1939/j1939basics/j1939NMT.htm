Network Management

 

On a J1939 network, each ECU has a unique address. The address may exist only once. The device address is represented by an 8-bit value. This is also coded in the CAN identifier. The device address is contained in each sent PGN. In J1939, 255 device addresses are defined, whereby two addresses have a special meaning:

Address

Description

0..253

Valid addresses for communication

254

NULL address

A device has no valid address.

255

GLOBAL address.

For global addressing for specific Parameter Groups or for the transport protocols.

Each ECU has a predefined address. Another identification characteristic of an ECU is the device name. This is a 64-bit value that serves to describe the functionality of a device and also provides information about the manufacturer.

Address Claiming

Before a device may participate in sending on a network, it must log onto the network with a valid address. For this, the predefined device address together with the device name is sent to the network in the boot sequence of an ECU. This procedure is called Address Claiming. The AddressClaim Parameter Group handles the sending of this information. If the predefined address of an ECU is already present on the network, the device name serves as a prioritizing aid. Here, the device has the higher priority whose 64-bit device name has a smaller numeric value.
