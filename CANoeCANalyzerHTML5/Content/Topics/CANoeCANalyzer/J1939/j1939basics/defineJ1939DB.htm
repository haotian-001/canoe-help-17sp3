Define a J1939 Database

 

When defining a J1939 network, some special features have to be taken into account, e.g. the CAN identifier also contains addresses (which do not exist in pure CAN). Thus there are more configuration possibilities that must be taken into account for a J1939 network.

Various programs in the Vector toolchain use a J1939 database in special ways. To be able to use a database with J1939, the value J1939 must be assigned to the network attribute ProtocolType. This attribute is already defined in the included database J1939.dbc. If a database is created new, the attribute must be defined as described in the following table:
