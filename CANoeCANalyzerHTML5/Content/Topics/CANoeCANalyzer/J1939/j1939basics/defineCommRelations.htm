Define Communication Relationships

 

The setup of communication relationships is absolutely necessary for the definition of a network and the code generation. Here it is specified which messages are sent and received by the nodes on the network.

The content of a Parameter Group is identified with the Parameter Group Number (PGN). On a J1939 network it is possible that a Parameter Group is sent the same PGN by different nodes. These Parameter Groups distinguish themselves in their transmitter addresses and can be assigned to a send node using these.

Furthermore, it is possible to specify whether a Parameter Group is only received if it was sent by a particular node or whether a Parameter Group is accepted regardless of the sender.

Which of these possibilities is used depends on the particular application.

With the assignment of messages to a node, the selection and display of Parameter Groups in CANoe is simplified and it is clearer.

Assigning send Parameter Groups

If a Parameter Group should be sent by a node, the Parameter Group is inserted into the list of Tx messages. During the assignment of send Parameter Groups, two possibilities can be distinguished:

  • The sender address of the message is defined: A Parameter Group with defined sender address can only be sent by the node with the corresponding sender address.
  • The sender address of the message is not defined: The Parameter Group can be assigned by several nodes as Tx message. The node address of the respective node is used as the sender address.

By specifying the sender address, the behavior on the recipient side can be influenced. With a defined sender address, the recipient accepts the message only from the specified sender; otherwise only the Parameter Group number is considered.

If the sender address of the assigned message is defined, then it should match the node address (NmStationAddress). Special case: a node functions as a gateway.

A J1939 Parameter Group can have only one sender address. If a Parameter Group should be sent by several nodes, an instance of the message must be created for each sender. The CAN identifier of the message distinguishes itself only by the sender address, the Parameter Group number is identical.

By copying a J1939 message, the new instance of the desired Parameter Group can be created. After that, the Parameter Group can be assigned to the sender node and the sender address adjusted. The name of the message must be adjusted since it must be unique. It is also possible to create a new Parameter Group; then the signal and message attributes must also be edited.

For specific Parameter Groups a new instance of the message must also be created for each recipient. For each recipient, the various instances receive the corresponding recipient address.

Assigning Receive Parameter Groups

Each node can be assigned messages that it should receive. For the code generation this assignment is absolutely necessary. In CANoe the selection and display can be simplified.

A Parameter Group that should be received by a node is inserted into the list of Rx messages of the node. Using the assigned Rx messages, it is specified which messages a node should receive and from which nodes these messages must be sent. It is also possible to specify that all Parameter Groups will be received with a particular PGN, regardless of the sender address. This distinction will be made using the sender address of a Parameter Group. If the sender address of a Parameter Group is not defined, then the Parameter Groups are received regardless of the sender address.

Note

In CANdb++, the Rx signals must be assigned that should be received. The corresponding messages are thus entered into the list of Rx messages.
