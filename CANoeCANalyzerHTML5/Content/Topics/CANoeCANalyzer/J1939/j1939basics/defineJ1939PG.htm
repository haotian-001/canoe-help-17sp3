Define a J1939 Parameter Group

 

A J1939 Parameter Group is essentially defined the same way as a normal CAN frame. However, there are three particularities to heed:

  • The message type of a J1939 Parameter Group is J1939 PG (ext.ID).
  • The DLC of a J1939 Parameter Group can be larger than 8 bytes. A J1939 Parameter Group with a DLC greater than 8 must be transmitted with a suitable transport protocol.
  • A J1939 Parameter Group has an extended CAN identifier that must correspond to the J1939 identifier format.

The signals can be assigned to the message as usual. However, signal attributes can also be defined . If the DLC is greater than 8, the signals can also begin at a bit position that is greater than 64.

For J1939 messages whose length is variable, the DLC should be set to the maximum possible value.

The PGN of the Parameter Group must be defined. In the CANdb, the data page to the PGN is counted in, so that the PGN includes 18 bits. In addition it is also necessary to specify the priority of the message. The priority consists of 3 bits.

The transmitter address of a Parameter Group does not absolutely have to be defined. It is possible to use the null address (FEh) here. '*' is then displayed for the address. Thus you can see that the transmitter address of the Parameter Group is not defined. For specific Parameter Groups (PDU Format 1) the same applies for the recipient address. For many applications is is not necessary to define the address, e.g. if the database is used for analysis. In addition it can be necessary that a node must be able to receive the message from all senders, and thus determines only at runtime from which sender address the Parameter Group will be sent.

Note

For the included database (J1939.dbc), the addresses of the messages are not defined.
