Wait Functions

 

Note

The following wait functions are part of the J1939 Test Service Library (J1939TestServiceLibraryNL.dll), which is no longer being developed and should therefore no longer be used. Instead, you can use the wait functions described on page Testing in a J1939 Network.

Wait functions can be used to interrupt the execution of a CAPL program. The execution is resumed at that point, if an event occurs. The test can be performed step by step. This makes the CAPL programs easier. The wait functions of the J1939 Test Service Library interrupts the CAPL programs until the specified Parameter Group is received or a timeout occurs.

The following example waits for the Parameter Group EEC1 from the source address 0 and verifies the signal EngineSpeed.

Example

long lret;
pg EEC1 rxPG;

lret = J1939TestWaitForPG( 0, 0xfe, EEC1, 500 );

switch(lret) {
  case 1:
  if (J1939TestGetWaitEventPGData(engParam ) == 0) {
    if (rxPG.EngineSpeed.phys < 100) {
      TestStepFail( "TestCase 1", "Wrong Signal Value");
    }
  }
  else {
    TestStepFail( "TestCase 1", "Get event data failed");
  }
  break;
  default:
    TestStepFail( "TestCase 1", "timeout or constrain violation ");
    break;
}
