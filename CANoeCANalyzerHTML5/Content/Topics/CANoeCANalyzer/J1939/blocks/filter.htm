CAN Filter

 

The filter of the standard CANoe is extended with the option J1939. The configuration is analogous to the standard filter.

Cross Reference

Functions that are not J1939 specific are described in the help of CANoe.

Buttons and Options

Button

Command

[Events...]

Messages from a J1939 database are filtered by their PGN instead of the message ID.

[Raw data...]

You can enter the numeric value of a PGN by adding a P, e.g. Start ID FF00P and Last ID FFFFP for all non-specific proprietary PGNs.

For PGN format 1 (00xxP..EFxxP) you have to set the destination address xx to 0.

[Nodes...]

You can select J1939 network nodes and filter them by their J1939 addresses.

Filter Settings

On page J1939 in the upper right part of the filter you can set the J1939 message and node attributes that are to be filtered.

To define a filter you have two possibilities. The static method filters by the defined Address, the dynamic method however observes the Address Claim messages and filters by the 64 bit device name. You can toggle between theses two methods with the check box in the column Dynamic Filter.

For a filter defined with the button [Events...] you can freely set the address, the database name and the node name for both the receive direction and the transmit direction. For a filter defined with [Nodes...] you can replace the wildcards (*) in the columns of the device name with specific values, or replace specific values with wildcards. Afterwards a modified device name is marked with a * before its database name.

Note

If you use the dynamic filtering the Address Claim messages must reach the filter.

If that's not the case, e.g. because the Address Claim messages are sent before measurement start or filtered out by another filter, the filter cannot evaluate the messages of the nodes.

J1939 Node Filter

In addition to the filter described here, the J1939 option offers an additional node filter. With the J1939 Node Filter you can filter Parameter Groups that are sent or received by particular nodes.
