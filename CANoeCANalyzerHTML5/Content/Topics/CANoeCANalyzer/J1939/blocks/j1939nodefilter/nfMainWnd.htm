Node Filter (Deprecated)

 

With the J1939 Node Filter you can filter on PGs that are sent or received by specific nodes.

Example

Three filter rules are listed in the filter displayed above. The filter type is set to "Pass filter".

Rule 1: All PGs that are send by the node with address #5 and received by the node with address #10 are allowed to pass through.

Rule 2: All PGs that are sent globally (destination address 255) by nodes with addresses #220, #221, #222,...., #230 are allowed to pass through.

Rule 3: All PGs having address from #20 through #30 as a destination address are allowed to pass through. The sender address is ignored (*).

[Add filter rule...]: A new filter rule will be added to this filter. This rule will be configured using the Node selection dialog.

[Delete filter rule]: To delete a rule, first select it under Source. Then click on the [Delete filter rule] button.

Edit filter rule: To be able to edit a filter rule, open the Node selection dialog by double clicking on the rule to be changed and then edit the rule.

Filter type: To adjust the filter type, choose the selection Pass filter (for a pass-through filter) or Stop filter (for a block filter).

[Channel...]: This button opens the dialog for the channel selection.

[OK]: With this button you can close the dialog.

[Help]: Press this button if you need help about the J1939 Node Filter.

Note

If no rules are specified for the filter, then:

  • All messages will be allowed to pass through a block filter, since there is no rule for blocking.
  • All messages will be filtered (blocked) for a pass-through filter since no rules are found to justify passing a message through.
