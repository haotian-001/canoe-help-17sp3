Manual Object-Oriented Memory Access

 

On this page objects can be specified and their values written to or read from the memory with the touch of a button.

Each line describes an object. Absolutely necessary values in the object table are the object ID and the length of the object. The name, unit, and an assigned environment variable are optional. The column Space specifies how the object ID is to be interpreted. If value 0 (SPN SPACE) or a SPACE defined in the database is selected, an object can be selected from a list of possible IDs or from a list with names.

If an object is selected from the list, it is marked as "connected with the database". Different icons show if an object is connected to a database or not:

Symbol

Description

object is connected to a database

object is currently not connected to a database

If an object is connected to a database, the object length, unit, and conversion values (factor, offset) are then extracted from that database. The object length and the unit cannot be changed then. If the object is not connected to a database, the unit and the physical value cannot be displayed. The raw value and the object length are modifiable.

A connection to the database can be established or released with a right mouse click on the icon in column DB. For this the object ID for the specified SPACE must be defined in the database.

If objects with a coherent identifier range (e.g. 80, 81, and 82) are selected, then these objects are edited via a single access procedure. Thus bus traffic is reduced; especially if authentication is necessary for each new memory access.  

In the environment variable column the name of an environment variable can be specified, which on changing of the object value receives the new raw value. For objects with a length of 32 bits or less, the data type of the environment variables may be in the Integer or Float database. In case of longer objects, the data type Data must be selected.

With a click on the line number, the entire line is marked and can either be moved using the mouse or deleted with the <Del> key. Using the context menu, it is possible to copy and insert cell contents as well as to undo operations again.

The single columns can be shown or hidden by a right mouse click on the appropriate column header.

For the read and write operations, only those objects are used that are marked by a selected box.

Here too there is an Output Window in which the status of the executed action is specified.

The following buttons are available:

Button

Action

Reads data of the specified length out of the ECU. The data in the memory window is overwritten.

Writes the data in the memory window into the ECU.

Starts or stops broadcast messages of a single ECU or all ECUs

Deletes the content in the Output Window

The window synchronization is activated and mark that instant of time at which the current selected message occurred.

If the mouse has a wheel, then by pressing the <Ctrl> key, the view of the table can be enlarged or reduced and by pressing the <Shift> key, the table can be scrolled.
