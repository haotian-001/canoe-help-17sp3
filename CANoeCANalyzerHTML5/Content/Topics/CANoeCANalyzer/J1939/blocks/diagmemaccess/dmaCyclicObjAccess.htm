Cyclic Object-Oriented Memory Access

 

In addition to manual access, memory objects can also be read cyclically. But for the column Duration and two missing buttons, this page is identical to the manual object-oriented memory access page.

For objects with an activated check box in the first column, a cyclic read access is executed during the measurement. During the measurement the read access to an object can be activated or deactivated by using the check box. The cycle time in the column Duration (in milliseconds) and the object ID in the column ID can be changed too. In contrast it is not possible to change the environment variable which is assigned to an object, if the cyclic read access is activated.

If several objects have the same cycle time, only one timer is created for these objects.

In addition, the following fields are available:

Button

Action

Starts or stops broadcast messages of a single ECU or all ECUs

Deletes the content in the Output Window

The window synchronization is activated and mark that instant of time at which the current selected message occurred.
