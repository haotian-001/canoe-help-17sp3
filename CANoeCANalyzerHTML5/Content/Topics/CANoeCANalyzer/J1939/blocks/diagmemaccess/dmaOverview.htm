J1939 Diagnostic Memory Access

Ribbon|Diagnostics & XCP Tab|J1939|Diagnostic Memory Access|View / New

General

With the J1939 diagnostic memory access window you can read and write parts of the memory as well as execute a boot load procedure. For this the diagnostic messages DM14, DM15, DM16 and DM18 are used.

Structure

Note

The window consists of four page. In addition to direct access to a memory area, object-oriented access is also possible (manual or cyclic). The settings that apply for all types of memory access, you can make on the Configuration page.

To minimize the network traffic, you can stop the transmission of broadcast messages of a single ECU or all ECUs. For this the diagnostic message 13 is sent to the ECUs.

Subsequently a status messages is cyclically sent to the ECUs to avoid that these ECUs change to their previous states. If required you can start the transmission of the broadcast messages. The ports to be used you can set on page Configuration.
