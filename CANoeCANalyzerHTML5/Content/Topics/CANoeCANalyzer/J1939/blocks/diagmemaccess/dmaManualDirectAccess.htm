Manual Direct Memory Access

 

On this page it is possible to read out a memory area, write to it or delete it; it is also possible to execute a boot load procedure.

The data to be read or written is also displayed in the memory window. The window is divided into three columns. The left column displays the offset with respect to the specified address. The offset always begins with zero. The middle column displays the data in hexadecimal form and the right column in ASCII form. The data can be edited in the middle and right columns. After editing, the data is displayed in red. With the help of the context menu or the key combination <Ctrl>+<Z> operations can be undone and restored with <Ctrl>+<Y>. Using the context menu, the number of data bytes per line can be set.

The result of the action carried out is specified in the Output Window. In case of errors that are reported by the ECU with the DM15, the error number is also displayed next to the error text.

In addition, the following fields are available:

Button

Action

Load a file into the Data Window. The previous content is lost.

Store the data shown in the Data Window in a file.

Address

Memory access occurs at this address in the ECU. From the list box, one of the last used addresses can be selected.

Length

Length of the data in bytes that should be read, written or deleted.

Read data of the specified length out of the ECU. The data in the memory window is overwritten.

Write the data in the memory window into the ECU.

Delete the specified memory address (via address and length) in the ECU. The data in the memory window play no role.

Boot load procedure: the data in the memory window is written to the ECU. Then the ECU continues its execution at the specified address.

On successful operation a system or environment variable is set. This system or environment variable must be specified in the Configuration dialog. Thus, for example, a CAPL program can be activated that can now communicate with the downloaded program via the diagnostic message 17.

Starts or stops broadcast messages of a single ECU or all ECUs

Deletes the content in the Output Window

The window synchronization is activated and mark that instant of time at which the current selected message occurred.
