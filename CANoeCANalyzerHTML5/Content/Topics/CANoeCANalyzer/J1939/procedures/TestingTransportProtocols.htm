Testing Transport Protocols

 

The J1939 and ISO11783 interaction layers in CANoe send and receive the transport protocols BAM, RTS/CTS, Fast Packet and ETP automatically. To enable you to test your ECU against errors in the transport protocols, CANoe gives you the option of manipulating the automatic nature of the interaction layers. For simple test cases, you can use CAPL functions for blocking or delaying to change the transmission behavior of individual messages, for example by skipping DT messages. For more complex test scenarios, on the other hand, you can freeze the execution of the transport protocols and continue or stop them at a later time. This offers you a wide range of options, because you can intervene in the connection at a very specific point in time, create and send TP messages after your requirements, and then let the connection continue.

Tests Implementation

For the CAPL functions J1939ILFreezeTp, J1939ILUnfreezeTp as well as J1939ILKillTp you should set a delay of at least 1 ms between the messages so that the functions can intervene before the interaction layer sends further messages. You set the delays with J1939ILSetNodeProperty (analog ISO11783_ILSetNodeProperty).

Example

on start
{
  // Setup delays to allow immediate freeze
  J1939ILSetNodeProperty("TPDTLatency", 1);
  J1939ILSetNodeProperty("CTSLatency", 1);
  J1939ILSetNodeProperty("EoMALatency", 1);
}

To freeze the connection at a defined time, you can use TestWaitForJ1939PG to wait for a specific parameter group and then call J1939ILFreezeTP:

/*
Freezes the BAM connection after a TP.DT occurred
*/
void FreezeTpBam()
{
  long result;
  long timeout;
  long idx;

  timeout = 100; //ms

  // Send PGN with TP
  J1939ILSetMsgEvent(Node_A, FreezePGN);

  // Wait for 2 occurrences of TP.DT (0xEB00) and freeze the connection afterwards.
  for(idx = 0; idx < 2; idx++)
  {
    result = testWaitForJ1939PG(0xEB00, timeout);
    if(result != 1)
    {
      testStepFail("PGN not received");
      return;
    }
  }
  // After TP.DT occurred, freeze the connection.
  J1939ILFreezeTp(Node_A, 1, Node_A.NmStationAddress, 0xFF);

  // Send manipulated TP.DT frame
  //[...]
  testWaitForTimeout(50);

  // Resume connection
  J1939ILUnfreezeTp(Node_A, 1, Node_A.NmStationAddress, 0xFF);
}

You can then inject your own messages into the connection and continue the connection with J1939ILUnfreezeTP or terminate it with J1939ILKillTp.
