Define a J1939 Node

 

For the definition of a J1939 node, parameters for the network management can be specified and send and receive Parameter Groups can be assigned. In contrast to a standard CAN node, a J1939 node has additional attributes. These attributes are necessary for the code generation.

A J1939 node has

  • a node address in the range from 0 to 253
  • and a 64-bit device name (device name)

These properties can be set via node attributes.

Note

In the SAE J1939 specification, the terms node and ECU are used in another way than in the CANdb. In the SAE J1939 specification, the ECU respectively the Controller Application (CA) has an address and a 64-bit device name; in the CANdb, by contrast, the node has the properties.

The node address is set with the node attribute NmStationAddress (see table below). For a node with network management, the preferred address is set here. The node address may not be used multiple times in the database.

For the network management the device name is also required. This consists of a 64-bit value that is composed of several components.
