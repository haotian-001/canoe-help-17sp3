Fast Packet Protocol (CAPL J1939)

 

The support for maritime applications offers functionality to analyze and simulate the Fast Packet Protocol, so CANoe can send and receive this transport protocol.

In order to recognize the Fast Packet Protocol, the message attribute SingleFrame within the CAN database must be set to 0. If this attribute is missing or not set to 0 and the DLC is greater than 8, CANoe select a suitable transport protocol according to the destination address (broadcast: BAM / peer-to-peer: CMDT).

Use

The Fast Packet Protocol can transfer 1 to 223 bytes. Because in CAPL the packets and the complete Parameter Group are received, a special handling of the messages is necessary.

The on pg function is called for a single packet with DLC 8 and the complete Parameter Group with DLC 1..223. In general only the complete message is needed. For Parameter Groups with DLC > 8 the CAPL function getThisMessage must be used. This function copies a received message, which is transferred with a transport protocol, into a variable. For message which does not use a transport protocol the DLC is set to 0.

Example

In the example the Parameter Group ConfigurationInformation is received. The Parameter Group must be defined in the database with a DLC > 8. The DLC of the variable rxPG is > 0, if the complete message is received.

on pg ConfigurationInformation
{
  pg ConfigurationInformation rxPG;
  getThisMessage( rxPG, this.DLC );

  if (rxPG.DLC > 0) {
    // handle PG rxPG here
  }
}
