Extensions for Maritime Applications

 

GNSS Monitor

The GNSS (Global Navigation Satellite System) monitor is used for graphical and numeric display of degrees of longitude and latitude as well as to specify the height of a GNSS message. In addition, this data can be logged with other information in the protocol.

GNSS Simulator

The GNSS Simulator can be used to simulate a GNSS receiver. To do this, a geometric figure or a position file is used to determine a path that will be traced by the virtual GNSS receiver when the simulation is started. GNSS positions thereby generated are transferred cyclically with configurable messages.

GNSS Node Layer

The GNSS node layer can be used to simulate a GNSS receiver. To do this, different CAPL functions are used to determine a path that will be traced by the virtual GNSS receiver when the simulation is started. GNSS positions thereby generated are transferred cyclically with configurable messages.

Fast Packet Protocol

The Fast Packet Protocol is supported for analysis and simulation. Therefore it is necessary to define which Parameter Groups are transmitted with Fast Packet Protocol. For this purpose the message attribute SingleFrame is used in the database. If the attribute value is set to 0 for a Parameter Group, the Fast Packet Protocol is used for this Parameter Group.

This is considered for the Trace Window interpretation as well as for the signal access in the Graphics Window and Data Window.

The Fast Packet Protocol is also used in send direction for the simulation of network nodes. Out of CAPL Parameter Groups with SingleFrame=0 are always sent with Fast Packet Protocol. If the J1939 Interaction Layer is used, the network attribute ProtocolType=NMEA2000 must be defined.
