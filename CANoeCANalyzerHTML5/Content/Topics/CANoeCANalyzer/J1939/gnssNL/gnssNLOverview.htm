GNSS Node Layer

 

Function

The GNSS Node Layer (J1939) (Global Navigation Satellite System) is used to simulate a GNSS receiver. These virtual receivers can be used to supply other devices on the network with position data. GNSS positions are transferred cyclically with configurable messages.

Setting up the Node Layer

A node layer can be assigned to each node in the simulation structure of CANoe. CAPL functions can be used to obtain access to node layer.

For a node of GNSS to be able to use the node layer, a few settings must be made in the simulation structure and database:

  • A node attribute must be inserted in the database with the name NodeLayerModules and with type String. A node must then be inserted in the same database. The attribute NodeLayerModules must be set to GNSS_NL.vmodule for this node. If a number of node layers will be used by a single node, the names of the individual ones must be separated by commas.
  • A node must be inserted in the simulation structure. The node from the database must be assigned in the configuration dialog.

You can check which node layers are being used by a node in the configuration dialog under Module.

Using the Node Layer in a CAPL Program

The GNSSStartUp function can be used to set up a node in the network. From this time on, settings can be made, waypoints can be added and the speed and course of the GNSS receiver can be determined.

The GNSSStartSimulation function is used to start the simulation. The virtual GNSS receiver then moves on the path defined by the waypoints or course. The speed of the receiver is determined by the GNSSSetSpeed function. When the receiver reaches a waypoint to which a speed is assigned, this speed is received by the receiver. During the simulation, additional waypoints can be added or a new course can be selected.

The GNSSStopSimulation function ends the simulation and the node is removed from the network with GNSSShutDown.

Events

The node attempts to call functions in the CAPL program with various events. These functions must be defined by the user in the CAPL program. The function name and signature must be precisely maintained during this process.

The node issues a warning in the Write Window and when the measurement begins if one of the event functions GNSSAppErrorIndication, GNSSAppAddrClaimed or GNSSAppCmdAddrIndication is not defined.
