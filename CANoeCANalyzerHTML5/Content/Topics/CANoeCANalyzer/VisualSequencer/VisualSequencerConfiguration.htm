Configuration

 

Clicking the Configuration button () opens the Advanced Settings dialog.

Settings

Execution

Here you can define, where to execute the Visual Sequence.

  • Standard:
    The sequence will be executed in the simulation environment.
  • OnBoard:
    The sequence will be executed on a Vector Interface (e.g. VN1630 family). In this mode the defined wait and cycle times are met extremely precise.

Please consider the following restrictions for OnBoard execution:

  • Some commands cannot be used or only with restrictions (see notes on the command's page).
  • You cannot use the signal layer.
  • Within a sequence commands may only access buses that are accessed by one shared interface.

The following sequence for example can only be executed OnBoard if the channels CAN1 and CAN2 are both assigned to the same interface.

Send CAN Raw Frame | CAN1::0x100 | = | AA BB

Send CAN Raw Frame | CAN2::0x200 | = | CC DD

Signal Layer

In CAN simulations, the CANoe Interaction Layer provides signal-based access to the bus. The Use Signal Layer setting lets you specify whether this intermediate layer is used when a sequence is executed.

In order to use the signal layer, the corresponding send node needs to be simulated for each of the signals used in the sequence (command: Set).

If the signal layer is not used (setting: Send messages manually), cyclical messages that transport signals need to be sent separately (commands: Set CAN Cyclic Frame and Send CAN Frame).

Command Configuration

This section contains specific settings for individual commands.

  • Wait For Key: Waits until the following key is pressed
    This defines the key event that the Wait For Key command waits for before the next command is executed.
  • Check: Output negative results only
    This defines that an output is generated only if the result of the Check command is negative.
    If the option is inactive, check commands always generate an output.

Command Logging

A log can be written while the sequence is running (see commands: Check, Write and Write Text).
You have the following options:

  • Log to Write
    Output is written to the Write Window.
  • Log to File
    Output is written to a CSV file (comma separated values).
  • Column separator
    The symbol that defines the boundary between two columns in the file.
  • Decimal symbol
    The symbol that is used as decimal symbol in the file.
  • Decimal places
    Defines the number of decimal places to log.
  • Log with time stamp
    All output is preceded with the time (seconds) elapsed since measurement start.
  • Name Display
    Defines the name display of symbols.
    The levels are as described in Name Display.
