Option AMD/XCP

 

  

X Calibration Protocol (XCP) / CAN Calibration Protocol (CCP)

XCP and CCP are widely used calibration and measurement protocols that are standardized by the ASAM Working Group. XCP can be used on different bus systems, while CCP can be used only on CAN.

Option AMD/XCP enables access to ECU-internal values and enables their usage in special test cases. In CANoe, ECU-internal signals are mapped as system variables and are thus available in all analysis windows, in CAPL, and in the Test Feature Set. CANoe acts as an XCP or CCP Master and can communicate simultaneously with several slaves.

For Analysis you can decode XCP/CCP messages via the Protocol Observer.

Supported Protocols and Bus Systems

  • CCP
  • XCP on CAN
  • XCP on CAN FD
  • XCP on Ethernet
  • XCP on FlexRay
  • XCP on LIN

For CCP/XCP on CAN, XCP on FlexRay and XCP on LIN the respective bus options are required.

AUTOSAR Monitoring and Debugging (AMD)

Effective with release 10 the Vector MICROSAR Classic AUTOSAR stack offers the generation of A2L files for the BSW modules (Basic Software Components) and the SWC (Software Components) and thereby provides the symbolic information.

You can read internal states and data flow out of a control unit and analyze it together with bus data. Option AMD/XCP uses the established XCP and CCP protocols to read the data from the control unit.
