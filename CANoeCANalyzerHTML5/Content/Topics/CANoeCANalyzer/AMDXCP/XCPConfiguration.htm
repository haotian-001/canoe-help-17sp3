XCP Configuration

 

All devices contained in this configuration that are configured for communication via XCP or CCP are displayed on the left side in the XCP/CCP Window.
The respective settings/parameters are displayed on the right side.

Add Device

To add another device you need a valid A2L file. You can open this file with in the toolbar.
Alternatively, you can also import the entire XCP device configuration for an ECU. Select therefore the file type XCP Configuration (XCPCFG).

If the A2L file contains both an XCP and a CCP definition you must select the protocol type, here. If only one definition exists it will be selected automatically. If the A2L file contains different transport layers, e.g., FlexRay and CAN, you must select a transport layer. This selection cannot be changed afterwards.

Note

An A2L file can be added in CANoe using drag and drop. This operation creates a device and opens the XCP/CCP Window.

Context Menu of the Device

Activate/Deactivate Device
Activates/Deactivates the device for the following measurement. If the device is completely deactivated, you cannot re-activate it during the measurement; also XCP CAPL commands won't re-activate it.

Delete Device
Deletes an device from the configuration.
At the same time all system variables assigned to the device are also deleted.

Import Device Settings from CANape
Imports CANape project files (CANape.ini) to take over XCP/CCP settings relevant for CANoe.

Reset Device to A2L Settings
Reloads XCP/CCP settings from the A2L file.

Rename Device

Add Measurement Group
Adds a measurement group in the tree view.

Import Parameter Settings
Imports the parameters and their measurement settings.

Measurement Groups

Measurement groups allow the configuration of multiple parameter groups with different measurement settings. In this way you can easily change between the parameters. The measurement group active in each case is highlighted in bold face. If a measurement group is selected in the tree, all parameters of the A2L file are displayed.

You can add and delete measurement groups via the toolbar. You can rename measurement groups via the context menu.

You can switch between measurement groups during a running measurement. The XCP/CCP connection to the device must not be established.
You can activate a measurement group by double clicking on the measurement group, via context menu or by using the CAPL function xcpActivateMeasurementGroup.

Context Menu of the Measurement Group

Activate/Deactivate Measurement Group
Activates/Deactivates the measurement group for the following measurement. If the measurement group is deactivated, you can re-activate it during the measurement; also xcpActivateMeasurementGroup can re-activate it.

Delete Measurement Group
Deletes a measurement group from the configuration.
This command is only available for a deactivated measurement group.

Duplicate Measurement Group
Duplicates a selected measurement group.

Rename Measurement Group

Export Parameter Settings
Exports the parameters and their measurement settings.

Parameter List

If you have selected a device in the tree view on the left side, all XCP/CCP-relevant setting options are displayed, grouped according to subject, on the right side. You can change the settings or reset them to the default value with in the toolbar.

If you have selected a measurement group or a folder in the tree view on the left side, the so-called parameter list is displayed on the right side. Here you can configure the parameters to be measured and set an automatic measurement method.
By selecting multiple lines, you can change settings for multiple parameters simultaneously.

You can use drag and drop to move parameters for displaying in the Graphics Window or Data Window or the State Tracker. With <Ctrl>+<C> you can copy the name of the associated system variable to the Windows clipboard.

  • Configuration of the Column View

You can configure the view of the columns directly using the right mouse button with the Column Chooser context menu command of the table heading.

With a click of the left mouse button on a table heading, you can sort the list by the content of the column.

You can use the column filter to filter the parameter list by the following fields: Configured, Auto Read, Measurement, Cycle/Events, and Group.
Move the mouse over the table heading so that the column filter can be activated . You can select several values for the Cycle/Event and Group columns, for example, to display parameters from different groups together.

  • Description of the Parameter List Columns

Column

Description

Configured

Activates and deactivates access to a parameter.

If this option is activated, a system variable is created for accessing the parameter.

If the option is deactivated, the associated system variable is deleted.

Name

Name of the parameter from the A2L file.

Read only parameters are marked as such using the icon of the name column.

Auto Read

The read mode is started automatically after connection to the device.

Measurement

Set read mode

  • Polling
    Cyclic measuring as soon as CANoe is connected to the ECU.
    Note that polling of many parameters results in a high busload. If many parameters are involved, choose the DAQ measuring method, if possible.
    In order to measure only the necessary parameters, you have the option of activating/deactivating individual polling parameters during the measurement via CAPL and the XCP/CCP configuration dialog.
  • DAQ
    The control unit sends the parameters independently. Whether the DAQ measurement can be used depends on the implementation of the XCP slave in the control unit and the settings in the A2L file. Static and dynamic DAQ lists are supported.
  • Cyclic
    The control unit sends the parameters independently. This measurement method selects a DAQ event together with a factor (pre scaler) that matches the desired cycle time best.
  • OnConnect
    One-time measuring on establishment of connection to the XCP slave. This method is suitable, e.g., for reading configuration parameters.

Cycle/Event

Set cycle update

  • When Polling and Cyclic
    The cycle time of the polling measurement [ms].
  • With DAQ
    The XCP event with which the parameter is measured. The selection is dependent on the settings in the A2L file.

Value

Displays the last measured value of the parameter. Using the value can be queried from the ECU when an XCP/CCP connection exists.

RAM Address

Memory address of the parameter.

Comment

Comment of the parameter from the A2L file.

Group

Group from the A2L file of the selected parameter.

With an active XCP/CCP connection, you can change the cycle time and the automatic reading for parameters that are measured via polling.

If the XCP/CCP connection is not active, you can configure the measurement methods of all parameters as you wish while the measurement is running.

Search

With in the toolbar you open the search input on the right side above the parameter list of the XCP/CCP Window.

All currently visible columns are searched.

  • You can search for multiple terms simultaneously by separating them with a space character.
    The first hit per column will be highlighted.
    The term can be any part of the search result. The search is case insensitive.

  • You can restrict the search to a single column by entering the search term in the form column:term (without space character). You can specify a part of the column name as well.

    Example

    Search in column Name for ampl.

    Input: Name:ampl

  • You can combine multiple conditions with + or excluded with –. For this purpose enter the subsequent term without a space character after the + or – sign.

    Example

    Search rows where the column Name contains the term ampl and the column Measurement is not set to DAQ.

    Input: Name:ampl -Measurement:DAQ
