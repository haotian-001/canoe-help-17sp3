XCP on Ethernet

 

Option AMD/XCP supports XCP communication via UDP and TCP. This is possible both with and without the Option Ethernet. You can use the VN5610 or an Ethernet adapter of the Computer as the network interface. Note that the Ethernet adapter of the RT computer is used in distributed mode because the realtime simulation is executed there. Analogously, when VN8900 is used, note that its Ethernet adapter will be used.

Below is a description of all variants with the relevant possible settings.

CANoe Option AMD/XCP without Option Ethernet

Ethernet Adapter of the Computer

The correct adapter is selected automatically via the Windows TCP/IP configuration. From the destination address, which is set in the XCP/CCP Window for the device, Windows determines the adapter that can communicate with the same subnet:

In this case, an adapter that is connected to subnet 192.168.1.* is searched for. If the ECU is correctly connected, you must check the Windows network settings if no communication is possible.

Ethernet Adapter of the Computer in Distributed Mode

If CANoe is being used in distributed mode, the network settings of the RT computer are relevant and not those of the user computer. Use a remote desktop connection to check how the Windows network settings are configured. Ensure that the ECU is connected to the Ethernet adapter of the RT computer.

VN8900

If you are using a VN8900, the ECU must be connected to its Ethernet adapter. The settings of the adapter can be directly edited in CANoe.

Note

Without the .Ethernet option, you cannot see XCP frames in the Trace Window. The measured XCP values are nevertheless displayed as system variables. If errors occur you will find warnings and error messages in the Write Window.

CANoe Option AMD/XCP with Option Ethernet

If you have the .Ethernet option, you can set via Simulation ribbon tab the TCP/IP Stack dialog of CANoe.

Use TCP Stack of the Operating System

In this case the configuration is determined by the Windows settings just like in the case without theOption Ethernet. However, Ethernet packets with corresponding XCP protocol disassembly will be displayed in the Trace Window. Only for communication with localhost is this not possible for technical reasons.

Use CANoe internal TCP/IP Stack

With this setting, the network configuration in Windows is no longer relevant. Instead, you must set the IP address that Option AMD/XCP is to work with in the CANoe TCP/IP stack on the desired channel. Additionally, it is possible to create VLANs and to configure IP addresses there. If you have made multiple IP configurations, the VLAN or Ethernet channel that must be used for communication with the XCP ECU is determined automatically.

The assignment between the CANoe Ethernet channel and the Ethernet hardware results from the Network Hardware Configuration dialog. Either an Ethernet adapter of the computer or a channel of the VN5610 can be assigned here.

Note that the operation of distributed mode or standalone mode is possible in this case, as well. The Ethernet adapter of the corresponding device or connected VN5610 is displayed. Any Windows network settings are not relevant and are overwritten by the CANoe TCP/IP stack settings.

Ethernet packets with corresponding XCP protocol disassembly are displayed in the Trace Window.
