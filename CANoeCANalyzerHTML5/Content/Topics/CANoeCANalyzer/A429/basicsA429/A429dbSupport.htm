Database Support

 

ARINC 429 words can be written in a database (DBC format). The application data are then available as signals within CANoe. In addition, all network information can be represented symbolically.

The label is used as a unique key within the database. A new database can be created with the Aerospace Message Editor. Alternatively CANdb++ can be used (only for 8Bit long labels). The template A429_template.dbc is to be used for this.

Network Attributes

The database contains a network object that is described using the following attributes. These attributes are created automatically.

Name

Data Type

Values

Description

BusType

String

Default: “ARINC429”

This attribute ensures that the database is interpreted as a network description for ARINC 429.

This attribute must not be changed by the user!

Baudrate

Integer

Value range: 10500..110000

Default: 100000

Bitrate for the ARINC 429 bus

This value is not used for initialization of a channel. Rather it serves only as a comment.

A429Parity

Enumeration

Value range: unknown, odd, even

Default: odd

This value is not used for initialization of a channel. Rather it serves only as a comment.

ProtocolType

String

Default: "Aerospace"

This attribute activates "extended multiplexing". This enables a protocol interpretation using DBC signals.

This attribute must not be changed by the user!

ARINC Word Attributes

The following attributes belong to an ARINC word and may be changed by the user.

Name

Data Type

Description

A429CycleMin

 Integer

Indicates the minimum transmitting interval in microseconds.

The default value is specified by the user. This value is used as a comment.
A429Cycle Integer

This value is used to specify the transmitting interval of the message in microseconds.

This value is used for initialization in CAPL and in the Interactive Generator (IG).

With A429Cycle > 0, a default cycle time is specified for which A429CycleMin ≤ A429Cycle ≤ A429CycleMax applies.
A429CycleMax

Integer

Indicates the maximum transmitting interval in microseconds. The default value is specified by the user.

This value is used as a comment.
A429TransportDelayMax Integer

Indicates the maximum transport delay in microseconds. The default value is specified by the user.

This value is used as a comment.
A429EquipmentId Hex

This is the Equipment Identifier as part of the Information Identifier.

This value is displayed in the Equipment Identifier column of the Trace Window.

Value range: 0x0..0xFFF

Default: 0x0
A429Protocol

Enumeration

Value range:

  • BC (Broadcast)
  • AIM
  • A619 (ARINC 619 - ACARS)
  • WB1 (Williamsburg 1)
  • WB2 (Williamsburg 2)
  • WB3 (IEEE 802 compliant MAC)

Default: BC

Only the ARINC words are available for the different communication mechanisms. Generally, only broadcast is used. Bit-oriented protocols are also used in practice. In this case, the label is no longer used to describe the word content but rather to identify the receiver (SAL: System Address Label).

If this value deviates from BC, the display in the Trace Window is changed.
A429SpecSource Enumeration

Value range:

  • ARINC (issued by ARINC)
  • GAMA (issued by GAMA)
  • Private (private specification)

Default: ARINC

The definition of the ARINC word can originate from different standards and specifications. This value is used as a comment.

A429ExtendedID

Enumeration

Specifies whether the label is 8Bit or 10Bit long (including the SDI).

Value range:

  • Default (8Bit)
  • Extended (10Bit)

Default: Standard

If the message is created in the database, the value 4 (4 bytes) is entered as the message length. The bit numbering within the message starts at 0 and goes to 31 (parity bit).
