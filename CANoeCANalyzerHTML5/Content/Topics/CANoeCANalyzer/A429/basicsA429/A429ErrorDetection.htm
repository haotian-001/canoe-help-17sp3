Bus and Line Errors

 

With the VN0601, errors can only be detected if a transmission of ARINC words also takes place. The error detection for a Rx channel is dependent to a large extent on the setting of the channel parameters.

Rx errors

Error Description

Cause

Details

Gap Violation

The minimum gap for two consecutive ARINC words (gap) has been fallen below.

The value for the minimum gap can be changed in the hardware configuration.

The Gap Violation message is output in the Trace Window. The error was caused with the ARINC word that chronologically followed the error message.

The system variables GapErrors and WordErrors are increased.

Parity Error

The received data do not match the parity bit.

The evaluation of the parity can be changed in the Hardware Configuration.

The Parity Error message is output in the Trace Window. The data for the ARINC word are displayed in the Trace Window, but may also be incorrect.

The system variables ParityErrors and WordErrors are increased.

Bit length exceeded/Bit length too short

The duration of an individual bit of the ARINC word is outside the defined tolerance band.

The tolerance band can be changed in the hardware configuration.

The Bit length exceeded or Bit length too short message is output in the Trace Window.

The system variables BitRateErrors and WordErrors are increased. The receipt of the message is interrupted and the complete ARINC word is not available.

Invalid frame format

A format violation occurs when an edge has been detected before completion of the last bit of the ARINC word.

The Invalid frame format message is output in the Trace Window.

The system variables FormatErrors and WordErrors are increased. The received ARINC word is available in its entirety.

Invalid duty factor

The signal shape on the bus does not conform to the specification.

The individual phases of the bit time deviate more than 5%.

The Invalid duty factor message is output in the Trace Window.

The system variables DutyFactor and WordErrors are increased. This error is not detected in Auto mode.

Bitrate out of tolerance

The measured bitrate over one ARINC word deviates by more than 1%.

The deviation is based on the set bitrate in the hardware configuration.

The Bitrate out of tolerance message is output in the Trace Window.

The system variables WordLenErrors and WordErrors are increased. This error is not detected in Auto mode.

Bitrate range violation

The signal coding deviates from an RZ coding.

The Bitrate range violation message is output in the Trace Window.

The system variables CodingErrors and WordErrors are increased.

Tx errors

Error Description

Cause

Details

Unexpected line state

When an ARINC word is transmitted, the transmitted logic level does not match the received level.

This error pattern can result from a cable short-circuit or another active transmitter.

The Unexpected line state message is output in the Trace Window.

The system variables LevelErrors and LineErrors are increased. The ARINC word could not be transmitted but is displayed in the Trace Window.

Line is not idle

A check before the transmitting of an ARINC word has determined that the line is not idle.

This error pattern can result from a cable short-circuit or another active transmitter.

The Line is not idle message is output in the Trace Window.

The system variables IdleErrors and LineErrors are increased. The ARINC word could not be sent but is displayed in the Trace Window.
