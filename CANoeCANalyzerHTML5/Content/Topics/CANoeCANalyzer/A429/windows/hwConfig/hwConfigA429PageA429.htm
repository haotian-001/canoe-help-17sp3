AFDX » Network Hardware Configuration - ARINC429

Ribbon: Hardware Tab|Network Hardware|ARINC429

Configuration options are available for every defined channel. Not connected channels are advertised with . A is used if the channel is not yet enabled to be used by CANoe.

With the context menu the assigned communication direction is changed between Rx and Tx. A measurement may be started only if the number of available hardware channels is sufficient. In the mode Simulated bus there is no such restriction. The assignment of connected hardware to a specific application channel may be modified with the Application Channel Mapping dialog.

VN0601 (For a connection to ARINC 429 the ARINC interface VN0601 is necessary.)

The VN0601 is clocked internally at 64 MHz. All internal time measurements are based on this clock, and the bitrate is also derived from it. The minimum time resolution is 15.625 ns. The bitrate is an integer ratio of the clock rate. This produces a “grid” that is taken into account in the dialog for all time-related specifications.

  • Deactivated: The hardware does not perform a parity calculation for the channel.

  • Even: The hardware sets the parity bit in such a way that an even parity results.

Word gap

Valid range 1..128

The word gap for the channel occurs in bit times and is displayed in microseconds, based on the set bitrate.

Rx direction

Definition

The following definitions apply to the receive direction:

Bitrate: The upper and lower limit of the bitrate displayed in the hardware configuration dialog relates to the maximum/minimum time length of an individual bit. The maximum deviation of an individual bit is (± 2.5%). The VN0601 uses the set desired value (± 1.0%) to determine whether the bitrate of the complete message is within the tolerance range.

Parity: When the message is received, VN0601 checks the parity to determine whether it is even or odd. If the parity is deactivated on the VN0601, the hardware does not perform a parity check.

Word gap: The word gap for the channel, specified in bit times, is dependent on the upper and lower limit of the bitrate. The time between two messages is calculated from this. The word gap is additionally displayed in microseconds, based on the set bitrate.

Note

In Simulated bus mode Rx channels are still available but will not trigger any bus events. Therefore no configuration settings are available.
