Interactive Generator (A429)

 

With the Interactive Generator (IG), valid ARINC-429 words can be transmitted cyclically and event-based. For an existing database the payload can be processed byte-oriented as well as signal based. For already transmitted or received ARINC-429 words, it is possible to import the properties of these messages from the Trace Window to the IG.

The properties of ARINC-429 words can be displayed and changed via the A429 page of the Interactive Generator and the associated data columns.

Several specifics exist for ARINC-429 words that are explained hereafter.

Copying and Pasting ARINC-429 messages from the Trace Window

Previously transmitted or received ARINC-429 messages can be copied using the Copy to clipboard context menu of the Trace Window and imported into the Interactive Generator using the Paste | Paste A429 Word list entry.

The following assumptions and special treatments are applied automatically when messages are pasted:

  • For received (Rx) ARINC-429 messages, the first Tx channel is used.
  • If the same database is assigned to multiple channels, the message with the first applicable transmit channel is pasted.
  • If more than one ARINC-429 word is present on the clipboard, duplicates are not imported. An ARINC-429 word is recognized as a duplicate if the following properties match: network name, message name, DBC name, label, and payload.

Signals

If the signal sequences in the signal list of the messages are active, the signals of a message are set with the cycle time of the message or updated at each individual transmission.

Cyclic transmission of multiple instances of a message with identical properties

If multiple ARINC-429 messages with the same ARINC-429 identifier (label) on the same transmit channel are defined in an Interactive Generator, only one message declaration (line in the transmit list) can be cyclically transmitted simultaneously. If the cyclic transmission of such a message is activated, the cyclic transmitting of copies of this message (same label) is stopped. This is visually shown by only one activated check box in column Cycle Time.

Setting of the parity and minimum transmit gap

The currently set values of the corresponding Tx channel serve as the presetting for parity and gap time. These are applied for the selected message if the check box in the Channel Defaults data column is activated. If the check box is deactivated, the values for parity and gap time can be changed for each message, deviating from the ARINC-429 channel settings. The settings of the Tx channel for parity and gap time are then ignored.

If the burst transmit mode is used for event-based ARINC-429 messages( Burst data column), individual messages are transmitted taking into account the current gap time setting.
