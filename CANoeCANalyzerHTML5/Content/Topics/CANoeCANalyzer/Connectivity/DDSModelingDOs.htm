Distributed Objects (DOs) for Data Distribution Service (DDS)

 

DDS Modeling with Distributed Objects

DDS participants are modeled in CANoe using the new communication concept and distributed objects (DO).

Distributed objects of type consumed value and provided value are supported. Distributed objects of the consumed value type are used to participate in the communication in read mode; a consumed value thus corresponds to a DDS Reader. Distributed objects of the type provided value are used to participate in the communication in write mode; a provided value thus corresponds to a DDS Writer.

DDS only knows complex data types, therefore only values of type struct for consumed or provided value/data are allowed. To participate in the DDS communication, first the data type (in the example below HealthData) is defined. This data type is used in an interface (here IMonitor) as consumed value (or provided value). Finally an object (here Monitor) of the interface is built. The Monitor object contains a consumed value and thus can receive data of type HealthData. The Sensor object is of type reverse<IMonitor> and contains healthData as provided value, thus it is used to send the data.

Example

struct HealthData {
float heartRate;
[impl2phys=x*100.0, unit="%"]
float oxygenSaturation;
}

  [CommunicationPattern="SendReceive",
  Binding="DDS",
  DDS::Domain::ID=0,
  DDS::Reader::Reliability=RELIABLE,
  DDS::Reader::History=KEEP_ALL,
  DDS::Domain::Monitoring=ALL]
  interface IMonitor
  {
    // Topic subscribed by CANoe
    [DDS::Topic::Name="/Monitor/healthData"]
    consumed data HealthData healthData;
  }

object Monitor : IMonitor; // Subscribes
object Sensor : reverse<IMonitor>; // Publishes

Data Types and Attributes of DDS Modeling

The following data types and attributes can be used for DDS modeling:

Name

Possible Values

Description

Data types

Struct

These data types are supported for DDS in conjunction with distributed objects.

Data types within struct

Unsigned Integer and Integer (8, 16, 32, 64 bit), Bool, Double, Float, String, Struct, Array, List, Bytes

Not yet supported currently: Arrays of arrays, lists of lists, and nested combinations of both.

These data types can be used within a struct for DDS.

The following attributes can be used to configure DDS modeling:

Name

Possible Values

Description

Binding=<0>

DDS

Binds member to DDS binding.

CommunicationPattern=<0>

PublishSubscribe, SendReceive, SharedMemory

Communication pattern of the communication endpoint. Must always be set to SendReceive for modeling with DDS.

AutoConnect=<0>

true, false
  • true: Automatic connection after the start of the measurement.
  • false: Manual connection after the start of the measurement.

Default: true

DDS::Domain::ID=<0>

int64

The Domain Id. Should be set explicitly.

Default: 4294967295 (0xffffffff)

DDS::Domain::NetworkInterface=<0>

String

Network adapter or network adapter address (e.g. an IP address) to be used by the respective domain. This is achieved by creating a temporary Cyclonedds configuration file that exists only during the measurement. This attribute is incompatible with attribute DDS::Domain::Config.

DDS::Domain::Config=<0>

String

Filename or path to a Cyclonedds configuration file to be used for the domain. Only one configuration file can be specified per domain. Possible options can be found here. This attribute is incompatible with attribute DDS::Domain::NetworkInterface.

DDS::Domain::Monitoring=<0>

ONLY_REMOTE, ALL, NONE

Sets the mode of the domain monitoring i.e. sets when the DDS::DDSMonitor.readerAvailabilityChangedEvent and DDS::DDSMonitor.writerAvailabilityChangedEvent events are triggered. In ONLY_REMOTE mode, only events are triggered for DDS Reader and Writer that are not part of CANoe. Mode ALL triggers events for all DDS Reader and Writer. Mode NONE ensures that no Domain Monitoring events are triggered.

This does not affect the events that can be activated and deactivated with the attributes listed below.

Default: ONLY_REMOTE

DDS::Reader::ShowSubscriptionMatchedEvent=<0>

true, false

Indicates whether the SubscriptionMatchedEvent event is displayed in the Trace Window.

Default: true

DDS::Reader::ShowRequestedIncompatibleQoSEvent=<0>

true, false

Indicates whether the RequestedIncompatibleQoSEvent event is displayed in the Trace Window.

Default: true

DDS::Writer::ShowPublicationMatchedEvent=<0>

true, false

Indicates whether the PublicationMatchedEvent event is displayed in the Trace Window.

Default: true

DDS::Writer::ShowOfferedIncompatibleQoSEvent=<0>

true, false

Indicates whether the OfferedIncompatibleQoSEvent event is displayed in the Trace Window.

Default: true

DDS::Topic::Name=<0>

String

The topic name. There can be only one data type per domain.

Default: "Vector/CANoe"

DDS::Topic::Typename=<0>

String

An alternative name for the type, unless the default type name implied by the type is to be used.

DDS::Log::Level=<0>

ERROR, WARNING, INFO

Sets the global log level of the DDS binding. Should be set only once.

Default: ERROR

DDS::Key=<0>

String

Sets a member of a struct as DDS key. Currently only simple data types are allowed as key. Currently only one key per interface member is allowed.

DDS::Topic::ROS2Prefix=<0>

NO_PREFIX, RT_PREFIX

Adds ROS2 prefixes to a topic name.

The RT_PREFIX option inserts 'rt' before the topic name.

The NO_PREFIX option does nothing.

Default: NO_PREFIX

The following configuration settings (Quality of Service (QoS)) can be set on consumed values (DDS Reader) and provided values (DDS Writer) per attribute: Reliability, History, Durability, Lifespan, Liveliness. Further QoS will be implemented on request.

The attributes of the QoS settings can be found in the following table:

Name

Possible Values

Description

DDS::Writer::Reliability=<0>

DDS::Reader::Reliability=<0>

BEST_EFFORT, RELIABLE

Reliability QoS of a provided (DDS Writer) or consumed value (DDS Reader).

Default Writer: RELIABLE

Default Reader: BEST_EFFORT

DDS::Writer::Reliability_MaxBlockingTime=<0>

Time

Duration that the DDS Writer (provided value) can block if its queue is full of unconfirmed messages.

Default: 100 ms

DDS::Writer::History =<0>

DDS::Reader::History =<0>

KEEP_LAST, KEEP_ALL

History QoS of provided (DDS Writer) or consumed value (DDS Reader).

Default: KEEP_LAST

DDS::Writer::KEEP_LAST_Depth =<0>

DDS::Reader::KEEP_LAST_Depth =<0>

int64

Setting the history QoS of a provided (DDS Writer) or consumed value (DDS Reader).

Default: 1

DDS::Writer::Durability=<0>

DDS::Reader::Durability=<0>

VOLATILE, TRANSIENT_LOCAL, TRANSIENT, PERSISTENT

Durability QoS of provided (DDS Writer) or consumed value (DDS Reader).

Default: VOLATILE

DDS::Writer::Lifespan=<0>

DDS::Reader::Lifespan=<0>

Time

Lifespan QoS of a provided value (DDS Writer) or consumed value (DDS Reader).

Default: Infinite

DDS::Writer::Liveliness=<0>

DDS::Reader::Liveliness=<0>

AUTOMATIC, MANUAL_BY_PARTICIPANT, MANUAL_BY_TOPIC

Liveliness QoS of a provided (DDS Writer) or consumed value (DDS Reader).

Default: AUTOMATIC

DDS::Writer::Liveliness_leaseDuration=<0>

DDS::Reader::Liveliness_leaseDuration=<0>

Time

Setting the liveliness QoS of a provided (DDS Writer) or consumed value (DDS Reader). The time span tells how often an 'alive' message is expected/sent.

Default: Infinite

Note

The time span 9223372036854775807 ns is interpreted as infinite by the DDS binding.
