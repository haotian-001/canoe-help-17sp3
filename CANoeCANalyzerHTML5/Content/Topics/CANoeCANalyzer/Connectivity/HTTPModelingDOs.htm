Distributed Objects (DOs)

 

HTTP Modeling With Distributed Objects

The modeling of an HTTP client in CANoe is done using the new communication concept and distributed objects (DOs).

Only distributed objects of the consumed method type are supported. Here it is tried to set the body of the HTTP response to the return value. However, if an unsuccessful HTTP response is returned, i.e. a status code different from 200, a binding error is triggered with the corresponding status code.

The elements that can be used for modeling are listed in the table below:

Attribut [HTTPBinding::Body=<0>]

String of the form {{x}},{{y}}… where x,y... must be arguments of the consumed method.

Allows to specify which function argument should represent the body of the HTTP request. If not specified, arguments not specified for Path/Query/QueryParam will be used.

Attribut [HTTPBinding::Query=<0>]

String

Specifies an arbitrary query for an HTTP call. Cannot be used with QueryParam at the same time.

Attribut [HTTPBinding::QueryParam=<0>]

String of the form {{x}}, where x must be an argument of the consumed method.

Applies as query the string "x={x}", where {x} is the value of x.

JSON Serialization of Members

  • All return values and function arguments, that are used for the HTTP body of method participants that have the serialization attribute set to JSON or Default (local or inherited) are serialized with the JSON Serialization.
  • If JSON serialization is selected, the HTTP header field with Accept key and application/json value is added to the message.

Google Protocol Buffers (GoogleProtobuf) Serialization of Members

  • All return values and function arguments used for the HTTP body, from method participants with the serialization attribute set to GoogleProtobuf (local or inherited), are serialized using GPB Serialization.
  • If GoogleProtobuf serialization is selected, the HTTP header field with Accept key and application/octet-stream value is added to the message.

Plain Serialization of Members

  • All return values and function arguments of data type string from method participants with the serialization attribute set to plain (local or inherited) are serialized as UTF8 encoded strings.
  • All return values and function arguments of data type bytes from method participants with the serialization attribute set to plain (local or inherited) are transmitted unchanged as a byte array.
  • Other data types are not supported with this serialization.
  • If Plain serialization is selected, the HTTP header field with Accept key and application/octet-stream value is added to the message.
