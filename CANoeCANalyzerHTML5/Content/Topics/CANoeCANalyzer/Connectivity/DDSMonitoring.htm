DDS Monitoring

 

Monitoring Events

The DDS binding, which is used to model DDS participants in CANoe, triggers certain monitoring events in CANoe. Currently there are the following events and data types, which are described in more detail below:

The display of these events in the Trace Window can be activated or deactivated using various attributes.

SubscriptionMatchedEvent — DDS::DDSMonitor.subscriptionMatchedEvent [▲ back]

This event is triggered when a consumed value (DDS Reader) from CANoe finds a DDS Writer that uses the same topic and domain and has compatible Quality of Service (QoS) settings. The event is also triggered if a previously found DDS Writer disappears again. The event can be triggered per consumed value, i.e. per DDS Reader of CANoe.
