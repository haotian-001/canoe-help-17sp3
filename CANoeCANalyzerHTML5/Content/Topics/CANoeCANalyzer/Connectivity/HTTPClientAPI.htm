HTTP Client API

 

Introduction

Using the Client API allows sending client requests within the HTTP communication. The protocol-specific information of the HTTP protocol is available here.

The _HTTP::Client distributed object provides methods for sending the client requests. The usage of data types from the _HTTP::DataTypes namespace are described in more detail in HTTP Data Types.

To use the Client API, the HTTP binding must be activated. Therefor a vCDL with the content import module "HTTP" must be present in the Communication Setup.

Example

The following is an example of how to call the methods for C#, CAPL and Python.

C#

using System;
using Vector.Tools;
using Vector.CANoe.Runtime;

public class ClientAPIExample : MeasurementScript
{
  public override void Start()
  {
    var requestMessage = new _HTTP.DataTypes.HTTPRequestMessage()
    {
      HTTPMethod = "GET",
      RequestUri = "http://api.mathjs.org/v4/?expr=2%2B3*sqrt(4)"
    };
    _HTTP.Client.HTTP_SendRequest.CallAsync(requestMessage, (httpResponse) =>
    {
      Output.WriteLine(String.Format("HTTP Response Status: {0} {1}",
        httpResponse.StatusText,
        httpResponse.StatusCode.ImplValue));
    });
  }
}

import vector.canoe
import application_layer

@vector.canoe.measurement_script
class ClientAPIExample:

  #Notification that the measurement starts.
  def start(self):
    requestMessage = application_layer._HTTP.DataTypes.HTTPRequestMessage()
    requestMessage.HTTPMethod='GET'
    requestMessage.RequestUri = 'http://api.mathjs.org/v4/?expr=2%2B3*sqrt(4)'
    application_layer._HTTP.Client.HTTP_SendRequest.call_async(requestMessage,
      lambda httpResponse: vector.canoe.write(f'HTTP Response Status: {httpResponse.StatusText} {httpResponse.StatusCode}'))
