MQTT Client API

 

Introduction

The MQTT Client API provides a way to send and receive MQTT messages. To use the MQTT Client API, the MQTT service must be activated in the Connectivity dialog and the MQTT module must be imported into the vCDL (import module "MQTT").

The MQTT client API provides the consumed and provided methods. To use these, a distributed object must be created by the _MQTT::ClientAPI::IClientAPI interface. The consumed methods can be called on the object in the application model or in the Application panel, for example, to establish the connection with the broker or to publish a message. The provided methods are called on the object under the respective condition (according to the method name). The reaction to the call of these functions can be implemented in the application model. See also the Examples below. The syntax and meaning of the methods and their parameters are explained in the Methods chapter.

Communication Direction

consumed

Description

Establishes the connection with the MQTT broker.

Parameter

Name

Data Type

Description

ClientId

String

Name/designation of the client.

CleanSession

Bool (optional)

Specifies whether to reuse the previous session, if any.

If this parameter has the value true

  • the previous session is deleted when the connection is established
  • the current session is deleted when the connection is terminated.

KeepAliveInSeconds

Uint16 (optional)

Time span in seconds after which the client sends a ping request. This checks whether the connection is still active.

Username

String (optional)

Username (may be required by the broker).

Password

String (optional)

Password (possibly required by the broker).

LWT

_MQTT::DataTypes::MQTTPublishMessage (optional)

Message sent by the broker, when the connection with the client was unexpectedly terminated.

Examples

In the following you can see a use case of the MQTT Client API. In the data source (vCDL file), only a single object of the client API is created here for better comprehensibility. However, it is possible to create multiple objects and use them independently, even in addition to using self-defined communication participants (so-called high level clients). The specified code blocks for the application models in vCDL, C#, CAPL and Python have the same functionality. You can use the language you prefer.

Some of the methods are not used in the example. In this case, orient yourself to the other consumed or provided methods and use the overview of the MQTT Data Types.

vCDL

version 1.4;

import module "MQTT";

namespace ClientAPIExample

{
  _MQTT::ClientAPI::IClientAPI ExampleClient;
}

C#

using Vector.Tools;
using Vector.CANoe.Runtime;

public class ClientAPIExample : MeasurementScript {
  public override void Start()
  {
    ExampleClient.MQTT_Connect.Call("ClientAPIExampleClient", false, 60, null, null, null);

   // make the argument for MQTT_Subscribe
    var subscriptionRequest = new _MQTT.DataTypes.SubscriptionRequest();
    subscriptionRequest.TopicFilter = "livingroom/#"; //topic filter for all topics starting with "livingroom/"
    subscriptionRequest.RequestedQos.ImplValue = 0;

    ExampleClient.MQTT_SubscribeTopic.Call(subscriptionRequest);

    // make the argument for MQTT_Publish
    var publishMessage = new _MQTT.DataTypes.MQTTPublishMessage();
    publishMessage.Payload.SetBytes(System.Text.Encoding.ASCII.GetBytes("57 %"));
    publishMessage.PublishTopic = "livingroom/humidity";
    publishMessage.QoS.ImplValue = 0;
    publishMessage.Retain.ImplValue = false;
    publishMessage.DUP.ImplValue = false;

    ExampleClient.MQTT_Publish.Call(publishMessage);
  }

  public override void Stop()
  {
    ExampleClient.MQTT_Disconnect.Call();
  }

  [OnCall(ExampleClient.MemberIDs.MQTT_OnPublishReceived)] // function is called when a published message is received on a topic the client has subscribed on
  public void onPublishReceived(_MQTT.DataTypes.MQTTPublishMessage receivedMessage) // user defined implementation for the case of receiving a message. The name of the function can be anything.
  {
    if (receivedMessage.PublishTopic == "livingroom/humidity")
    {
      var humidity = System.Text.Encoding.ASCII.GetString(receivedMessage.Payload.GetBytes());

      Output.WriteLine("Humidity of " + humidity + " received."); // write message to Write Window
    }
    else
      {Output.WriteLine("Message received.");
    }
  }
}

Python

import vector.canoe
import application_layer

@vector.canoe.measurement_script
class ClientAPIExample:
    def start(self):

      application_layer.ClientAPIExample.ExampleClient.MQTT_Connect.call("ClientAPIExampleClient", False, 60, None, None, None)

      # make the argument for MQTT_Subscribe
      subscriptionRequest = application_layer._MQTT.DataTypes.SubscriptionRequest()
      subscriptionRequest.TopicFilter = "livingroom/#" # topic filter for all topics starting with "livingroom/"

      subscriptionRequest.RequestedQos = 0

      application_layer.ClientAPIExample.ExampleClient.MQTT_SubscribeTopic.call(subscriptionRequest)

      # make the argument for MQTT_Publish
      publishMessage = application_layer._MQTT.DataTypes.MQTTPublishMessage()
      publishMessage.Payload = bytes("57 %%", "ascii")
      publishMessage.PublishTopic = "livingroom/humidity"
      publishMessage.QoS = 0
      publishMessage.Retain = False
      publishMessage.DUP = False

      application_layer.ClientAPIExample.ExampleClient.MQTT_Publish.call(publishMessage)

    def stop(self):

      application_layer.ClientAPIExample.ExampleClient.MQTT_Disconnect.call()

    @vector.canoe.on_call(application_layer.ClientAPIExample.ExampleClient.MQTT_OnPublishReceived) # function is called when a published message is
received on a topic the client had subscribed on
    def onPublishReceived(self, receivedMessage): # user defined implementation for the case of receiving a message. The name of the function can be anything.

      if receivedMessage.PublishTopic == "livingroom/humidity":

        humidity = str(receivedMessage.Payload, "ascii")

        vector.canoe.write(Humidity of " + humidity + " received.") # write message to Write Window

      else:

        vector.canoe.write("Message received.")

In this example, a distributed object of the MQTT client API is created and connected to the broker in the application model at the start of measurement. The object then subscribes to a topic filter and then publishes itself to a topic within this filter. At the end of the measurement, the connection with the broker is terminated. The object reacts to receiving a message and issues a corresponding notification in the Write Window.

You can also call the consumed methods of the MQTT client API in the Application panel. For example, you can perform a Disconnect and Reconnect after the start of the measurement. When publishing a message, the payload must be entered in bytes. To do this, first enter the number of bytes and then the values of the bytes in the Application panel.
