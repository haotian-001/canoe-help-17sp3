HTTP Protocol

 

Introduction

Hypertext Transfer Protocol (HTTP) is a protocol for transmitting application-layer data over a computer network. While the most common application is loading web pages from the WWW in the browser, another common application is communication between a client and a server, which do not necessarily have to be on the same network.

For the usage of HTTP 1.1 CANoe offers two possibilities. One option is modeling based on distributed objects, described in HTTP Modeling with Distributed Objects. Here, HTTP-specific properties are abstracted with attributes and a possible response from the server is represented directly. A more detailed interaction with a HTTP server is possible with the Client API, described in HTTP Client API.

HTTP messages are split into different methods, with the following subset supported when modeling using distributed objects:

  • GET

    This method queries resources from the requested URI.

  • POST

    Messages with the POST method are usage to transmit content from a client to a server for further processing. The type of message accepted by the server is defined by the server.

  • PUT

    Used to upload a record to a server. If a resource already exists at the specified URI, it will be replaced.

  • DELETE

    Used to delete a specified resource on the server.

Configuration

To activate the HTTP protocol, in the CANoe Options dialog under Extensions |Connectivity activate the HTTP check box. There, also select the desired Mode via the list (see below). Use the button in the Configuration column to make further settings, e.g. the configuration of a proxy to be used.

The supported modes are:

  • ConnectivityInterface

    If the distributed mode is used, it is only necessary that the user computer (the computer on which the graphical interface is executed) has an Internet connection.
