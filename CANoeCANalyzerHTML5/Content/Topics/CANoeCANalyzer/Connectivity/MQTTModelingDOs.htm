Distributed Objects (DOs)

 

MQTT Modeling with Distributed Objects

The modeling of an MQTT client in CANoe occurs according to the new communication concept and distributed objects (DOs).

The elements which can be used for modeling are listed in the following table:

Designation

Possible Values

Description

Member types

Data, event

These member types are supported for MQTT in connection with distributed objects.

Data types

Unsigned integer, integer, double, float, UTF8 string, struct, union, array, bool, bytes

These data types are supported for MQTT in conjunction with distributed objects.

Attribute

[Binding=<0>]

MQTT

Binds member to the MQTT binding.

Attribute

[CommunicationPattern =<0>]

PublishSubscribe

MQTT is a publish/subscribe pattern, which is why only this value is permissible here.

Attribute

[AutoConnect=<0>]

True, false

True – Automatic connection following the start of measurement.

False – Manual connection following the start of measurement.

Default: True

Attribute

[AutoSubscribe=<0>]

True, false

True – Automatic subscribing with MQTT broker following connection.

False – Manual subscribing to MQTT broker.

Default: True

Attribute

[MQTTBinding::ClientId=<0>]

String

Assignment of DO member to a client.

(see exceptions under Supported Scope of the MQTT Protocol).

Attribute

[MQTTBinding::Topic=<0>]

String

provided Member:

topic name for PUBLISH control packet.

consumed Member:

topic filter for SUBSCRIBE control packet.

(see exceptions under Supported Scope of the MQTT Protocol).

Attribute

[MQTTBinding::Username=<0>]

String

Defines the user name to authenticate with the broker.

Attribute

[MQTTBinding::Password=<0>]

String

Defines the password to authenticate with the broker.

Attribute

[MQTTBinding::CleanSession=<0>]

True, False

CleanSession flag for CONNECT control package.

Default: True

Attribute

[MQTTBinding::Retain=<0>]

True, False

Retain Flag for PUBLISH control package.

Default: False

Attribute

[MQTTBinding::KeepAlive=<0>]

Uint16

KeepAlive interval in seconds for CONNECT control package.

Default: 0

Attribut

[MQTTBinding::OnConnectionLost=<0>]

Ignore, StopMeasurement, RestoreConnection

Defines how to handle a loss of connection to the broker on the client side:

  • StopMeasurement
    Automatic termination of the measurement.
  • RestoreConnection
    Automatically attempt to reconnect all DO members of the client.
  • Default
    Ignore

Attribut

[MQTTBinding::Connection:: ConnectTimeOut=<0>]

Uint32

The time in seconds after which the connection attempts to the broker are stopped.

Attribut

[MQTTBinding::Connection:: ReconnectMinTime=<0>]

Uint32

Lower limit of the time interval in seconds that is waited between two connection attempts.

Attribut

[MQTTBinding::Connection:: ReconnectMaxTime=<0>]

Uint32

Upper limit of the time interval in seconds that is waited between two connection attempts.

Attribut

[MQTTBinding::Publish::ResendTime=<0>]

Uint32

The time in seconds between two attempts to send a QoS1 PUBLISH message.

Attribut

[MQTTBinding::Publish:: MaxAttemptsToResend=<0>]

Int32

The number of attempts to resend an unanswered PUBLISH message. .

Attribut

[MQTTBinding::QoS=<0>]

QoS0, QoS1

Defines the used Qualitiy of Service-Level.

Default: QoS0

Attribut

[SecurityManager::SecurityProfile=<0>]

String

Corresponds to the ID of the connectivity profile to be used in Security Manager.

(At this point, in Security Profile Explorer: right click on Vector Security Source/Connectivity Authentication/Your Security Profile | option Copy Security Profile ID to Clipboard appears).

Attribut

[MQTTBinding::Authorization::AuthParamName=<0>]

String

Corresponds to the Name field of the TLS configuration to use in the selected Security Manager PKI profile.

Attribute

[Serialization=<0>]

Default, JSON, GoogleProtobuf, Plain

(For MQTT : Default = JSON)

Describes the serialization format of the MQTT payload.

JSON Serialization of Members

All data and event members with the serialization attribute set to JSON or Default (locally or inherited) are serialized with the JSON Serialization.

Google Protocol Buffers (GoogleProtobuf) Serialization of Members

All data and event members with the serialization attribute set to GoogleProtobuf (locally or inherited) are serialized with the GPB Serialization.

Plain Serialization of Members

  • All data and event participants of data type string for which the serialization attribute is set to plain (local or inherited) are serialized as UTF8-encoded strings.
  • All data and event participants of the data type bytes for which the serialization attribute is set to plain (local or inherited) are transmitted unchanged as byte arrays.
  • Other data types are not supported with this serialization.
