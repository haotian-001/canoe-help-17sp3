System Under Test (SUT)

 

SUT

The SUT is the connected device to be tested which is to be connected with CANoe via MQTT broker in the Connectivity Hub.

The URI with which the SUT is to connect can be taken from the configuration dialog under the Endpoint item.

In order for the Connectivity Hub to identify the SUT, the SUT certificate must also be provided when establishing the TLS. An SUT can also be comprised of several different single devices which use the same SUT certificate.

SUT example

The following example shows accessing of the MQTT broker in the Connectivity Hub by a connected SUT.

A Raspberry Pi with access to the Internet serves as the connected device in this case. An MQTT client is implemented on the device via JavaScript. NodeJS is used as the execution environment.

JavaScript

// Imported namespaces
const fs   = require("fs");
const mqtt = require("mqtt");

//CFS URI from the broker
const brokerAddress = <Endpoint>;

//create mqtt client
var mqttClient = mqtt.connect(
  brokerAddress,
    {
      key:                fs.readFileSync(<SUT-KEY-PEM>),
      cert:               fs.readFileSync(<SUT-CERT-PEM>),
      ca:                 fs.readFileSync(<SUT-CA-CERT>),
      rejectUnauthorized: true,
    });

//suscribe to topic "rooms/living/devID00002/temperature"
mqttClient.subscribe("rooms/living/devID00002/temperature");

mqttClient.on("message", (topic, message) => {
  if (topic == "rooms/living/devID00002/temperature") {
    //double the temperature and return it as speed
    var newSpeed = parseFloat(message) * 2;
    mqttClient.publish("rooms/living/vID00002/Speed", newSpeed.toString());
  }
  //add more code here
});

The application subscribes to the topic rooms/living/devID00002/temperature with the MQTT broker.

If the temperature sensor PUBLISHES a change, the application calculates a speed of rotation and sends a PUBLISH message to a ventilator subscribed to the topic rooms/living/vID00002/Speed with the MQTT broker.

The following placeholders must be replaced accordingly:

<Endpoint>
The endpoint which can be taken from the configuration dialog, e.g. cfs.vector.com.

<SUT-KEY-PEM>
The path to the file that contents the private key for the SUT certificate, e.g. sut-key.pem.

<SUT-CERT-PEM>
The path to the file that contents the SUT certificate, e.g. sut-cert.pem.

<SUT-CA-CERT>
The path to the CA certificate, e.g. ca-cert.pem.

External dependencies

The MQTT packet is required for execution. The packet can be obtained via Node Packed Manager (NPM) or the Bash shell:

npm install mqtt

The SUT certificate is also required in the project directory.
