Use Application Messages

 

The following example shows how to use ASN.1:

Example

ASN1Module DEFINITIONS AUTOMATIC TAGS ::= BEGIN

ASN1Protocol ::= SEQUENCE {
  booleanField BOOLEAN OPTIONAL,
  bitStringField BIT STRING { bit1(0), bit2(1) },
  sequenceField SEQUENCE { enumeratedField ENUMERATED { off (0), on (1) } },
  arrayField SEQUENCE ( SIZE(0..10) ) OF SeperatlyDefinedChoice
  date DATE,
  classId    MyClass.&id ({Class-Set}),
  classValue MyClass.&Value ({Class-Set}{@id})
}

SeperatlyDefinedChoice ::= CHOICE {
  numberChoice INTEGER { namedNumber(123) },
  ia5StringChoice IA5String
}

MY-CLASS ::= CLASS
{
  &id ClassKey UNIQUE,
  &Value
} WITH SYNTAX {&Value IDENTIFIED BY &id}

ClassKey ::= INTEGER SIZE (0..255) { one(1), two(2) }

classOne MY-CLASS ::= { INTEGER IDENTIFIED BY one }
classTwo MY-CLASS ::= { BOOLEAN IDENTIFIED BY two }

Class-Set MY-CLASS ::= { classOne | classTwo }

END

Token designators of an ASN.1 protocol

To access the fields of an ASN.1 based protocol with the Packet API you have to derive the appropriate token designator from the ASN.1 module.

The following scenarios are possible, relating to the shown example:

  • If ASN1Protocol is selected as highest data type then the token designator of a field is the field name.
    Referred to the example you could access the first protocol field with the token designator booleanField.
  • If the protocol field is a SEQUENCE you can access the field by appending a point and the field name to the token designator.
    Referred to the example a valid token designator could be sequenceField.enumeratedField.
  • If the protocol field is a SEQUENCE OF you can access an element of the field by appending the index of that elements in brackets to the token designator.
    Referred to the example you could access the first element with arrayField[0].
  • If the protocol field is a CHOICE you can access the data type of the selection by appending a point and the token of the data type to the token designator.
    Referred to the example you could use a valid token designator arrayField[0].numberChoice.
  • If a SEQUENCE field is specified by an Information Object Class, a distinction must be made if it is the key of the class (UNIQUE attribute in the class definition) or not. As with normal fields, you can access the key field via the name (classId in the example). Similar to CHOICE, an additional selection designator must be specified for fields whose content depends on the key field. The selection designator is the name of the Information Object (classOne, classTwo) that determines the data type of the field. In the example the designator is classValue.classOne if the classValue is of type INTEGER.
  • If the protocol field is of data type TIME or of a sub data type DATE, TIME-OF-DAY, DATE-TIME or DURATION, you can access the time and date parts by appending the name of that part to the token designator.

Access ASN.1 data types with CAPL

With a protocol designator and a valid token designator that are defined during import you can use the CAPL functions of the Packet API. For example you can get and set values of protocol fields of a packet with the Get… and Set… functions. The display of those values depends on the ASN.1 data type.

Additional information

  • With C2xInitProtocol a packet can be created with the ASN.1 based protocol as payload. At first the packet is empty, that means no protocol fields are available. DEFAULT values from the ASN.1 module are not used to create and initialize the DEFAULTS.
  • The number of elements for SEQUENCE OFs cannot be set. Instead the number of elements is set during writing to an element. E.g. if the SEQUENCE OF contains two elements and is set with arrayField[4].ia5StringChoice the SEQUENCE OF is extended to five elements. Then the third and fourth element must be set with a value.
  • The selection of a CHOICE cannot be set explicitly. It's done automatically if a value is assigned to a data type of the CHOICE. E.g. a C2xSetTokenInt with arrayField[0].numberChoice set the CHOICE arrayfield[0] to numberChoice.
  • If you set a field that depends on a key field (e.g. classValue.classTwo), the key field is set accordingly (e.g. classId to the value two). If a key field is set to a new value, all dependent fields will be deleted.
  • The Information Object Class can only be used with PER encoding. It is not supported by the ASN.1 Import Wizard.
  • When calling C2xCompletePacket, unset mandatory values are initialized with default values. Then the packet is coded with the assigned ASN.1 encoder and written to the payload of the underlying protocol.
  • CAPL do not provide the conversion of UTF-8 string to or from the used character set.
  • If CANoe receives a message with a corrupt ASN.1 layer this message is marked as erroneous and decoded as far as possible in the Detail view of the Trace Window. However it is not possible to access the field of the ASN.1 layer in CAPL.
