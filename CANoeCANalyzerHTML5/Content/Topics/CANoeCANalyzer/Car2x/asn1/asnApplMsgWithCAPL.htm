Process Application Messages with CAPL

 

This example shows how you can create an application message as a packet with CAPL, fill it with data and send it. It is shown too how you can read and analyze an application message with a CAPL event handler.

Create CAPL data structures and methods

Because the application message is described with ASN.1, the ASN.1 specification has to be imported to CANoe.

The following simple ASN.1 description is used for this example (transmitted in protocol BTP with destination port 1000):

SIMPLE-PDU-Descriptions {
}

DEFINITIONS AUTOMATIC TAGS ::=

BEGIN
SimplePdu ::= SEQUENCE {
  -- header is mandatory
  header SimpleHeader,
  -- data is an optional field
  data SimpleData OPTIONAL
}

SimpleHeader ::= SEQUENCE {
  -- mandatory sample field
  id INTEGER(0..100), -- 7 bits
  -- optional sample field
  frequency INTEGER (0..255) OPTIONAL --units of 0.1 Hz
}

SimpleData ::= SEQUENCE {
  simpleEnum ENUMERATED { first (0), next (1), last (2) },
  value INTEGER (0..900000000)
}
END

After the import the following CAPL files are available in folder Vector_CN_ASNV in the directory of your CANoe configuration:

  • ASNV_Template_SimplePdu.cin
  • ASNV_Template_BaseDatatypes.cin

Include CAPL files

The file C2x_Basics.cin is an include file with help functions that is part of the CANoe.Car2x sample configurations. Include this file and the file ASNV_Template_SimplePdu.cin to your CAPL program as follows:

includes
{
  #include "C2x_Basics.cin"
  #include "..\Vector_CN_ASNV\ASNV_Template_SimplePdu.cin"
}

Now the following data structures and functions are available in CAPL that you can use to send and receive application messages.

Data structures

struct SimplePduSimpleHeader {
  INT isValidFlag; //necessary if this struct is used as an optional field of a SEQUENCE
  LONG id; //INTEGER, 0...100
  struct OPT_LONG_ASN frequency; //INTEGER, OPTIONAL, 0...255
};

struct SimplePduSimpleData {
  INT isValidFlag; //necessary if this struct is used as an optional field of a SEQUENCE
  INT simpleEnum; //ENUMERATED, first(0), next(1), last(2)
  LONG value; //INTEGER, 0...900000000
};

struct SimplePdu {
  INT isValidFlag; //necessary if this struct is used as an optional field of a SEQUENCE
  struct SimplePduSimpleHeader header; //SEQUENCE
  struct SimplePduSimpleData data; //SEQUENCE, OPTIONAL
};

Fields that are marked as optional contain a validation flag isValidFlag in its structure. If you want to fill optional fields with data (as it is done in the example above with field SimplePduSimpleData data), then set isValidFlag to 1 for the appropriate field.

A comfortable and efficient way to transfer the generated data structures to the CAPL Browser is provided by the Car2x Network Explorer. This browser shows the generated data structures in a clear tree view, in addition you can search within the data structures and generate CAPL code.

Functions

// create a packet for SimplePdu-message
LONG API_InitSimplePduPacket()

// create a packet for SimplePdu-message and initialize with valid data
LONG API_InitSimplePduPacket(struct SimplePdu value)

// create a secured packet for SimplePdu-message and initialize with valid data
LONG API_InitSimplePduPacket(struct SimplePdu value, char packetTypeDesignator[],
                             char certificateName[])

// set the data of a packet for SimplePdu-message
int API_SetSimplePduParams(LONG packetHandle, struct SimplePdu value)

// get the data of a packet for SimplePdu-message
int API_GetSimplePduParams(LONG packetHandle, struct SimplePdu value)

Send application message in CAPL

You can create a packet of the application message and a timer for cyclic sending of the packet in the CAPL event on start.

variables
{
  // timer variables
  msTimer timer_SendSimplePdu;

  // cycle time for sending SimplePdu-message
  int gTimerCycleTime = 1000;

  // handle for SimplePdu packet (data)
  LONG gSimplePduHandle = 0;
}

on start
{
  Appl_Init();
}

void Appl_Init()
{
  // create a packet for SimplePdu
  gSimplePduHandle = API_InitSimplePduPacket();

  // start sending SimplePdu-packet (cyclic)
  setTimer(timer_SendSimplePdu, 5);
}

on timer timer_SendSimplePdu
{
  SendSimplePdu();
  setTimer(timer_SendSimplePdu, gTimerCycleTime);
}

You can fill and send the generated data structures (struct SimplePdu) as follows:

void SendSimplePdu()
{
  int errorFlag = 0;

  // data-structure for the SimplePdu
  struct SimplePdu simplePdu;
  simplePdu.header.id = 1;
  simplePdu.header.frequency.isValidFlag = 1;
  simplePdu.header.frequency.value = 100;

  // set validFlag to 1 to indicate that this optional field is used
  simplePdu.data.isValidFlag = 1;
  simplePdu.data.simpleEnum = 1;
  simplePdu.data.value = 10000;

  // fill packet with data of SimplePdu-structure
  API_SetSimplePduParams(gSimplePduHandle, simplePdu);

  // 1000 is the BTP destination port that is defined in ASN.1-import
  // The function SetPacketData_BTP is defined in the file C2x_Basics.cin
  if (SetPacketData_BTP(gSimplePduHandle, 1000, 0, 0) != 0) {
    errorFlag=1;
  }

  // send packet if no error occured
  if (errorFlag == 0)
  {
    if (C2xCompletePacket(gSimplePduHandle) != 0) {
      write ("C2xCompletePacket error: cannot encode SimplePdu message");
    }
    else {
      C2xOutputPacket(gSimplePduHandle);
    }
  }
}

In this example the optional field simplePdu.data is filled and used, that's why simplePdu.data.isValidFlag is set to 1.

Analyze application message in CAPL

For the reception of an application message you need to register a receive function (for WLAN events) in the CAPL event on prestart with C2xReceivePacket first. If an application message is received, you can read it by calling the function API_GetSimplePduParams in the receive function (in this example OnC2xPacket) and then analyze the application message.

on preStart
{
  // init the receive function
  C2xReceivePacket("OnC2xPacket");
}

/******************************************************/
//
// function is called when a packet is received
//
/******************************************************/
void OnC2xPacket (LONG channel, LONG dir, LONG radioChannel, LONG signalStrength, LONG sigQuality, LONG packet)
{
  struct SimplePdu simplePdu;

  if(API_GetSimplePduParams(packet, simplePdu) == 0) {
    // check if data (optional field) is set for the SimplePdu. The isValidFlag is always 1 for structs which are not optional
    if(simplePdu.data.isValidFlag == 1) {
      write("SimplePdu data: %i", simplePdu.data.value);
    }
  }
}
