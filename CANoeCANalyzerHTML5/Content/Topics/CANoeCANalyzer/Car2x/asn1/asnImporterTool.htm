ASN.1 Import Wizard

Windows Start Menu|CANoe|Tools|ASN.1 Import Wizard

To use an Â»ASN.1 based protocol in CANoe, you have to convert the ASN.1 module into a Car2x database using the tool ASN.1 Import Wizard. This database can then be added to a CANoe configuration.

The tool provides a graphic user interface to easily configure the settings. If you have configured all necessary settings, you can start the import.

First configuration page

On this page you select the file with the ASN.1 module (ASN.1-Specification). If this module uses so-called imports (other ASN.1 modules), you have to add these modules to the selected one. Furthermore, you must select a directory in which the created files are to be saved.

The options of the area Options are used to send a packet with an ASN.1 based protocol. To receive that packet, the ASN.1 based protocol has to be imported with the same options.

  • Encoding: Select the format of the fields of the ASN.1 based protocol. The following formats are supported
  • Packet Encoding Rule (Â»PER) (unaligned)
  • Basic Encoding Rule (Â»BER): The DER variant is used for coding, for decoding each BER variant is accepted.
  • Underlying Protocol: Select the protocol that is used to transmit the ASN.1 based protocol.
  • First and second selector: The function depends on the underlying protocol.

Example

If you have selected btp as the underlying protocol, you can define that the ASN.1 based protocol is transmitted on a specific destination port. Then CANoe interprets the payload with the set ASN.1 description only if a packet is sent or received with the configured values.

Second configuration page

On this page you select the ASN.1 data type that contains all fields of the ASN.1 based protocol. The name of the data type (it could be that the name of the ASN.1 module is displayed to the left of the data type) is used as protocol designator which is used to identify the ASN.1 based protocol in the CANoe configuration.

Note

If you use CAPL functions of the Packet API you have to use the protocol designator to access the fields of the ASN.1 based protocol.

Import procedure

After clicking [Import], two files are created in the selected import directory:

  • The file ASNV_Configuration.xml is the database that you can add to your CANoe configuration. If this file already exists before the import, it is not created again. Instead, a new entry for the imported ASN.1 protocol is added within the file.
  • The file ASNV_Module_{protocol designator}.xml contains the ASN.1 protocol converted into an internal CANoe format.

Notes

  • You can analyze the created database with the Car2x Network Explorer.
  • If you want to move or copy the Car2x database, all XML files must always be moved or copied together.
