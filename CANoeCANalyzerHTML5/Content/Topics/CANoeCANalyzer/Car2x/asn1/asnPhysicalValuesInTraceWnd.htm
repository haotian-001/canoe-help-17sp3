Show Physical Values with ASN.1 in Trace Window

 

The ASN.1 standard does not allow to describe the physical interpretation of raw values that are transmitted as INTEGER. therefore the ASN.1 Import Wizard supports the following extension for the INTEGER data type so that the interpreted values can be shown in the Trace Window:

INTEGER PHYSICAL { FACTOR(f) OFFSET(o) UNIT("u") }

Note

This extension is no standard ASN.1 and is not supported by other ASN.1 tools.

Part

Type

Meaning

Default

Usage

f

floating-point number

specify the conversion from transmitted raw value to physical interpretation of the value

[physical value] = [raw value] * f + o

1.0

optional

o

floating-point number

0.0

optional

u

string

specifies the physical unit

""

optional

Example

SEQUENCE
{
x INTEGER (0..100),
y INTEGER PHYSICAL { FACTOR(0.1) OFFSET(-5) UNIT("m") } (0..100)
}

With transmitted raw values x=35 and y=35 the sequence is displayed as follows:

x 35 35
y -1.5m 35
