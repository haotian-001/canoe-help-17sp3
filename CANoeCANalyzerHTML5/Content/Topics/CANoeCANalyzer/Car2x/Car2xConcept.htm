Car2x for CANoe | Concept

 

Overview

CANoe Option Car2x supports the basic concepts of CANoe and extends them with Car2x-specific functions.

For analysis of Car2x communication between vehicles and with the infrastructure, CANoe Option Car2x supports the IEEE 802.11p (WLAN) interface.

Serving as the communication module is the VN4610 or alternatively radio modules of different manufacturers. CANoe supports the 802.11p radio adapters Cohda Radio MKx, NEC LinkBird V2, V3 and V4, DENSO WSU-01 and WSU-015, Delphi CRPU and Continental CCU MK2 and MK3. Data are exchanged between the radio adapters and CANoe via Ethernet.

Analysis

Option Car2x supports and extends the basic analysis of CANoe.

Existing analysis windows are extended to interpret and analyze WLAN packets. Protocol information such as header and payload can be displayed in detail. Geographic coordinates that are received in a Car2x network can be comfortably displayed in a map too.

The analysis of data can be made more comfortable by using filter. Ethernet packets that are not interesting for analysis can be filtered out of the data stream.

Simulation and Stimulation

With Option Car2x for CANoe, Car2x scenarios can be simulated and the device under test (DUT) can be stimulated.

WLAN packets can easily be created, configured, and sent. Application payload written in ASN.1 can be interpreted too to access the data fields of the packets. This simplifies the simulation of Car2x networks and the test of ECUs.
