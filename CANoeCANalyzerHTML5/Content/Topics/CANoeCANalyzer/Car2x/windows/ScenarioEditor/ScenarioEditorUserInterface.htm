Scenario Explorer

In this area, the structure of a scenario is displayed in a hierarchical structure. The first level contains the created routes, the reference points and the stations are assigned to them as subelements, and the sensors are assigned to the stations. The measuring tapes are located on the same level as the routes.

When selecting an element in the scenario explorer, the properties of the selected element are displayed in the Properties.

Reference points, stations and sensors can be moved to a suitable target using drag and drop (left mouse button). If <Ctrl> is pressed (also when dragging and dropping with the right mouse button), the element is copied and not moved.

Duplicate Route Dialog

Via the context menu of an existing route, another route can be created with a certain distance to the existing one.

Here, the desired distance in meters, the direction of travel (same direction or oncoming) as well as the position (left or right, starting from the direction of travel of the existing route) relative to the existing route can be specified.

Timeline

In the timeline, so-called attributes such as Speed are displayed for stations.

 

You can create Keypoints for attributes to determine the value of an attribute at the time of the respective keypoint.

Once created, a keypoint can be moved along the timeline of the scenario (drag with left mouse button and move left/right). To snap the keypoint to a full second, hold the Ctrl button while dragging the keypoint or enter the exact value in the Properties.

Each newly created attribute has a fixed key point at time 0, this cannot be moved or deleted and is displayed in red.

Speed is an attribute that is generally available for stations, but user-defined attributes can also be created.

 

 

If the attribute is configured with Constant interpolation, the value of the attribute remains the same until a new keypoint defines a new value. Linear interpolation means the attribute value will change linearly between two keypoints, depending on scenario time between the keypoints.

You can access attributes in CANoe via CAPL.

Note

The rules for naming identifiers apply to the naming of attributes: only English alphanumeric characters and '_' are allowed. The following can be used as the physical data:

  • typeA signed integer (8 bytes long)
  • double precision floating point number (8 bytes long)

Application Events

From the perspective of the scenario editor, Car2x/V2x application events are a group of predefined station attributes with editable temporal value progression (via keypoints of the respective attribute). Structurally related, supported application events can be created via the context menu command Create Car2x/V2x-Event ... of the timeline.

After selection, a dialog opens in which application events can be created.

Properties

The properties of the selected element, i.e. the route, the reference point, the station, the sensor or a keypoint, are displayed here. These properties can be edited by the user.

Map

The routes and stations are displayed graphically on the road map of the application.

A route is defined on this map by waypoints. Simply click on the map to create waypoints. To do this, the scenario editor must be in edit mode, which can be accessed via edit (ribbon|group routes).

The points can be selected with the mouse and moved by dragging on the map, which is only possible in edit mode.

The start position of the stations can be set via the property Offset in the properties or by dragging with the mouse.

 

 

Right-click a point on the map to display the context menu and the coordinates of the point. Select the coordinates to copy them to the clipboard.

Additionally you can create measuring tapes via the context menu. These have two measurement cursors to which the distance is displayed once in the map, but also in the properties of the respective tape measure.

To move the measurement cursors of a scale, the edit mode does not have to be activated.

 

Tape measures can be locked in the center of a station or a sensor. This can be done either via the context menu, if the measurement cursor has already been placed correctly, or via drag and drop and <Ctrl> on the desired target. If the measurement cursor is moved manually from its position, the locking is released.
