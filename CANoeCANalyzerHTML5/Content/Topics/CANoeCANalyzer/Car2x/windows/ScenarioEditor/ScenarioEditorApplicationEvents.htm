Application Events

 

Application Events

With the help of the dialog different events can be created.

The right-hand side of the dialog describes concrete properties of the selected application event as well as the temporal extension (validity period) of the event in the scenario time:

  • StartTime - Time in seconds when the event is activated (transmitted).
  • Duration - Duration of the event in seconds. No specification means that the event is sent permanently and periodically active from the scenario time StartTime.
  • The property groups DENM or BasicSafetyMessage describe individual components or behavioral characteristics of the application event from the perspective of the corresponding Car2x/V2x message.

The attributes (components) of the created application events are inserted into the corresponding attribute group Events of the selected station. For BSM events, only one attribute group is created; for DENM events, a separate attribute group is created for each message. The attribute groups must have a unique name and start with Events.

At runtime the attributes of the Car2x/V2x message can be read / changed in the corresponding OnPreTx() CAPL function.

BSM (CN & US)

Additional, predefined attributes can be added to the attribute group in the Timeline after the event is created. The following attributes are available:

  • VehicleEventsFlags
  • PathHistory
  • Events
  • EventPosAndTime
  • MessageCount

DENM (EU)

For DENM events the listed properties can be set. The value for the EventZone corresponds to the maximum length of the EventZone. The value can be positive (EventZone upstream from the EventPosition) or negative (EventZone downstream from the EventPosition). If the value is 0, no EventZone will be created for the event. If no value is set for the OriginatingStationId and the SequenceNumber (value remains at 0), the values are automatically calculated when the event is created.

Additional, predefined attributes can be added to the attribute group in the Timeline after the event is created. The following attributes are available:

  • Events
  • EventPosAndTime
  • PathHistory
  • ActionIdOriginatingStationId
  • ActionIdSequenceNumber
  • CauseCode
  • EventZone
  • RelevanceDistance
  • RelevanceTrafficDir
  • StationType
  • SubCauseCode
  • Termination

In addition to the values in the scenario editor, values from the stack are set at runtime if the corresponding node attribute is activated in the Car2x Network Explorer.
