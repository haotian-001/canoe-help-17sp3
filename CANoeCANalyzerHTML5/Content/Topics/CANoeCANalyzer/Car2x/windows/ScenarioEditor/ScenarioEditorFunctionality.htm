Functionality

 

Use in CANoe

Scenarios created with the Scenario Editor can be loaded into CANoe via the Scenario Manager. You can open the Scenario Manager via Ribbon|Ribbon Tab Environment|Scenario Manager.

In the window of the Scenario Manager, the stations of the scenario are listed. Stations of the same name as a node of a loaded database are identified as the node. These stations are displayed with a yellow icon to the left and the preceding database name. Stations that have the same name as a GeoObject of a loaded ADAS database are displayed with a blue icon on the left side.

You can open the Scenario Editor via the button [Scenario Editor].

If a scenario was loaded via the Scenario Manager in the respective CANoe configuration, it is used when opening the scenario editor.

Scenarios and CAPL with Car2x

The structure and time history of the attribute values of the stations in the scenario can be read with the following CAPL functions. With the help of the callback functions you can react to events in each station of the scenario and display further user-defined actions in the program.

The following requirements must be met to be able to use the values of the attributes of the stations specified in the scenario file as well as the scenario callback functions in CAPL.

  • Scenario file must be loaded in the Scenario Manager
  • The Car2x Interaction Layer library (C2X_IL.dll) must be assigned to the CAPL node in the CANoe configuration.

To be able to use the automatic setting of the content/ signals of the Car2x messages CAM, DENM, BasicSafetyMessage from scenario data, these messages must be assigned to the CAPL node in the database.

Current values of the attributes of a scenario station can be read with the functions C2xGetStationAttributeDouble and C2xGetStationAttributeInt64, corresponding to the data type of the attribute. When the scenario is started, the return value of the functions corresponds to the current value in the scenario at the time of the call.

Attribute values of calling stations/ CAPL nodescan be read with the following functions:

  • float C2xGetStationAttributeDouble(char[] attributeName);
  • int64 C2xGetStationAttributeInt64(char[] attributeName);

If attribute values of other stations defined in the scenario are needed, the functions C2xGetStationAttribute must be used with the additional parameter stationName:

  • float C2xGetStationAttributeDouble(char[] stationName, char[] attributeName);
  • int64 C2xGetStationAttributeInt64(char[] stationName, char[] attributeName);

If the property Trigger of the attribute is activated (set to True), the reaching of each timeline keypoint of the attribute is reported by the callback function OnStationAttributeTrigger.

  • void OnStationAttributeTrigger(char attributeName[]);

If the attribute with the set Trigger belongs to another station in the scenario, this is reported additionaly by the Callback function OnStationAttributeTriggerAll:

  • void OnStationAttributeTriggerAll(char stationName[], char attributeName[]);

The scenario can be started or stopped via the user interface of the Scenario Manager as well as via the CAPL functions C2xStartScenario() and C2xStopScenario(). If the scenario is stopped, the attribute values of the scenario stations keep the value at stop time.

Use the function C2xIsScenarioStarted() to find out if the scenario is currently running.

The callback function OnStartScenario() is called, if the scenario was started via the Scenario Manager user interface or by calling C2xStartScenario().
