Raw Frame

 

In the Raw Frame window all data bytes of the complete packet are shown and interpreted as hexadecimal and ASCII value.

To change the data bytes click on the byte position you want to change, then insert the hexadecimal value you want to insert. It is also possible to insert the value as ASCII character. Therefore click in the ASCII column and enter your values.

Changed values are highlighted in color. To commit the changed value press <Enter> or click in another window. To discard your input press <Esc>.

Note

Consider that manually inserted data are overwritten if you have selected <Actual channel> or <Adapter Ath<x>> for the MAC addresses in the configuration dialog.

It can be selected how many bytes are displayed within one line (use list Bytes per line). It is also possible to change the length of the packet (use edit field Packet length).

You can use the comfortable copy/paste mechanism too. Therefore copy a stream of numbers in the clipboard, set the cursor to the position you want to insert it and paste it with the keys <CTRL>+<V>. That allows to insert raw data of a packet from Trace Window or other applications in the WLAN Packet Builder directly.

Note

If the length of the data packet exceeds the maximum length of 1514 bytes, the data that exceed the allowed length are highlighted and automatically truncated after a commit with <Enter>.

If you want to avoid that copied data are automatically modified, you should deactivate the automatic calculation of the checksum and length fields before the copy process.

Note

If the cursor position is in the hex column, only hexadecimal numbers can be added. If there are other characters within the stream, the complete data will be discarded.
