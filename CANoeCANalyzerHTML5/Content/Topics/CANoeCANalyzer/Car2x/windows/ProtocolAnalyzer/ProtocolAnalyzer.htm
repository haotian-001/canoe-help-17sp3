Car2x Protocol Analyzer

Ribbon|Ribbon Tab Analysis |Group Car2x |Protocol Analyzer

 

The Car2x Protocol Analyzer checks data and timing of all Car2x frames against an extensible set of rules. The check requires no action on the part of the user and is performed automatically when receiving, sending or playing (replay) Car2x frames, unless the Car2x Protocol Analyzer has been disabled manually.

The detected rule violations are displayed in the Car2x Protocol Analyzer window. The textual description of the rule violation itself (Finding) is presented with the frame's contextual data such as time, direction, ITS station affiliation, etc., as well as a required state description.

The CAPL Protocol Analyzer API of the Car2x Interaction Layer is also immediately informed about the rule violation via corresponding callback functions. Furthermore, it is possible to export the list of rule violations for further processing.

Configuration

More advanced settings are available in the Car2x Protocol Analyzer Settings dialog (accessible via the first button in the toolbar).

Using the Car2x Protocol Analyzer

The Car2x Protocol Analyzer can be activated or deactivated separately for each CANoe or CANalyzer configuration file via the Ribbon|Analysis tab|Car2x|Protocol Analyzer group. If the Car2x Protocol Analyzer is active, all Car2x frames sent and received during the measurement are analyzed, including frames from replay files and frames that were played in offline replay mode.

All detected rule violations are displayed in the list in the right part of the Car2x Protocol Analyzer window. The description of the rule violation is in the Finding column and the description of the expected state/behavior is in the Rule column.
