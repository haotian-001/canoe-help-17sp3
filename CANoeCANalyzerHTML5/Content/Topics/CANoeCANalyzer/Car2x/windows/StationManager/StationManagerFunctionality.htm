Car2x Station Manager » Functionality

Functionality

 

In the secure Car2x communication, various road users and the infrastructure are distinguished through certificates (identification via HashedId8) or the MAC address. A road user or an ITS station carries out certificate changes at regular intervals. This changes the MAC address and the HashedId8 in the packets. After the certificate change, this can be regarded from that time on as a new communicating ITS station starting to send.

But since data such as speed and position from a certain ITS station (a node defined in the database) is to be monitored in CANoe (e.g. in the Data Window or Graphics Window), the defined signals of this ITS station would no longer be recognized as data of the assigned database node and would thus no longer be updated for it is considered as another station.

With the Car2x Station Manager, you may configure the assignment of certificates and MAC addresses to ITS stations or database nodes. You may determine which certificates (HashedId8) and MAC addresses belong to an ITS station and to which database node this station is to be assigned to. This ensures that the data of a station with a changed pseudonym is still assigned to the signals of a node configured in a measurement window. This way the measurement can continue without any extensive changes.

Start of the Measurement

When first starting the measurement, packets will be received that are not yet assigned to a database node. Therefore you must execute the assignment in the Car2x Station Manager.

Changes of the Pseudonym During Runtime

If the ITS station changes its certificate (HashedId8/ MAC address) and the certificate is not yet known to the Car2x Station Manager, the newly created ITS station must be assigned to an already existing one to assign the certificate to the desired station. If a packet signed with this HashedId8 is received in the future, the packet will automatically be assigned to this station.

Automatic Assignment

The Car2x Station Manager also offers the possibility to execute automatic assignments of received packets based on the movements of the vehicle. This way, the manual assignment during runtime is no longer necessary. This configuration is especially suitable if only a small number of vehicles is present since the error-proneness of the automatic mapping is relatively low.

Note

The configuration that you may set has an impact on the Map Window, the Trace Window as well as on the Graphics Window, the Data Window and the State Tracker.

Map Window

In the Map Window, the ITS stations are shown with their assigned database node in the detailed view. However, these assignments may not be changed there since they may only be changed in the Car2x Station Manager. You may open the Car2x Station Manager via a button in the Toolbar or via a double click on an ITS station.

Trace Window

In the Trace Window, the connection of the ITS stations is perceptible from the fact that a node is displayed as a station in a column. Furthermore, you may activate the highlighting so that the packets of an ITS station are displayed in the same color as defined in the Car2x Station Manager.

Data Window, Graphics Window and State Tracker

In the Graphics Window and in the Data Window as well as in the State Tracker, the signals of the database nodes that are to be monitored are listed. The Car2x Station Manager ensures that the database nodes are linked to the ITS stations and that this way the signals of the right ITS station are displayed.
