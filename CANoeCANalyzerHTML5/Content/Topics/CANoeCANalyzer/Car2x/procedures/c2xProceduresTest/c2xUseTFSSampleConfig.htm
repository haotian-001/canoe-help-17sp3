Use Test Service Library Sample Configuration

 

The Test Service Library sample configuration demonstrates the use of the Car2x Test Service Library.

The functions of the Car2x Test Service Library are used in the configuration in a Test Unit. This contains a simple ECU test that tests the sending of a DENM message for EEBL (Emergency Electronic Brake Light) and a test that demonstrates the use of the Test Service Library in the scenario.

For the ECU test, the following is used:

  • A CAN network with a CAN database containing a message for triggering an EEBL (Electronic Emergency Brake Light).
  • A WLAN network containing an EU_ApplMsg database are used.

Within the test unit there are several test cases for the ECU test, that are all similar in structure. First a CAN message is sent. Depending on the transmitted values, a DENM message with the cause code for EEBL is then sent via a CAPL node simulating an ECU. In the individual test cases, the cause code of the EEBL is checked via different functions of the Car2x Test Service Library. Also, a test case list is used which executes a test case with multiple parameters.

For the test in which a scenario is used, first the scenario is started via the Scenario API. Then two test cases are executed. In the first test case, it is checked whether the station DUT sends both a CAM message and a DENM message with correct CauseCode. Here the C2xTestJoinMessage function of the Car2x Test Service Library is used. In the second test case the positions of all stations are checked. The C2xTestJoinSignalInRange function is used for this.
