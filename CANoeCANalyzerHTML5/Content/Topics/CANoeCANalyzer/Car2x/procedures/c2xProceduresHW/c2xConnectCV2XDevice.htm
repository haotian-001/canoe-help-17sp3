Connect a C-V2X Device

 

Activate the C-V2X Physical Layer

Communication via C-V2X (3GPP Rel. 14 Mode 4 (PC5)) uses its own frame format, which differs from IEEE 802.11p (DSRC). Therefore a switchover of the physical layer is necessary. Switching is done in the CANoe Options dialog under Bus Systems/Protocols on the Car2x page in the Physical Layer area via the Technologie list. The upper protocol layers (GeoNetworking, BTP, DSMP, ASN.1 Application Layer) are reused and are still described and parameterized in the database.

C-V2X Hardware Configuration

If the measurement in CANoe is performed on the real bus instead of a simulated bus, a connected C-V2X device is required. The integration and assignment of C-V2X devices to application channels in CANoe (Ath1, Ath2, …) is done via the V2X Hardware Configuration dialog. If the communication interface of a C-V2X device is assigned to an application channel (e.g. Ath1), you can make other settings, such as radio channel, in the Network Hardware Configuration dialog for the selected application channel.

GNSS

In addition to the C-V2X interface, the C-V2X devices provide the GNSS position data and the GNSS time source, which are available via the system variable GNSS::Ath<N>, where <N> is the number of the application channel.

C-V2X RMTPCD (Remote Packet Capture Daemon)

To enable CANoe option .Car2x to communicate with a C-V2X device via a LAN interface of the CANoe computer, you must install software (RMTPCD - Remote Packet Capture Daemon) on the C-V2X device.

The following steps are necessary for the installation:

Step by Step Procedure

  1. Download the installation package for your specific C-V2X device from https://vector-worldwide.com/vi_car2x_interfaces_en.html.
  2. Follow the instructions in the ReadMe.txt of the installation package.
  3. Make sure that the computer running CANoe.Car2x and the C-V2X device can communicate via LAN/Ethernet.
  4. Open the V2X Hardware Configuration dialog and add the C-V2X-device.
