Step by Step Procedures

The following describes the procedure for creating a traffic scenario with the Scenario Editor, load it into CANoe, and run it at the start of measurement.

It is also explained how CAPL can be used to react to events defined via the Scenario Editor.

The procedure is described using the following traffic scenario:

Two vehicles drive in a row. After a certain period of time, the first vehicle (Sender_HardBraking) brakes fully, to which the following vehicle (DuT) should respond.

Car2x Configuration in CANoe

Step

Description

Screenshot/ Example

1.

Create Configuration

Create a new configuration or load an existing one. Save it in the file system.

You can create a new configuration via File|New|Car2x.

—

2.

Add Car2x Database

In the Simulation Setup, add a Car2x database.

3.

Create Nodes

The database should contain nodes that correspond to the stations in the Scenario Editor.

In this example, two nodes are created in the database:

  1. DuT: The device under test
  2. Sender_HardBraking: A vehicle in front performing an emergency stop

Both nodes are created via the Car2x Network Explorer.

For this example, a US database (SAEJ2735_MsgSet) is used, the two nodes send a message of type BasicSafetyMessage cyclically every 100ms.

The sending behavior of the node can be further influenced in the database. In addition, the stack can be activated in the database for the node, this fills additional tokens for many application messages. More detailed information about which tokens are filled can be found in the respective help page: EU stack, US stack and CN stack.

4.

Synchronize Nodes

To display the nodes also in the Simulation Setup, you can use the Node Synchronization. For this, select the Node Synchronization via the context menu of the added data basis.

 

 

 

The two nodes are now inserted into the Simulation Setup.

Car2x Scenario

Step

Description

Screenshot/ Example

1.

Open the Scenario Editor

Open the Scenario Editor via the Toolbar in CANoe.

2.

Create a Scenario

Create a scenario by creating a route and stations (vehicles).

In the screenshot on the right, a route with 2 stations was created in the tool.

The station DuT is the device under test, the station Sender_HardBraking represents a vehicle that applies emergency braking at a certain point in time.

Note: The names of the stations must match the names of the nodes in the database.

3.

Create station attributes and events

For the stations, attributes for the speeds of the vehicles are created in the time line of the tool. For the station Sender_HardBraking, event attributes are created.

4.

Save the scenario

After creating the scenario, save it to the file system. It is recommended to save the file relative to the CANoe configuration or in folders below it.

—

5.

Load scenario

The created scenario is added to the configuration via the Scenario Manager in CANoe. You can open the Scenario Manager via the toolbar in CANoe under the tab Environment.

6.

Check mapping

In this window, click the folder icon in the toolbar and then select the saved scenario file.

If the names of the stations of the scenario correspond to those of the nodes in the database, they are displayed in the Scenario Manager with an orange icon.

7.

Start Measurement

Start the measurement to run the scenario in the Map Window.

8.

Access Attributes and Events

To access the application events of a specific station from a node, add a CAPL program e.g. to the DuT node in the Simulation Setup.

Here you can use a callback function to determine whether a station has triggered a certain application event like the emergency stop in our example.

void OnStationAttributeTriggerAll(char stationName[], char attributeName[])
{
  int64 flags = 0;
  if (strncmp(stationName,"SAEJ2735_    MsgSet::Sender_HardBraking",
    strlen(stationName)) == 0 &&
    strncmp(attributeName,"Events", strlen(attributeName)) == 0)
  {
  // Insert here the CAPL code to be executed when
  // the station Sender_HardBraking has sent an application event.
  // E.g.: VehicleEventsFlags of the event for the station
  flags = C2xGetStationAttributeInt64(stationName, "VehicleEventsFlags");
  }
}
