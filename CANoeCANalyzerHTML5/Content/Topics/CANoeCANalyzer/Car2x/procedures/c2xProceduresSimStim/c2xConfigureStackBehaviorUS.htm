Configure Stack Behavior US

 

C-V2X Stack

C-V2X Stack means that the setting of certain fields in the frame as well as cycle time calculations are automatically taken over by CANoe. The C-V2X stack is currently only implemented for the BasicSafetyMessage application message according to standard SAE J2735 2020-07.

Configuration of the C-V2X Stack in the Car2x Network Explorer

How to configure the C-V2X stack in Car2x Network Explorer is described in Configuring the sending behavior.

C-V2X Stack and Scenario Editor

When the C-V2X stack is activated, by default fields are set in the frame. By using scenarios, the frames can be further completed with dynamic fields such as Latitude, Longitude, Speed and Heading. If attributes are created and set in the Scenario Editor that were previously filled by the C-V2X stack, the data from the Scenario Editor is applied. However, if the database attribute Use GNSS Syvars is activated for a node, real position data and path history from connected C-V2X hardware are used.

C-V2X Stack and CAPL

When the C-V2X stack is activated, by default fields are set in the frame. By using CAPL, the frames can be further completed with dynamic fields such as Latitude, Longitude, Speed and Heading. This can be done e.g. in an OnPreTx CAPL callback. If fields are set in CAPL that were previously filled by the C-V2X stack, the data from CAPL is applied.

Path History

The Path History fields are automatically calculated and BasicSafetyMessage added in the following cases:

  • If the database attribute Stack Values and Use GNSS Sysvar are active for a concrete node and valid position data is present in GNSS.Ath<N> system variables.
  • Alternatively from scenario route data if a scenario exists for this node and Use GNSS Sysvar database attribute is not active.
  • Alternatively, if a CAPL function C2xAddGeoPos is called in the CAPL code with valid position data.

Cycle time

The cycle time of the BasicSafetyMessage is constant 100ms, if database attribute Stack Timing is active.
