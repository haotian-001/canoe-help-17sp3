Create V2x Stations with CAPL

 

V2x Scenario API

Via the V2x Scenario CAPL API it is possible to add stations dynamically to a scenario during a measurement. This is especially helpful if vehicles (stations) are to be used in the scenario whose position data is generated by another data source such as an external simulation program in a HIL. Data such as position, heading, speed are then constantly updated at the generated station. Based on this data, the Car2x stack builds valid Car2x messages that can be used for stimulation.

If no scenario is loaded in the Scenario Manager window or via CAPL when the station is created, a temporary scenario is created when a station is added. The station must be created before scenario start.

Adding is done via the API function C2xCreateStation. Just like stations that are statically added to a scenario in the Scenario Editor, the dynamically added station must have an associated node in the Car2x database and in the simulation setup. The name of the station in the call to the C2xCreateStation function must correspond to the name of the database node.

The procedure for creating and using scenarios is described on this help page.

Set Base Data for Dynamic Stations

Since dynamically added stations do not receive their data such as position, orientation and speed via routes defined in the scenario editor, these values must be set programmatically. The CAPL function C2xSetStationBaseAttributes can be used for this purpose (see example).

Set Attributes for Stations

In the scenario editor it is possible to define keypoints for attributes of a station at a certain time and to set values for them. See help for the Scenario Editor.

For dynamic stations such attributes can be created programmatically and values can be assigned for them. The C2xSetStationAttributeInt64 or C2xSetStationAttributeDouble functions can be used for this purpose.
