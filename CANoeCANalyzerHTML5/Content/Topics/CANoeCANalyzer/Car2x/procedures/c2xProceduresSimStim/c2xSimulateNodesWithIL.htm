Simulate Car2x Nodes

 

To simulate Car2x nodes, the Car2x Interaction Layer offers the following possibilities.

Note

Without the configuration of a network node of the database, you can only use the functions of the Car2x Interaction Layer if you included the Car2x Interaction Layer.

Database and Car2x IL API

For basic cyclic transmission of Car2x application messages, the Car2x Interaction Layer offers the possibility to configure both the protocol layers (including the Security Layer) and the transmission behavior in the database. In addition, the stack can be activated in the database for the node, this fills additional tokens for many application messages. More detailed information about which tokens are filled can be found in the respective help page: EU stack, US stack and CN stack. By means of CAPL functions (Car2x IL API) you can influence the cycle time during the measurement, specifically activate and deactivate the transmission and change the values of individual elements of the application message. For this purpose, a callback function enables the update of the content of the application message immediately before the transmission.

Generated CAPL Include Files (Car2x Network Explorer)

For the comfortable access to individual elements of the application messages to be sent and received, you can generate a CAPL include file for each application message providing specific data structures and methods by means of the Car2x Network Explorer. For access to the individual elements, the Car2x Network Explorer also generates the necessary CAPL code lines.

Packet API

For access to the fields of the individual protocol layers, the Packet API offers extensive functions. Besides the access to the protocol layers of the application message (that are transmitted by means of the Car2x Interaction Layer), you can also combine and send packets individually. Furthermore, a Callback function enables the access to received packets. Packet API functions are also suited to access elements of the application message (without the generated CAPL Include Files).

Security API

For the specific requirements regarding the Security Layer, the Security API provides you with a variety of functions.

Time API

When dealing with time stamps in different formats, you are supported by the conversion functions of the Time API.

Map Window API

For the visual analysis of signals and other parameter, the Map Window API provides you with functions to visualize them in any way in the Map Window. See also: Car2x CAPL Functions.

Station API

Special Functions to read scenario based stations.

Scenario API

Special Functions to react to changes in the status of scenarios or to change the status manually.
