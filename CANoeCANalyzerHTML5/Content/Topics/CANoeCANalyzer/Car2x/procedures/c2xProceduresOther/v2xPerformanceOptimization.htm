Optimize Performance for V2X Measurements

 

General Recommendations

  • 64 Bit: Option Car2x supports the 64-bit variant throughout. This enables the addressing of more main memory. In particular when used together with other bus systems, please observe the notes on 64-Bit/32-Bit. The 64-bit variant can be activated in the Options dialog under Measurement/General section Execution Environment.

  • When using functions of the Car2x Interaction Layer in network nodes, take care that when you create different packets with C2xInitPacket, you also release them again after use with C2xReleasePacket.

  • You can also take general performance optimization measures in the Options dialog under Measurement/Performance.

Additional Recommendations for Specific Use Cases

(Longterm) Logging

  • Remove the Map Window from the Measurement Setup.

  • Disable the Protocol Analyzer, especially the checks for the Security Layer.

  • Disconnect unneeded analysis windows in the Measurement Setup.

  • Remove large signal collections within active analysis windows.

  • Chronological display mode in the Trace Window.

  • Deactivate unneeded network nodes in the Simulation Setup.
    Note: If necessary, it may be useful to leave an analysis network node active in order to fill System Variables and record them as well.

Analysis

When analyzing large logging files, it can be helpful to split or filter them beforehand. The following options are available for this purpose:

  • In the offline source list, the configuration dialog can be used to restrict the time range for offline replay.

  • Using appropriate filtering, shorter sequences of packets can be exported from the Trace Window by selection.

  • Larger logging files can be filtered using offline mode, an IP Event Filter in the Measurement Setup, and a logging block.
    Note: Always use offline mode to filter logging files, since the reference to the original absolute time is lost when using a replay block in online mode.

Simulation of Multiple ITS Stations (more than 10)

Use signed packets only for ITS stations whose packets are also evaluated. Simulate the ITS stations used only for increasing channel load without signature.
