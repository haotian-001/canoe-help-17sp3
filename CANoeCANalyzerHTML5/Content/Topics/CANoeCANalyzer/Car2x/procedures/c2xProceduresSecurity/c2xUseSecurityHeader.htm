Use Security Header

 

CANoe can automatically decode certificates transmitted in the Security Header.

Cross Reference

  • Receive and Analyze Signed Messages
  • Send Signed Messages
  • Use Certificates

In order to evaluate the validity of the signed messages or in order to work with signed messages in CAPL, you must execute the following preparations.

  1. Open a configuration with at least one WLAN (Ath) channel.
  2. Check the used versions of the standardsETSI TS 103 097 and IEEE 1609.2, and adapt them if necessary.

  3. Enter the used certificates in the Car2x Certificate Manager. You can add existing certificates or create new test certificates.

    • For the analysis, at least the root certificate must be present and marked as trustworthy. In addition, all certificates whose signatures are to be evaluated from measurement start must be present. To more easily identify pseudonym certificates in particular, a name can be assigned to each certificate that is then displayed in the analysis windows. Signatures of certificates not present can only be evaluated when the missing certificates are received and can be assigned to a root certificate.
    • If signed messages are to be sent from CAPL, you must also enter the private key for the pseudonym certificates in the Car2x Certificate Manager. Observe the information on protection of private keys on the help page for the Car2x Certificate Manager. In addition, you should assign a name that can be used to reference the certificate and the key.
  4. The Car2x Certificate Explorer is used to explore all certificates known in CANoe, that is, the certificates configured in the Car2x Certificate Manager and the certificates received during the measurement. It shows the trustworthiness, the certificate hierarchy, and the content of the certificates.

  5. In order to use the Car2x Security function in CAPL, you must assign the Car2x Interaction Layer to the CAPL node. You can then send and receive signed messages.

    Cross Reference

    The Car2x Security Demo (ETSI TS 103 097) and the V2x Security Demo (IEEE 1609.2) contain detailed examples for the use of the CAPL functions.
  6. In the Trace Window, you can analyze the Security Header. A summary of the most important fields as well as a complete tree view of the protocol layer is located in the detail view below the GEO-SH – Secured Message node (ETSI TS 103 097) respectively the WAVE Security Services node (IEEE 1609.2).
