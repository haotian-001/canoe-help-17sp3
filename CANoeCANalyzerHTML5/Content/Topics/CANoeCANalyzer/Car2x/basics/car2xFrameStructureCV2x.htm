Structure of C-V2x Frames

 

Packet length

The packet length of a C-V2X packet comprises the number of data in bytes that are output in the Trace Window in the detail view, starting with the C-V2X Data Link Layer up to the end of the ASN.1 payload data. The C-V2X Data Link Layer is also counted.

C-V2X Data Link Layer

To provide simplified access to the selected managed data in MAC, RLC (Radio Link Control) and PDCP (Packet Data Convergence Protocol) headers of the C-V2X packet, a C-V2X Data Link Layer is generated and inserted into the Car2x packet.

Country specific payload

The value of the message family byte contained in the C-V2X layer determines according to which of the three possible C-V2X standards (Europe, US, China) the subsequent headers and payload data are structured. The structure of the headers is described in the protocol overview.
