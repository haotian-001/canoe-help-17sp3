Timestamp Analysis

 

In CANoe Car2x and the protocols used, a variety of different timestamps in different formats are used. The following two tables are intended to provide an overview.

Timestamp of the EU/US/CN protocols

Timestamp of the EU/US/CN protocols

Format

Intended use / description

EU

GEO-EH Timestamp

32 Bit uint Timestamp = TAI milliseconds since 01.01.2004 00:00:00.000 UTC Modulo 32

Time at which the generation of the geographic position takes place. No complete absolute time stamp due to modulo operation. In the option Car2x, the display in the Trace Detail View is calculated back to the full time stamp based on the measurement start time. Is automatically set in the option Car2x in the CompletePacket.

Security Header - Generation Timestamp

64 Bit uint Timestamp= TAI microseconds since 01.01.2004 00:00:00.000 UTC

Time at which the message using the certificate was generated. Is automatically set in the option Car2x in the CompletePacket.

Security Header - Start Timestamp

32 Bit uint Timestamp = TAI seconds since 01.01.2004 00:00:00.000 UTC

Time from which the certificate is valid. Is created when generating of the certificate. The duration, which is also included in the certificate, can be used to determine the validity period of the certificate.

CAM Generation Delta Timestamp

16 Bit uint Timestamp = TAI milliseconds since 01.01.2004 00:00:00.000 UTC Modulo 65 536

Time at which the CAM was generated. Is calculated automatically when the EU stack is switched on. No complete absolute time stamp due to the modulo operation.

DENM Detection Time

64 Bit uint Timestamp = TAI milliseconds since01.01.2004 00:00:00.000 UTC

Time at which the DENM (or the event) is updated. Is calculated automatically when EU Stack is enabled.

DENM Reference Time

64 Bit uint Timestamp = TAI milliseconds since 01.01.2004 00:00:00.000 UTC

The time when DENM (or the event) was discovered for the first time. Is calculated automatically when EU Stack is enabled.

Time from which the certificate is valid. Is created when generating the certificate. The duration, which is also included in the certificate, can be used to determine the validity period of the certificate.

BSM UTC Time

Sequence of Integers DDateTime ::= SEQUENCE { year DYear OPTIONAL, month DMonth OPTIONAL, day DDay OPTIONAL, hour DHour OPTIONAL, minute DMinute OPTIONAL, second DSecond OPTIONAL, offset DOffset OPTIONAL -- time zone }

Used in the Full Position Vector of the BasicSafetyMessage. Displays the full UTC timestamp.
