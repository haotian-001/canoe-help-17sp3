C-V2x Hardware

 

C-V2x Communication

C-V2x here refers to the direct communication between traffic participants or traffic infrastructure units (RSUs) in accordance with 3GPP Release 14 Mode 4 (PC5). The following conditions apply:

  • No cellular base stations are involved or necessary.
  • A valid GNSS signal (fix) must be physically available for successful communication, since PC5 mode uses accurate GNSS time as a common time base for time-slot based resource assignment.

C-V2x Hardware

It is possible to connect a third-party C-V2X hardware with CANoe. Supported is the Craton2 device from Autotalks - Autotalks Craton2 EVK CUT3.

Commissioning the C-V2x Hardware

The connection between CANoe and C-V2x hardware is made via Ethernet (UDPv4), using the computer's Ethernet network card.

In order for CANoe to communicate with C-V2x hardware, additional software (RMTPCD: Remote Packet Capture Daemon) must be installed on the device. More information can be found on this web page.

The configuration of the C-V2x hardware is done via the V2x Hardware Configuration dialog.

A detailed procedure for commissioning and configuration is described on the Connect a C-V2x Device help page.
