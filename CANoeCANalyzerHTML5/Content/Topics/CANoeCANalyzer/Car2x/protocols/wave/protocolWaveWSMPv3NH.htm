Protocol Information

The WAVE Short Message Protocol (IEEE 1609.3 - 2016) is a network protocol that contains basic information for communication between individual ITS stations.

This protocol contains the Network Header of the WAVE Short Message Protocol.

Protocol Dependencies [▲ top]

The data are interpreted as WAVE Short Message Protocol v3 if the following conditions are fulfilled

  • An Ethernet packet with Ethertype = 0x88DC is received.
  • The version field in the WSMP header has the value 3.

Optional Fields [▲ top]

The optional fields can be in any order.

In the table below, it has been assumed for the offset determination that the Transmit Power appears first, followed by the Data Rate and the Channel Number and finally the Channel Load. If the fields appear in a different order, the offsets change accordingly.

As soon as an optional field is available, the field Count WAVE Information Element Extensions is added automatically. This field gives information about how many optional fields are used.

Packet API [▲ top]

The optional fields can be added with the CAPL function C2xAddToken and removed again with C2xRemoveToken. The C2xAddToken function always creates the underlying tokens. In other words, for a C2xAddToken on the Transmit Power token, the following tokens are created automatically: Transmit Power ID, Transmit Power Length, and Transmit Power Content.

Note

During the initialization of the WAVE Short Message Protocol, the field Ethertype = 0x88DC (WAVE Short Message) is set in the Ethernet Header.
