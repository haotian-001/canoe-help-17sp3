Protocol Information

The WAVE Service Advertisement protocol (IEEE 1609.3) is a network protocol that provides information about services on other channels. After receipt of a WSA message, an ITS station is therefore always informed about which services are offered on which channel.

Protocol Dependencies [▲ top]

The data are interpreted as WAVE Service Advertisement when the following conditions are met:

  • A WLAN management frame with subtype "Action" is received (Frame Control = 0xD0xx).
  • The frame body begins with the following values:
  • Category Code = "Vendor Specific" (127)
  • Organizationally Unique Identifier (OUI) = 0x0050C2 (IEEE Registration Authority)
  • Individual Address Block (IAB) = 0x4A4 (IEEE P1609 WG)
  • Management ID = 3 (IEEE 1609.3)
  • Content Descriptor = 0x01 (WSA)

Header Format [▲ top]

Only the first start byte is fixed in the WAVE Service Advertisement protocol. All other fields are optional and can recur multiple times in some cases. For this reason, it is not possible to give an exact fixed definition of which field is located at which byte position. Byte positions are therefore not specified for optional fields and subfields of optional fields in the following table.

The following table presents in detail the available fields, the subfields they contain, and whether and how often they can be repeated.
