Protocol Information

The GeoNetworking protocol is a network layer protocol that provides packet routing in an ad hoc network. It makes use of geographical positions for packet transport. GeoNetworking supports the communication among individual »ITS stations as well as the distribution of packets in geographical area (see ETSI EN 302 636-4-1).

This protocol contains the Extended Header of the GeoNetworking protocol.

time stamp of Long Position Vector:

Timestamp = TST(TAI) elapsed TAI milliseconds since 2004-01-01 00:00:00,000 UTC mod 232

The time stamp is interpreted in the Detail view of the Trace Window as follows:

UTC+0 YYYY-MM-DD HH:MM:SS.sss

The time stamp is not unique due to the modulo operation. Its value range covers a period of 49 days, 17 hours, 2 minutes, and 47,296 seconds. Typically one is interested in times around the measurement point, so the period of about 24 days before and after the measurement point is used for the conversion to the absolute UTC time stamp. In addition CANoe considers specific leap seconds.

Packet API [▲ top]

Note

During initialization of the Extended Header, the Common Header is also initialized which then initializes the Basic Header. During initialization of the Basic Header, the Packet API sets the field Ethertype=0x8947 (GeoNetworking) in the Ethernet header and the field Header Type = 2 (GeoUnicast) in the Common Header.
