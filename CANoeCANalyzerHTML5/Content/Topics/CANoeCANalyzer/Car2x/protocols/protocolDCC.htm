Decentralized Congestion Control (DCC)

 

To avoid traffic overload over air a DCC mechanism is used as described by ETSI TS 102 687 V1.2.1 for the region EU. This approach limits the messages that can be transmitted within a given time period by enforcing a minimum time gap between consecutive messages. This minimum time gap depends on the measured busy ratio of the specific radio channel (CBR) and the so called Ton-time (how long the radio channel is blocked by this message). Therefore, if 2 messages are triggered at the same time, even with no congestion on air (CBR = 0) the second message must be delayed by at least 50msec (for small messages) or even 100msec for larger messages to fulfill the requirements. Before CANoe Version 12.0 the test engineer was responsible to care about necessary time gaps and all messages not fulfilling this rule were just skipped. Since 12.0 an internal queuing mechanism has been introduced, which delays all messages automatically and transmits those in a specific order respecting the QoS-priority (traffic class) and age. This DCC queue can be operated in 2 modes, which can be configured in the Options dialog:

  1. Overwrite mode: if a new message should be sent and a similar message is already in the DCC-queue, the old message is just replaced by the newer one. This prevents sending of old information, if newer ones are available and also reduces potential traffic. The overwrite mode is intended to provide newest information with smallest latencies.
  2. Insert mode: in this mode all messages are stored in the DCC-queue, even if similar messages do exist there, until the message queue becomes full. In this case the oldest similar message will be overwritten, all others are still kept in the queue. This mode should be used, if transmit message losses should be minimized.

If the DCC-queue size limit is reached, the oldest message with the lowest priority will be discarded, so even in the insert-mode messages may be skipped from transmission, if the send rate or the congestion is high.
