Protocol Information

To protect the data of a C-V2X package, it can be secured by the protocol layer China Security Header. The CnSh layer consists of information about the signer, information about the payload, the payload itself and finally the signature (about the information about the payload and the payload itself).

Protocol Dependencies [â² top]

In the data frame with DSMP layer, the CnSh layer is interpreted if the first byte of the DSMP user data has the value 3. Then the application message is interpreted (if present).

Trace Window [â² top]

Since the data structure of the CnSh layer is unclear, a summary of the most important transmitted values is displayed in the detail view of the trace window under Summary. In some cases values are also displayed that do not appear in the message but are known to CANoe, such as the signature status or the certificate name. The complete transmitted data structure can be found under Message Tree. The user data is displayed interpreted according to the signature.

Invalid signatures are displayed in the Trace Window with an error symbol. If the signature of a message cannot be verified, a note appears in the Protocol Info column. This note remains even if a certificate is received at a later time, which could be used to check the validity of the older message.

Signature

Status

signatureStatus

â

â

The status of the signature is displayed in the detail view of the trace window.

0x1000..0x1FFF: unknown

The signature cannot be verified because only the digest is known but not the certificate or because the validity of the certificate cannot be verified.

0x2000..0x2FFF: invalid

The signature is invalid either because the signature does not match the certificate or because the certificate is invalid.

0x3000..0x3FFF: valid

The validity of the signature is based on the fact that one of the certificates used from the certificate chain (certificate or root certificate) is marked as trustworthy in the Car2x certificate manager.

The status of the signature (value of the token) can be read with the C2xGetTokenInt function.
