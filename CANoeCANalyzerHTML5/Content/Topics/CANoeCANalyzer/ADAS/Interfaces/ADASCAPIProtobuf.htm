C-API / Protobuf

 

Short Description

The programming examples demonstrate the use of the C-API. In the ADAS project, the cclSetGpbOSI3Data2 function is used to import protobuf bytestreams into CANoe. This allows third-party simulation environments to be connected to CANoe by transferring their own protobuf bytestreams with the respective environment data to CANoe. It should be noted that the transmitted Protobuf structures must comply with the OSI standard so that they can be correctly interpreted and processed by CANoe.

Procedure

In the ADAS project of CANoeCCL.sln, an ADAS.dll is first compiled, which is later integrated into the sample configuration. The integration of the DLL is done in the Options dialog|Programming|C Libraries. This way, the behavior programmed in the DLL is started at the start of measurement.

In the sample implementation, a GPBData.data file is read first, which contains an exemplary protobuf bytestream. This encapsulates data in the SensorData as a top-level message. The type of the top-level message must also be passed as a parameter to cclSetGpbOSI3Data2 so that it can be interpreted correctly by CANoe.

The following dataTypes apply to the three supported top-level messages:

  • SensorData: 0
  • GroundTruth: 1
  • SensorViewConfiguration: 2

The selected bytestream is then sent periodically to CANoe. Here the data of the byte stream is extracted and written to the corresponding communication objects, which in turn are displayed in the analysis windows such as the Trace Window.
