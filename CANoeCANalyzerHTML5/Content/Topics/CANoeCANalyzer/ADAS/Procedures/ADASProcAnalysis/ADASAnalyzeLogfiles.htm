Analyze Logging Files

 

You can create logging files for ADAS communication objects and replay them later in offline mode to analyze the ADAS objects.

Create Logging Files

To create a logging file, activate the Logging Block in the Measurement Setup. If there is no Logging Block yet, you can insert a new Logging Block via the ribbon. Afterwards you can determine e.g. the name and the target directory of the logging file via [Logging file configuration...]. Select here BLF as file format. After the configuration you can start the measurement. In the background, the ADAS objects are recorded via the Logging Block and saved in the specified logging file.

Replay Logging Files

To do this, switch to offline mode. You can then specify the logging files to be replayed in the Measurement Setup under [Offline file list]. Add the previously recorded logging file here. You can then start the measurement and observe (e.g. in the Trace Window) that the logged ADAS objects are now being replayed.

Replay Logging Files Online

To do this, switch to online mode. You can then use the StartDOReplayFile function in a CAPL Application Model to load a logging file that contains ADAS objects. For a correct interpretation of the data, the vCDL file of the configuration used to record the logging file must be available and loaded.
