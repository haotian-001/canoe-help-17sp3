Use Test Feature Set API

 

The Test Feature Set API helps you to test your ADAS function. For this purpose, possibilities are provided to wait for events to occur. The exact procedure differs for CAPL and C# test code. The ADAS System DYNA4 sample configuration demonstrates how to use the API.

CAPL Procedure

In CAPL, there are dedicated wait functions for the distance and speed of Detected Objects. An overview of all ADAS TFS CAPL functions can be found on the help page Test Feature Set CAPL Functions.

Basically, the system first waits until a defined event occurs, e.g. a Detected Object approaches the reference point to less than 30m. If required for your test case, the function TestGetWaitADASDetectedObject can be used to access the name of the Detected Object and the function lookupDistObj can be used to return a reference to the Detected Object.

The reference can now be used to access individual fields of the Detected Object. If a Detected Object was successfully waited for, the return of your ADAS function can be checked via CANoe SAB and the test case result can be set.

Example

enum VWaitThreshold
{
  kUnderThreshold = 0,
  kOverThreshold = 1
};

distObjRef ::ADAS::IDetectedMovingObject detMovingObj;
char buffer[100];

if(TestWaitForADASDetectedObjectDistance(kUnderThreshold, 30, 1000)) //Wait for 1s for Detected Object below 30m distance
{
  if(TestGetWaitADASDetectedObject(buffer) == 0) //Get the name of the Detected Object that triggered the Wait Condition
  {
    detMovingObj = (distObjRef ADAS::IDetectedMovingObject) lookupDistObj(buffer); //Get the Detected Object with the CAPL Function lookupDistObj
    if(detMovingObj.IsValid) //Check if the Detected Object is valid
    {
     //Do something with the Detected Object, e. g. check the dimension of the object or log the Id
     // Check return of ADAS Function with CANoe SAB and set the test result
    }
  }
}

C# Procedure

In C#, waiting is done via the generic Wait method of the Execution class, in which a criterion is called. In this criterion, member methods of the sensors can be used to check whether Detected Objects are within a defined distance or have a certain speed. All methods of the Sensor API can be found on the help page ADAS CAPL Functions. If the wait is successful, the return of your ADAS function can be checked via CANoe SAB.

Example

[TestCase("Wait For Distance Under 30m", "")]
public void WaitForDetectedObjectWithinDistance()
{
  if (Execution.Wait(WaitDistCriterion, 3000) == 1)
  {
  //... Check the return values of the ADAS Function with CANoe SAB
  }
  else
  {
  Report.TestStepFail("Wait failed"); //Report the failure of the Test
  }
}
[Criterion]
[OnUpdate(radar.MemberIDs.sensor_info)]
public bool WaitDistCriterion()
{
  var detObjNames = radar.GetDetectedObjectsConditional.Call_Phys( _ADAS.DataModel.ECriterion.Distance, 30, _ADAS.DataModel.ETriggerCondition.Lesser); //Returns the names of Detected Objects with Distance less than 30m
  return detObjNames.Length > 0 ? true : false; //True if at least one DetectedObject is within 30m
}
