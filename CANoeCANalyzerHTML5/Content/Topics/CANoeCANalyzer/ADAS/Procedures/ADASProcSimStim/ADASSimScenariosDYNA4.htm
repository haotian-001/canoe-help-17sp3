Simulate Scenarios with DYNA4

 

Create a DYNA4 scenario

To create a DYNA4 scenario, you must have a DYNA4 Studio license. In DYNA4 Studio you can create any scenarios you like and then later integrate them into CANoe. On this web page you will find a Quickstart Tutorial for DYNA4.

Integrating a DYNA4 scenario in CANoe

DYNA4 Studio offers the possibility to provide the scenario in a compiled modeling library. The modeling library can be included as an application model in the Communication Setup.

In this case, only a DYNA4 run license is required for execution.

A detailed procedure can be found in the video Tutorial with CANoe as Master on this webseite.

Closed - Loop: Influencing a DYNA4 scenario from CANoe

Loading the modeling library automatically creates some DYNA4-specific system-defined system variables in CANoe, including the following:

  • DYNA4ExternalFunctionInput::Active
  • DYNA4ExternalFunctionInput::AccPedalPos
  • DYNA4ExternalFunctionInput::BrakePedalPos

If DYNA4ExternalFunctionInput::Active is set to 1, the first person vehicle of the scenario can then be accelerated (DYNA4ExternalFunctionInput::AccPedalPos) or braked (DYNA4ExternalFunctionInput::BrakePedalPos) using these two other variables.

This can be done in any CAPL /C# network node, test node or application model.

C# Example Code:

// Callback on update of the ADAS algorithm
[OnUpdate(ADASAlgorithm.MemberIDs.isWarningActive)]
public void OnUpdateADASAlgorithm()
{
  if (ADASAlgorithm.isWarningActive.ImplValue == 1)
  {
    DYNA4ExternalFunctionOutput.Active.Value = 1;

    // Send the values to the DYNA4 scenario -> Closed Loop
    DYNA4ExternalFunctionOutput.BrakePedalPos.Value = 1.0; // Full Brake
    DYNA4ExternalFunctionOutput.AccPedalPos.Value = 0.0; // No Throttle
  }
}

Using DYNAanimation

DYNAanimation is a software that models and displays scenarios created with DYNA4 in a 3D environment. DYNAanimation is not necessary to start a measurement with a DYNA4 modeling library. However, if you want to be able to visually follow exactly what is happening in the scenario, DYNAanimation offers a high level of detail (much higher than the Scene Window can, for example).

Comparison of Scene Window and DYNAanimation (same scenario):

If you want to use DYNAanimation, proceed as follows.

Step by Step Procedure

  1. Download the DYNAanimation installation package.
  2. Install DYNAanimation via the file DYNAanimation.msi.
  3. In CANoe, the AnimationProjectLauncher of DYNAanimation must be called.

on start
{
 startDA("YourPathToTheDYNA4Project");
}

//
// Load and start the Scenario, open DYNAanimation
//
void startDA(char AnimationProject[])
{
  char startDAcommand[260];
  char cfgPath[260];
  char ParameterStr[520];

   getAbsFilePath("", cfgPath, elcount(cfgPath));
  strncpy(startDAcommand,"cmd /c start /min YourPathToTheAnimationProjectLauncher\\AnimationProjectLauncher5.2.exe",elcount(startDAcommand));

  if (strlen(AnimationProject) > 0)
  {
    strncpy(ParameterStr," -projectPath ",elcount(ParameterStr));
    strncat(ParameterStr,"\"",elcount(ParameterStr));
    strncat(ParameterStr,cfgPath,elcount(ParameterStr));
    strncat(ParameterStr,AnimationProject,elcount(ParameterStr));
    strncat(ParameterStr,"\"",elcount(ParameterStr));
  }
  else
  {
    strncpy(ParameterStr,"",elcount(ParameterStr));
  }

  SysExec(startDAcommand, ParameterStr);
}

  1. When CANoe measurement starts, DYNAanimation opens in a separate window.
