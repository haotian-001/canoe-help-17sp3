Stimulating an ADAS Algorithm With the CANoe Sab

 

The SIL Kit generates a SIL Adapter based on a functional system interface description of the application under test and provides access to the system under test on a pure software level.

In simple terms, SIL Kit allows you to exchange data between CANoe and your development environment to stimulate and test your ADAS algorithm directly in the development environment.

Exemplary procedure:

I. Describe the functional system interfaces

  1. Describe the functional system interfaces of the software under test as distributed objects (DO) in a vCDL file.

  2. A SIL Adapter for data exchange with CANoe is generated for these distributed objects.
    The adapter is provided as a set of source files and libraries.

  3. Instrument the SUT source code using the SIL Adapter in the development environment.

  4. If you start the instrumented SUT in a virtual execution environment, you can read and write the values of the software under test from CANoe.

II. Making a Mapping

In an application model, the values of the incoming ADAS OSI objects must be mapped to the SilRadar object:

OnUpdate(LRRadar77GHz.MemberIDs.sensor_info)]
public void OnUpdateRadarDo()
{
  var detectedObjects = LRRadar77GHz.GetDetectedObjects.Call();

  for (int i = 0; i < detectedObjects.Length; i++)
  {
    _ADAS.DataModel.IDetectedMovingObject detObj =
    DORegistry.LookupDistributedObject<_ADAS.DataModel.IDetectedMovingObject>(detectedObjects[1], "ADAS");

    if (detObj.moving_object.header.tracking_id.value.ImplValue.Value == trackingIdOfRadarObjectOfInterest)
    {
      SilRadar.X.Value = detObj.moving_object.baseInfo.position.x.ImplValue.Value;
      SilRadar.Y.Value = detObj.moving_object.baseInfo.position.y.ImplValue.Value;
      SilRadar.Z.Value = detObj.moving_object.baseInfo.position.z.ImplValue.Value;
      SilRadar.Speed.Value = detObj.moving_object.baseInfo.velocity.x.ImplValue.Value;
      SilRadar.MeasurementStatus.Value = (int) detObj.moving_object.header.measurement_state.ImplValue.Value;
   }
  }
}

III. Create SIL Adapter

With the SIL Adapter you access the software to be tested. How you can create a SIL Adapter with the command line tool sil-adapter-builder is described on the help page of the SIL Adapter. You will also learn how to set up your CMake or Visual Studio code project to automatically generate the SIL Adapter when the vCDL files have changed.

Sample configurations can be found in the SIL Adapter Basic Demo included in CANoe.

IV. Instrumenting the SUT source code using the SIL Adapter in the development environment

To use the SIL Adapter, proceed as follows.

  1. Add the generated files in the development environment to the software project under test, e.g. via the CMake files generated in the same way.

  2. You can then replace all calls of functional system interfaces with calls to the SIL Adapter, e.g. to access hardware or other software components.

#include "IO.hpp"
#include "TypeLib/TypeLib.hpp"
namespace MyAdasAlgorithm
{
  void InitializeIO() { Vector::CANoe::TypeLib::Init(); }
  void ShutDownIO() { Vector::CANoe::TypeLib::ShutDown(); }

  int GetRadarObjectTrackingID() { return ADAS::SilRadar.TrackingID; };
  int GetRadarObjectMeasurementState() { return ADAS::SilRadar.MeasurementStatus; };

  double GetRadarObjectPosX() { return ADAS::SilRadar.X; };
  double GetRadarObjectPosY() { return ADAS::SilRadar.Y; };
  double GetRadarObjectPosZ() { return ADAS::SilRadar.Z; };
  double GetRadarObjectSpeed() { return ADAS::SilRadar.Speed; }

  void SetOutputOfAlgorithm(const ADASState& state)
  {
    ADAS::OutputOfAlgorithm.isWarningActive = state.IsWarningActive;
    ADAS::OutputOfAlgorithm.distanceToObject = state.DistanceToObject;
  }
}
