Simulate ADAS Scenarios with the Scenario Editor

 

The following describes the procedure for creating a scenario with the Scenario Editor, loading it into CANoe, and playing it back when the measurement is started.

Also explained is how to react in C# to messages sent via the Scenario Editor.

The procedure is described using the following scenario:

Example

Two vehicles drive behind each other. The first vehicle (Vehicle_2) makes an emergency stop after a certain time, to which the following vehicle (DuT) should react. The DuT detects the vehicles ahead by a sensor, which is attached to the front part of the vehicle. The detected vehicles are managed and sent as DetectedObjects in a list at the sensor.

1.

Create Configuration

First create a new ADAS configuration or load an existing one. Save this in the file system.

You can create a new Car2x configuration via File|Sample configuration|ADAS.

—

2.

Add ADAS Database

If not already present, add the ADAS database in the Communication Setup.

—

3.

Create Sensors

In the Communication Setup, open the vCDL Editor and add a new sensor in the data source.

Give your sensor a unique name. Select ISensor as the base interface and ADAS as the namespace.

Finally, a unique SensorId must be assigned. The SensorId must match the SensorId assigned in the Scenario Editor.

4.

Create Vehicles

Create two new vehicle objects via the vCDL Editor, select for these the IGeoObject interface.

Give your vehicles a unique name. The name must match the name of the station in the Scenario Editor.

Create Scenario

Step

1.

Open the Scenario Editor

Open the Scenario Editor via the Toolbar in CANoe.

2.

Create a Scenario

Create a scenario by creating a route, stations (vehicles) and a sensor on the DuT.

Note: Sensors cannot be created via the toolbar. To do this, you must right-click on the station to which the sensor is to be added and select Add Sensor from the context menu.

The station DuT is the device to be tested, the station Vehicle_2 represents a vehicle performing an emergency braking at a certain point in time. The sensor cone of the sensor DuT_Sensor1 represents the perceptual area of the vehicle. Only stations that are within the sensor cone are detected and sent as a message at the sensor.

Note: The names of the stations must match the names of the GeoObjects in the database.

3.

Create station attributes

For the stations, attributes for the speeds of the vehicles are created in the timeline of the scenario editor.

4.

Save the scenario

After creating the scenario, save it in the file system. It is recommended to save the file relative to the CANoe configuration or in folders below it.

—

5.

Load scenario

The created scenario is added to the configuration via the Scenario Manager in CANoe. You can open the Scenario Manager via the toolbar in CANoe under the tab Environment.

6.

Check mapping

In this window, click the folder icon in the toolbar and then select the saved scenario file.

If the names of the stations of the scenario correspond to those of the nodes in the database, they are displayed in the Scenario Manager with an blue icon.

Sensors are not displayed in the Scenario Manager.

7.

Start Measurement

Start the measurement to run the scenario and to display Detected Objects in the Scene Window.

8.

Access Attributes and Events

To access the detected objects of a vehicle/sensor, add a C# script to the system and communication structure in the Application models section.

In this, you can use a callback function to determine whether the sensor detects new objects and if the detected objects can be returned via the Sensor API.

In addition, you can manage start & stop of the scenario code-based via the Scenario Manager API.
