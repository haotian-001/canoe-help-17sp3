Create a Configuration

 

Create configuration and analyze messages

To create an ADAS configuration and then analyze it, perform the following steps.

Step by Step Procedure

  1. Start CANoe.
  2. Create a new configuration using the template under File|Sample Configurations|ADAS.
  3. Customize and create sensors.
    Within the vCDL, a sensor is already preconfigured with SensorId 1. You can edit this in the System and Communication Setup using the vCDL Editor. See also the section Working with the vCDL Editor further down this page.
  4. Create a scenario, either via the Scenario Editor, DYNA4 or via third-party simulation environments.
  5. Optional: Add a new application model in the Communication Setup. Here you can react to changes to the sensor via callbacks and evaluate its object list programmatically.
  6. Start the measurement with <F9> and observe the trace window.

Note

In the ADAS Feature Set the underlying data structures of the communication objects correspond to the ASAM OSI3 standard. Object lists are transferred according to the SensorData scheme, therefore all ADAS configurations require a sensor with set and matching SensorId.

Working with the vCDL Editor

Step by Step Procedure

  1. In the Communication Setup click on New Data Source|ADAS.

  2. In the following Save as Dialog, assign a name and path for your vCDL template; in this example ADAS_Template.vCDL. Click on Save. The vCDL is created and pre-filled with ADAS-relevant data.

  3. Open the created data source ADAS_Template.vCDL via a double click or the Edit button.

  4. The vCDL Editor opens. As an example and commented out, you can already see here how you can create a sensor. Comment in the two lines:

    • [_ADAS::DataModel::SensorId=1]

    • object SampleSensor : _ADAS::DataModel::ISensor;

    Assign a unique SensorId as well as a unique name for the sensor that matches your project. You can repeat this for as many sensors as you like.

    Save your changes and close the vCDL Editor.

  5. A pop-up notifies you that the data source has been changed. Reload the data source using the Load now button.

    In the subsequent Load Data Source dialog, click the Finish button.

  6. In the application layer objects, you will now see your sensor and the associated methods under ADAS. You can now close the Communication Setup and work with the Sensor object.
