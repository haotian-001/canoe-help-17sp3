Create and configure message sequences

 

The tree view on the left side of the dialog contains both message templates and elements for flow control.

Message Templates

Via drag and drop the message templates can be inserted either into the list of messages to be transmitted once (Sequence) or into the list of cyclical messages (Cyclic transmission). A double-click on a message template inserts the message into the Sequence list box. If the <Ctrl> key is pressed at the same time, the message is inserted into the cyclical transmission list.

Messages can also be inserted into the list of messages to be sent once or the list of messages to be sent cyclically by selecting the corresponding command from the local context menu of a message template (right-click on a message template).

Configuration of Messages

After the message template is inserted into one of the list fields, a dialog for configuration of the message opens automatically. The settings on the configuration dialog vary depending on the type of message template.

The following message templates are available:

Template

Service

Description

SDO

SDO protocol

Executes a SDO download. Besides the specification if the transmission type (normal or block transfer) the transmitted data can be specified too. If block transfer is not selected the data length (0<data length<5 byte) is used to automatically switch from expedited transfer to segmented transfer.

PDO

PDO message

Sends the selected PDO. The data to be sent for the PDO can be specified.

HBEAT

Heartbeat message

Sends a heartbeat message.

GUARD

Guarding message

Sends a guarding response. A guarding request can be executed with the RTR message template.

NMT

Network management

Sends a NMT command. In addition to the node ID, the NMT command must be specified.

SYNC

SYNC message

Sends a SYNC message.

EMCY

Emergency message

Sends an emergency message. In addition to the error code and the error register, additional object-specific settings can be made.

RTR

Remote Frame

Sends a remote message. This can be configured freely.

Identifier Configuration

In addition to the configuration of the parameters specified above, the identifier of the message is also specified on the configuration dialogs.

If the CANoe configuration is assigned a database, the list of message templates contains node-specific messages for the message templates SDO, PDO, HBEAT, GUARD, NMT and EMCY. For example, the message template SDO is expanded to include the default SDOs of the nodes contained in the database.

For the configuration of these concrete message templates, the identifier are already configured but can be modified. In addition, additional information such as a list of all objects present on the nodes is available for the configuration of concrete messages.

The configuration dialog for SDO transfer provides different options to define the message IDs to be used.

If the check box Adjust automatically is selected both identifier are calculated dependent on the node ID. In this mode it is not possible to enter the identifier manually, only the identifier of the list are selectable.

The lists for the identifier are activated alternating. That means if an ID is selected in ID (Tx) the appropriate ID is set in ID (Rx) and vice versa.

The calculation is done referring to the Predefined Connection Set defined in /2/. If an applicable database is assigned to the configuration, the Node ID list shows a list of all nodes that are defined in the database. It is also possible to enter a node ID that is not in the list. In selecting a node, the objects 0x1200..0x127F are read from the DCF files and all valid SDO Server Identifier pairs are written to the appropriate ID lists. If the default SDO Server (object 0x1200) is described in the DCF file then its identifier is added to the lists too. Identifier that differ from the Predefined Connection Set are handled in the same way. If the IDs are not described in the DCF or are described incomplete, the default identifier dependent on the node ID of the node are added to the list with the note (Default).

If the check box Adjust automatically is not selected, it is possible to select one of the predefined identifier in the lists and as well as enter any valid identifier. For extended format identifier the following format must be used:e.g. 1234x.

Data Configuration

Data can be configured in the Data definition dialog. Open this dialog with [...] next to the data input field. Several formats (see table) are available to configure the data. The check box Adjust automatically refers to the length of the configured data.

  • Adjust automatically selected

The data length is determined dependent on the used message template. If a SDO message template is used, the length is determined on the object description or the entered data. For a PDO message template the length is calculated on using the mapping table.

An object description is available if the assigned database of the selected node references a DCF file and if a description for the object (index/subindex) exists that is configured in the message template. Otherwise the data length is determined on the data length of the selected data format and the entered data.

On format Byte array the data must be entered in hexadecimal format. If the length is an odd number of half bytes the length is rounded to full number of byte. In this case data are filled with an leading null in the missing half byte.

On format String value the data length is the number of entered characters On format File contents the data length is the length of the specified file.

  • Adjust automatically not selected

The data length is not automatically calculated, is must be entered by the user.

Note

The display and specification of numeric values (hexadecimal or decimal) on the configuration dialogs and in the CANopen generator occurs depending on the setting in CANoe. Values that make no sense for a hexadecimal display are marked with a leading (dec.). Here only decimal values can be specified. Values that can be specified hexadecimal only are displayed with the prefix (hex.).

Configuration of Cyclical Messages

If the message template is inserted into the Cyclic transmission list box, a cycle time and a delay time must be specified on the configuration dialog.

The delay time specifies with which delay the sending of the cyclical message begins after the start of the transmission. If no delay time is specified, the start time is automatically delayed by a send cycle.

Flow Control

In addition to the messages, it is also possible to insert control elements for flow control into the message sequence:

Control Element

Description

PAUSE

Interrupts the message sequence for a specified time.

CYCLETX

Starts or stops the transmission of the cyclical messages.

Note

Control elements cannot be inserted into the list of cyclical messages.
