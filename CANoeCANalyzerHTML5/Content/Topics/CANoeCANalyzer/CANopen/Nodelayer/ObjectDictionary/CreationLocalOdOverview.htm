Generating the Local Object Dictionary (deprecated)

 

The functions in this section enable the user to create an object dictionary. For a CANopen ECU, all parameters and data are present in an object dictionary. Each individual entry is addressed uniquely via an index (16 bit) and a subindex (8 bit). The significance of the individual object dictionary entries is already predefined via the communication profile (see /2/) and the device profile. Via PDOs, only data can be exchanged that is also present as entries in the object dictionary.

Note

The CANopen node layer is still supported for capability with versions up to and including 9.0 SP6.

It was replaced by the CANopen Simulation Concept that is supported starting with CANoe 10.0.

Note

The object dictionary can only be modified if the node was not yet started with coStartUp.

Generally there are two possibilities for creating an object dictionary. In the simplest case, an EDS or a DCF is available for the CANopen ECU to be simulated. Then the object dictionary can be created via the function call coODCreateFromFile.

As an alternative, there is also the possibilities of creating object dictionary entries via individual function calls. When creating an entry, the following information is required:

  • Index
  • Subindex
  • Data type of the object
  • Permissible access types
  • Initial value

The permissible access types are coded as follows:

Code

Meaning

0

only read

1

read and write

2

only write

3

read and write - can only be mapped in a RPDO

4

read and write - can only be mapped in a TPDO

5

only read - can only be mapped in a TPDO

6

only write - can only be mapped in a RPDO

7

is handled in the application

The sequence in which entries are generated in the object dictionary is only relevant with respect to the data type of an object. If an object should be generated with a manufacturer-specific data type (not listed in the table of data types) then the object that represents this data type must be created first.

Frequently entries in the object dictionary are also realized as fields. This means that objects of the same type are combined under the same index. They distinguish themselves only in the subindex. Such a field can include up to 254 subobjects.
