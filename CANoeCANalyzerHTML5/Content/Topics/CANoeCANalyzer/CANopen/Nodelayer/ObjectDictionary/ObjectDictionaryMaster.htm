Object Dictionary for Master (deprecated)

 

In order to configure the node layer as master, the object dictionary must also contain the following objects:

Store Parameters (required for configuration manager)

Storage of the parameters in nonvolatile memory

Index

0x1010

Type

Array of Unsigned32

Description

If the value 0x65766173 (ASCII "save") is written to a subindex, then the parameters are stored in a nonvolatile memory and stored in binary fashion in a directory CO_Data in a file 'Nodename'.bin.

The node layer only supports the storage of all parameters (subindex 0x01).

 

Restore Default Parameters (optional)

Restore the default configuration of the node

Index

0x1011

Type

Array of Unsigned32

Description

If the value 0x64616f6c (ASCII "load") is written to a subindex, then the default parameters are restored.

The node layer only supports the restoration of all parameters (subindex 0x01).

 

Concised Configuration Storage (required for configuration manager)

Storage of node configurations in concised format

Index

0x1F22

Type

Array of Domain

Description

With this object, the configuration of a node can be stored in concised format. The subindex corresponds to the node ID. On boot-up, the master transmits the configuration to the respective nodes.

 

NMT Startup (required)

Defines the start behavior of the node

Index

0x1F80

Type

Unsigned32

Description

This object specifies the start behavior of the node.

Bit 0

0 = node is slave

1 = node is master

Bit 1

0 = only start assigned nodes

1 = start all nodes

Bit 2

0 = the node goes independently into the state Operational

1 = the node remains in the state Pre-operational

Bit 3

0 = assigned nodes are started automatically

1 = node is started by Request NMT (0x1F82)

Bit 4

0 = if for a node an error event occurs, then only this node is handled

1 = if an error control event occurs for a required node, then a 'reset all nodes' is executed for all nodes

Bit 6

0 = if an error control event occurs for a required node, then procedure is according to Bit 4

1 = if an error control event occurs for a required node, then 'stop all nodes' is executed for all nodes

 

Slave Assignment (required)

Assignment of the slave node

Index

0x1F81

Type

Array of Unsigned32

Description

With this object the slave nodes are assigned that the master must manage. The subindex corresponds to the node ID.

Byte 0

Bit 0

0 = the node is not a slave

1 = the node is a slave

Bit 2

0 = there is no reaction to an error control event

1 = there is a reboot of the node for an error control event

Bit 3

0 = optional slave

1 = required slave

Bit 4

0 = a 'reset communication' is always executed

1 = 'reset communication' is not executed if the node is in the Operational state

Bit 5

0 = user software version is not checked

1 = user software version is checked

Byte 1

Bit 0..7

8 bit value for retry factor

Byte 2..3

Bit 0..15

16 bit value for guard time in milliseconds

 

Request NMT (optional for NMT master)

Execute state change of a node

Index

0x1F82

Type

Array of Unsigned8

Description

With this object the state of a node can be changed. The new state must be written to the subindex that corresponds to the node ID.

 

Request Guarding (optional for NMT master)

Activation of the guarding

Index

0x1F83

Type

Array of Unsigned8

Description

With this object, the guarding for a node can be activated. The subindex corresponds to the node ID.
