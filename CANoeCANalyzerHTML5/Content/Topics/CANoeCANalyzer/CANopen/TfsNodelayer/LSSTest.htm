LSS Tests

 

To test the LSS functionality of a device you have the following functions available. Dependent on the device to be tested the functions can be arranged into groups.

Testing LSS Slaves

The device to be testes is a LSS Slave. A request of a LSS Master is simulated and it is waited for the correct answer from the LSS Slave.

  • coTfsLssSendSwitchStateModeGlobalRequest
  • coTfsLssSendSwitchStateSelectiveSequence
  • coTfsLssSendConfigureNodeIdRequest
  • coTfsLssSendConfigureBitTimingRequest
  • coTfsLssSendActivateBitParTimingRequest
  • coTfsLssSendStoreConfigurationRequest
  • coTfsLssSendInquireVendorIdRequest
  • coTfsLssSendInquireProductCodeRequest
  • coTfsLssSendInquireRevisionNoRequest
  • coTfsLssSendInquireSerialNoRequest
  • coTfsLssSendInquireNodeIdRequest
  • coTfsLssSendIdentifyRemoteSlaveSequence
  • coTfsLssWaitForIdentifySlaveResponse
  • coTfsLssSendIdentifyNonConfRemoteSlaveRequest
  • coTfsLssWaitForIdentifyNonConfSlaveResponse
  • coTfsLssSendFastScanRequest

Sequentially Testing of a LSS Master

The device to be tested is a LSS Master.

You can use these functions if the functionality of the LSS Master is known and you know the sequence of the LSS requests of the LSS Master. Within tests with these functions, the sequence of the requests must be kept.E.g. it is not possible to wait for a request A or B.

  • coTfsLssWaitForSwitchStateModeGlobalRequest
  • coTfsLssWaitforSwitchStateSelectiveSequence
  • coTfsLssWaitForConfigureNodeIdRequest
  • coTfsLssSendConfigureNodeIdResponse
  • coTfsLssWaitForConfigureBitTimingRequest
  • coTfsLssSendConfigureBitTimingResponse
  • coTfsLssWaitForActivateBitTimingRequest
  • coTfsLssWaitForStoreConfigurationRequest
  • coTfsLssWaitForInquireVendorIdRequest
  • coTfsLssWaitForInquireProductCodeRequest
  • coTfsLssWaitForInquireRevisionNoRequest
  • coTfsLssWaitForInquireSerialNoRequest
  • coTfsLssWaitForInquireNodeIdRequest
  • coTfsLssWaitForIdentifyRemoteSlaveSequence
  • coTfsLssSendIdentifySlaveResponse
  • coTfsLssWaitForIdentifyNonConfRemoteSlaveRequest
  • coTfsLssSendIdentifyNonConfSlaveResponse
  • coTfsLssWaitForFastScanRequest

Parallel Testing of a LSS Masters

The device to be tested is a LSS Master.

If you have an unknown LSS Master it is advisable to use the functions of this group. With these it is possible to wait for several LSS requests. Furthermore it is possible to answer a LSS request with more than one LSS response. For each LSS request you can provide the designated LSS response.

  • coTfsLssClearMasterResponseList
  • coTfsLssWaitForMasterRequestMessage
  • coTfsLssAddMasterResponseSwitchStateGlobal
  • coTfsLssAddMasterResponseSwitchStateSelective
  • coTfsLssAddMasterResponseConfigureNodeId
  • coTfsLssAddMasterResponseConfigureBitTiming
  • coTfsLssAddMasterResponseActivateBitTiming
  • coTfsLssAddMasterResponseStoreConfiguration
  • coTfsLssAddMasterResponseInquireVendorId
  • coTfsLssAddMasterResponseInquireProductCode
  • coTfsLssAddMasterResponseInquireSerialNo
  • coTfsLssAddMasterResponseInquireRevisionNo
  • coTfsLssSetMasterResponseIdentifyRemoteSlaveMask
  • coTfsLssAddMasterResponseIdentifyRemoteSlave
  • coTfsLssAddMasterResponseInquireNodeId
  • coTfsLssAddMasterResponseIdentifyNonConfRemoteSlv
  • coTfsLssAddMasterResponseIdentifyFastscan
