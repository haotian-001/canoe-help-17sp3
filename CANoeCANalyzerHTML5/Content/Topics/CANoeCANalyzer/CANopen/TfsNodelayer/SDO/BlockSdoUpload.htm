SDO Block Upload

 

The initialization phase for the SDO block upload consists of two calls of coTfsSDOBlockInit. With the first call, the client subcommand initiate upload request (cs = 0) is transmitted. Directly after the second call of this function (client subcommand = start upload (cs = 3)) the SDO server starts sending the first SDO block.

The call of coTfsSDOBlockUploadSegmentResponse must occur without temporal distance directly after the call of coTfsSDOBlockInit. Otherwise, it is very likely that not all data segments will be detected. With this function a block of this block is confirmed after receipt of all segments. The function must be called only once per SDO upload block and confirms the receipt of the individual segments.

The SDO block transfer is concluded with coTfsSDOBlockEnd. The possibly-transmitted CRC checksum can be checked with coTfsSDOCalcCrc.

The complete SDO block upload can also be executed with the help of the command coTfsSDOBlockUpload.
