SDO Block Download

 

The SDO block download is initialized with coTfsSDOBlockInit. The block size can be called with coTfsSDOGetBlockSize and CRC checksum support with coTfsSDOChkCrcSupport.

The individual block segments are sent using coTfsSDOBlockDownloadSegmentRequest. The last call of this function must wait for the confirmation (acknowledge) of the server and if necessary, re-transmit segments that have already been sent. A block can contain up to 127 individual segments. The confirmation also contains the block size desired by the server. This can be called up via coTfsSDOGetBlockSize.

As many blocks are transmitted as are necessary in order to transmit the desired data set.

Only in the last segment of the last block must the parameter isLastSegment of coTfsSDOBlockDownloadSegmentRequest be set to 1 in order to introduce the SDO block end phase.

The SDO block download is concluded with coTfsSDOBlockEnd. Here the CRC checksum is also transmitted that can be calculated with coTfsSDOCalcCrc.

The complete SDO block download can also be executed with the help of the command coTfsSDOBlockDownload.
