Object Dictionary Tests

 

This test modules allows checking existent and hidden object dictionary entries.

A simple but not complete check can be done with coTfsODChkStdEntries. In this test only necessary and recommended objects are checked. Because the detailed device description is unknown, only the basis functionality of the device can be checked. A more detailed check is described in the following paragraph.

 

For checking the object dictionary the necessary parameter are saved in a internal list of the test module. This list is uses to realize the possible complexity of the test in CAPL. For a complete test the following functions are available:

  • First the list should be reset with coTfsODClearAllEntries.
  • The all entries should be added with coTfsODAddEntry. Because only one permissible value can defined for the function call, the user has the possibility to add more single entries with coTfsODAddOptEntryValue. A complete value range can be defined with coTfsODAddOptEntryValueRange.
  • Depending on the device it is possible that the object dictionary contains objects, which only differ by the subindex. With coTfsODAddEntrySubIndexRange identical entries can be defined only differing by their subindex.
  • After complete definition of the internal list, the functions coTfsODChk and coTfsODChkNotExist can access the elements. The first function checks if the given objects exist, executes a read or write access (depending on the access type) and compares the read data with the default values. The second functions checks if the object does not exist in the object dictionary.
  • According to the complexity of the configured entries, the test of the object dictionary needs some time.  So the user can resume with the test after occurrence of an error or stop the test. coTfsODChkBreakOnError defines which of the both possibilities is used.

Additionally the test sequence coTfsODChkSingleEntry is available. It allows the existence check of a single object but only with one permissible value. For more complex tests, the test sequences described above should be used.
