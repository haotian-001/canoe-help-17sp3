Monitoring

 

By using the monitoring functions you can monitor SDO and NMT messages that are sent between CANopen devices in the network. The functions check if SDO requests, SDO responses and NMT responses (Boot-up message) are processed and sent in a defines time span.

In contrast to the SDO timing functions messages that are not sent from CANopen test module are monitored too. Therefore the SDO timing functions are obsolete and should not be used any longer.

To monitor a node set the node ID of a single node (coTfsMonitorIncludeNodeId) or a range of node IDs (coTfsMonitorIncludeNodeIdRange). If you want to exclude nodes from monitoring, use the functions coTfsMonitorExcludeNodeId and coTfsMonitorExcludeNodeIdRange.

If you have added nodes to be monitored or excluded nodes from monitoring, you have to start the monitoring process with coTfsMonitorActivate. to take over theses changes.

You can stop the monitoring at any time with coTfsMonitorDeactivate. All values that are monitored by this time are added to the test report. These are the minimum, maximum and average response times of the nodes, the number of monitored events and the number of failed or late responses.

If the test module has writing access to an active test report, events are logged that have caused a violation of the defined time values (coTfsMonitorSetSdoTimeout and coTfsMonitorSetNmtTimeout). This is independent of other running tests.
