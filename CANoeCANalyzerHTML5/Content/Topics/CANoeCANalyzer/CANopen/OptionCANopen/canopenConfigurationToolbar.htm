Toolbars

 

Toolbar of the Main Window

Symbol

Function

Description

New CANopen Node from Electronic Data Sheet

Adds a new CANopen node based on an EDS file. A device configuration file is created automatically to mirror the contents of the object directory.

New CANopen Node

Adds a CANopen node without an EDS file, i.e. with an empty object directory. A device configuration file is created automatically to mirror the contents of the object directory.

New Group

Adds a new group in which CANopen nodes can be combined as a logical group.

Delete

Deletes the selected CANopen nodes or groups.

Import CANopen Configuration

Imports a CANopen configuration.

Export CANopen Configuration

Exports the entire CANopen configuration of the current network into a file. This can be imported into another CANoe configuration, for example.

CANopen Settings

Opens the CANoe Options dialog with the CANopen specific settings.

Create Simulation Nodes

Creates a simulation node for each CANopen node in the Simulation Setup if it does not yet exist and creates a DBC database.

Create and Attach DBC Database

Creates a DBC file for the current CANopen configuration and adds it as a database for the network. Existing files may be overwritten or removed.

Scan Network

Searches CANopen nodes on a network and reads in their settings. If a suitable EDS file is found in the EDS directories (see button EDS Folders), it is automatically assigned to the found node. If no suitable EDS file was found, all communication relevant objects of the node are read.

Read All Entries of Selected Nodes

Reads out the object directory of all selected nodes or all nodes of the network via SDO upload. Only objects contained in the Device Configuration File and not marked as WO (Write Only) are considered.

Write All Entries of Selected Nodes

Writes all values to the object directories of the selected nodes or all nodes of the network via SDO download. Only objects that are not marked as RO (Read Only) are considered.

Store Parameter

Sends the command for saving the current values in the object directory.

Restore Parameter

Sends the command for resetting the current values in the object directory to the default values. An NMT reset is required in order for the CANopen node to accept the default values.

Toolbar of the PDO/SRDO Connections Page

Symbol

Function

Description

Read Entries of all PDOs/SRDOs

Read the object entries of the PDO/SRDO configuration of all nodes.

Write Entries of all PDOs/SRDOs

Write the object entries of the PDO/SRDO configuration of all nodes. The network should first be set to the NM state PreOperational.

PDO View

Representation of the data exchange in the network according to PDOs.

PDO Object View

Representation of the data exchange after objects mapped in PDOs.

SRDO View

Representation of the data exchange in the network according to SRDOs.

SRDO Object View

Representation of the data exchange after objects mapped in SRDOs.

—

Filter of current view
  • Tx-Rx – Only connected PDOs/SRDOs
  • Tx only – Not connected Tx PDOs/SRDOs
  • Rx only – Not connected Rx PDOs/SRDOs
  • All – Unfiltered view

Connect another Rx PDO/SRDO

Add another Rx PDO/SRDO to the selected Tx PDO/SRDO. The Tx CAN ID is then entered for the selected Rx PDO/SRDO.

Disconnect Tx and Rx PDO/SRDO

With the selected Rx PDO/SRDO a free CAN ID is entered and the Valid flag is removed. If the Tx PDO/SRDO is not assigned to another Rx PDO/SRDO, the Valid flag on the Tx side is also removed.

Set Valid Flags for selected PDOs/SRDOs

Sets the Valid flag of the Tx and Rx PDO/SRDO.

Remove Valid Flags for selected PDOs/SRDOs

Removes the Valid flag on the Rx side. If no further Rx PDO/SRDO with the Valid flag set is assigned, the Valid flag on the Tx side is also removed. With SRDOs, the connection is disconnected because the transmission direction is bound to the valid state.

Creates a counterpart for PDOs and SRDOs

For all valid but not connected Tx and Rx PDOs and SRDOS a CANopen simulation node is created which receives respectively sends these PDOs.
