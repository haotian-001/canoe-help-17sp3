CANopen with CANoe | Concept

 

CANopen is a CAN-based communication protocol for the networking of devices from different manufacturers. A CANopen device contains a logical CANopen node represented by a unique node ID on the network and is characterized by an object directory in which process data and parameters are organized.

In the CANopen Configuration window, you can edit the contents of the object directory of all CANopen nodes of the network simultaneously and consistently with one another. A copy of the object directory in the form of a so-called device configuration file is created for this purpose. These files are generated in the _CANopen_<Configuration Name>\<Network Name> subfolder. Using the SDO protocol (service data object), the contents of the files can be compared to the object directory of the nodes. CANoe represents the entries in the object directory using system variables in the CANopen namespace. Access from CAPL or display in the Graphics Window is possible in this way, for example.

All CANopen specific functions which can be activated in the window for CANopen nodes are based on standardized entries from the so-called communication profile area of the object directory (index 0x1000 to 0x1FFF). Their functions are described in CiA documents DS 301 and DSP 302. For example, the heartbeat cycle time for the producer is configured in object 0x1017 subindex 0. If a network is selected in the Configurations view, the heartbeat configuration of all nodes can be edited simultaneously in the Details view on the Heartbeat page.
