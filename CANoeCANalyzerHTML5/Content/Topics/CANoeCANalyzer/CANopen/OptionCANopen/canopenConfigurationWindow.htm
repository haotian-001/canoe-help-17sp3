Toolbar (CANopen Configuration)

View: Configurations

In the Configurations view, you can see all CAN networks and the CANopen nodes and node IDs contained within them. You can see the transmit, receive and safety PDOs (process data objects) of a node below the node. By selecting an element, you can make changes in the Details view.

View: Details

Different details are available depending on the selected element in the Configurations view.

Selected Element

Available Pages

Description

Network

General

Name
Name of the network from the Simulation Setup or name of the CAN channel.

  • DBC only for configured node IDs
    When generating the database, only CAN frames of the nodes contained within the CANopen Configuration window are taken into account. If the option is deactivated, the standardized CAN frames of the maximum possible 127 nodes are entered.
  • DBC for extended interpretation
    The generated database can also be used to interpret SDO commands. If you deactivate this option, the database will be considerably smaller and load faster.
  • Folder for DBC and DCF files
    In this folder, all CANopen specific files are generated, such as the database and the device configuration files. You can copy the path of the folder, but you cannot change it.

PDO/SRDO Connections

On this page you can see the Tx PDOs and Rx PDOs as well as SRDOs of all nodes of a network. You can connect them with each other and set communication parameters such as transmission type and cycle time. You can also configure the mapping objects, that is, the content of the transferred frames. The functions and use cases on this page are described on the PDO/SRDO Connections page.

SYNC

The synchronous transmission of PDOs requires the configuration of a SYNC frame on the network. From this page, you can specify the nodes which are to send the SYNC frame, the CAN ID to be used and the cycle time to be followed. With the Apply globally button, this configuration is copied to all nodes. Non-uniform SYNC configuration values are displayed in red.

You can read out and overwrite the SYNC-specific settings of the network with and .

Heartbeat

You can configure heartbeat monitoring on the network. You can see the cycle time with which nodes generate their heartbeat frame and the timing with which the consumers monitor it. If 0 is entered, a node does not receive the frame. You can process values simultaneously by selecting multiple cells in the table.

You can read out and overwrite the heartbeat configuration of the network using and on the toolbar. It is also possible to filter the table so that only those nodes for which heartbeat monitoring is active are displayed. Incorrect values (e.g. consumer time less than producer time) are displayed in red.

CMT

If the CANopen network is configured centrally by a device, this configuration data can be displayed, read and written on the CMT page for all devices.

LSS

You can configure node ID and bitrates with help of LSS.

Node

General

Name
You can specify the name of a CANopen node here. This affects the system variables created for the node.

  • Node ID
    Unique ID of the node [1 to 127] on the network. When the Node ID is changed, all objects whose default value includes the Node ID can be set to the new default value (e.g., the COB-ID of the first TPDO defined as 0x180 + $NODEID). Please check the PDO/SRDO and heartbeat settings after resetting.
  • Electronic Data Sheet
    Description file from which the object directory of the node was copied. The path is for information only and cannot be changed.
  • Device Configuration File
    File which contains the contents of the object directory of the node. Using the button in the input box, it is possible to import the contents of another file and edit the file with CANeds.
  • CANopen Simulation
    Using this check box, you can activate and deactivate the CANopen simulation of the node.
  • Simulation Node
    Here, you can assign a node to the CANopen node in the Simulation Setup which is to execute the simulation.

Object Dictionary

On this page, you can view and edit all entries of the object directory of a node. For complex objects, the sub-elements that are not used according to sub-index 0 are displayed inactive (grayed). Using the context menu of an entry, you can edit the entry directly with CANeds and copy the associated system variable name to the clipboard. You can drag and drop the associated system variable to the Graphics Window or State Tracker.

You can read the current value from the node via SDO and write a new value using and .

TPDO/RPDO

TPDO/RPDO

You can set the communication parameters of the TPDO or RPDO in the top section of the page. The corresponding object, 0x1800 or 0x1400, is edited in doing so. In the bottom section, you can configure the objects mapped to the PDO (0x1A00 or 0x1600). The selection menu in the Index column shows you all mappable objects. Using the context menu, you can add or remove mapping objects.

SRDO

SRDO

You can set the communication parameters of the SPDO in the top section of the page. The corresponding object, 0x1301, is edited in doing so. In the bottom section, you can configure the objects mapped to the SRDO (0x1381), whereby you must map an object and an inverted object as a pair. If the same object is mapped for both, inversion generally occurs in the CANopen stack.
