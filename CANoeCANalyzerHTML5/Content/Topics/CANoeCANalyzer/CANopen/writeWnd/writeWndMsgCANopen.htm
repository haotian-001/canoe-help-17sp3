Messages of the Modules for CANopen/CANopen Safety specific Analysis [▲ back]

Code

Type

Description

CO0001

 Error

Error in <node dcf>[<object number>]: SCT, RFT or SRVT is not a number

The node description has no numeric value for the specified object. The value must be of type UInt8 (SRVT) or UInt8 (Refresh/SCT). The object is not monitored during measurement.

CO0002

 Error

Error in <node dcf>[<object number>]: <multiple SCT> <multiple SRVT> <multiple RefreshTimes> for SRDO defined

The specified SRDO in the DCF object is defined differently in another DCF object. To process an error monitoring of the SRDO anyway, the smallest value in deviation of zero is used as reference value.
CO0003

Information

SRDO configuration of <node dcf> is inconsistent. Non-monitored SRDO are marked with '!' within the interpretation columns

The SRDO configuration is not correct and the SRDO is not monitored.

Possible reasons:

  • The Identifier is used in more than one pairs, e.g. SRDO1 [102, 101]; SRDO2 [102; 111].
  • SCT, SRVT (Rx) or RefreshTime (Tx) have a value of 0.
CO0004

Information

Not every node has dcf in <dbc name>. Those are not monitored by the hook

The network cannot be monitored completely due not all nodes have a node description. Time measurement is executed but cannot be verified. A ? is displayed after the time stamps.
CO0005

Information

No DCF files found in <dbc name>

The database contains no node description (see CO0004).

Messages of the CANopen Node Layer Module [▲ back]

Type

Description

Error

<CANopen Node Layer>: VIACAN::CreateMessageRequest failed
<CANopen Node Layer>: VIACAN::CreateStatusRequest failed
<CANopen Node Layer>: VIAService::GetBusInterface failed

An internal error occurred.

With the current state of technology it is not possible to develop completely error free complex programs. This applies to CANoe.CANopen too.

To localize and fix rare system errors, the program uses diagnosis mechanisms to report these errors. Please note the text of the error message and contact Vector Support. With detailed information the support may help you fast to solve the problem.

Error

<NodeLabel>: Please save the CANoe configuration before starting the measurement

The CANopen node layer is used in a CANoe configuration that is not saved yet. The CANopen node layer derives internal used paths from the storage location of the configuration. So the configuration has to be saved before measurement start.

Error

<NodeLabel>: The file <filename> was not found, please add it to the 'user files' in the CANoe options

This error mainly occurs on using CANoe in distributed applications. Depending on the configuration the CANopen simulation uses external files (DCF files, simulated nonvolatile memory). These files must be made available with the mechanism of User Files on using the distributed mode.

Information

<NodeLabel>: EDS/DCF file '<filename>' <various error messages>

This message refer to load the specified EDS/DCF file and inform about possible errors/warnings in the EDS/DCF file.

Information

<NodeLabel>: Warning, CANdb Name not determined for the node!
<NodeLabel>: Warning, CANdb Attribute "CO_ID" not determined for the node or no valid ID assigned!

To differentiate error messages of several simulated CANopen node, the node name gets a prefix. To define this prefix several conditions must be fulfilled. The necessary steps are described here.

Information

<NodeLabel>: Warning, callback function coOnError( dword, dword, dword, dword, dword, dword ) is not defined
<NodeLabel>: Warning, callback function coOnUploadResponse( dword, dword, dword, dword ) is not defined
<NodeLabel>: Warning, callback function coOnDownloadResponse( dword, dword, dword ) is not defined
<NodeLabel>: Warning, callback function coOnNodeChangedState( dword ) is not defined
<NodeLabel>: Warning, callback function coOnEmergency( dword, dword, dword ) is not defined
<NodeLabel>: Warning, callback function coOnBootUp( dword ) is not defined
<NodeLabel>: Warning, callback function coOnConfigReady( dword, dword ) is not defined
<NodeLabel>: Warning, callback function coOnUploadIndication( dword, dword ) is not defined
<NodeLabel>: Warning, callback function coOnDownloadIndication( dword, dword, dword, dword ) is not defined
<NodeLabel>: Warning, callback function coOnLssEvent( dword, dword, dword ) is not defined

To use the complete functional range of the CANopen node simulation, several callback functions are necessary. These callback functions must be defined in the appropriate CAPL node with the specified signature.

Information

<NodeLabel>: Warning, node ID <Node-ID> already used by another node!

The specified node ID is already used by another simulated node. This may point to an error in the configuration and may lead to unpredictable behavior in the CANopen network.

Information

<NodeLabel>: Warning, node-Id <Node-ID> differs from the previously determined node ID <Node-ID> on call of coODCreateFromFile()!

DCF files as well as EDS files can be used to define the object dictionary of simulated nodes. In EDS files the CAN identifier are derived from the node ID. So this node ID must be set on creating the object dictionary. Due the object dictionary can not only be created via this mechanism, the node ID must be set on starting the node. This message may point to an error in the simulation configuration.

Information

< NodeLabel >: Array object [<object index>] of type <data type> was replaced by array object of type <data type>
<NodeLabel>: Array object [<object index>] of type <data type> was replaced by object [<object index>,<object sub index>] of type <data type>
<NodeLabel>: Array object [<object index>] of type <data type> was replaced by object of type <data type>
<NodeLabe>: Object [<object index>,<object sub index>] of type <data type> was replaced by object of type <data type>

An object has been defined multiple times on creating the object dictionary. The last definition has the highest priority and overwrites previous definitions.

Information

<NodeLabel>: Rx COB-ID <CAN identifier> of the default SDO server [<object index>,<object sub index>] differs from the CiA DS 301 specification (<CAN identifier>)
<NodeLabel>: Tx COB-ID <CAN identifier> of the default SDO server [<object index>,<object sub index>] differs from the CiA DS 301 specification (<CAN identifier>)

The configured CAN identifiers of the default SDO server differ from the CAN identifiers defined in the standard CiA301. This may be correct, depending on the used device profile.

This message is an information that other connected CANopen devices or applications must use these CAN identifiers.

Information

<NodeLabel>: Environment variable '<environment variable name>' of type float is connected with non-float object [<object index>,<object sub index>]

The data type of the environment variable that is used for the connection differs from the data type of the object. This may be correct and the intention of the user.

This message is an information of possible conversion errors of the data.

Information

<NodeLabel>: Number of elements of structured object [<object index>] decreased - sub object(s) <lowest sub index> to <highest sub index> deleted

This message may occur if a structured object has a sub object 0. If this object is set to an value smaller than the previous value during measurement, the specified sub objects are deleted and not longer available.
