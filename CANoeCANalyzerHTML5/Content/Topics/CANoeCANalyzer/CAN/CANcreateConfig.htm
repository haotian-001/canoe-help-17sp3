Create CAN Configuration

 

The basis for working with CANoe is the so-called configuration, a CFG file in which all settings are stored. You can create a configuration manually from a template (described below) or as a copy of an existing configuration.

Alternatively, you can also use the Model Generation Wizard, which generates a configuration based on a CAN database and the selected Interaction Layer (IL), inserts the contained network nodes in the simulation setup and generates control and display panels to be able to control a rest bus simulation.

Create Configuration From Template

Step by Step Procedure

Assign Database (DBC)

  1. In File Ribbon Tab select the New command.
    The available templates are displayed. The templates differ in the number of channels and the utilized baudrates.
  2. Select the CAN 500kBaud 1ch template, and click on Create Configuration From Template.
    A configuration with network CAN and channel CAN 1 is created.
  3. Open the Simulation Setup via the Simulation ribbon tab.

  4. Select the context menu command Add.... of the Databases entry.

  5. In the Open dialog, select the desired database (DBC) and apply it the network CAN by clicking [Open].

Add and Configure Nodes

  1. Select the context menu command Insert Network Node in the system view of the Nodes entry.
    The node ECU 1 is inserted in the system view and in the graphical overview.
  2. Select in the graphical view for node ECU 1 the context menu command Configuration....
  3. In the Node Configuration dialog, on the Common page, under Network node, select the desired node from the inserted database.
  4. Accept the settings for the node from the database with [OK].

  5. Repeat these steps for all nodes you want to include from the DBC in your configuration.

Information on Interaction Layer

Definition - What is the Interaction Layer?

The Â»Interaction Layer (IL) is a modeling library that models the send behavior of the network node automatically at runtime based on the attributes of network nodes, frames, or signals.

In conjunction with the Interaction Layer in a configuration, there are three basic variants, which are described in more detail below:

  1. IL Already Stored in Database
  2. IL not stored in database, CAN Default IL is used
  3. IL not stored in database, CAN Default IL is not used
    (this third variant deviates from the standard method and is reserved for special cases)

1. IL Already Stored in Database

If an Interaction Layer in the node attributes is referenced in your database, you see this module information in the Simulation Setup:

  • in the tooltip of the node
  • in the Node Configuration dialog on the Components page

Because the network nodes use an IL, you can now start the measurement in order to monitor the bus traffic in the Trace Window. To do so, open the Trace Window using the Analysis Ribbon Tab . Important: Application channels must be assigned in the Vector Hardware Config dialog.

To set signal values and send them to the bus, you can open the Node panel at the network node (using icon or context menu), change the values, and send the values to the bus using the Update button.

2. IL Not Stored in Database, CAN Default IL Is Used

If your database does not contain a reference to an IL, a CAN Default Interaction Layer is used when generating the configuration. The CAN Default IL is selected on the Bus Systems page (Options dialog).

The Vector-IL is selected by default. The CANoe Interaction Layer (CANoeILNLVector.dll) is then referenced in all network nodes.

3. IL Not Stored in Database, CAN Default IL Is Not Used

You can also program the general send behavior of the frames with the help of CAPL. However, this is the conventional method of sending and is only still used to map special cases.

The mapping of the send behavior using an IL is recommended.

If you want to generate a configuration without IL support nevertheless, you must activate the None option in the CAN Default Interaction Layer area on the Bus Systems page (Options dialog) before generating the configuration.
