Create CAN Database (CAN Communication Matrix)

The use of a CAN database is recommended for the development, analysis, and testing of CAN networks. This database contains the network architecture and the communication data of the distributed network nodes. For sending, it is customary to use an Interaction Layer (IL), which takes over the sending of frames based on the attributes of network nodes, frames, or signals in the communication matrix.

A CAN database is created in accordance with the CAN specifications with the following editors of Vector Informatik GmbH:

AUTOSAR Explorer (file format ARXML)

Start CANoe and open the AUTOSAR Explorer using the Tools Ribbon Tab. This editor provides the following options:

  • Create AUTOSAR CAN database
  • View and analyze database
  • Function for validation of the communication matrix according to the AUTOSAR schema:
    Menu path View|Schema Validation

Cross Reference

You can find more detailed information in the help of AUTOSAR Explorer.

CANdb++ Editor Standard (file format DBC)

Start CANoe and open CANdb++ using the Tools Ribbon Tab. This editor provides the following options:

  • Create database
  • Consistency check of certain CAN properties for CAN communication
  • Menu path Options|Settings|Consistency Check
  • Menu path File|Consistency Check

Cross Reference

You can find more detailed information in the help of CANdb++.

PREEvision Function Designer CAN (file formats DBC, XML, ARXML)

The PREEvision Function Designer CAN by Vector Informatik GmbH enables a complete network design.
