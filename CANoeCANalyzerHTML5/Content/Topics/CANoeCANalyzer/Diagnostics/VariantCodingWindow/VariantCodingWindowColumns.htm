Columns

Ribbon|Diagnostics & XCP Tab|Variant Coding

The columns in the Coding Data area have the following meaning:

Column

Description

ECU/Coding Domain/Parameter

Name or qualifier of the respective ECU (/diagnostic description), coding domain or (sub-)parameter. By clicking on the arrow buttons you can expand or collapse nested data structures.

DID

Display of the Data Identifier.

Segment Numer

Display of the Segment Number.

Specified Value

Specified value for the relevant parameter, is initialized with the default value stored in the diagnostic description and written to the connected ECU when writing. Differences between ECU value and specified value are highlighted in color.

ECU Value

Value read from the connected ECU during the last operation Read for the relevant parameter or value written during the last operation Write (if writing was successful according to the diagnostic response).

Unit

Unit belonging to the respective parameter, if defined in the diagnostic description.

Last Operation

Result of the last operation (read, write) performed. The following results are possible:

  • —
    So far, no operation has been performed.
  • Pending
    An operation is in the queue.
  • Reading
    Operation Read was started and is not yet finished.
  • Read
    Variant coding data was successfully read from the connected ECU.
  • Writing
    Operation Write was started and is not yet finished.
  • Written
    Variant coding data was successfully written to the connected ECU.
  • Error
    An error occurred while performing the last operation.

Selection

By activating the relevant check box, you can select the parameters, coding domains and controllers to which the Read or Write operations should be limited. The check box in the column header allows you to select or deselect all elements at once.
