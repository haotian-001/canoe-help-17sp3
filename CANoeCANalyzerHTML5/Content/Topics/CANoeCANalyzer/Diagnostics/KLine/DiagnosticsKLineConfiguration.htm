K-Line Settings

Ribbon: Diagnostics & XCP Tab|Diagnostics/ISO TP Configuration|Administrate Diagnostic Descriptions|Assignments|K-Line Settings

Configuration using Vector Interfaces

K-Line Channel Configuration using Vector Interface

To use a K-Line channel please select a K-Line channel in the Options dialog under Measurement|General|Channel Usage.

Assign a K-Line application channel to the used hardware in the Vector Hardware Config dialog.

In the Diagnostics/ISO TP Configuration dialog now the Vector hardware based K-Line channels are visible - beside the COM interface - and can now be configured for diagnostics.

Configuration of the K-Line ECU simulation using Vector Interface

To simulate a K-Line ECU please select a K-Line channel in the Options dialog under Measurement|General|Channel Usage.

Assign a K-Line application channel to the used hardware in the Vector Hardware Config dialog.

In the Diagnostics/ISO TP Configuration dialog now the Vector hardware based K-Line channels are visible - beside the COM interface - and can now be configured for the simulation.

Note

For the initialization of an ECU simulation, the ECU must be configured in the Diagnostics/ISO TP Configuration dialog. The initialization itself must be done with CAPL.

Configuration of the K-Line Monitoring using Vector Interface

To observe K-Line communication please select a K-Line channel in the Options dialog under Measurement|General|Channel Usage.

Assign a K-Line application channel to the used hardware in the Vector Hardware Config dialog.

In the Diagnostics/ISO TP Configuration dialog now the Vector hardware based K-Line channels are visible - beside the COM interface. Assign a diagnostic description or standard diagnostic description to the K-Line channel. Under Usage of the diagnostic description select the Interpretation Only option.

Preliminary Notes

Once you added a diagnostic description to the K-Line channel you can configure following parameters for the K-Line diagnostic channel.

Note

All numerical parameters must be provided in decimal format.

Parameters from Diagnostic Description for Transport and Diagnostic Layer

Override manually

If the diagnostic description has an interface matching to the network, the default parameters from the interface of the diagnostic description are used. These parameters (see below) you can adjust manually, after activating the Override manually check box.

Initialization

  • None
    The channel will not be initialized (no init pattern is being sent). Only the diagnostic requests will be sent as K-Line frames at the configured baudrate.
  • Fast
    Additional timing parameters for fast initialization.
  • 5 Baud
    Additional timing parameters for 5 baud initialization.

Addressing

Force 4 byte header
If this check box is activated, 4 byte headers will be used for all diagnostic frames irrespective of actual frame length.

  • Tester
    Address of the diagnostic tester.
  • ECU
    Address of the diagnosed ECU.
  • Functional
    Functional address of the diagnosed ECUs.

Timing

  • Idle time
    Delay before a new initializing of an ECU.
  • P3 min. time
    Minimum distance to the next diagnostic request in ms.
  • P4 min. time
    Minimum distance between two request bytes in ms.

Access timing values
Specification of the use of the AccessTimingValues service.

  • Do not query
  • Query and log
  • Query and use

Physical Layer

  • Port
    Serial port used for communication with the ECU
  • Baudrate
    Baudrate of the diagnostic tester.
  • UART configuration

    Configures the format of a data byte:
  • Length: 7/8/9 bits
  • Parity: no/odd/even
  • Stop Bits: 1 or 2
  • Use echo bytes (RS232 only)
    It is expected that transmitted bytes will be reported.
