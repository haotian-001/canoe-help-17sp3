Basic Diagnostics

 

Using the basic diagnostics, you can exchange diagnostic information with the ECU also without a diagnostic description (CDD, ODX/PDX, MDX).

Note

Basic diagnostics is supported in CANalyzer pro and CANoe run/full as of version 7.2 SP3 and CANalyzer exp as of version 7.5.

Depending on the protocol you want to use (KWP or UDS) select the corresponding basic diagnostic description.

For simple use cases, basic diagnostics is an alternative to the process-oriented approach using diagnostic description files.

Supported Use Cases

  • Description of simple diagnostic services (UDS & KWP) with the Basic Diagnostic Editor.
  • Sending/receiving of defined requests/responses on CAN, LIN, FlexRay, K-Line and Ethernet (DoIP/HSFZ) by means of
  • Diagnostic Console and
  • CAPL/TFS
  • Symbolic interpretation of basic diagnostic services as well as their parameters in the Trace Window.

Basic diagnostics thus represents an extension to the process-oriented approach with CANdela diagnostic descriptions for simple applications.

Using Basic Diagnostics

In order to use basic diagnostics, you must add a basic diagnostic ECU to the diagnostic configuration.

Outside of the measurement, you can now define/modify the diagnostic services in the and commit them to the diagnostic console. Saving the configuration also commits these changes.

While measurement is stopped you can now code the diagnostic services in the Basic Diagnostic Editor. The services coded in this way are transferred to the Diagnostic Console by pressing the Commit button or saving the configuration.

After the measurement starts, you can call up the services in the Diagnostic Console,in CAPL and also in CAPL test modules/ test case libraries.

Note

On the same CAN bus or CAN network you can work with ECUs configured using diagnostic description files as well as with multiple basic diagnostic ECUs at the same time. The TP parameters of these ECUs must be different though.
