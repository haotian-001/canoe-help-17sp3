Diagnostic Parameters Window

Ribbon|Diagnostics & XCP Tab|Configuration|Diagnostic Parameters|Open Window

General

In the Diagnostic Parameters Window, you can insert diagnostic response parameters and configure the associated requests. After that you can send requests interactively via mouse click, or cyclically immediately after the measurement starts, and the parameter values contained in the responses will be displayed both in the Diagnostic Parameters Window and in the other analysis windows (the State Tracker, Data Window, and Graphics Window).

  • Columns
  • Toolbar

Insert Parameters

To allow diagnostic parameters to be inserted into the Diagnostic Parameters Window, it is necessary to configure a corresponding diagnostic description with usage as a tester with physical requests. You can configure the diagnostic descriptions with the Diagnostics/ISO TP Configuration dialog.

Note

Diagnostic descriptions that are configured as diagnostic testers with functional group requests are not supported by the Diagnostic Parameters Window.

You can insert parameters into the Diagnostic Parameters Window by dragging and dropping them from other analysis windows (the State Tracker, Graphics Window, and Data Window) or with the Symbol Selection dialog ( button). You can configure the displayed columns with a right click on the column heading and then selecting the Column Chooser context menu command, in which the column in question is moved between the Diagnostic Parameters Window and the Customization dialog with drag and drop. You can also change the column order by dragging and dropping the column header.

General Notes

Note - Display Diagnostic Parameters

To display diagnostic parameters in the Diagnostic Parameters Window, activate the corresponding diagnostic observer in the Diagnostics/ISO TP Configuration dialog:

  • Additional Diagnostic Settings | ISO TP Observer or FlexRay TP Observer | Performance

Note - Session and enabling the ECU

Before the request is sent, no automatic session change and no automatic unlocking of the ECU with security access requests occurs. You must use other means to ensure that the ECU is located in the corresponding session and enabled so that the requests sent by the Diagnostic Parameters Window can be answered by the ECU with positive responses and the desired parameters can thus be displayed (e.g., by using the Diagnostic Session Control.

Note -Responses are only sent to the Sender

Please note that, after every sending of a request, all responses are signaled only to the sender. Diagnostic requests that are sent from the Diagnostic Console are thus invisible to a CAPL program, as are the received responses. Similarly, no responses to requests sent from a CAPL program are displayed in the Diagnostic Console.

Note - Cycle Times

Please note that, depending on the bus system selected, the associated receipt and/or send buffer may fill up if the selected cycle times are too short. To avoid this situation, the cycle time should be set higher than the P2 time (client time) for the ECU in question. If the ECU typically responds significantly more quickly, you can reduce the cycle time accordingly.

If a request is not sent to an ECU, or an ECU has repeatedly failed to respond to a request, a corresponding error is shown, and further cyclic sending of the request is halted.

When selecting the cycle time, please also note that a new request – even for different parameters – can only be sent to one and the same ECU from the Diagnostic Parameters Window if a response has been received to the last physical request sent to this ECU or the P2 timeout (client timeout) has occurred. Especially if the ECU answers with a negative response with NRC 0x78 (requestCorrectlyReceived-ResponsePending), undesired delays may occur when requests are sent, and corresponding warnings may appear in the Diagnostic Parameters Window. The tooltip of the warning/error icon on a parameter provides information about how often sending of the corresponding request failed or was delayed, the response was not received, or a negative response was received. (NRC 0x78 is not evaluated as a negative response in this context.)
