Off-Board Diagnostics

 

For error detection with off-board diagnostics, one or more control units are connected to an external diagnostic tester that executes the error detection functions (e.g., CANoe). The diagnostic communication occurs via a 1:1 connection between the tester and the control unit using standardized diagnostic protocols.

During active diagnostics, the tester and the control unit typically exchange large quantities of complex data. The interpretation of these data depends not only on the (vehicle) manufacturer but also on the model of the control unit, its year of manufacture, or even on the individual control unit itself.

Working with Diagnostic Descriptions

The diagnostic data for a control unit are described in a standardized diagnostic description (CDD, ODX/PDX, MDX), taking into consideration the specification, control unit software, and test device parameterization.

The diagnostic description can be assigned to a CANoe configuration. This enables the sending of diagnostic requests as well as a diagnostic interpretation and the output of data with symbolic designators.

Based on the diagnostic descriptions, the Diagnostic Console contained in the Diagnostic Feature Set provides interactive access to all diagnostic requests. These can be selected, parameterized, and displayed with their response.

The Fault Memory Window enables you to read out the fault memory of a control unit and to delete entries (all entries or individual entries) from the fault memory (DTC - Diagnostic Trouble Code).

You can use the Diagnostic Session Control to change interactively from one status of a control unit to another (session change, security access) and to control the communication management settings of a control unit.

  • With its CANdela product family, Vector provides a comprehensive and process-oriented solution for the diagnostic development process for optimizing the diagnostics in the motor vehicle. The diagnostic data for a control unit are described in a standardized diagnostic description (*.cdd). These file forms the basis for the diagnostic functionality in software tools, such as CANoe and for the generation of diagnostic software components for integrating diagnostics in control units.

Standard Diagnostics: Working with reduced CDD Diagnostic Description

Let's assume:

  • You don't have any diagnostic description files for the control unit to be diagnosed.
  • But you have the diagnostic protocol (KWP, UDS) as well as the essential diagnostic communication parameters (such as CAN identifier and timeout times) of the ECU.

For this case CANoe provides reduced CDD diagnostic descriptions. These contain a standardized scope of diagnostic requests. This method is referred to in CANoe as Standard Diagnostics.

Basic Diagnostics: Working with simple customized diagnostic descriptions

You can use the Basic Diagnostics to exchange diagnostic information with the control unit even without diagnostic description files (CDD, ODX/PDX, MDX). With help of the Basic Diagnostic Editor you can therefore define services based on KWP2000 or UDS.
