OEM-specific Extensions — VW TP Add-on

 

Note

The observer is only available if the Vector CANoe Add-on VAG starting with Version 2.0 is installed!

Please contact Vector Support to get the add-on.

VW TP 2.0 Observer Usage

The VW TP 2.0 observer interprets messages, which are sent on the CAN bus according to the Volkswagen transport protocol 2.0. The results are displayed in the Trace Window:

  • Transmit and receive address of TP messages
  • TP protocol information for every CAN message (depending on messages type, e.g. proposed ID for channel setup messages)
  • IDs and name/qualifier of the diagnostic services
  • Names/qualifier/values for diagnostic parameters

 

  1. For analyzing messages in the Trace Window using the VW TP 2.0 observer, the observer has to be activated in the configuration dialog.
  2. The observer identifies dynamic channel setup (see protocol specification) and can monitor data transmissions.
  3. If the VW TP 2.0 interface was configured for a diagnostic description (Diagnostic Description Settings: Assignments) and if address as well as application type correspond to received/transmitted data, this is are interpreted according to the diagnostic descriptions and displayed in the Trace Window.

Note

Please also refer to samples and documents which come with the Vector CANoe VAG Add-on.

Configuration VW TP 2.0 Observer

The VW TP 2.0 observer can be activated in this dialog (Use settings found for diagnostic description and CAN databases) and thus enables the diagnostic interpretation.

Additionally the observer can also monitor CAN bus connections without being configured explicitly. For this purpose activate the option In addition observe dynamic channel setup on the following networks. If messages are sent on the selected buses with IDs in the range (hex) 0x200 to 0x2FF these messages are interpreted as setup messages for dynamic channels. The defined CAN IDs are also used for identifying channel messages.

If diagnostic descriptions are configured with VW TP 2.0 interface parameters the received/sent data is interpreted accordingly if ECU address and application type correspond to parameters set in this dialog.

Settings Transport Layer VW TP

This dialog allows the configuration of a diagnostic interface to communicate with ECUs using the Volkswagen transport protocol 2.0:

  • Address: Tester target address or ECU respectively; diagnostic testers typically use address 0.
  • Suggested ID: These values are proposed for the ECU by the tester during dynamic channel setup.
  • Application Type: For diagnostic applications typically 1 is used.
  • Block Size: Protocol parameter which defines the occurrence of Acknowledge messages.
  • Segment Size [Byte]: Because VW TP contains no length information, transferred data is segmented to be able to limit the ECU reception buffer. This setting fixes the maximum length of a segment.
  • T1, T3: Encoded values of timing parameters (see protocol definition) must be entered here. The decoded values are automatically displayed beside the editor field.

The VW TP 2.0 observer uses these parameters for diagnostic data transmission and diagnostic data interpretation.
