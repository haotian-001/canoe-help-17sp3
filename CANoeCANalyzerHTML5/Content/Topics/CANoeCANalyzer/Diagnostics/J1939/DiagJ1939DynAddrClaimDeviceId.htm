Settings for Address Claiming / J1939 Device Name

As soon you activate in the J1939 Transport Layer dialog under Address Claiming the Enable Address Claiming check box, there are overtaken the addresses you specify here in the J1939 Tester dialog under Settings for Address Claiming.

Here are provided the address fields that are described on the following page: Device Name

Note

CANoe cannot use any wildcards (see below) here because all fields must be defined. CANoe tries to get the initial TP address. In case of failing the search is repeated with following addresses.

Wildcards

  • A Device Name consists of 10 fields, one of which is defined as reserved and is therefore always 0. The remaining 9 fields are permanently assigned in a specific node.
  • Typically, however, only the values of a subset of the fields, and not all of the fields, are known exactly when diagnostics are performed. Thus, for example, the function of a group of devices is the same, but never their Identity Number.
  • In order to now deal with a different device identification (and thus different Device Names) in a CANoe configuration, fields can be masked in the configuration.
  • During address claiming, the first node whose Device Name matches the configured Device Name based on the mask is then accepted as the destination and its TP address is used.
