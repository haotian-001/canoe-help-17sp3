Diagnostic Console (Diagnostic Feature)

Ribbon|Diagnostics & XCP Tab|Diagnostic Window|Diagnostic Console (Diagnostic Feature)

General

The Diagnostic Console is a Diagnostic Feature that you can use within the Diagnostic Window. In the Diagnostic Console it is possible to directly send individual diagnostic requests of an ECU, display and evaluate the corresponding diagnostic responses.

  • Columns
  • Toolbar
  • Diagnostic Console Configuration

Basics

The Diagnostic Console can be configured as Diagnostic Feature within the Diagnostic Window. To use the Diagnostic Console, the corresponding diagnostic description must be available in CDD, ODX/PDX or MDX format.

Note

Please note that after each request is sent, all responses are reported only to the sender. Diagnostic requests sent from the Diagnostic Console are therefore invisible to a CAPL program, as are the responses received to them. Similarly, no responses to requests sent from a CAPL program are displayed in the Diagnostic Console.

The Diagnostic Console can also be used for diagnostic descriptions configured as Functional Group Requests.

Note

When opening the Diagnostic Console for a functional device, please note the following: For each device of the functional group whose response is to be displayed in the Diagnostic Console, a corresponding physical device must be created.

Window

The Diagnostic Feature Diagnostic Console is only available in online mode.

The displayed requests can be sent once or cyclically with an adjustable cycle time. To send, select the corresponding message and click the button in the Send column. Alternatively, you can also send a request by double clicking on the service identifier. Further settings that can be specified for a request are described under Columns.

The User-defined messages function allows you to preassign diagnostic requests with parameter values and save them under a descriptive name. For example, if you frequently use the same diagnostic request with different parameter values depending on the use case, you can now access the appropriate service immediately without having to enter parameters again.

Raw Messages are used to specify requests independently of the diagnostic description. They can either be derived from an existing request or freely built.

If Vehicle System Groups are defined in the diagnostic description, it is possible to switch to the view of the Vehicle System Groups.
