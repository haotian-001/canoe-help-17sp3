Columns (Diagnostic Console - Diagnostic Feature)

Ribbon|Diagnostics & XCP Tab|Diagnostic Window|Diagnostic Console|Columns (Diagnostic Console - Diagnostic Feature)

The Diagnostic Console columns can be displayed individually. Display the Column Chooser by a right click on the column header and select the desired columns. You can also change the position of the columns by drag and drop.

The columns have the following meaning:

Column

Available for

Description

ECU/Service Diagnostic Class/ Parameter

 

*

Name or qualifier of the respective ECU with specification of the selected variant, request, diagnostic class, vehicle system group or (sub)parameter. By clicking on the arrow buttons you can expand or collapse nested data structures.

Note

For raw messages or user-defined messages, the qualifier is editable. It must be unique within the ECU. If this is not the case, the corresponding request cannot be sent, the qualifier is displayed with a red frame. To have all qualifiers (and not the names) displayed, click on the Show Qualifier button in the toolbar .

Request

If there is a matching read request for a write request (SID = 0x2E), the parameters can be initialized via the button next to the qualifier of the write request by previously sending and reading the response of the associated read request. This function is not available for raw messages and user-defined messages. 

PDU / Raw Value

Request

Raw values of the PDU.

For raw messages all bytes are editable, for user-defined messages and services the signature is not editable. Parameters: Display of the raw value.

Parameters

Display of the raw value.

Send

Request

A button is displayed with which a request is sent. If a value is entered in the Cycle Time column, the button is used to start or stop cyclical sending.

Note

Data_Packet requests of the type ReadScheduleFast cause the ECU to send an endless response stream, which leads to a timeout. Although the corresponding message is acknowledged, the ECU continues sending until a StopScheduleSending type request is sent to the ECU.

Parameters

A check box is displayed for parameters. For optional parameters, this can be edited. An optional parameter is only sent if the check box has been activated.

Functional Group Request

Request

Activate this check box to send the request with the addressing for functional requests. If this is not permitted according to the diagnostic description, this is indicated by a red frame and corresponding information in the tooltip.

Note

Please note that for a diagnostic tester which is defined for functional group requests, the check boxes are activated and cannot be edited.

Suppress Positive Response Messages

Service, User-Defined Message

If the check box is activated, the Suppress Positive Response Message Indication bit is set when a request is made.

Note

The option can only be activated if it is supported by the corresponding service in the diagnostic description. Otherwise the option is inactive.

Cycle Time

Request

For cyclic sending of a request, the cycle time is specified here in seconds. The cycle time must be at least 0.1 s.

Note

When selecting the cycle times and the number of requests to be sent, the timings of the ECU must be taken into account. The diagnostic requests are sent sequentially. Too many requests lead to a delay and the specified cycle times cannot be adhered to.

Value

Parameter

Symbolic value for the parameter concerned, is initialized with the default value stored in the diagnostic description. If an invalid value is entered, it is not accepted when the cell is exited.

Note

For optional parameters, the value can only be entered if the check box in the Send column is activated.

Iteration Parameter

Number of iterations.

Unit

Parameter

Unit belonging to the respective parameter, if defined in the diagnostic description.

Signatur

Service, User-Defined Message

Display of the signature according to the diagnostic description.

SID

Request

Display of the service identifiers.

DID

Request (only if specified)

Display of the data identifiers.

Position

Parameter

Byte:Bit position.

Length

Request

Length of the requests in byte.

Parameter

Length of the parameter in byte:bit.

Type

Parameter

Data type according to diagnostic description.

Comment

*

If present in the diagnostic description.
