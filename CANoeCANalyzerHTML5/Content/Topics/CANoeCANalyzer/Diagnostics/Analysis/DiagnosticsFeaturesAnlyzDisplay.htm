Filter Diagnostic Objects using CAPL Filter in the Measurement Setup

The ISO TP observer generates events that are being sent to the Measurement Setup. There they usually arrive in all branches; if several Trace Window have been configured the TP event will be displayed in all of them.

To show events only in selected Trace Windows, a CAPL filter node can be inserted before the window. In this node the following function must be implemented at which two different signatures are being accepted.

  • In the first case the symbolic names of the sending node and the receiving node are handed over in the same way as they are shown in the Trace Window.
  • In the second case these values are left out since they aren't needed.

The return value determines how the event will be treated: "0" leads to the result that the event will be stopped while "1" leads to the result that it will be passed through. Certainly, the return value can also depend on the arguments that have been handed out to the function.

Note

Diagnostic events are reported as TP events and can be addressed like TP events.

Option1: with source & target

long CallbackTPDataIndication( dword timeStart, dword timeFinish, WORD channel, dword CanIdTx, dword additionalTx, dword CanIdRx, BYTE data[], char source[], char target[]) // optional parameters
  {
    return 1; // forward event; returning 0 would stop the event
  }

Option 2: without source & target

long CallbackTPDataIndication( dword timeStart, dword timeFinish, WORD channel, dword CanIdTx, dword additionalTx, dword CanIdRx, BYTE data[])
  {
    return 0; // stop event; returning 1 would forward the event
  }

Example

variables {
  long gbForward = 1; // Default: forward events. Toggle via keyboard or environment variable, e.g.
}

long CallbackTPDataIndication( dword timeStart, dword timeFinish, WORD channel, dword CanIdTx, dword additionalTx, dword CanIdRx, BYTE data[] char source[], char target[])
{
  int i;

  write( "[%.1f-%.1f] Received %d bytes '%s' -> '%s' (Ch%d 0x%x/%d->0x%x): "
      , timeStart / 100.0, timeFinish / 100.0
      , elcount(data)
      , source, target
      , channel, CanIdTx, additionalTx, CanIdRx);
  writeEx( 1,1, "[");
  for( i = 0; i < elcount(data); ++i)
    writeEx( 1, 1, "%02x ", data[i]);
  writeEx( 1,1, "]");
  if( !gbForward)
    write( "Not forwarding event to Trace Window.");
  return gbForward;
}

Display of Diagnostic Parameters

Diagnostic parameters which are defined via CDD, ODX/PDX, MDX diagnostic descriptions or the OBD-II function can be displayed in the Data and Graphics Window. A parameter is either configured by using the Symbol Selection dialog or by dragging data from the Trace Window. You can overwrite the name displayed in the legend.

Note

  • The displayed value is taken from the diagnostic primitive if it is recognized in the Measurement Setup.
    Thereby it is necessary to activate the ISO TP observer and the diagnostic interpretation as well as using a diagnostic description file (CDD, ODX/PDX, MDX).
  • If a non-numeric parameter is displayed in the Graphics Window its value is shown as -1.0.
  • The value range of a diagnostic parameter is not stored in the diagnostic description; therefore the display cannot be refreshed automatically.
    Instead, the range 0.0...10.0 is proposed which should be adjusted manually before a measurement.
    Alternatively the display can be adjusted automatically after receiving some parameter values.
  • If a parameter from an iteration is displayed, the first parameter found in the primitive will be evaluated.
    So far it isn't possible to determine the number of the iteration from which the parameter is taken.
