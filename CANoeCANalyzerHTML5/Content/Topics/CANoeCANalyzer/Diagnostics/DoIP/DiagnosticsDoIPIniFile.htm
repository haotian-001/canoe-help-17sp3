DoIP INI File

 

Edit

  • In this configuration file, you can adjust the starting values for the Diagnostics over IP/HSFZ configuration as well as any timing settings that cannot be modified in the configuration dialogs.
  • This file should only be modified when the application is closed (i.e. not running).
  • When using distributed mode or standalone mode, the DoIP.INI file on the corresponding VTP Device determines the system behavior. Via Software Update you can copy the DoIP.INI file from the user computer to the VTP Device.
  • The file location of the INI files you find via the Options dialog (General|File Locations). Next to Location of application data click the Open button.

[Connection]

Key (DoIP)

Description

VEHICLE_TCP_Data

ECU TCP data port (TCP_DATA)

The vehicle accepts TCP connection on this port.

TESTER_UDP_Data

Tester UDP data port (UDP_TEST_EQUIPMENT_LISTEN)

The tester receives Vehicle Announcement Messages on this port.

VEHICLE_UDP_Data

ECU UDP data port (UDP_VEHICLE_LISTEN)

The vehicle receives Vehicle Identification Requests on this port.

ForceTesterTCPSendPort

Defines a specific TCP source port for packets sent by the tester.

ForceTesterUDPSendPort

Defines a specific UDP source port for packets sent by the tester.

ForceVehicleUDPSendPort

Defines a specific UDP source port for packets sent by the vehicle.

Key (HSFZ)

Description

HSFZ_DiagnosticsPort

ECU TCP data port

The vehicle accepts TCP connections on this port.

HSFZ_IdentificationPort

Tester UDP data port

The tester receives vehicle identification messages on this port.

HSFZ_ControlPort

ECU UDP data port

The vehicle receives messages of the car identification service on this port.

[Timing]

Key

Description

T_DoIP_Ctrl

Length of time in ms the tester waits for a response to a previously sent control message.

A_DoIP_Announce_Wait

Maximum length of time in ms a DoIP ECU waits before sending a response to an Identification Request.

A_DoIP_Announce_Interval

Length of time in ms between two consecutive Vehicle Announcement messages.

A_DoIP_Announce_Num

Number of consecutive Vehicle Announcement messages.

A_DoIP_Response

Length of time in ms the tester waits for acknowledgment of a Request message.

T_TCP_Generic_Inactivity

Length of time in ms a DoIP ECU holds an inactive TCP socket open before closing it.

T_TCP_Initial_Inactivity

Length of time in ms after which a TCP socket is closed again if the DoIP ECU does not receive a Routing Activation message.

T_TCP_Alive_Check

Length of time in ms after which a DoIP ECU closes the TCP socket if it does not receive a response to an Alive Check message.

T_TCP_Diagnostic_Message

Length of time in ms after which a request or response is considered lost if no Diagnostic Message Acknowledgment message was received. After that time, the request or response may be repeated.

T_TCP_Vehicle_Discovery

This length of time in ms is only relevant, if a Vehicle Announcement / Vehicle Identification Response message with VIN/GID sync. status incomplete (0x10) was received.
In this case the value indicates the length of time a vehicle can take to perform the VIN/GID synchronization.

[Observer]

Key

Description

Active

Activates/deactivates (1/0) the interpretation of the messages on the DoIP channel.

[Specification]

Key

Description

DisableIdentification

Deactivates/activates (1/0) the Vehicle Identification process.

DisableRouteActivation

Deactivates/activates (1/0) the Routing Activation process.

EnableRouteActivationType

Activates/deactivates (1/0) the Activation Type attribute in the Vehicle Routing Activation Request datagram.

RouteActivationType2Byte

This value is only relevant, if the key EnableRouteActivationType is unequal 0
Defines the length of the Activation Type attribute:
0 means only 1 byte
1 means 2 bytes

StartControlTimerOnSendRequest

Reserved for internal use. Should not be modified.

SendRoutingActivationWithOEM

Activates/deactivates (1/0) the transmission of the optional Reserved for OEM-specific use attribute in a Routing Activation Request message. The content of this attribute can be defined using the key OEMspecific (see below).

[RoutingActivationRequest]

Key

Description

ActivationType

This value is only relevant, if the key EnableRouteActivationType is unequal 0. It defines the contents of the ActivationType attribute in a Routing Activation diagnostic request.

Reserved

The content of the attribute Reserved by this part of ISO 13400 can be defined by this key.

OEMspecific

This value is only relevant, if the key SendRoutingActivationWithOEM is unequal 0 and defines the contents of the optional attribute Reserved for OEM-specific use in a Routing Activation Request message.

[VehicleAnnouncement]

Key

Description

EID

The content of the attribute EID in a Vehicle Announcement / Vehicle Identification Response message can be defined by this key.

GID

The content of the attribute GID in a Vehicle Announcement / Vehicle Identification Response message can be defined by this key.

FurtherActionRequired

The content of the attribute Further Action Required in a Vehicle Announcement / Vehicle Identification Response message can be defined by this key.

VINGIDsyncStatus

The content of the attribute VIN/GID sync. Status in a Vehicle Announcement / Vehicle Identification Response message can be defined by this key. This value indicates whether VIN and GID are already synchronized (0) or not (0x10). In latter case the tester will retry Vehicle Identification after T_TCP_Vehicle_Discovery ms.

[VehicleIdentification]

Key

Description

IPv6MulticastAddress

Defines the IPv6 multicast address used in a Vehicle Identification Request.
