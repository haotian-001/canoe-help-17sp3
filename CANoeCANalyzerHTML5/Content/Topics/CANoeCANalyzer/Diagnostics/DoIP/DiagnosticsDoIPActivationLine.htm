Activation Line

 

Starting with CANoe 9.0 SP3, the DoIP Activation Line can be activated automatically at the start of measurement , and deactivated at the end of measurement. The Activation Line is controlled by a writable system variable of type long that has the possible values 0 (inactive) and 1 (active).

Suitable system variables you can configure automatically for Vector I/O hardware from the Vector I/O Configuration dialog, e.g. the standard DOUT port of a VN1630 or VN1640 interface.

If such a system variable is found in the configuration, you can select it on page DoIP/HSFZ Main Settings. At the start of measurement it will be set to 1, at the end of measurement it will be set to 0.

When the Activation Line is configured and a diagnostic request is sent from the DCW or FMW before the configured delay has passed after the start of measurement, the following error is printed in the control:
DoIP Activation Line startup time did not pass yet - try again later.

From CAPL, the configured system variable and delay can be retrieved with the function DiagGetCommParameter and the following parameters:

  • DoIP.ActivationLineStartuptime
  • DoIP.ActivationLineSysvar
  • DoIP.ActivationLineSysvarNamespace

In a CAPL test module the function TestWaitForDoIPActivationLineStartup can be called to make sure that the configured delay has passed.
