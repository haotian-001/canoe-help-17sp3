Diagnostics over Internet Protocol (DoIP)

 

Note

  • The Diagnostic over IP/HSFZ Diagnostic Access (as known as legacy DoIP access) is available due to compatibility reasons with CANoe versions up to and including 8.0.

    With this Diagnostic Access only Ethernet network adapters of the user computer can be used, accessing the VN5600 network interfaces is not possible.

    Additionally, only communication of the diagnostic layer is visible in the Trace Window, which makes debugging more difficult.

  • With Option Ethernet, Eth channels should be used instead in order to benefit from the full DoIP/HSFZ functionality.

    Only by using the Eth channels it is possible to use VN5600 network interfaces and to observe the communication on all three layers (Ethernet packets, DoIP-PDUs and diagnostic messages) in the Trace Window.

  • The Diagnostic over IP/HSFZ Diagnostic Access is deprecated and therefore should no longer be used.

    Vector reserves the right to remove this access in a future version from CANoe.

Cross Reference

The current DoIP modeling library DoIP.vmodule in CANoe supports also HSFZ (High-Speed-Fahrzeugzugang). You can switch between both types of Ethernet diagnostic communication. Use therefore the list in the Diagnostics/ISO TP Configuration dialog.

Constant improvements in ECU functionality and memory capacity have resulted in the need for a higher capacity network connection for diagnostics and ECU flashing than what has been provided up to now by e.g. CAN-based diagnostic connections. Diagnostics over Internet Protocol (DoIP), conforming to the ISO 13400 standard, can now be used for this purpose with ECUs.

When it is used, the diagnostic tester is typically connected to an ECU or a diagnostic gateway via an Ethernet interface, which enables it to quickly exchange large amounts of data with ECUs when e.g. reading out the fault memory state including environment data or when running flash processes.

The DoIP function is available free of further cost with every CANoe pro installation from version 7.1 SP4 or CANalyzer pro installation since version 7.6.SP3, and supports the following basic use cases:

  • Diagnostics with a real DoIP-capable ECU
  • Diagnostics with a real ECU network, e.g. via a DoIP/CAN gateway
  • Simulation of DoIP ECUs or DoIP/network gateways using the DoIP modeling library (DLL file).

DoIP support in includes the following features:

  • DoIP diagnostic channel:
    When using diagnostic description files (CDD, ODX/PDX, MDX), CANoe makes a diagnostic channel available e.g. to CAPL or XML testers whose communication and diagnostic parameters are populated manually or with data from the description files.
  • DoIP modeling library (DoIP.vmodule) gateways:
    The DoIP modeling library (*.vmodule), which is used to simulate DoIP ECUs and gateways, is located in the Exec64 folder of the CANoe installation.
    The HSFZ (/DoIP) UDS Simulation sample configuration gives an example of its implementation in a gateway node.
  • Support for the Diagnostic Console and the Fault Memory Window
  • Diagnostic configuration on DoIP-capable ECUs.
  • Diagnostics on an ECU network via a DoIP gateway / simple switching of an entire network via DoIP
  • Observation of diagnostic objects in the Trace Window using specified diagnostic description files (CDD, ODX/PDX, MDX).

Notes on Connecting the ECU to CANoe

To run diagnostics on a real ECU/gateway with CANoe using DoIP, you have the possibility to connect it to a Vector Ethernet-Interface, like VN5610, or to an interface of your computer. Open the Diagnostics/ISO TP Configuration dialog (Diagnostics & XCP ribbon tab). There you select via Diagnostics over IP the interface you want to use: Either under ECU qualifier|DoIP Settings|Adapter or under Gateway to <Network>|Adapter.

With CANoe.Ethernet option you have the additional possibility to assign a diagnostic description to an Ethernet Network. In this case the used interface is listed (Hardware ribbon tab|Network Hardware|Ethernet).

Attention: Next to errors caused by badly configured firewall settings, one main disadvantage of using the interface of your computer is the missing hardware synchronization. Cause of that you have inexact time stamps, especially if you work with different networks.

Depending on the hardware, you'll need to use either a standard Ethernet patch cable (e.g. when using a switch) or a crossover cable.

Notes on Firewall Settings

DoIP communication uses the TCP and UDP protocols.
If there are problems with DoIP communications, these may be caused by a firewall installed locally on your computer. If the firewall is blocking DoIP packets, this may cause anything from errors to a complete failure of DoIP communications.
You or your IT team will need to make sure (e.g. by adjusting the firewall rules) that the firewall is not blocking the required TCP or UDP ports.
Which TCP/UDP ports and network addresses need to be un-blocked depends on your configuration; you will have entered these in the diagnostic configuration for each network and/or ECU.
