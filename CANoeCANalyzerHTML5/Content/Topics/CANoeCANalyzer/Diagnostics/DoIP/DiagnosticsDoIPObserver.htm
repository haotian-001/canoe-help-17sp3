DoIP/HSFZ Main Settings

Ribbon: Diagnostics & XCP Tab|Diagnostics/ISO TP Configuration|Additional Diagnostic Settings|DoIP/HSFZ Main Settings

DoIP/HSFZ Behavior

Protocol

The protocol version is used in the header of DoIP PDUs. Via this list you select a protocol version for Diagnostics.

  • Diagnostics over IP (DoIP)
  • Interpretation according to HSFZ (High-Speed-Fahrzeugzugang)

Activate DoIP/HSFZ interpretation

If you activate this check box DoIP and HSFZ messages are interpreted according to the selected diagnostic description. Recognized service primitives are displayed in the Trace Window.

Send 0xFF...

If you activate this check box, the DoIP tester will send 0xFF as protocol version in Vehicle Identification diagnostic requests and accept any DoIP protocol version supported by CANoe in the response. This setting is not relevant for HSFZ.

Activate legacy DoIP access

If you activate this check box, you can use the legacy Diagnostic over IP (DoIP) access in your configuration. The DoIP access is deprecated, but can be used in case the option .Ethernet is not available.

Note

It is not recommended to use the legacy DoIP access in parallel with DoIP/HSFZ on Ethernet channels.

Note

  • The Diagnostic over IP/HSFZ Diagnostic Access (as known as legacy DoIP access) is available due to compatibility reasons with CANoe versions up to and including 8.0.

    With this Diagnostic Access only Ethernet network adapters of the user computer can be used, accessing the VN5600 network interfaces is not possible.

    Additionally, only communication of the diagnostic layer is visible in the Trace Window, which makes debugging more difficult.

  • With Option Ethernet, Eth channels should be used instead in order to benefit from the full DoIP/HSFZ functionality.

    Only by using the Eth channels it is possible to use VN5600 network interfaces and to observe the communication on all three layers (Ethernet packets, DoIP-PDUs and diagnostic messages) in the Trace Window.

  • The Diagnostic over IP/HSFZ Diagnostic Access is deprecated and therefore should no longer be used.

    Vector reserves the right to remove this access in a future version from CANoe.

Activation Line

The Activation Line can only be configured when suitable system variables are found in the configuration. See Activation Line (DoIP).

I/O Port

Via this list you select the system variable that shall be used to control the Activation Line. Alternately you select the No activation line used command.

Communication delay after activation: X ms

Time that shall be waited after setting the system variable at the start of measurement before CANoe tries to connect to the DoIP entity.

Note

CANoe enforces this delay only when the diagnostic windows (Diagnostic Console, Fault Memory Window and Diagnostic Session Control) are used. From CAPL it is possible to ignore this time. It can be retrieved in CAPL using DiagGetCommParameter, so a program can honor the delay, if necessary.
