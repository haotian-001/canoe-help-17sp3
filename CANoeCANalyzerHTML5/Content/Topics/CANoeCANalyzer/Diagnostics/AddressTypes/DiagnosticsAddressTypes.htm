Addressing Types

 

Addressing a ECU — 1:1 Relationship

With physical addressing (physical request), a control unit is addressed by means of a CAN identifier sent by the tester. In other words, there is a 1:1 relationship between the tester and the control unit. The control unit that serves as receiver is known.

Addressing Multiple ECUs — 1:n Relationship

CANoe enables multiple control units in a network to be diagnosed at once. The receiver does not have to be known.

An entire network with multiple diagnostic descriptions (CDD, ODX/PDX, MDX, Basic Diagnostics) can be accessed using Functional group requests (functional addressing).

Functional Group Requests

In CANoe a diagnostic description can act as a functional diagnostic description. This means that a diagnostic description specifies certain services that can be received by multiple control units at the same (functional) ID and can be evaluated similarly (functional group).

If diagnostic services are sent on the basis of a functional diagnostic description (functional network requests), a single request for all control units is sent to the network using the functional ID stored in the diagnostic description. All control units answer to this functional ID.

The network nodes do not have to be configured using their own (specific) diagnostic description. The responses of the individual control units are interpreted with this specific diagnostic description, if present. If such a description does not exist for a control unit that sends a response, the functional diagnostic description is used for interpretation.

Typically, the requests here are for:

  • Session handling
    e.g., switch all control units in the network to normal/default session diagnostic mode, or
  • TesterPresent
    The tester holds the control units in diagnostic mode by a cyclic message, even if no other request from the tester is otherwise received.

These are available in an additional Diagnostic Console and a further Fault Memory Window. They can also be used in CAPL.
