ISO TP Observer Use Case

 

  1. To allow use of the ISO TP observer to analyze messages in the Trace Window you can
  • Assign databases to your configuration that define the ISO TP typical attributes for the network or the network nodes,
  • Assign diagnostic description files (CDD, ODX), and
  • Configure additional TP settings manually (see menu path Diagnostics/ISO TP Configuration...|Additional Diagnostic Settings|ISO TP Observer).

The TP parameters of all the above-mentioned setting options for the interpretation of TP traffic are used ("set union") as long as they are not contradictory.

Note

If the different configuration methods specify conflicting settings for TP parameters within the same network (e.g. different sending and receiving CAN IDs for the same network node in the database and the diagnostic description files) this can lead to unpredictable interpretation results.

When using both databases and diagnostic description files as part of the same configuration, please take particular care to ensure that the corresponding TP parameters are compatible.

The most important attributes in connection with the database approach are:

  • TpAddressingMode
  • TpTxIdentifier
  • TpRxIdentifier
  • TpBaseAddress or TpNodeBaseAddress
  • Diagnostic station address

Note

For a detailed description of the attributes and applicable attribute values, please refer to CANdb++|Vector Tool Chain Attributes|Transport Protocol (TP) and Diagnostic Attributes and the ISO TP documentation (OSEK TP).
  1. To activate the interpretation in the Trace Window, select Activate ISO TP observer, enable Diagnostic Interpretation in the dialog Diagnostics/ISO TP Configuration...|Additional Diagnostic Settings|ISO TP Observer.
  2. The messages in the network are then interpreted in the Trace Window according to the specified databases or diagnostic descriptions.

Example

For a sample configuration that demonstrates the usage of the ISO TP observer (pure TP traffic, without diagnostics) please refer to OSEK TP Multi Channel. The sample database OSEKTP_Demo.dbc contains TP attributes for nodes NodeA to NodeD and for the network itself. TP connections are established between pairs of nodes (NodeA/NodeB and NodeC/NodeD). The value of the TpTxIdentifier attribute for one communication partner corresponds to the TpRxIdentifier attribute for the other partner, and vice versa.

Example

The interpretation of diagnostic messages in the Trace Window based on assigned diagnostic descriptions is demonstrated in UDS Diagnostic System Configuration.
