ISO TP Observer - Advanced

Ribbon: Diagnostics & XCP Tab|Diagnostics/ISO TP Configuration|Additional Diagnostic Settings|ISO TP Advanced

With the check boxes in this dialog you can select the messages that shall be displayed in the Trace Window:

Messages displayed in the Trace Window

  • Survey compliance of STmin (minimal separation time)
    A warning is issued when a Consecutive Frame (CF) is seen earlier than STmin after its predecessor. The first CF after the First Frame is not checked.
  • Expect offset STMin before the first CF (consecutive frame)
    A warning is issued when the first Consecutive Frame is seen earlier than STmin after the Flow Control frame from the receiver.
  • Supervise sequence number series
    A warning is issued when the sequence number in the Consecutive Frames (CF) is not correct. The first CF must have the sequence number configured in the observer INI file (cf. FirstSN), and no value may be repeated or skipped.
  • Report frames unexpected according to protocol
    A warning is issued when the type of a protocol frame is unexpected at that time during a data transmission. For example:
    • A Consecutive Frame (CF) without First Frame (FF).
    • A FF or Single Frame (SF) while a transmission waits for a CF or Flow Control (FC).
    • The protocol information in the frame is not known.
  • Report frames sent by a wrong sender
    A frame sent to the target node of a transmission is not sent by the source node of that transmission.
  • Warn on nested transmissions
    While a transmission is running, a frame is observed that seems to belong to a different transmission between the same transport protocol nodes. For example, a Single Frame is received during a segmented transfer.
  • Check for padding value
    Here you can specify which padding value is expected by the observer for not used bytes in connection with the transport protocol.
  • Show the expiration of ISO TP protocol timers in the Trace Window
    A warning is issued when a protocol timer expires. For example, if no Consecutive Frame (CF) is sent after a Flow Control Clear To Send (FC.CTS) frame from the receiver.

Additional Settings from the ISO TP Configuration File

Additionally, settings that are manufacturer specific and therefore read from the observer INI file are displayed in the area on the bottom of the dialog:

  • Blocksize
    • Default: 5
    • Use this Consecutive Frame block size when the evaluation of Flow Control frames has been deactivated, or no Flow Control frames shall be used. After this many Consecutive Frames from the sender another Flow Control frame will be expected.
  • EvalFC
    • Default: 1 (active)
    • Are Flow Control frames considered at all?
  • STMin
    • Default: 20
    • STmin expected for Consecutive Frames. If Flow Control frames are evaluated (default), the value from Clear To Send frames will be used, not this one.
  • WFTMax
    • Default: 5
    • Issue warnings when more Flow Control Wait frames (FC.WT) are sent by the receiver of a transmission.
  • ZeroPaddingValue
    • If padding is checked, the padding bytes must have this value.
  • EvalAllFC
    • Default: 0
    • If 0, STmin and Block Size are taken only from the Flow Control Clear To Send (FC.CTS) frame after the First Frame, otherwise these values are updated with every FC.CTS.
  • FirstSN
    • Default: 1
    • The first Consecutive Frame after the First Frame is expected to have this sequence number. If a different value is received, a warning is issued.
  • TpUseFC
    • Default: 1
    • Is the usage of Flow Control frames expected?
      For example, a warning is issued when a Consecutive Frame is sent without previous Flow Control Clear To Send frame, unless this value is set to 0.
  • ZeroPadding
    • Default: 0
    • If non-0, protocol frames will be checked for the values of their padding bytes. A warning is issued when the bytes are not equal to the ZeroPaddingValue.
