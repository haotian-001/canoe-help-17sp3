ISO TP Observer

 

The ISO TP observer interprets messages transmitted on the CAN bus in accordance with ISO Transport Protocol ISO/TF2 (ISO/DIS 15765-2 (2001-11-01)) and with any diagnostic descriptions (CDD, ODX/PDX, MDX) provided, displaying the interpreted message contents such as:

  • Sender and receiver addresses of TP messages
  • TP protocol information for each CAN message (First Frame, Consecutive Frame, Flow Control, etc.)
  • IDs and name/qualifier of the diagnostics services
  • Names/qualifiers/values of diagnostic parameters

in the Trace Window.

The ISO TP observer is an absolute prerequisite for diagnosis interpretation

Interaction between the TP Observer and the Diagnostic Interpretation Settings

The observer is configured from the following sources:

  • Settings in the dialog Diagnostics/ISO TP Configuration: Transport Layer

The values specified here are used for the interpretation for the corresponding diagnostic description file.

Note

  • The values in the diagnostic description file itself remain unchanged regardless of any manual modifications made in this dialog and its pages.

    You can restore the default values stored in the diagnostic description file to the dialog at any time by clicking the [Get Defaults] button.

  • Only if the parameters set here fit the actual transmission can the diagnosis data be interpreted!
  • From the assigned databases (.dbc files) the settings for the defined nodes are taken over (there must be no nodes in the Simulation Setup!), whereby the attributes for as for the OSEK_TP.DLL are interpreted.
  • If none of these settings fits the CAN message, it is checked for additional TP parameters insofar as these parameters were activated.
