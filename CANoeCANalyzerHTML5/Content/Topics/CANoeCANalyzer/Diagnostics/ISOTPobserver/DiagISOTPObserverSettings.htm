ISO TP Observer Settings

Ribbon: Diagnostics & XCP Tab|Diagnostics/ISO TP Configuration|Additional Diagnostic Settings|ISO TP Observer

The ISO TP Observer interprets messages sent over the CAN bus according to the ISO transport protocol ISO/TF2 (ISO/DIS 15765-2 (2001-11-01)) and displays the results in the Data columns of the Trace Window; see also CAPL Filter in the Measurement Setup.

Via the Activate ISO TP observer, enable diagnostics interpretation check box you can activate the ISO TP Observer and thus enable diagnostic interpretation.

Note

Since CANoe v5.2 only the integrated Observer is shipped.

It completely replaces the Hook DLL that has formerly been shipped.

Performance

Activate ISO TP observer, enable diagnostic interpretation

If you don't need diagnostic interpretation deactivate this check box to save processor capacity. In this case no diagnostic observer events are visible in the Trace Window.

Detection of TP Messages

If the TP is not configured sufficiently in the database you can use additional parameters. If you activate Use additional TP parameters, you can select one of the following parameters:

  • No Additional Detection
    Except the CAN identifiers that are defined in the used diagnostic descriptions no other identifiers are used to recognize TP messages.
  • Normal
    Here you can additionally edit a Base address and an Rx mask.
  • Extended
    Here you can additionally edit a Base address and an Rx mask.
  • Normal Fixed or Mixed
    With this parameter no further settings are available.

Interpretation of TP Messages

Interpretation according to KWP2000 standard, if no CDD matches.

The ISO TP is most commonly used for diagnostics. For the definition of the data exchanged between a client (usually a tester) and one or more servers (usually ECUs), the Keyword Protocol 2000 (KWP2000) (cf. ISO 15765-3, ISO 14230-3) was defined. CANoe can interpret data transferred via the ISO TP according to this standard, i.e. the name of the service primitive and its mnemonic will be displayed.

Note

Please note that most manufacturers use diagnostic descriptions that differ from the standard!

The KWP2000 interpretation will only use the definitions in the standard, i.e. not a full interpretation of the diagnostics is possible.

In order to start the KWP2000 interpretation of transported data, it is sufficient to activate the check box Interpretation according to KWP2000 standard, if no CDD matches if the TP parameters settings for the observer are correct.

If the check box is activated, it will be tried to interpret the transmitted service primitives even if there is no diagnostic description available. Note that if data can be interpreted using a description, it will not be tried to interpret it according to KWP2000.
