Default Functions to Configure Diagnostic Descriptions

If you click in the tree view an element on level 3 the following administration functions are displayed on the right above the table as buttons. (These Functions are also provided when you put a right click on an element on level 3 and open the corresponding context menu):

Remove (Diagnostic Description)

Remove diagnostic description form the ECU or the element in the tree view.

Duplicate (Diagnostic Description)

Copies the diagnostic description and inserts it at the same position in the tree view as a duplication.

Note

This function makes it easy to assign the same diagnostic description file to multiple ECUs that support identical diagnostic services. To prevent naming conflicts, a new identifier is assigned automatically, which you can then modify. The communication parameters are copied from the original diagnostic description. They must be modified manually each time in the Diagnostic Description Settings|Protocol Settings dialog to prevent conflicts (e.g., the CAN ID used for physical requests to an ECU cannot be used/configured more than once on the same network.)

This function is not available for OBD-II ECUs.

Generate (Diagnostic Description)

Generates the diagnostic description.

Note

Like Duplicate, this function makes it possible to assign the same diagnostic description file to several ECUs on a network. In this case, however, the duplicates are created using the CANdesc diagnostic attributes and/or transport protocol attributes (TP) defined for the network nodes in the database and automatically assigned to the diagnostic configurations/list entries. Manual adjustment of the communication parameters is no longer necessary. For each network node that is found, whose CANdesc diagnostic attributes are not yet used or whose TP parameters are not already contained in the list, a new list entry with the network node's name is created. Once they are generated, these new list entries can be modified or removed manually, as is the case with other existing entries.

This function is not available for OBD-II ECUs.

Important Note

Changes to databases (*.dbc) – e.g. addition of new nodes or modification of node attributes already inserted into a diagnostic configuration via generation – are not automatically replicated in the diagnostics configuration.

In this case, you will need to remove the diagnostic descriptions for the modified ECUs and generate them again.

Open (Diagnostic Description)

Open diagnostic description in the globally configured application for this type of diagnostic description (e.g. CANdelaStudio for CDD, ODXStudio for PDX/ODX).
