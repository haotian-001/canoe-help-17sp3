ODX/PDX Diagnostic Descriptions

Ribbon: Diagnostics & XCP Tab|Diagnostics/ISO TP Configuration|Administrate Diagnostic Descriptions|ODX/PDX Diagnostic Descriptions

Diagnostic descriptions in ODX/PDX (Open Diagnostic Data Exchange/Packed ODX) format can be used as of CANoe version 6.0.

Note for ODX versions

  • ODX 2.0.1 has been supported since CANoe 6.0.
  • ODX 2.2 has been supported since CANoe 7.1.

All of the functions developed for CANdela descriptions are available; however, the following ODX format properties should be taken into account:

  • ODX diagnostic descriptions can be spread over several files. For this reason, it is important to specify all of the required files, including, in particular, definition of communication parameters.
  • An index file simplifies administration of these files. This index file is part of PDX packages (zip archives). A PDX package can be created according to the procedure described here.
  • An ODX/PDX diagnostic description file can define the diagnostic functionality of several ECUs (but not "variants"). For this reason, the ECUs to be used must be specified explicitly.
  • The contents of the ODX/PDX diagnostic description are converted to the CANdela schematic. This makes it possible to display diagnostic classes and instances, making the diagnostic services easier to use.
  • If several responses are defined for one request, only one is selected (for interpretation or in ECU simulations).
  • Because the ODX/PDX diagnostic description functionality is still being developed, CANoe does not support all features yet. Please visit the Vector Informatik Web site to get the latest support information.

Please note that ODX/PDX diagnostic description files can be quite large. As a result, they may take a long time to read.

If more than one ECU are found in the selected ODX/PDX diagnostic description the Select ECU from diagnostic description dialog appears. In this dialog these ECUs are listed. Select the ECUs you want to add to your diagnostic configuration.
