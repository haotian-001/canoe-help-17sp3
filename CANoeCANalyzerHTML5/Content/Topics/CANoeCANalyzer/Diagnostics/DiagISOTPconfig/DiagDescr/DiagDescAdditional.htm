Settings Additional Descriptions

Ribbon: Diagnostics & XCP Tab|Diagnostics/ISO TP Configuration|Administrate Diagnostic Descriptions|Assignments|Additional Descriptions

An Additional Diagnostic Description is a diagnostics definition that is used to extend a so called Master Description. This can be used to add diagnostics services to a diagnostic description that cannot or shall not be changed, e.g. because it is provided by an OEM.

The Additional Diagnostic Descriptions share the communication channel of the Master Description and therefore no additional parameterization is required.

The Additional Diagnostic Descriptions can be used in the diagnostic windows (Diagnostic Console, Fault Memory Window and Diagnostic Session Control) as well as in CAPL and for interpretation of diagnostics messages by the Diagnostics Observer in the Trace Window.

Add Additional Descriptions

Via the Add Additional Descriptions list you can add Additional Diagnostic Descriptions to an existing Master Description. Like for the Master Description, you may choose from:

  • Diagnostic Description (CDD, ODX/PDX, MDX)
  • Basic Diagnostic Description (UDS)
  • Basic Diagnostic Description (KWP)
  • Standard Diagnostic Description (CDD)

In the table you will see the Master Description and the Additional Diagnostic Descriptions attached to it. The descriptions are tried in top-down order when the data of a diagnostics event is interpreted. You can change the order of the descriptions in the list via drag and drop.

The settings for the Master Description cannot be changed in the table; you may change them in the Transport Layer and Diagnostic Layer pages, if the description allows this.

If a diagnostic description added as Additional Diagnostic Description contains several variants, language definitions or target groups to choose from, you can specify these in the table.

Note that in CAPL, you can address the Additional Diagnostic Descriptions via DiagSetTarget() or DiagInitEcuSimulation() by the ECU qualifier <ECU qualifier of „Master“ description>.<ECU qualifier of Additional Description>, e.g. ECU1.AdditionalDescription1.

This is also the name used in the Diagnostic Console, Fault Memory Window and Diagnostic Session Control.

Buttons above the Table

If you click in the tree view on Additional Descriptions the following administration functions are displayed on the right above the table as buttons:

Remove (Diagnostic Description)

Remove diagnostic description form the ECU or the element in the tree view.

Note that you cannot remove the Master Diagnostic Description here; to do so, select the respective network overview page in this dialog and remove it there. Note that removing a Master Diagnostic Description will remove all related Additional Diagnostic Descriptions as well.

Open (Diagnostic Description)

Open diagnostic description in the globally configured application for this type of diagnostic description (e.g. CANdelaStudio for CDD, ODXStudio for PDX/ODX).
