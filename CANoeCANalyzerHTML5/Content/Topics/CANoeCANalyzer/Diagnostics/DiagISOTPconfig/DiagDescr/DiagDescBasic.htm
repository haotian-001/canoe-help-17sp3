Basic Diagnostic Description with UDS, KWP

Ribbon: Diagnostics & XCP Tab|Diagnostics/ISO TP Configuration|Administrate Diagnostic Descriptions|Basic Diagnostic Description with UDS, KWP

If you click in the Diagnostics/ISO TP Configuration dialog in the tree view in level 3 on a basic diagnostic description you see this sub dialog:

Cross Reference

Default Functions to Configure Diagnostic Description

In this dialog you can configure the following settings for the selected basic diagnostic ECU:

Diagnostic Description

ECU Qualifier

Here the ECU qualifier is displayed which will be used by CANoe to refer to the current diagnostic description. The qualifiers for each diagnostic description configured in a CANoe configuration must be unique.

When a description file is added, as default the ECU qualifier as defined in the description will be used. Yet it is also possible to manually overwrite this qualifier here. The qualifier name may only contain letters A…Z and a…z, numbers and underscores _.

Furthermore it is ensured that unique qualifiers are assigned to all ECUs.

Note

In particular this has effects on CAPL programs.

When you configure the target ECU (DiagSetTarget or DiagInitECUSimulation) you have to specify the correct qualifier!

It is also the name used in the Diagnostic Console, Diagnostic Session Control the Fault Memory Window and the Symbol Explorer.

Available Options

Interface

Here you can configure the interface/diagnostic access which will be used for communication with the basic diagnostic ECU.

Basic diagnostic ECUs can use interfaces on CAN, LIN, FlexRay, Ethernet and K-Line. When choosing LIN or FlexRay, you also must add LDF or FIBEX databases to the configuration as additional addressing parameters are read from the network nodes defined in those databases.
