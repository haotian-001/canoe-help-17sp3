Administrate Diagnostic Descriptions

Ribbon: Diagnostics & XCP Tab|Diagnostics/ISO TP Configuration|Administrate Diagnostic Descriptions

Functions Level 2

If you click in the Diagnostics/ISO TP Configuration dialog in the tree view an element on level 2 you see this sub dialog:

This sub dialog is divided in three areas:

  • Diagnostic Descriptions
    Here you can administrate diagnostic descriptions, i.e.. add, remove, open, ... More detailed descriptions you find below.
  • On-board Diagnostics (OBD-II)
    Via this list you select the following settings for OBD:
  • No OBD-II functionality
  • Use 11 bit (Normal) addressing
  • Use 29 bit (NormalFixed) addressing
  • Gateway Settings
    Enable diagnostics via common DoIP/HSFZ gateway for this network check box.

Note

  • The Diagnostic over IP/HSFZ Diagnostic Access (as known as legacy DoIP access) is available due to compatibility reasons with CANoe versions up to and including 8.0.

    With this Diagnostic Access only Ethernet network adapters of the user computer can be used, accessing the VN5600 network interfaces is not possible.

    Additionally, only communication of the diagnostic layer is visible in the Trace Window, which makes debugging more difficult.

  • With Option Ethernet, Eth channels should be used instead in order to benefit from the full DoIP/HSFZ functionality.

    Only by using the Eth channels it is possible to use VN5600 network interfaces and to observe the communication on all three layers (Ethernet packets, DoIP-PDUs and diagnostic messages) in the Trace Window.

  • The Diagnostic over IP/HSFZ Diagnostic Access is deprecated and therefore should no longer be used.

    Vector reserves the right to remove this access in a future version from CANoe.

Enable diagnostics via common DoIP/HSFZ gateway for this network

Use this option to activate Diagnostics over IP centrally for the entire network. This function provides a simple way of modifying existing configurations to support Diagnostics over IP. All you need to do is add a corresponding DoIP gateway to the configuration, which then forwards the diagnostic requests and corresponding responses to the respective ECUs and testers. Additionally you have to assign a logical DoIP address to each ECU.

Example

A sample configuration for a CAN network with a corresponding DoIP gateway is included in the installation.

Add Diagnostic Description

Via the Add Diagnostic Descriptions list or the context menu of an element on level 2 in the tree view you can add different diagnostic descriptions:

  • Diagnostic Description (CDD, ODX/PDX, MDX)
  • Basic Diagnostic Description (UDS)
  • Basic Diagnostic Description (KWP)
  • Standard Diagnostic Description (CDD)

Loading a diagnostic description may take some time depending on the size. While loading the ECU address in the description is analyzed. If you want to assign a description twice in one single network using the same addressing parameters, you can add it as additional description.

Open, Configure, Remove Diagnostic Descriptions

Table of Diagnostic Descriptions

If you click in the tree view an element on level 2, in the table on the right all loaded diagnostic descriptions are listed with their current configurations. The following columns are displayed:

  • ECU Qualifier
    ECU qualifier which will be used by CANoe to refer to the current diagnostic description.
  • Usage
    See: Usage of the Diagnostic Description
  • Origin
    Filename of the diagnostic description file (without path and file type).
  • Settings
    The most important TP parameters for the diagnostic channel are:
    Addressing mode, CAN ID for diagnostic requests to the ECU, CAN ID for diagnostic responses from the ECU.

Buttons above the Table

If you click in the tree view an element on level 2 and select an diagnostic description in the table the following administration functions are displayed on the right above the table as buttons:

Remove (Diagnostic Description)

Remove diagnostic description form the ECU or the element in the tree view.

Open (Diagnostic Description)

Open diagnostic description in the globally configured application for this type of diagnostic description (e.g. CANdelaStudio for CDD, ODXStudio for PDX/ODX).

Configure Diagnostic Description

With a double click on a diagnostic description in the table on the right you open the corresponding sub dialog for configuration. That way you automatically switch on the corresponding subordinated element in the tree view:

Depending on the type of diagnostic description there are differences that are described on the corresponding help topics:

  • Diagnostic Description
    with CDD, ODX/PDX, MDX
  • Basic Diagnostic Description
    with UDS, KWP
  • Standard Diagnostic Description
    with reduced CDD
