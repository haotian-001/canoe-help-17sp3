FlexRay TP Observer

Ribbon: Diagnostics & XCP Tab|Diagnostics/ISO TP Configuration|Additional Diagnostic Settings|FlexRay TP Observer

The FlexRay TP observer interprets messages sent over the FlexRay bus according to a specific Transport Protocol (TP) and diagnostic description and displays the results in the columns of the Trace Window.

Currently the following transport protocols are supported:

  • ISO 10681-2 FlexRay Transport Protocol STD Version 2.2 2008-11-28
  • AUTOSAR FlexRay Transport Layer V2.1.1 R2.1 Rev 0014
  • BMW FlexRay Transport Protocol (2007-04-16 FlexRay_LH_Part3_SW)

Note

In order to be able to select the appropriate protocol variant, the appropriate FlexRay package (AUTOSAR FlexRay Support Package or BMW FlexRay TP Support Package) must be installed!
If none is installed, then only the ISO 10681-2 FlexRay TP can be interpreted.

Performance

Activate FlexRay TP Observer, enable diagnostic interpretation

Via this check box you can activate the FlexRay TP Observer and thus enable diagnostic interpretation in general for FlexRay.

Note

Only frames that are marked in the FIBEX database to be of any Diagnostics or TPL type are considered.

Use diagnostic settings instead of TP settings, if possible

If the TP is not configured sufficiently by the diagnostic description, you can define additional parameters here. If you activate this check box, these settings will be overwritten by the appropriate settings of the diagnostic description.

Detection of TP Messages

  • Protocol type
    If both TP packages are installed you can select here the used variant.
  • Connection mode
    With this parameter you select the used protocol sub variant.
  • Message length
    With this parameter you select the maximum message length in one segmented transfer.
  • Address format
    Here you can select 1 byte or 2 byte addresses that are used inside the protocol control information.
