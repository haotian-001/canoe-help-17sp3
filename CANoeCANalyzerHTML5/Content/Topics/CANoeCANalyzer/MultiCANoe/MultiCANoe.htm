Overview

Multi CANoe is a special operation mode of CANoe. This mode allows you to exchange data and to operate multiple CANoe instances synchronized on the same computer or on different computers. The individual instances are independent of each other and have their own configuration and realtime area. The measurement is started synchronously in all instances. All network interfaces are connected via the HW-Sync line.

With Multi CANoe you can extend the performance of CANoe to meet the requirements for very large test systems.

Functionality [â² back]

An Multi CANoe configuration (= Multi CANoe cluster), consists of a master and one or more slaves. The master is the clock for time synchronization and ensures data exchange between all CANoe instances (= participants) of the Multi CANoe cluster.

Multi CANoe works only in the Real Bus working mode.

Time Synchronization

Almost all Vector VN interfaces allow a hardware based time synchronization via a common line. One interface sends a pulse which is recognized almost immediately by all connected interfaces. The more interfaces have to be synchronized, the longer the transmission takes and the weaker the edge of the pulse becomes. With a Multi SYNCbox Active you can amplify the input signal and forward it to up to 10 active outputs. This creates a constant delay between the input and output signals. You can compensate this delay by measuring it and entering it into the CAN.ini of the master.

Data Exchange

Basically the configurations of the single nodes run independently. For time synchronization, the time stamp of the reference channel of the master is forwarded to all slaves in addition to the sync pulse.

Data can be exchanged between the instances via defined system variables. For this purpose, an FDX connection via Ethernet is automatically established in the background.

Measurement Start

The measurement start of a Multi CANoe cluster is synchronous. As soon as you start the measurement in a participant, a request is sent to the master and the master sends a start command to all slaves. Each slave starts its configuration and reports the status back to the master. In this phase the hardware is already addressed, but no received events are processed by CANoe yet. As soon as all slaves have reported the measurement start to the master, the master sends a pulse via the HW-Sync line. This pulse is the time 0 for all instances. From this time on, the received events are processed.

Messages

If you have activated Multi CANoe in the Options dialog, the status bar displays the text Multi CANoe, the role of the current instance ([M]aster or [S]lave) and the state of the cluster:

Incomplete

Not all slaves of the cluster have connected to the master. The measurement cannot be started.

Invalid

At least one participant is invalid. Reasons are a wrong CANoe version of the participant or a wrong work mode. Multi CANoe can only be used with the Real Bus work mode. The measurement cannot be started.

Connected

All slaves are connected to the master, the measurement can be started.

Starting

One slave has initiated the start of the measurement and the remaining participants start the measurement.

Running

The measurement of the Multi CANoe cluster is in progress.

Stopping

One participant has initiated the stop of the measurement and the remaining participants stop the measurement.

Creating a Multi CANoe Cluster [â² back]

To create a Multi CANoe cluster, you must define the participants of the cluster and, if needed, the system variables to be exchanged

Determine the Participants of the Cluster [â² back]

A participant of the Multi CANoe cluster is described by the values Alias, IP Address and Port. In addition, you specify which participant is the master.

You make these settings in the Options dialog.

Note

The defined participants must be identical in all participating CANoe instances of the Multi CANoe cluster.

Therefore, it is best to define all participants in one CANoe configuration and export this definition to a file. You can use the available import function to read in the definition of the participants in all participating CANoe instances.

The Multi CANoe settings are stored globally in the CAN.ini and are independent of the loaded configuration.

Defining the System Variables [â² back]

Information can be exchanged between participants via system variables. For each system variable to be exchanged, specify which participant may write the system variable.

Note

The system variables to be exchanged must be identical for all participants.

It is therefore best to define all system variables in a CANoe configuration, save the system variables in a file (*.vsysvar) and import them into all participating CANoe instances.

For runtime reasons, it is advisable to keep the quantity of system variables to be exchanged low, since each update of a system variable leads to data exchange between all participants.

Note

Struct Definition with variable layout are not supported as system variables.

Note

With Multi CANoe, you cannot use the Vector Hardware Manager to configure a HW-Sync on the VN Interfaces locally.

If your VN interfaces already have a local HW-Sync setting from a previous project, please remove it. Otherwise, CANoe will not display the Network Hardware Configuration dialog above.

CANoe Instances on the Same Computer

You can run multiple CANoe instances on the same computer, making it possible to perform more complex simulations and take advantage of the computer's full processing power. You must assign a unique FDX port to each instance when running multiple instances.

In order to allow different CANoe instances on one computer to access different Vector VN interfaces, you have to adjust the application name with which the channel assignment is done.

For this you have to start CANoe in the first step via the command prompt to set the application name at program start via command line parameter:

CANoe64.exe /m <ApplicationName>

In the next step you have to adjust the channel assignment in the Vector Hardware Config dialog or the Vector Hardware Manager.

Note

For Ethernet networks in network-based mode, this step is not necessary as the required network names can be configured independently in each CANoe configuration. It's also possible to simulate different simulation nodes for the same Ethernet network, but each instance must simulate a unique set of nodes, and a simulation node cannot be split across different CANoe instances.

Required Version and Licenses [â² back]

For Multi CANoe, the same CANoe version must be installed on all computers (main version, SP version and hotfix numbers must be identical). Otherwise the measurement start will be aborted.

Multi CANoe requires at least

  • minimum one CANoe pro or CANoe run license for the master
  • one CANoe pro or CANoe run license for each option that is used in a configuration
  • one CANoe Slave license for each additional participant
