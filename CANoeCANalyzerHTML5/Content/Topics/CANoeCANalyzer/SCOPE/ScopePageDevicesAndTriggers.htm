Devices and Triggers

 

The Devices and Triggers page on the left side of the Scope Window has a tree view made up as follows:

Scopes
Shows each Scope device.
Use the context menu of the Scopes item to add and delete Scope devices: New scope or Delete all scopes.

Scope <n>: Scope device
The name contains a sequential Scope number and the Scope channels mapping.
Below the configured trigger conditions are displayed.
When you deactivate a Scope device, the corresponding Scope device, along with all its trigger settings, will be ignored during the measurement.
You can use the context menu to open the Scope Hardware Configuration dialog as well as add or delete trigger conditions.

Trigger Condition (CAN Frame/CAN Any Error Type, I/O Trigger and Digital Trigger)
The name consists of the trigger type and a short description of the parameter.
When you deactivate a trigger condition, that trigger condition will be ignored during the measurement.
Use the context menu to add or delete trigger conditions.

  • Configuration
    With trigger conditions that use frame identifiers, for instance, you can select either a specific frame ID or a range of frame IDs.
  • Trigger channels
    Trigger conditions generally apply to all application channels mapped to a Scope device.
    However, you can use the Channels... command in the context menu to select individual application channels. You can only select application channels if they are assigned to this particular Scope device.
  • Consistency alerts
    The consistency of the Scope hardware configuration and trigger conditions is checked automatically, and alerts are issued automatically.
    Alerts appear in the Write Window. In addition, error and warning symbols are shown as tooltip in the configuration tree.
