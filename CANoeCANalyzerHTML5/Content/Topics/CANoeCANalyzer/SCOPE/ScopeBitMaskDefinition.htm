Bitmask Definition

 

Click on (selection via ) of the Eye Diagram's toolbar in the Scope Window to open the Bitmask Definition dialog.

In addition to the eye diagram you can also analyze the eye diagram. To do this, a bitmask must be defined.

Appearance

The appearance of the bitmask can be changed using four parameters. Changes to a data point (tuple) are visible with immediate effect in an eye diagram, which has been created before.

  • Line width/Line color
    Specify the line width and a line color for the bitmask.
  • Symbol/Symbol color
    Specify a symbol and a color for the data points (tuples) of the bitmask.
  • Alpha-Blending/Background color
    Specify the Alpha-Blending for the background color.

Data Point Definition

A bitmask can be defined using 6 data points (tuples) for CAN bus systems and 10 tuples for FlexRay systems.

To add more data points you can edit the XML import file and then import it via the [Import] button.

A tuple is defined as {nominal bit time[%]; voltage[mV]}. Changes to a tuple are visible with immediate effect in an eye diagram, which has been created before.

  • Time [%]
    The nominal bit time (NBT) always depends on the current baudrate for the respective measurements.
    Valid values: 0 ... 100 %
  • Voltage [mV]
    The voltage which must not be fall down or must not be exceeded during nominal bit time.

Example: Bitmask with green lines and red data points (tuples)

Import / Export

All appearance settings and data point definitions you can import and export. Use therefor the corresponding buttons.

Note

Before you can edit the XML file create an XML file via the [Export] button.

To define more complex bitmasks you have to edit data point entries in XML file before import.

Default Values

Via this button you can restores the Vector default settings for a network depended bitmask.
