Hardware Configuration

Devices and Triggers Page|Scope Device|Context Menu|Configuration

In the Scope Hardware Configuration dialog you configure each single scope device, that is connected.

You can also open the dialog with a double click on a Scope device on the Devices and Triggers (Scope) page.

Active Configuration

The user can add, delete, undo and redo custom settings for selected device. When choosing another device type for current setting the active configuration is transferred to selected device. For example, the channel mapping of a 4-channel scope is transferred to the first 4-channels of an 8-channel scope.

Scope

Device Type

Select the Scope hardware type from the list.

When using PicoScope devices of 5000 series you can adjust the voltage measurement range resolution.

Note

  • When using PicoScope 6403E-34 voltage resolution is 8-bit and not adjustable.
  • In addition to the real Scope hardware you can use a virtual Scope in simulation mode and real mode. You can configure it like a real oscilloscope. Currently only the CAN bus system is supported. See also Bitmask Analysis (CAN - Scope) sample configuration.

Voltage Resolution

The resolution of voltage measurement range can be adjusted. The following settings are available: 8 Bit and 12 Bit. If a higher resolution is used, the adjustment of minimum sampling interval will be restricted.

For details see the specification of the PicoTech hardware.

Analog Channel Mapping

This is where you configure the signal sources that are connected to the Scope hardware.

  • Channel
    Name of the Scope channel (corresponds to the name of the Scope hardware).
  • Signal source
    Choose one of the listed signal sources:
  • LIN, CAN, FlexRay , SENT or PSI5 application channel
    The application channels listed here correspond to those configured in the CANoeOptions dialog.
    With CAN, you'll need to differentiate between the Low and High lines.
    With FlexRay you can select channel A or B. You have then to configure both signals (BP and BM).
  • I/O line
  • None
    Nothing connected.
    Measurement data on this channel are not acquired.
  • Label
    User-defined name for the signal source.
  • Probe
    The scaling factor needs to correspond to the value entered here.
  • Range Scope, Range Signal, Tolerance
    In scope hardware dialog the current channel mapping is represented in a grid view. For each channel you can set the voltage range of the input signal.
  • The input range values are represented in the Range signal column.
  • The input voltage range of the scope device can either be adjusted with the internal measurement range of the scope device, represented in column Range Scope or the amplification of the probe represented in column Probe.
  • The input voltage range is automatically validated. Valid ranges are:
  • CAN High-Speed: 5 V
  • LIN: 20 V
  • FlexRay: 5 V
  • SENT: 5 V
  • PSI5: 5 V
  • The measurement tolerance related to the voltage input range is represented in grid column Tolerance.
  • Samples per bit
    Number of samples per bit.
    This value is shown only when the signal source is an application channel. It corresponds to the current data acquisition settings and the baudrate of the application channel.

Digital Ports

Port

Name of digital ports P0 and P1. Each port clusters 8 digital measurement channels.

Threshold

Pre-Setting for level trigger condition. The user can select predefined values from list or is able to edit the thresholds manually. e.g. 1V or 1mV.

Hysteresis

User can select a hysteresis value from list. The hysteresis corresponds to given threshold.

Enabled

Activate or deactivate the entire port.

Digital channels (D0…D7)

Click corresponding row cell for Port 0 or Port 1. Dialog Digital Channel Properties opens. Configure each digital channel.

  • Enabled: Activate or deactivate digital channel on current port.
  • Label: Description for digital channel.

Acquisition Settings

Acquisition time

You can specify the recording time automatically or manually.

An automatically generated value takes the baudrate of the relevant application channels into account and thus is not determined until recording begins.

Minimum samples per bit

The desired minimum number of sampling points per bit. This is a guide value for the selection of a suitable sampling period for each oscilloscope used.

If the guide value for an oscilloscope cannot be complied with, a warning is output on the Devices and Triggers (Scope) page.

Sampling period

You can either specify the sampling period yourself or you can have this done automatically using the minimum number of sampling points per bit.

The number of sampling points is calculated automatically depending on the recording duration and sampling rate. The sampling rate and maximum number of sampling points depend on the oscilloscope type.

Note

To simplify operation, there are configuration options where all recording settings are specified automatically at the start of acquisition:

  • Recording duration: In this case, the recording time is specified using the baudrate of the relevant application channels.
  • Sampling rate: The sampling rate is specified based on the Scope hardware capabilities and the guide value for the sampling period.

Trigger Settings

For the trigger settings, you can choose between a pre-trigger and a minimum post-trigger. The pre- and post-trigger times are set as a percentage in the range of 10 to 90% of the recording duration.

Pre-trigger

With a pre-trigger, the pre-trigger time is defined as a percentage of the total recording duration.

Triggers which occur in the set pre-trigger time are not taken into account. Following expiration of the pre-trigger time, trigger events are recognized. Starting at the trigger point, the measurement data is assigned to the post-trigger phase. In this trigger mode, the measurement data can be uniquely assigned to a phase (see Figure 1).

Figure 1: Trigger event and measurement data acquisition

If the trigger point occurs after the pre-trigger time, data from the pre-trigger phase which has already been recorded is discarded (gray box in Figure 1).

Example

The setting pre-trigger = 10% yields a pre-trigger time of 1 ms with a recording duration of 10 ms. Thus 10% of the measurement data remains in the pre-trigger phase and 90% in the post-trigger phase.

Note

If this trigger is used, it is possible that a Trigger Now may not function. The reason for this is based on the fact that a trigger cannot be initiated during the pre-trigger time. The better choice for a Trigger Now is min. post-trigger.

Min. post-trigger

A minimum post-trigger time is defined for this trigger mode.

In principle, the pre-trigger time and post-trigger time can vary in this mode. Trigger events are recognized during the entire pre-trigger time. A variable post-trigger time which does not undershoot the minimum set value is yielded starting at this point in time (see Figure 2).

Figure 2: Trigger event and measurement data acquisition

Example

The setting min. post-trigger = 10% means that the pre-trigger time can be a maximum of 90% of the recording duration, depending on the occurrence of the trigger event.

Situation 1

If the trigger event occurs before the pre-trigger time expires, the post-trigger phase is extended accordingly. If the trigger event occurs after 4 ms with a recording duration of 10 ms, the post-trigger phase is extended to 6 ms.

Situation 2

If the trigger event occurs after the pre-trigger time expires, the duration of the post-trigger phase does not change. If a trigger event occurs after 9.5 ms with a recording duration of 10 ms, the pre-trigger time is 9 ms and the post-trigger time is 1 ms.

Note

This trigger can be perfectly combined with a Trigger Now.
