Triggers

Prerequisites

To enable triggering, you need

  • a running measurement
  • a connection to a Scope device, and
  • an activated trigger (toolbar: ) to use predefined trigger conditions.

Trigger Types

You can use the following trigger types individually or in combination; the first trigger that occurs "wins":

  • Manual triggering
    Click (Trigger now) in the toolbar to trigger immediately.
  • Trigger on predefined condition
    To trigger in reaction to a bus event, you need a suitable trigger condition (see Devices and Triggers (Scope) page).
    A trigger condition is based on specific event types and event parameters.
    You can create as many trigger conditions as you like, all of which are linked with an OR operator.
  • Trigger via CAPL
    You can use the scopeTriggerNow() CAPL function to trigger via a CAPL program.
    This lets you trigger on e.g. signal or system values.

Trigger Modes

You can switch the trigger mode at any time via the toolbar:

  • Single
    The trigger is not reactivated after it is triggered; all preconfigured trigger conditions are also deactivated.
  • Repeat
    The trigger is automatically reactivated after triggering; all preconfigured trigger conditions remain active.
    The blind time in Repeat mode is at least 100 ms.

Trigger Conditions

You can define trigger conditions for CAN, CAN FD, LIN and FlexRay.

Trigger conditions can be added on the Configuration page. You can select the trigger conditions in the context menu of a device node of a configured scope device.

Trigger Condition Configuration (CAN, CAN FD, LIN)

Trigger conditions for CAN, CAN FD and LIN are configurable in a separate configuration dialog shown below (here described with a CAN trigger condition).

Frame Identifier

  • Frame
    You can define single trigger conditions via their message identifier or their symbolic representation in an attached database.
  • Range
    You can define trigger conditions for message identifier ranges.

Online Filter Settings

  • Frame (only for CAN/CAN FD)
    In this list you can select All, CAN or CAN FD. Depending on what you select the list below shows different commands.
  • Bit fields
    Used to scale the trigger event to defined bit sequence. Adjusted scaling (bit sequence) is used for Online Eye Diagram.

Trigger Channels

You can define the application channels where the defined trigger condition is evaluated on.

Trigger Condition Configuration (FlexRay)

Trigger conditions for FlexRay are configurable in a separate configuration dialog shown below.

Frame Identifier

  • Slot ID
  • Single
    You can define single trigger conditions via their message identifier or their symbolic representation in an attached database.
  • Range
    You can define trigger conditions for message identifier ranges.
  • Base Cycle, Cycle Repetition
    You can define repetitive time slots (Cycle Repetition) from a starting point (Base Cycle) on which a trigger condition is evaluated.
  • Channel Mask
    You can define a channel mask (A, B, AB) where to trigger condition is evaluated on.

Online Filter Settings

  • Bit fields
    Used to scale the trigger event to defined bit sequence. Adjusted scaling (bit sequence) is used for Online Eye Diagram.

Set Trigger on Frame Flags

Sync, Startup, Payload Preamble, Null Frame
You can define special frame flags (Sync Flag, Startup Flag, Payload Preamble, Null Frame) that can have the states 0, 1 or Ignore.

Trigger Channels

You can define the application channels where the defined trigger condition is evaluated on.
