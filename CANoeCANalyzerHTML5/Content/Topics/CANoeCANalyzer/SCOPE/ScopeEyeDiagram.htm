Eye Diagram

 

Analysis Requirements

  • Supported bus systems:
  • CAN and CAN FD (starting with CANoe 9.0)
  • FlexRay (starting with CANoe 9.0 SP2)
  • LIN (starting with CANoe 9.0 SP4)
  • Analysis of single frames that can be selected in the Trace view.
  • Analysis of multiple frames filtered by frame type (CAN, CAN FD, FlexRay, LIN), application channel control unit (ECU) or bit fields.
  • Analysis of data imports starting with CANoe version 8.5.

Note

When preparing an eye diagram, the number of sample points has to be observed. If this is chosen too low for a bit, a maximum deviation may occur in positive or negative direction of the sampling time.

During the creation of the eye diagram, an edge slope measurement is carried out. This edge slope measurement cannot be done if there are not enough of sampling points. The legend of the Eye Diagram reveals whether the edge slope measurement has been successful. If the edge slope measurement has not been successful, no time will be displayed for the falling and rising edges.

Analysis Restrictions

In the following cases an analysis is not possible or faulty. In all these cases an error notification will be displayed.

  • General
  • Unsupported bus systems.
  • Incomplete recorded frames.
  • Old data imports from CANoe version < 8.5.
  • Sampling rate of the scope is too low.
  • Failed compensation of the time drift between a CAN hardware and the PicoScope.
  • CAN
  • Frames where the dominant or recessive level has been disturbed (e.g. using CANstress) the parsed data differs from the transmitted data of the CAN controller. The significance of the bits has been artificially disturbed.

Configuration

On the Eye Diagram page you can configure the Eye Diagram.

Online Options

During data acquisition it is possible to create an eye diagram. The analyzed bit range corresponds to the bit field definition (see Trigger Configuration) of triggered message. By using Persistency Mode it is possible to overlay several measurements with the same bit field definition. The newest measurement is especially highlighted colored. Analyzed trigger events are listed in the bit legend of eye diagram. Each event entry encloses the analyzed bit range. Additionally the eye diagram analysis automatically preforms a transition time measurement for falling and rising edges for each bit of defined bit field. It also exposes min /max voltage of measured bits.

  • Online Analysis
    Activate or deactivate Online Analysis mode via this check box.
  • Persistency Mode
    Activate or deactivate Persistency mode via this check box.
  • Amount of persistent records
    Enter the amount of persistent records (up to 20) in this input box.

Analysis Options

The analysis can base on a single frame or on dedicated filter settings.
The following options are available:

  • Selected frame in Trace view
    You can select a frame in the Trace view to create the Eye Diagram.
  • Filter settings
    You can select multiple filters (see below) to create the Eye Diagram.

Filter Settings

When you selected Filter Settings the following options are available:

  • Frame
    Depending on the bus system you can select the corresponding frame type that shall be used for the Eye Diagram.
  • Channel
    You can filter frames by their channel.
  • ECU
    You can filter frames by their ECU (only with database).
  • Bit fields
    You can define which sections of the frame shall be used for the Eye Diagram.

Buttons

  • [Clear]
    The last created Eye Diagram is deleted.
    This is automatically done when the current measurement is deleted or altered.
  • [Apply]
    The creation of the Eye Diagram is started.
  • [Cancel]
    The current creation is canceled.

Analysis in the Eye Diagram

After the Eye Diagram is created there are several ways to analyze the data:

  • superposition of all bits
  • presentation of the ideal bit as background legend (with Sync, TSEG1, sample point, TSEG2).
  • the bit graphs can be selected in the legend individually or grouped
  • the bit graphs can be selected in the Eye Diagram individually or by multiselect
  • The bit graphs selected in the Eye Diagram can also be shown in the diagram and in the Trace view (see figure). Therefore you have to select the legend's context menu command Show selected bits in measurement curves.

CAN

Display of the ideal Bit as background legend (with Sync, TSEG1, sample point, TSEG2).

Bitmask Definition

Via the toolbar of the Eye Diagram you open the Bitmask Definition dialog. Open therefor the list of the Options for bit analysis button and click the Bitmask configuration command. With the defined bitmask you can make a bit analysis during creation of the Eye Diagram.

Activate the bit analysis. Open therefor the list of the Options for bit analysis button and click the Enable bit analysis command.

FlexRay

For a FlexRay analysis you can include BBS bits. Therefor activate the Analyse BSS (FlexRay) option. (See list of the Options for bit analysis button ). If you don't activate this option the BSS bits are not analyzed. They are also not contained in the legend of the Eye Diagram.

LIN

For this bus system the Bit analysis feature is currently not available.

Eye Diagram Import/Export

Import/Export as Data (Format CEYE)

The entire eye diagram consisting of several measurement with defined bit sequence can be imported and exported. On import the complete eye diagram analysis is replayed. The data acquisition algorithm for replayed measurements is the same as in online mode.

Import and export capabilities are accessible via the toolbar menu of an eye diagram or by usage of the context menu of the bit legend.

Export as an image file

The Eye Diagram can be exported as a screenshot including all drawing objects. Use the context menu commands of the Eye Diagram to start the export:

  • Copy to Clipboard as
    The following formats are supported:
  • Bitmap (*.bmp)
  • Vector Graphics (*.emf)
  • Save View as Image
    The following formats are supported:
  • PNG
  • GIF
  • JPEG
  • TIFF
  • BMP
  • EMF
