Function Overview

 

CANoe.FlexRay offers the following features:

FlexRay Features

Display and analysis possibilities that are typical for FlexRay

Additional columns with FlexRay-specific information can be shown in the Trace Window.

FlexRay-specific information can be shown in the Bus Statistics Window.

FlexRay Bus Simulator

The complete FlexRay communication can be simulated without having a real bus (interface) available.

Transmission and reception of FlexRay frames and PDUs

FlexRay frames and PDUs can be received, analyzed, processed and transmitted in CAPL programs.

Support of "Cycle Multiplexing"

If a FIBEX database is integrated, then FlexRay frames in the same slot but with different "Cycle Multiplexing" will be distinguished, and in the Trace Window they will also be displayed in different rows.

Signals (for the Graphics Window, panels and for access in CAPL) are differentiated according to their allocation to multiplexed frames (or extracted from the correct frames and integrated in the correct frames).

When transmitting, it is possible in CAPL to transmit FlexRay frames with different cycle repeat rates and base cycles ("Cycle Multiplexing") in the same time slot.

The filters also support "Cycle Multiplexing" for FlexRay. In addition to the slot ID, the base cycle and the cycle repeat rate can also be set in the filter.

Support of  "In-Cycle Repetition"

If a FIBEX database is included, signals transmitted in several slots are also identified. When receiving a signal the integrated signal layer analyzes the corresponding slots and displays via a CAPL query the right. This mechanism works also on transmission.

StartCycleEvent

At the beginning of a communication cycle an event is sent that - in addition to the start time of the current cycle - also contains the valid Network Management vector.

Support of Signal Variables in CAPL

With the signal operator $ it is easy to read and to assign signal values. The signal objects work together with the signal layer.

Example: $<signal name> = value and variable = $<signal name>

Automatic Buffer registration for active database nodes

When a database node is assigned to CAPL node all Tx frames are configured for transmission automatically. In the Tx buffer configuration it is possible to adapt the configuration accordingly.

Support of the FlexRay Network Management mechanism

The CAPL Send function can also be used to set the NM bit. This causes the first two data bytes in the payload area to be interpreted as the NM vector.
A special receive function permits use of the NM vector in individual CAPL nodes.

Tx Acknowledge

If FlexRay frames are transmitted the FlexCard Cyclone II/SE and the VN interfaces produce Tx events with the full information of the frame at transmission time.

Filter to suppress "planned" Null Frames

Null Frames are transmitted in unused gabs of a slot with cycle multiplexing. These Null Frames are filtered automatically to reduce the busload. It is possible to deactivate the filter.

Event Highlighting in the Trace Window

FlexRay events (e.g. StartCycle, Error, ...) can be configured with different colors for text and background in the Trace Window.

Hardware Time Synchronization with other bus systems

The network interfaces support the hardware time synchronization with other network interfaces. This ensures that the time stamps of the various bus systems will be synchronized with each other and thus will be comparable, in multi-bus implementation.

Support of up to 32 FlexRay channels

Up to 32 hardware FlexRay network interfaces are supported. You can set the number of available cluster/measurement channel/FlexRay network interfaces in the Options dialog (Measurement|General).

Note

If you are using more than one channel, you can only use VN interfaces (one per channel).
The first channel must also be a VN interface.

Asynchronous and Mixed Mode

In Asynchronous Mode, you only need to specify the baudrate in order to read data from the bus. Transmission is not possible in this mode. The mode is supported by the VN interface and the FlexCard Cyclone II/SE. In this mode, network start-up can also be monitored.
