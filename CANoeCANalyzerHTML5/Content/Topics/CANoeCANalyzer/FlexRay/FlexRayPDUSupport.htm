PDU Support

 

Introduction

Note

Linking the FIBEX+ or FIBEX 3.x format to CANoe and CANalyzer has resulted in a number of new functions and differences in use.

The differences are summarized on this help page.

The interpretation of FlexRay data is fundamentally different. FlexRay frames play a subordinate role, and signals are largely evaluated using PDUs.

Main functions:

  • PDU display
  • Automatic PDU update bit evaluation
  • Use in CAPL
  • PDU based remaining bus simulations

Activating PDU Support

CANoe automatically activates PDU support when a PDU database is assigned. Databases in FIBEX+ or FIBEX 3.x format are accepted as databases with PDU descriptions.

Options

Network Interface

A special filter can be configured in the FlexRay Interface Hardware configuration dialog. Activating the "Show PDU with disabled update bit" setting enables you to process PDUs with disabled update bits (e.g. for display in the Trace Window). If this setting is not activated, only PDUs with enabled update bits are processed.

  • Default setting: Setting is deactivated.
  • This setting is only available when a PDU database has been assigned.

Measurement Setup

  • Trace Window
    The Trace Window displays PDUs with symbolic names and assigned signals. A row corresponds to exactly one PDU (frame type: PDU). Because a PDU can be transmitted in different slots, the current transmission can be determined by means of the Identifier column. This shows the current slot and the assigned timing.
    In addition to the PDU display, FlexRay frames are also displayed as raw data frames (Raw Frames).
    The new Update Bit column shows the value of the assigned update bit.

    Note

    Variable update bit values can only be monitored if the option "Show PDUs with disabled update bit" is activated in the Options dialog.
  • Filters
    The filter can be used to customize the configuration of the measurement windows (e.g. the Trace Window).
    It can be used to filter raw frames and PDUs:
  • Raw frames can be filtered by entering a concrete ID or range.
  • PDUs can be configured by means of symbolic assignments made in the Symbolic Selection Dialog.
    For each PDU, the Identifier column shows the slot number for each assigned timing.
    If the PDU is transmitted on different cycles in one slot, the column will contain the same slot number multiple times.

  • PDUs can be configured by means of symbolic assignments made in the Symbolic Selection Dialog.
    For each PDU, the Identifier column shows the slot number for each assigned timing.
    If the PDU is transmitted on different cycles in one slot, the column will contain the same slot number multiple times

    Note

    A simple way to filter out all raw frames from a Trace Window is to create a blocking filter with an identifier range of 1…2047.
  • Symbol Explorer and Symbolic Selection Dialog
    The symbolic data sinks, such as the Data and Graphics Windows, are also configured via the PDU context. The Symbol Explorer and symbolic selection dialog (in the context of the Data Window, Graphics Window and filters) are structured in line with the PDUs:
    The predefined PDU signals in the Symbol Explorer now include the Update Bit signal. In the context of a PDU, the signal can be pulled into the Data and Graphics Windows using drag and drop.

CAPL

  • Receiving PDUs
    PDUs can be received using the on FrPDU <PDU name> event handler.
  • Transmission of PDUs
    You can use FrPDU to create a FlexRay PDU transmission object.
  • Functions
  • FrSetSendPDU: Configures the hardware to transmit the specified PDU.
  • FrUpdatePDU: Updates the PDU payload in the assigned FlexRay frames.
  • Signal-oriented models
    The PDU mode also supports signal-based access within CAPL. The signal layer integrated in CANoe allows direct signal access in the transmit and receive directions in CAPL programs.
    The PDU mode also supports signal-based access within CAPL. The signal layer integrated in CANoe allows direct signal access in the transmit and receive directions in CAPL programs.
  • Signal Transmission
    $FIBEX_Signal_name = newValue;
    This signal assignment directly updates the send buffer that contains the signal.
    The corresponding update bit is automatically set to 1 in PDU mode.
  • Signal Receipt/Evaluation
    Variable = $FIBEX_Signal_name;
    This assignment updates the CAPL variable with the most recent signal value from the signal layer.
  • Configuration
    For signals to be transmitted, the corresponding send buffers must be registered. CANoe carries out the registration automatically via the CAPL program's node context. If this is the case, all slots in which frames/PDUs are sent are registered automatically.
    It is possible to modify the transmission behavior of automatically registered frames/PDUs and to deactivate individual slots.
    The configuration dialog is accessed via Configuration|Hardware Configuration|Tx Buffer.
    The list shows the individual PDUs in PDU mode. Because, on average, up to 5 PDUs are transmitted in one slot, it is important to note that in Version 6.1.24 SP1, deactivating a single PDU has no effect. All PDUs assigned to a slot must be deactivated in order for the slot to be deactivated.
    The [Modify] button offers simple modification options for influencing basic behavior:
  • Activation/deactivation of all start-up and sync flags
  • Configuration of buffer properties (event-triggered, time-triggered)

Logging

The log records the FlexRay frames, as has always been the case. These can be played back as PDUs in offline mode using a corresponding database.

Note

A configured filter block (Pass Filter for PDUs) inserted before the logging block can be used to ensure that frames are not passed to the logging block (see also Filters).

Offline Analysis

You can execute specific evaluations using the measurement data sinks or via CAPL programs in the measurement setup:

Example

on FrPDU PDU_BVS_Status
{
  int ub=0;
  if(ub ^ this.updatebit)
  {
    write("%d",this.updatebit);
    ub=this.updatebit;
  }
}
