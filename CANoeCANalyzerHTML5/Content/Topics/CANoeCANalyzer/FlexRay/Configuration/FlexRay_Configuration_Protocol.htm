FlexRay Configuration - Protocol

Ribbon: Hardware Tab|Network Hardware|FlexRay|Protocol

Here you can specify the protocol parameters for the bus timing.

Use these protocol parameters in order to determine the register values of the CC, which you can also display by selecting Controller in the FlexRay branch on the left side.

The protocol parameters values can be taken from the database or can be set manually. If Use Database Setting is not activated, differences to the default database settings can be displayed. The option Database/Custom is valid for all FlexRay devices.

You can change the parameter values of the manual configuration by clicking on the values in the Custom column.

Colors of the Parameters

Parameter Color

Meaning

Saddle Brown

Protocol constant or physical layer relevant parameter

Medium Blue

Global protocol related parameter

Navy

Local protocol related parameter (of a DB node)

Silver

Parameter directly depends on other parameters and is set/derived automatically.
Neither the manual parameter can be modified, nor is the parameter read from the DB.

Buttons

The following controls are available:

  • [Use Database Settings]
    This button allows to select if the cluster parameters should be taken from the database or if manual values should be used.
  • [Options...]
    The following buttons are available in this tab.
  • [Import FPIâ¦]
    Clicking on this button opens a file selection dialog where you can open an FPI file. The protocol parameters specified in this file will be imported into the manual configuration. It is the user's responsibility to ensure that the imported values are protocol conformant and consistent.
  • [Export FPIâ¦]
    Clicking on this button opens a file selection dialog where you can save an FPI file. The specified protocol parameters of the manual configuration will then be exported into this file.
  • [Reset]
    If you select this button, all protocol parameter values of the manual configuration are then reset to the values of the database configuration.
  • [Check Parametersâ¦]
    Clicking on this button starts a consistency check. It is checked whether the parameters keep to the protocol constraints.
  • [Show Differences]
    If Use Database Settings is not selected the activation of this button shows the differences between the manual and the database configuration.
  • [Database]
    Here you can choose the reference database. The network parameters of this reference database are used to configure the hardware.
  • [Controller]
    You can select a database node with this list. If Use Database Settings is selected, then the selected node decides which values are used for the local cluster parameters of the FlexRay device. The first entry of the list distinguishes the possibility that the global cluster parameters are calculated automatically from the tool. The local cluster parameters of the FIBEX database are evaluated in the following order:
  1. If a node is selected the local parameters are taken from this node.
  2. If the database contains a node (ECU) named CANoe, DENoe, CANalyzer, or DENalyzer, then the local cluster parameters are taken from that node.
  3. If automatic calculation of local parameters is selected, then the local cluster parameters are re-calculated in any case. The local cluster parameters from the database are ignored.

The name of a selected database node is displayed the following way: <ECU name>::<Controller name>.

Note

FPI stands for FlexRay Protocol Information. This is a proprietary file format and it is not standardized.
