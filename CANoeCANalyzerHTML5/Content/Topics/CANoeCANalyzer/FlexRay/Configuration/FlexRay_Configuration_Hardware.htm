Hardware Configuration

Ribbon: Hardware Tab|Network Hardware|FlexRay

On the left side of the dialog all FlexRay clusters (measurement channel/FlexRay network interface) used in the current configuration are listed in a tree view.
A maximum of 32 channels is available. You can adjust the number of available clusters/measurement channels in the Options dialog.

For FlexRay the following pages are available:

  • Mode

    The operation mode of the FlexRay interface can be configured on the Mode page.

  • Controller

    The Controller configuration is used to set the controller registers for the FlexRay hardware.

  • Protocol

    In the Protocol view, the contents of the register values of the controller for the bus configuration are displayed as protocol parameters.

  • Options

    On the Options page several settings of the driver and/or interface can be configured. This encompasses settings for

    • Transmit or update acknowledgments
    • Null frame and PDU filtering
    • Automatic restart of the communication controller after synchronization is lost
    • Wake-up sending

  • Trigger output

    VN Trigger I/O

  • Trigger input (only available for VN interfaces)

    This page is NOT available in the hardware configuration dialog! The settings must defined in the CAN.ini file. See VN Trigger I/O.

  • Tx buffer

    Shows the automatic Tx buffer definitions.

  • Key Slot Config

    Configures the two Startup/Sync slots

Note

The Controller and Protocol pages are different views of the same data (FlexRay bus configuration). Changes (in the manual settings) on a page are also correspondingly displayed on the other page.
