Key Slot Configuration

Ribbon: Hardware Tab|Network Hardware||FlexRay|Key Slot

To start-up a FlexRay bus there are at least two start-up/sync frames necessary. The VN interface is able to transmit two start-up/sync frames. Thus, they can start-up a FlexRay cluster by themselves without the need of any external start-up node. Both configured key slots are sent at least as a Null frame beginning with [Start (measurement)].

Thus, you are able to test and analyze a real ECU, which does not perform a start-up (Integrating Nodes, which do not transmit a start-up/sync frame).

If additionally the Tx buffer list, a Frame Panel, CAPL program, or modeling library (DLL file) defines a Tx frame in one or both of these key slots, then the slot will contain a data frame with a valid payload.

Both key slots can operate (independently) in the following modes:

Mode

Description

Off

The key slot is not used.

A frame sent from the interface in this slot is not marked as a start-up/sync frame.

Note

The execution of the CAPL function FRSetKeySlot can change this setting again!

Automatic

This is the default mode. The key slot will possibly be active and is selected according to the following rule:

Any frame that is active in the Tx buffer list or entered in any Frame Panel will be selected for this key slot, if it is marked in these lists as start-up and/or sync frame (and if it is not already selected for the other key slot).

Note

The settings of the start-up/sync flags of any Frame Panel overrides any settings of the Tx buffer list!

The execution of the CAPL function FRSetKeySlot will override any settings of the Tx buffer list and the Frame Panel (and may therefore also override this 'automatic' setting)!

Manual

In this mode the key slot will explicitly be defined here. Thus, the properties of the key slot are defined by the appropriate controls of this dialog:

Key Slot Property

Meaning

Type

The frame in this slot can be sent as a start-up or only as a sync frame.

Slot

This is the explicit slot number of the static segment that will be used as the key slot. (Regardless if the database, Tx buffer list and/or Frame Panel plans a start-up/sync frame in this slot or it does not.)

Channel Mask

This is the explicit channel mask (A, B, A+B) that is used for the key slot.

Leading Coldstart

If this check box is deactivated, then the sent key slot cannot be the first frame that is sent for the cluster's start-up. Thus, the key slot can act only as a following start-up or integrating sync frame.

By using the button [Select ECU...] the key slot settings can be loaded from the key slot setting of an appropriate node definition in the database.

Note

The execution of the CAPL function FRSetKeySlot cannot override this key slot setting!

Note

  • If the interface is selected to operate in the "asynchronous mode" (see VN interface mode page), then only one manual key slot can be configured.
  • The key slot number in the asynchronous mode can differ from the slot numbers that are used in the synchronous mode.
  • The frame of the (asynchronous mode) key slot will be sent as a Null frame only.
  • If the key slot is enabled for the "asynchronous mode", then all cluster parameters must be specified (instead "Baudrate only")!
