FlexRay Configuration: VN Interface Trigger I/O

Ribbon: Hardware Tab|Network Hardware|FlexRay|Trigger-I/O

On this tab, you can activate the trigger output and the trigger input of the VN interface.

Trigger Output

Three independent trigger outputs can be used. Each of the outputs can separately be enabled and configured.

Trigger Conditions

The following conditions can (simultaneously) be defined as a trigger time per trigger output:

  • Start of the static segment
    The trigger output is activated at a cycle start. A specific cycle multiplexing can be given in form of a repetition factor and an offset. Cycle repetition 1 and offset 0 means "every cycle".
  • Start of the dynamic segment
    The trigger output is activated at the start of the dynamic segment. A specific cycle multiplexing can be given in form of a repetition factor and an offset. Cycle repetition 1 and offset 0 means "every cycle".
  • Symbol
    The trigger output is activated when a symbol (MTS, CAS) is received. A channel mask must be specified (1: A or 2: B).
  • Error
    The trigger output is activated if (can be selected as an option) the following errors occur:

    • SFBM

    Sync Frames Below Minimum

    SFO

    Sync Frame Overflow

    CCF

    Clock Correction Failure

    PERR

    Parity Error

    EDA

    Error Detection on Channel A

    LTVA

    Latest Transmit Violation Channel A

    TABA

    Transmission Across Boundary Channel A

    EDB

    Error Detection on Channel B

    LTVB

    Latest Transmit Violation Channel B

    TABB

    Transmission Across Boundary Channel B
  • Slot
    The trigger output is activated if the time point of the specified slot is reached.
  • Wake-up
    The trigger output is activated when a wake-up is received.
  • POC State
    The trigger output is activated, if the communication controller enters the specified POC (Protocol Operation Control) state.
    The following states can be selected:

    • DEFAULT_CONFIG

    STATUS_READY

    NORMAL_ACTIVE

    NORMAL_PASSIVE

    HALT

    CONFIG

    WAKEUP_STANDBY

    WAKEUP_LISTEN

    WAKEUP_SEND

    WAKEUP_DETECT

    STARTUP_PREPARE

    COLDSTART_LISTEN

    COLDSTART_COLLISION_RESOLUTION

    COLDSTART_CONSISTENCY_CHECK

    COLDSTART_GAP

    COLDSTART_JOIN

    INTEGRATION_COLDSTART_CHECK

    INTEGRATION_LISTEN

    INTEGRATION_CONSISTENCY_CHECK

    INITIALIZE_SCHEDULE

    ABORT_STARTUP

    STATUS_STARTUP_SUCCESS

The trigger output can also be set by a CAPL function: FRSetTrigger

Trigger Input

On measurement start the activation of the trigger input is displayed in the Write Window.

PIN 2 of the 8-pin connector is used. Please refer to the interfaces manual for a detailed description of the pins (FlexRay_Interface_Family_Manual.PDF, page 15).

Use the settings to determine the trigger edge:

  • Rising
  • Falling

Trigger Events in CANoe

On occurrence of a trigger a system variable is set. On activating a trigger in the hardware configuration these system variables are available "everywhere" in the tool. You can change the name of the system variable. All trigger system variables are in the FrTrigger namespace.
