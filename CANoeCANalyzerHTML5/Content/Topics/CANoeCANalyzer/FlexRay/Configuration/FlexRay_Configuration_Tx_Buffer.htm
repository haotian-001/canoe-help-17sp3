FlexRay Configuration: Tx Buffer

Ribbon: Hardware Tab|Network Hardware|FlexRay|Tx Buffer

This page lists all Tx frames of the database, which shall be transmitted by the nodes of the database. It is important that the database nodes are assigned to the CAPL nodes. The buffers can here be configured. For activated frames send buffers on the FlexRay interface are defined accordingly.

This configuration dialog can be opened from two different locations:

  • Alternative 1: Via this page and the Tx Buffer Configuration button you open the Transmit Buffers Configuration dialog.
  • Alternative 2: In the Simulation Setup open with a right click on a node and the context menu commandConfiguration the Node Configuration dialog. On the PowerTrain_FlexRay page you find the current settings for the Tx Buffer. Via the Configuration button you open the Transmit Buffers Configuration dialog.

Depending on the database model (frame-based or PDU-based) the dialog only displays frames or displays frames and their contained PDUs. The PDU list will be updated when a specific frame is selected.

Dialog layout when only frames are displayed:

Dialog layout when frames and PDUs are considered:

In the leftmost tree view of the dialog the contents of the frame list can be filtered. The selection of

  • All Frames
    Shows all frames. Even those that do not have any Tx relations
  • Tx Nodes
    Shows all frames with Tx relations.
  • Concrete Node from Database
    Shows only those frames that are sent by this node.

In the area on the left bottom of the dialog the configured key slots are displayed:

  • If you select the All Frames or the Tx Nodes entry an overview of all ECUs with key slot and its usage (Sync, Startup,Leading Coldstart) are displayed.
  • If you select a single node you can change the settings of the key slot and its usage.
    The Key Slot Usage list provides the following settings:

Not configured

No key slot will be used.

Sync

For this key slot the Sync flag will be configured.

Startup

For this key slot the Sync and Startup flag will be configured.

Startup, LCS

For this key slot the Sync and Startup flag will be configured.
The Leading Coldstart property in the controller will be activated.

With setting the check box of the column Active the appropriate Tx buffer in the interface can be created. Within the tree view the check box enables or disabled all Tx buffers of the appropriate filter list.

The check box of the tree view distinguishes three states:

  • Activated: All frames are enabled
  • Deactivated: All frames are disabled
  • Three-State: Some Frames are activated while other frames are deactivated

If frames are sent by more than one node, then the deactivation/activation is applied to the lists of all nodes, which contain this frame.

Configuration columns of the frame list:

Column

Meaning

Active

Activates or deactivates the corresponding Tx buffer.

Name

Name of the Frame (FIBEX+ does not define frame names. Therefore "..." will be displayed.)

Tx Node

Name of the database node that transmits this frame

Slot

Slot ID that contains this frame

Offset

Base cycle of the frame's cycle multiplexing

Rep.

Cycle repetition of the frame's cycle multiplexing

Type

Service type (that is defined in the FIBEX file)

Channel Mask

FlexRay channel (A or B)

Payload Length

Length of the frame's payload area in [16 Bit words]

TT

If enabled, then the Tx buffer is set to time-triggered mode (automatic repeated transmission). Otherwise to event-triggered mode (Null frames will be sent, if the buffer is not updated).

PP

Sets or unsets the Payload Preamble bit of the frame's header.

Used for PDU

If set, then the Tx buffer can only be used by PDU updates (otherwise only frame updates are allowed).

Database

Name of the database that defines this frame

The following menu commands will be applied only to frames, that can be configured:

  • Enable All
    Activates all frames
  • Disable All
    Deactivates all frames
  • Flags
    • Delete startup flags
      Deletes the SUPF flag of all frames
    • Set startup flags according to Database
      Sets the SUPF flag for all frames that are defined as startup frames in the database
    • Delete Sync Flags
      Deletes the Sync (and SUPF) flag of all frames
    • Set Sync Flags according to Database
      Sets the Sync flag for all frames that are defined as Sync frames in the database
  • Dynamic/static frames
    • Set to Time-Triggered
      Sets all dynamic/static frame's Tx buffers to time-triggered transmission (automatic repeated transmission)
    • Set to Event-Triggered
      Sets all dynamic/static frame's Tx buffers to event-triggered transmission (Null frame is sent on missing update in static segment)
  • Service Types
    Activates/deactivates all frames of a specific service type

Note

To create a send buffer for a frame which is not defined in the database use the CAPL function FrSetSendFrame or the FlexRay Frame Panel.

Notes

  • A frame can only be activated when the database node is assigned to a node in the Simulation Setup and the simulation node is active. If the simulation node is deactivated the frame will also be deactivated and displayed in gray. If the database node is not assigned to a simulation node the frame is deactivated and displayed in gray. Frames, whose database node is assigned to a simulation node and the simulation node is also active in the Simulation Setup, are displayed in a normal way and may be activated and deactivated.
  • If the Real Mode is selected, then an interface channel (e.g. "FlexRay 1") must be attached to the measurement channel of the Simulation Setup.
