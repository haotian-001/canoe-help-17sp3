FlexRay Configuration - Partial Remaining Bus Simulation (PRBS)

 

This FlexRay feature configures the Partial Remaining Bus Simulation. In a PRBS only those nodes will be simulated that send frames to the real existing ECUs. Goal of this feature is to minimize the overall number of simulated nodes and sent frames.

The set of real existing ECUs is named System under Test (SUT). In this configuration and status dialog for the PRBS the nodes for the SUT can be selected from all attached databases. The dialog can be opened from the context menu of a FlexRay bus: [Setup PRBS...]

By pressing the [Apply] button the current selections for the PRBS will be stored and all nodes of the Simulation Setup will be activated or deactivated accordingly. Additionally all Tx buffers from the transmission lists of the nodes will also be activated or deactivated. If any node is missing in the Simulation Setup, but is required for the partial remaining bus simulation, then the PRBS can not be defined.

Nodes, which are activated and whose Tx buffers are completely activated, participate fully in the simulation. Nodes, which are activated but have any deactivated Tx buffer, participate only partly in the simulation. All deactivated Tx buffers are sent to nodes that are not part of the SUT. Therefore they are not required. Is a node not sending any frame to a node of the SUT, then the node will be deactivated (together with all of its Tx buffers).

While selecting or deselecting nodes for the SUT the dialog shows per node the current state and the planned and pending modifications for the simulation.

Status Display

The columns of the dialog have the following meaning:

  • Selection
    These check boxes select a node as a member of the SUT.
    The node will therefore be deactivated after applying the selection.
  • Node
    Name of the node from the database
  • Database
    Name of the database in which the node is defined.
  • Mode
    Shows the mode of the node, which is set after applying the current SUT selection.
    The modes are:
  • <empty>
    The node is not required and will therefore be deactivated.
  • <SUT>
    The node is member of the SUT and will therefore be deactivated.
  • <full>
    All Tx frames of the node are sent to the SUT.
    The node and its Tx buffer list will therefore be fully activated.
  • <partial>
    Some (but not all) Tx frames of the node are not sent to the SUT.
    Thus, a few (but not all) Tx frames are sent to the SUT.
    The node will therefore be activated, but some of its Tx buffers will be deactivated.
  • State
    Shows whether the node is activated, deactivated, or missing in the current simulation.
  • Modifications
    Shows a summary of all actions when applying the partial remaining bus simulation.

[Modify...]

This menu contains the following actions:

  • Clear the Remaining Bus Simulation
    The selection of the SUT nodes will be cleared.
    Nevertheless, all nodes will be deactivated after applying this setting.
    This means that nothing will be simulated.
  • Full Remaining Bus Simulation
    The selection of the SUT nodes will be cleared.
    Nevertheless, all nodes and Tx buffers will be activated after applying this setting.
    This means that everything will be simulated.

[Options...]

The options dialog for the PRBS allows modifying the following settings:

  • Frame Types (default: all selected)
    Here some frame types of the FIBEX database can be ignored. Frames of not selected frame types will not be simulated. That means that those Tx buffers will be deactivated (after applying the current SUT again
  • Set the PRBS completely (default: on)
    If this switch is off, then only those changes will be required that have to be applied due to changes at the SUT. I.e. if in an active partial remaining bus simulation any node is manually be deactivated in the Simulation Setup, then the PRBS dialog (with this disabled option) will not show any pending modifications (although the node is partly or fully required). This setting may be required in order to hold manual changes active unless changes in the SUT require changes for this node or any Tx buffer.
  • Consider Recursion (default: off)
    In a first step only those nodes and Tx buffers will be activated that are sent to nodes of the SUT. In a second step additionally those nodes and Tx buffers will be activated that are sent to already simulated nodes. The second step will iteratively be repeated until any other node or Tx buffer will not be activated.

[Apply]

This button applies all planned and pending changes (activating/deactivating nodes and Tx buffers; see Modifications column) to the current simulation.

[Close]

This button closes the dialog without applying any pending modifications to the current simulation.
