How to view FlexRay bus events and signals?

 

Using the configuration created in "How to create a configuration for analysis?", you can now start via Home ribbon tab|Start to observe the FlexRay traffic in the Trace Window when using a simulated bus.

If you want to use a real bus, then some prerequisites as described in "How to assure a cluster start-up?" must be fulfilled.

Bus events and signals will be displayed in the following windows:

Trace Window

The Trace Window not only displays valid FlexRay frames, but all types of FlexRay bus events and errors with their time stamp.

Note

For a full list of FlexRay events and errors, please see Columns for FlexRay.

By expanding a FlexRay frame, you can view its signals as defined in the FIBEX file associated to this FlexRay channel. Additional FlexRay-specific columns can be added to the Trace Window using its configuration dialog.

Data Window

To view frame signals numerically, either open an existing Data Window in the Measurement Setup or create a new one. You can then add signals to this window in one of the following ways:

  • Via context menu command [Add signals…]
  • Per drag and drop from the Symbol Selection dialog
  • Per drag and drop from the Trace Window.

Graphics Window

To view frame signals graphically over a timeline, either open an existing Graphics Window in the Measurement Setup or create a new one. You can then add signals in the same way as for the Data Window (see above).

Panel

You can also create your own panels using the Panel Designer utility, which can be started via the Tools ribbon tab|Panel Designer.

Example

  1. drag and drop the Output Box from the toolbox to the opened panel.
  2. Select a FlexRay signal to be displayed via the Symbol Selection in the Properties dialog.
  3. Now save your panel (File|Save).
  4. With the File|Add Panel to Configuration you can add the created panel to your configuration (if not already done during saving).

After that the panel will be displayed on the currently active desktop.

After [Start (measurement)] it should now be possible to view the signal’s last value using the panel. The value will be updated and stored whenever the signal is received.
