How to increase determinism?

 

Here "indeterminism" is related to unpredictable update time points for frames. Although the programmer assumes the update takes place according to a relative and always equidistant trigger event (periodical timer), the real update time point jitters extremely, which consequently leads to slot misses (see "How to avoid slot misses?").

The determinism can extremely be increased by using the distributed mode solution. Hereby disturbances of Click on the blue link to load the sample configuration in CANoe. are eliminated (jitters are minimized). The complete computation power is available for CANoe. Thus, RT systems are strongly recommended to be used for remaining bus simulations with FlexRay.

A similar solution in order to increase the deterministic execution and to increase the execution speed for one CAPL program is to use the Execution of CAPL Code on Interface feature, which is only available for Click on the blue link to load the sample configuration in CANoe. and certain interface types.

Note

For the CAPL programs it is extremely important to specify update time points not by using "on timer" events (which are possibly asynchronous to the FlexRay time).

Rather the FlexRay synchronous notification handlers like "on FRStartCycle", "on FRFrame", or "on FRSlot" should be used.
