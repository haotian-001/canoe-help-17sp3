How to assure a cluster start-up?

 

If you want to use a real bus, then four prerequisites must be fulfilled:

  1. Your FlexRay bus can do a start-up. This is assured when already two real start-up nodes are present and are working. One or both of these start-up nodes can be simulated by CANoe depending on your current network interface hardware. Therefore CANoe must be configured in a way that it sends one or two start-up frames.

    Note

    • If CANoe does not send any start-up frame, then you need two external start-up nodes.

      When CANoe sends one start-up frame, then you need one external start-up node.

      If the network interface (e.g. VN) is able to send two start-up frames, then you need no external start-up node.

    • Exactly two frames (of the static segment) of the Tx buffer list, a CAPL program (refer FRSetSendFrame), or a Frame Panel can be sent as a start-up and/or sync frame.

      Refer also the Key Slot Configuration of the network interface for the selection of the used slot numbers.

  2. The FlexRay TDMA parameters that are defined in your FIBEX or CHI file matches exactly those definitions that are used to implement the external nodes.

    Note

    • The network interface hardware is allowed to be used in a so-called "asynchronous mode" (see Network Interface Mode).

      This mode requires only a few parameters (baudrate, physical layer, etc.) to be set and a FIBEX or CHI file is not required, but is restricted to receive frames only.

  3. Select the connected network interface type in a list of the hardware configuration dialog.

    Note

    • The Controller Configuration or Protocol Configuration dialog you can choose the database settings to be used for the TDMA parameters or a manual setting that enables the import of an appropriate CHI file.

      If you choose the database settings, then you can also select the database node from which the local settings will be used.

  4. Assure that the bus channel A is connected to the channel A of the bus network interface. The same applies for channel B. If both channels are exchanged, then any received frame will be interpreted with a wrong CRC and therefore it will be rejected or interpreted as an erroneous frame.

    Note

    • The Controller Configuration or Protocol Configuration dialog you can choose the database settings to be used for the TDMA parameters or a manual setting that enables the import of an appropriate CHI file.

      If you choose the database settings, then you can also select the database node from which the local settings will be used.

    • A cross-connection of the network interface channel A to the network interface channel B is also not allowed!

If all prerequisites are fulfilled, the network interface of CANoe can synchronize to the real FlexRay bus when pressing [Start (measurement)].
