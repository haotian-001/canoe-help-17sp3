How to implement ECU simulations that have to respond on diagnostic requests?

 

If you add a diagnostic description database to a FlexRay bus in the Diagnostics/ISO TP Observer configuration dialog (Configuration | Diagnostics/ISO TP Configuration…), then you can

  • assign the node or functional group that is target for the requests and
  • configure the underlying FlexRay transport protocol that is to be used.

You can assign more than one diagnostic description database (with optionally different targets) to one FlexRay bus.

Then you can simulate responses on ECU diagnostic services in CANoe by

  • inserting a network node into the Simulation Setup
  • assigning one of the network nodes defined in the FIBEX database to the network node ("Common" Page|CANdb Name) in the Simulation Setup
  • adding the appropriate transport protocol module ("Modules" Page) to a simulated node in the Simulation Setup
  • implementing the appropriate CAPL Callback Interface (CCI) for communication with the corresponding transport protocol in the CAPL program for this node

Examples for CCI implementations are the CAPL Callback Interface (CCI) reference implementations, see Diagnostics: Connection of the Communication Layer.
