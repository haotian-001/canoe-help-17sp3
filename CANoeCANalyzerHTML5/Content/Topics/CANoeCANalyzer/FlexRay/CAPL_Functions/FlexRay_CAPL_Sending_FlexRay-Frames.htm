Sending FlexRay Frames

 

It is possible to send FlexRay frames in the current version using CAPL functions and with the FlexRay Frame Panel .

There are the following functions in CAPL:

  • FRSetSendFrame
    This function can be called optionally in on preStart to configure an appropriate send buffer in the Communication Controller (CC) before the measurement starts.
  • FRUpdateStatFrame
    This function can then be used to update the send buffer in the CC for a frame in the statistic segment. For the purpose of the update, all data are taken from an FRFrame object.
  • FROutputDynFrame
    This function can then be used to update the send buffer in the CC for a frame in the dynamic segment. For the purpose of the update, all data are taken from an FRFrame object.
  • FRSendFrame
    This function can then be used to update the send buffer in the CC. All data have to be specified via parameter.

The functions can be used without a database. It is also possible to transfer the parameters from a database using the symbolic name of the frame.

Note

The functions FrSetSendFrame and FrSendFrame may also use symbolic frame names from the database instead of the first five parameters.

If the send buffer is activated via the Tx buffer list the FrSetSendFrame function in on PreStart is obsolete. The function can be used optionally to change the data of the send buffer (override).

Example

A start-up/sync frame in slot 60 is sent in the following CAPL example. The calculation and buffer update are triggered each time at the start of the cycle.

 

variables
{
  // The gMsg1 message is the sync message for the static segment
  // that is sent on both channels.
  FRFrame ( 60, 0, 1) gMsg1;
  const BYTE gMsg1Flags    = 3;  // state-driven with sync and start-up
  const BYTE gMsg1Channel  = 1;  // send on cluster/FR interface 1
  const BYTE gMsg1ChanMask = 3;  // send on channel A+B
  const BYTE gMsg1Len      = 32; // 32 Byte user data
}
on preStart
{
  // Optionally prepare buffer for message gSta1Msg:
  gMsg1.MsgChannel     = gMsg1Channel;
  gMsg1.FR_ChannelMask = gMsg1ChanMask;
  gMsg1.FR_Flags       = gMsg1Flags;
  FRSetPayloadLengthInByte(gMsg1, gMsg1Len);
  FRSetSendFrame( gMsg1 );
}
DoFRCalc ()
{
  int   i;
  for (i = 0; i < gMsg1.FR_PayloadLength * 2; ++i)
  {
    gMsg1.byte(i) = i;
  }
}
DoFRSend ()
{
  // Update the sync message buffer gSta1Msg:
  FRUpdateStatFrame(gMsg1); // or FROutputDynFrame(gMsg1);
}
on FRStartCycle *
{
  DoFRCalc(); // calculate message contents
  DoFRSend(); // commit buffer for sending
}

Note

If an earlier time is provided as a send slot (e.g. 2), then the buffer update will already be too late in the current cycle, so that the frame calculated in the current cycle is not sent until the next cycle!
