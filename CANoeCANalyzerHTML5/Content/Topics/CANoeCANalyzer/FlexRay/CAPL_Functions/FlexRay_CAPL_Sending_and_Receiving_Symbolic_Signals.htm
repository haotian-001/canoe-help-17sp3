Receiving and Transmitting Symbolic Signals in FlexRay Frames

 

This requires that there is a FIBEX Database assigned to the configuration in which appropriate FlexRay frames are defined with a signal layout.

Receive

All frames/PDUs are cached in FlexRay after they are received. With the aid of the function

  • <double> = GetSignal (<Signal-Name>)or
  • with a direct access to the signal variable

it is possible to call up the last received value of a given signal.

Example

...
on FRFrame GearBoxInfo // receive FlexRay Frame "GearBoxInfo"
{
  int gearinfo;
  gearinfo = $Gear;  // read out the actual signal value
}
...

Note

The CAPL compiler automatically converts Double values to Integer values, or vice versa.

For the signal types in the frames, Integer or Double (floating comma) values must be used.

Transmit

With the aid of the function

  • SetSignal (<Signal-Name>, <double>) or
  • assigning the value directly to a signal variable.

a signal configured for transmission can be set/updated in a FlexRay frame or PDU.

Example

variables
{
  int gEngTemp;
  msTimer tEngTempTimer;
}
...
on preStart
{
  // register FlexRay frame 25 (EngineData) for sending:
  FRSetSendFrame(EngineData /*frame name*/, 0 /*flags*/);
}
...
on timer tEngTempTimer
{
  if (gEngTemp < 75.)
 {
    gEngTemp += 2;
    $EngTemp = gEngTemp; // Changing signal value
 }
  setTimer(tEngTempTimer,1000);
}
...

The frame update with the FlexRay Communication Controller is carried out at the end of the relevant event procedure (in the example above, at the end of the function on timer tEngTempTimer).

Notes

  • The Motorola FlexRay Controller V9 supports only one frame update per transmission cycle. Further updates within the same cycle are ignored. This means that all the signals of a frame have to be updated in the same higher-level event procedure!
  • To use the signal layer for sending you first have to assign a database node - with a send list that includes the signals to be send - to the CAPL node in the Simulation Setup. The CAPL node must be assigned to the correct ECU in the database that the frames can be registered automatically. The registered frames can be checked in the Tx buffer list.
