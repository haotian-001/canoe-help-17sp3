Gateway Mode

 

Overview

Vector VN network interfaces provide a particular gateway mode. In this mode all frames are automatically routed between the two networks.
You can control this routing by means of specific CAPL commands. While the routing is interrupted you can use CANoe to insert any signal sequences in the target network.
Automatically routed frames have no cycle delay; both networks run absolutely synchronous.

One-Channel-Gateway (A-A-Gateway)

  • Only channel A is supported in each case.
  • Channel A of the second FlexRay channel is connected at the FR B connector of the gateway.
  • The network parameters for both networks must be identical.

Two-Channel-Gateway (AB-AB-Gateway)

Note: This mode is only supported on network interfaces with two FlexRay channels (VN7572, VN8972).

During measurement the routing can be flexibly switched on/off for static frames. The routing of the dynamic part can be switched off at the start of measurement. By default, all frames are routed automatically.

Activating the Gateway

To use theVN network interfaces as a gateway, the channel mapping has to be adjusted in the Vector Hardware Config dialog (Windows Control Panel|Vector Hardware).

Activation in CANoe

To activate the gateway in CANoe you have to switch on the gateway mode in the hardware configuration dialog. The gateway mode supports either synchronous mode or mixed mode.
If a channel has been configured for gateway operation the corresponding second channel has to selected under Gateway Mode Parameters.

CAPL Functions

The gateway can be controlled with the following CAPL functions:

  • FrEnableGateway
  • FrGwBypassStatic
  • FrGwBypassDynamic
