How to Analyze an Ethernet Network

 

You analyze the Ethernet traffic of an Ethernet network via the Measurement Setup, since all Ethernet packets received by the Ethernet interface are guided through the Measurement Setup. In the Measurement Setup, various analysis windows and filters for evaluation are available to you.

You can also evaluate a recorded logging file in the offline mode.

Note

You can test step-by-step instructions on this page with the sample configuration EthernetBasic.cfg.

Ethernet Network Interface

Compared to other bus systems, you must pay attention to how the Ethernet interface is connected to the Ethernet network. Since Ethernet switches are usually used, Ethernet traffic varies on each Ethernet link. Therefore, pay attention to how and where the Ethernet interface is connected to analyze the desired Ethernet traffic.

Options available for the Vector Ethernet interfaces are described on the page How to Configure a Vector Ethernet Interface

Interpretation of the Ethernet Packets

In CANoe, the Ethernet packets are interpreted as much as possible. The nested protocol headers in the Ethernet packet are evaluated. The protocol fields can be evaluated in the detail view of the Trace Window or in CAPL. A large number of protocols listed in the Protocol Overview are supported.

When interpreting protocols, CANoe identifies various errors, for example wrong checksums or incorrect length fields. They are marked in the Trace Window with . The detail view and the Protocol validation column contain additional information.

Some settings of the Ethertype or UDP/TCP port numbers are required for a concrete network. These settings can be made via the Options dialog or by assigning a database.

See also: How to Create an Ethernet Configuration

With application protocols, for example SOME/IP or AUTOSAR PDU, some PDUs are split and transmitted in several Ethernet packets (TCP or IP fragmentation) or one Ethernet packet contains several PDUs. Since it is insufficient to evaluate an individual Ethernet packet, CANoe tracks the Ethernet packets of a connection, generating from it new events for the individual application protocols. This evaluation is supported for the following protocols:

  • AUTOSAR PDU
  • AUTOSAR UDP NM
  • SOME/IP
  • DoIP
  • XCP
  • AVB/RTP
  • HTTP

Events generated from the Ethernet packets have a or icon. The time stamp is identical to that of the Ethernet packet.

CANoe detects various errors in these events. Once an error is detected, the event is marked with . The events can be evaluated only to a certain size, for example 64 Kilobyte, then CANoe limits the length, indicating it with . The alternating shows then the original and the available length.

Ethernet in the Trace Window

You can configure the columns arbitrarily, not all Ethernet columns are contained in the predefined column configurations.

Using Column Filter

Since Ethernet usually transfers a lot of data, the Trace Window quickly becomes confusing. The column filters help you, for example, to display only certain protocols or IP addresses.

You can activate several column filters at the same time.

Using Predefined Filter

Another way to improve the clarity of Ethernet is to use the predefined filters of the Trace Window. These filters can be used to filter certain event categories. If, for example, you want to concentrate on AUTOSAR PDU or SOME/IP, it can be helpful to hide the Ethernet packets.

Step by Step Procedure: Hide Ethernet Packets

  1. Open the Trace Window.
  2. Open the predefined filters with in the toolbar.
  3. Deactivate Ethernet events under Bus Systems to hide all Ethernet packets.

Using Analysis Filter

You can also use the analysis filters of the Trace Window. These filters can be configured specifically for Ethernet packets. Groups can also be created that can be easily activated/deactivated.

Step by Step Procedure: Hide special Ethernet Packets

  1. Open the Trace Window.
  2. Open the analysis filter with in the toolbar.
  3. Drag and drop the desired Ethernet packet to the Pass filter.
  4. Activate the pass filter. Now only the events that correspond to this filter conditions are displayed.
  5. With a double click on the set filter you open the Filter Conditions dialog.
  6. Adapt the filter conditions.
  7. Close the dialog with [OK].

Alternatively you can also use the context menu to set filter.

Detailed View of Ethernet Packets

With the detailed view you get a more detailed interpretation of the packet information. The checksum of Ethernet packets is displayed in the detailed view only if the value is provided by the network hardware.

Step by Step Procedure: Open Detail View

  1. Open the Trace Window.
  2. Select a line with an Ethernet packet ().
  3. Open the detail view with in the toolbar.
