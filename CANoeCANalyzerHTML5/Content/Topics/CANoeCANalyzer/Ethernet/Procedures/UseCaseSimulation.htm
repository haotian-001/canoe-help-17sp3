How to Simulate an Ethernet Network

 

CANoe allows you to simulate one or several Ethernet nodes. The simulated nodes can also use other bus systems or be created as gateway nodes.

Typically, there is a real ECU which requires one or several Ethernet remote peers to implement or test embedded software.

Simulating Ethernet Nodes without Database

If there is no database, the simulated nodes can be created in CANoe. The IP of the node must be configured for the TCP/IP Stack. This includes the MAC address, IPv4 or IPv6 addresses, and possibly the VLANs.

The simulation behavior can be implemented in CAPL or C#. The TCP/IP API is used to establish communication via UDP or TCP.

The application data must be generated by means of programming in CAPL/C#.

Step by Step Procedure

  1. Open the Simulation Setup via the Simulation ribbon tab by pressing the button.
  2. Insert a new network node via the context menu of the network trunk.
  3. Open the TCP/IP Stack dialog via the context menu command TCP/IP Configuration... of the newly inserted block.
    Here you can see the IP configurations of all simulated nodes. The current network node is preselected.
  4. On the VLAN Configuration page, use [Add...] to create one or several VLANs and assign VLAN IDs.

In the following steps, IP can be configured for each VLAN.

  1. For each Ethernet network to which the network node is assigned, there is a page with the IP Configuration.
    Specify a unique MAC address under Ethernet Settings. Manually assigned MAC addresses start with 02:…
  2. Specify now the IP address. To do so, activate IPv4 or IPv6.
  3. Add a new IP address or edit the preset IP address. Make sure to assign IP addresses and net mask or prefix such that the network nodes are in the same subnet and can reach each other. If only VLANs are used, then the IPv4 and IPv6 on this adapter can be deactivated.
  4. Each configured VLAN has its own subpage with the corresponding IP configuration. As in the previous step, you can specify the IP address here.
    Make sure that each VLAN is in the same subnet.
  5. Close the dialog by clicking [OK].
  6. Double click the network node to open a file selection dialog.
  7. Specify here a new filename for the CAPL file.
    Alternatively, a C# file can also be specified here.
  8. Close the dialog by clicking [Open], the CAPL Browser will be opened.
  9. Enter the following CAPL code to send an UDP broadcast packet by pressing a key:

    10. variables
    {
      UdpSocket socket;
    }

    on start
    {
      socket = UdpSocket::Open( 0, 40000 );
    }

    on key '1'
    {
      socket.SendTo( ipGetAddressAsNumber("255.255.255.255"), 40001, "Hello", 5 );
    }

  1. Compile and save the CAPL program.
  2. Start the measurement in the simulated mode (Home ribbon tab).
  3. Press the key <1>. The UDP broadcast packet will be sent.

Simulating Ethernet Nodes with Database

CANoe can use an ARXML or Fibex database to simulate a network node up to the signal level. You can specify signal values for the simulation, for example via panels, Signal Generators or programming. CANoe can perform the transmission via Ethernet according to the definition from the database.

You can assign interaction layer VMODULE files to the network node. They are configured from the database and provide various simulation functionalities. For Ethernet, the following interaction layers are available:

  • AUTOSAR PDU IL (AsrPDUIL.vmodule)
    This interaction layer (IL) generates AUTOSAR PDUs and fills them with signal values. The PDUs are sent according to the defined sending type. This IL also calculates E2E and protects SecOC.
  • AUTOSAR Eth IL (CANoeILNL_AUTOSAR_Eth.vmodule)
    This interaction layer (IL) transmits the PDUs or SOME/IP messages to one or several receivers via UDP/TCP connections. In addition, this IL is responsible for Service Discovery.
  • SOME/IP IL (SOMEIP_IL.vmodule)
    This interaction layer (IL) has the same function as the AUTOSAR Eth IL, however, it only supports SOME/IP and does not support AUTOSAR PDUs.
  • AUTOSAR UDP network management IL (AsrUdpNm.vmodule)
    This interaction layer (IL) is responsible for UDP network management.

Depending on the required functionality, one or several interaction layers can be assigned to the network node. For AUTOSAR PDUs or SOME/IP, for example the AsrPDUIL.vmodule and the CANoeILNL_AUTOSAR_Eth.vmodule must be assigned to the network node.

Step by Step Procedure

  1. Add a database in the Simulation Setup: See How to Create an Ethernet Configuration
    For this, you can use, for example the ARXML file from the CANoe sample configuration PDUBasicAutosar (Open folder).
  2. Open the Simulation Setup via the Simulation ribbon tab by pressing the button.
  3. Insert a new network node via the context menu of the network trunk.
  4. Open the Node Configuration dialog via the context menu command Configuration... of the newly inserted block.
  5. Select a node from the DB Node selection list, for example the VGW node.
  6. Go to the Components page.
  7. Open the file selection dialog by clicking [Add...].
  8. Select here the interaction layer AsrPDUIL.vmodule from the C:\ProgramData\Vector\AddOn Packages\Vector AddOn\vModules directory and close the dialog by clicking [Open].
  9. Repeat the procedure with the interaction layer CANoeILNL_AUTOSAR_Eth.vmodule from the exec64 directory of the CANoe installation.
  10. Close the Node Configuration dialog by clicking [OK].
  11. Open the TCP/IP Stack dialog via the context menu command TCP/IP Configuration... of the newly inserted block.
    Here you can see the IP configurations of all simulated nodes. The current network node is preselected.
  12. Check the IP configuration of the network and the VLANs. The IP addresses and VLANs have been read from the database. The configuration cannot be changed or expanded.
  13. Close the dialog by clicking [OK].
  14. Start the measurement in the simulated mode (Home ribbon tab).
    If you have used the ARXML from the PDUBasicAutosar example, SOME/IP Service Discovery messages are now displayed in the Trace Window. If they are interpreted only as UDP, then the SOME/IP UDP port 30490 still needs to be configured in the protocol settings. See How to Create an Ethernet Configuration.
    For PDUs to be sent, along with the sample database a consumer is also required; this consumer registers at the provider with the Service Discovery.
  15. Stop the measurement.
  16. Configure the second IC node from the database by repeating steps 3 to 10.
  17. Start the measurement in the simulated mode (Home ribbon tab).
    The IC node registers now at the VGW provider and the EMotorStatus PDU is sent cyclically.
  18. Open the Symbol Explorer via the Environment ribbon tab by pressing the button.
  19. Open the Symbol Panel via the context menu command Change Value... of a signal.
  20. Change the signal value.
    The new value will be sent by the IL.
  21. You can also drag and drop the signal from the Symbol Explorer into the Data Window or Graphics Window. The last transmitted signal value will be displayed there.

Creating the Remaining Bus Simulation from the Database with Model Generation Wizard

Model Generation Wizard allows you to create a complete configuration for a remaining bus simulation from a database. The generated configuration includes:

  • the Simulation Setup with the network nodes from the database. The required interaction layer VMODULE files and a CAPL program are assigned to the nodes.
  • panels to influence the signals, service states, and network management.

Note

Add-ons for various OEMs are available. The Model Generation Wizard can create the remaining bus simulation, depending on the selected OEM.

Step by Step Procedure

  1. Open the Model Generation Wizard via the Simulation ribbon tab by pressing the button.
  2. Select the desired modeling from the OEM selection list, for example Vector Modeling.
  3. Select a database.
    For this, you can use, for example the ARXML file from the CANoe sample configuration PDUBasicAutosar (Open folder).
  4. Select an output directory.
  5. You can influence generation on the Settings page.
  6. On the Nodes page, you can exclude individual nodes from generation. Nodes which are available as real ECUs can be deactivated here.
  7. Switch to the Generation page and press the [Start Generation] button.
    The Disclaimer dialog appears. Confirm it by clicking [I accept].
  8. Once the generation is completed, close the Model Generation Wizard.
  9. Start the measurement.
  10. If the network management is defined in the database, the network must be started by means of the Ignition switch on the MainPanel panel.
