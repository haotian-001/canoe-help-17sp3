How to Enable Encrypted DoIP Communication via TLS

 

With CANoe version 14, you can enable encrypted communication via TLS for the built-in diagnostic channel.

Note

With CANoe version 14, DoIP communication via TLS is not supported in the following cases:

  • In combination with the CAPL Callback interface.
  • For networks where diagnostics via common DoIP/HSFZ gateway was activated.

Enabling Encrypted Communication via TLS for the Diagnostic Tester

Step by Step Procedure

  1. Optional: By default, the diagnostic windows like Diagnostic Console, Fault Memory Window and Diagnostic Session Control will use the Shared CANoe Stack for communication. If you want to force the diagnostic tester to use an individual stack, you need to create a dedicated simulation node for this purpose in the Simulation Setup.
  2. Open the Security Configuration dialog and change to the Stacks page.
  3. At the stack over which the diagnostic tester should communicate, enable the Use Security check box and select the desired security profile for the TLS communication. Close the dialog clicking the OK button.
  4. Open the Diagnostic/ISO TP Configuration dialog. At the network where the diagnostic tester should communicate, add a diagnostic description. Make sure that in the Usage of the Diagnostic Description area, the Diagnostic tester check box is activated.
  5. Switch to the DoIP/HSFZ Settings page for the diagnostic description just added.
  6. In the Tester area, for Adapter select an IP address at the TCP/IP stack you want to use for the diagnostic tester, i.e. the stack where you enabled the Use Security check box in step 3. After that the selected security profile for this stack and a new check box Enable TLS should be visible.
  7. In the Tester area activate the Enable TLS check box.

Enabling Encrypted Communication via TLS for the Diagnostic ECU Simulation

Step by Step Procedure

  1. Create a simulation node for the diagnostic ECU simulation, if no simulation node is available.
  2. Open the Security Configuration dialog and change to the Stacks page.
  3. At the stack over which the simulated ECU should communicate, enable the Use Security check box and select the desired security profile for the TLS communication. Close the dialog clicking the OK button.
  4. Open the Diagnostic/ISO TP Configuration dialog. If the diagnostic description is not already added for the diagnostic tester at the network where the ECU should communicate, add a diagnostic description. Make sure that for this diagnostic description in the Usage of the Diagnostic Description area, the Simulation by check box is enabled.
  5. Switch to the DoIP/HSFZ Settings page for the diagnostic description just added. In the ECU area, for Adapter select an IP address at the TCP/IP stack you want to use for the diagnostics ECU simulation, i.e. the stack where you enabled the Use Security check box in step 3. Now, the selected security profile for this stack and a new check box Enable TLS should be visible, as well as a input box Certificate.
  6. In the ECU area activate the Enable TLS check box.
  7. Enter the certificate name to be used by the ECU simulation and be careful not to add leading or trailing blanks by mistake. You can find the certificate names which can be used in the Security Manager when selecting the corresponding security profile.
