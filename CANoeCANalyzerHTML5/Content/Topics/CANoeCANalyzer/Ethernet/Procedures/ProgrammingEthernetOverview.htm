CAPL Function Overview

 

Link Layer

The CAPL functions of the link layer can be used for analysis, stimulation, test and fault injection. For the simulation, CAPL functions of the internet and transport layers or the application layers should be used in order to make the generation of CAPL programs easier.

Analysis and Simulation/Stimulation

on ethernetPacket can be used to receive and evaluate individual Ethernet packets. Selectors allow the protocol fields to be read. The ethernetPacket object is used to generate Ethernet packets and output( ethernetPacket ) is used to send them.

To generate a busload, ethStartPacketGenerator can be used.

If AUTOSAR I-PDU are transmitted via Ethernet, ethernetPacket::PDUCountand ethernetPacket::GetPDUcan be used to read a PDU.

The handler on ethernetStatus can inform a CAPL program if an Ethernet link is established or removed. The Eth object or the Ethernet statistics system variables can be used to query the Ethernet link status at any time.

Test

In test configurations/modules, testWaitForEthernetPacket can be used to wait for specific Ethernet packets.

Internet and Transport Layer

Using the TCP/IP-CAPL functions, UDP and TCP sockets can be generated and it is possible to communicate with them. The simulated node or the test module is always an active terminal station. To establish a connection, the IP addresses and port numbers of the local and remote endpoints must be specified. The configured TCP/IP stack establishes the connection. Afterwards, the CAPL program can send the user data via UdpSendTo or TcpSend and receive them in CAPL callback functions.

The TLS/IP-CAPL functions also allow the establishment of encrypted connections.

Application Layer

For the different application protocols, individual functions are made available in CAPL.

on pdu and on signal can be used to evaluate AUTOSAR I-PDUs or signals. It is even possible to read out signal values with $ signal names. An ARXML database should be used to be able to use the symbolic designators of the PDUs and signals. In order to be able to send AUTOSAR I-PDUs, the interaction layers AsrPDUIL2 and CANoeILNL_AUTOSAR_ETH must be integrated. They are completely configured from the assigned database node. The transmission of PDUs can be triggered by triggerPDU and the $ signal name.

Values from SOME/IP events and notifications can be received with an on service signal and read with a $ service signal name. An ARXML or Fibex database must be integrated. To be able to send SOME/IP events, methods or notifications, the CANoeILNL_AUTOSAR_ETH or SOMEIP_IL must be integrated. The assignment with a $ service signal name can trigger the transmission of an event. For methods, other CAPL functions are available.

The AUTOSAR UDP Network Management can be simulated using the AUTOSAR UDP NM Interaction Layer.

The AVB Interaction Layer can be used to send and receive audio and video streams via AVTP and RTP.
