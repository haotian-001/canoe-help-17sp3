Stimulate Ethernet Packets

 

To establish functioning communication with Ethernet, usually it is necessary to send several Ethernet packets or to establish a connection to the receiver. To send a UDP packet, it is necessary to query the MAC address of the receiver beforehand using an ARP request.

Therefore, a TCP/IP Stack is usually used with Ethernet. The application lets the TCP/IP Stack handle the lower protocol layers and also generate and send individual Ethernet packets. Only application data is transferred to the TCP/IP Stack or written into the socket and the TCP/IP Stack ensures that the data is transmitted to the receiver.

During the development of a ECU, it is often helpful to send individual Ethernet packets and selectively influence certain protocol fields, for example during debugging or to generate erroneous Ethernet packets.

Sending an Individual Ethernet Packet with Ethernet Packet Builder

The Ethernet Packet Builder can be used to create individual Ethernet packets in a table view and edit all protocol fields of a packet. User data can be edited in a hex view. The Ethernet packets can then be sent at the push of a button.

Step by Step Procedure

  1. Open the Simulation Setup via the Simulation ribbon tab by pressing the button.
  2. Insert a new Ethernet Packet Builder via the context menu of the network trunk.
    You can also insert an Ethernet Packet Builder in the System View via the context menu of the Interactive Generators.
  1. Double-click the Ethernet Packet Builder to open it.
  2. Select a UDP packet via the selection list of [New Packet].
  3. Select the packet and edit the protocol fields (e.g. MAC and IP addresses, port numbers) in the Packet Information area.
  4. Start the measurement.
  5. Press the [Send] button in the Ethernet Packet Builder to send the Ethernet packet.

Sending an Individual Ethernet Packet with CAPL

With CAPL, sending Ethernet packets can be controlled by events, for example by pressing a key, with timer or upon receiving a different Ethernet packet. The protocol fields in the protocol headers can be set with an assignment.

Step by Step Procedure

  1. Open the Simulation Setup via the Simulation ribbon tab by pressing the button.
  2. Insert a new network node via the context menu of the network trunk.
  1. Double click the newly inserted block to open a file selection dialog.
  2. Specify here a new filename for the CAPL file.
  3. Close the dialog by clicking [Open], the CAPL Browser will be opened.
  4. Insert the following CAPL code:

    5. on key '1'
    {
      ethernetPacket pkt;

      pkt.source.ParseAddress( "02:00:00:00:00:01" );
      pkt.destination.ParseAddress( "02-00-00-00-00-02" );

      // initialize packet with IPv4 and UDP protocols
      pkt.udp.Init();

      // set IPv4 addresses
      pkt.ipv4.source.ParseAddress( "192.168.1.1" );
      pkt.ipv4.destination.ParseAddress( "192.168.1.255" );

      // set UDP ports
      pkt.udp.source = 40001;
      pkt.udp.destination = 40002;

      // set UDP payload
      pkt.udp.SetData( 0, "Hello", 5 );

      // calculate UDP and IPv4 checksum and send Ethernet packet
      pkt.CompletePacket();
      output( pkt );
    }

  1. Compile and save the CAPL program.
  2. Start the measurement.
  3. Press the <1> button. The Ethernet packet will be sent by the CAPL program.
