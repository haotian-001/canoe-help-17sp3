How to Log Ethernet Traffic

 

Ethernet, as the other bus systems, is recorded with the Logging Block. The Ethernet packets and partly further information like the link status and faulty Ethernet packets are recorded.

As usual, raw data is recorded for Ethernet. The interpretation of higher protocols, which are displayed in the Trace Window as additional lines with the same time stamp, is not stored. This information is generated again from the Ethernet packets by CANoe during replay or import. This applies to AUTOSAR PDU and SOME/IP data, for example.

Compared to Windows network adapters, Vector Ethernet interfaces record additional information, such as reliable timestamps, frame duration and checksum. Make sure that the Extended Ethernet Logging is activated (Options dialog). In addition, you can synchronize VN interfaces via hardware cable.

The following logging formats are supported for Ethernet:

  • BLF
    All Ethernet bus events and the events of other bus systems are recorded. If the Extended Ethernet Logging is activated, additional information, such as frame duration and frame checksum is also recorded.
  • ASCII
    The same information as with BLF is recorded. With Ethernet, however, the ASC logging files are significantly larger than with BLF.
  • MF4 (frame logging)
  • PCAP
    Only Ethernet packet events are recorded, other bus systems are not recorded. Besides, no information on direction and channels is recorded with the Ethernet packets.
    Use this format only if you cannot use the other formats. Also, the filename should contain information such as the recorded channel and port.
  • PCAPNG
    Only Ethernet packet events are recorded, other bus systems are not recorded. No direction is recorded with the Ethernet packets.

Step by Step Procedure: Logging the Ethernet Data Traffic

  1. Insert a Logging Block via the Analysis ribbon tab.
  2. For this Logging Block, open the Logging File Configuration dialog by pressing the button on the Analysis ribbon tab.
  3. Specify a target directory and a target file
  4. Select BLF as file format.
  5. Close the dialog by clicking [OK].
  6. Open the Options dialog via the Quick Access Toolbar ().
  7. On the Ethernet page, select the Ethernet logging mode Extended Ethernet Logging to record all information of the Vector Ethernet interfaces.
  8. Close the dialog by clicking [OK].
  9. Start measurement to record bus traffic in the configured logging file.

Note

If the Extended Ethernet Logging is used, the Ethernet packets can no longer be replayed with CANoe 9.0 SP2 and older.
