Getting Started With Ethernet VN Devices

 

Compared to serial bus systems such as CAN, FlexRay or LIN, Ethernet-based systems are typically based on a network topology. This often makes measurement as well as simulation and test tasks more complex than we know from serial bus systems. In addition, Ethernet-based systems often use much higher data rates. The freely configurable Vector Ethernet VN devices have been designed to meet the above-mentioned requirements. For example, complex hardware filters can be defined to provide tools such as CANoe or CANalyzer with only the data that is of interest in the current application. Depending on the measurement tasks, the Ethernet VN devices can be freely configured to meet the various measurement, simulation or test use cases. This free configurability brings a new level of complexity to the setup of Ethernet VN devices that we have never seen before. While the assignment of an Application Channel provided by the tool was sufficient for the setup of a CAN interface, several steps are necessary for the configuration of the Ethernet VN devices.

This site describes the procedures for configuring the Ethernet VN devices as well as the precautions required in the Vector tools to be able to operate the Ethernet VN devices. The focus is on the following use cases:

  • Measurement
  • Simulation
  • Diagnostics
  • Media Conversion

Acronyms

Term

Description

PHY

A PHY is the component that implements the physical layer portion of the Ethernet. A PHY connects a link layer device (often called Medium Access Control) to a physical medium such as copper cables.

Vector Ethernet VN devices offer different number of ports. Behind each port a PHY is installed.

TAP

A Test Access Point:

  • is used to measure data on an Ethernet cable
  • connects two physical ports of the Ethernet VN device with each other. Packets received on one port are automatically forwarded to the other port. The latency is thereby vanishingly small.
  • also forwards all packets to the application (CANoe)

Prerequisites

Hardware

For the consideration of the use cases, a VN5640 device from Vector is used. Of course, the VN5640 can be replaced by any Vector Ethernet VN device if it provides a sufficient number of ports.

System Design

When considering the various use cases, the following network topology serves as a basis:

The example network consists of four simple ECUs (VGW, IC, LRR and CAMF) each connected to a switch. Switch_1 is integrated in the ADAS ECU, while Switch_2 is an external, independent switch. With the free port Switch_1 provides a diagnostic port.

Note

In the automotive environment, switches are typically installed in the ECUs (see in this example ADAS ECU).

Measurement

Example

The communication between Switch_1 and Switch_2 is to be measured with CANoe.

This measuring point (M) enables measurement of all Ethernet packets communicated via this link. Ethernet packets that are communicated exclusively via other links (e.g. VGW sends to IC) are not captured. You have to set up further measuring points for this.

Cabling

Direct intervention using a Y-cable (as we know it from CAN) is not possible, since there is always a point-to-point connection between two Ethernet Physical Layer connections (PHYs). Furthermore, the connection between Switch_1 and Switch_2 must be broken, and the Vector VN device must be interposed.

System setup and Cabling

Note

Two ports on the Ethernet VN device are required for each link to be measured.

Hardware Configuration

Step by Step Procedure

A Test Access Point (TAP) must be configured on the Ethernet VN device.
Please proceed as follows to set up a TAP:

  1. Make sure that the Ethernet VN device is connected to the computer.

  2. Open the Vector Hardware Config dialog from the Windows start menu or Control Panel, Hardware and Sound group|Vector Hardware.

  3. Select the desired Ethernet VN device with the right mouse button and click on the context menu command Ethernet device configuration.

  4. The Vector Ethernet Device Configuration dialog is opened. If there is already a hardware configuration stored on the device this configuration will be displayed in the middle part of the dialog.

  5. Create a new configuration via the File ribbon tab|New.

Note

This command will only create an empty configuration in the configuration dialog. The current configuration stored on the Ethernet VN device will not be overwritten.

  1. Select a link segment in the Segments Window and drag and drop it into the blue network placeholder on the Layout page.

  2. A network with the default name Ethernet1 is created automatically. In this use case we analyze the ADAS network. Therefore select the network on the Layout page and change the network name in the Properties Window on the right side of the dialog. Alternatively, you can also double-click on the name on the Layout page.

    For information on further link settings in the Properties Window, please refer to the help of the Vector Ethernet Device Configuration dialog. To do this, simply press <F1>.

Note

CANoe automatically creates the necessary reference to the Ethernet VN device when both sides use the same Ethernet network names. The network name must therefore also be specified when creating the CANoe configuration.

  1. Assign the desired physical ports of the Ethernet VN device to the inserted link. Use the ports to which the desired network nodes (in this example the real switches Switch_1 and Switch_2) are connected to the Ethernet VN device. Select the desired ports from the Ports Window on the left side of the dialog and assign them to the inserted link segment via drag and drop.

    Assigning the two physical ports means that the inserted link segment will henceforth function as a TAP. This is displayed in the Properties Window on the right side of the dialog when the Link segment is selected.

Note

Additional links must be added to the same network (ADAS) if further links of this network must be measured.

  1. Give the assigned ports the names of the connected switches. You can change the name in the Properties Window on the right side of the dialog when the port is selected. Alternatively, double-click on the port on the Layout page.

    The specified name is displayed on the Layout page of the dialog. The number displayed in square brackets corresponds to the number of the port you have selected from the Ports Window before. You can find the numbers on the Ethernet VN device housing too. Changing the port number is only possible indirectly. If another port of the Ethernet VN device is to be used, you must delete the already assigned port and must assign a new port. For information on further port settings in the Properties Window, please refer to the help. To do this, simply press <F1>.

  2. Now select the ports one after the other on the Layout page and choose the corresponding Channel Mode in the Properties Window on the right side of the dialog.

Note

If the port on the ADAS device on Switch_1 is operated as IEEE 100BASE-T1, Slave, for example, IEEE 100BASE-T1, Master must be set on the used Ethernet VN device port.

Otherwise, no link can be established.

  1. Save the current configuration via the Home ribbon tab| Write. The configuration is then stored as default configuration on the Ethernet VN device, a corresponding message is displayed. Additionally, the configuration can be stored in a file via the File ribbon tab|Save.

  2. Close the Vector Ethernet Device Configuration dialog.

  3. The configuration of the Ethernet VN device is complete, the second step is now to create the CANoe configuration.

CANoe Configuration

Step by Step Procedure

Proceed as follows to create a CANoe configuration:

  1. Start CANoe.

  2. Create a new CANoe configuration via the File ribbon tab|New. Select the template Ethernet (Simulation Setup). A new CANoe Ethernet configuration will be created. The configuration already contains all windows necessary for analysis.

  3. Assign the exact network name to the CANoe configuration that you previously specified in the hardware configuration (in this example ADAS). To do this, select the Ethernet1 entry in the System View of the Simulation Setup with the right mouse button and click on the context menu command Rename....

  4. The changed network name is then displayed in the Simulation Setup.

Note

CANoe is now automatically connected to the Ethernet VN device via the same network name when the measurement is started. If the names are not identical, the two network names must be mapped via Hardware ribbon tab|Channel Mapping.

  1. Open the Port Configuration dialog via the Hardware ribbon tab|Port Configuration.

    With the hardware connected, CANoe recognizes the configured segments together with assigned ports. The ports previously configured on the Ethernet VN device are displayed in gray in the left area of the dialog.

  2. Packets received on a physical port or sent via a physical port are not displayed in CANoe until the port has been activated in CANoe. To do this, select the port in the Measurement Ports group with the right mouse button and click on the context menu command Activate.

  3. Close the Port Configuration dialog with [OK] after all desired ports have been activated.

  4. Start the measurement via the Home ribbon tab|Start.

Note

Packets received on a port are marked as Rx packets. If a packet is sent from the Ethernet VN device via a port (in the TAP case this corresponds to forwarding a packet from one port to the other) this packet is marked as Tx packet. Both the direction and the name of the port are displayed in the Trace Window.

Simulation

Example

The ECUs LRR, CAMF, IC and the Switch_2 are not available and are to be simulated with the aid of CANoe. The network topology should be retained, so the ECUs LRR and CAMF should continue to be connected to the ADAS ECU (Switch_1) via Switch_2. With an abstracted network topology, the ECUs could also be connected directly to Switch_1 if the number of ports is sufficient.

Cabling

Two ports are required on the Ethernet VN device. The simulation of Switch_2 is realized directly on the Ethernet VN device.

LRR and CAMF are connected to Switch_2 via so-called virtual ports. IC is connected to the real ADAS ECU via a direct connection. This results in the following schematic representation.

Step by Step Procedure

  1. Make sure that the Ethernet VN device is connected to the computer.

  2. Open the Vector Hardware Config dialog from the Windows start menu or Control Panel, Hardware and Sound group|Vector Hardware.

  3. Select the desired Ethernet VN device with the right mouse button and click on the context menu command Ethernet device configuration.

  5. Create a new configuration via the File ribbon tab|New.

  1. Select a switch segment in the Segments Window and drag and drop it into the blue network placeholder on the Layout page.

  1. A network with the default name Ethernet1 is created automatically. This name can be changed if necessary. In our use case we analyze the ADAS network. The network name can be changed in the Properties Window on the right side of the dialog if the network is selected on the Layout page. Alternatively, you can also double-click on the name in the Layout page.

    For information on further switch settings in the Properties Window, please refer to the help of the Vector Ethernet Device Configuration dialog. To do this, simply press the <F1>.

Note

CANoe is connected to the Ethernet VN device via the network name. The network name must therefore also be specified when creating the CANoe configuration.

  1. Select the switch segment on the Layout page and change the name to Switch_2. The name can be changed in the Properties Window on the right side of the dialog.

  2. Assign Switch_2 the physical port of the Ethernet VN device to which the ADAS ECU (Switch_1) is connected. Select the desired ports from the Ports Window on the left side of the dialog and assign it to Switch_2 via drag and drop.

  3. Assign the name of the connected network node (in this example this is port 3 of Switch_1) to the assigned port. The name can be changed in the Properties Window on the right side of the dialog when the port is selected on the Layout page. Alternatively, you can double-click on the port on the Layout page.

    The specified name is displayed on the Layout page of the dialog. The number displayed in square brackets corresponds to the number of the port you have selected from the Ports Window before. You will find this numbering also on the Ethernet VN device itself. Changing the port number is only possible indirectly. If another port of the Ethernet VN device is to be used, you must delete the already assigned port and must assign a new port. For information on further port settings in the Properties Window, please refer to the help. To do this, simply press <F1>.

  4. Now select the port on the Layout page and choose the corresponding Channel Mode in the Properties Window on the right side of the dialog.

Note

If the port on the ADAS device on Switch_1 is operated as IEEE 100BASE-T1, Slave, for example, IEEE 100BASE-T1, Master must be set on the Ethernet VN device port used.

Otherwise, no link can be established.

Assigning virtual ports for the simulated nodes to the Switch_2 segment is not necessary. CANoe will automatically create the virtual ports required for the connection of the simulated nodes at the start of measurement (see CANoe configuration below).

  1. Select a link segment in the Segments Window and drag and drop it into the blue ADAS network in the middle Layout page.

  1. Assign a port to the link segment, change its name, and set the appropriate channel mode (see steps 9 to 11).

  1. Save the current configuration via the Home ribbon tab|Write. The configuration is then stored as default configuration on the Ethernet VN device, a corresponding message is displayed. Additionally, the configuration can be stored in a file via the File ribbon tab|Save.

Diagnostic

Example

A direct access to the diagnostic port at Switch_1 is to take place. For this, a direct connection must be set up in the Ethernet VN device.

Cabling

Only one port on the Ethernet VN device is required for the direct connection of a tool. Diagnostic applications, for example, can be realized via a standard Ethernet cable (100Base-Tx / 1000Base-T). Corresponding Physical Layer connections (PHYs) including RJ45 connectors are already provided on the Ethernet VN devices for this purpose.

Note

The diagnostic application was selected here only representatively. Of course, any Ethernet control device can be connected directly to the Ethernet VN device and thus connected to the tool.

Hardware Configuration

A direct connection must be configured in the Ethernet VN device. A direct connection

  • comprises a physical port of the Ethernet VN device. The packets received on this port are forwarded to CANoe.

  • allows to send Ethernet packets to the connected ECU..

To set up a direct connection, please proceed as follows:

  1. Make sure that the Ethernet VN device is connected to the computer.

  2. Open the Vector Hardware Config dialog from the Windows start menu or Control Panel, Hardware and Sound group|Vector Hardware.

  3. Select the desired Ethernet VN device with the right mouse button and click on the context menu command Ethernet device configuration.

  4. The Vector Ethernet Device Configuration dialog is opened. If there is already a hardware configuration stored on the device this configuration will be displayed in the middle part of the dialog.

The network name can be changed in the Properties Window on the right side of the dialog if the network is selected on the Layout page. Alternatively, you can also double-click on the name on the Layout page.

    For information on further link settings in the Properties Window, please refer to the help of the Vector Ethernet Device Configuration dialog. To do this, simply press <F1>.

Note

The Vector tools are connected to the Ethernet VN devices via the network name. The network name must therefore be specified when creating the CANoe configuration in the tool.

  1. Assign the desired physical port of the Ethernet VN device to the inserted link. Use the ports to which the CAMF ECU is connected to the Ethernet VN device. Select the desired port from the Ports Window on the left side of the dialog and assign it to the inserted link segment using drag and drop.

    Assigning the physical port means that the inserted link segment now functions as a direct connection. This is displayed in the Properties Window on the right side of the dialog when the link segment is selected.

Accordingly, CANoe requires only a virtual port that the application automatically creates at the configured link segment on measurement start. No further configuration step is required. Media Conversion

The configuration of a Media Converter is analogous to the configuration of the measurement use case. Only when assigning the physical ports to the segment (see step 10 of the hardware configuration), the corresponding physical ports, for example IEEE 100BASE-T1 (Automotive Ethernet), Master and IEEE 100BASE-TX (Standard Ethernet), must be selected.
