How to Test an Ethernet ECU

 

With Ethernet, the basic concepts for test sequences are supported in CANoe. In tests, it is possible to affect the Ethernet communication at various protocol levels, for example to stimulate protocol errors.

Link Layer

As in network nodes, CAPL and C# APIs in test units or test modules can be used to send and receive individual Ethernet packets. The status of the Ethernet link can be affected with ethSetLinkStatus. The status of the Ethernet link and the statistical values can be queried via the Eth object.

The Vector network interfaces provide a stress function which can be used to cyclically send an Ethernet packet. The ethStartPacketGenerator CAPL function controls this packet generator.

In test sequences, the testWaitForEthernetPacket CAPL function can be used to wait for certain Ethernet packets.

Internet and Transport Layer

The TCP/IP API in CAPL and C# can be used in test units or test modules as in network nodes. In this case, the tester is the terminal station of a connection.

To make a fault injection with the TCP/IP Stack, the CAPL callbacks OnIpSendPrepare and OnIpReceivePrepare can be implemented. This allows you to change the protocol fields of Ethernet packets which are sent or received by the TCP/IP Stack.

Application Layer

To test AUTOSAR PDUs, the general PDU function can also be used for Ethernet.
