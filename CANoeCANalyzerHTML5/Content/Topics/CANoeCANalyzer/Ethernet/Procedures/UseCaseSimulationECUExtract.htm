How to Simulate an Ethernet Network with AUTOSAR ECU Extract

 

In an AUTOSAR ECU extract, an individual ECU is defined. Usually, the ECU should be implemented or tested for this ECU extract. For the remaining bus simulation, however, remote peers for the ECU are required; these stations are not included in the ECU extract.

To enable remaining bus simulation for CANoe with an AUTOSAR ECU extract, an additional database node is created during reading. This node has the name of the ECU with the addition _SimulatedPeerNodes.

If an ARXML contains only one ECU, this ECU is identified as an ECU extract. CANoe collects all connections defined without remote peer and assigns them to the _SimulatedPeerNodes database node. IP addresses which lie within the same subnet as the addresses of the actual ECU are assigned to this node. Services consumed by the ECU are provided by the _SimulatedPeerNodes database node. Conversely, services provided by the ECU are consumed by the _SimulatedPeerNodes database node.

Note

IP addresses selected by the _SimulatedPeerNodes database node do not correspond to the IP addresses in the real network. To enable remaining bus simulation with the network settings of the real network, a complete ARXML is required.

The remaining bus simulation with an AUTOSAR ECU extract is created as with a complete ARXML, see How to Simulate an Ethernet Network.
