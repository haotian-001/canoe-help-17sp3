Network Settings

 

On this page you can modify the settings for each Ethernet channel.

This page is available for

  • CANoe TCP/IP Stack with activated Use CANoe internal TCP/IP stack option
  • Network nodes with activated Individual TCP/IP stack instance option

Ethernet Settings

Enter the MAC address that is used as sender address for Ethernet packets sent from this network node. Consider that this MAC address can be used only once in the network.

Note

If a new network node is inserted a generated MAC address is assigned. Here it is ensured that this MAC address is not assigned in the simulated network yet.

If the MAC address conflicts with the MAC-ID of a connected hardware, you have to change the MAC address.

If a node from a FIBEX file is selected for the network node and the MAC address is defined there, the MAC address cannot be changed.

  • Use MAC-ID of Ethernet adapter

    Note

    This option is available only on selection of the CANoe network stack.

    If you use this option you don't need to configure the MAC address manually, it is allocated from the network adapter.

  • MAC address
    Here you can manually enter the MAC address.

IPv4 Settings

  • IPv4 Enabled
    Here you enable IPv4 on the TCP/IP stack. This setting is enabled by default.
  • Possible settings of the IP address
    If the node obtains its address from a DHCP sever it is not ready to communicate via the TCP/IP stack until it received a valid IP address. Therefore it cannot communicate immediately after measurement start.
  • Use the following IP address
    The TCP/IP stack uses the given IP address, subnet mask and default gateway.
  • Obtain IP address automatically with DHCP
    The TCP/IP stack tries to obtain IP address, netmask and default gateway from a DHCP server.
  • Obtain preferred IP address with DHCP
    The TCP/IP stack tries to obtain the following preferred IP address from a DHCP server.
    It may happen, that the DHCP server doesn't assign the requested IP address but a different one.
  • Default-Gateway
    Enter the IPv4 address of the default gateway in the network. If an Ethernet packet is sent to an IPv4 address in another sub net, the packet is sent to the MAC address of the default gateway.

  • IPv4 Address and Mask
    Enter one or more IPv4 addresses and the net masks to identify the sub nets. Consider that the IPv4 addresses can be used only once in the network.
    Use [Add] to enter a new IPv4 address and [Remove] to remove an IPv4 address.

    Note

    If a new network node is inserted a generated IPv4 address conform to the scheme 192.168.<channel number>.<running number> is assigned. Here it is ensured that this address is not assigned in the simulated network yet.

    But if the address conflicts with the address of a connected hardware, you have to change the address.

IPv6 Settings

  • IPv6 Enabled
    Here you enable IPv6 on the TCP/IP stack. This setting is deactivated by default.
  • Standard-Gateway
    Enter the IPv6 address of the default gateway in the network. If an Ethernet packet is sent to an IPv6 address that is not in the local link, the packet is sent to the MAC address of the default gateway.
  • IPv6 Address and Prefix
    Enter one or more IPv6 addresses and the prefixes to identify the sub nets. Consider that the IPv6 addresses can be used only once in the network. If you don't assign a link local address, a link local address is automatically assigned on measurement start by auto configuration.
    Use [Add] to enter a new IPv6 address and [Remove] to remove an IPv6 address.

Note

If a node from a FIBEX file is selected for the network node, the IPv4 and IPv6 settings cannot be changed.

Advanced Settings

Maximum Transmission Unit (MTU) sets the maximum transmission unit of the interface. By default this value is set to 1500 byte. It can be reduced down to 72 byte.

MTU: maximum Ethernet payload size.
