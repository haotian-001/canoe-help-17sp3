Ethernet Migration Wizard

 

VN Ethernet interfaces from firmware version 11.1 support network-based access. It is recommended to replace configuration using channel-based network access with configurations using network-based access.

When loading a channel-based CANoe configuration that is in Real Bus mode, a window appears from which you can start the migration wizard. Prerequisite is that a driver version 11.2 or higher is installed. You can also perform the migration later. However, the window appears every time a non-migrated configuration is loaded.

Support/Knowledge Base

If you have problems during or after the migration, please contact Vector Support or use the Vector Knowlege Base (Category: CANalyzer/CANoe|Options|Ethernet (IP)|Migration Wizard).

Migration Wizard

The Migration Wizard helps you in 3 steps to migrate your channel-based CANoe and interface configuration to a network-based configuration.

On the Welcome to the Migration Wizard page, click [Next>] to begin the migration with step 1.

With [Cancel] you can cancel the migration at any step.

Step 1: Select Device

In the first step, you must select the interfaces (devices) which provide the application channels needed by CANoe (e.g. VN5640 (001146) provides Eth1, VN5640 (006065) provides Eth2 and Eth3, ...).

The required application channels are displayed in Required tool application channels.

The Available Devices list displays the interfaces that are either currently connected or were connected to your computer in the past. The application channels configured on the interfaces are also displayed.

With [...] you can add your own VASET files that can contain one or more interfaces and the application channels configured on them.

Select from the interface list which interfaces (devices) provide the required application channels. Provided application channels are displayed bold in Required tool application channels. The Migration can only be performed if all application channels are provided.

Example

Click [Next>] to see an overview of issues that may occur during migration.

Step 2: Issue Report

In the second step, you get an overview of problems that may occur during migration.

For example hardware filters will be migrated, but are disabled because the filter concept has changed. You might have to redefine the filters to take full benefit of the new filter concept.

It could also happen that you will be instructed to use another hardware connector instead of the one you were using now.

Click [Next>] to create a network-based hardware configuration.

Cross Reference

You can find information about the new filter concept in the Ethernet Device Configuration help.

Step 3: Done

With [Finish] you will conclude the migration process. The new interface configuration files (*.xml) will be saved next to your CANoe configuration. In addition the interface configuration will be included in your CANoe configuration.

The interface configuration contains the serial number of the interface on which it should be running. If you want to use the configuration on another interface, you have to load the configuration file manually in the Ethernet Device Configuration and write it to the corresponding interface. If you save this configuration back to the configuration file (*.xml), the new interface is used when you load the CANoe configuration.

Cross Reference

You can open the Ethernet Device Configuration help directly in the dialog.

Save the CANoe configuration to take over the changes.

Manual Migration

Instead of using the Migration Wizard, you can also make a manual migration.

First select the network-based access in the Options dialog.

Open in the Network Hardware Configuration/Vector Hardware Config dialog for your interface via the context menu the Ethernet Mode Configuartion dialog and select Network-based mode.

Open for your interface via the context menu the Ethernet Device Configuration and configure the interface.

If you save the configuration in an XML file you can included it into your CANoe configuration in the Network Hardware Configuration dialog.
