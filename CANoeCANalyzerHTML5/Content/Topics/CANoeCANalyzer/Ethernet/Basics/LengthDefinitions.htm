Length Definitions

 

Packet Length

The packet length specifies the number of data in bytes that are typically sent from the interface hardware to CANoe.The packet length depends on the network type.

For Ethernet the packet length includes the fields beginning at the destination MAC address to the end of the payload or the padding bytes. A packet length less than 60 byte are not allowed for Ethernet (Ref. IEEE Std 802.3TM-2008).

Frame Length

The frame length includes the number of bytes of data contained in the Ethernet header, the Ethernet user data and the frame checksum. The frame length is 4 bytes longer than the packet length.

Ethernet Size

Note

The Ethernet size is supported for the network type Ethernet only.

The Ethernet size is specified by the packet length minus the Ethernet header. It might be called as payload length of the Ethernet packet also. The Ethernet header has a length of 14 or 18 bytes and includes the MAC addresses of sender and receiver, the field Ethertype and the field VLAN-Tag (if available).

Payload Length

The payload length specifies the number of data in bytes that are contained in the highest protocol that is interpreted by CANoe. So the payload length always refers to the protocol that is displayed in the Protocol column of the Trace Window.

For application protocols with no own protocol header (e.g. SOME/IP), the payload length must be set to the same length as the previous protocol payload length.

If CANoe does not interpret a further protocol within an Ethernet packet, the payload length corresponds with the Ethernet size and eth is displayed in the Protocol column of the Trace Window.

Physical Length

The physical length specifies all data of a packet that are transmitted on the physical medium. Typically not all physically transmitted data are sent by the network adapter, so CANoe does not support this size.
