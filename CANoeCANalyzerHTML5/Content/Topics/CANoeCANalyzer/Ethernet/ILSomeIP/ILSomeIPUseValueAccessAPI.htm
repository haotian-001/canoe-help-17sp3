SOME/IP IL: Value Access API

 

Based on the following example, it is explained how you can use the Value Access API to access values of a SOME/IP message in the CAPL program of a simulated node.

The example was composed in a non-standardized notation. To actually use it in CANoe, the example would have to be transferred to a FIBEX database, and this database is then assigned to the network to which the simulated node is connected.

Example

EVENT onValueChanged(ComplexDatatype newValue)
STRUCTURE ComplexDatatype {
  enumeratedField ENUMERATION { off = 0, on = 1 },
  utf8String UTF8String,
  arrayField[0..100] UNION {
    numberChoice INT32,
    floatChoice FLOAT64
  }
}

Derive the Designator of an Event or a Method Call

To use the Value Access API to access a value of an event or of a method call that is contained in a SOME/IP message, you must derive the associated designator from the FIBEX database. The rules are oriented towards commonly used programming languages such as C.

The following cases occur in relation to the presented example:

  • The parameters of an event or method call have a designator. In the presented example, the designator newValue is valid for the event onValueChanged.
  • If a designator belongs to a STRUCTURE, then you can access their fields by appending a dot to the designator for this STRUCTURE followed by the field name. Therefore, in the example shown, the designator newValue.enumeratedField would be valid for the event onValueChanged.
  • If the designator belongs to an array, then you can access an array element by appending the index of the element in square brackets to the designator. For example, you could use newValue.arrayField[0] to access the first of the possible hundred UNIONs within the event onValueChanged.
  • If the designator belongs to a UNION, then you can access the data type of a selection by appending a dot to the designator followed by the designator of the choice. In the example shown, the designator newValue.arrayField[0].numberChoice would be valid for the event onValueChanged.

Access to Values of a SOME/IP Message via CAPL

SOME/IP messages can be received and sent in the CAPL program of a simulated node.

A message contains an event or a method call here. The valid designators for accessing the values of an event or method call were described in the previous section. The values can be passed to the SomeIpSetValue… and SomeIpGetValue… functions (e.g. SomeIpSetValueLong and SomeIpGetValueLong).

Functions with the suffix Data() always set and read the raw data. The representation of the values that are returned by the CAPL functions with other suffixes in response to requests, and the representation expected when setting values, depends on the associated data type. Additional Information

  • The number of elements in an array cannot be set. Instead, the number is incremented as necessary when writing to an element. If the array previously contained two elements, and newValue.arrayField[4].numberChoice is set, then the array then gets five elements. However, the third and fourth element must still be assigned a value.
  • The choice of a UNION cannot be explicitly set. This is done automatically, when a data type of UNION is assigned a value. For example, setting newValue.arrayField[0].numberChoice sets the UNION arrayField[0] to numberChoice.
  • In CAPL, there is no conversion of a UTF8String or UTF16String from or to the character set used in the CAPL Editor. That is, a SomeIpSetValueString(messageHandle, "newValue.utf8String", "ß") would not return the desired result, because the character set of the CAPL Editor (mostly latin1) is not converted to UTF8.
