Protocols

Test

CANoe supports a large number of protocols used in the automotive industry:

Application Layer - Automotive

AUTOSAR PDU Multiplexing, DoIP, HSFZ, PDU Marshalling, SOME/IP, XCP, PGT, UDP NM

Application Layer - General

DHCPv4, DHCPv6, HTTP, NTP/SNTP, TFTP, SNMP, RTP, RTCP

Session Layer

TLS, DTLS

Transport Layer

TCP, UDP

Internet Layer

ICMPv4, ICMPv6, IGMP, IPv4, IPv6, IPsec Authentication Header

Link Layer

ARP/RARP, Ethernet, NDP, SRP, MAC Control

The complete list of the supported protocols can be found in the Protocol Overview.

Application protocols, depending on the use case, require additional information which must be specified in a database:

A separate TCP/IP Stack can be configured for test configurations and test modules.

CAPL

  • TCP/IP API allows the use of UDP/TCP sockets via the configured TCP/IP Stack.
  • CAPL functions for reading and changing the Ethernet link status
  • CAPL object for querying the Ethernet link statistics
  • Handler for received Ethernet packets and access to the included protocols
  • Sending Ethernet packets
  • Handler for AUTOSAR I-PDU events
  • Handler for signals in AUTOSAR I-PDU events
  • Handler for service signals in SOME/IP events
  • Handler for Ethernet link status
  • Waiting function for Ethernet packets

.NET

  • TCP/IP API allows the use of UDP/TCP sockets via the configured TCP/IP Stack.
  • Sending and receiving Ethernet packets
  • Access to the definition of the database

Analysis

Simulation/Stimulation

The windows and blocks in the Measurement Setup provide extended functions for Ethernet.

Observing Several Packets of Ethernet Traffic to Evaluate Distributed Information

  • Fragmentation of AUTOSAR I-PDU, SOME/IP, HTTP, WebSocket, XCP and DoIP via TCP
  • Fragmentation of AUTOSAR I-PDU, SOME/IP and XCP with IPv4 and IPv6
  • Fragmentation of SOME/IP with SOME/IP TP
  • SOME/IP Service Discovery

Trace Window

  • Columns for Ethernet
  • Protocol interpretation in the detail view
  • Ethernet link status event
  • Protocol-specific events, for example AUTOSAR PDU, SOME/IP, DoIP, XCP
  • Predefined event filters
  • Analysis filters
  • Coloring lines, depending on the protocol
  • Coloring events, depending on the IP address, conversation, service ID, etc.

Graphics Window, Data Window, and State Tracker

  • Signals from AUTOSAR I-PDUs and SOME/IP event or notifications defined in the database can be displayed.

Logging

  • BLF and ASC can be recorded with extended information of the Vector Ethernet interfaces.
  • PCAP and PCAPNG can be recorded.

CAPL Program

  • Handler for received Ethernet packets and access to the included protocols
  • Handler for AUTOSAR I-PDU events
  • Handler for signals in AUTOSAR I-PDU events
  • Handler for service signals in SOME/IP events
  • Handler for Ethernet link status

Bus Statistics Window

  • Statistical values for received and sent Ethernet packets for each Ethernet channel

Further Analysis Functions

  • Signals from AUTOSAR I-PDUs and SOME/IP events, and SOME/IP notifications can be used in panels. The required definition is read from the ARXML or Fibex files.
  • Protocol Monitor
    The MAC and IP addresses of the Ethernet and SOME/IP traffic are evaluated. The identified participants are displayed graphically and communication relations for SOME/IP are shown.
  • Ethernet link statistics with statistics system variable.
  • In the offline mode, besides the multibus logging formats, such as BLF and ASC, also PCAP and PCAPNG can be replayed.
  • Media Stream Control can be used to replay in a panel an audio or video stream transmitted with AVTP or RTP.

A separate TCP/IP Stack can be configured for each node in the Simulation Setup. The network settings can also be applied from the definition of an assigned database node.

AUTOSAR Eth Interaction Layer (AUTOSAR/IP IL)

  • Sending and receiving AUTOSAR I PDUs according to the definition of an assigned database node
  • Sending and receiving SOME/IP events, notifications and methods according to the definition of an assigned database node
  • SOME/IP Service Discovery

SOME/IP Interaction Layer (SOME/IP IL)

  • Sending and receiving SOME/IP events, notifications and methods according to the definition of an assigned database node
  • SOME/IP Service Discovery

AVB Interaction Layer (AVB IL)

  • Sending AVTP streams as a talker
  • Receiving AVTP streams as a listener

AUTOSAR UDP NM Interaction Layer

  • Handles network management states.
