Protocol Information [▲ back]

The Dynamic Host Configuration Protocol (DHCP) can assign IP addresses to one or more nodes. This allows the integration of a new node into an existing network without a manual configuration. The used transport protocol is UDP.

Protocol Validation [▲ back]

Ethernet packets containing the DHCPv4 protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[DHCPv4-001]

DHCPv4 packet length is too small. Must be at least 236 bytes

The present packet is too short. A DHCPv4 packet must consist of at least 236 bytes to be able to contain the header completely.

[DHCPv4-002]

Unable to parse all DHCPv4 options. Error at byte #N of DHCP message

The DHCPv4 options of the message (Options field) could not be read in. The byte position of the error is specified.

Programming [▲ back]

Initialization

During initialization of the DHCPv4 protocol, the fields SourcePort=67 (server) and DestinationPort=68 (client) in the UDP header are also set.
