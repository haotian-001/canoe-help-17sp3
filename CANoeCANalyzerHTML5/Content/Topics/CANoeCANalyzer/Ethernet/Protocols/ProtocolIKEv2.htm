Protocol Validation [▲ back]

Ethernet packets containing the IKEv2 protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[IKEv2-001]

IKEv2 packet length is too small. Must be at least 28 byte

The present packet is too short. An IKEv2 packet must consist of at least 28 bytes to fully accommodate the header.

[IKEv2-002]

IKEv2 length is too small. Must be at least 28 byte

The packet length specified in the header (Length field) is smaller than the smallest possible length of a UDP packet (8 bytes).

[IKEv2-003]

IKEv2 length is bigger than available data

The packet length specified in the header (Length field) is greater than the present packet.

[IKEv2-004]

IKEv2 version is invalid. Version must be 2.0

The version specified in the header (Version field) is invalid. Only packets with version 2.0 are recognized as valid.

[IKEv2-005]

IKEv2 exchange type unknown

The Exchange Type specified in the header is unknown.

[IKEv2-006]

IKEv2 payload header length is too small

The length specified in a payload header (Payload Length field) is less than the smallest possible length for this payload type.

[IKEv2-007]

IKEv2 payload has not enough data

The length specified in a Payload Header (Payload Length field) is greater than the present package.

[IKEv2-008]

IKEv2 invalid flag is set

An invalid flag was set in the Flags field. Only the Response, Version and Initiator flags are allowed.

[IKEv2-009]

IKEv2 next payload is unknown

The payload type specified in a Next Payload field is invalid.

Programming [▲ back]

The IPsec Encapsulating Security Payload header of an Ethernet packet is accessed via the selector (CAPL) or the property (.NET) ikev2. Only the protocol fields of the IKEv2 header can be accessed. Subsequent protocol headers cannot be accessed directly. These must be set via IKEv2 Data.

Initialization

During the initialization of the IKEv2 Header, the fields Source Port=500 and Destination Port=500 in the UDP header are also set.
