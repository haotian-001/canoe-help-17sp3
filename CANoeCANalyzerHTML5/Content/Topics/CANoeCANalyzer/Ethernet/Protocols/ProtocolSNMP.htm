Protocol Information [▲ back]

The SNMP-protocol (Simple Network Management Protocol) is a simple and very common protocol of the application layer to configure and monitor managed devices on IP-networks and was originally specified in the RFC 1067.

The communication happens typically between so called agents (client-side) and managers (server-side).
The SNMP-protocol exists mainly in 3 different versions, v1, v2c and v3; v2c is currently only partially and v3 not supported here.
The communication is connectionless via so called „well known UDP ports“

  • 161 to query agents.
  • 162 to send notifications (so called traps).

Header Format [▲ back]

The SNMP packets are described by ASN.1 and the protocol fields are encoded in the Type-Length-Value format. The SNMP protocol specifies the following packet types: GET-REQUEST, GETNEXT-REQUEST, GETBULK, SET-REQUEST, GET-RESPONSE and TRAP.

The packets (besides GETBULK and TRAP messages) are built up using the following schema :

SNMP Packet Header

PDU Header

PDU Body

Version Nummer

Community Name

Paket Type (Get, GetNext, …)

RequestID

Error Status

Error Index

Variable Binding 1

Variable Binding 2

...

Variable Binding n

There is no fix length information, but instead each field is described by ASN.1 (Type-Length-Value-Format).

GETBULK and TRAP messages have a differing PDU header (see Optional Protocol Fields).

Protocol Field

ASN.1 Data Type

Description

Length

—

Total length (in bytes) of the SNMP message that is coded with the ASN.1 type SEQUENCE.

Version

INTEGER

Version of the SMNP protocol as integer.

Community

OCTET STRING

Community name as a string used to assign access rights. Typical values are public for read-only access or private for read/write access.

Packet Type

—

ASN.1 type of the SNMP message:

Value

Packet Type

Description

0xA0

GET-REQUEST

Is used to request a specific data record, a unique ID must be specified.

0xA1

GETNEXT-REQUEST

Runs through the following data records, e.g. tables.

0xA2

GET-RESPONSE

Receives the answer to one of the previously sent packets.

0xA3

SET-REQUEST

Changes one or several data records.

0xA4

TRAP

Unsolicited message if an event occurs, from the agent to a manager.

0xA5

GETBULK-REQUEST

Calls the defined number of data records at once, similar to several GETNEXT-REQUESTS.

Protocol Validation [▲ back]

Ethernet packets containing the SNMP protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[SNMP-001]

Version not found

The present SNMP packet does not include a Version field.

[SNMP-002]

Initial sequence not found

The present SNMP packet does not begin with a field of the SEQUENCE type.

[SNMP-003]

Version v# not supported

The SNMP version of the present packet is not supported. Only versions up to v2 are supported.

[SNMP-004]

Community not found

The present SNMP packet does not include a Community field.

[SNMP-005]

Decoding token Id # not supported

The present packet includes an unknown field type and cannot be interpreted.

Programming [▲ back]

Initialization

During initialization of the SNMP protocol, the fields SourcePort=161 and DestinationPort=161 in the UDP header are also set. The used port number can be configured in the Options dialog.

Note

The direct access to protocol fields is not supported.
