Protocol Information [▲ back]

The Internet Group Management Protocol (IGMP) is used by IPv4 systems to report their IP multicast group memberships to neighboring multicast routers. It is required to be implemented by all hosts wishing to receive IP multicasts.

CANoe supports all three versions of IGMP.

Protocol Validation [▲ back]

Ethernet packets containing the IGMP protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[IGMP-001]

IGMP checksum is not correct

The checksum (Checksum field) of the IGMP layer is not correct.

[IGMP-002]

IGMP packet length is too small. Must be at least 8 bytes

The present packet is too short. An IGMP packet must consist of at least 8 bytes to be able to contain the header completely.

Programming [▲ back]

Initialization

During initialization of the IGMP protocol, the field Protocol=2 in the IPv4 header is also set.

Options

The IGMP protocol specifies different message types with different protocol fields (see Optional Protocol Fields table). Using the selectors (CAPL) or the properties (.NET) MembershipQuery and MembershipReport, ethernetPacket.<protocol>.<field>.IsAvailable can be used to check whether the message is of the appropriate type, and ethernetPacket.<protocol>.<optional-structure>.Init can be used to set the message to the appropriate type.

With nested selectors (CAPL) or properties (.NET), protocol fields of the appropriate message type are accessible.
