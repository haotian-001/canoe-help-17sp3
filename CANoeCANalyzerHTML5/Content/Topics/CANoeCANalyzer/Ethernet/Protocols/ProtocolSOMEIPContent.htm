Protocol Information [▲ back]

The protocol Scalable service-Oriented MiddlewarE over IP (SOME/IP) is a service-oriented Ethernet protocol from the automotive industry. This protocol allows the individual control units or software components to offer so-called services (service provider) or to make use of services from other control units (consumer). It is possible, for example, to call methods, read or write field values or to subscribe to certain events on the occurrence of which the subscribers are informed.

The protocol Service Discovery (SOME/IP-SD) which is specified within the SOME/IP protocol is used for the communication of the available services of a provider or for a required service of a consumer as well as for subscribing or unsubscribing from SOME/IP services.

With the Transport Protocol (SOME/IP-TP), SOME/IP offers the possibility of a segmentation of large messages via UDP.

The SOME/IP Interaction Layer allows a simplified access.

CANoe offers additional possibilities to use SOME/IP in configurations.

Protocol Validation [▲ back]

Ethernet packets containing the SOME/IP protocol are analyzed and their consistency is checked. Detected irregularities are shown in the Protocol Validation column of the Trace Window.

SOME/IP length is bigger than available data!

Programming [▲ back]

The access to SOME/IP of an Ethernet packet is performed via the selector (CAPL) or the property (.NET) SOMEIP.

Initialization

During initialization of SOME/IP, UDP is used by default as transport protocol, the fields SourcePort and DestinationPort are set to a SOME/IP port-number that you have configured in the Options dialog.

Note

The direct access to protocol fields of SOME/IP is not supported.
