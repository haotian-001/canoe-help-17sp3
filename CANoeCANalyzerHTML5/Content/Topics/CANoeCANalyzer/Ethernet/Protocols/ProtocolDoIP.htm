Protocol Information

Diagnostics over IP (DoIP) is a proprietary protocol for the diagnostic access to a vehicle via Ethernet. It is based on its predecessor protocol HSFZ.

CANoe offers further possibilities to use DoIP in Configurations.

Note

The DoIP header is correctly interpreted in those TCP packets of a TCP data stream in which the DoIP header lies at the beginning of the TCP payload.

Protocol Validation [▲ back]

Ethernet packets containing the DoIP protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[DoIP-001]

DoIP payload length is bigger than available data

The specified payload length (Payload Length field) in the header is bigger than the present packet.

[DoIP-002]

DoIP packet length is too small. Must be at least 8 bytes

The present packet is too short. A DoIP packet must consist of at least 8 bytes to be able to contain the header completely.

[DoIP-003]

Inverse DoIP version does not fit to the version

Protocol version (Protocol Version field) and inverse protocol version (Inverse Protocol Version field) are inconsistent. One value must be the ones complement of the other.

[DoIP-005]

Resync necessary to interpret data! Some data was not interpreted due to a length error in the previous PDU

A length error in a previous PDU prevents the interpretation of further data.

Programming [▲ back]

Initialization

During initialization of the DoIP protocol, the fields SourcePort=13400 and DestinationPort=13400 in the UDP/TCP header are also set. The used port number can be configured in the Options dialog.
