Protocol Information [▲ back]

The Address Resolution Protocol (ARP) is a network protocol that allows the assignment of network addresses (IP address) to hardware addresses (MAC Media Access Control address). It is mostly used for IPv4 addressing in Ethernet networks. For IPv6 networks the NDP protocol is used.

RARP (Reverse Address Resolution Protocol) offers the way vice versa: the assignment of MAC addresses to IP addresses. This allows the detection of IP addresses for devices that have no permanent MAC address.

The packet format is identical for both protocols.

Protocol Validation [▲ back]

Ethernet packets containing the ARP protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[ARP-001]

ARP packet length is too small. Must be at least 28 bytes

The present package is too short. An ARP packet must consist of at least 28 bytes to be able to contain the header completely.
