Protocol Information [▲ back]

The MACsec protocol (Media Access Control Security according to 802.1AE) enables authorized systems to guarantee the confidentiality and integrity of data at the network layer.

Protocol Validation [▲ back]

Ethernet packets containing the MACsec protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[MACsec-001]

MACsec packet length is too small. Must be at least 8 byte

The present packet is too short. A MACsec packet must consist of at least 8 bytes to be able to evaluate the header.

[MACsec-002]

MACsec length is bigger than available data

The packet length is too short to contain all MACsec data.

[MACsec-003]

MACsec version is invalid. Version Flag should not be set.

The version flag is not set.

[MACsec-004]

MACsec invalid combination of flags is set

The combination of the set TCI flags is invalid. If the ES or SCB flag is set in the TCI, the SC flag must not be set.

[MACsec-005]

MACsec short length is too big

The Short Length is greater than 63.

Programming [▲ back]

Initialization

During the initialization of the MACsec protocol, the Ethertype field in the Ethernet header is set to the value 0x88E5.

Header Access

The MACsec header of an Ethernet packet is accessed via the MACsec selector (CAPL) or the MACsec property (.NET) . The sci, scsi and scipi fields are only available if the Secure Channel (tcisc) flag is set in the Tag Control Information. The Integrity Check Value field can only be accessed if the Changed Text (tcic) flag in the Tag Control Information is not set.
