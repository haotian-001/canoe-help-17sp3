AUTOSAR UDP NM

 

The AUTOSAR UDP network management protocol is used to switch ECUs from operating mode to sleep mode.

A AUTOSAR PDU is cyclically sent with broadcast via UDP to keep ECUs in the operating mode.

Configuration

The UDP port numbers must be configured to enable CANoe to evaluate AUTOSAR NM. On the Bus Systems page (Options dialog), these settings can be found in the Protocol Identification area. The UDP port number can be entered here at UDP NM.

Analysis

CANoe evaluates the UDP transmissions and extracts the NM PDU. They are shown, for example in the Trace Window as its own PDU events.

Simulation

The AUTOSAR UDP network management IL (AsrUdpNm.vmodule) is available for simulation. This interaction layer (IL) manages the NM state of a simulated ECU and sends the NM PDUs. It must be configured together with AUTOSAR Eth IL (CANoeILNL_AUTOSAR_Eth.vmodule) (see also How to Simulate an Ethernet Network).
