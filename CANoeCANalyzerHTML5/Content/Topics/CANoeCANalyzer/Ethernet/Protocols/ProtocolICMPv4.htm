Protocol Information [▲ back]

The Internet Control Message Protocol (ICMP) is part of an IP implementation and it is used for controlling tasks. Applications that use ICMP are e.g. Ping and Traceroute.

With Ping you can check the IP communication between two computers. An Echo Request message is sent to a node which responds with an Echo Reply message.

With Traceroute you can observe the way of an IP packet through all routers. For this datagrams with manipulated Hop Limit are sent and the Time Exceeded message as well as the Destination Unreachable messages are evaluated.

Protocol Validation [▲ back]

Ethernet packets containing the ICMPv4 protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[ICMPv4-001]

ICMP payload length is N bytes, but it is expected to be M bytes

The payload of the present ICMPv4 message has the wrong size. Special ICMPv4 types (Type field) require a defined payload length.

[ICMPv4-002]

ICMP frame is too small. For the given ICMP type it must be at least N bytes

The present packet is too short. Depending on the ICMPv4 type (Type field), a packet must have a specified minimum length to be able to contain the ICMPv4 message completely.

[ICMPv4-003]

ICMPv4 checksum is not correct

The checksum (Checksum field) of the ICMPv4 layer is not correct.

Programming [▲ back]

Initialization

During initialization of the ICMPv4 protocol, the field Protocol=1 in the IPv4 header is also set.

Options

The ICMPv4 protocol specifies different message types with different protocol fields (see Optional Protocol Fields table). Using selectors (CAPL) or properties (.NET) of the appropriate message type, ethernetPacket.<protocol>.<field>.IsAvailable can be used to check whether the message is of the appropriate type, and ethernetPacket.<protocol>.<optional-structure>.Init can be used to set the message to the appropriate type.

With nested selectors (CAPL) or properties (.NET), protocol fields of the appropriate message type are accessible.
