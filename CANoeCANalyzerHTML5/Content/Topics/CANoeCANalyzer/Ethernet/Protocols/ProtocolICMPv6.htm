Protocol Information [▲ back]

The Internet Control Message Protocol (ICMP) is part of an IP implementation and it is used for controlling tasks. Applications that use ICMP are e.g. Ping and Traceroute.

With Ping you can check the IP communication between two computers. An Echo Request message is sent to a node which responds with an Echo Reply message.

With Traceroute you can observe the way of an IP packet through all routers. For this datagrams with manipulated Hop Limit are sent and the Time Exceeded message as well as the Destination Unreachable messages are evaluated.

Protocol Validation [▲ back]

Ethernet packets containing the ICMPv6 protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[ICMPv6-001]

ICMPv6 packet is too small

The present packet is too short. Depending on the ICMPv6 type (Type field), a packet must have a specified minimum length to be able to contain the ICMPv6 message completely.

[ICMPv6-002]

ICMPv6 checksum is not correct

The checksum (Checksum field) of the ICMPv6 layer is not correct.

Programming [▲ back]

Initialization

During initialization of the ICMPv6 protocol, the field NextHeader=58 in the IPv6 header is also set.

Options

The ICMPv6 protocol specifies different message types with different protocol fields (see Optional Protocol Fields table). Using selectors (CAPL) or properties (.NET) of the appropriate message type, ethernetPacket.<protocol>.<field>.IsAvailable can be used to check whether the message is of the appropriate type, and ethernetPacket.<protocol>.<optional-structure>.Init can be used to set the message to the appropriate type.

Note

During initialization of the message types Multicast Listener Query, Multicast Listener Report, Multicast Listener Done and Version 2 Multicast Listener Report, the field HopLimit=1 in the IPv6-Header is set, the IPv6 Extension Header HopByHopOptions is added and in this header the Router Alert option is inserted.

With nested selectors (CAPL) or properties (.NET), protocol fields of the appropriate message type are accessible.
