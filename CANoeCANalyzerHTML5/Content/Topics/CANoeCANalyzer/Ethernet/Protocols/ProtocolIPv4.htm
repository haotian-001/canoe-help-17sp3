Protocol Information [▲ back]

Besides IPv6 the IPv4 protocol (Internet Protocol Version 4) is the lowermost protocol of the internet protocol family respectively the TCP/IP model that is independent of the transmitting medium. With Ethernet IPv4 uses the Ethertype 0800.

Protocol Validation [▲ back]

Ethernet packets containing the IPv4 protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[IPv4-001]

IPv4 header length is too small. Must be at least 20 bytes

The header length specified in the header (Header Length field) is smaller than the smallest possible length of an IPv4 header (20 bytes).

[IPv4-002]

IPv4 header length is bigger than available data

The header length specified in the header (Header Length field) is bigger than the present IPv4 header.

[IPv4-003]

IPv4 total length is bigger than available data

The packet length specified in the header (Total Length field) is bigger that the present packet.

[IPv4-004]

IPv4 total length is smaller than IPv4 header length

The packet length specified in the header (Total Length field) is smaller than the specified header length (Header Length field).

[IPv4-005]

Checksum is not correct

The checksum (Checksum field) of the IPv4 layer is not correct.

[IPv4-006]

IPv4 packet length is too small. Must be at least 20 bytes

The present packet is too short. An IPv4 packet must consist of at least 20 bytes to be able to contain the header completely.

Programming [▲ back]

Initialization

During the initialization of the IPv4 protocol, the field Ethertype=0800h in the Ethernet header is also set.

Example

In this example an Ethernet packet with IPv4 protocol is created and sent. The Ethertype is output in the Write Window before sending.

CAPL

ethernetPacket pkt;

// initialize packet IPv4
pkt.ipv4.Init();

// print ethertype ("IPv4 has ethertype 0x0800")
write( "IPv4 has ethertype 0x%04X", pkt.type );

// send Ethernet packet
output( pkt );

C#

EthernetPacket pkt = new EthernetPacket();

// initialize packet IPv4
pkt.IPv4.Init();

// print ethertype ("IPv4 has ethertype 0x0800")
Output.WriteLine("IPv4 has ethertype 0x{0:X4}", pkt.Ethertype);

// send Ethernet packet
pkt.Send();

Header Access

The read and write access to the protocol fields of the header is done by means of selectors (CAPL) or properties (.NET) which have been named according to their designators.

Note

If a higher layer protocol is initialized in an empty packet which does not explicitly require IPv6, the initialization of IPv4 takes place by default on the internet layer.

Example

In this example, a new packet is generated and sent with IPv4 and several protocol fields are filled with specific values.

By this, two variants for the assignment of IP addresses are demonstrated:

  • Direct parsing of an address from a string
  • The assignment via objects of the type ip_Address (CAPL) or System.Net.IPAddress (.NET)

CAPL

ip_Address 255.255.255.255 broadcast;
ethernetPacket pkt;

// initialize packet with IPv4 protocol
pkt.ipv4.Init();

// set IPv4 addresses
pkt.ipv4.source.ParseAddress( "192.168.1.1" );
pkt.ipv4.destination = broadcast.IPV4Address;

// set time to live
pkt.ipv4.ttl = 32;

// add optionTimestamp
pkt.ipv4.optionTimeStamp.Init();

// calculate IPv4 checksum and send Ethernet packet
pkt.CompletePacket();
output( pkt );

C#

using System.Net; // for type IPAddress
...
EthernetPacket pkt = new EthernetPacket();

// initialize packet with IPv4 protocol
pkt.IPv4.Init();

// set IPv4 addresses
pkt.IPv4.Source.ParseAddress("192.168.1.1");
pkt.IPv4.Destination.Value = IPAddress.Broadcast;

// set time to live
pkt.IPv4.Ttl.Value = 32;

// add optionTimestamp
pkt.IPv4.OptionTimeStamp.Init();

// calculate IPv4 checksum and send Ethernet packet
pkt.CompletePacket();
pkt.Send();

Options

The IPv4 protocol specifies options which can be included in the packet or not. ethernetPacket.<protocol>.<field>.IsAvailable allows you to check whether an option is available in the header, ethernetPacket.<protocol>.<optional-structure>.Init allows you to add the option and ethernetPacket.<protocol>.<optional-structure>.Clear can be used for the removal.

Example

In this example, the option TimeStamp is stamped to each IPv4 packet which is sent by the TCP/IP stack of the node if it is not already available in the packet.

CAPL

long OnIpSendPrepare(dword socket, ethernetPacket * packet)
{
  if ( packet.ipv4.IsAvailable() && !packet.ipv4.optionTimeStamp.IsAvailable() )
  {
    packet.ipv4.optionTimeStamp.Init();
    packet.CompletePacket();
  }
  return 1;
}

C#

[OnIpSendPrepare]
public bool AddOptionTimeStamp(Socket socket, EthernetPacket pkt)
{
  if (pkt.IPv4.IsAvailable && !pkt.IPv4.OptionTimeStamp.IsAvailable)
  {
    pkt.IPv4.OptionTimeStamp.Init();
    pkt.CompletePacket();
  }
  return true;
}
