Protocol Information [▲ back]

The MKA protocol ((MACsec Key Agreement) is a part of MACsec and is used for authentication and key exchange between two parties. It uses the EAPOL protocol.

Protocol Validation [▲ back]

Protocol Validation ID

Message

Description

[MKA-001]

MKA packet length is too small. Must be at least 4 byte

The present packet is too short. An MKA packet must consist of at least 4 bytes to be able to evaluate the header.

Programming[▲ back]

Ethernet packets containing the MKA protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Initialization

During the initialization of the MKA protocol, the PacketType field in the EAPOL header is set to the value 5.

Header Access

The access to the MKA header of an Ethernet packet is done via the mka selector (CAPL) or the MKA property (.NET).
