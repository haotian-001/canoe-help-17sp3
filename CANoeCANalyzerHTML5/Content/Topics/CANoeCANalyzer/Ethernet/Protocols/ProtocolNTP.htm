Protocol Information [▲ back]

The Network Time Protocol (NTP) is used to synchronize clocks within computer systems with packet-based communication networks. Normally the connectionless transport protocol UDP is used. This was designed to get reliable time value of networks with variable latency of the packets.

In general SNTP (Simple Network Time Protocol) and NTP are similar protocols, that are used for different application areas. A Whereas NTP servers or clients ensure a high accuracy and avoids high time steps, SNTP is only useful for simple application without big requirements to accuracy and reliability.

Protocol Validation [▲ back]

Ethernet packets containing the NTP/ SNTP protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[NTP-001]

NTP packet length is too small. Must be at least 12 bytes

The present package is too short. A NTP packet must consist of at least 12 bytes to be able to contain the header completely.

Programming [▲ back]

Initialization

During initialization of the NTP protocol, the fields SourcePort=123 and DestinationPort=123 in the UDP header are also set. The used port number can be configured in the Options dialog.

Options

The NTP protocol specifies different message types with different protocol fields (see Optional Protocol Fields table). Using the selector (CAPL) or the property (.NET) Control, ethernetPacket.<protocol>.<field>.IsAvailable can be used to check whether the message is Control message, and ethernetPacket.<protocol>.<optional-structure>.Init can be used to set the message to the appropriate type (Mode=6 in NTP header).
