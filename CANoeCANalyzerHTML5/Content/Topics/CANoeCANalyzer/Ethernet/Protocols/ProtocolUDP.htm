Protocol Information [▲ back]

UDP (User Data Protocol) is a network protocol with minimal protocol header. It is a connectionless protocol and doesn't use data segmentation, that means the packet size in CANoe is restricted to the maximum Ethernet size. With a checksum transmission errors can be identified. With UDP a service on the target system can be addressed with port numbers.

Protocol Validation [▲ back]

Ethernet packets containing the UDP protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[UDP-001]

UDP length is too small. Must be at least 8 bytes

The packet length specified in the header (Length field) is smaller than the smallest possible length of a UDP packet (8 bytes).

[UDP-002]

UDP length is bigger than available data

The packet length specified in the header (Length field) is bigger that the present packet.

[UDP-003]

UDP packet length is too small. Must be at least 8 bytes

The present packet is too short. A UDP packet must consist of at least 8 bytes to be able to contain the header completely.

[UDP-004]

Checksum is not correct

The checksum (Checksum field) of the UDP layer is not correct.

Programming [▲ back]

Initialization

During initialization of the UDP protocol, the field Protocol=17 in the IPv4 header, or NextHeader=17 in the IPv6 header, is also set.

Note

If the UDP protocol is initialized in an empty packet, the initialization of IPv4 takes place as standard on the internet layer.

Example 1

In this example a new Ethernet packet is generated to which the UDP protocol with its default initialization is added and which is sent afterwards. During the initialization of the UDP protocol, the IPv4 protocol is automatically added to the packet and the Protocol field of the IPv4 protocol is output in the Write Window before sending.

CAPL

ethernetPacket pkt;

// initialize packet with IPv4 and UDP protocols
pkt.udp.Init();

// print protocol type ("UDP has protocol type 17")
write( "UDP has protocol type %d", pkt.ipv4.protocol);

// send Ethernet packet
output( pkt );

C#

EthernetPacket pkt = new EthernetPacket();

// initialize packet with IPv4 and UDP protocols
pkt.UDP.Init();

// print protocol type ("UDP has protocol type 17")
Output.WriteLine("UDP has protocol type {0}", pkt.IPv4.Protocol.Value);

// send Ethernet packet
pkt.Send();

Example 2

In this example, a new packet is initialized and sent with the UDP protocol as shown in example 1. Since the IPv6 protocol, which is compatible with UDP, is already available in the packet on the internet layer, IPv4 is not initialized.

CAPL

ethernetPacket pkt;

// initialize packet with IPv6 and UDP protocols
pkt.ipv6.Init();
pkt.udp.Init();

// print protocol type ("TCP has protocol type 17")
write( "UDP has protocol type %d", pkt.ipv6.next);

// send Ethernet packet
output( pkt );

C#

EthernetPacket pkt = new EthernetPacket();

// initialize packet with IPv6 and UDP protocols
pkt.IPv6.Init();
pkt.UDP.Init();

// print protocol type ("UDP has protocol type 17")
Output.WriteLine("UDP has protocol type {0}", pkt.IPv6.Next.Value);

// send Ethernet packet
pkt.Send();

Header Access

The read and write access to the protocol fields of the header is done by means of selectors (CAPL) or properties (.NET) which have been named according to their designators.

Example

In this example, a new packet is generated and sent with UDP and IPv4 and several protocol fields are filled with specific values.

By this, two variants for the assignment of IP addresses are demonstrated:

  • Direct parsing of an address from a string
  • The setting of an endpoint using an object of the type ip_Endpoint (CAPL) or the assignment via an object of the type System.Net.IPAddress (.NET) and UInt16 (.NET)

CAPL

ip_Endpoint 255.255.255.255:40002 broadcastEp;
ethernetPacket pkt;

// initialize packet with IPv4 and UDP protocols
pkt.udp.Init();

// add optionTimestamp, resize and calculate checksum of IPv4
pkt.ipv4.optionTimeStamp.Init();
pkt.CompletePacket();

// set IPv4 addresses and UDP ports
pkt.ipv4.source.ParseAddress( "192.168.1.1" );
pkt.udp.source = 40001;
pkt.SetDestinationIPEndpoint( broadcastEp );

// set UDP payload
pkt.udp.SetData( 0, "Hello world", 11 );

// calculate UDP and IPv4 checksum and send Ethernet packet
pkt.CompletePacket();
output( pkt );

C#

using System.Net; // for type IPAddress
EthernetPacket pkt = new EthernetPacket();

// initialize packet with IPv4 and UDP protocols
pkt.UDP.Init();

// add optionTimestamp, resize and calculate checksum of IPv4
pkt.IPv4.OptionTimeStamp.Init();
pkt.CompletePacket();

// set IPv4 addresses and UDP ports
pkt.IPv4.Source.ParseAddress("192.168.1.1");
pkt.UDP.Source.Value = 40001;
pkt.IPv4.Destination.Value = IPAddress.Broadcast;
pkt.UDP.Destination.Value = 40002;

// set UDP payload
pkt.UDP.SetRawData(System.Text.Encoding.ASCII.GetBytes("Hello world"));

// calculate UDP and IPv4 checksum and send Ethernet packet
pkt.CompletePacket();
pkt.Send();
