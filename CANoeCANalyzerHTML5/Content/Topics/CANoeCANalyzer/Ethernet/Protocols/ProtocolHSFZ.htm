Protocol Information [▲ back]

The High Speed Fahrzeugzugang (HSFZ) protocol is a proprietary protocol for a diagnostic access to a vehicle via Ethernet. It is the predecessor of DoIP.

CANoe offers further possibilities to use HSFZ in Configurations.

Protocol Validation [▲ back]

Ethernet packets containing the HFZ protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[DoIP-001]

HSFZ payload length is bigger than available data

The payload length specified in the header (Payload Length field) is bigger than the present packet.

[DoIP-002]

HSFZ packet length is too small. Must be at least 6 bytes

The present packet is too short. A HSFZ packet must consist of at least 6 bytes to be able to contain the header completely.

[DoIP-005]

Resync necessary to interpret data! Some data was not interpreted due to a length error in the previous PDU

A length error in a previous PDU prevents the interpretation of further data.

Programming [▲ back]

Initialization

During initialization of the HSFZ protocol, the fields SourcePort=7811 and DestinationPort=6811 in the UDP header, or SourcePort=6801 and DestinationPort=6801 in the TCP header, are also set. The used port number can be configured in the Options dialog.
