Protocol Information [▲ back]

The Dynamic Host Configuration Protocol (DHCP) can automatically assign IP addresses to one or several nodes. By this, the automatic integration of a new computer in an existing network is possible without manual configuration. UDP is used as transport protocol.

Protocol Validation [▲ back]

Ethernet packets containing the DHCPv6 protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[DHCPv6-001]

DHCPv6 packet length is too small. Must be at least 34 bytes (relay message)

The present packet is too short. A DHCPv6 packet of type Relay-forward or Relay-reply must consist of at least 34 bytes to receive the header completely.

[DHCPv6-002]

DHCPv6 packet length is too small. Must be at least 4 bytes (client/server message)

The present packet is too short. A DHCPv6 packet must consist of at least 4 bytes to be able to contain the header completely.

Programming [▲ back]

Initialization

During initialization of the DHCPv6 protocol, the fields SourcePort=546 and DestinationPort=547 in the UDP header are also set. The used port number can be configured in the Options dialog.
