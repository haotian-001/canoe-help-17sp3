Protocol Information [▲ back]

AH (IPsec Authentication Header) is a supplement to the IPv4 protocol which enables a safe transmission of encrypted data on the internet layer. The IPsec Authentication Header complements the IPv4 Header.

Note

The IPsec Authentication Header is only available as protocol for IPv4. For IPv6 the corresponding functionality is provided by the IPv6 Authentication Header.

Protocol Validation [▲ back]

Ethernet packets containing the AH protocol are analyzed and checked for consistency. Observed irregularities are displayed in the Protocol Validation column of the Trace Window.

Protocol Validation ID

Message

Description

[AH-001]

AH length is bigger than available data

The length specified in the header (Length field) is bigger than the present packet.

[AH-002]

AH packet length is too small. Must be at least 12 bytes

The present packet is too short. An AH packet must consist of at least 12 bytes to be able to contain the header completely.

Programming [▲ back]

You can access the IPsec Authentication header of an Ethernet packet via the selector (CAPL) or the AH property (.NET).

Initialization

During initialization of the IPsec Authentication header, the field Protocol=51 in the IPv4 header is also set.
