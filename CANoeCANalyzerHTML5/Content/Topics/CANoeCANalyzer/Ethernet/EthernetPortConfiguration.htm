Port Configuration

Ribbon: Hardware Tab|Port Configuration

This dialog is only available with a loaded network-based configuration or with the activated Access network via Ethernet ports option in Options dialog. Here, you can configure

  • which ports are to be measured
  • the assignment of simulated ports to a hardware segment, which controls where the simulation participates in the Ethernet topology

Views

The content of the dialog depends on

  • If CANoe is already connected to the interface(s) that are configured with the required network(s), the Port Configuration dialog shows the ports and segments defined on the hardware in combination with the simulation ports defined in CANoe.
  • If no interface is attached or if the required network cannot be found, you can configure manually.

Note

The graphical view on the right is only available in real bus working mode.

Configured Ports

All ports contained in the configuration are displayed here sorted by network and category in a tree view.

You can find a detailed description of the terms and example on the Network-based Access page.

Network

All measurement and simulation ports of the available networks are displayed here.

Measurement Ports

Real hardware is connected with measurement ports. They are added as physical or virtual ports in the Ethernet Device Configuration.

Here you can activate measurement ports whose data you want to measure.

Add measurement ports manually if no interface is connected or if the required network cannot be found.

Measuring ports that are not available in the hardware are marked with a yellow exclamation mark. You can start the measurement, but these measuring ports will not provide any data. You can delete these ports via the context menu. The port will only be deleted when you close the dialog with [OK].

Note

All simulation ports are also listed here as inactive.

In the Options dialog you can specify that the events of all simulation ports are not forwarded for evaluation. Activate singe simulation port here to forward their events.

Simulation Ports

The ports that were automatically assigned, e.g. by inserting network nodes in the Simulation Setup, are displayed here. They have the name of the network node and can neither be edited nor deleted. You may also create so-called virtual ports as simulation ports in the Ethernet Device Configuration.

Assign segments for the simulation to the simulation ports.

Context Menu

Automatic Segment Selection
Automatic selection of segments. Automatic segment selection is only useful if you have only created one segment in the Ethernet Device Configuration. If more than one segment exists in the Ethernet Device Configuration, the segment name should be selected explicitly.


If no port is connected or if the required network cannot be found, you can also add a segment manually.

Optimize the Required Hardware Resources (Reduce Number of Used Ports)
If you activate this option, CANoe optimizes the hardware resources used for the simulation.

CANoe uses simulation ports to interact with the Ethernet network. The number of simulation ports depends on the number of nodes, Packet Builders, Replay Blocks or other simulation components used. The VN Ethernet interfaces allow only a limited number of ports that can be opened for each network or segment. To avoid that a simulation cannot be executed due to this hardware restriction, CANoe can optimize the number of simulation ports used, by combining ports into one port. Only simulation ports that are connected to the same switch segment are combined. If this option is used, hardware features such as filtering and TSN (Time Sensitive Networking) are no longer available for the individual simulation ports.

You can also generally activate this option in the Options dialog.
