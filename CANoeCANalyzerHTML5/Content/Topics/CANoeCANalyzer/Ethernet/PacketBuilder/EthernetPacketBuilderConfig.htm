Configuration

 

Default Settings

In this dialog you can specify the default values that should be used if a new packet is created and added to the packet list:

Default Source MAC Address and Default Destination MAC Address

Here you can select the default MAC addresses from the following entries:

  • <Current channel>

The MAC-ID of the adapter is used that is assigned to the channel entered for the packet (e.g. adapter MAC-ID of channel Eth 1).

Note

  • If you select this setting consider that manually inserted data are overwritten in the section Raw Frame.
  • Enter the MAC address as six two-digit hexadecimal numbers separated by colons, e.g. the broadcast MAC address FF:FF:FF:FF:FF:FF.
  • <Config manual>

You can enter any MAC address.

  • <Broadcast>

The broadcast MAC address (FF:FF:FF:FF:FF:FF) is used.

  • <Adapter Eth <x>>

The MAC-ID of the adapter is used that is configured to channel Eth x.

Note

If you select this setting consider that manually inserted data are overwritten in the section Raw Frame.

Default Source Port / Default Destination Port

Here you select the default settings for the used ports.

Default Source IPv4 Address / Default Destination IPv4 Address

Here you select the default settings for the IPv4 addresses.

Note

Enter the IP address in dot-decimal notation as four numbers in the range 0..255 separated by dots.

Default Source IPv6 Address / Default Destination IPv6 Address

Here you select the default settings for the IPv6 addresses. All syntax defined in the RFC 3513 are supported for IPv6 addresses. On leaving the edit field a validation check is processed for the entered addresses.

Entered IPv6 addresses are always formatted to the Vector style:

  • hexadecimal values
  • uppercase
  • all leading zeros dropped
  • without zeros compressor
  • no hybrid notation
