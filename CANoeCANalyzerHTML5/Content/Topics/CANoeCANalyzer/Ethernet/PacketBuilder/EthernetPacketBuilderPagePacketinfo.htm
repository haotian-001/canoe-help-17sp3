Packet Information

Raw Frame

 

Packets can be configured in this area. Dependent on the detected protocol the packet will be interpreted. The supported protocols are listed in the protocol feature matrix.

In this window the following information is displayed:

Column

Description

1

Name of the field (e.g. Ether Type)

2

Configurable value of the field

Different representations of the values are possible depending on the interpretation. A MAC address is display in MAC address representation, the IP addresses as well. For bit fields, the position within the bit field is highlighted. Otherwise the values are displayed in decimal or hexadecimal (0x) format.

3

Position of the value in the frame (e.g. [0/6]: starts at byte 0, length=6 Bytes)

If there are e.g. 3 bit, this column shows the start byte [15/1], the bit length in display in the interpretation column. The position results from the field order.

4

Additional interpretation of the value if the protocol is supported

The interpretation is automatically done based on the inserted data

For an Ethernet packet you can change the values Source (Source MAC address ), Destination (Destination MAC address ) and Ether Type. To set the Source MAC address and Target MAC address you can also use the MAC-ID of an available adapter.

To change a value you must double-click with the left mouse button on the value or the row containing the value. In the opening input field insert the value your value, then press [Enter] or click in another part of the window to commit the value. At several values you are supported with a list to select a value.

To discard your changes press [Esc].

Fields for Checksum and Length

Checksums and length fields are automatically calculated and set if this is defined for the protocol. This setting is available for each protocol in each packet. If you want to change these values manually you have to deactivate automatic calculation with the button on the right side of the dialog. The calculated values are then only shown in the interpretation column but not transferred to the corresponding value field.

Depending on the protocol, the following buttons are available:

Button

Description

Activates/deactivates the automatic calculation of the checksum fields

Activates/deactivates the automatic calculation of the length fields

Note

If the calculation of checksum and length field are activated, the data content of the packet is getting modified if you change the protocol.

Protocol Errors

If a row is marked in red, a protocol error has been detected. The reason is displayed in the interpretation column and the column is marked with the appropriate icon for erroneous packets in the packet list.

Payload

Below the interpretation part a payload section is available, which displays the real payload. The layout is similar to the raw frame section but here you see only these data which are not interpretable by a higher protocol.
