Packet List

Packet Information

Raw Frame

 

In the packet list an overview of all packets and its information is listed. The following columns show you the packet information:

Column

Description

Packet Description

Description of the packet, which can be changed manually

Source

Address of the sender

Destination

Address of the receiver

Protocol

Detected protocol of the packet

Packet length

Length of the packet

Payload length

Length of the payload

For network-based access only:

Send to Port

Port on which the packet is sent

Reverse Direction

No
Send direction from the selected port to a connected segment.

Yes
Send direction from the selected port away from the connected segment, e.g. towards an externally connected ECU respectively network or towards CANoe.

You can add new packets with the button [Add Packet]. With a click on the arrow you can insert a packet with the appropriate pre-configured protocol directly (e.g. a UDP datagram). With the button [Delete] you can remove packets from the list.

Within the list you can change the sequence of the packets to send via drag and drop. Existing packets you can duplicate with <Ctrl>+<C> and <Ctrl>+<V>.

You can also change the description and the packet length of a packet. Therefore you have to double-click with the left mouse button on the description in column Packet Description of a packet or in column Packet length to change the length. To change the length of the packet payload double click on column Payload length. The packet length is adapted automatically.

In a configuration with network-based access you can set a port on which the packet should be sent. With a double click on a table cell in the Send to Port column you can open an overview with all available ports. Typically at simulation ports it is sent in the direction of the connected segment (reverse direction = no). For measurement ports it is send away from the connected segment (reverse direction = yes) to an externally connected ECU respectively network.

Note

If a port configured in the Ethernet Packet Builder is no longer available, the name is displayed in red.

To send the packet finally, select it and click the button [Send].

If a packet is erroneous it is marked with the icon . Therefore the packets are checked for checksum, packet length and packet order. If the packet is error-free, it is marked with .

Note

A packet is marked error-free, if the checks are successfully executed and no errors are detected, but it cannot be guaranteed that the packet is error-free in every case!
