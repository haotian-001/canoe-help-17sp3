Debugger: Document View

 

Here, you see the open source code spread over individual pages.

If you have deactivated Debug mode via the toolbar , this is indicated with and a corresponding tooltip. If you have opened a program with a compiler error, this is also visualized with the same icon and a tooltip.

Connection to program node

If you change a program outside of CANoe, the source code is automatically updated in the Debugger Window directly or at the latest after measurement stop. If a source code file no longer belongs to the main program, this file will be closed automatically.

If you remove a program node, test module or test configuration the associated Debugger Window is also removed.

Breakpoints

If debug mode is active, you can set a breakpoint to the left of the line number in the document view with a mouse click. A program is interrupted at this breakpoint before it executes the command in the line in which the breakpoint was set. Breakpoints are marked in red for network nodes and orange or red in test modules / test configurations.

If a breakpoint cannot be reached, it is marked as not active by an open circle .
Examples are

  • Empty lines
  • Comments
  • Variable declarations
  • Function definitions

You can insert only one breakpoint per line. If a line contains multiple commands, the breakpoint relates to the first command.

Clicking the breakpoint removes it without a prompt.

Breakpoints are retained even after the Debugger Window is closed and are stored in the CANoe configuration.

You can find a list of all breakpoint in the Breakpoint Window.

Context Menu of a Source Code File

Open...
Opens a dialog with the available source code files.

Close
Closes the active source code file.

Close All but This
Closes all open source code files except the active source code file.

Edit... (not available for test configurations)
Opens the active source code file in the CAPL browser or .NET Editor.

Context Menu of the Source Code

Insert/Remove Breakpoint
Inserts a breakpoint or removes an existing one.

Enable/Disable Breakpoint
Enables/disables a breakpoint

Break Test/Simulation
For a breakpoint you can select, if the test module or the test configuration alone or also the simulation should be stopped.

Remove All Breakpoints
Removes all breakpoints.

Disable All Breakpoints
Disables all breakpoints.

Toolbar commands

Add/remove monitoring
Adds/removes the selected variable to/from the Watch area.

Copy
Copies the selected text to the clipboard.

Select All
Selects the whole source code.
