Debugger

 

General

The icons may vary depending on the context in which the window is opened.

The debugger supports troubleshooting

  • in Simulation Setup: of network nodes and test modules that were developed in CAPL and .NET.
  • in Measurement Setup: of program nodes that were developed in CAPL.
  • in Test Setup for Test Modules: in test modules that were developed in CAPL and .NET.
  • in Communication Setup: of application models that were developed in CAPL and .NET.
  • in Test Setup for Test Units: in test units that were developed in CAPL and .NET.

During runtime of a CANoe simulation, the debugger can be used to analyze the program flow and to check values of variables.

The debugger enables you to analyze programs that run during CANoe measurement. In the document view of the Debugger Window, you can set breakpoints in the original source code at which the program flow will be stopped. Depending on the type of program and CANoe measurement (real/simulated), the remaining simulation can continue running or will also be stopped. Single step functions enable you to trace the further sequence step by step.

As soon as a breakpoint is reached, all variables of the program are output with their current values in the Variables and Watch areas. After each single step, value changes can be checked in this way.

To resume the simulation, you actuate the Start icon () or the Play key .

Note

  • For test modules and test units, debugging is also possible for measurements with real hardware.
    For network nodes, debugging is only possible in simulated mode, because the complete simulation is stopped.
  • The debugger does not support .NET programs in the Measurement Setup.

  • For .NET programs, Visual Studio can also be specified as the debugger in the Options dialog.

Opening

Using

You can open the CAPL debugger

  • Via the Debug... context menu command for a network node in the Simulation Setup or for a test module (CAPL or .NET) in the Simulation Setup or for a program node in the Measurement Setup.
  • Via the Debug... context menu command for a test module (CAPL or .NET) in the Test Setup
  • Via in the toolbar of the Test Configurations for Test Units or via the Debug... context menu command of an included test configuration
  • Via in the test run dialog of a test module or a test configuration
  • Via in Actions area of application models in Communication Setup

The source code of the program and the related include files are displayed in the document view of the opened Debugger Window.

You can use the Tools ribbon tab to open the individual Debugger Windows.
