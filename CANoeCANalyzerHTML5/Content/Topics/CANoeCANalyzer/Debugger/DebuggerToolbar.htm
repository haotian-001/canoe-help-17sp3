Debugger: Toolbar

 

You can use the toolbar to operate the CAPL debugger.

The icons may vary depending on the context in which the window is opened.

Symbols

Command

Starts the debugging or continues a program until the next breakpoint.

Step Into: In contrast to Step Over, also changes to sub-functions if present in the source code. This also applies to functions in include files. The debugger cannot change to functions of C/C++ DLLs or to functions integrated in CANoe. In this case, the function of Step Into is identical to that of Step Over.

Step Over: Continues the program and, if applicable, the simulation until the next command is reached in the current function. If the end of a function is reached, the program is stopped at the next breakpoint.

Opens/closes the Variables area.

Opens/closes the Watch area.

Opens/closes the Breakpoint area.

Opens the active source code file in the CAPL browser or .NET Editor.

Opens the test run dialog for the respective test module or the test configuration window for the respective test configuration.

Activates and deactivates debug mode.

Opens a dialog with the available source code files.

Opens a dialog with search functions.
Alternative you open the dialog with <Ctrl>+<F>.

Step by Step Procedure: Seach for Line Number

If you searching for a special line number you can use the following function:

  1. Via the key combination <Ctrl>+<G> you open the Go to line dialog.
  2. There you enter a line number.
  3. With OK the cursor jumps to that line.
