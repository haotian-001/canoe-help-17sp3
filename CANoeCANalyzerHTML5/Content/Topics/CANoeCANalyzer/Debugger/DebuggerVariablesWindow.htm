Debugger: Variables

 

The Variables area lists all variables contained in the program in a tabular form. In doing so, variables that are valid at the current breakpoint are black, and all others are grayed. Examples of grayed variables are local variables that can be used only in other sub-functions.

Name

Name of the variable.

Value

Variable value at the time of a breakpoint or for a single step.

Data type

Data type of the variable. The full class name is displayed for objects.

Watch

Selection of whether or not the variable is to be displayed in the Watch area.
The selection is saved in the configuration.

Function

Function or event handler in which the variable is declared and thus valid.

For global CAPL variables, variables is displayed.

When you double-click a variable, it will be highlighted in the Document view.

With Search you can search for variables.

Value display

  • Variables with simple (scalar) data types and variables of type char[] have one line each.
  • Variables of type enumeration have one line each. Their value is displayed as an enumerator (if this type of variable exists for the value).
  • Floating-point numbers are displayed in the value column. Integers are displayed as hexadecimal numbers if the global setting in CANoe is hexadecimal.
  • Variables with array types (except char[]) can be expanded in the first line. There is a sub-line for each element in the array. Multi-dimensional arrays can be expanded multiple times.
    The Value column displays the combined value in square brackets separated by a comma.
    Examples: [1, 3, 5] (type long[]), [[2.0, 1.3], [1.2, -3.14]] (type double[][]).
    The Name column displays the sub-lines of the array index.
  • Variables with struct types can be expanded in the first line. There is a sub-line for each member in the struct. If, in turn, the member is an array or struct, this line can be further expanded.
    The Value column displays the entire value within square brackets and separated by a comma.
    Example: ["Vector", 3.0, 1] for a struct { char name[20]; double val1; long val2; }.
    For sub-lines, the Name column displays the name of the member.
  • Variables whose type is an associative array can be expanded in the first line. A sub-line that can be further expanded exists for each element in the array.
    The Value column displays the complete value in square brackets and separated by a comma, in which case the "Key: Value" format is used for individual entries.
    Examples: ["Vector": 1, "Stuttgart": 2] for a variable "long x[char[]]"; [1: 1.1, 2:1.2] for a variable "double y[long]".
    For sub-lines, the Name column displays the value of the key.
  • Variables whose type is a frame type can be expanded in the first line. There is a sub-line for each member in the type. If database information is available and the global setting of CANoe is "symbolic" (and not numeric), the signals of the message are displayed. Otherwise, the bytes of the message are displayed.
    The Value column displays nothing in the main line.
    If database information is available and the global setting is "symbolic", the Data type column displays the name of the message, in addition to the CAPL data type. Otherwise the CAPL data type and the ID of the message are displayed.
    Examples: "CAN frame: EngineState"; "FlexRay-Frame: (1, 0, 2)".

If a value does not fit in a column, a tooltip containing the value is displayed. For lines that contain sub-lines, the tooltip is displayed with multiple lines. The maximum number of lines displayed is 5; if there are more than 5 sub-lines, the fourth line displays ... instead of the value and the fifth line contains the last value.

Note

The global settings hex/dec and num/sym affect the display. num/sym sets for message objects, if the data field is displayed as byte array or if the signal values are displayed.
