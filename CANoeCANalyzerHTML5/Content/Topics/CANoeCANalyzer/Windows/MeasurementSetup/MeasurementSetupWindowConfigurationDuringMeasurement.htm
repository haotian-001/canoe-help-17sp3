Measurement Setup: Configuration during Measurement

 

You have the possibility to activate/ deactivate specific analysis windows, function blocks or whole analysis branches during measurement. With it you can deactivate analysis branches which are not used any longer, to resolve performance problems or to configure isolated issues.

Analysis windows can be activated/ deactivated by activating a broken hot spot or inserting a breakpoint before the window using the context menu.

Following analysis windows can be activated/ deactivated during measurement:

  • Data Window
  • Graphics Window
  • Trace Window
  • Statistics Window
  • Bus Statistics Window
  • Logging block

Function blocks can be activated/ deactivated by selecting the block and choosing the context menu command Block active.

Function blocks to be activated/ deactivated are:

  • Stop and Pass Filter
  • Stop and Pass Filter for variables
  • Channel filter
  • Trigger
  • MOST Filter
  • MOST Application Filter

Particularities

  • If a function block cannot be activated/deactivated during measurement (e.g. CAPL block), the hot spots which are in the flow chart on the left side of the function block in the Measurement Setup, can also not be activated/ deactivated.
  • Logging files of Graphic, Data Window and the Logging block must have a unique name within the configuration. Otherwise blocks/ windows cannot be activated during measurement. If there is a naming conflict you are requested to configure a unique logging filename for a specific block/ window.
  • You have the possibility to change the logging filename for deactivated blocks/ windows during measurement.

Note

Block/ windows which are not listed above cannot be activated/ deactivated during measurement.

Note

The space key for

  • Activating/ deactivating hot spots,
  • Activating/ deactivating function blocks

cannot be used during measurement.
