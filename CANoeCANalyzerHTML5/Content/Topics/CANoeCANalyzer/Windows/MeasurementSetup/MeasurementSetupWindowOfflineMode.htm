Offline Mode

 

Select Offline Mode

You can switch between online and offline mode in the following ways:

  • By double-clicking the switch symbol
  • Using the context menu of the switch symbol
  • By pressing the <Space> with the switch symbol selected
  • By using the Home ribbon tab

Data Source in Offline Mode

In offline mode, recorded measurement values from a logging file (e.g., BLF, ASC, MF4) are used.

Note on logging files

  • In the sample configuration version, a maximum of 1000 messages can be read in.
  • Here you find further notes for logging file formats.

The Simulation Setup is not active in offline mode. Test Setup is not displayed unless it has been activated for Offline-Mode.

Control Offline Mode

You can control the process of a logged measurement via the Home ribbon tab.

Symbol

Function


Start

Replays the logged measurement data of a logging file. The entire measurement data will be displayed in the analysis windows.

Stop

Stops the replay of the logging file.


Step

Starts or resumes the stepwise replay of the logged measurement data.


Step-length of time step for stepwise data output.

In Offline Mode, you can also use the units h, m, s, and ev by entering them in the field after the corresponding number. The unit ms is hidden because it is the default value.

Animate

Activates an automatic stepwise data output.

Note

The Animate command has nothing to do with the Animation Factor, set in the Options dialog.

  • The animation factor acts only in online mode in the case of pure simulation mode without hardware.
  • The Animate command acts only in offline mode.

Break

Pauses the stepwise data output in the analysis windows.

Note on Step Mode

  • The last message of a step always lies outside of the specified time span. This prevents a situation where no message is displayed in the current step if the next event has a larger time stamp.
  • If a number of events are specified using the ev unit, exactly this number will be read from the logging file. However, additional events (e.g., via observers or the MOST AMS service) can be displayed in the Trace Window.
  • If the logging file has been completely read in, a corresponding note appears in the Write Window at the next single step.
