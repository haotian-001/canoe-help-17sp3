Channel Mapping

Breakpoints

 

In the offline mode, you can adjust the channel assignment for the evaluation. It may be necessary if more channels than you require for your evaluations were recorded or if the channel sequence of the files does not match the configuration.

Channel mappings are assembled in the so-called channel mapping sets. These are defined independently of each other. A channel mapping set can be defined as standard; it is automatically assigned to all compatible offline sources if another set has not yet been assigned to them.

The individual channel mappings are defined by selecting the bus of both the source and the target channels.

Note

If no buses are available in the current configuration (see Channel Usage), no changes can be made to the channel mapping.

Configure Channel Mapping Sets

You can add, delete and define channel mapping sets as default either from the toolbar or the context menu.

You can rename a channel mapping set by clicking on a selected set or pressing <F2> and entering the new name.

Context Menu

Add Set
Adds a new channel mapping set.

Remove Set
Removes selected channel mapping set.
A multiple selection is possible.

Set as Default Set
Defines the selected channel mapping set as default.

Columns

In the column Bus Type you can select from all bus systems that have at least one configured channel in the configuration.

The column Source Channel may contain any channel number between 1 and 255, regardless of the number of configured channels. These correspond to the channels in the logging file, whereas not all channels named in the list need to exist in the currently configured logging file, too.

The Destination Channel however has to be one of the configured channels. Here you can configure the channels of the active configuration on which the messages should be sent. Ignore may be selected here so that the messages of certain channels will be ignored.

The default behavior is to use the channel number from the logging file. Messages from a channel with a channel number larger than the maximum number of supported channels in CANoe (e.g. 32 for CAN) are discarded. That means that all messages in a logging file are forwarded to CANoe without any channel mapping taking place, as long as there is not a matching line in the channel mapping configuration dialog.
