Processing Sequence

The processing of the offline sources can be configured for the offline mode via the following settings:

According to time stamp without reference to date/time (standard setting)

The offline sources are processed according to their time stamp, date and time are irrelevant. This means that all files start with 0 and the events are arranged according to the time interval from the start. It is automatically considered whether the files have absolute or relative time stamps.

Example

In this example, three logging files are processed. A measurement was split to two files, thus the header contains the same time. The files are processed only according to their time stamp.

According to date/time and time stamp

The offline sources are processed according to date/time and the time stamp. It is automatically considered whether the files have absolute or relative time stamps.

This option is only available if all offline sources have absolute time stamps (date and time).

Example

In this example, three logging files are processed. A measurement was split to two files, thus the header contains the same time. The files are processed according to date/time and their time stamp.

According to source's sequence

The offline sources are played in the order in which they are displayed in the Offline Mode Window. You can change the order there by drag and drop.

Example

The files Log_1, Log_2 and Log_3 could be listed e.g. in the order Log_2, Log_3, Log_1 in the Offline Mode Window and would be played in exactly this order during the offline measurement.

Additional Information on ASC Files

CANoe logging files contain the start of the measurement with date and time in the file header. Each saved event within the file contains a time stamp.

It is distinguished whether the time stamps of an event are saved absolutely to the start of a measurement or relatively to the previous event. The header contains the information about whether the time stamps were saved absolutely or relatively.

Time Range

You can use the entire time range from all offline sources or select a time section.

Time format

You can select between

  • Time in seconds
  • Measurement time
  • Date and time (if stored)

To select the time range you can use the fields for Start and End or the time scale.

If the right slider for the end of the section is moved all the way to the right, then it is interpreted as the end of all offline sources. If you add a new logging file which extends the period of a measurement, then the value for the end of the section is automatically extended to the new end of the measurement.

Miscellaneous Settings

Skip simulation time from 0 to first event in logging files

In some logging files, the first event can have a very high time stamp:

  • when logging was started after a long time (e.g. due to trigger settings) or
  • when the first event was logged after a long time due to filter settings.

If you use such a file in the offline mode, it may take a while for the first event to be run. This particularly occurs when using cyclic timers in CAPL (on start).

You can avoid this by activating the Skip simulation time from 0 to the first event in the logging file option. Then, the time timeNow() will already have the time stamp of the first event in on start. Alternatively, you can also start the timer when the first event is processed instead of at the measurement start.
