J1939 Parameter Groups in the Data Window

 

J1939-specific features are taken into account when displaying signals from the J1939 Parameter Groups.

Consideration of Addresses

The representation of a signal from a J1939 Parameter Group depends on how its send and destination address are defined. If the send or destination address (or both) are defined as placeholders (0xFE or 254, respectively) (which may be the case if a signal comes from a J1939.dbc database), the signal is displayed as a separate group in which a subline is created for each SA/DA communication.

Signal Interpretation

When interpreting and displaying a J1939 signal, the SigType database attribute is taken into account.

PGN

  • The cell interpretation is set to Text (via the context menu)
  • A J1939 Parameter Group with the corresponding PGN exists in the database.

Note

If a configuration contains symbols that are not yet supported in the new default view, the Tabular page Branched page (= default view) are displayed.

In this case it is recommended to reconfigure all signals from J1939 parameter groups on the Branched page.
However, it is possible to configure signals from J1939 Parameter Groups also on the Tabular page. For this case, see the following section for the description of the Tabular page.

J1939 Parameter Groups (Tabular Page)

The configuration of the Data Window occurs as in standard mode. Instead of messages, Parameter Groups are selected.

Address Input

If a signal from a J1939 Parameter Group is displayed in the Data Window, attention must be paid whether there are several transmitters of this Parameter Group.

If the address of the Parameter Group is not defined (as is the case for Parameter Groups of the standard database J1939.dbc), the signal is only displayed correctly if there is a transmitter. If the signal is sent by several transmitters, the Data Window always displays the last value received.  The result is a "jumping" signal in the Data Window that always switches between the values of the two transmitters.

To receive a correct signal flow, the sender address must be defined. For this, click Edit... on the context menu of a signal. Now the transmitter address can be entered on the signal dialog after the database connection was interrupted.

Signal Interpretation

The value of a J1939 signal is interpreted according the SigType database attribute.

The Data Window can display string values. The display of the line must be changed to Text or Symbolic and the SigType attribute must be set to a string type. As physical value the number of characters are displayed.

For a PGN or DTC signal type the Parameter Group name or signal name from the database is displayed.

J1939 Indicator

In addition, an indicator can be activated on the context menu. If the indicator is active, the validity of a transmitted signal is evaluated. In CANoe several signal states are distinguished. These are displayed with a multi-color LED in the Value column. The following table shows the assignment of the LED colors to the signal states:

Color

State

red

error, the error code is displayed

white

Parameter Group was not received

green

valid signal, the current signal value is displayed

gray

not available, the last available value is displayed

yellow

other states

  • reserved
  • parameter specific

At measurement start the LED color is white until the first Parameter Group is received that contains this signal. If valid signals (green) or errors (red) are received, the current signal value changes. For transmitted signals in Parameter Groups that are not available (gray), the last displayed value is retained. This has the advantage that only valid signal values are displayed during analysis.

This mechanism is only supported if suitable attributes are specified in the database during the definition of the signals in the database.

J1939 Source Address (SA)/J1939 Destination Address (DA)

In these columns the addresses of the signal Parameter Groups are displayed for J1939 signals. The presentation depends how the address is defined in the signal configuration.

  • concrete value: the defined address is always displayed
  • wild card *: "--" is displayed from measurement start till reception of the first signal, then the address of the last received signal is displayed. If a signal has the value "not available", the address does not change.
