Application Panel

Ribbon: Simulation Tab|Application Panel

Select Application Layer Objects

You can open the Select Application Layer Objects dialog via Application Panel|New of the Simulation ribbon tab.

All application layer objects contained in the configuration are listed here.

Select the desired application layer object and confirm the selection with [OK] (multiple selection is possible).

A separate application panel is opened for each selected application layer object.

The Application Panel can be used to display and stimulate participants (COs) and distributed objects (DOs) from the application layer if the Simulated (CANoe) state is set for them in the Communication Setup (Properties).

Application Panel

General

In the Application Panel, you can display and stimulate application layer objects at runtime. You can call Getters and Setters of fields as well as methods. You can trigger events and set data values and filed values.

For COs, you can establish the connection to a service (service discovery). For reference DOs and reference DO member, you can set the target DO.

The calls and associated responses of the communication counterpart are displayed in the history.

Save calls that you want to reuse with their parameters as favorites. You can also play these as a sequence, for example.

Note

You can also open the Application Panel via the Symbol Explorer, the Data Window or the Graphics Window (context menu Change value...). Only the main view for methods, functions, fields or data is displayed here so that you can quickly display and change the value for the selected element at runtime.

Favorites

On the left side below you can see your saved favorites.

The Favorites area allows you to quickly find and call frequently used member of a service (method, event, field) or of a DO (data, event, field, method).

Note

Only the member's send side can be added as a favorite.

All parameter values and settings (e.g. counterpart or reference target) are saved. You can also use a favorite as a template and adjust the stored values before resending in the Stimulation view. With in the toolbar of the main view your changes are saved and applied. Alternatively, you can create a new favorite with the changed values using .

In addition to the context menu and the toolbar in the main view, you can also add element to the favorites using drag and drop.

Repeats all stimulation steps stored under Favorites/Folders one after the other. You can change the order with drag and drop in the tree.

Stop Sequence
Stops a running sequence.

If you select a stimulation call from the favorites, it can be executed again by clicking the corresponding stimulation button (e.g. [Set] or [Call]).

Sequences

You can play all stimulation calls stored under Favorites as a sequence. You can use folders to structure your favorites and thus also play back only a certain set of stimulation calls as a sequence. Subfolders are also played back when a sequence is played back.
The order of the elements here corresponds to the execution order within the sequence. You can adjust the order using drag and drop.

You can rename folders as well as individual stimulation calls.

A sequence can be paused during its execution.

If you select Favorites or a subfolder, you can use Wait for return in the main view to activate that when playing the sequence, the return value of the currently played element should be waited for before the next element is played.

Note

This setting only affects stimulation calls that have a return value (e.g. method calls).

An indicator () in the main view shows at runtime of a sequence which element or which subfolder is currently being played. Likewise, the actions (e.g. Call) are displayed.

In the main view a sequence can be started with [] and stopped with [].

Note

Playing sequences allows you to automatically execute several stimulation calls in a predefined sequence. There is a slight delay between the individual calls so as not to overload the system.

Main View

In the main view on the right side, you will get the following information and stimulation options, depending on the selection in the navigation tree:

Participants (COs)

Here you get an overview of the available services and their roles (consumer/provider).

Service

Here you get the following information and stimulation options for the service:

  • Role
    The role of the endpoint in the participant (consumer or provider)
  • Endpoint
    The endpoint in the participant
  • Service state
    Display of the service state. You can start the service with [Provide Service] or stop it with [Release Service] (only for provider).
  • Counterpart
    Selection of the endpoint of the counterpart
    The selection affects all methods, events and fields of this service.
  • Connection state
    Display of the connection state to the service of the selected counterpart.
    You can establish the connection to the service of the selected Provider counterpart with [Request Service] or terminate it with [Release Service] (only for consumer).

Method

Methods can be stimulated on the consumer consumer side. On the Provided side you will see the values from the last call.

Counterpart

Selection of the endpoint of the counterpart.

Value

  • Consumer
    In the value area you can set the call parameters of the method.
  • Provider
    In the value area the last called call parameters are displayed.

Stimulation Call

  • Consumer
    With [Call] you call the method with the set parameters at the set counterpart.
    The response is displayed on the Response page.

Field

You can query the value of a field on the consumer side with [Get] and set it with [Set].
On the provider side, the value for a specific consumer counterpart or for all consumer counterparts (Provider Value) is displayed. You can change and set this value. Note that Get, Set and the Field notifications are optional.

Counterpart

Selection of the endpoint of the counterpart

Value

  • Consumer
    In the value area you can set the value for the field or display the current value.

  • Provider
    In the value range you can see the value that the field has for the selected counterpart. Here you can set the value that the field is to receive

Stimulation Call

  • Consumer
    With [Set] you send the set value to the selected counterpart.
    After calling [Get] the current value of the selected counterpart is displayed here.
  • Provider
    With [Apply] the set value is sent for the selected counterpart or for all counterparts (Provider Value). If a notification is configured for the field, the consumers are notified now.

Event

Events can be stimulated on the provider side for a specific consumer counterpart or for all consumer counterparts (Provider Value). On the consumer side, the values from the last call are displayed for a specific provider counterpart.

Counterpart

Selection of the endpoint of the counterpart.
You can select either a specific or all consumer endpoints.

Value

  • Consumer
    In the value area the parameter values of the last call are displayed for the set counterpart.
  • Provider
    In the value area you can set the parameters of the event.

Stimulation Call

  • Provider
    With [Trigger] you trigger the event with the set parameters at the set counterpart or at all counterparts.

/ Distributed Objects (DO and Reference DO)

Here you get an overview of the interface and binding of the distributed object.

For reference DOs, the active target DO is displayed, which you can change here.

Method / Field / Event / Data

The current values of the member are displayed here. Depending on the direction (Consumed/Provided/Internal), the members offer different stimulation options.

Note

Note that stimulation and display are dependent on the used data model. For example, you can only stimulate a field set if the field's setter is configured in the data model.

Method

The Request page displays the call parameters. With [Call] you call the method with the set call parameters. The Response page displays the return values.

Field

You can query the value of a field with [Get] and set it with [Set]. With [Apply] you can set the value of the field.

Event

With [Trigger] you trigger the event with the set parameters.

Data

With [Apply] the set parameters are sent.

Embedded Member / Reference Member

Here you get an overview of the interface and binding of the embedded member or reference member.

For reference members the active target DO is displayed, which you can change here.

Note

Nested references, i.e. reference members in reference DOs or reference members within other reference members, are not supported by the Application Panel.

Member Container

Here you get an overview of the member container interface.
You can change the active index of the container displayed here..

Note

Multidimensional containers are not supported by the Application Panel.

History

In addition to measurement start and stop, all executed stimulation calls and their responses are displayed in a history on the right side below in the application panel.

  • Select a line to display the call parameters or the received data in the main view.
  • A stimulation call can thus be easily repeated.
  • If a line with a stimulation call is selected, the corresponding response line is highlighted. Display return values by selecting the response line in the value area.
  • The history can be deleted at any time via the context menu.
