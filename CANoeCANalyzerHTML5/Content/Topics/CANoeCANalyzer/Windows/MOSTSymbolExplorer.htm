MOST Symbol Explorer

(CANoe) Ribbon|Analysis Tab|Symbol Explorer
(CAPL Browser) Ribbon: Layout Tab|MOST Symbols

You can insert the symbols from the MOST Symbol Explorer in the Data or Graphics Window using drag and drop.

An XML function catalog for MOST can be added to a configuration by inserting the XML file in the System view of the Simulation Setup.

Note

The window position is not stored in the configuration.

Additional Qualifiers with drag and drop

The input of the symbol attributes depends on which mouse button you use to drag the symbols to a Data or Graphics Window:

  • Left mouse button
    If you drag the symbols to a window using the left mouse button, all other settings, such as the channel or displayed name, are assigned with default values.
  • Right mouse button
    If you drag the symbols to a window using the right mouse button, another dialog opens prior to the configuration of the window, where you can set additional attributes. In addition to limiting the channel and InstanceID in the message, you can enter the name in the form it will appear in the signal list.

For structured parameter types, such as Array or Stream, you can enter an index that specifies, for example, what number array element is to be displayed in the Data or Graphics Window.

Search Option

You can search for a text of your choosing in the MOST Symbol Explorer (the search takes in all available texts). Press <Enter> or to start the search.
The search function does not support the wildcard character ("*").

Tree View

The MOST Symbol Explorer can not only be opened via the Analysis ribbon tab but also from within different windows and function blocks. The displayed tree view depends on opening the MOST Symbol Explorer.

MOST Signal Selection

Use the MOST Symbol Explorer to choose

  • a MOST bus statistic signal (ControlMessageRate [fr/s], ControlMessages [total], …),
  • a MOST system signal (Light, Lock, SBC, MPR, ...) or
  • a signal parameter from the XML function catalog

and insert them into the Data or Graphics Window.

  • The tree view shows all parameters that are selectable in the current configuration.
    The MOST bus statistic signals and the MOST system signals are available independent of function catalogs and are always listed directly under the topmost nodes Bus Statistic Signals and System Signals.
    Next are the XML function catalogs that are assigned to the current configuration. The several levels of the tree contain
  1. Function Catalogs
  2. FBlocks
  3. Functions
  4. OpTypes
  5. Parameters and possibly Sub-Parameters on further levels.

For the Graphics Window you only may select parameters with scalar data type or scalar enumeration type.

In addition to that, in the context of the Data Window string parameters and bit fields may be selected, too.

  • For an expedient restriction of the parameter selection on parameters that correspond to system states that usually are valid throughout the measurement, the display mode of the tree view can be switched by a selection list.
    • The Show all Parameters setting displays all parameters.
    • The Show status parameters / Property values only setting reduces the tree view to the status messages and all parameters beneath.

The following settings are only available if you open the MOST Symbol Explorer via a window and not via the Analysis ribbon tab:

  • The parameter name and the data type with value range of a parameter selected from the function catalog are displayed in the Current Selection area. The parameter name can be edited; as a default it is identical to the parameter address. To switch back to the default after assigning a user-defined name, you just have to delete the name from the edit field. After leaving the edit field the parameter address appears again.
    If the parameter address contains an array index, this index can be edited by pressing the [Index] button or editing directly in the edit field. Please note that after assigning a user-defined name the array index is no longer visible but still being used.
  • In the Additional Message Qualifiers area you can assign an instance number and a channel number to the signal, for these numbers are not defined in the function catalogs.
    For the predefined MOST system signals only the channel number can be set.
    When adding signals always the last used instance-/channel number is proposed.
    "*" in the entry field is up to the wildcard value 0xFF.

If this dialog is used for changing already configured signals, the tree view is deactivated and the parameter name of the signal is fixed. This means that the signal itself cannot be changed; just the edit fields for the signal name and the message attributes are active and show the current value.

MOST Message Selection

All messages of the loaded XML function catalogs are displayed in the tree view:

  • Level 0: XML function catalog
  • Level 1: FBlock
  • Level 1b: Instance (This level appears only if the instance-dependent interpretation of function blocks is active).
  • Level 2: Function
  • Level 3: OpTypes

You can select an individual message (Level 3), or all OpTypes of a function (Level 2), or all OpTypes of all functions of an entire function block (Level 1).

In the latter two cases the appropriate number of lines

  • will be inserted in the send list of the MOST Interactive Generator (one for each resulting message) or
  • are added (one for each resulting message) to the MOST Filter's message list.

Context Menu

Sort Functions Alphabetically
Sorts Block, Function, and OpType in alphabetical order.

Sort Functions by ID
Sorts Block, Function, and OpType by ID.

Show IDs
The IDs of FBlock, Function, and OpType are generally helpful and provide additional information regarding the classification of the respective symbols in the function catalog. For easier navigation via keyboard entry of symbol names, the display of IDs can be disabled.

Show FBlock Instances

Show Tooltips
The MOST Symbol Explorer provides tooltips for a more detailed description of the symbols available for selection. These provide information about attributes such as method or property and data types, and displays comments that are present in the function catalog for the symbols. Because the latter are quite extensive or poorly maintained in many function catalogs, the tooltips can also be disabled if desired.

 

Note on CAPL Browser

From the MOST Symbol Explorer you can insert via drag and drop

  • symbolic parameter names and addresses
  • symbolic message names or parts thereof

from XML function catalogs. In CANoe XML function catalogs must be inserted in the Simulation Setup in order to be able to use the MOST signal or message selection dialog.
