Export/Conversion

Ribbon: Tools Tab|Logging File Conversion
Context Menu Logging File: Export…
Context Menu Trace Window: Import/Export|Export…/Export Selection…
Context Menu Graphics Window (Diagram): Import/Export|Export…/Export Clipping...

With the help of this dialog you can…

  • …save a logging file under a new format.

    Data of the logging file can be saved signal based or message based.

  • …save the contents of the Trace Window to a file.

    Data of the Trace Window can be saved signal based or message based.

  • …save the contents of the Graphics Window to a file.

    Data of the Trace Window can be saved signal based.

Note

Effective with version 6.0 it is possible to use long symbolic names (> 32 characters). These are displayed for the ASCII logging if in the standard toolbar the icon sym is active.

The produced ASCII files can be read in older versions of the program and, however, messages with long symbolic names are ignored.

If you like to read ASCII files without exceptions in older versions, please, log numerically, while you deactivate the icon sym.

The newly created file can then be opened automatically in another program for further processing.

Signal-based/Message-based Export or Conversion

The display of the dialog depends on Export or Conversion functionality.

Destination File/Source File/Destination Format

Signal-based export (logging file/Trace Window/Graphics Window):
Only the selected signals in section Signal Information will be written in the destination file.

Message-based export (logging file/Trace Window):
The complete file will be saved, therefore the signal selection feature is inactive.

Note

  • Format
    The Format shows information on the selected target format.
  • Status
    The Status indicates the progress of the saving process in percent complete.

The following file types are available for the destination file:

Format

Action

Signal-based Formats

  • These formats do not contain messages but single signals.

CSV

Character Separated Values (ASCII format)

  • File type: CSV Signal Logging Files
  • Context: logging file/Trace Window/Graphics Window

MAT

MATLAB® format (binary format)

  • File type: MATLAB Signal Logging Files
  • Context: Graphics Window/Trace Window/logging file
  • In MATLAB® format names must conform to the MATLAB® convention for variable names, i.e. they must begin with an alpha character and may only contain alphanumeric characters or underlines ( _ ).
    Invalid characters are replaced by an underline before saving.

  • CANoe supports MATLAB® format Level 4 and Version 7.3 (HDF5 based)

MDF/MF4

Measurement Data Format (binary format, signal-based format, MDF version 2.0–4.1)

  • File type: MDF Signal Logging Files
  • Context: logging file/Trace Window/Graphics Window

Note

  • Up to and including CANoe 14, only the MDF extension was supported for signal-based MDF logging files. Importing a signal-based logging file with the MF4 extension will fail.

Message-based Formats

  • These formats contain messages; therefore you cannot select single signals.
  • These formats are only available for the Trace Window and for logging files.

ASC

Standard ASCII representation

  • File type: ASCII Frame Logging Files
  • This format stores the data as a readable text file.
  • The ASCII format is used for data exchange with third party programs or to include trace data in documents.

BLF

Binary Logging Format (binary format)

  • File type: BLF Frame Logging Files
  • The BLF format supports all bus systems and protocols as well as system variables.
    For new logging files we strongly recommend to use the (binary) BLF format.

MF4

Measurement Data Format (binary format, MDF version 4)

  • File type: MDF Frame Logging Files
  • This format supports all bus systems and protocols as well as system variables. CAN, LIN and FlexRay will be stored according to the ASAM standard. All other bus systems and protocols are stored Vector-specific.

XML

XML representation

  • File type: XML Disassembly Logging Files
  • This format exports data exactly as they are shown in the Trace Window.
  • It is used to display message sequences in other applications and cannot be re-imported.

Note

  • Starting with CANoe 13.0, XML export is switched off by default. The export can be switched on in the [System] section of the in CAN.ini file with the switch ExportXMLEnabled=1.
  • The colors used for export are, except for the background, the same as for the Trace Window. You may change these colors in the Options dialog (Windows/Blocks|Trace Window|Event Highlighting).
  • Additionally, you may override these settings by changing the XSLT stylesheet in the Export and Logging Settings dialog (XML Settings|XSLT Stylesheet).

PCAP/PCAPNG

Packet Capture (binary format)

  • File types:
  • PCAP Ethernet packet Logging
  • PCAPNG Ethernet packet Logging
  • These formats are only supported by the Ethernet bus system.
  • For PCAP, channel number and transmit direction (Rx, Tx) are not supported.
  • For PCAPNG, the Direction is not supported.

Time Range (Conversion only)

Here you can set the time range within the logging file (source file) that should be converted.

Time format

You can select between

  • Time in seconds
  • Measurement time
  • Date and time

To select the time range you can use the fields for Start and End or the time scale.

Signal Information

Here you can load signal information of the source (Trace Window/Graphics Window) or the source file (logging file), respectively.

With signal-based export you may select single signals. You can configure whether all signals or only selected signals shall be saved/exported.
You can also use your last selection, select all signals or cancel your selection.

Remember signal selection

You can keep the signal selection for another export. With the Remember signal selection check box, the export dialog saves in memory the unique signal names of all selected signals. If a new source file (e.g., another logging file) is selected or the export is performed again, the signals from the new logging file are extracted with the help of the database and compared with the previously saved signal names. Found signals are then selected again automatically.
In this way you can always export the same signals for multiple sources.

Note

The signal selection is stored neither in the CANoe configuration nor in the logging file.
If a previously selected signal is not found, no error message will be displayed.

Signal List Context Menu

Via the Save signal selection context menu command you can save the currently selected signals in a text file.

Via the Load signal selection context menu command you can load saved signals selections from a file and apply it to the list of extracted signals.

Note

The functions for saving and loading the signal selection are available only for logging file conversion and exports from Logging Blocks.

Advanced Settings

Here you can open via [Settings...] a configuration dialog in which you can set global specific settings of the selected target format. Additionally you can select an action that shall be performed after successful export. Furthermore you can create your own actions under Global Export Settings and define them as default action for target formats.

Export Selection/Clipping

If you open the dialog in the Trace Window via Export Selection... context menu command, only the marked data of the Trace Window will be exported.

If open the dialog in the Graphics Window via Export Clipping... context menu command, only the visible data of the Graphics Window will be exported.
