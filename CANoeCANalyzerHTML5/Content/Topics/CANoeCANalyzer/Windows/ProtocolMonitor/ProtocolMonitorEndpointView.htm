Endpoints View

 

The hierarchical tree view shows the endpoints that have been observed in the network so far. Depending on the setting, the endpoints are listed according to their affiliation to network nodes or networks.

Only the endpoints that belong to the currently active network selection are displayed.

With <ESC> you cancel a selection.

Filtering & Hiding

You can refine the view further

  • by filtering endpoints according to certain criteria
  • by hiding single endpoints (see Symbols below)
  • by using the context menu commands Show All Endpoints and Hide All Endpoints you can toggle the visibility of all endpoints at once

Note

Endpoints can be invisible because they have been either filtered out or hidden (see example below).

Filters for the Trace Window can be created via the context menu.

Examples

Example: Filtering with deactivated filter binding

The filters for all protocol layers and Only Relevant Layers have been activated, the filter binding for the Endpoints View is not activated.

  • If an Ethernet packet with IPv4 and UDP protocols is observed, the Ethernet and IPv4 endpoints are filtered out.
    The Ethernet and IPv4 endpoints are therefore not shown in the Diagram View, and marked with the filter symbol in the Endpoints View.
  • The UDP endpoints are not filtered out and are therefore visible in both views.

 

Example: Hiding

If the UDP endpoint 40000 is hidden by clicking on the symbol, it is no longer visible in the Diagram View and marked with the symbol in the Endpoints View.

Example: Filtering with activated filter binding

If the filter binding is activated in the Endpoints View with the filter symbol , then only the UDP endpoints are displayed in the Endpoints View.

Filter

If the filter binding is activated, then only those endpoints of the currently active network selection are displayed that are not filtered out.

If the filter binding is not activated, then all endpoints of the currently active network selection are displayed.
Filter Selection

Here you select according to which criteria endpoints are to be filtered out. All selected filter criteria are combined with each other. The selection directly affects the Diagram View.

You can apply the following filters:

Only Relevant Layers

If the filter is activated, then the endpoints that do not belong to the highest protocol layer in the stack are filtered out.

In addition, the endpoints that do not belong to a connection of the highest protocol layer in the stack are also filtered out.

Bind To Connections

If the filter is activated, then the endpoints that do not belong to the protocol stack of a displayed (i.e. non-filtered) connection are filtered out.

Node Types

 
  • Broadcast

If the filter is not activated, then the broadcast network node is filtered out.
  • Multicast

If the filter is not activated, then the multicast network node is filtered out.

Protocols

 
  • Data Link, Network, Transport, Session, Presentation, Application

If the filter is not activated, then the endpoints of the corresponding OSI layer are filtered out.

The endpoints of individual protocols can also be filtered out. The protocols are listed under the OSI layer to which they are assigned.

Example: Bind To Connections

  • The endpoint filters are activated for all protocol layers.
  • The endpoint filters Only Relevant Layers and Bind To Connections are not activated.
  • The filter binding for the Endpoint View is activated.
  • For the connection filters in the Connection View, only SOME/IP is activated for the protocols.

In the Endpoint View and the Diagram View all endpoints are displayed.

If the Bind To Connections filter is activated, the endpoint stacks containing the endpoints of the displayed SOME/IP connections are displayed in the Endpoint View and the Diagram View.
