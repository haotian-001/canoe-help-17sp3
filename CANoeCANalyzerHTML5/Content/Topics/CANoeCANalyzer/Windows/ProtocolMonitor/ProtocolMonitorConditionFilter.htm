Stop and Pass Filters

 

Open the area for the Stop and Pass Filters via the splitter on the left edge of the Connections View.

The Stop and Pass Filters work in combination with the predefined filters that you can configure and activate via the toolbar of the Connections View. Connections that pass the predefined filters are then processed by the Stop and Pass Filter.

A Stop and Pass Filter can contain any number of conditions. The conditions are combined with a logical operator:

  • AND operator: the filter affects the connections that fulfill all conditions.
  • OR operator: the filter affects the connections that fulfill at least one of the conditions.

Filters and conditions can be created or deleted (<Del>) both from the toolbar and from the context menu.

Via the context menu of a filter you can

  • convert a Pass Filter into a Stop Filter (and vice versa)
  • change the logical operator (AND, OR) of the filter
  • rename the filter (<F2>)

Filters and conditions can be copied (<Ctrl>+<C>) and pasted (<Ctrl>+<P>) via the context menu.

You can create as many Stop and Pass Filters as you want, but you can only activate one filter at a time.

Conditions

Two basic types of conditions are distinguished: property conditions and combination conditions.

Property conditions and combination conditions can be created within a Stop and Pass Filter as well as within a combination condition.

Property Condition

A property condition specifies for which values of a certain property a filter affects a connection.

A property condition always consists of a specified property, an operator and a value.

Double-click on the property condition or click the icon to open the editing dialog of a property condition

Combination Conditions

You can combine conditions with logical operators to create combination conditions. The used logical operator determines how the combination condition will be evaluated.

Operator

Description

AND

And combination : the combination condition is fulfilled if all combined conditions are fulfilled.

OR

Or combination : the combination condition is fulfilled if at least one of the combined conditions is fulfilled.

Both property conditions and combination conditions can be combined in combination conditions.

Creating Filters by Means of Text Input

The Protocol Monitor Query Language allows the description of filters in text format. Clicking on in the toolbar opens the text input area, which allows you to compose the filter directly. You can also read a text representation of an existing filter here.

As an aid, a list of valid input options is displayed when typing. If the entered text is incorrect, then the input field is highlighted in red and a tooltip provides further information about the input error.

Clicking <Enter> creates a Pass Filter from the entered text.

Clicking <Ctrl>+<Enter> creates a Stop Filter from the entered text.

Note

You cannot edit an existing filter with the text input. A new filter is always created.
