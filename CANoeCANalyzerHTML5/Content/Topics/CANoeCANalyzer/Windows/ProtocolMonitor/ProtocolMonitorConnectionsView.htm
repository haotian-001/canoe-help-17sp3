Connections View

 

The overview of all connections is used to analyze the network communication. You can select the displayed columns via the toolbar.

The content of this view can be copied to the clipboard in text form via the context menu or with <Ctrl>+<C>. With <ESC> you cancel a selection.

Filters for the Trace Window can be created via the context menu.

Column Filters

For each column you can configure a column filter to filter the displayed connections based on their properties.

When you move the mouse button over a table heading, a filter icon appears in the upper right corner. By clicking on the filter icon, you can configure the filter or reset it. The configured column filters only affect the Connections View and not other views.

You can reset all column filters via the context menu.

Stop and Pass Filters

Open the Stop and Pass Filters area via the splitter on the left edge of the Connections View.

Toolbar

Symbol

Function

Filters

If the filter binding is activated, then only those connections of the currently active network selection are displayed that are not filtered out.

If the filter binding is not activated, then all connections of the currently active network selection are displayed.

Predefined Filters
Filter Selection

Here you select according to which criteria connections are to be filtered out. All selected filter criteria are combined with each other. They also work in combination with active Stop and Pass Filters. The selection directly affects the Diagram View.

You can apply the following filters:

Only Highest Layers

If the filter is activated, then the connections that do not belong to the highest protocol layer in the stack are filtered out.

Depending on the selection of the displayed layer for the endpoints, connections of different layers can be displayed.

Statuses

 
  • Status 'Down'

If the filter is not activated, then connections in the 'Down' status are filtered out.
  • Status 'Pending'

If the filter is not activated, then connections in the 'Pending' status are filtered out.

Protocols

 
  • Data Link, Network, Transport, Session, Presentation, Application

If the filter is not activated, then the connections of the corresponding OSI layer are filtered out.

The connections of individual protocols can also be filtered out. The protocols are listed under the OSI layer to which they are assigned.

Column

Description

OSI Layer

OSI layer to which the connection is assigned. The OSI layers are identified with the following symbols:

Data link layer

Network layer

Transport layer

Session layer

Presentation layer

Application layer

Time

Last time at which the connection was observed.

Network

(Physical) network in which the connection is located.

VLAN ID

If the connection is in one or more virtual networks, their VLAN IDs are displayed here.

Protocol

Protocol to which the connection belongs.

Status

Connection status: Up, Down or Pending.

Initiator

Endpoint that was the source when this connection was first observed.

For the initiator, its node as well as the parent endpoints of the different OSI layers can be displayed.

Direction

Directions with which the connection has been used so far.

Addressee

Endpoint that was the destination when this connection was first observed.

For the addressee, its node as well as the parent endpoints of the different OSI layers can be displayed.

# Validation Results

Number of validation results that exist for this connection.

# Events

Number of all events that affected the connection.

# Events to Addressee

Number of events at which the connection was used from initiator to addressee.

# Events to Initiator

Number of events at which the connection was used from addressee to initiator.

Filtered

Indicates with a filter symbol that the connection has been filtered out via the filter settings.

Example

The filters for all protocol layers and Only Highest Layers have been activated.

If an Ethernet packet with the protocols IPv4 and UDP is observed, only the information of the UDP connection is listed with activated filter binding ().

With deactivated filter binding the information of the Ethernet connection, the IPv4 connection and the UDP connection is listed.
