Concept

 

Most protocols, such as those in the TCP/IP reference model, follow a layered architecture (cf. ISO/OSI reference model). Each layer has a specific task, from pure bit transmission to the transmission of application data, and thus provides the infrastructure for the next higher layer. Most protocols can be assigned to a specific layer; some protocols also fulfill the tasks of several (adjacent) layers. In real communication, for example, there is rarely just one protocol involved, but usually an entire protocol stack.

What all protocols have in common is that they allow information to be transported from a source to a destination. The source and destination form the endpoints of the communication. To transport information, there must be a connection between the endpoints.

The layered architecture described above means that endpoints and connections themselves are also layered, following the protocol hierarchy. Hierarchically stacked endpoints and connections are the core element types of the Protocol Monitor.

Example of the hierarchy in the representation of the endpoints

This example is intended to clarify the terms parent, child, build on, lowest, highest and root.

Grouping element: Node

Ethernet endpoint 20:00:00:00:01:04

Endpoint of the lowest layer = root endpoint

Parent endpoint of the IPv4 endpoint 192.168.1.4

IPv4 endpoint 192.168.1.4

Child endpoint of Ethernet endpoint 20:00:00:00:01:04

IPv4 endpoint builds on Ethernet endpoint

Parent endpoint of UDP endpoint 50001

UDP endpoint 50001

Child endpoint of IPv4 endpoint 192.168.1.4

UDP endpoint builds on IPv4 endpoint

In the example, the UDP endpoint is the highest layer.

Endpoint

An endpoint is a communications endpoint that can act as the source or receiver of a connection. Endpoints can also represent multicasts or broadcasts, in which case they do not correspond to a unique receiver.

Each endpoint belongs to a protocol and is part of a hierarchy. An endpoint therefore usually has a parent endpoint, and can itself be the parent endpoint for other child endpoints that build on it.

Connection

A connection symbolizes the communication between two endpoints. A connection can contain multiple frames in the Trace Window.

Each connection belongs to a protocol and is part of a hierarchy. A connection therefore usually has a parent connection, and can itself be the parent connection for other child connections that build on it.

The assignment of the roles initiator and addressee of a connection by the Protocol Monitor is based on the first observed frame belonging to a connection. The source of the first frame automatically becomes the initiator and the destination becomes the addressee.

Network Node

A network node corresponds to an ECU and usually includes one or more endpoints of the lowest protocol layer. The Protocol Monitor assigns endpoints to those network nodes that are defined in the CANoe configuration or defined in an attached database. Endpoints corresponding to multicasts or broadcasts are also grouped under two specific network nodes, but which do not correspond to any ECU.

If an endpoint is observed that cannot be assigned to a known network node, it is assigned to a new, unknown network node (). The unknown network node gets a name that corresponds to the name of its root endpoint. In case of name equality with already existing nodes, an increasing index is appended.

Network

A network is a grouping of endpoints that form a broadcast domain and can reach each other directly. Two types of networks are distinguished:

  • a physical network belongs to a specific bus system, such as Ethernet;
  • a virtual network, such as a VLAN, is a grouping within another (virtual or non-virtual) network.

Networks can thus form a hierarchy.

Fully Qualified Name

Each network node, network, endpoint and connection in the Protocol Monitor has a fully qualified name that uniquely describes it. This name incorporates the entire hierarchy of the object.

Network node

The fully qualified name corresponds to the node name.

Network

The fully qualified name is composed of the name of the physical network with the VLAN IDs based on it.

Example

Multimedia::17::11 denotes the virtual network with VLAN ID 11, which is located in the virtual network with VLAN ID 17, which in turn is located in the physical network named Multimedia.

Endpoint

The fully qualified name is composed of the fully qualified name of the network in which the endpoint is located and the protocol layers based on it. Each protocol layer is qualified with the protocol name and the ID of the endpoint.

Example

Multimedia::Ethernet[02:00:00:01:04]::IPv4[192.168.1.4]::UDP[50001] denotes the UDP endpoint 50001, which builds on the IPv4 endpoint with address 192.168.1.4, which in turn builds on the Ethernet endpoint with address 02:00:00:01:04. The endpoint stack is located in the physical network Multimedia.

Connection

The fully qualified name is composed of the fully qualified name of the network in which the connection is located, the protocol, the initiator and the addressee of the connection. Initiator and addressee are qualified without network and separated with a comma.

Example

Multimedia::IGMP[Ethernet[02:00:00:01:03]::IPv4[192.168.4.3],Ethernet[01:00:5E:00:00:16]::IPv4[224.0.0.22]] denotes the IGMP connection in the network Multimedia that is established between the IPv4 endpoint 192.168.4.3 (which builds on the Ethernet endpoint 02:00:00:01:03) and the IPv4 endpoint 224.0.0.22 (which builds on the Ethernet endpoint 01:00:5E:00:00:16).
