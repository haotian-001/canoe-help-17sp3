Creating Filters for The Trace Window

 

The Protocol Monitor builds a model of the network from the observed data traffic. However, it does not display all the details of the data traffic as the Trace Window does.

However, you can use the Protocol Monitor to create filters for the Trace Windows of the open CANoe configuration. This filter will filter those events that belong to a specific element of the Protocol Monitor's network model (for example, an endpoint or a connection).

For endpoints, connections and networks you can create filters for the Trace Window using the context menu command Create Filter for Trace Window. Here you also select the Trace Window from a list and thus open the configuration dialog for the trace filter. The filter consists of one or more filter groups in the analysis filter area of the Trace Window.

With the Filter type you can define whether a Pass Filter or a Stop Filter is to be created.

Note

For some elements you can either create no Stop Filters or no filters at all. The corresponding entries in the context menu or in the dialog are then deactivated

You can determine the filter name yourself. It is used for easier identification of the created filter groups. It does not have to be unique.

Note

When creating filters, it is not taken into account whether there are already existing filters with the same name.

Example

In the CANoe configuration there are the two Trace Windows with the name CAN Trace and Ethernet Trace.

With a right click on a TCP endpoint in the Diagram View the filter dialog for the Trace Window Ethernet Trace is opened.

The filter name is set to TCP 38565.

After closing the dialog with [Create] the two new filter groups Pass: TCP 38565 (as Source) and Pass: TCP 38565 (as Destination) are displayed in the Trace Window Ethernet Trace:
