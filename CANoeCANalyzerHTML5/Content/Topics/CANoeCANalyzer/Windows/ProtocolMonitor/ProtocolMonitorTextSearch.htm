Text Search

 

To keep an overview in large networks, the Protocol Monitor offers a text search. You can open the text search via the icon in the Diagram View.

If you enter a search text, only the objects whose properties contain the desired search text will be displayed. If you do not enter a search text, then all objects will be listed.

Columns

Column

Description

Type

Specifies the type of the found object:

  • Network Node
  • Network
  • Endpoint
  • Connection
  • Validation Result

Protocol

Specifies the protocol or bus of the found object.

Object Name

Contains a description of the identity of the found object.

Matches

Contains the properties because of which the found object is a search hit.

If the search text is empty, all properties of the object are listed.

Search Settings

You can open the search settings via the icon in the text search:

Setting

Function

Extended Mode

If this setting is activated, then you must enter a search text in the format of the Protocol Monitor Query Language. This also allows more complex search queries to be realized.

If the entered text is incorrect, the input field is highlighted in red and a tooltip provides further information about the input error. As an aid, a list of valid input options is displayed when typing. In the footer of the text search, the search text is displayed that was used to create the currently displayed search hits.

Regardless of the data type, the properties are all interpreted and searched as text.

If this setting is not activated, the entered search text is used directly.

Options

Match Case

If this setting is activated, then only search hits are found where the upper and lower case letters match those in the entered search text.

This setting has no effect in Extended Mode, where case sensitivity is not taken into account.

Match Whole Word

If this setting is activated, then only search hits are found where the entered search text corresponds to a whole word.

This setting has no effect in Extended Mode, where all hits are considered regardless of word boundaries.

Types

 

With these settings you can reduce the object types to be searched:

  • Network Node
  • Network
  • Endpoint
  • Connection
  • Validation Result

Properties

The text search considers the following properties:

Property

Type

Description

Name

Network Node, Network, Validation Result

Name of the object. For validation results the description text is searched.

Protocol

Network Node, Network, Endpoint, Connection

Protocol to which the object belongs. For networks, the bus type is used. For network nodes, the bus types of all networks in which the node was observed are used.

Validation Results

Endpoint, Connection

Validation results that exist for this object. The description text of the validation result is searched.

ID

Endpoint

Protocol specific ID of the endpoint.

Initiator

Connection

Initiator of the connection, i.e. the initial source endpoint.

Addressee

Connection

Addressee of the connection, i.e. the initial destination endpoint.

Ethernet

Is Locally Administered?

Endpoint

Indicates whether the MAC address is locally administered.

  • true: MAC address is locally administered
  • false: MAC address is not locally administered

AUTOSAR PDU Multiplexing

Rx PDUs

Endpoint

List of PDUs (name and frequency) received from the endpoint.

Tx PDUs

Endpoint

List of PDUs (name and frequency) sent from the endpoint.

PDUs

Connection

List of PDUs (name and frequency) sent over the connection.

ARP

Requests

Connection

List of ARP requests (sender IP address, destination IP address and frequency).

Replies

Connection

List of ARP replies (sender IP address, sender MAC address and frequency).

MAC Control

Operation Code

Connection

List of operation codes (name and frequency; if the operation code is 0x0001 ("Pause") then the last transmitted value of the Quanta field is also displayed).

SOME/IP

ECU Resets

Endpoint

Number of detected ECU resets.

Only for ServiceDiscovery endpoints that are not part of the Multicast network node.

Offers

Endpoint

List of provided services (name, SD status, number of consumers).

Only for ServiceDiscovery endpoints that are not part of the Multicast network node.

Subscriptions

Endpoint

List of subscribed services (name, status, list of subscribed event groups (name and status)).

Only for ServiceDiscovery endpoints that are not part of the Multicast network node.

Role

Endpoint

Role of the service endpoint:

  • Consumer: consumes the service
  • Provider: provides the service

For service endpoints only.

Service

Endpoint, Connection

Name of the service.

For service endpoints and service connections only.

Instance ID

Endpoint, Connection

Instance ID of the service.

For service endpoints and service connections only.

Version

Endpoint, Connection

Version of the service interface.

For service endpoints and service connections only.

Endpoint

Number of received notifications.

For consumer endpoints only.

Methods

Endpoint, Connection

List of method calls (name and frequency).

For service endpoints and service connections only.

Events

List of events (name and frequency).

For service endpoints and service connections only.

Fields

Endpoint, Connection

List of field accesses (name, frequencies of get, set and notify).

Unknown Method IDs

Endpoint, Connection

List of calls with unknown method IDs (method ID and frequency).

For service endpoints and service connections only.
