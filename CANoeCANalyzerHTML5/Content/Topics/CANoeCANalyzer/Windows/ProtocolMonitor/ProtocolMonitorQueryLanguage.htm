Protocol Monitor Query Language

 

With the Protocol Monitor Query Language you can describe conditions for the selection of objects. You can formulate conditions for object properties (=property conditions), such as names, timestamps or network membership. In addition, you can combine conditions using logical operators (= combination conditions).

You can use the Protocol Monitor Query Language in text search and for creating Stop and Pass Filters.

As an aid, a list of valid input options is displayed when typing. If the entered search text is incorrect, the input field is highlighted in red and a tooltip provides further information about the input error.

A property condition always consists of a property, an operator and a value.

Properties

It depends on the context which properties you can use.

  • In the Stop and Pass Filters of the Connections View, only properties of connections are considered.
  • In the text search, all objects are considered (i.e. network nodes, networks, endpoints, connections and validation results).
    Regardless of the data type, the properties are all interpreted and searched as text.

Combination Conditions

You can combine conditions with logical operators to create combination conditions. You can combine both property conditions and combination conditions. In this way you can create complex conditions.

Operators

Operator

Description

AND

And combination: the combination condition is fulfilled if all combined conditions are fulfilled.

OR

Or combination: the combination condition is fulfilled if at least one of the combined conditions is fulfilled.

Example

The following expression is used to obtain all objects where the network node ADAS is involved as initiator or addressee:

"Initiator Node" = ADAS

OR

"Addressee Node" = ADAS
