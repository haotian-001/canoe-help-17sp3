09-0009

The specified check cannot be used in the current bus context. You might be able to eliminate this error by setting the bus context using the SetBusContext CAPL function before using the check.

09-0010

09-0011

09-0012

Execution of a test module / test unit was interrupted. The test is therefore incomplete.

09-0014

Execution of a test module / test unit was canceled because the measurement was stopped. The test is therefore incomplete.

To execute this test module / test unit in its entirety, please restart it.

09-0015

'MainTest' function is not allowed in CAPL libraries.

A CAPL library for the test module contains a MainTest routine. Such a routine is not allowed in CAPL libraries. Please delete the MainTest routine. CAPL libraries consist only of test cases and test functions.

09-0016

Test module / test unit is already running. It will not be started again.

An attempt was made to restart the test module / test unit even though it is already being executed.

Please review the start condition configuration: Test Module Configuration Dialog / Configuring Test Units.

09-0017

<function> ignored. There is no test case running!

Test steps can only be documented in the test report while a test case is running.

Please make sure that a test case is running when the function is called.

09-0018

Endless loop in test sequence detected. Test sequence stopped to prevent system instability.

This situation can arise when the old constraint behavior (4) is used.

We recommend using the default behavior for constraints.

09-0019

Invalid context of a wait condition detected. Test sequence stopped to prevent system instability.

This situation indicates that there is a programming error in a user-defined DLL.

09-0020

Wait call ignored, because this procedure cannot be suspended.

Wait functions can only be used in test sequences; waiting is not possible in event handlers.

09-0021

FiberManager (RT): <number> suspended (not finished) test modules / test units deleted.

A test module / test unit that was still active when the measurement ended was aborted at the end of the measurement. This indicates that the test sequence is incomplete.

09-0022

A unique definition for the CAPL test case and/or CAPL test function could not be found. Please review your CAPL libraries to make sure that the test case and/or test function is defined only once.

Test cases and test functions that have the same name but different signatures â i.e. differing numbers and types of transmitted values â are allowed.

09-0024

Parameter type of test case <test case name> is invalid. Arrays only allowed for char parameters.

The test case contains a parameter that is not a char parameter, but which is nevertheless defined as an array (e.g. double xyz[]). This is not allowed.

Please review the definition of this test case and modify the parameter type as needed.

09-0025

This test case contains an int parameter. During execution, it was assigned a value that is too long for this integer type.

In the parameter definitions, please specify the long or int64 type instead of the int type.

09-0026

Parameter overflow in test case <test case name>. Use type int64 instead of long in the CAPL test case.

This test case contains a long parameter. During execution, it was assigned a value that is too long for the long type.

In the parameter definitions, please specify the int64 type instead of the long type.

09-0027

Ambiguous definitions of <function name> in CAPL files were found. Definition was taken from CAPL file <filename>, <test module filename> was ignored.

A unique definition for the CAPL test case and/or CAPL test function could not be found. Please review your CAPL libraries to make sure that the test case and/or test function is defined only once. Test cases and test functions that have the same name but different signatures â i.e. differing numbers and types of transmitted values â are allowed.

09-0028

Environment variable cannot be used to start the test module. Unsupported type.

The test module is being started with a data environment variable. Use of this data type to start a test module is not supported.

Please specify a different environment variable in the test module configuration dialog, or modify the environment variable type in the database.

09-0029

System variable cannot be used to start the test module / test unit. Unsupported type.

The test module / test unit is being started with an integer array or float array system variable. Use of these data types to start a test module / test unit is not supported.

Please specify a different system variable ( Test Module Configuration Dialog / Configuring Test Units), or modify the system variable type in the system variable configuration dialog.

09-0030

No test function <name> with matching signature could be found.

An XML test module contains an invalid CAPL test function call. The corresponding test function was not found in the configured CAPL libraries. Please review the CAPL test function calls in the XML test module. When doing so, please make sure that the number and types of transmitted values are correct.

09-0031

No test case <name> with matching signature could be found.

An XML test module contains an invalid CAPL test case call. The corresponding test case was not found in the configured CAPL libraries. Please review the CAPL test case calls in the XML test module. When doing so, please make sure that the number and types of transmitted values are correct.

09-0032

No value for <path> found in environment. Using default working directory for syscall.

The Windows environment variable for this working directory was not found. The path test environment or CANoe configuration path is therefore being used as the working directory.

09-0033

<path> can't be set as working directory. Using default working directory for syscall.

The specified path cannot be used as a working directory. The path test environment or CANoe configuration path is therefore being used as the working directory.

09-0034

The test execution was aborted due to a overflow of the report queue!

Too many report events or high load situations may lead to lost report events. Since this would make the test report invalid the test will be stopped immediately.

In the CAN.INI file you can adjust the following parameter to enlarge the report queue:

[SYSTEM]
MAXNRBLOCKS=130

The default setting is 100, therefore you should chose a value >100.

You can raise this value at will, but please keep in mind, that the memory consumption will rise as well; i.e. as long as there is enough free memory you may raise the value.

09-0035

Unapplied changes of test unit variant properties have been discarded.

The variant properties of a test unit have been changed. They haven't been committed before measurement start by pressing the [Apply changes] button.

Therefore the changes have been discarded at the start of measurement and the previous values have been restored.

09-0040

An error occurred during the expansion of field codes inside the report file path. The expanded report file path may contain invalid path specification or invalid path characters. See also the <detail> part for further error description. The temporary report file is still available at the same location as the CANoe configuration.

09-0045

.NET test module '...': No test group could be found.

This error occurs, when the execution of the test results in number of test groups/test cases that is different from the readout at compile time (display in the test execution dialog).

A possible cause can be that several test modules use .NET assemblies with identical names from different folders. In this case the .NET runtime can only load one assembly and executes this assembly for all test modules.

09-0047

Temporary screenshot folder could not be moved from <temp_location> to <final_location>

The screenshot folder used during reporting could not be relocated from the temporary location (<temp_location>) to the final report location (<final_location>).

Please check the location of the report file and the writing access to that path.

09-0048

An error occurred during the report post processing. See <log file>

During the XSLT report post processing an error occurred. See log file for further details.

09-0052

The report path could not be created <path>

The path for the report could not be created.

The lightly cause for this is that the final report path is read-only. Please verify the writing access for the report path.

09-0053

CANoe system environment does not match the vTESTstudio configuration used for building the test unit.

The used CANoe system environment differs from the vTESTstudio project configuration.

It might be, e.g. that a signal, system variable or parameter has been deleted or renamed in the CANoe configuration and the changes have not been reproduced in the vTESTstudio project.

Open the *.vtuexe files in the vTESTstudio project and import the CANoe system environment. Thereafter the test unit must be built. Finally restart the measurement in CANoe. The problem should be solved.

09-0054

A test design artifact of the test unit cannot be converted.

The test unit has been generated with an older version of vTESTstudio. This requires that some test design artifacts must be converted. This conversion failed. If possible try to generate the test unit with the appropriate vTESTstudio version.

09-0055

A test design artifact uses features of a newer CANoe version.

For each CANoe version a recommended corresponding vTESTstudio version is provided. The error message indicates that a test unit has been generated with a vTESTstudio version newer than recommended for the current CANoe version. A feature is used in vTESTstudio which is not supported by the used CANoe version. Remove the usage of the new feature from the vTESTstudio project and (re-)build the test unit.

09-0056

A test design artifact with an unknown version cannot be processed.

The test unit contains a test design artifact with an unknown version. This indicates an internal error.

Please contact Vector Support.

09-0058

The test unit cannot be compiled.

If a vTESTstudio editable parameter file is used it might be that it has been changed in a way that the test unit cannot be compiled. In example by changing the type of a parameter in an incompatible way or by adding an incompatible "parameter value" to a parameter of type "automatic".

Load the editable parameter file in the vTESTstudio project. It is a good starting point to create a new test unit and load the editable parameter file in this test unit. After validation a comprehensive error message should deliver information on the problem. In case no error occurs, take over the modification of the editable parameter file to the original one in the appropriate test unit.

If no parameter file is used or no changes has been done the problem is an internal error.
Please contact Vector Support.

09-0060

Test unit <name> was built with an older vTESTstudio version which is not supported with CANoe 15.

With CANoe 15 the minimum required vTESTstudio version is 2.2. The test unit was built with an older version.

It is recommended to rebuild the test unit with vTESTstudio 2.2 or newer.

09-0062

The test unit contains obsolete features.

The test unit contains features that are marked as obsolete and will no longer be supported in future versions.

Use a newer version of vTESTstudio to re-build the test unit to get more detailed information and to adapt the test design.

09-0064

The test unit contains obsolete features.

The test unit contains features that are marked as obsolete and will no longer be supported in future versions.

Use at least vTESTstudio version 5.0 to re-build the test unit to get more detailed information and to adapt the test design.

09-0067

Test Unit <name> was created with a .NET framework version that is not compatible with the version selected in CANoe.

The configured .NET framework version used to create this Test Unit in vTESTstudio is not compatible with the framework version in CANoe.

To run the test unit, you must change the .NET framework version in CANoe settings.
