FlexRay

 

[14] FlexRay

ID

Notification / Description

14-0002

FlexRay <Channel>: Unable to map PDUs for frame <Frame>, driver status = <State>

The driver cannot map the PDUs into the frame. Possible reason:

  • Wrong payload length
  • PDUs are overlapping
  • PDUs are placed outside the frame's payload
  • Update bits are placed inside of PDUs
  • Update bits are placed outside the frame's payload

14-0003

FlexRay <Channel>: To be able to send sync. frames, pKeySlotUsedForStartup and pKeySlotUsedForSync must be changed.
pKeySlotUsedForStartup is patched to <Value> and pKeySlotUsedForSync to <Value>.

Both parameters must be adapted in order to send key slots for start-up/sync.

14-0004

FlexRay <Channel>: pMaxDynamicPayloadLength and pLatestTx are recalculated to allow the sending of all configured dynamic frames.
pMaxDynamicPayloadLength = <Value>, pLatestTx = <Value>,
new pMaxDynTxBufPayloadLength = <Value>, new pLatestTx = <Value>

The parameter will be calculated according to the currently registered Tx frames of the dynamic segment. If the parameter shall not be calculated and should be used from the current cluster parameters, then the following entry in the CAN.INI file disables this automatic calculation:

[FlexRay]
Calculate_pLatestTx = 0

Note: If there is more than one node present in the database, then the parameters of a specific node can explicitly be selected, refer to protocol parameter.

14-0005

FlexRay <Channel>: xlFrPduConfiguration: Unable to register frame <Frame>, driver status = <State>, <Info>

The driver cannot map the PDUs of the frame. Possible reason:

  • Invalid configuration data for the frame

14-0006

FlexRay <Channel>: Unable to register PDU <Name>, driver status = <State>

The driver cannot handle the PDU. Possible reason:

  • Invalid configuration data for the PDU

14-0007

FlexRay <Channel>: pAllowHaltDueToClock is changed to 1 due to can.INI-settings.

The parameter is changed to 'true'. Therefore it is possible to detect the interface is not synchronizing with the cluster.

14-0008

FlexRay <Channel>: Cluster parameters can not be loaded, driver status = <State>

Invalid cluster parameters. At least one of the values is invalid..

14-0012

FlexRay: Unable to activate the channel(s), driver status = <State>

Interface error. Please disconnect and connect the interface or reboot the computer.

14-0013

FlexRay <Channel>: PDU transmit error, driver status = <State>. PDU count = <Num>. PDUs: <PDU list>

Possible reasons:

  • PDU or its Frames could not be registered for sending; see also 14-0002, 14-0032, 14-0040, 14-0041
  • The interface is currently not synchronous to the cluster.
  • PDU Transmit for a raw frame is invalid:
    If a frame that is normally used for PDUs is being defined as a raw frame, then a special flag was set when the frame was registered (see Tx buffer list, FRSetSendFrame). If the (raw) frame is used in a Frame Panel, this flag was set automatically. Now, it is impossible to update any of the frame's PDUs. Only (raw) frame updates are still allowed!
  • The Tx slot is already used by another application that uses the same interface in parallel!

14-0015

FlexRay: No interface card found!

Possible reasons:

  • Interface is not connected
  • Driver is not installed (correctly)
  • In the driver settings the application channel was not assigned (correctly)
  • For USB devices: disconnect and connect device again

14-0017

FlexRay <Channel>: A Sync. frame (<Frame>) can not be registered for sending if the measurement is active.

During running measurement a sync frame should be registered for sending. Sync frames must be registered before measurement starts.

14-0019

FlexRay <Channel>: Restarting CC.

The communication controller of the network interface has been reset per CAPL function, or the automatic restart after synchronization lost is enabled in the CANoe interface options and the halt state was detected.

14-0020

FlexRay <Channel>: Restarting CC, pWakeupChannel = <Value>, pWakeupPattern = <Value>, gdWakeupSymbolTxIdle = <Value>, gdWakeupSymbolTxLow = <Value>.

The communication controller of the network interface has been reset per CAPL function with changed wake-up parameters.

14-0021

FlexRay <Channel>: Channel can not be deactivated, driver status = <State>.

Interface error. Please disconnect and connect the interface or reboot the computer.

14-0022

FlexRay <Channel>: Channel interface can not be found.

See 14-0015.

14-0023

FlexRay <Channel>: Key slot <Slot ID> can not be configured twice.

The same key slot should be configured more than once. The measurement is started. The double configuration is ignored.

14-0024

FlexRay <Channel>: Frame <Frame> has a sync. flag, but all key slots are already defined.

More key slots should be configured than possible. The measurement will be started with the maximum number of possible key slots.

14-0025

FlexRay <Channel>: The frame <Frame> can not be registered because it shall be transmitted in a key slot, but the key slot is not set up for channel <Channel Mask>.

If the key slot is configured manually and a single channel mask is set (e.g. "A"), then on the other channel ("B") a normal frame cannot be sent.

Workaround: Change the channel mask to "AB" in key slot configuration.

14-0026

FlexRay <Channel>: Another application has connected to the channel.

This hint occurs when more than one application uses this interface channel.

14-0027

FlexRay <Channel>: Another application has deactivated the channel.

This hint occurs when more than one application uses this interface channel.

14-0029

FlexRay <Channel>: Key slot configuration error. Driver data: <Info>

Possible reasons:

  • Any of the configured key slots is not placed in the static segment.
  • Any of the configured key slots is already used by another application on the same interface in parallel.
  • See also 14-0023, 14-0024

14-0030

FlexRay <Channel>: Transmit error for frame <Frame>, invalid channel number.

A frame has to be sent on an invalid channel.

14-0031

FlexRay <Channel>: Transmit error for frame <Frame>, invalid channel mask.

A frame has to be sent with an invalid channel mask.

14-0032

FlexRay <Channel>: Transmit error for frame <Frame>, frame not registered for sending.

A frame has to be sent, but the frame cannot be registered. See also 14-0017.

14-0033

FlexRay <Channel>: Transmit error for frame <Frame>, frame has PDUs and cannot be transmitted as frame.

If a frame that is used for PDUs should be sent as a raw frame, then a special flag must be set when the frame is registered (see Tx buffer list, FRSetSendFrame). If the (raw) frame is used in a Frame Panel, this flag is set automatically.

14-0034

FlexRay <Channel>: Transmit error for frame <Frame>, error in auto-increment- or frame-data.

A frame with invalid configuration data has to be sent.

14-0035

FlexRay <Channel>: Transmit error for PDU <Name>, invalid channel number.

A PDU should be sent on an invalid channel.

14-0036

FlexRay <Channel>:Transmit error for frame <Frame>, invalid payload length <Num> bytes, but must be <Num> bytes.

The frame should be sent in the static segment, but the frame's payload length does not match.

14-0038

FlexRay <Channel>: Missing channel assignment. Please assign an application channel in the driver settings!

See 14-0015.

14-0039

FlexRay <Channel>: Channel interface not found!

See 14-0015.

14-0040

FlexRay <Channel>: The frame <Frame> conflicts with other frame(s).

The frame's timing (Cycle Multiplexing) collides with other frames.

14-0041

FlexRay <Channel>: Invalid configuration for in-cycle-repetitioned frame <Frame>.

An In-Cycle-Repetition frame must have Base Cycle = 0 and Cycle Repetition = 1.

14-0100

FlexRay <channel>: Frame <frame> has invalid channel mask.

In gateway mode only channel mask A is supported.

14-0101

FlexRay: Can not configure channels <channel1> and <channel2> as gateway channels.

One or both channels have an invalid channel number.

14-0102

FlexRay <channel>: Frame <frame> cannot be configured (inconsistent with slot disturbance configuration).

Frames can only be sent in slots for which the hardware bypass has been deactivated.

14-0103

FlexRay <channel>: Bypass for frame <frame> can not be configured.

The hardware bypass can only be configured for frames from the static segment and for channels of a FlexRay gateway.

14-0104

FlexRay <channel>: Error in key slot configuration. The key slots for gateway channels must be identical.

The key slot configuration for both gateway channels must be the same.

14-0105

FlexRay: The channels <channel1> and <channel2> can not be used as gateway channels. They are not mapped to the same interface.

In the driver configuration dialog both gateway channels must be assigned to the same interface channel.

14-0106

FlexRay <channel>: The gateway bypass can not be activated in prestart.

The bypass is already activated.

14-0107

FlexRay <channel>: The gateway bypass for this channel can not be activated.

The channel number is invalid.

14-0108

FlexRay <channel>: The gateway bypass for the dynamic segment can only be activated in prestart.

The bypass for the dynamic segment can only be deactivated at the start of measurement.

14-0109

FlexRay <channel>: The gateway bypass for the frame <frame> in the dynamic segment can not be configured.

The hardware bypass can only be configured for frames from the static segment and for channels of a FlexRay gateway.

14-0110

FlexRay <channel>: Frame <frame> cannot be send, because the gateway bypass is enabled.

Frames can only be sent if the bypass is deactivated.

14-0111

FlexRay <channel>: Cannot configure key slot <slot> for the gateway channel.

An error occurred while configuring the key slot.
