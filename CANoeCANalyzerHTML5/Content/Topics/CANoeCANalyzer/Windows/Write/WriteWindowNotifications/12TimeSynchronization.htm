Time Synchronization

 

[12] Time Synchronization

ID

Notification / Description

12-0001

Time-Sync: Software time synchronization is required but not activated (or drivers need to be restarted).

When multiple network interfaces are used in one configuration, their clocks need to be synchronized because otherwise their time stamps will drift apart.

Some network interfaces only support software time synchronization. If you use such an interface in a configuration, you need to activate software time synchronization. You can do this in the Control Panel's Vector Hardware dialog under General Information|Settings. Once you've activated software time synchronization, you need to restart the network interface drivers. You can make sure this happens by re-booting the computer. However, you can also do this by removing the network interfaces from the computer and then reconnecting them.

12-0002

Time-Sync: Software or hardware time synchronization is required but neither is activated (or drivers need to be restarted).

When multiple network interfaces are used in the same configuration, their clocks need to be synchronized because otherwise their time stamps will drift apart.
Hardware time synchronization generates more precise time stamps than software time synchronization does. You can activate global hardware time synchronization (Hardware ribbon tab|Network Hardware|Hardware Sync).
Activation of software time synchronization is described in message 12-0001.

12-0003

Time-Sync: Software and hardware time synchronization are both activated.

Hardware time synchronization generates more precise time stamps than software time synchronization does. Because both synchronization mechanisms are activated, the time stamps are less precise than they would be if only hardware synchronization were activated. However, activating both mechanisms gives better precision than only activating software time synchronization.

The only time both mechanisms should be activated at the same time is when one of the network interfaces in the configuration only supports software time synchronization and at least two of the other interfaces also support hardware time synchronization.

12-0004

Time-Sync: Deactivating hardware time synchronization. (<= 1 HW participating in HW sync).

Only one (or none) of the network interfaces in the configuration supports hardware time synchronization. This means that hardware time synchronization is not needed, so it is being deactivated.

12-0005

Time-Sync: Deactivating software time synchronization. (HW sync active for all channels or <= 1 HW participating in SW sync).

All of the network interfaces in the configuration support hardware time synchronization, or the configuration uses only one network interface. This means that software time synchronization is not needed, so it is being deactivated.

12-0006

Hardware time synchronization is required but not activated.

Several network interfaces are used in the current configuration, at least one of them does not support software time synchronization (e.g. VN8900). For this reason the hardware time synchronization has to be activated.

With the current settings the time stamps are not synchronized and therefor not meaningful. The working mode with not synchronized channels is not recommended.

12-007

You are using several network interfaces that cannot be synchronized with each other.

If, for example, you use two network interfaces in the current configuration, one of which only supports hardware time synchronization and the other only supports software time synchronization, they cannot be synchronized with each other. Therefore neither hardware nor software time synchronization can be activated and the time stamps of the channels involved will drift apart during the measurement.

Since the remaining bus simulation requires a common simulation time for all channels, a different channel assignment should be used.

12-0011

HW-Sync: Manual configuration faulty. Using automatic configuration:

The manual settings used for hardware time synchronization are incorrect. For this reason, automatically derived values are being used for the current measurement. Please check the hardware time synchronization settings (Hardware ribbon tab|Network Hardware|Hardware Sync).

12-0012

HW-Sync: Generation of sync edge on sync line failed: <channel>

The »sync source (i.e. the network interface that generates the sync edge) cannot generate a sync edge on the sync line.
This may have the following causes:

  • The network interface being used as the sync source does not support generation of sync edges.
    To avoid this error, please configure a different interface as the sync source.
  • There was an error accessing the driver.
    To avoid this error, please update the network interface drivers.

12-0013

HW-Sync: Channel reactivated: <channel>

Sync events were received from a network interface that has not issued any sync events for a longer period of time.
Events from this interface will be synchronized again from now on.

12-0014

HW-Sync: Master clock continuously timed out: <channel>

The network interface with the »master clock is not generating any sync events. This makes synchronization impossible.
Please make sure that all network interfaces participating in hardware time synchronization are connected to sync lines.

12-0015

HW-Sync: Channel continuously timed out: <channel>

One of the network interfaces participating in hardware time synchronization is not issuing any sync events. Synchronization of events from this interface is therefore impossible.
Please make sure that all network interfaces participating in hardware time synchronization are connected to sync lines.

12-0016

Synchronization error. Please check firmware version of network interface.

Timestamps cannot be generated. Please update the firmware version of the network interface.

12-0017

You are using several network interfaces that cannot be synchronized with each other.

If, for example, you use two network interfaces in the current configuration, one of which only supports hardware time synchronization and the other only supports software time synchronization, they cannot be synchronized with each other. Therefore neither hardware nor software time synchronization can be activated and the time stamps of the channels involved will drift apart during the measurement.

Since the remaining bus simulation requires a common simulation time for all channels, a different channel assignment should be used.

12-0021

MOST <channel>: Unexpected timestamp (<0): <original timestamp> at <CANoe/CANalyzer time>

The plausibility check failed because the original timestamp has a value of < 0. For this reason, the current CANoe time was used for this event.

12-0022

MOST <channel>: Unexpected timestamp (>2 min in advance): <original timestamp> at <CANoe/CANalyzer time>

The plausibility check failed because the value of the original timestamp is more than two minutes in the future compared to the current CANoe time.

For this reason, the current CANoe time was used for this event.

12-0023

MOST <channel>: Unexpected timestamp (>1 h break): <original timestamp> at <CANoe/CANalyzer time>

The plausibility check failed because the original timestamp and the most recent timestamp are more than an hour apart.

For this reason, the current CANoe time was used for this event.
