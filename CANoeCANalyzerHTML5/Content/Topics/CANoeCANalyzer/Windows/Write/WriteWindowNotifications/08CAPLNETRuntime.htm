CAPL/.NET Runtime

 

[08] CAPL/.NET Runtime

ID

Notification / Description

08-0001

Node <name>: Output to bus system <bus> channel <channel> not allowed!

The node is not permitted to send any frames on the requested channel.

Possible causes:

  • The node is not assigned to the appropriate network in the Simulation Setup.
  • The channel was incorrectly set in the code.

The simulation behavior can be specified via the CANoe options (compatibility).

08-0002

Node <name>: Output to bus system <bus> not allowed before START or after END!

You can send frames on the requested bus only during the measurement. In the on preStart CAPL event procedure, the sending of frames, for example, is not allowed.

08-0019

CAPL/.NET debugger is active for node <node name>.

The CAPL or .NET debugger is active for the node. Please consider that using the debugger can considerably slow down the simulation.

08-0020

Breakpoint during measurement start / stop ignored

For technical reasons, the execution of the CAPL or .NET program cannot be stopped during the measurement start and stop. Breakpoints that are reached, for example, from the on start CAPL event procedure are therefore ignored.

08-0021

Breakpoint was not called from MainTest and therefore halts the simulation

A breakpoint that should actually stop only the test and not the simulation was reached from an event procedure instead of from the test run itself (that is implemented by the MainTest function in CAPL). The simulation was therefore stopped.

08-0022

The string "<s>" contains special characters which don't fit to the CAPL program

You want to call a CAPL help function with a string parameter from a .NET or XML test module or test unit using special characters that cannot be displayed in the encoding of the CAPL program.
Save the CAPL program in a different encoding or do without the special characters.

This error message can also be displayed if you are using a VN8910 or VT6001 on which the appropriate code page has not yet been installed. In this case, contact Vector Support so that you can install the code page.

08-0023

The string "<s>" is too long in the encoding of the CAPL program

You want to call a CAPL help function with a string parameter from a .NET or XML test module or test unit, but the CAPL char array is not large enough for the entire string.
Either use a shorter string or enlarge the CAPL array.

08-0024

The out-parameter string "<s>" contains special characters which don't fit to the computer

You want to call a CAPL help function with a string output parameter from a .NET or XML test module or test unit using special characters that cannot be displayed in the encoding of the host.
Use another host or do not use special characters.

This error message can also be displayed if you are using a VN8910 or VT6001 on which the appropriate code page has not yet been installed. In this case, contact Vector Support so that you can install the code page.

08-0025

The out-parameter string "<s>" is too long in the encoding of the computer

You want to call a CAPL helper function with a string output parameter from a .NET or XML test module or test unit, but the output string is too long for the passed character array.
Either use a shorter string or enlarge the CAPL array.

08-0026

Breakpoint ignored because the program is already halted

A breakpoint in a program has been reached while the program has already been halted due to another breakpoint that has been reached.

This can happen in test modules and test units if the first breakpoint that has been reached halts the program, but not the simulation, while the second breakpoint has been accessed by an event procedure.

In this situation the second breakpoint will be ignored; the program in the debugger still halts at the first breakpoint.

08-0040

Access to the parallel port with the CAPL function <name> is not available any more.

See Parallel Port.

08-0041

Overflow using the timeNow() function.

The old timeNow function can only be used correctly for measurements up to approximately 12 hours.

For longer measurements you should use timeNowInt64 or timeNowNS instead.
