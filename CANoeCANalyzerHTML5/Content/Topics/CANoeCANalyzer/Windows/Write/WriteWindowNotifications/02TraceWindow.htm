Trace Window

 

[02] Trace Window

ID

Notification / Description

02-0001

Trace Window '<Trace Name>': Fading out of events with unchanged values, as well as fading out of unmodified data are not supported for additional events.

To implement the hiding of events without data change and the hiding of unchanged data, information must be stored for each event. When there are many different events, this cannot be absolutely guaranteed due to memory use.

02-0002

Trace Window '<Trace Name>': The maximum number of <number> events in fixed mode has been reached. Additional events cannot be displayed anymore. To reduce the number of different events, please place event or channel filters in the Measurement Setup before the Trace Window..

The maximum number of events/lines in the display buffer of the fixed mode is limited. In the case of many different events, this may cause that not all events can be displayed in the Trace Window. The filters in the Trace Window only act as display filters. The hidden events nevertheless occupy a space in the display buffer and can be made visible again by deactivating the display filters.

If many different events are sent and most of them are hidden by the trace filters, this message may appear although only a few events are visible in the Trace Window. This can be the case, for example, with system variables, since these are often hidden using the trace filters. If you start the measurement, then all system variables are reset to their initial values and thus occupy one place each in the display buffer of the fixed mode, although they are not displayed. There is not enough space for other events.

Solution

Insert an Event (e.g. CAN Filter) or Channel Filter before the respective Trace Window in the Measurement Setup. This filters the data stream before the Trace Window, so that unwanted events do not occupy any space in the display buffer.

02-0003

Trace Window '<Trace Name>': Not all new events can be displayed in fixed position mode.

In the case of short data history and events with high memory use, in particular, it is possible for no more RAM to be available for a new event in the data history. Instead of deleting one existing line in fixed mode, the new event is not saved.

You can normally prevent this situation with a larger data history.

02-0004

Trace Window '<Trace Name>': Not all new events can be displayed in chronological mode.

In the case of short data history and events with high memory use, in particular, it is possible for no more RAM to be available for a new event in the data history. Instead of deleting the existing data history in scrolling mode, the new event is not saved.

You can normally prevent this situation with a larger data history.
