VT System

 

[52] VT System

ID

Notification / Description

52-0036

Disturbance of VT System communication detected!
Trying to recover.

The cyclical transmission of measurement values from the VT System to CANoe is interrupted for a short time because a large number of errors has occurred. During the interruption no commands can be sent to the VT System (e.g. relay switching). If this is attempted, error 52-0044 is reported. An attempt will be made to resume data transmission after this interruption.

The cause of this problem is usually a longer blockage of the computer, which can occur e.g. when a USB device is connected or some other time-consuming I/O operation is carried out.
Other programs with long processing times can also cause such problems.

It is advisable to avoid such processes while running a CANoe measurement. If this message reappears constantly, it might be the case that the physical connection has been lost or the VT System switched off.

52-0037

Module left operational status!
Check the following messages for further information.
Stopping measurement…

A VT system module has exited its normal operational status, causing an immediate termination of the measurement.

Possible causes for this are e.g. overheating, excessive voltage, excessive current or the internal load of the VT1004 module being operated with negative voltages.

Once the measurement ends, an additional message usually appears providing further information.

52-0038

Datagram not accepted (WC == 0).
VT System configuration may be incorrect.
Restarting the measurement may solve the problem.

An error has occurred in the communication between the VT System and CANoe.

The most likely reason for the appearance of this message is that the VT Systemwas switched off during measurement. If this is the case, the measurement needs to be restarted once the VT System is switched on again. This is to ensure that communication with the VT System is functioning correctly again.

52-0044

Some Error occurred when trying to transfer settings to VT System!
Setting may be lost!

Communication problems between the VT System and CANoe may have caused some settings not to be transferred correctly.

To make sure that the VT System is in the correct state, you may need to transmit the relevant settings again.

52-0045

VT System: Error occurred when initializing! Configuration may be corrupt or a start value violates a constraint.
(Hint: Check for further messages in the Write Window.)

An error occurred during initialization of the VT System.

Possibly a start state for a VT System setting that was set in the VT System Control Window violates a VT System constraint. In this case further messages are output in the Write Window to identify the affected setting.

This may also be caused by a defective VT System configuration in CANoe. In this case it may be necessary to create a new configuration in the VT System configuration dialog.

52-0067

VT System: Error when trying to scan for VT System modules.
(Hint: This could be caused e.g. by plugging the Ethernet cable into the wrong port of a backplane)

In most cases this error is produced by an Ethernet cable that has been plugged into the wrong connection of a VT System backplane. Information on the backplanes and the installation descriptions contain notes regarding the correct connection:

  • Please be sure to refer to the notes and technical data in the VT System Manual (PDF)
    (https://www.vector.com/vtsystem-manual)    .
  • If you currently don't have a connection to the internet, please refer to the manual revision which was valid at the time of this release under VTSystem Manual (PDF).

Another possible cause is a faulty module. In this case a note is output that determines the (number of the) backplane that contains the faulty module. Notes on numbering the backplanes:

  • The 1st backplane is the one that is directly connected to the computer or VT6000.
  • The 2nd backplane is the one that is directly connected to the 1st backplane,…).

In this case you must remove the faulty module to be able to operate the VT System again.

52-0090

Use the VT System Firmware Updater to update the firmware.

See page VT System: Firmware Update for further information.

52-0112

The VT System configuration uses cycle times < 1ms that are meant to be used with Extended Real Time. Extended Real Time is not active or not supported: Affected values are probably not updated correctly!

At least one measured value of the VT System configuration is set to a transfer cycle time < 1ms. These transfer cycle times can only be used in a sensible way if Extended Real Time (ERT) is available. ERT is currently not available. ERT should be activated if possible or else a transfer cycle time >= 1ms should be configured for the affected values. General information about how to use ERT in combination with the VT System can be found on the page VT System and Extended Real Time.

52-0113

The VT System configuration uses cycle times < 1ms that are meant to be used with Extended Real Time. It could not be established whether Extended Real Time is active: If Extended Real Time is not active, affected values are probably not updated correctly!

At least one measured value of the VT System configuration is set to a transfer cycle time < 1ms. These transfer cycle times can only be used in a sensible way if Extended Real Time (ERT) is available. Currently it is not clear whether ERT is available. If this is not the case, please see 52-0112.
