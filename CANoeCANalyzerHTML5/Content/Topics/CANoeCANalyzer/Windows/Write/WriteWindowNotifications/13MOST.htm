[13] MOST

ID

Notification / Description

13-0002

MOST: Driver queue overflow.

Tx requests were lost. To avoid this problem, please reduce the transmission rate.

13-0003

MOST: Invalid channel in API <function name>.

The only channels allowed are the ones specified in the configuration. Please check your CAPL programs.

13-0004

MOST: Cannot send message.

Transmission of this message/packet is not supported by the connected hardware or in the current operating mode.

13-0005

MOST <channel>: Allocation table not available.

Allocation table is currently not available.

13-0006

MOST <channel>: Streaming of synchronous channels is not supported by the connected network interface type.

Streaming of synchronous channels is only supported by VN2610 Vector MOST interfaces.

13-0007

MOST <channel>: The current driver version does not support streaming of synchronous channels.

The drivers currently installed do not support streaming of synchronous channels. Please update the drivers.

13-0008

MOST <channel>: Function <function name>: wrong number of parameters.

The function was called with an incorrect number of parameters. Please check your CAPL programs.

13-0009

MOST <channel>: Function <function name>: wrong return value type.

The wrong return value type was specified for this function. Please check your CAPL programs.

13-0010

MOST <channel>: Parameter <parameter number> of function <function name>: wrong type.

An incorrect value type was used for one of the parameters when the function was called. Please check your CAPL programs.

13-0011

MOST <channel>: Parameter <parameter number> of function <function name>: wrong dimension.

An incorrect dimension was used for one of the parameters when the function was called. Please check your CAPL programs.

13-0012

MOST <channel>: Running <interface name> in node and spy mode on the same channel will lead to unpredictable time jumps during the simulation.

To separate node and spy modes, please take the following steps:

  1. Add an additional MOST channel to the configuration.
  2. Reconfigure the existing channel (which is in node and spy mode) so that only one of these two modes is active (e.g. node mode).
  3. Configure the newly added channel to the other mode (e.g. spy mode) and assign this new channel to the same network interface as the existing channel.

13-0013

MOST <channel>: <interface name> is running in spy mode without time synchronization.

This causes the time stamp for events on this network interface to drift in relation to other event sources. To avoid this problem, please activate hardware or software time synchronization.

13-0014

MOST <channel>: Hardware time synchronization is not supported for <interface name> in node mode.

The network interface does not support hardware time synchronization in node mode. To prevent the time stamps from drifting in relation to other event sources, please activate software time synchronization.

13-0015

MOST <channel>: Address Handler can not resolve destination address of message <numerically displayed message> (<CANoe/CANalyzer time>, <NetState>)

The Address Handler cannot find the functional address of this message in the system. The message is sent being to target address 0xFFFF.

13-0016

MOST <channel>: Transmission in bypass mode not possible!

The connected network interface has closed the bypass and cannot transmit. This can happen if the interface is configured to run with the bypass closed (typically with an activated spy), or if the FOT at the interface input is in the NoLock state.

13-0017

MOST <channel>: Tx queue full (<CANoe/CANalyzer time>; Control channel)

The message cannot be sent because the Tx queue (CMS) is full. To avoid this problem, please reduce the transmission rate.

13-0018

MOST <channel>: AMS Tx service is busy.

The message cannot be sent because the Tx queue (AMS) is full. To avoid this problem, please reduce the transmission rate.

13-0019

MOST <channel>: Tx queue full (<CANoe/CANalyzer time>; Asynchronous channel)

The packet cannot be sent because the Tx queue (asynchronous channel) is full. To avoid this problem, please reduce the transmission rate.

13-0020

MOST <channel>: AMS Tx Service is disabled.

The message cannot be sent because the AMS service is disabled. Please activate the AMS service (Hardware ribbon tab|Network Hardware|MOST <Channel>) or transmit via CMS.

13-0021

MOST <channel>: Asynchronous channel is disabled!

The packet cannot be sent because the asynchronous channel is disabled. Please activate the asynchronous channel (Hardware ribbon tab|Network Hardware|MOST <Channel>|Setup).

13-0022

MOST <channel>: <number> Tx receipt(s) were lost.

To avoid this problem, please reduce the transmission rate.

13-0023

MOST <channel>: Unexpected Tx receipt.

The network interface issued a Tx receipt, but the corresponding Tx request does not exist.

13-0024

MOST <channel>: Tx receipt error.

General Tx receipt error.

13-0025

MOST <channel>: Wake up is not available because AbilityToWake is set to 'Off'.

Please activate AbilityToWake via the MostNBSetAbilityToWake(long wakeStatus) CAPL function.

13-0028

MOST <channel>: Cannot open connection.

General connection error.

13-0034

MOST <channel>: No network adapter available!

The computer does not have a network adapter.

13-0035

MOST <channel>: Failed to establish socket connection for network adapter <IP address> (<error id>)!

For more information, please see the Error code list.

13-0036

MOST <channel>: Failed to establish socket connection for spy mode (<connection>; <error id>)!

For more information, please see the Error code list.

13-0037

MOST <channel>: Failed to establish socket connection for node mode (<connection>; <error id>)!

For more information, please see the Error code list.

13-0038

MOST <channel>: Failed to establish socket connection (<error id|connection>)!

For more information, please see the Error code list.

13-0039

MOST <channel>: TelLen of CMS Tx request does not match message length (<TelLen> + <x> != <message length>). Message will not be sent.

When transmitting messages, please make sure the message length is TelLen + x.

13-0040

MOST <channel>: TelLen of CMS Tx request does not match message length (<TelLen> + <x> != <message length>). Adapting message length.

When transmitting messages, please make sure the message length is TelLen + x.

13-0041

MOST <channel>: TelLen of CMS send request does not match message length (<TelLen> + <x> != <message length>). Adapting TelLen.

When transmitting messages, please make sure the message length is TelLen + x.

13-0042

MOST <channel>: <CANoe/CANalyzer time>: Unexpected hardware sync timestamp: newtime - oldtime = <time difference>; <event type>.

The difference between the current time stamp and the most recent valid time stamp is not plausible.

13-0043

MOST <channel>: Failed to set SBC (<SBC value>). Only possible when interface hardware is in 'NetInterfacePowerOff' state.

The MOST system needs to be shut down before the SBC value can be set. Once the MOST system is re-booted, it will use the new SBC value.

13-0044

MOST <channel>: Failed to set frequency value (<frequency value>). Only possible when interface hardware is in 'NetInterfacePowerOff' state.

The MOST system needs to be shut down before the frequency value can be set. Once the MOST system is re-booted, it will use the new frequency.

13-0045

MOST <channel>: Ethernet connection to interface hardware currently overloaded.

To avoid this problem, please reduce the transmission rate.

13-0046

MOST <channel>: Send queue error: Cannot find corresponding Tx request for a Tx receipt.

The network interface issued a Tx receipt, but the corresponding Tx request does not exist.
