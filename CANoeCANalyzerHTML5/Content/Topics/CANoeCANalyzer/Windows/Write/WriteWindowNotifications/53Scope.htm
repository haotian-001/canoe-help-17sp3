[53] Scope

ID

Notification / Description

53-0001

USB communication error, please reconnect the Scope hardware!

This message occurs when already established USB communication with Scope hardware fails. In this case the Scope hardware must be reconnected.

53-0002

Hardware time synchronization is active. To use the Scope option the hardware time synchronization has to be deactivated!

To make Scope captures the hardware time synchronization has to be deactivated since the sync line is used for Scope triggering. via Hardware ribbon tab|Network Hardware|Hardware Sync.

53-0003

New Scope option version is available.
Unfortunately old settings for Scope Window have to be discarded due to technical reasons!

This message occurs when the loaded configuration contains settings for old Scope Window. Unfortunately these settings are discarded due to technical reasons and as the result the Scope Window has to be configured again.

53-0004

Cannot establish USB communication with Scope hardware.

This message occurs when USB communication with the Scope hardware cannot be established. In this case the Scope hardware must be reconnected.

53-0005

Cannot interrupt USB communication with Scope hardware.

This message occurs when established USB communication with the Scope hardware cannot be terminated. In this case the Scope hardware must be reconnected.

53-0006

Scope capturing only possible on real bus!

This message occurs on an attempt to make capture with Scope hardware when the application runs not on a real bus. To capture within simulation mode configure the current scope device as "VirtualScope".

53-0007

Registration or license for Scope option is missing!

This message occurs on an attempt to make capture with Scope hardware when either software registration or license for Scope option is missing.

53-0008

Maximum buffer size for Scope option data reached. Please delete previously captured data!

This message occurs when the maximum buffer size for captured data reached or there is no RAM memory available. Previously captured data has to be deleted.

53-0009

Maximum buffer size for Scope option data reached. Please delete previously captured data or increase the buffer size!

This message occurs when the user-defined buffer size for captured data reached or there is no RAM memory available. Buffer size for the captured data has to be increased via Scope Configuration|General in Scope Window. Alternatively, previously captured data can be deleted.

53-0010

Scope hardware did not trigger (check Scope trigger cable connection)!

This message occurs when sync pulse on the sync line has been generated but the Scope hardware did not trigger. Trigger cable connection has to be checked. Elsewhere the USB connection with the Scope hardware has to be re-established via toolbar button Connect Scope hardware in Scope Window.

53-0011

Installed Scope hardware driver version is older than required. Please install driver version <version> or newer!

This message occurs when the currently installed PicoScope driver is older than required. The newest driver version can be found under https://www.picotech.com/.

53-0012

Clock drift compensation of Scope hardware failed. <description>

This message occurs when clock drift of the Scope hardware cannot be compensated. As a consequence the signals decode fails and there is no logical interpretation available. Check that used probe corresponds to the settings in the Scope Hardware Configuration dialog in Scope Window.

CAN-specific: Please ensure that the calculation of Start Of Frame time stamps in Options dialog (Bus Systems/Protocols|Bus Systems) is active.

53-0013

No scope hardware found for <scope name>!

This message occurs when driver for Scope hardware is not installed or Scope hardware is not connected. Ensure that the driver is installed (it can be found under https://www.picotech.com/) and check that Scope hardware is connected.

53-0014

Scope hardware driver error <error description>

This message occurs when an error occurs during USB communication with Scope hardware. In this case the Scope hardware must be reconnected.

53-0015

Expected scope hardware not found for <scope name>(<scope hardware type>)!

This message occurs when the type of connected Scope hardware does no correspond to the hardware type configured in the Scope Hardware Configuration dialog in Scope Window. Modify the settings in the Scope Hardware Configuration dialog correspondingly or connect appropriate Scope hardware.

53-0016

Clock synchronization of Scope hardware failed (sync pulse event not detected)!

This message occurs when trigger sync pulse is not reported by the used network interface. By CAN this can be caused by full Tx queue. Check the connection of the sync cable.
