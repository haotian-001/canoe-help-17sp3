ID

Notification / Description

46-0001

IPBPacketLib: Loading IPBPacketLib.DLL failed

The file IPBPacketLib.DLL does not exist or it is the wrong file version.

46-0002

IPBPacketLib: Initialization failed (<error text>)

An error occurred during initialization of the IPBPacketLib.DLL.

46-0003

Loading signal protocol DLL (<filename>) failed

The signal protocol DLL must be configured in the hardware dialog. The path must be inserted relative to the configuration file.

46-0004

IPNetworkStack: Loading IPNetworkStack.DLL failed

The file IPNetworkStack.DLL does not exist or it is the wrong file version.

46-0005

IPNetworkStack: Initialization failed (<error text>)

An error occurred during initialization of the IPNetworkStack.DLL.

46-0006

IPNetworkStack: Warning: <warning text>.

Warning message of the TCP/IP network stack, measurement is still possible.

46-0007

SOME/IP Deserialization: Limit reached, PDU contains too many parameters

It was not possible to deserialize a SOME/IP PDU, because the cache of CANoe is not big enough.

The size of the cache can be modified in the can.ini with the entry [IP] DeserializationCacheEntrySize.

46-0008

UdpOpen was called with an IP address but without a valid port number.

In this case no implicit bind will be performed. Please give a valid port to perform implicit bind or bind explicit with IpBind().

Also refer to UdpOpen.

46-0009

TcpOpen was called with an IP address but without a valid port number.

In this case no implicit bind will be performed. Please give a valid port to perform implicit bind or bind explicit with IpBind().

Also refer to TcpOpen.

46-0010

IP reassembly begins to reuse buffers.

When reassembling fragmented IP datagrams, the available buffers have run out. This is possible if either

  • many fragmented messages are transmitted at the same time or
  • individual fragments of several transmissions have been lost.

In this case the IP reassembly starts to reuse the oldest buffers.

The number of available buffers can be adjusted in the can.ini file with the key IpFragmentationBufferLimit in the section [Ethernet].

46-0020

Car2x Security: Configured certificates not matching the active standard are ignored.

The Certificate Manager contains certificates for protocol versions that do not match with the version of the used Security Layer.

46-0119

46-0123

46-0125

46-0135

46-0136

Function can only be used in bus context of type Ethernet

46-0144

Car2x Security: Security Header not initialized.

46-0145

46-0146

Car2x Security: Protocol type|version mismatch.

46-0147

Car2x Security: Certificate "<Name>|<HashedId8>" not allowed to sign messages.

The certificate type is not suited to sign messages.
