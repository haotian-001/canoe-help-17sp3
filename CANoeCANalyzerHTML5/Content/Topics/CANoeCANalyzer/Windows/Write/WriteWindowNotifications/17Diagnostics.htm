[17] Diagnostics

ID

Notification / Description

17-0001

Construction of diagnostic network <name> failed - unknown details

The communication network for diagnostics could not be created on the given bus, because a fatal error occurred. Please check the settings of the diagnostic descriptions.

17-0002

Construction of diagnostic network <name> failed: <error message>

The communication network for diagnostics could not be created on the given bus, because an error occurred.

Possible error messages:

  • Builder could not create transmitter node (type:IsoTp)
    The creation of an ISO transport protocol node failed. It is possible that an old version of the OSEK_TP.DLL was loaded into the system. Please check the DLL version via the File ribbon tab. Ensure that no or only the current OSEK_TP.DLL from the CANoe directory is loaded as modeling library (DLL file) in you configuration! Possibly update the OSEK_TP.DLL used in your configuration.
  • Further messages:
    It is possible that there are inconsistent parameter settings. Please check the settings of the diagnostic descriptions.
  • Construction of diagnostics network on //DoIP/.13400 failed: Resource could not be built
    The initialization of the DoIP.DLL failed. Please check the DLL version via the File ribbon tab. Ensure that no or only the current DoIP.DLL from the CANoe directory is loaded as modeling library (DLL file) in you configuration! Update the DoIP.DLL used in your configuration, if necessary.

17-0003

K-Line access is not possible in simulation mode! Diagnostic descriptions on <name> will be ignored.

CANoe does not support simulation of K-Line bus systems. The corresponding diagnostic descriptions are therefore hidden during the measurement. Switch to real bus mode in order to activate these descriptions.

17-0004

COM port <name> is configured for diagnostics on K-Line, but this port is not available!

This message indicates that at least one diagnostic description is configured for diagnostics on K-Line on the given COM port (e.g. "COM1"). On measurement start it was detected that this port is not available.

It is possible that the configuration was created on a different computer on which the assignments of COM ports is different. Or a driver for serial interfaces was uninstalled or deactivated.

It is recommended to use the Windows system control to check the configuration of serial interfaces on the executing computer (consider distributed mode!). Alternatively you can select an available port in the configuration dialog of the diagnostic description.

17-0005

Cannot load diagnostic description: <reason>

The indicated problem occurred while attempting to load the diagnostic description.

17-0006

Diagnostic description <name> is configured to use baudrate <rate>!

The baudrate indicated in the diagnostic description is outside the range supported by CANoe.

17-0007

<Port> reports missing support for baudrate <rate> (diagnostic description <name>)!

The baudrate indicated in the diagnostic description is not supported by the utilized port. Use a different port, e.g., an integrated port or a K-Line network interface of Vector Informatik GmbH.

17-0008

Diagnostic description <name> at bus <name> is not defining valid FlexRay Tx Frames/PDUs for the Tester!

Please configure appropriate frames/PDUs in the Settings for the FlexRay Transport Protocol.

17-0009

Some FlexRay TP parameters for the diagnostic network <name> at bus <name> are not valid

Please configure appropriate properties and frames/PDUs in the Settings for the FlexRay Transport Protocol.

Especially if multiple tester frames/PDUs are selected, then the repetition factor of the frames' cycle multiplexing must use the same factor for all frames. Otherwise, the TP layer cannot guarantee the correct sending order of the frames.

17-0010

Some functional FlexRay TP parameters for the diagnostic network <name> at bus <name>" are not valid

Please configure appropriate frames/PDUs in the Diagnostic Description Settings: FrTP Parameter.

Especially if multiple tester frames/PDUs are selected, then the repetition factor of the frames' cycle multiplexing must use the same factor for all frames. Otherwise, the TP layer cannot guarantee the correct sending order of the frames.

17-0011

Some FlexRay nodes have several diagnostic descriptions assigned on the bus <name>

If a node has several diagnostic descriptions assigned on the same bus, this may lead to problems when simulation this node, since requests are interpreted by only one description during a simulation. Currently it is not predictable which one of the descriptions this will be.

17-0012

Some diagnostic descriptions use the same FlexRay TP parameters and therefore their requests/responses cannot be distinguished on the bus <name>

To assign physical requests and responses on the bus clear without ambiguity to a diagnostic description for interpretation, the used ECU addresses of the TP layer have to be different.
Please configure appropriate ECU addresses in the Settings for the FlexRay Transport Protocol: FrTp Parameter.

17-0013

The direct sending of byte streams is not supported by the CAPL Callback Interface for diagnostics.

The DiagSendRequestPDU CAPL function is not supported by the currently used port.

17-0016

The following network adapter is not available for DoIP: '<adapter>' (configured for: <diagnostic descriptions>). A different adapter will be selected automatically.

In the diagnostics configuration for DoIP descriptions a network adapter with given name was configured that is no longer available. For example a USB network adapter might have been removed.

Please open the Diagnostics/ISO TPâ¦ configuration dialog and assign a different adapter to the descriptions if the automatic selection of an adapter does not work as expected.

17-0017

Eth<n> is set to "Phy Bypass", i.e. no sending is possible on this channel! (Configured for: <diagnostic description>)

Since the global setting for the given Ethernet channel is physical bypass, no sending is possible because all passing Ethernet packets are just recorded for analysis.

It is possible to set the channel to MAC Bypass to allow sending; though this results in a slightly slower speed for analysis.

17-0018

No DoIP communication is possible on network adapter '<name>' because the application's TCP/IP stack has been activated globally! (Configured for: <diagnostic descriptions>)

In the global options for TCP/IP, the internal stack has been activated. Therefore network adapters managed by the operating system and 'localhost' are not available for DoIP. In order to use these adapters, activate the TCP/IP stack of the operating system.

17-0019

One or more diagnostics devices have been assigned to <channel>. Please reassign them, if necessary.

When a CANoe configuration is loaded in CANalyzer, diagnostic descriptions assigned to a CANoe simulation bus must be moved to a channel in CANalyzer. This warning is issued when more than 1 suitable channel is available.

17-0020

Since Eth channels are configured, diagnostics interpretation of DoIP messages is only done on Eth channels.

The DoIP diagnostics interpretation can only be performed either for the DoIP Diagnostics Access or for Eth Networks.

If you wish diagnostics interpretation of DoIP Diagnostic Messages, you should add the corresponding diagnostic description in the configuration dialog Diagnostics/ISO TPâ¦ at the used Eth network.

17-0021

Diagnostic object too small or not enough iterations. Use DiagResize or increase number of iterations.

The parameter could not be set, because the size of the diagnostic object is not as required.

Please initialize the object correctly prior to that with DiagResize() or increase the number of iterations with DiagSetParameter().

17-0022

Adding diagnostics channel to network <name> failed with error <text>. Please check diagnostics configuration.

There was an error during the initialization of the diagnostics channels for the given network. For example, it is tried to use an identifier for several channels, which is not possible.

17-0023

Unified language settings for description(s) <diagnostic descriptions> to <language>

Since the same ECU document can only be held in one language, all descriptions using this document must be set to the same one. This message indicates the descriptions for which the settings were changed, and the value set.

17-0024

Diagnostics channel for description <Qualifier> is closing, therefore access not possible.

A connection oriented diagnostics communication channel has been closed, and this operation has not finished yet. Therefore the attempted action on this closing channel, e.g. the sending of another request, is not possible yet.

The channel has to be reopened after the closing has completed.

17-0025

ECU simulation was activated on channel K-Line <n>, therefore the diagnostic windows for <ECU qualifier list> will not work.

On a K-Line channel, tester and ECU simulation cannot operate at the same time.

On the given channel an ECU simulation was activated, therefore the diagnostic windows (Diagnostic Console, Fault Memory Window and Diagnostic Session Control) that attempt to send on this channel will not work.

17-0026

Activation Line was (un)set via system variable <variable>.

At the start and stop of measurement, the system variable that is configured to control the Activation Line (DoIP) is set to 1 and 0 respectively.

17-0027

Physical network request description %s is no longer supported.

Please contact Vector support for details.

17-0030

The communication parameters of the selected diagnostic interface (CAN FD) for <ECU qualifier> are not compatible with the assigned CAN hardware channel (classic CAN).

A simulation node is not able to communicate due to inconsistent settings in the Diagnostics/ISO TP Configuration dialog and in the Network Hardware Configuration dialog. Please check whether CAN FD mode should be used instead of CAN or a different diagnostic interface should be selected for the diagnostic description with the mentioned ECU qualifier.
