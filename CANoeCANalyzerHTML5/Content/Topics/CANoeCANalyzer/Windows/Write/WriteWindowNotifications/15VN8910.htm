VN8900 Network Interface

 

[15] VN8900 Network Interface

ID

Notification / Description

15-0001 / 15-0002

The connection to the configured RT server could not be established…".

Under Home ribbon tab|Group Measurement|Vector Tool Platform|CANoe/CANalyzer|Distributed Mode it was configured that CANoe should run in distributed mode with a certain RT server (e.g. VN8900).

The attempt to establish a connection to the simulation environment on this device failed. For possible reasons see Troubleshooting.

15-0010

File synchronization: <filename> – unknown application file class!

Transfer of this file type to the device is not supported.

15-0011

File synchronization: <filename> – file not found on user computer!

The file was not found in the user computer file system (in the case of user files, see search rules under CAPL Functions: File Search Procedure).

15-0012

File synchronization: <filename> – file not found on remote device!

The file was not found in the device's file system after it was transferred to the remote device.

15-0013

File synchronization: <filename> – could not read file on user computer!

Please check whether read permissions are set correctly for this file on the user computer.

15-0014

File synchronization: <filename> – could not read file on remote device!

The file could not be read in the device's file system prior to its transfer back to the user computer. If this file was accessed by CAPL during measurement, make sure that it was closed properly afterward.

15-0015

File synchronization: <filename> – could not save file on user computer!

The (user) file could not be saved to the file system on the user computer. Check whether the disk space and write permissions on the user computer are sufficient.

15-0016

File synchronization: <filename> – could not save file on remote device!

The file could not be saved to the file system on the remote device after it was transferred there.

15-0017

File synchronization: <filename> – could not commit file on remote device!

It was not possible to integrate ("commit") the file into the file system on the remote device after it was transferred there.

15-0018

File synchronization: <filename> – could not transmit file to remote device!

A connection error occurred while the file was being transferred to the remote device.

15-0019

File synchronization: <filename> – could not create synchronization folder on remote device!

The folder in which transferred files are to be saved on the remote device could not be created.

15-0020

File synchronization: <filename> – maximum file size exceeded!

File sizes must not exceed 63 MB when transferring files for use in distributed environments.

15-0030

Filename containing a path is not allowed in a distributed environment: <filename>

The use of path names that include drive names and folders cannot be used in connection with CAPL file access in a distributed simulation environment. The path setting is specified in User Files (Options dialog).

15-0031

File access failed since filename was not predefined: <filename>

In a distributed simulation environment (e.g. with VN8900 as RT server), all files that are to be accessed via CAPL have to be specified in User Files (Options dialog).

15-0032

Function <CAPL function name> is not available in a distributed environment.

Some file functions (particularly functions for querying or modifying file paths) are not supported in a distributed function. The path names are derived from the specifications made in User Files (Options dialog).

15-0060

Offline mode is not supported in standalone mode. Standalone configuration will be switched to online mode.

See configuration requirements for execution in standalone mode. The automatic switch to online mode only affects the downloaded/exported standalone configuration.

15-0061

Simulated bus mode is not supported in standalone mode. Standalone configuration will be switched to real bus mode.

See configuration requirements for execution in standalone mode. The automatic switch to real bus mode only affects the downloaded/exported standalone configuration.

15-0062

The configuration uses features that are ignored in standalone mode.

See configuration requirements for execution in standalone mode. The automatic deactivation of features only affects the downloaded/exported standalone configuration.

15-0068

The variant of the RT kernel used on the RT server (<RT Kernel Architecture RT server (32-bit/64-bit)>) differs from the variant configured in the CANoe configuration (<RT Kernel Architecture CANoe Configuration (32-bit/64-bit)>).

The 64-bit variant of the RT kernel can only be used on the user computer and on some devices (e.g. on a VN8914).

Please read the notes on 64-bit/32-bit.
