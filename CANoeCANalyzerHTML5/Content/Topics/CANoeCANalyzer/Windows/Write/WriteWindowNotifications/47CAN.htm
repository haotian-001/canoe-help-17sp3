[47] CAN

ID

Notification / Description

47-0001

CAN Statistics: The hardware filter on channel <channel number> affects the statistic values!

This message occurs when hardware filter is configured for the concerned CAN channel. In this case the statistics for CAN may be incorrect.

Configuration of the hardware filter can be modified via Hardware Configuration dialog.

47-0003

Measurement could not start, because channel <channel number> doesn't support CAN FD!

The measure failed to start because at least one of the CAN channels does not support CAN FD. But the channel is configured for CAN FD.

Possible causes:

  • The driver is outdated. Please install the latest driver.
  • The network interface does not support CAN FD.

47-0004

CAN FD Flags of loaded configuration reset.

Suppose that in a configuration CAN FD flags are set. In the current version of CANoe CAN FD is not activated. If the configuration is loaded with this version of CANoe, CAN FD settings are set to default values. The CAN FD values of the configuration are reset.

47-0024

The filter for the NACK error frames has been enabled!

For the CAN channel the NACK error frame filter was activated.

The filter is activated when the transmission error counter ≥ 128 and the transmitter is in error passive state.

The filter can be enabled/disabled in Options dialog (Measurement|Performance|CAN Error Frames). The linked help page provides further information.

47-0025

The filter for the NACK error frames has been disabled!

For the CAN channel the NACK error frame filter was deactivated.

The filter is deactivated when the transmission error counter < 128 and the transmitter is in error active state.

The filter can be enabled/disabled in Options dialog (Measurement|Performance|CAN Error Frames). The linked help page provides further information.

47-0035

CAN: CANoe is configured for ISO driver. A non CAN FD ISO driver is installed for at least one channel.

This misconfiguration may lead to a canceled measurement start or to wrong frame lengths used by the CAN bus simulator.

If you want to use the NON-ISO CAN FD driver:

  • Up to and including CANoe 9.0 SP4:
  • Please change the INI switch in CAN.ini:

    [CAN]
    ENABLE_CANFD_ISO = 0

  • Starting with CANoe 9.0 SP5:
  • Configure the channel for ISO CAN FD in the Network Hardware Configuration dialog.
  • Otherwise you can install the ISO CAN FD driver.
    The driver is available in the Vector Download Center; it is part of the "Vector Driver Setup".

47-0036

CAN: CANoe is configured for non ISO driver. A CAN FD ISO driver is installed for at least one channel.

This misconfiguration may lead to a canceled measurement start or to wrong frame lengths used by the CAN bus simulator.

If you want to use the ISO CAN FD driver:

  • Up to and including CANoe 9.0 SP4:
  • Please change the INI switch in CAN.ini:

    [CAN]

    ENABLE_CANFD_ISO = 1

  • Starting with CANoe 9.0 SP5:
  • Configure the channel for NON-ISO CAN FD in the Network Hardware Configuration dialog.
  • Otherwise you can install the NON-ISO CAN FD driver.
    The driver is available in the Vector Download Center; it is part of the "Vector Driver Setup".

47-0038

Driver error: Events from invalid channel received.

Please install the latest driver or change the settings in Options dialog (Measurement|Time Source & Event Sorting|Simulation Time Source) from All appropriate channels (legacy behavior) to Auto: Computer-based timer.

47-0039

CAN: The network interface or driver doesn't support the Bus Off feature.

The Bus Off feature is only available for certain network interfaces and drivers. The availability can be checked in the feature list of the network interface on the driver CD.

47-0041

CAN: Loading a CANoe configuration (from an older CANoe version) with an assigned AUTOSAR 4.x database

CANoe 9.0 (and newer) fully supports the PDU layer of AUTOSAR 4.x databases (for CAN and Ethernet). Thus, in CAPL the new PDU object is available for transmission and reception.

Since CANoe 8.5 can also load AUTOSAR 4.x databases, but does not support the PDU layer (for CAN and Ethernet), it is not possible to use a CANoe 8.5 configuration in CANoe 9.0 (or a later version) without manually modifying the CANoe configuration (especially the CAPL programs).

Manual modifications steps needed:

  • Exchange the IL by an IL that supports the AUTOSAR PDU concept.
  • Modify your CAPL programs by using PDU centric methods like the PDU object, the on PDU handler and the TriggerPDU function.
