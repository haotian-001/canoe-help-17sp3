44-0032

Ethernet Driver Warning: Error getting TCP offload parameters: <warning text>.

At start of the measurement CANoe queries the Ethernet adapterâs IP offload capabilities in order to determine if the IP offload setting is configured. Offloading in this respect means that certain routines specific to the processing of the IP protocol are carried out by the Ethernet adapter hardware itself. However, if enabled affected protocol fields like IP checksums are no longer visible to the operating system or applications. To bypass this behavior CANoe disables the offload setting so that the TCP/IP stack is fully responsible for providing the checksums. This ensures that Ethernet packet data captured, processed and shown in CANoe is in accordance with what was physically transmitted on the network.

If CANoe fails to retrieve or set the offload configuration from the hardware the respective warning is shown. Although providing this information is mandatory for a Windows Ethernet adapter driver not all vendors fulfill that requirement. This is specific to the adapter vendor and cannot be influenced by Vector.

As a result Ethernet packets captured or sent by CANoe can appear to have wrong checksums.

Update the network adapter driver with a current version from the vendorâs web site or use another adapter to avoid this problem.

44-0101

Ethernet Driver: Reception overrun â packets are lost!

Not all received data could be processed in CANoe, some packets are lost. Since several network types share receive buffers, the loss may possibly affect another network type, for which a notification is then also output in the Write Window. Please reduce the number of received packets or make more computing power available on your computer.

Notes to reduce data:

  • Create hardware filters to process only relevant packets CANoe.
  • Adjust reception latency (Hardware|Network Hardware|Driver|VN56xx|Receive Latency) to your needs in the Vector Hardware dialog (Windows start menu path: Control Panel|Vector Hardware).
