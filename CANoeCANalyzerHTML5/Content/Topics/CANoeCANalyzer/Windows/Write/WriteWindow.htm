Write Window

Ribbon|Home Tab|Write

General

Output in the Write Window includes important system messages that occur over the course of the measurement (e.g. triggering of Logging function).
Also all messages send with the write() CAPL function are displayed here.

Messages of the Write Window serve to provide supplemental notes on your measurements as well as - in the event of problems - a basis upon which troubleshooting can be performed by our support.

Note

In a configuration a Write Window is included automatically. The window can not be deleted but inserted on every desktop.

Note

  • Max. 250 characters can be displayed in one line in the Write Window.
    If the line is longer, the line will be truncated. However, logging files record the complete line.
  • CAPL functions can output a maximum of 1024 characters per line to the Write window; longer texts must be split over several lines.
    If the line is longer then 4096 characters, a warning message will be displayed in the Write Window.

Bus-/Protocol-specific Settings

MOST

  • After measurement start the Write Window contains valuable information on which mode is used for operating the MOST channels.
  • If a measurement stops right after starting the measurement there might be a problem with the connection to the network interface or a problem with missing licenses. The Write Window will show detailed information.
  • Some erroneous actions by CAPL or user interaction can only be detected during a measurement but should not lead to a measurement stop. These may include CAPL function calls with illegal parameter values, sending on an unavailable channel or the attempt to send while the bypass is closed.
    In these cases information is printed to the Write Window indicating which actions were tried to execute.
