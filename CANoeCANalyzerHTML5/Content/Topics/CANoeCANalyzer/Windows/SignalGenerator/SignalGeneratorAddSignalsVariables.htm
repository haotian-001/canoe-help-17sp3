Signal Replay: Add Signals/Variables from Logging File

 

The Add Signals and Variables from Logging File dialog you open via the Signal Replay page, if you add a logging file or via the context menu command of an existing logging file.

In this dialog, you can search a file for signals and variables.

Only one channel from each logging file is used. If the file contains signals for several channels (e.g. CAN1 and CAN2), only the signals from the channel that is read first are evaluated.

Setting a target channel determines which channel is used to output the imported signals. This makes it possible to allocate the channel imported from the logging file to a channel in the current configuration. The database in which the signals are defined must be available to the target channel.

Use the [Start] button to search the file for signals and variables.
The signals and variables found are displayed in the list.

In the case of very large logging files, the search is paused after two or three seconds. The desired signals may already be listed in the results. If this is not the case, use the [Resume] button to continue the search. The progress bar indicates what percentage of the logging file has been searched.

Use the [Pause] button to stop the search when the desired signals are found.

Select the desired signals and variables from the list (you can make multiple selections) and confirm your selection with [OK].

The selected signals and variables are imported and appear as an expandable list below the corresponding logging file. After this you can deactivate and activate the Signal Generators in the Active column.
