MOST Audio Routing Window

Ribbon|Analysis Tab|Audio Routing

The MOST Audio Routing Window shows the distribution of connection labels in the resource allocation table of the connected node.

Analog – Line IN/OUT

  • LineOut/Headphone lets you listen to audio channels
    When you select a connection label, the routing engine of the connected node is programmed so that the relevant channels are routed as audio signals to the LineOut/Headphone output of the network interface. You can control the volume using the slider and the mute button. With MOST150 the bypass of the interface must be inactive to listen to a certain audio channel.
  • Line IN lets you feed audio data onto the ring
    When you select a connection label from the allocation table, the routing engine of the connected node is programmed so that the signals at the LineIn input of the network interface are routed to the associated channels on the ring. You can control the amplification of the input signal using the slider and the mute button.

Digital – S/PDIF IN/OUT

  • S/PDIF Out lets you transmit digital audio signals to the S/PDIF output.
    Once you select a connection label, the routing engine of the connected node is programmed so that the associated channels are routed to the S/PDIF Out output of the network interface. You can mute the signal using the mute button. (MOST150 only)
  • S/PDIF In enables you to feed in digital audio data from the S/PDIF In input of the network interface. Once you select a connection label from the allocation table, the routing engine of the connected node is programmed so that the signals at the S/PDIF In input of the network interface are routed to the associated channels. You can mute the signal using the mute button. (MOST150 only)
  • Channel Management
    The channel management allows manually allocating bandwidth of the synchronous data area on MOST. You can select up to 8 bytes per frame for a connection label on MOST25 and up to 152 bytes per frame on MOST150. With Deallocate Label you can free arbitrary connection labels on MOST25. On MOST150 de-allocation is restricted to labels previously allocated with Allocate Channels through the connected network interface.

Note

  • You can use Channel Management, for example, to induce stress situations on the ring by allocating almost all channels manually, fragmenting the allocation table or de-allocating channels prematurely.
  • In OptoLyzer, the AnalogOut 1 port is used as the Headphone output.
    The Analog In 1 port is used as the LineIn.
  • The data displayed and commands executed always relate to the network interface allocated to the channel that is selected on the page tabs.
  • With MOST150, you do not need to allocate channels explicity. As soon as the Line In or S/PDIF In are routed, a new label is created in the allocation table which you can then use to route the Line Out or S/PDIF Out.
