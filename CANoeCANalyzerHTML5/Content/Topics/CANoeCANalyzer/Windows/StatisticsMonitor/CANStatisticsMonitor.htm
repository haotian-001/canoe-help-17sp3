CAN Statistics Window

Ribbon|Analysis Tab|Statistics|View / New 

General

The CAN Statistics Window displays statistics about bus activities during measurement.

This Statistics Window can be inserted into the Measurement Setup via the context menus of the relevant function blocks (Trace etc.). To show the window use the Show Window context menu command. All CAN bus events that arrive at the input of the Measurement Setup are evaluated for this purpose.

If a database is assigned to a CAN channel, statistics can also be evaluated individually for each available CAN node and partly for available CAN messages.

CAN statistics can also be evaluated by using statistics system variables.

Various statistic CAPL functions are available.

In the Options dialog you can activate/deactivate the calculation of all bus statistics values.

Hints for busload calculation

Calculation by the Hardware

The total busload (row Busload) is calculated on the CAN hardware and considers all events on the CAN bus, including Error Frames.

Note

If there is no value available from the hardware (e.g. in offline mode, when no statistic events are logged), the calculation from the software is used for the total busload counter.

Calculation by the Software

The busload for the counter of a single network node is calculated by the software. In that case are considered only events that passed the Acceptance Filter.

Note

Because of technical reasons the Error Frames are not considered in the software calculation.

The difference between hardware calculated busload and software calculated busload you can see in the row Unknown events.

Example

If all messages with the ID 100 are blocked by the Acceptance Filter, these could not be considered by the software calculation. If these messages represent 5% of the total busload, these 5% are not listed in the row of the network node that has sent this messages but in the row Unknown events.

Chip State

Error status information on the individual CAN controllers and the momentary readings of their respective error counters. The following states may occur:

  • ACTIVE (normal state)
    The microcontroller reacts to bus errors by sending out an Error Frame consisting of 6 dominant bits.
  • WARNING
    The microcontroller has reached the warning stage (96 Error Frames).
    This is displayed in the Bus Statistics Window. The controller is still Error Active.
  • PASSIVE
    The controller has entered the Error Passive state.
    It reacts to bus errors with an Error Frame consisting of 6 recessive bits.
    Please refer to the CAN specification for further details on this behavior.
  • OFF
    The microcontroller has been disconnected from the bus.
    It is not reactivated until a restart has been executed

An available CAN channel can be selected from the CAN channel selection list. The selection list shows the current CAN channel, followed by the corresponding name of the CAN network in the Simulation Setup. Selecting the CAN All option in the channel list provides an overview of the Current/Last columns for all available CAN channels.

The Current/Last column shows the current or most recently derived statistical value. The Min, Max and Avg columns show the corresponding minimum, maximum and average statistical values. All channels can be reset when CAN All is selected; otherwise only the selected CAN channel is reset.

In the configuration dialog (context menu/right mouse button) you can adjust the windows refresh rate. In the Available Statistics area you control via the check boxes which statistics are shown and which not.

If a measurement is running, the currently displayed values can be paused by clicking the pause toolbar button or by the context menu command Pause Statistics.

Note

When using virtual CAN channels the following data is not evaluated:

  • The frame duration is not emulated. Therefore no busload data and no data for the minimum transmission distance are available. The displayed burst length may be wrong.
  • Controller State (Active, Error passive, Bus off, ...) is not emulated.
  • Chip State always shows Active after starting with virtual channels. Since the controller can always send and receive messages it is correct to display Active.
  • When Error Frames occur the display doesn't change; it still shows Active. In this case the indication is not absolutely correct.

Via the context menu or the toolbar you have further functions:

Configuration
Opens the configuration dialog.

Collapse All
Collapses all statistics.

Expand All
Expands all statistics.

Reset All Statistics
Resets all statistics.

Pause Statistics
Freezes all statistics during a meassurement.

Adapt Columns Width
Adapts the widths of all columns to the current values.

 

Sort
Sorts statistics Alphabetically or Logically.

 

Copy to Clipboard
Copies selected values to the clip board.
