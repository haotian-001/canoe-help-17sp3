Export Settings

Context Menu of the Graphics Window diagram: Import/Export|Export|Advanced Settings
Context Menu of the Trace Window/the logging file: Export...|Advanced Settings

In the Export Settings dialog you can configure various properties for export.

Format

Signal Format

Choose whether a signal name should be supplementally qualified and saved with the name of its associated message and database name.

Variable Format

Choose whether the variable name should be supplementally qualified and saved with the name of its database.

Delimiter

Specifies the separator used between the individual parts of a qualified signal or variable.
You can enter up to 3 characters.
Blank characters are considered.

Signal Values

  • Write raw values
    Specify here whether the signal raw value should be output when exporting. If this option is disabled, the physical values are output.
    This setting has no effect when exporting from the Graphics Window since in such case the Graphics Window setting is used.
  • Write symbolic values
  • Save J1939 signals with invalid values
    Here you determine whether invalid values for J1939 signals shall be exported.
    If deactivated, invalid values will be ignored and not exported. This option is only relevant for the export.
    The source file must be loaded again to consider the changed option correctly.

ASCII Settings

Number of Decimal Places

Decimal places for time stamps:

In this input box you can enter the number of decimal places with which time stamps should be written in the ASCII logging file

Value range: 1...9
Default value: 6

Note

  • Up to and including CANoe version 5.1 the time stamps were written with 4 decimal places. Starting with CANoe version 5.2 on the default is 6 decimal places.
    When changing the value please take into consideration how many decimal places other 3rd-party tools can handle.
  • These settings are configuration independent.
  • These settings are effective for the test report, too.

Precision of numeric values:

In this input box you can enter the number of decimal places in double values of system variables in ASCII logging files.

Value range: 1...14
Default value: 6

Number Format

Here you can choose, if the numbers will be logged in a decimal or hexadecimal format.

Time Format

Here you can choose, if the absolute or relative time stamps of the messages should be logged.

Representation

Here you can choose, if the value descriptions (symbolic values) of the value tables defined in the database should be logged or not.

Error Handling

Mark corrupted lines: If this option is activated, rows before or after which data loss occurred are marked with a *.

CSV Settings

Format

Here you can set options for saving in CSV format:

  • Column separator
    Specifies the separator used when saving.
    You can enter up to 3 characters.
    Blank characters are considered.
  • Decimal symbol
    Specifies the decimal symbol that is used when saving.
  • Decimal places for time
    Indicates the digits after the decimal point that are used when saving.
  • Decimal places for signal values
    Indicates the digits after the decimal point that are used when saving.
  • Export global header
    With this option you can select, if a global header should also be saved in the file or not.
    The header contains the application name, the configuration name, the date, and the time.
  • Quote all text cells
    With this option, quotation marks are placed around all texts when saving. Numeric signal values remain unchanged.

Time Axis

  • One time axis for each signal
    Creates for each signal a separate time axis.
  • Common time axis for all signals
    Creates a common time axis for all exported signals. One line is created for each existing point in time. In some cases values do not exist for all signals at a particular point in time. Which signal value is to be saved can be set under Signal Values.
  • Calculate new common time axis for all signals
    Uses the set time interval (in seconds) to calculate a new common equidistant time axis for all exported signals. In some cases values do not exist for all signals at a generated point in time. Which signal value is to be saved can be set under Signal Values.
  • Export time [s] relative to measurement start
    Exports the time relative to measurement start.
  • Export absolute date and time
    Exports the absolute time in 2015-06-15T08:01:47,156000 format.

Signal Values

Depending on the selected time axis settings, you can make the following settings.

  • Current value
    Saves the current value. If no signal value exists for the point in time, the line remains empty.
  • Last value
    If no signal value exists for the point in time, the last valid value will be saved.
  • Next value
    If no signal value exists for the point in time, the next valid value will be saved.
  • Linear interpolation
    If no signal value exists for the point in time, it will be calculated by interpolation from the last and next values. This does not apply to digital signals and value tables.

MAT Settings

Format

  • Limitation of signal name length
    Here you can specify the length of the signal name.
    The signal names are shortened during export to the appropriate length and can then also be used in old MATLAB versions. Select the None entry if you don't want to adjust the length of the signal name.

    Note

    In MATLAB® format names must conform to the MATLAB® convention for variable names, i.e. they must begin with an alpha character and may only contain alphanumeric characters or underlines ( _ ).

    Invalid characters are replaced by an underline before saving.
  • 0: no limitation
  • 19: MATLAB version up to and including 4.x (19 characters)
  • 31: MATLAB version 5.x and higher (31 characters)
  • 63: MATLAB version 6.5 and higher (63 characters)
  • Version
    Data can be saved as
  • MATLAB version 7.0 or earlier or
  • MATLAB version 7.3 (HDF5-based).

Time Axis

  • One time axis for each signal
    Creates for each signal a separate time axis.
  • Calculate new common time axis for all signals
    Uses the set time interval (in seconds) to calculate a new common equidistant time axis for all exported signals. In some cases values do not exist for all signals at a generated point in time. Which signal value is to be saved can be set under Signal Values.
  • Export time [s] relative to measurement start
    Exports the time relative to measurement start.
  • Export absolute date and time
    Exports the absolute date/time as a MATLAB® serial date number.

Signal Values

Depending on the selected time axis settings, you can make the following settings.

  • Current value
    Saves the current value. If no signal value exists for the point in time, the line remains empty.
  • Last value
    If no signal value exists for the point in time, the last valid value will be saved.
  • Next value
    If no signal value exists for the point in time, the next valid value will be saved.
  • Linear interpolation

    This does not apply to digital signals and value tables.

XML Settings

XSLT

These settings concern the XML disassembly export.

  • Execute XSLT conversion
    If this option is activated, the XML file is converted by an XSLT stylesheet after exporting it.
  • Open XSLT conversion result
    Opens the converted file with the associated default application.
  • XSLT stylesheet
    Complete path of the stylesheets, which is used for conversion.

Columns

The configured columns of the Trace Window are used by the export. In contrast, if exporting from a logging file the default bus system/protocol columns of the trace can be selected.

Actions

An action defines an application that is called after saving the target file.

This requires input of the paths of the Application, the Working directory and the Program arguments. If you wish to pass the path of the generated target file in the program arguments, you can do this by inserting the character string %OUTPUT%.

Example 1

You would like to define Microsoft Excel as an action.

  • Choose [Add...] and enter the name Excel.
  • Confirm your dialog selection with [OK].
  • Choose the [...] button to the right of the Application text box.
  • Find the Excel program (e.g.: C:\Program Files\Microsoft Office\Office\EXCEL.EXE) in the file selection list and select it.
  • Confirm the dialog selection with [OK].

Example 2

You would like to define Vector's vSignalyzer as an action.

  • Choose [Add...] and enter the name vSignalyzer.
  • Confirm your dialog selection with [OK].
  • Choose the [...] button to the right of the Application text box.
  • Find the vSignalyzer program (e.g.: C:\Program Files\Vector\vSignalyzer\Exec\vSignalyzer32.EXE) in the file selection list and select it.
  • Confirm the dialog selection with [OK].
