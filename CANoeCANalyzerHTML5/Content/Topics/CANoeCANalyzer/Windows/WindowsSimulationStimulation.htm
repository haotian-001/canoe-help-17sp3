Configuration

Communication Setup

The Communication Setup is the central starting point for communication configuration. You configure:

  • the application layer (application models)
  • the communication layer (communication, bindings)
  • the transmission medium (hardware, bindings)

Simulation Setup

In the Simulation Setup, the overall system is displayed graphically with networks, devices and all network nodes.All options for parameterizing the Simulation Setup are selected in this window.

General [▲ back]

Application Panel

In the Application Panel, you can display and stimulate application layer objects at runtime.

Start Value Window

In the Start Values Window, values can be specified for symbols (system variables, signals, distributed objects). The values are set either at the start of the measurement or when the value is entered.

Signal Generator

Signal Generators are used to define a signal's sequence and measure an ECU's reaction.

The relevant message is transmitted here according to a defined transmission model:

  • via Interaction Layer
  • via schedule table
  • via CAPL or .NET programs

Signal Panel

The signal panel makes it easy to parameterize signals at measurement time.

The relevant message is transmitted here according to a defined transmission model:

  • via Interaction Layer
  • via schedule table
  • via CAPL or .NET programs

IL Configuration

The dialog evaluates the transmission type description of an interaction layer module (CANoe Interaction Layer, OEM Interaction Layer) from the database and displays it. The OEM-specific variants are taken into consideration here.

Symbol Panel

You can use the Symbol Panel to display and edit symbol values during the simulation.

The relevant message is transmitted here according to a defined transmission model:

  • via Interaction Layer
  • via schedule table
  • via CAPL or .NET programs

Panels

Panels are graphical elements via which values of signals and variables can be changed and displayed.

The relevant message is transmitted here according to a defined transmission model:

  • via Interaction Layer
  • via schedule table
  • via CAPLor .NETprograms

Note

With CANoe run, only display elements can be created and configured. Combined control and display elements can only be used for display purposes. See also Limited mode.

A429 [▲ back]

A429 Interactive Generator (A429 IG)

The A429 Interactive Generator (A429 IG) allows you to send ARINC-429-Words defined in a database or parameterized by means of raw data via the ARINC 429 bus. You can set signal values within the words manually or automatically via Signal Generators.

ADAS [▲ back]

Scenario Editor

The Scenario Editor is an application with which traffic scenarios can be created via a graphical interface.

AFDX [▲ back]

AFDX Interactive Generator (AFDX IG)

With the AFDX Interactive Generator you can configure and send messages during measurement.

CAN [▲ back]

CAN Interactive Generator (CAN IG)

The CAN Interactive Generator (CAN IG) allows you to send configurable CAN frames via CAN. You can set signal values within the frames manually or automatically via Signal Generators.

PDU Interactive Generator (PDU IG)

The PDU Interactive Generator (PDU IG) allows you to send configurable AUTOSAR PDUs over any network, that supports sending AUTOSAR PDUs (CAN, Ethernet FlexRay). You can set signal values within the AUTOSAR PDUs manually or automatically via the global Signal Generators.

CANopen [▲ back]

CANopen Interactive Generator (CANopen IG)

The CANopen Interactive Generator enables the creation of message sequences that can be sent once or cyclically. Messages can be selected from a list of CANopen specific message templates and combined into a sequence. The messages are configured via dialogs.

Car2x [▲ back]

Scenario Editor

The Scenario Editor is an application with which traffic scenarios can be created via a graphical interface.

WLAN Packet Builder

The WLAN Packet Builder allows you to easily create and configure WLAN packets to send them to the network.

Ethernet [▲ back]

Ethernet Packet Builder

The Ethernet Packet Builder allows you to easily create and configure Ethernet packets to send them to the network.

Ethernet Interactive Generator (Ethernet IG)

The Ethernet IG allows you to easily send data via UDP and TCP.

PDU Interactive Generator (PDU IG)

The PDU Interactive Generator (PDU IG) allows you to send configurable AUTOSAR PDUs over any network, that supports sending AUTOSAR PDUs (CAN, Ethernet FlexRay). You can set signal values within the AUTOSAR PDUs manually or automatically via the global Signal Generators.

FlexRay [▲ back]

FlexRay Frame Panel

The FlexRay Frame Panel can be used to configure and flexibly modify frames and signals during the measurement.

FlexRay PDU Panel

The FlexRay PDU Panel can be used to configure and flexibly modify PDUs and their signals during the measurement.

PDU Interactive Generator (PDU IG)

The PDU Interactive Generator (PDU IG) allows you to send configurable AUTOSAR PDUs over any network, that supports sending AUTOSAR PDUs (CAN, Ethernet FlexRay). You can set signal values within the AUTOSAR PDUs manually or automatically via the global Signal Generators.

ISO1183 [▲ back]

GNSS Simulator

The GNSS Simulatorcan be used to simulate a GNSS receiver. To do this, a geometric figure or a position file is used to determine a path that will be traced at a variable speed when the simulation is started. GNSS positions thereby generated are transferred cyclically with configurable messages.

J1587 [▲ back]

J1587 Interactive Generator (J1587 IG)

You can use the J1587 Interactive Generator to set signal values, define signal patterns, and send appropriate messages.

When messages are sent by the Interactive Generator, any defined transmission model is bypassed. For example, if in parallel with the Interactive Generator you are also using the interaction layer to transmit a particular message, the behavior on the bus may not correspond to the expected behavior.

Only available if minimum one J1708 channel is used.

J1939 [▲ back]

GNSS Simulator

The GNSS Simulatorcan be used to simulate a GNSS receiver. To do this, a geometric figure or a position file is used to determine a path that will be traced at a variable speed when the simulation is started. GNSS positions thereby generated are transferred cyclically with configurable messages.

LIN [▲ back]

LIN Interactive Scheduler

With the LIN Interactive Scheduler (LIN ISC) you can execute a simple LIN Masternode simulation. In addition to functions for influencing scheduling, the ISC offers the possibility to control the state of a LIN network.

MOST [▲ back]

MOST Interactive Generator

With the MOST Interactive Generator (MOST IG), you can configure and send messages and packets.
