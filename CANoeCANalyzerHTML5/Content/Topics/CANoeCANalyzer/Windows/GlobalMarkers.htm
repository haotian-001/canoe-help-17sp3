Markers

Marker Context Menu

General

Markers make it easy to tag and locate different time points in a measurement.

Adding Markers

  • Via New Marker... in the context menu of measurement area or marker bar
  • Via a double-click on the marker bar
  • State Tracker only: Via a simple trigger configuration in the Marker column of the State Tracker signal view.
  • Clicking the [Set] button in the Marker column adds a marker to all measurement time points that correspond to the specified conditions.

Note for Marker Groups

  • Markers generated via a specific condition (e.g. trigger) belong to a marker group. All markers in a group have the same color and the same name, and are differentiated by means of a numerical postfix, e.g. Marker[1], Marker[2],…
  • You cannot move markers of a group because they are firmly linked to a condition. The condition is shown as a tooltip for the marker.

Removing Markers

  • Clicking Delete in the context menu of the marker bar deletes the selected marker or marker group.
  • Clicking Delete All in the context menu of the marker bar deletes all markers configured in the window.

Note for Marker Groups

You cannot delete individual markers from a marker group. Only the entire group can be deleted.

Scope:
In the Scope Window marker groups can be deleted with Delete All. In the Scope Window for every measurement a group will be added. If you delete a measurement, the corresponding group will be deleted.

Hiding and Showing Markers

  • Clicking Hide in the context menu of the marker bar hides the selected marker or marker group.
    The marker disappears from view, but can be made visible again by choosing Show from the context menu of the marker bar.

In the Trace Window the Show menu command is only available in chronological mode.

  • Clicking Hide All in the context menu of the marker bar hides all of the markers.

Moving and Editing Markers

Clicking Edit... in the context menu of the marker bar allows you to edit the name and color of the marker, and it is also possible to enter a comment. This comment is shown as a tooltip for the marker (when the mouse pointer is moved over the marker in the marker bar).

You can move markers around with your mouse. If you move the mouse pointer over a measurement signal, the marker "snaps" to the sampling points.

In the Trace and Test Trace Window markers can not be moved with the mouse. They must be set explicitly new.

You cannot move marker groups. The position of marker groups is fixed because they are generated via a configured trigger condition. A key symbol by the marker indicates that it cannot be moved.

Note

Markers which were set in the Trace Window or Test Trace Window can be displayed, but not moved, in the Graphics Window and State Tracker. These markers are indicated by a small lock .

If a marker is set to a time stamp which does not fit any time stamp in the Trace or Test Trace Window, in the Graphics Window or State Tracker the marker is then displayed on the next fitting event in the Trace or Test Trace Window. Thus there can be multiple markers for a single event. A tooltip displays the markers.

Only those markers which are in the analysis area are displayed. If you navigate to another marker outside this area, the trace content is reloaded.

Navigate between Markers

With Next Marker, Previous Marker in the context menu, you can navigate to the next or previous marker. With Go to Marker you open an dialogs for selecting a marker.

In the Trace Window the menu commands for navigation are only available in chronological mode.

Time Interval Measurement

Choosing Time Interval Measurement in the context menu of the marker bar allows you to configure time intervals between markers or the end of the measurement. The time interval measurement is shown in a flag by the marker. You can configure up to 3 interval measurements per marker.

Persistence

All markers are removed when measurement begins. The markers are available only after the measurement ends or when it is paused.

Limitations

For performance reasons, a maximum of 10000 markers is allowed per configuration. A marker group (generated via a trigger condition e.g. in State Tracker) can only contain up to 100 markers. If the condition applies more than 100 times, the 100 measurement time points that are closest to the end of the measurement are marked.
