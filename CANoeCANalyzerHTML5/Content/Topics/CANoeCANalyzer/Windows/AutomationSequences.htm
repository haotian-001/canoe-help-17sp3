Automation Sequences

Ribbon: Simulation Tab|Automation

General

Here you can configure and control automation sequences.

The control can also be carried out via the state indicator in the status bar.

Configuration

The following types of automation sequences are available:

Visual Sequences

Visual sequences let you easily (graphically) create bus-relevant sequences.
You can create sequences with system variables or signals. You can send messages or check the values of signals and system variables.

The visual sequences contained in the configuration are shown here in a table.

Column

Description

State

The icon indicates the state of the visual sequence:

  • running
  • stopped
  • paused
  • Incorrect file. A tooltip shows you more detailed information.

Active

Activate the visual sequences here that you want to execute.

Name

Name of the visual sequence. The name can be changed by double-clicking in the cell or using <F2>.

Control
(Buttons)

Here, you can do the following to the visual sequence activated in the list.

  • Start
  • Stop
  • Pause

The buttons can only be actuated while the measurement is active. A visual sequence is stopped automatically when the measurement is stopped.
Autostart

If this option is activated, the visual sequence is started automatically when the measurement is started.
Repetition

Here, you specify how many times the visual sequence is to be repeated.

  • periodic
    The sequence is executed again and again until it is stopped manually or the measurement ends.
  • once
    The sequence is executed exactly one time.

To select the repetition, click in the desired cell.

Toolbar

Button

Description

Creates a new visual sequence.

In the New Visual Sequence dialog, enter a name for the visual sequence.

Confirm with [OK]. The Visual Sequencer opens.

The new sequence is then also available in the Simulation ribbon tab.

Imports visual sequences. You can import files of VSQ format (exported Visual Sequence) or GEN (exported send list of the CAN Generator).

Select the files to import (VSQ, GEN) and confirm with [OK].

The new sequences are also available in the Simulation ribbon tab.

Deletes a visual sequence.

This sequence is then also no longer available in the Simulation ribbon tab.

Opens the Visual Sequencer.

Context Menu

You can also use the context menu to create, import, edit, and remove visual sequences.

Macros

You can use macros to record and execute user actions in panels and Diagnostic Consoles.

The macros contained in the configuration are shown here in a table.

Column

Description

State

The icon indicates the state of the macro:

  • running
  • stopped
  • recording
  • Incorrect file. A tooltip shows you more detailed information.

Active

 Activate the macros here that you want to execute.

Name

Name of the macro. The name can be changed by double-clicking in the cell or using <F2>.

Control
(Buttons)

Here, you can do the following to the macros selected in the list.

  • Start
  • Stop
  • Record

If you wish to record a macro, you need to add the desired macro to the macro list. The macro should not be write-protected.

During recording, all panel actions that you perform are recorded.

Macros recorded during the measurement are saved with the time stamp of the CANoe system time. Macros recorded when no measurement is running are saved with the time stamp"0.000".

You can also execute macros outside of the measurement. When you do so, the time stamps of the macros are ignored and the macro is executed as quickly as possible. No supplemental user actions are possible during playback if no measurement is running.

Note

It is not possible to continue playback when a measurement stop or measurement start occurs.
Playback of a macro is automatically terminated on a measurement stop, a measurement start, or when the configuration is closed.

Autostart

If this option is activated, the macro is started automatically when the measurement is started.

Repetition

Here, you specify how many times the macro is to be repeated.

  • periodic
    The macro is executed again and again until it is stopped manually or the measurement ends.
  • once
    The macro is executed exactly one time.
  • custom.
    The macro is repeated a specified number of times.

Customer Macro Settings

On the right side of the table you can make the following settings for a macro activated in the table. The repetition setting is ignored outside of the measurement.

  • Customer repetition
    Here, you enter the number of times the macro is to be repeated if you have selected a user-defined repetition in the table.
  • Execute with animation
    Time stamps are ignored in Animation mode. All events are transmitted in accordance with the selected animation factor, which must be at least 1 ms.
  • Read only
    If a macro is write-protected, you cannot start any recordings for it.is automatically disabled.
  • Warn on overwrite
    A warning appears if you begin recording a macro for which a source file already exists. If you don't want to see any warnings and want to overwrite the file immediately, you can deactivate warnings for the selected macro here.
  • Record
    You can specify which events to record here. You can record panel entries and diagnostic requests either together or separately. When you record panel entries, changes to signals are also recorded. In the case of signals, the value labels (symbolic values) from the value tables defined in the database are recorded.

Toolbar

Button

Description

Creates a new macro (ASC).

In the New ACS Macro dialog, select a macro name and a new macro file. Select a name for the macro and select a file. As long as a macro recording is not yet running, you can use Start macro recording immediately to specify whether the recording for this macro is to be started immediately.

Note

The newly created macro file does not actually exist until the recording has been made.

Adds macros (ASC).

Select the macro files to add and confirm with [OK].

The macros will be added to the configuration.

Deletes a macro.

The macro file is not deleted in the process.

The file is opened in the editor for ASC files specified in the Options dialog (with consideration of the specified structure).

Context Menu

You can also use the context menu to create, add, edit, and remove macros.

Example of Macro Application

The macro in this example is one that creates a reproducible test that requires defined initial values.

First you create the desired macro. Then you record the required starting states before starting the measurement. Once you have recorded the defined initial values, you start the measurement and record your test.

At first, the recorded macro contains only events with the time stamp 0.000. These are the initial values recorded before the measurement. Afterwards, events are executed with time stamps based on the CANoe system time.

To play back the reproducible test, configure your macro to As Recorded playback mode. Activate it to begin at the start of the measurement by selecting Playback at Measurement Start.

Start the Measurement.

Macro playback starts automatically. The defined initial values (time stamp 0.000) are played back as quickly as possible, i.e., the initial values are set. Afterwards, the reproducible test is executed according to its recorded time stamps.

.NET Snippets

.NET Snippets provide a similar functional scope as visual sequences, but are programmed in a .NET language (e.g., C#) instead of being formed visually/symbolically.

The .NET Snippets contained in the configuration are shown here in a table.

Column

Description

State

The icon indicates the state of a function of a .NET Snippet:

  • running
  • stopped
  • Incorrect file. A tooltip shows you more detailed information.

Snippet

Displays the snippets and the functions they contain.

Control
(Buttons)

Here, you can do the following to the individual functions of a .NET Snippet

  • Start
  • Stop

Toolbar

Button

Description

Creates a new .NET Snippet.

In the New .NET Snippet dialog, choose a snippet name and a new snippet file.
Confirm with [OK]. The .NET Snippet is added to the configuration and an editor with a predefined snippet opens. You can specify the editor in the Options dialog.

Adds .NET Snippets.

Select the snippet files to add and confirm with [OK].

The .NET Snippets will be added to the configuration.

Removes the .NET Snippet from the configuration.

Note that all snippet functions programmed in the file will also be removed.

The file opens in the editor for .NET programs specified in the Options dialog (e.g., Visual Studio).

Compiles the active .NET Snippet, after which any errors in the program are displayed.

If the Developer mode button is activated, the snippets are loaded into a separate AppDomain. If no function was started, snippets can then be edited and recompiled during the measurement.

Advantage:

If all functions of a snippet are stopped, the snippet can be edited in the editor and recompiled during the measurement.
If the button is deactivated, the snippets can no longer be edited after they have been started one time. Instead, this can be done only after the end of measurement.

Disadvantage:

In this case the execution of the snippets takes longer.

Context Menu

You can also use the context menu to create, add, edit, compile, and remove .NET Snippets.
