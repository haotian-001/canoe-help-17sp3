Application Models

 

Application Models

Here, you can see model files (e.g. CAPL programs) which should be executed during measurement.

Via the toolbar you can add new files.

Via the context menu you can delete files.

Content Overview

Application Models

Overview of application models including state and location.

File

Display of the location of the selected file.

Properties and Actions

Application Models

Properties

—

Actions

 

Off

Elements are switched off and do not take part in communication.

On

Elements are simulated from CANoe and take part in communication.

Add

Adds a new file to the list. You can add CAPL programs, C Sharp programs or DLLs.

Multiple files can be selected.

In the Properties Window of the Communication Setup you can select for endpoints whether an access in a C Sharp program is possible via a generated C# API.

You can also add new files with drag and drop.

File

Properties

Name

Name of the application model

State

Here you select whether the model should be used for the communication:

  • On: Is used
  • Off: Is not used

Qualification of Participants

Participants whose qualification can be shortened during programming.

This property is only displayed if communication objects are included in the configuration.

Example

Example

  • Unassigned participant
    <CO>.<side>.<consumer>.<provider>.<action>
    Mirror.providerSide.Console.Mirror_Driver.SetProviderImplementation
  • Assigned participant
    <CO>.<action>
    Mirror.SetProviderImplementation

With [...] you open the Select Participants dialog. Here you select the participants whose qualification should be shortened.

Actions

Edit

With a click on the link you open the file in a editor.

These editors can be configured in the Options dialog.

Debug

With a click on the link you open the file (CAPL, .NET) in the debugger.

Remove

Removes the selected file.

Details

No further information is displayed here for the selected application model.
