Measurement Point

 

You can observe values of real endpoints in CANoe.

Note

To observe the real communication in CANoe, endpoints must be real connected.

In the Communication Setup the following options are available:

Observing the State

Use the following settings if you want to observe values that are set in a Provider/Server.

  • Set the state for the desired participant/endpoint (Provider/Sender) to Remote.
  • Activate the measurement point
  • with a click on the circle of the graph in the Communication Setup
  • via the participant properties, Measure contained endpoints option
  • via the endpoint properties, Measure option

    • The new state is displayed in the graph of the Communication Setup.

    Note

    For participants this action is only possible if at least one Provider/Server endpoint is contained. Only on these endpoints measurement points can be selected.
  • Apply the changes with [ Apply].

    • In the Symbol Explorer, you can now see by the symbols (and ) in the view of the communication objects that real values are observed. The same symbols are also displayed in the Trace Window.

Observing the Communication via a Measurement Point

Use the following settings if you want to observe the communication between to endpoints via a measurement point.

  • Click in the graph of the Communication Setup on the circle between two participants/endpoints.

    • The new state is displayed in the graph of the Communication Setup.

  • Set the state for the desired participants/endpoints to Romote.
  • Apply the changes with [ Apply].

    • In the Symbol Explorer a new Measurement Points category is displayed in the view of the communication objects . You can see by the symbols (and ) that real values are observed. The same symbols are also displayed in the Trace Window.

    Example

    In the following figure the Measurement was activated between the Mirror_Driver and Console participants. In the Symbol Explorer for the Mirror communication object the Measurement Points|measure[Console,Mirror_Driver] category was created in the view of the communication objects. The communication between both endpoints is marked with an additional .

    Info

    The Console and Mirror_Driver participants include the following endpoints that communicate with each other:

    • Mirror_Driver participant: Services::Mirror[Mirror_Driver] endpoint as provider ()
    • Console participant:: Services::Mirror[Console] endpoint as consumer ()
