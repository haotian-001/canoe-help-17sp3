Qualification in Application Model

Application models (*.cs) in which the qualification of the participant can be shortened during programming.

Example

  • Unassigned participant
    <CO>.<side>.<consumer>.<provider>.<action>
    Mirror.providerSide.Console.Mirror_Driver.SetProviderImplementation
  • Assigned participant
    <CO>.<action>
    Mirror.SetProviderImplementation

With [...] you open the Select Application Models dialog. Here you select application models in which the qualification of the participant can be shortened.

With a click on the name you select the application model. in the graphical overview.

Generate C# API

Here you can select whether an access to an endpoint in a C Sharp program is possible via a generated C# API. You can reduce generation time and memory consumption by activating this option for required endpoints only. The availability of the endpoints in the analysis windows is independent from this option. E.g. endpoints can also be displayed in the Graphics Window with deactivated option.

Measure contained endpoints

You can only activate this option, if you set the state of the participant to Real (Device).

If you activate this option, all in the participant contained Provider/Server endpoints are set to Real (Device) state and the Measure option (see under Endpoints) is set. Now you can observe values for these endpoints.

The option is automatically activated, if at least one contained Provider/Server endpoint hast been set to Real (Device) state and the Measure option (see under Endpoints) has been set too. The participant gets also the Real (Device) state.

Participants and Endpoints

Participant

In the graph you can see participants communicating with each other.

Via the blue links in the connect participants, you select the respective participant.

The symbol in the middle of the connection line displays that you observe the communication on a measurement point between two participants. You can switch the measurement on or off directly in the graphic with one click.

The symbol on a Provider/Server displays that you observe values set on a Provider/Server. The state of the Provider/Server must be set to Real (Device).

Communication Objects

Information which communication objects belong to the participant.

Endpoint

In this graph you can see the selected endpoint and all other endpoints with the same communication object.

Via the blue links in the connected endpoints, you select the respective endpoint.

The symbol in the middle of the connection line displays that you observe the communication on a measurement point between two endpoints. You can switch the measurement on or off directly in the graphic with one click.

The symbol on a Provider/Server displays that you observe values set on a Provider/Server. The state of the Provider/Server must be set to Real (Device).
