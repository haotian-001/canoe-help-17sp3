Toolbar and Navigation Path

 

Toolbar

The toolbar provides the following symbols:

Symbol

Function

Load data source

Here, you select single data sources for the CANoe configuration.

Available file types:

  • AUTOSAR (*.arxml)
  • Vector communication data model (*.vCODM)
  • Vector Communication Description Language (*.vCDL)
  • *.XML

Note

Working with AUTOSAR extracts and system descriptions.

Create new vCDL data source

Here you can create new vCDL files. The newly created vCDL files are automatically added as data source.

Depending on the selection in the list, a corresponding application protocol is activated.

With Default you create a vCDL file without application protocol information.

With [...] you can make further settings for some protocols in the Options dialog.

You can edit the vCDL files in the vCDL Editor.

Activate application protocol

By activating an application protocol, you activate

  • interfaces with which you can define protocol-specific application layer objects
  • the binding with which you can define the transfer of the application layer objects

If you want to delete an activation, you must delete the vCDL files for the corresponding protocol as data source.

Add application model

You can add CAPL programs, C Sharp programs or DLLs.

Multiple files can be selected.

In the Properties Window of the Communication Setup you can select for endpoints whether an access in a C Sharp program is possible via a generated C# API.

You can also add new files with drag and drop.

Add VTT model

Here, you can select a VTT model file (*.vttsut) that was generated from vVIRTUALtarget. A VTT model contains data sources and application models.

You can also import VTT models with drag and drop.

Smart Charging Configuration

Use the list to activate the application protocol for which you want to define application layer objects.

Click [Configure...] to open the Smart Charging Configuration dialog. If you have not selected an application protocol before, all possible application protocols are activated. After defining your smart charging objects and closing the dialog, a new namespace is displayed under Application Layer Objects.

If you want to delete the activation, you have to delete all related objects in the Smart Charging Configuration dialog.

 / 

Show namespaces/clusters

If you have created DOs for smart charging in the Smart Charging Configuration dialog, then you can select whether you want to display the DOS sorted by namespaces or clusters under Application Layer Objects.

Validate data model

Validated the loading process.

Shows or hides properties and actions.

Navigation Path

Beneath the toolbar, the search and the path to the currently selected element is shown. This path is especially helpful for very long lists, if the superordinate elements are not located in visible area.

By clicking on an element in the navigation path, you can navigate to this element in the tree view.

Symbol

Function

Navigates back to the last selection.

Navigates forwards to the previous selection

Navigates to the superior folder.

 

In the Search... field you can enter a term to search for in the content overview. Wildcards * are supported..
