Video Configuration

Ribbon: Hardware Tab|Video Configuration

In the Video Configuration dialog, you can manage and configure Video Windows.

Note

You can configure up to four Video Windows.

The most commonly used video formats are supported in the Video Window for the Online Source depending on the current system configuration and the DirectShow components already installed on the system. Thus, for example, uncompressed AVI, compressed AVI (with corresponding installed Codec, e.g., DivX), MPG, MPEG, WMV, and MP4.
The AVI format is available for the Recorder File.

If the recording of a video file is paused and restarted during an active measurement, visible artifacts may occur in the image at the intersection points of the video file. You can prevent occurrence of these artifacts by setting the pre-trigger time (see Recorder below) to a value greater than 0.

Recorder

Active

Here, you select whether the recording is active during an online measurement.
If this setting is not active, the recording cannot be subsequently started manually (toolbar or state indicator) or via the program (CAPL function).

Use video trigger (manually, by CAPL or COM)

This option allows you to start or stop a video recording manually (within the toolbar), via CAPL or COM (StartRecording, StopRecording).

  • Pre-trigger time

    • Indicates how long video data are to be buffered as long as the recording has not yet been started or is paused. When the recording is started manually (toolbar or state indicator) or via the program (CAPL function), the video data buffered before the triggering event are taken into account and written to the recording file.

    Note

    If a pre-trigger time greater than 0 is set, the individual video segments must be copied as a unit when the measurement is stopped. As a result of this, global functions such as the starting of a new measurement, a change of configuration, or the closing of the application can be delayed.
  • Post-trigger time

    • Indicates how long video data will continue to be written to the recording file after the recording has been paused manually (toolbar or state indicator) or via the program (CAPL function).

  • Trigger recording on measurement start

    • The recording starts at the start of the measurement.

The video file can be split by means of field functions, for example according to file size and time, see section Recorder file.

Use trigger/splitting of Logging Block

This option allows you to use the settings of a Logging Block for recording a video file.

  • The trigger settings of the Logging Block will be applied.
  • The settings from the logging file of the selected Logging Block will be applied for splitting.

In the selection list, all configured Logging Blocks are displayed.

If the option is activated, then

  • each trigger of the Logging Block activates a trigger for the Video Window, i.e. the Logging Block and the Video Window always record the same sections.
  • the video file is also split when the logging file is split. The video file can be additionally split by means of further field functions, for example according to file size and time, see section Recorder file.
  • pre and post recording times from the Logging Block will be applied.

Note

  • Simultaneous video recordings are not supported. For simultaneously recorded logging files there is only one video file (which ends with _MultiTrigger).
  • If you have activated the option Use trigger/splitting of Logging Block, then you cannot use any video-specific triggers (manual, CAPL, COM).
  • In the Logging Block, you can determine a test trigger using a verdict. In this case, the logging file will be saved only if the selected verdict is fulfilled, otherwise the logging file will be deleted. The behavior of the video file and the logging file for the selected trigger is identical. The video file is deleted if the logging file is deleted.

Field Codes

With the field codes it is possible to generate and increment filenames.

Advantages:

  • Existing files can be protected against overwriting.
  • After the end of measurement you don't have to constantly enter new filenames.
  • The logging recording can be split into several (smaller) logging files
  • which are easier to be sent
  • which are easier to be analyzed

Note

Please note that the following settings can result in many hard-drive accesses and may lead to an increased system load:

  • Field function Increment at each Trigger is configured and recording is often started and stopped
  • Field function Increment by File Size is configured, the file size is small, the video resolution is relatively high

In both situations, the video files are very often finished and started.

Increment at each Trigger

{IncTrigger|001}

Each trigger can be distributed to an individual video file. The filename is incremented after each trigger.

You can predefine a start value in the field code.

Increment by File Size

{IncSize|001|100MB}

With this function the video recording can be split into several (smaller) video files.

There is a new video file when the entered size of the file is reached; the filename is incremented automatically. The size has to be entered in MB

You can predefine a start value in the field code.

Increment after time

{IncTime|001|01d01h00m}

With this function the video recording can be split into several (smaller) video files.

A new video file is generated when the entered time is reached; the filename is incremented automatically. The beginning of the time slot is either a start of measurement or a new incoming message. The time can be set in days, hours and minutes:

  • Days (d): 0-99
  • Hours (h): 0-23
  • Minutes (m): 0-59

You can predefine a start value in the field code.
