Video Window: Synchronization

 

Video files displayed in the Video Window can be synchronized with data in other Analysis Windows when the measurement is stopped.

The maximum synchronization precision can be achieved with video files that were recorded during an online measurement using the video window. In this case, in addition to the video file an identically named synchronization file with filename extension *.VideoSync is recorded. With this supplement to the video file, precise synchronization between the CANoe measurement time and the camera hardware is ensured by storing the associated measurement time instant of the measurement for each video frame.

Note

For a typical frame rate of 30 frames/second, synchronization data for a maximum of 12 hours of video are stored per window. That is, a precise synchronization is possible for the last 12 hours of the measurement. If the video file is longer, the video position in areas for which no more data exist is derived from the measurement time through a direct conversion. This enables a synchronization same as before, but it is less precise.

Offline Source

If you select a video file in the Offline Mode Window with associated synchronization file as the offline source, the synchronization file will be played again automatically within the scope of offline mode so that synchronization data also exist for the offline analysis.

Note

If no synchronization file is present, the video file can nevertheless be used as an offline source. In this case, the beginning of the video file is assigned to the beginning of the measurement. The video position is derived from the measurement time through the direct conversion. This enables a synchronization, but it is less precise.

Imported Video Files

Synchronization data are generally not played for imported files. If you wish to take into consideration existing synchronization files, use offline mode.

Offset

In the toolbar or the Configuration dialog, you can manually assign an offset for all video files that can be used to align the video file relative to the measurement. This offset can be used, for example, in the following cases:

  • You import a video file that was recorded with another program in parallel with a CANoe measurement and thus the start time of the measurement does not coincide with the start time of the video file.
  • Depending on the camera used, an undefined time span elapses from the time instant of the frame recording until the processing of the frame in CANoe. Because the measuring time is not determined until the processing of the frame in CANoe, the video may be shifted relative to the measurement

In order to set an offset for alignment of the video file, proceed as follows:

  1. Stop the measurement.
  2. Activate the window synchronization with in the CANoeHome ribbon tab.
  3. Find and select a recognized event in one of the Reference Windows (e.g., Trace Window).
  4. Using the button in the toolbar, adjust the offset correction until you see the frame that belongs to the selected reference event.

The following can be synchronized:

  • Video files recorded in CANoe (configuration dialog: Recording )
  • Video files in Offline Mode Window
  • Imported video files (note the precision without an existing synchronization file)

Note

The maximum synchronization precision always depends on the frame rate of the video file (e.g., at a frame rate of 30 frames/second, even when a synchronization file is used, the maximum synchronization precision of a bus event in the Trace Window is 33 ms).
