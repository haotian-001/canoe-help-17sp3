Smart Charging Configuration

 

In the Smart Charging Configuration dialog, you can define and edit distributed objects (DOs) for Smart Charging.

Note

Note that changes you make in this dialog are saved in the CANoe configuration as a vCODM file.

Toolbar

Overview of Protocols and Network Configuration

The overview on the left shows the application protocol (CCS, CHAdeMO, OCPP) and Ethernet Peer Setup. The display depends on the activated protocols.

Note

If you have not yet activated an application protocol for Smart Charging via the Communication Setup toolbar (), the available application protocols are activated the first time you open the Smart Charging Configuration dialog.

A CSS- or OCPP-DO always has a corresponding IP stack, which you can configure under Ethernet Peer Setup.

Content Overview

If you have selected an application protocol in the overview on the left, all available DOs are displayed in the middle view.

The distributed objects (DOs) managed here each model either a vehicle (EV), a charging station (EVSE/ChargingStation) or a charging station management system (CSMS).

You can add or delete DOs either via the toolbar or the context menu. The individual cell contents cannot be edited here.

If you have selected Ethernet Peer Setup in the overview on the left, the IP stacks for the individual DOs are displayed in the middle view (only with CCS or OCPP application protocol).

The IP stacks are automatically created or deleted when CCS- or OCPP-DOs are created or deleted. The displayed information cannot be changed here, but you can change some data via the Properties view.

Properties

Depending on the selection in the overview on the left or the content overview, the IP stack properties or the protocol-specific DO attributes are displayed here.

For distributed objects, you can determine the runtime behavior of the simulation here. More information about the DO attributes can be found on the following pages:

  • CCS
  • CHAdeMO
  • OCPP

For IP stacks, you can specify the MAC and IP addresses, as well as the port here. This data is used for communication between DOs or between DO and hardware.

Validation

You can open the Validation view using the splitter at the bottom of the dialog.

In the Validation view, errors that occurred during creation/editing are displayed. The validation refers to all data in the dialog. With a double-click on an entry you navigate directly to the displayed object.

You can start the validation from the toolbar of the dialog or here in the splitter.
