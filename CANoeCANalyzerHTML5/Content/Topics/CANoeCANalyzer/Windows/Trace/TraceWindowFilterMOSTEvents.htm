Analysis Filter for MOST Events

Context Menu Path: Analysis Filter|Add Condition|Add MOST Event...

With the MOST Filter Condition dialog, you can either use an existing MOST event in the Trace Window as a template for the filter or create a new filter.

To edit an existing event from the Trace Window, move this event from the Trace Output Window using drag and drop so that it becomes defined as an analysis filter and sequence filter. The filter dialog can then be opened using the Edit Condition... context menu command.

Note

When you drag and drop a message the analysis filter is set per default to filter all protocol types (CMS, AMS, MHP) as well as node and spy messages. This concentration on application-relevant content facilitates a quick creation of filters. If too many messages have been filtered you can restrict(/open) the filter in the filter dialog.

For a new filter, open the dialog directly using the menu path Add Condition|Add MOST Events....

Filter Conditions

For addressing purposes, the Channel and the Source or Destination address can be specified

The function blocks can be specified individually. If a database exists, the available values from the database are listed for the various elements. Automatic consideration is given to interrelationships of the database during selection.
You can also enter the values for the elements numerically. If the database contains this value, it is updated in the input field accordingly.

Direction

It is possible to distinguish between sent Tx, received Rx, and Spy events.

Protocol

Here, you have the option to filter by one or more protocols.

  • CMS
    Filters the message frames of the control channel
    AMS atoms, MOST message types: CtrlFrame, CtrlSpyFrame
  • AMS
    Filters messages of the Application Message Service
    Message types: AMSMsg, AMSSpyMsg
  • MHP
    Filters individual packets of the MOST High Protocol
    Message types: Packet(MHP), PacketSpy(MHP)
  • MHP Combiner
    Filters events of the MOST High Protocol Combiner
    Message types: MHPBlock(Spy), MHPPacket(Spy), MHPConnection(Spy)
  • MHP Combiner
    Filters events of the MOST High Protocol Observer
    Message types: MHPError(Spy), MHPWarning(Spy)

You can use the wildcard function (*) in all input fields. This applies all possible values in the filter.
A wildcard selection is possible for FBlock, Function and Optype. If you have assigned a value as a wildcard in the function block, all subsequent values must be specified directly as numerical values.
