Predefined Filter for MOST Events

Predefined Filter|MOST Events

In the MOST Filter dialog you can set filters on defined messages and events.

Messages

These settings determine which messages are to be filtered.

An activated check box means that this message type passes the filter and is displayed in the Trace Window.

The check box AMS atoms (CMS) should be activated, if analyzing mainly the communication on bus level.

If working on the application level, it is advisable to activate the AMS check box and to deactivate AMS atoms (CMS).

With the Packet check box the display of packets of the asynchronous channel can be disabled.

Note

When using the VN2600/VN2610 or VN2640 of Vector as network interface the asynchronous channel can be deactivated in the hardware configuration dialog. Thus the CPU as well as the USB bandwidth are discharged, whereas the filter described here solely filters out the packets.

MOST Events

These settings determine the filtering of event types.

An activated check box means that this event type passes the filter and is displayed in the Trace Window.

Default Settings

Here you can save your currently selected settings as the default MOST filter settings, and also load or delete the default settings.

If a Trace Window is inserted in the Measurement Setup with standard settings, then the stored filter settings are used. If no standard settings were stored, the factory settings are used instead.

If a Trace Window is inserted in the Measurement Setup with MOST settings, then the Vector factory settings are taken over for this window.

By loading the standard settings there is the possibility of checking the stored settings, as well as later taking over the settings to an already existing window.
