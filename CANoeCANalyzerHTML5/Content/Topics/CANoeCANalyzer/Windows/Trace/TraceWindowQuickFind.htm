Context Search

Context Menu: Context Search

The context search feature makes it easy to navigate between the messages of a channel in the Trace Window (sequential mode). The Trace Window must be paused during measurement. The following functions are available, depending on the selected message:

J1939

Search Criteria

Description

Next J1939 PG from the same source

The search looks for the next parameter group with the same source address as the current selected parameter group.

Previous J1939 PG from the same source

The search looks for the previous parameter group with the same source address as the current selected parameter group.

J1939 request from selected parameter group

If a request parameter group is selected, the search looks for the response parameter group. If another parameter group is selected, the search looks for a a request parameter group.

Next J1939 transport protocol from the same source

The search looks for a J1939 transport protocol transmission of the current selected source address.

Previous J1939 transport protocol from the same source

The search looks for a J1939 transport protocol transmission of the current selected source address in reverse order.

Start of J1939 transport protocol

The search looks for the start of a J1939 transport protocol transmission of the current selected source address.

End of J1939 transport protocol

The search looks for the end of a J1939 transport protocol transmission of the current selected source address in reverse order.

J1939 SDM of selected SHM

If a Safety Header Message (SHM, J1939-76) is selected, the search looks for the corresponding Safety Data Message (SDM, J1939-76).

J1939 SHM of selected SDM

If a Safety Data Message (SDM, J1939-76) is selected, the search looks for the corresponding Safety Header Message (SHM, J1939-76).

MOST

Command/Report

If you select a MOST message that represents a command (e.g. OpType "Get"), the context menu in the Trace Window contains the following entries:

Context Search

Function

Report

Switches to the report that very probably belongs to the selected command. Filters (e.g. MOST Application Filter) and filter settings applied to the Trace Window may influence the result.

First Report

Switches to the first report that may belong to the selected command.

Previous Command

Switches to the previous command of the same type.

Next Command

Switches to the next command of the same type (same FBlockID, FunctionID and OpType, same InstID, or Wildcard).

If you select a MOST message that represents a report (e.g. OpType "Status"), the context menu contains the following functions:

Context Search

Function

Command

Switches to the command that very probably belongs to the selected report.  Filter blocks (e.g. MOST application filter) and filter settings applied to the Trace Window may influence the result.

First Command

Switches to the first command that may belong to the selected report.

Previous Report

Switches to the previous report of the same type.

Next Report

Switches to the next report of the same type (same FBlockID, FunctionID and OpType, same InstID, or Wildcard).

Note

  • The context search results depend on correct chronological sorting in the Trace Window.
  • The first context search function (for finding the associated report or command) is not available when you use node position or group addresses.

MOST High

If you select a message that belongs to a MOST High transmission, the following functions are available in the context menu:

Context Search

Function

MOST High Packet

Switches to the MHP packet event that is generated by the MHP combiner when the connection ends.

Previous MOST High Block

Switches to the previous MHP block event.

Next MOST High Block

Switches to the next MHP block event.

AFDX

Search Criteria

Description

Next Error

Switches to the next AFDX message containing an error

Previous Error

Switches to the previous AFDX message containing an error

Next sequence number

Switches to the next AFDX-message with the next sequence number in the selected virtual link

Previous sequence number

Switches to the previous AFDX-message with the next sequence number in the selected virtual link

Note

If no matching AFDX-message could be found in the current Trace Window, previous or further Trace Windows will be searched.
