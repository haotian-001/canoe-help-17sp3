Columns for CAN / CAN FD

Context Menu Path: Configuration|Column Configuration|CAN / CAN FD

Columns for CAN

Field

Title

Function

Time

Time

Absolute time from the measurement start, or relative to the previous event.

Channel

Chn

CAN channel.
The channel number on which the message was sent/received.

Identifier

ID

CAN identifier of the message.

Name

Name

Symbolic name of the message.

ID / Name

ID / Name

Depending on the global settings in the Home ribbon tab, either the symbolic name or the CAN identifier of the message.

Dir

Dir

Tx = Transmit message,
Rx = Receive message,
TxRQ = Transmit request.

(NAK) Only for Error Frames with Error Code "Not Acknowledge" and passive Error Flag.

DLC

DLC

Length of the data field in decimal representation.

Data

Data

The data in decimal or hexadecimal representation.
For remote frames the Data column indicates Remote Frame.

At the interpretation of the OSEK TP protocol square brackets are displayed to distinguish between TP protocol bytes, unused bytes and user data. TP protocol bytes and not used bytes are displayed in brackets.

Attribute

Attr

Additional attributes:
WU = Wake-up,
TE = Transceiver Error,
XX = Transceiver Error and Wake-up

Protocol time

Frame Duration

Duration of the complete event in milliseconds and bit times. It also contains the Inter-Frame-Space (3 bit times). This value is only displayed if the Start-of-Frame time stamp is available – see hint for Start-of-Frame column.

Type

Event Type

The text CAN Frame or CAN FD Frame is displayed, depending on the protocol with which the message is sent.

Specific 6

Bus Idle

Time between End of Frame of previous bus event and Start of Frame of current bus event in milliseconds. Thereby it is the Bus Idle duration between the current and the preceding bus event. Note, that the End of Frame time stamp includes also the 3 bits of the Inter Frame Space. This value is only displayed if the Start of Frame time stamp is available – see hint for Start of Frame column.

Specific 7

Bus Busy

Accumulated time from the end of the last Bus Idle phase up to the end of the current bus event in milliseconds. Thereby this is the duration of the last not interrupted (by Bus Idle) events burst up to the end of the current bus event. This value is only displayed if the Start of Frame time stamp is available – see hint for Start of Frame column.

d:hh:mm:ss

d:hh:mm:ss

Absolute time from the measurement start in d:hh:mm:ss.000 format.

Diff Time

Diff Time

Time relative to the previous event.

In chronological mode, the column displays the time difference to the visible, previous row.
This means that the value depends on the sorting and the filter settings.
In the fixed mode, DiffTime displays the difference to the previous-event.

Bustype

Bustype

Specification of the bus system (CAN, LIN, MOST, FlexRay,…).
J1939 messages are specified as bus system CAN, because J1939 is a protocol, not a bus.

Send Node

Send Node

Display of the send node entered in the database for the corresponding message.
For J1939 this column remains empty!

Bus

Bus

Bus name.

Database

Database

Database in which the corresponding message is defined.

Counter

Counter

Indicates the number of times the event has appeared since measurement start.

Start of Frame

Start of Frame

Timestamp at the start of the bus event.

Note:
For some CAN hardware this value has to be calculated, otherwise no value is displayed. The Start-of-Frame calculation can be activated via Options dialog (Bus Systems/Protocols|Bus Systems).

Specific 15

Arb. Field ID

The frame ID in binary format.

Data ASCII

Data ASCII

The data bytes will be interpreted as character code.
This representation is suitable, for example, if texts are being transmitted in a transport protocol.

Comment

Comment

If you add a comment to a message in CANdb++ Editor it will appear here. Comments also appear to PDUs (Protocol Data Unit) for J1939 standard.

Diff Time (Ref. Event)

Diff Time (Ref. Event)

Time difference between the current event and the selected reference event.
This column contains only data in chronological mode.

Date and Time

Date and Time

Displays the absolute time stamp (Date/Time) for the corresponding message.
In offline mode this corresponds to the time the message was logged.

You can specify the display format (Date and Time or Time) in the Options dialog.

Logging files generated with CANoe always contain date and time of measurement start in the Header.
If a logging file does not contain this information, then date and time of the offline measurement start are displayed in this column.

Data length

Data length

Interpreted value of Data Length Code (DLC).

With CAN FD a DLC of 9 is not 9 databytes long but 12.

Furthermore, this column currently displays a value of 8 (12).
That means 8 data bytes are valid, but the message has a length of 12.
The CAN controller can read currently only 8 bytes of data.

XCP/CCP Command

XCP/CCP Command

XCP/CCP command names.

Responses on a command are displayed

  • for success case like this Ok:command name
  • in error case like this ERR
    (with CCP currently like this Err:command name)

See Protocol Observer.

XCP/CCP Parameter

XCP/CCP Parameter

Parameter of XCP/CCP commands or the response are displayed as Sub-Lines and in a separated column. If you select a parameter, the corresponding bytes in the Data column are highlighted.

See Protocol-Observer.

Diff Time (last Occ.)

Diff Time (last Occ.)

Time difference between the current event and the previous event of the same type. This column display the period of time since the last reception of a message, system variable,...

In fixed mode, the column has the same meaning as DiffTime.

BRS

BRS

Bitrate Switch: Indicates whether the bitrate in a CAN message is switched between arbitration and data phase. If the function use BRS is activated a 1 appears.

ESI

ESI

Error State Indicator: Indicates whether the CAN controller is in state Error Active = 1, or Error Passiv = 0.

Update Bit

Update Bit

Flag for displaying the PDU update bit (AUTOSAR PDUs and FlexRay PDUs).

If the update bit is configured in a PDU, the update is set automatically when a data update is performed. It is reset after a transmission.

Diff Time (last Occ.)

Diff Time (last Occ.)

Time difference between the current event and the previous event of the same type. This column display the period of time since the last reception of a message, system variable,...

In fixed mode, the column has the same meaning as DiffTime.

Validation State

Validation State

Flag to indicate the SecOC validation state.

  • 1 = successful
  • 0 = not successful.

The flag is only used when security is activated.

Error Event Details

ECC (Error Code Capture Register)

The ECC is evaluated and displayed as text in the Data column.

The next lines contain an extended error interpretation. For CAN core based interfaces the error position, the frame length from SOF to error position, the CAN ID, the DLC and the data field are visualized.

Note

The validity of ID, DLC and data field depends on the error type and the error position.
