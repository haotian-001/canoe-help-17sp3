Columns for MOST

Context Menu Path: Configuration|Column Configuration|MOST

Field

Title

Function

Time

Time

Synchronized time stamp (see also: time stamps).

Channel

Chn

Channel information with the prefix M.

Identifier

ID

Message ID.

Name

Name

This name is taken from the DBC file, if available.

ID / Name

ID / Name

Depending on the general display option (see database support):

  • Symbolic mode: Name
    This name is taken from the DBC file, if available.
  • Numeric mode: Display of the message ID.

Dir

Dir

Tx = Transmit message,
Rx = Receive message,
TXRQ = Transmit request.

DLC

DLC
  • System message: Length of the valid data in bytes.
  • Control message: Not available - see TelLen.
  • AMS message: Length of the user data in bytes.
  • Packet: Length of the whole packet in bytes.

Data

Data

Transported data of the event (HexDump) .

Attribute

RType

Subtype of the MOST Control Frame: RemoteRead, RemoteWrite, Alloc, Dealloc, GetSource.

Source

Source

Depending on the general display option:

  • Symbolic mode: Device name, if available (see database support).
  • Numeric mode: Source Address

Destination

Destination

Depending on the general display option:

  • Symbolic mode: Device name, if available (see database support).
  • Numeric mode: Destination Address

Original Time

Original Timestamp

Original time stamp generated by the network interface, with an offset of 0 at the start of measurement (see also: time stamps).

Type

Type

Type of MOST event:

  • CtrlFrame – Message of the Control Channel
  • AMSMsg – Message of the Application Message Service
  • CtrlSpyFrame – Message received by the Control Spy.
  • AMSSpyMsg – Message of the Application Message Service, received by the Control Spy.
  • AMSSegErr – Report of a segmenting error by the receiver of an AMS message
  • AMSExtErr – The sender of an AMS message did not receive any Tx receipt from the MOST transceiver.
  • Packet – Packet of the Asynchronous Channel
  • SpyPacket – Packet received by the Async Spy.
  • LightLock – LightLock event
  • Register – Change of a chip register in the MOST hardware
  • RegData – Read result of a chip register
  • HWMode – Change of MOST hardware operating mode
  • Trigger – Trigger event of the MOST hardware
  • NetState – Change in network status

With messages received by the spy, the type name contains the term "Spy", e.g. CtrlSpyFrame.

Specific 0

Disassembly

Symbolic interpretation of the event. For Control Frames and AMS messages the data bytes are interpreted from FBlock to OpType based on the available databases (CANdb or XML). Parameter values are shown symbolically in the Disassembly Column of the Trace Window. The drop-down expanded view of messages is not necessary. Parameter values can now be read directly from passing messages. The number of displayed parameters is limited to 6 to preserve the performance and clarity of the Trace Window.

Specific 1

State

Status information for MOST messages.

Specific 2

AckNack

Acknowledgment information for MOST messages.

Specific 3

CRC

Cyclic block check code of the Control Frame or packet.

Specific 4

ASCII

ASCII representation of the data bytes.

Specific 5

DbType

Type of database the MOST frame is disassembled with. This is "DBC" for CANdb++ database or "XML" if the disassembly is based on the XML description of the MOST Function Catalog.

Specific 6

FBlock.InstID

Symbolic or numeric display of the functional address of messages satisfying requirements ranging from function catalog format to OpType (Control and AMS messages).

Specific 7

Function

Symbolic or numeric display of the function name for messages satisfying requirements ranging from function catalog format to OpType (Control and AMS messages).

Specific 8

OpType

Symbolic or numeric display of the OpType for messages satisfying requirements ranging from function catalog format to OpType (Control and AMS messages).

Specific 9

ParamBytes

Decimal or hexadecimal display of the parameter bytes for messages that are conform to the function catalog. If not, this field remains empty.

d:hh:mm:ss

d:hh:mm:ss

Absolute time from the measurement start in d:hh:mm:ss.000 format.

Diff Time

Diff Time

Time relative to the previous event.

In chronological mode, the column displays the time difference to the visible, previous row.
This means that the value depends on the sorting and the filter settings.
In the fixed mode, DiffTime displays the difference to the previous-event.

Bustype

Bustype

Specification of the bus system (CAN, LIN, MOST, FlexRay,...).

Bus

Bus

Bus name.

Database

Database

Database in which the corresponding message is defined.

Counter

Counter

Indicates the number of times the event has appeared since measurement start.

Start of Frame

Start of Frame

Absolute time at the start of the bus event.

Prio

Prio

Message Priority.

Specific 10

TelLen

Length of user data in bytes

Specific 11

TelID

Specifies the kind of telegram:
0: Single transfer
1: First telegram of a segmented transfer
2: Second..(n-1). telegram of a segmented transfer
3: Last telegram of a segmented transfer
8: MOST High Protocol - user data
9: MOST High Protocol - control data
A,B: MAMAC Packets

Specific 12

PIndex

PIndex is a node-based counter (1 byte) which increments per message sent. All low-level retries of a message have the same PIndex.

Data ASCII

Data ASCII

The data bytes will be interpreted as character code. This representation is suitable, for example, if texts are being transmitted in a transport protocol.

Diff Time (Ref. Event)

Diff Time (Ref. Event)

Time difference between the current event and the selected reference event.
This column contains only data in chronological mode.

Date and Time

Date and Time

Displays the absolute time stamp (Date and Time) for the corresponding message. In offline mode this corresponds to the time the message was logged.

You can specify the display format (Date and Time or Time) in the Options dialog.

Logging files generated with CANoe always contain date and time of measurement start in the Header.
If a logging file does not contain this information, then date and time of the offline measurement start are displayed in this column.

Diff Time (last Occ.)

Diff Time (last Occ.)

Time difference between the current event and the previous event of the same type. This column displays the period of time since the last reception of a message, system variable,...
In fixed mode, the column has the same meaning as DiffTime.

The Trace Window shows the messages in a tree view. In overwriting mode or when the Trace Window is paused the parameter names and values can be shown on supplemental lines on which messages are expanded.

Parameters are only interpreted, when the the TelID is 0 or 1. Only then the concerned frame contains the bytes corresponding to the signal description in the database.  If the TelID is 2 or 3 the messages are part of a segmented transmission. A disassembly of the complete message is performed when the AMS is active. Messages with other TelIds are part of protocols for which no disassembling of parameters is currently provided.

Note

If you activate the CANoe/CANalyzer’s AMS Service the assembled message is also shown. Clicking the message expands it so that all parameters are visible.

The AMS Service is activated in the Network Hardware Configuration dialog.
