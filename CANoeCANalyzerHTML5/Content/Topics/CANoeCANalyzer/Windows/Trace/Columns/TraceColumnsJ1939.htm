Columns for J1939

Context Menu Path: Configuration|Column Configuration|J1939

Columns for J1939

Field

Title

Function

Time

Time

Absolute time since measurement start, or relative time to the ongoing event

Channel

Chn

CAN channel: number of the channel on which the message was sent/received

Identifier

ID

CAN identifier of the message

Name

Name

Symbolic name of the parameter group

ID/Name

PGN/Name

Depending on the global setting, either the symbolic name or the parameter group number

Dir

Dir
  • Tx: sent parameter group
  • Rx: received parameter group
  • TxRq: send command

DLC

DLC

Length of the data field

Data

Data

Data in decimal or hexadecimal format

Attribute

Attr

Additional attributes

  • WU: Wake-Up
  • TE = Transceiver Error
  • XX = Transceiver Error and Wake-up

Source

Src

Source address of the parameter group

Destination

Dest

Destination address of the parameter group

Frame Duration

Frame Duration

Duration of the complete event in milliseconds and bit times

It also contains the Inter-Frame-Space (3 bit times). This value is only displayed if the Start-of-Frame time stamp is available – see hint for Start of Frame.

Type

Event Type

The text CAN Frame or CAN FD Frame is displayed, depending on the protocol with which the message is sent.

Specific 6

Bus Idle

Time between End-Of-Frame of previous bus event and Start-Of-Frame of current bus event in milliseconds

Thereby it is the Bus Idle duration between the current and the preceding bus event. Note, that the End-Of-Frame time stamp includes also the 3 bits of the Inter-Frame-Space. This value is only displayed if the Start-of-Frame time stamp is available – see hint for Start of Frame.

Specific 7

Bus Busy

Accumulated time from the end of the last Bus Idle phase up to the end of the current bus event in milliseconds

Thereby this is the duration of the last not interrupted (by bus idle) events burst up to the end of the current bus event. This value is only displayed if the Start-of-Frame time stamp is available – see hint for Start of Frame.

d:hh:mm:ss

d:hh:mm:ss

Absolute time since measurement start in d:hh:mm:ss.000 format (24h)

Diff Time

Diff Time

Time relative to the previous event.

In chronological mode, the column displays the time difference to the visible, previous row.
This means that the value depends on the sorting and the filter settings.
In the fixed mode, DiffTime displays the difference to the previous-event.

Send Node

Send Node

Notation of the node that sends the parameter group

The notation depends on the settings for monitoring the dynamic address changing, configurable in the Options dialog.

If the address monitoring is active the notation is derived as follows.

  • If a node with the appropriate J1939 device name or address is defined in the database, the node name is used as notation.
  • If an Address Claiming parameter group is received, the notation is derived by using parts of the J1939 device name.
  • Otherwise the notation is derived by ECU plus address, e.g. ECU 5.

If the address monitoring is not active the notation is derived as follows.

  • The database is searched for a node with the appropriate NmStationAddress node attribute by using the source address. Its node name is used as notation then.

Counter

Counter

Indicates the number of times the event has appeared since measurement start

Start of Frame

Start of Frame

Time stamp at the start of the bus event

Note

For some CAN hardware this value has to be calculated, otherwise no value is displayed. The Start-of-Frame calculation can be activated in the Options dialog (Bus Systems/Protocols|Bus Systems).

PGN

PGN

Parameter group number of the parameter group

Data Page

Data Page

Data Page of the parameter group

Prio

Prio

Priority of the parameter group

Data ASCII

Data ASCII

Data are displayed in ASCII format

J1939 Interpretation

J1939 Interpretation

Interpretation of J1939 parameter groups

  • Transport protocol
  • Address Claiming
  • Request
  • Acknowledgment
  • Diagnostic Memory Access (DM14, DM15)

For option ISO11783 the following parameter groups are interpreted:

  • Extended transport protocol
  • Virtual Terminal
  • Process data
  • File server

J1939 Violation

J1939 Violation

Output of violations of the J1939 specification (see J1939 checks)

Diff Time (Ref. Event)

Diff Time (Ref. Event)

Time difference between the current event and the selected reference event

Note

Data are available in chronological mode only.

Date and Time

Date and Time

Absolute time stamp (date and time) for the corresponding message, in Offline mode this corresponds to the time the message was logged.

You can specify the display format (Date and Time or Time) in the Options dialog.

Logging files generated with CANoe always contain date and time of measurement start in the Header.
If a logging file does not contain this information, then date and time of the offline measurement start are displayed in this column.

Receive Node

Receive Node

Notation of the node that received a parameter group

The notation is derived as described in column Send Node. For global parameter groups this column is blank.

J1939 Sequence Diagram

J1939 Sequence Diagram

Representation of message flow based on the type of the Sequence Diagram:

  • <-130->: A node with address 130 sends a broadcast message.
  • 38->129: A node with address 38 sends a message to a node with address 129.
  • 38<-130: A node with address 130 sends a message to a node with address 38.

To receive a correct representation, you should set a fixed width font, e.g. Courier New in the Trace Window involved.

Diff Time (last Occ.)

Diff Time (last Occ.)

Time difference between the current event and the previous event of the same type. This column displays the period of time since the last reception of a message, system variable,...
In fixed mode, the column has the same meaning as DiffTime.

Signals of an Expanded Parameter Group

The line of a parameter group can be expanded. The first line contains information about the parameter group, e.g. CAN ID, priority, addresses. Afterward the signals are listed, that are defined in the database for this parameter group.

For signals with defined SigType attribute, the signal value is interpreted. For example, string signals can be displayed.
