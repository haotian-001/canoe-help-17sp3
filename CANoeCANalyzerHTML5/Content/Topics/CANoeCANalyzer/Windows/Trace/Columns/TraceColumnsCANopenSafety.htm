Trace Window: CANopen Safety

Context Menu Path: Configuration|Column Configuration|CANopen Safety

To select CANopen Safety specific columns select CANopen Safety in the list box Columns layout. It provides additional columns besides the CANopen specific columns.

Error

In addition to the CANopen specific error types a V (for Violation) can be displayed here. The exact cause can be found in the column Interpretation. If an error cannot be ruled out a question mark can be displayed.

Interpretation

The columns displays the frame that belongs to the SRDO, including the ID of the corresponding frame in brackets. Protocol violations are logged too.

Output

Description

SCT expired

Exceeding the Safeguard Cycle Time

SRVT expired

Exceeding the Safety Relevant Validation Time

Frame order error

Violations of the order rule: it is monitored for each SRDO whether the first and second transmission as well as two succeeding transmission take place in the correct order. If this error occurs no more errors are output.

Inconsistent DLC

The data length code is inconsistent. Data might be incorrect too but due to the higher priority of the DLC error only the DLC error is output.

Inconsistent Data

An error was detected during bit-wise inversion.

Refresh Time

This column displays the time span between the both first frames of succeeding SRDOs of one ID (value range 0..65353 ms).

Time (SRVT)

This column displays the time span between the both frames of one SRDO (value range 0..255 ms).

Time (SCT)

This column displays the time span between the second frames of succeeding SRDOs of one ID (value range 0..65353 ms).

Output

Description

?

The SRDO cannot be verified due to no reference data as DCF files. The label is only available for safety identifier (index range 0x101..0x180).

!

The SRDO is not observed because the available DCF files contain different definitions of the SRDO.
