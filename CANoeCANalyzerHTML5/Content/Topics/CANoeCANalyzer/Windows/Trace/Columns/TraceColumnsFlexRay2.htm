FlexRay Type Column

 

The following events are displayed in the Type column:

Event

Description

Data Frame

Indicates a FlexRay frame with valid data.

Null Frame

Indicates a FlexRay Null Frame with invalid data. The null indicator bit in the frame header is 0.

Frame Error

Indicates erroneous frame receipt. The received frame is damaged or is outside its transmission time window. For more detailed information on the error, see the Frame State (Ab.) and Frame State columns.

Raw Frame

Indicates that a frame was received, not a PDU!
These events are only displayed with certain FIBEX databases with appropriate PDU information.

PDU

This is an interpretation of a PDU, not a frame.
These events are only displayed with certain FIBEX databases with appropriate PDU information.

StartCycleEvent

The time stamp of this event indicates the beginning of a cycle.

Status

This indicates a notification of a POC (Protocol Operation Control) state change of the FlexRay Communication Controller.

Symbol

A symbol was received. This can be one of the following symbols: wake-up, CAS or MTS.

Error Event

Indicates a FlexRay bus error event.

The table below contains possible error events and their meanings:

Error Event Type

Meaning

Driver receive queue overflow

The driver is supplying more frames than the application can retrieve.

In this case frames may be lost on receipt.

FlexCard Overflow Error

Card buffer overflow

More frames are being received than the application can retrieve.

In this case the measurement must be terminated and restarted.

CC state change

Occurs when the CC state changes.  

Possible display:

  • CONFIG state entered
  • NORMAL ACTIVE state entered
  • NORMAL PASSIVE state entered
  • HALT state entered
  • READY state entered
  • STARTUP state entered
  • WAKEUP state entered

POC Active

Corresponds to vPOC!ErrorMode.

The communication controller has assumed the corresponding state.

POC Passive

Corresponds to vPOC!ErrorMode.

The communication controller has assumed the corresponding state.

POC Comm. Halt

Corresponds to vPOC!ErrorMode.

The communication controller has assumed the corresponding state.

Sync. Frames below minimum

Not enough sync frames have been received for the clock synchronization of FlexRay.

The number of sync frames is specified separately for channels A and B and the last even and uneven cycle.

Sync. Frames overflow

Too many sync frames have been received for the clock synchronization of FlexRay.

The number of sync frames is specified separately for channels A and B and the last even and uneven cycle.

Clock Correction Failure

The following causes (or combinations thereof) are possible:

  • Missing offset correction
  • Missing rate correction
  • Offset correction limit reached
  • Rate correction limit reached

As well as the cause of the error, the number of received sync frames is also displayed.

NIT Failure

This is generated if E-Ray reports one of the following errors:

  • SENA Syntax Error during NIT Channel A (vSS!SyntaxErrorA)
  • SBNA Slot Boundary Violation during NIT Channel A (vSS!BViolationA)
  • SENB Syntax Error during NIT Channel B (vSS!SyntaxErrorB)
  • SBNB Slot Boundary Violation during NIT Channel B (vSS!BViolationB)

CC Error

Eray error register content  

The following values or combinations of values can be set:

  • PERR parity error
  • IIBA illegal input buffer access
  • IOBA illegal output buffer access
  • MHF message handler constraints flag
  • EDA error detected on channel A
  • LTVA latest transmit violation channel A
  • TABA transmission across boundary channel A
  • EDB error detected on channel B
  • LTVB latest transmit violation channel B
  • TABB transmission across boundary channel B

One or more of the listed abbreviations is displayed in the Trace Window.

Example: CC error (EDA )
