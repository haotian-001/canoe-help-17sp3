Protocol time

Frame Duration

Duration of the complete event in microseconds (refers to the physical length of the packet).

If you use Vector hardware for sending and receiving, the frame duration

  • of the Rx packet is measured by the hardware.
  • of the Tx packet is calculated based on its length and the underlying speed of the connection.

Minimal discrepancies are possible in this case for identical frames. These depend on the transmission speed and have the following orders of magnitude:

  • 0.008 microseconds at 1000 Mbits
  • 0.040 microseconds at 100 Mbits
  • 0.400 microseconds at 10 Mbits

Note

The transmission time is provided by the Ethernet interface. Therefore, the Frame Duration is only available in online mode with real bus working mode.

In offline mode the transmission time depends on the CANoe version that has been used to create the logging file. In logging files created by CANoe 17 the Frame Duration will always be available. In older CANoe versions you have to enable the Ethernet Extended Logging in the Options dialog.

Date and Time

Date and Time

Absolute time stamp (date and time) for the corresponding message, in Offline mode this corresponds to the time the message was logged.

You can specify the display format (Date and Time or Time) in the Options dialog.

Logging files generated with CANoe always contain date and time of measurement start in the Header.
If a logging file does not contain this information, then date and time of the offline measurement start are displayed in this column.
