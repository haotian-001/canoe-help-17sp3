Columns for FlexRay

Context Menu Path: Configuration|Column Configuration|FlexRay

Columns for FlexRay

Field

Title

Function

Time

Time

Absolute time from the measurement start, or relative to the previous event.

Channel

Channel

FlexRay channel A, B or AB and the channel number on which the message was sent/received.

Identifier

Slot

Identifier/Slot ID of the message.

Name

Name

Symbolic name of the message.

Dir

Dir

Tx = Transmit message,
Rx = Receive message,
TXRQ = Transmit request.

DLC

Length

Length of the data field.

Data

Payload

The data in decimal or hexadecimal representation.

Original Time

Original Time

Original time stamp that is generated by the interface, with an offset of 0 at the start of measurement.

Type

Type

Type of message:
Data Frame, Null Frame, Frame Error, Raw Frame, PDU, StartCycleEvent, Status, Symbol, HWSync, Error Event

Error

Frame State (Ab.)

Interpretation of the Frame State.
The abbreviations used here depend on the FlexRay hardware in use; they are explained below.

Specific 0

Cycle

Current cycle.

Specific 1

Frame State

Status field of the message.

Specific 2

Schedule

dynamic/static

Specific 4

Sync Flag

Message is used for time synchronization in the network.

Specific 7

PP Flag

Payload Preamble Bit.

Specific 8

Header CRC

CRC field in the message header.

Specific 9

Sync Time

When using multiple interfaces, time will be synchronized, so that the events are comparable. The modified time stamp is the sync time stamp.

d:hh:mm:ss

d:hh:mm:ss

Absolute time from the measurement start in d:hh:mm:ss.000 format.

Diff Time

Diff Time

Time relative to the previous event.

In chronological mode, the column displays the time difference to the visible, previous row.
This means that the value depends on the sorting and the filter settings.
In the fixed mode, DiffTime displays the difference to the previous-event.

Bustype

Bustype

Specification of the bus system (CAN, LIN, MOST, FlexRay,...).

Send Node

Send Node

Send node of the massage of the database.

Bus

Bus

Bus name.

Database

Database

Database in which the corresponding message is defined.

Counter

Counter

Indicates the number of times the event has appeared since measurement start.

Specific 10

Frame CRC

CRC field of the complete message (trailer).

Specific 11

Spy Flag

Message has been received in asynchronous mode by the bus scan module (Spy).

Specific 12

Startup Flag

Message is used for network start.

Specific 13

Update Bit

If the message is an interpreted PDU (depends on FIBEX version) this field represents the update bit of the PDU.

Specific 14

Reserved Bit

Reserved Bit.

Specific 15

Category

PDU- or Frame-Type.

Data ASCII

Data ASCII

The data bytes will be interpreted as character code.

This representation is suitable, for example, if texts are being transmitted in a transport protocol.

Diff Time (Ref. Event)

Diff Time (Ref. Event)

Time difference between the current event and the selected reference event.
This column contains only data in chronological mode.

Date and Time

Date and Time

Displays the absolute time stamp (Date and Time) for the corresponding message.

In offline mode this corresponds to the time the message was logged.

You can specify the display format (Date and Time or Time) in the Options dialog.

Logging files generated with CANoe always contain date and time of measurement start in the Header.
If a logging file does not contain this information, then date and time of the offline measurement start are displayed in this column.

XCP/CCP Command

XCP/CCP Command

XCP/CCP command names.

Responses on a command are displayed

  • for success case like this Ok:command name
  • in error case like this ERR
    (with CCP currently like this Err:command name)

See Protocol Observer.

XCP/CCP Parameter

XCP/CCP Parameter

Parameter of XCP/CCP commands or the response are displayed as Sub-Lines and in a separated column.

If you select a parameter, the corresponding bytes in the Data column are highlighted. See Protocol-Observer.

Diff Time (last Occ.)

Diff Time (last Occ.)

Time difference between the current event and the previous event of the same type. This column displays the period of time since the last reception of a message, system variable,...
In fixed mode, the column has the same meaning as DiffTime.
