Trace Window: Columns for CANopen

Context Menu Path: Configuration|Column Configuration|CANopen

To select CANopen specific columns select CANopen in the list box Columns layout. The columns defined in the standard column set of CANopen are displayed in the Trace Window. The additional columns have the following meaning:

Transfer Data

The data transmitted by the current service is displayed in the set numeric basis (hex or dec). In contrast to the standard column Data, these are only the pure reference data here. With an expedited SDO, for example, only 1-4 bytes are displayed instead of the 8-byte raw data.

Generally this data is output as raw values, with the following exceptions:

Service

Meaning

Abort SDO

Error code from abort message as 32 bit value

Guard request

Output of the identifier RTR

Guard response or heartbeat

The symbols T0 and T1 display the current value of the toggle bit.

The device state is specified in plain text.

Expedited SDO

For the transmission of mapping entries in a SDO, the mapped object is displayed. This makes the evaluation of mapping changes easier.

Example:
08 01 00 62 [6200.01] [1600.01] Initiate...

Error

This column contains notes about errors. Error types are here:

Abbreviation

Meaning

A

SDO Abort

E

Emergency

G

Guard-Error: unknown status in guard response or heartbeat

L

Error in a LSS service

P

General protocol error such as an incorrect DLC

R

Use of a reserved bit or byte

S

Error in the SDO protocol

This column is helpful for quick detection of error situations since with very many messages, you only have to check this column.

Interpretation

In this column, the services NMT, SDO, LSS, EMCY, and sometimes heartbeat are explained in plain text. Displayed is the name of the service and the associated service parameters, insofar as these are not already displayed in the column Transfer Data. Error codes of the emergency message and the SDO abort whose values are specified in the specification /2/ are output as plain text messages.

If the transmitted characters contain at least one readable character, then the data content is also specified as ASCII string. Characters that are not readable are displayed as a dot.

In the Interpretation column, it is also possible to display the object name of an object written or read by SDO.
