Analysis Filters

 

Note

  • The selected filter settings are stored in the configuration. If filter settings are changed, the corresponding events are loaded into the analysis area of the Trace Window.
  • Pinned events are not considered by Trace filters. They are always displayed at the top in the Trace Window.
  • Configured column filters can be taken over in user-defined column layouts.
  • Note that the filters have different performance levels. Therefore, if possible, always work with the filter with the highest performance.
    In terms of performance, the filters have the following ranking:
  • Predefined filters
  • Analysis filter
  • Sequence filter
  • Column filter

You can open the analysis filters in the Trace Explorer via the icon in the toolbar of the Trace Window.

The analysis filters only affect the view of events displayed in the Trace Window. No events are deleted from the data stream.

Filter Groups

The analysis filters can be divided into different filter groups. Accordingly, you can easily switch back and forth between different trace outputs.
You can add, rename and delete filter groups via the context menu of the Trace Explorer.

Note

The individual filter groups exist in parallel. This means they can cancel each other out, depending on their definition.

Filter Action Principle

  • A filter group that is not activated is not taken into account. All events are displayed in the Trace Window.
  • A Stop or Pass Filter is generally not taken into account when it is deactivated.
    This means an event passes a deactivated filter and is displayed in the Trace Window.
  • An activated filter group in which no Stop or Pass Filter is activated allows all events to pass. All events are displayed in the Trace Window.
  • An activated filter group with activated Pass Filter allows only the events that are defined in the Pass Filter to pass In the example only Gateway_1 is displayed in the Trace Window.

    Note: If you have defined the above-shown filter group 1 for filter group 2, all events are displayed nevertheless.
  • An activated filter group with activated Stop Filter allows all events up to the events that are defined in the Stop Filter to pass.
    In the example Door_r is not displayed in the Trace Window.

    Note: If you have defined the above-shown filter group 1 for filter group 2, all events are displayed nevertheless.
  • Stop and Pass Filters act sequentially within a filter group. This means an event passes the filter group only when it passes both the Stop Filter and the Pass Filter.
    In the example Gateway_1 is selected as the Stop and Pass Filter. An event is no longer shown in the Trace Window.

    The Stop Filter does not allow Gateway_1 to pass. All other events would actually be displayed in the Trace Window.
    The Pass Filter allows only Gateway_1 to pass. Only Gateway_1 would actually be displayed in the Trace Window.
    Because Gateway_1 was also defined as the Stop Filter, an event is no longer displayed.

You can configure Stop and Pass Filters via

  • the context menu of the Trace Explorer
  • drag and drop from the Trace Output Window
  • drag and drop from the Symbol Explorer

You can

  • add, remove and edit filter conditions

If you add a signal, environment variable or system variable as a filter condition, you can configure value filters for it. This enables you, for example, to filter a message only if its assigned signal has a defined filter value.
If you add a signal as filter condition, a value filter is set automatically.
Value filters of environment variables or system variables can be of type integer, float, or string.

Value Filter

Title: Value of <symbol name (signal, environment or system variable)>.
Logical operator: With the logical operator you can link the conditions. Possible values are AND and OR.
Number format: Here you can choose between HEX and DEC.
You can configure a maximum of 10 conditions. The interpretation depends on the » symbol type (e.g. signal) and cannot be changed.
With you can add a new line. With you can delete a selected line.

The following numerical relations are possible:

  • equals
  • is not equal
  • is greater than
  • is greater than or equal to
  • is less than
  • is less than or equal to
  • Interval (the endpoints are included)

Following relations are possible as strings:

  • equals
  • is not equal
  • contains
  • does not contain

The following wildcards are supported:

  • * stands for an arbitrary number of characters.
    dow*Do finds e.g. WindowLeftDown and WindowRightDown
  • ? stands for exactly one character.
    dow????Do finds e.g. WindowLeftDown
  • <empty> stands for an empty string.

  • When adding this filter a dialog open where you can select the direction (Rx/Tx).
    You can change the settings later in the same dialog (context menu).
  • add bus/protocol specific filters (see below)

Symbols and Their Meaning

Symbol

Meaning

Pass filter: Here you can specify the events that should be displayed in the Trace Window.

Defined value filter.

Show Changed Events Only

If you have enabled this option, an event is only displayed if its value has changed compared to its predecessor. If you disable this option, all events will be displayed again regardless of whether its value has changed or not.

Note

This option is only available with activated chronological mode ( in the toolbar).
