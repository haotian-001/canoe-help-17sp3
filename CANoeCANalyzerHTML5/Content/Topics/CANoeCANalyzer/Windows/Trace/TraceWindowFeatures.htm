Function Overview

 

The Trace Window offers the following functions:

Window Configuration
  • You have numerous options for configuring the Trace Window. In addition to the basic settings you can also configure the columns.
  • With the splitter function of the Trace Output Window, you can analyze two different periods of time in parallel.
  • Signal values can be displayed symbolically.
  • With the Trace Highlighting function, you can highlight important events and messages in the Trace Window.
  • With the fading out of unmodified data, you can set that rare events or unchanged bytes will be fade out slowly.
    In the global configuration dialog you can activate/deactivate this fading for all new Trace Windows.
  • Configure the event highlighting for MOST by setting the highlight colors for events and messages to make them easier to find.
  • In addition you can set the Event Filter for MOST messages and events.
  • Messages with different multiplexors can be displayed in fixed mode position as separate rows. This function enhances the clarity when working with multiplexor signals.
    In the global configuration dialog you can set this functionality for CAN based protocols and FlexRay.
  • You can drag and drop signals and messages from the Trace Window into a Graphics Window or another Trace Window. This requires that the contents of the Trace Window remain unchanged. You can use this function, for example, when the measurement is completed or in Pause mode. When you drag a message to the Graphics Window, all the signals of this message are inserted.
  • The number of decimal places to be displayed for time stamps in the Trace Window can be specified in the global configuration dialog.
  • The display of special signal information (e.g. signal comments) can be activated/deactivated in the global configuration dialog.
  • Since every bus event, bus system or high-layer protocol can output different information, there are multiple columns in the Trace Window containing the corresponding data. Individual columns may remain empty, however, since the corresponding information is not provided in every message.
    Columns available for CAN / CAN FD, J1587, J1939, LIN, MOST, FlexRay, ISO/OSEK TP, Diagnostics, Variables.
  • The Trace Window offers a range of additional views.
  • You can define the order of the signal display.
  • Unsorted Events (Message, Error Frame,…) are displayed with a red exclamation mark: .

Recording and Evaluation
  • The result of the analysis filter is comparable to an output filter that matches a specific message (a message ID). The same applies, by analogy, to the variable filter.
  • Pinned events are displayed at the top of the Trace Window. This makes it easier for you to monitor selected messages, especially when numerous messages and variables are displayed in the Trace Window.
  • A Reference event can be used like a Bookmark in the Trace Window. After you set a reference event the Difference view will always show a comparison of the last selected event with the reference event.
  • Markers make it easy to tag and locate events.
  • If you mark a signal of a message in the Trace Window, the position of the signal is highlighted in the data field. Thus you can easily find out in which range of the data field the signal has been transmitted.
    This indication is only available in pause mode or after measurement stop.
  • You can use the search function to search through the complete buffer of the Trace Window for messages, signals or signal values.
  • The Event Highlighting for MOST uses color highlighting to help you find important events and messages faster.
  • By defining special Event Filters for MOST messages and events you can create "specialized" Trace Windows to simplify analysis.
  • MOST status information on control and AMS messages are displayed in the State column of the Trace Window and can also be called up by means of the CAPL selector MOST_State.
  • You can use the context menu, the keyboard or the toolbar to control the Trace Window functions.
  • You can export the contents of the Trace Window; partially or completely. There are various export formats available so that the exported data can be used for further evaluation in other programs.
  • It is also possible to convert previously exported data to a different format, for example to enable further processing of the same data in different programs.
  • Selected rows in the Trace Window can be copied to the clipboard and then pasted into a Graphics Window, another Trace Window or a different application (as text).
