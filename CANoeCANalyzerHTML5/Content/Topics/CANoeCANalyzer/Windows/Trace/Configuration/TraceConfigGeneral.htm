On this page you can configure basic settings for the Trace Window.

Display Mode

The display mode setting determines how the rows in the Trace Window are to be written.

  • In chronological mode each new row is inserted below the previous row, and when the window is full it scrolls upward.
    This display mode shows the chronological sequence of messages very well. With high busloads, however, this display mode may cause performance problems since it requires a lot of processing.
  • In fixed Position mode each message (type) is assigned to a specific line the first time it occurs, and all further messages of the same type are written to the same line.

Car2x

In the fixed mode the messages are displayed using special criteria. The message data are updated only if the following data are available and identical (the protocol designator respectively the token designator is given in parenthesis):

  • WLAN (wlan)
  • Frame Control - Low Byte (frameControl)
  • MAC address of the receiver (address1)
  • MAC address of the sender (address2)
  • Ethernet (eth)
  • ethernet type (type)
  • GeoNetworking: Common Header (geo_ch)
  • protocol of the next layer (nextHeader)
  • header type (ht)
  • header subtype (hst)
  • GeoNetworking: Extended Header (geo_eh)
  • GeoNetworking address of the Long Position Vector (lpvGNAddr)
  • GeoNetworking address of the Short Position Vector (spvGNAddr)
  • Basic Transport Protocol (btp)
  • sender port (source)
  • destination port (destination)
  • WAVE Short Message Protocol v3 Network Header (wsmp_n)
  • with loaded database: DSRC Message ID of the J2735 application layer
  • PS ID
  • Transport Protocol ID

Time

  • If you select absolute as the time setting, the time display will indicate the specific point in time at which the message was received (beginning from the measurement start).
  • The relative time setting, on the other hand, indicates the time in relation to the preceding message.
    In chronological mode this is the message received directly before the current message, whereas in fixed position mode the relative time is displayed in relation to the previous message of the same type.

Signal Sorting

Here you can set, if the signals should be displayed

  • Sorted corresponding to the order in the database.
  • Sorted by their start bit in ascending order.
  • Sorted by their names in ascending order.

The sorting is valid for the current Trace Window, and is available for standard CAN, LIN and FlexRay signals.

Fading of Unmodified Data

The fading enhances the clarity in the Trace Window, the visibility of frequent events and changed data will be improved.

With the Fade out old events option, you can activate/deactivate slowly fading of rare events in the fixed position mode ( in the toolbar). To enhance the clarity, you can delete faded events from the Trace Output Window via in the toolbar.

With the Fade out bytes without changes option, you can activate/deactivate the slowly fading of data bytes without value changes.

Note

In the Options dialog you can set, if the fading should be activated/deactivated for all new Trace Windows generally.
