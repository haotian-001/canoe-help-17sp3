MOST Highlighting - Message List

Context Menu Path: Configuration|View|MOST Highlighting

For each message to be highlighted, an entry must be added to the message list. The description of a message is contained in one line.

The different buttons can be used to add and delete messages and to resort the messages in the list.

Structure of the Message List

Column

Description

Active

In this column you can activate or deactivate single messages

Through the short cut menu you can activate or deactivate all messages in the message list

Font and background color

In these two columns you can select the colors of the messages, in which they are displayed in the Trace Window. A left mouse click on the respective cell opens a dialog, in which the color for font and background respectively can be selected.

Address:

Here you can define the source and destination address of the respective message.

SA

Source address

DA

Destination address

Message/Packet:

The [New] button can be used to add a new message via the XML selection dialog. Application packets are those packets, which are sent on the asynchronous channel vie the MOST High Protocol (MHP). This is only possible after an XML function catalog has been added to the configuration. In case the configuration contains an XML function catalog the input of FBlock, Function and OpType can be done via a choice list.

FBlock

Function block, defines an application (e.g. booster or CD player)

Inst

Instance of the function block.

Function

Function within the given FBlock.

OpType

Operation within a specified function.

TelID

This field only affects individual frames within the messages. TelID describes the protocol as well as the message type within the protocol to which the message belongs. The following applies:

  • 0: Normal CMS message
  • 1: First AMS telegram
  • 2: Second .. (n-1) AMS telegram
  • 3: Last AMS telegram
  • 8: MOST High user data telegram
  • 9: MOST High control data telegram
  • CMS (0..3)
  • MHP (8..9)
  • CMS

Note

Since the MOST High Protocol can be used for transmission on the control as well as on the asynchronous channel, all messages/packtes with TelID 0x08 and 0x09 respectively on both channels will be considered for highlighting.

TelLen

Describes the number of payload data bytes used. It only affects the individual frames or telegrams. It is automatically adapted to the data that is entered, insofar as the maximum number of payload data bytes for the corresponding telegram (see TelID) is not exceeded. The following applies:

  • A maximum of 12 payload data bytes for AMS telegrams; 11 payload data bytes maximum for broadcast/groupcast addressing
  • A maximum of 11 payload data bytes for MHP telegrams on the control channel
  • A maximum of 1006 payload data bytes for MHP telegrams on the asynchronous channel

If the maximum number of payload data bytes is exceeded, the individual telegrams are:

  • Blocked by an pass filter
  • Allowed by a stop filter

Changes made to this field affect the "Data" field, unless a wildcard ("*") has been entered there:

  • If fewer bytes are entered there, the remaining bytes are hidden
  • If more bytes are entered there, the superfluous bytes are deleted

Ack

Only for messages on the control channel.

Three states that can be declared here:

  • active: Ack bit must be set in the message
  • inactive: Ack bit must not be set in the message
  • "don't care": Ack bit is ignored

Nack

Only for messages on the control channel.

Three states that can be declared here:

  • active: Nack bit must be set in the message
  • inactive: Nack bit must not be set in the message
  • "don't care": Nack bit is ignored

Channel

MOST channel on which the message/packet was sent or received.

 

If the message is described in an embedded XML function catalog, the symbolic names are displayed in the fields. The corresponding function catalog has to be embedded in the configuration. If no description is available for parts of the message, the parts that are not included in the function catalog are displayed numerically and color shaded.

Example

The FBlock in question is described in the function catalog, but not the selected function. The name of the FBlock is shown, but the information in the Function field is displayed numerically and color shaded.

Direction:

Sets the filter direction.

Rx

Received messages / packets are included in the filtering process.

Tx

Sent (transmitted) messages / packets are included in the filtering process.

Spy

Spy messages / packets are included in the filtering process.

Combiner

Along with the individual telegrams of an application message, the combined messages of the Application Message Service (AMS) and the MOST High Protocol (MHP) can also be filtered.

Both message types can be activated/deactivated by clicking in the relevant cell:

  • activated means that this message type is also included in the filtering process
  • deactivated means that this message type is not included in the filtering process

Note

Since the MOST High Protocol can be used for transmission on the control as well as on the asynchronous channel, all combined events (MHPBlock, MHPPacket and MHPError) on both channels will be considered for highlighting.

Loading and Saving Message List

You can store the message list in a file (.mhm), in order to load in another Trace Window. Note, that only the content of the message list is stored, column widths or similar are not considered.

Loading and saving of the message list is done via the context menu of the message list. In case there are already messages in the message list, you can decide whether

  • to discard the existing messages, or
  • to append the new messages to the message list

General

Not only specific values, but also "wildcards" or masking patterns can be used for the individual parameters of a message. Wildcards are specified by means of an asterisk "*" and indicate that the given parameter may have any value. A masking pattern is specified by means of an underscore "_". Masking is carried out at nibble level, which means you have to specify a "_" for each nibble.

Example

Masking of the FBlock:    "0_"  (include all function blocks from 0x00..0x0F)

Masking of an address: "3__" (include all group addresses)

Clicking on the last, gray line in the message list adds a new application message to the list. In this case, all fields are filled with a "*" (wildcard). The fields of the new message can be filled even without an embedded function catalog.

Note

The user-defined event highlighting has a higher priority than the default event highlighting. That means, that an event, which is configured for the user defines as well as for the default event highlighting, will be highlighted in the colors of the user-defined event highlighting.

Inside the message list the first message has the highest, the last message the lowest priority.

Example for highlighting the combiner events and the respective atoms

Add a new message to the message list and set the desired values for FBlock .. OpType. Duplicate the message and remove the checks for "AMS" and "MHP" from the first message. Set the value of TelID from the second message to "all others". Finally choose the desired colors for font and background for both messages.

Context Menu of the Message List

The context menu of message list is displayed if you execute a right mouse click in on of the cells of the message list.
The following menu commands are available:

Read configuration from file…
Opens a file dialog to select the file (.mhm) from which the configuration (message list) should be read.

Write configuration from file…
Opens a file dialog to select the file (.mhm) to which the configuration (message list) should be written.

Activate selected message(s)
Activates the selected message(s) in the message list.

Deactivate selected message(s)
Deactivates the selected message(s) in the message list.

Activate all messages
Activates all messages in the message list.

Deactivate all messages
Deactivates all messages in the message list.

Set font color…
Opens a color selection dialog to select the desired font color for the message to be displayed in the Trace Window.
If multiple lines are selected, the color is set for all selected lines (messages).

Set background color…
Opens a color selection dialog to select the desired background color for the message to be displayed in the Trace Window.
If multiple lines are selected, the color is set for all selected lines (messages).
