MOST Event Highlighting

Context Menu Path: Configuration|View|MOST Highlighting

In the MOST Highlighting dialog you can set colors for the event highlighting MOST.

The user-defined event highlighting allows the coloring of specific MOST messages in the Trace Window. This can be done for each Trace Window individually.

Advantages

  • MOST messages can clearly be highlighted in the Trace Window
  • MOST messages can easily be found in the Trace Window

Note

  • The event highlighting has the following priorities:
  1. CAPL (highest priority)
  2. user-defined event highlighting
  3. default event highlighting (lowest priority)

That means, that an event, which is highlighted in CAPL and configured for the user defines as well as for the default event highlighting, will be highlighted in the colors defined in CAPL.

  • It is also possible to highlight one or more MOST message directly via the context menu of the Trace Window.

With [Trace Highlighting...] you open the Options dialog. Here you can configure the default colors for MOST events.

Functions and Controlling

Buttons

The following buttons are provided to assist you in compiling the message list:

  • [New]
    Inserts a new message in the message list.
    A left mouse click on the button opens the MOST Symbol Explorer, which enables messages to be selected directly from an integrated XML function catalog.
  • [Clone]
    Duplicates the marked message. The duplicate is added at the end of the message log.
    This function is very useful if a message of the same type is to be filtered with different data bytes.
  • [Delete]
    Deletes the marked message from the message list.
  • [Up]
    Moves the message marked in the message list one row up.
  • [Down]
    Moves the message marked in the message list one row down.
  • [Layout]
    Arranges the column widths in the message list automatically to ensure the best possible visibility for the entries.
