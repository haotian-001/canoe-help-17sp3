On this page you can make the following settings:

Font and Decimal Places

You can configure the fonts and the decimal points in the Options dialog.

Standard Highlighting

To configure the standard event highlighting you are forwarded to the Options dialog.

Specific events can be highlighted depending on different categories to find them in the Trace Window more quickly.

Note

The highlighting of events is generally available for all options. Special features for specific options are listed here.

A429

For A429 you can set colors for the following events:

Event

Description

System Address Label (SAL)

The message is described in a database as System Address Label (attribute A429Protocol != “BC”).

Receive Errors / Error Events

Rx or Tx error

Gap Violation

The ARINC Word causes a gap violation.

Signal Value out of Min-/Max-Range

The signal value is out of the minimum and maximum values defined in the database. Signal values are visible in the fixed mode of the Trace Window only.

AFDX

For AFDX you can set colors for the following event groups:

Group

Priority

Description

Error

 0+

highlighting according to different error categories, highlighting can be set for these error categories independently

Error Category

Description

Invalid MAC Frame

see criteria in Validity of an AFDX frame

Wrong Line

InterfaceID of the MAC address does not fit the Ethernet interface (Line A resp. Line B), on which the packet has been received

Integrity Check (IC)

no AFDX sequence number (SeqNum) or violated sequence order during active integrity check

Redundancy Management (RM)

an expected redundant packet is not received or received to late during active redundancy check

Fragmentation

reassembling of IP fragments to a packet failed because

  • at least one fragment is missing,
  • order is incorrect, or
  • the expected fragment sizes are inconsistent

Higher Layer Protocol

contains all protocol errors of the IP and UDP protocol and erroneous constants referred to ARINC664p7

General

 0

highlighting according to the different kinds of normal frames/packets

Line

 1

highlighting according the Line the frame was send or received

Direction

 2

highlighting according the frame was send or received

Port Type

 3

If there is a database where the frame is described the frame/packet is highlighted according whether it is sent by Service Access Point (SAP) or communication port.

Priority

The priority states, which order is used to determine the message color. There are also priorities within the groups. The priority corresponds with the position in the list, so a higher position means a higher priority.

Example

If both groups with priority 0 (Error and General) are activated, the frame/packet will colored according the Error color group, when it has any error. Otherwise it is a normal frame/packet and will be colored according the General group.

To use Line coloring with coloring of erroneous frames, deactivate the General group. Likewise deactivate the General and Line group to have Direction coloring with Error coloring.

ARINC 825

For A429 you can set colors for the following message events:

Event

Description

Peer-To-Peer (NSC, TMC)

A message with a LCC = {NSC|TMC} was registered. For Data Loading Messages (ARINC 826), a different color is used.

Peer-To-Peer (NSC, TMC) Service Error

With this color a negative acknowledge with Node Services or with Software Data Loading is highlighted.

Periodic Health Message (TMC)

Message with LCC = {TMC}; Function ID = 125; SMT-Bit = 1

ARINC 826 SDL (loader)

Software Data Loading: Message of the Loader; Client FID = 127

ARINC 826 SDL (target)

Software Data Loading: Message of the Target; Server FID = 127

Direct Message Channel (DMC)

Message with LCC = {DMC}

Protocol Error

The following errors are recognized:

  • wrong DLC (with LCC = {NSC|TMC})
  • Fragmentation error (Software Data Loading)
  • CRC error (High Integrity Message)

Zero Sequence Number (High Integrity)

The sequence number of a High-Integrity-Message has the value 0.

Message Level Functional Status

The bit FSB and the DLC are analyzed for LCC = {NOC|EEC}.
