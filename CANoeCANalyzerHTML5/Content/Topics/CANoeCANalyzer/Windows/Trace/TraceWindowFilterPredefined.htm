Trace Window: Predefined Filters

 

Note

  • The selected filter settings are stored in the configuration. If filter settings are changed, the corresponding events are loaded into the analysis area of the Trace Window.
  • Pinned events are not considered by Trace filters. They are always displayed at the top in the Trace Window.
  • Configured column filters can be taken over in user-defined column layouts.
  • Note that the filters have different performance levels. Therefore, if possible, always work with the filter with the highest performance.
    In terms of performance, the filters have the following ranking:
  • Predefined filters
  • Analysis filter
  • Sequence filter
  • Column filter

You can open the predefined filters in the Trace Explorer via the icon in the toolbar of the Trace Window.

Filter Concept

A multi-level filter concept is used for the predefined filters. With a left-click on an icon you open the selection for the filter settings.

Symbol

Meaning

Do not Save Events

Events are not saved in the buffer of the Trace Window and thus cannot be displayed later.

You can use this setting selectively to remove unimportant events from the data flow in order to free up memory space for data of interest.

Hide Events

Events are saved in the buffer of the Trace Window but they are hidden in the Trace Window.

The saved events can be displayed later.

Display Events

Events are saved in the buffer of the Trace Window and displayed in the Trace Window.

The saved events can be hidden later.

Left-click an entry to open a field with further selection options where you can set the filters.

Fields in which a further selection exists are displayed as a link (underlined in blue).

Example

The area of the predefined filters is divided as follows:

Measurement Setup Filters

Here you can navigate directly from the Trace Window to the filters configured in the Measurement Setup (in the Trace branch). The symbol at the beginning of the column indicates, if the filter is active () or not ().

Bus Systems

Here, the following bus systems and their events are listed, depending on the installed option.

  • AFDX events
  • ARINC 429 events
  • AUTOSAR PDU events
  • CAN events
  • Ethernet events
  • FlexRay events
  • J1708 events
  • LIN events
  • MOST events
  • WLAN events

Variables

Here, you can specify whether the current values of assigned system and environment variables should be displayed.
Individual namespaces can be enabled/disabled for the system variables.

Diagnostics

To simplify diagnostics of controllers, it is possible to display and filter specific diagnostic interpretation in a Trace Window. So you can focus on certain controllers during analysis.

Using this check box it is possible set the interpretation for specific diagnostic responses and requests of the configuration.

Note

To display diagnostic interpretation, the check box for display TP information in the Trace Window as well as the TP observer shall be activated.

XCP/CCP Protocol Observer

For CAN and FlexRay you have for each ECU a predefined filter. With Ethernet you find only one Filter for all XCP packages. See further Protocol-Observer.

Observer

Here you can set, if all TP information should be displayed.

  • DoIP packet events
  • FlexRay TP observer
  • Protocol messages
    Display protocol information. 
  • Transported data
    Display transported data of a segmented transmission.
  • Timeouts
    Display timeout information.
  • FSA messages
  • HSFZ messages
  • ISO-TP observer
  • Transported data
    Display of transported data in case of a segmented transmission.
  • Protocol messages
    Display of protocol information, e.g. Single Frame, First Frame, Flow Control Frames.
  • Warnings
    Display of protocol warnings.
  • Information
    Display of further protocol information.
  • J1939 TP observer
  • K-Line observer
  • MOST high observer
  • PDU marshalling events
  • SOME/IP messages

System Messages

  • Test Feature Set
    The check box allows the display of Test Feature Set information, such as the start/stop of test modules/test cases as well as their verdicts.
  • CAPL
    Output, generated by a CAPL program, e.g. using the WriteEx function is displayed in the Trace Window.
  • Internal
    Display of the controller’s status.

Communication Objects

Here, you can select which information of the communication concept should be displayed.

  • Signal/Event
    Communication without status information and methods:
    PDUs, signals, fields, events
  • State
    State information () of PDUs, services and events.
  • Method
    Methods of communication objects and fields.

Distributed Objects

  • Value
    Value changes of data, event and field members.
    The interfaces of distributes objects (DO) can also contain methods, the calls are then also visible in the trace window, but can be switched invisible via the Method property of the predefined filters.
  • State
    State information () of data, event and field members.
  • Method
    Methods of method and fields members.
  • Dynamic
    Dynamic distributed object events (e.g. distributed object created/removed or container changes).
