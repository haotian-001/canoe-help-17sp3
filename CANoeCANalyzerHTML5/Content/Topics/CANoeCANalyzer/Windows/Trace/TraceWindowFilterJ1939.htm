Filter Condition | J1939

Context Menu Path: Add Condition|J1939

With option J1939 you can filter for parameter groups instead for CAN identifier. You can create analysis filters with single parameter groups or use topic-based filter conditions for specific subject areas of the J1939 protocol. A mixed usage is possible too.

Analysis Filter with Single Parameter Groups

Single parameter groups can be added to the analysis filter as follows:

  • with drag and drop from the Trace Window output
  • with drag and drop from the Symbol Explorer
  • via the context menu from the database

If you add a parameter group from the Trace Window output the source and target address (if applicable) are initially considered by the analysis filter. So it is possible to add parameter groups that are not defined in the database.

If you add a parameter group from the Symbol Explorer or the database the source and target address are considered as configured in the database. If an address is configured as fixed value this value is used in the analysis filter. If an address is configured as wildcard this address is initially ignored.

Initially the priority is ignored regardless how the parameter group was added.

After adding the parameter group you can configure it. For this open the configuration dialog via the context menu command Edit condition... of the parameter group. In addition to the CAN standard configuration settings you can specify the source address, target address (if applicable), and priority.

After each change that is made to a sender or destination address, the database is searched to find a message that fits the defined criteria best.

Consideration of Dynamic Address Change

You can define, for both the sender and destination address, whether the dynamic address change should be considered.

  • If the dynamic address change should be ignored, you must define the sender or destination by its address (a number from 0 to 255 (or from 0x0 to 0xFF)).

    Here, the search is always for messages that are sent from the address 0x3.

  • If the dynamic address changes should be considered, you must define the address by the node name from the database. Therefore you have to activate the network management respectively the tracking of dynamic address change. Further information.

    Example

    Here, the search is always for messages that are sent from the InstrumentCluster node. The current address of the node is not considered here.

Topic-based Filter Conditions for specific Subject Areas of the J1939 Protocol

Via the context menu of the analysis filter you can add more filter conditions that are available for the evaluation of specific subject areas of the J1939 protocol.

Independent of the address the following PGNs are filtered, in the header of the PGN as well as in the data portion of an acknowledge (ACK – PGN E800h) or request (RQST – PGN EA00h).

J1939 Network Management

  • CA (FED8h)
  • WSMEM (FE0Ch)
  • WSMSTR (FE0Dh)
  • NAME (9300h)
