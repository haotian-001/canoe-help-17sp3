Filter Condition | CANopen Services

 

When the option CANopen is activated, you can filter for certain CANopen services instead of for individual CAN identifiers.

Note

You can add CANopen analysis filters using the context menu of the Trace Explorer. It is not possible to use drag and drop from the Trace Output Window or Symbol Explorer.

Requirements

The CANopen specific filters are only available under certain conditions:

  • Basically the option CANopen in CANoe has to be active.
  • Additionally, the database of the configuration has to contain certain attributes.

Further information on the topic of activation

Activating the CANopen Functionality

Filter Consistency

Avoid combining CANopen networks with different standards (e.g., CiA 301 or CiA 447). The filter always operates according to the first profile type found in a database and ignores subsequent databases that might possibly define a different profile type.

Correct Sequence for SDO Transfer

The SDO transfer involves a transport protocol that requires certain message sequences. SDO protocol violations lead to the filter may not operate as expected.

Basic Information on Filters

Changing the Parameters of a Filter

With a double click on a created CANopen analysis filter (or via the context menu), the associated configuration dialog opens. There, you specify the filter criteria. You can make the filter more or less selective.

Further information on the topic of analysis

Analysis | Concept

Default Values when Inserting - Caution: Identical Filters are not Possible

If you create a new filter, it is initially assigned the default filter criteria. These default values are selected in such a way that the filter delivers a maximum set of results corresponding to the category.

You can make additional settings to limit the filter and further reduce the set of results.

A filter has a unique signature based on its settings (parameters). You can insert only one filter with an identical signature in the filter list. However, this rule applies only within a filter type (Stop Filter or Pass Filter).

Assuming there is a PDO filter for which the default values were left, you can not use a second PDO filter as long as you have not changed the criteria of the previously used filter.

Basic Parameters of a Filter

Each filter contains basic parameters. Those parameters are present in all filters as long as they are relevant to the type of the filter.

Name

Name of the filter as it will be displayed in the list of analysis filters.
It is technically possible to use filters with identical names. To avoid problems, you should change the proposed default name if you are working with several filters of the same category. A summary of the most important filter settings is available in a short text. These tooltips appear when you touch with the mouse pointer. In this way, you can distinguish identically named filters, if necessary, without having to open the filter criteria in each case.

Channel

The CAN channel to filter by.
For systems with multiple CAN channels, you can limit the analysis to a certain CAN channel. You can choose an individual channel or all channels by selecting CAN all. A combination of multiple channels can be realized using multiple filters.

Node ID (not for SDO)

The default setting * (wildcard) corresponds to all nodes (1-127 or 1-16), depending on which profile (CiA301, CiA447) is defined in the databases. The selection list contains all nodes to which the CO_DCF- attribute is assigned in the database and the corresponding DCF file could be loaded successfully. The CAN identifier defined for this node is read from the corresponding COB-ID objects described in the node's DCF file.

In addition, you can enter a node ID of an unlisted node into the field. In this case, the filter operates according to the CiA standard used (database attribute: CANopenApplProfile) based on the Predefined Identifiers described there.

CAN Attributes

Rx, Tx, TxRq, Data, and Remote are basic CAN-based attributes that can be included in the filter.

Additional Parameters for Filters

In addition to the basic settings described above, additional parameters are offered depending on the filter category.

EMCY

It must be noted here that the EMCY COB-ID in the object directory is taken into consideration. If the object (0x1014) is not present, the filter operates according to the Predefined Connection Set.

NMT

The network management filter contains the two parameters NMT status and Broadcast. You can use these parameters to specify the node states and target nodes. The Broadcast check box decides whether or not the filter is to also consider global NMT commands.

Note

You can additionally configure the Node ID parameter for this filter with the value 0. This means that no filtering occurs for a node-specific NMT command. If you activate the Broadcast switch simultaneously, only global NMT commands will be filtered.

NMT Error Control

You can parameterize the node monitoring filter with any combination of the valid node statuses BootUp, Stopped, Pre-Operational, and Operational.

PDO

All necessary information is read from the corresponding COB-ID objects (0x1400 – 0x15FF / 0x1800 – 0x19FF), and both received and transmitted PDO messages are evaluated. If no access to the object directory is possible for one or more nodes (DCF file unknown), the Predefined Connection Set Identifiers will be used when the CiA301 profile is being used.

SDO

This filter enables filtering of the complex SDO transfer, which in some cases consists of many messages. Three transfer modes are supported:Expedited, Segmented, and Block Transfer. You can also configure the filter with regard to the transfer direction or the addressed objects. This filter also generally uses the determined COB-IDs of the referenced DCF files. For this, the objects 0x1200 – 0x127F are read in and added to the list of SDO channels.

For all nodes that are not described, depending on the used profile, the predefined SDO-channel is added to the SDO-channel list. You can use the SDO list filter to reduce the SDO channels to simplify the selection. A summary below the list enables a quick overview of the number of SDO channels taken into consideration in the filter. The channel list contains a context menu that you can open with the right mouse button. There, you can easily activate or deactivate all channels or invert the current selection.

By clicking the column heading, you sort the column and can thus list all activated channels in order, for example. You can set the fields for Index and Subindex to *. This * is a wildcard and stands for any values. The wildcard also is inserted automatically when you leave the field empty.

Special Considerations in the Trace Window with Analysis Filters

Continuous Mode versus Stationary Mode

For technical reasons a filter in continuous mode of the Trace Window during active measurement is applied only to messages, that arrive starting from the activation time of the filter onwards, but not to messages that came in previously. This may result in a partially filtered Trace Window content.

To avoid such partial filtering, there are two possibilities:

  • Empty the Trace Window before activating the filter, or...
  • ...pause the measurement. In stationary mode, the filter is applied to all messages.
