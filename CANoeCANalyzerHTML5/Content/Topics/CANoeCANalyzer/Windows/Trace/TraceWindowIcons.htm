Ethernet Icons [▲ back]

Icon

Type

SOME/IP SD action (offer, subscribe eventgroup, …)

Signal

PDU

Ethernet packet

Ethernet packet with protocol error of the layers 3-7.

Note

Protocol errors of layer 3..7 are displayed. If using the Vector Ethernet interface VN5610 protocol errors of layers 1 and 2 are additionally displayed for Rx packets.

More information are displayed in the column Data and the detailed view.

Status event.

A state event is displayed in the Trace Window if the Ethernet connection changes, e.g. the cable is plugged, and on measurement start. It shows the connection state, e.g. link-up.

More information are displayed in the detailed view.

Ethernet packet with one or more protocol-specific packets.
For selected protocols, Ethernet packets are interpreted in greater depth and the contained packets are displayed as additional events in the Trace Window; see protocol-specific Ethernet packet.
The icon signals that an additional Ethernet packet has been generated from this Ethernet packet. The Ethernet packet is displayed directly after the Ethernet packet in continuous mode.

Protocol-specific Ethernet packet.
The packets are generated from CANoe for selected protocols, e.g., for SOME/IP, DoIP, XCP, and HSFZ. For this, UDP and TCP packets are evaluated and the packets transferred therein are displayed as additional events in the Trace Window. These packets have the same time stamp as the Ethernet packet from which they were generated; see Ethernet packet with one or more protocol-specific packets.

Faulty protocol-specific Ethernet packet.
These Ethernet packets are faulty, e.g., a coding error has been detected. Additional information is available in the detail view.

Shortened protocol-specific Ethernet packet.
CANoe limits the maximum length of Ethernet packets. This icon is used to indicate shortened Ethernet packets. The Length column shows the originally transferred length and the length of the data available in the Trace Window.
