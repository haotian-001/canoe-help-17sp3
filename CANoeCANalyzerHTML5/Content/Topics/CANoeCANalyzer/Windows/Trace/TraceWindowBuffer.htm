Buffer Concept

 

Each Trace Window has a main buffer and an analysis area. The main buffer contains a large portion of the measurement or the entire measurement. This is dependent on the configured size of the main buffer and the duration of the measurement. The analysis area is an excerpt of the main buffer that corresponds to the current filter settings (predefined and analysis filter). Through a suitable filter selection, you see a corresponding time range of the measurement. Older events can be subsequently reloaded from the main buffer using the Buffer Navigator.

Events that occur only rarely are saved separately if they conform to the selected filter settings. If these filter settings are changed following the measurement, it is possible that other events falling within the same time range are no longer contained in the main buffer. A system comment will be displayed to inform you of this.

Usage/Navigation

Chronological mode

If the number of recorded events won't fit into the analysis range, this is indicated at the left edge of the Trace Output Window. The first event of the measurement is displayed at the top.The bar represents the size of the analysis range.

With a right mouse click on the bar you open the Buffer Navigator (orange). You can use it to move the analysis area upward/downwards, to jump to a dedicated time stamp or to jump to the oldest/newest time stamp.

The loading of events to the analysis area may take some time. You can cancel this operation in the Buffer Navigator before the analysis area is completely filled. You can use Reload analysis buffer content to manually load the analysis area.

Beside the navigation via the menu of the Buffer Navigator, following possibilities are available:

Navigation to a specific time stamp

Shift the bar upwards to load older events from the comprehensive buffer.
To shift the analysis area, you can use the following steps:

  • Shift the bar with pressed left mouse button to shift the analysis area.
  • Shift the bar with <Ctrl> + pressed left mouse button to shift the analysis area.
    The more you move the mouse the faster the analysis range is moved. The same behavior is available with pressed middle mouse button (dependent on your mouse configuration).
  • Use key combinations.

Shift the navigation area by 50%

To shift the analysis area by 50%, you can also use

  • a click with the left mouse button in the navigation bar above or below the Navigator.
  • <Page Down>, <Page Up> keys. To do so, scroll with the keys to the upper or lower stop. Then, release and press the key again. Movement of the position indicator of the scroll bar indicates shifting of the analysis area.
  • orbutton of the scroll bar. To do so, scroll with the button to the upper or lower stop. Then, release and click the button again. Movement of the position indicator of the scroll bar indicates shifting of the analysis area.
  • key combinations

Your selected navigation time point are saved in a navigation history. Take and in the toolbar to navigate to the time point in this history.

Fixed position mode

In fixed position mode there is no Buffer Navigator available.

Prioritization

The storage capacity of the comprehensive buffer is limited. When the storage capacity is exceeded, older events are overwritten by the recording of newer events. If there are multiple Trace and Test Trace Windows in the configuration, the process of deleting older events can be delayed as long as possible for a specific Trace or Test Trace Window by prioritizing that window.

The prioritization can be activated in the Trace Configuration dialog.

Configuration

In the Trace Configuration dialog location and the size of the pagefile are displayed.
The buffer settings are configured in the Options dialog.

Tip: The default program settings should be suitable for almost all cases, so you don't normally need to make any changes here.
