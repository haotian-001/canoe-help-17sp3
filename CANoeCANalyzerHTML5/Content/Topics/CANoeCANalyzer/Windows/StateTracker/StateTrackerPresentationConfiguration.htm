Presentation of CAN Frames

Similar to Signals and variables, CAN Frames are represented as rectangles. The length of the rectangle results from the frame duration. If the frame duration cannot be determined, only update points are displayed as vertical lines.

Configuration of CAN Frames

There are 4 ways to configure CAN messages in the State Tracker:

  • Network Section
    All frames that are transmitted in the network will be displayed.
  • Node Section
    All Tx frames of the configured node will be displayed.
  • 1…n IDs (range of IDs)
    All frames whose ID is in the configured range will be displayed.
  • Single Message
    Only the configured frames will be displayed.

The configuration of the CAN messages is done via the context menu.

Displaying Bursts of CAN Frames

If bursts are reported the particular Frames are highlighted (default color is yellow).
Display of the bursts and the color can be adjusted for CAN messages in the configuration dialog on the Measurement Objects page.

Configuration of the Value Presentation

Display of the values can be configured depending on the object type. The setting can either be done

  • in the configuration dialog on the Measurement Objects page or
  • via the context menu command Set Value Mode of the measurement object in the legend.

Analysis Outside a Running Measurement

Since version 7.6 SP3, the State Tracker supports the writing of data to the system buffer (data history). Data is not written to the system buffer by default. You can activate it in the configuration dialog under Optimizations.

After the measurement has run, you can view and analyze the data in the State Tracker. How much of the signal history is actually displayed depends on the number of signal updates and the amount of RAM buffer allocated to the signals. If it is not possible to load all of the data to the signal's RAM buffer, the affected signal is marked with the warning History data.

You can solve this problem by making the visible area smaller.

It can take anywhere from a few milliseconds to several minutes for the data to refresh, depending on the system used, the volume of data and the number of signals. Data is only refreshed when needed, e.g. when you increase the size of the visible area or jump to the start of a measurement. Data is always loaded from right to left on the time axis so that the most recent data is always visible.
