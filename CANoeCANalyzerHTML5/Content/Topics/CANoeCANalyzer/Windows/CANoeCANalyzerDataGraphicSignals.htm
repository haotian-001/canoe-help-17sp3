Missing Measurement Values for Signals in Data and Graphics Window

 

Unique channel assignment of received signals

Are you experiencing the problem that received signals are not displayed in the Data Window or Graphics Window? This can be due to an incorrect channel or database assignment:

Retaining names of signals

The identification rules for signals have been revised. The names of the signals are retained even after changes in the database or relocation of the database. This saves a lot of work because it is easier to identify signals. The signals can be assigned to a new configuration more quickly.

When signals are received the database is assigned to the correct channel

The originally assigned channel of a signal is retained even when the configuration changes. Thus, if you use the signal in a different scenario with different channels, you must adapt the database assignment or reconfigure the signals (delete and reinsert them). Otherwise, the Data Window or Graphics Window may not display any measured values although the signal appears to be present and active in the window.

Example

  • Assume you are working with a network to which the CAN 1 channel and a database are assigned. Signal1 is defined in the database. Consequently, Signal1 is assigned to the CAN 1 channel. You have also inserted Signal1 in the Graphics Window. This configuration functions correctly after starting the measurement. The curve of the signal is shown in the Graphics Window.
  • Now you would like to import a recorded measurement to this network using a logging file. In the recording, however, Signal1 was received on the CAN 2 channel. When playing the logging file, Signal1 is present and active in the Graphics Window, but no measurement values or curve is displayed because only signals on the CAN 1 channel continue to be received.
  • To solve this problem it is not sufficient to assign the database to the network with CAN 2 in addition to the network with CAN 1.
  • You must remove the database from the network with CAN 1 and then assign the database to the network with CAN 2 in order for the measured values of the logging file to be correctly displayed there.

Note

  • The display of received signals in the Data Window or Graphics Window is dependent on the channel. As a result, data of the same signal from several different channels are not combined in the display.
  • The database always considers the channel defined originally for a signal. An additional insertion of the database on another channel is not sufficient. The signal remains assigned to the original channel.
  • In order to determine which channel is assigned to a signal in the Data Window, point the mouse at a signal. The tooltip of a signal then indicates the name of the network and thus implicitly also the channel. If you display the legend in the Graphics Window, you will also receive a list of the signals including the tooltip.
