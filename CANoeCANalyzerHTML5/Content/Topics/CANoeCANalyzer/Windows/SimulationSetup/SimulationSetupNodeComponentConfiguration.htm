Simulation Setup: Component Configuration

Context Menu of a Node in the Simulation Setup: Component Configuration…

In the Component Configuration dialog you can configure component specific parameters for the components (IL/NL). These are parameters for the component itself, for the network, for the nodes or for ECUs.

The Component Configuration dialog is active if a component has been assigned to a node in the Simulation Setup that allows configuration via parameters; e.g. ISO11783_IL that is also part of the sample configuration Virtual Terminal Demo.

Several filter possibilities are available in the dialog. With the Filter menu you can filter out the different parameter types. The two lists below can be used to restrict the view to dedicated components and nodes. The Value column can be used to set the parameters. Bold values differ from the default values specified by the component.

Sending Behavior of Simulated Nodes

Available with included CANoeILNL_AUTOSAR_ETH.vmodule.

In the CANoeILNL_AUTOSAR_ETH group, you can set the sending behavior of the simulated nodes. The setting only acts on the sending behavior of the notification messages of events and fields. For SendBehavior, select one of the following Values:

  • AccordingToStandard
    The notification messages will be sent according to the SOME/IP specification as follows:
  • Event notifications: at a value update
  • Field notifications: at a value change
  • AccordingToStandardAndDatabase
    Default for all simulated nodes to which a node from the database is assigned.
    Same sending behavior as with the AccordingToStandard setting. In addition, the notification messages are transmitted cyclically. The cycle times are taken from the database.
  • AccordingToStandardAndCyclic
    Default for all simulated nodes to which no node from the database is assigned.
    Same sending behavior as with the AccordingToStandard setting. In addition, the notification messages are transmitted cyclically. The cycle times are taken from parameterDefaultCycleTimeInMs. All notification messages are transmitted with this cycle time. Cyclic transmission does not take place if the value 0 is entered in parameter DefaultCycleTimeInMs.
  • NoAutomaticSending
    There is no automatic transmission of the notification messages. They can only be sent via explicit CAPL calls (Trigger Event or Commit Field).

Static TCP Routing

Available with included CANoeILNL_AUTOSAR_ETH.vmodule.

In the CANoeILNL_AUTOSAR_ETH group, you can automatically establish TCP connections (client side) for PDUs between two sockets defined in the database. For AutomaticTCPBehaviour, select one of the following Values:

  • Off (default)
    No TCP connection is establishes automatically.
  • AtILStartAndStop
    With the start of the Interaction Layer (via measurement start or ILControlStart) an attempt is made to establish all connections of the node.
