Simulation Setup: Concepts

 

Working in the Simulation Setup

Via the context menu (right mouse button) of the Simulation Setup you have access to standard operations like copy, cut, paste etc. You can also apply these functions on different bus systems.

By means of the tabbed pages at the bottom of the Simulation Setup, you can easily change between the different networks of your configuration.

Additionally you can easily shift objects. Just select the object with the left mouse button and drag and drop it by pressing the left mouse button (drag and drop.

For a simple indication and display of the real and the simulated bus the following color code applies:

  • real Bus

…is displayed as a dark blue line.

  • simulated Bus

… is displayed as a red line.

If a node outputs a message to a network, a check is performed to determine whether the node actually has a link to this bus. The behavior of this function can be adjusted (see Configuration: Compatibility).

Gateway

Several nodes can be assigned to one ECU, which makes modeling of gateways possible. A gateway is a special ECU, which is used as connection between two or several buses. The information exchange between the networks is done via the nodes of a gateway. This can concern several networks (of the same type) as well as a system of different bus systems (CAN, LIN,…).

Gateways can be created or configured by different means:

  • in the configuration dialog of a node

Here several networks can be assigned to one node. In this way such a node gets additionally the function of a gateway. You can open the configuration dialog e.g. via the context menu (right mouse button) configuration of a node in the Simulation Setup. The node configuration dialog will appear and you can select the tabbed page network. There you can assign the desired networks to the selected node.

  • in the menu of the Simulation Setup

Please select copy via the context menu of the node, that shall become a gateway. Afterwards please change via the tabbed pages at the bottom of the Simulation Setup to the network that shall be connected. There you place the mouse on the line, which represents the network. Via the point connect of the context menu the connection between the two networks can be created (gateway). The configuration of the gateway can now be made e.g. via the context menu command configuration (see above).

  • in the system view

Here you can start copy via the context menu of a node. Afterwards you open the network tree, that you want to connect and place the mouse on nodes. Via the insert connection command of the context menu you establish the connection between the two networks (gateway). The configuration of the gateway can now be made e.g. via the context menu command configuration (see above).

System View

The system view can be displayed on the right in the Simulation Setup. For this purpose please use the context menu (right mouse button) of the Simulation Setup. The system view shows a tree view sorted according to networks. It offers a drag and drop support, with which you can modify your Simulation Setup fast and easily.

Additionally there is a context menu available for all items of the tree view, with which you have fast access to important and frequently needed functions. You can, for example

  • …execute a system verification (context menu at networks)
    The result is displayed in the Write Window. (For details see System verification at the end of this page.)
  • …define a new gateway (context menu of a node)
    (see above: Gateway)
  • …execute a simplified "Copy/Cut-and-Paste" action
    (see above: Gateway)
  • ...remove all elements of a certain node type (e.g. generators)

Object View

The object view shows you all substantial information about each active object (node, gateway,…). Additionally it is displayed, whether the selected block is active (for the Generator, the Interactive Generator (IG) and the Replay Block).

The object view can be displayed on the right of the Simulation Setup. Please use the context menu (right mouse button) of the Simulation Setup to do so.

System verification

You can start a system verification by selecting the System verification command in the context menu (right mouse button) of a selected network (see above).

The System verification checks your Simulation Setup according to the following rules:

  • 1st Rule

All network nodes that are defined in a database must be included in the assigned network.

Example

The database Comfort defines the network node console and is assigned to the network Comfort in the Simulation Setup. To execute this rule, there must be a CAPL block included in the Simulation Setup; this CAPL block has to be assigned to the database node console.
  • 2nd Rule

All gateways that are defined in the databases must be included as gateways within the Simulation Setup.

Example

The databases Comfort and PowerTrain both contain the network node gw and are assigned to the networks Comfort and PowerTrain in the Simulation Setup. These nodes are interpreted as gateway definitions. To meet this rule, there must be a CAPL block in the Simulation Setup that has to be assigned to a database node and has to be included as a gateway. It does not meet this rule, when a CAPL block is included as a node in one of the networks while no database is assigned to this network.
  • 3rd Rule

All gateways that are defined in the databases must be included as gateways in the assigned networks within the Simulation Setup.

Example

This rule is slightly different to the 2nd rule. It does not meet to include a gateway – that is defined in the databases – in the available networks of the Simulation Setup. The concerned CAPL block must be included as a gateway in the networks to which the databases are assigned to.
