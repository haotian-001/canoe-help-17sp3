Simulation Setup: Network Node Configuration

 

With this dialog you configure a Network node in the Simulation Setup.

Common

Settings

  • You can give the node a title in the Title field. It is only used for displaying in the Simulation Setup. There is no other semantic meaning. The database assignment is done using the node database name.
  • In the Network Node box choose the node name of a network node that was predefined in the database. This name is then shown in the first line of the function block in the Simulation Setup. If no name is selected the default name <<default>> is set to call the user's attention to a possible error.

    		•

    Note

    The database name of test modules is always set to << default >> and cannot be changed. Test modules stored with their database name in older configurations can only reset the name to << default >> or retain the current name.
    The same rule applies for network nodes in the test setup.
  • The behavior of a network node can be influenced by an appropriate selection in the State line.

    • Node Type

    Display

    Notes

    real node

    off

    This node really exists and is connected to the real bus. It is shown in the Simulation Setup with a blue connection line to the real bus, and its status line identifies it as off.
    In this case the network node model and all other options have no effect on the measurement.
    Network nodes of the simulation can be deactivated with this option.

    simulated node

    simulated

    Simulated nodes are shown with red connection lines to the simulated bus, and their status lines identify them as sim.
    During a measurement the node's functionality is simulated by the associated CAPL program.
  • In the Execution field you can select how the assigned programs shall be executed.
  • Normal:
    The program (CAPL, C# ,…) will be executed in the simulation environment.
  • On network interface (CAPL-on-Board):
    CAPL programs can be executed directly on the VN interfaces. Please refer to the CAPL Code Execution Modes for Improved Realtime Characteristics.
  • [Extended…] provides access to further special settings for executing a node application:
  • With Start Delay and Drift/Jitter the time behavior of the simulated network node can be influenced.
    The Start Delay acts as an activation delay, while Drift and Jitter affect the individual timers of the node.

Node Specification

  • The [File] button opens a file selection dialog. Here you select the CAPL program that describes the functionality of the network node. The relevant file is indicated in the text box beneath this.

    		•

    Note

    Nodes and ECUs can be simulated without a CAPL program. This is quite useful if the functionality to be simulated or tested is exclusively implemented by the use of external modules (e.g. modeling libraries (DLL files) or LIN nodes).
    In this case it is not necessary to select a source file.
    An already configured source file can be removed by deleting the filename in the text field.
  • The [Edit] button opens the CAPL Browser, with which you can edit the selected program.
  • All CAPL programs must be compiled before the measurement start. This is done by clicking [Compile] or by in the Environment ribbon tab.

Simulink

To add a Simulink model compiled for HIL Mode to the Network Node use the Components tab to reference the module description file, see HIL Mode - CANoe Configuration (MATLAB Integration).

This page is only available with activated Legacy Simulink integration check box. The dialog only remains for compatibility reasons and will be removed in a future version. Please rely on the above described method to configure Simulink models.

This page shows the configuration of the Simulink model for a specific node.
The model itself can be specified in the first line of this page.

  • If already available the required model viewer information can be selected by the usage of the [...] button on the right side.
  • The specification of the Simulink Coder model or vModule can be done in the last line of this page.

See also additional information on the CANoe Simulink Model Viewer.

ERT

This page is only available in ERT network nodes and allows to control specific settings for the Extended Real Time extension:

  • The priority of an ERT network node controls the relative realtime priority to all other ERT network nodes. If several ERT network nodes are scheduled at the same time, nodes with higher priority are executed first. Higher priority nodes can preempt already running nodes with a lower priority.
  • The Data consistency of an ERT network node can be Write-Through or Snapshot. The Write-Through mode is optimized for speed, the data within a handler (e.g. a CAPL handler) can change anytime. The Snapshot mode guarantees that the data within a handler will be consistent and does not change during the execution of the handler.

  • The CAPL Timing Variables measure the time delta between two CAPL timer calls and the execution time of the timer handler. To activate the variables, a CAPL script must be selected in the node specification.

See additional Information for ERT network nodes.

Components

All components used are shown here; additionally you can add further components (e.g. modeling libraries (DLL files)) here.
The following details are shown: the network on which the component is available, the name of the component, and the path of the stored file.

Cross Reference

Buses

This page shows all available and assigned networks.
When you select an entry with the left mouse button you can afterward move it by clicking one of the buttons [>>] or [<<] and thus change the status to assigned or available.

Note

If a bus system has node-relevant settings, a dedicated page will appear on which you can edit this settings.
The name of the page refers to the network name.

FlexRay

This page provides a quick overview of the node's send configuration. The Key Slot configuration from the database is shown. The active frames are configured automatically on measurement start. By default all frames are active that are assigned to the Application type. Protocol frames (TP,NM,…) are deactivated by default.

With [Configuration] you open the transmit buffer configuration. There you can configure the transmit properties of the frames in detail.

CANopen

You will find the CANopen page in the node configuration of every CAN simulation node. Here you can indicate the desired node ID and the device configuration file. If CANopen support is then activated, the simulation node behaves like a real CANopen device and sends a boot-up notification at the start of a measurement, for example.

See also Simulation of CANopen Nodes
