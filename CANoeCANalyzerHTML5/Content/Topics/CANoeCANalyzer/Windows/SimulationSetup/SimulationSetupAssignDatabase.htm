Simulation Setup: Assigning Databases

 

Note

Depending on whether the network is a CAN, LIN or FlexRay network, you select the corresponding database in XML, ARXML, or DBC format.

In the Simulation Setup, you can add one or more databases to the current network:

  1. Click an empty spot in the Simulation Setup to open the context menu, and then click System View.

  2. Select the context menu command Add.... for the Databases entry of the corresponding network.

  3. In the Open dialog, select the desired database and apply it to the corresponding network by clicking [Open].

In the system view you can remove an added database by clicking the Remove... button to remove an added database.

Additional Notes for Databases

Note

  • The database name shall be started with letters and not with numerics.
    Background: The database name is used in CAPL as qualifier.
  • If a database name is started with a digit, the workaround is to set an underline character (_) ahead.
  • If a database name contains only digits the database cannot be used.
  • After you have run the Add command, see above, the Open dialog appears and you can select a database.
    Exception: For FlexRay and LIN networks only single selection is possible further on.
  • If a LIN channel is used, LIN-specific LDF databases can be used.
  • If a FlexRay channel is used, FIBEX (XML) files can be used for it as well as DBC databases.
  • Under File type, you can specify whether you want to display all databases, or just the LDF, DBC or XML files.
  • LDF databases and FIBEX (XML) files are opened in read-only mode and therefore cannot be modified.
  • CAN, MOST
  • When assigning databases additional choices or errors may occur, e.g. the bus system does not match or the database is already assigned to another network and thus can be linked only.
    In this cases the following dialog Include databases with an overview of the selected databases, possibly errors and the possible actions (e.g. Insert, Link) is displayed.
    With this dialog it is possible to choose for each database how to include it.
  • MOST
  • With MOST devices it is also possible to link databases with all MOST devices using the Include databases dialog.
    Here you select     to either attach the database to all devices or to insert the database just to the active device.
  • When removing a database from a MOST device, you can decide to remove the database from all devices.
    When you confirm the database removal, the Remove database dialog is opened.
    Select [Yes] to remove the database from all devices or [No] to remove the database only from the active device.
    If the Don't show this dialog again check box is set the dialog will not opened anymore and the database will automatically be removed from all/one devices depending on the last user choice.

Note

  • Typically, databases serve to describe networks. The protocol type of the network can be specified explicitly in the database in this case with the CANdb++ database attribute BusType. If this attribute does not exist, the protocol type CAN will be adopted implicitly as long as the database contains message definitions. If the database contains neither the CANdb++ database attribute BusType nor message definitions, the type Undefined or All will be adopted. Such databases are only suitable for the definition of environment variables.
  • In general, the protocol type of a database that is to be inserted must agree with the bus system of the current network.

Example

  • A network to which the channel CAN 1 is assigned is of bus system CAN.
  • A network to which a database of protocol type LIN is assigned is of bus system LIN.
  • A network to which a database of protocol type Undefined or All is assigned also has no explicit bus system.
    If an additional channel is assigned to this network, however, then the bus system will be implicitly specified by it.

When databases are assigned, the agreement between the protocol type and the network type is checked.
The following combinations are allowed:

  • The protocol type of the database and the network type agree.
  • The database has the protocol type Undefined or All; this means that it only contains environment variables. This database can be assigned to any network, but at most one single network; thus it cannot be assigned to several networks. Nevertheless the environment variables have to be globally unique and are visible from all networks.
  • The network type is undefined.
    Any database can be assigned to this network.
    After the assignment, the bus system corresponds to the protocol type of the database.

Any given database can only be accepted into the configuration one time. In particular, a database cannot be inserted twice into one and the same network. You can, however, link one database with multiple networks. The simplest way to do this is to insert the database into the desired network. A warning will appear telling you that the database already exists and that only one link to the already existing database can be inserted. Confirm the message to create the link. Another approach is to drag an existing database onto the databases of another network using drag and drop. As you drag with the left mouse button, you must hold down the <CTRL>+<SHIFT> keys to perform the link action. If you drag with the right mouse button, a context menu will appear when releasing the button in which you can select the link action. The assignment of databases to channels is implicitly guaranteed by the networks configured in the Simulation Setup as well as their assigned channels.

  • A network contains the database Powertrain as well as the channel CAN 1.
    The database Powertrain is thus assigned to the first CAN channel.
  • A network contains the database Comfort as well as the channel CAN 2.
    The database Comfort is thus assigned to the second CAN channel.
  • If a third database is assigned to both networks through a link, it is thereby also assigned to both channels.
