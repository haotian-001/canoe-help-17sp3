Simulation Setup: Renaming a Database

 

In the System View of the Simulation Setup you will see a tree representation of the active configuration.

If you move the mouse pointer to a database there, and then press the right mouse button, a context menu appears in which you can call the Rename… command. With this command you can give the selected database a new name.

  • After you have called the command the Rename dialog is opened.
  • Here you enter the new/changed name of the database.
  • When you click the [OK] button the new database name for the Simulation Setup is accepted, and it is shown in the System View.

Important Note

  • Please observe proper notation for database names. Database names are case sensitive. The filename and its notation do not need to agree with the internal CANoe identifier (name) or its notation which you can change in this dialog.
  • For example, if you have tied the call of a message to a specific database in CAPL programs ("message database01::mymessage m1" instead of "message mymessage m1") the program might not be compiled error-free any longer. In this case you must either change the program or undo the renaming.

Note on Names/Descriptors and on Zip/Unzip

Some Zip programs have problems with certain characters, e.g., umlauts (mutated vowels; "ä", "ö", "ü"), "ß", space.

  • These characters are altered during a zip/unzip operation without any feedback/confirmation.

  • This results in configurations that do not work (properly) any longer after unzipping.

  • Please use only names/descriptors that do not contain any of these characters.
