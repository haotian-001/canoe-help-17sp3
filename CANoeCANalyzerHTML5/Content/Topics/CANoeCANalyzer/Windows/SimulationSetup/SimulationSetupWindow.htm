Simulation Setup

 

General

In the Simulation Setup, the overall system is displayed graphically with networks, devices and all network nodes.
All options for parameterizing the Simulation Setup are selected in this window.

Description of the Simulation Setup

In the Simulation Setup, the overall system is represented graphically with all defined networks, devices and all Network nodes. You can switch between the different networks (CAN1-CAN2, CAN-LIN, etc.) via the defined Gateways. The simulated bus from Phase 1 and Phase 2 is represented by a red horizontal line. The dark blue line above it symbolizes the physical bus from Phase 3. The two buses (real and simulated) are connected to one another through the expansion card. Parameterization of the card (baudrate selection, acceptance filtering) is performed using the Hardware ribbon tab|Network Hardware or the context menu of the network card icon, which is opened by selecting the icon with the right mouse key or by selecting it with the cursor keys and then pressing the <F10> key.

Simulated network nodes are represented by red connection lines to the simulated bus in the simulation configuration, and they are labeled in the node's status line with sim. During a measurement, the functionality is simulated by the assigned CAPL program. The bus behavior of the network node does not differ from that of a real controller with the same functionality. Real nodes, however, are represented by black connection lines to the real bus. The network node models of real nodes do not have any effect on the measurement.

If you click on the network display with the right mouse key you obtain the context menu for the bus (network).

The connection to the measurement configuration is symbolized by the ">>"-block at the far left of the Measurement Setup. Both the data on the bus and all Environment variables are fed into the data flow plan of the measurement configuration at this point, where they can be analyzed and observed during runtime.

Vector Simulation Node

The Vector Simulation Node is a virtual node on a bus. This node is automatically generated by CANoe if the configured database (AUTOSAR/FIBEX) contains messages to which no sender or receiver is assigned:

  • If a message does not contain a sender, the Vector Simulation Node serves as sending node, so that the message can be sent via the interaction layer
  • If a message does not contain a receiver, the Vector Simulation Node serves as receiving node for the message.

In windows like Symbol Explorer, Graphics Window or Trace Window, this node is displayed under the name VectorSimulationNode.
