Analysis Windows: Valid/Invalid Signals

 

Invalid signals can occur in the following windows:

  • Graphics Window
  • Data Window
  • Signal generators and signal replay
  • Visual Sequencer
  • State Tracker
  • Start Value Window

The behavior does not apply to signals for which filtering is possible in the Trace Explorer. Because the Trace Window is message based, signals cannot be examined independently of their message. The same applies to signals in the Interactive Generator, their corresponding Signal Generators, and the signals in the user-defined filter of the trigger.

Background

In CANoe, an attempt is made to sustain the linking of a signal that is configured in an analysis window to the definition in a database as long as possible. That is, even if changes are made in the database that directly or indirectly affect the signal, an attempt is made to consider these changes and to sustain the link. For example, it is possible to shift the signal from one message to another if the signal is unique within CANoe.

If the configured signal is not assigned uniquely to a signal of a database, the signal is displayed as an invalid signal in the window (icon, tooltip, grayed type). In the case of an invalid signal, settings can no longer be changed. However, it can be deleted from the window. For an invalid signal, the differentiation features used for the explicit qualification do not change until the signal becomes valid.

Note

Please note that system variables can not be moved from one namespace to another, even if the name of the system variable is unique. Moved system variables will be displayed as invalid.

When does a signal become invalid

FAQ | Signals are invalid after replacement of a database that attaches to several networks

A database attaches to two networks. In the analysis window, the same signal is configured from each of the two networks. Because CANoe must consider the different network, the signals configured in the analysis window behave differently if they are handled in a different order. That is, the signals can become invalid.

In order to replace databases that are assigned to more than one network, start by deleting the old database on the networks in the Simulation Setup and then add the new database to the same networks in reverse order.

Example

Replacement of database A with database B on network 1 and network 2. Signal A is contained in both databases. For this reason, the signal name is not unique and the network must be considered for differentiation. In the Graphics Window, Signal A is configured from each of the two networks.

Correct procedure, note the order!

  1. Remove database A on network 1
    Signal A, network 1: Invalid
    Signal A, network 2: Valid -> Signal A
    The signal of network 2 is prepared to keep the link to the database, irrespective of the networks, because the signal name is unique. The signal of network 1 is invalid.
  2. Remove database A on network 2
    Signal A, network 1: Invalid
    Signal A: Valid
    When the database is removed from network 2, both signals are invalid.
  3. Add database B on network 2
    Signal A, network 1: Invalid
    Signal A: Valid
    When the database is added on network 2, the second signal becomes valid because the signal name is unique. The signal in the Graphics Window is linked to the signal from the database.
  4. Add database B to network 1
    Signal A, network 1: Valid
    Signal A: Valid -> Signal A, network 2

Result: The signals configured in the Graphics Window now correspond exactly to the signals before these changes were made.

Incorrect procedure

  1. Remove database A on network 1
    Signal A, network 1: Invalid
    Signal A, network 2: Valid -> Signal A
    The signal of network 2 is prepared to keep the link to the database, irrespective of the networks, because the signal name is unique. The signal of network 1 becomes invalid.
  2. Remove database A on network 2
    Signal A, network 1: Invalid
    Signal A: Invalid
    If the database of network 2 is removed, both signals are invalid.
  3. Add database B on network 1
    Signal A, network 1: Valid -> Signal A
    Signal A: Valid
  4. Add database B on network 2
    Signal A: Valid -> Signal A, network 1
    Signal A: Valid -> Signal A, network 1

Result: The signal of network 2 was assigned to the signal of network 1.

Reason

As long as a signal name is unique within CANoe, the maximum possible flexibility with respect to database changes should be provided. Thus, for example, it must be possible for a signal with a unique name to remain valid independent of the database. (The database has been replaced, but contains a signal with the same name.)

FAQ | Signals are invalid after replacement of a database that attaches to one network

A database attaches to one network. This database is removed and another is added that contains a signal with the same signal name. A signal from the first database is configured in the analysis window.

To replace databases that are assigned to one network, start by deleting the old database in the Simulation Setup and then add the new database on the same network.

Example

Replacement of database A with database B on network 1. Signal A is contained in both databases. In the Graphics Window, signal A is configured from database A.

Correct procedure, note the order!

Database A was added to network 1:

  1. Remove database A on network 1
    Signal A: Invalid
  2. Add database B on network 1
    Signal A: Valid

Result: The signal configured in the Graphics Window remains valid and now corresponds to the signal from the new database.

Incorrect procedure

Database A was added to network 1:

  1. Add database B on network 1
    Signal A: Valid -> Signal A, database A
    If the second data is added first, the signal name is no longer unique and the database must be considered when linking the signal.
  2. Remove database A on network 1
    Signal A, database A: Invalid
    Even if the first database is now removed, the database name was needed nevertheless for differentiation purposes, and the configured signal becomes invalid.

Result: The signal configured in the Graphics Window becomes invalid.
