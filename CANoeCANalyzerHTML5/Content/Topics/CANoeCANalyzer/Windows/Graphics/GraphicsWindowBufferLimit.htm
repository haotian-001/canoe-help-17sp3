Graphics Window: Buffer Limit

Ribbon: File Tab|Options|Windows/Blocks|Graphics Window: Buffer Limit

General

The buffer limit in kB defines the maximum memory allocation by all Graphics Windows. Above all, this makes sense if you have input a relatively large time span as the buffer size. Without a maximum limit, the operating system will continue to demand more and more memory over the course of the measurement. With page swapping this may lead to severe loads of the overall system.

The buffer limit does not restrict how much measurement data can be stored, but instead how much data can be displayed without having to compress or truncate it.

Compression

After the measurement, the system always loads as much data as needed for display in the window from the entire program memory. If the size of this data exceeds the buffer limit, the graphs in the window are compressed. This means that not all measurement points are displayed. However, CANoe, attempts to maintain a clear symbol chart as it removed measurement points. Compressed graphs are identified by their lighter color and a notice found below graph area.

Compressed graphs do not show samples and cannot be used to determine values. If you would like to use the measurement cursor, use the Zoom In until No Compression function to reduce the size of the display.

Data is not loaded during the measurement. When the buffer limit is reached, old data is removed from the graphs to make room for new data that is being saved.

Available Memory

To get an idea of the memory available at any given time, this system information is displayed with the following meanings:

  • Physical memory: Available main memory
  • Total memory: Maximum usable memory of CANoe

It is difficult to determine a precise relationship between the required memory, available memory and system behavior, since the available physically memory is subject to severe fluctuations (number of opened applications, resource requirements, etc.).CANoe may require a large portion of the available memory for the measurement data alone.

It is certainly possible to specify a memory limit that is greater than the currently available physical memory. However, please note that the operating system can only provide this extra memory with the help of page swapping as soon as the main memory is no longer sufficient.

However, set the memory limit as low as possible to leave more memory available for the actual data.

System Behavior

If the buffer limit is insufficient for all Graphics Windows, an error message is written to the Write Window for which the in-sufficient memory problem occurred. The following error message appears:

Insufficient system memory: The ring buffer of <<Signal>> was truncated !

<<Signal>> refers to the symbol for which the insufficient memory was found.

Default: 20.480 kB = 20 MB

Estimation of the Buffer Limit

If you want to work without compression at large display size or if you want to see all data during the measurement, you have to check, if the set buffer limit for the displayed symbols in the Graphics Window is big enough.

The buffer limit required depends on the number of symbols, sampling rate and measurement time, but not on symbol size. Please note that the symbols of all Graphics Windows should be considered, since the buffer limit is a global value.

There are different formulas for computing the buffer limit depending on the sampling rate. You must check to see how fast the sampling rate is for each individual symbol.

Case 1: Sampling rate > 6.4 milliseconds

In this case the buffer limit is calculated by the formula:

Buffer limit = (Measurement time / 6.6s) * 8 kB * Number of signals

Case 2: Sampling rate < 6.4 milliseconds

A special case occurs when the sampling rate is so fast that more than 156 measurement data are logged per second, i.e. fewer than 6.4 milliseconds lie between the individual measurement time points. In this case the buffer limit is calculated by the following formula:

Buffer limit per signal = (Sampling rate * Measurement time) / 128 kB

If all symbols have a fast sampling rate the following formula applies:

Buffer limit of all signals = Sum [over all signals] (Sampling rate of the signal * Measurement time) / 128 kB

Case 3: Rough estimate

You can also make a rough estimate for all symbols by just considering the fastest sampling rate in the calculation.

Buffer limit = (Number of signals * Fastest sampling rate * Measurement time / 128) kB

Application case

In a Graphics Window 4 symbols are each configured with 16 bits.

The measurement time is 13 hours (approx. 47000 seconds), sampling is 2 times per second (500ms).

Check: Sampling rate > 6.4 milliseconds = 500ms

Buffer limit = (Measurement time / 6.6s) * 8 kB * Number of signals

Buffer limit = (47000 / 6.6s) * 8 kB * 4

Buffer limit = 228000 kB
