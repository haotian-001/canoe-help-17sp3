Graphics Window: Special Features when Working with Grouped Symbols

Context Menu Path: Configuration...|Signals/Axes

The following special features should be noted when working with grouped symbols:

General

  • Any symbols can be grouped, i.e. bit number and value type do not have to match.
  • The common axis/group can be disabled in the legend. Individual symbols can be disabled, even if they belong to a common axis/group.
  • The common axis/group can be collapsed/expanded to save space.
  • A symbol can only belong to one common axis/group.
  • Symbols of a group can only be sorted within the group with the keys.
  • Copying/moving common axis/groups into other Graphics Windows is not supported.

Y-Axis Views

  • With the All Y-axes display option, symbols within a common axis only have one Y-axis in comparison to all other symbols.
  • If the Separate views display option is selected, each grouped symbol is displayed as a non-grouped symbol in a separate diagram.

Value Ranges/Scaling

  • The Default all Y scaling function sets the standard values of the common axis/group. For common axis these are the lowest minimal value and highest maximum value of the grouped symbols from the database.
  • The Store scalings function is used to save the value range for each common axis/group and associated symbols.
  • The Get scalings function is used to restore the value range for a common axis/group and all symbols in that group stored with the Store scalings function.
  • The Move signals up/down function is used for grouped symbols to move all symbols of the group upward / downward.
