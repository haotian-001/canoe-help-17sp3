ISO11783 Specific Windows

Short Description

Virtual Terminal Window

The Virtual Terminal Window provides a ready-to-use virtual terminal instance without any additional CAPL programming. VT window offers the possibility to visually control the data masks as well as the contents of object pools. In addition, the ISOBUS Shortcut Button (ISB) is supported.

In passive mode, you can observe the bus traffic of a real Virtual Terminal. If problems occur, you can analyze the object pool in CANoe Virtual Terminal in more detail.

Interactive Task Controller

The Interactive Task Controller can be used to access the process data dictionary of an implement. The process variables can be set or read out. The process variables are selected symbolically after the device description has been transferred by the implement. When a task is started, the process variables of the default treatment zone are transmitted to the implement and the peer control assignments are set.

Task Controller Window

The Task Controller Window provides an operational Task Controller instance that simulates the ISO11783 communication of a real Task Controller without programming. The TC Window independently logs on to the bus, sends its status message, responds to requests sent by add-on and diagnostic devices, and receives all uploaded device descriptions (DDOPs). From the loaded device descriptions (DDOP), the TC window calculates the process variable directory for each connected attachment and displays it clearly. This provides you with a graphical interface with which you can read, change and query all process variables (including those relevant to Section Control).In addition, the TC window offers a convenient way to analyze the time history of individual selected process variables.

File Server Window

The File Server Window provides a ready-to-use file server instance without any additional CAPL programming. The window displays connected clients, opened files and read/write operations as well as a log file with detailed information about the communication.

J1939 Scanner

With the J1939 Scanner in the Measurement Setup active nodes in the network can be displayed and analyzed. The source addresses of the received Parameter Groups and the Address Claiming messages are evaluated by the J1939 Scanner. From this information the J1939 Scanner detects the active nodes and the state.

J1939 Filter

The J1939 Filter lets you filter J1939 messages. It has a number of advantages over the CAN filter.

J1939 DTC Monitor

With the J1939 DTC Monitor in the Measurement Setup you can easily display and request error codes (DTC â Diagnostic Trouble Codes) and diagnostic messages. With the support of Freeze Frames (extended information from fault memory) you can use the J1939 DTC Monitor for emission related or power train diagnostics.

J1939 Diagnostic Memory Access

With the J1939 diagnostic memory access window you can read and write parts of the memory as well as execute a boot load procedure. For this the diagnostic messages DM14, DM15, DM16 and DM18 are used.

GNSS Monitor

The GNSS Monitor in the Measurement Setup is used for graphical and numeric display of degrees of longitude and latitude as well as to specify the height of a GNSS message in the NMEA2000Â® or J1939 protocol.

GNSS Simulator

The GNSS Simulatorcan be used to simulate a GNSS receiver. To do this, a geometric figure or a position file is used to determine a path that will be traced at a variable speed when the simulation is started. GNSS positions thereby generated are transferred cyclically with configurable messages.

General Windows with ISO11783/J1939 Support

Screenshot

Short Description

Trace Window

The purpose of the Trace Window is to record bus activities during measurement. All the messages received at the input of a Trace block are displayed as lines of text in the Trace Window.

Data Window (J1939)

The Data Window is used to display signal values.The signal value is displayed as a physical value, as a raw value and with a progress bar by default.

Graphics Window (J1939)

In the Graphics Window symbols (signals, variables, and diagnostic parameters) are displayed graphically in an XY diagram. You can configure the time or any symbol as the X-axis.

Statistics Window (J1939)

The Statistics Window displays statistics about bus activities during measurement.

This window can be inserted into the Measurement Setup via the context menus of the relevant function blocks (bus statistics etc.). All bus events that arrive at the input of the Measurement Setup block are evaluated for this purpose.

If a database is assigned to the channel, statistics can also be evaluated individually for each available network node.

CAN Interactive Generator (CAN IG)

The CAN Interactive Generator (CAN IG) allows you to send configurable CAN frames via CAN. You can set signal values within the frames manually or automatically via Signal Generators.

Map Window

With the Map Window you can display and evaluate logged position information from configured GNSS receivers on a map. In addition, you can use CAPL or .NET to draw your own objects on the map.

With the following options, the basic scope of the Map Window is extended by additional functionalities.

  • With the Option Car2x you can display location coordinates of ITS objects and events received in a Car2x network on the map.

  • With the Option J1939 or the Option ISO11783 you can display and evaluate position information from J1939 or NMEA parameter groups. The Fast Packet Protocol is supported as well.
