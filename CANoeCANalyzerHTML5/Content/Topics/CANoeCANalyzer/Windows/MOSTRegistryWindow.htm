MOST Central Registry Window

Ribbon|Analysis Tab|Central Registry

The MOST Central Registry Window displays the bus registry of the MOST ring in a clear manner. It also offers additional analytical possibilities.

It is divided into three areas:

Registry area

The Registry Area contains a list field that displays the registry. Each entry contains a node address, an FBlock ID, an instance ID and a symbol that proves whether the displayed information is secure (see section Correctness / Completeness of the Displayed Information). If a catalog is assigned to the configuration, then the names of those FBlocks are displayed as well.

Below this is a row of tabs that make it possible to switch between all of the configured channels.

The [Advanced] button at the lower edge of the area serves to open the area containing the analytical functions.

There are several additional buttons next to it:

  • With the button, the currently displayed registry can be copied to the clipboard.
  • With the button, a registry can be imported from the clipboard.

Using these two buttons, a registry can be imported and then saved as reference. After importing a second registry, these two registries can be compared using the difference view. Note, that this button can only be clicked if no measurement is running, the window is set to registry view, no history entry is selected and the window is not paused.

  • With the button, the registry displayed at the time the button is clicked can be frozen. This makes it possible to analyze the registry at length, without any changes that may arise affecting the currently displayed state.

On the registry list a context menu can be opened:

  • The menu commands Pause, Copy to clipboard and Import from clipboard have the same effect as the corresponding buttons described above.
  • The menu command Synchronize Windows has the effect, that windows like the Trace Window, the Data Window and the Graphics Window are synchronized with the Registry Window using time stamps. Once activated, e.g marking a message in the Trace Window causes the Registry Window to display the registry (history entry), that was known at the point of time the message was received. Conversely selecting a history entry in the Registry Window causes the Trace Window to mark the message, that has caused the history entry in the Registry Window.

Advanced area

Views

You can choose between three different views:

  • The current registry view shows the currently observed registry and the selected entry (see section Registry).
  • The reference registry view shows the saved registry.
  • The comparison view compares the reference registry (required state) with the one being observed currently (current state). All entries from both registries are displayed. Differences are highlighted in color:
    • An entry displayed in red indicates that the entry is required in the reference registry, but is missing from the current registry.
    • An entry displayed in green indicates that the entry is not required in the reference registry, but is present in the current registry.

Note

The comparison view and reference registry views are only available once a non-empty registry has been saved.

Registry

In this area you have access to a history view that shows the development of the registry over time (see the section Obtaining Registry Information). The selection list shows the time stamp of the given entries.

[Query Registry Now] can be used to query the complete registry manually with the MOST NetworkMaster, provided the currently selected channel offers node functionality (see Hardware configuration: MOST|Setup).

Below there is a status text indicating whether Automatic Query of the registry is activated for the selected channel (see Hardware configuration: MOST|Application Socket|Automatic Query of the Central Registry).

Difference

If the Only show differences option is selected, the Difference view only displays the differences between the current registry and the reference registry. If this option is not activated, the congruent entries are also displayed.

Reference

The [Save] and [Delete] functions can be used to save the registry currently displayed or to delete the saved registry.

Status bar

The status bar at the bottom of the Registry Window applies to the complete registry displayed and is divided into five sections:

  • Completeness of the registry (s.a. correctness/completeness of the displayed information)
  • : The displayed registry is complete.
  • : The displayed registry may be incomplete.
  • Timestamp

The time stamp of the displayed registry.

  • Differences
    Format: +x / -y

+x stands for the number of FBlocks contained in the current registry, but not in the reference.-y stands for the FBlocks contained in the reference, but not in the current registry.

Note

This display is only active when a reference registry has been saved.
  • Pause

The icon is displayed when the Pause button is activated or when one of the entries in the history (except the most recent entry) is selected.

  • Newness of the displayed information

The * icon is shown whenever new information has been received in the background while Pause Mode is activated.

Obtaining registry information

The Registry Window is totally passive. It listens for all the Ring Scan messages sent by the devices in the ring and for all internal events of the CANoe simulated devices (local registry).

Ring Scan messages:

  • Netblock.FBlockIDs.Status

Every node sends this message on request from the MOST NetworkMaster. This message is also sent as the replay to InstanceID change notifications.

  • NetworkMaster.Configuration.Status
  • If the transmitted status is not OK the entire current registry is deleted.
  • If the transmitted status is OK a new entry is added to the history.
  • If the transmitted status is invalid the transmitted FBlock is deleted from the registry.
  • NetworkMaster.CentralRegistry.Status

The MOST NetworkMaster sends this message in reply to corresponding requests from devices in the ring.

Correctness/completeness of the displayed information

If the Registry Window observes a Netblock.FBlockIDs.Status message, for example, the FBlocks contained in the message are added to the displayed registry. However, since this message does not necessarily mean that the information is actually added to the Central Registry by the MOST NetworkMaster (for example, if the MOST NetworkMaster is busy and cannot accept the message), the FBlocks are temporarily evaluated as unsure and are marked with the icon. The information is only sure, if the message is acknowledged.

On the other hand, all FBlocks contained in NetworkMaster.CentralRegistry.Status messages have already been entered in the Central Registry. These are evaluated as sure and are marked with the icon.

The completeness of the displayed registry is only guaranteed when a NetworkMaster.CentralRegistry.Status message is observed that contains the complete registry, or when one of the simulated devices includes the MOST NetworkMaster built into CANoe. On completion of the Ring Scan this simulated device sends an internal registry event to the Registry Window containing the complete Central Registry. If a real MOST ring is analyzed by a spy-node, for example, this analysis-node receives FBlockIDs.Status messages from all nodes except the node containing the MOST NetworkMaster (those messages are internal and not tansmitted via the bus). Therefor the displayed registry is incomplete until the CentralRegistry.Status message described above is received.
