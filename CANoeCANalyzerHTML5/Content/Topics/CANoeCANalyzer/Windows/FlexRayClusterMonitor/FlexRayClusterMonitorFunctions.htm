FlexRay Cluster Monitor: Function Overview

 

The FlexRay Cluster Monitor offers the following functions:

Network monitoring

The data traffic of the cluster is constantly being compared with the schedule from the FIBEX database.

Deviations of the data traffic are highlighted in the node and frame view using different diagnostic icons.

Quick error diagnostics

The direct comparison of actual and desired values allows for quickly locating errors in the network model.

The desired values are taken from the database, whereas the actual values are calculated in the on-going measurement. The direct view onto the network parameters and the running statistics support the analysis.

For example, if a certain frame is not sent from the database contrary to the communication model. this deviation is highlighted in the frame view as well as at the corresponding node.

FIBEX data model

The communication model is represented node-based and subdivided by channels A and B. Frames that are not defined in the database, but are still being transmitted on the bus, are assigned to a special node ("undefined frames"). This also allows for quickly locating deviations in the communication model.

Examples of typical deviations include:

  • A frame is not being transmitted
  • A frame is transmitted as Null Frame while a data frame is being expected

History Window

Use the context menu (right mouse button in the Node view or the Frame view) to show the History Window. The window registers state changes of the statistic date of the frames and shows them together with the time stamp of the last actualization of the data.

Node filter

The surveillance of individual nodes can be configured with the node filter. In case of an active filter all frames transmitted by this node will be filtered and the target data will be recalculated.

Frame type filter

The analysis of the network can be configured using a frame type filter. This allows for excluding certain frame types from the statistical analysis. Frame types, which are filtered, and the corresponding nodes are highlighted by means of corresponding icons. In case of an active filter, the desired values of the node are corrected downward.

Listing of supported frame types:

  • APPLICATION
  • NM
  • DIAG-STATE
  • DIAG-REQUEST
  • DIAG-RESPONSE
  • TPL
  • XCP_PRE_CONFIGURED
  • XCP_RUNTIME_CONFIGURED
  • SERVICE
  • OTHER (also specified for frames that are not defined in the database or not assigned to a customer)
