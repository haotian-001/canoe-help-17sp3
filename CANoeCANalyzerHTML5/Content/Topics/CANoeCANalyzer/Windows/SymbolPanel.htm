Symbol Panel

 

General

You can use the Symbol Panel to display and edit symbol values (signals and system variables) during the simulation. You can also add symbols to the Symbol Panel while a measurement is running. The Symbol Panel supports the following symbol values: strings (string), integers (int) and floating-point numbers (float).

You can open the Symbol Panel for a selected symbol via the context menu of the Symbol Explorer, the Data Window or the Graphics Window, or via the Simulink Model Viewer. Alternately, you can drag and drop a symbol from one of the windows mentioned above into the Symbol Panel. If the Symbol Panel is already open and you select a new symbol, the symbol assignment in the Symbol Panel is updated.

Information Shown

Attributes Shown

The Symbol Panel shows the following symbol information:

  • Symbol type
  • Signal:
  • System variable:

    If Invalid Symbol Selection is shown, this means that the type of the selected symbol is not supported. Please select a signal or a system variable to change its symbol value.

  • Symbol name
  • Current value
    The symbol value is shown as a physical value or value table value.

The lower part of the Symbol Panel shows the following attributes:

  • Min
    The minimum symbol value defined in the database.
  • Max
    The maximum symbol value defined in the database.
  • Initial value
    The default value for the signal.
  • Cycle time
    The signal cycle time defined in the database; alternately, the signal cycle time of the message that contains this signal.

Control Elements

Note

You can edit the new symbol value using the following controls:

  • New value
    You can enter a value directly into this field. This field also displays new values that result from adjusting the settings in other controls.
  • Slider
    Here you can use your mouse to specify a value within a defined range.
  • With Apply new value implicitly you define that the set new value is set directly with <Enter> or when leaving the field. If you have not activated the option, then you must click on [Apply New Value] to set the value.
  • Clicking [Apply Initial Value] will set the default value of the selected symbol. This behavior is independent of the Apply new value implicitly option.
  • If a new/modified signal value has not (yet) been sent, an update symbol () appears next to the field containing the new value.
    The appearance of the update symbol next to a signal means that the value shown is not the same as the current bus value, i.e. that the new signal value has not yet been sent to the bus. As soon as the signal is sent to the bus, the update symbol disappears. A prerequisite for successful signal transmission is the availability of an interaction layer (IL).
  • Several sources can influence a symbol (e.g. simulated network nodes, Replay Blocks and Signal Generators). Thus, it is possible that the new value you have specified and sent is not shown as the current value. This can happen if the value you specified was modified by another source.
  • The values that signals and system variables are to assume on measurement start can be configured individually with the Start Values Window.
