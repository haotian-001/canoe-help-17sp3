Test Configuration

 

General Information

You control the sequential execution of one or more test units in this window. The window can be opened from the Test Setup for Test Units – either via the context menu of the test configuration or by double-clicking.

The window is part of the normal window management and can be opened or brought to the front at any time via the Test ribbon tab.

Test Run

Test Units

This view displays the entire test run including test groups, test cases, and test sequences. It is compiled sequentially from the configured test units. The test units are read in for this, at the latest when the execution dialog is opened. If errors occur in the process, they are written in the Write window.

The following is specified in the list for each test cases, test sequence, and test group:

  • name
  • result of the last test execution (verdict or"not executed")
  • a check box for selecting or deselecting the element
  • the time required for executing the element

When the test configuration starts, all selected elements are executed sequentially in the order they appear in the list. Elements that are not executed receive the result "not executed", which is also noted in the report.

Buttons (on the left side):

  • Start
    Starts the execution of the test configuration.
    This is only possible during the measurement. Also, the test configuration must not already be running.
  • Stop
    Aborts a running test configuration.
    The verdict of an aborted test configuration is inconclusive or fail (in compatibility mode).
    Note:
    In vTESTstudio, it is possible to define whether test cases and test sequences will be aborted immediately or executed until the end if the verdict fail occurs. On error in test system the currently executed test case / test sequence will always be aborted immediately.
  • Configuration
    Opens the configuration dialog of the test configuration.
  • Debug
    Opens the Debugger.
  • Search
    Search for texts or verdicts (Failed/Passed) in the test sequence. For the text search you can select whether the search is to be case-sensitive and whether you want to search for whole words.
    Previously searched terms are listed above the selection of verdicts.
    With and you can navigate step by step to the individual search results.
    With you can select all search results.

Context Menu

  • Expand All / Collapse All
    All selected elements and sub-elements will be expanded/collapsed.
  • Activate / Deactivate / Invert Activation
    All selected elements and sub-elements will be activated/deactivated or the selection will be inverted.
  • Select by Verdict
    All passed/failed elements of the selected elements and sub-elements will be selected.
  • Execute Selected Elements
    All selected elements and sub-elements will be executed.
  • Synchronize Windows
    Activate/deactivate windows synchronization.

Note

The runtimes displayed in this window do not correspond to the simulation time but to the real elapsed processing time. Due to this reason, it is possible that the displayed times do not correspond to the times you would get by calculating the runtime from start and end time as listed in the Test Trace.

In this view, you can add, remove, and configure test units.

Context Menu and buttons (on the right side):

  • Add test unit…
    You can select a test unit built with the vTESTstudio via a file selection dialog. A built test unit has filename extension VTUEXE.
    Alternatively you can add the vtestunit.yaml format from the CANoe4SW Server Edition environment here.
  • Edit
    Opens the vTESTstudio project containing the test unit in vTESTstudio. In the Options dialog, specify your current vTESTstudio installation as the editor for test units under Tools.
    The editing of a test unit is only possible if the vTESTstudio project is on the same computer.
  • Compile
    Compiles the test unit.
  • Exclude from Creation
    The test unit is excluded from the creation and thus neither compiled nor executed.
  • Report
    Opens the test report of the last execution:
  • Open Test Report in CANoe Test Report Viewer
    If you selected the recommended Test Report Viewer Format in the global CANoe options, the following option is available:
  • Display the test report in CANoe Test Report Viewer.
  • Open Test Report in HTML Browser
    If you selected the XML/HTML test report format in the global CANoe options, the following options are available:
  • Display the HTML test report in an HTML browser.
  • Custom transformation
    Select the XSLT file in the following dialog. The result of the last pass of the test unit (stored in the XML file) will be transformed to HTML by using the selected XSLT and the HTML result is shown in the browser. The new HTML file is in the same folder as the XML file and the previously generated HTML file. to distinguish between the HTML files, the new one gets added the name of the used XSLT file.
  • As a further option up to five last used XSLT files are listed. If you select one of them, a new transformation is started as described under Custom transformation.
  • Configuration...
    Opens the configuration dialog of the test unit.
  • Rename
    Allows a new name to be specified for the test unit.
  • Remove
    Removes the test unit from the CANoe configuration.
  • Edit Sources
    Parameter files that are attached to the executable test unit as editable files are opened in the editor configured for that purpose. For this, you specify, e.g., Excel as the editor for test unit parameter files in the Options dialog. Changes to the editable files are applied to the test at the next measurement start when the test unit is next compiled. The VTUEXE file itself remains unchanged.

Execution Variants / Variant Property Profiles

You set the values of the execution variants in the lower part of the window. If variant properties are declared in the vTESTstudio project and defined as "editable in CANoe" for the test unit, you can specify the values here that will be used to execute the test. Variant properties defined in vTESTstudio as "Not editable in CANoe" for the test unit will be displayed as "read only" values here. The variant properties of the test unit just selected are always displayed here.

With Show variant properties for selected test units only you can select whether all variant properties of all test units are shown or only the variant properties of the selected test units. You can even select multiple test units.

If variant properties of multiple test units are shown identical variant properties of different test units will be grouped. Individual test units and their current value are shown in a lower hierarchy level below the variant property. On the higher hierarchy level you can change values for all test units in this group. On the lower hierarchy level you can change values of individual test units.

Changes made here can be applied to or made undone for all test units in one operation. Click [Apply] or [Revert] for this.

Variant properties that have been set can be saved and selected in a profile without having to adjust all values individually.

These profiles can also be exported individually or together and then re-imported, whereby the saved and existing variant properties are checked. If necessary, a warning is issued that some variant properties no longer exist in the configuration or that new ones have been added since the export.

Toolbar for Variant Property Profiles

Description (from left to right):

  • Selection of the current profile
    Selection list of all profiles available in the configuration.
  • (*) is only visible if unsaved changes exist in the current profile.
  • Save as new profile
    A window appears in which you can enter the name for the new profile or cancel the process. If you have selected a name, a new profile is created with the current variant properties.
    This button is only active if changes that have not been applied are present.
  • Delete profile (does not work with [Default])
  • Apply and save in profile
    Corresponds to the previous "Apply" button, with the addition that the current values are saved in the current profile.
    This button is only active if there are changes that have not been applied.
  • Undo
    Undoes all changes that have not been applied.
    This button is only active if there are changes that have not been applied.
  • Import
    A file open dialog appears where you can select a *.varprof file and import the profiles it contains. If there are differences in the existing variant properties (new ones have been added to the configuration, or some have been removed), a warning message appears. This is also in the Write window.
  • Export
    The button expands and you can choose whether only the current profile or all profiles should be exported. Then a file save dialog appears, where you can define the path and filename for the export. The files are saved in *.varprof format (an XML format).
  • Show variant properties only for selected Test Units
    Same functionality as before, only hidden behind a button.
