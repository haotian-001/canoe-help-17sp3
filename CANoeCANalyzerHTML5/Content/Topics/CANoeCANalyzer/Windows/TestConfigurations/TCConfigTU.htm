Configuring Test Units

Context Menu of the test unit: Configuration...

Common

Here, you see the name under which the test unit is displayed in the CANoe configuration. The name does not have to match the test unit name from vTESTstudio. However, it must be unique within the CANoe configuration. You can change the name directly in the list of test units.

Test Report

Test Report

If an overall test report for all test units is not to be created (see setting in the configuration dialog of the test configuration), you can specify via the Enable test report generation check box whether or not a test report is to be created for this test unit.

Output Options

Test Report Path

Specify the filename of the test report file in the Test report path input box:

  • use your keyboard to type in a file path in the field
  • use the Field Codes button to create the test report file path
  • use the […] button to open a file selection dialog

An existing file will be overwritten when the test configuration is executed.

Test Report in XML/HTML Format

If you selected the XML/HTML test report format in the global CANoe options, you can activate an additional test report in HTML format by selecting Create HTML test report using the following XSLT stylesheet. For that purpose the XML test report file is transformed into an HTML test report file after the test has been performed. This HTML test report has the same filename as the XML test report, except that the extension is HTML instead of XML.

If you don't want to use the standard transformation script, you can select a different XSLT transformation stylesheet by clicking the [...] button.

If you want to used a different XSLT stylesheet once, you can select the XSLT file in the execution dialog in the Test Report menu and used it for an immediate transformation of the last generated XML file.
