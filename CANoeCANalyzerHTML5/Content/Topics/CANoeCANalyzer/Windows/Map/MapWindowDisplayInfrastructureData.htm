Display of Infrastructure Data

 

The Map Window automatically displays received infrastructure data in the map:

MAP (MAP Data Message)

Geometric data (typically static) of intersections are displayed. The tooltips of the individual elements contain additional data, e.g. the type and the width of a lane:

Object/Element

Display

Comment

Intersection ID

The IntersectionID is displayed at the reference position refPoint. The IntersectionID should be unique. All SPaT data are displayed according to the IntersectionID.

Vehicle Ingress Lane

Ingress lane for motor vehicles The stop line is located at the border of the lane element (node) which is closest to the intersection (first node).

Vehicle Egress Lane

Egress lane for motor vehicles

Various other lanes

e.g. crosswalk, bike, etc.

SPaT (Signal Phase and Timing Message)

Dynamic states of traffic light signals are displayed. As a precondition for the display a MAP with the associated IntersectionID has been received already.

For display of the signal state (MovementPhaseState) always the borders of the ingress and the egress lane which are located closest to the intersection (first node each) are connected with a line. The tooltip contains additional data, e.g. the signal group (SignalGroupID).

Priority for public transport and emergency vehicles

In addition to the infrastructure data of the intersection, the priority for public transport and emergency vehicles is automatically displayed in the Map Window:

SREM (Signal Request Extended Message)

For each signal request an arrow is drawn away from the requesting station. The lanes belonging to this request are outlined in the color of the request type.

The request is displayed in the detail view in the format: Request by S. <StationId> for I. <IntersectionId>.

The following RequestTypes are displayed:

SSEM (Signal Status Extended Message)

For each Signal Status, an arrow is drawn pointing in the direction of the requesting station. If the position of the station is not known, the arrow is drawn at the intersection. The lanes belonging to this request are filled with color of the signal status.

The Signal Status is displayed in the detail view in the format: Status for S. <StationId> by I. <IntersectionId>.
