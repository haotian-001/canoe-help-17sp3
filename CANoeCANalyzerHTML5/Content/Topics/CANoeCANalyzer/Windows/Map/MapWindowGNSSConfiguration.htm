GNSS Configuration

Ribbon: Hardware Tab|GNSS Configuration

This dialog allows you the configuration of GNSS receivers for usage in CANoe. The received data will be available as system variables in CANoe. You can visualize this data directly in the Map Window.

Note

You can configure up to 4 GNSS receivers.

Managing GNSS Receivers

With the button you can add a GNSS receiver to the configuration.

With the button you can delete a GNSS receiver from the configuration.

Via the Configuration menu you can import and export GNSS configurations as XML file. This functionality is useful to reuse an existing GPS configuration in different CANoe configurations.

Configuring GNSS Receivers

On the left hand side all configured GNSS receivers are displayed.

On the right hand side the settings of the selected GNSS receiver are displayed and can be configured there.

General Settings

Receiver name

Use this name to identify a certain GNSS receiver more easily.

System variables

For every GNSS receiver you have configured two system variable structures are created. These contain a basic set as well as an extended set of GNSS information. If the corresponding option is selected, enhanced GNSS information will be generated, too.

Data Source

In CANoe you can select GNSS data from three different data sources:

GNSS Receiver

If you have a GNSS receiver, you can connect the device via the COM interface. You will need to specify the corresponding COM interface channel in the dialog.

Alternatively you can start an automated scan for GNSS receivers connected to the computer. If you are using CANoe in distributed mode the scan will be performed on the realtime module. Incoming GNSS data from the GNSS receiver will be available as system variables in CANoe.
With [...] you open the GNSS Receiver Search dialog, to display the connected GNSS receiver.

Save .gpslog files in folder (logging of GNSS data)

This option is only available for real GNSS receivers. When activated the outputs of the GNSS receiver will be written to a GPSLOG file. These include all relevant NMEA messages coming from the receiver. In addition you can choose the directory in which the GPSLOG files will be stored. For every measurement in CANoe a new file will be created. The file is named using the receiverâs name, the current date and time.

Note

The specified directory will be interpreted with relation the computer running the realtime area of CANoe. If you use the distributed mode of CANoe you have to specify a directory on the remote computer.

If you use a realtime module (e.g. a VT6000 or a VN8900) the settings for the logging folder will have no effect. Instead the logging files will be stored in a specific folder on the remote device. To download the logging files from such a device use the Standalone Manager built into CANoe.

If you want to analyze or process the data at a later date with CANoe, e.g. convert GNSS information into a readable CSV format, you can use the Logging Block to log GNSS data in BLF or ASC format.

GNSS Logging File

If you are logging GNSS data in a GNSS log file with GPSLOG format, you can specify this file as the data source. Logged GNSS data is displayed as GNSS events directly on the card and/or in the Trace Window.

J1939 Messages

If you have selected J1939 or NMEA messages as the data source, the messages used are those defined by J1939 or NMEA as default GNSS data carriers.

You will require the corresponding J1939 optionor ISO11783 option.

Note

  • I you have selected J1939 Messages as data source, J1939 messages are used both in Online Mode as well as in Offline Mode.
  • For other data sources in Offline Mode the GNSS system variables from the logging file are used.

Via the Settings button you can open the J1939 Settings dialog where you can do further Settings.

None

If this data source is selected, no GNSS data will be generated. In this case the GNSS system variables will be created but not written to by CANoe. Instead they can be written manually (e.g. by signal generators or a CAPL scripts).

GNSS Receiver in Offline Mode

If you play a logging file in offline mode of CANoe, you can use the contained GNSS data by the GNSS integration. To do so you have to configure one or more receivers in the GNSS configuration dialog.

An additional configuration of the data source is not required.

  • If you have selected J1939 Messages as data source this setting will be used both in online mode as well as in offline mode. I.e. in offline case GNSS information form the J1939 messages in the logging file will be extracted and assigned to the GNSS receiver.
  • GNSS receivers with other data sources get the GNSS information directly from the logging file. The data will be assigned to the configured GNSS receivers automatically.

Note

In offline mode you can change the values of GNSS system variables via a CAPL node (inserted on the left of the vertical line in the Measurement Setup). That way you can write the GNSS system variables with data.

Example: GNSS information is transmitted with CAN signals and mapped to a GNSS system variable in offline case.

{
  SysBeginVariableStructUpdate(sysvar::GPS::GPS1);

  @GPS::GPS1.Longitude = getSignal("GPS_x");
  @GPS::GPS1.Latitude = getSignal("GPS_y");
