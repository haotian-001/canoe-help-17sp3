Integrate Local OpenStreetMap® Map Data

 

The Map Window provides the possibility to use OpenStreetMap® offline map data. The data is stored as tiles. You can either use sample tiles or create tiles tailored to your requirements.

Local OSM map data is especially useful if you do not have direct access to the internet during the usage.

Use Sample Tiles

You can select local map data on the page Map Window (Options dialog). In the area OpenStreetMap® click on Local map data and select the path corresponding to the used protocol.

Sample Tiles for EU Protocols

You can find the local map data under OSM_Tiles parallel to the configuration files Car2xSystemDemo.cfg and Car2xSignalSendingDemo.cfg. The tiles show a map section of Stuttgart.

Sample Tiles for US Protocols

You can find the local map data under OSM_Tiles parallel to the configuration files V2xSystemDemo.cfg and V2xSignalSendingDemo.cfg. The tiles show a map section of Detroit.

Create OSM-Tiles

You can generate tiles from free available data extracts packages from OpenStreetMap project. To generate the tiles, the free software Maperitive is needed. Afterward, the tiles can be used easily in the Map Window.

Note

The license terms for the use of OpenStreetMap maps are available here.

Additional Software

Maperitive (http://maperitive.net/) is a FREE desktop application for drawing maps based on OpenStreetMap and GPS data.

Note

Maperitive uses a big amount of memory (RAM), therefore it is recommended that you install the software on a computer with at least 4 GB RAM.

Step by Step Procedure

  1. Download the latest version of Maperitive (ZIP file).
  2. Extract the ZIP file into any folder.
  3. Start program (file Maperitive.exe).

Get the Map Data

Maperitive is delivered without any map data. Therefore, map data must be downloaded or accessed online.
A possible source for map data is, for example:

  • https://download.geofabrik.de/

This provider makes free map data available and offers the possibility to create user-defined map data.

It is recommended to narrow as much as possible the source map data to reduce the tiles extraction time.

Example

Map data for Stuttgart and its region

https://download.geofabrik.de/ -> Europe -> Germany -> Baden-Württemberg -> Regierungsbezirk Stuttgart -> stuttgart-regbez-latest.osm.pbf

In order to access the maps online, click Download OSM Data in the Menu Map of Maperitive. The maps are then downloaded automatically. If access is not possible due to missing proxy settings, enter the following into the command line of Maperitive.

  • set-setting name=web.proxy.host value=YourProxyName
  • set-setting name=web.proxy.port value=YourProxyPort
  • set-setting name=web.proxy.username value=YourProxyUsername
  • set-setting name=web.proxy.password value=yourProxyPassword
  • set-setting name=web.proxy.use-proxy value=true

Once configured, the settings are saved in the User Settings file of Maperitive and are used each time Mapertive is started.

Open the Map Data

In case you want to load downloaded map data, you can open it in Maperitive after the download (the file has the extension OSM.PBF).

Note

Maperitive extracts the map data and does a realtime rendering of the map. This may take a while (depending on the size of the map data several minutes).

Set Geometry Bounds

To reduce the extraction time and the amount of map data use the possibility to set map bounds (menu command Map | Set Geometry Bounds). A red rectangle appears which defines the geometry bounds. Clicking on the rectangle to activate it and modify the size and position of the geometry bounds.

Example

The region around the headquarter of the Vector Informatik GmbH was selected.

Generate Tiles

To extract the map data of the specified geometry bounds and convert them into tiles type following command into the Command prompt field at the lower edge of the Maperitive program window:

generate-tiles minzoom=1 maxzoom=18 tilesdir="D:/tiles/" <ENTER>

This will start the tiles generation with a minimum zoom of 1 and a maximum zoom of 18. The destination folder for the generated tiles is D:/tiles/. See Maperitives documentation for a full explanation of all available commands (Help | Maperitive Documentation | generate-tiles Command).

Note

The tiles generation may take a while (depending on the size of the map respectively the geometry bounds, and the defined zoom levels). E.g. the tiles generation of the region around the headquarter of the Vector Informatik GmbH (see example above) needed about 45 minutes on a standard computer.

Use Tiles in the Map Window

After generating the tiles the folder can be copied into the folder which you have defined as OSM-offline folder in the CANoe Options dialog. Then the Map Window uses the offline map data.

Note

Depending on the tiles generation parameters it may be possible that no map data is available in any regions and zoom levels. This is marked with the icon in the upper left corner of the tile.
