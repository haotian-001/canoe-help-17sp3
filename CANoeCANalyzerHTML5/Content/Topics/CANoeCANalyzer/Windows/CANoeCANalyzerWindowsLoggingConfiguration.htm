Graphics Window / Data Window: Signal Logging Configuration

Context Menu Graphics Window / Data Window: Logging Configuration…

In the Graphics Window or Data Window you can record (log) the configured symbols during the measurement.
In the Signal Logging Configuration dialog the following settings are available to configure the logging file.

  • Destination folder
    Here you select the folder to which the logging file is to be written.
  • Destination file
    Here you specify the name of the logging file. Field functions support you when specifying the name of a logging file.

    • Field Codes

    With the field codes it is possible to generate and increment filenames and directories.

    Advantages:

    • Existing files can be protected against overwriting.
    • After the end of measurement you don't have to enter new filenames constantly.
    • The logging recording can be split into several (smaller) logging files
    • which are easier to be sent
    • which are easier to be analyzed

    System

    Field Code

    Description

    User Name

    Windows login name.

    Computer Name

    {Computername}

    Name of the computer.

    Local System Time

    {LocalTime[|Format]}

    Current local system time.

    The default of the time stamp format can be set in the global options. In the field code you can overwrite the default setting.

    CANoe

    Changes

    • The {IncMeasurement} field code has been replaced by the {MeasurementIndex} field code.
    • Only for Logging Blocks:
      The {Date} {Time}{YYYY}{MM}{DD}{hh}{mm} and {ss} field codes have been replaced by the {MeasurementStart} field code.

    Field Name

    Field Code

    Description

    Configuration Name

    {ConfigName}

    Name of the CANoe configuration file.

    Measurement Index

    {MeasurementIndex}

    The measurement index can be set in the global options and is incremented on each measurement start.

    The measurement index is stored in a separate file for system parameters independent of the configuration.

    Measurement Start

    {MeasurementStart[|Format]}

    Time stamp of the measurement start.
    The default of the time stamp format can be set in the global options. In the field code you can overwrite the default setting.

    Program Version

    {Version}

    CANoe version.

    Logging

    Field Name

    Field Code

    Description

    Increment at each Trigger

    {IncTrigger|001}

    Each trigger can be distributed to an individual logging file. The filename is incremented after each trigger.

    You can predefine a start value in the field code.

    Increment at each Named Trigger

    {IncTrigger|001}_{TriggerCondition[|length]}

    Combination of the field codes Increment at each Trigger and Trigger Condition Name.

    Increment by File Size

    {IncSize|001|100MB}

    With this function the logging recording can be split into several (smaller) logging files.

    There is a new logging file when the entered size of the file is reached; the filename is incremented automatically. The size has to be entered in MB

    You can predefine a start value in the field code.

    Increment after time

    {IncTime|001|01d01h00m}

    With this function the logging recording can be split into several (smaller) logging files.

    A new logging file is generated when the entered time is reached; the filename is incremented automatically. The beginning of the time slot is either a start of measurement or a new incoming message. The time can be set in days, hours and minutes:

    • Days (d): 0-99
    • Hours (h): 0-23
    • Minutes (m): 0-59

    You can predefine a start value in the field code.

    Logging Block Name

    {LoggingBlock}

    Name of the logging block.

    Logging Condition Name

    {TriggerCondition[|length]}

    Name of the trigger condition.

    Depending on the selected trigger mode, different names are resolved:

    • Single Trigger: SingleTrigger
    • Toggle Trigger: ToggleTrigger
    • Entire Measurement: NoTrigger
    • Test Trigger: Name of the corresponding test element that activates the trigger.
      Example: Suppose you have a test configuration PowerTrain with a test unit GearBox and a test case GearCheck that activates the trigger. Then, the field code is resolved to PowerTrain_GearBox_GearCheck. If the trigger would react on test units, the field code would be resolved to PowerTrain_GearBox. If the trigger would react on test configurations, the result would be PowerTrain.

    Independent from the trigger mode, you can optionally define a certain maximum length. If the resolved name is longer than the specified value, the string is cropped accordingly.
  • File Format
    For logging the data you can use the MDF format (MDF Signal Logging Files, *.mdf, Version 2.0–4.1, signal-based).

Measurement Data Format (binary format)

  • With this format signals are extracted from messages and binary stored in signal-based MDF files (MDF Signal Logging Files).
  • These MDF Signal Logging files can easily be imported into CANape, vSignalyzer, CANoe's Graphics- and Data-Window or third party tools.
  • Support for statistic information, CAN signals, LIN signals, FlexRay signals, GPS signals, J1939 signals and J1587/J1708 signals. MOST signals are not supported.
  • Always absolute time stamps are logged.

Note

This formats only logs signals; message information will be lost.

Advanced Settings

  • With Log entire measurement you can specify whether the symbols configured in the Graphic/Data Window shall be logged automatically beginning with the measurement start. Alternatively you can always manually start/stop the logging via the Analysis ribbon tab symbol .
  • With Warn before overwriting logging file you can specify whether a warning prompt should appear at the start of measurement before overwriting an existing measurement file.
  • With View logging file(s) after measurement stop the logging file(s) will be opened after measurement stop automatically in an editor.
  • Global logging settings: With [Settings...] you open the Options dialog to make all settings for all logging blocks of the configuration.
