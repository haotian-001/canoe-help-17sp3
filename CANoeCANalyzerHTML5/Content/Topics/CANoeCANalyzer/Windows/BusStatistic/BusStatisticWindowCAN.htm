Bus Statistics Window: CAN

 

The Bus Statistics Window (also) displays CAN-specific data and values.
For CAN the following information are available in the Bus Statistics Window:

  • Busload [%]
  • Peakload [%]
  • Std. Data [fr/s]
  • Std. Data [total]
  • Ext. Data [fr/s]
  • Ext. Data [total]
  • Std. Remote [fr/s]
  • Std. Remote [total]
  • Ext. Remote [fr/s]
  • Ext. Remote [total]
  • Error Frame [fr/s]
  • Error Frames [total]
  • Chip State

The momentary and maximum busloads since the measurement start are displayed in percent. The CAN controller averages these parameters according to the configured Sampling period, which can yield different values depending on the setting.

Error State (Chip State)

Also the Bus Statistics Window displayed are error status information on the individual CAN controllers and the momentary readings of their respective error counters. The following states may occur:

  • SIMULATED
    CANoe is in simulated mode. Bus access (transmission and receipt of messages) is simulated by CANoe. The hardware is not accessed, which means that the controller status cannot be displayed.
  • ACTIVE
    Normal state.
    The microcontroller reacts to bus errors by sending out an Error Frame consisting of 6 dominant bits.
    The CAN controller state is Error Active.
  • WARNING
    The Warning state in the Bus Statistics Window indicates a special CAN controller state. Active means that the CAN controller is reacting to bus errors and that it is in the Error Active state.
    In this case, however, the warning stage (96 Error Frames) has been reached, which is documented in the Bus Statistics Window as Warning.
  • PASSIVE
    The controller has entered the Error Passive state.
    It reacts to bus errors with an Error Frame consisting of 6 recessive bits. Please refer to the CAN specification for further details on this behavior.
  • OFF
    The microcontroller has been disconnected from the bus.
    The CAN controller is in the Bus Off state.
    It is not reactivated until a restart has been executed.

Note

When using virtual CAN channels the following data will not be evaluated:

  • Controller state (Active, error passive, bus off, ...) will not be emulated

    After starting chipstate always shows Active on virtual channels. Since the controller can always send and receive messages it is correct to display Active.

    When Error Frames occur however the display doesn't change but remains as Active. In this case the display in the Bus Statistics Window isn't absolutely correct.
  • Baudrate will be ignored
  • Transmission period of a message is nearly zero.

The following functions are possible:

  • Error Frames can be generated
  • Messages with standard and extended identifiers can be sent
  • TXRQ is being generated
