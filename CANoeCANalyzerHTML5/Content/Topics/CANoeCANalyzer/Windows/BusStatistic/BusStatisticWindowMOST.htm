Bus Statistics Window: MOST

 

The Bus Statistics Window shows some statistical data and the last valid bus state like frame/s, frames(total), number of changes of Light & Lock and so on. These values are calculated by the Vector MOST driver.

For a channel operated in node mode, only the frames addressed to this node are included in the calculation. However the values are calculated before any filter or other CAPL node can influence the number of the received frames. If the spy is activated for this channel, only the Spy messages will be counted.

Edit the sampling rate

The sampling rate with which the statistic analysis is processed may be configured in the CAN.INI file:

[MOST]
// Sampling rate of bus statistic (value in ms)
// -1: Off, 100-.. valid values (default=1000)
Statistic = 1000

The Bus Statistics Window (also) displays MOST specific data and values.

For MOST the following information are available in the Bus Statistics Window:

  • Frames [fr/s]
    Messages/frames - per second.
    More precisely: frames transmitted on the control channel.
  • Frames [total]
    Received messages/frames - total.
    More precisely: frames transmitted on the control channel.
  • Lgt&Lck [L&L/s]
    Light & Lock errors - per second.
    The MOST Interface reports L&L errors whenever it recognizes that there is no light (input) or that the synchronous timing of the ring broke down. An error is recognized and reported (only) once, e.g. when the light is switched off. As a result, the MOST Interface cannot receive any more, but it doesn't continue reporting errors.
  • Lgt&Lck [total]
    Light & Lock errors - total.
    The MOST Interface reports L&L errors whenever it recognizes that there is no light (input) or that the synchronous timing of the ring broke down. An error is recognized and reported (only) once, e.g. when the light is switched off. As a result, the MOST Interface cannot receive any more, but it doesn't continue reporting errors.
  • Buffer Lvl. [%]
    Load of the OptoLyzer's input buffer.
  • Packets [pkt/s]
    Received packets - per second.
    More precisely: packets received on the asynchronous channel.
  • Packets [total]
    Received packets - total.
    More precisely: packets received on the asynchronous channel.
  • Coding Errors [CE/s]
    Detected coding errors - per second.
  • Coding Errors [total]
    Detected coding errors - total.
  • Frequency [kHz]
    Frequency of the MOST ring.
  • State
    Light & Lock states:
    • no Light
    • no Lock
    • Lock
    • stable lock
    • critical unlock
  • AMSMessages [msg/s]
    Received AMS messages - per second.
  • AMSMessages [total]
    Received AMS messages - total.
  • AMSTelegrams [tel/msg]
    Average number of telegrams per received AMS message.
  • AMSDuration [ms/msg]
    Average transmission time of an AMS message.
  • AMSRxError [total]
    Number of erroneous received AMS messages.
  • AMSTxError [total]
    Number of erroneous transmitted AMS messages.
