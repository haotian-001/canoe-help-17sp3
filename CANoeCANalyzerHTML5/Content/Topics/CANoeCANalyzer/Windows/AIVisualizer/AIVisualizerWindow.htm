AI Visualizer

Ribbon|Analysis Tab|AI Visualizer|View / New

With the AI Visualizer (AI=Artificial Intelligence), data such as images and AI results exchanged between an AI component and CANoe can be visualized.

Toolbar

Symbol

Description

Enlarges the image section.

Reduces the image section.

Fits the input image into the window.

Shows/hides rulers at the top and left edge of the window.

Shows/hides borders around detected objects.

Shows/hides the input image.

Window

The right pane graphically displays viewable data exchanged between CANoe and an AI component. This includes, for example, input images for the AI component or outlines of found objects by the AI. For each object to be visualized, an element is created in the AI Visualizer.

In the splitter on the left, the elements are listed in a tree view and their properties are displayed.

In the tree view, you can show and hide individual elements.

In the sample configuration AISilTestingAnalysis you will find detailed information on how to address a System Under Test (SUT) with an AI component via CANoe.

Creating Elements

The elements can be created dynamically in CAPL or C# using a DO interface (IGraphicalElementFactory). To use the API, the GraphicalElementsAPI.vcdl file must be loaded in the Communication Setup. The VCDL file can be found in the sample configuration AISilTestingAnalysis in the DataSources directory (open folder).

After that the API is available with the following methods:

interface IGraphicalElementFactory
{
  internal method string GetOrCreateElementContainer(in string id);
  internal method string GetOrCreateElement(in string id, in string container, in EElementType elementType);
}

The elements are organized in containers, which are also created programmatically. First create a container (IGraphicalElementContainer), to which you then assign the created elements.

The following element types can be created via the API:

You can define color values and create text using the ColorData and TextData structures.

C# example of creating a container with a rectangle

var MyContainer1 = GetOrCreateElementContainer("MyContainer1");
if (MyContainer1 != null)
{
  MyContainer1.container_data.position.x.ImplValue = 10:
  MyContainer1.container_data.position.y.ImplValue = 10;
  MyContainer1.container_data.dimension.width.ImplValue = 200;
  MyContainer1.container_data.dimension.height.ImplValue = 200;

  Var MyRectangleElement = GetOrCreateElement("MyRectangleElement", "MyContainer1", _GraphicalElements.DataModel.EElementType.Rectangle)
                           as _GraphicalElements.DataModel.IRectangleElement;

  MyRectangleElement.element_data.position.x.ImplValue = 10;
  MyRectangleElement.element_data.position.y.ImplValue = 10;
  MyRectangleElement.element_data.dimension.width.ImplValue = 150;
  MyRectangleElement.element_data.dimension.height.ImplValue = 100;
  MyRectangleElement.shape_data.stroke_size = 4;
  MyRectangleElement.shape_data.fill_color.R.ImplValue = 100;
  MyRectangleElement.shape_data.fill_color.G.ImplValue = 255;
  MyRectangleElement.shape_data.fill_color.B.ImplValue = 0;
  MyRectangleElement.shape_data.fill_color.A.ImplValue = 0;
}

For more application examples, see the AISilTestingAnalysis sample configuration in the PanelHelper_DataMediator.cs file, #region Auxiliary methods section (open folder).
