MOST FBlock Monitor

Ribbon|Analysis Tab|FBlock Monitor

The MOST FBlock Monitor displays the MOST device assignment, the reported functions and the most recently observed property values for all function blocks whose communication on the MOST ring has been observed by the spy.

The user interface is divided into four sections:

System view

The system view shows a list of all devices and their function blocks. This list comprises the devices and function blocks identified on the basis of different messages observed in the MOST ring Obtaining Information. Function blocks are displayed in black if the respective function block has actually received status messages and the existence of the function block can therefore be considered as proven or if the NetBlock or MOST NetworkMaster has reported the existence of the function block. Otherwise the function blocks are displayed in gray, especially after the start of a new measurement if the previous list is not cleared from the display.

Toolbar:

Close the function block tree.

Open the complete function block tree.

Update the list of function blocks. For this purpose the NetworkMaster.CentralRegistry.Get message is sent and the corresponding status message is evaluated. This button can only be used when a measurement is in progress and the network interface bypass is deactivated.

Clear all function blocks from the tree that are currently displayed as not available (gray). This update can be useful when function blocks have been taken over from a previous measurement but the system for the new measurement has been modified significantly. It can also be used for systems in which the instance IDs of the function blocks change in the running system.

Catalog view

The Function Catalog view displays all the function blocks described in the function catalog. When a function block is selected the function view displays all the functions described in the catalog. These can be added to a custom view via drag and drop or by means of the commands in the context menu. In this case the relevant instance ID of the function block has to be selected, since it is not known in the Function Catalog view.

Function view

In the Function view, all methods, properties and property values observed in the ring are displayed.

When the System view is activated, the functions displayed are related to the function block selected in the System view. When the Function Catalog view is activated, all the functions defined in the catalog for the selected function block are displayed. If a custom view is selected, all the functions configured in this view are displayed.

All parameter names, their most recently transmitted values (column: value), the transmitted raw data (column: raw data) and the unit and function class defined in the function catalog are displayed. The Value column shows the physical values calculated on the basis of the conversion formula from the function catalog. Conversion formulas describe, for example, the decimal position for fixed-decimal values, the symbolic values in enumerations and the encoding of strings.

  • COLOR CODE: The color code in the second column indicates:

A valid status message has been observed since the start-up of the network.

The function was only identified in the list (FktIDs.Status) but no corresponding status message has been observed yet.

No information is available yet regarding the given function.

The function has been clearly identified as not available (for example, a request to this function returned a "FctID not available" error).
  • GET: The [Get] button can be used to request the current value of a property. This is only possible if the network interface is configured as a node and the Get message does not have to be parameterized.
  • CHECKBOX: When a custom view is activated, it is possible to set a notification for the selected property (see also Notification Shadow Service). This means you can observe the state of function blocks that normally are not transmitted (or are not regularly transmitted) on the bus. When this check box is activated, and from this point on whenever necessary according to the MOST specification, CANoe sends a corresponding Notification.Set(Function, DeviceID) message to the function block. For this to work, the property has to support the notification. Whether this is the case cannot currently be determined based on the function catalog. From this point on, whenever there is a change in status the corresponding function block sends the current value to the node address where CANoe is running.

Note

A resulting load increase on the bus may affect the system. Apart from the busload, no other effect on other applications is to be assumed since the information is sent directly to the CANoe address and cannot normally be received by other devices.

The parameter values are not supported by all function classes. For example, the parameters for methods and some property classes have no significance outside of the specific report message.

Toolbar:

Close the function tree.

Open the complete function tree.

The list of functions related to the selected function block can be actively updated. (This is done by sending the FktIDs.Get message to the function block.)

Context Menu

The context menu for the column headings can be used to select the columns to be displayed. The context menu for the selected property rows can be used to add the rows to a custom view. This can also be done via drag and drop.

List of custom views

The list of custom views can contain any number of user-configured views. Unlike the System view, custom views can also display properties from different function blocks. These are added to the custom view either via drag and drop or by means of the context menu for the properties in the function view.

By giving the custom view an appropriate name it is possible to describe the use case for which the view has been configured. Custom views can be used, for example, to observe a certain selection of functions from a bigger function block. Another use case is the collection of all the relevant properties from different function blocks for a specific application (a control unit). By selecting the given view from the list of custom views the overview of selected functions can quickly be call up. User-configured views can also be transferred to other configurations (Export/Import in context menu). By means of the context menu for the column headings you can show or hide individual columns and also group the individual properties according to function blocks with descriptive separators.

Obtaining Information

The list of function blocks available in the system and displayed in the system view is obtained as follows:

  • All FBlockIDs.Status messages from the devices are evaluated, such as they are returned as responses to a ring scan by the MOST NetworkMaster.
  • All messages from the MOST NetworkMaster relevant to the system configuration (e.g. CentralRegistry.Status/Error) are evaluated.
  • The functional addresses (FBlockID, InstID) of the observed status messages are analyzed.
  • The error messages 'FBlockID/InstID not available' are taken into account.

The information for the Function view is obtained as follows:

  • All FktIDs.Status messages from a function block are evaluated.
  • All property status messages are analyzed.

Completion and updating of information:

As mentioned, the MOST FBlock Monitor only displays information sent by the devices on the bus, independent of the tool, since the last system start-up. To complete or update this information, the MOST FBlock Monitor offers several options, provided CANoe is running as a node.

  • Function block list in the System view: message NetworkMaster.CentralRegistry.Get is sent
  • Function list in the Function view: message FktIDs.Get is sent to the respective function block
  • [Get] button: a GET message is sent to the respective function block
  • Set notification: in custom views it is possible to set notifications for selected properties. CANoe then determines the device address of the function block instance – if it is not yet known – and adds a corresponding entry to the target device's notification matrix via Notification.Set(Function,DeviceId) (see also Shadow Service and Notification Shadow Service)

Activating/deactivating:

The MOST FBlock Monitor can be opened via the Analysis ribbon tab. The MOST FBlock Monitor only runs on a MOST channel if the corresponding data collector is activated in the hardware configuration dialog. The Control Spy should also be activated on this MOST channel in order to obtain as much information as possible.

By default, the data collector is activated for the MOST 1 channel in a new configuration.

On computers with low CPU performance or limited memory or in use cases with many bus systems attached the required processing capacity can be reduced by deactivating the data collector. The MOST FBlock Monitor then displays a corresponding message in the title bar of the Function view.
