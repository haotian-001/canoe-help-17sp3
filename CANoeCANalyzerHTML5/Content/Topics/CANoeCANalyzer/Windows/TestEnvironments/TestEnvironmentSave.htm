Save / Save As

Context Menu of a Test Environment: Save / Save As…

To save a Test Environment select it with the right mouse button in the Test Setup for Test Modules and select the context menu command Save.

The test environment is now saved with all settings (activated/non-activated blocks, expansion status of the directories).

If the test environment was not yet saved, you will be asked to specify the storage location and the filename.

If you want to save a test environment under another name, select the function Save as... in the same place.

All test environments can be saved at the same time using the context menu in the Test Setup for Test Modules.

If you save or close the CANoe configuration, you will be asked for the filename for all test environments that were not yet saved in a file.

Note

  • Operation Save

If the test environment is loaded from older configuration or it is saved in older version it will keep its old version as long as it is changed by user explicitly.
When saving the test environment "Warning" dialog is opened if the test environment version is older than the current version.
Select [Yes] to keep the original version or [No] to save the configuration in latest version.
If the check box Don't show this dialog again is set the dialog will not opened anymore and the test environment will automatically be saved in version selected by user.
Caution:
Saving to a newer version: If multiple CANoe installations use the same test environment, then ensure that the current test environment version is not newer than the oldest used CANoe version.
Saving to an older version: The test environment can be saved successfully to an older version if only compatible features are used in the test environment and in all referenced sub-components.

  • Operation Save As

If the test environment version is changed in the Save as dialog user has to confirm/deny the version change operation in the "Warning" dialog opened after Save as dialog is closed.
Select [Yes] to change the version or [No] to keep the original version.
If the check box Don't show this dialog again is set the dialog will not opened anymore and the test environment will automatically be saved in version selected by user.
Caution:
Saving to a newer version: If multiple CANoe installations use the same test environment, then ensure that the current test environment version is not newer than the oldest used CANoe version.
Saving to an older version: The test environment can be saved successfully to an older version if only compatible features are used in the test environment and in all referenced sub-components.
