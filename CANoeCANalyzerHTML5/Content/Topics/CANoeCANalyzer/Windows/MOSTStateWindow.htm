MOST State Window

Ribbon|Analysis Tab|State

The MOST State Window gives the user a perspective on important system and node states on a MOST ring. In the MOST State Window one status line is shown for each MOST channel in CANoe.

Red icons are used for error states (e.g. no Light) so that the user can recognize at a quick glance whether something is wrong with the state of the ring. Green icons are shown for states that do not represent any errors g (e.g. Lock).

All states are unknown before the first measurement start. When the measurement is stopped the displays are no longer updated, and they show the last momentary states.

The node position is shown numerically. If the node position is unknown (e.g. no NPR register event received yet) then '---' is shown. Otherwise it is represented as a decimal number. In case the bypass is closed, the "virtual" node position is displayed in brackets and italic (e.g. [3]). The "Virtual" node position [i] indicates, that the connected interface is behind the node with position i-1. In case [i] = MPR the connected interface is just before the timing master.

The node addresses and group addresses are also shown numerically. If an address is unknown a '?' is displayed. If it is known it is shown as a hexadecimal number. The addresses can be changed in the respective input fields during measurement. Therefore select on field with the mouse and set the desired address (hexadecimal). For group addresses only values from 0x300..0x3FF can be set.

The register values of SBC and MPR are also shown numerically. If a value is unknown a '?' is displayed. Furthermore, invalid SBC register values (value range: [0x06..0x0F]) are highlighted by red background color of the cell.

The supplemental services AMS (Application Message Service) and AS (Application Socket) are defined with the channel options in the CANoe hardware configuration. If a service is activated it is shown with the icon. If it is deactivated it is shown with the icon.

Actions

  • Wake-up
    ... You can wake-up the MOST ring optically, by executing a double click on the light icon .
  • Twinkle (only VN2600/2610)
    ... You can twinkle the power LED of the attached MOST interface, by executing a double click on the MOST channel (e.g. "MOST 1")

Context Menu

The context menu in the MOST Status Window is referenced to the respective MOST channel. The following entries are available:

  • Trigger ring scan  (only as Slave or Spy)
    ... through a Network Change Event (NCE) the ring scan by the MOST NetworkMaster is triggered
  • Wake-up
    ... optical wake-up of the MOST ring
  • Shutdown (only MOST150/MOST50, only when bypass is not active)
    ... shuts down the MOT ring via the shutdown flag
  • Tx Light off
    ... enables or disables the output for the MOST signal
  • Bypass active
    ... activates or deactivates the bypass. In case the bypass is active, this is marked by a check box on the left of the menu command.
  • Set timing mode: Master/Slave
    ... set the respective timing mode
  • Static Master (only MOST150, only timing mode: master)
    ... sets the master mode (static / non-static)
  • CtrlSpy active (only VN2600/VN2610)
    ... activates or deactivates the spy on the control channel. In case the spy is active, this is marked by a check box on the left of the menu command.
  • AsyncSpy activ (only VN2600/VN2610)
    ... activates or deactivates the spy on the asynchronous channel. In case the spy is active, this is marked by a check box on the left of the menu command.
  • Set node address
    ... activates the input for changing the node address
  • Set group address
    ... activates the input for changing the group address
  • Apply to hardware configuration
    ... stores the current hardware settings to the hardware configuration. Thereby the settings of the timing mode, bypass*, CtrlSpy, AsyncSpy, node and group address are applied.
  • Twinkle (only VN2600/VN2610)
    ... Twinkle the power LED of the attached MOST interface

* The bypass state is only considered, if it was set via the context menu of the MOST State Window.
