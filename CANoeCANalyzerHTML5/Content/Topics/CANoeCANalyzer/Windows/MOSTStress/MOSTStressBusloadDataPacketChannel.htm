MOST Stress Window: Busload on the Data Packet Channel

 

Data Packet

The settings for busload on the data packet channel are organized into these areas:

Busload

Here you configure how high the busload should be (packets per second) and the total number of packets to be sent.

  • The busload can be set to a value between one packet per second and a maximum.
    This maximum, shown to the right of the input box for reference, will depend on the packet size, bus frequency (typically 44.1 or 48 kHz) and value of the SBC register.
  • If you specify a total number of packets, exactly this number of packets is generated at the configured rate, and afterwards the stress mode is ended.
  • The total number of packets may lie between 1 and 2147483647.
  • Just to the right of the input box is the Unlimited option.
    It enables sending of an unlimited number of packets.
    If this option is activated the busload is generated continuously.

Packet

Here you configure the packet to be used to generate the busload.

  • You can configure the packet's destination address, length, priority and contents (Hexdump).
  • The packet length is automatically adjusted as you edit the Hexdump.
    However, you may manually change the length to a different value afterwards.
  • The [Defaults] button resets the contents (Hexdump) and packet length to their default values.
    This action does not reset the destination address or priority.
  • In the Counter column you can add a passing counter to the generated packets. The width of the counter can be set from 1 to 4 bytes. The counter's position in the packet can be selected in that way that devices receiving these packets will not be influenced by the counter value.

Low Level Retry Parameter (MOST25)

If a message is not accepted by the receiver, the MOST chip attempts to resend the message after a brief wait time (Retry delay).
This is subject to a maximum number of send attempts.

  • Retries may occur if the destination address does not exist in the MOST ring, or if the receiver's Rx buffer is still full and therefore it cannot accept the message.
  • If sending to addresses that do not exist on the ring, or if the addressed device cannot accept the stress mode messages quickly enough, retries will further increase the busload. For this reason it is advisable to set the number of retries (send attempts) to 1. This means that only one send attempt is made per message. This is a way to ensure precise maintenance of the settings made under Busload.

Rx Buffer Disable

In order to selectively induce NAcks on the data packet channel, the button [Disable] can be used to prevent automatic emptying of the reception buffer. Only a maximum of 255 packets or a data quantity of approximately 3 kbytes can now be received. After that, the MOST chip rejects all further data packets sent to the device (the Not Acknowledge bit is set).

  • When the disable is activated, the button [Start disable] changes to the button [Stop disable], which can be used to reactivate the automatic emptying of the reception buffer.

Note

When the disable is deactivated , any data packets received during the disable will be sent to CANoe. These are then assigned the time stamp of the lock cancellation and not the time stamp of their receipt.

The disable also acts on the MOST Ethernet channel (MEP).
