Creating an SCC Configuration

 

Example setup

This section uses an example to describe the creation of an SCC simulation. In this example, a charge point simulation will be set up in such a way that it can communicate with a real vehicle ECU.

Note

Configuring a vehicle simulation can be done in a similar way.

To set up a charge point simulation, follow these steps:

Step by Step Procedure

  1. Open CANoe and create a new configuration. Choose the Ethernet template.
  2. Open the Configuration|Network Hardware… dialog, and choose a suitable Ethernet adapter.
  3. Insert a new network node in the Ethernet network.
  4. Open the network node configuration for the created node.
  5. Under Components, insert the SCC_ChargePoint.vmodule file, which is located in the folder %programdata%\Vector\SmartCharging Components\CCS\v4\Exec32. Close the node configuration dialog.
  6. Open the TCP/IP configuration dialog via Simulation|TCP/IP Stacks… and select the EVSE node. In the section Stack Selection, choose Individual TCP/IP stack instance.



    Next, select the page Ethernet and configure a valid, link-local IPv6 address (starting with FE80:) and MAC address. Make sure that these are not being used by any other adapter on your system.

  7. Save the configuration to a suitable directory.
  8. Create a Ethernet_SCC_ChargePoint.xml file in the same directory.
    Open the file with a text editor and enter the following (actual content will be added in a later step):
    <?xml version="1.0" encoding="UTF-8"?>
    <SCCConfiguration>
    ..<EVSEConfiguration ID="0">
    ....<EVSEID>...(enter ID here)...</EVSEID>
    ..</EVSEConfiguration>
    </SCCConfiguration>
  9. Create a CAPL file for the network node in your configuration directory. Insert the following function:
    on start

    {

      SCC_StartSimulation();

    }
  10. Connect the external device to the selected Ethernet adapter.
