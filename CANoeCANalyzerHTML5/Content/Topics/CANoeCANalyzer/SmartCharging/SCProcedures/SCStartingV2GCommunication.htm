Starting the V2G Communication

 

Advancing to the V2G phase

With the steps in the previous sections, the basics configuration has been finished. It is now possible to initiate communication, starting with the SLAC process.

However, to reach the higher layer of communication (Vehicle2Grid TP), which controls the actual charging progress, additional steps may need to be performed. One reason is that the progression from the SLAC phase to the SECC Discovery phase depends on the successful establishment of a PLC link between the two HomePlug Green PHY chips, as denoted by the variable link status.

Link status handling (Charge Point)

When setting up a charge point, as described in the previous steps, the link status is needed only for information purposes, or to implement custom handling. VS_Host_Action.Ind, as described below, may be used but the simulation itself does not react on the result. However, no further messages will be received until the link is successfully established.

Link status handling (Vehicle)

When settings up a vehicle, the simulation module needs to be notified about the link establishment. Once it is aware that link status = 1, it will start sending the SECC Discovery Request. One of the following methods must be implemented.

VS_Host_Action.Ind

Enabling VS_Host_Action.Ind as described in CCS Hardware Integration, an existing HomePlug Green PHY chip automatically informs about the connection status.

Simulated link status

If SLAC shall be used, but no real HomePlug Green PHY chip is present (e.g. when CANoe is run in simulated mode), or link status detection is not desired, the link status can be set via CAPL.

Fixed delay

As an alternative to the simulated link status, a delay can be specified manually in the XML configuration file. The simulation will then behave as if the link status was switched to 1 after the delay time, starting with the end of the SLAC process. Please refer to <SLAC_DelayTime> .

Note

Practice-approved values for a delay time between the end of the SLAC process and the start of SECC Discovery (when link status = 1) are 6-10 seconds. This corresponds to the time the HomePlug Green PHY chips needs to reboot.

No SLAC usage

If your real ECU is connected directly via Ethernet (no PLC link / no HomePlug Green PHY chip), the SLAC process can be skipped. To implement this:

  • Set link status to simulated in the sample configuration.
  • If you are using the vehicle simulation, set <UseSlac> = 0 in the configuration file to skip the SLAC process.
  • The charge point simulation will always respond to a SECC Discovery Request even if no SLAC takes place.

Extended configuration

With the steps described above, communication will now advance to the Vehicle2Grid TP sequence, which is visible in the Trace Window as TCP/IP messages. Finally, to advance to the charge loop, some mandatory parameters must be configured.

To do this, open the created XML configuration file again and add the following new elements inside <EVSEConfiguration ID="0">. (It is assumed that the ISO 15118‑2 protocol is chosen.)

  • Services: <ChargeService>, <ServiceList>
  • SASchedule: <SAScheduleList>

Element syntax

Please refer to the sample configuration for the correct syntax; you may also reuse the elements defined in its EVSE_configuration file.

Result

If everything has been configured correctly, when starting the measurement the communication should advance until the V2G charge loop. This is visible by V2G messages (blue) in the Trace Window, and the appearance of the message pairs ChargingStatus, CurrentDemand or MeteringStatus.
