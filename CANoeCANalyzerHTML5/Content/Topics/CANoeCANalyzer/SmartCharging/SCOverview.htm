Overview DIN70121/ISO 15118‑2

 

The ISO/IEC TC 22/SC 3 N - Road vehicles — Vehicle-to-Grid Communication Interface specification defines the communication protocol between a vehicle and a charge point.

This help page describes the scope of functions which implements the aforementioned specification and related modeling libraries.

It consists of the following components:

  • Charge point modeling library (SCC_ChargePoint.vmodule)
  • Vehicle modeling library (SCC_Vehicle.vmodule)
  • CANoe Observer for display of SCC events in the Trace Window

Charge point module

The charge point modeling library is responsible for processing the following tasks:

  • Handling of the SLAC protocol for EVSEs
  • Receipt of SCC Requests and automatic sending of SCC Responses
  • Simulation of charge point statuses (e.g. EVSEStatus)
  • Sending of arbitrary SCC and SLAC messages

Vehicle module

The vehicle modeling library is responsible for processing the following tasks:

  • Handling of the SLAC protocol for EVs
  • Receipt of SCC Responses and automatic sending of SCC Requests
  • Simulation of vehicle statuses (e.g., EVStatus)
  • Sending of arbitrary SCC and SLAC messages

Monitor module

The Monitor is a passive module that tracks SCC communication. In contrast to the Observer module, it is a modeling library that provides direct access to the data via system variables. Use for:

  • Man-in-the-middle analysis of a running SCC connection
  • Offline analysis of SCC logging files (usage of Replay Block is mandatory)

SCC Observer

The integrated SCC Observer is used to clearly represent Smart Charging related messages in the Trace Window of CANoe.

  • Display of SCC messages with message name and SessionID
  • Display of all XML elements of the payload as nested signals
  • SCC messages fragmented by TCP are assembled
  • Display of SLAC MME Frames and SECC Discovery messages and their respective signals
