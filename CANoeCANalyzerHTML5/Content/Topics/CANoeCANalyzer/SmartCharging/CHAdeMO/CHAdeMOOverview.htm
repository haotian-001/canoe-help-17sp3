CHAdeMO Overview

 

The  CHAdeMO 3.0/ChaoJi type 2 standard specifies a DC charging standard mainly used in the Asian market. It is the successor of CHAdeMO 2.x and takes a step towards unification of the various DC charging standards in the world.

This help page describes the functional scope of the simulation module implementing the above specification.

The initial release of the CHAdeMO protocol implementation supports:

  • CAN messages H‘100, H‘101, H‘102, H‘108 and H‘109

  • CHAdeMO 0.9

  • CHAdeMO 1.0

  • CHAdeMO 1.2.5

  • CHAdeMO 2.0

  • CHAdeMO 3.0

It consists of the following components:

  • Charger and vehicle modeling library

  • Charger and vehicle interface definition

Modeling library

The simulation module implements the charger and vehicle as so-called DO-Bindings.

CAN fields and HW signals are accessible through DO data members.

Supported CHAdeMO (CAN-) communication data can be presented by DOs in the Trace if CHAdeMO is activated in the communication setup (monitoring).

Interface definition

A predefined interface is used for the CHAdeMO communication objects. Please do not modify the data model (the vCDL file).
