CCS Hardware Integration

 

This page describes how to configure the hardware, the steps necessary to convert a configuration from channel-based to network-based, as well as common problems encountered during the conversion.

QCA Settings

Open the Vector GreenPHY Configurator tool from your CANoe installation or download the latest version from the Vector Download Center.

Make sure that the following settings are applied:

  • configured role = EV or EVSE

  • firmware version = 1.2.5

  • host indications are enabled

The Vector GreenPHY Configurator does not only support the QCA chip but also PLCs from other manufacturers.

Converting a Configuration

  1. Make the following settings in the Options dialog under Bus Systems/ Protocols|Ethernet in the Network Access area:
  1. Activate the Network-based accerss option.
  2. Deactivate the Allow the use of Simulation Ports in the Measurement Setup check box.
  1. Configure your hardware in the Vector Hardware configuration dialog (Hardware|Network Hardware|Driver|[HW Name]):
  1. Switch Ethernet access mode configuration to Network-based mode.
  2. Open Ethernet device configuration and make sure that the network name matches the name in the Simulation Setup or Communication Setup.
  1. Open the Port Configuration dialog and activate in the Measurement Ports area only the HW-Port.
  1. If you see other unused ports in the Port Configuration dialog, you can deactivate them in the Simulation Setup. To do this, simply right click on the corresponding simulation node and disable the Port Generation option.

Troubleshooting

  • Network Hardware|Hardware Configurations contains a XML configuration:

    Please remove all XML configurations from this dialog.

  • Make sure, that in Ethernet device configuration (see 2.a) the Channel Modus of the HW Ports are set to Auto-Negotiation (restricted to 100 MBit â RJ45).

  • Network name has not been defined correctly:

    See 2.b.

Note

If the network used in the Simulation Setup is renamed during the changeover, the corresponding configuration XML for the Simulation DLL must also be renamed. If the CANoe Test Package EV is used, the network name in the file public_UserParameter.vparam must be customized (see also the documentation for the Test Package).
