Test Functions

 

Overview

Test Support

Test functions provide means to influence the SCC protocol outside the bounds of the regular protocol flow. The functions are always available, indifferent to the protocol’s state machine, or if a protocol instance is even running. This means in particular that test functions can be used in passive mode (refer to Simulation Modes and States).

At the moment, all supported test functions are related to the free (e.g. state-machine unrelated) sending of messages. Free Sending of SLAC Messages describes the sending of SLAC messages (MME frames), while Free Sending of SCC Messages describes the sending of SECC Discovery and V2G messages.

Note

Only one of the protocols (SLAC, SDP, V2G) can be used at the same time. Preparing a new message will discard a previously prepared message, regardless of its protocol type.

Step by Step Procedure

To send a message using the test functions, there are three steps:

  1. Create the respective message using one of the Create-functions from Free Sending of SLAC Messages or Free Sending of SCC Messages.
  2. If desired and available, set further parameters using the function SCC_SetOptionalParameter() resp. SCC_SLAC_SetAdditionalParameter().
  3. Send the prepared message using SCC_SendPreparedMessage(). For V2G messages that support signatures, you can choose to send the message with or without signature.

Free Sending of SLAC Messages

Overview

The following functions provide means to send SLAC messages (MME frames) without regarding the connection’s protocol state.

Setting Parameters

As implied above, the parameters for the message to be sent are set via different methods.

  • Variable parameters that depend on the individual SLAC run are set directly in the Create-function.
  • Additional parameters that normally should not be modified can be overridden one by one using the function SCC_SLAC_SetAdditionalParameter(). Lists of available optional parameters are found in the following section below each Create-function. section within the configuration file, where the parameter is to be found.

Note

  • Each Create-function generates a valid message according to DIN70121:2014-12 / SAE J2847/2.

  • Overriding additional parameters will most likely invalidate the message.

    Use this possibility only to create invalid messages on purpose.

Free Sending of SCC Messages

Overview

The following functions provide means to send SCC message without regarding the connection’s protocol state.

Setting Parameters

As implied above, the parameters for the message to be sent are set via different methods.

  • Simple, mandatory parameters are set directly in the Create-function.
  • Simple, optional parameters are set one by one using the function SCC_SetOptionalParameter(). Lists of available optional parameters are found in the following section below each Create-function.
  • Complex parameters (e.g. Services, Schedule Lists), both mandatory and optional, are read from an additional XML configuration file. Instead of the actual parameters, the functions are passed an ID value of the target section within the configuration file, where the parameter is to be found.

Note

  • Only those Create-functions which actually depend on the XML configuration file are passed an ID value.

Configuration of the Test Functions

All complex parameters for the test functions (i.e. all parameters having an XML structure of their own) are configured using the same XML file as the simulation module (e.g. Ethernet_SCC_Vehicle.xml for the vehicle module). However, these parameters are defined in one or more sections labeled TestConfiguration. In contrast to the vehicle or charge point simulation, these sections are not only read once, but the ID of the section has to be supplied each time a function needs to access these elements. This way, the test functions can easily be called with many different parameter values.
