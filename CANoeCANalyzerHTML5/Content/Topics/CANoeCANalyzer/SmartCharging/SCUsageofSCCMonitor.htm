Usage of SCC Monitor

 

Mode of Operation

Distinction from SCC Observer

The monitor module operates similar to the SCC observer. It tracks SCC messages on the selected bus, assembles fragments, and provides the data for analysis. The difference is that while the SCC observer serves for data extraction to the Trace Window, the monitor provides direct access to the data inside the messages.

FAQ

Please note that due to technical limitations, a data object from an SCC message provided by the SCC observer cannot be used like a regular signal, e.g. a CAN signal. In particular, it cannot be dragged to other CANoe windows, and its value cannot be access via CAPL code. The SCC monitor is designed to fill this gap.

Adding the monitor

The SCC Monitor is not active by default. Every SCC configuration can be extended by the Monitor modeling library in a similar way to adding a new simulation module.

Use Cases

Usage of the SCC Monitor is recommended for the following applications:

Man-In-The-Middle

To track the communication between two real remote stations, it is possible to set up a CANoe configuration using both EV and EVSE simulation module. The simulation modules have to execute SLAC in both directions, but need not participate in V2G communication.

The drawback is that analysis of the data is only possible in the Trace Window, as none of the simulation modules is the receiver of any SCC message.

To compensate this, add the SCC Monitor to get live access to the data.

Offline analysis

When replaying a logging file, the SCC Monitor offers the possibility to evaluate the data and timings, run tests and check for errors.

Important Note

Because the Monitor is a component that is only available in online mode, to analyze a logging file it must be played using the Replay Block in the Simulation Setup.
