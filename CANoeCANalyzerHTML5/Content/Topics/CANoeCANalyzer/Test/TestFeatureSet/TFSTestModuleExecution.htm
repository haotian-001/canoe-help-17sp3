Test Module Execution

 

Test modules can be started interactively or automatically:

  • The following variants for starting test modules are suitable for automated execution:
  • Starting the test module via "Start measurement"
  • Starting the test module with a modified environment variable
  • Starting the test module via key combination
  • Starting the test module as part of a sequence of test modules in the test setup

These variants can be specified in the Configuration dialog or by the placement of the test module in the test setup.

  • The following variants for starting test modules are suitable for interactive execution:
  • In the Execution dialog (recommended).
    The Execution dialog is opened by double-clicking on the test module or via the test module context menu.
  • In the Simulation Setup: via the button on the module.
  • In the test setup: via the context menu.

In the Execution dialog for XML test modules you can specify which test cases are to be executed.

Note

  • The execution of a test module is documented in the Write Window
    (e.g., the start and end of the execution of a test case; test case and test module results).
  • The start and end of test case are also displayed in the Trace Window.
    This enables you to quickly gain an overview of which activities belonged to a test case.
  • The execution of a test module is documented in the Test Report.
    The test report can be configured separately for each test module.
