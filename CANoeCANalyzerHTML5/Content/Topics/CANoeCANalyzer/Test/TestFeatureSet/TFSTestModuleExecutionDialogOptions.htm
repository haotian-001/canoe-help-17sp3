Test Module Execution Dialog Options

Context Menu: Execution…

You can set various options for execution of test modules here:

Test Execution

  • A test module can be executed several times in succession (XML test modules only).
    You can define the number and duration of the test runs. If you restrict the duration to a certain amount of time (days, hours, minutes) the current test run will be executed completely even if the specified duration already ended.
    Regardless of repetitive execution only one test report will be generated for all test runs, in which the single test sequences are marked.
    If a test module – that has been configured for repetitive execution – has finished, the verdict remains unchanged if a complete test run has been done. This means that effective with the 2nd test run a test module may be stopped manually without getting the verdict failed.
  • Execution in random order (XML test modules only).
    With this option the test cases will be executed in random order instead of a sequential processing.
    The structure within the test module remains as it is given by the test group. This means that randomizing the test cases only happens within a test group. Therefore Preparation and Completion sections remain effective for the same part of the test module. Furthermore all test cases activated in the execution dialog will be executed exactly once.
    In the test report all test cases will appear in the order of the actual execution, numbered according to their appearance in the test report.

    • With multiple execution and random order you can additionally choose whether the sequence shall be re-determined for each pass or the same random order has to be taken for all passes.

Verdict Impact & Debugging

  • Influence of the verdict (XML test modules only)
    Here you define the test module behavior in case of a verdict Failed.
    You can either stop a test case as soon as the test case Failed or you can stop the whole test module. You can also turn off the influence of the verdict (verdict-independent test sequence).
  • Debug mode
    In debug mode, the test module is paused automatically after the execution of each test module or each erroneous test case. The user must continue the execution by clicking the Start button.
  • Pausing mode (XML test modules only)
    You can specify whether a pause takes place after each test function/control function or only after test cases. A pause is only possible after test cases in CAPL/.NET test modules.
