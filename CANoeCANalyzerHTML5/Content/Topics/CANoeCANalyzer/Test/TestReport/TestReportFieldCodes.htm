Report Field Codes

Toolbar of the Test Configuration Window: Configuration...

In the Report area of the configuration dialog for a test configuration you find the Field Codes button. With this button you can insert in the file path different field codes that will generate dynamic report file paths. At the end of the test the field codes will be replaced with their evaluated values and the final string will represent the report path.

The example text will give you a hint on how the final report file path will look like. Some of the field code samples may contain already the final values, like the field code {ConfigName}, but most of them will be evaluated during the test run.

Note

  • CANoe creates the resulting report path provided that the specified path is accessible and the user has write access.
  • The report path may contain field codes to specify folders. Moreover the system variable field code may be used for creating dynamic sub paths.
  • Field codes may be used also using the CAPL function testReportFileName.
  • See Restrictions for limitations when using report field codes.

The measurement index can be set in the global options and is incremented on each measurement start.

The measurement index is stored in a separate file for system parameters independent of the configuration.

  • The {Timestamp} field code has been replaced by the {TestStop} field code.

    The {Date} {Time}{YYYY}{MM}{DD}{hh}{mm} and {ss} field codes have been replaced by the {TestStart} field code.

Field Code

Description

Duration

{TestDuration}

Duration of the test in the format hh-mm-ss.

Test Execution

{TestRun[|Length]}

Counter for test execution.
This value is incremented by each test run.

You can optionally set the desired length to specify the number of digits.

Test Cycle

{TestCycle[|Length]}

Number of executed test cycles. The field code can contain a length to specify the desired number of digits.

Test Start/
Test Stop

{TestStart|[Format]}

{TestStop|[Format]}

Date and time of the test start/end.
You can set the default time stamp format in the global options. In the field code you can overwrite the default setting.

Optionally, any format can be used within the field code. To do so, please use the following descriptors:

  • Year: YYYY
  • Month: MM
  • Day: DD
  • Hour: hh
  • Minute: mm
  • Second: ss

Apart from the descriptors you can use any other character to combine a text with time and date.

Example:

{TestStart|Started on DD.MM. at hh.mm}

Verdict

{Verdict}

Final test verdict.

Variant Property

{varprop::variant[|length]}

Value of an evaluated variant property at the end of the test. Only the variant properties for the activated test units are evaluated.

Optionally you can define a certain length. If the variant property falls below this length the value is filled up with "_" (underscore).

Restrictions when using Field Codes

  • Report file path may not contain invalid path characters either before or after field code evaluation.
  • UNC paths may not contain field codes for specifying the server or share name.
  • Length of the field codes must be a numerical value.
  • Only system variables of the type int, float and string are supported.
  • Any invalid path character in the value of a system variable field code will be replaced with underscore character _.
  • If an error occurs during the evaluation of the field codes either used directly in reporting options dialog or via CAPL function, the report file will not be available.

Examples

Report Numbering as in Previous CANoe Versions:

e.g. d:\configs\test1\report_{TestRun|4}.xml
-> d:\configs\test1\report_0001.XML
-> d:\configs\test1\report_0002.XML
-> d:\configs\test1\report_0003.XML

Length:

eg. d:\configs\test1\report_{TestCycle|3}_{TestRun|3}.xml
-> d:\configs\test1\report_003_005.XML
-> d:\configs\test1\report_003_006.XML
-> d:\configs\test1\report_004_007.XML

Field Codes in Sub Path Specification:

e.g. d:\configs\{TestStart|DATE YYYY-MM-DD}\{Verdict}\report_{MeasurementIndex}.xml
-> d:\configs\DATE 2015-04-21\pass\report_028.XML
-> d:\configs\DATE 2015-04-22\fail\report_078.XML
-> d:\configs\DATE 2015-04-23\inconclusive\report_102.XML

System Variables Field Codes:

e.g. d:\configs\report_{@ns::myvar}.XML
-> d:\configs\report_myvalue.XML

System Variables Field Codes with Length Specification:

e.g. d:\configs\test\report-{@ns::myvar|10}-{TestRun|3}.XML
-> d:\configs\test\report-myvalue1__-012.XML
-> d:\configs\test\report-myvalue2__-013.XML
-> d:\configs\test\report-myvalue3__-014.XML
