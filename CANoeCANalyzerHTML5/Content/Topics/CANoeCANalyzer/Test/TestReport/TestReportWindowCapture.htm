Window Capture

 

The Window Capture feature lets you capture the content of many analysis windows (AFDX VL Statistics Window, CAN Statistics Window, Data Window, FlexRay Statistics Window, Graphics Window, LIN Statistics Window, Scope Window,State Tracker, Test Trace Window, Trace Window, Video Window) and also panels during a test and integrate such window captures into the test report. This means you can view – in the test report – e.g. an image of the signals observed in the Graphics Window at the time the test was run.

This feature can be used in both XML and CAPL test modules/units.

  • The relevant CAPL function is TestReportAddWindowCapture.
  • The relevant pattern in XML test modules is windowcapture.

Note

  • In order for window capture to work, CANoe must be visible in the foreground and must not be obscured by other applications. Only those areas that are visible on the screen are captured successfully. It works best if you maximize the main CANoe window.
  • Windows are not captured if you disable the output of comments and details in the test module Report Filter configuration.
  • If you use default names for the windows to be captured, the capture may fail if you are running the test in a CANoe installation that is set to a different language. This is because the default names are adapted to the GUI language on loading. As a result, the windows are not found in the test module/unit under the specified name. If you want to use your test module/unit in several different CANoe installations with different language settings, you should assign your own unique window names. When naming the windows, please also read Tips: Separate windows for different test cases below.

Window Configuration

Window content is captured in line with the window's GUI configuration and its current size on the screen. Content that is outside of the visible area is not captured. The window must be open (or minimized) while the test is running. The window should not be larger than the visible screen, as otherwise not all of its contents are captured. During the test, the windows may flicker briefly as they are captured.

Particularities of Individual Windows

  • Graphics Window
    The graph and the legend are captured. Ideally you should configure the display options to show the y-axes and their labels, using the All y-axes or Separate views settings for y-axis display.
    As a parameter, you can pass a list of signals, variables or diagnostic parameter names to be enabled for the screen capture.
    If no name is defined, all currently visible curves are captured. Names must be separated with a semicolon. The following syntax is supported:
    Signals: [channel::][network::][node::][message::]SignalName
    Environment variables: EnvVarName
    System variables: sysvar::Namespace::SysVarName
    Diagnostic parameter: [ecu::][service::][(reqest | response)::]ParameterName
    It is possible to provide the name only. The first matching signal, variable or diagnostic parameter will be used. The test report will contain a warning if a name cannot be resolved.
    Hint: Please refer to the More Object Info column in the Graphics Window to determine the full name.
  • Statistics Window
    You can specify the page (channel selection) to be captured. If you don't specify a page, the currently active page is captured. To capture all pages, repeat the call and specify the different page names.
  • Trace Window
    Only the Trace area is captured. The Detail view – even if enabled – is not captured because it only contains data in pause mode, which doesn't make sense while a test is running.

Filenames and Paths

If you do not specify a filename, the image filenames are generated automatically. You have the option of specifying a filename yourself. By default, all image files are saved to one subfolder per test report. This folder has the same base name as the test report and is created in the same location within the file system. If the Add number to filename automatically option is enabled for the test report, the incrementation is also applied to the image filenames. If incrementation is not enabled, the images, like the test report, are overwritten on each run.

Explicit Paths

Optionally, you can specify filenames with an explicit, relative or absolute path. The specified folder must already exist, otherwise an error message is generated. Relative paths are calculated relative to the folder in which the test report is saved (not the default image folder). Incremental test report numbering is ignored when an explicit path is specified.

Warning:
Images saved into folders specified in an explicit path are overwritten without confirmation on each test run. If you want to retain images from several test runs, you should either not specify a path and enable the Add number to filename automatically option for the report, or save the relevant images between test runs.

File Formats

If the specified filename does not include a filename extension, the images are saved as PNG files by default (.png is automatically appended). If you specify an extension for a supported file format, then this format is used. PNG, GIF, BMP and JPG/JPEG are supported. If you only specify one valid filename extension, e.g. .gif, the subsequent filenames are generated automatically but the images are saved in this format.

Ideally you should use PNG (default) or GIF format. JPEG is not very suitable for window capture, as the compression generates artifacts, particularly at the sharp edges of text or graphs. BMP format may result in very large files as it is an uncompressed format.

Note

  • Separate windows for different test cases
    If you want to repeatedly capture e.g. the Graphics Window in different test cases using different configurations such as a different signal selection, you should add several Graphics Windows to your test setup.
    Configure each of them for the desired task and give it a corresponding name.
    You can then use these windows in the different test cases.
  • Test report and images in the same folder
    By specifying the relative path "./" before the filename, you can save the image files to the same directory as the test report.

Warning:
In this case, the image files are overwritten on each test run, even if the name of the test report itself is incremented!
