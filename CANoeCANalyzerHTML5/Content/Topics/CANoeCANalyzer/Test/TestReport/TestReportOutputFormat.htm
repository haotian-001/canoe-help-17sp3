Test Report Output Format

Ribbon: File Tab|Options|General|Test Feature Set

CANoe writes the test report to a file while the test is being implemented. This file contains all information of the test report in a structured, machine-readable form.

In the global CANoe options, you select the file format for reports:

  • CANoe Test Report Viewer (recommended)
    Using CANoe Test Report Viewer you can visualize and search test reports intuitively.
    Several possibilities for filter, grouping and navigation as well as user-defined views provide easy analysis of test reports.
  • XML/HTML Format (previous format)
    XML/HTML output format allows creating test reports in XML format. Using an XSLT stylesheet file transforms the XML test reports to HTML. Thus you can adjust the look of your test report to your needs.

CANoe Test Report Viewer

Using CANoe Test Report Viewer you can intuitively view and search test reports. This output format is stored in a *.vtestreport file and can be viewed with CANoe Test Report Viewer installed with CANoe. There you can use certain filters, grouping and navigation features as well as user-defined searches. The CANoe Test Report Viewer provides the following features:

  • Queries
    CANoe Test Report Viewer enables query definitions on the complete data of the test report. Even complex and nested queries can easily be defined. The defined queries can be saved and re-used later; e.g. a list of all test cases that contain a particular background check.
  • Predefined queries
    CANoe Test Report Viewer provides several predefined queries relevant for testing, e.g. a list of all failed test cases, a list of all trace items grouped by result of the linked test cases.
  • Filtering
    Different filters can be activated in CANoe Test Report Viewer to hide non-relevant information in certain views. Thus the data needs not to be reduced during test execution but remains completely available at any time during execution and analysis.
  • Grouping results
    Together with filtering, grouping is used for well-arranged results, e.g. grouping by verdict etc.
  • Easy navigation within a report, e.g. between different information or time sections.
  • Usability
    Analyzing test reports is done intuitively and efficiently by having a selectable and navigable execution tree, views in different pages and query, filter an navigation possibilities as mentioned above.
  • Support of existing workflows based on XML reports
    The CANoe Test Report Viewer can export the most important information to XML files that are compatible to the XML report format. This contains all information down to test case level without details on execution of the test case implementation.

XML/HTML

In the configuration dialog of the test configuration, you can specify which information is to be transferred to the XML test report. You can use this to limit the size of the XML file for very long tests. For example, you can disable the automatic reporting of wait operations.

Because the HTML test report is generated from the XML test report, a limitation of this type always affects the HTML test report indirectly, as well. The XSLT style sheet used to generate the HTML test report can likewise be used to limit the information selection. The standard XSLT style sheet onepage.xslt has a few definitions for this purpose at the start of the file. These can be used to change the appearance of the HTML test report. You will find the style sheet in the exec32/testunit_report directory of the CANoe installation directory.

The HTML test report is generated from the XML file with the help of an XSLT style sheet at the end of a test run. By using different XSLT stylesheets, it is possible to generate different types of test reports, e.g., short test reports that show only the most important information in an overview, or detailed test reports. Which XSLT style sheet will be used can be specified in the configuration dialog of the test configuration or in the configuration dialog of the test unit.

The XSLT style sheet onepage.xslt is part of the CANoe installation and is located in the exec32/testunit_report directory of the CANoe installation directory. This style sheet generates an HTML test report that displays all information on a single HTML page. This style sheet is also used when the style sheet specified in the configuration menu is not found.

You will find other style sheets in the Exec32/testunit_report directory of the CANoe installation. You can use these as a basis for your own style sheets and adapt them to your requirements. In order to use the style sheets, a few rules may have to be followed or certain requirements met (e.g., installation of Java). In addition, the style sheets may not be suitable for all applications. More detailed information is provided in the accompanying documentation files (text file with the same name as the style sheet, but with extension .txt).

Following the test implementation, HTML test reports can also be generated from an XML test report by means of a transformation using an XSLT style sheet. To do so, you must manually call the XSLT processor as described under Tips.
