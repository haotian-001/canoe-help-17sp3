External Test Report References

 

On the level of test modules, test units, test groups and test cases you can add references to external resources to the report. In the test report the link appears under the (optional) title. The following types of references are supported by CANoe and labeled appropriate in the report:

  • URL
    an URL that links to any external resource and can be opened in a browser.
  • DOORS
    an URL that links to a requirement in the DOORS Requirements Management Tool.
  • eASEE
    an URL that links to an object in eASEE.

This functionality can be used both in XML test modules and in CAPL test modules / test units.

  • The appropriate CAPL function is TestReportAddExternalRef.
  • The appropriate element in XML test modules is externalref.

Note

  • URLs
    Open the resource you want to link to in your browser, copy the URL from the browser's address field and paste it in your test module.
  • DOORS (effective with DOORS version 8.1)
  • You can both link to single requirements and to modules in DOORS: choose Copy URL in the context menu of the module/requirement to copy the DOORS URL in the clipboard. Alternatively you can find the URL in the properties dialog of the module/requirement.
  • The browser forwards DOORS links to the DOORS front end, therefore for opening the URL the DOORS client software has to be installed on the computer on which the report is viewed (the client is not necessary for executing test or for creating a report).
  • eASEE
  • The eASEE documentation describes how to create links to objects in eASEE.
  • To be able to open eASEE links, eASEE has to be installed on the computer on which the report shall be viewed.
    This is not necessary for test execution and creating the report.
