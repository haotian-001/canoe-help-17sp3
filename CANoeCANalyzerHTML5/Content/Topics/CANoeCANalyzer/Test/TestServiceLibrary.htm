Test Service Library (TSL)

 

CANoe can be used to monitor message cycle times, the reaction time that an ECU needs to send an answering message upon receipt of a message, the validity of signal values in messages etc.

You can make deductions about the quality of the control devices being tested using test result statistics, such as the number of deviations reported during the test period.

CANoe makes special functions (checks) available for these tests, which are consolidated in the Test Service Library. They can be used in CAPL and XML test modules as well as in test units. Test specification details are partially drawn from the CANdb++ database, in which data such as target values can be stored. Thanks to this functionality, a minimal amount of test code is needed.

The Test Service Library also includes functions which make it easier to stimulate control devices (stimulus functions). It is possible, for example, to use these functions to generate different signal forms.
