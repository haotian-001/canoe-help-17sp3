Tips & Tricks

 

Tip:
Editor for XML test modules

The editor for XML test modules can be chosen in the Options dialog.

Tip:
Editor for .NET test modules

The editor for .NET test modules can be chosen in the Options dialog.

Tip:
Subsequent generation of HTML test reports from XML test reports 

Test reports in the XML format can subsequently be converted into HTML. For that purpose the XSLT processor has to be called manually on command line level.

sabcmd <XSLT File> <XML File> <HTML File>

Thereby the HTML file is generated from the given XML file by using the XSLT stylesheet.
Example:

sabcmd file://report\onepage.xslt file:///d:\project\report.XML file:///d:\project\report.html

The sabcmd application (the XSLT processor) is located in the exec64 directory of your CANoe installation.
Please take care of the correct path entries – be sure to use "file://" for relative and "file:///" for absolute paths.

There are also complex XSLT stylesheets such as multipage that entail further operations in addition to the basic XSLT transformations, as for example copying additionally needed files. For further information and hints on how to use these stylesheets manually, please refer to the text file of the same name which is located in the directory of the stylesheet and the comments in the first lines of the stylesheets.

Tip:
Avoiding rounding errors 

When working with floating-point numbers (e.g. with according signals) you always have to work with relation comparison (<, > etc., <range> (in XML )) for these values always may contain rounding errors. Apart from that floating-point numbers are mostly used to express physical values that never can be measured exactly.

Tip:
Line breaks in descriptive texts 

It is possible to explicitly insert line breaks in the descriptions of test modules, test groups, test cases, and test steps in the test report. For this purpose, insert the following at the desired position in the descriptive text:

  • \n in <description> elements of XML test modules or
  • \n in a CAPL function call, such as TestModuleDescription.

If you selected the XML/HTML test report format in the global CANoe options, these positions are replaced with a <br /> by the XSLT style sheet, which inserts a line break when transforming the XML test report into an HTML test report.

This conversion can also be deactivated in the XSLT style sheet. For this purpose, the variable DoLinebreakTransformation in the file include\varglobal.xslt must be set to "false".
