Test Service Library: Introduction

 

The Test Service Library contains a collection of prepared test functions (checks) and value generation functions (stimuli) which make it easier to assemble tests. It is based on the basic functions included in the Test Feature Set (TFS). It makes special functions available that assist with solving specific, and possibly very special, test problems effectively and efficiently.

  • Checks

    Check functions run parallel to the defined test procedure or simulation node and continuously test whether certain conditions are being met.

    Any violation of a check condition detected by the check is reported via a callback function. In test modules/units, the checks are implemented as constraints or conditions. Feedback is not necessary in this case. The checks influence the test results directly in this case, and the execution is logged in the test report.

  • Stimulus Generators

    These commands are used to generate a specific series of values for a signal or an environment variable, which are used to stimulate a device.

    The stimulus generators create value series which are applied to signals or environment variables cyclically. The cycle time sets the sampling rate for the generated value series and thus the rate of change of the drain.

    The specified value series are repeated periodically, so that a value series of any desired length can be created. The time required to execute one value series until it is repeated is referred to as the period.

The use of checks and stimuli is not restricted to test modules/units. They can also be used in CAPL simulation nodes. However, it is recommended that checks and stimuli only be used in test modules/units.

The usage of TSL functions differs slightly, depending on what kind of test module or simulation node they are being used in:

  • Usage of TSL functions in XML test modules
  • Usage of TSL functions in CAPL test test modules/units
  • Usage of TSL functions in CAPL simulation nodes

Checks and stimulus generators are always created and parameterized before usage. In the case of a generator, the sink for the value series and the cycle time of the generated value series are specified.
