Timer Precision

 

The Test Service Library use the standard timers of CANoe.

  • The internal precision of the timers is nanoseconds.
  • The standard precision for the CAPL API is milliseconds.

To set a higher precision for the CAPL API the function ChkConfig_SetPrecision can be used. This function can change the precision between 1ms and 1 ns.

Note

This does not affect the checks and stimuli that were created before changing the precision.

All time stamps that are given to the Test Service Library by the CAPL API have the unit of the current precision.

Checks

  • The create and start functions of the specific checks contain time information.
  • The statistical queries always return the time information with the precision that was used to create the check.
    With another function ChkQuery_Precision it is possible to convert the time information to the standard precision (unit = milliseconds).

Stimulus Generators:

Only the create functions of the stimuli contain time stamps.
