.NET Debugging

 

Overview

The .NET debugging support integrated in CANoe helps you debug test modules that contain .NET code. It simplifies the process by automating actions that you need to execute frequently. This includes starting Visual Studio, loading relevant source files, and starting and initializing the debugger. For remote debugging, this also includes starting the remote debugging monitor on the RT System and automatically starting up Visual Studio with the correct user rights.

In the Options dialog you can configure .NET debugging.

Notes on Remote Debugging

If you're using an RT System (such as the VT6000), you need to use remote debugging instead of normal debugging. The process is the same as for local debugging, except that you need to take a number of steps at the start to set up the system. These are described in VT6000: Remote Debugging.

Installing the Remote Debugging Monitor

To enable automatic starting of the RT System's remote debugging monitor via CANoe, you have to ensure that the remote debugging monitor is in the correct folder. The folder name is:

  • D:\Tmp\msvsmon\2003\ for Visual Studio 2003
  • D:\Tmp\msvsmon\2005\ for Visual Studio 2005
  • …

This means you can install different versions of the remote debugging monitor in parallel if you are debugging with several Visual Studio versions.

Location of test module source files

Because a local version of Visual Studio needs to run under a different user account for remote debugging (user rtdebug), it has no access to the folders belonging to the user who is currently logged in. If the source files for the test module you're debugging are located e.g. in a folder on the desktop, you won't be able to load them.

Please make sure that the required files are all located in a folder that can be accessed by all users of the computer.

Use

In order to use CANoe's .NET debugging function, you first need to start the measurement. The debug button is then available in all Test Execution dialogs that contain .NET code. Clicking this button opens the previously configured Visual Studio version, loads the source files specified in the test module and then connects the Visual Studio debugger with CANoe. If you're debugging in distributed mode, the remote debugging monitor is also started automatically on the RT device.

Depending on the options you've set, the test module may then start automatically. You can then debug it in Visual Studio. You can also opt to deactivate automatic execution. This is useful if you want to insert pauses in Visual Studio before starting. You then start the test module manually in the CANoe Test Execution dialog.
