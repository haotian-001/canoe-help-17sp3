Precondition and Break Condition

 

Test modules and test cases may contain a section on precondition and break condition.

Precondition

The preconditions are checked before test execution.

If the preconditions are not fulfilled the test group or the test case respectively is not executed — including corresponding preparation and finalization step.

Note

The verdict is not changed.

If the preconditions are fulfilled the test group or the test case respectively is executed. The preparation step is displayed as separate section in the test report.

Not executed test cases are visualized in the test report as skipped, as well as they were not activated in the Execution dialog.

The precondition is optional and can be implemented as following:

<precondition>
CAN/LIN/FlexRay signal, environment variable, system variable,
<vareval>, <and>, <or> or <xor> element
</precondition>

It checks either a single value or it checks a list of values grouped by a conjunction.

Break Condition

For the test sequence a special control function is used to abort the test case or the whole test module. As well as the precondition a single value or a list of values grouped by a conjunction can be checked thereby.

<assure terminate="testcase/testmodule" title="…">
  CAN/LIN/FlexRay signal, environment variable, system variable,
  <vareval>, <and>, <or> or <xor> element
</assure>

The title attribute has to be defined; the terminate attribute is optional. The default value thereby is testcase.

In case of an abortion the verdict is set to Failed. The remaining test sequence is not executed any more. Only the finalization section is still executed. In the test report the control function is visualized in the test sequence with the corresponding verdict.

Note

This special control function can only be used in the sequence of a test case and not in the preparation and finalization section.

Set up XML Variable Conditions and Conjunctions

To check a value of a XML variable please use the vareval element:

<vareval name="name of XML variable">value</vareval>

The value can be specified directly or with a relation, e.g. as for signals.

Example

<vareval name="Var1">
  <gt>4</gt>
</vareval>

To check a list of values they have to be grouped with a conjunction. There are three different conjunctions possible.

Syntax

Short Description

<and>
CAN/LIN/FlexRay signal, environment variable, system variable or <vareval>
</and>

All conditions have to be fulfilled, that the whole condition is applied.

<or>
CAN/LIN/FlexRay signal, environment variable, system variable or <vareval>
</or>

At minimum one condition has to be fulfilled, that the whole condition is applied.

<xor>
CAN/LIN/FlexRay signal, environment variable, system variable or <vareval>
</xor>

Only one condition has to be fulfilled, that the whole condition is applied.

Example

<testgroup title="Test cases of variant comfort">
  <precondition>
    <vareval name=”ActualVariant”>Comfort</vareval>
  </precondition>
  <testcase>
    <preparation>
      <initialize title="Start engine" wait="150ms">
        <cansignal name="Ignition">On</cansignal>
      </initialize>
    </preparation>
    <assure title="Check if engine is running" terminate="">
      <and>
        <cansignal name="EngineRunning">1</cansignal>
        <cansignal name="EngineSpeed"><gt>0</gt></cansignal>
      </and>
    </assure>
    <!—Do concrete tests ->
  </testcase>
</testgroup>

The precondition assures the controller's variant.

The <assure> break condition assures, that the motor runs.
