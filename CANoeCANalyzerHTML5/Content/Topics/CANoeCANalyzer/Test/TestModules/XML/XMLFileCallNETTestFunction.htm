Call .NET Test Function

 

In the test module configuration dialog, you can add .NET assemblies to the XML test module via the Modules tab. The assemblies contain test functions that can be called from the XML test module.

How to Define Test Functions in .NET Test Modules:

using System;
using Vector.Tools;
using Vector.CANoe.Runtime;
using Vector.CANoe.Threading;
using Vector.CANoe.TFS;
using NetworkDB;

public class NETTestFunctionLibrary
{
  [TestFunction]
  public void SetSignal<S>(int d)
  where S : Signal
  {
    RuntimeObject<S>.Value = d;
  }
}

In order to create the .NET test libraries, you need to reference the different assemblies.
You need to include all the assemblies that are required to develop pure .NET test modules.

How to Reference the Test Function Programmed in .NET Above in an XML Test Module:

<testcase title="TC2" ident="ID2">
  <nettestfunction name="SetSignal" title="Close the window">
    <netparam name="S" type="signal">
      <cansignal name="CyclicSignal" />
    <netparam>
    <netparam type="int" name="d">0</netparam>
  </nettestfunction>
</testcase>

  • The name attribute indicates the name of the test function used in the .NET program.
  • The title attribute is mandatory.
    It can be used to give the test function a title for use in the user interface and the protocol.
  • The class and assembly attributes are optional.
    They can be used to provide a unique description of the test function.
  • Whole numbers, floating-point numbers, character strings, signals, environment variables and system variables are permitted as parameter types.
    These are automatically mapped to the corresponding .NET types:
  • int: UInt64, UInt32, UInt16, Int64, Int32, Int16
    Careful with value overruns!
  • float: Double
  • string: String

Note on Variables

It is also possible to use a variable (<var>) instead of a set value. The data type of the variable has to correspond to the .NET parameter type.

  • signal: Vector.CANoe.Runtime.Signal
  • envvar: Vector.CANoe.Runtime.EnvironmentVariable
  • sysvar: Vector.CANoe.Runtime.SystemVariableBase

Note on Value Types and Generics

Unlike value types (int, float, string, etc.), these types are declared as generics in the test function.

In order to guarantee type safety, you need to use constraints with these generics (in the example: where T: Signal).

You can use the following specific types: Signal, EnvironmentVariable and SystemVariableBase.

The Following Definition is Required to Enable Using the Same Test Function for All Three Types in the Same Way:

[TestFunction]
public void SetSignal<S>(int d)
where S : class, IRuntimeValue
{
  RuntimeObject<S>.Value = d;
}
