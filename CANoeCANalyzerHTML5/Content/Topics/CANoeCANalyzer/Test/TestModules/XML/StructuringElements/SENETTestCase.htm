.NET Test Case

 

Name

.NET Test Case

Description

Allows to reference test cases programmed in .NET in an XML test module.

Remarks

They allow to parameterize and invoke test cases that are programmed in .NET.

In CANoe's Test Module Configuration dialog, .NET assemblies can be added to the XML test module via the Modules tab. The assemblies contain test cases that can be called from the XML test module.

The test cases are defined as in .NET test modules, except that no Main function is required.

For the users of this type of XML test module there is no apparent difference between test cases in XML and .NET, as the .NET test cases can also be deactivated in the user interface.

Syntax

<nettestcase name="SimpleTest" title="Title" title=" Displayed name of the CAPL test case" class=" NETTestCaseLibrary" assembly="TestCaseLibrary" " variants="A" >
  <netparam name="ParameterName" type="float|int|string|signal|envvar|sysvar"> Parameter Value </netparam>
</nettestcase>

Attributes
  • name (mandatory)
    The name attribute indicates the name of the test case used in the .NET program.
  • title (optional)
    The attribute can be used to give the test case a title for use in the user interface and the protocol.
    This is particularly useful when the same .NET test case is called several times with different parameters from an XML test module.
  • ident (optional)
    This attribute is used to assign a unique ID to a test case. This ID is also used in the test report.
  • class and assembly (optional)
    These attributes can be used to provide a unique description of the test case.
  • variants (optional)
    Specifies the variants in which the CAPL test case is executed.
