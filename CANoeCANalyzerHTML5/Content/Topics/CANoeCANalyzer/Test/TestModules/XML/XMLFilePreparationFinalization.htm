Preparation and Finalization

 

The test module, each test group and each test case may contain a section on preparation and finalization. Both are optional and can be implemented independently of one another.

<preparation>
  List of comments, variable definitions, system variables definitions, test functions and control functions
</preparation>

<completion>
  List of comments, test functions and control functions
</completion>

The variable definitions, test functions and control functions in the preparation section are always executed before the actual contents of the test module, test group or test case. Constraints and conditions defined in the same context (e.g., conditions defined in the same test case) will not apply yet. However, constraints and conditions defined in the surrounding contest (e.g., constraints defined for the parent test group) will apply.

Note

  • Variables can only be implemented in a preparation section, as they can only be defined and described there.
  • Only system variables created in an XML test module can be changed by the XML test module. System variables can only be created in a preparatory step.

The test functions and control functions in a finalization section are processed after the respective test module, test group or test case has been processed. Constraints and conditions that have been implemented in the same context will not be monitored at the same time as the preparation section. The finalization section will be executed even if the test case is aborted because of an error.

If a test case is not executed during a test run because it was deactivated by the user, the preparation and finalization sections contained in it will also not be executed. The same logic applies to test groups: If all test cases belonging to a test group are deactivated, then the preparation and finalization sections for the test group will not be executed.
