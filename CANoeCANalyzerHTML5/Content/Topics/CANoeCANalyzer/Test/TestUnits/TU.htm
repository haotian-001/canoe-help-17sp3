Test Units

 

In vTESTstudio, you can implement tests with the help of CAPL and C# files as well as in tabular and graphic forms. In so doing, the different test languages can be easily combined. The executable entity programmed in vTESTstudio is called a test unit. It comprises all files required for the test. A test unit created in vTESTstudio has the filename extension *.VTUEXE. This can be configured and executed in CANoe.

In addition to the file name extension *.vtuexe, there is also the file format *.vtestunit.yaml for describing a test unit. This file format can be used to describe test units for CANoe4SW Server Edition.

A test unit is configured within a test configuration. A CANoe configuration can contain any number of test configurations. A test configuration, in turn, can contain 1â¦n configured test units. All test units within a test configuration are executed sequentially one after the other.

A test configuration can be tested either automatically at the start of the measurement or interactively by the user. A started test configuration normally terminates at the end of the test run or when forced by the user or at the end of the measurement.

The assignment of a test unit to buses happens already in vTESTstudio. A test unit is assigned automatically to all buses configured in the vTESTstudio project.

The user can activate or deactivate the test configuration in the Test Setup for Test Units before the start of the measurement (setting can be recognized from the corresponding check box). A deactivated test configuration cannot be started during the measurement. The automatic start function is also disabled.

Each execution of a test configuration generates a test report.
