Executing Test Units

Context Menu of the test configuration: Execute Test Units

Test configurations with one or more test units can be started interactively or with an automated start:

  • For automated execution, the following start options are appropriate and can be specified in the configuration dialog of the test configuration:
  • Start of test configuration at the start of the measurement
  • Start of test configuration when an environment variable changes
  • Start of test configuration at the press of a button
  • An interactive execution of the test configuration is possible in the execution dialog. To open this execution dialog, double-click the test configuration in the Test Setup for Test Units or select it using the context menu of the test configuration.

You can select which test cases and test sequences are to be executed in the execution dialog.

You can monitor the test execution of a test configuration in the Test Trace Window. All test steps and actions are displayed there while the test is running.
Additionally two »system variables will be generated automatically for each test configuration, which can be used to query and show the current execution status and the current »verdict:

Variable Name

Values

RunState
  • 0|Not Running
  • 1|Running
  • 2|Finished

VerdictSummary
  • 0|None
  • 1|Passed
  • 2|Inconclusive
  • 3|Failed
  • 4|Error in Test System

Note

  • The execution of a test configuration is represented in the Trace window (e.g., start and end of the execution of a test case and results of test cases).
  • The execution of a test configuration is also recorded in the test report.
    It is possible to configure an overall test report for all test units of a test configuration or a separate test report for each test unit.
