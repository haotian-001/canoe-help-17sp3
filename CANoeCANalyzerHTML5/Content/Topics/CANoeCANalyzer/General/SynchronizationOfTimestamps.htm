Synchronization of Time Stamps

 

Time synchronization means the adjustment of hardware time stamps of events coming from different network interfaces each with a separate hardware clock. This is done to achieve that events virtually occurring at the same time, have the same time stamp, despite the fact that in real systems no two hardware clocks ever run at the same speed and have no offset.

Typically Vector interface hardware offers several channels, for connection to various bus systems such as CAN, LIN, FlexRay and Ethernet. When all channels are on one network interface, a time synchronization is not needed, because the time stamps for the events of the respective channels are derived from the same clock.

As soon as more than one network interface is used, a synchronization between the time stamps of events from different interfaces is necessary; because each interface has its own clock.

Vector offers two possibilities to synchronize the time stamps between different network interfaces.

  • Software Time Synchronization
    A software algorithm synchronizes the time stamp of the events of each network interface to the computer.
  • Hardware Time Synchronization
    A party line is used for signaling between all synchronized network interfaces to enable a precise synchronization. Not every interface supports this way of synchronization.

Software Time Synchronization

A software algorithm synchronizes the time stamp of the events of each network interface to the computer.

Prerequisites

There are no further HW requirements. E.g. a connection between interfaces is not required.

Notes

Most of the Vector network interfaces support software times synchronization, except of the following:

  • VN8911, VN8912, VN8912A, VN8914

Installation/Activation

  • Activate the time synchronization in the Vector driver
    (Start|Settings|Control Panel|Vector Hardware|General information|Settings|Software time synchronization: YES)
  • Deactivate the Hardware time synchronization in the Network Hardware Configuration dialog in CANoe

Notes

  • The precision of the Software Time Synchronization is limited.
    With Vector interfaces the synchronization is quite good.
  • Due to adaptive algorithms with third party hardware, the maximum precision will only be achieved a couple of seconds after measurement start. As soon as an exact synchronization is crucial for the specific measurement, start the measurement and start calculations with the synchronized time stamps some seconds later.

Hardware Time Synchronization

A party line is used for signaling between all synchronized network interfaces to enable a precise synchronization. Not every interface supports this way of synchronization.

Prerequisites

For synchronization a connection between the network interfaces is used, the so called party line.

Notes

Most of the Vector network interfaces support hardware times synchronization, except of the following:

  • VN1610, VN1611, VN7610

Installation/Activation

  • Connect the party line to all active interfaces
  • Deactivate the time synchronization in the Vector driver
    (Start|Settings|Control Panel|Vector Hardware|General information|Settings|Software time synchronization: NO)
  • Configure HW Sync as active in the Network Hardware Configuration dialog of CANoe.
  • Shortly after measurement start there will be an information in the Write Window whether the time synchronization works properly or not.

Notes

  • As soon as problems with the synchronization occur, an according information will be written to the Write Window and in any active Logging file.
  • Due to closer coupling between the interfaces via the party line, the hardware time synchronization is much more precise than the software time synchronization. The maximum precision is reached 1 or 2 seconds after measurement start. The typical jitter between the time stamps is less than ±1 µs and the offset is virtually eliminated.

Timestamps

Synchronized Time Stamp

  • The synchronized time stamps typically are shown in the leftmost column of the Trace Window.
  • In CAPL you have access to these time stamps by using the selector time of almost every bus event. The resolution in CAPL is 10 µs or 1 ns with float messageTimeNs(...), starting with 0 at the start of measurement.
  • Calculation of time differences between events from different network interfaces should be done using the synchronized time stamps.
  • When using a hardware time synchronization the precision is high enough for every use case.

Original Time Stamp

  • For MOST and FlexRay it is possible to get the original unsynchronized time stamp.
    The original time stamps can be shown in the Trace Window when the column Original Time is selected. Besides correction of the offset, so that the time stamp shows approximately 0 on measurement start, this time stamp is the unmodified time stamp generated by the network interface.
  • The selector MsgOrigtime, for the events supporting original time stamps, allows access to the original time stamp, with a unit of 10 µs.
  • The original time stamp should be used, when exact time differences between events of the same network interface have to be calculated.
  • However when using hardware time synchronization, the synchronized time stamps should be exact enough for all use cases.
