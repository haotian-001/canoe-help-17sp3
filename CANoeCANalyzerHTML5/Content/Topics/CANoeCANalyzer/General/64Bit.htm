Notes on 64-Bit/32-Bit

 

Overview

New in version 13.0:

  • The CANoe application (and thus the analysis area with the graphical user interface) is now only available as a 64-bit variant for the user computer.

New in version 15 SP3:

  • The RT kernel (i.e. the real-time area) is available in a 64-bit variant and in a 32-bit variant for the VTP Device.

On the Realtime Execution (RT Setup) page you will find an explanation of the operating modes and terms.

Topic

64-Bit RT Kernel

32-Bit RT Kernel

Deployment
  • Only for processors and operating systems in 64-bit architecture.

For processors and operating systems in 32-bit architecture or 64-bit architecture

Availability
  • The 64-bit variant is available on the user computer and for realtime execution in standalone mode or in distributed mode on selected devices (e.g. VN8911, VN8914, VT6020, VT6060 and RT Rack).

  • For realtime execution in standalone mode or distributed mode on devices that support only 32-bit; includes VN8900, VT6000 and RT Rack.

    The 32-bit variant is also available on the user computer.

Memory
  • The 64-bit variant can use more main memory; usually the complete built-in main memory.
  • The 32-bit variant can use a maximum of 4 GB main memory.
  • The CAPL programs in the Simulation Setup of CANoe or test programs can therefore continue to use a maximum total of approx. 3 GB memory during realtime execution.

Use Cases
  • Applications that require a lot of main memory, especially
  • large databases
  • some CAPL analysis programs
  • When working with others who have older versions of CANoe.
  • When working with VN8900.
    Exception: there are already all DLLs for both systems.

Installation

When CANoe is installed, both the 32-bit variant and the 64-bit variant of the RT kernel are automatically installed.

Configuration

The variant of the RT kernel is stored in the configuration file. By default, the 32-bit variant is used. To switch the variant, open the General dialog (File Ribbon Tab|Options|Measurement|General).

Settings in the CAN.ini file

The setting in the configuration file can be overwritten with a CAN.ini switch. Insert the following entry in CAN.ini within the [SYSTEM] section:

EnforceRtkTargetArchitecture=<Value>

<Value> can have the following values:

  • 0: not set, the variant from the configuration file is used.
  • 1: 32-bit variant, the variant from the configuration file is ignored.
  • 2: 64-bit variant, the variant from the configuration file is ignored.

If you want to check the current setting, open the Help page (File Ribbon Tab|Help). In the Product Details section, the configured variant is displayed in the Configured RT kernel line.

Binary Extensions

If you are using the 64-bit variant, you may need to customize self-developed extensions and programs that are added in the form of DLLs. This is because a 64-bit program cannot load 32-bit DLLs, and vice versa, a 32-bit program cannot load 64-bit DLLs. This applies in particular:

  • .NET test modules

    When you create in Visual Studio you must select Any CPU as the Platform Target (this is the default). You can also access other .NET libraries whose Platform Target is x86 unless you call them in static constructors, when initializing static variables, or in the StructuredMain function.

  • CAPL-DLLs that you want to use from CAPL programs in the test setup

    You must create such DLLs yourself in a 64-bit variant and either integrate them using #pragma library or configure them in the CAPL-DLL dialog (File Ribbon Tab|Options|Programming|CAPL-DLL).

If you have any questions, please contact Vector Support.

Assigning Modeling Libraries

When you include modeling libraries, you must ensure that you configure the correct variant (32-bit or 64-bit) of your modeling library according to the execution environment used.

  • You can select module descriptions (*.vmodule) instead of the DLL. This always ensures the correct variant of the modeling library.
  • If possible, you should always select a module description file (*.vmodule) instead of the DLL. This always ensures the correct variant of the modeling library.
  • If you use one of the supplied modeling libraries from the Exec32 or Exec64 folder of the CANoe installation directory, the correct variant of the DLL is automatically used.
  • If you configure a DLL from any other directory, the variant must match so that the configuration can be executed correctly.

If you have any questions, please contact Vector Support.

Limitations

The following further restrictions exist in the 64-bit variant compared to the 32-bit variant:

  • Not all modules and CAPL DLL files are available as 64-bit variants.
  • MOST is not supported.
  • Only some of the sample configurations are supported (see compatibility overview).
  • Extensions that you have created yourself, such as CAPL DLL files for the Measurement Setup, must be adapted yourself. This is due to the fact that a 64-bit program cannot load 32-bit DLL files.
