Macro Recorder

 

Macros permit reproducible and automated testing.

With the help of the Macro Recorder in CANoe macros can be made to record and replay user actions on panels and Diagnostic Consoles. If the actions of Diagnostic Console and the Fault Memory Window are recorded, then diagnostic data sent to the controller are included in the recording.

Note

Macros are being used to influence simulations (not analysis). Therefore macros can only be recorded and executed in online mode.

Once you have started to record the macro every action you execute on the panels with the mouse, and every request sent via the Diagnostic Consoles is recorded. Furthermore, explicit opening and closing of the diagnostic channel are also recorded, and this affects sending of Tester Present requests. The Macro Recorder can also be used to replay the recorded macros.

Each macro is saved in a *.asc file. The ASCII format of the macro file conforms to ASCII Logging Format:

5.1543 Env_DoorWindow_up  := 1

0.9326 Env_DoorWindow_up  := 0

1.7398 Env_DoorWindow_down  := 1

0.9893 Env_DoorWindow_down  := 0

4.0247 1 Console::Console_2::Light = 0

0.9351 2 Console::Console_2::Light = 1

6.7700 L2 RightMirror::MirrorTemp::MirrorTempRight = 6

9.9156 DiagRequest[Vector_Door] 1A 90

You can open the macro file in a standard text editor and make changes to it (e.g. change time stamps) while preserving its underlying structure.

With changes of the signal values the bus system, the channel, the node name, the message name and the signal name will be stored in the file.

<Time stamp> <bus system+channel> <Nodename::Messagename::Signalname> = <Signal value>

4.0247        1                 Console::Console_2::Light                = 0

6.7700        L2                RightMirror::MirrorTemp::MirrorTempRight = 6

For Diagnostic Requests the qualifier of the specific ECU is entered; it is used to access diagnostic descriptions. Besides recording data, control commands may also be recorded:

<Time stamp>  DiagRequest[<ECU qualifier>]  (<Request Data>|Open|Close|TPon|TPoff)

14.8213       DiagRequest[Vector_Door]      Open

15.1213       DiagRequest[Vector_Door]      TPon

15.9156       DiagRequest[Vector_Door]      1A 90

18.990        DiagRequest[Vector_Door]      TPoff

19.2810       DiagRequest[Vector_Door]      Close

Cross Reference

Using the macro recorder for diagnostic purposes

For every macro that is being executed during a measurement, an animated symbol is shown in the CANoe status bar. When the execution is stopped or finished, the symbol is no longer shown.

Notes on Integration of Macro data in CANoe configurations

  • Macros are assigned to a CANoe configuration, i.e. all macro properties (e.g. macro name) are saved to a CANoe configuration and are therefore unavailable if the macro file is added to a different CANoe configuration.
  • In the Macro Configuration dialog you can assign macros to the CANoe configuration or remove them from the CANoe configuration.
