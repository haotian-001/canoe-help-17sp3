CANoe/CANalyzer Feature Matrix

 

CANoe/CANalyzer Product Family — Products, Editions and Variants at a Glance

CANalyzer

CANalyzer is the universal software analysis tool for ECU networks and distributed systems.

CANalyzer makes it easy to observe, analyze, and supplement data traffic in CAN, FlexRay, LIN, or MOST systems.

With powerful functions and user-programmability, all needs are covered from simple network analysis to advanced troubleshooting of complex problems.

Product Variants

  • CANalyzer exp

    The expert variant is ideal for all standard applications, and it provides all functions and extensions without limitation. However, this variant does not support creating and executing CAPL programs.

  • CANalyzer pro

    The professional variant provides all functions and extensions without limitations. It supports all use cases from simple observation of the bus traffic to complex analysis, stimulation and testing of heterogeneous systems.

Test Bench Edition (TBE)

The Test Bench Edition variant is used for test and production. It is only available with license type Test Bench, which allows remote access and execution on a virtual environment. The feature set of the TBE is identical to the corresponding CANoe/CANalyzer variant.

The following Test Bench Edition variants are available:

  • CANalyzer exp TBE

  • CANalyzer pro TBE

  • CANoe pex TBE

  • CANoe run TBE

  • CANoe pro TBE

CANoe

CANoe is a comprehensive software tool for the development, testing and analysis of entire ECU networks and individual ECUs.

It supports you throughout the entire development process – from planning to startup of entire distributed systems or individual ECUs.

CANoe's versatile functions and configuration options are used by network designers, development and test engineers at OEMs and suppliers.

Product Variants

  • CANoe pex

    The project execution variant provides a graphical user interface exclusively. Simulation, test cases and results are easily controlled without the need to specifically evaluate the underlying messages.

  • CANoe run

    The runtime variant is suitable for users who want to quickly and easily test their ECU in interaction with a specified remaining bus simulation. Configurations cannot be changed, analysis functions are fully available and network nodes can be easily connected and disconnected.

  • CANoe pro

    The professional variant is intended for users who want to take advantage of the full range of CANoe functions. Simulation models can be created with CAPL and .NET. Test cases are easy to model with the Test Feature Set.

CANoe4SW

CANoe4SW is the comprehensive tool for development, test, and analysis of software in cyber physical systems – individual software components as well as subsystems and entire distributed systems.

It supports software developers and testers in all markets (medical, railway, automotive, …) throughout the entire development process of distributed systems and IoT devices. Virtual execution environments on the computer, in virtual machines, or in the cloud enable software to be tested for any targeted OS based on Windows or Linux, e.g., Ubuntu, CentOS, SUSE.

The System Under Test is accessed by its functional system interfaces at a pure software level. This enables early "black-box" SIL testing independently of hardware availability and integrates seamlessly in CI/CT environments.

In addition, the support of connectivity protocols such as MQTT provides access to IoT devices and backend software running in the cloud.

CANoe4SW Server Edition

CANoe4SW Server Edition allows the completely automated execution of CANoe configurations in a server environment based on Windows or Linux. In CANoe, each configuration can be exported as CANoe4SW Server Edition configuration.

A CANoe4SW Server Edition configuration always uses the simulated bus, an access to the network hardware is not intended.

CANoe4SW Server Edition includes different tools which are required for the automation.

CANoe/CANalyzer Demo Version

  • The purpose of the demo version is to give you a basic impression of the program.

  • The feature set is nearly the same as in the full version.

  • The demo version supports the same bus systems as the normal version.

Please note the following restrictions

  • Timer

  • The demo version uses a simple Windows timer.

  • Therefore, it is not possible to draw conclusions from the realtime behavior of the demo version to the productive behavior in CANoe.

  • Test

  • The test report contains the remark "Demo Version" multiple times.

  • The maximum number of test cases is restricted to 10.

  • The maximum runtime length of test modules is restricted to 60s.

  • When you try to exceed these limits the measurement is stopped and it is documented in the report.

  • Simulation Setup

  • The number of nodes in the Simulation Setup is limited to 4.

    You can load a Simulation Setup with more than 4 nodes but you cannot start it.

  • The configurations you set up with the demo version cannot be run with the run or pex variant.

  • You may skip the settings of the bus parameters and the message setup (right-clicking the bus icon in the Simulation Setup).

  • Hardware

  • You cannot connect real hardware to the demo version, because the installed demo driver is used.

  • I/O Hardware

  • You cannot access real I/O hardware.

    For demonstration purposes you can select the two PortLink DLLs pl_demo.DLL and da_demo.DLL.

    Existing CANoe configurations can be edited, e.g. you can remove the PortLink DLLs that aren't supported from the configuration.

  • Windows TCP/IP Stack

  • Access to the Windows TCP/IP Stack (e.g. via CAPL) is not possible.

  • Add-ons

  • You can use the demo Version with the LabVIEW add-on.

    Please note restrictions like the maximum number of nodes.

  • You can use the demo Version with the MATLAB add-on.

    Please note restrictions like the maximum number of nodes.

  • You cannot use the demo version together with the SIL Kit framework.

  • Import/Export

  • You cannot read extracts from logging files.

  • You can import a maximum of 1000 messages.
