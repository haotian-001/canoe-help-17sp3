Installation on a Network Drive

 

If the CANoe application is installed and started on a network drive following problems can occur:

Problem Description

General:
Global settings are not stored. The additional tools (CAPL Compiler, CAPL Browser, Panel Designer, etc.) do not work or do not work proper.

Cause

CANoe uses the install directory (Exec32) to store files temporary. In case of you have no write access these misleading situation can occur.

Solution

Write access to the installation directory is needed. Please contact the system administrator.

Problem Description

COM server:
Script or other programs which communicate with CANoe using the COM interface do not work.

Cause

The COM server is not registered on the local computer, if the installation program was not executed on the local computer.

Solution

Register the COM server on the local computer using following command line:

canoe32.exe -regserver

Note: Administrator rights on the local computer are necessary.

Problem Description

Help:
The help cannot be opened.

Cause

The standard security settings of the Internet Explorer inhibit the display of the HTML based help, if it is located on a network drive. Detailed information can be found under:

https://support.microsoft.com/en-us/topic/ms05-026-a-vulnerability-in-html-help-could-allow-remote-code-execution-e482d0f1-6967-b4f5-02e0-2e8c19554d10

Solution

A workaround is to edit the Windows registry. Please contact the system administrator.

Problem Description

Global program settings: Global program settings edited by several users, will be overwritten.

Cause

A lot of program settings are stored in the CAN.INI file located in the CANoeapplication data directory. If several users share the same CANoe installation on the network drive, so the CAN.INI file and thus the global program settings are shared just as well.

Solution

Global program settings should not be changed or it should be ensured, that only one user disposes the network drive installation at any time.

Problem Description

CANoe does not start - new from version 6.0 -

Cause

CANoe uses the Microsoft .NET Framework from version 6.0 on. The .NET Framework default security settings inhibit the execution of .NET programs on a network drive (respectively not confidential sites).

Solution

You can allow the program start on the network drive from the installation directory. Therefore please execute the program EnableNetworkStart from the installation directory of your local computer.

Note

It is not recommended to operate CANoe from a network drive.
