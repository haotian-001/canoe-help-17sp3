Database Assignment When Loading Old Configurations

Ribbon: File Tab|Open

Assigning Databases to Several Channels

Prior to Version 7.2 it was only possible to assign databases to either a single channel or to all channels of a bus system. For example, if three CAN channels were in use, a CAN database could be assigned to either CAN 1, CAN 2 or CAN 3, or to CAN All.

Version 7.2 offers a more precise way of assigning databases. You can now, for example, assign a database to CAN 1 and CAN 3 without assigning it to CAN 2. It is no longer possible to assign a database to CAN All, but you can still assign a database to all channels by assigning it to them individually.

When you load a configuration created with a version earlier than 7.2, in which a database is assigned to all channels, that database will remain assigned to all channels.

Simulation Setup

Earlier versions already made it possible to assign a CAN database to a subset of channels in the Simulation Setup. When resolving ambiguities, however, the real assignment was still to CAN All. When you load such a configuration in CANoe versions 7.2 and later, the displayed Simulation Setup is corrected automatically so that the database is available in all CAN networks. If you don't want that database to be assigned to all of the networks, you can, of course, remove it from some of them.

If you don't want the correction to be made and want to keep the assignments made in the old Simulation Setup, you can set the CompatibleDBAssignment option to 0 in the [System] section of the CAN.INI file.

Database types Undefined and All (i.e. containing only environment variables), when previously assigned to more than one network are now, as before, only assigned to the first network.

Search Order for Resolution of Ambiguities

When resolving ambiguities, databases are searched in the order that they are assigned to the relevant channel. Prior to Version 7.2 databases assigned to all channels were implicitly given lower priority. This meant that when a message was received on CAN 1, a database assigned to CAN All was searched after all of the databases assigned only to CAN 1, even if these were further down the list.

From Version 7.2, the search order always follows the exact order of databases for the relevant channel. When you open an older configuration, incompatibilities concerning the resolution of ambiguities are prevented by placing those databases that were previously assigned to all channels behind all the other databases in the list.

Important Note

Please note that these changes may occasionally require modifications to the configuration. If, for example, you have defined a message symbolically in CAPL without qualifying it sufficiently, under certain circumstances a different type of message may now be derived. To find potential errors of this sort, you need to activate the warning option database symbol name is ambiguous on the CAPL tab of the Options dialog.
