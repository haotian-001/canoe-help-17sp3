Drift and Jitter

 

The Drift and Jitter options change the behavior of timers that are used in a network node model.

Drift sees to it that the running times of all timers that are started in the specific node are lengthened or shortened by a fixed percentage.

With Jitter the timer running times are changed randomly. Here the user specifies the interval within which the fluctuation should lie. The fluctuation is thereby uniformly distributed.

The parameter values are input in the relevant text boxes in percent. The maximum bandwidth for this is from â100.00% to +100.00%.

If the supplemental attributes Drift and Jitter have been input in the database for this node, checking the option From CANdb causes the database values to be used.

  • The key word for Drift in the database is: CANoeDrift.
  • For Jitter the lower limit is defined by CANoeJitterMin,
  • and the upper limit with CANoeJitterMax.

To learn how to define user-specific attributes in the database please refer to the CANdb++ Editor help function.

Settings for Drift and Jitter can also be changed directly from the specific network node model using the relevant CAPL functions. Please refer to CAPL Help regarding this.

The relevant functions are:

  • Drift: setDrift, getDrift
  • Jitter: setJitter, getJitterMax, getJitterMin
