CANoe/CANalyzer Compatibility

 

General

CANoe/CANalyzer compatibility mode makes it possible to use configurations in the same way in both tools. This means that a configuration created in CANalyzer can be loaded directly in CANoe. Likewise, CANoe configurations can be loaded and used in CANalyzer. However, a configuration created in one tool and loaded in the other affects only the functions that are available in both tools. Functions that are not available are not loaded.

To prevent the original configuration from being corrupted, a configuration loaded in the other tool can only be saved under a new name (File|Save As…).

CANalyzer Compatibility in CANoe

When a CANalyzer configuration is loaded in CANoe, only one block is loaded from the transmit branch. If there are several blocks, the first activated block is loaded. If all blocks are deactivated, the first block is loaded. On loading, the analysis branch of the Measurement Setup is preserved. This includes the database and channel assignments. A semantic analysis of CAPL programs is not carried out.

  • The CANalyzer Measurement Setup and transmit branch are mapped to the CANoe Simulation and Measurement Setups.

  • Note about CAPL programs

CAPL programs in the transmit branch typically have the following structure:

on message *
{
  output(this);
}

Because CAPL programs are not validated when a CANalyzer configuration is loaded, this structure results in an infinite loop and should therefore be removed from the CAPL programs. The CAPL compiler will show a warning notification in the Write Window if this happens.

  • The following CANalyzer functions are accepted in CANoe
  • The first active block of the transmit branch
    Program Nodes, Generators, Interactive Generators and Replay Blocks are accepted in CANoe
  • Channels and their database assignments for all bus systems
  • All analysis windows in the Measurement Setup
  • Panels
  • The following CANalyzer functions are not accepted in CANoe
  • CANalyzer receive branch
  • Filters and channel filters

CANoe Compatibility in CANalyzer

CANalyzer first switches to CANoe compatibility mode. This is indicated by a CANoe icon in the status bar. If you execute an action that makes the configuration incompatible with CANoe, a warning message appears. If this happens, you have the option of forcing compatibility mode. If you do this, you will not be able to add further blocks to the transmit branch. Alternatively you can ignore the warning message and continue working with the configuration, but it will no longer be compatible with CANoe. The CANoe icon disappears from the status bar to indicate this. You can also specify whether compatibility should be forced in the global options dialog under Settings|Compatibility.

It is not possible to force compatibility from an incompatible configuration. In this case you would need to remove the blocks from the current configuration.

When a CANoe configuration is opened in CANalyzer, the Measurement Setup is loaded in its entirety. The database and channel assignments are preserved. However, it is not possible to depict the contents of the CANoe Simulation Setup in the CANalyzer transmit branch. The transmit branch therefore remains empty.

  • The CANoe Simulation Setup and Measurement Setup are loaded in CANalyzer - the transmit branch remains empty

  • The following CANoe functions are accepted in CANalyzer
  • All analysis windows in the Measurement Setup except State Tracker
  • System variables
  • CANoe diagnostic settings are accepted in CANalyzer and used to configure the ISO TP observer
  • The following CANoe functions are not accepted in CANalyzer
  • Test setup with test modules
  • Test configuration with test units
  • Simulation Setup nodes and blocks
  • Environment variables
  • Filters for system and environment variables
  • State Tracker
  • I/O configuration
  • VT System configuration
