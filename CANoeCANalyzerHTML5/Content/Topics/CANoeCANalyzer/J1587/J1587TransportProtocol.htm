J1587 Transport Protocols

 

Overview

J1587 specifies two transport protocols, whose main characteristics are explained here in short.

  • Multisection – PID 192 (Page 1) / PID 448 (Page 2)
    Using the multisection parameter it is possible to transmit a parameter, whose message length would exceed the maximum of 21 bytes (as specified by J1708). The parameters' data is split up into several fractions, applied a sequence number, and then transmitted consecutively.
    The multisection protocol allows the transmission of a single, "long" J1587 Parameter. Its length (data portion - without MID, PID, length byte & checksum) must in between 18 to 239 bytes (page 1 parameter) and 17 to 226 bytes (page 2 parameter) respectively.
  • Transport Protocol with flow control (RTS / CTS) —
    PID 197 (CMP – Connection Management PID) & PID 198 (CDP – Connection Mode Data Transfer PID)
    This protocol provides for a handshake mechanism between two bus sharing units, where the receiving node acknowledges the submitted packets and requests the send node to continue its transmission.
    This transport protocol permits sending of any form-free byte sequences, which are interpreted by the respective receiver.
    The length of the data sequence fractioned and submitted with this transport protocol is limited to 3825 bytes.

Treatment and Usage of Transport Protocols within CANoe

  • Parameter Monitor, Diagnostics (DC) Monitor
    Parameter submitted via the multisection protocol are treated and displayed like unsectioned parameters.
  • CAPL
    The procedure to select a specific transport protocol to send J1587 parameters is described here.
  • Trace Window
    In addition to the display of all packets their contents are reassembled to a single (long) message which is shown at the end of a successful transport protocol submission.
  • Logging Block
    The output files of a Logging block list the assembled long messages resulting out of a transport protocol submission. In ASCII logs these are marked J1587+ to distinguish them from messages physically on the bus.
