New Features in Version 14

 

New Features in Version 14 SP2

General

  • General

  • ASAM XIL API

  • Improvement of the signal resolution.

  • System variables can now be given the same name as bus signals.

  • Communication Concept (SOA)

  • Simplified import process of data sources especially when a data source was changed.

  • Changes to the data model can be published at any time, making it unnecessary to accept/reject all changes in the Communication Setup.

  • Use cases were renamed to scenarios and the functionality was improved

  • PTP synchronization of the CANoe time is now possible with an external PTP master.

    The start time in logging files and in the Trace Window corresponds to the UTC time of the external master.

CAPL / CAPL Browser

  • CAPL

  • New/Extended CAPL Functions for CAN

  • New/Extended CAPL Functions for Car2x

  • New/Extended CAPL Functions for Communication Concept

  • New/Extended CAPL Functions for Security

  • New/Extended CAPL Functions for SmartCharging

  • New/Extended CAPL Functions for System Variables

Analysis

  • Trace Window

  • PTP synchronization of the CANoe time is now possible with an external PTP master. The start time in logging files and in the Trace Window corresponds to the UTC time of the external master.

  • Trace Window (Ethernet)

  • TC10 Sleep/Wakeup events are displayed and can be filtered.

  • IKEv2 key exchange events are displayed.

  • The IPSec AH protocol is visualized and higher protocols deserialized.

  • The IPSec ESP protocol is visualized and the encrypted data is shown.

Automotive

  • Option AMD/XCP
  • The XCP Observer now also displays the Overload flag, which is used by the XCP slave to mark DAQ packets after a data loss.

  • Option CAN

  • If a CAN frame is destroyed by an error frame, the payload data still transmitted can be queried via CAPL (getPayloadData).
    In addition, the data can be queried via selectors of the error frame event.

  • Option Car2x

  • General

  • The accuracy of the absolute measurement start timestamp has been increased in case the GNSS time is provided by the VN4610.

    This enables the absolute timestamps of the individual events in the Trace Window in Date and Time column being displayed with an accuracy of 1 ms. This applies to measurements with real bus as well as for import into the Trace Window and offline replay of BLF logging files.

  • The replay of logging files of format PCAPNG is supported.

  • The Data Window supports signals of type OCTET STRING, BIT STRING and IA5String.

  • CN: In the Map Window RSU and event positions for application messages RSI and MAP are visualized.

  • CN: The Certificate Manager supports the generation of Chinese PK12Is (SM2 certificate chains)

  • Car2x Interaction Layer

  • In case the GNSS time is provided by the VN4610, this time is used for the calculation of timestamps in Tx packets.

  • Packet API:

    With the new function C2xParsePacket an already encoded application message can be decoded after copying it into a Car2x packet. This enables to use and modify externally generated and encoded application messages.

  • Station API:

    By means of the new functions C2xSetStationAttributeInt64 and C2xSetStationAttributeDouble the attribute values of scenario stations can be set for individual keypoints during runtime.

  • EU: The Car2x Stack responds automatically to certificate requests (in the Security Header).

  • Sample Configurations

  • The sample configuration Test Service Library (Car2x) demonstrates the use of the Car2x Test Service Library functions.

  • CN: The sample configuration Basic Security (C-V2X) demonstrates the use of the Chinese security.

  • EU: New sample configurations are pre-configured with typical V2x scenarios.

  • Option Ethernet
  • Core Functionality
  • Support of extended sequence numbers in IPSec.
  • AUTOSAR serialization parameters for individual data member in an AUTOSAR event is now supported.
  • Network-based Access
  • The Windows network adapter can be used in network-based access mode.
  • Configuration issues are clarified with an information icon in the Port Configuration dialog.
  • Trace Window
  • TC10 Sleep/Wakeup events are displayed and can be filtered.
  • IKEv2 key exchange events are displayed.
  • The IPSec AH protocol is visualized and higher protocols deserialized.
  • The IPSec ESP protocol is visualized and the encrypted data is shown.
  • Programming
  • The CAPL Protocol API now supports optional fields for DHCPv4 and TFTP and IPv4/NDP/TCP options.
  • Testing
  • High level TC8 protocol tests (UDP, TCP, SOME/IP) can be tested using IPv6.

Diagnostics

  • DoIP over TLS
  • Additional options for the connection setup.

Industrial / IoT

  • Option CANopen
  • CANopen DBC databases do not require a CANopen license any more. The CANopen columns in the Trace Window are filled when a CANopen license is available.

Security

  • Observer for Transport Layer Security (TLS) - Available since version 14:
  • Display of the decrypted data in the Trace Window
  • Configuration of the Observer
  • Automatic (CANoe is terminal)
  • Input of the Master Secrets
  • Evaluation of a special UDP frame
  • Offline analysis of TLS communication
  • Observer configuration analog to real mode
  • For endpoint simulation, CANoe writes the Master Secret to the log file, e.g. for later analysis.
  • Support of Ethernet packet protection via IPsec Authentication Header (AH) - Available since version 14:
  • Different IPsec cipher suites available and configurable via Security Profiles
  • Compatible with higher layer protocols (SOME/IP, DoIP, TLS, HTTP, etc.)
  • IKEv2 protocol support (Internet Key Exchange v2)
  • Certificate based peer authentication
  • Dead Peer Detection, IKE Fragmentation and IKE Rekeying supported
  • Configurable via Vector Security Manager
  • Import of StrongSwan IPsec configuration possible
  • Full control of Security Policy Database

Smart Charging / E-Mobility

  • Option SmartCharging
  • GB/T 27930 Interaction Layer: better detection of protocol violations in the phase Charger suspends the charging process

Testing

  • VT System

  • The VT System can now also be operated with the new driver based HW sync.

  • VT6x04 System Variables

  • The System Variables RelayLowToVBatt, RelayLowToGnd, RelayHighToVBatt, RelayHighToGnd were replaced by two new system variables RelayCanHigh, RelayCanLow.
    This reduces dependencies between system variables.

  • Backplanes VT8006A + VT8012A

  • Automatic opening of the AGND-DGND connection when VT7001A or VT7101 is connected.

  • Display of warnings when ground connections are erroneous.

  • Both backplanes can now also be slaves for the HW sync.

  • A further relay on the backplane can now be switched by CANoe via CAPL.

  • The command line interface of the User FPGA Manager was enriched by various functions and multi-project capability.

  • Sample Configurations

  • New sample configuration for LV124 tests and VDA320 tests.

  • Update of the RS232 sample configuration.

  • Option For EtherCAT
  • Option For EtherCAT can now also be used on VN8900 devices.

  • Option Sensor

  • For sensors that are controlled via SENT or PSI5, the supply voltage can be switched on and off via CAPL (Switch Supply Voltage).

Truck / Agriculture

  • Option ISO11783
  • Virtual Terminal Interaction Layer
  • Support for color palette objects (including transparency and alpha channels) extends the implementation of the version 6 of the standard ISO11783-10.
  • Task Controller Interaction Layer
  • The new CAPL function TCIL_GetDeviceParameters provides parmeters of the client device which are contained in the Device XML element (DVC) of the device description.
  • Option J1939
  • J1939 Interaction Layer
  • Provides improved processing for violations of the J1939 transport protocol, widely compliant with the J1939-82 Conformance Test (2015).
  • Support for J1939-76 (Functional Safety)
  • J1939.dbc and J1939Base.dbc
  • New Safety Header Message (SHM) with all relevant signals.
  • Trace Window
  • Parsed content of the SHM message is displayed in a clearly readable way in the J1939 Interpretation column.
  • Violations of the J1939-76 standard (wrong checksum or sequence number, missing SHM, delayed SDM etc.) are reported in the J1939 Violation column.
  • GB/T 27930 Interaction Layer
  • Better detection of protocol violations in the Charger suspends the charging process phase.

New Features in Version 14

General

  • General

  • Logging

  • In the header of logging files comments can be added now. Thus logging related information can be documented.

  • New Format for the Help

  • New browser-based HTML5 format for Help replaces outdated CHM format:

    You can display Help content in multiple browser tabs/windows simultaneously.

  • More user-friendly interface:

    Full text search and index search are carried out via a single input box, which is always displayed in the upper right corner.

  • Improved display of search results:

    Searched text is highlighted in context.

    Page title, text excerpt, and the memory path of the help page are displayed.

  • See also Tips for using the Help.

  • Communication Concept (SOA)

  • Extension of the Communication Setup

  • Support in the Import Wizard for the separation of SUT and test environment.

  • Simple switching of use cases (measure/environment simulation/test the tester).

  • Navigation control with integrated search field.

  • View for communication details for communication objects (CO) and distributed objects (DO).

  • New editor for vCDL files, as plugin in the Vector Tools Environment.

  • The length of dynamic arrays (COs and DOs) can be set via panels.

  • Custom Binding/Mapping of DOs can be configured with the Symbol Mapping dialog.

  • The Binding Info of DOs is displayed in the Symbol Explorer, in the Data, Graphics and Trace Window.

  • The Participants and Application Layer Objects view in the Symbol Explorer have a new column that displays the data type of COs and DOs.

  • Vector Tool Platform

  • RT hardware (e.g. a device from the VN8900 device family) can be rebooted via Platform Manager or the Vector Tool Platform dialog in CANoe. The feature is also available via the Standalone Manager command line interface. In the same way the RT kernel of CANoe can be restarted on the RT hardware.

  • Status information related to the RT hardware can also be reported on standard output rather than using an INI file in order to simplify evaluation.

  • With the Event Control you can send events.

Analysis

  • Map Window
  • Map Window API: CAPL and .NET are available to draw in the Map Window.
  • New callback function: OnMapItemClicked. This function is called on clicking on an element that has been drawn by means of the Map Window API.
  • Vector provides an own OSM Tile Server for the Map Window.
  • Trace Window
  • The column filters have been revised. Besides multiple selection via a list, you can now configure more than two events in the custom filters.
  • Trace Window (Ethernet)
  • DoIP events are now displayed up to a size of 64 kB.
  • All special IPv6 addresses are now displayed in the detailed view.

Automotive

  • Option Car2x
  • Application Messages (EU)
  • Support of SREM and SSEM messages
  • Update of IVI to version ISO/TS 19321:2020 / SAE-J2735
  • Update of MAP/SPaT to version ISO TS 19091:2019
  • C-V2X
  • Support of the Autotalks HW Craton2 as third party C-V2X communication hardware
  • Chinese V2X Protocol Stack
  • Support of DSMP (since CANoe 13.0)
  • Support of Security Header (incl. signing + verification)
  • Update of application messages (BSM, MAP/SPaT, etc.) in the database
  • The Car2x Interaction Layer can be configured for the Chinese protocol stack.
  • The Scenario Editor can be used as basis for scenarios.
  • BSM as well as MAP/SPaT are visualized in the Map Window automatically.
  • Driving Tests
  • Network nodes can be configured in the database to use the current position data of e.g. a VN4610 directly in the Car2x stack. This allows the retrofitting of a conventional vehicle to a 'Car2x vehicle' (without the need to use CAPL) and the utilization for driving tests.
  • The current position (e.g. from a VN4610) can be used as starting point for a scenario.
  • Scenario Editor
  • The Help provides videos for an easy start with the Scenario Editor.
  • Map Window
  • Vector OSM Tile Server : Vector provides an own OSM Tile Server. In CANoe it is possible to configure this server as the tile provider, so that problems regarding the download of tiles belong to the past.
  • The help provides a description how infrastructure data (e.g. traffic light signal states) is drawn into the Map Window.
  • New callback function: OnMapItemClicked . This function is called on clicking on an element that has been drawn by means of the Map Window API in the Map Window.
  • Sample Configurations
  • New sample configurations contain pre-configured Chinese V2X scenarios, based on T/CSAE 53-2017
  • The EU Car2x System Demo has been revised and uses the current features and Car2x stack functions.
  • Option Ethernet
  • Core Functionality
  • Support for IPsec (Transport Mode, Authentication Header [RFC 4302] and IKEv2 Key Exchange [RFC 7296]).
  • More accurate calculation of the busload (taking into account IFG, preamble and SOF).
  • The number of templates has been reduced. There is one template for the Simulation Setup and one for the Communication Concept. Both templates use network-based access
  • Network-based Access
  • Visual support of measurement and simulation ports has been introduced in the Port Configuration dialog , including the display of configuration errors.
  • It is now possible to configure filters for measurement and simulation ports in the IP Filter . This allows port information to be filtered in the measuring setup for the other blocks such as Trace and Logging etc.
  • Trace Window
  • DoIP events are now displayed up to a size of 64 kB.
  • All special IPv6 addresses are now displayed in the detailed view.
  • Programming
  • In .NET the MAC addresses can now be specified in lowercase.
  • It is possible to change the payload type for RTP messages in CAPL.
  • Testing
  • TC8: Implementation of all SOME/IP-ETS tests are now included in the TC8 sample configuration. The SOME/IP-ETS tests are based on the TC8 test specification v3.0.
  • Fuzzing: Fuzz-testing solution for the SOME/IP-SD protocol has been included as sample configuration.
  • Option LIN
  • LIN Interactive Scheduler (LIN ISC)
  • Creation of application-specific schedule tables for test purposes.
  • Extended fault injection for LIN TP.

  • Option For EtherCAT
