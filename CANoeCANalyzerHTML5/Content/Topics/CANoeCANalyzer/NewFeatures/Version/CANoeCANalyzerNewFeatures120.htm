New Features in Version 12.0 SP4

Analysis

Automotive

  • Option Ethernet
  • Port-based Network Access (starting with VN firmware version 11.1)

  • The Ethernet Packet Builder supports port-based network access.
  • At measurement start a link status event is generated for each used Ethernet port.
  • Fix an issue which may prevent the execution of the .NET callback function OnEthernetPacket for each port in the analysis area of CANoe.
  • Improved display of ports in the Port Configuration dialog.
  • Modification of default network names in Ethernet1, Ethernet2, etc.
  • The configuration of a signal protocol DLL has been moved into Options dialog.
  • Fix an issue which may lead to the deactivation of time synchronization in case multiple network interfaces are used.
  • Stability improvement when opening another configuration in case the current configuration utilizes the global TCP/IP-Stack and port-based access.
  • Stability improvement when opening an invalid Ethernet interface configuration.
  • Fix an issue which may lead to a loss of some configuration contents in case the configuration is stored for an old version (Save As).
  • Fix an issue which may lead to the display of unused networks in Port Configuration dialog.
  • Fix an issue which may lead to the display of unused networks in Application Channel Mapping dialog.
  • SOME/IP Interaction Layer
  • Fix an SOME/IP server issue which may lead to utilize incorrect UDP/TCP ports.
  • Trace Window
  • Fix an issue which may prevent the interpretation for SOME/IP events in case the Communication Concept is used.
  • Fix an issue which may lead to using a wrong sender information in case of drag-and-drop of an event into the analysis filter.

Simulation / Stimulation

  • Ethernet Packet Builder
  • The Ethernet Packet Builder supports the port-based network access.

Truck / Agriculture

New Features in Version 12.0 SP3

General

  • ASAM XIL API
  • The XIL API can now be used directly without FDX description.
  • Communication Concept (SOA)
  • AUTOSAR Adaptive Platform databases with schema 19-03 can now be imported.
  • FIBEX databases are now supported.
  • Abstract interface definitions can be imported from ARXML databases.
  • The Communication Object Editor can be used to edit the SOME/IP Binding.
  • Usability improvements in the Communication Setup.
  • Offline/Logging
  • ASCI loggings contain a unique ID of the measurement to be able to navigate from the Vector CANoe Test Report Viewer.
  • Metadata are now also written to MDF/MF4 files.
  • In the file list of the Offline Mode Window several files can now be activated and deactivated at the same time.
  • Quick Start Guide
  • The CANoe Quick Start Guide was integrated in the help.
    To print topics easily a PDF version is available with all topics of the Quick Start Guide.
  • Symbol Mapping
  • The two new mapping settings OnTimerValueChanged and OnTimerValueUpdated allow a cyclic mapping of the source symbol after a given timer elapsed.
    Thus, values from models with very short cycle times (high required computing power) can be mapped to signals with longer cycle times.
  • Size and position of the Symbol Mapping dialog are stored in the configuration.
  • VTP/VN8900
  • In distributed mode, the user currently occupying the RT server is displayed. You can also configure that the connection is automatically terminated at the end of the measurement and re-established before the next measurement start to avoid conflict situations.
  • VN8911 and VN8914 devices that are equipped with a current image (version ≥ 3.0) and special WiFi sticks can be used in distributed mode or in standalone mode via WiFi.
  • You can export and import the channel configuration on RT hardware (e.g. VN8900). This is possible via the user interface or it can be automated using the Standalone Manager command line interface.

Panel Designer

  • Graphical User Interface (GUI)
    The user interface of the Panel Designer is now High-DPI capable. On high-resolution monitors, symbols and texts are adapted to selected resolution and scaling.
  • Input/Output Box

    The display can be set as default.
  • Method Call Control

  • When copying a control, the new control gets a unique name (control type with unique index).
