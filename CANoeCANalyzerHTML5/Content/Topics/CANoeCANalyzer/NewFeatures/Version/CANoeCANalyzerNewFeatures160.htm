New Features in Version 16

 

New Features in Version 16 SP3

General

  • General

  • Communication Concept (SOA)

  • The Application Panel supports now member container of distributed objects (lists or arrays). It is possible to select an item and modify its values.

  • Container of distributed objects are displayed in the Communication Setup.

  • Multi CANoe

  • The Multi CANoe cluster communication has been extended by a life sign, which enables the detection of a connection loss.

  • The Multi CANoe cluster can be used in a IPv6 network. You can add the IPv6 address of the Multi CANoe master and slave in the Options dialog.

  • MATLAB

  • Enhanced logging information:

  • In order to streamline the support process and to reduce iterations between Vector and the customer the code generation writes more and additional information to the build log file.

  • CAPL

  • New/Extended CAPL Functions for Ethernet

  • New/Extended CAPL Functions for FlexRay

  • New/Extended CAPL Functions for ISO11783

  • New/Extended CAPL Functions for J1939

  • New/Extended CAPL Functions for SmartCharging

ADAS

  • ADAS Feature Set

  • The ADAS Test Feature Set (TFS) function names were renamed. Thus, it might be necessary to adjust the CAPL or C# code.

Analysis

  • Protocol Monitor

  • When the Endpoint name cannot be display completely in the Diagram View, the complete name will be shown in the tooltip for this endpoint.

  • The Detail View has been improved. The details of the selected object are displayed together with the high-level information of connected objects. If you want to see the details of a connected object, you can select it immediately in the Detail View.

  • Trace Window

  • Ethernet

  • The Web Socket unmasked payload data is now also shown in the Trace Window column Payload Data ASCII, in addition to the detailed view. This way you can inspect the payload data without having to select each event separately.

  • The filter conditions for the SOME/IP and the SOME/IP-SD filters are now also available from the context menu of the filter groups of the analysis filters.

Automotive

  • Option Car2x

  • General

  • For C-V2X, up to 8 C-V2X channels are now supported. Previously, 4 channels were supported.

  • Car2x application channels can be used together with the SIL Kit.

  • Car2x Protocol Analyzer

  • Fundamental checks for the Basic System Profile R1.6.2 (EU) are supported for CAM and DENM. Detected errors and warnings are highlighted in the Trace Window. Detailed information can be found in the Car2x Protocol Analyzer window. For most checks, a reference to the corresponding standard or BSP requirement is available (tooltip in the overview area). This helps development and test engineers to detect weaknesses in their implementation, check them in the standards and profiles, and finally improve the quality of V2X communication of their devices.

  • Map Window

  • The station type of the Chinese ITS stations is now displayed with a matching symbol.

  • V2X Stack (EU)

  • By means of an offset for an ITS station on a route, a PathHistory can be generated already at the start of a scenario.

  • For transmitting the CAM, the following signal structures are additionally filled with values derived from the scenario: yawRate, longitudinalAcceleration and curvature.

  • GeoBH Lifetime is now correct for DENMs that use repetitions.

  • For update DENMs, the DetectionTime is now adjusted each time before sending.

  • The relevanceTrafficDirection and the relevanceDistance of a DENM are now set every time the message is sent. This allows starting different scenarios with different events during the measurement runtime.

  • Sample Configurations

  • System Configuration (Car2x) now also demonstrates the Car2x Protocol Analyzer in a separate desktop view. It is also possible to trigger errors intentionally using a panel in order to be able to check them in the Car2x Protocol Analyzer Car2x Protocol Analyzer.

  • Option Ethernet

  • Performance

  • The performance of the SOME/IP stack has been improved.

  • Security

  • A step-by-step guide for configuring software-based MACsec has been added to the help description.

  • The remaining bus simulation can be configured to support software-based MACsec for one DUT pro Ethernet network. The DUT should be connected to a direct link or to a switch with only one physical port.

  • The AUTOSAR and SOME/IP modeling libraries provide callback functions to monitor the (D)TLS state. The OnSomeIpEstablishedTLSConnection/OnAREthEstablishedTLSConnection and OnSomeIpClosedTLSConnection/OnAREthClosedTLSConnection can be used to track the opening and closing of (D)TLS connections.

  • SOME/IP

  • The SOME/IP service instance can be set at runtime with the CAPL funtion SOMEIP_ChangeServiceInstanceID.

  • With the new CAPL function SomeIpFillValues/AREthFillValues it is now possible to activate optional field, resize arrays and fill fields with values of an entire SOME/IP message.

  • Notification in the Write Window about the aborted serialization at an overflow of the length field.
    If the remaining bus simulation serializes a Some/IP object containing an array, struct or union, it should be checked that the serialization length fits into the length field. If not, it would cause an overflow of the length field. If not, this would result in an overflow of the length field.

  • The modeling library now provides functionality to block the automatic sending of the response by means of the return value. <OnSomeIpMethodRequest> with the appropriate return value can be used for fault injection and testing purposes.

  • The database reader now supports an alternative way to describe events without parameters (usually used for describing fire and forget methods in classic AUTOSAR). The alternative description allows events without parameters to be described as a variable data prototype without a reference.

  • Trace Window

  • The Web Socket unmasked payload data is now also shown in the Trace Window column Payload Data ASCII, in addition to the detailed view. This way you can inspect the payload data without having to select each event separately.

  • The filter conditions for the SOME/IP and the SOME/IP-SD filters are now also available from the context menu of the filter groups of the analysis filters.

  • Protocol Monitor

  • When the Endpoint name cannot be display completely in the Diagram View, the complete name will be shown in the tooltip for this endpoint.

  • The Detail View has been improved. The details of the selected object are displayed together with the high-level information of connected objects. If you want to see the details of a connected object, you can select it immediately in the Detail View.

  • Option FlexRay

  • With the new CAPL function frChannelUsesCANoeNetworkSimulator it is now possible to get the information whether a given channel uses the built-in network simulator of CANoe.

  • Option LIN

  • New COM method AutogenerateNodes() automatically generates a LIN network based on the assigned database.

Diagnostics

  • Diagnostics

  • The new Diagnostic Feature Diagnostic Console can be configured as part of the Diagnostic Window. In future versions of CANoe this new Diagnostic Feature will replace the old window Diagnostic Console.

Industrial / IoT

  • Connectivity

  • DDS support as binding within the communication concept:
  • Modeling of DDS topics and DDS participants in vCDL.
  • DDS monitoring and DDS debugging (basic scope).

  • Use cases:

    - Simulation/Test of DDS based systems.

    - Integration with ROS2 environments.

  • MQTT:
  • You can authenticate your client simulated in CANoe against a local broker.

  • Option CANopen

  • If a CANopen DBC was generated, this generation process is performed automatically when the configuration is stored via Save As at a different location.

Testing

  • Testing

  • Check statistic for Error Frame Count now holds bus information.

  • Together with vTESTstudio 7 SP3

  • Extensions for Python as test design language:

  • Access to variants and parameters in parameter files.

  • Python functions and test cases with parameters of enumeration types can be used in the Test Table Editor and the graphical editors.

  • Several further improvements as drag and drop from the Symbol Explorer and additional reporting functions allow an efficient test implementation in Python.

  • TestUnitBuildCLI now provides JSON format as output. With this format the build results can be easily processed without parsing every single build message, e.g. get a list of all failed test units.

  • CANoe Test Report Viewer

  • Indication of partially executed tests:

    Icons for grouping elements like test groups or test fixtures now feature an indicator if only a part of their test cases were executed.

  • Invocation of mini toolbar via mouse right-click:

    The mini toolbar available in all content views can now not only be invoked by hovering, but also by right-clicking on any element.

  • Navigation to activities by Verdict:

    New ribbon buttons allow to quickly navigate over all activities with a verdict that is worse or equal to an adjustable limit.

  • NuGet package for Data API:

    The Data API now comes as NuGet package instead of individual assembly files. The package is available in the CANoe Test Report Viewer installation and also part of the Data API sample project. The package is compatible with both .NET Framework 4.8 and .NET 5 or later.

Truck / Agriculture

  • Option ISO11783

  • Interaction Layer Iso11783 IL, VT IL, TC IL and FS IL:
  • The following CAPL functions allow arbitrary manipulations of the J1939 transport protocol:
  • Iso11783IL_FreezeTp / Iso11783IL_UnfreezeTp / Iso11783IL_KillTp
  • VTIL_FreezeTp / VTIL_UnfreezeTp / VTIL_KillTp
  • TCIL_FreezeTp / TCIL_UnfreezeTp / TCIL_KillTp
  • FSIL_FreezeTp / FSIL_UnfreezeTp / FSIL_KillTp
  • New templates for Iso11783 configurations:
  • New configuration templates are available, containing Iso11783 database, preconfigured Virtual Terminal, Task Controller, File Server and CAPL filter modules.
  • Virtual Terminal:
  • Deviations from requirements of the AEF Conformance Test have been resolved.

  • Option J1587

  • No time delay at the start of measurement if multiple J1587 channels are present.

  • In the Parameter Monitor the multi-digit values for PID are no longer truncated.

  • Option J1939

  • J1939-22 (CAN FD):
  • Map Window supports J1939-22.
  • New templates for J1939 configurations:
  • New configuration templates are available, that include a J1939 database.
  • J1939 Interaction Layer:
  • Additional CAPL functions allow arbitrary manipulations of the J1939 transport protocol:
  • J1939ILFreezeTp / J1939ILUnfreezeTp / J1939ILKillTp
