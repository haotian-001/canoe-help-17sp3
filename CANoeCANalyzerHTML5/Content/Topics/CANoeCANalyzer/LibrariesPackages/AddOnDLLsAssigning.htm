Assigning Modeling Libraries

 

Introduction

The following describes how modeling libraries (interaction layer, transport protocol, network management, and customer-specific expansions) and test modules are integrated into a CANoe configuration.

The standard installation of CANoe contains a few sample configurations, into which such a modeling library has already been added; e.g. AUTOSAR network management for CAN and FlexRay.

Note

For OEM-specific add-ons, you install the add-on beforehand.

Where are the modeling library files located?

There are two standard paths where modeling libraries can be found:

1. <%programdata%>\Vector\<sub path>

  • This installation path is used by new add-ons and CANoe itself.
  • The modeling libraries for different variants (64/32 bit, Linux) and the corresponding module description file are always installed here.
  • This default path must also be used to easily configure modeling libraries in vTESTstudio and then load them in CANoe during test execution.

2. <CANoe setup directory>\Exec64 bzw. Exec32, Modules_Linux-x64

  • The module description files in the Exec64 directory.
  • The different variants of the modeling libraries are located accordingly in the Exec64/Exec32/Modules_Linux-x64 directory.

Cross Reference

For more information on loading modeling libraries, see Search Sequence for Modeling Libraries.

Which version of the modeling library is currently installed?

You can find the version of the modeling library loaded in CANoe on the File ribbon tab|Help|Details.

There are various options for assigning a modeling library file. These are presented in the following.

Assigning a Module Description File or DLL File via nodes in the Simulation Setup

Note

The usage of a module description file is described here. If this is not available, the DLL file can also be used directly.

You can assign the module description file to all nodes of the Simulation Setup at once, by using the network node.

Assigning to all nodes

  1. Open the context menu of the network node, and click Assign Module to all Network Nodes.
  2. In the following dialog, select the module description file and confirm with [OK]. It will be assigned to all nodes.

Conversely, you can remove existing module description file with the Remove Module from all Network Nodes context menu command.

It may be necessary to assign a module description file to an individual node only. This is done in the Simulation Setup as described below:

Assigning a DLL file or module description file to an individual node

  1. Open the context menu of an individual network node, and click Configuration.
  2. In the following dialog, open the Components page.
  3. Click the [Add] button to open a dialog for the module description file selection.
  4. After confirmation with [OK], the module description file will be assigned only to the particular node.

You can select unneeded module description file in this dialog and then click the [Remove] button to remove them.

Note

In the selection dialog you will find corresponding shortcuts to the standard paths for modeling libraries on the left under the heading Vector CANoe [Version].

Note

You can also use the above-described method to assign module description files or DLL files for test modules via the context menu of the node in the test setup.

Assigning a Module Description File or DLL File for CAN using the CANdb++ Editor

Note

The usage of a module description file (starting with CANoe 12.0 SP3) is described here. If this is not available, the DLL file can also be used directly.

In the CANdb database, you can assign the value with the name of the module description file to the NodeLayerModules attribute. The attribute can contain a comma separated list of filenames (with filename extension; but without path) and without spaces (!).

Assigning a module description file as the default for all nodes

  1. Open the CANdb++ Editor.
  2. Select the View|Attribute Definitions menu command.
  3. Open the context menu, and click New.
    Note: Only one attribute of the same name is permitted.
  4. Enter the name NodeLayerModules for the attribute on the Definition page.
  5. Open the Object Types list, and click Node.
  6. Open the Value Type list, and click String.
  7. In the Default input field, specify the filename of the module description file.
  8. After confirmation with [OK], the module description file will be stored in all nodes.
  9. Then save the database.

Assigning a module description file to an individual node

  1. Open the CANdb++ Editor.
  2. Select the View|Overview menu command.
  3. With a double click on a node your open the associated dialog.
  4. You can adapt the attribute for the node on the Attributes page. For example, you can change the name of the module description file for the particular node in the Value input field, if required.
  5. After confirmation with [OK], the module description file will be assigned to the corresponding node.
  6. Then save the database.
