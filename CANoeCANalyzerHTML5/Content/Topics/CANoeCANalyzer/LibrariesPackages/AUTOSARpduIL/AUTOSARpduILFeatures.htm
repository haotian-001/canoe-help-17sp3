Global System States [▲ back]

There exist two global system states that are handled by the AUTOSAR IL modeling library:

Ignition

The Ignition state is represented by the global system variable sysvar::_ILControl::Ignition, which can be set to 0 (off) or 1 (on).

  • When the Ignition is on, then the IL will prevent an attached NM modeling library at the same node to release the bus.
  • If Ignition is switch from off to on, then the IL will request the bus at an attached NM modeling library at the same node.
  • Otherwise, when the Ignition is going off, then the attached NM modeling library at the same node will release the bus after the NM sleep timer expires (see Usage of the Interaction Layer).

Notes

  • When an NM modeling library at the same node releases the bus, then the IL modeling library will be set to waiting (see Interaction Layer State Model). Thus, the IL stops sending.
  • When an NM modeling library at the same node requests the bus (possibly due to switching the Ignition to on), then the IL modeling library will go to running (see Interaction Layer State Model). Thus, the IL starts sending.

The Ignition state can also be controlled by the CAPL functions ARILSetIgnitionState and ARILFaultInjectionSetIgnitionState.

Power

The Power state is represented by the global system variable sysvar::_ILControl::Power, which can be set to 0 (off) or 1 (on).

  • When the Power is off, then the IL modeling library will be disabled (like calling ARILControlSimulationOff). Thus, the IL will not send anything.
  • The IL can be operated only, when the Power state is on.

The Power state can also be controlled by the CAPL functions ARILSetPowerState and ARILFaultInjectionSetPowerState.

Sending Model [▲ back]

Fault injection is possible for the PDU sending model (see Timing Modification).

PDU Transmission

The Interaction Layer controls the sending of the PDUs. PDUs can be sent periodically, sporadically or both.

Transmission Mode

The AUTOSAR system description defines for each PDU a True and a False transmission mode, which is activated upon the data filters of all signals of this PDU. The conditions of all data filters of one PDU are or-ed. Thus, depending on the signal values and their changes a dedicated transmission mode is activated.

Cyclic-Timing

The cyclic-timing defines the transmission period for an optional cyclic sending of the PDU.

Event-Timing

The event-timing defines (if activated) whether the PDU is transmitted regardless of the cycle period when any transfer property of a signal or signal group fires/triggers. Optionally the event-timing can define a repeated transmission after the event occurred.

Message Counter [▲ back]

Some PDUs or signal groups contain a message counter (SQC for Sequence Counter). The SQC is an optional counter and will be incremented before each sending onto the bus.

The End2End protection info of the PDU or signal group defines by its profile name the calculation rules of the signal and the position and size of the signal in the payload.

Notes

  • On each transmission of a PDU, a possibly defined message counter is incremented by the AUTOSAR PDU IL.
  • The MC/SQC of a signal group is always incremented regardless of the update bit. For signal groups with update bits, the MC/SQC of the signal group can optionally only be incremented, when the update bit of the signal group is set., then this can be achieved by decoupling the update bit from the SQC by calling the function ARILSetOperationMode with appropriate parameters.

Fault injection is possible for the MC/SQC calculation (see Fault Injection and Disturbance).

CRC Value [▲ back]

Some PDUs or signal groups contain a CRC (Cyclic Redundancy Check) value. It is calculated from the other bytes of the messages including the message counter and optionally a given start value or additional data ID.

The calculation of the CRC is based on different polynomial and optionally additional start values and/or data IDs. The End2End protection info of the PDU or signal group defines by its profile name the calculation rules of the signal and the position and size of the signal in the payload.

Note

The CRC is calculated by the AUTOSAR PDU IL automatically on each sending.

Fault injection is possible for the CRC calculation (see Fault Injection and Disturbance).

Update Bit [▲ back]

These special signals are defined within the AUTOSAR System Description. It can also possibly be identified as a signal with a dedicated naming convention:

The signal name has the suffix _UB.

Notes

  • Update bit signals are automatically set or reset by the IL according to an update of the corresponding application signal or signal group. Thus, update bits are 0 as long as the corresponding application signal or signal group has not been updated. If the corresponding application signal or signal group has been updated, then the update bit is automatically set once and automatically been reset directly after the transmission of the corresponding PDU. This is called Normal/Real Update Bit Mode and is active by default.
  • In contrast to the Normal Update Bit Mode also the Plug&Play Update Bit Mode exists. This mode must explicitly be activated (see function ARILSetOperationMode. When the Plug&Play Update Bit Mode is active, then update bits (application signal update bits and signal group update bits) are always continuously set to 1 regardless of any update or PDU transmission.

The AUTOSAR PDU IL provides fault injection functions in order to disturb the automatic handling of the update bits (see Fault Injection and Disturbance).
