AUTOSAR PDU Interaction Layer

 

General

Function Overview

This is a description of the functions of the of the AUTOSAR PDU Interaction Layer (short AUTOSAR PDU IL), which is based on the module description file Enables support for different RT kernel architectures (64 bit/32 bit/Linux) without having to change the configuration. The interaction layer is responsible for managing the transmission model and performs the following tasks:

  • Controls the transmission of PDUs dependent to the send types of signals and PDUs.
  • Provides the possibility to send PDUs on request (using a signal or a PDU as triggering object).
  • Provides a fault injection interface to influence the sending of PDUs.
  • Automatically calculates SQC and CRC of the E2E protection in PDUs.

Caution: Release and Compatibility

Prerequisites

Always use the VMODULE file from your global add-on installation directory ("%PROGRAMDATA%\Vector\AddOn Packages\Vector AddOn\vModules")! Possibly the vModule and the DLLs/SOs referenced in the vModule are automatically updated by installing an OEM specific add-on.
  • Starting with version 9.0 SP3 (9.0.86) for usage with CAN and Ethernet.
  • Starting with version 9.0 SP4 (9.0.111) for usage with FlexRay.

Tip

Get application callbacks (hooks) on transmit activity of the Interaction Layer

It is possible to get an application callback within CAPL before the IL sends a PDU to the bus. In this callback it is possible to prevent the sending of the PDU or to change the data of the PDU.

dword applPDUILTxPending(dword busContext, dword longID, dword shortID, char name[], dword & aPayloadLength, byte data[]);

See Callback Interface functions for details.
