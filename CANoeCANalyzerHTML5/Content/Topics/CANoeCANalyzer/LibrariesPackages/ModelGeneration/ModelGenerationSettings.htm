Settings

Ribbon: Simulation Tab|Model Generation Wizard

With the [Generate] button you start the model generation with the current settings described below. The model (CANoe configuration) will be generated in the folder you define in the Output Path input box. The generation process is shown in the Output area.

Note for the settings display

The displayed settings depend on the selection under

  • OEM
  • Panels
  • Variant
  • Database

This help page describes basic OEM-specific settings.

Not all setting options are available for all OEM-specific model generations.

Generation Parameters

OEM
In this list per default the model generation with the CANoe IL is available. With each installed OEM add-on an OEM specific variant of the model generation is available.

Create Configuration
When you activate this option a new configuration will be created. Please select the network database and verify the output root folder.

Extend Configuration
When you activate this option a new bus will be added to the configuration that is currently loaded and active in CANoe. Please select the network database; the output root folder is retrieved from the CANoe configuration automatically.

Note

Use the Extend Configuration mode only with configurations that were initially created by the Model Generation Wizard in the Create configuration mode. Don’t modify any configuration settings, e.g. channel settings, before the model generation is finished.

Panels (check boxes)

  • Modern
    Generates new user editable panels with signal and (optionally) disturbance controls per message. Corresponding load buttons are generated for the Rx and Tx direction of each node in the main panel.
  • Node Panels
    Generates load buttons for the CANoe intrinsic node and signal panels in the Main Panel.

Note

These buttons work only during measurement. Optionally additional disturbance panels can be generated.
  • No Panels
    There is not any panel generated. (This selection overrides all other selections).
  • Simple Main Panel
    Generates a simple Main Panel without any sub-panels for the signal and message updates or receptions. Disturbances are also not available. (This selection overrides the Modern and Node Panels selections).
  • Diagnostic Panels
    This will enable the selection of a diagnostic description and generates the Diagnostic Console and Fault Memory window in CANoe.
  • PRBS Mode
    This feature allows to select a target node set. According to that selected nodes only the PDU panels, respectively message panels, will be generated for those PDUs that are sent to or are received from those node set. This allows to dramatically reduce the amount of generated PDU, resp. message, panels.

Variant (check boxes)

Note

The following selections are only a hint for the model generation. In fact the database with its node attributes defines which NM has to be used.
  • OSEK NM
    Per default any NM node uses the module description file Enables support for different RT kernel architectures (64 bit/32 bit/Linux) without having to change the configuration. OSEKNM01.vmodule that implements the OSEK NM for the OEM. The required DLL is referenced in the module description file (*.vmodule).
  • AUTOSAR NM
    Per default any NM node uses the AsrNM33.vmodule that implements the AUTOSAR NM for CAN or uses the AutosarFlexRayNM32.vmodule that implements the AUTOSAR NM for FlexRay. The required DLL is referenced in the module description file (*.vmodule). The appropriate module description file (*.vmodule) is chosen upon the database format (and/or the cluster type).
  • AUTOSAR NM with PN
    Per default any NM node uses the AsrNM33.vmodule that implements the AUTOSAR NM for CAN or uses the AutosarFlexRayNM32.vmodule that implements the AUTOSAR NM for FlexRay. The required DLL is referenced in the module description file (*.vmodule). The appropriate module description file (*.vmodule) is chosen upon the database format (and/or the cluster’s type).
    This selection allows selecting a VSDE extension file that specifies the NM filter masks and partial networks (PNs) for each NM node.
  • Without NM
    Per default each node does not use any NM protocol. Thus, a NM module description file (*.vmodule) will not be used. Nevertheless, a NM panel is generated in order to display the nodes state.

Settings (check boxes)

Create CANoe Configuration
Depending on the generation mode, a new CANoe configuration is created or an existing one is modified. If disabled, then any existing CANoe configuration file (*.cfg) will not be modified, nor newly created. Nevertheless, if the option is disabled, other files (e.g. CAPL, panels, etc.) will still be created, deleted or modified.

Cleanup output path before generation
Before the generation process starts the output folder is cleared.

Copy DLLs
All required modeling libraries (module description files, node layers) are copied into an Exec32 folder in the output path. It is highly recommended to disable this setting for using the modeling libraries from their original installation directories.

Use 32-bit RT kernel
When using a CANoe starting with 17.0, the MGW creates a CANoe configuration, where the execution environment in CANoe is set to the 64-bit RT kernel by default. This setting allows to create a CANoe configuration, where the execution environment is set explicitly to the 32-bit RT kernel.

Use 32-bit RT DLLs instead of vModules
By default for node layers like IL or NM DLL the VMODULE files are referenced. This allows switching the runtime kernel architecture of CANoe from 32-bit to 64-bit. If this switch is enabled, then instead of the VMODULE files directly the 32-bit DLLs are referenced. In this case the runtime kernel of CANoe cannot be used in 64-bit mode! This switch is only available for compatibility reasons.

Global System State Control
Creates handling of global ignition switch signal.

SysVar naming in version 0
Create system variables with old naming as in former add-ons.

Global Time Support
Create CAPL code for sending (master) or receiving (slave) AUTOSAR Global Time synchronization messages (if CANoe ≥ 9.0 SP4).

Use SysVars for big Rx/Tx data signals
If activated, Rx/Tx panels are supporting byte array controls for big signals (> 64 bit). If activated the communication between signal and panel is handled by CAPL and a dedicated system variable (per direction).

Select Generation Input

Database
Enter DBC, FIBEX or AUTOSAR database in this input box. The [Select...] button allows the selection of as dedicated network contained in the database.

Output Path
Enter optionally the output path (if not already set). You can use the [Select...] button.

Database Nodes

Via the check boxes select the nodes you want to be considered during generation (default: all). You can use also the [Select All] or [Unselect All] button.

Extended Parameters (multiple selection is possible)

  • IL used
    This setting is enabled by default and will attach the Interaction Layer vModule to each node.
  • IL disturbance control
    This setting is enabled by default and will create appropriate disturbance controls in the message panels. These controls allow to change the cyclic sending of a message, and to disturb the automatic CRC/CHK and message counter calculation of the Interaction Layer.
  • IL disturbance panel
    If this setting is enabled, then the disturbance controls are generated in dedicated disturbance panels and not inside the Message Panels.
  • Copy DB
    If this setting is enabled, then the databases will possibly copied into the folder hierarchy of the output folder for the generation (if not already present there).
  • NM panel
    This setting is enabled by default and will create a new NM panel in order to control each nodes NM behavior.
  • Use node name in CAPL message name
    This setting is enabled by default. It will use in CAPL programs qualified names for messages by taking the Tx node name into account. Some databases require disabling this setting, because the node name is also used in value tables of signals.
  • Configure SecOC
    Allows to select a security profile on the Sources TAB page. With this selection the Secured On-Board Communication is supported in the final model.
  • New NM control
    This setting enables the generation of the new NM control in the NM panel, that supports also partial networking bits (Default is on; requires CANoe ≥ 9.0 SP4).
  • Create FR-CAN diagnostic gateway node and diagnostics CAN bus:
    This option is only of interest and function when adding a FlexRay bus.
  • Force AsrPDUIL2 model
    Use alternative AUTOSAR PDU IL (AsrPDUIL2.vmodule) using a more compatible CAPL interface (for a combined usage with a CAN IL in gateway nodes).
  • Inhibit AutoSAR 4.2 FlexRay PDU model
    Although the database is based on AUTOSAR version 4.x (or greater), the AsrPDUIL.vmodule or AsrPDUIL2.vmodule is not used. Instead the previous FlexRay OEM-specific IL DLL is used and CANoe’s new multibus PDU Layer is disabled for FlexRay buses.
  • Enable FlexRay AB mode
    With this setting the interpretation of redundant frames and PDUs in a FlexRay configuration with channels A and B can be controlled by one instance. If the dual channel support is enabled, PDUs and frames on both channels, A and B, will be configured with channel mask AB (instead of two instances with mask A and B separately).
  • Append channel to bus and DB name for ECU extract
    When using an ECU extract for generating, the name of the network in the Simulation Setup and the alias name of the database in the Simulation Setup is set to <Cluster name>_<Channel No.>, where the Cluster name is the name of the bus in the database.
  • Append channel to bus and DB name for system description
    This activates the same behavior as the switch Append channel to bus and DB name for ECU extract also for a system description (resp. all other types of databases except ECU extracts).
  • Prevent gateway nodes
    Gateway nodes are assigned as different nodes to each bus separately. Thus, they are not configured to be gateways in CANoe and there is a CAPL program available for each bus.
  • Prevent renaming of DB name
    The alias name of the database in the Simulation Setup is not changed according to the switches Append channel to bus and DB name for ECU extract or Append channel to bus and DB name for system description. Thus, those switches only rename the network name in the Simulation Setup.
