Model Generation Wizard » Panels

Generated Panels

 

The following panels will be created at the generation of a CANoe configuration with the Model Generation Wizard. The creation of the panels depend on the panel settings.

On the help page you find information about the handling of these panels.

Main Panel (with CANoe IL)

The Main Panel displays all nodes of the configuration. The node buttons allow opening the new output or input panels or the node panels from CANoe. The output and input panels shows message buttons in order to open the Message Panel with all signals of the message and appropriate disturbance functions. The Main Panel establishes also the possibility to change the ignition status in order to start the remaining bus simulation. The columns with the buttons for the Tx/Rx signals and the node panels are optional. Their availability depends on the selected panel settings for the generation process.

Global States

Ignition (KL15)

For all ECUs the status of the KL15 can be used to control the sending of the simulation node. The ILs can be started and stopped depending on the status of the clamp 15. If enabled all nodes start sending and NM nodes set a network request. This switch is the recommended switch to start and stop the remaining bus simulation.

Power (Kl30)

Sets the global state of the clamp 30.

Comfort

NM Panel CANx
This button opens the NM Panel. The button and also the NM Panel itself are not available, if the appropriate setting for the generation process was disabled.

Close all panels

This button closes all panels.

Close Node Panels

This button closes all node panels.

Enable IL of Node

The check boxes in the Enable IL of Node area allows to disable the Interaction Layer of the node. If the Interaction Layer is disabled, the node will not send any application messages.

Tx Signals /Rx Signals

These buttons open the appropriate panels with the buttons for each transmitted or received message of that node.

Node Signal Panels

These buttons open the appropriate node panel of CANoe.

CAN IL Control

Enable IL
This check box enables/disables the ILs simultaneously in all nodes.

Message Panel (with CANoe IL)

The Message Panels display modification controls for each signal of the message and if available the timing modification of the IL.

Cyclic Timing

The cyclic sending of a message (if the message is defined as a cyclic message in the database) can be disabled or enabled. The cycle period can also be modified or reset to the database default. See CAPL functions ILFaultInjectionResetMsgCycleTime.and ILFaultInjectionSetMsgCycleTime.

Event Timing

Send Once

This button triggers the immediate transmission of a message.

Application Signals

The lower part of the Message Panel shows all signals of the message. The physical value of a signal can be set. Their current valid value is displayed. For signals with value tables also their symbolic value can be selected and is displayed.

NM Panel

The NM Panel displays the current simulation state of the Network Management (NM) and interaction layer (IL) per node and allows manipulating the state via check boxes and buttons.

NM Panel CANx (Comfort)

Check Boxes

Network Request (first check box)

This check box allows modifying the network request status of the node’s NM and normally displays the current status.

Application Activity (second check box)

This control displays the current application activity of the node. It also allows modifying the application’s activity state dynamically. A yellow state shows an active application activity. If active, the application activity prevents the NM to release the NM request, e.g. after disabling the ignition status (KL15).

IL Activation (third check box)

This control displays the current activation state of the node’s IL. It also allows modifying the IL state dynamically. A green state shows an active IL. An active IL normally means the node is active.

NM Deactivation (forth check box)

This control allows deactivating the NM simulation of a specific node. A red state shows a deactivated NM. A deactivated NM does not send any NM message.

NM Simulation State (text to the right of the check boxes)

This text control displays the current state of the node’s NM state machine, which is defined by the NM protocol.

Buttons

[NM Request All] / [NM Release All]

These buttons allow setting/resetting the Network Request status of all nodes simultaneously.

[IL Request All] / [IL Release All]
These buttons allow setting/resetting the IL Activation state of all nodes simultaneously.
