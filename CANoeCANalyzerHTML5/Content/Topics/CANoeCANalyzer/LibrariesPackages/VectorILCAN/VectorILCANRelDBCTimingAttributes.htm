Transmission of the message is repeated n times after the first transmission (i.e. it is sent a total of n+1 times).

The repetitions are carried out at a fast cycle time (attribute: GenMsgCycleTimeFast).

Resend N times

If all signal values are set to their inactive value (attribute: GenSigInactiveValue), the message is sent n times with the inactive signal values after the last signal is set.

The transmissions are carried out at a fast cycle time (attribute: GenMsgCycleTimeFast).

Notes

  • If no send type is set for a message (GenMsgSendType == NoMsgSendType), the message send type results solely from the send types of the message signals.
  • If no send type is set for a signal (GenSigSendType == NoSigSendType), this signal does not contribute to the resulting send type of its message.
  • If the send type of all the signals of a message is set to NoSigSendType, the message send type must be set to Cyclic. The message is then sent cyclically at fixed transmission intervals.
  • To reduce the busload you can limit the transmission of messages to a minimum interval for the send types OnEvent and IfActive (attribute: GenMsgDelayTime). After a message has been sent the time span specified in GenMsgDelayTime must elapse before the same message can be sent again. If this time span has not elapsed yet, the next transmission of the message is delayed until the specified minimum interval is reached. The time grid for transmission is not affected by observance of this minimum interval.

Note

In principle the minimum interval is always monitored. This is completely independent of the actual send type. Only purely cyclic messages will not be affected by this monitoring at all.

The CANoe interaction layer has an auto-start function. This means that, as long as you are creating the model and make no changes to it, the IL is started automatically. Several functions enable you to influence the automatic behavior of the CANoe interaction layer. The auto-start function of the CANoe interaction layer can be disabled by calling up the ILControlInit function in the pre-start.
