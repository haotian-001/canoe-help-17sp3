Basics - CAN Interaction Layer

 

Concept

Various timings for a Message can be defined based on a database description. The CANoe interaction layer controls the transmission accordingly.

Data updates are possible at any time. However, the data transfer takes place only at the times allowed by the Interaction Layer State Model.

Timing Types

The transmit method is defined by the message send type and the send types of the signals placed on the message. The message transmit behavior additionally includes the times for cyclic transmissions, transmit delays, signal inactive values, etc. that are used to control the message transmissions.

The following send types are supported by the CANoe interaction layer:

Basic Send Types

Send Type

Description

Cyclic

The message is sent at a fixed cycle time independent of signal values.

OnChange

The message is sent when a changed signal value is written to the message.

OnWrite

The message is sent when the value of a signal is rewritten to the message.

IfActive

The message is sent at a fast cycle time if a signal of the message is set to an active value. As soon as all values are inactive, the message is no longer sent.

Combined Send Types

Send Type

Description

Cyclic and OnChange

The message is sent cyclically. If a modified signal value is written to the message, it is sent additionally, outside of the cycle time. The defined cycle time is not changed by this action.

Cyclic and OnWrite

The message is sent cyclically. If the value of a signal value rewritten to the message, it is sent additionally, outside of the cycle time. The defined cycle time is not changed by this action.

Cyclic and IfActive

The message is sent cyclically. If the value of a signal is set to an active value the message is sent at a faster cycle time until it returns to an inactive value.

For the send types OnChange, OnWrite and IfActive it is possible to define repetition.
