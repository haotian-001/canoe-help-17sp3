Database Settings AUTOSAR 3

 

Note

These settings are only valid for AUTOSAR 3 databases in conjunction with the CANoe interaction layer.

AUTOSAR 4 databases must use AUTOSAR PDU IL.

The transmission behavior of messages and signals sent with the CANoe interaction layer are described in the database. In AUTOSAR databases the send types are defined in a different way to DBC databases. The CANoe interaction layer supports a subset of all possible send types which can be described with the AUTOSAR format:

General

Minimum Delay

Deﬁnes the Minimum Delay Time (MDT) between successive transmissions of this I-PDU in seconds.

Transmission Mode True Timing

Cyclic Timing

Time Offset

This attribute speciﬁes the time until ﬁrst transmission of this I-PDU.

Time Period

Period of the repetition of cyclic transmissions.

Event Controlled Timing

Repetition Period

The repetition Period speciﬁes the time in seconds that elapses before the PDU can be sent the next time (Minimum repeat gap between two pdus). The repetition Period is optional in case that no repetitions are conﬁgured.

Number of Repetititons

Deﬁnes the number of repetitions for the Event transmission mode.

Transmission Mode False Timing

Cyclic Timing

Time Offset

This attribute speciﬁes the time until ﬁrst transmission of this I-PDU.

Time Period

Period of the repetition of cyclic transmissions.

Event Controlled Timing

Repetition Period

The repetition Period speciﬁes the time in seconds that elapses before the PDU can be sent the next time (Minimum repeat gap between two pdus). The repetition Period is optional in case that no repetitions are conﬁgured.

Number of Repetitions

Deﬁnes the number of repetitions for the Event transmission mode.

Data filter which are assigned to signals in the related message switch between True and False timing. At least one filter must be set to apply the True timing.

Data Filter

always

No ﬁltering is performed so that the message always passes.

maskedNewDiffersMaskedOld

Pass messages where the masked value has changed:

(new_value&mask) !=(old_value&mask) new_value: current value of the message

old_value: last value of the message (initialized with the initial value of the message, updated with new_value if the new message value is not ﬁltered out).

maskedNewDiffersX

Pass messages whose masked value is not equal to a speciﬁc value x:

(new_value&mask) != x new_value: current value of the message.

maskedNewEqualsX

Pass messages whose masked value is equal to a speciﬁc value x:

(new_value&mask) == x new_value: current value of the message.

never

The ﬁlter removes all messages.

newIsOutside

Pass a message if its value is outside a predeﬁned boundary:

(min > new_value) OR (new_value > max).

newIsWithin

Pass a message if its value is within a predeﬁned boundary:

min <= new_value <= max.

oneEveryN

Pass a message once every N message occurrences.

Algorithm: occurrence %

period == offset Start: occurrence = 0.

Each time the message is received or transmitted, occurrence is incremented by 1 after ﬁltering.

Length of occurrence is 8 bit (minimum).
