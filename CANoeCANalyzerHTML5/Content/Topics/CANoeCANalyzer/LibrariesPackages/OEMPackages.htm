Overview - Modeling Libraries / Add-ons

 

General

Cross References

The functionality of CANoe can be further expanded by supplemental Modeling Libraries. Since not all expansions are needed for every task, this can reduce the complexity of the main program.

Note

To support different variants without having to change the configuration, the modeling libraries are configured using a module description file (*.vmodule). Whenever possible, a module description file should be used. (For more information on the architecture of the RT kernel, see Notes on 64/32 bits).
  • Assigning of Modeling Libraries
  • Search Sequence for Modeling Libraries
  • Module Description File
  • Model Generation Wizard
  • IL Configuration
  • PDF Manuals for Standard Modeling Libraries

CANoe supports the following types:

Modeling Library

CAPL DLL

This component expands the functionality of exactly one specific simulated node, e.g. for:

  • Interaction Layer (IL)
  • Network Management (NM)
  • Transport Protocols (TP)

Further information:

  • Assigning of Modeling Libraries

These DLLs directly expand the functionality of CAPL and allow complex tasks to be implemented in a programming language other than CAPL, such as C or C++.

To use such a DLL, add it to the CAPL DLL list in the Options dialog (Programming|CAPL DLL).

Further information:

  • Configuration: CAPL DLL
  • Assigning a Windows DLL in CAPL

Which OEM Add-ons Are Installed?

OEM specific modeling add-ons are not part of the CANoe installation. Please contact your Vector key account manager to get the appropriate add-on.

You want to know, which OEM add-ons are installed on your computer? Open the File ribbon tab and click Help. Under Details are listed all installed add-on.
