Installation of the Vector Tool Platform on a Computer

 

Note

The use of an own installation is not recommended, because only a limited support can be provided due to manifold system settings of the computer.

General

CANoe offers the possibility to execute realtime relevant simulation parts on any computer (RT server), independent of the graphical CANoe user interface on the actual user computer (distributed mode).

This page explains how to run any computer on which the Vector Tool Platform (VTP) has not yet been preinstalled as an RT server.

Installation of Vector Tool Platform

In order to be able to operate any computer as an RT server, it is necessary to install the Vector Tool Platform on this computer. The following prerequisites are mandatory and restrictions apply.

Prerequisites

The following technical requirements are necessary to install VTP on a (standard) computer:

  • (Standard) computer with Windows operating system ( ≥ Windows 7).
  • Correctly installed drivers for any bus hardware connected to the (standard) computer.
  • Microsoft .NET Framework (at least version 4.7.1) and Visual C++ redistributable for Visual Studio 2013 and 2017 (available as a download from Microsoft or by installing the VTP Platform Manager on the computer).
  • At least the system partition (C:) and a data partition (D:) exist on the (standard) computer.
  • The user on the (standard) computer has administrator rights.
  • The (standard) computer is connected to the same network as the user computer.

Installation

The following steps are necessary to install on a (standard) computer:

Step by Step Procedure

  1. On the computer to be operated as RT server, execute the installation file Vector VTP Control Server.msi. (It is located in the installation directory of the Vector Platform Manager in the subdirectory Installer Additional Components. Examplt: C:\Program Files (x86)\Vector Platform Manager\Installer Additional Components\Vector VTP Control Server.msi.

    Note

    The following files and directories are created:

    • C:\VtpControlServer\ (corresponds to the subdirectory ..\VtpDrivers\x32\VectorToolPlatform\VtpControlServer\ of the Vector Platform Manager installation)
    • C:\rth\versions.json (File from ..\VtpDrivers\x32\VectorToolPlatform\common\rth\ of the Vector Platform Manager installation)
    • D:\VtpCs\Logging\LoggingConfig.json (newly created default logging configuration)
  2. On the (Standard) computer to be run as RT server start the following applications:
    • C:\VtpControlService\ControlServiceConsole.exe
    • C:\VtpControlService\ControlServer.exe

      • Note

      • ControlServer.exe requires administrator rights (right click in Explorer: Run as administrator) and should not be called with the parameters --firstrun , --install and --uninstall, as this might have influence on a possibly parallel CANoe installation on the (standard) computer.
      • If an automatic start of VTP with the boot of the (standard) computer is desired, the applications can be manually included in the autostart process (under Windows 10 the Windows User Account Controls (UAC) must be deactivated and the Windows Registry Key HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\EnableLUA set to 0x0).
  3. On the user computer (distributed mode), start CANoe and configure the distributed mode with the new RT server (confirm a possible request to install/update CANoe to the RT server). After the update, the (standard) computer can be operated as RT server with the restrictions (see below) and can be reached via CANoe or the Vector Tool Platform.

Restrictions

After installing VTP (see Installation above), the following limitations apply:

  • Driver Update: Any request for installation should be rejected. If an attempt is made to install updates, the ControlServer will prevent this on devices that are not equipped with an operating system image supplied by Vector and an error message will appear.
  • Installation in Autostart: An autostart installation, if desired, must be performed manually. The unsuccessful system update does not do this.
  • Reboot: The triggering of a system reboot is ignored by the ControlServiceConsole.

Deinstallation

The following steps are necessary to uninstall the installed VTP installation (see Installation above):

Step by Step Procedure

  1. Close the following applications on the (standard) computer or the RT server:
  • C:\VtpControlService\ControlServiceConsole.exe
  • C:\VtpControlService\ControlServer.exe

    • Note

    The ControlServerConsole can be terminated by right-clicking on the upper right corner of the window while holding down the <Ctrl> key.
  1. Uninstall the Vector VTP Control Server application from the Control Panel.

    Note

    The following directories and subdirectories are deleted:

    • C:\VtpControlServer\
    • C:\rth\
    • C:\CANrtk\
    • C:\ProgramData\Vector\VectorToolPlatform\
    • D:\RTApplications\
    • D:\RTApplications_temp\
    • D:\VtpCs\
  2. Remove a possibly created automatic start of
  • C:\VtpControlService\ControlServiceConsole.exe
