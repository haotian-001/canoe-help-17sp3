Multi CANoe

This section is only visible if Multi CANoe instances have been created on the Execution Environment tab.

The selected RT kernel for a Multi CANoe cluster is shown. A different instance can be selected via the drop-down list. Default means that no Multi CANoe instance is selected.

RT Server

CANoe offers the possibility to execute the realtime relevant simulation parts on a separate computer (RT server), i.e. separate from the graphical user interface (user computer), refer to distributes mode.

Current RT Server

If an RT server is currently configured, the server and connection type (USB or TCP/IP) are displayed next to Current RT server.

Under Current occupation of selected device as RT server, you can see whether the device selected under Connection|Device Selection is already occupied as RT server, so that the current instance of CANoe cannot connect to the RT kernel on the device. Possible assignment states are here:

  • The device is currently not used as RT server and is therefore available for the current instance of CANoe.
  • The device is used by another instance of CANoe/CANalyzer (on the same or another user computer) as RT server in distributed mode. In addition, the user (user name, computer) that occupies the RT server is specified here, so that this user can be informed if necessary.
  • The device is used in standalone mode. To deactivate the standalone mode, please switch to the CANoe/CANalyzer|Standalone Mode page.

To use the device selected under Connection|Device Selection as RT server, select Configure currently selected device as RT server.

The CANoe Write window shows the output of a connection setup or disconnection, both in case of success and in case of error (see also Troubleshooting). If the software versions of CANoe on the RT server and on the user computer are different, a request appears to update the version on the RT server (see Updating the Execution Environment).

Connection Control

In this section you can configure when CANoe should connect to and disconnect from the RT server.

Activate Immediately connect to the RT server at program start and after configuring an RT server to automatically establish the connection to the RT server already at program start or after configuration of an RT server. If the connection cannot be established immediately (e.g. because the RT server is still occupied), a further connection attempt is made automatically as soon as the RT server becomes available. Please note that you must activate this option if your user license is stored on the RT server or a device connected to it.

Activate Automatically disconnect from the RT server after measurement end and reconnect before measurement start to automatically disconnect from the RT server at the end of the measurement and automatically reestablish the connection before the next measurement start. This option is mainly used to share the RT server with other users as long as you do not need it for a measurement.

Depending on the status of the current connection to the RT server, further actions are available under Manually connect to or disconnect from RT server. These actions are interesting, for example, if you have configured the automatic connection setup at the end of the measurement as described above, but still need a connection to the RT server for a short time outside of the measurement for configuration purposes:

Depending of the current state of the RT server connection, further operations are possible:

  • [Connect]: The connection to the currently configured RT server will be established.
  • [Disconnect]: The connection to the currently configured RT server will be terminated.

Time Settings

Activate Automatic adjustment of the RT system time at the start of measurement to adjust the RT server system time to the system time on the user computer at the start of a measurement.
