Standalone Mode: Configuration Settings

 

Note

Note that the settings you make in this dialog are configuration settings. These are therefore saved in the configuration open in CANoe.

In order for these settings to take effect on the standalone device, you must therefore first download the current configuration to the device â just like for all other configuration settings.

Logging Block

Here you can activate or deactivate an upstream Logging Block for standalone mode and configure the logging properties (e.g. the name of the logging file) themselves ([Configure...] button).

When Use standalone logging has been activated, Trigger Blocks and CAPL Blocks can be inserted optionally.

Trigger Block

With this Trigger Block logging can be started and stopped with CAPL (trigger, startLogging, stopLogging).

When the Trigger Block is configured using the [Configure...] button, you can specify a pre-trigger time (logged events before occurrence of the trigger condition) and a post-trigger time (logged events after completion of the Trigger Block). Furthermore you can use Stop after n Trigger Blocks to specify that the measurement will be stopped after a certain number of trigger starts. In the Buffer size input box, you can limit the number of messages that will be saved during the pre-trigger or post-trigger time.

CAPL Filter

This CAPL block can be used, in particular, for filter purposes and is added upstream of the Logging Block.

It is useful, for example, to specify a CAPL file via the [Configure...] button that is also upstream of the Logging Block in the Measurement Setup. With the exception of functions that have been specifically tailored to the Measurement Setup, all CAPL functions are also supported in standalone mode. It must be noted in particular that â just like when using a CAPL block in the Measurement Setup â messages are only forwarded to the Logging Block if this is explicitly initiated by calling CAPL function output.

Storage Settings for Logging and Test Report Files

You can use Create logging and test report file on external volume to specify that the logging file and test report file are to be saved on an external data medium connected via USB (e.g., USB memory stick) instead of in the internal memory of the standalone device (VN8900).

Notes

Regarding storage location of the logging file the following has to be considered:

  • USB memory devices that require special drivers can not be used on the VN8900.
  • External USB data media must not be removed during the measurement, because this can corrupt the stored data. It is recommended that the data medium be removed only after shutting down the power supply of the VN8900.
  • On the external volume, logging files are written to folder Logging and test report files re written to folder TestReports. Both folders will be created if necessary.
  • If storage on external volume is configured but no external memory device can be found or no write access is granted to the destination folder there the logging file is written to the internal volume.
  • Approximately 700 MB of memory is available for logging files, test report files, and standalone configuration files on the internal data medium (Flash module) of VN8900. However when software is updated, additional memory space is needed temporarily on the Flash module, so that logging or test report files will be deleted if required.
  • Devices of VN8914/VN8911 type can be optionally equipped with an additional SD memory card. In this case, the SD memory card always is used for logging on the internal disk; the available space depends on the size of the installed SD card.

Further notes about internal and external storage for VN8900 you find on the page Storage.

Automatically delete old logging and test report files

When logging or test reports are used in standalone mode, the oldest logging or test report files can be automatically deleted as soon as the amount of memory occupied on the corresponding partition (e.g., internal memory or USB stick) exceeds a defined limit. You can use Automatically delete old logging and test report files to activate this function.

Notes

  • Note that this function operates at the file level. Thus only files can be removed but not older sections within a file.
    For long-duration measurements it is recommended that a field function is selected in the configuration of the logging file for automatic incrementing of the filename when a certain limit is reached (e.g. the File size ... MB field) so that the logging is distributed among multiple files and the oldest files can then be deleted if required.
  • This feature is only available if the standalone measurement is done on a VN8910 or realtime module (VT6000 family). In all other cases this feature will be automatically disabled on measurement start in standalone mode.
  • Start deleting at â¦ % partition usage
    Defines the allocation of the partition in % from which old logging and test report files should be deleted.
    As long as the allocation of the partition is under this limit no logging and test report files will be deleted.
  • Stop deleting at â¦ % partition usage
    Defines up to which allocation of the partition old logging and test report files should be deleted.
    Deleting is stopped as soon as defined allocation (rate) is reached.
    The value must be smaller than the value in the Start deleting at â¦ % partition usage field.
