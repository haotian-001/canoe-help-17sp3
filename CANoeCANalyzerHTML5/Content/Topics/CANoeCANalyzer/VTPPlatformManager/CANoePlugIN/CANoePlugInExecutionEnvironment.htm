Update RT Kernel

Under Current Version of the CANoe RT kernel on the device, the software version that is currently installed on the selected device is displayed.

As of CANoe 15 SP3, the execution environment can be installed as a 32-bit and as a 64-bit application on a VTP Device or packed into an archive file. You can choose between the variants 32-bit only, 64-bit only or 32-bit and 64-bit. Two different archive formats are supported. RTENV files can only contain the CANoe RT kernel and are still offered to maintain compatibility with older CANoe versions. VTPAPP files, on the other hand, can contain any RT applications for the Vector Tool Platform (VTP).

The software version on the device can be updated in different ways:

  • In CANoe:
  • Update the RT kernel to the currently used CANoe version: Click on the [Update] button and select the desired variant.
  • In addition, CANoe checks the software version when a device is selected via Home ribbon tab|Group Measurement|Vector Tool Platform and suggests an update if necessary.
  • A check of the software version with suggested update also takes place when establishing a connection to the RT server in distributed mode in CANoe.
  • When using Platform Manager or CANoe:
  • Update based on a packed installation: Click on the [Update] button and select the desired archive file. In this case, all variants of the execution environment contained in the archive file are installed.
  • When using Standalone Manager:
  • In the case of devices that are not accessible from CANoe and on which a CANoe version < 11.0 SP3 is running, a CANoe software update must be performed once via the Standalone Manager using an RTENV file. Please refer to the corresponding notes on the help pages for the Standalone Manager. Subsequent software updates can then be performed using one of the above methods.

Notes

When you start the software update, there is generally no connection between the CANoe installation on the local computer and the RT server environment (since updates are usually carried out because of a difference in versions). If there is still an active connection (e.g. when reinstalling the same version), it is temporarily terminated during the update.

If the device is running in standalone mode, the software update requires to stop measurement and deactivate standalone mode first. An appropriate message box appears on start of software update in this case. After the software update you can activate the standalone mode again (page CANoe/CANalyzer|Standalone Mode). Before starting the software update, make sure that the Vector Tool Platform (VTP) is installed on the device in a certain minimum version. If this is not the case, you will be asked to perform a VTP installation or update first. If this is aborted, the further update of CANoe cannot take place on the device. Please note that for the VTP update it is assumed that the latest version of the Vector Platform Manager is installed on your computer (is installed with CANoe by default).

When the software update starts, the first thing that happens is that relevant data is requested from the RT server. After this, all the files that are needed to simulate the realtime area and to manage the environment (in particular, the connection to CANoe) are copied to the RT server and installed there in the next step.

Progress is shown in a separate window. You can cancel the update as long as the installation of files on the device has not begun. If you click the [Cancel] button when the files are already being installed, the installation will simply continue to run on the device.

Notice!

Do not switch off the device during the installation!

Doing so could damage the system files on the device.

Pack RT Kernel

You can assemble an execution environment for VN8900 for each version of CANoe installed on the local computer and pack it into an archive file. This archive file can be used to adapt the software version on a device to the version of an existing standalone configuration at a later time or at another location.

Two different archive formats are supported. RTENV files are still offered to maintain compatibility with older CANoe versions. VTPAPP files are independent of CANoe and will replace RTENV files in the future.

A list shows all versions of CANoe/CANalyzer currently installed on the local computer. To pack an installation, select the appropriate entry from the list and click [Pack]. Then select a format (RTENV or VTPAPP) and a directory in which the archive file is to be stored.

Network Ports of RT Kernel

CANoe's basic communication with the RT kernel on an RT server connected via IP takes place via TCP. The RT server can be reached under TCP port number 2809 by default. If this port number cannot be used (e.g. due to a conflict with another application or IT specifications), you can assign a different value under Basic Communication (TCP). Click Apply to save the port number on the RT server.

Hardware Synchronization

The hardware synchronization on the selected device can be activated or deactivated here.

Hardware synchronization should be activated if multiple network interfaces are used (e.g., a plugin module VN8970 and a VN1600 interface connected to the VN8900 via USB).

Multi CANoe

Notes

  • This area of the configuration of the execution environment is only intended for the use with Multi CANoe. If you run the VTP device without Multi CANoe, this area is not required.

If more than one Multi CANoe instance will run in distributed mode and they have their RT kernel on this device, all RT kernel instances must be created in this section.

Use cases

  • Multi CANoe is not in use
    No manual configuration is required in this area. Leave this part of the configuration empty.

  • One Multi CANoe instance only
    No manual configuration is required in this area. The default RT kernel is always available and will be used.

  • Two or more Multi CANoe instances on one device
    To distinguish multiple RT Kernel instances for Multi CANoe on one device, all instances in this section must be configured with an arbitrary name. Click [Create...] and enter a name for the new RT Kernel instance. Repeat this for all instances that run their RT kernel on the connected device.

    Example

    In this example, the Multi CANoe instance is linked to its Multi CANoe RT kernel instance using two command line parameters to start CANoe.

    The Multi CANoe cluster consists of two Multi CANoe instances. Both work in distributed mode and their RT kernels are both located on the same RT server (e.g. RT Rack) with the IP address 192.168.0.10. This requires that the 2 CANoe instances are started with the following command line parameters:

    start "" "%CANoe_InstallDir%CANoe64.exe" /f "%~dp0MultiCANoe_Config_01.cfg" /m MultiCANoe01 /c MultiRTK01@192.168.0.10
    start "" "%CANoe_InstallDir%CANoe64.exe" /f "%~dp0MultiCANoe_Config_02.cfg" /m MultiCANoe02 /c MultiRTK02@192.168.0.10

    /m specifies the name of the Multi CANoe instance which is also used as application name in the hardware setup.
    /c specifies the name of the Multi CANoe RT kernel instance, as defined in this dialog.

    Hint: After starting CANoe, make sure that the RT server is connected to your computer in distributed mode and that the Multi CANoe option is selected. Both may be disabled at startup if the connection to the RT server is not available at that time.
