CANoe | Concept

 

Introduction

CANoeis a universal development, test and analysis environment, which is made available to all project participants over the entire development process. The system producer is supported in functional distribution, functional checking and integration of the overall system. The supplier obtains an ideal test environment by simulation of the remaining bus and environment.

Architecture

Realtime Area

Time-critical sequences are processed in the real-time area. The incoming bus data are read in via a network interface, provided with a time stamp there, and then processed promptly. The sending of the data to the bus also takes place in the real-time area.

All blocks (e.g., CAPL programs, test modules, test units, generators) that you insert in the Simulation Setup or Test Setup are processed in the real-time area. For this reason, you should only insert blocks there that use little computing time and thus do not influence the reaction of the system.

To prevent the real-time area from being influenced by time-consuming analysis, the incoming data are forwarded from the real-time area to the analysis area via a buffer.

Analysis Area

Non-time-critical sequences are processed in the analysis area. The data from the buffer are forwarded to the individual analysis windows via the Measurement Setup. The data are thereby processed from left to right and from top to bottom. In the Measurement Setup you can insert various function blocks, e.g., for data reduction, in addition to the analysis windows.
