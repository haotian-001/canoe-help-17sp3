CANstress

CAN networks are highly tolerant in respect of disturbances of the bus communication and failures. With CANstressD (Digital) or CANstressDR (Digital and Resistor Network) you disturb the CAN bus, its physical properties and the logical levels (recessive and/or dominant) in a targeted, reproducible way.

CarMaker Interface

Open Setup Folder

Via the CarMaker interface it is possible to start single tests or test series in CarMaker and monitor selected simulation data.The control and stimulation of the CarMaker test runs is done via the APO interface provided by CarMaker. This can be called from CANoe using special CAPL functions.

This component is included in the setup of CANoe. In the setup folder Installer Additional Components you find the corresponding setup files.

FDX Protocol

CANoe FDX (Fast Data eXchange) is a protocol that can be used for easy, fast, and real-time data exchange between CANoe and other systems via an Ethernet connection. The protocol gives other systems read/write access to system variables, environment variables, and bus signals of CANoe. In addition, it is possible to send control commands (e.g., for starting and stopping the measurement) to CANoe or to receive status information via FDX.

Functional Mock-up Interface (FMI)

The Functional Mock-up Interface (FMI) defines a standardized interface for the coupling together of complex dynamic simulation models from various modeling tools (e.g., Simulink). One of its purposes is to enable and simplify validation and testing of the interaction of components within a simulated environment.

GPIB

GPIB is a protocol for networking external instruments. These instruments include measuring devices, power supplies and function generators. This standard was originally developed by Hewlett Packard under the name HP-IB. Later HP-IB was renamed GPIB (General Purpose Interface Bus) by the IEEE Committee as. Other equivalent names are IEEE 488 and IEC 625..

The IOpiggy is a further piggy-back module for the VN8900 and VN7570, complementing the CANpiggy and LINpiggy. This piggy makes measuring and generating I/O signals simple and easy.

LabVIEW Integration

By using the ActiveX library that is included in LabVIEW, CANoe can be activated and addressed via the COM Interface. Therefore the COM Interface ensures that all objects (and their methods, properties and events) are available in LabVIEW.

MATLAB Integration

Open Setup Folder

The purpose of this interface is to extend the node modeling capability of CANoe by adding the strength of the MATLAB/Simulink environment. Now models can be used to describe the functionality of a network node in CANoe. This allows CANoe to simulate nodes which are impossible or difficult to model using CAPL alone.

This component is included in the setup of CANoe. In the setup folder Installer Additional Components you find the corresponding setup files.

PortLink

PortLink is used to link CANoe environment variables to analog and digital I/O ports on I/O cards. The value of an environment variable is either read to a port or output by a port.

RS232

In CANoe you can control the RS232 interface via CAPL functions. If your computer does not have a built-in RS232 interface, you can use a USB to RS232 adapter. Be sure to install the corresponding adapter driver when doing so. You can also use other network interfaces with the RS232 API. The requirement for this is that Windows must recognize these serial network interfaces as serial COM ports. One example would be IrDA, as most are represented as a virtual COM port in Windows.

Scope

Option Scope is an integrated oscilloscope solution for CANoe, based on a USB scope hardware. The PicoScope 5444B-034 hardware supports up to 2 CAN, 4 LIN or 2 FlexRay channels. The scope is triggered using the sync line of Vector's interface hardware. With bus-specific trigger conditions the cause of protocol errors can be located much faster than with a traditional oscilloscope.

The StandaloneUtilities contain the additional tool Standalone Manager supplied with CANoe. You can install the Standalone Manager (independently, i.e. without CANoe) on any other computer. You can use it to control the standalone mode of VN8900 devices connected to this computer.

VN Family

The VN1600 product line succeeds the market-proven USB CANcaseXL interface. While the VN1610/VN1611/VN7610 with 2 channels focuses on minimum size and weight, the VN1630/VN1640 with 4 channels is designed for flexibility and I/O support.
The VN8900 system is a systematically advanced development of the proven products of the XL-Interface family and can be used for a wide variety of applications. A particular focus here is on parallel access to multiple bus channels and I/Os with exacting realtime and latency requirements.

VN7570

The VN7570/VN7572 is a high performance PCI Express based network interface. It offers up to eight channels of which four may be assembled with CAN, LIN and up to one (VN7570) or two (VN7572) FlexRay transceivers. This makes the VN7570/VN7572 highly flexible and applicable for a wide variety of applications.

✔

VN8900

The VN8900 combines the ease of use (plug & play, USB – as e.g. VN1630) with real-time capability. For that purpose in CANoe the real-time-relevant parts of the simulation- and test environment are executed on the VN8900 device (see Realtime Execution (RT Setup)). In CANalyzer the send branch and its blocks are executed on the VN8900 device. Thereby you achieve improvements with respect to latency times and precision of timers.

XILAPI

Start Setup

The ASAM XIL API is a standard that specifies software interfaces for access to test systems. This API allows different test systems to be programmed in a standardized manner. Furthermore, investments in test scripts remain protected even when a change is made from one test system to another.

This component is included in the setup of CANoe. In the setup folder Installer Additional Components you find the corresponding setup files.
