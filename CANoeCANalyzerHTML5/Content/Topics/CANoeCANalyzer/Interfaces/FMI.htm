Functional Mock-up Interface (FMI)

Ribbon: Environment Tab|Tool Couplings|Functional Mock-up Interface

The Functional Mock-up Interface (FMI) defines a standardized interface for the coupling together of complex dynamic simulation models from various modeling tools (e.g., Simulink). One of its purposes is to enable and simplify validation and testing of the interaction of components within a simulated environment.

Data exchange via the FMI takes place using so-called Functional Mock-up Units (FMUs), which must be available in the form of a ZIP file (*.fmu extension). An FMU file is essentially an XML file with the variable definitions and a mathematical description of the model. This description can exist either in the form of C code or directly as executable binary files for the respective supported platforms.

An application can act as an FMU-Provider or as an FMU-Consumer:

  • FMU-Provider: provides an FMU
  • FMU-Consumer: incorporates other models

There are two FMU types:

  • Co-Simulation
    The FMU file contains all components for autonomously performing the calculation steps of the model as a function of the simulation time. This also includes the solver needed for solving a system of differential equations, if necessary.
  • Model Exchange
    An FMU of this type only provides the equations of the model via a C interface. If the FMU describes a system of differential equations, the FMU consumer must have its own solver to solve it.

As a consumer CANoe currently supports only Co-Simulation FMU files, depending on the RT kernel variant, in Win32 or Win64 binary format and in FMI versions 1.0 or 2.0.

Model variables from a utilized FMU file are mapped onto system variables under CANoe. This enables a flexible data exchange within the CANoe simulation context.

At the start of a measurement in CANoe, the simulation of the integrated modules is initiated. A simulation step is performed at defined time intervals, thereby triggering the calculation of the model variables. Input parameters of the model can be used to import values from CANoe (or other FMU files) into the simulation model for the calculation.

You use the Functional Mock-up Units dialog to add and configure an FMU file.
