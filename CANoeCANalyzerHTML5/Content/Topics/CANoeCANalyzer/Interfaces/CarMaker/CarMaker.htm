CarMaker Interface

Ribbon: Environment Tab|Tool Couplings|CarMaker Interface

Note

To use the CarMaker Interface in CANoe, the CarMaker component must be installed.

This component is included in the setup of CANoe. In the setup folder Installer Additional Components you find the corresponding setup files.

Open Setup Folder

The CarMaker interface consists of a configuration dialog and a modeling library (DLL file).

The DLL file must be configured as a component at a simulation node and provides special CAPL Functions for communication with CarMaker.

Using the configuration dialog you can create different groups of CarMaker Quantities. You can subscribe to these groups for monitoring using their names. During the measurement, you can switch between the configured groups, whereby only the last subscribed group is active.

You can also configure the cycle time for the periodic update of the internal state of the CarMaker interface in this dialog.

Configuration

How to set up the data exchange between CANoe and CarMaker:

1. Select a CAPL network node in the Simulation Setup for the communication with the CarMaker interface. Add the CarMaker component (CarMakerControl.vmodule) from the Exec64 directory to this node and create a CAPL file.

2. Open the CarMaker configuration dialog and enter the host name or IP address of the CarMaker host computer and the name of the user running CarMaker.

Note

CarMaker must already be started and should be in idle state.

  • If only one CarMaker instance is running on the host computer, you can also leave the User Name input box empty.
  • If you run CarMaker on your own computer, you can use the values localhost and localuser.
  • If the communication with CarMaker can be established successfully, the tree of available quantities is automatically filled with the elements provided by CarMaker.

3. Create groups of CarMaker quantities that you want to monitor by dragging elements from the tree of available quantities into the desired group.

Note

Since the values of all elements in the active group are updated cyclically, very large groups should be avoided if possible and, if necessary, divided into smaller groups.

4. Specify a cycle time with which the internal state of the CarMaker interface should be updated.

Note

A short cycle time improves the response time of the communication, but can lead to an increased system load.

5. Accept the settings by leaving the dialog via the OK button. After you have closed the dialog, the corresponding CarMaker system variables are automatically created. These can then be read during the measurement.

Note

The CarMaker system variables cannot be written. To write individual CarMaker quanta, use the corresponding CAPL function.

6. In the CAPL file, you can use the various CarMaker functions to control the connection and communication.
