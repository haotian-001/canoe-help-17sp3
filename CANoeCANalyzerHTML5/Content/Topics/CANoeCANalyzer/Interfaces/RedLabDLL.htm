RedLab DLL

 

Supported Devices and Functions

Most devices of the Meilhaus-RedLab device family are supported by this DLL, including the following:

  • RedLab 1008
  • RedLab 1208 LS and FS
  • RedLab 1208 HS – 4AO
  • RedLab 1608 FS
  • RedLab 1024 LS

Access to the analog and digital inputs/outputs as well as the counters of these devices is provided in CANoe.

Note

In order to make use of the RedLab devices, you need to have installed the Instacal software delivered with the devices.

You can also download the software from https://www.meilhaus.de/en/.

Configuration

In order to access the RedLab devices in CANoe you need to first connect them to the computer via USB. You can then configure the devices using the Instacal software.

Automatic Configuration

  1. Now open CANoe. Make sure that there is no pl_redlab.ini file in the application data folder. To open this folder you can use the [Open] button on the Program Settings|Locations page in the CANoe Options dialog.
    This step is necessary as CANoe will automatically generate a configuration file only if no such file is already present.
  2. Once you have ensured that there is no pl_redlab.ini file in the application data folder, open the I/O hardware dialog by choosing the Hardware ribbon tab|Other.
  3. Click on the [I/O Device Drivers…] button and activate the DLL pl_redlab.dll.
  4. A configuration file pl_redlab.ini is then created, with settings adapted to the connected RedLab devices. Additionally, the corresponding system variables are created in CANoe.

Manual Configuration

You can modify the configuration by editing the pl_redlab.INI file, which is made up of the following elements:

Keyword

Meaning

Example

[Boardx]

[Boardx] Determines which RedLab device the following parameters refer to.
The related index refers to the board ID assigned in Instacal.

[Board0]
The following parameters refer to the first device configured in Instcal.

AI_NAME_x and
AO_NAME_x

Determines the name of the analog input/output.
The channel index is specified by the x parameter. If there is no name set for a channel (i.e. the line is missing in the configuration file) the corresponding channel is not used in CANoe.

AI_NAME_3=AnalogInput3
This creates a system variable in CANoe named AnalogInput3 containing the data measured at analog input 3 of the RedLab device.

AI_RANGE_x and
AO_RANGE_x

Determines the scaling for the measurement value of an analog input/output.
The possible scalings are described in the Universal Library documentation (also available on the Meilhaus website).

AO_RANGE_0=101
The analog output with index 0 is set to scaling 101 (=UNI5VOLTS); this means that the output values are scaled to a range of 0 V…5 V.

DIO_NAME_x_y

Determines the name of a digital input/output.
The x parameter stands for the port at which the channel is located, and the y parameter for the channel index.

DIO_NAME_10_5=DigitalIn
The name of channel 0 at port 10 (=FIRSTPORTA) of the RedLab device is set toDigitalIn and a corresponding system variable is created in CANoe.

DIO_DIRECTION_x

Specifies the direction of all pins at digital port x.
A 1 stands for ouputs and a 2 for inputs.

DIO_DIRECTION_10=2
All digital channels at port 10 (=FIRSTPORTA) of the RedLab device are configured as outputs.

COUNTER_NAME_x

Specifies the name of a counter.
The counter index is specified by the x parameter.

COUNTER_NAME_0=Counter0
The name of the first counter is set to Counter0 and a corresponding system variable is created CANoe.

TRIGGER_MODE

Specifies which trigger mode is to be used.
A value of 0 stands for cyclical capture (i.e. no triggering), 1 for digital triggering and 2 for analog triggering.

TRIGGER_MODE=2
An analog trigger is used, i.e. measurement values are only captured when the trigger condition is met.

TRIGGER_TYPE

Specifies the trigger type.
With value 0 the trigger reacts to rising edges, with 1 to falling and with 2 to both edge types.

TRIGGER_MODE=2
The trigger is set off both by a rising and by a falling signal edge.

TRIGGER_CYCLE_TIME

Cycle time in ms used for the capture of measurement values when cyclical trigger mode is selected.

TRIGGER_CYCLE_TIME=100
Measurement values are captured every 100 ms.

TRIGGER_PIN

Index of the pins to be used for triggering.

TRIGGER_PIN=0
If an analog trigger is selected, the first analog input is used as the trigger pin.

TRIGGER_PORT

Port of the pins to be used for triggering.
This value is only required when a digital trigger is used.

TRIGGER_PORT=10
The digital pin at port 10 (=FIRSTPORTA) is used for triggering.

TRIGGER_VOLTAGE

Voltage threshold at which the analog trigger is set off.

TRIGGER_VOLTAGE=2.5
The analog trigger is set off when the measured voltage goes above or below a value of 2.5 V (depending on the trigger type configuration).

Note

The documentation of the Universal Library is part of the RedLab software package. It can be downloaded from the Meilhaus website http://www.meilhaus.de. This documentation includes scaling details for the various RedLab devices.

Triggers

The capture of measurement values can be cyclical or trigger-controlled. Cyclical capture is used by default. To activate a trigger, you need to insert additional lines in the configuration file pl_redlab.INI. The trigger configured here is independent of the settings in Instacal, which allows a much more flexible trigger configuration.

  • Cyclical capture (no trigger)
    This mode is activated by default. It corresponds to the configuration TRIGGER_MODE=0. In this mode, the measurement values are transmitted to CANoe cyclically. The cycle time can be specified using the TRIGGER_CYCLE_TIME keyword. Please note, however, that if the specified cycle time is too short it may not be possible to maintain it consistently due to hardware constraints.
  • Digital trigger
    This mode is activated with the configuration line TRIGGER_MODE=1. In addition, you need to specify the trigger type to be used by means of the keyword TRIGGER_TYPE.
  • TRIGGER_TYPE=0: The trigger is set off by a rising signal edge.
  • TRIGGER_TYPE=1: The trigger is set off by a falling signal edge.
  • TRIGGER_TYPE=2: The trigger is set off by a rising and by a falling signal edge.

The pin to be used for signal monitoring also has to be specified. This is done using the keywords TRIGGER_PIN and TRIGGER_PORT.

  • Analog trigger
    This mode is activated with the configuration line TRIGGER_MODE=2. In addition, you need to specify the trigger type to be used by means of the keyword TRIGGER_TYPE.
  • TRIGGER_TYPE=0: The trigger is set off when the signal value rises above the threshold value.
  • TRIGGER_TYPE=1: The trigger is set off when the signal value falls below the threshold value.
  • TRIGGER_TYPE=2: The trigger is set off when the signal value goes above or below the threshold value.

The threshold value is specified using the keyword TRIGGER_VOLTAGE. The unit of measure for this value is Volt. With TRIGGER_PIN you specify which analog input is to serve as the trigger.

Counter

For every counter of the connected RedLab devices two system variables are created. While the first one holds the current value of the counter the second one can be used to reset the counter. It has the same name as the first variable but is enhanced with the suffix "_reset". To reset the counter it suffices to change the value of this variable.

Performance

The rate at which measurement values of the RedLab device can be captured and transmitted to CANoe depends heavily on the number of channels used. The fewer channels you define in the pl_redlab.INI configuration file, the faster the transmission speed for the defined channels will be. It is advisable, therefore, to specify only the channels that you actually require in the configuration file. You can temporarily deactivate channels that you don't need by commenting out the corresponding settings with a semicolon.

The counters of the RedLab measurement devices are addressed using their corresponding index (beginning with 0). The RedLab 1008 only has one counter, which is address via index 0.

Sample Configuration

This sample shows a simple configuration of a RedLab 1008. You need to have set up the device in Instacal beforehand and assigned ID 0 to it.

Two analog outputs with a scaling of 0 V...5 V are created. These are linked with system variables AnalogOutput1 and AnalogOutput2 in CANoe. Two analog inputs with a scaling of ±10 V and system variables AnalogInput1 and AnalogInput2 are also created. In addition, two digital inputs and two digital outputs are configured at ports AUXPORT and FIRSTPORTA. The event counter of the RedLab 1008 is also part of the configuration. It is represended in CANoe by system variable Counter.

[Board0]

; Names and ranges of the analog inputs. See Universal Library for possible range values.
AI_NAME_0=AnalogInput1
AI_RANGE_0=1
AI_NAME_1=AnalogInput2
AI_RANGE_1=1

; Names and ranges of the analog outputs. See Universal Library for possible range values.
AO_NAME_0=AnalogOutput1
AO_RANGE_0=101
AO_NAME_1=AnalogOutput2
AO_RANGE_1=101

; Names of the digital channels. Note: directions of the ports are specified later.
DIO_NAME_1_0=DigitalOutput1
DIO_NAME_1_1=DigitalOutput2
DIO_NAME_10_0=DigitalInput1
DIO_NAME_10_1=DigitalInput2

; Directions of the digital I/O ports. Use 1 for output, 2 for input.
DIO_DIRECTION_1=1
DIO_DIRECTION_10=2

; Names of the counters.
COUNTER_NAME_0=Counter

; Trigger configuration.
TRIGGER_MODE=0
TRIGGER_CYCLE_TIME=100
