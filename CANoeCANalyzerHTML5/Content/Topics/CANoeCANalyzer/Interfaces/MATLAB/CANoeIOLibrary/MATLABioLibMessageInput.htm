Message Input

 

Note

The access to signals is provided through the CANoe Interaction Layer.
Please refer to Setup MATLAB CANoe IL for information about preliminary steps.
For further information see CANoe Interaction Layer.

  

Description

The Message Input block serves as input block for all signals of a database message. The values of the signals are output by the block either as a vector or as a Simulink signal bus.

  

Parameters
  • Sample time
    These settings according to the Simulink Semantics:
  • 0: continuous sample time
  • -1: inherited sample time

Continuous sample time is set by default. If the Message Input block is used e.g. in a triggered subsystem, the Sample time must be set to inherited, since in triggered subsystems only constant blocks or blocks with inherited sample time are allowed.

  • Signal values
    Specifies whether the signal exists in the Simulink model as raw values or physical values.
  • Data type
    Data type of the Output port of the block. The only permissible data type here is inherited, i.e., the data type is derived from the blocks connected to this block.
  • Output type
    This parameter defines whether the values of the signals are output as a vector or as a Simulink signal bus.
  • Select message
    Specify message from database.
