CANoe I/O Library

 

The CANoe I/O library is a collection of S-Functions blocks.

The library provides the following features:

  • Data I/O
  • Trigger sources
  • Event sinks
  • Model execution control
  • CANoe simulation control
  • Calibration using the XCP Protocol
  • Convenience functions for building models

I/O Library

Description

Signal Input

The Signal Input block is the basic input block for a database signal.

Signal Output

The Signal Output block is the basic output block for a database signal.

Signal Block

This block allows the user to collect a number of Signal Input and Signal Output blocks into one subsystem and configure them by one step.

Bus Block

This block allows the user to connect to a Simulink signal bus. This eases the handling of many input and/or output signal.

Message Input

The Message Input block serves as input block for all signals of a database message. The values of the signals are output by the block either as a vector or as a Simulink signal bus.

Message Output

The Message Output block serves as output block for all signals of a database message. It contains the values of signals either as a vector or as a Simulink signal bus.

Simulation Step

The Simulation Step block is used for synchronization and data exchange between CANoe and Simulink.

Analysis Step

The Analysis Step block is used for synchronization and data exchange between CANoe and Simulink.

CAPL Call

The CAPL Call block can be used to trigger a Simulink subsystem from CAPL.

CAPL Handler

The CAPL Handler block can be used to handle a Simulink event in CAPL.

Data Environment Input

The Data Environment Input block is used for an environment variable of type data.

Environment Step

The Environment Step block is used to designate a Simulink model that is executed only when one of its CANoe environment inputs changes.

System Variable Input

The System Variable Input block is the basic input block for CANoe system variables.

System Variable Output

The System Variable Output block is the basic output block for a CANoe system variable.

System Variable Struct Input

The System Variable Struct Input block is used for input the system variables of type Struct.

System Variable Struct Output

The System Variable Struct Output block is used for output the system variables of type Struct.

XCP Calibration

The XCP Calibration block enables calibration of the model parameters using XCP on CAN or Ethernet.

CANoelib Parameters

In the CANoelib Parameters block, settings can be made that apply to all blocks. An example of these are default values for block parameters.

Connect Open Ports

The Connect Open Ports block is a block with whose help all open ports of the model can be connected to a System Variable Input block or a System Variable Output block.

SOMEIP Signal Input

The SOMEIP Signal Input block is the basic input block for a SOME/IP signal.

SOMEIP Signal Output

The SOMEIP Signal Output block is the basic output block for a SOME/IP signal.

SOMEIP Signal Block

The SOMEIP Signal Block is the input and output block for multiple SOME/IP signals.
