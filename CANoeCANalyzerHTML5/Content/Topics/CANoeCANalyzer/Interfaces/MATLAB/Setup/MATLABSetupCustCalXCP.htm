Customizing for Calibration using XCP

 

In this chapter all steps are described which are necessary to enable XCP calibration for a compiled model. It is assumed that the model uses the CANoe target cn.TLC.

Note

Either the calibration or the parameterization feature can be used but not both at the same time.

Simulink Settings

The CANoe MATLAB Integration Package requires the following settings of the Simulink model. Open the menu path Simulation | Configuration Parameters to open the settings dialog.

Optimization

The Inline Parameters option on the Optimization page must be enabled. It is also recommended to disable the Block Reduction setting.

Simulink Block Settings

All parameters must be defined as ASAP2.Parameter in the MATLAB Workspace to appear in the generated A2L file. If the model contains the XCP Calibration block all Workspace parameter referenced by the model will be converted to ASAP2.Parameter objects automatically during the code generation process.

Code Generation

During the code generation process a calibration database file (A2L) is generated. It can be found in the code generation subfolder next to the MATLAB model:

MODEL_NAME_cn_rtw\MODEL_NAME.A2L

A calibration tool like CANape can use the Linker Address Map file to update the addresses in the A2L file after the model was built. The Map file is typically located in the Release folder:

MODEL_NAME_cn_rtw\Release\MODEL_NAME.MAP

XCP Time Stamps

If the Transport Layer XCP on Ethernet is selected for calibration in the XCP Calibration block, the individual XCP frames contain a time stamp. The resolution of the time stamp is 1 ms.

If the time stamp for the model is to be deactivated, the MIP_CONFIG_XCP_TIMESTAMPS_DISABLE parameter can be set to on in the MATLAB Command Window with the help of the following command:

set_param('model','MIP_CONFIG_XCP_TIMESTAMPS_DISABLE', 'on')
