HIL Mode - Simulink Configuration

 

This page describes the steps which are necessary to run a MATLAB model together with a CANoe configuration. It is recommendable to start CANoe and open the desired configuration at first. If a new configuration should be created consider using the Model Generation Wizard (see Configuration Setup from Scratch). The reason is that CAN signals or system variables can be selected easily in the Simulink model if a CANdb is present.

It may be helpful to copy the Simulink model file (MDL, SLX) into the same folder as the CANoe configuration (CFG file). If you want to use the generated DLL/SO, create a subfolder named Exec32 for example. The Simulink Coder can be configured to copy the DLL/SO into this folder.

Simulink Settings

The CANoe MATLAB Integration Package needs some settings of the Simulink model which are described here. Open the menu path Simulation|Configuration Parameters to open the settings dialog.

Note: Improvement performance when exchanging many signals

If a large number of signals are to be exchanged between CANoe and the Simulink model, not for every signal should be used a Signal Input block or Signal Output block. Instead, the signals should be collected in one Signal block or a Bus block. This improves the performance of the data exchange between CANoe and the Simulink model.

Simulink Coder Settings

The Simulink Coder must be configured with the correct target for CANoe. Otherwise the CANoeMATLAB Integration Package blocks can’t be used. There are different targets which allow to work with various compilers. For the configuration of these compilers, see Compiler Configuration.

There are three different targets, for the three supported compilers, available:

  • cn.tlc (Microsoft Visual C++ Compiler)
  • Standard Realtime: Build for Windows 32-Bit/64-Bit with the Visual Studio Compiler.
  • Extended Real Time: Build for Windows 32-Bit with the Visual Studio Compiler.
  • Only supports Microsoft Visual Studio 2013 Professional
  • Execution in CANoe is only supported up to and including CANoe version 16 SP5.
  • The target folder for the code generation is MODEL_cn_rtw.
  • cn_mingw.tlc (Mingw-w64 Compiler)
  • Standard Realtime: Build for Windows 32-Bit/64-Bit with the Mingw-w64 Compiler.
  • The target folder for the code generation is MODEL_cn_mingw_rtw.
  • cn_linux.tlc (Clang Cross-Compile Toolchain)
  • Standard Realtime: Build for Linux 64-Bit with the Clang Cross-Compile Toolchain.
  • Extended Real Time: Build for Linux 64-Bit with the Clang Cross-Compile Toolchain.
  • Execution in CANoe is only supported starting with CANoe version CANoe 17.
  • The target folder for the code generation is MODEL_cn_linux_rtw.
  • For limitations with the Linux RT, see Compiler Configuration.

CN Code Generation Options

The CN target has some specific options for the code generation process. First of all the Output Directory can be set. If nothing is specified the DLL/SO will be generated in the .\MODEL_cn*_rtw\ folder (with subfolders for the respective architecture) relative to the folder containing the Simulink model. You can specify a subfolder of your configuration like ./Exec32 for example to generate the DLL/SO in the .\MODEL_cn_rtw\Exec32 folder.

  • Export CAPL Functions:
    With this option activated three special CAPL functions are exported by the generated DLL. Refer to MATLAB CAPL Function Overview for more details.
  • Perform Simulation Step at t(0):
    If the model does not contain blocks with states (e.g. the model contains only algebraic blocks) this option can be selected to get a value for the simulation step at t=0.
  • Check CANoe block configuration at compilation:
    Uses the Model Advisor to check settings of all CANoe blocks (see Test of CANoe Blocks).
  • Generate CANoe Model Viewer description file: Generates information required for viewing the MATLAB model within CANoe. This feature is only supported for MATLAB R14 and newer.
  • CreateSystem Variable blocks for unconnected ports:
    Connects each unconnected port on the top level of the model with a system variable input or output block (depending on whether it is an inport or an outport). For each block, a new system variable <modelName>::<blockName>_<portNumber> is created.
  • Create System Variables for performance measurement
    Creates system variables for measurement of the performance of the Simulink model. The created system variables are created in namespace
    <modelName>::Performance and are called minSampleTime, maxSampleTime, avgSampleTime and StepTime.
    minSampleTime, maxSampleTime and avgSampleTime specify the minimum, maximum, and average execution times, respectively, of a simulation step of the Simulink model, in microseconds. StepTime specifies the total time from the beginning of a simulation step to the beginning of the next simulation step, in microseconds.
  • Suppress creation of System Variables
    Specifies whether system variables are to be created by the DLL/SO if these are not defined in CANoe. If this option is not selected, system variables that are used by the model but are not defined in CANoe are created by the DLL/SO. If the option is selected, these system variables are not created. This option relates only to system variables configured in system variable blocks. System variables that are defined by parameter assignment of Simulink blocks or the Simulink signal analysis, or that are defined implicitly in the model for performance measurement are always created.
  • Target 64 bit CANoe
    Specifies whether the DLL is to be generated for the 32-bit or 64-bit version of CANoe. The DLL cannot be loaded by the other CANoe version in each case.
  • Debug build
    Specifies whether the DLL/SO file is to be built as a debug version. If this check box is activated, debug information is also generated during the build process.
  • Build 64-bit Realtime DLL
    Builds a DLL/SO for the 64-bit runtime kernel. This option cannot be selected with the 32-bit options below at the same time.
  • Build Standard Realtime DLL
    Builds a standard CANoe DLL/SO. This option cannot be selected with the 64-bit option above at the same time.
  • Build Extended Real-Time DLL
    Builds a DLL/SO for Extended Real Time (ERT). An ERT compatible device must be used for this DLL/SO to be used by CANoe. Find the required Compilers and CANoe versions for using Extended Real Time above.

Note

Please note that currently only signal blocks and non-struct-system-variable blocks are supported by ERT.

This option is incompatible with the following functions

(see check boxes in the screenshot below):

  • Debug build
  • Exclude specific Blocks from Parameterization
  • Build Standard Realtime DLL
  • Build 64 Bit Realtime DLL
  • Generate vModule file
  • Enable Simulink Signal Analysis:
    Enables analysis of all named Simulink signals. With CANoe 7.0 or newer system variables are created for each signal below the namespace MODEL::Signals. This option requires the activation of the C-API for signals. Signals should be considered as Test Points, otherwise Simulink might remove them when optimizing the model. Please refer to the Simulink help on Test Points.
  • Enable Parameterization from CANoe:
    If this option is activated, values of model parameters will be accessible from CANoe. This option requires activation of the C-API for parameters.
  • Parameterization and Signal Analysis also affect referenced models:
    If this option is activated, the Enable Parameterization from CANoe and Enable Simulink Signal Analysis options allow the access to the parameters or signals in referenced models. Otherwise only parameters or signals of the root model can be accessed.
  • Exclude specific Blocks from Parameterization:
    Allows to filter out some blocks from the parameterization. A standard regular expression can be given in the Blocks to exclude from Parameterization input box. This feature is available starting with Visual C++ 2013 (12.0).
  • Generate vModule file (*.vmodule):
    Generates a module description file of type vModule which references the built DLL/SO. If the output directory is not specified, the file will be generated in the same folder as the Simulink model and references to both 32-bit and 64-bit DLL/SO built in the default path. Otherwise, it will be generated in the specified output directory and references to the DLL/SO generated in the same folder.

Note

Only non-complex scalar or vector parameters are supported by the parameterization and signal analysis feature. Both make use of the C-API interface.
