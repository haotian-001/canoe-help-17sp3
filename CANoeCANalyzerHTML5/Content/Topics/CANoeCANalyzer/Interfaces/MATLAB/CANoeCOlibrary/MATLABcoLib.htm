Communication Object Library

 

The Communication Objects library is a collection of S-Functions blocks.

The library provides the following features:

Communication Objects

Short Description of the Block

CO Function Answer

Can be used to define the behavior and data of an automatic function return.

CO Function Call

Is used to execute a function call.

CO Function Calling

Can be used to trigger Simulink subsystems on function call.

CO Function Return

Can be used to trigger Simulink subsystems on function return.

CO Latest Call

Is used to get the data of the latest function call.

CO Latest Return

Is used to get the data of the latest function return.

CO Signal Input

Is the basic input block for a communication object signal.

CO Signal Output

Is the basic output block for a communication object signal.

CO Signal Struct Input

Is used for input the signal of type Struct.

CO Signal Struct Output

Is used for output the signal of type Struct.

Union Data Type

A Union data type, as part of a signal struct, is represented in Simulink as a bus with all union elements. Additional, the first bus parameter represents the current union discriminator.

  • a discriminator value of 0 means that none of the union elements is valid.
  • a discriminator value larger 0 means that the according union element is valid.

This implicit discriminator is always available regardless whether the struct contains an explicit discriminator value or not.

String Data Type

A String data type is represented in Simulink as an array of type uint8. A string will be stored within this array as null terminated string. Therefore, the string has a maximum number of characters equal to the size of the array -1.

The length of the array, connected to an input port or output port of a block with a sting value, can be less or equal to the string length. Furthermore, the null termination sign is optional. For example, a string with a size limitation of five is used. In this case the uint8 array may have a size of one up to five elements. As the null terminated string is optional, valid values are:

  • [uint8('12')]
  • [uint8('12') 0]
  • [uint8('1234')]
  • [uint8('1234') 0]

If a struct contains a string, it is represented in Simulink as bus element. This bus element is a null terminated array too.

Bus Creation

A struct data type is represented in Simulink as a bus with necessary subbuses. All blocks which uses signals of type Struct or Function Port Data offer a Create bus functionality to create the corresponding buses. Once a bus is created it will be generated automatically on opening the Simulink model. The bus creation information is stored in files. These files are in directory BusScripts which is in the same directory as the Simulink model. If a bus is no longer needed, the according file can be deleted.
