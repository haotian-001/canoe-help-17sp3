MATLAB Integration Package

Ribbon: Environment Tab|Tool Couplings|MATLAB Integration Package

This Help describes the CANoeMATLAB Integration Package. It was designed to combine the strengths of both simulation tools. CANoe is a total network simulation tool designed to both model nodes on a network and the network that binds them. Traditionally the functionality of these network nodes are modeled using the CAPL language provided with CANoe. The purpose of this interface is to extend CANoeâs node modeling capability by adding the strength of the MATLAB/Simulink environment. Now models can be used to describe the functionality of a network node in CANoe. This allows CANoe to simulate nodes which are impossible or difficult to model using CAPL alone. Another feature would be to use a Simulink model to generate an outside stimulus to an existing CAPL node.

Modeling Concepts

It is important to understand how the MATLAB Integration Package communicates with CANoe in order to easily design a model. It is easiest to think of the CANoeMATLAB Integration Package as an extension or a complete replacement of existing CAPL code. The application of a CAPL node can be completely implemented with Simulink. It is possible to design an entire ECU in Simulink. Examples of this can be found under Sample Configurations. For definition of concepts see Definition of Concepts. Please continue with the Setup if you are familiar with Software Layers of an ECU.
