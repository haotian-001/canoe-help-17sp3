AUTOSAR NM Observer

 

Introduction

The AUTOSAR NM Observer enables you to perform an analysis (online and offline) of the protocol behavior of control units with respect to the AUTOSAR Network Management (NM) on CAN buses. This allows you to detect incorrect behavior at an early stage.

You can activate and configure the AUTOSAR NM Observer using the NM Configuration dialog.

Method

Overview

The AUTOSAR NM Observer observes the bus traffic (NM messages and application messages) and attempts to determine the current state of the control units from these observed messages. This determined state is written in system variables. You can use the State Tracker to visualize the corresponding states.

For certain events, status messages are generated (e.g., if an NM message is missing) and displayed in the Trace Window.

The system variables of the Observer are in the AUTOSAR_NM namespace. This namespace must not be used for your own system variables. When the Observer is activated, fixed system variables (per CAN network) that reflect the observed state are created automatically.
After the configuration is saved, the system variables are displayed in the System Variables Configuration dialog.

Detailed Method of Functioning

The AUTOSAR NM protocol defines a state machine, including the conditions for the state transitions. The manner in which NM messages are sent is also dependent on the state. The AUTOSAR NM Observer monitors the arrival of these messages with timers and determines based on their arrival or absence if the current state is still valid or if a transition to another state is necessary. The timers and state machines used by the Observer are described below.

Timers

The values used for timers result to a large extent from the AUTOSAR NM protocol. However, specific time periods are also used for the Observer. Some of these specific values serve exclusively for visualization of certain bus events.

Timers from AUTOSAR NM Specification

These timers are defined in the INI file ([TIMER] section) or the database (DBC). The same definitions from the DBC or INI file are used for the NM simulation with the aid of the NM-DLL. Refer to the Technical Reference: AUTOSAR NM CAN 3.3.0 for more information.

Timer is restarted with each received and sent NM message. When the timer expires in Ready Sleep state, a transition to Prepare Bus Sleep state occurs.

Default value: 2000 ms

NM_WAIT_BUS_SLEEP_TIME

Default value: 1500 ms

(DBC: NmAsrRepeatMessageTime)

Time the node is in Repeat Message state.

Default value: 3200 ms

NM_IMMEDIATE_NM_CYCLE_TIME

Cycle time of NM messages in Repeat Message state when messages with shortened cycle time are being sent.

NM_MAX_START_DELAY

When there is a new request of a partial network (transition to Requested mode), all nodes of the partial network must be in Repeat Message or Normal Operation state after this time.

Maximum permissible time period that application messages of a control unit may be sent before its first NM message (after exiting the Bus Sleep state).

For a node the maximum permissible time period that may elapse between sending its own NM message or receiving an NM message relevant to it and sending its application message. The selected time must be greater than NM_TIMEOUT_TIME.

Default value: 100 ms

The system variable for indicating the occurrence of an NM message with setCarWakeup bit is in Active state for this time period; used for visualization only

Default value: 500 ms

Effect of Arbitration on Timers

In order for a message to be sent on the CAN bus, the arbitration must first be won. This can produce time delays that the AUTOSAR NM Observer must take into consideration. For this reason, for example, the timer values used for monitoring the cycle time are greater than the values designated by the protocol. In such cases, the values used are determined based on the following scheme

Timer value = Protocol value * Tolerance factor

This tolerance factor (NM_OBSERVER_TIME_TOLERANCE, default value: 1.2) is used for monitoring of the following times:

  • CANNM_MSG_CYCLE_TIME
  • NM_IMMEDIATE_NM_CYCLE_TIME
  • ACTIVE_TIME

Limitations

NM state changes in control units often do not immediately affect the sent bus messages. Due to its completely passive method of functioning, the AUTOSAR NM Observer cannot detect such state changes until the effects on bus messages are present. As a result, there may be a time offset between the current NM state of the control unit and the state determined by the Observer. In rare cases, the Observer indicates the transitions too late, even if the control unit is functioning properly and the delays on the bus are not excessive (this pertains especially to the Normal Operation and Ready Sleep NM states). However, the overly long observed hold time in this state is limited to one NM cycle time.

Configuration and Input Parameters

The AUTOSAR NM Observer can be activated for the individual CAN networks in the NM Configuration dialog. The nodes to be monitored can be selected individually.

The Observer obtains its settings from the same INI files that are used for AUTOSAR NM simulation (see Technical Reference: AUTOSAR NM CAN 3.3.0). If the corresponding files already exist, they can be opened in the configuration dialog. Additional settings are taken from the database (DBC). The Observer also requires other parameters, which can be added to one of the two INI files (local/global). If these parameters are not assigned, the appropriate default settings are used.

The additional parameters are as follows:

[OBSERVERTIMER]
; all timers are in milliseconds
NM_MAX_START_DELAY=200
MAX_APPL_BEFORE_NM_MESSAGE_TIME=0
MAX_APPL_AFTER_NM_MESSAGE_TIME=10
MIN_ACTIVE_TIME=0
ACTIVE_TIME=100
WAKEUP_TIME=500
; timer tolerance
NM_OBSERVER_TIME_TOLERANCE=1.2

For an explanation of the parameters, refer to AUTOSAR NM Observer-specific timers

Except for NM_MAX_START_DELAY, the values of all timers can be overwritten in the node-specific sections [node name].

Note

For a simulation, only the nodes to be simulated have to be specified in the INI files. By contrast, the AUTOSAR NM Observer requires all nodes to be specified.

Evaluation Sequence

Default values are defined for almost all parameters. These are overwritten by corresponding information in the INI files or the DBC.

The evaluation sequence for this is as follows, beginning with the lowest priority:

  1. Default values
  2. Global INI file (general)
  3. Database (DBC)
  4. Local INI file (general)
  5. Global INI file (node-specific)
  6. Local INI file (node-specific)

Because of this evaluation sequence, the creation of a local INI file is recommended. Its structure is described in the Technical Reference: AUTOSAR NM CAN 3.3.0 in the pages about the INI files.
