Coupling of two CANoe Instances using the FDX Protocol

 

Note

The coupling of two CANoe instances requires the use of UDP (IPv4 or IPv6) as transport layer.

The CANoe FDX protocol was originally designed in such a way that the communication always starts at the FDX client. This means that the FDX client sends a UDP datagram to CANoe first, whereupon CANoe sends as response one or multiple datagrams back to the sender. The necessary protocol settings that CANoe should use for this FDX client (e. g. IP address and port number of the FDX client, byte order, FDX version) are automatically derived from the datagram of the client. This means that the CANoe configuration remains largely free from the specific protocol settings of the FDX client.

If you want to couple two CANoe instances, this procedure does not work any more because now one of the CANoe instances must send the first UDP datagram and therefore needs the specific protocol settings of the remote station. In order to still establish a coupling of two CANoe instances the following CAPL functions have been added to CANoe:

  • FDXClientHandleUdp
  • FDXConfigureSequenceCounting
  • FDXDisableFreeRunningMode
  • FDXEnableFreeRunningMode
  • FDXSetByteOrder
  • FDXSetProtocolVersion

Example

The following CAPL example sets the protocol settings for a second CANoe instance and sends cyclically the FDX data groups 310,311 and 312 to this second instance.

variables
{
  long fdxClientHandle = 0;
}

void StartFdxCommunication()
{
  char FDXClientAddrString[64] = "127.0.0.1"; // loopback device
  dword FDXClientPort = 2810;
  dword FDXClientAddr;
  // get fdx client handle
  FDXClientAddr = IpGetAddressAsNumber(FDXClientAddrString);
  fdxClientHandle = FDXClientHandleUdp(FDXClientAddr, FDXClientPort);

  // configure protocol parameters
  FDXSetProtocolVersion(fdxClientHandle, 2, 0); // FDX protocol version 2.0
  FDXSetByteOrder(fdxClientHandle, 1); // Encoding uses little endian
  FDXConfigureSequenceCounting(fdxClientHandle, 2); // Disable sequence counting

  // Activate cyclic transmit of data groups 310, 311 and 312 with a cycle time of 10ms
  // and different start offsets (1ms, 2ms and 3ms)
  FDXEnableFreeRunningMode(fdxClientHandle, 310, 4, 10000000, 1000000);
  FDXEnableFreeRunningMode(fdxClientHandle, 311, 4, 10000000, 2000000);
  FDXEnableFreeRunningMode(fdxClientHandle, 312, 4, 10000000, 3000000);
}

void StopFdxCommunication()
{
  // Deactivate cyclic transmit of data groups 310, 311 and 312.
  FDXDisableFreeRunningMode(fdxClientHandle, 310);
  FDXDisableFreeRunningMode(fdxClientHandle, 311);
  FDXDisableFreeRunningMode(fdxClientHandle, 312);
}
