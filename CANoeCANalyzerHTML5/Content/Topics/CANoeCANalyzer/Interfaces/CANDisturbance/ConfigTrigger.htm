Configuring Triggers

 

You can define various triggers with the CAN Disturbance Interface:

  • Software Trigger
  • Frame Trigger with and without trigger enable function
  • External Trigger
  • Error Frame Trigger
  • Scope Trigger
  • Inverted Trigger
  • Send Direction Trigger
  • Missing Bit Trigger

Software Trigger [â² back]

The software trigger can be activated with the CAPL function CanDisturbanceTriggerNow. This is implemented by sending a user-defined sequence or a frame-based sequence on the bus. It is sent as soon as the sequence is configured on the CAN Disturbance Interface.

CAPL Example

Frame trigger with and without trigger enable function [â² back]

Use a frame trigger when a user-defined or frame-based sequence should only be sent in case of a certain frame. The frame trigger can be configured with the CAPL function CanDisturbanceTriggerEnable. There are two variants of this function.

Variant 1: Outputting user-defined sequences

Use this variant if you want to disturb a bit in a frame, for example.

Example: Disturbing the acknowledge slots of a CAN frame

Proceed as follows:

  1. Set the trigger position for the frame trigger to the CRC delimiter.
  2. Configure the frame trigger for a certain frame.
  3. Define a sequence that will send a recessive signal instead of a dominant signal.
  4. Configure the trigger for the CAN Disturbance Interface.

CAPL example: CanDisturbanceTriggerEnable (frame and sequence)

Variant 2: Frame-based sequence (outputting entire frame as a sequence)

Use this variant if, for example, you want to generate a burst from multiple frames by sending another frame right after a certain frame on the bus. Alternatively, you can also simulate a faulty control unit that starts sending within a frame.

Example: Generate burst on bus

Proceed as follows:

  1. Set the trigger position for the frame trigger to the bit 2.IFS.
  2. Configure the frame trigger for a certain frame.
  3. Convert a message into a frame sequence.
  4. Configure the trigger for the CAN Disturbance Interface.

CAPL example: CanDisturbanceTriggerEnable (frame and frame sequence)

Note

A frame trigger can additionally be linked to the value of a digital input. If the value corresponds to the configured value and the frame trigger condition is fulfilled, the sequence will be sent. If only the frame trigger condition is fulfilled, the sequence will not be sent. Valid values for this trigger condition are ActiveHigh or ActiveLow.

DIN0 and DIN1 are available as digital inputs. However, it is not possible to use both inputs for a trigger enable function at the same time.

External Trigger [â² back]

An external trigger is a trigger using the digital DIN0 or DIN1 inputs in order to send a user-defined or a frame-based sequence to the bus.

The trigger is activated as soon as the value of the configured digital input corresponds to the configured condition. The following trigger conditions can be set:

  • Active_High
  • Active_Low
  • Rising_Edge
  • Falling_Edge

The variants for a user-defined sequence and a frame-based sequence are also available for the external trigger.

Use a user-defined sequence to send small disturbance sequences to the bus. The sequence will be sent as soon as the value of the digital input corresponds to the configured value.

Use a frame-based sequence to send an entire frame to the bus. The sequence will be sent as soon as the value of the digital input corresponds to the configured value.

CAPL examples:

  • CanDisturbanceTriggerEnable (external trigger and sequence)
  • CanDisturbanceTriggerEnable (external trigger and frame sequence)

Error Frame Trigger [â² back]

You can use the CAN Disturbance Interface to trigger based on an active or passive error flag and to generate disturbances within the error delimiter.

The variants for a user-defined sequence and a frame-based sequence are also available for the error frame trigger.

CAPL examples:

  • CanDisturbanceTriggerEnable (error frame and sequence)
  • CanDisturbanceTriggerEnable (error frame and frame sequence)

Scope Trigger [â² back]

The scope trigger is activated and deactivated via the CANDisturbanceInterface<DeviceId>::Scope::Enable system variable.

If you have activated the scope trigger, a pulse will be output at the Scope Trigger output as soon as the CAN Disturbance Interface outputs a user-defined or a frame-based sequence. This enables you to trigger e.g. an oscilloscope.

You can configure the polarity, duration and delay time of the pulse. The delay time is the time during which no pulse is output even when there is a sequence output.

The corresponding system variables are also available in the namespace CANDisturbanceInterface<DeviceId>::Scope.

Inverted Trigger [â² back]

The inverted trigger is configurable for frame trigger and multiple frame trigger.

This trigger is usable, if all frames should be disturbed but one or more frames should not.

The trigger is configured like a frame or multiple frame trigger.

To configure the inverted trigger together with a defined trigger use the parameter flags of the following functions:

  • CanDisturbanceTriggerEnable (frame and sequence)
  • CanDisturbanceTriggerEnable (frame and frame sequence)
  • CanDisturbanceTriggerEnable (Multiple frames and sequence)
  • CanDisturbanceTriggerEnable (Multiple frames and frame sequence)

Send Direction Trigger [â² back]

The send direction trigger is configurable for frame trigger and multiple frame trigger.

The trigger is configurable for receive (Rx) and transmit (Tx) direction. If neither Rx nor Tx are configured both directions are used.

To configure the direction trigger together with a defined trigger use the parameter flags of the following functions:

  • CanDisturbanceTriggerEnable (frame and sequence)
  • CanDisturbanceTriggerEnable (frame and frame sequence)
  • CanDisturbanceTriggerEnable (Multiple frames and sequence)
  • CanDisturbanceTriggerEnable (Multiple frames and frame sequence)

Missing Bit Trigger [â² back]

It is possible to configure that a defined frame trigger or multiple frame trigger also send the sequence, if the trigger position not reached.

The trigger can be used e.g., if the CRC delimiter should be disturbed for all CAN and CAN FD frames. In this case it is not possible to define for all frames a valid offset within the CRC field. The offset could be 14, 16 or 20.

If the trigger position is set to 21, without the bit missing trigger only CAN FD frames with data length greater than 16 data bytes are disturbed at the CRC delimiter. If the trigger position offset is set to 14 all frames will disturbed, but for CAN FD frames not at the CRC delimiter.

With a missing bit trigger, the offset can be set to 21 of the CRC field. In this case the trigger is sent the sequence within the next bit also, if the trigger position is not reached. In this example the next bit is the CRC delimiter.

To configure the missing bit trigger together with a defined trigger use the parameter flags of the following functions:

  • CanDisturbanceTriggerEnable (frame and sequence)
  • CanDisturbanceTriggerEnable (frame and frame sequence)
  • CanDisturbanceTriggerEnable (Multiple frames and sequence)
  • CanDisturbanceTriggerEnable (Multiple frames and frame sequence)
