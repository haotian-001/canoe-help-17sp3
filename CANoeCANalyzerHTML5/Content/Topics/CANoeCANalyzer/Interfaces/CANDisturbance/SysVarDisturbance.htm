System Variables

 

There are device-specific system variables containing information on the current configuration status, and device-independent system variables. The system variables are available only if you have activated the disturbance functionality in the Network Hardware Configuration dialog for at least one channel.

Device-Independent

The device-independent system variables are enumerations with the respective possible values for trigger, trigger source, DIO channels, etc.

The system variables are located in the general namespace CanDisturbance::Enums.

All system variables in this namespace are read only and have the value Invalid(-1). The sole purpose of the variables is to act as an enumerator.

They are assigned to groups in the following table. The group defines what the system variable can be used for.

Device-specific

The CAN Disturbance Interface-specific system variables are located in the general namespace CANDisturbanceInterface<DeviceID>. In this context, the placeholder <DeviceID> represents the device ID that you set for the device in the Network Hardware Configuration dialog.

Whenever the number of possible values is limited, these system variables contain value tables with the corresponding possible values.

Configuration

Type: int

Value table:

  • 0: Settings not store in configuration
  • 1: Settings store in configuration

When the value is set to 1, the values for resistances, capacitors, switches and the scope configuration are saved in the configuration. Otherwise these values will not be saved and default values will be used when the configuration is called again.

ResetConfiguration

Type: int

Value table:

  • 0: None
  • 1: Reset

Resets the configured values to the values currently stored in the configuration.
