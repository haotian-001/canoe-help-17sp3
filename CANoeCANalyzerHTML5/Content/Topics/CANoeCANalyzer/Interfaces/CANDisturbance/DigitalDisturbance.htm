Digital Disturbances

 

You can use the CAN Disturbance Interface to output dominant or recessive signals on the CAN/CAN-FD bus in order to generate digital disturbances. A distinction is made between user-defined sequences and frame-based sequences.

User-Defined Sequences

User-defined sequences consist of multiple segments, each of which is defined by a time period and a bus level value. The bus level of a user-defined sequence can have any of the following values:

  • Dominant level ("d")
  • Recessive level ("r")
  • Recessive level ("R") that can overdrive a dominant level from another bus-sharing unit

In CAPL, this sequence is represented by the data type (class) CanDisturbanceSequence. A user-defined sequence can have a length of up to 4096 segments. Each segment can have a length between 1 and 65535 FPGA ticks. One FPGA tick corresponds to 6.25 ns resulting in a maximum possible segment length of approx. 409 µs.

If you want to add a segment to a sequence, you have to use the AppendToSequence function of the CanDisturbanceSequence data type. This function expects the length and the value of the segment as parameters.

Example

A segment is to have a length of 2 µs and a dominant bus level:

CanDisturbanceSequence.AppendToSequence(320, ‘d‘);

This call adds a segment to the current sequence.

If a user-defined sequence is not to be output immediately, you can combine it with frame triggers, external triggers or an error frame trigger. The user-defined sequence is sent only when the corresponding trigger condition is fulfilled.

Frame-Based Sequences

A frame-based sequence represents an entire frame transmitted on the bus. Within the sequence, you can define a digital disturbance for each bit of the frame, e.g. a disturbing pulse within the bit or an inversion of the bit. As with user-defined sequences, you can configure recessive overload of a dominant level.

This figure shows the assignment of CAN/CAN FD frame fields according to the selectors of the CanDisturbanceFrameSequence CAPL data type (class). This maps a frame-based sequence.

The frame-based sequence contains all fields of a CAN/CAN FD frame (from BaseID to EndOfFrame EOF). A field is represented by the CanDisturbanceFrameSequenceField CAPL data type.

The fields of a CAN/CAN FD frame can contain one bit (e.g. FDF) or multiple bits (e.g. BaseID).

The BitSequence selector of the CanDisturbanceFrameSequenceField data type can be used to access the individual bits. The selector returns an array of bits. The number of bits used by a field can be queried using the FieldLength selector.

The BaseID field of a CAN/CAN frame comprises 11 bits represented by fields ID28 to ID18.

To access the ID28 bit of a CanDisturbanceFrameSequence object, for example, the IDBase selector must be used. The selector returns an object of the CanDisturbanceFrameSequenceField data type. This data type has the BitSequence selector, through which the individual bits can be accessed.

Each bit is written by the CanDisturbanceBitSequence data type. This sequence is comparable to the user-defined sequence. The difference here is that a maximum of eight segments can be defined. The values of the individual segments can be determined using the SegmentValue or SegmentLength selector.

Note

When a CanDisturbanceFrameSequence object is created, no segments are defined and set to 0 for a CanDisturbanceBitSequence. This means that this bit or field is not used for the frame-based sequence.

Example 1

canDisturbanceFrameSequence frameSequence; //Define a empty frame sequence
word length
byte value;
//Access to the bit sequence which represent the ID28 bit
CanDisturbanceBitSequence bitSequence = frameSequence.IDBase.BitSequence[10];
//Get length and value of first segment of the sequences of ID28
If(bitSequence.SegmentCount > 0)
{
  length = frameSequence.IDBase.BitSequence[10].SegmentLength[0]; //length is equals 0
  value = frameSequence.IDBase.BitSequence[10].SegmentValue[0]; //value is 0
}

The CanDisturbanceFrameSequence object has all fields possible for a CAN/CAN FD frame at all times. Configuration can easily be carried out with the SetMessage method of the object and a CAPL CAN message object. Here, all required CanDisturbanceFrameSequenceField objects are set to the values from the CAPL CAN message object.

Example 2

canDisturbanceFrameSequence frameSequence; //Define a empty frame sequence
int deviceId = 1; //Unique id of the Disturbance interface
message 0x100 msg; //Configuration object for the frame sequence

word length
byte value;
//Configure message object for the frame sequence
msg.dlc = 4;

//Configure frame sequence object with message object
frameSequence.SetMessage(deviceId, msg);
//Access to the bit sequence which represent the ID28 bit
CanDisturbanceBitSequence bitSequence = frameSequence.IDBase.BitSequence[10];

//Get length and value of first segment of the sequences of ID28
If(bitSequence.SegmentCount > 0)
{
  length = bitSequence.SegmentLength[0]; //length is equals 320
  value = bitSequence.SegmentValue[0]; //value is ‘d’
}

The length is specified in FPGA ticks. A tick corresponds to 6.25 ns at 160 MHz. If a baudrate of 500 kBaud/s is used, a bit is 2,000 ns long. This results in 320 ticks (ticks = 2,000 ns/6.25 ns) for the length of a segment if this is to represent a bit.

It is possible to modify the CanDisturbanceBitSequence objects using SetMessage after configuration.

The modifications make it possible to

  • Invert bit values (protocol test)
  • Define glitches within a bit (e.g. sample point test)
  • Shorten or lengthen the bit (e.g. sample point test)

Example 3

canDisturbanceFrameSequence frameSequence; //Define a empty frame sequence
int deviceId = 1; //Unique id of the Disturbance interface
message 0x100 msg; //Configuration object for the frame sequence

word length
byte value;
//Configure message object for the frame sequence
msg.dlc = 4;

//Configure frame sequence object with message object
frameSequence.SetMessage(deviceId, msg);

//Insert a glitch after 50 % and 10% length and shorten the bit by 5%
if ( frameSequence.IDBase.BitSequence.SegmentCount > 0)
{
  length = frameSequence.IDBase.BitSequence[10].SegmentLength[0]; //length is equals 320 ticks
  value = frameSequence.IDBase.BitSequence[10].SegmentValue[0]; //value is ‘d’
  frameSequence.IDBase.BitSequence[10].SegmentCount = 3;
  //Set segment lengths
  frameSequence.IDBase.BitSequence[10].SegmentLength[0] = length*0.50; // Set to 160 ticks
  frameSequence.IDBase.BitSequence[10].SegmentLength[1] = length*0.10; // Set to 32 ticks
  frameSequence.IDBase.BitSequence[10].SegmentLength[2] = length*0.35; // Set to 128 ticks

  //Set segment values
  frameSequence.IDBase.BitSequence[10].SegmentValue[0] = value; // Set value ‘d’
  frameSequence.IDBase.BitSequence[10].SegmentValue[1] = value == ‘d’?‘R’:‘d’; //Invert bit
  frameSequence.IDBase.BitSequence[10].SegmentValue[2] = value; // Set value ‘d’
}

If a frame-based sequence is not to be output immediately, you can combine it with frame triggers, external triggers or an error frame trigger. The frame-based sequence is sent only when the corresponding trigger condition is fulfilled.
