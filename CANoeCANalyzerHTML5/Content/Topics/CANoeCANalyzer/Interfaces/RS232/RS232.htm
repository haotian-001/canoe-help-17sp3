RS232 API

 

In CANoe you can control the RS232 interface via CAPL functions. If your computer does not have a built-in RS232 interface, you can use a USB to RS232 adapter. Be sure to install the corresponding adapter driver when doing so. You can also use other network interfaces with the RS232 API. The requirement for this is that Windows must recognize these serial network interfaces as serial COM ports. One example would be IrDA, as most are represented as a virtual COM port in Windows.

Note

  • Starting with CANoe 7.1 several RS232 CAPL functions have been declared deprecated. These functions have been fully removed with CANoe 12.0.
  • Notes on using USB-RS232 Converter.

Differences between the current version and the obsolete RS232 CAPL API:

  • You do not longer require a separate CAPL DLL, and hence the #pragma library CAPL command is not applicable.
  • You do not longer require a v24.INI configuration file.
  • Serial interfaces are detected automatically, i.e. manual action is no longer required (e.g. the entry of numbers in files).
  • You can open all available ports from CAPL.
  • Blocking write operations are obsolete.
  • Reception of data blocks.
  • Parallel sending and receiving.
  • Error handling via callback handler.

Changes if programs are to be ported:

  • The CAPL DLL rs232vc.DLL must be removed (Options dialog).
  • The CAPL DLL rs232vc.DLL may not be integrated in the includes section via #pragma library.
  • Blocking send variants such as Rs232WriteBlock/Rs232WriteByte should be replaced by the non-blocking function RS232Send.
