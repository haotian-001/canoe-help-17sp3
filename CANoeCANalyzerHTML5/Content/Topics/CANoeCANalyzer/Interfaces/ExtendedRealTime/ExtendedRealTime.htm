Extended Real Time (ERT)

 

The Extended Real Time (ERT) extension enables the execution of CAPL programs and MATLAB simulation models under realtime conditions up to typical cycle times of 100µs. ERT additionally supports a CAN/CAN-FD network interface. By directly addressing the I/Os of the VT system from ERT, they can also be used under realtime conditions. To configure and use ERT on a VT system, there is more information and detailed instructions on this help page.

For this purpose, a VTP device is divided into 2 logical areas, one for regular simulation execution and one for realtime execution. The use of ERT during simulation requires adjustments in the CANoe Simulation Setup. Parts of the simulation that are not executed in ERT can be used as usual. Data transfer of variables between ERT and regular simulation execution can be automatic or selective.

Extended Real Time can be activated on some VTP devices in addition to Distributed Mode or Standalone Mode.

Activation of Extended Real Time

Two steps are necessary for ERT activation:

Step

Description

1.

Activation on the VTP device

  • Activate via the Vector Platform Manager or CANoe|VTP Tool Platform|Device Settings|Extended Real Time (ERT).
    The VTP device restarts.

2.

Activation in CANoe

  • CANoe|File|Options|Measurement|Performance

  • Activate ERT network node.

  • Selective data exchange can be activated in the check box below.

Using Extended Real Time

After activation, ERT Network Nodes can be added to the CANoe Simulation Setup.

Only the programs created as ERT network nodes are executed in the ERT. All other network nodes are executed in the regular simulation environment.

By default, all updates of ERT variables are synchronized with other network nodes. This can have a negative impact on the realtime performance of the ERT network nodes if the number is high. For optimization, selective synchronization of ERT variables can be enabled in the performance options. This allows manual definition and focus on the required synchronized ERT variables. This is activated by the Sync ERT Update option in the System Variable Configuration dialog.

Activating CAN modules in Extended Real Time

Some CAN modules can be used either in Extended Real Time or conventionally in the RT kernel. The selection is possible per module via the Vector Platform Manager or CANoe|VTP Tool Platform|Channels/Network Interfaces. By selecting a module, a check box Available in ERT is displayed on the right side. For different devices there are different default settings here. A restart of the device is necessary to change the assignment.

Note

Extended Real Time supports all integer and double types. Also system variable arrays of the types Integer (32 bit signed) Double and Data can be used.

For an overview of CAPL functions supported in ERT, see this overview.

Note

CAPL DLLs are not supported for CAPL in ERT.

To monitor the performance of ERT, you can use the Performance Monitor.

Note

With CANoe 17 and VTP 5.0 there are some new features for the use of Extended Real Time (ERT).
