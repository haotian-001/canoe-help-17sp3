VN8900: Replacing a Hardware Module

 

Changing hardware modules

When a VN8900 module is replaced by a different one (e.g. replacement of a VN8950 module by a VN8970 module), some steps are required to ensure correct interworking of CANoe and the VN8900 device. The detailed actions to be done depend on the versions of the operating system image and CANoe installed on the VN8900 device, but in general the following steps are required:

  1. Update software on the VN8900 device

    This step is only necessary if the software version on the VN8900 device is different from the version installed on the computer.
    Details on performing a software update.

  2. Replace the old VN8900 module by the new one

    To replace the VN8900 module by a new one, follow the steps below:

    1. If not already done, unplug the power supply from the VN8900 device.
    2. Loosen the screws of the VN8900 module, located on the front panel.
    3. Pull the VN8900 module from the VN8900 device (you can drag it by holding the screws)
    4. Push in the new VN8900 module into the VN8900 device. Pay attention to the correct orientation of the module.
    5. When you completely pushed in the new module, tighten the screws located on the front panel.
    6. Plug in the power supply again.

  3. Update the VN8900 driver on the computer

    This step is only necessary if the current VN8900 driver on the computer does not support the new module. Download the driver package from the Vector homepage and execute the setup program. In the selection dialog ensure that the VN8900 driver is selected for installation.
    Note that on a 64 bit operating system the 64 bit version of the VN8900 driver is required.

  4. Install the driver for the new module on the VN8900 device

    This step is only necessary if the operating system image on your VN8900 does not yet contain an installed driver for the new module (e.g., a VN8970 driver). You can check this by using the VN8900 driver update feature available in CANoe. If the module driver is not installed perform a driver update.
    Note that you need the 32 bit version of the driver for the VN8900 module even if you connect it to a computer running a 64 bit operating system.

  5. Update channel assignment in Vector Hardware Config panel

    The mapping of application channels (e.g.CANoe CAN 1) to the hardware channels (e.g. VN8911 1 Channel 1) on the new VN8900 module reflects the channel assignments that had been configured with the old VN8900 module. Because the application channels are as-signed to the hardware channels in the same order as before a CAN application channel may now be assigned to a FlexRay hardware channel (i.e., a channel equipped with a FlexRay piggy). If you try to start measurement in this situation you will receive an error message.
    Therefore it is required to check and potentially update channel assignment for all application channels in Vector Hardware Config panel (available via Windows panel) to reflect the new situation. If you use the device in distributed mode or in standalone mode, you can also directly change the channel mapping on the device (Home ribbon tab|Group Measurement|Vector Tool Platform|Tool Platform|Channel Mapping). In the example above, assign a FlexRay application channel to the FlexRay hardware channel and move the CAN channel to an appropriate CAN hardware channel (i.e., a channel equipped with a CAN piggy).
