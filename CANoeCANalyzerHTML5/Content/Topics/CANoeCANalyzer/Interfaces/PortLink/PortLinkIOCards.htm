PortLink: I/O Cards

 

Note

In the Options dialog (Extensions|I/O Device Drivers) you can set the device DLLs that should be used for the configuration.

At this time the following DLLs are available:

DLL

Description

pl_nidaqmx.DLL

Supports most National Instruments I/O cards and data acquisition devices.

pl_redlab.DLL

Supports the Meilhaus RedLab device Family.

I/O Ports

Each I/O card has a number of ports. PortLink supports analog inputs and outputs and digital inputs or outputs.
With some cards the number of ports may change when the card configuration is changed. For example, changing a port from Single Ended to Differential reduces the number of available analog ports by one-half.

  • Analog I/Os
    The value range of an analog port depends on the I/O card and the port type.
  • Digital I/Os
    Digital ports are always single-bit ports. With some cards the direction of the digital ports can be switched.

Card Installation

Installation of an I/O card is very dependent on the card type. Therefore we refer you to the relevant manuals from your card producer. Usually it is necessary to install a card driver. For cards from National Instruments you must also install their "Measurement and Automation Explorer". Among other things this is used to set the device numbers of the cards.

Device Identification

In PortLink an I/O card is referred to as a device.
Each Device can be identified uniquely by its device number and by the device DLL identifier. The device number is obtained in various ways:

  • DLL for National Instruments cards:
    The device number corresponds to the device number that can be set in the configuration of the National Instruments card.
  • Demo DLL with function generators:
    The device number is permanently set in the DLL and cannot be changed.

Port Identification

Each port can be identified uniquely by its port number, device number and device DLL identifier.
