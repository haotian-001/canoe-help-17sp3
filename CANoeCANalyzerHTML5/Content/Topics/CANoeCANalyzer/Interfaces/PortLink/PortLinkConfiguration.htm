Ribbon: Hardware Tab|Other

Port List

Listed here are all ports existing in the computer. List entries consist of the following columns:

  • Type
    Port type:
    Analog input (AI), analog output (AO),
    digital input (DI) or digital output (DO).
  • Acquisition
    This column is only filled out for input ports and describes the acquisition type for the respective port.
    Possible acquisition modes:
  • Polling
    Measurement values are acquired in an update interval to be specified.
  • Event
    Driver triggers an event when changing an acquired measurement value.
  • Stream
    Data from the driver is continuously acquired with a certain frequency and made available in data blocks.
  • Port name
    The port's identifier
  • Port #
    Number of the port on the card
  • Device name
    Alias name of the card – the alias name is set in the DLL
  • Device #
    Number of the card
  • Sampling
    Sampling frequency for ports that support the acquisition mode Stream.
  • Update [ms]
    Update interval for the port.
    For Polling ports the update interval can be set in ms.
    For Stream ports the update interval is derived from the configuration of the card and can't be changed.

Sorting the List Entries

You can sort either list by clicking a column header.
You can return the Port List to its standard sorting sequence by clicking the header of the DLL alias column.

Exit Configuration Dialog

  • No changes were made
    If no changes were made the configuration can only be exited by [Cancel].
  • Changes were made
    If changes were made and the user attempts to exit the dialog with [Cancel] a dialog appears warning that all changes will be lost.
    If changes were made and the dialog is exited with [OK] all changes are accepted and saved in the configuration file.
  • Accept changes
    If changes were made they can be saved without exiting the configuration by pressing the [Accept] button.
