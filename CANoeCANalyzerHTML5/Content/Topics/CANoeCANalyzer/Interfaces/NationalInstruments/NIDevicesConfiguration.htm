Configuration

 

Installation

You will need National Instruments Measurement and Automation Explorer (MAX) in order to configure devices and for use in CANoe. MAX an further required software components are available at NI; see NI-DAQmx Driver (https://www.ni.com/dataacquisition/nidaqmx.htm). When installing the NI-DAQmx driver you should use at least version 9.5.

The MAX can be used to configure both, the devices themselves and the measurement tasks for the particular devices. A task covers one or more ports that can be used with that task. Settings are made on a task-specific basis and affect all the relevant ports.

Please note that the supported types of acquisition and output will depend on the devices used. For more detailed information, refer to the description of the device in question.

The MAX can be used to assign names to tasks and the ports they contain. These names are used to identify and locate tasks and their ports at a later date.

Step by Step Procedure: Installation

  1. Install the NI-DAQmx driver.
  2. Open the NI Measurement and Automation Explorer (NI MAX).
  3. Create one or more Measurement Tasks.
  4. Save your settings and close NI MAX.
  5. Start CANoe and open the Configuration dialog (Other in Hardware ribbon tab).
  6. Click on [I/O Device Driversâ¦].
  7. Activate the pl_nidaqmx.dll I/O device driver and close the dialog with [OK].
  8. Now the configuration dialog shows all available NI tasks and ports.
    You can now use these ports in CANoe via system variables (namespace IO).

Note

When using distributed mode (VT6000, RT Rack,â¦), installation and configuration of NI MAX both have to be done on the RT server.

Creating a Task

Step by Step Procedure: Creating a Task

  1. Create a new task in the Measurement and Automation Explorer under the NI-DAQmx â Tasks folder.
  2. Select the task type. The following types are supported:
  • Analog acquisition
  • Analog output
  • Digital I/O
  • Counter I/O

Depending on the application in question, counter-I/O works with either analog or digital signals and the corresponding variable types.

  1. On the next page, select the ports you want to use with this task. Please note that, for the analog input, you can only ever create one task per device and that this task may only ever contain ports that are found on the same physical device.
  2. Assign a name to the task.

General Task Settings

Once you have created a new task, you can modify the names of the individual ports and make other general settings, such as the voltage range for analog ports or invert lines for digital tasks, in the task's configuration view.

Note

  • Changes in MAX or pl_nidaqmx.ini will be adopted in CANoe after reloading the configuration.
  • Depending on the used NI hardware some settings in the MAX may not be available / allowed. You can start all created tasks in NI MAX once to verify the the MAX configuration. Please keep in mind that you have to stop all tasks in NI MAX before you can use them in CANoe. As a precaution you should close the MAX before using NI hardware in CANoe.

Task Timing Settings

The task timing types that can be set in the Measurement and Automation Explorer are described below. The supported acquisition and output types are specified for each task timing type.

Note

A particular task timing type should be selected in accordance with the following criteria:

  • 1 Sample, also referred to as polling, should only be selected if the I/O hardware used does not support any other operating mode, as is the case with certain devices connected via USB. Generally it is more effective if the device driver provides information to CANoe, rather than to poll the data on a regular basis. It would also make good sense to use this setting if you only need the input data very rarely.
  • 1 Sample (HW timed) is recommended for acquiring signals which change sporadically. You usually have to configure the clock line separately. This timing type is used only rarely.
  • With the N Samples type, signal acquisition/generation ends after a data block has been processed. This means that, depending on the buffer size, data rate, and duration of the measurement, it may end before the CANoe measurement does. It is quite unusual to see this timing type used in CANoe.
  • Therefore, Continuous is the preferred type for continually acquiring/generating signals for the entire duration of a measurement. The block length is set under Samples to be read. Higher values do deliver better performance, but also mean longer latency times between the creation and transmission of a signal value. The frequency of data transfer depends on the buffer depth and the data rate (or the rate at which the input signal changes).
    Notice: Set the buffer size so that each input signal is updated once per millisecond, approximately. This would equate to a buffer depth of 5 samples for a digital input which changes at a rate of up to 5 kHz. If you measure more than 4 or 5 such inputs simultaneously, it is advisable to further increase the buffer depth. This is particularly necessary if significant busloads are present at the network interfaces (CAN, LIN, MOST, FlexRay) at the same time.

Information About Acquiring And Outputting PWM Signals

In NI terminology, the acquisition and output of PWM signals is referred to as semi-period measurement and pulse train generation respectively.

When measuring PWM input signals, the NI driver writes the widths of the active and inactive half pulses to the buffer as floating-point numbers. Therefore, it is obvious that the buffer sizes must be set to an even number. If you have configured an odd buffer size, CANoe will increase it by one sample.

When acquiring analog and digital inputs and outputs, the port provided in CANoe corresponds exactly to a channel configured in MAX. When acquiring PWM input signals, not only the configured port containing a continuous sequence of the half-period samples is available, but also other virtual ports:

  • Value: the current value of the active half-period (DutyCycle)
  • Period: the current value of the period
  • Ratio: the current value of the active half-period (DutyCycle) as a ratio of between 0 and 1
  • RatioPercent: the current value of the active half-period (DutyCycle) as a ratio of between 0 and 100

The Raw port corresponds to the channel configured in MAX. Only the Value and Period virtual ports are generated as inputs by default. You can configure the virtual ports to be used by configuring the task accordingly in the pl_nidaqmx.INI configuration file.

The logical active half-period (high or low) can also be selected via the configuration file (polarity).

PWM signals are usually output by writing the active half-period (DutyCycle) in seconds. The half-period can be output in seconds, but if an appropriate setting is made in the configuration file, it can also be output as a ratio (Ratio or RatioPercent). The default setting is to output it as a time unit in seconds. Additionally you can change the frequency of the PWM signal. Please note the restriction for the NI daqmx driver that the output signal may only be changed once during a period. This means that you cannot change both frequency and pulse-width modulation within a single period. In this case the pulse-width modulation/the frequency can be changed after a waiting time that has to be greater than the period of the signal. Otherwise the changes are not assumed.

Further Settings: File pl_nidaqmx.ini

Using the pl_nidaqmx.ini file you can perform further settings that are not available via MAX. This file is in the Application Data directory of your CANoe installation. As a rule, you do not need to edit this file.

Note

  • When using distributed mode (VT6000, RT Rack,â¦), changes on the pl_nidaqmx.ini file have to be done on the RT server.
  • During a CANoe software update the pl_nidaqmx.ini file is transferred from the user computer in distributed mode to the RT server (and thereby overwrites an existing pl_nidaqmx.ini).
    If you change the pl_nidaqmx.ini on the user computer in distributed mode and start a software update afterwards, the updated file will be available on the RT server, too.

Possible values are:

  • 1: The most recent buffer value is available as a scalar input signal.

TaskControl=0

This setting enables you to switch on/off tasks dynamically during measurement. This option is only available for tasks that run in event or stream mode. When the option is active a system variable is generated with which you can switch on/off the task on runtime.

Possible values are:

  • 0 (default)
    Task control inactive.
  • 1
    Task control active.
    Task will be started automatically on measurement start.
  • 2
    Task control active.
    Task will not be started automatically on measurement start.

OwnThreadMode=0

With this option an own thread can be generated for tasks with request mode 1 value per request. This option can be necessary for USB devices to achieve a better performance in CANoe.

Possible values are:

  • 0 (default)
    Automatic mode.
    Threads are generated automatically when needed (for example of USB devices).
  • 1
    Inactive.
    No thread is generated for this task.
  • 2
    Active.
    A thread is generated for this task.

ThreadCycleTime=20

Cycle time in milliseconds for generated thread with the OwnThreadMode option.

UseSimulatedDevice=0

With this option simulated devices can be used in CANoe.

  • 0 (default)
    Simulated devices will not be used in CANoe.
  • 1
    Simulated devices will be used in CANoe.

ReserveTaskResources=0

Reserves the resources for the particular task before measurement start, which is needed under certain circumstances. A possible use case could be if more than one angular measurement is used with an NI 9401.

  • 0 (default)
    Resources are not reserved.
  • 1
    Resources are reserved before measurement start.
