Import

On tab Import you can add, edit, and remove Functional Mock-up Units (FMUs) for the configuration. CANoe accesses the variables of the FMU data model using system variables. As a result, CANoe can control the data exchange with the FMU during a simulation.

Toolbar

  • With the Add FMU button, you add one or more FMUs.
  • With the Remove FMU button, you remove previously selected FMUs from the configuration.

Configured FMUs

All FMUs of the current configuration are listed in the area on the left. General information and configurable settings of the selected FMU are displayed on the right.

General Settings

  • Step size [s]
    In this input box, you specify the duration (in seconds) of a calculation step of the FMU that will be called periodically during the simulation.
  • Activate debug outputs
    With this check box, you activate or deactivate the output of debug information during the simulation.

General Information

  • File
    This field shows the path to the file containing the FMU packet.
  • Interface version
    This field shows the FMI version of the FMU.
    Versions 1.0 and 2.0 are supported.
  • FMU type
    This field shows the type of the FMU. Only Co-Simulation is supported at present.
  • GUID
    This field shows the GUID of the FMU.

Model Variables

Select the variables you want to use for the simulation. The table groups the model variables according to FMI type (Output, Local, Parameter, Input).

The table contains the following columns (properties):

  • Model Variable
    The variable name within the FMU.
  • CANoe System Variable
    The system variable that synchronizes CANoe with the model variable.
  • Data Type
    The FMI data type of the model variable.
  • Active
    Use this check box to select whether the variable is to be considered during the simulation.
    Note: CANoe generates a system variable only when the option is activated.

Buttons

  • Select None
    All check boxes in the Active column get deactivated.
  • Select All
    All check boxes in the Active column get deactivated.

Export

You can configure a Functional Mock-up Unit (FMU) based on the loaded CANoe configuration (i.e. existing system variables and signals) on tab page Export. Afterwards, the FMU can be exported as a file which contains the model description and required libraries for data exchange.

Toolbar

  • With Export FMU you can export the configured FMU to a file.
  • The Merge FMU button can be used to add the input and output variables of a previously exported FMU to the current export configuration. Referenced signals or variables that are not available in the current configuration are marked accordingly. Other export settings, such as the FMU type or the interface version, are not taken over from the FMU.

Note

An FMU can only be read in if it has been exported via the export dialog. An arbitrary FMU typically does not have the required internal format.

Export Settings

  • Identifier
    In this field, you specify the name for the FMU.
    Please note: this is only required for interface version 2.0 as version 1.0 uses the predefined name CANoeFMIv1. This cannot be modified.
  • Description
    Use this field to add descriptive information.
  • Interface version
    With this control you select the FMI version of the FMU.
    Versions 1.0 and 2.0 are supported.
  • FMU type
    With this control you define the type of the FMU.
    Co-Simulation and Model Exchange are supported.
  • Step size [s]
    In this input box, you specify the duration (in seconds) of a calculation step of the FMU that will be called periodically during the simulation.
  • FMI port
    In this input box, you specify the TCP port to be used by the FMU client for communication with the FMU.
    The default is port number 4660.
  • Decoupled time base
    During measurement it is attempted to synchronize the time between CANoe and the FMU consumer, whereby the FMU consumer might be delayed. Use this option to deactivate time synchronization if necessary.
  • Separate FDX file
    If the exported FMU is initialized in FMU Consumer, FDX communication is established automatically between FMU and CANoe. This usually takes place outside of the measurement, and measurement is then started via FMU. However, if the FMU is not supposed to be initialized in FMU Consumer until a measurement is already running in CANoe, then in order to establish the FDX communication, it is necessary for the FDX description to already exist in CANoe. This can be achieved with the Separate FDX File check box.
    If this check box is activated, an FDX description file is also generated during the FMU export in addition to the FMU file. This FDX file is automatically added to the current CANoe configuration if it is not already included in it. Please note that in the process the configuration may be modified but not automatically saved.

Export Symbol Table

Add the symbols (variables and signals) to be provided by the exported FMU for simulation.

The table contains the following columns (properties):

  • Activate/Deactivate check box
    Only activated symbols are considered for export.
  • Variable/Signal
    The system variable/signal name within the FMU.
    You can add a symbol by entering a name (auto-completion looks for available Names), selecting from the dropdown control in the field or using drag and drop from the Symbol Explorer control integrated in the dialog.
    Please note: symbols of type Struct, Data, Integer Array, Float Array and 64 Bit Integer (signed and unsigned) as well as variables used for analysis only are not supported. Nevertheless it is possible to add single struct members and array elements.
  • Category
    With this selection control, you can define the data exchange type of the symbol.
    Read-only symbols only support categories Local and Output, writable symbols Input and Parameter additionally.
  • Data Type
    Shows the data type of the variable/signal.
  • Byte Size
    Displays the byte size of the symbol.
    For String symbols you can modify this value.

Symbol Explorer

Use this integrated tool to drag and drop symbols into the symbol table. It is especially required for adding single array elements.
