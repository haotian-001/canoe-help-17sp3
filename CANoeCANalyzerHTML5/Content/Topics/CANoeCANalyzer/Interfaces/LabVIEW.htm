LabVIEW Integration

Ribbon: Environment Tab|Tool Couplings|LabVIEW Integration

Note

You must have the LabWindows CVI Runtime installed on your computer in order to use the CANoe LabVIEW integration. This runtime can be downloaded from the National Instruments website free of charge.

Enabling the LabVIEW Integration

To enable the LabVIEW integration for CANoe, open the CANoe options dialog via Options dialog. Go to page External Programs|LabVIEW and activate the LabVIEW integration. Now a new command Environment ribbon tab|LabVIEW Configuration is available.

Troubleshooting

Note

If you receive the error notification Could not load library DLL, an update of the NI DAQ mx driver might solve the problem.

Configuration

In this dialog you configure the LabVIEW integration for CANoe. With this integration the shared network variables provided by LabVIEW can be used as system variables in CANoe.

This is how to realize data exchange between LabVIEW and CANoe:

  1. To get started enter the IP address of the computer running LabVIEW into the corresponding input box.
    If LabVIEW is running on the same computer as CANoe, use the IP address 127.0.0.1.
  2. Press [Scan for variables] to establish a connection to LabVIEW and request all available shared network variables.
    The variables will then be listed in the table below.
  3. In the table, select all variables that you would like to use in CANoe.
    For each selected variable a system variable will be created automatically when the dialog is closed.
  4. While the namespace of the variable is fixed, you may change the name of the variable under which it will appear in CANoe.

After closing the dialog, all selected variables will be created automatically.
You can then read and write these system variables during the measurement to communicate with LabVIEW.

Distributed Mode and Standalone Mode

The LabVIEW integration can be used in a single computer setup as well as in CANoe's distributed mode and standalone mode. In such a scenario the IP address of the computer running LabVIEW might alter between the time you create the CANoe configuration and the performed measurement.

To cover this case the configuration dialog of the LabVIEW integration provides the possibility to enter a second IP address. While the first address is used when scanning for variables, the second address is used to communicate with LabVIEW during the measurement.

Note

The connection to LabVIEW works with all VN8900 devices and VT modules that have support for NI GPIB. These are: VN8911, VN8914, VT6051, VT6051A, VT6011, VT6060 and VT6020.

Supported Data Types

The following data types are supported:

  • Signed Integer (8 bit, 16 bit, 32 bit and 64 bit)
  • Unsigned Integer (8 bit, 16 bit, 32 bit and 64 bit)
  • Float (Single and Double Precision)
  • Array (from the basic data types mentioned above)
  • String

Performance

Latency and Throughput

Under optimal conditions (i.e. low CPU and network load) the following values for latency and throughput are to be expected:

Latency
The latency for the transmission of a single value (double, 8 bytes) between CANoe and LabVIEW is approx. 1 ms. This is true for both directions, i.e. from CANoe to LabVIEW as well as from LabVIEW to CANoe.

Throughput

Transmitting 100,000 samples (integer, 4 bytes) per second causes a network load of approx. 34 Mbit/s. Thus e.g. 100 variables can be transmitted in a 1 ms raster on a 100 Mbit network.

Please consider the following hints to maximize the performance of the data transmission between CANoe and LabVIEW:

  • Variables that are being sent in a loop should be initialized and closed outside the loop:
  • To read or write multiple variables a serial structure should be used instead of a parallel one. Create such a structure by connecting the error inputs and outputs of the blocks:
  • When the same value is sent multiple times from LabVIEW, the transmission rate is reduced to 10 ms. Values that change continuously are not affected by this mechanism.

COM Access to CANoe

By using the ActiveX library that is included in LabVIEW, CANoe can be activated and addressed via the COM interface. Therefore the COM Interface ensures that all objects (and their methods, properties and events) are available LabVIEW.
