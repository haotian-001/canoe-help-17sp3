Configuration

Ribbon: Hardware Tab|Vector I/O|IOpiggy

The Vector I/O Configuration dialog is used to configure the IOpiggy.

Connector Overview

A standard configuration is used for the IOpiggy in this case. All measurement values are captured cyclically, using a fixed cycle time. On the Acquisition page you can edit the cycle time.

Analog Trigger (only in Custom mode available)

Configuration settings for the analog trigger. The trigger source is always A0 (pin).

  • Trigger Level
    Trigger level of the analog trigger, in volts.
  • Edge
    Edge used for trigger activation.
    You can select from Rising, Falling or Both.

Digital Trigger (only in Custom mode available)

Configuration settings for the digital trigger.

  • Trigger source
    Specifies the source for the digital trigger.
    You can select several trigger sources here.
    The trigger is activated when the corresponding edge is detected on at least one of the selected trigger sources.
  • Edge
    Edge used for trigger activation.
    You can select from Rising, Falling or Both.

PWM

PWM Output

Parameters for PWM output. The PWM signal is output as soon as measurement starts. You can still modify the frequency and duty cycle during the measurement using the corresponding system variables.

  • Frequency
    Frequency in Hz for the PWM output directly after the measurement starts.
  • Duty cycle
    Duty cycle in % for the PWM output directly after the measurement starts.

PWM Capture (only in Custom mode available)

  • Timeout
    Timeout for the PWM measurement.
    If an edge is not detected during the specified time period, the system variables for the PWM measurements are reset.

Digital Levels

Input Thresholds

Threshold: Trigger threshold in volts to differentiate between High and Low.

Output Levels

High Level: Output voltage in volts for the High Level.

Holding Time (only in Custom mode available)

You can use the Output(hold) function to ensure that the output value for a High or Low value does not fall below a minimum hold time, even if the system variable changes more quickly than this. This function is only available for digital output at pin 9.
