Diagnostics

Symbol

Description

Configuration group

Diagnostics/ISO TP Configuration

In this dialog you can add diagnostic descriptions for control units (ECUs).

With a click on the symbol you open the Diagnostics/ISO TP Configuration dialog.

Basic Diagnostic Editor

Using the Basic Diagnostic Editor, you can describe simple diagnostic services (UDS & KWP) for control units on CAN, LIN, FlexRay and K-Line.

You can add and remove basic diagnostic ECUs in the Diagnostics/ISO TP Configuration dialog.

With a click on the symbol you open the Basic Diagnostic Editor for a configured basic diagnostic ECU.

Diagnostic Parameters Window

You can add and remove diagnostic descriptions for ECUs in the Diagnostics/ISO TP Configuration dialog.

Here you open the Diagnostic Parameters Window, to configure automatic diagnostic requests to ECUs for that you have added diagnostic descriptions in the configuration and to display parameter values from the corresponding diagnostic responses.

Control group

Diagnostic Console

The Diagnostic Console makes it possible to send single diagnostic requests directly to an ECU and receive and analyze the corresponding diagnostic responses.

You can add and remove diagnostic descriptions for ECUs in the Diagnostics/ISO TP Configuration dialog.

Here you open the Diagnostic Consoles of the ECUs for that you have added diagnostic descriptions in the configuration.

Fault Memory Window

The fault memory list of an ECU can be read directly with the Fault Memory Window. You can also delete individual entries in the fault memory list (DTC - Diagnostic Trouble Code).

To read out the fault memory list of an ECU, the respective database must be loaded in a diagnostic description format (CDD, ODX or MDX).

You can add and remove diagnostic descriptions for ECUs in the Diagnostics/ISO TP Configuration dialog.

Here you open the Fault Memory Windows of the ECUs for that you have added diagnostic descriptions in the configuration.

Diagnostic Session Control

The Diagnostic Session Control allows to switch an ECU interactively between its different diagnostic states (changing a session, security access) as well as to control its communication management settings.

You can add and remove diagnostic descriptions for ECUs in the Diagnostics/ISO TP Configuration dialog.

Here you open the Diagnostic Session Control of the ECUs for that you have added diagnostic descriptions in the configuration.

OBD-II Window

The OBD-II Window makes it possible to execute On-Board Diagnostics (OBD) according to the SAE J1979 standard.

You can activate/deactivate the OBD-II functionality in the Diagnostics/ISO TP Configuration dialog.

Here you open the OBD-II Windows of the networks for that you have activated the OBD-II functionality.

Diagnostic Window

Here you open and configure the Diagnostic Windows with the respective diagnostic features for ECUs whose diagnostic descriptions you have added in the Diagnostics/ISO TP Configuration dialog.
The following diagnostic features are available:

  • Variant Coding
    The diagnostic feature Variant Coding you can use in the Diagnostic Window.
    The diagnostic feature Variant Coding is used to read, write and compare variant coding data of an ECU. Security aspects from possibly configured security sources are taken into account.
  • ECU Control
    The ECU Control is used to change the diagnostic session for an ECU, to enable the security access in the ECU and to authenticate the tester at the ECU. Security aspects from any configured security sources are taken into account. Furthermore, ECU resets can be executed, DTC setting services and communication services can be sent.
  • Diagnostic Console
    The Diagnostic Console is a diagnostic feature, that you can use in the Diagnostic-Window.
    In the Diagnostic Console it is possible to directly send individual diagnostic requests of an ECU, display and evaluate the corresponding diagnostic responses.

Tools group

CANdelaStudio

Using CANdelaStudio, you can view, edit and export diagnostic data of control units in the automotive industry and in related areas.

Here you can open and edit in CANdelaStudio the inserted diagnostic descriptions of your configuration.

ODXStudio

Using ODXStudio, you can easily navigate through the ODX model, just like a web browser. ODX is a complex data format used to store information relevant for diagnostics in a structured way.

With a click on the symbol you open the ODXStudio.

Test group

Import DiVa Project

DiVa enables automated, data-based creation of reproducible test cases for ECU-specific software with broad and detailed test coverage. Test specifications are automatically generated utilizing the prescribed diagnostic specification files as well as test modules and test environments for CANoe.

Here you import DiVa projects in CANoe.

J1939 group

J1939 DTC Monitor

With the J1939 DTC Monitor in the Measurement Setup you can easily display and request error codes (DTC â Diagnostic Trouble Codes) and diagnostic messages. With the support of Freeze Frames (extended information from fault memory) you can use the J1939 DTC Monitor for emission related or power train diagnostics.

Here you open J1939 DTC Monitors added in the Measurement Setup.

Additionally you can insert J1939 DTC Monitors in the Measurement Setup.

J1939 Diagnostic Memory Access

With the J1939 diagnostic memory access window you can read and write parts of the memory as well as execute a boot load procedure. For this the diagnostic messages DM14, DM15, DM16 and DM18 are used.

Here you open J1939 Diagnostic Memory Access Windows added in the Measurement Setup.

Additionally you can insert J1939 Diagnostic Memory Access Windows in the Measurement Setup.

J1939 OBD-I/M Monitor

The J1939 On-Board Diagnostic Inspection and Maintenance Monitor (short: J1939 OBD-I/M Monitor) allows to configure diagnostic messages of type DM7 and send them to ECUs in the network.

Here you open J1939 OBD-I/M Monitors added in the Measurement Setup.

Additionally you can insert J1939 OBD-I/M Monitors in the Measurement Setup.

J1587 group

J1587 DC Monitor

The J1587 Diagnostic (DC) Monitor examines transmitted diagnostic messages (PIDs 194â¦196), displays their contents, and enables requests in order to retrieve diagnostic data.

Here you open J1587 DC Monitors added in the Measurement Setup.

Additionally you can insert J1587 DC Monitors in the Measurement Setup.

XCP/CCP group

XCP/CCP Window


  • Add ECU to the configuration.
    Select an A2L file or XCP device configuration file (XCPCFG) for this.

  • Import AMD Configuration
    You can import AMD configurations to enable easier use of AMD modules of the MICROSAR Classic Stack in CANoe.
    AMD configurations add, for example, CAPL test programs, A2L files, and Trace Windows to existing configurations.

With a click on the upper button you can open the XCP/CCP Window.

Via the lower button you can select an A2L-/XCPCFG file and import an AMD configuration.

Split Button

You can recognize a split button by the split functionality displayed on the left.

You open the corresponding window using the Symbol. Other functions are available via the Text.

Please note the provided information since in some cases the other functions are only available under certain preconditions.

ASAP2 Viewer

Using the ASAP2 Viewer you can create, modify and update controller description files in the ASAP2 format.

With a click on the symbol you open the ASAP2 Viewer.

ASAP2 Updater

Using ASAP2 Updater you can update the address and data type information in an ASAP2 file based on the entries of a Linker MAP file.

With a click on the symbol you open the ASAP2 Updater.
