Global Logging Comment

Ribbon: File Tab|Options|Windows/Blocks|Logging|Logging Comment|Set Current Comment...

The Global Logging Comment dialog (Options dialog) can be used to specify the logging comment. Settings that you make here apply to all logging blocks that have not been explicitly configured differently.

For information on local settings for an individual logging block, see the Logging Comment help page.

Logging comments can consist of several attributes. The set of these attributes is the same for each logging block and can only be set in the Global Logging Comment dialog.

Toolbar

Symbol

Function

Adds a new attribute to the comment. This functionality is only available outside the measurement.

Deletes the currently selected attribute. This functionality is only available outside the measurement.

Moves the currently selected attribute up. This functionality is only available outside the measurement.

Moves the currently selected attribute down. This functionality is only available outside the measurement.

Field Codes

Inserts a field code into the currently selected input field. Field codes can be used to generate parts of the comment automatically.

The values are determined each time the logging file is closed and inserted into the comment text.

In the preview fields you get a preview of the comment text with resolved field codes.

The Logging Comment Field Codes page contains a detailed list of the available field codes.

Columns

Entry Field

Description

Attribute

The name of the attribute.

This value can only be changed outside the measurement.

Default Value

The default value written by this logging block.

This value is persisted when the configuration is saved.

The default value can only be changed outside the measurement.

Current Value

The value for the current or next measurement.

This value is not persisted and will be reset to the default value at the end of the measurement.

The default value can be taken over as the current value by pressing the button .
