Configuration: Map Window

Ribbon: File Tab|Options|Windows/Blocks|Map Window

Via Global Map Window Configuration in the toolbar of the Map Window you can open the corresponding settings dialog. Here you can, among other things, configure the map material that is used by the CANoeMap Window.

Map Window Settings

General map settings

Enable DirectX acceleration

Activate this option to increase the performance of the map display.

Note

  • The performance of the displays in the map window is also determined by the graphics card used and whether it is designed for DirectX. If you are using a less powerful graphics card and notice problems displaying content in the map window, you can disable the option DirextX acceleration. Without DirectX acceleration, however, the performance of the map display is lower than when using DirectX.

Map provider settings

Requirements for the Map Display

  • To display the map, the map provider from which the map material is to be obtained must first be configured. To do this, tiles, which each represent a square-shaped map section, are loaded from the tile server of the map provider. In addition to the two preconfigured map providers Vector (OSM) and HERE, you can also enter and save tile URLs from other providers under Custom in order to obtain the map material from their tile server. You can find a small selection of OpenStreetMap® -based tile URLs on the OpenStreetMap®website.
  • The loaded tiles are stored in a directory on the hard disk, so that new tiles only have to be loaded if necessary. To force the loading of the latest map material it is therefore necessary to click Open cache folder... and delete the tiles stored in the cache folder.
  • Please note that the respective providers may restrict your access to the tile servers if you use them too often. To prevent automatic reloading of map material, you can switch to Cached mode if you have already collected enough tiles in the cache. To avoid possible restrictions of third party providers when using CANoe, Vector Informatik GmbH offers its own tile servers starting with version 14.0, which are called with the preconfigured card provider Vector (OSM).
  • The tiles stored in the cache for OpenStreetMap® in CANoe 12.0 can no longer be used in CANoe 12.0 SP2 due to a format conversion by OpenStreetMap®. From this version on, tiles are stored in a different format.
  • An active Internet connection is required for map display with map data from the Internet. Local offline map data is required for the map display of OpenStreetMap® data without an Internet connection. Please note that this data has a different format than the cached tiles. You can also create and integrate offline map data yourself.
  • You can configure which map material is to be used. Make sure that all relevant regulations regarding the use of the map material are complied with.

Vector (OSM)

Starting with version 14, Vector Informatik GmbH provides its own tile servers. With the new map provider Vector (OSM), the map material is obtained directly from Vector's own servers. The offered map material is based on data from OpenStreetMap®.

Besides the possibility to get the map material online, there is also the option to load the map material exclusively from the cache, e.g. if you have already collected enough tiles in the cache. To do this, switch to the Cached mode to load only data from the Vector Cache and prevent access to the Vector Tile Servers.

OpenStreetMap® (offline)

This preconfigured provider is only offered from CANoe 13.0 onwards for compatibility reasons in order to be able to use previously generated offline map material from OpenStreetMap®. To do this, enter the directory containing the local map data from which the map is to be loaded.

To use online map material, you must create a custom map provider and configure it with a valid tile URL.

See also the help page Integrate Local OpenStreetMap® Map Data.

HERE

HERE is available as another preconfigured map provider. This is an online map provider that uses the standard HERE tile URL to obtain the online map material.

In some circumstances, frequent access to the HERE servers may limit your connection to the HERE map material. This will result in a slower loading of the tiles. In this case, you may need to create a new HERE account to continue to quickly download new map tiles. If you already have a HERE account, you can select your desired authentication method under Credentials. Here you have the possibility to authenticate yourself either with a single API key or with App-ID and App code. If you do not have any access data, select None as your Credentials. In this case, the tiles will be loaded without the benefits of a custom HERE account. As an alternative to a HERE account, you can also switch to another map provider. You can configure this provider under Custom.

As with Vector (OSM), HERE offers the option to load the map material only from the cache. To do this, switch to Cached mode to prevent access to the HERE servers.

Custom

In addition to the preconfigured map providers, you can also create and configure your own custom providers. A prerequisite for this is that you know the tile URL of the respective provider. Such tile URLs can often be found by a simple Google search (e.g. by searching for openstreetmap tile url). A small selection of OpenStreetMap® -based tile URLs can be found here.

For a better differentiation of the different providers, you can edit the name of the provider in addition to entering the Tile-URL. This two pieces of information are saved after confirmation with [OK] and the new provider is then also available for subsequent sessions. With Delete Selected Provider you can delete a previously saved provider. The saved providers are stored in a version-independent XML file under ... AppData/Local/Vector/MapCache/mapwindow_settings_custom.xml, so that they are retained across versions, e.g. when installing a new version.

As with Vector (OSM) and HERE, the loaded tiles of the custom providers are stored on the hard disk and only reloaded when needed. The tiles of the different providers are also stored in different directories under ... AppData/Local/Vector/MapCache/Custom ProviderName/ to prevent mixing of map material from different providers.

The name of the cache folder corresponds to the selected name of the provider (with the additional prefix Custom). Here too, you can navigate to the relevant directory by choosing Open Cache Folder....

To reuse a cache folder, you may need to create a custom provider and give it the same name (folder name without prefix). The tile URL can also be left empty if you want to use only the maps from the cache folder.

Note

When entering a Tile URL, you should make sure that it contains at least the strings {x}, {y} and {z}. These are used as placeholders for the tile numbers (X, Y) and zoom level (Z) and are mandatory for a request to the tile servers.

Example

When CANoe is newly installed, the custom provider OpenStreetMap Template is already set. This contains the template Tile-URL:

http://tile.INSERT_SERVER_NAME.org/{z}/{x}/{y}.png

You can replace INSERT_SERVER_NAME with openstreetmap, for example, to obtain online map material from OpenStreetMap®. You can also rename OpenStreetMap Template to OpenStreetMap to make it clear that it is now a different (valid) provider. Confirm with [OK] to save the new provider. All open map windows should now receive the map material from OpenStreetMap®.

You can also use Open Cache Folders... to navigate to the corresponding directory. This folder is called Custom OpenStreetMap and consists of the previously selected name OpenStreetMap and the additional prefix Custom. This cache folder can be shared and reused on another computer. To do this, a new custom provider named OpenStreetMap must be created there.

As an alternative to creating a new provider, you can also copy the mapwindow_settings_custom.xml to the appropriate directory on the new computer to adopt the providers stored in it.

Image

The Map Window offers the possibility to use own images as map. To do this, select the Image provider. Subsequently, further fields appear in the Options dialog in which you can select the Image File to be used and specify the coordinates of the corner points P1 (top left) and P3 (bottom right). Note that the image file must be stored in the folder .../AppData/Local/Vector/MapCache/Images/. If the specified data is valid, a small preview will be displayed. Finally, you can give a name to the setting made under Image Provider and save it for later use. These configurations are saved in .../AppData/Local/Vector/MapCache/mapwindow_settings_image.xml, so you can easily transfer the settings to other computers by copying the images folder as well as the XML file.

Proxy settings

Use web proxy

Activate this option to enter your credentials. Enter the text for the Server and Port in the general form Servername:Port. Then enter your username and password.
