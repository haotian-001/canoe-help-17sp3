Configuration: Logging

Ribbon: File Tab|Options|Windows/Blocks|Logging

Default Logging Format

Here you set the default logging format for writing logging files.

  • New logging files will have set this logging format as default (e.g. in the file selection dialog of the Logging block).

  • The format preset after installation and recommended by Vector is the binary logging format BLF.

  • The preset default logging format will only be used for writing files.

    Places where logging files are being read e.g. a source file for the offline mode or a Replay Block are not affected.

General Logging Settings

  • Stop measurement on logging error

    If this option is enabled, the measurement will be stopped in case of an error in logging; otherwise, only the affected logging block will be stopped.

  • Open logging file with default viewer

    If this option is enabled, the logging file is opened in the default application, which is specified in the Options dialog.

Logging Comment

Here you can specify the comment that is written to the logging file, or edit the comment of the logging file last written by this block after the measurement.

Comments are only supported for the logging formats BLF, MF4, and MDF.

  • Click on [Set Current Comment...] to open the Global Logging Comment dialog where you can set the comment for new logging files.

  • Click on [Change Last Comment...] to open the Change Global Logging Comment dialog where you can edit the comments of the last measurement.

    This button is only active if logging files supporting comments were written during the last measurement.
