Configuration: Trace Window

Ribbon: File Tab|Options|Windows/Blocks|Trace Window

The settings made in this dialog apply to all Trace Windows.

Analysis Range

The analysis range can be changed in the data history settings.

The link takes you to the options for the data history.

Digits after Decimal Point

Here you can specify the number of digits after the decimal point with which time stamps should be displayed in the Trace Window.

  • The default value is 6 digits after the decimal point.
  • The maximum number is 9 digits after the decimal point.

Furthermore you can specify the number of digits after the decimal point with which floating-point numbers should be displayed in the Trace Window.

  • The default value is 4 digits after the decimal point.
  • The maximum number is 9 digits after the decimal point.

Displayed Signal Information

Here, you can specify which signal information should be displayed in the Trace Window:

  • FlexRay signal groups
  • Signal multiplexing information
  • Signal comment
  • Signal raw value

Multiplexors

Here you can set, if in fixed position mode messages with different multiplexor values should be displayed in different rows. Ethernet PDUs

The Duplicate PDUs at different ports option allows you to specify whether messages measured at different measurement or simulation ports should be displayed in different lines in fixed mode.

Settings

With the Enable fading for new Trace Windows option, you can enable/disable the hiding of rare events or unchanged data for new Trace Windows.

You specify which unchanged data are to be hidden in the individual Trace Windows in the Trace Configuration|General dialog.

You can activate/deactivate tooltips that pop up if the text of an event cannot be shown completely in the according column of the Trace Window.

With the Automatic refresh of detail view during measurement option you can set if the detail view of the selected event in fixed position mode should be updated automatically.

Under Format of Date and Time column you can specify the display format of the Date and Time column in the Trace Window. You can choose between Time and Date and Time.

Event Highlighting

This function is used to highlight important events and messages to find them quicker in the Trace Window.

The events are arranged for each bus system.

To highlight an event in the Trace Window in color, select the respective check boxes.

You can select the desired color by clicking on the color button in the Text Color and Background Color columns.

For evaluation the selected event is displayed in the Preview field in the chosen text and background color.

The [Reset] button can be used to reset the selected event or all events of a selected group or bus system to the standard colors.

Note

  • The setting is machine dependent. When highlighting events, all Trace Windows of a configuration are affected.
  • Window-specific color settings in the respective Trace Window or highlighting events using CAPL overwrite the color settings of the global options dialog. In other words, an event configured for both window-specific and standard highlighting is displayed in the window-specific colors.
  • Window-specific event highlighting is only supported by MOST and must be set separately for each Trace Window.
