CAPL Compiler

When you create a new configuration, both check boxes are not activated by default.
When you load an old configuration (program version 3.2 and earlier), both check boxes are activated by default.
The configuration behaves automatically like in program version 3.2 (or earlier).

  • Suppress compilation to work without CAPL sources (.CAN)
    Activate this check box, if you want to relay the configuration without CAPL source files (.CAN). If activated it is not possible to compile any CAPL programs in the configuration. If databases or other components have changed, on measurement start a compilation is required. In this case the measurement start is aborted with the following message: CAPL compilation is turned off in this configuration.
    To relay a configuration that works without compiling the following conditions must be met:
  • Same CANoe version, same service pack as in the preparation of the CFG file, thus the CAPL compiler is identical.
  • Identically configured databases (their time stamps and on which networks they are configured).
  • Identically configured modeling libraries (DLL files) and their time stamps.
  • Identically defined system variables (name, type, value table; but not the value).
  • Time stamp of the source file and all include files that are referenced by it (only if the source file exists).
  • Identical execution target (computer, CAPL-on-Board, Vector Realtime Boost).
  • If CAPL debugging is switched on: Time stamp of the debug information.
  • Same network names (it may not be added new networks or channels).
  • Identical settings in Options dialog (Measurement|Performance|Threading in Measurement Setup).
  • Share equal string constants
    Normally, the compiler saves only one string for all identical string constants. If, for example, the Vector string constant is used multiple times in a program, it is nevertheless stored only once. If you deactivate this option, each string constant will be saved individually and you can manipulate a constant during runtime without affecting the other uses of the constant. Tip: It is easier if you copy the constant in the program to its own char[] buffer beforehand and then change and reuse its content.
  • Treat DWORD variables like LONG during comparison
    If the check box is activated, DWORD variables are treated during comparison like LONG variables, that means that they are signed.
    If the check box is not activated, DWORD variables are always positive (no sign) and can therefore never be less than zero.
  • Global database access
    In CANoe configurations normal nodes are assigned to a (single) bus (Gateways are assigned to more buses) and every bus has one database assigned.
    If the check box is activated, every node can read all databases that are available in the current configuration. This is important during the compilation of CAPL programs, if several databases of the configuration use the same Identifiers (IDs) or the same symbolic names.
    If the check box is not activated, every node can just access (see) the one database that is assign to its bus.
  • Use case sensitive callback search
    If this option is activated, CANoe will observe the case sensitivity during the search for the large number of callback functions in CAPL programs (e.g. OnXcpEvent or OnTcpReceive ), i.e. functions with a different notation (e.g. OnTCPReceive) will not be called as a callback.
    This option is provided for the compatibility with CANoe versions older than 13.0. Activate the option if you have several overloaded functions with different notation in one CAPL program and if one of them is expected to be the callback function.
  • Default PDU Size
    Here you define the size of the data range of a CAPL PDU object as long as it is either not defined in the database or it is a type-free PDU. If the size of a PDU can be determined using the database this size will be used for the CAPL PDU object.

Warnings and Error Handling

You can activate and deactivate various CAPL Compiler warnings on this page.
The selected settings are saved in the configuration currently loaded.

  • Click [Set Default Values] to reset all warning settings to their default values.
  • With Write Window output you determine the detail level of the warnings that are output to the Write Window:
  • None
    No compiler warnings are output to the Write Window.
  • Only warning count
    For each node with compiler warnings the number of warnings is output.
    Thereby you'll easily find out the nodes with warnings after compiling all nodes.
  • Full warning output
    All warnings are output completely to the Write Window.
