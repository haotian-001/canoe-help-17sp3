.NET Debugging

.NET Debugger

In this list you can select an internal debugger or a Visual Studio debugger. In the list all Visual Studio versions that are supported from CANoe as .NET Debugger are displayed. The selected Visual Studio must be installed on your computer.

If you have selected a Visual Studio debugger, the following options are available:

Load required .NET sources automatically

The .NET program will be loaded automatically in the selected Debugger.

You can open the Debugger via

  • the context menu command Debug... of a test module in the Simulation Setup
  • the context menu command Debug... of a test module in the Test Setup
  • in the test execution dialog

If a test module references only one .NET assembly, the source files should be in the same folder as the assembly. The debugger will not start if this is not the case.

If the folder structure can't be arranged as described above, you will have to deactivate this option. In this case, you will need to manually load the source files in Visual Studio at the start; they will remain open during the rest of the debugging process.

Start test module after attaching Debugger

The test module will be started automatically, after loading into the selected Debugger. If you deactivate this setting, the debugger will start but the test module will not be executed. This can be useful e.g. if you want to insert pauses in the .NET source file before running the test module.
