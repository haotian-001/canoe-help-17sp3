Configuration: AUTOSAR Database

Ribbon: File Tab|Options|General|AUTOSAR Database

On this page you can configure the signal and signal group names with AUTOSAR databases.

Used Language Attributes – FIBEX/AUTOSAR

Localization

With this setting the localized display of database symbols – such as long names, comments, etc. – is configured and activated. If the selected language is not available for any database symbol, then the first available variant will be displayed.

Display Signal and Signal Group Names – AUTOSAR

With I-Signal (default setting) or System Signal you can set from where in the network description file the signal and signal groups get their symbolic names.

With the symbolic name signals and signal groups can be qualified and configured in all analysis windows, CAPL programs, modeling libraries,…

The selected option

  • will be effective after reloading the configuration

  • is independent of the configuration file and is stored on the user computer

Note for Name Changes

If the name of I-Signal and System Signal differ, after a modification the already configured signal will loose the assignment to the database. The consequences are:

  • already existing CAPL programs cannot be compiled and must be adapted

  • the affected signals must be configured again in the analysis windows

Control handling of SOME/IP PDU without wervice context – AUTOSAR

If you activate this setting, so-called "dummy" services are automatically created and assigned to PDUs for which a SOME/IP transformation has been specified in the database but which are not assigned to a SOME/IP service.

Control handling of overlapping NM FlexRay Frames - AUTOSAR

FlexRay network management frames whose slot cycles overlap are split into several frames when this option is activated. If necessary, new frames/PDUs are created and optimally distributed to the free slot cycles found in the schedule table.
