Configuration: Save/Open

Ribbon: File Tab|Options|General|Save/Open

Configuration

If the When saving the configuration save a backup of the old configuration is activated, whenever the configuration is saved a backup copy is made with the filename extension BAK. No backup copy is made with the Save as function.

Save Configuration Additionally in Version

Here, you can specify whether an older version of the configuration is to be saved in addition to the current version whenever a configuration is saved.

If you select this option, you can then select the desired configuration version.

Note

The saving of an older version occurs automatically as soon as the current configuration is saved. Existing files are overwritten without any prompt for confirmation.

Besides the configuration, existing test setups and old panels are also saved in the selected old version. The name assignment follows the same pattern as during configuration.

Storage Location

The older configuration is stored in the same directory as the configuration in the current format. If the filename or storage location of the configuration in the current format is changed, the name and the storage location of the configuration in the older format are also changed.

Name Assignment

The file that contains the older version of the configuration is named as follows: Configurationname_verXX.CFG.

  • Configurationname: Name of the configuration file in the current format.
  • _verXX:Version number (e.g. _ver70 for version 7.0)

Auto Save Settings

Automatically save recovery information every XX minutes
With this input box you can specify the time interval to check the actual configuration for unsaved changes. If there are unsaved changes at that points in time the application will save recovery information for you. This recovery information can be used to restore your configuration in case of a operating system shutdown or a tool malfunction. Values between 1 and 120 minutes are possible. The default is 10 minutes.

If the Automatically save recovery information during measurement check box is activated, the application also saves recovery information during a running measurement. Activating this option is not recommended, cause it may have an performance impact for the running measurement.

Directory for recovery information:
Her you can specify the location where the recovery information shall be saved. The default is your windows temporary folder ("%TEMP%). The default path can be restored by deleting the content of the edit box.

Note

This configured path is stored in the can.INI [System] section under the key AutoSaveDirectory and can be influenced there. If the key does not exist, the default windows temporary folder ("%TEMP%) will be used.

File Associations

You can specify whether configuration files (CFG) should always be opened with CANoe here.

Activate Open configuration files (CFG) with CANoe/CANalyzer or CANoe4SW to link the data type for configuration files (CFG) to CANoe/CANalyzer; you need administrator rights for this operation.

Note

  • The option differentiates between CANoe and CANalyzer configurations and starts the most recent version of the appropriate application in each case.
  • The file type dialog under Windows shows the shortcut of CFG with the CANtools Version Selector.

FAQ

Possible causes for the CFG extension not being associated with the CANoe program may be:

  • The association of CFG files with the CANtools Version Selector is only carried out successfully during installation if the CFG extension was not already associated with a different program beforehand.
  • After installation, the association can be set in CANoe under Options|General|Save/Open. It is not necessary to retract the previous association; the current program association is overwritten.
  • When Microsoft Office or one of its service packs is installed, it can happen that the CFG extension is once again associated with the Microsoft Office application. To reset this association, activate the Option in CANoe again.
  • If the association is not created automatically you can also use the CANtoolsLauncher.EXE tool under the C:\Program\Common Files\Vector directory. Call the function CANtoolsLauncher.EXE -regserver in the command window to create the association between CFG extension and CANoe program. The association can be removed using -unregserver command.
  • Setting up the association manually in the Windows Explorer
