Configuration: MCD A2L Files

Ribbon: File Tab|Options|External Programs|MCD A2L Files

New Technology available!

Merely due to compatibility reasons ASAM MCD3 via CANape is currently still available.

Please use CANoe option .AMD/XCP instead.

A2L Files

In the bottom line you can add new modules. With you can remove single rows. You can edit the rows to change the contents.

Note

Currently mixed configurations of CCP and XCP modules are not possible.

In principle there are two different procedures:

1. Standard procedure: Connect existing CANape project (CANape.INI)

  • Module Name
    Must be unique and conform to the device name in the CANape project.
    This name is used to reference the module in CAPL code.
  • A2L File
    Configure the same A2L file as configured in the CANape project.
  • CANape Channel
    "Default" setting.
  • CANape Working Directory
    The directory must contain the CANape.INI file.
    Seed & Key DLLs must be placed in this directory, if needed.

2. Generic approach (CAN only): Connect to a logical CAN channel

  • Module Name
    A freely definable unique name for the current module.
    This name is used to reference the module in the CAPL code.
  • A2L File
    A freely definable A2L file.
  • CANape Channel
    CAN1…8 setting.
    The channel refers to the logical CANape CAN channel.
    Use the Vector Hardware Configuration dialog to assign a channel to a certain hardware.
  • CANape Working Directory
    Seed & Key DLLs must be placed in this directory, if needed.
    On first start a generic CANape.INI file will be created.
