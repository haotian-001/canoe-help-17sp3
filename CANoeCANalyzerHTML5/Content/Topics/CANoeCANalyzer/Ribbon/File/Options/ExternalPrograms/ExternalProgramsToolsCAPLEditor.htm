Editing CAPL Code with External Editors

Ribbon: File Tab|Options|External Programs|Tools|CAPL File Editor

If you enter in CAPL file editor a Visual Basic script (*.vbs), CANoe runs the script instead of starting the editor.

Passed Parameters:

The name of the Browser project file is passed as a parameter. This file contains all the settings required to open or compile the CAPL program. The name of the CAPL program to open is contained in the setting

[Command1]
File=<FileName>

To compile the CAPL program, call up the compiler CAPLcompExec.EXE.
CAPLcompExec.EXE accepts the following parameters:

  • Name of the Browser project file (/f<name>).
  • Name of a logging file (/p<name>).

If this parameter is not passed the CAPL compiler messages are written to the standard output.

  • Output format for messages (/t<format>).

By default the messages are output in CAPL Browser format.
Format: <Function name> (<Line>,<Column>),Message.

If msdev or cw32 is used as the format parameter, the messages are written in this format: <Filename>(<Line>) : Message
