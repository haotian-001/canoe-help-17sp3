GL Logger

Ribbon: File Tab|Options|Extensions|GL Logger

GL Logger Mode

Activating GL Logger Mode

Setting Network Parameters
  • If a logger is to be used as interface hardware, you must activate GL logger mode in Options dialog (Extensions|GL Logger).
  • In GL logger mode, you can use a logger as network interface only. A mixed operation with other interfaces is not possible.
  1. Open the GL Logger Configuration (Hardware ribbon tab|Network Hardware)
  2. Set the IP Address and the Port of the logger.
  3. Click Check Connection.
  • If the logger is detected, its name and a corresponding photograph are displayed.
  • If a connection to the logger cannot be established, check the network connection, the IP address, and the port.

Options

Mode

Short Description

Supported Hardware

Monitoring Mode

In Monitoring Mode, the logger transfers all incoming data such as messages and Error Frames. During the measurement, the logger does not record any data in the ring buffer. Classifications are also continued during the measurement.

GL2000, GL3000, GL4000, GL5000

Signal Sampling Mode

In Signal Sampling Mode, the logger transfers sampled signal values of predefined signals. Each signal is mapped onto a system variable that is created in the GLLogger namespace. During the measurement, the logger continues to record data in the ring buffer.

GL3000, GL4000, GL5000

Configuration of CANoe

Configuration of the Logger
  1. Create a new configuration with the help of a template.
  2. Select the GL logger mode (see above).
  3. Set the number of CAN, LIN, FlexRay, and MOST150 channels.
  4. If you want to use databases, you must create networks in the Simulation Setup and assign the databases to them.
  5. Open the network hardware configuration dialog and set the IP address and the port of the logger.
  6. Click Check connection.
    If the logger is detected, its configuration data are listed.
  1. Open the Vector Logger Suite.
  2. Create a new project or open an existing project via File ribbon tab (Create Configuration/Load Configuration) and add interface functionality to it.
  3. Activate interface mode on the Hardware|Interface Mode page.
  4. Set the IP address of the logger and adapt the subnet mask and port, if necessary. Ensure that the settings match your computer or network.
  5. Optionally, you can change the channel and the CAN ID of the marker message that indicates the start and end of interface mode in the logging file.
  6. If necessary, you can specify in the Transmit Behavior area which messages sent by the logger are to be displayed in CANoe. Likewise, you can specify whether Error Frames are to be displayed.
  7. Download the configuration to the logger (see Vector Logger Suite manual).

More Information
  • The CAN, LIN, FlexRay, and MOST150 bus systems are supported. The licenses necessary for CANoe can be obtained from an additionally inserted interface card or a license dongle.
  • Sending of messages is not possible. For this reason, the Simulation Setup is deactivated. Messages sent from the logger configuration (both real and virtual) continue to be sent after the start of the measurement. They are displayed as Tx messages if they were correctly configured in the logger.
  • The use of databases is possible. The data received from the logger can thus be symbolically visualized and analyzed in CAPL blocks.
  • A mixed operation of loggers and other network interfaces is not possible.

Signal Sampling Mode

Configuration of CANoe

Configuration of the Logger
  1. Create a new configuration with the help of a template.
  2. Activate the GL logger mode (see above) and select Signal Sampling Mode.
  3. Open the GL Logger Configuration (Hardware ribbon tab|Network Hardware) (see above).
  4. Select one of the two operating modes:
    a) Dynamic creation of system variables
    Here, a system variable is created in the GLLogger namespace at the start of the measurement for each signal supplied by the logger. The system variables are only available during the measurement. This operating mode is suitable for calling all existing information of the logger without additional effort.
    b) Use of selected signals
    Here, you can select certain signals from a list of available signals for which corresponding system variables are created in the GLLogger namespace when the dialog is exited. These can be used in the configuration and saved with it.
  5. After the start of the measurement, current signal values are assigned cyclically to the system variables in the GLLogger namespace.
  1. Open the Vector Logger Suite.
  2. Create a new project or open an existing project via File ribbon tab (Create Configuration/Load Configuration) and add interface functionality to it.
  3. Activate the Signal Sampling Mode.
  4. Set the IP address of the logger and adapt the subnet mask and port, if necessary. Ensure that the settings match your computer or network.
  5. Select the signals that the logger is to make available later during operation.
  6. Download the configuration to the logger (see Vector Logger Suite manual).

More Information
  • Because the current signal values are transferred cyclically from the logger, it is possible that not all value changes will be displayed by the system variables. This is the case if a signal changes its value at time intervals that are shorter than the sampling cycle time.
