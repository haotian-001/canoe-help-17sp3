Configuration: Connectivity

Ribbon: File Tab|Options|Extensions|Connectivity

Services

In this area you control the connectivity services. For each Service you select a Mode. After that you can make settings via the button in the Configuration column.

  • MQTT

    By activating this check box, you instruct CANoe to use an MQTT broker in one of the following modes, which you can select via List in the Mode column:

  • Connectivity Hub mode

    Activates the MQTT broker provided by the Connectivity Hub. An Owner Certificate is required.

    Configuration

    Proxy Settings

    • None

      No proxy is used.

    • Default

      The system proxy is used.

    • User-Settings

      The user-defined proxy is used.

    Certificate Settings

    Here a Owner Certificate should be selected. If none has been imported yet, you can do this via the Import button. In addition you can delete or export already existing owner certificates via the corresponding buttons.

    Instance Settings

    In this area you can add instances you want to use for measurement. The Add button sends a request to the Connectivity Hub, which adds an instance. Normally it is sufficient to work with one instance. You can also enter a comment for an instance, delete or export it.

    Note

    Exporting an instance gives you a SUT Certificate, that you can use to authenticate a System Under Test (SUT) to the Connectivity Hub.

    Webhook Token

    If you need a webhook token in form of a JSON web token for authentication with the Vector Connectivity Hub, you can generate one here.

    Note

    Tokens are valid for 30 days and have to be send as a bearer token with the HTTP request.

  • CANoeHosted mode

    Activates the local MQTT broker provided by CANoe.

    Configuration

    • Port

      Input box for the port,the CANoe hosted broker should be started on.

    • Verbosity Level

      Via this list (None, All, MQTT) you define, how much monitoring/status information will be displayed by the CANoe hosted broker.

    • Publish::MaxAttemptsToResend

      Activate this check box, to define via the input box the number of attempted resends on a PUBLISH for the CANoe hosted broker.

    • Publish::ResendTime [s]

      Via this input box you define the time after which a PUBLISH is resend in seconds for the CANoe hosted broker.

  • WAN mode

    Instrucs CANoe to use an external MQTT broker.

    Configuration

    • Connection

      Via the URI input box you define the URI of the broker running on the local computer or in the network.

    • Local Broker

    • Start broker at the start of measurement

      Activate this check box, to use a local MQTT broker and to start it at the start of the measurement.

    • Path

      Via this input box you define the path to the local MQTT Brokers executable. Additional commands can be added to the path, if necessary.

  • HTTP

    By activating this check box you execute an asynchronous HTTP requests.

  • Connectivity Interface mode

    Configuration

    Proxy Settings

    • None

      No proxy is used.

    • Default

      The system proxy is used.

    • User-Settings

      The user-defined proxy is used.

  • Connectivity Hub

    By activating this check box you instruct CANoe to activate the Connectivity Hub Service, that gives you the ability to use .e.g. Webhooks in CANoe.

  • Connectivity Hub mode

    Configuration

    Proxy Settings

    • None

      No proxy is used.

    • Default

      The system proxy is used.

    • User-Settings

      The user-defined proxy is used.

    Certificate Settings

    Here a Owner Certificate should be selected. If none has been imported yet, you can do this via the Import button. In addition you can delete or export already existing owner certificates via the corresponding buttons.

    Instance Settings

    In this area you can add instances you want to use for measurement. The Add button sends a request to the Connectivity Hub, which adds an instance. Normally it is sufficient to work with one instance. You can also enter a comment for an instance, delete or export it.

    Note

    Exporting an instance gives you a SUT Certificate, that you can use to authenticate a System Under Test (SUT) to the Connectivity Hub.

    Webhook Token

    If you need a webhook token in form of a JSON web token for authentication with the Vector Connectivity Hub, you can generate one here.

    Note

    Tokens are valid for 30 days and have to be send as a bearer token with the HTTP request.

User Proxy Settings

In this area you can set user-defined proxy settings. If you select this option, you must specify a URI in the URI input box and, if necessary, a certificate.

Note

To use the User-Proxy-Settings the proxy has to be selected in the individual Service Configuration, if applicable.

Note

The proxy access data is persistent and is retained both when CANoe is restarted and when the configuration is switched.
