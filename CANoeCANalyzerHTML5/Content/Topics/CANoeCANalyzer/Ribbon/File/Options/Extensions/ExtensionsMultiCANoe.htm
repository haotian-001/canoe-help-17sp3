Configuration: Multi CANoe

Ribbon: File Tab|Options|Extensions|Multi CANoe

Note

These settings are configuration-independent settings of CANoe.

Multi CANoe Settings

Here you configure the participants of a Multi CANoe cluster.

Note

The defined participants must be identical in all participating CANoe instances of the Multi CANoe cluster.

With Activate Multi CANoe, you activate Multi CANoe for this CANoe.

Activate Use IPv6 to use Internet Protocol 6 (IPv6) addresses.

Start timeout in s

Depending on the configuration, the start of the measurement on the individual CANoe instances can take different lengths of time. In this input field you set how long CANoe waits until all nodes have successfully started the measurement.

Define Participants

A participant of the Multi CANoe cluster is described by the values Alias, IP Address and Port. In addition, you specify which participant is the master.

Column

Description

Alias

The alias is used to distinguish multiple instances with the same IP address. An alias must be unique.

The alias is also used in the definition of the system variables.

IP Address

Multi CANoe uses the Windows IP stack for data exchange.

Note

Note that all participants must be on the same IP subnet.

Port

Multi CANoe uses FDX over UDP/IP for synchronization and data exchange in the background. If several participants of the Multi CANoe cluster run on the same computer, you must use different port numbers for these participants.

If you do not use a standard port and FDX is activated (see below Several CANoes on One Computer), then you must ensure that the same port is also set in the FDX Settings.

Symbol

The current CANoe instance.

Configured CANoe instance.

The current CANoe instance is not configured as a participant of the Multi CANoe cluster.

With [Import...] and [Export...] you can import and export all settings, e.g. to distribute them to the different participants of a Multi CANoe cluster.

Several CANoes on One Computer

You can run multiple CANoe instances on one computer. This allows you to run either all the participants or only some of the participants in a Multi CANoe cluster on one computer.

To distinguish and address the participants that are operated on the same computer, you must define the application name and the Port uniquely.

Port

If FDX is activated in the XIL API & FDX Protocol dialog, you must ensure that the same settings apply to Multi CANoe (port number and IP version).

Multi CANoe

XIL API & FDX Protocol

IP Address

It is also possible to use localhost as IP address (127.0.0.1 or ::1). In this case all participants must run on the same computer and use localhost as address.
