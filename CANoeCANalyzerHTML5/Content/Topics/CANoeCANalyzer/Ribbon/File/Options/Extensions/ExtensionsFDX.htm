Configuration: XIL API & FDX Protocol

Ribbon: File Tab|Options|Extensions|XIL API & FDX Protocol

XIL API Settings

  • With the Enable XIL API server option you can set, if the XIL API server is activated for the current configuration.
  • In the Port number input box you can set the number of the TCP port that should be used for the XIL API (TCP/UDP Ports used by CANoe).

FDX Settings

  • With the Enable FDX check box you can set, if the FDX protocol is activated for the current configuration.
  • With the Transport Layer list you configure the protocol to be used as transport layer for the FDX protocol: UPD/IPv4, UDP/IPv6, TCP/Ipv4 or TCP/IPv6.
  • In the Port number input box you set the number of the UDP port or TCP port to be used for the FDX protocol. (Table of the TCP/UDP ports used by CANoe).

FDX Description Files

To enable data exchange via FDX, you must specify in CANoe before start of the measurement which signals or variables are to be exchanged and the manner in which this is to occur (e.g., with regard to grouping).

  • With the [Add…] button you can add description files in XML format to the list of FDX description files. The files selected in the list are read at the start of the measurement and an assignment is made to the signals and variables from the CANoe configuration or the databases used therein.
  • With the [Remove] button you can remove the selected files from the list.
    With a double-click on a line in the list you can change the filename or path.
  • With the [Edit] button you open the FDX Editor. If you have previously added FDX description files, the files whose check boxes are activated are opened directly in the FDX Editor for editing.
