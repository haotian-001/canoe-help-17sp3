User Files

Scope

If you want to read files in distributed mode using the corresponding CAPL functions (e.g. openFileRead, openFileWrite or getProfileInt) in the Simulation Setup or in the Test Setup for Test Modules programs, you have to first define these files here. The files specified and activated in this dialog are transferred to the remote device at the start of measurement (if available). CAPL then accesses the files on the remote device during measurement. Files that are only created during measurement may also be predefined as mentioned above. If they are not predefined they will be implicitly registered as user files for the current measurement. All files created or changed via CAPL functions on the remote device are transferred back to the predefined location on the user computer once measurement ends. In case of implicitly registered user files the file path given as parameter to the CAPL file function is interpreted as the file path on the user computer to which the file is copied on measurement end (with non-absolute paths being considered as relative to the configuration directory).

Cross Reference

For more information on using user files in a distributed environment see VN8900: Model and Configuration Requirements.

If there is no distributed simulation environment the definition of user files in this dialog is not required. However, if there is a definition for a user file, the file is first searched at the location given here before the other search rules according to CAPL Functions: File Search Procedure are applied.

Use

To add a file to the list of user files, select it from your file system using the [Add...] button. You can also specify filenames that do not yet exist. You can remove selected files from the list using the [Remove] button. To change the filename or path, just double-click the corresponding entry in the list.

Note

You cannot add files to the list that have the same name but different paths.

CAPL Functions

To use user files in CANoe, the following CAPL functions are available:

  • GetUserFilePath: Determination of the absolute file path of a user file.
  • RegisterUserFile: Dynamic registration of a user file during measurement.
  • TestWaitForUserFileSync: Synchronization of a user file during measurement (can be used in test setup).

The CANoe .NET API also has the corresponding methods (Vector.CANoe.Runtime.GetUserFilePath, Vector.CANoe.Runtime.RegisterUserFile, Vector.CANoe.Threading.WaitForUserFileSync).
