Network Access

With the network-based access option you activate the network-based mode of operation in CANoe.

Please note that network-based mode of operation is not supported by all Vector interfaces.

Update signals without port qualification

  • always
    Events of all ports are used to update a signal without qualified Ethernet ports.
  • RX packages only
    Only RX packages received by the interface are used which leads to less duplicated updates
  • never
    Signals without qualified Ethernet ports are not updated.

Optimize the required resources by connecting Simulation Ports with a simulated switch where possible

For VN Ethernet Interfaces, the simulation ports of the same segment are connected via a simulated virtual switch within CANoe.

Since the VN Ethernet interfaces only allow a limited number of ports per network or segment, this option can be used to run the simulation even if this limit would be exceeded. The simulated switch is then connected to the hardware switch by dynamically adding a virtual port named ConcentratorVP_<number>.

This option also improves the performance of CANoe by reducing the number of events received from outside of CANoe.

This also applies if you use SIL Kit. When using SIL Kit, the virtual port is named according to the network.

Allow the use of simulation ports in the Measurement Setup

If this option is disabled, the events of all simulated ports will not be forwarded to the GUI components for evaluation.

In order to still receive events from certain nodes, these must be additionally activated as measuring ports in the Port Configuration dialog.

Check modeling libraries for compatibility with network-based mode

If this option is activated and network-based access is used, all configured modeling libraries are checked for compatibility with network-based access.

If a modeling library is incompatible, then a warning is shown in the Write Window. However, the measurements can still be performed.

Statistic System Variables Schema

If you activate this option, all Ethernet system variables will be grouped into a struct system variable per channel and port to improve the performance.

Note

Activating this option also changes the identifiers for the system variables. For this reason, all affected elements in the Graphics or Data Window may have to be redefined.

Note for Version 17

As of version 17, the following setting is no longer supported.

The setting is available for configurations from older versions for compatibility reasons.

Enable the usage of a signal protocol DLL by enabling the Enable signal protocol check box.

Enter the name directly in the edit field DLL filename (the file is searched for in the configuration directory and in the installation directory (directory Exec32) of CANoe) or use the button [...] to open a dialog for manually searching this file.
