Configuration: CANopen

Ribbon: File Tab|Options|Bus Systems/Protocols|CANopen

CANopen Settings

SDO Timeout
You can use this input box to enter the maximum wait time in milliseconds that is to be used to wait for responses from the SDO command.

Set NMT status to PreOperational before SDO Downloads
If several objects of one or more nodes are written via SDO download (e.g. the entire PDO configuration), the CANopen network is set to the PreOperational state via NMT message before the download.

Save DCF compact
The CiA306 specification optionally provides a more compact storage format for DCF files, which you can activate via this check box.

[EDS Directories]
The EDS Directories button allows you to specify where CANoe searches for EDS files in order to display them in selection dialogs and assign devices during scanning.

Name Scheme for CANopen System Variables

The naming scheme for the generation of the CANopen system variables is freely configurable. You can use your own characters and combine the available field codes as you like.

Input Box

Available Field Codes

Namespace for Nodes

Node ID, Node Name

Namespace for Structured Objects

Index (hexadecimal), Object Name (Name of the main object of the structured oject)

Variable for Objects

Index (hexadecimal), Sub Index (hexadecimal), Object Name

The generated names must be C identifiers so that you can access the variable in CAPL. If this condition is not fulfilled, you receive a warning notification after the respective input box.

If the naming schema creates identical names for two objects, they are automatically assigned an index.

The Default Scheme button resets the input boxes to their default settings. In earlier CANoe versions the index and subindex were included in the name, with the Compatible with CANoe < 12.0 button this naming scheme can be set.
