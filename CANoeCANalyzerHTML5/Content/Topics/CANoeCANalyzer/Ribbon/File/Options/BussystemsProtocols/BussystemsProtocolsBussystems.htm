CAN Default Interaction Layer

CAN signals can only be sent when the send node is contained in the Simulation Setup and there is an interaction layer configured for it.
If there is no interaction layer configured, you can switch on the default interaction layer here.

Transmit Behavior CANoeILNLVector.dll and CANoeILNLDefault.dll

  • The Default Interaction Layer (CANoeILNLDefault.dll) sends each message based on event triggering. Thus, on any signal update the message will be sent.
    A message can not be sent cyclically with the Default IL.
    The default interaction layer ignores NM, TP and diagnostic messages.
  • The CANoe Interaction Layer (CANoeILNLVector.dll) sends a message cyclically and/or even triggered according to the message and signal attributes in the database. Also the cycle period can be switched (if defined) according to the activity state of any signal.
    A message can be sent with the CANoe Interaction Layer:

    • Basic Send Types

    Additional Properties

    spontaneous
    (sporadically / OnEvent)

    NoOfRepetitions = n AND for each signal a property from {OnWrite, OnChange, None}.

    cyclic
    (periodical)

    Cyclic period (normal period and if applicable fast period).

    dual cyclic
    (normal and fast cyclic)

    See cyclic additional properties.

    on active cyclic
    (IfActive)

    For each signal one property that defines the InActive value.

    See also cyclic additional properties.

    Basic send types can be combined, e.g. spontaneous and cyclic

Start-of-Frame Time Stamp Calculation for CAN

In online mode

  • If SOF time stamps are provided by the hardware, you do not need this option. It has no effect.

Notes

The calculation of SOF can negatively affect the throughput performance.
  • If the CAN hardware does not provide SOF time stamps, you must activate this option in order for the SOF time stamps for CAN messages to be calculated.
  • If this option is not selected, you will have limitations
  • in the Statistics Window (the busload cannot be calculated)
  • in some columns of the Trace Window (see bus/protocol-specific settings)
  • when using the message selectors in CAPL

In offline mode

  • If a valid value for the SOF time stamp is available, this option has no effect. You do not need this option.

Notes

The calculation of SOF can negatively affect the throughput performance.
  • If a valid SOF time stamp has not been logged, you must select this option in order for the SOF time stamp for CAN messages to be calculated.
  • If this option is not selected, you will have limitations
  • in the Statistics Window (the busload cannot be calculated)
  • in some columns of the Trace Window (see bus/protocol-specific settings)
  • when using the message selectors in CAPL
  • If the baudrate wasn't logged, you can enter a value here. In the table you can set the baudrates of the discrete channels.

The CAN channels configured under Channel Usage will be listed. You can use the baudrates of the hardware configuration (default setting) or you can insert own baudrates if you want to read in a logging file in the offline mode that was logged with different settings or hardware.

To set a new baudrate, deactivate the check box within the Use Hardware Settings column. After that you can enter a new baudrate for a channel in the Baudrate column via double click.

Note

The baudrate settings in this dialog have no effect on the hardware configuration. They are used solely for the statistics calculation in offline mode.

FlexRay Frame and PDU Interpretation – Dual Channel System (AB), Ambiguous PDUs

With this settings you control the interpretation of frames and PDUs of FlexRay configurations with channels A and B. If you change the settings, the database will be reloaded.

Enable Dual Channel Support

  • If active, PDUs – on both channels, A and B – will be configured with channel mask AB.
    Channel postfixes (xy_Ch_A, xy_Ch_B) for PDUs will be created for ambiguous objects only.
  • If not active, all frames/PDUs automatically get a channel postfix.
    Redundant objects will be split into a channel A object and a channel B object.
  • On creating a new configuration, dual channel support is active by default.

Use Postfixes for Ambiguous PDUs on Channel(s)

If dual channel support is not enabled you can define the postfix settings for frames/PDUs here:

  • A and B
    Channel postfixes for frames/PDUs will be generated on channel A and B.
  • A only
    Channel postfixes for frames/PDUs will be generated on channel A only.
    PDUs on channel B will keep their original name.
  • B only
    Channel postfixes for frames/PDUs will be generated on channel B only.
    PDUs on channel A will keep their original name.
  • No Postfixes
    No channel postfixes are created for frames/PDUs.

FlexRay Signal and Payload Initialization

Initialization of Signals

When the Initialize signals with database values option is enabled, signals are automatically transmitted with an initial value during a simulation. An explicit data update overwrites these initial values later.

A FIBEX database makes the following signal attributes available as initial values:

  • Default value
  • NOT-VALID lower limit
  • NOT-VALID upper limit

Click the [Set as Default] button to use the current settings as default settings for new configurations.

Click the [Reset] button to revert the current settings back to the default settings.

Default Initialization of the Payload

These settings are used to set the payload area of a FlexRay frame to a default value. The settings affect unused data areas or signals that have not been initialized. The data bytes can be set either to 0x00 or 0xFF.

Click the [Set as Default] button to use the current settings as default settings for new configurations.

Click the [Reset] button to revert the current settings back to the default settings.

FlexRay Settings

PDU Transmission

This setting is used to specify the principle transmission behavior of the PDUs.

For single PDUs, the following settings can be overwritten using the FlexRay PDU panel or CAPL (FRUpdatePDU):

  • Send Always:
    A single PDU update results in a continuous transmission of the PDU. This is carried out in accordance with the PDU's schedule. If the PDU has an update bit, then it is set automatically before each transmission.

  • n-times:
    The updated PDU is transmitted exactly n-times. If an update bit is configured, it is also set automatically for each transmission in this case.

Click the [Set as Default] button to use the current settings as default settings for new configurations.

Click the [Reset] button to revert the current settings back to the default settings.

Protocol Identification

For protocols Ethernet, TCP, and UDP it can be specified, which protocol follows in the next higher layer. For this the values of the column Protocol Identification are assigned to the protocols in the next higher layer.

Notes

  • The protocols are displayed only if they are available in the current configuration. At least one Ethernet or WLAN channel has to be configured.
  • TCP/UDP Ports used by CANoe

With [Edit...] a configuration dialog for setting the values opens. Alternatively the dialog can be opened with a double click on a protocol.

Single values are separated by commas, value ranges are defined with a minus, e.g. 80, 8000-8888. With [Default] the value is set to its default value.
