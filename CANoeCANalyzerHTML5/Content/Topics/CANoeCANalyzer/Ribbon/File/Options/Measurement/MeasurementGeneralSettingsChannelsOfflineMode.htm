Channels in Offline Mode

Ribbon: File Tab|Options|Measurement|General

In offline mode messages from a previously recorded logging file are replayed into the Measurement Setup.

This involves playing-in each message on the channel on which it was logged. Thus, the channels in offline mode match the channels used during logging to the logging file. The number of channels should be defined such that it matches the number of channels configured when logging to the logging file.

Behavior when using undefined channels:

  • The channel setting does not result in any filtering of messages in the data flow plan.
  • If messages assigned to an undefined channel are played in, they are passed through the Measurement Setup.
  • This is also the case when transmitting from a Generator or Replay Block in the Measurement Setup on an undefined channel. The transmit request is passed through.
  • CAPL blocks represent an exception. These program blocks do not send messages to which an undefined channel is assigned.
