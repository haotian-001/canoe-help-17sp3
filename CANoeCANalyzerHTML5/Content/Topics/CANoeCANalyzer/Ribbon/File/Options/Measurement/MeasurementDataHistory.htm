Configuration: Data History

Ribbon: File Tab|Options|Measurement|Data History

Here you can make settings for the data history of the analysis windows. CANoe saves measured data from Trace, Test Trace and Graphics Window in a global buffer. The data can be swapped to the hard disk here.

Usage of Memory(RAM)

With the slider, you can specify the size of the data history in the memory:

short – middle – long – very long – Custom use

Depending on the operating system and the real available main memory, CANoe uses a maximum of the displayed amount of RAM. The RAM requirement of CANoe increases continuously during the measurement until the data history reaches the set maximum size. When the maximum size is reached, older elements in the data history are overwritten.

Note

  • With a very large use of the main memory, the next measurement start can take considerably longer.
  • If other programs with large main memory requirements are executed, you should set the usage so that enough real main memory is available for all programs. Otherwise, the operating system will attempt to move memory areas to the hard disk.

Usage of Swap File

Note

  • If high data rates are involved, you should select the fastest hard disk possible with sufficient space for the swapping. Do not place the swap file on an external storage medium connected to a slow USB port, for example.
  • Please do not create the pagefile on the system partition (usually "C:\"). Use a different partition (e.g. "D:\") instead. This prevents potential performance problems related to interactions with virus scanners, windows pagefiles, indexing services and NTFS compression.
  • Note: The new size will be applied on the next measurement start.

With the slider, you can specify the size of the swap file. If no swap file is used CANoe uses the only the main memory for the data history.

  • Location of Swap File
    Here you can set where the swap file is to be created.

Data History Settings

Here you can specify how CANoe is to respond when the buffer is full:

  • Delete older data gradually
    The oldest data is deleted from the windows and buffers bit by bit. As much data is retained as possible.
    Lines are never lost in fixed position mode in the Trace Window, and you can also specify in a configuration dialog of a Trace Window that its data is to be deleted as late as possible (after other windows). If relocating to a slow hard disk, considerable drops in speed can occur, especially with computers which have only one processor.
  • Delete all older data immediately
    All old data is deleted, and the measurement is continued with empty buffers. This can prevent drops in speed during the rest of the measurement. In online mode it is possible that current data for CAPL analysis programs or Logging Blocks could also be lost during the deletion process, because the process requires a certain amount of time. In offline mode, on the other hand, this type of data loss does not occur.
  • Stop measurement
    Select this option if you want to analyze all measured data in the windows and don't want to lose any old data in any case.

Size of Visible Analysis Range for Trace/Test Trace

With the slider, you can specify the size of the visible analysis area: very short — short — middle — long — very long.

The length of the data history depends on the configured size of the analysis area.

The calculated sizes are dependent on the number of Trace and Test Trace Windows, the system buffer settings, and the equipment of the working computer.

The new size of the visible analysis area is applied only at the next measurement start.

The approximate number of events/frames that a window can save with the current settings before the buffer is full is displayed below the setting options. If you want to analyze all the measured data in the windows after the measurement, you should make the settings so that this quantity of saved data is sufficiently large.

Note

For a data rate of 10,000 frames per second and a measurement duration of 4 hours, 144 million frames must be saved.
