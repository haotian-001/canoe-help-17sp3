Channels in Online Mode

Ribbon: File Tab|Options|Measurement|General

In online mode with working mode Real Bus, messages are transmitted from the Simulation Setup to one or more real buses, and they are received in the Measurement Setup by one or more real buses.

Thus, the defined channels match the controller chips in the real buses. A chip is allocated to each defined channel in order to permit a start of measurement.

Allocating Channels to Controller Chips

In the CAN hardware configuration of your computer's Control Panel the channels are allocated to the CAN chips that are registered in the system.

The chip allocation is only relevant in online mode. In offline mode, where the messages are played in from a file, it is irrelevant.

If you wish to modify the standard chip allocation, then go to your computer's panel (via the Start menu path Settings|Control Panel) and open CAN Hardware. The CAN Hardware Configuration Dialog then appears, in which you can change the chip allocation of the channels.

The chip allocation is also shown in the Hardware Configuration dialog.

Behavior when using undefined channels

  • The channel setting does not result in any filtering of messages in the data flow plan.
  • When receiving on chips that are not allocated to a defined channel, the received messages are passed through the Measurement Setup.
  • This is also the case when transmitting from a Generator or Replay Block in the Measurement Setup to an undefined channel. The transmit request is passed through the Measurement Setup.
  • However, if the transmitting block is integrated in the Simulation Setup, an error is reported in the Write Window as soon as the transmit request is on the undefined channel. In such a case the message is not transmitted.
  • CAPL blocks represent an exception. These program blocks do not transmit messages to which an undefined channel is assigned.
