Application Channel Mapping

Ribbon: Hardware Tab|Mapping

General

  • In the Application Channel Mapping dialog you define the mapping of application channels to network interface channels or the mapping of application network names to hardware network names. Additionally you can deactivate networks and their assigned channels.
  • When using the SIL Kit, you define in this dialog which application channels are connected to the SIL Kit.
  • For the CAN, Car2x, LIN, FlexRay and Ethernet networks, you can also assign the application channels to the network simulator of CANoe instead of to the network interface channels. This allows a mixed operation of real and simulated channels for complex remaining bus simulations.
  • This dialog automatically appears if a channel mapping error is detected on measurement start.
  • If you leave the dialog with [OK] the activation state of the networks will be transferred to the configuration and all channel mapping changes will be stored locally on your computer. The assignment of channels to the SIL Kit or the CANoe network simulator is stored in the configuration.
  • Channels that are not assigned to a network can be temporarily deactivated for the following measurement. This modification will not be saved in the configuration.
  • Network name mapping modification will not be saved in the configuration.

Supported Hardware

The assistant supports Vector XL- and VN-hardware families.

VN8900 devices are supported in interface mode as well as in distributed operation mode. The displayed channel mapping refers to the currently used execution environment. In distributed mode the channels set and usable on the device are displayed (see also Home ribbon tab|Group Measurement|Vector Tool Platform|CANoe/CANalyzer|Channel Mapping), while in interface mode the channels are displayed on the local computer.

Automatic or manual reassignment to other hardware devices is not possible.

Note on COM Interface

To allow for completely automated scenarios without the need for user input the automatic check is always inactive if the measurement is started via the COM interface.

User Defined Application Name

If CANoe is started with a user defined application name not equal to CANoe, the title of this dialog includes the user defined application name.

Note

Channel mapping is only available for channels using channel-based network access.

Network name mapping is only available for channels using network-based access.

Manual Channel Mapping

To be able to start a measurement the proper network interface channel has to be mapped to each application channel or the proper hardware network name has to be mapped to each application network name. All mappings can be checked and modified in this dialog.

For each application channel the mapped network interface channel or for each application network name the mapped hardware network name is shown. A red symbol marks a faulty mapping. A tooltip describes the reason of the faulty mapping.

For each channel the network interface type and the transceiver type are shown. If a channel description has been edited for a used channel it will be shown in an additional column. Effective with driver version 9.6 you can edit channel descriptions in the Vector Hardware Configuration. Effective with driver version 11.1 you can define hardware network names for the Ethernet networks, which makes network name mapping possible.

You can change the channel mapping in the Hardware column. In this column you can also select the assignment to the SIL Kit or the CANoe network simulator

The list of available network interface channels shows only compatible bus system channels. The symbols reflect the channel availability (unmapped/already mapped). If you select an already mapped channel its former mapping is canceled and None is assigned to the application channel. If Warn, if hardware channel is already used by another application channel is activated in Options|General|Messages you'll be asked at first whether to confirm or deny the assignment. This warning can be disabled permanently.

Automatic Channel Mapping

Clicking Automatic Remapping assigns unmapped or incorrect mapped channels to an appropriate network interface channel, if possible.

Revert All Channel Mapping Changes

Clicking Revert Mapping resets all channel mapping to their initial state.

Manual Deactivation of Networks

If a network is not required for a measurement, it can be deactivated in the Active column. This can be useful if the application channel of the network cannot be mapped to a network interface channel and the simulation can perfectly be performed without this network. Application channels without assigned network cannot be deactivated.

Automatic Deactivation of Networks

Clicking Deactivate Unmapped deactivates all networks whose application channel has no valid channel mapping.
