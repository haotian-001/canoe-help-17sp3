Configuration: Name Display

Ribbon: File Tab|Options|Appearance|Name Display

Note

Settings for the name display are taken into consideration

  • in the Data Window

  • in the legend of the Graphics Window

  • in the State Tracker.

Signals

You can configure the name display of signals (symbols) directly in the list.

Display Details

In this column, you specify how much symbol information is to be displayed. Each symbol type has its own display levels, i.e., not every display level is available for each symbol.

Possible display levels:

  • Auto: All information is displayed to identify the symbol as unique.
  • Name: Only the symbol name is displayed.
  • Custom...: With this display level, you can left-click the entry to open a field with additional selection options where you can, in turn, set the qualifiers that are to be displayed.

Qualifier for Different Symbol Types

If a qualifier is not available (e.g. no Tx node defined in database) then this qualifier will not be displayed.

Position

In this column you determine where additional information on the symbol shall be shown.

  • In Front
    The information is shown before the symbol name (Separator: "::").
  • Behind
    The information is shown behind of the symbol name (Separator: "//").

Truncation of Symbol Name

If there is insufficient space for the complete name display of the symbol, you can specify how the name is to be shortened in this column

  • Non
    The name display is not shortened by the program. If there is too little space, the symbol name may not be visible.
  • Center
    The complete name display will be displayed to the extent possible. If there is too little space, the additional information to the left or right of the symbol name is shortened to begin with. The truncation is indicated with "…". If there is still too little space, shortening occurs centered behind the first letter of the symbol name. The ending of the symbol name remains visible as long as possible.
  • From right
    The complete name display will be displayed to the extent possible. If there is too little space, the additional information to the left or right of the symbol name is shortened to begin with. The truncation is indicated with "…". If there is still too little space, shortening occurs from the right. The beginning of the symbol name remains visible as long as possible.

Preview for Signals

In the Displayed Symbol Name column, you are shown a preview of the symbol name according to your settings in the Display Details and Position columns. Note that a realistic preview is not possible for the Auto display level, because all databases would have to serve as the basis here.

In the Truncated column, you are shown a preview of the shortened symbol name corresponding to your settings in the Truncation of Symbol Name column.

System Variables

You can configure the name display of system variables directly in the list.

Display Details

In this column, you specify how much information of the system variable is to be displayed. Each system variable type has its own display levels, i.e., not every display level is available for each type.

Possible display levels:

  • Auto: All information is displayed to identify the symbol as unique.
  • Name: Only the name of the system variable is displayed.
  • Custom...: If you choose this display level, the configuration dialog for system variables opens automatically. In this dialog, you can select the namespaces you want to display (except statistics variables, XCP system variables, and VTS system variables).
  • Important
    This display level displays only the namespaces that were set in the program as important for the display. You cannot influence the selection of the namespaces.

Position

In this column you determine where additional information on the system variable shall be shown.

  • In Front
    The information is shown before the symbol name (Separator: "::").
  • Behind
    The information is shown after of the symbol name (Separator: "//").

Truncation of Symbol Name

If there is insufficient space for the complete name display of the system variable, you can specify how the name is to be shortened in this column

  • Non
    The name display is not shortened by the program. In this case the settings in the Truncation of Namespaces column have no effect. If there is too little space, the name of the system variable may not be visible.
  • Center
    When there is too little room, the namespaces are initially configured according to the setting in the Truncation of Namespaces column. If there is still too little space, shortening occurs behind the first letter of the symbol name. The ending of the name of the system variable remains visible as long as possible.
  • From right
    When there is too little room, the namespaces are initially configured according to the setting in the Truncation of Namespaces column. If there is still too little space, shortening occurs from the right. The beginning of the name of the system variable remains visible as long as possible.

Preview for System Variables

In the Displayed Symbol Name column, you are shown a preview of the system variable name according to your settings in the Display Details and Position columns. Note that a realistic preview is not possible for the Auto display level, because all databases would have to serve as the basis here.

In the Truncated column, you are shown a preview of the shortened system variable name corresponding to your settings in the Truncation of Symbol Name column.
