CANoe4SW Server Edition Configuration Settings

Ribbon: File Tab|CANoe4SW Server Edition Export|Configure...

In the CANoe4SW Server Edition Configuration Settings dialog, you can configure settings for the logging in the CANoe4SW Server Edition environment.

Note

The settings configured here are saved in your CANoe configuration.

At the export to a CANoe4SW Server Edition configuration, all these settings are also saved in the configuration file (*.rtcfg).

Logging Block

Here you can activate or deactivate a Logging Block for CANoe4SW Server Edition and configure the logging properties (e.g. the name of the logging file) themselves ([Configure...] button).

When Use CANoe4SW Server Edition logging has been activated, Trigger Blocks and CAPL Blocks can be inserted optionally.

Trigger Block

With this Trigger Block logging can be started and stopped with CAPL (trigger, startLogging, stopLogging).

When the Trigger Block is configured using the [Configure...] button, you can specify a pre-trigger time (logged events before occurrence of the trigger condition) and a post-trigger time (logged events after completion of the Trigger Block). Furthermore you can use Stop after n Trigger Blocks to specify that the measurement will be stopped after a certain number of trigger starts. In the Buffer size input box, you can limit the number of messages that will be saved during the pre-trigger or post-trigger time.

CAPL Filter

This CAPL block can be used, in particular, for filter purposes and is added upstream of the Logging Block.

It is useful, for example, to specify a CAPL file via the [Configure...] button that is also upstream of the Logging Block in the Measurement Setup. With the exception of functions that have been specifically tailored to the Measurement Setup, all CAPL functions are also supported in standalone mode. It must be noted in particular that – just like when using a CAPL block in the Measurement Setup – messages are only forwarded to the Logging Block if this is explicitly initiated by calling CAPL function output.
