Term

Definition

Hardware segment

A hardware segment is used for gaining access to an Ethernet network. A hardware segment may comprise several hardware channels and provides an application channel for the application (e.g. CANoe). The following hardware segments are available:

  • Switch
    Comprises one or more hardware channels. The switch learns MAC addresses and transfers packages according to the learned rules.
  • TAP
    Comprises two hardware channels and is a simple measurement point (see below for additional information).
  • Direct link
    Comprises exactly one hardware channel and is typically used for diagnosis applications.

TAP
(Test Access Point)

A TAP basically provides the functionality of a Y-cable, which you may be familiar with from CAN applications. I.e., a TAP is (usually) a measurement point between two endpoints.
You can configure one (e.g. VN5610) or several (e.g. VN5640) TAPs on the Vector network interfaces. When doing this, the following options are available:

  • PHY Bypass
    Packages are transferred between the two hardware channels with merely any latency. It is not possible to send packages using CANoe.
  • MAC Bypass
    Packages are transferred between the two hardware channels with minimum latency. In addition, packages can be sent using CANoe.
  • Bypass Off
    Packages are not transferred automatically between the two hardware channels. If necessary, it is up to you to make sure that the packages received on one hardware channel are output on the other hardware channel (using CAPL, .NET or similar). With this option you can precisely filter for packages or manipulate packages.

Ethernet default configuration

The Vector Hardware Config dialog can be used to configure the VN hardware and to store the settings permanently on the hardware. Open the Vector Hardware Config dialog using the [Driver…] button in the Network Hardware Configuration dialog, and set any required options.

  • Define number of segments (there is one hardware segment per application channel available)
  • Configure hardware channels (PHY settings)
  • Configure TAPs

When closing the dialog, the configuration is stored automatically to the VN hardware.
