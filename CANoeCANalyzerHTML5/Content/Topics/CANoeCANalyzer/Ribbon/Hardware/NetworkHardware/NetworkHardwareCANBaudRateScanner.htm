Baudrate Scanner

Ribbon: Hardware Tab|Network Hardware|CAN|Setup|[Scan...]

The baudrate scanner attempts to determine the baudrate of a network automatically. It does this by using variable baudrates and checking whether a message can be processed successfully via the given channel.

A scan is registered as successful if it was possible to send or receive a message correctly, depending on the mode used. If it was not possible to process a message successfully within the given time span, the scan continues with the next baudrate in the given baudrate interval.

Mode

Active Mode

Active mode is used when scanning the baudrate of a network on which there is currently no communication. In this case, CANoe is an active participant (transmitter) in the network, sending messages via the given channel. The process ends when a message has been sent successfully and thus the correct baudrate is known. If the wrong baudrate is used the other devices on the bus will not be able to receive the message. In this case, CANoe (as the transmitter) does not receive an acknowledgment and therefore does send an Error Frame in turn. The scan then continues with the next baudrate.

Passive Mode

Passive mode is used when scanning the baudrate of a network on which there is already communication going on. In this case, CANoe is an passive participant (receiver) in the network, receiving messages via the given channel. The process ends when a message is received successfully. In this mode, no Error Frames are sent by CANoe as the receiver. In passive mode the scan continues if no message is received within the given time span.

Use Cases

A single ECU is connected to a CAN channel.

With a passive baudrate scanner CANoe doesn't recognize any baudrate on the channel.

In contrast, the active baudrate scanner works perfect.

Reasons:

As a passive participant CANoe generally doesn't send an Acknowledge. As a consequence of this the sole transmitter sends an Error Frame prior to the EOF (since no Acknowledge has been received) and no communication comes about by means of which CANoe could recognize the baudrate correctly.

The passive scanner with only one connected device cannot arrive to a result.

Scope

Here you have to define which baudrate range shall be checked. The scope differs in the numbers of baudrates to be checked and their distance to each other. You can decide between following settings:

  • With the standard scope the commonly baudrates will be checked. They are 33,333, 50, 83,333, 100, 125, 250, 500 and 1.000 kBaud.
  • With the user-defined scope you can define a specific baudrate range which shall be checked. All baudrates between 0 and 1.000 kBaud are valid.
  • The last possible scope checks all possible baudrates between 0 and 1000 kBaud.

Message

ID

To scan in the active and passive mode, you can use any CAN identifier between 0 and 2047 (0x7FF). Extended IDs must have a trailing x and must be between 0x and 536870911x (1fffffffx hex).
Additionally for the passive mode it is possible to react on any identifier, that is all messages on the bus will be received by the baudrate scanner to determine the baudrate.

Note

These functions are only available in connection with Vector CAN Interfaces.
