Ethernet Network Hardware Configuration

 

An Ethernet interface is required to enable measurements in an Ethernet network in real mode by means of CANoe. Vector offers for Ethernet various network interfaces which have a different number of channels and different configuration options.

Procedure for Network-Based Access

VN Ethernet interfaces from firmware version 11.1 support a network-based access.

Application note AN-IND-1-023_Ethernet_VN_Family_From_Firmware_Version_11.1.pdf describes how the concept behind the Ethernet VN interfaces was changed.

VN Ethernet interfaces with network-based access support flexible definition of networks, segments and ports. This allows you to use these devices in combination with CANoe for a wide range of use-cases such as analysis, tests and simulation.

The configuration of the CANoe can be done in the in the Network Hardware Configuration dialog.

Step by Step Procedure

  1. Open the Network Hardware Configuration dialog via the Hardware ribbon tab|Network Hardware.
  2. Press the [Drivers…] button to open the Vector Hardware Config.
    All connected Vector interfaces are displayed under Hardware.
    (Alternatively, you can open Vector Hardware Config via Windows Control Panel with Vector Hardware.)
  3. Select an Ethernet interface, information on the selected interface will be displayed on the right side.
  4. In the Ethernet Mode Configuration dialog you can check if the Ethernet interface is still in channel-based mode. Open the dialog with the context menu command Change Ethernet Mode to select the network-based access.
  5. Use the Device Configuration context menu command to open the Device Configuration dialog.
  6. Define the Switches and/or Links (TAP or Direct Connection) that you want to use by dragging them onto the networks on the Layout page.
  7. Give a meaningful name to an Ethernet network so that its use is unique. A network defined in CANoe and a network defined in the Ethernet device configuration automatically find each other when they have the same name.
  8. Add the Ports (Physical) to the Switches and Links and configure their properties. These Ports can be used as measurement ports in CANoe to analyze the Ethernet traffic.
  9. Close the Ethernet Device Configuration dialog by clicking [OK].
  10. Close the Device Configuration.
  11. Close the Network Hardware Configuration dialog by clicking [OK]

The networks that have been defined in the Ethernet Device Configuration are now ready to be used in CANoe. The ports that have been defined are now available in working mode real bus in the Port Configuration dialog.

Procedure for Channel-Based Network Access

Depending on the use case, there are various options for connecting an Ethernet interface to an Ethernet network. Since an Ethernet switch, forwarding Ethernet packets to the corresponding Ethernet links is usually used, Ethernet traffic varies on each link. Depending on where the Ethernet interface is connected, only a part of the Ethernet traffic is visible for the analysis in CANoe.

Thus, the Vector Ethernet interfaces provide the following settings via the Vector Hardware Config:

  • Direct Connection (Direct access in the Network Hardware Configuration dialog)
    One Ethernet channel per Ethernet network in CANoe is used. If this channel is connected to an Ethernet switch, CANoe only sees the Ethernet packets which are forwarded to this Ethernet link. To allow an Ethernet switch to forward to this Ethernet link anything at all besides multicast or broadcast, CANoe must actively send Ethernet packets
  • from its own MAC address.
    Therefore, the direct mode is usually used for remaining bus simulation and tests.
  • Measurement (Y cable in the Network Hardware Configuration dialog)
    The Ethernet link between two real Ethernet ECUs is broken and guided through the Ethernet interface. Thus, two Ethernet ports are required on the VN interface which passes Ethernet traffic with minimum delay.
    In CANoe, two channels are assigned to one network. Each Ethernet packet exchanged between the real ECUs can be measured.
  • Simulation (Switch in the Network Hardware Configuration dialog)
    The Vector Ethernet interface functions as an Ethernet switch. All real ECUs are connected to the Vector Ethernet interface. Since all real ECUs are connected to the Ethernet interface, all Ethernet packets exchanged by the ECUs can be measured in CANoe.

Vector Ethernet Interface in the Direct Connection Mode

You must set and configure your Ethernet interface in the Network Hardware Configuration dialog. Vector interfaces are configured with Vector Hardware Config.

Step by Step Procedure on VN5610

  1. Open the Network Hardware Configuration dialog via the Hardware ribbon tab|Network Hardware.
  2. Press the [Drivers…] button to open the Vector Hardware Config.
    All connected Vector interfaces are displayed under Hardware.
    (Alternatively, you can open Vector Hardware Config via Windows Control Panel with Vector Hardware.)
  3. Select an Ethernet interface, information on the selected interface will be displayed on the right side.
  4. Use the Device Configuration context menu command to open the Device Configuration dialog.
  5. Set properties required for an Ethernet network, for example 100BASE-T1.
  6. Make sure that for Direct Access the Bypass setting is set to Off.
  7. Close the Device Configuration dialog by clicking [OK].
  8. The channels of the Ethernet interface are listed below the interface and can be opened with [+].
  9. Open the context menu of the channel and select the application channel which should be used in CANoe.
  10. Close the Device Configuration.
  11. Close the Network Hardware Configuration dialog by clicking [OK].

Vector Ethernet Interface in the Measurement Mode

The Measurement mode is configured in the same way as the Direct Connection mode. Only the Bypass setting must be set to On.
