On this page you define the bit timing; additionally you can change the CAN controller mode.

A bit time is shown that is subdivided into three segments: SYNC (orange), TSEG1 (blue) and TSEG2 (green).

CAN Controller Mode

You can set the CAN controller mode via the list field. The following settings are available:

  • Normal
    The CAN controller participates in bus communication.
  • Ack Off
    The bus is influenced by the measurement since the CAN controller on the interface card represents the interface between the analysis software and the CAN bus. Particularly the CAN controller sends its acknowledge for correctly recognized messages by sending a dominant level (on the bus) in the corresponding slot of the CAN message. To reduce the influence on the system you can switch off this functionality. Thereby the node is no longer visible on the bus. Please keep in mind that – when you switched off acknowledge – communication on the bus only may happen if at least one other network node sends an acknowledge.
  • Restricted
    This mode is only supported by CAN FD enabled network interfaces and drivers.
    It s possible to receive valid data and remote frames. On valid frames, the CAN controller sends an acknowledge. If the CAN controller recognizes an error, it will not send error frames or overload frames, but it switches to bus integration state. In this mode it is possible to perform measurements on the bus without a CAN interface affecting the measurement by Error Frames (e.g. measuring an Error Flag sent by a real ECU). Sending is not possible in this mode.
    A Write Window notification will be displayed if the CAN controller switches to bus integration state due to an error.

TX Self-ACK

Vector network interfaces can automatically generate an acknowledge when transmitting a CAN message. This allows the simulation of a network without connecting other active CAN control units. In this case, the CAN channel used must still be terminated by an electric load resistor. If the Self-ACK is not supported, an error message will appear in the Write Window at the start of measurement.

Preview Synchronization Edge

To visualize the influence of the synchronization jump width (SJW) parameter, you can use the slider to adjust the position of the synchronization edge and to examine their influence on the bit timing.
In CAN mode you must activate the Preview synchronization edge option.

The position of the slider defines time point of a synchronization edge on the bus (i.e. the beginning of a bit) relatively to the bit time of the controller. The upper area of the figure shows the nominal bit timing, i.e. a bit on the bus is represented as it is expected by the controller. The lower area of the figure shows the internal controller timing, i.e. the bit time interval from the perspective of the controller. The length of this bit interval depends on the time point of the arriving synchronization edge:

  • If the edge falls within the Sync range of nominal timing, the controller runs synchronously.
  • If the edge falls within the TSEG1 range of nominal timing, re-synchronization must be performed.
    In this case the TSEG1 segment must be lengthened up to SJW (Synchronization Jump Width) BTL cycles.
  • If the edge falls within the TSEG2 range of nominal timing, re-synchronization must be performed.
    In this case the TSEG2 segment must be shortened by up to SJW (Synchronization Jump Width) BTL cycles.
  • If no edge falls within the nominal timing this bit time is not used for re-synchronization.

Bit Timing/Bus Timing Register

The parameters BTR0 and BTR1 define the bit timing. These parameters are compatible to the bus timing registers of the Philips SJA1000 controller.

  • Baudrate
    You can enter the baudrate manually or use the Baudrate Scanner to calculate the baudrate automatically. Click [Scan…] to start the Baudrate Scanner.
  • Bus Timing Register 0/1
    You can edit the BTR0 and BTR1 manually, select them from the BTR list, or you can have them calculated automatically after editing the baudrate.
    Use the data sheet of the Philips SJA1000 controller for the manual input of the BTR values.
  • Clock frequency
    Frequency at which the CAN controller is clocked. The minimum length of a BTL cycle is derived from this frequency. This determines which baudrates can be set exactly.
  • Samples
    In the list field you can select the possible number of samples.
  • Prescaler
    The prescaler is automatically specified. It defines the scaler that is applied to the quartz frequency of the CAN/CAN FD controller The length of a BTL cycle is determined from the resulting frequency.

CAN FD Mode

Use Database Settings

If you use a database that contains information about baudrate and bit timings you can select the Use database settings option to accept this values. The DBC and AUTOSAR ARXML formats are supported.

Note

Enabling this option is done automatically when a database is added.

Automatic activation only occurs under the following criteria:

  • for newly created configurations CANoe version > 16 SP3,

  • when the channel is newly added to the configuration,

  • if the Use DB Settings option is explicitly enabled when a database is present.

The automatic setting is not done…

  • for channels of an existing configuration where this option was not active,

  • for channels where settings were made and subsequently a database was added,

  • if the settings from the database cannot be used.

Arbitration Phase / Data Phase

  • Clock frequency
    Frequency at which the CAN controller is clocked. The minimum length of a BTL cycle is derived from this frequency. This determines which baudrates can be set exactly.
  • Baudrate
    You can enter the baudrate manually.
    Two baudrates must be configured for CAN FD (one register set in the controller for each baudrate):
  • Configuration of the baudrate in the arbitration phase. This corresponds to the baudrate of the classic CAN protocol.
  • Configuration of the baudrate in the data phase. This allows transmission of the data phase at a higher baudrate (> arbitration baudrate). Attention must be paid during configuration that the selected sample point corresponds to the sample point of the actual connected CAN FD ECUs. If the timing is different, communication errors may possibly be detected.
  • Sample point
    You can select the sample point manually or within the Bit Timing Configuration dialog. You can open the dialog with the [...] button on the right side of the Sample point input field.
    All bit timings are shown that are possible for the given baudrate. The currently given values are selected.
  • Prescaler
    The prescaler is automatically specified. It defines the scaler that is applied to the quartz frequency of the CAN/CAN FD controller The length of a BTL cycle is determined from the resulting frequency.

If the baudrate changes, the bit timing parameters (sample point and prescaler) will automatically be adjusted to the best matching values.
