VN5610

On this page you can define the settings for your VN5610 network interface for configuration at the start of measurement (online mode, real bus).

With the lists in area Settings you can specify the properties for the network configuration.

List

Description

Bypass

Here you select if the Ethernet interface forwards incoming Ethernet packets from one channel to the other channel, outputs the packets and vice versa.

Select one of the following options:

  • OFF: Ethernet packets are not forwarded.
  • PHY Bypass: Ethernet packets are forwarded., packets cannot be sent from CANoe, but the lag of time during forwarding is minimal.
  • MAC Bypass: Ethernet packets are forwarded., packets can be sent from CANoe that are integrated to the current network traffic.

If you select Bypass the destination channel of the bypass is displayed next to the list.

Ethernet channel

Here you set the configuration of the Ethernet channel.

Select one of the following options:

  • BR, Slave, 100MBit: Physical Layer BroadR-ReachÂ®, Clock Mode Slave, transmission speed 100 Mbit/s
  • BR, Master, 100Mbit: Physical Layer BroadR-ReachÂ®, Clock Mode Master, transmission speed 100 Mbit/s
  • IEEE, Auto, 100BASE-TX: Physical Layer IEEE802.3, active Auto-Negotiation, transmission speed 100 Mbit/s
  • IEEE, Auto, 1000BASE-T: Physical Layer IEEE802.3, active Auto-Negotiation, transmission speed 1000 Mbit/s
  • IEEE, Auto, 100/1000BASE-T: Physical Layer IEEE802.3, active Auto-Negotiation, transmission speed 100 Mbit/s or 1000 Mbit/s
  • IEEE, Fixed, 100BASE-TX: Physical Layer IEEE802.3, inactive Auto-Negotiation, transmission speed 100 Mbit/s

Note

The Ethernet interface requires some time to setup the connection when it is configured the first time after a power-cycle or the settings have changed. During this period no messages will be sent or received and will be lost.

With the [Twinkle] button you can identify the Ethernet interface that is assigned to this channel. The status LED of the assigned Ethernet interface flashes four times.

VN5640

On this page you can configure the network interface VN5640.

Transparency Settings

The transparency default configuration of the VN hardware can be overwritten by CANoe at the start of a measurement. To activate this function activate the check box Use these link transparency settings (sleep/wakeup, link transparency)â¦, and configure the settings.

If the check box is activated Sleep Wakeup Signals are forwarded to the corresponding connector.
If the check box is deactivated Sleep Wakeup Signals on one hand of the connection are not forwarded to the corresponding connector.

Link transparency

  • Enabled: When one hand of the link changes it's state e.g. from up do down the corresponding channel is set link down, too.
  • Disabled: When one hand of the link changes it's state e.g. from up do down the corresponding channel does not change his state automatically.

When should the transparency settings be overwritten at the start of a measurement?

  • If you want to change the behavior for link transparency that you have configured as default for the selected hardware.
  • If you want to change the behavior for link transparency for selected channels without changing the behavior of the remaining channels.

Settings

The Ethernet default configuration of the VN hardware can be overwritten by CANoe at the start of a measurement. To activate this function activate the check box Use this Ethernet configuration during measurementâ¦, and configure the settings.

See also Getting Started with VN5640 (PDF)

When should the Ethernet default configuration be overwritten at the start of a measurement?

  • If you want to make sure that CANoe automatically configures the hardware correctly at the start of a measurement. The CANoe configuration can be used for any VN hardware, if it supports the desired settings (e.g. number of used hardware channels).

  • If you want to manipulate an existing network topology after the start of a measurement.

    		•

    Example

    • You have configured a measurement point (TAP) which operates in PHY Bypass mode (packages are transferred with minimum latency between the two endpoints connected to the TAP).
    • After the start of measurement, additional packages shall be sent and certain packages shall be filtered by use of CANoe.
    • For this, you configured a TAP with a deactivated bypass (Bypass Off) in CANoe.
    • As a result, CANoe will switch off the bypass at the start of measurement. After the measurement has been stopped, the system will continue to operate without any disturbances introduced by the CANoe application, because then the PHY Bypass mode is set again.

Transfer Mode

Here you can define the transmission behavior of the hardware segment. After your selection, the relevant parameters for the configuration are displayed. At the start of measurement CANoe checks whether the selected settings can be implemented. If there are any incompatible settings, the start of the measurement is aborted and an error message is displayed.

Example

In the Vector Hardware Config dialog (opened via [Driverâ¦] button) you have assigned an application channel to a hardware segment TAP. After that, if you select the transfer mode Switch in the Network Hardware Configuration dialog and configure three hardware channels, the start of the measurement is aborted, because the hardware segment only provides two hardware channels.

Background: CANoe cannot change the segmentation on the hardware and the allocation of hardware channels.

The following transfer modes are available:

  • Switch
    Use this setting to configure up to 16 hardware channels. The switch learns MAC addresses and the packets are transmitted based on these rules.
  • Y Cable
    Use this setting to configure a measurement point (simple TAP or media converter) including two hardware channels.
  • Direct Access
    Includes exactly one hardware channel and is typically used for the direct access to the network.

  • Direct Access (with TAP)
    In older CANoe versions (< version 9.0) measurement points were implemented using two application channels. If older configurations are to be used on recent VN hardware, select the operating mode Direct Access (with TAP).

    		•

    Note

    Use this mode only, if you cannot adapt the old configurations. In new configurations measurement points should always be configured in Y Cable mode.

Bypass

Set the bypass method to be configured in the TAP at the start of measurement.
This option is available in the following transfer modes:

  • Y Cable
  • Direct Access (with TAP)

Eth x.x (<application channel>.<hardware channel>)

Configure the hardware channels, which are assigned to the hardware segment in the Vector Hardware Config dialog (opened via the [Driverâ¦] button). The number of hardware channels to be configured is set using the buttons [+] and [-]. At the start of measurement, the hardware channels for the hardware segment are configured on the hardware in ascending order, i.e. from left to right. When configuring, the first free hardware channel will be used for which the setting can been made.

The first matching hardware channel for Setting 2 is the hardware channel 13, since all previous channels are IEEE 100BASE-T1 (BroadR-Reach) channels.
