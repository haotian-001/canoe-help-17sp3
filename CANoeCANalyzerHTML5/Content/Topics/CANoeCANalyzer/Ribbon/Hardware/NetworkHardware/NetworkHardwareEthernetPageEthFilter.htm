Network Hardware Configuration: Filters

Ribbon: Hardware Tab|Network Hardware|Eth|Filters

Settings

Filter

 

Only available with channel-based network access.

On this page you can define independent acceptance filters for the IEEE 802.3 fields Destination MAC Address, Source MAC Address and Ethertype to see only relevant packets and/or improve the performance. Further you can use IPv4 addresses and TCP/UDP ports as filter criteria.

Acceptance Filters

With the button [Add...] you open the dialog Add filter, in which you can define new filters and add them to the list of active filters. For a new filter you have to specify the following values:

  • Field: select the type of the filter in the
  • Codelist: acceptance code, the relevant bits (specified by the acceptance mask) of the received packet are compared with that code
  • Mask: acceptance mask, which specifies the bits of the acceptance code that are compared

Note

During the measurement the defined filters are applied in the same order as they are defined in the filter list.

With the button [Edit...] you can modify the values of the current selected filter in the list.

If the list is empty, no acceptance filter are defined so that all network packets on this channels are blocked and no incoming network data can be received.

You can remove one or all defined filters with the buttons [Remove] and [Remove all].

If you activate the Accept all packetscheck box, all filters in the list are deactivated.

Procedure to Accept or Reject an Ethernet Packet

Note

The acceptance filtering does only have any effects on received packets (Rx).

The acceptance mask indicates which bit of the corresponding field value in the received packet should be compared with the acceptance code. If the bit in the mask is 0, then that particular bit is irrelevant for the comparison. Otherwise the bit of the ID is compared with the corresponding bit of the acceptance code. If they are equal the packet is received, else it is filtered out.

Example

Filter definition: Destination MAC address, Code 00:16:81:00:00:00, Mask FF:FF:FF:00:00:00

Effect: Only packets with a Destination MAC address beginning with 00:16:81 (regardless the following 3 bytes) are accepted.

Example

Filter definition: Ethertype filter, Code 0x800

Effect: Every packet with an Ethertype field value of 0x800 (IPv4 protocol) is accepted.

The comparison of the bits is done for all defined filters in the active list. If one of them accepts the Ethernet packet, it is passed. If no one accepts it, the packet is rejected.
