MOST High Observer Configuration

Ribbon|Analysis Tab|High Observer

The MOST High Protocol Observer detects violations of the MOST High Protocol (MHP). It analysis all MHP connections and generates MHP Error events in case of timeout and retry violations or illegal communication flows. These events can be shown in the Trace Window or received in CAPL programs placed in the Measurement Setup. The Observer also collects statistical information about NegAcks as well as block and frame requests.

The MHP Combiner collects all payloads which are transported over a MOST High connection. After completion of a block (see MOST High specification for the terminology) an MHP Block event is shown, which contains some statistical data about the transmission as well as the time stamp of the last frame belonging to this block.

A MHP Packet event is generated after transmission of a MHP packet. It contains the payload of the transmission if the Combiner was active and statistical information about errors and warnings if the Observer was active. The Trace Window can display a disassembly of the user data of the MHP transmission if the message is described in the function catalog and as long as certain limitations in size are not violated (see Technical Background). A MHP Connection event is generated at the end of a MHP connection, which contains statistical information about all packets and blocks that were transmitted.

Note

The observer and combiner events are only available in the Measurement Setup and can be analyzed in CAPL.

The MHP Observer and Combiner are purely passive, so that in case of erroneous MHP connections the communication is neither disturbed nor corrected.

Configuration

The settings for the MHP Observer and Combiner can be modified via the Analysis ribbon tab|MHP Observer Configuration. The settings in this dialog are stored in the MHPObserver.INI configuration file contained in the installation folder of your CANoe installation. It is also possible to save the file to the folder containing the configuration. In this case the local INI file is used instead of the file in the installation folder.

You can specify the settings for MHP versions 2.1, 2.2 and 2.3 in the configuration dialog. The settings are located on the following tabs:

MOST High Observer

On this page you can activate the Observer and modify the timeout and repetition settings. For each value tolerances (min / max) can be set, beyond which the Observer reports an error. Timeouts are given in [ms].

In addition, you can specify whether only errors (default) or errors and warnings are to be reported by the Observer. If you don't want certain errors to be reported, you can deactivate them here. Since the Observer may report several errors in the case of an incorrect MHP message, it is possible to limit the number of error messages (default: only one error message per situation).

MOST High Combiner

On this page you can activate the Combiner and specify the maximum amount of data to be collected by the combiner. These settings are independent of the MHP version.

General

The settings on this page are related to the Observer and the Combiner. You can modify the number of MHP connections to be monitored at a time and specify a timeout after which a connection will no longer be monitored.

Important Note

Changes to the default Combiner settings may increase the need of memory (RAM) of CANoe dramatically and may have impact on the performance of the tool.

Technical Background

As soon as the Observer or the Combiner is activated, they examine the control messages and the asynchronous packets of each MOST channel for MHP connections (TelIds 8 and 9). This works in node mode, spy mode or both.

MHP connections are identified based on the MOST channel, Node/Spy-Attribute, Source Address, Destination Address, Function Block, InstanceId, Function and OpType. Multiple connections can be observed in parallel.

The MHPPacket event is suited to display statistical information and the payload in the Trace Window and as soon as an attached function catalog describes the corresponding message, a disassembly of the payload can be shown. Without function catalog support a hex dump of the data is shown. The detail view of the Trace Window shows either hex dump or disassembly according to the state of the ‘sym’ button in the toolbar.

As soon as the maximum amount of data (default: 64 kByte) is reached, the combiner disposes of the collected user data and stops collecting in order to prevent memory shortage caused by huge transmissions, as used e.g. for flashing ECUs. The detail view then shows an appropriate hint. To collect data exceeding this limit, CAPL nodes can be used.
