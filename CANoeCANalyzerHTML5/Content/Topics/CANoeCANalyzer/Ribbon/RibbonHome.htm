Start of Measurement without Logging

Via this button you can also start the measurement without writing any logging file, without individually deactivate each Logging Block in the Measurement Setup.

When the measurement is started via

  • all active Logging Blocks are deactivated
  • all inactive Logging Blocks remain inactive

At the end of the measurement, all blocks that were already activated before the measurement are reactivated.

Offline Mode

Replays the logged measurement data of a logging file. The entire measurement data will be displayed in the analysis windows.

Stop

You can stop the measurement by

  • this symbol
  • pressing <Esc> (see also Options dialog)
  • exceeded post-run time for the Logging block
  • calling the stop() function in any CAPL program

Under heavy load conditions, stopping can take a while. This is due to either buffers and queues that have to be flushed at the end of the measurement or simulation nodes that defer the stop. Double clicking the stop button prevents the buffers from being flushed and thus yields an immediate stop.

Animate

Activates an automatic stepwise data output in Offline Mode.

Instead of reading the data as quickly as possible from the source file, during animation the messages are read from the offline source file with delays. This results in a slow-motion presentation of events. You can set the value of the delay factor in the file CAN.INI.

To do this, enter the following line in the section [OFFLINE]:

AnimationDelay = nnnn

  • The value nnnn describes the delay factor in milliseconds.
    The default value is 300 ms.
  • The key combination is <F8>.

Note

  • The animation delay means that after nnnn ms the next event will be read from the source file.
  • The readout of the events is independent of the stored time stamps.
  • The time stamps of the source file are displayed in the Trace Window.
  • You can only enter positive integers.

Note

The command has nothing to do with the animation factor, set in Options dialog.

  • The animation factor acts only in online mode in the case of pure simulation mode without hardware.
  • The Animate command acts only in offline mode.

Step-length of time step

Online Mode - Working Mode Simulated Bus

In online mode you can enter the length of the time step in ms. The largest possible step is 60000 ms.

Offline Mode

In offline mode, you can also use the units h, m, s, and ev by entering them in the field after the corresponding number. The unit ms is hidden because it is the default value.

You can enter the step-length for the stepwise data output directly in the field, or you can choose an existing entry from the list.

Note on Step Mode

  • The last message of a step always lies outside of the specified time span. This prevents a situation where no message is displayed in the current step if the next event has a larger time stamp.
  • If a number of events are specified using the ev unit, exactly this number will be read from the logging file. However, additional events (e.g., via observers or the MOST AMS service) can be displayed in the Trace Window.
  • If the logging file has been completely read in, a corresponding note appears in the Write Window at the next single step.

Vector Tool Platform

Here you can configure devices running the Vector Tool Platform (for example, VN8900 series devices) that you want to use in standalone mode or distributed mode. The realtime relevant part of the configuration runs on the external device, which leads to improved realtime properties.

With a click on the symbol you open the Vector Tool Platform configuration dialog.
