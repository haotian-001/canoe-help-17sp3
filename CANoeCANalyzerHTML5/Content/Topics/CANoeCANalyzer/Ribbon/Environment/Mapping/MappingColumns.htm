Symbol Mapping: Columns

 

Column

Description

Activation

If a mapping relation is marked active, the source and destination of the mapping relation are mapped on measurement start, dependent on the settings in the Mapping column.

Icon

The symbol (variable/signal) is displayed here:

  • : system variables
  • : signals
  • : Values of communication objects
  • : Values of distributed objects
  • : namespaces of system variables

Destination/Source

The Destination and Source columns contain the two symbols of the mapping relation that are mapped on measurement start.
The following symbols are available:

  • System variables
  • Signals
  • Values of communication objects
  • Values of distributed objects
  • Namespaces of system variables

Mapping

In this column you define when the source symbol should be linked to the target symbol:

  • OnChange: The source symbol will be only mapped on the target symbol, if the source value changes and will be different to the target value.
    Valid for signals and system variables, namespaces of system variables.
  • OnUpdate: The source symbol will be always mapped on the target symbol. It is irrelevant, if the symbol values (source or target) are identical or different.
    Valid for signals, system variables, namespaces of system variables.
  • OnTimerValueChanged: The source symbol will only periodically be mapped on the target symbol after the given timer elapsed, if the source value changes and is different to the target value.
    Valid for signals and system variables, namespaces of system variables.
  • OnTimerValueUpdated: The source symbol will always periodically be mapped on the target symbol after the given timer elapsed. It is irrelevant, if the symbol values (source or target) are identical or different.
    Valid for signals, system variables, namespaces of system variables.

Cyclic Timer (ms)

In this column, you specify how often a mapping between the source and target symbols should happen. The given timer value is only considered when the mapping is set to OnTimerValueChanged or OnTimerValueUpdated.

Factor

The mapping of the values is done using a linear conversion:

<target value> = <source value> * factor + offset

x/+/= show how the target value is calculated from the source value, factor and offset.

On creating a new mapping relation the initial factor is 1 and the offset is 0. This implements a 1:1 mapping of the values.
