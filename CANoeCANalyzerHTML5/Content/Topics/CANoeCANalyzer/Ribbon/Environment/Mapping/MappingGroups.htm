Symbol Mapping: Groups

 

Groups

You can combine the mapping relations in groups in order to improve the overview. Each group can be assigned an individual name.

If a group is activated, all activated mapping relations of this group will be executed. If a group is deactivated, all of its mapping relations will be regarded as deactivated and therefore not be executed.

You can create groups using in the toolbar.

Mapping Sets

You can use mapping sets to configure various mapping relations in groups and change them during the measurement.

Active mapping relations that have been configured in the Static Mapping Set will always be executed.

Active mapping relations that have been defined in a Dynamic Mapping Set can be activated during measurement using the CAPL function SymbolMappingSetDynamicMappingSet. There can be only one dynamic mapping set active at the same time.

You can create mapping sets using in the toolbar. You need to activate the mapping set in the Options dialog in order to create it.

Note

If you activate the dynamic mapping set in the Options dialog, all previously created groups will be moved to a static mapping set. All newly created mapping sets are dynamic.

Binding Groups

Binding groups are used especially for members of distributed objects whose binding is set to Mapping. With this binding you can conveniently link the DO member value to e.g., a VT System system variable or an XCP parameter.

With such a DO member, the value is transferred to another symbol by the mapping:

  • A consumed data, event or field member can be configured as the target of the link if it is bound by mapping. The value of such a member is normally read-only in the configuration and can only be used as source in a normal mapping relation because it is set by the binding.
  • A provided data, event or field member can be configured as source of the link if it is bound by mapping. The value of such a member can only be used as target in a normal mapping relation.
  • A prerequisite is that the communication pattern is not PublishSubscribe because the subscription cannot be transferred by mapping.
  • Method members cannot be configured either as source or target of the link; they cannot be bound by mapping.

If the bound DO member is the source of the link, the Mapping and Cyclic Timer columns are read-only. The value is always transferred to the target when the DO member is to be transferred, which depends on the value of the TxTrigger attribute.

Note that the raw value of the DO member is always linked. Therefore, you cannot specify a linear conversion for binding groups; instead, use a linear conversion from raw to physical value of the DO member.

You can add the name of the binding group to the DO member or its DO in the MappingBinding::Group attribute. Then the binding group is automatically deactivated when you switch the binding for all such members, e.g. by changing the use case in Communication Setup. It is also automatically reactivated when you switch the binding back to Mapping for such a member.

Binding groups can be created using the toolbar. They are only available if you use DOs in the configuration.
