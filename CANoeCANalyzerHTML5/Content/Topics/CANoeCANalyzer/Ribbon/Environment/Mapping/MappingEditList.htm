Symbol Mapping: Editing the Mapping List

 

Note

Cyclical assignments are only allowed for system variables.

The following conditions have to be met:

  • The assignments have to be done with Factor=1 and Offset=0.
  • At least one mapping must contain the OnChange mapping relation.

Groups and Mapping Sets

You can insert and delete groups and mapping sets via the toolbar.

You can move and copy groups within the dialog with drag and drop. To copy them press additionally <Ctrl>.

With a double click in the Name column, you can rename each group and each mapping set.

To create mapping sets, you must activate them in the Options dialog.

Adding/Deleting Mapping Relations

You can insert variables, signals, values of application layer objects or system variable namespaces in the list as follows:

  • Keyboard
    Start entering (a part of) the name of the symbol in the Destination or Source columns.

    A list of all available symbols containing the typed text string is displayed. You can select the desired symbol using the up and down arrow keys. The symbol is inserted with the <Tab> or <Enter> key.

    Note

    For list display, note the settings for array elements of distributed objects in the Options dialog.

    With <Del> you can delete all selected mapping relations from the mapping list.
    With <Ctrl>+<C>, <Ctrl>+<X>, <Ctrl>+<V> you can copy and cut configured mapping relations and add it to a new line in the mapping list.
    With <Ctrl>+<a> you can select all lines of the mapping list.


  • This button is used in the Destination and Source columns to open the following menu:

  • Add Application Layer Object...

    Adds values of application layer objects.

    Note

    For list display, note the settings for array elements of distributed objects in the Options dialog.
  • Add variable…
    Adds system variables.
  • Add signal…
    Adds signals.
  • Add System Variable Namespace...
    Adds a system variable namespace.

    • The Symbol Selection dialog is opened. In so doing, the displayed symbols are pre-filtered to contain just the relevant types (system variables, signals, values of application layer objects, system variable namespaces).

  • Symbol Explorer
    You can find it at the right of the Symbol Mapping dialog.

    You can drag signals, system variables, values of application layer objects and system variable namespaces from the Symbol Explorer to the list of mapping relations.
    Single symbols can be dropped directly on the Destination and Source columns of already configured lines.
    To add new lines to the list, you can drag one or several symbols and drop them either on an empty line or on the area below mapping list.
    In contrast to opening the Symbol Selection dialog via , multiple symbols can be selected here. These symbols are then appended as new lines to the list of mapping relations.

    In the Symbol Explorer you can also copy a symbol into clipboard. For this, use the context menu or keyboard (<Ctrl> + <C>). After that you can paste the symbol in the columns Destination or Source in an existing mapping relation. For this, use also the context menu or keyboard (<Ctrl> + <V>).

  • Context Menu
  • Toolbar

Editing Factor and Offset

You can enter factor and offset in the equally named column.

  • If several rows are selected and you change the factor or offset in one of the selected rows, this new value is automatically set in all selected rows.
  • With the <Ctrl>+<a> key combination you select all rows in the list.
  • Using the context menu you can access the Formula Assistant for easier input of factor and offset.
