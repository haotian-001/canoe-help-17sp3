Configuration group

Measurement Setup

The data flow is displayed graphically in the Measurement Setup.

All options for parameterizing the Measurement Setup are selected in this window.

With a click on the symbol you open the Measurement Setup.

Offline Mode Window

In the Offline Mode Window, you can add and remove message-based logging files, video files and image series which should be run in the offline mode.

With a click on the upper button you open the Offline Mode Window.

Via the lower button you can configure the offline mode.

Split Button

You can recognize a split button by the split functionality displayed on the left.

You open the corresponding window using the Symbol. Other functions are available via the Text.

Please note the provided information since in some cases the other functions are only available under certain preconditions.

Filter

In the Measurement Setup, you have various filters available that you use to specify which data are to be allowed in the analysis window and which data are to be explicitly blocked.

You can add and delete filters in the Measurement Setup.

Here you open event filter added in the Measurement Setup.
Available event filter:

  • CAN Filter
  • J1939 Filter
  • J1587 Filter
  • FlexRay Filter
  • LIN Filter
  • MOST Application Filter
  • MOST Filter
  • AFDX Filter
  • IP Filter
  • A429 Filter

Logging

For an analysis after the measurement, you can also record data and replay it later.

You can add and remove logging blocks in the Measurement Setup.

For logging files configured in the Measurement Setup you can here

  • start or pause the data recording during the measurement
  • configure the Trigger conditions for the data recording
  • configure the logging file

Bus Analysis group

Trace Window

The purpose of the Trace Window is to record bus activities during measurement. All the messages received at the input of a Trace block are displayed as lines of text in the Trace Window.

Here you open Trace Windows added in the Measurement Setup.

Additionally you can insert preconfigured Trace Windows in the Measurement Setup.

Graphics Window

In the Graphics Window symbols (signals, variables, and diagnostic parameters) are displayed graphically in an XY diagram. You can configure the time or any symbol as the X-axis.

Here you open Graphics Windows added in the Measurement Setup.

Additionally you can insert Graphics Windows in the Measurement Setup.

Data Window

The Data Window is used to display signal values.The signal value is displayed as a physical value, as a raw value and with a progress bar by default.

Here you open Data Windows added in the Measurement Setup.

Additionally you can insert Data Windows in the Measurement Setup.

State Tracker

The State Tracker is an analysis window that shows bit values and states. It is particularly suitable for displaying digital inputs/outputs as well as status information like terminal states or network management states.

Here you open State Trackers added in the Measurement Setup.

Additionally you can insert State Trackers in the Measurement Setup.

FlexRay Cluster Monitor

The Cluster Monitor shows the statistics and bus activities of a FlexRay cluster during a measurement. The real data of the network traffic are compared with the data from the FIBEX model, and deviations or differences are registered and displayed.

Here you open a FlexRay Cluster Monitor added in the Measurement Setup.

Additionally you can insert the FlexRay Cluster Monitor in the Measurement Setup.

System Scanner

With the System Scanner active nodes in the network can be displayed and analyzed.

Here you open System Scanners added in the Measurement Setup.

Additionally you can insert System Scanners in the Measurement Setup.

Possible System Scanner

  • J1939 Scanner
  • CANopen Scanner

J1587 Parameter Monitor

The J1587 Parameter Monitor shows all parameters which are transmitted from the nodes of a J1587 network. The parameters are ordered by the sender (alt. sending) nodes. A symbolic representation is available if the assigned database contains the necessary information.

Here you open J1587 Parameter Monitors added in the Measurement Setup.

Additionally you can insert J1587 Parameter Monitors in the Measurement Setup.

BAP Channel

BAP (control and display protocol) is a protocol that was developed for standardizing interfaces in the Comfort and Infotainment area.

With a click on the symbol you open the dialog for the BAP configuration.

Statistics Window

The Statistics Window displays statistics about bus activities during measurement.

Here you open Statistics Windows added in the Measurement Setup.

Additionally you can insert Statistics Windows in the Measurement Setup.

Possible Statistics Windows

  • AFDX Statistics Window
  • CAN Statistics Window
  • Bus Statistics Window
  • FlexRay Statistics Window
  • LIN Statistics Window

More Analysis group

Map Window

With the Map Window you can display and evaluate logged position information from configured GNSS receivers on a map. In addition, you can use CAPL or .NET to draw your own objects on the map.

With the following options, the basic scope of the Map Window is extended by additional functionalities.

  • With the Option Car2x you can display location coordinates of ITS objects and events received in a Car2x network on the map.

  • With the Option J1939 or the Option ISO11783 you can display and evaluate position information from J1939 or NMEA parameter groups. The Fast Packet Protocol is supported as well.

Here you open Map Windows added in the Measurement Setup.

Additionally you can insert Map Windows in the Measurement Setup.

Video Window

You can use the Video Window to record and play back video files.

Here you can

  • configure new and existing Video Windows
  • open Video Windows

Scene Window

ADAS objects can be visualized with the Scene Window. Sensors, detected objects and ground truth data are displayed.

With a click on the symbol you open the Scene Window.

AI Visualizer

With the AI Visualizer (AI=Artificial Intelligence), data such as images and AI results exchanged between an AI component and CANoe can be visualized.

With a click on the symbol you open the AI Visualizer.

Scope Window

Signal level on CAN, LIN and FlexRay bus are displayed graphically here.

With a click on the symbol you open the Scope Window.

GNSS Monitor

The GNSS Monitor in the Measurement Setup is used for graphical and numeric display of degrees of longitude and latitude as well as to specify the height of a GNSS message in the NMEA2000Â® or J1939 protocol.

Here you open GNSS Monitors added in the Measurement Setup.

Additionally you can insert GNSS Monitors in the Measurement Setup.

Protocol Monitor

The Protocol Monitor observes data traffic in the network and interprets a variety of protocols supported by CANoe. The connections between endpoints are displayed graphically and in tabular form. In a hierarchical tree view, all endpoints are broken down by their affiliation to network nodes or to networks.

Here you can activate and deactivate the Protocol Monitor.

  • Use Activate Protocol Monitor to activate the evaluation of the network traffic and open the window. If the Protocol Monitor is activated, the Ethernet communication of the nodes during measurement is evaluated. You do not need to open window for this. Even if the window is displayed on different desktops, the same content is always shown.
  • Use Deactivate Protocol Monitor to deactivate the evaluation of the Ethernet communication and close the window.

MOST group

MOST Symbol Explorer

In the MOST Symbol Explorer you select the symbol you want to insert in a Data or Graphics Window. You can insert the selected symbols in the windows using a drag and drop operation.

With a click on the symbol you open the MOST Symbol Explorer.

MOST Central Registry Window

The MOST Central Registry Window displays the bus registry of the MOST ring in a clear manner. It also offers additional analytical possibilities.

With a click on the symbol you open the MOST Central Registry Window.

MOST Audio Routing Window

The MOST Audio Routing Window shows the distribution of connection labels in the resource allocation table of the connected node.

With a click on the symbol you open the MOST Audio Routing Window.

MOST FBlock Monitor

The MOST FBlock Monitor displays the MOST device assignment, the reported functions and the most recently observed property values for all function blocks whose communication on the MOST ring has been observed by the spy.

With a click on the symbol you open the MOST FBlock Monitor.

MOST System Viewer

The MOST System Viewer provides an overview of all connected MOST devices. To obtain this information, the MOST System Viewer monitors the ring scan of the MOST NetworkMaster and other events on the MOST ring.

With a click on the symbol you open the MOST System Viewer.

MOST High Observer Configuration

The MOST High Protocol Observer detects violations of the MOST High Protocol (MHP). It analysis all MHP connections and generates MHP Error events in case of timeout and retry violations or illegal communication flows. These events can be shown in the Trace Window or received in CAPL programs placed in the Measurement Setup. The Observer also collects statistical information about NegAcks as well as block and frame requests.

With a click on the symbol you open the MOST High Observer configuration dialog.

MOST State Window

MOST State Window gives the user a perspective on important system and node states on a MOST ring. In the MOST State Window one status line is shown for each MOST channel in CANoe.

With a click on the symbol you open the MOST State Window.

Car2x group

Protocol Analyzer

The Car2x Protocol Analyzer checks data content and timing of all Car2x frames against an extensible set of rules.

With a click on the symbol you open the Protocol Analyzer.

Certificate Explorer

With the Car2x Certificate Explorer you can analyze certificates that are known at runtime.

With a click on the symbol you open the Certificate Explorer.
