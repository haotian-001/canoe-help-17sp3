PDU Interactive Generator (PDU IG)

The PDU Interactive Generator (PDU IG) allows you to send configurable AUTOSAR PDUs over any network, that supports sending AUTOSAR PDUs (CAN, Ethernet FlexRay). You can set signal values within the AUTOSAR PDUs manually or automatically via the global Signal Generators.

A configuration can contain any number of PDU IG windows. You can insert or remove a PDU IG window from the configuration using this button:

  • To insert, click the PDU Interactive Generator entry under New. A new window with the name PDU Interactive Generator appears.

  • To remove, click on the button with the three dots behind the name of the PDU IG window and select the Remove command.

You can give each PDU IG window its own name. For more information, see the description for configuring the PDU IG.

Only available if minimum one network is used, that supports the sending of AUTOSAR PDUs (CAN, Ethernet, FlexRay).

ISO11783 Virtual Terminal Fenster

The Virtual Terminal Window provides a ready-to-use virtual terminal instance without any additional CAPL programming. VT window offers the possibility to visually control the data masks as well as the contents of object pools. In addition, the ISOBUS Shortcut Button (ISB) is supported.

You can add and remove an ISO11783 Virtual Terminal in the Simulation Setup.

Here you open ISO11783 Virtual Terminals added in the Simulation Setup.

ISO11783 Interactive Task Controller

The Interactive Task Controller can be used to access the process data dictionary of an implement. The process variables can be set or read out. The process variables are selected symbolically after the device description has been transferred by the implement. When a task is started, the process variables of the default treatment zone are transmitted to the implement and the peer control assignments are set.

You can add and remove an ISO11783 Interactive Task Controller in the Simulation Setup.

Here you open ISO11783 Interactive Task Controllers added in the Simulation Setup.

CANopen Network Management (CANopen NMT)

Note

You can send NMT commands from the toolbar only if you have a CANopen Interactive Generator in the assigned network.

Only available during measurement.

send command NMT Start (NMT Operational)

send command NMT Pre-Operational

send command NMT Stop

send command NMT Node Reset

On clicking one of the buttons, a menu is opened. In this menu you can consecutively select on which channel you want to send the NMT command and which node is the receiver.

  • Channel
    You have to select a channel only if you have assigned several CAN hardware channels.
  • Node selection
    You can select one or more nodes to be the receiver of the NMT command:
  • All nodes
    Nodes that are described in the assigned CAN database
  • Nodes...
    Nodes that are not described in the assigned CAN database - in this case you can send NMT commands in the dialog Send CANopen NMT command only

GNSS Simulator

The GNSS Simulatorcan be used to simulate a GNSS receiver. To do this, a geometric figure or a position file is used to determine a path that will be traced at a variable speed when the simulation is started. GNSS positions thereby generated are transferred cyclically with configurable messages.

You can add and remove a GNSS Simulator in the Simulation Setup.

Here you open GNSS Simulators added in the Simulation Setup.
