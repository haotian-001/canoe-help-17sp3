Security Configuration

Ribbon: Simulation Tab|Security Configuration

In this dialog, you can define which Security Profiles shall be used by CANoe. There are different security use cases like Secured Onboard Communication (SecOC), Diagnostics, Backend Access, TLS and TLS observation and IPSec. Depending on the security use case, you have to assign a Security Profile to networks, stacks or the TLS Observer.

You can create and manage Security Profiles in the Vector Security Manager. Only the ID of the profile is stored in the CANoe configuration. At no time the data of the profile is exported to CANoe.

Search in Security Profile

In all Security Profile drop down lists, you can search for Security Profiles by selecting the drop-down list and entering a search text. All profiles including the entered string are shown in the profile list. All profiles that are not relevant will be hidden.

Toolbar

  • Remove Security Profile
    Use this button to remove the selected Security Profile assignment.
  • Refresh List of Security Profiles
    Synchronizes the list of available Security Profiles with the Vector Security Manager.
  • Open Security Manager
    Use this button to open the Vector Security Manager.
    The Security Profiles are managed and configured in this tool.

SecOC and Diagnostics

On this page, you can assign Security Profiles to networks of the Simulation Setup. The Security Profile is used for SecOC, Diagnostics and Backend Access.

The list of Security Profiles contains the following columns:

Column Header

Description

Network

Contains the name of the CANoe network.

Channel

Describes the assigned channel of the network.

Use Security

Use this check box to activate or deactivate the security function.

If the check box is deactivated, SecOC, Diagnostics and Backend Access are not possible. The assignment of the Security Profile is retained, however.

Security Profile

The list contains all available Security Profiles that can be used for SecOC and Diagnostics.

TLS and IPSec

On this page, you can assign Security Profiles to simulation nodes. Assigning a Security Profile to a stack allows you to do TLS or IPSec at this simulation node. Only Security Profiles compatible with TLS or IPSec are offered.

The list of PKI-Security-Profiles contains the following columns:

Column Header

Description

Stack

Contains the name of the stack.

Used by Node

Lists all node names using the stack.

Use Security

Use this check box to activate or deactivate the security function. If the check box is deactivated, TLS or IPSec is not possible. The assignment of the Security Profile is retained, however.

Security Profile

The list contains all available Security Profiles that can be used to simulate TLS endpoints or to do IPSec protected communications.

TLS Observer

On this page, you can assign a Security Profile to be used to observe TLS communications. All TLS communications are observed with this Security Profile. Only Security Profiles compatible with TLS are offered.

Column Header

Description

Use Security

If the check box is deactivated, no TLS observation is performed. The assignment of the Security Profile is retained, however.

Security Profile

The list contains all available Security Profiles that can be used for TLS observations.

MACsec

On this page you can assign a security profile to be used for securing Ethernet communication with MACsec. This security profile must contain valid MACsec port configurations. Each port that is to be protected with MACsec must be assigned a MACsec port configuration from the security profile.

Column Header

Description

Network

Here the name of an Ethernet network configured in the hardware is displayed. In CANoe this is the name of the simulation network, in CANalyzer the name of the application channel.

Measurement Port

The name of a physical port that was previously activated as a measurement port in the CANoe Port Configuration dialog is displayed here.

Use Security

Use this check box to activate or deactivate the security function.

When the check box is deactivated, the port does not support MAcsec. However, the MACsec port configuration assignment is retained.

MACsec Port Configuration

The list contains all available MACsec port configurations from the security profile that can be used.

Connectivity

On this page you can see which security profiles are available for connectivity links. This page is a display page only and lists all the security profiles that have been assigned in the VCDLl code.

Column Header

Description

Security Profile

The list contains all available security profiles that can be used for connectivity.
