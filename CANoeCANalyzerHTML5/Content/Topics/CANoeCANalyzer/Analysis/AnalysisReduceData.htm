Reduce Data

 

For a clear presentation of the data and to achieve a higher data throughput, you have the following options in CANoe:

Filter in Measurement Setup

In the Measurement Setup, you have various filters available that you use to specify which data are to be allowed in the analysis window and which data are to be explicitly blocked.
For example, you can reduce your data by individual signals or by the data stream from whole channels.

All filters can be used as stop and Pass Filter:

  • Stop filter: The configured data are filtered out from the data stream; all other data are allowed by the filter.
  • Pass filter: Only configured data are allowed by the filter; any other data are filtered out.

Filter Overview of Measurement Setup

Note

Filter can be inserted in the analysis area and the real-time area.

In the real-time area they optimize the throughput.

The earlier the data is reduced, the more performance is gained. Therefore, place the filters as far forward as possible in the measurement setup.

In the CANoe options under Performance, you can also activate a way to filter data before it reaches the analysis area.

Short Description

Channel Filter

With a Channel Filter it is possible to completely block or open the channels of all registered bus systems (CAN, LIN, MOST, FlexRay).

The Channel Filter appears in the Measurement Setup as a small block with a number of red and green lines. These lines symbolize the individual channels of a bus system and their momentary states. A green line means an open channel, while a broken red line indicates a blocked channel.

Event Filter

CAN Filter

The CAN Filter lets you filter CAN messages. It has a number of advantages over standard filter:

  • Special CAN parameters for filtering
  • Ability to add/remove filters during a measurement
  • Ability to manipulate parameters during a measurement

The CAN Filter appears in the Measurement Setup as small block with the label CFB.

LIN Filter

The LIN Filter lets you filter LIN frames. It has a number of advantages over standard filter:

  • Special LIN parameters for filtering
  • Ability to add/remove filters during a measurement
  • Ability to manipulate parameters during a measurement

The LIN Filter appears in the Measurement Setup as small block with the labelLFB.

MOST Application Filter

The MOST Application Filter can be used to filter various MOST messages and events. The filter always ignores non-MOST messages and events. It has a number of advantages over MOST Filter:

  • Message and event oriented filtering
  • No CAPL programming required

The MOST Application Filter appears in the Measurement Setup as small block with the label MAFB.

MOST Filter

The MOST Filter can be used to filter a wide variety of MOST messages and events. The filter always allows all non-MOST messages and events (e.g. CAN messages) to pass.

The MOST Filter appears in the Measurement Setup as small block with the label MF.

FlexRay Filter

The FlexRay Filter lets you filter FlexRay frames. It has a number of advantages over standard filter:

  • Special FlexRay parameters for filtering
  • Ability to add/remove filters during a measurement
  • Ability to manipulate parameters during a measurement

The FlexRay Filter appears in the Measurement Setup as small block with the label FFB.

J1939 Filter

The J1939 Filter lets you filter J1939 messages. It has a number of advantages over the standard filter:

  • filtering for J1939 parameters groups
  • filtering for static addresses
  • filtering for dynamic addresses by using the J1939 device name
  • filtering for priority

The J1939 Filter appears in the Measurement Setup as small block with the label JFB.

IP Filter (Pass and Stop Filter)

The IP Filter lets you filter Ethernet packets and remove them from the data stream.
The IP Filter only affects Ethernet packets. All other events, e.g. CAN messages, are not filtered, rather they are passed.

The IP Filter appears in the Measurement Setup as small block with the label IFB.

J1587 Filter (Pass and Stop Filter)

The J1587 Filter lets you filter J1587/J1708 messages. It has a number of advantages over the CAN Filter:

  • filtering for J1587 parameters
  • filtering for J1708 message identifications

The J1587 Filter appears in the Measurement Setup as small block with the label 1587.

AFDX Filter (Pass and Stop Filter)

With the AFDX Filter you can remove AFDX messages and frames from the data stream.

A429 Filter (Pass and Stop Filter)

With the A429 Filter you can remove ARINC words from the data stream.

All Protocols (Standard filter)

The filter is used to intentionally reduce the volume of data. Furthermore, individual messages and network nodes can also be configured in the filter. Depending on the installed option, the access to several database is possible, e.g. CAN, LIN, MOST.
Moreover, the type of messages to be filtered can be configured, whereby it is also possible to observe Error Frames.

Pass filter appear in the Measurement Setup as small blocks with the label PF, Stop Filter with the label SF.

Variables Filter

Pass/Stop Filter

With this filter variables (environment and system variables) can be filtered.

Pass filter appear in the Measurement Setup as small blocks with the label PE, Stop Filter with the label SE.

Programs in Measurement Setup

With CAPL you can program your own filter and trigger conditions.
A program node appears in the Measurement Setup as small block with the label P.

Trigger Conditions in Measurement Setup

You can use different trigger conditions to influence the logging of data in a logging file in the Measurement Setup.
A trigger appears in the Measurement Setup as small block with the label T.

Filter in Trace Window

In the Trace Window you can reduce the amount of data for the analysis using various filters both during and after the measurement.
For example, you can access predefined filters, filter for individual signals and signal values, or set filters for various columns.

Hardware Acceptance Filter

The network hardware uses acceptance filtering to control which received messages are passed on to CANoe.
This filtering occurs at the very beginning of the data flow.
