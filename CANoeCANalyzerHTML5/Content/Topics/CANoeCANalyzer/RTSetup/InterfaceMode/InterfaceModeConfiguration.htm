Configuration and Operation

 

A VN8900 device is used in interface mode when the following requirements are met:

  • You have assigned application channels that are used in the CANoe configuration to hardware channels of the VN8900. The channel assignment for the interface mode is set in the Vector Hardware Configuration on the computer. The channel assignment on the device, which you can change under Home ribbon tab|Group Measurement|Vector Tool Platform|Tool Platform|Channel Mapping page, has no effect in interface mode.
  • CANoe is not configured for operation in distributed mode, i.e. no RT server is configured under Home ribbon tab|Group Measurement|Vector Tool Platform|CANoe/CANalyzer|Distributed Mode.

Notes

  • The channel assignment for the interface mode is set in the Vector Hardware Configuration on the computer. The channel assignment on the device, which you can change under Home ribbon tab|Group Measurement|Vector Tool Platform|Tool Platform|Channel Mapping page, has no effect in interface mode.
  • Since CANoe 12.0, an assignment of all channels used in the CANoe configuration to a VN8900 no longer causes it to be used automatically in distributed mode.

In order to prevent unintentional use in interface mode (with the above-mentioned effects, e.g. with regard to realtime characteristics), CANoe performs a check at various points before a VN8900 connected via USB is used in interface mode, which could also be operated in distributed mode:

  • If a VN8900 is connected via USB when starting CANoe and no distributed mode is configured, you will be asked if you prefer to configure the device as an RT server in distributed mode. You can revise this decision later by changing the setting under Home ribbon tab|Group Measurement|Vector Tool Platform|CANoe/CANalyzer|Distributed Mode.
  • If a VN8900 is configured in interface mode when starting a measurement in CANoe, you will be asked if you want to cancel the measurement and open the dialog Home ribbon tab|Group Measurement|Vector Tool Platform to configure the device as RT server for distributed mode.

In both cases, you can prevent the demand from appearing again via Show this dialog no longer. You can reactivate the checks via Options|General|Messages.
