Interface Mode

 

Introduction

In interface mode, the realtime relevant parts are not executed on the VN8900 but together with CANoe on the user computer. The VN8900 is used like a standard network interface (see VN1640). In this respect, the interface mode corresponds to the standard operation of CANoe on a single computer, but in the context of VN8900 devices, which can also be operated in distributed mode or standalone mode, it is rather the exception.

Architecture

Realtime Area

  • The realtime area is located on the user computer.
  • Actions such as control and test execution, realtime simulation and model execution, measurement, analysis, logging, bypassing are performed on the user's computer.
  • The details correspond to the standard operation.

Analysis Area

  • The analysis area is located on the user computer.
  • The details correspond to the standard operation.

Schema

Use Cases

The interface mode can be useful in the following use cases:

  • The properties of the CANoe configuration do not allow operation in distributed mode (see Requirements for Configurations ). This is especially true in the case of dependencies to operating system components or other applications that are not available in the embedded image of the VN8900.
  • You have a device of type VN8910 or VN8910A. Since CANoe 11.0 this device is only supported in interface mode.
  • In addition to the VN8900, you want to connect further network interfaces (possibly also another VN8900) to the computer via USB and use them simultaneously in a CANoe configuration.

    An alternative could be distributed operation using an additional network interface connected to the VN8900 via USB (e.g. VN1600, but not another VN8900, see device manual for details).
  • You want to use several VN8900 devices in a CANoe configuration which are connected via IP and which you have integrated in the driver configuration (Vector Hardware Configuration) on the computer as Network Devices.
  • Your CANoe configuration has very high demands on the throughput in the realtime part, which cannot be handled by your VN8900 device.

    An alternative to the interface mode in this case could be distributed operation with a more powerful VN8900 device (e.g. VN8914 instead of VN8911).

Note

Note, however, that the interface mode also has some limitations:

  • The realtime properties (e.g. jitter and latency) are generally better in distributed mode than in interface mode. Conversely, this may mean that a CANoe configuration with high realtime requirements (e.g. with FlexRay channel) cannot be used satisfactorily in interface mode.
  • In contrast to operation in distributed mode or stand-alone mode, network interfaces connected to the VN8900 via USB cannot be used. For use in interface mode, they must be connected directly to the computer.
  • The VN8900's internal network adapters cannot be used as an Ethernet channel during measurement.
