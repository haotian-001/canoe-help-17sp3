Error Messages

 

The following error messages can occur while using standalone mode:

Downloading the Standalone Configuration

  • The configuration uses functions that are not supported in standalone mode.
    To set up standalone mode, deactivate the specified functions in your configuration, if possible (see also Configuration Requirements).
  • An error occurred during compilation.
    As is the case with a normal measurement start, the programs used in the Simulation Setup are compiled, if necessary, prior to download. Check the specified program and correct the error.
  • Hardware and application channel check has failed.
    Check whether the network hardware on the device is connected and the application channels are assigned correctly. Also take note of the output in the Write Window.
  • A suitable license is not available on the device.
    Check the license installed on the device.
  • An error occurred while saving the configuration to the device.
    The device may not have sufficient disk space available for the configuration.
  • An error occurred while reading files appended to the configuration.
    When a standalone configuration is downloaded or exported, the following are provided with the generated standalone configuration file:
  • CAPL DLLs,
  • Modeling libraries (DLL files incl. associated INI files),
  • User files configured for access in CAPL
  • diagnostic descriptions

An error occurred while reading one of these files from its original memory location.

  • Initialization of Simulink module failed.
    Check the configuration of the MATLAB/SimulinkÂ® modules used in the Simulation Setup. Also take note of the output in the Write Window.
  • Dynamic modules are missing.
    Check the configuration of the dynamic modules (e.g. hook DLLs).
  • An internal error occurred during the download.
    Please contact Vector Support.

Initializing Standalone Mode

  • Configuration file version on the device does not match CANoe version.
    The configuration file on the device was generated using an older version of CANoe and can no longer be used with the current CANoe version. Please load the configuration to the device again.
  • Configuration file could not be loaded on the device.
    An error occurred while trying to read the configuration file on the device. Downloading the configuration to the device again could solve this problem.
  • An error occurred while decoding the configuration file on the device.
    It was possible to read the configuration file on the device, but it could not be decoded. The file may be damaged. Downloading the configuration to the device again could solve this problem.
  • An internal error has occurred.
    Please contact Vector Support.

Starting the standalone measurement

  • A suitable license is not available on the device.
    The license is checked whenever the configuration is downloaded and each time measurement is started. Check the license installed on the device.
  • Initialization of the configured hardware channels failed.
    Check whether the network hardware is connected correctly to the device. Restarting the device may also eliminate this problem.
  • An error occurred while loading a network node.
    In some cases, e.g. when a CAPL DLL is integrated into the CAPL program using a #pragma library directive, the problem can be eliminated by re-compiling the program while the device is connected.
  • An error occurred while loading a node layer DLL.
    Check the configuration of the modeling library (DLL file) in the network nodes of the Simulation Setup.
  • An internal error has occurred.
    Please contact Vector Support.
  • Error loading test configuration.
    There was an error initializing the test configuration.
  • Error loading diagnostic descriptions.
    There was an error initializing the diagnostic descriptions.
  • Invalid path on local computer.
    The folder on the local computer, which has been specified as the destination for files transferred from the connected device, does not exist.
  • Error generating report file.
    An error occurred during the creation of the HTML report file. Please check the report path and the XSLT stylesheet file. You can find detailed error information in the log file at the same location as the test report (test unit) or test report (test module) file.
  • An error occurred while initializing code execution on network interface (CAPL-on-Board).
    Please check, that code execution on network interface is support by all used channels.

Further error messages

  • Configuration file could not be loaded on the device.
    Reload the configuration onto the device and then activate standalone mode.
  • Removing remote file <filename> failed.
    The lack of user rights is one possible reason for a failed deletion operation. It is also possible that the file is still being used or was deleted in parallel by another process. If the deletion operation was performed using command line option "âr", the specification of an incorrect filename may be another reason for the failure.
  • The standalone configuration filename is invalid.
    An invalid filename was specified for the current operation (changing the standard or current standalone configuration file, deleting a configuration file). Empty filenames, filenames with path information, and filenames with incorrect extension (other than RTCFG) are classified as invalid.
  • The operation is not allowed in the current state.
    Device state has changed since the operation has been triggered. A potential cause might be another instance of Standalone Manager or CANoe that has started a concurrent operation in the meantime.
  • The available memory on the remote device is not sufficient to complete the operation.
    The error could occur when accessing a file on the device but also when downloading a new configuration file. In the latter case, attempt to reduce the size of the standalone configuration (incl. the DLLs, user files, and diagnostic descriptions contained therein).
  • The available memory on the user computer is not sufficient to complete the operation.
    Try to reduce the size of the standalone configuration (including associated DLLs, user files, and diagnostic descriptions).
  • An error was logged on the connected device.
    The RT kernel on the device was terminated unexpectedly. Contact our Support and provide the error details displayed in the error message.
  • Reading file <filename> on the remote device failed.
    You may not have the required read permission.
  • Saving remote file <filename> on the local computer failed.
    You may not have the required write permission. It is also possible that the overwriting of an existing file is prevented because the file is still in use.
  • File <filename> on the remote device is currently in use.
    The file cannot be read, overwritten or deleted, because it has been already opened for write access. This error particularly occurs when attempting to copy the logging file, which is currently used during a standalone measurement, to the local computer or to delete it.
  • Cannot overwrite current configuration file while measurement is running.
    Exit the message before downloading the standalone configuration file.
  • Reading file <filename> on the local computer failed.
    You may not have the required read permission.
  • Saving file <filename> on the remote device failed.
    You may not have the required write permission. It is also possible that the overwriting of an existing file is prevented because the file is still in use.
  • Internal error saving file <filename>
    You may not have the required write permission or there is insufficient memory space available on the destination data medium. It is also possible that the overwriting of an existing file is prevented because the file is still being used.
  • The vFlash support report could not be saved.
    An error occurred when creating the support report. Additional error details are output with the error message in the dialog.
  • The files required to install vFlash could not be extracted.
    An error occurred when unpacking the vFlash environment file. Additional error details are output with the error message in the dialog.
  • There is no vFlash support data on the selected device.
    When creating the support report it was determined that no support files are stored on the device.

Note

The standalone mode is automatically deactivated if a fatal error occurs while standalone mode is being activated (during activation or when restarting the device after successful activation) or while a standalone measurement is starting.
