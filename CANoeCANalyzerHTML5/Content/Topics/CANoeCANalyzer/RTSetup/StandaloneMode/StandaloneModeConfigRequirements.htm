Configuration Requirements

 

Configurations that have their realtime specific portions loaded onto a VN8900 or an RT server for execution in standalone mode are subject to the following limitations.

The following functions are supported:

  • Simulation nodes and test modules in the Simulation Setup and in Test Environments for Test Modules.as well as Test Units.
  • Diagnostic functions and XCP functions.
  • Configurations that use CAN, LIN, MOST, FlexRay, K-Line or Ethernet channels.
  • Configurations using IOpiggy.
  • The use of modeling libraries, CAPL DLLs and C libraries.
  • The execution of Visual Sequencer programs.
  • The Replay Block. You can use a source file in BLF format.

Please note the following limitations:

  • The use of simulated bus mode while in Standalone Mode doesn't make much sense and is therefore not supported.
    In standalone mode only execution in online mode is supported.
  • You cannot download a configuration where files with the same name are loaded from different folders.

    Workaround: rename the file(s) to make the filenames unique, and change the configuration to use the new filenames.

If the configuration contains a function that is not supported, a corresponding error message is displayed during the download/export. Some of the functions that are not supported are automatically deactivated when the standalone configuration is created. In this case only a warning appears in the Write Window on download or export.

Further details you find in the Feature Matrix.
