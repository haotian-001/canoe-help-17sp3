Configuration Requirements

 

When CANoe is used in a distributed environment with decoupled realtime operations (e.g. on a VN8900), the same models and configurations are used that are normally used for execution on a computer, and consequently the same program properties are generally available.

There are, however, a few restrictions that apply when a configuration is used in distributed mode:

  • Synchronization of program files

Note

If you want to use a C library or a CAPL DLL in distributed mode on a RT server (e.g. with a VN8900 or an RT Rack, please mind that the DLL doesn't contain dependencies to other DLLs that are not part of the Windows Embedded Images on the device. Particularly debug versions cannot be executed if they contain dependencies to dynamic debug libraries.
  • CANoe automatically transfers modeling libraries (DLL files incl. the corresponding INI files), CAPL DLLs, C-Libraries and .NET assemblies to the RT server prior to the start of the measurement.
  • If DLLs are changed during an active connection â e.g. due to re-compiling a MATLAB model â the updated DLLs are automatically transferred on measurement start.
  • Generally you do not need to add or delete files manually on the RT server.
  • CAPL access to user files
  • Files that should be read by a CAPL program in the Simulation Setup or in the Test Setup for Test Modules need to be transferred from the user computer to the VN8900 or the RT computer before the measurement is started. In order to do this, the files must be specified in the Options dialog (Extensions|User Files), including the file paths in relation to their location on the user computer file system.
  • If the CAPL program has write access to a file, the file is transferred back to the user computer after the measurement ends.
  • The getAbsFilePath, setFilePath and setWritePath CAPL functions are not supported.
  • I/O channels are supported.
  • The VT System can be used in distributed mode.
  • Besides Vector I/O Hardware also other I/O Hardware (third party) can be used in distributed mode. The configuration must be done on the RT server.
  • Other I/O Hardware (third party) (except for Vector I/O Hardware) cannot be used on the VN8900 because there is no configuration possibility.
  • VN8900 supports IOpiggy.
  • GPIB
  • The VN8900 only supports adapters from National Instruments.
  • Properties not supported:
  • MCD is not supported.
