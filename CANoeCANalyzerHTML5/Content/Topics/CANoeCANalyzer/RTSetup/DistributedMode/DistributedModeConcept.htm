Distributed Mode

 

Introduction

CANoe offers the opportunity to execute the realtime relevant simulation parts on an individual computer (RT server) that is separated from the graphical user interface (user computer in distributed mode).

Thus on the one hand, the total performance of the system can be expanded more easily if necessary; on the other hand, influences above all of the graphic system on latency times and timer precisions are prevented.

The configuration of the simulation and the evaluation then occur on the usual workstation computer, for example, while the simulation runs on an individual computer/ device (distributed mode).

Architecture

In distributed mode, CANoe is divided into two components that are executed on different computers / devices in order to uncouple of the realtime operation.

Realtime area on RT server

  • The RT server is executed on an individual computer/device with the Microsoft Windows operating system. Different types of devices can be used as RT servers:
  • VN8900 (but not VN8910/VN8910A)
  • VT6000
  • RT Rack
  • Standard computer (after the corresponding installation)
  • In addition to the CANoe execution environment, the Vector Tool Platform (VTP) must be installed on the RT server. This allows access to the RT server in CANoe or in the Vector Platform Manager, e.g. to update the execution environment of CANoe on the RT server or to change device settings (e.g. network settings, channel assignment). The graphical user interface on the RT server is therefore usually not required and is not available for VN8900 devices, for example.

Evaluation part on user computer

  • This computer (user computer) is used to evaluate and display the data stream generated by the simulation or read by the bus hardware in the graphical user interface and to log data.
  • The evaluation part is typically executed on the usual workstation computer.
  • The application of CANoe then occurs from the evaluation part in as much the same manner as possible as in standard operation with one computer.

Functionality

These two computers/devices are connected with one another either via TCP/IP – typically via Ethernet, possibly via Wifi - or via USB (only with VN8900). All data for the configuration of the simulation or measurement is transmitted automatically to the RT server at the start. During measurement, all evaluation and logging data is transmitted promptly to the user computer. Keyboard, panel, and any other user interaction is also transmitted directly to the RT server during the simulation or measurement.

The TCP/IP network connection is uncoupled on both sides by a data buffer so that the realtime part is not dependent on the quality of the connection.

Schema

Use Cases
  • Your CANoe configuration has increased realtime requirements.
  • Your CANoe configuration has increased throughput requirements that you want to meet through the distribution between two powerful computers (user computer and VTP Device).
  • You have an RT Rack with a large number of connected hardware.

  • Recommended operating mode for VN8900 devices together with CANoe (unless the special requirements mentioned under interface mode are met).
