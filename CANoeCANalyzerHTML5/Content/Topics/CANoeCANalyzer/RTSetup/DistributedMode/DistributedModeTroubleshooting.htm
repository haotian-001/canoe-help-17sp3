Distributed Mode: Troubleshooting

 

Establishing a Connection

The following things can cause a failure to connect to the configured RT server ( e.g. VN8900 , VT6000, RT Rack):

  • The device/computer is not turned on or connected to the LAN or via USB.
  • The device/computer is switched off or not connected to a USB port of the user computer or to the LAN.
  • The simulation environment version on the RT server does not correspond to the CANoe version on the local computer.
    A software update can solve this problem.
  • The simulation environment on the RT server is running in standalone mode.
  • The simulation environment on the device is currently in use by another process (e.g. another instance of CANoe).
  • The RT kernel on the device is currently not running. You can start the RT kernel via Home ribbon tab|Group Measurement|Vector Tool Platform|Tool Platform|Applications.
  • A general connection error has occurred.
    A possible reason for this is that the device was detached from the computer or switched off during the attempt to establish a connection to it.
    This problem may also arise if there is no suitable or up-to-date driver on the RT server.
    A driver update can solve this problem.
  • The simulation environment on the RT server is not running.
    To solve this problem switch off the RT server and switch it on, again. If the problem isn't solved update the CANoe software on the device (see software update). Is the problem still isn't solved please contact Vector Support.
  • An internal error with unknown cause has occurred.
    In this case please contact Vector Support.

Checking Channel Assignments

At measurement start, the application channels used in the configuration and assigned to a VN8900 device are checked.

The following constellations lead to warning or error messages:

  • There is a connection to an RT server (distributed mode) on which application channels have been assigned to hardware channels on a VN8900 connected there. This scenario is not supported.
  • The application channels are assigned to a VN8900 device connected via USB that supports operation in distributed mode (e.g. VN8911, VN8914). However, distributed operation is not configured. In this case, a warning appears indicating that the measurement is being performed in interface mode (simulation kernel is running on the local computer). As a result, the realtime properties are generally worse than in distributed mode, where the simulation kernel runs on the RT server (e.g. VN8900).
    To configure the device as RT server, open the Vector Tool Platform dialog via Home ribbon tab|Group Measurement|Vector Tool Platform, select the device on the page Connection and change the corresponding setting on the page CANoe/CANalyzer|Distributed Mode.
  • The application channels are assigned to a VN8910/VN8910A device connected via USB, which since CANoe 11.0 no longer supports operation in distributed mode. A warning indicates that the device is now operating in interface mode (with a corresponding effect on the realtime properties).

Network Port Settings

The following network protocols and network ports must not be blocked by a firewall or similar. Please note the information in the table TCP/UDP Ports used by CANoe.

You can change the TCP port used by CANoe for basic communication with the execution environment on the RT server via Home ribbon tab|Group Measurement|Vector Tool Platform|CANoe/CANalyzer|Execution Environment (default value: 2809).
