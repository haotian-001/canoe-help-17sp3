Configuration and Operation

 

The operation of the distributed mode is very similar to the standard operation.

Preparation:

  • Both computers used must be set up appropriately. The RT server must be a device with an installed Vector Tool Platform. On devices shipped by Vector (VN8900, VT6000, RT Rack) the Vector Tool Platform is either already pre-installed or can be installed in the context of a driver update. Alternatively, you can also operate any computer as an RT server. Please refer to the instructions for installing the Vector Tool Platform on a standard computer.
  • In addition, make sure that the configuration satisfies the currently existing limitations.

Configuration of the RT server

Start CANoe on the user computer.
The configuration of a device/computer as RT server now takes place in the dialog under Home ribbon tab|Group Measurement|Vector Tool Platform. First select the device on the Connection page. Then select on the page CANoe/CANalyzer|Distributed Mode that this device should be used as RT server.

Configuration of the hardware channels

In CANoe, the channels and the hardware are set and programmed on the usual dialogs. However, the [Driver] button on the hardware dialog leads to the driver settings of the local computer (user computer) and not to the settings of the RT server (where in the case of VN8900 devices connected via USB, local settings may be synchronized to the device). The channel assignment on the RT server can be set under Home ribbon tab|Group Measurement|Vector Tool Platform|Tool Platform|Channel Mapping instead.

Note

  • Changes made in the Vector driver applet in the Vector hardware apply only to the user computer and do not affect the RT server.
  • If changes in the Vector hardware configuration (hardware assignment) on the RT server are desired, these must be made on the RT server.

Measurement

Start of the measurement is essentially as in the standard case. The data required for the simulation is thus copied automatically onto the RT server.
Some of the current function limitations lead to a error message with aborting of the measurement start. In such a case, adherence to these limitations should be checked.

Notes

The following aspects must be noted:

  • The display at the bottom left for the fill level of the buffer between realtime branch (simulation) and evaluation branch (interface) is now divided in two: it thus reflects the two buffers on the RT server and the user computer.
  • A filling of the buffer on the RT server usually indicates a network transmission problem.
  • A filling of the buffer on the user computer usually indicates a capacity problem on the user computer and can be solved with the usual interface optimization techniques.
  • Depending on the performance of the (Ethernet) network between the two computers, there is a certain temporal distance between the bus events and their display. This is especially noticeable if given an operation action e.g. on a panel, a corresponding reaction of the (CAN or LIN) bus is expected.
  • When CANoe is terminated on the user computer, the execution environment of CANoe is also automatically rebooted on the RT server. On restart of CANoe it automatically tries to connect to the RT server.
