RT Setup with Linux

 

Overview

From CANoe 13.0 the RT kernel (i.e. the realtime part) is also available in a Linux variant that can be used for the operation with CANoe4SW Server Edition.

Subject

Linux RT Kernel

Operation
  • Execution of CANoe4SW Server Edition configurations
  • Only for x64 architectures compatible with Intel

Availability

The Linux variant is available for the following distributions:

  • Ubuntu 18.04, 20.04 and 22.04
  • AlmaLinux 8.6

Use case
  • Can be easily automated in combination with CANoe4SW Server Edition via a command line interface
  • For applications that require a lot of main memory

Installation

Linux RT is currently only provided on request.

If you have any questions, please contact Vector Support.

Supported Features

Currently the following selection of features is available:

  • Only simulated mode is supported (real hardware is not accessible)
  • Bus systems: CAN, Ethernet and LIN buses are supported
  • CAPL programs with a limited set of functions
  • Python scripts
  • System variables are accessible

Binary extensions

Also under Linux it is possible to integrate your own developed extensions. However, these must be compiled as SOs (Shared Objects) and integrated via module description files.

If you have any questions, please contact Vector Support.

Integration of modeling libraries

Please note that if you integrate modeling libraries, the Linux variant must be referenced via a module description file (*.vmodule).

If you want to select one of the supplied modeling libraries, e.g. CANoeILNLVector.so, from the directory Modules_Linux-x64 of the CANoe installation directory, the corresponding module description file CANoeILNLVector.vmodule from the Exec64 directory must be included.

Currently the following modeling libraries are available for Linux:

  • CANoeILNLDefault.vmodule
  • CANoeILNLVector.vmodule
  • CANoeILNL_Autosar_Eth.vmodule
  • SomeIP_IL.vmodule

If you have any questions, please contact Vector Support.
