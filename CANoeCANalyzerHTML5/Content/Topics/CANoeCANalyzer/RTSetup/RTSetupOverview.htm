Realtime Execution (RT Setup)

 

CANoe is split into two components to decouple realtime operation, the real-time area and the analysis area. You can run a CANoe simulation in different modes. Both distributed and standalone modes already provide an improvement in realtime execution. This is made possible by splitting CANoe into two components: the simulation execution area (RT Kernel / RT Server) and the analysis area. There is no need to adapt the Simulation Setup when using distributed or standalone mode.

A further, additional improvement of the realtime execution in terms of latency and determinism is obtained by activating Extended Real Time (ERT). This is especially beneficial for high deterministic requirements.

Modes

Various options are available for realtime execution, which differ as follows:

Interface Mode

Distributed Mode

Standalone Mode

In interface mode, simulation execution, analysis and operation of CANoe are performed via the computer.

This is the default configuration.

In distributed mode, the CANoe simulation is transferred to a suitable network interface (VTP device) and executed there.

The operation, on the other hand, takes place via the connected computer.

In standalone mode, the CANoe simulation is transferred to a suitable network interface (VTP device) and executed there.

The simulation is started automatically or by means of a trigger.

Measurement recordings are stored locally on the network interface and are available for subsequent analysis.

To use distributed mode, standalone mode or Extended Real Time, a network interface of the Vector Tool Platform (VTP) must be selected.

These are grouped together to form the VTP devices.
