BAP

Ribbon|Analysis Tab|BAP

General

Documentation

BAP (control and display protocol) is a protocol that was developed for standardizing interfaces in the Comfort and Infotainment area. Originally defined by the OEM Software Initiative (HIS), it is currently in use in vehicles of the VAG Group.

The BAP documentation is only available in English and only after installation of the BAP add-on:

BAP_Manual.PDF

Configuration And Hardware

Analysis
  • BAP is based mostly on CAN communication. As a result, every standard CAN interface can be used for the protocol analysis or stimulation.
  • The BAP add-on that expands the analysis functions of CANoe to include this protocol is installed as part of the VAG add-on.

Note

The VAG add-on is not included in the CANoe installation. You can request this OEM add-on through your Vector Key Account Manager.

Once the BAP add-on is installed, the following functions are available:

Trace Window

  • Interpretation of CAN messages in the context of the BAP protocol.
  • Filtering and highlighting according to BAP criteria, e.g., according to logical control unit.

CAPL

  • Access to BAP messages and their signals in the Measurement Setup.

New Features

CANoe 11.0

  • BAP channels can be disabled individually without loosing the database configuration.

CANoe 8.5

Version 8.5 SP4

  • Extended ID range for logical devices according to BAP specification 1.7 (LSG ID > 63) is supported.
  • BAP signals in Data Window and Graphics Window can be renamed.
  • Improved input of CAPL functions and events handler for BAP in CAPL Browser.
  • Encrypted software configuration files (XM1) and function catalogs (XMB) can now be used in the 64-bit variant of CANoe.
    Requirement: VAG CAN Package > 3.0.2.

CANoe 8.2

Version 8.2 SP3

  • Encrypted software configuration files (.xm1) and function catalogs (.xmb) can now be assigned to BAP channel configurations (available since VAG CAN package version 3.0.1).
  • A software configuration file may contain more than one logical ECU version. It can now be chosen which one is used for a channel configuration.
  • BAP function catalogs can define a factor for conversion of raw parameter values to physical values. The factor is applied in Data, Graphics and Trace Window.
  • Display style of parameter values in Trace Window's sub-lines is now similar to CAN.
  • Beside online measurement and offline replay the BAP combiner is now also active when importing a log file to Trace Window. Hence, application layer messages are displayed in this case, too.
  • Drag and drop of parameters from sub-lines of BAP application layer messages in Trace Window works correctly now. Data, Graphics and filter area of Trace Window are valid drop targets.
  • Instructions document for running BAP protocol DLL (BAP1xx.dll) on setups with VN8900 hardware is available with VAG CAN package 3.0.1.

CANoe 8.1

Version 8.1 SP2

  • The visualization of long messages and the distinction between property and method was corrected.

CANoe 8.0

Version 8.0 SP3

  • Parameters of BAP messages can be configured to be displayed in Data and Graphics Window as well as filter condition in Trace Window.

CANoe 7.6

Version 7.6 SP2

  • The Trace Window provides an analysis filter for BAP messages.
    Trace contents can be filtered by drag and drop of BAP messages into the Trace analysis filter. The filters then can be configured in order to filter all messages of a certain logical device, a certain function etc. Heartbeat messages can be shown or hidden on the press of a button.
  • Several software configuration files can now be assigned to a CAN channel to accumulate a complete system description from these files.

CANoe 7.5

Version 7.5 SP2

  • CAPL now allows access to the parameters of BAP messages (e.g. bapParamSet, bapParamGet and bapIsParamAvailable).

  • With bapParamIsAvailable(), a check can be made to determine whether the message contains a particular parameter.
  • The functions can be used both on messages with a flat parameter layout and on Records, Arrays, and StatusAll messages. As a result, all parameters that are disassembled in the Trace Window can now be evaluated by the program.

CANoe 7.2

Version 7.2 SP5

  • Parameters of the type TString_6Bytes are now supported.

Version 7.2 SP1

  • The Function Class of a BAP message can now determined via CAPL. This allows the explicit correlation of the FunctionID to its meaning. Implementation of CAPL filters is now much easier and allows for example the filtering of heart beat messages.

Version 7.2

  • Color settings for BAP messages can now also be specified in the context menu of the Trace Window.
  • GetAll messages and Array parameters are now disassembled.
  • The way the BAP Sequence Observer identifies timeouts has been corrected. Unwarranted Sequence Observer error messages no longer appear in the Trace Window, and segmented transmissions are fully disassembled.
  • When BAP messages are exported from the Trace Window into an HTML file, the BAP sequence data and coloring are also retained for segmented messages.

Note

You can start the export via the context menu of the Trace Window. When doing so, choose XML as the target format.

CANoe 7.1

Version 7.1 SP4

  • CAPL now supports to comfortably implement filters for BAP messages in the Measurement Setup and the access to the BAP properties of CAN messages.
  • The disassembly of BAP messages in the Trace Window was improved especially when no function catalog is available.

Version 7.1 SP3

  • CAPL features for filtering, highlighting and the access of protocol properties are added to the BAP support.
  • Improves symbolic presentation of BAP messages, when no function catalog is available.
