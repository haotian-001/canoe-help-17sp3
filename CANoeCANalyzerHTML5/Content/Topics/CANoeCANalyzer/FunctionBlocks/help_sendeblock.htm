Transmit Block

 

The transmit block has the task of forwarding messages that arrive at its input to the card driver as transmit requests.

Transmit block parameters cannot be set by the user. Neither a menu nor a dialog appears when the block is selected by mouse click or by using the keyboard.

In typical applications, a program block or a Generator is inserted before the transmit block. With a program block the user can specify what is to be transmitted.

Example

Transmission of a message in response to a key press or in response to receiving a specific message.

The messages to be transmitted are passed in the transmit block through the card driver to the CAN expansion card, and after successful transmission they are returned with the actual time of transmission. In doing so, the message attribute Tx is preserved, so that transmit messages can be displayed and/or logged with the time of transmission. These messages are not retransmitted if they reach the transmit block again directly. This prevents unintentional creation of infinite loops, which would load the bus severely under certain conditions. If an infinite loop is to be programmed intentionally (e.g. as a base load) this can be achieved by copying a message in a program block.

When transmitting from CAPL programs the user can specify explicit from which of the two CAN controllers (bus connections) transmission is to be made. This is considered appropriately by the card driver.  If the controller is not set explicitly, first a search is made for the particular message. If successful the message is transmitted. If unsuccessful a search is conducted in Controller 2.

Note

If there is a transmit request for a message which is set up as a receive message, then a remote message is transmitted!  The message attributes Rx, Tx, TXRQ, DATA and REMOTE are ignored by the transmit block.

If the card driver receives a transmit request which cannot be executed because the particular message is missing in the setup of the specified controller or is missing in both controllers, then an error message (Send error) appears in the Write Window. The running program is not interrupted by this.
