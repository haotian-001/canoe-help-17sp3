LIN Disturbance Block: Response Disturbance View

 

Response disturbances are configured in the Header Disturbance view.

Configuration

  • Byte without disturbance
    Value of the data byte for disturbance configuration, displayed as a bit sequence with additional start and stop bits. If a database is available, the value is automatically derived from the initial signal values. If symbolic resolution is not possible, the value is set to 0xFF.
  • Byte
    If a Data byte is selected as the Disturbance byte in the Disturbance List, the data byte to be disturbed is displayed as a bit sequence.
    You can use the context menu to configure one dominant or recessive disturbance per data bit.
    You can configure only one dominant disturbance for the stop bit, and none for the start bit.
  • IB (Interbyte)
    If an Interbyte is selected as the disturbance byte in the Disturbance List the interbyte space to be disturbed is represented by a full bit length.
    You can use the context menu to configure a dominant disturbance for this interbyte space.
    The data byte that follows is also displayed as a bit sequence, but cannot be disturbed in addition.
  • Byte with disturbance
    If a Data Byte is selected as the disturbance byte in the Disturbance List, the byte value resulting from the disturbance is displayed.
    This is derived from the original byte value prior to the disturbance and the configured disturbance.
    If a disturbance is configured for the stop byte, no prediction is displayed for the byte following the disturbance.

Note

Please note hardware-related restrictions!
