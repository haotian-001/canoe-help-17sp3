LIN Disturbance Block: Disturbance List

 

Each disturbance to be executed by the LIN Disturbance Block needs to be added to the Disturbance List.

Configuration

  • Click the New icon in the toolbar to add a new entry for a Header, Response or Bus disturbance.
  • Click the Delete icon in the toolbar to remove the selected item(s).
  • Click the Up or Down icon in the toolbar to move an item up or down in the Disturbance List.
  • You can also select one or more items and press the <Del> key to remove them.
  • Click the right mouse button to access the context menu.
    To remove the selected item(s), select Remove.

Columns/Fields

  • Action
    Click the [Activate] button to activate the configured disturbance on the LIN bus.
    The button title reflects the currently configured functionality. The title changes dynamically as you modify the configuration. This button is only enabled during a measurement.
  • Exec.
    Specifies the number of runs (executions) for the configured disturbance.
  • Type
    Specifies whether the disturbance is a Header, Response or Bus disturbance.
  • Byte
    Specifies which byte or interbyte space of a Header or Response is affected.
  • Disturb on Header
    LIN frame ID (6 bits); the symbolic name is shown if a database is available.
    The disturbance is executed for the configured header.
  • Bus Level
    Specifies whether the disturbance is recessive or dominant.
  • Bit Count
    Specifies the length of the configured disturbance.
  • Channel
    Specifies which LIN channel the disturbance is executed on.

Note

  • You can modify the Disturbance List while a measurement is running.
  • The [Activate] button in the Action column may sometimes remain deactivated during a measurement.
    This is usually caused by an inconsistency in the configuration and is generally explained in a tooltip message.
    Possible errors:
  • The measurement is running in simulated mode.
  • The LIN Disturbance Block is deactivated in the test setup:
  • An invalid LIN channel is selected; etc.
  • A VN1611 network interface is used. A VN1611 does not support disturbance functionality.
