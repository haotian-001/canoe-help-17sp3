CAN Filter Types

 

Frame Filter

You can insert a Frame filter:

  • via the context menu (right mouse-click)
  • via the menu bar (Edit menu)
  • Add Frame from Databaseâ¦
    opens a symbol selection dialog that you can use to insert frames from a database
  • Add Frame Identifierâ¦
    opens a parameter selection dialog that you can use to configure a Frame filter using the frame identifier and the CAN application channel
  • Add Frame Identifier Rangeâ¦
    opens a parameter selection dialog that you can use to configure a Frame filter using a range of frame identifiers and the CAN application channel

The frame filter parameters view is divided into two areas:

  • the left side (main view) lists all Frame Filters:

    • Column

    Description

    Active

    Activates/deactivates a filter

    Name

    Shows the frame name from the database for frames that are present in a network description or database
    (not editable)

    ID

    Frame identifier

    Send Node

    Name of the send node

    Channel

    Application channel number.
    The selection options depend on the number of configured channels.
  • the right side displays the properties of the currently selected filter (properties view).
    It consists of the following groups:

    • Property

    Description

    Common

    Frames with modified or unmodified data field only.

    • Modified means filtering only if data field has been modified
    • Unmodified means filtering only if data field has not been modified
    • Ignore means the contents of the data filed is ignored.

    Database Attributes

    Shows the database, data length, frame name and publisher
    (not editable; available only for database frames)

    Frame Flags

    RTR bit value: Data, Remote or Tx request frames.
    Yes means filtering, No means no filtering.

Note

  • When using two or more Frame filters with the same frame identifier, the filters are used in a conjunction (OR relation).
  • A Frame filter overwrites the settings of the sending/receiving frame filter of a Node filter.

    Frames

    Data
    • Data set to yes and Remote to yes: ignore RTR
    • Data set to yes and Remote to no: filter frame if RTR bit is set to Data
    • Data set to no and Remote to yes: filter frame if RTR bit is set to Remote
    • Data set to no and Remote to no: ignore RTR

Event Filter

The Event filter filters non-frame-ID-based events as e.g.:

  • Error Frame

The event filter parameters are shown in a view:

Column

Description

Active

Activates/deactivates a filter.

  • Activated means filtering.
  • Deactivated means no filtering.
  • Ignore means the event is ignored.
