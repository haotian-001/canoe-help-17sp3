IP Filter: Protocol Filter

 

Protocol Filter

Port Filter

Packet Filter

 

On the Protocol Filter page, you can configure simple filter conditions for certain protocols. The following settings are configurable here:

Setting

Description

Active

You can use this check box to activate or deactivate the filter condition. Filter conditions that are added during a running measurement are initially in the deactivated state.

Channel

Here you select the channel on which the filter condition should apply.

Ports

This column is available for network-based access only. Here you select the measurement ports and simulation ports on which filtering should be performed.

Direction

Here you select the direction in which the packets should be filtered.

  • Tx - only sent packets are filtered
  • Rx - only received packets are filtered
  • Rx/Tx - both sent and received packets are filtered; default value

Protocol

Here you select the protocol on which filtering should be performed. The list contains all protocols current supported by CANoe.

The default value is *, i.e. all protocols are accepted.

Address

Here you input the address from which a packet is sent or the address to which a packet is sent.

You can use an IPv4 address, IPv6 address or MAC address (see address formats). The default value is *, i.e. all addresses are accepted.

Note

Currently the IP Filter does not support GeoNetworking addresses.
