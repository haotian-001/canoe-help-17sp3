IP Filter: Packet Filter

 

Protocol Filter

Port Filter

Packet Filter

 

On the Packet Filter page you can configure filter conditions for the protocol fields of Ethernet packets. The protocol fields in which address and numeric values may be input are configurable here.

You can enter the values for the protocol fields in the Value column on the right side of the table. The Ethernet packets are then either passed (with a Pass Filter) or blocked (with a blocking filter), if the values of the Ethernet packet's protocol fields match the values of the configured protocol fields. If you use value * for a protocol field this protocol field is mandatory and has to be in the Ethernet packet. The value can be arbitrary then.

You can specify a range for protocol fields that expect a numeric value. Write two points between the beginning and end of the range, e.g. 34900..35100.

In addition, you can configure the following settings:

Setting

Description

Active

You can use this check box to activate or deactivate the filter condition. Filter conditions that are added during a running measurement are initially in a deactivated state.

Channel

Here you select the channel on which the filter condition should apply.

Ports

This column is available for network-based access only. Here you select the measurement ports and simulation ports on which filtering should be performed.

Direction

Here you select the direction in which the packets should be filtered.

  • Tx - only sent packets are filtered
  • Rx - only received packets are filtered
  • Rx/Tx - both sent and received packets are filtered; default value

Ethernet Type

Field of the Ethernet header to specify the protocol the payload of an Ethernet packet is sent with

E.g. 0x0800 means filtering for IPv4 packets and 0x86DD means filtering for IPv6 packets.

IPv4 Source Address

IPv4 address a packet is sent from

IPv4 Destination Address

IPv4 address a packet is sent to

IPv4 Protocol

Field of the IPv4 header to specify the protocol the payload of an IPv4 packet is sent with

E.g. 0x6 means filtering for TCP packets and 0x11 means filtering for UDP packets.
