MOST Filter

Context Menu of a Hot Spot: Insert Event Filter|MOST Filter

Note

If configured in the same way, the MOST application filter, which has been available since version 6.0, can deliver the same results as the MOST Filter described here. The application filter has the advantage that it is much more flexible and does not require the use of different Filters.

The MOST Filter can be used to filter a wide variety of MOST messages and events. The filter always allows all non-MOST messages and events (e.g. CAN messages) to pass.

A MOST Filter can be added at any point in the Measurement Setup.

Options

Control Channel messages are subdivided into

  • AMS atoms
    AMS atoms are the MOST Control Frames into which a long AMS message is disassembled, and which are then sent onto the bus. (TelID 1, 2 or 3)
    Even messages with TelID=0 are AMS atoms for these non-fragmented transmissions can be displayed as an AMS message e.g. in Trace Window.
  • AMS messages
    AMS messages are the messages composed by the AMS service which may contain up to 64KB of useful data.
  • Other messages
    All MOST Control Frames of a different type (RemRead, Alloc, etc.) or which have another TelID (>3) are included under Other messages.

Note

  • Only AMS atoms can be written into logging file. In case you want to write MOST messages into a logging file, make sure that in each MOST Filter before a Logging Block the several AMS atoms are let through.
  • To use this in the context of the Trace Window display, the settings can be adjusted directly in the Trace Window via the MOST filter configurationpage.

Asynchronous Channel can be used to filter all other packets that are on the Packet Channel.

Control Channel and Packet Channel options may be applied to Node and/or Spy modes. The Node modes are Master and Slave. The Spy mode is the Bypass mode with activated control or asynchronous Spy.

Independent of the Node/Spy option, Light & Lock and Unconfigured Events may be filtered. Included under Unconfigured Events are all events not listed here (e.g. NetState, Register, etc.).

The filter type selection is valid for all options named above. The user may choose a Blocking or Pass filter.
