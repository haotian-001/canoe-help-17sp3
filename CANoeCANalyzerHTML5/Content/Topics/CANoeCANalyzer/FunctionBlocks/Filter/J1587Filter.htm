J1587 Filter

Ribbon|Analysis Tab|Filter|J1587 Filter
Context Menu of a Hot Spot: Insert Event Filter|J1587

General

The J1587 Filter lets you filter J1587/J1708 messages.

It allows

  • filtering for J1587 parameters and
  • filtering for J1708 message identifications.

The default setting is to filter every message with the specified PID, not considering the MID. If the PID should be filtered only for a specific MID, the MID can be entered in the column Address (MID) on the page J1587.

Window

Usage

The J1587 Filter is divided into three areas:

  • Menu Bar
    Here you configure the filter (pass or Stop Filter, filter active/inactive) and the view (depending on the currently displayed filter group), and you can add/remove filters.
  • Filter View
    All filter types are shown here:
  • Parameter Filter
  • Node Filter

The main view lists all filter. The columns are configurable depending on the filter type.
The properties view displays the settings of the currently selected filter.

  • Configuration Overview
    Here, in the lower part of the window, an overview of all configured and activated filters is shown.

Requirements

  • The option .J1587 has to be installed
  • At least one J1708 application channel has to be configured

The J1587 Filter is used in the analysis branch of the Measurement Setup. To insert a J1587 Filter , use theInsert Event Filter|J1587 context menu path of a hot spot that precedes the relevant evaluation block.

The main function of the J1587 Filter is to serve as either a Pass Filter or a Stop Filter. A Pass Filter that is not configured (i.e. is empty) does not allow any J1587/J1708 events to pass through, and thereby stops all J1587/J1708 event traffic.
