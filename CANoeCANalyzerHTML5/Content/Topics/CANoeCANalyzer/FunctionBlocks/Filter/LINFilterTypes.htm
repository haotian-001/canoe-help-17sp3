LIN Filter Types

 

Frame Filter

You can insert a frame filter:

  • via the context menu (right mouse-click)
  • via the menu bar (Edit menu)
  • Add Frame From Database…
    opens a symbol selection dialog that you can use to insert frames from a database
  • Add Frame Identifier…
    opens a parameter selection dialog that you can use to configure a Frame Filter using the LIN FrameId and the LIN application channel

The frame filter parameters view is divided into two areas:

  • the left side (main view) lists all frame filters:

    • Column

    Description

    Active

    Activates/deactivates a filter

    ID

    LIN frame ID

    Name

    Shows the frame name from the database for frames that are present in a network description or database
    (not editable)

    Channel

    Application channel number.
    The selection options depend on the number of configured channels.
  • the right side displays the properties of the currently selected filter (properties view).
    It consists of two groups: Events and Database Attributes:

    • Property

    Description

    Events

    Valid frames, frames without response, checksum errors, invalid responses.
    Yes means filtering, No means no filtering.

    Database Attributes

    Shows the database, frame length, frame type and publisher
    (not editable; available only for database frames)

Note

  • When using two or more Frame Filters with the same LIN FrameId, the filters are used in a conjunction (OR relation).
  • A Frame filter overwrites the settings of the sending/receiving frame filter of a Node filter.

Node Filter

You can only insert nodes from a database!
You can insert a Node filter…

  • …via the context menu (right mouse-click)
  • …via the menu bar (Edit menu)

The frame filter parameters view is divided into two areas:

  • the left side (main view) lists all Node filters:

    • Column

    Description

    Active

    Activates/deactivates a filter

    Name

    Node name
    (not editable)

    Node Address

    Optional: node address of the slave node

    Frames

    Tx: transmitted frames of the node are filtered;
    Rx: received frames of the node are filtered;
    Tx/Rx: all transmitted and received frames of the node are filtered.

    Channel

    Application channel number.
    The selection options depend on the number of configured channels.
  • the right side displays the properties of the currently selected filter (properties view).
    It consists of two groups: Events and Database Attributes:

    • Property

    Description

    Events

    Valid frames, frames without response, checksum errors, invalid responses.
    Yes means filtering, No means no filtering.

    Database Attributes

    Shows the database
    (not editable)

Note

  • When using two or more Node Filters with the same LIN FrameId, the filters are used in a conjunction (OR relation).
  • A Frame Filter overwrites the settings of the sending/receiving frame filter of a Node filter.

Event Filters

The Event Filter filters non-FrameId-based events as e.g.:

  • Receive error (except illegal response)
  • Sync error
  • Long dominant signal
  • Spike event
  • Wake-up signal
  • Go-to-sleep command
  • Schedule table change
  • Disturbance event

The event filter parameters are shown in a view:

Column

Description

Active

Activates/deactivates a filter.

  • Activated means filtering.
  • Deactivated means no filtering.
  • Ignore means the event is ignored.

Event Types

Name of the event type
(not editable)
