Logging Comment

Ribbon: Analysis Tab|Logging|Logging File Configuration |Logging Comment|Set Current Comment...

The Logging Comment dialog allows you to specify the logging comment. Settings you make here apply only to this single logging block.

Information on global settings for all logging blocks can be found on the Global Logging Comment help page.

Logging comments can consist of several attributes. The set of these attributes is the same for each logging block and can only be set in the Global Logging Comment dialog.

Field Codes

Inserts a field code into the currently selected input field. Field codes can be used to generate parts of the comment automatically. The values are determined each time the logging file is closed and inserted into the comment text. In the preview fields you get a preview of the comment text with resolved field codes.

The Logging Comment Field Codes page contains a detailed list of the available field codes.

Columns

Entry Field

Description

Attribute Name

The name of the attribute.

This value can only be changed in the Global Logging Comment dialog.

Attribute Selection

Sets whether the locally defined, the globally defined, or no value at all for this attribute is written to the logging file.

You set globally defined values in the Global Logging Comment dialog.

Default Value

The default value, which is written by the logging blocks.

This value is persisted when saving the configuration.

The default value can only be changed outside the measurement.

Current Value

The value for the current or next measurement.

This value is not persisted and will be reset to the default value at the end of the measurement.

The default value can be taken over as the current value by pressing the button .
