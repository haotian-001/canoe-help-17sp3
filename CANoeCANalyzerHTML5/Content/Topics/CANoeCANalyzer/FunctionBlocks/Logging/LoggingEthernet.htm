Logging Block: Ethernet

 

You can log the bus traffic in ASC, BLFBinary Logging Format, MF4, PCAP Packet Capture and PCAPNG format to logging files. You can then replay these files in offline mode or with the replay block, or read them into individual analysis windows (e.g. Trace Window) using the import function.

Extended Ethernet Logging

When Vector network interfaces are used (e.g. VN5610), you can log additional information. Only the BLF and ASC logging formats support this. For this, you must activate the Extended Ethernet Logging (Options dialog).

The following additional information is taken into account in the extended Ethernet logging:

  • Ethernet packet checksum
  • Transmission duration of an Ethernet packet [ns]
  • Hardware channel number, if the network interface is configured accordingly (only for channel-based configurations)
  • Port names of the measurement and simulation ports (only for network-based configurations)
  • Simulation indicator for frames originating from a CANoe simulation port (only for network-based configurations)
  • Forwarded Ethernet packets if the network interface is configured accordingly

Logging Differences Between Channel-Based and Network-Based Configurations

The data in a logging file depends on the type of configuration used to record it.

A network-based configuration records port names but not hardware channel numbers. For channel-based configurations, it is the other way around. In addition, the interpretation of the direction information (Dir column) of Ethernet packets is different.

Therefore, the following implications arise for offline analysis or import of channel-based logging files into a network-based configuration.

Direction Information After Import

During a measurement the direction information of Ethernet packets is determined and recorded. Depending on the selected configuration the following results:

Network-based Configuration Channel-based Configuration

Tx = transmitted by VN interface

Rx = received by VN interface

Tx = transmitted by CANoe

Rx = received by CANoe

The interpretation of the information after import or during offline analysis depends only on the configuration at the time of recording, not on the one used to view the data later.

Note

Use the PCAP logging format only if you cannot use the other formats. PCAP does not store essential network information, e.g. the file does not contain channel numbers and port names. If you still want to save Ethernet in PCAP format, please make sure to use a suitable filter before the logging block. If possible, this should only let through one channel and port or one hardware channel, in order to be able to assign the origin of the data in the network later.
