Logging Block: External Writer/Reader

 

You can specify external writers and readers for any number of log formats in order to write and read them in CANoe.

These logging files can then be read in Replay Blocks, as offline source files or as source files for export and conversion. Furthermore they can be written in logging blocks or as a target file for export or conversion.

These external writers and readers take the form of COM components that are implemented by the IStorage COM interface, and which transfer log data from a specific format into the BLF format.

A few settings must be adjusted in order to use external readers in CANoe.

Step by Step Procedure

  1. Adjust the [LOGGING_READER] section of the CAN.INI file as follows:

    2. <file format>=<descriptive string>|<external reader>|<read/write flag>

    Example

    [LOGGING_READER]

    CCO=MOST CCO|{1715a11b-d3b9-4e3e-ad9c-07451164f494}|Mostlog.dll|read

    OP2=MOST OP2|{1715a11b-d3b9-4e3e-ad9c-07451164f494}|Mostlog.dll|readwrite

    With the Read/Write flag you determine whether the component has only read access or write access as well. The following flags are available:

  • read

    Component supports read access only
  • write

    Component supports write access only
  • readwrite

    Component supports both read and write access.
  1. Register the library in which the reader is to be found using the following system command.
  • To do this, open an MS-DOS prompt and change to the directory in which the library is located.
  • Enter the following command: regsvr32 <filename of the library>

Example

regsvr32 MOSTlog.DLL

Error Messages

External reader is not configured in the INI file

Error Message

The format of the logging file <filename> is unknown.

Make sure that the relevant external reader is configured in the CAN.INI file!

Description

The configuration contains a logging file in the Replay Block, or as a source file for the offline mode, which is not in a standard vector format (e.g., *.blf, *, *.log, *.asc).

If an external reader has not been defined for this format in the CAN.INI file, the logging file cannot be loaded.

You can define the external reader to be used in the [LOGGING_READER] section of the CAN.INI file:

<file format>=<descriptive string>|<external reader>|<read/write flag>

Example

[LOGGING_READER]
CCO=MOST CCO|{1715a11b-d3b9-4e3e-ad9c-07451164f494}|Mostlog.dll|read
OP2=MOST OP2|{1715a11b-d3b9-4e3e-ad9c-07451164f494}|Mostlog.dll|readwrite

External reader is configured in the INI file but the component is not registered

Error Message

The external reader for the log format <file format> could not be loaded.

Make sure that the <external reader> component is registered in the system!

Description

The configuration contains a logging file in the Replay Block, or in the form of a source file for the offline mode, which is not in a standard vector format (e.g., *.blf, *, *.log, *.asc).

The external reader is specified in the CAN.INI file, but could not be opened because the relevant library is not registered with the system.

You can register the library that contains the external reader with the system in the following way:

regsvr32 <filename of the library>

Example

regsvr32 MOSTlog.DLL
