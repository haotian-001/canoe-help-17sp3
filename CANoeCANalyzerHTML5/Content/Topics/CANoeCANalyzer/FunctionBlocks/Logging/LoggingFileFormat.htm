Logging File Configuration

Ribbon|Analysis Tab|Logging|Logging File Configuration

In the Logging File Configuration dialog the following settings are available to configure the logging file.

  • Destination folder
    Here you select the folder to which the logging file is to be written.
  • Destination file
    Here you specify the name of the logging file. Field functions support you when specifying the logging filenames.

    • Field Codes

    With the field codes it is possible to generate and increment filenames and directories.

    Advantages:

    • Existing files can be protected against overwriting.
    • After the end of measurement you don't have to enter new filenames constantly.
    • The logging recording can be split into several (smaller) logging files
    • which are easier to be sent
    • which are easier to be analyzed

    System

    Field Name

    Field Code

    Description

    User Name

    {Username}

    Windows login name.

    Computer Name

    {Computername}

    Name of the computer.

    Local System Time

    {LocalTime[|Format]}

    Current local system time.

    The default of the time stamp format can be set in the global options. In the field code you can overwrite the default setting.

    CANoe

    Changes

    • The {IncMeasurement} field code has been replaced by the {MeasurementIndex} field code.
    • Only for Logging Blocks:
      The {Date} {Time}{YYYY}{MM}{DD}{hh}{mm} and {ss} field codes have been replaced by the {MeasurementStart} field code.

    Field Name

    Field Code

    Description

    Configuration Name

    {ConfigName}

    Name of the CANoe configuration file.

    Measurement Index

    {MeasurementIndex}

    The measurement index can be set in the global options and is incremented on each measurement start.

    The measurement index is stored in a separate file for system parameters independent of the configuration.

    Measurement Start

    {MeasurementStart[|Format]}

    Time stamp of the measurement start.
    The default of the time stamp format can be set in the global options. In the field code you can overwrite the default setting.

    Program Version

    {Version}

    CANoe version.

    Logging

    Field Name

    Field Code

    Description

    Increment at each Trigger

    {IncTrigger|001}

    Each trigger can be distributed to an individual logging file. The filename is incremented after each trigger.

    You can predefine a start value in the field code.

    Increment at each Named Trigger

    {IncTrigger|001}_{TriggerCondition[|length]}

    Combination of the field codes Increment at each Trigger and Trigger Condition Name.

    Increment by File Size

    {IncSize|001|100MB}

    With this function the logging recording can be split into several (smaller) logging files.

    There is a new logging file when the entered size of the file is reached; the filename is incremented automatically. The size has to be entered in MB

    You can predefine a start value in the field code.

    Increment after time

    {IncTime|001|01d01h00m}

    With this function the logging recording can be split into several (smaller) logging files.

    A new logging file is generated when the entered time is reached; the filename is incremented automatically. The beginning of the time slot is either a start of measurement or a new incoming message. The time can be set in days, hours and minutes:

    • Days (d): 0-99
    • Hours (h): 0-23
    • Minutes (m): 0-59

    You can predefine a start value in the field code.

    Logging Block Name

    {LoggingBlock}

    Name of the logging block.

    Logging Condition Name

    {TriggerCondition[|length]}

    Name of the trigger condition.

    Depending on the selected trigger mode, different names are resolved:

    • Single Trigger: SingleTrigger
    • Toggle Trigger: ToggleTrigger
    • Entire Measurement: NoTrigger
    • Test Trigger: Name of the corresponding test element that activates the trigger.
      Example: Suppose you have a test configuration PowerTrain with a test unit GearBox and a test case GearCheck that activates the trigger. Then, the field code is resolved to PowerTrain_GearBox_GearCheck. If the trigger would react on test units, the field code would be resolved to PowerTrain_GearBox. If the trigger would react on test configurations, the result would be PowerTrain.

    Independent from the trigger mode, you can optionally define a certain maximum length. If the resolved name is longer than the specified value, the string is cropped accordingly.

  • File Format
    The following logging file formats are available.

    Format

    Description

    BLF

    Binary Logging Format (binary format)

    • File type: BLF Frame Logging
    • Message logging
    • The BLF format supports all bus systems / protocols and environment variables.
      For new recordings you should select the new BLF format.
    • Extended information can be logged for Ethernet. Select the corresponding settings for this on the Ethernet page (Options dialog).

    ASCII

    Default ASCII description

    • File type: ASCII Frame Logging
    • Message logging
    • When ASCII format is selected, the data are saved as a readable text file.
    • The primary purpose of ASCII formats is to permit data exchange with external programs or the transfer of Trace data into documents.
    • Extended information can be logged for Ethernet. Select the corresponding settings for this on the Ethernet page (Options dialog).

    KME

    KVASER Memorator LOGGER Files

    There is an external program for the configuration of the KVASER Memorator LOGGER mode.

    MDF/MF4

    Measurement Data Format (binary format, MDF version 2.0–4.1)

    • File type: MDF Signal Logging
    • Signal logging
    • Signals are extracted from messages and binary stored in signal-based MDF files. Default value is the latest version.
    • These signal-based MDF files can easily be imported into CANape, vSignalyzer or into CANoe's Graphics Window.
    • Always absolute time stamps are logged.
    • Support for Environment variables, statistic information, CAN signals, LIN signals, FlexRay signals, GPS signals, J1939 signals and J1587/J1708 signals. MOST signals are not supported.

    Note

    • This formats only logs signals; message information will be lost!
    • Up to and including CANoe 14, only the MDF extension was supported for signal-based MDF logging files. Importing a signal-based logging file with the MF4 extension will fail.

    MF4

    Measurement Data Format (binary format, MDF version 4.1)

    • File type: MDF Frame Logging Files
    • This format supports all bus systems and protocols as well as environment variables. CAN, LIN and FlexRay will be stored according to the ASAM standard. All other bus systems and protocols are stored Vector-specific.

    PCAP/PCAPNG

    Packet Capture (binary format)

    • File types:
    • PCAP Ethernet packet Logging
    • PCAPNG Ethernet packet Logging
    • PCAP AFDX Frame Logging (only reading)
    • PCAPNG-AFDX-Frame-Logging
    • PCAPNG format is supported by the Ethernet and AFDX bus systems, PCAP only by the Ethernet bus system.
    • For PCAP, channel number and transmit direction (Rx, Tx) are not supported.
    • For PCAPNG, the Direction/ transmit direction is not supported, but various channels.

Advanced Settings

  • With Warn before overwriting logging file you can specify whether a warning prompt should appear at the start of measurement before overwriting an existing measurement file.
  • With the Show error message on data loss you can specify whether you should be informed of data losses.
  • With View logging file(s) after measurement stop the logging file(s) will be opened after measurement stop automatically in an editor (not available in standalone mode).
  • With Write start value file at logging start an additional start value file (SVC) is created for each generated logging file, which you can include in the Start Values Window. Via [Symbol Selection...] you select the symbols to be included in the start value file. The values of the symbols that are available in the measurement memory at the time the file is created are always saved.
  • Global logging settings: With [Settings…] you open the Options dialog to edit all settings for all logging blocks of the configuration.

Logging Filter

  • With Log bus events you can select if bus events should be logged (e.g. frames, messages, PDUs).
  • With Log system and environment variables you can select if environment and system variables should be logged.
  • With Log trigger events you can select if the Trigger block actions (Start/Stop trigger events) should be logged. Trigger events are displayed in Trace Window when the logging is imported.
  • With Log test events you can select if information on the activity of test modules and test units should be logged.
  • With Log internal events you can select if internal program events should be logged. Internal events are for instance CANoe status reports.
  • With Log statistic events you can select if system variables for statistics should be logged.
  • With Log CAPL events you can set if the output of the writeToLog CAPL function and program internal comments should be logged.
    Not available in standalone mode.
  • With Log diagnostic events you can select if diagnostic events should be logged.
  • With Log communication and attribute events you can select if events from the Communication Setup should be recorded.
  • Log FlexRay events
    For FlexRay you can select whether Frames, PDUs or both should be logged. This setting only influences valid FlexRay frames and PDUs. All events of other bus systems as well as other FlexRay events (Null Frames, Invalid Frames, Start Cycle Events,...) remain unaffected.
  • Frames
    Only frames are written to the logging file.
    When using a logging file containing only frames as input, PDUs will be extracted from the frames by using database information. This setting produces smaller logging files than Frames & PDUs. When a pass filter for PDUs is placed before the Logging block, all frames will be filtered, resulting in a logging file containing neither frames nor PDUs.
  • Frames & PDUs
    Frames and PDUs are written to the logging file.
    When using a logging file containing frames and PDUs as input, the PDUs contained in the file are used. The PDUs will not be extracted from the frames.
    When a Pass Filter for PDUs is placed before the Logging block, all frames will be filtered, resulting in a logging file containing only PDUs.
  • PDUs
    Only PDUs are written to the logging file.

Cross Reference

Logging files generated with CANoe always contain the date and time in the Header.
Additional information on the available logging formats can be found in the PDFs of the delivery.

Logging Comment

Here you can specify the comment that is written to the logging file, or edit the comment of the logging file last written by this block after the measurement.

Comments are only supported for the logging formats BLF, MF4, and MDF.

  • Click on [Set Current Comment...] to open the Logging Comment dialog, which allows to set the comment for new logging files of this block.

    This button is only active if the selected file format supports comments.

  • Click on [Change Last Comment...] to open the Change Logging Comment dialog, which allows you to edit the comment for the last file written by this block.

    This button is only active if this logging block has written a logging file that supports comments during the last measurement.
