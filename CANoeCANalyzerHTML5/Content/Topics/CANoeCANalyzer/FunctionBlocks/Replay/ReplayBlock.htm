Replay Block

 

The Replay Block provides the means for reproducing measurement sequences which have already been recorded. To do this, the user indicates a logging file. Messages contained in it are introduced into the data flow.

An important application is the replay of a logged data stream onto the bus. This involves setting the Replay Block to a specific bus system (Supported bus systems: CAN, LIN, MOST, FlexRay, Ethernet).

With CANoe you can also Feed environment variable sequences into the simulation configuration.

For data replay the Replay Block must be entered in the simulation configuration. When the [Select file] button is activated a file selection box appears for the replay file which is to be reproduced. Below this the user can specify how transmission is to occur. This area contains a selection of messages and time behavior. The file can be transmitted once or cyclically. With cyclic transmission the first message is started again after the end of file is reached.

Note

  • Effective with version 6.0 it is possible to use long symbolic names (> 32 characters). These are logged in the ASCII Frame Logging file (format ASC) if in the standard toolbar the icon sym is active.
    The produced ASCII Frame Logging files can be read in older versions of the program. However, messages with long symbolic names are ignored, because older program versions do not support long symbolic names.
    If you like to read ASCII Frame Logging files without exceptions in older versions, please, log numerically, while you deactivate the icon sym in the standard toolbar.
  • In the demo version, a maximum of 1000 messages can be replayed.

When a Replay Block is removed from the simulation or measurement configuration (by selecting the command Remove... in the context menu) the configuration is lost. This only affects the dialog settings. The replay file itself is not deleted.

Configuration

The configuration dialog contains the following pages:

  • General
  • Channel mapping
  • Bus-specific options:
  • CAN Options
  • LIN Options
  • MOST Options
  • FlexRay Options
