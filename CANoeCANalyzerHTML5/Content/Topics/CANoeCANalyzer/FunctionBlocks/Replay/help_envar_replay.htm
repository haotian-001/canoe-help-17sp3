Replay of Environment Variables

 

The Replay Block provides you with the option of feeding data into the simulation configuration from both binary and ASCII files.

Thus, you can edit ASCII logging files with a conventional text editor before you play them back in the Replay Block. In particular, you can also generate your own ASCII files with the sequences of values of certain environment variables, in order to feed them into the measurement through the Replay Block.

The Replay Block reads an ASCII file line-by-line. You enter the number format in the first line.

  • If the integer values in your file are to be interpreted as decimal values you write:

    • base dec

  • If you wish to enter the numbers in hexadecimal format you write:

    • base hex

You can comment out any line in the file with two slashes followed by a space character (// ). For lines which are to be fed back into the measurement from a Replay Block, you first enter the time of transmission in seconds (after start of measurement) in the first column. To play back an environment variable you write - in the second column - the name of the environment variable followed by a space character, the assignment operator :=, another space character and the variable value.

Example of an ASCII Replay File

// Number format: Decimal base dec

// Time   Name of EV      Value
1.0000    LightSwitch  := 1
2.0000    LightSwitch  := 2
4.0000    MotorSwitch  := 1
4.1000    SpeedEntry   := 100.0
4.2000    SpeedEntry   := 150.00
4.3000    SpeedEntry   := 200.00
4.4000    SpeedEntry   := 300.00
4.5000    SpeedEntry   := 500.00
4.6000    SpeedEntry   := 800.00
4.7000    SpeedEntry   := 2000.00
4.8000    SpeedEntry   := 2200.00

Besides environment variables, in the ASCII replay you can also feed messages and Error Frames into the ongoing measurement.
