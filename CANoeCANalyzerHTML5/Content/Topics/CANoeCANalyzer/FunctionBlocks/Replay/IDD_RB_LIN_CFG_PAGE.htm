Replay mode

Here you configure the way LIN events are replayed; supported LIN events:

  • Unconditional frame

  • Event-triggered frame – No Response

  • Event-triggered frame - Single Response

  • Dynamic frame

  • Configuration Request / Response

  • Diagnostic Request / Response

  • Wake-up Request

  • Transmission Error, i.e. header without any response

To send the header of a LIN frame the Replay headers option has to be selected. Responses for the sent headers can be configured in one of the following ways:

  • Take all responses from the Replay file

  • Take responses from the Replay file only for frames published by the Master node

  • Ignore responses from the Replay file for all frames

  • Use Sync-Break and Sync-Delimiter length from log file

    By enabling this option, the break length and the delimiter length from the first LIN event appearing in the replay file on a specific channel will be used for sending all subsequent LIN headers on the same channel.

    These modified timings will be effective until replay stops.

Note

In order to send headers LIN has to be in the Master mode. See: Master mode of the LIN hardware for more information.

Replay filter

Here can be configured a filter for certain events from the Replay file. Available settings:

  • Filter responses from Replay file for frames with Tx flag

  • Filter responses from Replay file for frames with Rx flag

  • Filter Wake-up signals
