A Replay Block is controllable by CAPL calls automatically even if the start/stop and pause/resume keys (see bottom of the page) are not defined.

You can start, suspend, resume or stop the Replay Block during the measurement by buttons of the Replay Block in the Simulation Setup:

  • - Start / Resume button
  • - Pause (suspend) button
  • - Stop button

General

  • Replay name
    Replay Block name used as a parameter for CAPL replay functions.
  • [Select file…]
    You can select the logging file to be replayed here using the file selection box.
    The filename is displayed in the line that follows.
    You can enter the message based file types like BLF Frame Logging File (format BLF), MDF Frame Logging File (format MF4) or ASCII Frame Logging File (format ASC) as file formats.

Note

  • If the messages you save in a MDF Frame Logging File (format MF4), have a time distance of more than 240 seconds they cannot be replayed with their correct time stamps (e.g. in offline mode and in the Replay Block).
  • MOST objects will be discarded on importing CLF files.
  • Here you find further notes for logging file formats.

You can use ASCII Frame Logging Files (ASC format) in the Replay Block to Feed environment variable sequences into the simulation configuration in CANoe.

Output modes

  • Standard
    Events are replayed with respect to their time stamps.
  • Step (not available for FlexRay)
    Replay is suspended on the start.
    It outputs one message on replayResume() CAPL call or on Pause/Resume key and enters the suspended state again.

Note

If the output mode Step is selected, the Replay Block is automatically set into the paused state after measurement start. The Replay Block must not be started explicitly.
  • Animated with … ms (not available for FlexRay)
    Timestamps of events are skipped in this mode.
    All events are sent with defined period in ms.
  • Repetitive output sending mode
    The file can be played back once or repetitive.
    With cyclic transmission reading resumes at the start after the end of file is reached.

Start options

  • Start replay on measurement start
    If this box is cleared, you have to start the replay by using either the start/stop key or by the replayStart() call.
    Otherwise replay starts on measurement start.
    Start timing conditions are taken in account in both cases.

Start timing conditions

Start timing conditions are active only for the standard mode.

  • immediately
    The first message is transmitted at the start of measurement.
  • with the first event time
    The times of transmission are defined by the original times within the file.
  • after timeout of … ms
    The time of transmission of the first message relative to the start of the measurement (or start on key / CAPL call) is entered in the edit box.

This option is not available for FlexRay.

Keys

  • Start/Stop key
    Define a key to start or stop the Replay Block.
    Replay is always started from the beginning as opposed to the Pause/Resume key.
  • Pause/Resume key
    Define a key to pause or resume the Replay Block.
    You can use a pause/resume key to step forward in the step mode.
    Only one message is transmitted on key press.
