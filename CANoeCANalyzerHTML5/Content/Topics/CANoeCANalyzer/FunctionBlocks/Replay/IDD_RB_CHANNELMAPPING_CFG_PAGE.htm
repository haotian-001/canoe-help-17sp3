Here you configure the allocation of the channels found in the logging file to the channels available in the configuration. The possible option settings are identical for all supported bus systems.

The Source column shows all channels that are configurable. These correspond to the channels found in the logging file, whereby not all channels named in the list need to also exist in the logging file.

In the Target column you can configure the channels of the active configuration on which the messages should be output. Channel selection is limited to the number of channels set in the configuration. Ignore may be selected here so that the messages of certain channels will not be sent.

If the number of channels used for a bus system is reduced in a configuration, even though the channel allocation shows mappings to channels which are eliminated by this action, then these mappings are highlighted in yellow in the "Channel Mapping" dialog and are automatically set to ignore, since it is not possible to send on unconfigured channels. However, these new values are not saved in the configuration until you confirm them with [OK] in the dialog.

Options

The [As in original] button causes all channels of the logging file to be mapped to the corresponding channels of the configuration (Channel1 -> Channel1, Channel2 -> Channel2, etc.). All channels in the left column for which there are no corresponding channels in the configuration are mapped to ignore, even if messages of such channels exist in the logging file.

The [Ignore all] button sets all channel allocations to Ignore.

If Show ignored channels is deactivated, all channels that are mapped to ignore are hidden.

Further Settings

With Send environment variables you can activate sending of environment variables.

With Send system variables you can activate sending of system variables.

Note

FlexRay frames can only be replayed on the channel where the Replay Block is attached. For this reason the selection of the target channel is restricted to this channel. In order to prevent sending of messages of other channels from the logging file "ignore" can be selected here.
