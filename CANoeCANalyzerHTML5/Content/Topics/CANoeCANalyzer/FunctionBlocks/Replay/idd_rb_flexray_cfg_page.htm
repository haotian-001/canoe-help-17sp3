FlexRay Options

 

Note

The FlexRay Replay Block can only be inserted into the Simulation Setup.

Functionality

The FlexRay Replay Block reads a logging file and sends the recorded FlexRay frames onto the assigned bus. The following points have to be considered:

  • In order to send FlexRay frames by the Replay Block, the appropriate Tx buffers must be enabled. The node-based configuration therefore has to be made in the page "FlexRay Options" of the Replay Block's configuration dialog (see below).

Note

Because this configuration is made database-oriented, the currently assigned FlexRay database of the Replay Block's bus must be the same as the database, which was used while recording the logging file.
  • In order to send FlexRay frames, the FlexRay bus must be synchronous. Sync-Frames in the logged recording will not be replayed with the sync attribute in the FlexRay header!
  • The replay for FlexRay happens cycle-oriented and not time stamp-oriented.
  • Starting the replay needs some preliminary lead, resp. start-up time in order to synchronize with the current cycle.
  • Environment and system variables will not be replayed.

Options

Replay Configuration

In this list you can select those nodes, whose transmitted messages should be replayed from the logging file.

Behavior

  • Stop at synchronization lost event
    When a sync-lost event is read from the logging file, then the replay is stopped.
  • Notify not supported events in Write Window
    Any event that is read from the logging file and that cannot be interpreted and replayed will be notified in the Write Window.
  • Send PDUs
  • If this option is enabled, then only FlexRay PDUs are re-played from the logging file.
    The Tx buffers must therefore be enabled for PDU updates.
    The replay of PDUs automatically leads to sending of the corresponding FlexRay frames according to the database.
  • If this option is disabled (default setting), then only FlexRay (raw) frames are re-played.
    The Tx buffers must not be enabled for PDU updates.
    The replay of FlexRay (raw) frames automatically leads to sending of the contained PDUs.
    If the logging file only contains FlexRay PDUs, then the FlexRay Frames will automatically be assembled from the appropriate PDUs.
  • Thus, this option only determines the possible source objects from the logging file, and which objects are actually updated for transmission.
