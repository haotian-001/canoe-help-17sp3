Test Trigger for Test Units

 

In Test Trigger for Test Units mode, you can here configure the time window for the logging of a execution period of a test element from the Test Feature Set. A test element can be a test configuration, a test unit, a test implementation entity, or a test case.

Note

  • The Test Trigger for Test Units mode is only effective, when using test units (from vTESTstudio).

  • For test modules, the Test Trigger for Test Units mode is not applicable.

The beginning of the corresponding test element triggers the start of logging and the end of the same test element triggers the stop of logging. The pre-trigger time refers to the start trigger and the post-trigger time to the stop trigger. If multiple test elements are executed at the same time, the trigger remains active until all test elements are finished, including their pre- and post-trigger times.

In addition, you can optionally specify a Verdict condition. In this way, a logging file will only be stored, if the verdict of the corresponding test element is equal to the specified one; otherwise, the logging file is deleted when the trigger is completed. If multiple verdicts are selected, the conditions are connected with a disjunction (OR). If multiple test elements are logged into a single file, the file is stored if at least one test element has one of the specified verdicts.

Example

You have two test configurations named Powertrain and Comfort and you want to log all events during both test executions in two separate ASCII logging files, each with a pre- and post-trigger time of 1s. In addition, you are only interested in the logging file, if the corresponding test configuration fails.

Trigger configuration:

  • Trigger mode: Test Trigger for Test Units
  • Scope: Test configuration
  • Verdict: fail activated
  • Time window:
    Pre-trigger time: 1000 ms
    Post-trigger time: 1000 ms

Logging file configuration:

  • Destination file: Logging_T{IncTrigger|001}_{TriggerCondition}
  • File format: ASCII Frame Logging

Suppose that the Powertrain test configuration is started first and has verdict pass, but the Comfort test configuration has verdict fail. Then you receive one logging file with the name:

  • Logging_T002_Comfort.asc

which contains all events from 1s prior to the start of the Comfort test configuration until 1s after its end. The file Logging_T001_Powertrain.asc is temporarily created, but deleted afterwards, because the verdict did not match the specified condition.
