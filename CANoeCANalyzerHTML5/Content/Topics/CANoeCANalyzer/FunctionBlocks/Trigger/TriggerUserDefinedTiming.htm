Timing

 

Various time conditions can also be selected for a user-defined condition. The default configuration has no time condition.

Timing of a user-defined condition

  • None
  • Timeout after: … cycle time(s)

You can enter any positive float number. The entered value must be equal to or greater than 0.01 milliseconds.

The length of a single cycle time is set in the message attribute CycleTime in the associated database. It is displayed as a static text below the edit box.

This timing type is unavailable if the message ID is not defined in any of the associated databases or its database does not have such an attribute; see below: user-defined condition with a timeout.

The resulting value of the timeout is stored in milliseconds (1 cycle time * N). The allowed minimum and maximum values of the timeout are the same as for timeout after … milliseconds.

  • Timeout after: … ms
    The entered value must be equal to or greater than 0.01 milliseconds.
    See below: user-defined condition with a timeout.
    By configuring the Delay or Timeout after:…ms the user-defined condition is classified as Continuous.
    See Set of user-defined conditions:Discrete vs. continuous conditions for additional information.
  • Delay…ms
    The feature specifies that after a successful evaluation the condition will be satisfied only after the required delay.
    By configuring the Delay or Timeout after:…ms the user-defined condition is classified as Continuous.
    See Set of user-defined conditions:Discrete vs. continuous conditions for additional information.
  • Absolute time interval
    The entry of an absolute time interval allows monitoring of a user-defined condition within the specified timeframe.
    Thereby you have to enter start and end time.

User-defined condition with a timeout

This is an user-defined trigger condition that has a timeout after … milliseconds.

It is configured in the Timing dialog.

A verification is done to determine whether the condition occurs regularly within a prescribed timeframe. If it does not occur within this time period the condition is met.

  • Initially, the timeout check is disabled at the start of the measurement.
  • The check is activated when the condition has occurred for the first time.
  • Each time the condition occurs within the prescribed timeframe the timer of the event condition is restarted (reset).
  • When the timer expires the condition is met.

Note

The time length values are limited to

  • 999,999,999,999.99 milliseconds or
  • 999,999,999.99999 seconds.

If you have to monitor a longer time period you must use a CAPL program to do so.
