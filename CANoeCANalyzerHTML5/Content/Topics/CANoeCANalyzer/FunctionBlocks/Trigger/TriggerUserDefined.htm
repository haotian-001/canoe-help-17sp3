Set of user-defined conditions

 

Here you can input the set of conditions used to search or trigger. This set can also be used as a global breakpoint in offline mode.

The set is organized in a tree form. Its node items are Groups and leaf items are Conditions.

  • Groups are represented by their logical operators (AND/OR)
  • Conditions are represented by a short textual description of the configured data.

The root of the tree is a predefined Group that cannot be removed.

Groups

Each group specifies a logical operator, which is applied to its immediate children items. The available operators are AND and OR.

Note

  • The whole set complies with the requirements when its root group is satisfied.
  • Only enabled groups and conditions participate in the evaluation of the set.
  • The evaluation of a single condition depends on its classification (see discrete vs. continuous user-defined conditions). You can visually distinguish between the conditionâs classifications according to their icons.
  • Group with the logical operator OR is satisfied if at least one of its child conditions or groups complies with the requirements.
  • Group with the logical operator AND is satisfied if all of its child conditions and groups comply with the requirements.

It could never comply with the requirement if there are at least two user-defined conditions, which never can be fulfilled on one runtime event. Usually, such conditions are Discrete and at least one of their following settings is not the same:

  • Bus system
  • Protocol type
  • Channel ID
  • Message ID

The set is automatically analyzed and a warning notification is issued when it could never comply with the requirements.

Conditions

Each user-defined condition allows specifying

  • a required runtime event (e.g. message, signal),
  • its timing (optional) and
  • its required data (if applicable).

The conditions are classified by a category of the required runtime event and by a way the event can be specified (symbolic or raw form).

The available types of conditions are:

  • Symbolic message condition
    Specifies a message from any of the associated databases.
  • PDU condition
    Specifies a PDU from any of the associated databases.
  • Symbolic signal condition
    Specifies a value of a signal. The signal should be chosen from one of the associated databases. Only signals assigned to a message (mapped signals) are allowed.
    AFDX signals of data type Opaque and String are currently not supported.
  • Symbolic predefined signal condition
    Specifies a value of a predefined signal. The signal should be chosen from messages that are defined in one of the associated databases.
  • System variable condition
    Specifies a value of a system variable. All system variables in system variables configuration dialog are available, including predefined system variables for bus statistics.
  • Predefined bus event condition
    Specifies a predefined bus event (e.g. CAN Error Frame)
  • Raw message condition (for AFDX and A429)
    Specifies a message and its data bytes. The definition should be entered manually.
  • PDU raw message condition
    Specifies a PDU and its data bytes. The definition should be entered manually.

Buttons / Context Menu Command

[New condition]

Here you can create a new condition of the required type. Depending on your choice a symbolic selection dialog or Raw condition pre-selection dialog will be opened. There you can select a desired symbolic data or enter some basic data in the case of raw messages.

The location of the new condition in the tree depends on the current selected treeâs item:

  • When the item is a group the new condition will be added as its last child.
  • When the item is a condition the new condition will be added as its next sibling.

New Group | AND
New Group | OR
(context menu only)

Here you can create a new group with the logical operator AND or OR.

The location of the new group in the tree depends on the current selected tree's item. When the item is a group the new group will be added as its last child. When the item is a condition the new group will be added as its next sibling.

[Editâ¦]

Opens an editing dialog for the selected condition.

[Duplicateâ¦]

Here you can duplicate the selected item of the tree. The new item is added as the next sibling of the source one.

  • In the case of a group all the sub items including sub-groups will be duplicated.
  • The root group cannot be duplicated.

[Delete]

Deletes the selected item of the tree.

  • In the case of a group all the sub items including sub-groups will be deleted.
  • You can use the <Del> key to achieve the operation without using the mouse.
  • Note, that the root group is predefined and therefore cannot be deleted.

Group operations

The operations over a group are available only via its context menu. To open the context menu select the desired group item in the tree and click over it with the right mouse button.

AND / OR

Sets the logical operator of the group to be AND or OR.

Enabled/Disabled items

Activate or deactivate the check box of a treeâs item in order to enable or disable it. Disabled items are ignored during the evaluation of the set. Sub-items of disabled groups are also disabled even if their check boxes are not cleared. To indicate such a case the icons of the items are grayed.

Working with drag and drop

Use drag and drop technique to move or copy (keep <Ctrl> key pressed) an item of the tree to a new location.

Except of the root, each item can be selected and dragged by the mouse to a new location inside of the tree.

The dragged item can be dropped into the highlighted destination if a restriction sign does not replace the mouse cursor. When the destination item is a group the dropped item will be added as its last child. When the destination item is a condition the dropped item will be added as its next sibling.
