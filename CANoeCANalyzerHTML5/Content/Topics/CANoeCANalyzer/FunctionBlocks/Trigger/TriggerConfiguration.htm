Trigger Configuration

Ribbon: Analysis Tab|Logging|Trigger Configuration

In this dialog you can configure the trigger conditions for logging measurement data to logging files.

Note

  • Whenever you activate the logging function the trigger is activated as well, automatically.
    Additionally you can place the trigger anywhere (hot spot) in the analysis branch of the measurement window.
  • InStandalone Mode, only one Trigger Block with limited functionality is available (toggle trigger triggered with CAPL).
  • To make the results of a measurement easier to understand, you should create multiple Logging Blocks in the evaluation branch, each with a different trigger condition. This results in several compact files, each of which is customized to address a special analytical question. If necessary, you can also configure a Logging Block to record the entire measurement.
  • For test triggers, it is strongly recommended to use the Increment at each Named Trigger field code in the Logging File Configuration dialog for generating the filenames. In this way, each test element is logged into a single file with a unique and meaningful name.

Note Filters in the Measurement Setup

Filters in the Measurement Setup are tailored to specific (superordinate) objects such as frames, PDUs, messages. Note that subsequent CAPL blocks usually address subobjects such as signals directly via the signal server and are therefore not subject to filtering. See Filter and CAPL Blocks.

Trigger Mode

The trigger mode defines the general conditions for a logging (Start point, endpoint, logging time period).
You can select the following trigger modes:

  • Single Trigger
    A certain event triggers logging
  • Toggle Trigger
    Certain events define the start and stop of logging
  • Entire Measurement
    The entire measurement is logged
  • Test Trigger for Test Units (only available with a Logging Block)
    The execution period of a test element defines the start and stop of logging.

Trigger Condition

If you have selected Single Trigger or Toggle Trigger as the trigger mode, you can now select from the following trigger conditions:

  • Start
    Triggering occurs at the start of measurement.
  • Stop
    Triggering occurs at the measurement stop.
  • CAPL
    Triggering is by a CAPL program.
  • User-defined
    The occurrence of a user-defined condition initiates triggering.

You can select none, one or several of these trigger types.

The [Define…] button does not become active unless the user-defined trigger condition is selected.

When you have chosen Toggle Trigger you can additionally chose whether to use the same trigger condition for block start and block end trigger or not. If you want to have the same trigger condition for both, activate the option Use combined toggle mode.

In Test Trigger for Test Units mode the following scopes are available:

  • Test configuration
    Triggering occurs at the start and end of each test configuration.
    Here, you can additionally choose whether triggering occurs for each execution cycle of a test configuration or once for all cycles.
  • Test unit
    Triggering occurs at the start and end of each test unit.
  • Test implementation entity
    Triggering occurs at the start and end of each test implementation entity (test fixture, test sequence, or test case). For nested test entities, e.g. a test case inside a test sequence, triggering occurs only for the topmost entity of the tree structure; in this example, the test sequence.
  • Test case
    Triggering occurs at the start and end of each test case.

In Test Trigger for Test Units mode, you can optionally choose from a set of verdict conditions. When the option Only with Verdict is enabled, a logging file will only be stored, if the verdict of the corresponding test element is equal to the specified one; otherwise, the logging file is deleted when trigger is completed. If multiple test elements are logged into a single file, the file is stored if at least one test element has one of the specified verdicts.

If the 5-valued verdict concept is activated in the Options dialog, you can choose from the following conditions:

  • none
    Tests with verdict none are logged.
  • pass
    Tests with verdict passed are logged.
  • inconclusive
    Tests with verdict inconclusive are logged.
  • fail
    Tests with verdict failed are logged.
  • error in test system
    Tests with verdict error in test system are logged.

You can select none, one, or several of these verdict conditions. If the 2-valued verdict concept is selected, you can choose only between pass and fail; the other three options are not available.

Note

  • The Test Trigger for Test Units mode is only effective, when using test units (from vTESTstudio).

  • For test modules, the Test Trigger for Test Units mode is not applicable.

Time

Here you can enter values for the pre-trigger and post-trigger times of the configured trigger. The configured pre-trigger and post-trigger times define the logging time period for this trigger.

For storing events during the pre-trigger time, a buffer of events is needed. In the Buffer size input box you define the number of events that the buffer should store. Events are internal events, messages and system variables.

Because the buffer has only a limited size, data may be lost if pre-trigger times are too long. Only the most recent data of the pre-trigger time are then logged, and an error message is output if the Notifications in Write Window option is activated. The data during the post-trigger time or, in the case of the toggle trigger, between the triggers is always logged.

If you wish to log all data beginning at the time the trigger is initiated, then activate the Infinite post-trigger time check box. The Stop after n Trigger Blocks box is then disabled.

Note that an infinite post-trigger time is only available in the Single Trigger mode. In all other trigger modes, this option is deactivated.

Note

  • Notes on time setting.
  • Notes on automatically stopping a measurement:
    • In the last line of the Time window area you can set how many triggers should be logged. If more than one trigger should be logged, activate the option Stop after n triggers. Now you can specify how many triggers should be logged (max. 9999).
    • After the post-trigger time of the nth trigger has elapsed the measurement process is stopped by the trigger, and the data are transferred from the ring buffer to the actual logging file. Depending on the amount of data this memory transfer operation may be associated with substantial waiting times.
    • The post-trigger time must lie in the range of 0 to approx. 6 h.
  • If the post trigger is set by using the setposttrigger CAPL function, the activated option Infinite post-trigger time gets overwritten in the configuration dialog, i.e. it will be ignored. Thus the setting Stop after n Trigger Blocks gets effective (even when grayed).

Advanced Options

Here you can specify additional options for all trigger modes:

  • Stop after n Trigger Blocks
    This options defines if the measurement should be stopped after a specified number of Trigger Blocks. After the post-trigger time of the nth trigger (max. 9999) has elapsed the measurement process is stopped by the trigger, and the data is transferred from the buffer to the actual logging file. Depending on the amount of data this memory transfer operation may be associated with substantial waiting times.
  • Create marker on trigger events
    Activate this check box to set a Marker for each trigger event. The marker name will be [1], [2], ... [N]. These specific global Markers are stored in a logging and will be read when using the logging in offline mode.
  • Notifications in Write Window
    When you activate this option, there will be trigger related notifications in the Write Window.

Manual Trigger (only available with a Logging Block)

  • Start/stop key
    Here you define with which key the logging will be started/stopped.
    Allowed keys: a-z, A-Z.
    You can start, suspend, resume or stop the Logging Block during the measurement by buttons of the Logging Block in the Measurement Setup:
    - Start / Resume button
    - Pause (suspend) button
  • Apply pre- and post-trigger settings
    When you control the logging with a key, the buttons of the Logging Block or specific CAPL commands, you start/stop the trigger (the logging functionality) immediately without any delay and you bypass all pre- and post-trigger settings.
    The settings of the pre- and post-trigger times will only be applied after you have activated the Apply pre- and post-trigger settings option.
  • If logging is inactive and the manual trigger key is pressed, logging will start and the pre-trigger time will be applied
  • If logging is active and the manual trigger key is pressed, the post-trigger time will be applied and logging will stop.
