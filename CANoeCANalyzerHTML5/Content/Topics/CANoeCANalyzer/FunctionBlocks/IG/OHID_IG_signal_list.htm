IG: Signal List

 

The signal values of a message are configured in the Signal List.

  • If you are working with an associated database, those signals appear which were assigned to the selected message in the Signal List.
  • If you are working without an associated database the Signal List remains empty.

Each signal is described by the following boxes

Field name

Comment

SB

Start bit of the signal, taken from the database

Signal name

Name of the signal, taken from the database

Raw value

Raw value of the signal, user-configurable

Phys Value

Physical value of the signal, user-configurable

Converting the raw value into the physical value and vice versa is completed using the conversion formula: [physical value] = ( [raw value] * [factor] ) + [offset]

Unit

Unit of the signal, taken from the database

Dec

Decrementing of the physical value by the physical step

Phys Step

Physical step, calculated with values of the database. User-configurable

Inc

Incrementing of the physical value by the physical step

Wave form generation

Open the configuration dialog for the Signal Generator using the Define button. A signal chart can be defined for the signal in this dialog.

Modification enabled

Only available if the IG is used as a Gateway.

  • If you activate this option, changed signal values will be sent via the gateway.
  • If this option is inactive, changed signal values remain unconsidered.

General

  • Ranges of possible configurable values are given in the status bar.
  • You only need to enter the raw value or physical value, since the corresponding value is always also calculated by the signal's conversion formula.
  • If intermediate values cannot be represented as raw values when entering physical signal values, they are automatically rounded up or down to the next closest value.
  • Depending on the signal type, the representation forms of raw and physical values may deviate between signals.

The following signal types exist:

  • Enumeration (Enum)
    Symbolic selection of defined value descriptions in the input box (Taken from the database)
  • Bit signal
    Check box permits easy toggling (no check mark = 0, check mark = 1)
  • Multiplexor signal
    When signal-multiplexing, various signals are transferred in one message depending on a multiplex value.
  • The signal that is contained in the multiplex value is designated as the multiplexor signal.
  • The signals that are transferred depending on the multiplex value are designated as multiplexed signals.

The multiplex value indicates the current valid signals from all possible message signals.
The following appears in the configuration dialog of the Interactive Generator:

  • Light green multiplexor signals
  • Dark green multiplexed signals

If the multiplex value is changed, the multiplexed signals are automatically updated.

Note

If defined values in the Interactive Generator in the database are changed, the corresponding field is shaded orange.

Example:
The signal value entered in the Raw Value or Phys Value fields is outside the value range that was defined in the database using a minimum and maximum value.
