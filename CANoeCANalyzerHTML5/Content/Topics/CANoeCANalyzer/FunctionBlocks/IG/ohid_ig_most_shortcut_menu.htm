MOST IG: Context Menu of the Transmit List

 

Open the context menu of the MOST IG transmit list using the right mouse button; the following functions are available:

Rename
Rename the [Send] button

Set FBlockID…
Sets the FBlockID for the selected message/messages. Value range: 0x00…0xFF

Set FunctionID…
Sets the FunctionID for the selected message/messages. Value range: 0x000…0xFFF

Set OpType…
Sets the OpType for the selected message/messages. Value range: 0x00…0x0F

Set InstanceID…
Sets the InstanceID for the selected message/messages. Value range: 0x00…0xFF

Set Destination…
Sets the destination for the selected message/messages. Value range: 0x0000…0xFFFF

Set Channel…
Sets the MOST channel for the selected message/messages.

Read configuration from file…
Opens a file dialog to select the file (.igm) from which the configuration (transmit list) should be read. Beside all messages, sequences and keyboard triggers are stored, too.

Write configuration to file…
Opens a file dialog to select the file (.igm) to which the configuration (transmit list) should be written. At first, the transmit list has to be stored in a file with the above mentioned command.

Create message sequence
Generates a transmission sequence from several selected lines. A title line is inserted above the selected lines and identified by both color as well as symbols in the first column. All messages in the sequence can now be transmitted in one step using the send button in the title line or the key inserted into the Triggering column.

Destroy message sequence
Destroys a generated message sequence. The messages in the sequence are not deleted.

Leave sequence group
Deletes a selected message from the message sequence. The message is not deleted from the transmit list.

send via AMS
Activates sending via AMS. With the AMS service, it is possible to send longer messages. Without AMS, the message length is limited to 12 data bytes. Messages that are too long are shortened accordingly and sent.

send via CMS
Activates sending via CMS (standard settings).

AMS -> CMS message sequence
Converts an AMS message into a CMS message sequence. The new CMS message sequence is then inserted into the transmission list directly after the AMS message. The transmission interval between the individual CMS messages is 50 ms. If the AMS message itself is part of a message sequence, it is deleted from that sequence.

Note

This context menu command is only available for AMS messages containing at least 2 telegrams (DLC > 12).
It is also possible to select multiple AMS messages and convert them into one CMS message sequence.
