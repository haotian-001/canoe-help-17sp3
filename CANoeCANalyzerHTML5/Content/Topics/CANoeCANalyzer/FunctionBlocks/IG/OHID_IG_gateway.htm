The IG as a Gateway

 

Note

  • The gateway function is not supported for ARINC-429 messages.
  • The gateway function is no longer available for new configurations.
    If you load an existing configuration that contains the old IG, this function may still be available.

The IG can be used as a gateway. It is possible to transmit single messages and the whole bus communication from one bus to the other one.
The signal values can be changed during the transfer.

You can make all the necessary settings to use the IGs as gateways in the CAN view of the IG configuration dialog.

Step by Step Procedure to transmit a message from one bus to another

  1. Select the CAN view in the transmit list.
  2. Select in the transmit list the messages to be transmitted.
  3. Select in the Message Parameters column in the Channel list the channel (output) for the bus communication.
  4. Activate the Gateway check box in the Triggering column.
    Now the gateway functionality is activated. All other fields of the Triggering column become inactive in the current row.
  5. Select the (input) channel for the bus communication in the Gateway pull down list of the Triggering column.
    When trigger channel and message channel are the same, the channel field of the gateway will be displayed red.

Example for transmitting a message from one bus to another

Note

The following example uses the CANSystemDemo sample configuration.

The EngineData Messages message should be transferred from CAN2 (PowerTrain) to CAN1 (Comfort):

Solution

  • Insert an Interactive Generator into the Simulation Setup on the network PowerTrain.
  • Insert the message into the transmit list using the [New] button.

The symbol selection dialog is opened. Select the messages mentioned above under the network PowerTrain.

  • Select the CAN view in the transmit list.
  • Select the CAN1 channel that should output the bus communication from the Message Parameter column in the Channel list field.
  • Activate the control field from the Triggering column in the Gateway list field and select channel CAN2, from which the selected message should be transferred.

After the measurement process starts, you can see that the selected message is now transferred from CAN2 to CAN1 from the PowerTrain and Comfort Trace Windows. The message ID is displayed in the Comfort Trace Window and not the symbolic name.

Procedure to transmit the complete bus communication from one bus to another 

  • Select the CAN view in the transmit list.
  • Enter an * in the Identifier field of the Message Parameters column.
  • Select in the Message Parameters column in the Channel list the channel (output) for the bus communication.
  • Activate the Gateway check box in the Triggering column.

Now the gateway functionality is activated. All other fields of the Triggering column become inactive in the current row.

  • Select the (input) channel for the bus communication in the Gateway pull down list of the Triggering column.

When trigger channel and message channel are the same, the channel field of the gateway will be displayed red.
