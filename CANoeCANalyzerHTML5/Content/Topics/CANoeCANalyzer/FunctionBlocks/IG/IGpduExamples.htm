Example 1: Add a PDU to the PDUs View

Note

The following example uses the System Configuration (CAN) AUTOSAR sample configuration.

Situation

The following PDUs from the assigned database PowerTrain should be triggered with the trigger conditions shown:

  • ABSdata triggers cyclically (50 ms cycle time). Update payload periodically every 50 ms.
  • EngineData triggers interactive (key press or Triger button) and update.

Solution

  • In the Simulation Ribbon Tab, click the PDU Interactive Generator button. In the list under New, select the PDU Interactive Generator command. A new window of the PDU IG will appear.
  • Deactivate all other active function blocks in the Simulation Setup.
  • In the toolbar of the PDU IG, click the button . The Symbol Selection dialog appears. In PowerTrain under PDUs, select the ABSdata and EngineData PDUs.
  • Configure the ABSdata PDU via the Timing column so that it is transmitted cyclically with a cycle time of 50 ms.
  • Activate periodic update by clicking the button in the Trigger column. In the Trigger Settings column, configure the PDU to update periodically every 50 ms.
  • Now configure the EngineData PDU so that it is triggered interactively by pressing a key. Leave the setting in the Timing column at the Event based (10 ms) setting.
  • Start the measurement.
  • Using the Analysis Ribbon Tab and the Trace list, open the Trace PowerTrain Trace Window.
  • The ABSdata PDU is transmitted cyclically every 50 ms.
  • Clicking the button of the EngineData PDU or pressing the defined trigger key trigers the PDU over the network.

Example 2: Configure the PDU so that it is triggered manually

To configure a PDU in such a way that it can be triggered manually, open the window of the PDU IG and add the desired PDU to the PDUs view.

Situation

A PDU should then always be triggered if the button in the Trigger column is clicked.

Solution

  • In the PDUs or Properties of the PDUs view, open the list in the Trigger column using the arrow button.
  • Change the trigger condition to Manual by activating the corresponding option.
  • In the Timing column select the Event based option.

Example 3: Configure the PDU so that it is triggered cyclically

To configure a PDU so that it is triggered cyclically, open the PDU Interactive Generator window and add the desired PDU to the PDUs view.

Situation

A PDU is to be triggered cyclically.

Solution

  • In the PDUs or Properties of the PDUs view, open the list in the Timing column using the arrow button.
  • Change the Timing condition to Cyclic by activating the corresponding option.
  • Define the cycle time of the PDU insert a value in milliseconds into the Periode input box and, if necessary, enter an offset below (also in milliseconds).
  • When the setting corresponds to your requirements, click outside the visible control to close it and accept your changes.

Example 4: Configure a PDU so that it is triggered with the push of a button

To configure a PDU so that it is triggered with the push of a button, open the window of the PDU IG and add the desired PDU to the PDUs view.

Situation

A PDU is to be triggered by pressing the P key.

Solution

  • In the PDUs or Properties of the PDUs view, open the list in the Trigger column using the arrow button.
  • Change the trigger condition to Manual by activating the corresponding option.
  • Enter a valid trigger key for the PDU (e.g. P) in the Trigger key input box.
  • In the Timing column, select the Event based setting and specify a debounce delay if necessary.
  • When the settings correspond to your requirements, click outside the visible control to close it and accept your changes.

Example 5: Configure a PDU so that it is triggered when a system variable value is changed

To configure a PDU in such a way that it is triggered when a system variable value is changed, open the window of the PDU IG and add the desired PDU to the PDUs view.

Situation

A PDU is to be triggered when the TriggerVariable system variable value is changed. This variable must already be contained in the configuration and may not be defined as Use for analysis only.

Solution

  • In the PDUs or Properties of the PDUs view, open the list in the Trigger column using the arrow button.
  • In the Trigger Settings column change the trigger condition to PDU/System variable by activating the corresponding option.
  • Open the Symbol Selection dialog using the [...] button.
  • Click the symbol for system variables in the top part of the Symbol Selection dialog if it is not already active.
  • In the list of system variables, select the TriggerVariable variable.
  • Close the dialog with [OK].
  • In the Timing column, select Event Based. If necessary, enter a debounce delay in the input box of the option.
  • When the settings correspond to your requirements, click outside the visible control to close it and accept your changes.

Example 6: Configure a Signal Generator for the signal of a PDU from an assigned database

You can configure a Signal Generator directly in the PDU IG. The Generator Control and Generator Type columns are available for this purpose.

Situation

The value of the CarSpeed signal is to be changed with a Signal Generator of type Sine.

Solution

  • If the PDU IG does not yet contain the ABSdata PDU, please first execute example 1: Add a PDU to the PDUs View.
  • In the Signals view, the CarSpeed signal appears as the top line if the view is sorted by the start bits of the signals. Click here on the arrow in the Generator Type column of the row and select the type Sinus in the appearing list.
  • In the Generator Control column, click on the button , if you want to make changes to the settings of the Signal Generator (e.g. amplitude, period, phase).
  • Start the measurement. The Signal Generator is started automatically and changes the value of the CarSpeed signal to a sine wave.

Example 7: Drag and drop PDUs

It is possible to move PDUs within the PDUs view in the window of the PDU IG by drag and drop them. If no PDU IG is available, do the Add a PDU to the PDUs view example first.

Situation

The position of the EngineData PDU is to be changed from Position 2 to Position 1.

Solution

  • Select the EngineData PDU.
  • Press and hold the left mouse button to drag the PDU above the ABSdata PDU. Free space for inserting a new line is displayed.
  • When you release the left mouse button, the PDU is placed above the ABSdata PDU and the line number changes from 2 to 1.

Example 8: Drag and drop PDUs from the Symbol Explorer

It is possible to move PDUs from an assigned database by drag and drop them from the Symbol Explorer to the PDU IG. If no PDU IG is available, do the Add a PDU to the PDUs view example first.

Situation

A PDU is to be added from an assigned database to the PDU IG.

Solution

  • Open the Symbol Explorer by clicking the Symbol Explorer button in the Environment Ribbon Tab.
  • Select the PDU from an assigned database in the Symbol Explorer.
  • Press and hold the left mouse button to move the PDU to the PDUs view of the PDU IG. The mouse pointer shows the insert symbol.
  • Release the left mouse button. The PDU is inserted into the PDU IG. The properties and signal values of the PDU are specified by the database entry of the PDU.

Example 9: Drag and drop PDUs from the Trace Window

It is also possible to move PDUs from the Trace Window to the PDU IG by drag and drop them.

Situation

A PDU (including all properties of the PDU) from the Trace Window is to be triggered with the PDU IG.

Solution

  • Select the PDU.
  • Press and hold the left mouse button in the Trace Window while dragging to the PDUs view of the PDU IG.
  • When you release the left mouse button, the PDU is placed in the position at which free space for inserting a new line is displayed. The properties of the PDU and the database link are the same as in the Trace Window.

Example 10: Customize columns using the toolbar

By default, not all property columns are displayed in the PDUs and Signals views. You can use the toolbar to customize the columns.

Situation

The DB comment property is not displayed in the PDUs view.

Solution

  • Click on the toolbar of the window.
  • The Customize dialog opens and shows the columns which can be added.
  • Select DB comment.
  • Press and hold the left mouse button and add the column in the position in the PDUs view at which the column is to appear by dropping it there.
  • Two arrows indicate the position at which the column is inserted after you released the left mouse button.

The same process is available for the Signals view. Use the button on the toolbar for this.

Example 11: Configure a PDU so that it is triggered when another PDU is received.

To configure a PDU to trigger when it receives another PDU, open the window of the PDU IG and add the desired PDU to the PDU view.

Situation

A PDU is to be triggered when another PDU is received.

Solution

  • In the PDUs view or PDU Properties view, in the Trigger column, open the list using the arrow button.
  • In the Trigger Settings column, change the trigger condition to PDU/System Variable by activating the corresponding option.
  • Open the Symbol Selection dialog via the [...] button.
  • Click the PDU icon at the top of the Symbol Selection dialog if it is not already active.
  • In the list of PDUs, select the PDU to which you want to trigger.
  • Close the dialog with [OK].
  • In the Timing column, select Event Based. If necessary, enter a debounce delay in the input field of the option.
  • If the settings match your needs, click outside the visible control to close them and apply your changes.
