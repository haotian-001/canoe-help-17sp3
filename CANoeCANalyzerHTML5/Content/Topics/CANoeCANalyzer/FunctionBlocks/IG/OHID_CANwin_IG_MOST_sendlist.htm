MOST IG: Transmit List

 

The transmit list is located in the top area of the configuration dialog of the MOST IG.

The various buttons can be used to add, delete and rearrange messages in the transmit list.

Every message that is to be sent via the MOST IG must be included in the transmit list. Each message is described in one line.

You can send the individual messages with the press of a button or a key from the send list during the measurement operation.

Setup of the transmit list

Send

With this button, located at the beginning of each row, you can send the corresponding message during the measurement operation.
The button can be renamed via the context menu.

If sending is disabled for a message the Send button is shown in gray.
This may be the case if:

  • The AMS service should be used although this service is inactive for the channel.
  • FBlock, FunctionId and OpType were not input completely

Normal messages can be sent out over the Application Message Service (AMS) or Control Message Service (CMS). This can be configured from the transmit list’s context menu for the selected messages. For this purpose, AMS must be activated for the channel being used (Network Hardware Configuration). If more parameter bytes have been configured than fit into a Ctrl frame, AMS automatically performs fragmentation for the transmission.

Note

In the Parameters area there is a check box for each individual message indicating whether AMS should be used.

Message Type

  • Normal message
    • FBlock
      … Function block, determines an application (e.g. amplifier or CD player)
    • Function
      … a function within the specified FBlock
    • OpType
      … an operation within the specified function

Together, these three fields define the type of a message.

If the message is described in an integrated XML-function catalog, the symbolic names are displayed in the fields. In that case the input of FBlock, Function and OpType can be done via a choice list. To this end, the corresponding function catalog must have been assigned to this message‘s transmission channel. If no description is available for parts of the message, the parts that are not contained in the function catalog are represented numerically and backlit in color.

Example:
The FBlock entered is described in the function catalog, but the selected function is not. Therefore, the name of the FBlock is displayed, but the information in the Function field is represented numerically and against a colored background.

  • Raw-/System messages
    Via the context menu of the [New] button you can insert a line for a raw-/system message in the send list (default: Normal). Now choose the type of the system message (Normal, Remote Read, Remote Write, Allocation, Deallocation or Get Source) from the list.
  • Raw packets
    Via the context menu of the [New] button you can insert a line for a raw packet.
  • Sequences
    A number of lines of the Send List can be grouped together to form a send sequence. To do this, mark the desired lines. Then a sequence can be created using the transmit list's context menu.
    This involves adding a title line above the selected lines and indicating in the first column, in color and by symbols, that these messages belong together.
    All messages of the sequence can then be sent out with an action by pressing the Send button in the title line, or by pressing the key configured under the Trigger column. Furthermore, individual messages can be sent separately when either the Send button or the key configured for the individual message is used.
    From the transmit list's context menu the sequence can also be re-triggered or a selected message can be taken from the sequence.

Addressing

  • Instance

In MOST rings with dynamic address allocation, the desired function block can be addressed by FBlock and InstanceId.

If the address FFFF is entered in the Destination field, the Address Handler uses the FBlock & InstanceId to determine the logical address of the device to which the message should be sent.

The following preconditions must be satisfied:

  • The Address Handler must be activated for the channel being used
  • The specific message has a Command Optype (OpTypes: 0..8), e.g. 'Set'
  • The ring is in the ConfigOK state

Note

  • System messages do not have the field instance. Therefore this field will not be used for system messages. It will be empty and it is not possible to alter it.
  • Even for CMS messages of the Command Optype it is now possible to enter 0xFFFF as the target address; the Address Handler then convert this using the FblockId and InstanceId and sends it to the correct device.
  • Destination
    … the physical address of the recipient.
    Nodes may be specified here as well as node position addresses or group addresses.
  • Channel
    … the channel on which the message is to be sent.

Triggering

  • Key
    Sending of the message can also be triggered by a key press. To do this the relevant key must be noted in this column. Valid keys are a to z and 1 to 0. If the box is left empty the message cannot be sent out by key press.
  • Time Distance
    The send interval for messages may be configured in milliseconds. The time entry always refers to the time interval to the previous message.
    If the time is entered in the title line of the sequence, then this interval is automatically entered in all message lines in this sequence.

General

  • The numerical values are represented in hexadecimal or decimal depending on the global setting. Entries can be made accordingly in hexadecimal or decimal. Selection lists containing all the entries of the assigned function catalogs are also offered for the fields of the message type.
  • If the last, gray row in the transmission log is clicked with the mouse, a new, blank row is added to the send list. This can also be filled out without an integrated function catalog.
  • The [Send] button can be renamed to document the purpose of the message to be sent and make the configured Interactive Generator easier to use.
    This is also a way to create a simple panel (see Typical MOST applications for the Interactive Generator).
    To rename the button open the context menu with the right mouse button and choose Rename. The name of the button can now be edited directly.
