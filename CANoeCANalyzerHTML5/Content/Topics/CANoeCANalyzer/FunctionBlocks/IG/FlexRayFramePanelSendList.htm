Transmit List

 

The transmit list is divided into the Static Frames and Dynamic Frames tabs.

You must enter a frame in the list for each data update to be carried out.

Fields for Static and Dynamic Frames

  • Action
    This button is used to transmit the set payload values to the network interface:

    • Tx Flag: Time Triggered
      Frame whose transmission is time-triggered. The FlexRay controller controls the actual transmission.
    • Tx Flag: Event Triggered
      Frame whose transmission is event-triggered. Transmission is initiated in addition to the simple data update.

  • Name
    Displays the symbolic name of the frame. If not present, the column remains empty.
  • ID
    Displays the symbolic name of the frame. If not present, the column remains empty.
  • Offset
    This field describes the basic cycle when using cycle multiplexing.
  • Repetition
    This field describes the repetition factor with the one frame when using cycle mutliplexing.
    For example, "2" means repeated transmission in every second communication cycle.
  • Cluster
    Specifies the FlexRay cluster on which the frame is being transmitted.
  • Channel
    Channel A, B or both are available.
  • PP
    Configures the Payload Preamble flag for the configured frame.

  • Tx Flag
    The following options are available:

    • Time Triggered
      Default setting for static frames.
      The communication controller transmits the existing data content in each cycle.
      It is not checked whether the data has already been sent.
    • Event Triggered
      Default setting for dynamic frames.
      The data content is transmitted exactly once.

    In the static range, the network interface transmits Null Frames after successful dispatch for all subsequent transmission times.
    In the dynamic range, the mini slot remains empty.

  • Initial TX
    On measurement start, a one-off update (transmit job) is executed.
    Based on further settings (Tx flag) the frame will be sent once or multiple.
    Only the definition of a frame in the transmit list of the Frame Panel leads to a (frame-based) TX buffer. Therefore, the Initial Tx flag is not relevant.
    If the frame is used in diagnostic context, Initial Tx should be inactive to avoid useless frames.
  • Length (Byte)

Fields for Static Frames Only

  • Startup
    Configures the startup Flag for the configured frame.
  • Sync
    Configures the Sync Flag for the configured frame.
  • NF – only for Vector Interfaces
    Configures the zero frame for the configured frame.
  • Length (byte)
    Contains the payload length of the network in bytes. The length is based on the payload length valid for the network (gPayloadLengthStatic) and can not be changed.

Fields for Dynamic Frames Only

  • Max Payload (byte) – not for Vector Interfaces
    The payload length can be changed in the dynamic range. In this column, the current payload length valid at the time of configuration is take over as the maximum value.
  • Act. Payload (byte) – not for Vector Interfaces
    Contains the payload length configured for the transmission time. The payload length can be changed during the measurement. However, it cannot overwrite the configured maximum value.
  • Length (byte) – only for Vector Interfaces
    Contains the payload length of the dynamic frame in bytes.

The Following Buttons are Available for Configuring the Transmit List:

  • [Add Frame...]
    Opens the Symbol Selection dialog.
    Select the desired frame and close the dialog with [OK]. You can also select and accept multiple frames.
    You can also add a frame manually by entering an identifier in the last row in the Frame ID column.
  • [Add Row]
    Adds an empty input line to the end of the transmit list.
    If in a new line neither a new frame is entered via the symbolic selection dialog nor an identifier is entered in the Frame ID column, the line is shaded red.
  • [Delete Row]
    Deletes the highlighted row from the transmit list.

Note

The entries in the columns are validated. The red color of the affected rows immediately identifies missing entries or inconsistencies that have occurred.

The cause is described in the tooltip.
