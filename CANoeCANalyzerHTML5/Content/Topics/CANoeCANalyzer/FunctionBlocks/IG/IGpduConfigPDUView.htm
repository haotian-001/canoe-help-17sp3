PDUs

 

Via the PDUs view you can configure AUTOSAR PDUs, that you want to send over the network. You can add AUTOSAR PDUs from an assigned database via the symbolic selection dialog and adjust their properties before sending.

To customize the columns use the toolbar button . A description of each column shows the table below.

The PDU Property view always displays all properties.

Column

Options

Displayed by default

Description

Row

—

Yes

Displays the row number of the PDU.

Trigger

Yes

If the trigger condition type is manual, the update of the PDU payload is triggered, when you click this button.

If the trigger condition type is periodic, this starts the periodic update of the payload of the PDU with the specified period duration.

Note: The PDU is not sent immediately, only its payload is updated! Whether and when the PDU is sent depends on the set timing: See description for Timing column.

—

If the trigger condition is periodic and currently active, this button appears. A click on this button stops the periodic update of the payload of the PDU.

Trigger Now

No

Triggers the update of the payload of the PDU exactly once. Whether and when the PDU is sent depends on the timing set. (See also note above and description of Timing column).

Trigger

manual/ on key

Yes

A click on the button updates the payload of the PDU. Additionally a trigger key can be defined for the PDU. Pressing the key then has the same effect as clicking the button.

periodic

No

A click in the button updates the payload of the PDU with the set period duration.

PDU

No

When the specified PDU is received on any channel, the update of the PDU payload is triggered.

system variable

No

When updating or changing the value (selectable) of the selected system variable, the payload of the PDU is updated. The variable must not be marked as Use only for analysis.

Name

—

Yes

The Name of the PDU from the assigned database.

Timing

external

Yes

The timing of the PDU is not affected. An external timing e.g. by the interaction layer controls the transmission of the PDU.

event based

Yes

The PDU is sent immediately in the event of a payload change, taking into account the set debounce delay. The debounce delay specifies the minimum time in milliseconds that must elapse before the PDU can be sent a second time.

cyclic

Yes

The PDU is sent cyclically with the set period (in milliseconds). The first transmission takes place after the time period specified with Offset (in milliseconds).

Length

—

Yes

Length of the PDU in bytes.

DB comment

—

Yes

Comment of the PDU from the database.

Comment

—

Yes

User-defined comment.

To show the Context Menu of a AUTOSAR PDU set within the PDUs view a right click on the row of the AUTOSAR PDU. The Context Menu shows following commands:

Add PDU from Database
Opens the Symbol Selection dialog, to insert a PDU from an assigned database.. A click on [OK] inserts the PDU in the PDUs view.

Search...
Opens the search input box for the PDUs view.

Cut
Cuts the selected PDU from the PDUs view.

Copy
Copies the selected PDUs into the clipboard.

Paste
Pastes the copied PDU from the clipboard under the current selected PDU in the PDUs view.

Delete
Deletes the selected PDUs from the.PDUs view.

Select all PDUs
Selects all PDUs contained within the PDUs view.

To change the position of one or more AUTOSAR PDU within the PDUs view, a drag and drop functionality is available.
