Example 1: Add CAN Interactive Generator

Situation

A CAN Interactive Generator should be added to the Simulation Setup.

Solution

  • Select the bus line in the Simulation Setup and open the context menu.
  • Select the Insert CAN Interactive Generator command to insert the block.

Alternative Solution

  • In the Simulation Setup open the System View (via context menu).
  • In the System View set a right click on Interactive Generators.
  • Select the Insert CAN Interactive Generator command to insert the block.
Example 2: Add Frames to Frames View

To add frames to the Frames view two options are available:

  • Add raw CAN/CAN FD frame
  • Add CAN frame from an assigned database
Example 3: Add Raw CAN/CAN FD Frame to Frames View

Note

The following example uses the System Configuration (CAN) sample configuration.

Situation

The following frames should be sent with the trigger conditions shown:

  • CAN frame with Id 0x7ff on CAN2 periodic (cycle time 33 ms)
  • Extended CAN FD frame with Id 0x900x on CAN2 interactive (key press or Send button)

Solution

  • Add a CAN Interactive Generator in the Simulation Setup of the PowerTrain network.
  • Disable all other active blocks.
  • Add the frames to the Frames view by using the button for the CAN frame and the button for the CAN FD frame of the toolbar.
  • Change the IDs of the CAN and CAN FD frame to 0x7ff and 0x900x within the ID column.
  • Configure the frame 0x7ff to be sent periodically with a period of 33 ms.
  • Activate cyclic triggering by clicking the button within the Send column.
  • Configure the frame 0x900x to be sent interactively by triggering on key press.
  • Change for both frames the channel from CAN 1 to CAN 2 within the Channel column.
  • Start the measurement.
  • Via the Analysis Ribbon Tab and the Trace list open the Trace Window Trace PowerTrain.
  • The frame 0x7ff is send with a period of 33 ms.
  • If the button of the frame 0x900x or the defined trigger key is pressed, the frame is send on the bus.
Example 4: Add CAN Frames from an Assigned Database to Frames View

Note

The following example uses the System Configuration (CAN) sample configuration.

Situation

The following CAN frames from an assigned database should be sent with the trigger conditions shown:

  • EngineData periodic (cycle time 33 ms)
  • GearBoxInfo interactive (key press or Send button)

Solution

  • Add a CAN Interactive Generator in the Simulation Setup of the PowerTrain network.
  • Disable all other active blocks.
  • Add the frames to the Frames view using the button of the toolbar.
  • The Selection of Frames and Bus Events dialog is opened. Select the two frames EngineData and GearBoxInfo under the network PowerTrain.
  • Configure the frame EngineData to be sent periodically with a period of 33 ms.
  • Activate cyclic triggering by clicking the button within the Send column.
  • Configure the frame GearBoxInfo to be sent interactively by triggering on key press.
  • Start the measurement.
  • Via the Analysis Ribbon Tab and the Trace list open the Trace Window Trace PowerTrain.
  • The frame EngineData is send with a period of 33 ms.
  • If the button of the frame GearBoxInfo or the defined trigger key is pressed, the frame is send on the bus.
Example 5: Configure Frame to be Sent Manually

To configure a frame to be sent manually insert a CAN Interactive Generator in the Simulation Setup and add a frame to the Frames view.

Situation

A frame should be send when the button is clicked in the Send column.

Solution

  • In the Frames view or Frames Property view, in the Trigger column open the list via the arrow button.
  • Change the trigger condition to Manual by activating the corresponding option.
Example 6: Configure Frame to be Sent Periodically

To configure a frame to be sent periodically insert a CAN Interactive Generator in the Simulation Setup. Add then a frame to the Frames view.

Situation

A frame should be send periodically.

Solution

  • In the Frames view or Frame Property view, in the Trigger column open the list via the arrow button.
  • Change the trigger condition to Periodic by activating the corresponding option.
  • Define the period of the frame by using the input box or the slider.
  • If setting fit your requirements click outside the visible control to close it and accept your changes.
Example 7: Configure Frame to be Sent by Key Press

To configure a frame to be sent by key press insert a CAN Interactive Generator in the Simulation Setup. Add then a frame to the Frames view.

Situation

A frame should be triggered with the key a.

Solution

  • In the Frames view or Frame Property view, in the Trigger column open the list via the arrow button.
  • Change the trigger condition to Manual by activating the corresponding option.
  • Insert a valid trigger key for the frame, e.g. a, in the On Key input box.
  • If the settings fit your requirements click outside the visible control to close it and accept your changes.
Example 8: Configuring the frame so that it is sent when a trigger frame is received.

To configure a frame to be sent when a trigger frame is received, insert a CAN Interactive Generator into the Simulation Setup. Then add a frame to the Frames view.

Situation

A frame is to be triggered by the trigger frame with CAN ID 0x100 on CAN channel 2.

Solution

  • In the Frames view or the Properties view of the frames , open the list in the Trigger column using the arrow button.
  • Change the trigger condition to Frame/System Variable by activating the corresponding option.
  • Enter the CAN ID 100 in the Frame/System Variable input field (if CANoe is in hexadecimal mode).
  • Select the channel CAN 2 in the selection field behind the input field Frame/System Variable (Only channels available in the configuration are displayed).
  • If the settings match your requirements, click outside the visible control to close them and accept your changes.
Example 9: Configure the frame so that it is sent when the value of a system variable changes.

To configure a frame to be sent when the value of a system variable changes, insert a CAN Interactive Generator into the Simulation Setup. Then add a frame to the Frames view.

Situation

A frame is to be triggered by the value change of the system variable TriggerVariable. This variable is already part of the configuration and is not defined as Use only for analysis.

Solution

  • In the Frames view or the Properties of Frames view, open the list in the Trigger column using the arrow button.
  • Change the trigger condition to Frame/System Variable by activating the corresponding option.
  • Open the symbol selection dialog via the [...] button.
  • Click on the system variable icon in the upper area of the symbol selection dialog if it is not already active.
  • Select the TriggerVariable variable from the list of system variables.
  • Close the dialog with [ok].
  • If the settings match your requirements, click outside the visible control to close them and accept your changes.
Example 10: Configure a J1939 parameter group so that it is sent with CAN FD.

To configure a J1939 parameter group to be sent according to J1939-22, add the corresponding frame to the Frames view and set the Type column to J1939-22 PG. The frame is now interpreted as a J1939 parameter group that is sent according to J1939-22. The sending method depends on the DLC values and Dest values:

.

  • DLC ≤ 60 bytes, Dest ≠ All: The J1939 parameter group is sent as a Contained-PG within a FEFF Multi-PG. Only one Contained-PG can be sent per Multi-PG.

    .

  • DLC ≤ 60 bytes, Dest = All: The J1939 parameter group is sent as Contained-PG within a FBFF Multi-PG. Only one Contained-PG can be sent per Multi-PG.

    .

  • DLC > 60 bytes, Dest ≠ All: The J1939 parameter group is sent with FD.TP (RTS/CTS). Successful transmission requires another node that supports FD.TP as a receiver.

    .

  • DLC > 60 bytes, Dest = All: The J1939 parameter group is sent with FD.TP (BAM).

    .

To configure a J1939 parameter group to be sent in a CAN FD network as a CEFF (CAN) or FEFF (CAN FD) frame, set the value in the Type column to Extended CAN Data or Extended CAN FD.

Example 11: Configure Signal Generator for Signal of CAN Frame from an Assigned Database

Signal Generators are only available for CAN frames from an assigned database.

Note

The following example uses the System Configuration (CAN) sample configuration.

Situation

The value of the EngSpeed signal of the EngineData frame should traverse the range of the signal.

Solution

  • Add a CAN Interactive Generator in the Simulation Setup of the PowerTrain network.
  • Disable all other active blocks.
  • Add the frames to the Frames view using the button of the toolbar.
  • The Selection of Frames and Bus Events dialog is opened. Select the frame EngineData under the PowerTrain network .
  • For the EngSpeed signal configure the Signal Generator Value Range (If the column is not displayed add the Generator Type column and Generator settings) via the button of the toolbar.
  • Start the measurement.
  • Via the Analysis Ribbon Tab and the Trace list open the Trace Window Trace PowerTrain.
  • The frame EngineData is send with a period of 33 ms.
  • The value of the EngSpeed signal sweep from min to max value of the signal.
  • To see the graphical representation open the Graphics Window and the fit the Y-Axis of the EngSpeed signal.
Example 12: drag and drop Frames

It is possible to drag and drop frames within the Frames view of the CAN IG. If no CAN IG is available execute first the example Add CAN Interactive Generator and Add raw CAN/CAN FD frame to Frames view.

Situation

The position from a frame should be changed from position 2 to 1.

Solution

  • Select the frame 0x900x.
  • Press and hold the left mouse button and drag the frame above the frame 0x7ff. A row maker will be displayed.
  • Release the left mouse button, the frame is dropped over the frame 0x7ff and the row number changed from 2 to 1.
Example 13: drag and drop Frames from Symbol Explorer

It is possible to drag and drop CAN frames from an assigned database from the Symbol Explorer to the CAN IG. If no CAN IG is available execute first the example Add CAN Interactive Generator.

Situation

A CAN frame from an assigned database should be added to the CAN IG.

Solution

  • Open the Symbol Explorer via the Environment Ribbon Tab and the Symbol Explorer button.
  • Select the CAN frame from an assigned database in the Symbol Explorer.
  • Press and hold the left mouse button and drag the frame to the send list view of the CAN IG. The mouse cursor will display the insert icon.
  • Release the left mouse button. The frame is inserted in the CAN IG. The frame properties and signal values are determined using the database entry for the frame.
Example 14: drag and drop Frames from Trace Window

It is also possible to drag and drop CAN frames contained within the Trace Window to the CAN IG.

Situation

A frame (including all properties of the frame) from the Trace Window should be send with the CAN IG.

Solution

  • Select the frame.
  • Press and hold the left mouse button within the Trace Window and drag over to the Frames view of the CAN IG.
  • Release the left mouse button, the frame is dropped on the position where the row marker is displayed. The frame properties and database association are the same as in the Trace Window.
Example 15: Customize Columns via Toolbar


By default not all property columns are displayed within the Frames view and the Signals view. To customize the columns you can use the toolbar.

Situation

The IDE property is missing within the Frames view.

Solution

  • Press within the toolbar.
  • The Customize dialog will open and show the columns could be added.
  • Select IDE.
  • Press and hold the left mouse button and add the column via drag and drop on the position within the Frames view.
  • Two arrows shows the position where the column is inserted after release the left mouse button.

The same procedure is available for the Signals view. Use therefore in the toolbar.
