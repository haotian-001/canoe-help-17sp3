Frames

 

Via the Frames view you can configure frames that you want to send over the CAN bus. You can add CAN frames from an assigned database via the symbolic selection dialog and adjust their properties before sending. Alternatively you add CAN or CAN FD frames and modify their identifier or further properties.

To customize the columns use the toolbar button . A description of each column shows the table below.

The Frame Property view always displays all properties.

Column

Options

Displayed by default

Description

Row

—

Yes

Displays the row number of the frame.

Send

Yes

If the trigger condition type is manual, this button sends the frame.

If the trigger condition type is periodic, this starts the periodic transmission of the frame.

—

If the trigger condition is periodic and currently active, this button appears. Pressing it stops the periodic transmission of the frame.

Send Now

No

This button sends the frame exactly once, even if a periodic transmission is defined in the Trigger column.

Trigger

Manual / On Key

Yes

The frame is sent on bus when the button  is pressed. Additionally a trigger key can be defined for the frame. If the key is pressed the frame is sent on bus. Manual and OnKey trigger conditions can be used together.

Periodic

No

The frame is sent on bus, with the defined period, after the button  is pressed.

Trigger frame

No

When the specified trigger frame is received on the specified channel, the frame is transmitted on the bus.

System Variable

No

When updating or changing the value (selectable) of the selected system variable, the frame is sent on the bus. The variable must not be marked as Use only for analysis.

Name

—

Yes

The name of the frame. If a CAN frame from an assigned database is used, this column is not editable for this frame.

ID

—

Yes

The identifier (ID) of the frame. If the ID has the suffix x, the ID is an extended ID (29-bit).

Channel

—

Yes

The CAN channel the frame should be send on.

Type

CAN Data

Yes

Frame is a standard CAN frame with an 11-bit identifier.

CAN Remote

—

Frame is a standard CAN remote frame with an 11-bit identifier.

CAN FD

—

Frame is a standard CAN FD frame with an 11-bit identifier.

Extended CAN Data

—

Frame is an extended CAN frame with a 29-bit identifier.

Extended CAN FD

—

Frame is an extended CAN FD frame with a 29-bit identifier.

Extended CAN Remote

—

Frame is an extended CAN remote frame with a 29-bit identifier.

DLC

—

Yes

The data length code of the frame.

For J1939 transport protocol frames this column shows the payload length.

DB comment

—

No

Displays the frame comment from the database.

BRS

—

No

If the frame is a CAN FD frame, the bitrate switch can be enabled / disabled using this column.

Comment

—

No

User-defined comment for the frame.

IDE

—

No

Displays the value of the IDE flag. If the frame is an extended CAN frame, the check box is activated.

RTR

—

No

Displays the value of the RTR flag. If the frame is a CAN remote frame, the check box is activated.

Protocol

—

No

Displays the protocol type of the frame.

Prio (GMLAN)

—

Yes (onlywith GMLAN DB)

Displays the GMLAN Priority of the frame.

SRC (GMLAN)

—

Yes (only with GMLAN DB)

Displays the GMLAN Source of the frame.

PID (GMLAN)

—

Yes (only with GMLAN DB)

Displays the GMLAN Parameter ID of the frame.

DP (J1939)

—

Yes (only with J1939 DB)

Displays the J1939 Data Page of the frame.

EDP (J1939)

—

Yes (only with J1939 DB)

Displays the J1939 Extended Data Page of the frame.

SRC (J1939)

—

Yes (only with J1939 DB)

Displays the J1939 Source of the frame. The predefined node addresses refer to the database values.

DST (J1939)

—

Yes (only with J1939 DB)

Displays the J1939 Destination of the frame. The predefined node addresses refer to the database values.

Prio (J1939)

—

Yes (only with J1939 DB)

Displays the J1939 Priority of the frame.

PGN (J1939)

—

Yes (only with J1939 DB)

Displays the J1939 Parameter Group Number of the frame.

To show the context menu set a right click within the Frames view. The context menu shows following commands:

Add Frame from Database
Open the symbol selection dialog, to insert a CAN frame from an assigned database.

Add CAN FD Frame
Add a CAN FD frame without database connection.

Add CAN Frame
Add a CAN frame without database connection.

Search...
Opens the search input box for the Frames view.

Cut
Cut the selected frame from the Frames view.

Copy
Copy the selected frames.

Paste
Paste the copied frames under the current selected frame.

Delete
Delete the selected frames.

Select All Frames
Select all frames contained within the Frames view.

To change the position of one or more frames within the Frames view, a drag and drop functionality is available.
