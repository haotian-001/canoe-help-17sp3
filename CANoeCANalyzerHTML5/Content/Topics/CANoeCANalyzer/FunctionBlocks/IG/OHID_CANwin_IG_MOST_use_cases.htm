MOST IG: Application Examples

 

In the following, three typical scenarios for using the MOST IG will be presented.

Example

Addressing Real or Simulated MOST Devices

Testing is to be carried out to determine whether a given real or simulated MOST device responds correctly to control commands (MOST messages).

In this example a MOST CD player is to be tested.

On principle, a panel may be created and the control buttons connected with CAPL programs; these can then compile and transmit the corresponding MOST messages.
However, the procedure is simpler (and faster) with the MOST IG.

An MOST IG must first be inserted in a MOST bus in the Simulation Setup. Then all relevant MOST messages can be entered in the transmit list via the configuration dialog.

In the case of a MOST CD-Player the following messages are used:

  • CDPlayer.Track.Set with data value 'Track: 7' to set the CD-Player to the seventh title of the inserted audio CD.
    Rename the send button to "Track 7".
  • CDPlayer.Deckstatus.Set with data value 'Mode: Play' to play back the selected title.
    Rename the send button to "Play".
  • CDPlayer.Track.Get to obtain information from the CD-Player on which title is currently being played.
    Rename the send button to "Get Info".

The reaction to the Set messages can be observed on the real CD-Player and the CD-Player's reaction to the Get request can be seen in the Trace Window.

Note

In a context menu the Send buttons can be named according to their functionalities. Here for example they are named "Track 7", "Play" or "Get Info". Afterwards the user can reduce the size of the configuration dialog so that only the Send buttons are visible. This is a way to get a simple operating panel.

Manual Simulation of MOST Devices 

MOST devices that essentially transmit MOST messages and do not have to respond to incoming messages within a certain period can be simulated manually with an MOST IG. This method is especially well-suited to the simulation of control units.

The messages that such a device would send can be configured in the transmit list of the MOST IG very easily. If necessary, the user can send the messages with the press of a button. For example, these may be control commands for the CD player that a control unit would send.

In the context menu of the Simulation Setup / Test Setup a name can be assigned to each MOST IG which then appears on the MOST IG title bar and the Windows task bar. This makes it easy to identify MOST IG when there are several of them.

Intervention During Operation of a MOST System 

The MOST IG can be used to intervene while a MOST system is operating. At the same time, any message can be selected from a function catalog and sent even at runtime.

Example:

The 'Play' command is sent to the CD player from a panel. Using the MOST IG, the message 'CDPlayer.DeckStatus.Set' is sent to the bus with data value 'Stop'. Now the behavior of the panel can be observed, as it is actually still in 'Play' mode

It is also possible to compile and send deliberately incorrect or invalid messages. In this way, it is possible to test the response of real or simulated MOST devices to such messages.

Example:

Sending the message 'CDPlayer.Track.Set' with invalid data value (e.g. invalid track number) to the CD player.
