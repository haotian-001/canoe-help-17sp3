MOST IG: Parameter Tree

 

Each message is assigned a parameter tree, in which the parameter values can be set. The parameter tree for the message marked in the send list is displayed.

The parameter tree is located in the bottom area of the configuration dialog of the MOST IG.

Setup of the parameter tree

Parameter

This column displays the parameters of the message that is currently active in the send list as a parameter tree. As is usual in Windows, the sub-branches can be shown by clicking on the '+' sign and hidden again by clicking on the '-' sign.

If a configured messaged is not described in an XML function catalog, a parameter tree is not shown for this message.

Special parameters

  • Remaining Bytes

If the Hex dump of a message cannot be fully disassembled with the help of the XLM Catalog specified in the configuration, then the superfluous data bytes are shown as Remaining Bytes in the message’s parameter tree. For example, this is the case if the Message-ID (FBlock, Function, OpType) does not appear in the catalog. In this special case all data bytes are shown as Remaining Bytes.

Limitations regarding parameter types

MOST IG supports most parameter types that might be used in the function catalogs. However, the following limitations apply:

  • For strings, only coding types ASCII and ISO 8859-15 are available (and No string!, see next item). However, entries are then not recoded, so the strings that are transmitted always (except for a few special characters that are identical with both coding types) follow the Windows ANSI character set.
  • Strings consist of at least two bytes: the byte for the coding type and the identifier for the end of the string. If these two bytes are also not to be transmitted, coding type No string! must be selected. Any string that is entered will be deleted, nothing will be transmitted for this parameter.

Value

The value of the corresponding parameter can be specified here. If possible values have been predefined in the function catalog, they can be selected from a list here.

The values entered are transferred when you press the <ENTER> key or exit the input field.

Unit

In this column, units for the corresponding parameters are specified (e.g. 'm/s' or '*10^3 Hz'). The cells in this column cannot be modified (exception: character coding in strings); the units of parameters have already been defined in the function catalog.

Check Box AMS

Checked if the message should be send using the application message service (AMS).
AMS allows to send messages with more than 12 data bytes.
If messages are send without using AMS, only the first 12 data bytes of a message will be transmitted.

Check Box Parameter Tree

This check box can be used to show/hide the display of the parameter tree. If the display of the parameter tree has been deactivated (hidden) by editing the hex dump, this check box can be used to reactivate it.

[hex] Button

The hex button can be used to specify whether the parameter values are represented in decimal or hexadecimal format. The corresponding numeric system is then used for the interpretation of values.

Placeholder for Device Addresses

With certain messages (e.g. Notification.Set) a device has to enter its own address. Of course, this address can be entered as a fixed value in each message. In this case, however, the address will no longer be correct if the device address changes. On the other hand, it is also possible to use a placeholder for the address which, is then replaced with the respective device address when the message is sent. This ensures that the message is always sent using the correct device address, even if the address undergoes a change. Placeholders can also be used for any parameter of type Number.

Possible placeholders:

  • NNNN: the node address of the device.
  • GGGG: the group address of the device.
  • PPPP: the node position address of the device.

Notes on the Status Bar

  • While parameter values are being input the Status Bar displays a note on possible input values, e.g. "signed integer, 8 Bit, [-128,127], value range: -5 – 10". That is, the data type of the parameter and the possible input values are shown.
  • The Value range item refers to the allowable value range as specified in the function catalog. Values outside of this range but within the range of the data type may be entered, but they are highlighted in color. This is intended to show that the value lies outside of the limits defined in the function catalog. Nevertheless the value can still be saved and sent out.
  • In the event of faulty entries the last value is retained, or if the box did not have any value, the smallest allowable value according to the function catalog is substituted.
