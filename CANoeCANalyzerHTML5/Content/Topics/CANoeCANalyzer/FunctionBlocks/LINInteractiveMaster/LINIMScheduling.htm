LIN Interactive Master – Scheduling

 

General

The LIN Interactive Master can be inserted in the Simulation Setup and becomes active when CANoe simulates the LIN Interactive Master node of a LIN network. This feature allows the interactive control of the internal LIN scheduler. Buttons are also provided for sending SleepMode and wake-up commands interactively.

The use of the LIN Interactive Master can be combined with a CAPL implementation of the LIN Interactive Master functionality. It is however not recommended to use CAPL functions that influence the behavior of the LIN Interactive Master or the internal LIN scheduler. By default such CAPL functions will be filtered out automatically. If necessary, the filtering of such CAPL functions can be deactivated in the Settings dialog.

Context Information

The top-half of the LIN Interactive Master provides the following information about the context of the LIN Interactive Master:

  • Master node
    The name is taken from the associated database.
    Whether this node is currently being simulated by CANoe is displayed in brackets next to the node’s name e.g. LIN_Master (simulated). Otherwise the text LIN_Master (not simulated) is displayed.
    To simulate a LIN Interactive Master node, the corresponding network node must be inserted in the Simulation Setup and be set to active.
  • Database
    This is the name of the database associated to the bus channel being simulated.
  • Baudrate
    Baudrate (in kbit/sec) used by the LIN Interactive Master simulation.
  • Header length
    Header Length (in bits) used by the LIN Interactive Master simulation.
  • Channel
    Channel mapped to the bus being simulated.
  • LIN Protocol
    LIN protocol version used by the simulated LIN Interactive Master.

If any parameters are not displayed, then the simulation of LIN Interactive Master is probably not configured correctly.

Schedule Tables

The schedule tables defined in the associated database are displayed by the LIN Interactive Master in a tree diagram.

Displayed columns

  • Tabelle/Slot
    Table name or slot name
  • Typ
    • In the case of frames: Frame type
    • In the case of commands: Command name
  • Delay
    Slot Delay Time (in milliseconds)

Available operations

The following operations can be performed from the context menu depending on which tree item is currently selected.

  • Tree Item: Schedule table
    • Send continuously
      The selected table will be sent continuously. Before the measurement start this operation specifies which table will be executed on starting the measurement.
    • Send once now
      The selected table will be executed once on completion of the current schedule table. Once this table has been completed, the internal LIN scheduler returns to the previous table. This operation might be used, for example, to execute a schedule table containing reconfiguration commands. Please note that this operation is only available during a measurement and if the database contains more than one schedule table.
    • Send once on wake-up
      The selected table will be executed once on each wake-up event. After that control resumes at a table configured to be sent continuously. This operation could be especially useful in reconfiguring a schedule table.
      Please note that this operation is only available if the database contains more than one schedule table.
  • Tree Item: Slot
    • Send continuously
      Causes the frame associated with the selected slot to be sent each time the slot is reached by the internal LIN scheduler.
    • Deactivate
      Slot time allocated for the selected slot will remain empty. An associated frame is no longer sent.
      Please note that this operation is available only if the selected schedule table contains more than one slot.
    • Simulate Collision Resolution
      This operation only applies if the selected slot has an associated Event Triggered Frame (ETF). When the selected slot is reached, the internal LIN scheduler enters into a conflict resolution state, i.e. headers for the associated frames are sent in the order listed in the LDF.
      Please note that this operation is only available during a measurement. The selected schedule table and the entire slot must be sent continuously.
  • Tree Item: Associated frame
    The operations below only apply if the selected frame is associated with a Sporadic frame.
    • Send continuously
      Causes the header of the selected frame to be sent each time the entire slot is reached.
    • Deactivate
      This deactivates the selected frame, i.e. the header of the frame is no longer sent.
    • Send once now
      Executes the selected frame once on next time the entire slot is reached. After that control resumes at an associated frame configured to be sent continuously, or the slot-time remains silent.
      Please note that this operation is only available during a measurement.

Status information

When an item is selected in the tree view additional information is displayed on a status bar in the block.

  • Tree Item: Schedule table
    • Table ID
      Serial number of the selected schedule table
    • Total cycle time
    • Activated slots
      Number of slots configured to be sent continuously and a total number of slots
  • Tree Item: Slot
    • Slot ID
      Serial number of the selected slot within its schedule table
    • DLC
      Size of an associated frame (in bytes)
    • Checksum model
      Name of the checksum model used by an associated frame
  • Tree Item: Associated frame
    • Position
      Serial number of the frame in the list of frames associated with the parent slot
    • DLC
      Size of the selected frame (in bytes)
    • Checksum model
      Name of the checksum model used by the selected frame

Wake-up & sleep push

Click on the corresponding button to send sleep or wake-up commands. Buttons are only available if their associated commands are possible. For example, the Sleep button is not available if the bus is currently in the Sleep state.

Note

  • In the following cases the functionality of this block is automatically disabled during the measurement.
  • LIN Interactive Master node of a LIN network is not being simulated
  • Associated database contains no schedule tables
  • Schedule table configured to be sent continuously contains no slots configured to be sent continuously
  • Master mode was deactivated in Network Hardware Configuration dialog
  • Operations on the Schedule are unavailable during Sleep mode.

Start Scheduler & Stop Scheduler

Click on the corresponding button to stop or start the LIN scheduler Note

Note

If the Scheduler is also influenced by other command sources, e.g. a CAPL program, the buttons may reflect the wrong scheduler state.

For example, the Stop Scheduler button might not be available even though the scheduler is running.

In this case, you can simply press the other button to make the desired command available.
