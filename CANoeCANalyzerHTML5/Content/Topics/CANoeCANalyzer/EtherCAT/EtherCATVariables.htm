System Variables

 

Namespaces

CANoe generates corresponding system variables for your EtherCAT® configuration. These variables can be found in the namespace ECAT.

For each configured master and for all slaves which are connected to them, the corresponding sub-namespaces are generated.

Input Values

The input values are used to receive slaves and to further process them in CANoe.

For these values, system variables with the name Task*_In are generated in each slave namespace. These system variables include the individual output values of the slaves as sub-elements. The sub-elements are grouped by the PDOs in which will be transmitted.

Output Values

The output values are used to transmit data from CANoe to the slaves.

In each slave namespace, a system variable with the name Out can be found if this slave is provided with at least one input value. This system variable is a structure that contains all EtherCAT® input values of the slave as sub-elements. The sub-elements are grouped by the PDOs in which will be transmitted.

Data Types

CANoe supports all common EtherCAT® data types:

  • Boolean data, 1 bit
  • Integers with a length of 1 to 64 bit, with and without leading sign
  • Floating-point numbers with a length of 4 or 8 bytes
  • Arrays of the above mentioned data types
  • Strings

Data types which cannot be assigned to one of the above mentioned categories are treated in CANoe as integers without leading sign.

Master Status Variables

SystemState

Value

Description

0 = OK

Conditions for this status:

  • LinkState is Connected and
  • MasterOpState is Op and
  • For all slaves: SlaveError is NoError and SlaveOpState is Op

1 = Error

If at least one of the conditions for the OK status is not met, the SystemState is set to Error.

MasterOpState

Value

Description

0 = Unknown

Conditions for this status:

  • LinkState is Disconnected or
  • MasterError is ConfigError

1 = Init

2 = PreOp

3 = SafeOp

4 = Op

5 = Bootstrap

Current state of the master.

MasterError

Value

Description

0 = NoError

No error

1 = StateError

MasterState is not Op

2 = SlaveMissing

At least one slave: SlaveState is Missing

3 = ConfigError

Bus configuration mismatch

LinkState

Value

Description

0 = Disconnected

The link is not connected.

1 = Connected

The link is connected.
