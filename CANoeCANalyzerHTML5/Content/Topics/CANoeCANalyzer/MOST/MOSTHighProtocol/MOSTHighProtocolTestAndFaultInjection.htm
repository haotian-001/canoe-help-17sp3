Test and Fault Injection of MHP Connections

 

The MOST High DLL (MOST_MHP.DLL) can also be used in CAPL test modules. In this case it has to be added to the test node configuration on the Module page. The DLL is located in the Exec32 folder of the MOST High demo. The MOST High functions described below can then be used in the test module.

variables
{
  long gTxPort;
}

testcase TC_MHP()
{
  byte data[200];
  gTxPort = MH_CreateTxPort("");
  MH_ReqTrans(gTxPort,0x171,
              0x52, //Navigation
              0x01, //InstanceID
              0xE01, //FunctionID
              0xC, //OpType
              data, elcount(data));

  //wait for reply of the receiver
  if (TestWaitForMostPkt(“Navigation.Waypoints.Status(START CONNECTION)”, 1000) > 0)
    TestStepPass("1", "Receiver accepts connection request");
  else
    TestStepFail("1", "Receiver didn’t reply");
}

Note

The TestWaitForMostPkt function can be used independently of the MOST High DLL to wait for MOST High packets as a way of testing MHP connections.

Fault Injection

The MOST High DLL has a set of CAPL functions and callbacks that can manipulate messages before they are transmitted or even suppress transmission altogether. Similarly, received messages can also be manipulated or discarded before they reach the MOST High DLL. The fault injection option has to be explicitly activated before any of the individual functions can be used.

on key 'f'
{
  MH_FI_Active(1);
}

long MH_FI_IndTxPreSend(dword handle)
{
  if (MH_FI_GetHighCmd() == 0xFD)
    return 0; //Prevent sending of READY FOR DATA
  else
    return 1; //Send every other message once only
}

The MOST High demo includes more comprehensive fault injection use cases.
