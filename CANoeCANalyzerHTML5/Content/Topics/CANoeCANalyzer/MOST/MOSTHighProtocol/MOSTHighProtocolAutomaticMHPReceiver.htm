Automatic MHP Receiver

 

Introduction

The automatic MHP receiver serves as a simple data sink (DSI) for incoming connections. It implements the receiver-side MOST High Protocol (MHP). The protocol parameters are set for this in such a way that as many incoming connections as possible are accepted regardless of the requirements specified for the transmission by the sender.

Use cases

  • Stimulation
    For the query of an FBlock property transmitted via MHP, the Get command with the MOST IG is sent via the control channel. The status response is received by means of MHP. For this purpose a receiver (DSI) must send a series of acknowledgments during the transmission. The automatic MHP receiver ensures that this occurs without the need for further configuration or programming effort.
  • Simulation
    During simulation of MHP-capable FBlocks or FBlock shadows, the automatic MHP receiver takes over the control of the MHP protocol without also having to simulate the receiver.
  • Test
    In addition, when control units are tested, often the only thing that matters is that a device under test (DUT) can respond to special requests with an MHP transmission and successfully transmit its data. As long as the data contents are of secondary interest, the automatic receiver can control the MHP connection without having to add an MHP simulation to the test script.

Data analysis

The received data are not saved. However, they are visible in the Trace Window as MOST High Combiner events and can be copied to the clipboard using the Detail viewof the Trace Window.

If an automated analysis of the user data is to be carried out, the MHP Combiner and MHP Observer events are evaluated in one CAPL node in the Measurement Setup. In this case, however, no influence can be exerted on the transmission or the MOST ring. If such influence is desirable, the use of a simulation node or test node with the help of MOST High DLL is recommended as an alternative. The extensive API of this DLL allows influence to be exerted on the transmission as well as active influencing of the MOST ring similar to that of a true control device.

Configuration

The automatic MHP receiver is configured using the Network Hardware Configuration dialog.

In CANoe, MHP receivers can also be programmed using the MOST High DLL. To avoid conflicts, the automatic MHP receiver reacts to an incoming Request Connection by first waiting to determine whether another simulated node accepts the connection. Only if this is not the case does the automatic MHP receiver send a corresponding Start Connection message, thereby accepting the connection, and maintain the connection by sending acknowledgments for the data.
