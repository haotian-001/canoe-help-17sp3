MOST: How To Send Messages and Packages (MDP & MEP)

 

You can send MOST messages and packages as follows:

MOST Interactive Generator (MOST IG)

The MOST Interactive Generator (MOST IG) offers the fastest way to send MOST messages or packets. The configuration user interface for entry of messages can be opened via the Simulation ribbon tab|MOST IG. To insert packets in the send list, use the drop down menu of the button [New].

Notes

  • If the Simulation ribbon tab does not offer MOST IG, this means that the configuration does not contain such a Generator. To insert a new block, the measurement must be stopped and the MOST IG must be inserted into the Simulation Setup as a block.
  • The Interactive Generator does not support sending of Ethernet packets (MEP).

CAPL

Alternatively, CAPL nodes can be used to send messages or packets, which is especially useful if messages have to be sent in reaction to events on the bus, e.g. as response to other messages.

As preparation a network node has to be inserted in the Simulation Setup and a CAPL file has to be assigned to this node.

Example: Sending a message on keyboard hit

(It would also be possible to achieve this exact behavior through suitable configuration with the MOST IG.):

on key ’m’
{
   // channel, destination address, message specification, instance id
   mostAmsOutput(1, 0x100,”NetBlock.DeviceInfo.Get(0x00)”, 0x00);
}

Example: Sending a packet (MDP) on keyboard hit

on key ’p’
{
   BYTE pktdata[1524] = { 1, 2, 3, 4, 5, 6, 7 }
   // channel, destination address, length, packet data
   OutputMostPkt(1, 0x101, 7, pktdata)
}

Notes

Typically, MOST High Protocol is used to communicate over the packet channel of MOST. CANoe comes with a modeling library (DLL file) implementing this transport protocol and therewith enhancing the CAPL capabilities. You can find more information in the Documentation MOST High Protocol MOST High Protocol Node Layer API.
Additionally a number of MOST Sample Configurations can be found here: Open directory

Example: Sending an Ethernet packet (MEP) on keyboard hit - only with MOST150 available

on key ’e’
{
   BYTE pktdata[1506] = { 1, 2, 3, 4, 5, 6, 7 }
   // channel, destination MAC address, length, packet data
   outputEthPkt(1, 0x123456, 7, pktdata);
}
