Simulation Concept

 

General

For the simulation of one MOST device in CANoe typically one MOST channel is used. The main window of the Simulation Setup exactly shows one MOST device with its function blocks (CAPL nodes). The function blocks communicate via the Application Socket with other function blocks. The Application Socket and further services are represented by the "device" symbol in the main window of the Simulation Setup. Depending on the configuration, CANoe provides further function blocks (NetBlock, MOST NetworkMaster, MOST PowerMaster).

The system view of the Simulation Setup gives an overview of all simulated MOST devices and other networks (CAN, LIN) respectively.

Notes

  • Unlike real bus networks (cp. CAN), a node in the main window of the MOST Simulation Setup represents only a part (e.g. a function block) of a device (ECU). When deactivating a node only this part becomes inactive. If a device shall completely be removed from the simulation, it can be deactivated in the context menu of the device symbol in the main window.
  • The Application Socket in CANoe only processes AMS messages, i.e. CAPL message variables of type mostAMSMessage.
