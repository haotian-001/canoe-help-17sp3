NetworkMaster

 

The MOST NetworkMaster is not part of the Application Socket. Nevertheless, CANoe provides for an implementation of a MOST NetworkMaster to permit fast setup of a MOST network.

The following properties apply to CANoe implementation of the MOST NetworkMaster:

  • The MOST NetworkMaster function block can be activated on every device. This is possible with adding the NWM directly in the Simulation Setup or via the configuration page of the Application Socket in the Network Hardware Configuration dialog (Hardware ribbon tab|Network Hardware).
  • Late registration or unregistration of a function block in the network results only in a complete system scan by the NetworkMaster if an invalid registration was made. Otherwise the function block is integrated into or removed from the Central Registry
  • The MOST NetworkMaster function block's functions SaveConfiguration and Boundary are not available.
  • The number of attempts of the NWM to resolve a duplicated functional address can be defined in the CAN.INI:

[MOSTApplicationSocket]
; Default: 20 attempts
NMAttemptsChangeInstID=20

MOST150

  • The Boundary function no longer exists.
  • The SystemAvail (0xA10) function is new for MOST150; accordingly, no error message FktID not available is returned and the user can simulate the functionality in CAPL.
  • If MOST150 is configured as a speed grade for the channel on which the NWM was placed, the CANoe implementation automatically behaves according to this MOST speed grade.

Configuration dialog

With [Use external implementation of MOST NetworkMaster] or [...] you can open a file selection dialog where you select the modeling library (DLL file) that contains your own MOST NetworkMaster implementation.

Note

A MOST NetworkMaster implementation has to base on the node layer interface. Vector may help you create such a MOST NetworkMaster. Please contact Vector Support.

The [File] button opens a file selection dialog. Here you select the CAPL program that e.g. describes the fault injection functionality. The relevant file is indicated in the text box beneath this.

The [Edit] button opens the CAPL Browser, with which you can edit the selected program.

All CAPL programs must be compiled before the measurement start. This is done by pressing [Compile] or by choosing the menu path Configuration|Compile all nodes.
