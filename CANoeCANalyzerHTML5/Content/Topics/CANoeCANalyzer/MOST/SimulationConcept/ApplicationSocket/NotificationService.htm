Notification Service

 

The communication partners of a MOST function block can use the MOST notification service to stay informed about changes in the function block states (properties).

The integrated notification services is of great advantage in simulating a function block using CANoe.

The service in the CAPL program for the function block must be activated using the MostAsNtfEnable() or MostAsNtfEnableEx() commands. MostAsNtfFunctionEnable() or MostAsNtfFunctionEnableEx() can be used to define which function block functions are to be bound to the notification service.

CANoe maintains the notification matrix for these functions. CANoe processes and replies to all commands issued via Notification.Set/Get and NotificationCheck.Set/Get messages.

Example

This example demonstrates the use of the notification service

The DeckStatus function (function ID = 0x200) for the notification service is activated in the CAPL program for the AudioDiskPlayer function block.

on preStart
{
// Activate notification service for this FBlock
MostAsNtfEnable();
// Enable DeckStatus function and set the callback for status reporting
MostAsNtfFunctionEnable(0x200, "SendStatus_DeckStatus");
}

A device with the account address 0x0102 registers as a notification client for the AudioDiskPlayer.DeckStatus by sending the command AudioDiskPlayer.Notification.Set(SetFunction, 0x0102, 0x200) to the AudioDiskPlayer CAPL function block.

CANoe receives the command, enters the client's node address into the notification matrix, and sends a status message containing the current DeckStatus value to the client.

The CANoe notification service uses the user-defined SendStatus_DeckStatus() function to compose and send the status message.

long SendStatus_DeckStatus(long destAdr)
{
  // build status message
  mostAMSMessage ADP_DeckStatus_Status msg;
  // get current property value from variable SysVarDeckStatus
  msg.DeckStatus = @SysVarDeckStatus;
  // send status message
  // destAdr == 0xFFFF: send to all notification clients
  // destAdr != 0xFFFF: send to destAdr
  MostAsNtfOutput(destAdr, msg);
  // message has been sent successfully
  return 0;
}

The user-defined SendStatus_DeckStatus() function can also be used to send changes in the DeckStatus to all notification clients via status messages.

on sysVar SysVarDeckStatus
{
  // property (represented by a variable) has changed
  // send status message to all notification clients
  // wildcard address 0xFFFF is used to address all notification clients
  SendStatus_DeckStatus(0xFFFF);
}

Further examples of how to use the CANoe notification service can be found in the supplied MOST option demos.

Further Properties of the CANoe Notification Service:

  • The MostAsNtfOutput() command can be used to send a message to all notification clients.
  • The MostAsNtfFunctionSet(), MostAsNtfFunctionClear() and MostAsNtfFunctionClearAll() CAPL functions can be used to write directly to the notification matrix.
  • The notification matrix can be read with the MostAsNtfFunctionListGetSize(), MostAsNtfFunctionListGetFunction(), MostAsNtfDeviceIDListGetSize(), MostAsNtfDeviceIDListGetDeviceID() and MostAsNtfFunctionCheck() functions.
  • The notification matrix can be of unlimited size.
  • Double entries are not possible in the notification matrix.
  • The notification matrix is deleted if the network status changes to ConfigNotOk.
