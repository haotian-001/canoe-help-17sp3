Timestamps

 

The resolution of the time stamp in CANoe depends on which network interface is being used. For Vector hardware, it is typically 1 µs. The resolution displayed in the Trace Window can be set in the Options dialog. In CAPL, the selector time and most CAPL functions deliver a resolution of 10 µs.

All events received prior to time synchronization being able to calculate a valid offset between the CAN bus and MOST ring are suppressed. It is for this reason that the first MOST frames displayed in a Trace Window may show a time stamp which predates the start of measurement by a few seconds.

Here you can find detailed information about synchronized time stamps.

Timestamps in COM

For all MOST events a 64 bit time stamp is delivered as two 32 bit Signed Integers (timeHigh, time). The value determines the time in 10 µs. A time stamp with the value 100000 means, for example, that the according event exactly occurred one second after measurement start.

Although the signatures are Signed Integer, both parts of the time stamp have to be interpreted as Unsigned values. In client languages that don't have Unsigned Integer types (e.g. Visual Basic) time stamps > 5.96 hours need a special treatment.

One possibility to process large time stamps is to save them as a Double value (or similar) internally in the client. By doing so, you have to take care that there may occur rounding problems especially with Double values and large time stamps.

Example

The following example shows a Visual Basic 6 function that processes both parts of the time stamp to a VB decimal value.
This data type can represent the complete value range of a time stamp without rounding errors
All basic mathematical functions can be used with this data type.

Private Function compose_timestamp(ByVal timeHi As Long, ByVal timeLo As Long) As Variant
'some local vars
Dim msbLong As Long
msbLong = &H80000000
Dim msbDec As Variant
msbDec = CDec(&H40000000) * 2
'high part
compose_timestamp = CDec(tmpHi And Not msbLong)
If timeHi < 0 Then
compose_timestamp = compose_timestamp + msbDec
End If
compose_timestamp = compose_timestamp * &H10000 * &H10000

    'lo part
compose_timestamp = compose_timestamp + (timeLo And Not msbLong)
If timeLo < 0 Then
compose_timestamp = compose_timestamp + msbDec
End If

    'unit adaption:
'10 µs to s
compose_timestamp = compose_timestamp / 100000
End Function

Note

When you perform the following replacement in the example (above)

  • Long replaced by Integer
  • Variant replaced by Object

you'll get a Visual Basic 7 function.
