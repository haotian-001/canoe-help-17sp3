OptoLyzer MOCCA compact 50e: Installation

 

Install the OptoLyzer MOCCA compact 50e driver according to the MOCCA user manual.

Requirements

At least you need hardware revision 3.1 and the following components of the package OptoLyz-er_MOCCA_Bundle_V01_05_00.exe:

  • USB driver
  • MOST_Tool_Foundation
  • OptoLyzer components
  • MOCCA adapter
  • Firmware

Notes

  • Note that a license for the Vector OptoLyzer Integration Package (OIP) is required, either on Vector hardware or a USB dongle, or as a license key!
  • The former variant MOCCA compact 50e (hardware revision 2.3) can still be used as interface. This support might be removed in future versions.

Synchronizing the time stamps of the node channel

The time stamps of node channel events are synchronized with time stamps of other connected interfaces by using a by a software algorithm.

Sync line for spy channel time stamps

Important Note

Works with hardware revision MOCCA compact 2.3 only.

If you want to synchronize the time stamps of the spy channel with the time stamps of any Vector network interfaces that are connected in parallel, you will need a sync line with matching circuitry connecting the Vector hardware and the LIN1 input (connector: Low speed CAN) of the MOCCA compact 2.3.

Important Note

The Sync input of the MOCCA interfaces is not compatible with Vector synchronization.
Never connect the sync line with this input directly without matching circuitry!

Matching Circuitry:

  • LIN1 (MOCCA 2.3 connector: Low speed CAN) is used to detect the edge of the synchronization master.
    The MOCCA interface is always the synchronization slave.

  • The Sync connector of the MOCCA interface provides 12 V for the matching circuitry.

    Important Note

    The Sync input of the MOCCA interfaces is not compatible with Vector synchronization.
    Never connect the sync line with this input directly without matching circuitry!
  • The Sync connector of the Vector network interface is used for the synchronization signal.

Using Computer Timestamps

To bypass time stamp synchronization completely (e.g. if the matching circuitry for spy channel events is not available) you have to edit the CAN.ini file as shown below. The events get their time stamp not from the interface clock but when they arrive in the CANoe event queue on the computer. Therefore the accuracy is reduced and depends on the current computer load. For precise measurements of time differences you should not use this setting!

; Channel MOST 1
[Mocca_50_1]
UseReferenceTime=1

; Channel MOST 2
[Mocca_50_2]
UseReferenceTime=1
