Access to Analog Audio Channels (Line In/ Headphone Out)

 

Analog audio data can be fed into and out of CANoe. With MOST, the term "routing" is commonly used to describe the transfer of audio data into and out of a MOST network. External audio devices can be connected via the MOST Line In/Out interface (e.g. VN2610 or VN2640). Stereo audio data are transmitted into and out of the MOST network at 4 bytes each, whereby each audio channel uses 2 bytes:

Left audio channel

Right audio channel

MSB

LSB

MSB

LSB

The routing can be done interactively via the MOST Audio Routing Window. The procedure is explained for each use case.

Alternatively the routing can be done in CAPL.

Therefore the functions MostSetSyncAudio(), MostSetSyncMute() and MostSetSyncVolume() are available.

MOST25 with VN2610

In CAPL, you have to specify individual synchronous channels for routing in the MostSetSyncAudio() function. The following applies in this case:

Example

Routing audio data from synchronous channels 0x04..0x07 to Line Out.

 long syncChannels[4];

 syncChannels[0] = 0x04; // MSB left audio channel
 syncChannels[1] = 0x05; // LSB left audio channel
 syncChannels[2] = 0x06; // MSB right audio channel
 syncChannels[3] = 0x07; // LSB right audio channel

 // activate routing of Line Out
 MostSetSyncAudio(mostGetChannel(), syncChannels, 1, 1);

The mapping of the individual bytes (MSB left, LSB left, ...) to the indices in the array variable (0, 1, ...) is valid for both Line Out as well as Line In.

Important Note

For Line In it is not strictly necessary to allocate all of the channels on which analog audio data is to be routed in advance. However, we strongly recommend to do so, since otherwise another device in the network may route its own data on the same channels.

MOST150 with VN2640

In CAPL, you have to specify label and width of label (number of synchronous channels) for routing in the MostSetSyncAudio() function:

Example

Routing audio data from synchronous label 0x016B (4 channels) to Line Out.

// activate routing of Line Out
MostSetSyncAudio(mostGetChannel(), 0x016B, 4, 1, 1);

The mapping of the individual bytes (MSB left, LSB left, ...) to the channels of the label is done automatically.

Use cases:

Line / Headphone Out: Connecting headphones or speakers

  • Routing of stereo signals (2 x 16 Bit Audio)

Execute the following steps in the MOST Audio Routing Window:

  1. The source has 4 allocated channels. These also appear in the allocation table and are marked with a connection label. Activate Line Out for the corresponding connection label.
  2. Deactivate the Mute check box and adjust the volume if necessary.

Corresponding example in CAPL for MOST25 with VN2610

Example

variables
{
  long syncChannels[4];
}

on key 'r'
{
  // the source has allocated channels 4,5,6 and 7
  syncChannels[0] = 0x04;
  syncChannels[1] = 0x05;
  syncChannels[2] = 0x06;
  syncChannels[3] = 0x07;

  // activate routing of Line Out
  MostSetSyncAudio(mostGetChannel(), syncChannels, 1, 1);
  // set volume and demute
  MostSetSyncVolume(mostGetChannel(), 1, 200);
  MostSetSyncMute(mostGetChannel(), 1, 0);
}

Corresponding example in CAPL for MOST150 with VN2640

Example

on key 'r'
{
  // the source has allocated lable 0x016B comprising 4 channels
  // activate routing of Line Out
  MostSetSyncAudio(mostGetChannel(), 0x016B, 4, 1, 1);
  // set volume and demute
  MostSetSyncVolume(mostGetChannel(), 1, 200);
  MostSetSyncMute(mostGetChannel(), 1, 0);
}
  • Routing of mono signals (1 x 16 Bit Audio)

Execute the following steps in the MOST Audio Routing Window:

  1. The source has 2 allocated channels. These also appear in the allocation table and are marked with a connection label. Activate Line Out for the corresponding connection label.
  2. Deactivate the Mute check box and adjust the volume if necessary.

Corresponding example in CAPL for MOST25 with VN2610

Note

The MOST Audio Routing Window configures the network interface for mono signals the way, that the signal is hearable on both headphones and speakers respectively. In the following CAPL example the signal is only routed to the left headphone respectively speaker output.

Example

variables
{
  long syncChannels[4];
}

on key 'r'
{
  // the source has allocated channels 4 and 5
  syncChannels[0] = 0x04;
  syncChannels[1] = 0x05;
  syncChannels[2] = 0xF8; // unused channel
  syncChannels[3] = 0xF8;

  // activate routing of Line Out
  MostSetSyncAudio(mostGetChannel(), syncChannels, 1, 1);
  // set volume and demute
  MostSetSyncVolume(mostGetChannel(), 1, 200);
  MostSetSyncMute(mostGetChannel(), 1, 0);
}

Note

Unused channels are marked with the special value '0xF8'.

Note about MOST150 with VN2640

Currently only labels comprising 4 channels can be routed.

Line In: Connecting an external audio source (e.g. CD player)

  • Routing of stereo signals (2 x 16 Bit Audio)

Execute the following steps in the MOST Audio Routing Window:

MOST25 with VN2610

  1. Channel Management: Allocate 4 channels. The channels with their corresponding labels will appear in the Allocation Table.
  2. Activate Line In for the corresponding connection label
  3. Deactivate the Mute check box and adjust the volume if necessary

Corresponding example in CAPL

Example

variables
{
  long syncChannels[4];
}

on key 'a'
{
  MostSyncAlloc(4); // allocate 4 channels
}

OnMostSyncAllocResult(long allocResult, long numChannels, long channels[])
{
  long i;

  if(allocResult == 0x01) // ALLOC_GRANT
  {
    for(i=0; i<4 && i<numChannels; ++i)
      syncChannels[i] = channels[i];

    // activate routing of Line In
    MostSetSyncAudio(mostGetChannel(), syncChannels, 0, 1);
    // set volume and demute
    MostSetSyncVolume(mostGetChannel(), 0, 200);
    MostSetSyncMute(mostGetChannel(), 0, 0);
  }
}

MOST150 with VN2640

  1. Activate Line In Routing (a label comprising 4 channels is automatically allocated and the data is routed).
  2. Deactivate the Mute check box and adjust the volume if necessary.

Corresponding example in CAPL

Example

on key 'a'
{
  // activate routing of Line In
  MostSetSyncAudio(mostGetChannel(), 0, 4, 0, 1);

  // set volume and demute
  MostSetSyncVolume(mostGetChannel(), 0, 200);
  MostSetSyncMute(mostGetChannel(), 0, 0);
}
  • Routing of mono signals (1 x 16 Bit Audio)

Execute the following steps in the MOST Audio Routing Window:

MOST25 with VN2610

  1. Channel Management: Allocate 2 channels. The channels with their corresponding labels will appear in the Allocation Table.
  2. Activate Line In for the corresponding connection label.
  3. Deactivate the Mute check box and adjust the volume if necessary.

Corresponding example in CAPL

Example

variables
{
  long syncChannels[4];
}

on key 'a'
{
  MostSyncAlloc(2); // allocate 2 channels
}

OnMostSyncAllocResult(long allocResult, long numChannels, long channels[])
{
  long i;

  if(allocResult == 0x01) // ALLOC_GRANT
  {
    for(i=0; i<2 && i<numChannels; ++i)
      syncChannels[i] = channels[i];
    syncChannels[2] = 0xF8; // unused channel
    syncChannels[3] = 0xF8;

    // activate routing of Line In
    MostSetSyncAudio(mostGetChannel(), syncChannels, 0, 1);
    // set volume and demute
    MostSetSyncVolume(mostGetChannel(), 0, 200);
    MostSetSyncMute(mostGetChannel(), 0, 0);
  }
}

Note

Unused channels are marked with the special value '0xF8'.
