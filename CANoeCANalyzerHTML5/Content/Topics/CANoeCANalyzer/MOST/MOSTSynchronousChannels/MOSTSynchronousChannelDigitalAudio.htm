Access to Digital Audio Channels (S/PDIF In and Out)

 

With CANoe and the VN2610 or VN2640 digital audio data can be fed in and out. With MOST, the term "routing" is commonly used to describe the transfer of audio data into and out of a MOST network. External S/PDIF capable devices can be connected to the VN2610's or VN2640's S/PDIF interface. S/PDIF data are transmitted into and out of the MOST network via the VN2610 or VN2640 at 8 bytes each, whereby 4 bytes are used per audio channel:

Left audio channel

Right audio channel

MSB

LSB

AUX

VUCP

MSB

LSB

AUX

VUCP

CAPL offers the following functions for routing digital audio data: MostSetSyncSpdif(), MostSetSyncSpdifLock() and MostSetSyncSpdifMode().

MOST25 with VN2610

In CAPL, you have to specify individual synchronous channels for routing in the MostSetSyncSpdif() function. The following applies in this case:

Example

Routing S/PDIF data from synchronous channels 0x04..0x0B to S/PDIF Out.

 long syncChannels[8];

 syncChannels[0] = 0x04; // MSB left audio channel
 syncChannels[1] = 0x05; // LSB left audio channel
 syncChannels[2] = 0x06; // AUX left audio channel
 syncChannels[3] = 0x07; // VUCP left audio channel
 syncChannels[4] = 0x08; // MSB right audio channel
 syncChannels[5] = 0x09; // LSB right audio channel
 syncChannels[6] = 0x0A; // AUX right audio channel
 syncChannels[7] = 0x0B; // VUCP right audio channel

 // activate routing of SPDIF In
 MostSetSyncSpdif(mostGetChannel(), syncChannels, 1, 1);

The mapping of the individual bytes (MSB left, LSB left, ...) to the indices in the array variable (0, 1, ...) is valid for both S/PDIF Out and S/PDIF In.

Note

For S/PDIF In it is not strictly necessary to allocate all of the channels on which S/PDIF data is to be routed in advance. However, we strongly recommend to do so, since otherwise another device in the network may route its own data on the same channels.

MOST150 with VN2640

In CAPL, you have to specify label and width of label (number of synchronous channels) for routing in the MostSetSyncSpdif() function:

Example

Routing S/PDIF data from synchronous Label 0x016B (4 channels) to S/PDIF Out

// activate routing of SPDIF Out
MostSetSyncSpdif( mostGetChannel() , 0x016B, 4, 1, 1);

The mapping of the individual bytes (MSB left, LSB left, ...) to the channels of the label is done automatically.

Note

Currently only labels comprising 4 channels (pure audio data) can be routed.

Use cases:

S/PDIF Out:

Connecting an external S/PDIF sink and routing the data from a S/PDIF source

 

Because the S/PDIF sink can receive and process all 8 data bytes of the S/PDIF signal, all S/PDIF data is routed. The VN2610/VN2640 has to be configured as S/PDIF Master.

Example code in CAPL for MOST25 with VN2610

variables
{
  long syncChannels[8];
}

on key 'r'
{
  // the source has allocated channels 0x00..0x07
  syncChannels[0] = 0x00; // MSB left audio channel
  syncChannels[1] = 0x01; // LSB left audio channel
  syncChannels[2] = 0x02; // AUX left audio channel
  syncChannels[3] = 0x03; // VUCP left audio channel
  syncChannels[4] = 0x04; // MSB right audio channel
  syncChannels[5] = 0x05; // LSB right audio channel
  syncChannels[6] = 0x06; // AUX right audio channel
  syncChannels[7] = 0x07; // VUCP right audio channel

  // configure network interface as S/PDIF Master
  MostSetSyncSpdifMode(mostGetChannel(), 1);
  // activate routing of S/PDIF Out
  MostSetSyncSpdif(mostGetChannel(), syncChannels, 1, 1);
}

Example code in CAPL for MOST150 with VN2640

// the source has allocated label 0x016B comprising 4 channels
// configure network interface as S/PDIF Master
MostSetSyncSpdifMode(mostGetChannel(), 1);
// activate routing of S/PDIF Out
MostSetSyncSpdif(mostGetChannel(), 0x016B, 4, 1, 1);

Important Note

Currently the VN2640 can only route labels comprising 4 channels (pure audio data). How-ever, the S/PDIF sink expects an S/PDIF signal with 8 data bytes. In this case, the non-transmitted data bytes (AUX and VUCP) are assigned a value of "zero" at the VN2610's/VN2640âs S/PDIF output.

Connecting an external S/PDIF sink and routing the audio data from a non-S/PDIF source (e.g. CD Player)

 

The CD player feeds in "only" audio data. However, the S/PDIF sink expects an S/PDIF signal with 8 data bytes. In this case, the non-transmitted data bytes (AUX and VUCP) are assigned a value of "zero" at the VN2610's/VN2640's S/PDIF output. The VN2610/VN2640 has to be configured as S/PDIF Master.

Example code in CAPL for MOST25 with VN2610

variables
{
  long syncChannels[8];
}

on key 'r'
{
  // the source has allocated channels 0x00..0x03
  syncChannels[0] = 0x00; // MSB left audio channel
  syncChannels[1] = 0x01; // LSB left audio channel
  syncChannels[2] = 0xF8; // unused channel
  syncChannels[3] = 0xF8;
  syncChannels[4] = 0x02; // MSB right audio channel
  syncChannels[5] = 0x03; // LSB right audio channel
  syncChannels[6] = 0xF8;
  syncChannels[7] = 0xF8;

  // configure network interface as S/PDIF Master
  MostSetSyncSpdifMode(mostGetChannel(), 1);
  // activate routing of S/PDIF Out
  MostSetSyncSpdif(mostGetChannel(), syncChannels, 1, 1);
}

Important Note

Unused channels are marked with the special value '0xF8'.

Example code in CAPL for MOST150 with VN2640

// the source has allocated label 0x016B comprising 4 channels
// configure network interface as S/PDIF Master
MostSetSyncSpdifMode(mostGetChannel(), 1);
// activate routing of S/PDIF Out
MostSetSyncSpdif(mostGetChannel(), 0x016B, 4, 1, 1);

Special case: Connecting a headphone or speaker to Line Out and routing audio data from a S/PDIF source

 

In this case, only the audio data from the S/PDIF source is routed to Line Out.

Example code in CAPL for MOST25 with VN2610

The S/PDIF source has allocated channels 0x04..0x0B. The audio data (MSB and LSB) is routed on channels 0x04 and 0x05 (left audio channel) and 0x08 and 0x09 (right audio channel).

variables
{
  long syncChannels[8];
}

on key 'r'
{
  // left audio data is routed onto channels 0x04 and 0x05
  // right audio data is routed onto channels 0x08 and 0x09
  syncChannels[0] = 0x04; // MSB left audio channel
  syncChannels[1] = 0x05; // LSB left audio channel
  syncChannels[2] = 0x08; // MSB right audio channel
  syncChannels[3] = 0x09; // LSB right audio channel

  // activate routing of Line Out
  MostSetSyncAudio(mostGetChannel(), syncChannels, 1, 1);
  // set volume and demute
  MostSetSyncVolume(mostGetChannel(), 1, 200);
  MostSetSyncMute(mostGetChannel(), 1, 0);
}

Example code in CAPL for MOST150 with VN2640

on key 'r'
{
  // the source has allocated lable 0x016B comprising 4 channels
  // activate routing of Line Out
  MostSetSyncAudio(mostGetChannel(), 0x016B, 4, 1, 1);
  // set volume and demute
  MostSetSyncVolume(mostGetChannel(), 1, 200);
  MostSetSyncMute(mostGetChannel(), 1, 0);
}

S/PDIF In:

Connecting an external S/PDIF source and routing the data to a S/PDIF sink

MOST25 with VN2610

Because the S/PDIF sink can receive and process all 8 data bytes of the S/PDIF signal, all S/PDIF data is routed. The VN2610 has to be configured as S/PDIF Slave.

Example code in CAPL

variables
{
  long syncChannels[8];
}

on key 'a'
{
  MostSyncAlloc(8); // allocate 8 channels
}

OnMostSyncAllocResult(long allocResult, long numChannels, long channels[])
{
  long i;

  if( (allocResult == 0x01) && (numChannels == 8) ) // ALLOC_GRANT and 8 channels allocated
  {
    for(i=0; i<numChannels; ++i)
      syncChannels[i] = channels[i];

    // configure network interface as S/PDIF Slave
    MostSetSyncSpdifMode(mostGetChannel(), 0);
    /*
    // In case the network interface is configured as TimingMaster there are two possibilities to configure the VN2610:
    // 1. The internal oscillator is used as MOST clock source
    MostSetClockSource(mostGetChannel(), 0);
    MostSetSyncSpdifLock(mostGetChannel(), 1);
    // 2. The MOST clock can be synchronized to the S/PDIF input signal
    MostSetClockSource(mostGetChannel(), 1);
    */
    // activate routing of S/PDIF In
    MostSetSyncSpdif(mostGetChannel(), syncChannels, 0, 1);
  }
}

MOST150 with VN2640

The VN2640 currently can only route the audio data of an S/PDIF signal to the MOST ring. The frequency of the MOST ring currently cannot be synchronized automatically to the S/PDIF frequency of the S/PDIF source. Therefore the VN2640 must be configured as S/PDIF slave. If the frequency of the S/PDIF source and the MOST ring differ, the VN2640 performs a resampling of the audio data.

Example code in CAPL

on key 'r'
{
  // configure network interface as S/PDIF Slave
  MostSetSyncSpdifMode(mostGetChannel(), 0);
  // activate routing of S/PDIF In
  MostSetSyncSpdif(mostGetChannel(), 0, 4, 0, 1);
}

Connecting an external S/PDIF source and routing the audio data to a non-S/PDIF sink (e.g. a stereo amplifier)

In this case, the sink (amplifier) can receive and process "only" the audio data. For this reason, only the S/PDIF audio data (each MSB and LSB) is routed. The VN2610/VN2640 has to be configured as S/PDIF Slave.

Example code in CAPL for MOST25 with VN2610

variables

{

  long syncChannels[8];

}

on key 'a'

{

  MostSyncAlloc(4); // allocate 4 channels

}

OnMostSyncAllocResult(long allocResult, long numChannels, long channels[])
{
  if( (allocResult == 0x01) && (numChannels == 4) ) // ALLOC_GRANT and 4 channels allocated
  {
    // skip unused S/PDIF data (AUX and VUCP)
    syncChannels[0] = channels[0]; // MSB left audio channel
    syncChannels[1] = channels[1]; // LSB left audio channel
    syncChannels[2] = 0xF8;        // unused channel
    syncChannels[3] = 0xF8;
    syncChannels[4] = channels[2]; // MSB right audio channel
    syncChannels[5] = channels[3]; // LSB right audio channel
    syncChannels[6] = 0xF8;
    syncChannels[7] = 0xF8;

    // configure network interface as S/PDIF Slave
    MostSetSyncSpdifMode(mostGetChannel(), 0);
    /*
    // In case the network interface is configured as TimingMaster there are two possibilities to configure the VN2610:
    // 1. The internal oscillator is used as MOST clock source
    MostSetClockSource(mostGetChannel(), 0);
    MostSetSyncSpdifLock(mostGetChannel(), 1);
    // 2. The MOST clock can be synchronized to the S/PDIF input signal
    MostSetClockSource(mostGetChannel(), 1);
    */
    // activate routing of S/PDIF In
    MostSetSyncSpdif(mostGetChannel(), syncChannels, 0, 1);
  }
}

Important Note

Unused channels are marked with the special value '0xF8'.

Example code in CAPL for MOST150 with VN2640

on key 'r'
{
  // configure network interface as S/PDIF Slave
  MostSetSyncSpdifMode(mostGetChannel(), 0);
  // activate routing of S/PDIF In
  MostSetSyncSpdif(mostGetChannel(), 0, 4, 0, 1);
}
