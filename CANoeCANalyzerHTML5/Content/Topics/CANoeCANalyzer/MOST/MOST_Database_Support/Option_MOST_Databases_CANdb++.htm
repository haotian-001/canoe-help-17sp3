CANdb++

 

MOST CANdb++ databases should only be used to describe function blocks (network nodes) in the system. Environment variables can also be defined in this context, such as for simulations and panels.

Describing MOST messages in CANdb++ databases is not recommended for this version of the MOST Option and is only supported to ensure compatibility with older versions. Furthermore, this support is not complete and cannot be assumed for newer features. If it appears to be necessary to use a CANdb++ database when creating a new configuration, we recommend only using databases generated automatically from the function catalog. You should always make changes to message descriptions in the original function catalog and then convert them again.

In older configurations, CANdb++ support may be needed in the following areas:

Creation of a CANdb++ database to describe MOST

CANdb++ databases are usually generated from XML function catalogs using the XML2DBC converter included with the software. Please note that the CANdb++ format cannot describe all of the parameter structures used in MOST systems. In particular, it is not possible to describe parameters with length/position or complex structures such as arrays and sequences.

Message ID

A message ID can be created for control messages that conform to the function catalog as well as for AMS messages.

ID = (FBlockID << 16) + (FunctionID << 4) + OperationType

The message ID can be used to search for MOST messages in the CANdb database and identify them in applications (e.g. CAPL program, Data Window).

The message ID is no longer needed in the current version. This property is only supported in order to ensure compatibility with older MOST Option versions.
