Function Catalogs in XML

 

The function blocks of MOST devices are described in XML Function Catalogs. The Option .MOST imports these function blocks from XML files.

This function catalog is usually created by the OEM and provided to all parties involved in the project. If you do not yet have a function catalog in XML format, contact your OEM.

For members of the MOST Cooperation, a function catalog editor is available for download on the MOST Cooperation intranet. It can be used to create new function catalogs. In addition, example catalogs of standard function blocks are available on the intranet.

Example catalogs can be found in the MOST subdirectories for sample configurations, e.g. Open directory.

Using XML Function Catalogs in CANoe/DENoe

XML function catalogs are assigned to MOST devices in the System View of the Simulation Setup. To do this, right-click on the Databases option in the System View. Choose Add... from the context menu. The Open dialog is displayed. You can select the XML files in this dialog.

Setting disassembly mode

With CANdb and XML there are two sources for the description of MOST messages. The key DisassemblyMode ([MOST] section) in the file CAN.INI (Exec32 directory of your CANoe installation) controls which database should be used for message disassembly in the Trace Window. The following modes are available:

0

CANdb only

1

XML Function Catalogs only

2

CANdb, XML Function Catalogs

(if CANdb does not describe the message, the XML Function Catalog will be used)

3

XML Function Catalogs, CANdb

(if the message is not described in the XML Function Catalog, CANdb will be used)

Validation

In the default setting CANoe does not validate the XML file relative to a DTD. If a validation should be made the XML file must contain a reference to the name of the DTD in the <!DOCTYPE> tag at the beginning. In CAN.INI, the FCatImportValidate entry under [MOST] must be set to 1.

[MOST]
FCatImportValidate=1

Note

Using XML catalogs without any DTD or without the official DTD of the MOST Cooperation might lead to unpredictable import results!

Import Errors and Warnings

The XML import of CANoe writes logical and semantical errors and other inconsistencies in the XML Function Catalog into an ASCII file if the entry FCatImportLog=1 is set in the [MOST] section of the profile CAN.INI. This default entry can be deactivated by deleting in the CAN.INI.

The name of the ASCII file is determined by the name of the XML file concerned (same directory, suffix .XML replaced by .log).

Using more than one XML Function Catalog

Option.MOST can use several XML Function Catalogs simultaneously.

If a MOST message is defined in more than one XML Function Catalog, the description with the highest precision will be used. The precision of a message description is internally defined as follows (4 = highest precision):

0

FBlock of message is not described

1

only FBlock of message is described

2

message is described up to ‘Function’

3

message is described up to ‘Operation’

4

at least one parameter of the message is described

If a message is described in XML Function Catalogs and in CANdb databases, CANoe Option MOST uses the description according to the disassembly mode:

  • Disassembly mode 2: If a message is described in a CANdb database, this description will be used.
  • Disassembly mode 3: If the message description in the XML Function Catalog is of precision 3 or 4, this description will be used. Otherwise the description in the CANdb database will be used.

Reading and Rereading XML files

With large XML files the import may take several seconds. During this time CANoe does not accept any commands and an hourglass cursor appears.
Furthermore, memory requirements for the XML Import are high (3-5 times the XML file size).

If the memory size becomes too large for a XML Import or the import takes too long you should reduce the function catalog to the function blocks existing in the system (e.g. using the Editor) or use individual XML files as are provided by the MOST Cooperation.

The XML Function Catalog will be reread from file when

  • a new configuration is loaded with another XML Function Catalog, or
  • a new XML file is assigned to a bus in the Simulation Setup, or
  • at start of measurement if the content of the XML file was modified since the last import.

The first time a catalog is imported with CANoe, a file with the extension .vcf is created in the same folder. This file speeds up subsequent imports of the function catalog. Please note that this file is connected to the exact CANoe version and is overwritten if the catalog is accessed by other versions. For this reason, this file is not suitable for distribution with CANoe configurations.
