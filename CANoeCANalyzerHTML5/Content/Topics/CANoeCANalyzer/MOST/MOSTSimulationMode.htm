Simulation Mode

 

CANoe offers a simulation mode in which CAPL nodes access a simulated bus instead of the real bus. With .MOST the operation in simulation mode approaches that of a real MOST ring. Properties that are considered include: Sequence of devices in the ring, message addressing and retries when transmission fails.
Some properties are not simulated such as arbitration or behavior of the ring when a node is connected/disconnected. This mode is very useful, e.g. to start sample configurations supplied with CANoe.MOST or to subject user-created models to initial testing without the connected hardware.

  • Each channel in the Simulation Setup represents a MOST device. Device settings such as Master/ Slave/ Spy or node addresses are taken from the Hardware Configuration Dialog.
  • The channels are interconnected in a ring shape by a virtual cable. The sequence follows the channel numbering. A Timing Master is either configured in a device, or else the first device to be configured as a Slave assumes this task. This allows the user to switch over – with little effort –from operation with a real bus to simulated operation resuming with the previously simulated nodes.
  • Messages are routed around the ring from the sender over the consecutive nodes until it reaches the receiver, and from there it continues until it arrives back at the Sender. Message addressing is considered in this process. For example, retries are executed if a message is sent to an address that does not exist in the ring. If these retries are also unsuccessful the sending node returns a Tx failed acknowledgment. A Tx acknowledgment is generated if the receiver receives the message successfully.
  • The receiving node for a message and all Spy nodes identify the messages as received (Rx).
  • Both the Control Channel and the Asynchronous Channel are supported.
  • All node operating modes are supported: Master/Slave, AllBypass, Spy
  • The startup behavior of the virtual MOST ring includes: Setting of Light & Lock status, assignment of node position addresses and setting of the SBC and frequency. These options may also be passed from channel to channel via the virtual cable.
  • It is possible to simulate a break in the ring. This is done by intentionally deactivating a node's emitted light. This causes the lights of all subsequent nodes to be deactivated. Only those nodes from the Master to the node whose light was deactivated still emit light.

MOST150:

  • The Ethernet channel is also simulated.
  • In MOST150 all system messages are eliminated, including Alloc/Dealloc.
  • The SystemLock flag is set during startup and the shutdown flag during shutdown.

You can activate/deactivate the virtual MOST ring in CANoe in the Options dialog (Measurement|General).
