Source Code Generation

 

Another function of the VT System control is the generation of source code for control of the VT System. This function records manually implemented operations and converts them to code. This code can then be inserted in test modules, for example.

Basic handling of code generator

The code generator can be opened using in the main toolbar. Alternatively, you can simply click the separator bar at the bottom edge of the interactive circuit diagram view. As soon as the code generator area is open, manual operations will be recorded. If the generator is closed, the recording is stopped. You can change this behavior in the settings dialog for the code generator.

The toolbar of the code generator

The toolbar of the code generator provides the following functions:

The settings dialog for the code generator

Here, you specify how the code is generated.

  • General settings
  • Insert comments in the generated source code
    With this setting you specify whether a suitable comment is to be generated for each generated code block.
  • Record commands while generator is in the background
    If this setting is selected, manually performed operations will also be recorded while the code generator area is closed.
  • Insert blank line after each code block
    This setting can be used to insert a blank line after each generated block of code to make the code easier to read.
  • CAPL settings
  • Use function or method syntax
    This setting specifies whether CAPL commands are generated in function style or method style. For an explanation of the differences between these two styles, go to the Differences between Functions and Methods page.
  • .NET settings
  • Generate initialization code
    If this setting is active, the code generator will also generate the declarations of all required variables, in addition to the actual operations.

Supported programming languages

The code generator currently supports the CAPL, C#, and VisualBasic .NET programming languages.

Limitations

For some operations, there is no direct counterpart in code form. In this case, the code generator merely generates a comment that explains how the desired function can be generated manually.
