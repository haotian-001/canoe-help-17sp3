Time Synchronization

 

All modules of a VT System are internally time synchronized. In CANoe, the time stamps originating from the VT System are transferred to the common time base so that they can be compared with time stamps from other sources. There are several ways to do this, which differ in requirements, configuration, and accuracy. The highest accuracy is achieved with hardware-based time synchronization. A number of requirements must be met before hardware-based time synchronization can be used. Hardware-based time synchronization can be used both from CANoe and via driver-based synchronization.

Driver-based Time Synchronization

Many newer Vector network interfaces support the configuration of time synchronization via the Time sync device configuration dialog. For these interfaces, the dialog can be opened via the Vector Hardware Config dialog.
Network interfaces that are available as a module for the VT System (VT6104, VT6204 and VT6306) do not yet support this configuration of time synchronization.

Configuring Hardware-based Time Synchronization in the Driver

In order to use driver-based time synchronization, all network interfaces involved must support configuration via the Time sync device configuration dialog. One of the interfaces participating in hardware-based time synchronization must be configured as Master for it in its Time sync device configuration dialog.

Note

The master of the hardware-based time synchronization can in turn be synchronized with another master via another protocol, such as PTP.

If VT System modules are used in the current CANoe configuration and at least one network interface is configured for hardware-based time synchronization as described, the VT System automatically configures itself as a slave of the hardware-based time synchronization. If everything is configured correctly and all prerequisites are met, the following message is output in the Write Window of CANoe at the start of measurement: VT System: Driver-based time synchronization is active. If the time synchronization of the VT System does not work, e.g. because the VT System is not connected to the party line, this is also reported in the Write Window.
