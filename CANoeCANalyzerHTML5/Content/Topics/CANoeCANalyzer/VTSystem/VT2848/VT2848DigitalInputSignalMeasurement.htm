Measuring the Digital Input Signal

 

The digital data stream of each channel's signal line is captured. This happens regardless of whether the channel is used as in input or an output. An adjustable switching threshold (range 0…40 V) is used to differentiate between the High and Low states. You can set the switching thresholds for groups of 8 channels collectively.

The signal is sampled every 50 µs. The bitstream is made available to CANoe via a system variable. This system variable contains the 20 most recently sampled values, each of which is either 0 or 1, representing Low or High.

The module can also measure PWM signals on channels 1-16. For PWM measurement, you need to specify the maximum duration of the measurement window. If less than three signal edges are detected during this time period, the PWM parameters cannot be determined. These three signal edges are needed to determine the High and Low phases of the PWM signal and to calculate the PWM frequency and pulse/pause ratio. If no three edges are detected within this time, a duty cycle of 0% or 100% is output. The set time for the measurement window should be longer than the period of the lowest frequency to be measured. In order to detect 0% or 100% duty cycle as quickly as possible, the time should not be set to too high a value, as the old measured values are still displayed in the meantime.

Example: PWM Measurement

In this example, a modulated voltage at the LED channel is measured. The channel is connected to an ECU output. The aim is to determine the PWM frequency and pulse/pause ratio (duty cycle) parameters. In the example, the threshold value is set to 2.5 V, LED is connected to channel 9 and the threshold value setting applies to channels 9 to 16 collectively. The namespace for the entire module is DigitalSignals.

CAPL

double freq;  // variable for reading the PWM frequency
double dc;    // variable for reading the PWM duty cycle

// set threshold for channels 9-16 to 2.5 V
sysvar::VTS::DigitalSignals.SetThreshold (eVTSThresholdGroupChannels9To16, 2.5);

// set maximum measurement duration to 100 ms, because the lowest PWM frequency is 100 Hz
sysvar::VTS::LED.SetPWMMeasurementDuration (0.1);

// wait 200 ms and read measured PWM parameters
TestWaitForTimeout (200);
freq = @sysvar::VTS::LED::PWMFreq;
dc = @sysvar::VTS::LED::PWMDC;
