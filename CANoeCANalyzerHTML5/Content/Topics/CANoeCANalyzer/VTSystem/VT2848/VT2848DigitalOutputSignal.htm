Outputting a Digital Signal

 

The VT2848 can output a digital voltage signal on each channel.

  • The High level of the signal is the potential connected to Vbatt or Vext.
    It can be up to 60 V.
  • The Low level is the ground potential connected to ECU GND.
    The current can be up to 200 mA (sink/source).

The following output modes are available:

  • Inactive
    A signal is not generated. The channel's input remains active. The channel can also be connected to the ECU's output pin in this mode.
  • Constant Value
    The output value (High/Low) is determined in CANoe by the value of the corresponding system variables and the specified output stage mode (High Side Switch, Low Side Switch or Push-Pull). This can be changed at any time via CANoe; however, the rate of change is limited to about 500â1000 changes per second due to the delay in transmission to VT System.
  • PWM Output
    The VT2848 generates a PWM signal in accordance with the selected output stage mode.
    The pulse/pause ratio (duty cycle) and frequency are specified via CANoe system variables. Only channels 33-48 are capable of PWM output.
  • Bitstream Output
    A specified bitstream is loaded in the module and is output independently in accordance with the selected output stage mode; see Bitstream Output. Only channels 33-48 are capable of bitstream output.

Example: Outputting a Digital Signal

This example demonstrates the possibility of directly controlling the state of a channel (Dout) that is connected to an ECU input. Because the VT2848 outputs digital signals, only two states are possible. In Push-Pull mode, it switches between these two states.

CAPL

// Configure output to "push pull"
sysvar::VTS::DOut.SetOutputMode (eVTS2848OutputModePushPull);

// Choose Vext as source for high value for channels 1-4
sysvar::VTS::DOut.SetOutputSource(eVTSOutputSourceGroupChannels1To4, eVTSOutputSourceVExt);

// set output mode to "constant output"
sysvar::VTS::DOut.SetCurveType (eVTSCurveTypeConstant);

// output 1, wait for 100 ms, then output 0
@sysvar::VTS::DOut::DigitalOutput = 1;
TestWaitForTimeout (100);
@sysvar::VTS::DOut::DigitalOutput = 0;
