PWM Measurement

 

The module can measure PWM signals. High and low levels are differentiated by means of comparators whose switching thresholds can be adjusted. So that a 0 % or 100 % duty cycle can be correctly detected, a suitable timeout must also be set. These values are displayed if no further edge has been detected after the set time. It is advisable to choose the set time to be greater than the period of the lowest frequency to be measured. However, so that a 0 % or 100 % duty cycle can be detected as quickly as possible, the value of the time set should also not be too large, as in the meantime the old measured values are also displayed.

The frequency and duty cycle (pulse-width modulation, pulse-duty factor) are measured. In addition, the number of measured values on the A/D converter (250 k samples/s) with a high or low level that have been detected is also output. The module also calculates the PWM signal's high and low levels as an average of the voltage during the high and low phases.

Example: Start PWM measurement

The following example demonstrates the measurement of a PWM signal with a threshold of 12 V and a measurement duration of 100 ms.

CAPL

double frequency;
double dutyCycle;

// Set PWM threshold to 12V
sysvar::VTS::LowBeamLeft.SetPWMThreshold(12);
// Set PWM measurement duration to 100ms
sysvar::VTS::LowBeamLeft.SetPWMMeasurementDuration(0.1);

// Wait a bit until the PWM measurement has finished
testWaitForTimeout(200);

// Get measured values
frequency = @sysvar::VTS::LowBeamLeft::PWMFreq;
dutyCycle = @sysvar::VTS::LowBeamLeft::PWMDC;

// Check the frequency and duty cycle
if( frequency > 50 && dutyCycle > 20 )
{
  testStepPass();
}
else
{
  testStepFail();
}
