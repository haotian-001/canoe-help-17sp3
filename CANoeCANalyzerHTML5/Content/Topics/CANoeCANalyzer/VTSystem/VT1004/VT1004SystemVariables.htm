It is not possible to read out the output values. When the corresponding system variables are read, only the last value to be set is returned.
However, this will not necessarily reflect the current state of the hardware.

The system variables can be used in CAPL, XML and .NET test cases. They can also be used in analysis windows and panels.

Example: Test Case in a CAPL Test Module

In this example, bus bar 1 is set up with Vbatt/Gnd and the lines are swapped on the module; as a result, Gnd is no longer on line b of the internal bus bar but is instead on line a. A short circuit is then created on the LowBeamRight channel between its line a and Gnd (via bus bar 1). After a timeout, a check is performed to determine whether the LowBeamLeft channel is switched off (average voltage must be below 2 Volt).

// swap lines of bus bar 1 (Gnd/+Vbatt is connected to bus bar 1;
// Gnd is now on line a, +Vbatt on line b) on module "Module_Lights"
@sysvar::VTS::Module_Lights::RelaySwapBusBar1A = 1;
@sysvar::VTS::Module_Lights::RelaySwapBusBar1B = 1;

// now short-circuit of line a of channel "LowBeamRight" to bus bar 1
// = short-circuit of line a to ground
@sysvar::VTS::LowBeamRight::RelayBusBar1A = 1;

// wait for some time
testWaitForTimeout (100);

// now check channel "LowBeamLeft"; it must be switched off
// = average value of voltage must not exceed 2 Volt
if (@sysvar::VTS::LowBeamLeft::Avg > 2.0)
TestStepFail ("23", "Low Beam Left is not switched off");
