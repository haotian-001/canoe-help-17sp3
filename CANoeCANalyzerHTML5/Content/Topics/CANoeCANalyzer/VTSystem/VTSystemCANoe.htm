Notes for Usage with CANoe

 

Usage in CANoe

The measurement and output values, along with many settings (e.g. for the relays), are made available in CANoe via system variables. The VT System configuration creates these automatically. They use the reserved VTS namespace.

A VTS sub-namespace is created for each VT System module channel and for each VT System module. The corresponding channel and module system variables are created in these sub-namespaces. You can specify the name of each channel and module; however, names must be unique across all modules and channels. The names are saved to the VT System configuration.

Example 1: System Variables and Namespaces

VTS::LowBeamRight::Avg   System variable Avg of the LowBeamRight channel
VTS::Light_Modul         Namespace of the Light_Module module

To prevent creating dependencies in the test script related to the way channels are distributed onto modules, channels are not created as sub-namespaces within the module namespaces.

In addition, a number of settings are adjusted using special CAPL functions that use a method syntax to call the system variables and namespaces of the VT System's modules and channels.

Example 2

sysvar::VTS::LowBeamRight::Avg.SetIntegrationTime (0.1);
sysvar::VTS::LowBeamRight.ResetMinMax ();

The vtsystem_configure command is used to performed these tasks for XML test modules. You can also use existing patterns in XML test modules to access the VT System's system variables directly. No differentiation is made between the VT System's system variables and system variables from other sources.

A detailed list of available system variables, CAPL functions and XML commands is available in the Help sections of the corresponding modules, VT1004, VT2004, VT2516 and VT7001.

To use the VT System in CANoe, you must activate it in CANoe in Options dialog (Extensions|VT System). Note that this setting is part of the current CANoe installation. The VT System must be activated before a VT System configuration associated with a CANoe configuration can be loaded. The VT System configuration can then be created, changed or loaded with a CANoe configuration when a VT System is not connected; it only has to be activated. However, a measurement cannot be started in this case. The VT System hardware is required for this purpose.
