System Variables

 

On a certain channel of the VT2816 module you can only measure one at a time, either current or voltage. Therefore the system variables with measurement values of current or voltage are only updated if the according measurement mode is active. This means that e.g. the AvgVoltage system variable is only updated if voltage measurement is active and AvgCurrent is only updated if the current measurement is active. In CAPL you can use the SetMeasurementMode command to set the measurement mode.

It is not possible to read back output values. When the various system variables are read, only the most recent value is returned. However, this value does not always represent the current state of the hardware.

The system variables can be used in CAPL, XML and .NET test cases. They can also be used in analysis windows and panels.

Example: Reading Values in a CAPL Test Module

This example configures an analog voltage measurement to ground with a range of Â±60 V and then reads out the value at the end.

// Configure channel for measurement (single-ended, range +/-60V)
sysvar::VTS::TempSensor.SetMeasurementMode (eVTS2816MeasurementModeVoltageLineAToGnd60V);

// Configure integration time of 10ms for average value
sysvar::VTS::TempSensor::AvgVoltage.SetIntegrationTime( 0.01);

// wait briefly until setting is applied
TestWaitForTimeout( 100);

// read average voltage
avgVoltage = @sysvar::VTS::TempSensor::AvgVoltage;
