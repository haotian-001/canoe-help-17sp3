Firmware Update

 

Introduction

In order to use the expanded functionality, you may have to update the firmware of your VT modules.

If CANoe requires a new firmware version, you are notified of this in an error message in the Write Window. Without this update, the respective module cannot be used in CANoe.

A standalone tool is available for updating the firmware, which can be found in the CANoe installation directory:
[CANoeInstallationFolder]\Exec32\VTSFirmwareUpdater.EXE

Note

Before starting the firmware update, you must close CANoe, because otherwise the network adapter may be blocked, thus preventing the firmware update.

In order that the firmware update can establish a connection to the VT System, the VT System backplane has to be connected directly to the computer on which the tool is executed. Especially when using a Realtime Module (e.g. VT6020) it is necessary, to modify the Ethernet connection temporarily while the firmware update tool is executed on the computer on which the CANoe GUI is running. Generally this is essential when using distributed mode, what means that CANoe runs distributed in two computers. You have to take care of this even when a VN8900 is used together with the VT System. The connection between the VT System backplane and the VT6020 and VN8900 (or similar) respectively is replaced by a connection to the computer on which the firmware update tool is executed. After executing the update the original Ethernet connection for standard operation has to be re-established.

With the current version of the VT6000 operating system (available with CANoe 8.0 SP4), the firmware update tool is also installed on the VT6000. As a result, firmware updates can in principle always be carried out without having to modify the hardware. This is described in more detail in the following section.

Firmware Update via the VT6000 Module

The firmware update tool can also be run on the VT6000 module. (However, as described above, the tool is not installed as standard with all versions of the VT6000 operating system). In this case, the following steps are necessary for carrying out a firmware update:

Step by Step Procedure: Firmware Update

Copy the required firmware update files to the VT6000. This can be carried out in one of the following ways:

  • Access via network: A folder for access via the network is shared on the VT6000 in the standard configuration. The name of the folder is \\<IP Address of VT6000>\Tmp. However, access to this folder can be prevented by security software on some systems. The shared folder on the VT6000 can be found at D:\Tmp\.
  • Copy the required firmware update files to a USB stick. The USB stick can be plugged into one of the two USB ports of the VT6000 in order to access the files from the firmware update tool.
  • Switch off the VT system and connect the VT6000 to another computer with a USB cable. For this purpose, the USB cable on the VT6000 must be connected to the Firmware Update connection. Now the flash memory of the VT6000 can be accessed directly. Copy the required firmware update files into the Tmp folder on the Data partition.
  1. Then switch on the VT system and establish a remote desktop connection (this is described on the page VT6000: Advanced settings).
  2. Start the Windows Explorer.
  3. When the firmware update tool has been installed, it can be found in the folder
    C:\Program Files\Vector VT System Firmware Updater.
  4. Start the tool VTSFirmwareUpdater there.
  5. Select the correct network adapter.
  6. Add the firmware update files to the list of available updates. Among other things, this is possible via the menu path File|Add File.

Then follow the steps described below in the tool to execute the update.

Executing the Firmware Update

When the firmware update is started, you must select the network adapter to which the VT modules are connected.

Please ensure that the backplane of the VT Systems is directly connected to the selected network adapter.

The Firmware Files list displays the loaded firmware update files. The latest firmware versions are always provided for the installed CANoe version.

Note

Newer firmware versions are downwards-compatible with older CANoe versions. A module with a newer firmware version can always be operated with an older CANoe version.

The Available Modules for Update list displays only the modules for which a firmware update can be performed. You should update the firmware only for the modules for which a newer firmware version exists. For this purpose, compare the version information in the New Version and Cur Version columns. If the version number in the New Version column is greater than that in the Cur Version column, you should place a check mark in the Active column.

Once you have selected all modules for the firmware update, you can use the button [Start Update All] to start the firmware update for all selected modules. The selected modules will be updated in parallel.

If an error occurs while updating a module, you should repeat the update for this module. If the update operation is aborted, a basic firmware version is loaded that you can use to repeat the firmware update.
