Outputting a Digital Signal

 

The VT2516 can output a digital voltage signal on each channel. The ECU interprets this as a sensor signal. High and Low levels can be set separately for each channel. They can range from 0 to 25 V. The current can be up to 30 mA (sink/source).

The following output modes are available:

  • Inactive
    A signal is not generated. In this case, the Signal Generator is disconnected completely from the signal line. The channel can also be connected to the ECU's output pin in this mode.
  • Constant Value
    The output value (High/Low) is specified via system variables in CANoe. It can be changed at any time via CANoe; however, the rate of change is limited to about 500–1000 changes per second due to the delay in transmission to the VT System.
  • PWM Output
    The VT2516 generates a PWM signal.
    The pulse/pause ratio (duty cycle) and frequency are specified via CANoe system variables.
  • Bitstream Output
    A specified bitstream is loaded in the module and is output independently (see Bitstream Output).

Example: Outputting a digital signal

This example illustrates exertion of direct control over the state of a channel, switch, that is connected to the ECU input. Because the VT2516 outputs digital signals, only two states are possible. These states are set in the program using 0 and 1. In the example, the High value (state 1) for the state is set to 10.5 V and the Low state (state 0) to 0.8 V.

CAPL

// set high and low states
sysvar::VTS::DOut.SetPWMVoltageHigh (10.5);
sysvar::VTS::DOut.SetPWMVoltageLow (0.8);
// set output mode to "constant voltage"
sysvar::VTS::DOut.SetCurveType (eVTSCurveTypeConstant);
sysvar::VTS::DOut.SetStimulationMode (eVTSStimulationModeVoltage);
// output 1, wait for 100 ms, then output 0
@sysvar::VTS::DOut::DigitalOutput = 1;
TestWaitForTimeout (100);
@sysvar::VTS::DOut::DigitalOutput = 0;

.NET (#C)
