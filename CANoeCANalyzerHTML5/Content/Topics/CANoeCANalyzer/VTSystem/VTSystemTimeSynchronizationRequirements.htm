Requirements of Hardware-based Time Synchronization with the VT System

 

You must observe the following rules to ensure that the hardware-based time synchronization works correctly together with the VT System:

  1. The party line connections of the modules in a VT System rack are connected via the backplane. Therefore, a maximum of one party line connection is required for one rack. If only one rack is used and only network interfaces that are VT System modules are used, then no party line connections are required.
  2. VT System modules that are not network interfaces are usually synchronized automatically via EtherCAT®. If you only use such modules, then no party line connections are required between several VT System racks.
  3. If you use several VT System racks, they often do not have to be connected via party line, because the synchronization is done via EtherCAT®. However, racks in which VT System network interfaces (VT6104A, VT6204 or VT6306) are operated may need to be connected via party line. This is always the case if it is not the first rack in EtherCAT® order.
  4. If you are using Vector network interfaces that are not VT System modules, then you must connect the VT System to these modules by Party Line. You must make a party line connection from the network interface to the first VT System rack in EtherCAT® order.
  5. It is very convenient to make the party line connection to a VT System rack via the trigger connector of the backplane. Pin Trigger 1 and the adjacent pin GND are used for this purpose.

Example 1

You are running a VT System rack with a VT6204 and some I/O modules like VT1004, VT2004 and VT2516. You do not operate any VT System network interfaces outside the rack.

Because of the backplane connections, you can use hardware-based time synchronization without additional cabling.

Example 2

In one setup, cascade two of the racks from Example 1. In addition, you operate a VN5640 Ethernet interface. Since each rack operates a network interface, you must connect both to the party line and also connect the VN5640 via party line. If more than two partners have to be connected via party line, the use of a Multi SYNCbox is recommended.

EtherCAT® Order

In EtherCAT® order comes first the VT System rack, which is directly connected via Ethernet cable to the system running the Runtime Kernel of CANoe. Then comes the rack connected to the first rack, and so on.

Additional Restrictions for Older Hardware

If you are still using a first-generation backplane (VT8006 or VT8012, both without A or B) in the first VT System rack, the synchronization connection for the VT System cannot be made through the backplane. Instead, you need a suitable VT System module. If you operate a VT System network interface in the first rack, no changes to the cabling described above are necessary, since a network interface can take over the task directly.

If no network interface is operated in the first VT System rack, the synchronization connection can still be established via a VT7001(A) or VT7101. For these modules, you must establish the party line connection via the two sync pins on the module instead of via the backplane.

If neither Network Interface VT System modules nor VT7001(A) or VT7101 are operated, you cannot use hardware-based time synchronization. Instead, you must use the software-based time synchronization.
