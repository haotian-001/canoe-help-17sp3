Bitstream Switching

 

For arbitrary switching of matrix paths, each channel can be operated separately in bitstream mode. The desired switching pattern is stored in a text file and loaded onto the module before the stimulation is started. The bitstream is defined in a file, and can have a maximum of 4096 bits.

You can load the bitstream from the file to the VT System module via the LoadWFBitStream CAPL function or the vtsLoadWFBitStream CAPL function onto the VT System module. In addition, you can set the time increment, number of repetitions, pause, start delay and start point parameters via the SetWFParams CAPL function or the vtsSetWFParams CAPL function.

  • The time increment parameter specifies the interval between two bits.
  • The number of repetitions parameter specifies how many times the bitstream is repeated.
  • If the bitstream is output repeatedly, the pause parameter specifies the length of the pause before each repetition.
  • The start delay defers the start of the bitstream switching by the given time.
  • The start point specifies at which point of the stream bit switching starts.

File Format

The file format is very simple. It contains the bitstream in the form of a string composed of the numbers 0 and 1. You can insert spaces and line breaks between the numbers to structure the bitstream. Comments can be inserted after //. All characters on this line are interpreted as comments and are ignored, up to the end of the line..

To run bitstream switching, the following steps are required, either manually or programmatically:

  • set Curve Type to Bitstream,
  • select all rows which shall be connected to the selected channel,
  • set the desired parameters for bitstream switching,
  • start the stimulation.

After start of the stimulation, all connected rows are switched synchronously with the configured parameters.

Note

While a PWM stimulation is running, the static setting of each matrix node is overruled if and only if the corresponding row is selected. After stop of the stimulation, the last state of the mode is maintained. The last static configuration of the node is set when Curve Type is changed to Constant.

Example 1

The following file defines the bit sequence 001100110010101100:

// Sample bitstream
0011
0011
0
0       // bits may be defined in one line
1 0 1 0
11 00   // definition in groups

Example 2: Bitstream Switching

In this example the rows 1 and 2 are switched on the matrix channel MyMatrixChannel_2 corresponding to the bit sequence defined in the file MyBitstream.txt with a time increment of 10 ms, a pause of 1s and 10 repetitions. The selection of rows 1 and 2 corresponds to the bit pattern 00II, therefore the argument for the SetOutputMask method has an integer value of 3.

CAPL

// This calls the method SetCurveType on VTS::MyMatrixChannel_2
sysvar::VTS::MyMatrixChannel_2.SetCurveType(eVTSCurveTypeBitStream);

// This calls the method SetOutputMask on VTS::MyMatrixChannel_2
sysvar::VTS::MyMatrixChannel_2.SetOutputMask(3);

// This calls the method LoadWFBitStream on VTS::MyMatrixChannel_2
sysvar::VTS::MyMatrixChannel_2.LoadWFBitStream("MyBitstream.txt");

// Configure waveform. Parameters:
   // TimeIncrement (time to hold each sample)       = 10ms
   // Pause (pause between two waveform repetitions) = 1s
   // NumberOfRepeats (number of repetitions)         = 20

sysvar::VTS::MyMatrixChannel_2.SetWFParams(0.010, 1.0, 20);

// Start stimulation
sysvar::VTS::MyMatrixChannel_2.StartSimulation();
