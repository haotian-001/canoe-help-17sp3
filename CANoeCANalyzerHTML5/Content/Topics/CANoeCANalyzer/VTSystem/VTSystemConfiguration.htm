VT System Configuration

Ribbon: Hardware Tab|Configuration

In this dialog you can create or modify a VT System configuration. You can add modules to or remove modules from your configuration (context menu). In addition, you can change the order of modules in the configuration via drag and drop.

Each module is represented as a node in the tree. The leaves of the node represent the module's different channels, whereas the module node itself represents the complete module. You can change the names of modules and channels. The channel name serves as the namespace for the active measurement values and settings of the channel. Likewise, the module name is used as the namespace for module-wide variable settings.

The available measurement values are shown for each channel. These can be activated and parameterized. A system variable is created for each of the activated measurement values.

Example

The Avg measurement value (this name is preset and is specified in the dialog) of the LowBeamRight channel (this name was set by the user for the given channel) can be read via the system variable VTS::LowBeamRight::Avg.

In addition there are comparable system variables in this namespace for writing values. These system variables are not displayed in the configuration dialog, however, since they cannot be configured or deactivated. They are always created automatically.

It is also possible to create or modify a VT System configuration and to create test configurations without having any hardware connected.
For this purpose, you must first activate the VT System in the Options dialog. Then create a VT System manually in the VT System configuration dialog using the Add Modules context menu command and configure it. Alternatively, you can also read in an existing configuration via the Import VT Config… context menu command.
You can only start the measurement when the corresponding VT System modules are connected for the configured modules.

Important Note!

Please note that the real VT modules have to be connected in exactly the same sequence (order, type, number) as they are configured in this dialog.

Menu Bar

Configuration

  • New Configuration
    Creates a new configuration based on the currently connected modules (which means it may also be empty).
  • Adapt to Connected Modules…
    Adapts the current configuration to the connected VT System modules by adding/removing/re-arranging the configured modules.
  • Import Configuration
  • Replace
    Imports a previously created VT System configuration file (VTCFG).
    The current configuration will be overwritten by the new configuration.
  • Merge
    Imports one or more VT System configuration files (VTCFG).
    The current configuration will be merged with all modules of the selected files.
  • Export Configuration…
    Exports the current configuration.
  • Analyze Performance…
    This function allows checking the current VT System configuration for performance issues.
    In order to do so the number of events generated per second by the VT System is analyzed.
  • Open Module Description Folder…
    Opens the folder that contains the module descriptions.

Module

  • Rename Module
    Renames the selected module or channel.
  • Add Module
    Adds a module of the selected type to the configuration.
  • Add Application-specific Module
    Adds an application-specific module of the selected type to the configuration.
  • Remove Module
    Deletes all selected modules.
  • Remove All Modules
    Deletes all configured modules.
  • Set all Modules Online
    Switches all configured modules to online mode.
  • Set All Modules Offline
    Switches all configured modules to offline mode.

Display Fields

Values

  • Active
    Activates or deactivates the transmission of the measurement value.
    A system variable is created for each active measurement value according to the following pattern.
    VTS::ChannelName::Name
  • Name
    Name of the measurement value.
  • Integration Time
    Integration time of the measurement value (not available for all measurement values).
  • Cycle Time
    Cycle time at which a measurement value is picked up from the module and updated in CANoe.

Constraints

  • Setting
    Setting on that the constraint is defined. The list of the possible settings depends on the selected channel.
  • Operator
    Defines the type of the constraint. The list of the possible operators depends on the selected settings. For the numerical values usual comparison operators are available. For relays it is set whether the opening or closing of the relay should be restricted.
  • Parameter
    Completes the constraints. Is the restricted setting a numerical value, a decimal comparison value can be entered for the selected operator. For relays the condition is displayed for that the selected operation is permitted.

With the [Add] button you can create additional constraints. This button is available if in the tree view on the left side a single channel is selected on that settings can be defined.

With the [Remove] button you can delete single settings. This button is available if minimum one setting is selected.

With the [Edit…] button you open the Relay Constraint Editor for editing parameters of the constraints. This button is available if a single setting with a complex parameter is selected; e.g. a constraint is defined on relay settings.

Pin Usage

Note: Only available for VT2710 modules.

On this tab, you can configure how the pins on both upper connections of the VT2710 should be used. You can assign the three pin groups 9–8, 7–4 and 3–2 to the different functions of the module:

  • Digital I/O
    The pins are used as digital inputs or outputs. You can set the direction separately for each pin.
  • UART
    The pins are used for the Tx or Rx lines of the UART.
  • SPI
    The pins are used for the different signal lines of the SPI interface.
  • RS485
    The pins are used for both signal lines of the RS485 interface.
  • User FPGA
    The pins are controlled by the User FPGA and are not available to CANoe (esp. Option .Sensor) anymore. Depending on the pin, you can choose what kind of hardware mode you want to use (e.g. digital output, digital input, RS232, RS485 or I2C).

It is possible to use different interfaces simultaneously. For example, pins 2–3 can be used for a UART channel. Pins 4–7 can be used in parallel for a SPI channel with a Chip Select. Pin 8–9 can then still be used for digital I/O.

Voltages

Note: Only available for VT2710 modules.

  • Internal voltage
    Here, you can configure the internal voltage source of the VT2710.
  • Voltage source
    This setting allows you to select for each of the two upper connections of VT2710 whether it should be supplied from the internal or external voltage source.
  • Output internal voltage at external
    If the internal voltage source is used, its voltage can be output at the external pin of the left or right upper connection of the VT2710.
  • LVDS Voltage Supply
    This setting activates or deactivates the voltage supply for the LVDS connections of the module and determines the output voltage.
