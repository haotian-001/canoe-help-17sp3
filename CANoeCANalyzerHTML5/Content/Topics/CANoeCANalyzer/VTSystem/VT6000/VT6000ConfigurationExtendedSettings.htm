Extended Settings

 

You cannot adjust all of the settings from the user computer (distributed mode). For example, system settings such as IP address changes or some Vector hardware settings can only be made directly in the VT6000. To adjust these settings, you need to create a remote desktop connection to the VT6000.

Remote Desktop Connection

You can easily create a remote desktop connection via the Vector Tool Platform dialog in CANoe.

  1. To do this, open the menu path Home ribbon tab|Group Measurement|Vector Tool Platform|Connection|General|Remote Desktop|Connect.

  2. To log in to the VT6000 using the remote desktop connection, use the following login:

    User Account: Administrator

    Password: (no password)

Two windows are usually opened on the VT6000 (Control Service Console of the Vector Tool Platform and RuntimeKernel.EXE). For further configuration the Control Service Console offers corresponding buttons, e.g. to open the Windows Explorer, the command line console, the panel or the Vector hardware configuration as well as to start and stop the RT kernel.

Changing System Settings

The VT6000 uses a write filter that protects the system from getting damaged when the module is switched off. Because of this write protection specific measures are necessary to change system settings. Without these steps the changes will get lost at reboot.

Change of network settings

Please follow the following steps to change the network settings of the VT6000:

  1. Create a remote desktop connection to the VT6000 (see above).
  2. Start the Windows Explorer in the Control Service Console.
  3. Navigate in the tree view of the explorer to Control Panel|Network Connections. In the right part of the explorer the existing network interfaces will be shown.
  4. Change the settings (e.g. IP address) of the appropriate network interface.

Please wait about 10 seconds after confirming your changes to ensure that all settings are stored in the Windows Registry.

  1. Navigate in the tree view of the explorer to drive C:\Scripts\WriteFilter and execute the file PersistIPConfiguration there.

Wait some seconds to ensure that all settings are stored on the memory chip of the system.

Note

When changing the network address you should note that the IP addresses of both network cards on the VT6000 are assigned to different sub networks.

Otherwise CANoe can not be connected with the module.

Changes in Vector hardware configuration

Please follow the following steps to change the Vector hardware configuration:

  1. Create a remote desktop connection to the VT6000 (see above).
  2. Open the Vector Hardware dialog in the Control Service Console window via the [Vector Hardware Configuration] button.
  3. Change the hardware configurations according to your needs.

Please wait about 10 seconds after confirming your changes to ensure that all settings are stored in the Windows Registry.

  1. Start the Windows Explorer in the Control Service Console.
  2. Navigate in the tree view of the explorer to drive C:\Scripts\WriteFilter and execute the file PersistVectorHardwareConfiguration there.

Wait some seconds to ensure that all settings are stored on the memory chip of the system.

Changing advanced system settings

In order to be able to persistently change other settings, such as changing system settings or installing new HW drivers, you must first disable write protection.

To do this, proceed as follows:

  1. Establish a remote desktop connection to the VT6000 (see above).

  2. Start the Windows Explorer in the Control Service Console.

  3. In the tree view of the Explorer, navigate to drive C:\Scripts\WriteFilter.

    Execute the file DisableWriteFilterAndReboot located there.

    The VT6000 is then restarted. After the reboot the write filter is disabled.

  4. Re-establish a remote desktop connection to the VT6000 and make the desired changes.

  5. After all changes have been made, you should reactivate the write filter for security reasons.

    To do this, navigate again in Windows Explorer to the C:\Scripts\WriteFilter folder and execute the EnableWriteFilterAndReboot file located there.

    The VT6000 is then restarted again. After the reboot the write filter is active and you can use the module normally again.
