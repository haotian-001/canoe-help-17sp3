Operation

 

Operating the distributed mode on the VT6000 realtime module is very similar to standard operation.

To run a simulation, take the following steps:

  1. Start the VT System and connect it to the user computer.
  2. Start CANoe on the user computer.

    3. In the Vector Tool Platform dialog (Home ribbon tab|Group Measurement|Vector Tool Platform) under Connection|Device Selection the available devices are displayed.
    Select the corresponding device.
    To use the selected device as RT server, activate Configure currently selected device as RT server under CANoe/CANalyzer|Distributed Mode|RT Server.
    Configure the connection under CANoe/CANalyzer|Distributed Mode|Connection Control.
    Your settings are applied directly.

  3. Now activate VT System in the VT System configuration dialog (Home ribbon tab|Hardware|VT System|Configuration).
    There, select the appropriate network adapter to which the VT System backplane is connected on the VT6000.
  4. Load the desired CANoe configuration onto the user computer (distributed mode).

    5. You may be alerted about a non-supported modeling library (DLL file) at this point.
    If the simulation can run without this modeling library and no CAPL program makes direct calls on it, you can ignore this message.

  5. Setting up the configuration is essentially the same as usual.

    6. The channels and hardware are assigned and programmed via the usual dialogs. Only the [Driver] button in the hardware dialog cannot be used in the usual way: in this case it leads to the driver settings of the local user computer (distributed mode) and not to the settings of the remote RT server.

  6. Starting the measurement is essentially the same as in standard mode. The data required for the simulation is copied automatically to the RT server.

    7. Some of the current function limitations lead to error messages and cause the measurement start to be aborted. In such a case, adherence to these limitations should be checked.

  7. Usage is essentially the same as usual.

Note

The following points are new and should be noted:

  • The display at the bottom left that shows the fill level of the buffer between the realtime branch (simulation) and the evaluation branch (interface) is now divided into two: it thus reflects the two buffers on the RT server and the user computer (distributed mode).
    Filling of the buffer on the RT server usually indicates a network transmission problem.
    Filling of the buffer on the user computer (distributed mode) usually indicates a capacity problem on the user computer (distributed mode) and can be resolved via the usual interface optimization techniques.
  • Depending on the performance of the (Ethernet) network between the two computers, there is a certain temporal distance between bus events and their display. This is especially noticeable if a (CAN or LIN) bus reaction is expected in response to an operation action e.g. on a panel.
  • The RT server is started automatically on the VT6000 when it is started up, and is ended when the VT System is shut down. No intervention is necessary.
  • The configuration used must adhere to the current limitations.
  • When working with realtime modules (VT6000 family), it is possible to use serial ports on USB-RS232 converters.
