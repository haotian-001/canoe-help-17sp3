VT6104/VT6204/VT6204A/VT6204B – Network Interface Module

 

General

The VT6104 network module has four network interfaces. You can configure the four channels in the driver as CAN or LIN channels, as required. You can also use relays to activate various signal paths and to insert errors such as short circuits and line breaks.

This is also valid for the VT6204 module. In addition, the VT6204 module allows you to operate FlexRay instead of CAN or LIN, if an appropriate piggyback is used. For the VT6204 and VT6204A this is possible on channel 1 only. The VT6204B allows operating FlexRay on channels 1 and 2.

The VT6104/VT6204 has two logical components: The actual network interface, which has four channels and offers the switching options for various signal paths and errors typical of the VT System. You configure the interface part using the Vector driver. The relay part is controlled via the VT System and its configuration. This part thus behaves like a VT System module with regard to its usage and configuration.

Notes on Module Series VT6104

The term VT6104 and the belonging description also cover variants of the module. There are two variants of the VT6104 module. Compared to the older variant (labeled VT6104), the newer variant (labeled VT6104A)supports the following additional features:

  • CAN FD
  • K-Line

However, there are no differences between the available relays and the system variables that control them and further features described here.

Switching Different Signal Paths

Usage, Functions, Technical Data

Relays can be used to switch connections to High and Low lines, other bus lines, Vbatt, Gnd and the bus bar. It is also possible to switch a 120 Ohm termination between Low and High, and to interrupt the High and Low lines. Additional relays can be used to switch the supplementary functions of the piggybacks.

To switch the relays, use each channel's system variables.

Relays for piggy supplementary functions:

  • Pin 3 to ECU ground: The ground line is at Pin 3, and in the case of galvanically isolated piggybacks, is connected separately to the plug. This line is typically connected to the ECU ground. This connection can be created internally using a relay, which obviates the need for complicated wiring at the plug.
  • Pin 9 to ECU battery voltage (Vbatt): Some piggybacks, especially opto-isolated ones, require a power supply at pin 9. These lines are then typically connected to the battery voltage that also powers the ECU. This connection can be created internally using a relay, which obviates the need for complicated wiring.
  • Pin 4 to piggy ground: This relay can be used for special functions with certain piggybacks, e.g. for 100 ohm termination in high-speed mode with the 5790c.
  • Pin 8 to piggy ground: Switches special functions like pin 4, e.g. reducing termination to 500 ohm with the 1054.

The pin assignments for the piggybacks and their supplemental functions are based on the pin assignments for the 9-pin CAN/LIN/FlexRay plugs that are typically used with other Vector interfaces, as described in the CANpiggy/LINpiggy/FRpiggy documentations.
