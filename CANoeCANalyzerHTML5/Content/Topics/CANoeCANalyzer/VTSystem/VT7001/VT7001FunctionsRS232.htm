RS232 API Functions

 

The VT7001/VT7101 module has three serial interfaces with which external devices can be addressed. They are primarily intended to operate power supplies and displays. Values such as those from a voltage measurement of a power supply can also be received, however.

The following commands are used solely to access the serial interfaces of the VT7001/VT7101. They are heavily oriented toward the API for accessing the RS232 interfaces of a computer.

With the exception of callbacks, all commands are called up on a system variable namespace which specifies the channel used. A serial interface is available on each of the two channels for operating external power supplies (standard designation: PS1 and PS2) and on the module channel (standard designation: M1_Sup1 resp. M1_Sup2).

Function

Short Description

SerialOpen

Opens a serial port.

SerialClose

Closes a serial port.

SerialConfigure

Configures a serial port.

SerialSend

Sends block of data asynchronously on serial port.

SerialReceive

Sets receiver buffer for serial port.

SerialSetOnSendHandler

Sets the callback handler for Send events.

SerialSetOnReceiveHandler

Sets the callback handler for Receive events.

SerialSetOnErrorHandler

Sets the callback handler for Error events.

RS232 Callback Handler

Short Description

<OnSerialSend>

Handler will be called after completion of request issued with SerialSend.

<OnSerialReceive>

Handler will be called when data has been received on serial port.

<OnSerialError>

Handler will be called if an error has occurred.

General information on use

Data can be transmitted and received between the opening and closing of an interface.

Example

byte data[2];
int length = 2;
data[0] = 1;
data[1] = 2;

// send two bytes

if ( 0 == sysvar::VTS::ECUPowerSupply.SerialConfigure(eVTSBaudRate9600, eVTSDataBitsEight, eVTSStopBitsOne, eVTSParityNone)) Write("Port has been configured.");
if ( 0 == sysvar::VTS::ECUPowerSupply.SerialOpen() ) Write("Port has been opened.");

TestWaitForTimeout( 10);  // Wait briefly to make sure settings are applied and port is ready

if ( 0 == sysvar::VTS::ECUPowerSupply.SerialSend( data, length) ) Write("Sending %d bytes to serial port.", length);
if ( 0 == sysvar::VTS::ECUPowerSupply.SerialSend( data, length) ) Write("Sending %d bytes to serial port.", length);

TestWaitForTimeout( 10);  // Wait briefly to make sure data is sent before closing the port

if ( 0 == sysvar::VTS::ECUPowerSupply.SerialClose() ) Write("Port has been closed.");

When using the serial interfaces, note that data could be lost during the transmission if the transmission speed is not adapted to match the capability of the computer. For this reason, it is important that you use the <OnSerialSend> callback when transmitting, as it indicates that data was actually transmitted. In addition, data could be lost if it is sent to a serial interface of the VT7001/VT7101 module in very quick succession.

Technical Limitations

The serial interfaces on the VT7001/VT7101 module only have Rx/Tx lines, i.e. a handshake is not possible. It is therefore not possible to change settings for the handshake.

Permissible transmission speed: 1200, 2400, 4800, 9600, 19200 and 38400 baud.
