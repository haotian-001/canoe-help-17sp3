Outputs

 

The VT7001/VT7101 has two separate outputs (OUT1 and OUT2) that each have their own ground line (GND1 and GND2). A variety of configurations can be achieved by combining these with the three power supply units (two external, one internal). In particular, you can control the ECU's two power supply inputs (clamp 15 and clamp 30) separately. You can also connect two different ECUs, execute measurements with ground offset or Vbat offset, etc. For a description of these different modes, see Interconnection Modes.

There are also two auxiliary outputs, OUT1 aux and OUT2 aux. These correspond to OUT1 and OUT2, but are not included in the measurements. You can make separate connections to them, as their purpose is to supply power to any additional components that may be needed. While these additional connections are fed from the same operating voltage, they should not add to the ECU's power consumption.

An additional connection (OUT1 bus bar) lets you connect the battery voltage Vbat to the other VT modules (typically via bus bar 1 or dedicated connections). This makes it easier to wire the VT System, as does the fact that the module has its own independent fuse (polyfuse).

Example: Outputting a Predefined Curve Form

An external power supply unit (namespace ExtSupply) is used in this example. The ECU is connected to OUT1 (namespace ECUpower) and GND1; the VT7001/VT7101 module is named PowerSupply. The output voltage is determined by a predefined curve (powercycle.TXT). The time increment for the curve's interpolation points is 50 Âµs; the curve is repeated eight times after a pause of 200 ms.

In principle, the generation of the curves functions as for the output of the arbitrary voltage curves with the VT2004. The file format is also identical. (However, the required function calls for starting a curve output are not identical!) Additional information on the curve output is therefore provided on the page Arbitrary Wave Forms (VT2004).

CAPL

// set mode to one power supply only -> external power supply 1
sysvar::VTS::PowerSupply.SetInterconnectionMode (eVTSInterconnectionModeSup1);

// load waveform from file
// factor is set to 0.2: 1 V control voltage -> 5 V output at power supply
sysvar::VTS::ExtSupply.SetRefVoltageMode (eVTSRefVoltageModeAnalogWaveForm, 0.2);
sysvar::VTS::ExtSupply.LoadWFVoltage ("powercycle.txt");
sysvar::VTS::ExtSupply.SetWFParams (0.00005, 0.2, 8);

// switch outputs on and start output curve
@sysvar::VTS::ECUPower::Active = eVTSOutputModeActive;
sysvar::VTS::ExtSupply.StartStimulation ();

Delays when Switching the Outputs

When switching the outputs of a VT7001/VT7101, unintended connections may theoretically appear. This applies to the state at which the two output pins are connected (Active = -1). In an unfavorable case, the connected power supply may become short-circuited due to minor differences in the switching behavior of the relays. To avoid this, there is a deliberate delay of 100 ms between opening and closing of the related relay during transitions into or from this state.

Up to and including CANoe 8.2 this delay of 100 ms also existed for normal activation/deactivation of the outputs of a VT7001/VT7101 module. Since this is an unnecessary restriction, this delay has been removed starting with CANoe 8.5. This means that fast activation/deactivation of the outputs is only limited by the latency of the triggering and the actual connection of the relay. This latency is at maximum 25 ms. If a configuration contains time-based assumptions that refer to the old connection behavior, you can re-enable this behavior by adding the following setting to the file CAN.ini:

[VT_System]
EnableVT7001FastOutputSwitching=0
