Using the VT System Offline Mode

 

As of CANoe 11.0, one or more VT System modules can be switched to an offline mode. In this mode, it is no longer necessary for the actual module hardware to be present in your VT System.

The VT System Offline Mode is completely independent of whether you operate CANoe with a real or simulated bus. This allows you to use VT System modules in offline mode, even if the remaining hardware (including other VT System modules) is operated in real (online) mode.

[Use Cases]

The VT System Offline Mode is especially helpful if you want to run a CANoe-configuration without the corresponding VT System hardware. To do this, simply switch the non-existing modules in your configuration to offline mode.

[Offline Module Behavior]

Modules that are in offline mode do not provide measurement values. However, if you need measurement values for the correct execution of your CANoe configuration, you can generate them yourself. CANoe provides you with an offline stimulation variable for each measurement value. If you write to this variable, the corresponding measurement value is also updated. The offline stimulation variables can be found in the Offline namespace of the respective VT System channel.

All stimulation variables of the offline modules can be written to as usual. However, this has no effect on the CANoe measurement or the connected VT System hardware. Any constraints configured are taken into account in offline mode as well as during normal operation.

The VT System CAPL functions can also be called for offline modules as usual.
