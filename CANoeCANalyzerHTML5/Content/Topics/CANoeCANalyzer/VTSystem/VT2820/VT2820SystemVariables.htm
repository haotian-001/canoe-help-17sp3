System Variables

 

You can use system variables to modify the following settings for channels 1 to 12 of the VT2820. Each of these has a normally open relay that is used to connect the two lines, as well as two normally open relays with which you can connect line a to one of the two bus bar lines.

Note

The two bus bar lines are often connected to ground and Vbatt.

To prevent a short circuit between the two lines, it is not possible to close both RelayBusBarA and RelayBusBarB for a given channel at the same time.

The namespace is the name of the channel in the VT System configuration:

System variables can be used to read or write the following settings for a VT2820 module. The settings apply to the entire module.

The namespace is the name of the module in the VT System configuration:

It is not possible to read back output values. When the various system variables are read, only the most recent value is returned. However, this value does not always represent the current state of the hardware.

The system variables can be used in CAPL, XML and .NET test cases. They can also be used in analysis windows and panels.

Example: Test Case in a Test Module

In this example, a short circuit is created between line a and GND via the bus bar. GND is connected to line b of the bus bar. After a short timeout period, the short circuit is ended and a short circuit is then created with Vbatt, which is connected to line a of the bus bar.

CAPL

// GND is connected to bus bar line b. Short circuit line a of channel "DoorSensor" to Gnd.
@sysvar::VTS::DoorSensor::RelayBusBarB = 1;

// wait until short circuit is recognized
TestWaitForTimeout( 100);

// end short circuit.
@sysvar::VTS::DoorSensor:: RelayBusBarB = 0;

// wait briefly
TestWaitForTimeout (100);

// Vbat is connected to bus bar line a. Short circuit line a of channel "DoorSensor" to Vbat.
@sysvar::VTS::DoorSensor:: RelayBusBarA = 1;

.NET (#C)
