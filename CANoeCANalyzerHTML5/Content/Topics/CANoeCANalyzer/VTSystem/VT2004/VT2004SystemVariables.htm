System Variables

 

There are two variants of the module VT2004. Among other features, the newer variant (label VT2004A) supports a wider range for voltage stimulation than the older variant (label VT2004). The following section lists the values for the new variant VT2004A; the values for the old variant are in brackets.

The following settings and output values can be read or written for each VT2004 channel via the system variables.

The namespace is the name of the channel specified in the VT System configuration:

It is not possible to read out the output values. When the corresponding system variables are read, only the last value to be set is returned. However, this will not necessarily reflect the current state of the hardware.

The system variables can be used in CAPL, XML and .NET test cases. They can also be used in analysis windows and panels.

Example: Test Case in a CAPL Test Module

In this example, bus bar 1 is set up with Vbatt/Gnd and the lines are swapped on the module. Gnd is now on internal bus bar line a. A short circuit is then created on the TempSensor channel between its line a and Gnd on the bus bar.

// swap lines of bus bar 1 (Gnd/+Vbatt is connected to bus bar 1;
// Gnd is now on line a, +Ubat on line b) on module "Sensors"
@sysvar::VTS::Sensors::RelaySwapBusBar1A = 1;
@sysvar::VTS::Sensors::RelaySwapBusBar1B = 1;

// now short-circuit of line a of channel "TempSensor" to bus bar 1
// = short-circuit of line a to ground
@sysvar::VTS::TempSensor::RelayBusBar1A = 1;

// wait and e.g. check fault memory
testWaitForTimeout (100);
