Performance Monitor

Ribbon: File Tab|Options|Measurement|Performance

When the Performance Monitoring function is switched on, a set of performance-critical system parameters are monitored during runtime of the measurement. Through an analysis of these parameters, conclusions can be drawn about the area of the measurement where problems are being caused.

For each monitored parameter there is an internal system variable that is updated in a defined (parameter-specific) time interval. The system variables exist only as long as the Performance Monitoring function is switched on, so that the function itself does not negatively affect the performance. Individual parameters can be dependent on other settings and are therefore not necessarily visible immediately after the function is switched on.

All system variables of the Performance Monitoring function can be found in the CANoe namespace. Depending on whether a parameter mainly influences simulation or analysis, the namespace Simulation or Analysis is generated, in which the system variables are then defined.

MainQueueLoad

This parameter determines the relative load of the main queue on both sides of the communication.

Thus, if results are generated faster on the simulation area (real-time area) than can be forwarded, the load increases here, and if more events are received on analysis area than can be processed, the load on this side increases.

Note: These variables only get updated if a significant queue load is measured.

Simulation::TimerAccuracy

variable

TimerAccuracy

This parameter shows the accuracy of a cyclic timer, i.e. the deviation between its expiry interval in real and simulated time.

This can be used to detect realtime problems in some cases. For this purpose, the interval must be set appropriately in the Options dialog; please also note the notes on interpreting the data there.

The parameter is only updated in the Real Bus working mode.

FreeDiskSpace<Drive>

This parameter determines the free disk space for all drives that store logging data in the current configuration.

These include drives that record data for internal reasons.

For each of these drives, a system variable is generated with the corresponding drive letter as a suffix. Because specific logging configurations become active only at the start of the measurement, the system variables are also not generated until this point in time.

Simulation::RxEventUpperDelay

variable

Double

s

RxEventDelay

These parameters show the maximum and minimum delay of a message since the last update of these parameters. If no message has been received since the last update, 0 is displayed.

The delay of a message is measured from the reception of a message to its processing in the simulation process.

The update interval of these parameters can be set in the CAN.ini in the section [RxDelayMonitor] with the value SysVarRecurrence in milliseconds.

Simulation::RxEventLowerDelay

variable

Double

s

The maximum usage of the CPU is limited for thread-specific parameters because a thread can fully utilize a maximum of one processor core. In a system with 4 cores, for example, the variable CPUUsageNetworkThread cannot exceed 25%. The usage values of the individual threads add up to the usage value of the process (they are thus measured relative to the total usage and represent percentages of the usage of the process). If the sum of the usage of the threads is significantly less than the usage of the process, there must be another thread in this process that is generating an unusually high load. The same applies to the sum of the processes: when these are running on the same system, their load adds up to that of the total system provided no other processes with high load are running.

Offline Mode

In offline mode, the system variables do not display any currently measured values, but the recorded values present in the played log file. Therefore, if there are no values in the played data, no values will be displayed.

Measuring the values in offline mode is not useful because the data is played back there as quickly as possible.
