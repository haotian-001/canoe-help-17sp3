Load Indicator

 

Load Indicator

The load indicator (buffer fill status), located in the left corner of the status bar, can be used during the measurement to observe the loading of your system. It represents the ratio of arriving events to processed events (e.g. display in the Trace Window).

The length of the bar indicates the fill level of the queue which buffers the real-time area (e.g. CAN hardware) from the Measurement Setup (e.g. Trace, Logging).

If the ratio between arriving and processed events is relatively balanced, then the load indicator has a green color. However, if it is shown in red, significantly more events are arriving at that particular time than can be processed.

If the buffer fill level reaches an alarming limit, then this is a clear indication that the system is overloaded. This might even lead to loss of events.

By deactivating individual analysis windows an attempt can be made to relieve the system of loading. Please also refer to page Performance Problems.

Edit

If necessary you can disable the display of the buffer level with the help of the CAN.INI entry ShowMainQueue=0 in the [System] section.

Load Indicator Network

During distributed operation with a VN8900 device or an RT server the fill level of the queue on the RT server, that separates the real-time area from the network connection (USB or Ethernet), is shown under the normal load indicator.

When the fill level of this buffer rises this is an indicator that the events that are processed in the real-time area cannot be transferred quick enough on the network or the USB connection to the user computer.
