Critical Memory Shortage - Measurement Stopped

 

The measurement has been stopped due to a shortage of memory.

The virtual memory used by CANoe has a exceeded a critical limit. The measurement has been stopped to prevent irregular termination of the program in the event that additional memory would be needed but can no longer be made available by the operating system.

Save your configuration and restart CANoe if possible in order to prevent data loss.

CANoe contains mechanisms for preventing too much memory from being utilized during the measurement. However, these cannot anticipate every situation. For example, the saving of many frames of different sizes in the Data History can also lead to fragmentation in the Windows heap.

During the measurement, large amounts of memory may be used for the Data History and scope measurements or by CAPL programs in the Measurement Setup. You may therefore be able to avoid the situation in several ways:

  • Use the 64-bit variant of CANoe.
  • In the Data History settings, set the utilization of the hard disk either to No with short data history or Normal use.
  • Check CAPL programs in the Measurement Setup to determine whether they use very large associative fields, for example.
  • Reduce the data measured with Option Scope.
