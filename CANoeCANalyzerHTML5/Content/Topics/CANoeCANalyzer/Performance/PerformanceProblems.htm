Performance Problems - Causes and Remedies

 

It depends on the configuration of your computer whether active programs running in the background cause performance problems.

Some of these programs are not visible to the user. They can be reconstructed by using the Task Manager|Processes. You have to press <Ctrl> + <Alt> + <Del>. Also you should check your start-up folder.

For a detailed analysis of performance problems you can activate performance monitoring of critical system parameters in the global options.

If this does not lead to the desired results, you can resolve the following problems on your own:

Configuration options

Problem

Possible Causes

Remedies

Time management is affected

animated or 3-dimensional mouse pointer

Deactivate the mouse pointer in: Control Panel|Mouse|Pointer.

Running measurement is affected, data loss

Screen saver

Deactivate the screen saver in: Control Panel|Display|screen saver.

Power Management

Deactivate the Power Management.

Slow realtime measurement

Windows animation, which can be observed when windows are maximized and minimized.

Deactivate this effect, for example, with the tool TWEAK UI, which you can download from the Microsoft homepage.

With this tool you can deactivate the animation under: Effects|General.

Reduced performance

Microsoft find fast indexer

Deactivate the find fast indexer in the Control Panel and take the indexer off your start-up folder.

Virus scanner

Setup your Virus scanner so that it doesn't start during a measurement.

Working at High Busloads

Problem

Possible Causes

Remedies

Reduced performance

Unnecessary opened measurement windows and evaluation blocks

Disconnect all these measurement windows and evaluation blocks.

By inserting a break at a Hot Spot (by double clicking on the hotspot or from its context menu) all function blocks and windows that appear after the hotspot in the data flow plan are disconnected, i.e. the data flow is terminated there.

Under some circumstances this can enhance overall system performance.

Trace Window in chronological mode

Select via the toolbar the fixed position mode; see also: Trace Window performance

Unnecessary statistic protocol

Deactivate the unneeded protocol in the context menu of the Statistics Window.

Data Window

Select Configure timings in the context menu of the Data Window and activate Update screen only cyclically with a Cycle time of max. 500 ms.

Many signals are configured in the Graphics Window

Select Settings and the [Measure] button in the context menu of the Graphics Window and give in a user-defined refresh of 200 ms...2 s.

High amount of data flowing through the data flowchart

Use the Filters.

Use the early filtering before the analysis area as well.

Unnecessary bus statistics

Deactivate the Bus statistics of the card.

Use the acceptance filtering.

Insufficient performance for simulation

Slow programs

Optimize programming

High frequency data exchange with external simulation participants.

Exchange data less frequently, but in larger blocks if necessary.

Data not required for the test

Use acceptance filter.

Use partial residual bus simulation.

Many networks with high busload

Use Multi CANoe.

Above-mentioned reasons and solutions are presented more detailed on the side working at high busloads.

Replacement by CAPL, use ERT.

Use asynchronous programming or background tasks., optimize programming, use ERT.

Use e.g. PCI Express plugin cards, a VN8900 or a VT system.
