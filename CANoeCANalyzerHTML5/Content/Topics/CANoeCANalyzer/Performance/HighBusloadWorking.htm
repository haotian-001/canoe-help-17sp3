Working at High Busloads

 

An abridged version of the following tips can be read on help page Performance Problems – Causes and remedies.

With high busloads, the data supplied by the network hardware may no longer be displayed in time in the analysis windows. However, in many cases you can optimize the settings of the measurement windows to avoid this data overflow:

  • To free up more computing resources, deactivate all measurement windows that you do not need. To disable data flow into the measurement window, simply double click the left side of the Hot Spots before the particular analysis block. The block will then be shown in gray, and the window will be deactivated.
  • Use filters to reduce the amount of data. The sooner the filter takes effect, the more performance it brings.
  • For the Trace Windowfunctions with a high computing power can be deactivated.
  • If you have configured a large number of signals in the Data Window you should select the Cyclic Symbol mode ("Configure timings" command in the Data Window's context menu and enter a cycle time of maximum 500 milliseconds. The window is then only updated cyclically and thereby demands less computing power.
  • If you have configured a large number of signals in the Graphics Window you should select a large user-defined Refresh (200 ms to 2 s) in the Measurement Configuration dialog ("Options" item in the context menu). This defines how often the Graphics Window display should be updated. Small values result in a continuously flowing display of the signal response, but they place high demands on computer resources and may lead to performance problems with slower computers. Although higher values reduce demand on computer resources, they also lead to a more erratic and delayed display of the signal response. Please also refer to the tips on optimizing the Graphics Window.
  • Reduce the amount of data flowing through the data flowchart by filtering out uninteresting messages with Filters, as close to the data source as possible.
  • Disable the network hardware bus statistics if you don't need it.
  • Use the acceptance filtering of the hardware as far as possible. Messages that the hardware filters out by restrictive acceptance masks are not fed into Measurement Setup (settings: Network Hardware Configuration).
