Process Data Component

 

General

List of Functions

The process data component is part of the ISO11783 Interaction Layer (ISO11783 IL), and it is needed to manage the process data directory and communication with a Task Controller or a Data Logger. This functionality is constructed based on a Device Description, which is described in a special file (ISO11783-10).

Operating Principle
  1. For each node in the simulation, there is a definition right in the database of whether there is a file that contains a process data directory description, and if so which file. In addition, the node attributes ISO11783PDDFilename and ISO11783PDDVersion are used.
    Alternatively, the CAPL command Iso11783IL_PDDLoadDeviceDescription can be used at runtime to load a process data directory and transfer it to the Task Controller. But then no system variables are available for the individual elements.
  2. The ISO11783 IL automatically reads in the specified file, analyzes it and generates a node process data directory (PDD). Afterwards, the ISO11783 IL generates a set of system variables, each of which corresponds to an element from the node-PDD.
  3. The ISO11783 IL automatically detects the Task Controller or data logger after sending the status message, and it sends the read-in node-PDDs to the Task Controller or Data Logger.
  4. Each change to a system variable is automatically transmitted to the associated element in the node-PDD. Moreover, an element may be changed by the CAPL command Iso11783IL_PDDSetValue.
  5. The Task Controller can request the current value of an element from the node-PDD with a Request Value command. The ISO11783 IL sends the value with the Process Data Acknowledge message.
  6. The Task Controller can set the current value of an element from the node-PDD with a Set Value and Acknowledge command. It may also be possible to use the Set Value command depending on the version of the Task Controller. In the process, the ISO11783 IL updates the relevant system variable and simultaneously calls the CAPL callback Iso11783IL_PDDOnDataChanged. In addition, the current value of a process variable from the node PDD can be queried with the CAPL function Iso11783IL_PDDGetValue.
