Elements of the Process Data Dictionary and System Variables

 

To illustrate the linking of the process data directory with the generated system variables, a process data directory is shown below as it is displayed in the Task Controller. The system variables generated for this process data directory are also displayed.

Note

System variables are only created automatically if the file with the process data directory is defined in the database (attribute ISO11783PDDFilename) or in the Component Configuration dialog of the node.

For each DeviceProcessData element, a system variable is generated with the name of the DeviceProcessData designator (e.g. "ApplicationRate", "FlowRate", "TankVolume", etc.). No system variables are generated for elements whose values do not change.

All system variables that describe the DPD elements of a device element are generated in the same namespace. The designator for the namespace matches the DeviceElement designator (e.g. "Section1", "Section2", etc.).

If multiple device elements are allocated to another device element (e.g. "Section 1" through "Section 5" are allocated to the "Sprayer", while "Sprayer" and "Tank" are allocated to the "Sprayer Device"), then the relevant higher-level namespaces are generated for the higher-level device elements (e.g. the namespaces "Sprayer", "Tank" and "SprayerDevice").

In turn, all higher-level namespaces are assigned the designator "PDD". This is under the namespace with the designator of the node from the database ("Sprayer").

Note

In renaming the system variables and namespaces, all blank characters are removed from the relevant designators. For example, "Tank Volume" becomes "TankVolume", and "Section1 " becomes "Section1", etc.
