ISO11783 IL CAPL Functions

ISO11783 IL Behavior [▲ back]

The ISO11783 IL automatically registers on the network after measurement start with the Address-Claimed message.

If configured, the object pool and process files are read in and transmitted to a Virtual Terminal or a Task Controller. TIM components must first be configured more precisely in CAPL in order to read in certificates and keys and then start initialization and authentication with another TIM ECU.

If the ISO11783 IL is not to be started together with the measurement start of CANoe, you can call the function ISO11783IL_ControlInit in the event handler on preStart. The IL can then be started at a later time with the function ISO11782IL_ControlStart.

ISO11783 Set Implementation Version [▲ back]

The implementation versions of the individual components can be set as follows:

  • With the database attributes ISO11783PDDVersion and ISO11783IOPVersion. (Object pool component and process data component only).
  • With the Component Configuration dialog of the node in the Simulation Setup. (Object pool and process data component only).
  • with CAPL functions Iso11783IL_OPSetProperty, Iso11783IL_PDDSetParameter and Iso11783IL_TIMSetProperty

ISO11783 IL Activate Functions [▲ back]

Network Management

The ISO11783 IL automatically performs address claiming at start-up.

If the node attribute NmJ1939AAC is set to 0, the ISO11783 IL goes into the Stopped state and sends an address PG Cannot Claim if a conflict occurs in the address claiming procedure.

If the node attribute NmJ1939AAC is set to 1, ISO11783 IL goes into the state Stopped (3) in case of a conflict. If the CAPL callback function ISO11783IL_OnChangedState is defined, it is called with state=3. Within this function, you can calculate a new address and start address claiming with a new address by calling ISO11783IL_ControlStart(<new address>).

J1939 Request

The ISO11783 IL does not answer a J1939 request automatically (exception: Request for Address Claimed), not even with a negative acknowledgment if a message is not supported. Automatic replies can, however, be activated by assigning different messages to the Tx list of the node.

You can influence the communication by using the CAPL callback function Iso11783IL_OnTxPrepare. With this function you can suppress and modify the Parameter Groups that are to be sent.

Acknowledgment

For the ISO11783 IL to send the acknowledgment message (PGN E800h), it must be assigned to the node as Tx message. If the ISO11783 IL finds this message in the transmit list of the node but the requested message is not entered in the transmit list of the node, a request is answered with a negative acknowledgment.

Fault Injection [▲ back]

You can influence the behavior of the ISO11783 IL from test units or test modules in order to provoke errors. In a remaining bus simulation, you can generate communication errors to which the SUT (System Under Test) must react.

The description of the corresponding CAPL functions can be found in the CAPL Function Overview - Fault Injection.

Manipulation of Transport Protocols

You have the option of intervening in the flow of Transport Protocols. With a series of CAPL functions, you can specifically block the transmission of transport protocol messages or delay them as required. With the functions for blocking and delaying individual messages, you can specifically test simple scenarios. However, if the test scenario becomes more complex, these functions are no longer sufficient. CANoe therefore offers you the option of freezing transport protocols and continuing execution in order to inject your own manipulated TP messages in a targeted manner. The functions Iso11783IL_FreezeTp, Iso11783IL_UnfreezeTp and Iso11783IL_KillTp are available for this purpose.

ISO11783 Diagnostics [▲ back]

In order for the ISO11783 IL to send diagnostic messages automatically, first activate ISO11783 diagnostic support. Via CAPL you can activate or deactivate errors which are then communicated via the ISO11783 diagnostic messages (DM1 and DM2).

The description of the corresponding CAPL functions you find in the Iso11783IL CAPL Function Overview - DTC Support.

See also Testing Transport Protocols (J1939/ISO11783).
