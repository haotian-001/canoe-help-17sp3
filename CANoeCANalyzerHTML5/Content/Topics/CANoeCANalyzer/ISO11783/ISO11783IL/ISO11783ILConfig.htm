Setting up the ISO11783 Interaction Layer

 

Setting up the ISO11783 Interaction Layer

The ISO11783 Interaction Layer (ISO11783 IL) is configured via the database.

Follow the next steps to activate using the ISO1783 IL by a node.
  • Define a node in the database.
  • For this node set all NmJ1939xxx attributes for the designated device name.
  • Set value Iso11783_IL.vmodule to the node attribute NodeLayerModules.
  • Add additional send messages if required.
  • If the ISO11783 IL should communicate with a Virtual Terminal:
  • Configure the node as described here: Configuration
  • If the ISO11783 IL should communicate with a Task Controller or a Data Logger:
  • Configure the node as described here: Configuration
  • If the ISO11783 IL has to communicate with a TIM Server or TIM Client:
  • Configure the node as described here: Configuration (TIM Component - ISO11783)

ISO11783 Network Management

The ISO11783 IL automatically performs address claiming at start-up. For this purpose the IL uses the device name that is defined in the database with the NmJ1939xxx node attributes. More information about network management and behavior in the event of address conflicts can be found on the following page: J1939/ISO11783 Interaction Layer Network Management.

Using the ISO11783 Interaction Layer in vTESTstudio

The CAPL functions of the ISO11783 Interaction Layer are not available in vTESTstudio by default. To use it in vTESTstudio, you have to insert the ISO11783_IL.dll from the CANoe folder Exec32 as a modeling library in the configuration of the test unit.

It is a good practice to copy the ISO11783_IL.dll from the CANoe folder Exec32 to a subfolder of the vTESTstudio project. If the test unit is executed in CANoe, the ISO11783_IL.dll from CANoe is used.
