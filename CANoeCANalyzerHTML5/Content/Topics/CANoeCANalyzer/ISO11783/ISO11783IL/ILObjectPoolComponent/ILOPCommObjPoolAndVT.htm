Changing and Transmitting Object Values

 

Values of objects from the object pool can be changed either via the automatically generated system variables or via the object pool API and transferred to the Virtual Terminal.

Note

System variables are only created automatically if the object pool file is defined in the database (attribute ISO11783IOPFilename) or in the Component Configuration dialog of the node.

Changing Object Values Using System Variables

Some objects in the object pool that are used for user inputs or displaying values (e.g. output number, input number, number variable, etc.) have a special attribute value. For these objects, the ISO11783 IL offers various ways to change the values and react to changes.

A system variable OBX00XYZ_value is prepared for the attribute. Each change to the system variables is transmitted to the relevant object in the object pool of the virtual terminal with Change Numeric Value command or Change String Value command. Simultaneously, the CAPL callback Iso11783IL_OPOnValueChange is called.

Changing Object Values Using Object Pool API

In the CAPL program, the Iso11783IL_OPSetNumericValue or Iso11783IL_OPSetStringValue is called. This changes the new value in the object pool and sends it to the relevant system variable. The Change Numeric Value command or Change String Value command is used to send the new value to the virtual terminal. The CAPL callback Iso11783IL_OPOnValueChange is called simultaneously.

Reaction to User Input in the Virtual Terminal

If the node receives a user input in the virtual terminal via a VT Change Numeric Value message or VT Change String Value message, both the affected object from the node object pool and the relevant system variable (if generated automatically) are updated. At the same time, the CAPL callback Iso11783IL_OPOnValueChange is called.
