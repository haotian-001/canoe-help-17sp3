Changing and Transmitting Object Attributes

 

Attribute values of objects from the object pool can be changed either via the generated system variables or via the object pool API and transferred to the Virtual Terminal.

The ISO11783 specification, Part 6, defines two methods for sending a new attribute value from one simulated node to a virtual terminal.

  • By the Change Attribute Command method (VT function AFh)

This method can only be used for attributes for which an attribute ID is defined.

  • By a specialized instruction such as:
  • Change Background Color (VT function A7h)
  • Change Numeric Value (VT function A8h)
  • Change Active Mask (VT function ADh)
  • Change Soft Key Mask (VT function AEh)
  • Change String Value (VT function B3h) etc.

For some attributes, only one method may be used, while for others either method may be used.

System Variables

If only one method is allowed for an attribute, this method is used by the ISO11783 IL. If both methods are permitted, then the ISO11783 IL uses the specialized instruction and not the Change Attribute Command method.

Object Pool API

If an attribute has an Attrbute ID defined, you can change the attribute value using the CAPL function CAPL-Funktion Iso11783IL_OPSetAttribute.

There are also several CAPL functions available that start with Iso11783IL_OPChange... that allow you to change certain attributes.
