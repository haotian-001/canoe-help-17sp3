Auxiliary Control Support

 

The concept of the Auxiliary Control (consisting of auxiliary inputs and auxiliary functions) is a component of the ISO11783 specification and is described in Part 6, Appendix.

  • Auxiliary inputs describe various physical input elements, such as switches, sliders or joysticks. These elements are used to accept user inputs directly (i.e. without virtual terminal) and route them to one or more nodes. It makes sense to use Auxiliary Inputs if, for example, user control is not convenient enough via a virtual terminal or if, for ergonomic reasons, it needs to be implemented differently (e.g. compare a virtual joystick to a physical joystick).
  • Auxiliary functions are the counterparts to the specific auxiliary inputs.

Currently, 15 different types of auxiliary inputs and 15 types of auxiliary functions are defined. For each auxiliary input type there is one auxiliary function type.

Auxiliary Input Objects

Each assigned auxiliary input object transmits its state to the ISO11783 Interaction Layer via the automatically provided system variables (generated by ISO11783 IL). For some auxiliary input types (Type ID 0, 1, 2, 3 and 4), one system variable with the designation value1 is provided, for all others (Type ID 5 to 14), two system variables with the designations value1 and value2 are provided. The system variable value1 (and if available - also value2) should be set at runtime (e.g. via CAPL or in a panel). Based on the value of the system variable value1 (and if available - also value2) ISO11783 IL calculates values for signals Value1 and Value2 and transmits them via the Auxiliary Input Status message, which is sent cyclically once per second and additionally upon each change.

The proper data setting of the system variables and subsequent transformation in the signals depends on the corresponding auxiliary input type and is explained in detail below:

Auxiliary Function Objects

In the case of auxiliary function objects, different system variables are provided for different functions. The goal here is to prepare the information sent with the Auxiliary Input Status message as well as possible. The following table offers an overview of the various function type IDs (per ISO11783-6) and associated system variables.
