Simulate Implement with ISO11783 Interaction Layer

 

The ISO11783 Interaction Layer Implement (short ISO11783 IL) is a library for quick and easy modeling of ECUs on the ISOBUS. In addition to functions for signal-based access to the ISOBUS (similar to those of the J1939 Interaction Layer), the ISO11783 IL offers many ways to simulate the entire communication between a simulated ISO11783 ECU and a virtual terminal, Task Controller or data logger in a very simple way. The simulation of a TIM Server or TIM Client is also possible with a few CAPL code lines. In addition, the ISO11783 IL simplifies simulation of an auxiliary input ECU considerably.

Functionality

  • Signal-based access to the ISOBUS
  • Sending of periodic messages
  • Sending of messages when setting and changing a signal
  • Sending messages in response to a request (ISO 11783-3)
  • Support of Address Claiming (ISO 11783-5)
  • Support of transport protocols
  • Data Transfer (ISO 11783-3)
  • Extended Data Transfer (ISO11783-6)
  • Fast-Packet
  • Communication with a virtual terminal and one or more auxiliary ECUs (Object Pool component)
  • Automatic read-in of an object pool and sending to a virtual terminal
  • System variables, which are automatically generated from the object pool, offer a simplified interface both for communication between the simulated node and a virtual terminal and for interfacing to an auxiliary ECU or simulating it
  • Object Pool API offers additional ways to influence the communication between a simulated node and a VT
  • Communication with a Task Controller (Process Data component)
  • Automatic read-in of a process data directory and sending to the Task Controller or data logger
  • System variables that are automatically generated from the process data directory offer a simplified interface for communication between the simulated node and a Task Controller or data logger.
  • Process data API offers additional ways to influence the communication between a simulated node and a Task Controller or data logger.
  • Peer Control support: Peer Control commands are handled and after successful assignment setpoint messages are sent to the setpoint user resp. are received from the setpoint source.
  • File Server API offers a CAPL interface for simulation of a File Server
  • Communication with a TIM (Tractor Implement Management) server or TIM client (TIM Component)
  • Simulate TIM client or TIM server (or both)
  • Manufacturer specific certificates can be used
  • Support of fault injection
