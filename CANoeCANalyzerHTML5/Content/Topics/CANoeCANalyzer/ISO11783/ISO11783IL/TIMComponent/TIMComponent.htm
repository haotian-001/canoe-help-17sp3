TIM Component

 

General

Function List

The TIM (Tractor Implement Management) component is a part of the ISO11783 Interaction Layer (ISO11783 IL) and is needed to implement communication between a TIM server and a TIM client or to test a TIM server or client. If the demonstration mode of the TIM component is not active, different certificates and an additional authentication DLL (AuthLib.dll) are required for the communication, which are not delivered with CANoe.

Setting Up a TIM Client

Setting Up a TIM Server

Follow the next steps to configure a database node for a TIM client:

  • Define a network node in the database.
  • For this node set all NmJ1939xxx attributes for the designated device name.
  • Set node attribute NodeLayerModules to value Iso11783_IL.dll.
  • Add following messages to the Tx list of the node:
  • Address Claimed (PGN 0xEE00)
  • TIM21 (PGN 0x2400)
  • AUTH21 (PGN 0x6F00) )
  • ISOBUS Compliance Certification (ICC, PGN 0xFD42)
  • Product Identification Information (PII, PGN 0xFC8D)
  • Add a new network node to the Simulation Setup in CANoe, open the configuration dialog of the network node and choose the TIM client node of the database.
  • Assign a CAPL file to the network node.

After configuration of the TIM client you can connect the TIM client to a TIM server with the function Iso11783IL_TIMConnectToServer.

Follow the next steps to configure a database node for a TIM server:

  • Define a network node in the database.
  • For this node set all NmJ1939xxx attributes for the designated device name.
  • Set node attribute NodeLayerModules to value Iso11783_IL.dll.
  • Add following messages to the Tx list of the node:
  • Address Claimed (PGN 0xEE00)
  • TIM12 (PGN 0x2300)
  • AUTH12 (PGN 0x7000)
  • ISOBUS Compliance Certification (ICC, PGN 0xFD42)
  • Product Identification Information (PII, PGN 0xFC8D)
  • Add a new network node to the Simulation Setup in CANoe, open the configuration dialog of the network node and choose the TIM server node of the database.
  • Assign a CAPL file to the network node.

After configuration of the TIM server you can start the TIM server with the function Iso11783IL_TIMActivateServer.

Authentication Library

To use the TIM client or TIM server without the demonstration mode (which can be activated / deactivated by Iso11783IL_TIMSetProperty) you must get the appropriate Authentication Library DLL. Rename the file to AuthLib.dll and copy it to the Exec32 folder of CANoe.

Test Support

Using the TIM CAPL Function within a Test

For all TIM CAPL functions there is a variant which can be used in a CAPL test file (e.g. in a CAPL test module or in vTESTstudio). For every TIM CAPL function there is an additional first parameter of type dbNode which defines the TIM node.

Using the ISO11783 Interaction Layer in vTESTstudio

Be default the CAPL functions of the ISO11783 Interaction Layer are not available in vTESTstudio. For use in vTESTstudio you must add the ISO11783_IL.dll from the exec32 folder of CANoe as a modeling library in the test unit configuration.

It is good practice to copy the ISO11783_IL.dll from exec32 folder of CANoe and store it in a subfolder of the vTESTstudio project. When the test unit is executed in CANoe then the ISO11783_IL.dll of CANoe is used.
