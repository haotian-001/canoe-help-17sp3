ISO11783 Node Layer

 

The ISO11783 Node Layer simplifies the simulation of a ISO11783 node with CAPL. The ISO11783 Node Layer offers the following functionality:

  • Address Claiming
  • Node table
  • Sending and reception of Parameter Groups
  • Transport protocols
  • Is used by generated CAPL programs

Furthermore the following APIs are available:

  • Object Pool API: Manages the communication with a Virtual Terminal and transmits an object pool file (IOP) automatically to the Virtual Terminal.

  • Process data API: Administers a device description file (ISO11783-10) and creates a process data dictionary. This is automatically transmitted to the Task Controller.

Cross Reference

The sample ISO11783SystemDemo shows the use of the ISO11783 Node Layer.

Include the Node Layer

To use the communication node layer with a node in the Simulation Setup, it must be included. The module description file ISO11783DLL.vmodule must be assigned to the node attribute NodeLayerModules.

Using the Node Layer

The following example shows how the ISO11783 node layer is used in a CAPL program:

Example

dword ecuHandle;
char  deviceName[] = { 0x01, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00 };

ecuHandle = Iso11783CreateECU( "default", deviceName );
Iso11783ECUGoOnline( ecuHandle, 0 );
// ...

char data[8];
// send parameter group
Iso11783TxReqPG( ecuHandle , 0xF001, 0xFF, 4, 8, data );
// ...

Iso11783DestroyECU( ecuHandle );
