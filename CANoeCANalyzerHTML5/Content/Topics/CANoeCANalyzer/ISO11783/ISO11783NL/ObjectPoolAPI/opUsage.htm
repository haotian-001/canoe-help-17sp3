Object Pool API - Usage

 

The initialization is done in CAPL before or after Iso11783ECUGoOnline. First an object pool filed must be loaded and activated. Vice versa is also possible, first activate, the load. The following example show the necessary CAPL calls to connect to the Virtual Terminal and transmit the object pool.

Example

handle = Iso11783CreateECU( gECUName );
Iso11783ECUGoOnline(handle, gECUAddress );

Iso11783OPLoad( handle, "ObjectPool.iop" );
Iso11783OPActivate( handle, "pool001" );

After loading the object pool file, the object pool must be activated with the CAPL function Iso11783OPActivate. Then the Object Pool API waits for a status message from the Virtual Terminal. After that the initialization procedure starts and the object pool is transmitted to the Virtual Terminal. As long as the Object Pool API is active the maintenance message is sent. If the connection to the Virtual Terminal is lost, the Object Pool API waits for a status message again.

Environment variables can be connected to objects (variable, input or output objects) in the object pool. The environment variables must be defined in the database and must have a appropriate type.

Example

Iso11783OPConnectEnvVar( handle, 1000, EnvVar_Value );

A change on a environment variable triggers a Change Value command with the value of the environment variable. If a Change Value Response is received, the new value is put to the environment variable.

CAPL functions for sending command to the Virtual Terminal are available. The commands a executed asynchronous, this means the Parameter Group is sent and the CAPL function returns immediately. On reception of the respond a callback function is called.

Virtual Terminal Version 3 Support

The Object Pool API supports the Virtual Terminal version 4 (ISO11783-6). Due the Virtual Terminal version 4 is still in process and can be changed in future, the default setting the Object Pool API is Virtual Terminal version 3. To activate Virtual Terminal version 4 support the following CAPL function must be called:

Example

Iso11783OPSetProperty( handle, "Version", 4 );
