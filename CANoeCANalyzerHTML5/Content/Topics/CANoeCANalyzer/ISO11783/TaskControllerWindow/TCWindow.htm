Task Controller Window

 

The Task Controller Window provides an operational Task Controller instance that simulates the ISO11783 communication of a real Task Controller without programming. The TC Window independently logs on to the bus, sends its status message, responds to requests sent by add-on and diagnostic devices, and receives all uploaded device descriptions (DDOPs). From the loaded device descriptions (DDOP), the TC window calculates the process variable directory for each connected attachment and displays it clearly. This provides you with a graphical interface with which you can read, change and query all process variables (including those relevant to Section Control).In addition, the TC window offers a convenient way to analyze the time history of individual selected process variables.

Inserting the Task Controller Window [▲ back]

To add a Task Controller Window to the Simulation Setup, open the context menu of the channel in the Simulation Setup by clicking on the bus with the right mouse button. Then click on the ISO11783 Insert Task Controller context menu command. This adds a node to the Simulation Setup. Double click on the node to open the Task Controller window.

Devices [▲ back]

This view displays all uploaded device descriptions. Each device description is displayed in its own tab. Process variables are indicated by a blue rectangle, while properties are indicated by a green rectangle.

There are two ways to collapse the number of items displayed:

  • Use the and buttons in the toolbar at the top to show and hide all process variables and properties.

  • You can also use column filters: When you move the mouse pointer over a table heading, a filter icon appears in the upper right corner.

Each process variable has a Set button and a Get button.

Using the Get button, request the current value of a process variable via the Request Value Command.

Use the Set button to set a process variable to a new value. Setting is done either by the Value Command or by the Set Value and Acknowledge Command - depending on the setting in the toolbar, which you can access via the buttons and . The arrow color of the button will adjust accordingly:

  • blue when using Value Command.
  • green when using Set Value and Acknowledge Command.

If the corresponding process variable is defined as read-only, the arrow color of the button turns gray. However, the button remains active so that you can check how the attachment responds to the change of a process variable read-only.

The toolbar also contains other buttons:

  • With the task is started. The status message is sent with StatusTotalsActive=1.

  • With the task is stopped. The status message is sent with StatusTotalsActive=0.

  • Using all stored device descriptions are deleted.

Process Variables for Section Control (Condensed State) [▲ back]

For process variables that map the Condensed Work State, there is a special graphical representation that shows the state of each section by a separate rectangle:

The colors have the following meaning:

  • Yellow: deactivated/off (0)
  • Green: activated/on (1)
  • Red: error (2)
  • Gray: undefined/not installed (3)

You can change the status of each section by clicking on the corresponding rectangle. The order is ascending with the left mouse button (gray -> yellow -> green -> red -> gray) and descending with the right mouse button (gray -> red -> green -> yellow -> gray).

Process Data Tables [▲ back]

Sometimes you need to analyze the time history of one or a few process data variables. For this task, the Task Controller Window provides the specialized Process Data Tables view.

The view is divided into two parts:

  • In the upper collapsible part you can see all device descriptions with their process data variables.

  • In the lower area you can create one or more tables and assign individual process variables to these tables via drag and drop.

Procedure: Drag a row with the process variable you want to analyze from the upper to the lower part with the mouse until the lower part is colored light green, and release the mouse button there:

During the measurement, a new row is inserted in the lower part each time one of the configured process variables is updated. The cell with the updated process variable will be highlighted in blue:

Configuration [▲ back]

In the Configuration view, you can set parameters that affect the behavior of the TC window. They are divided into the following groups:

Group

Parameters

Configuration

Supported TC version and other data submitted with the Version message.

Device Name

Here you can specify whether the Task Controller Window should either connect to a node from the Simulation Setup or whether the Task Controller Window should log on with the device name configurable here after J1939 standard.

If you want to use a node from the Simulation Setup, it must use the Task Controller Interaction Layer (TC IL). You can then configure the Task Controller individually via the CAPL commands of the TC IL.

Language

This is where you specify whether the language setting should be set is set by the primary Virtual Terminal. You also specify which data is transmitted with the Language Command message. Changing these properties in the active state causes the message to be sent immediately.

Passive Mode

Enable and configure passive mode.

Passive Mode [▲ back]

In passive mode, the ITC does not actively participate in the bus communication, but only listens to the messages on the bus and reconstructs from them everything that happens with the real, active Task Controller:

  • uploaded device descriptions
  • content of process variables
  • status of the task

Device descriptions uploaded to the real Task Controller can be analyzed and exported at any time - even after a communication failure or error in the real Task Controller.

This makes the Interactive Task Controller a true mirror of the real Task Controller.
