Interactive Task Controller

 

The Interactive Task Controller can be used to access the process data dictionary of an implement. The process variables can be set or read out. The process variables are selected symbolically after the device description has been transferred by the implement. When a task is started, the process variables of the default treatment zone are transmitted to the implement and the peer control assignments are set.

The window of the Interactive Task Controller is divided in two parts. The left side shows an overview of the available information. The device for which a device description is available are listed here. The view on the right side depends on the selected item.

There are several views available:

  • Devices overview
  • Device view
  • Task overview
  • Task view
  • Configuration

The expandable information view Information below shows the messages issued by the Interactive Task Controller. When selecting a line in the list and pressing <F1> the help will show a detailed description. When the measurement is stopped the windows synchronization is activated with the button [ ]. The marker is set to that instant of time at which the current selected message occurred.

The status bar of the window shows the operating mode of the Interactive Task Controller and allows the change of the mode.The defined waiting time (6 ms) after the Address Claim is displayed in a bar graph.

Operating Modes

The Virtual Terminal can be in the following operating modes:

  • Offline: The Interactive Task Controller is not active, it can be activated and deactivated during the measurement.
  • Online:The Interactive Task Controller is active and sends status messages.
  • Passive: The Interactive Task Controller is in passive mode.

The right mouse button opens a context menu which allows the activation and deactivation of the Interactive Task Controller.

Passive Mode

In this mode the Interactive Task Controller monitors the bus traffic, no messages are sent. After measurement start it waits for a status message from a real Task Controller. The passive mode can be activated and configured on page Configuration.

The Interactive Task Controller displays the same device descriptions that are sent to the real Task Controller.

Note

The Interactive Task Controller must be active during the initialization sequence of the real Task Controller so that the transmission of the device description can be analyzed. The passive Task Controller displayed the same value changes of the process data as the real Task Controller.

If a task file is specified in the settings the process data dictionary is stored in passive mode too. It is available if the configuration is loaded again.
