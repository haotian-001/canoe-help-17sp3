Configuration of the Interactive Task Controller

 

You can adjust various settings on page Configuration of the Interactive Task Controller.

Properties of the Interactive Task Controller

Task Controller version

Here you can select if the Interactive Task Controller is a version 1, 2, 3 or 4 Task Controller.

Task Controller Capability

Support Version Request

If you activate this option, the response for the Version Request message is enabled.

If you deactivate the option, a negative acknowledge is sent.

Support Structure Label Request

If you activate this option, the response for the Structure Label Request message is enabled. The implement can activate a device description stored in the Task Controller without anew transmission.

If you deactivate the option, a negative acknowledge is sent. The device description must sent to the Task Controller again.

Support Localization Label Request

If you activate this option, the response for the Localization Label Request message is enabled.

If you deactivate the option, a negative acknowledge is sent.

Boot time

Here you set the maximum number of seconds until the first Task Controller status message

A value of 255 (default value) means that this information is not available. This value is used in the Version Message if the Task Controller version is 3 or higher.

Supports TC-BAS, TC-GEO, peer control assignment, TC-SC

These options are used in the Version Message if the Task Controller version is 3 or higher.

Number of booms

Maximum number of section control booms that is supported. This value is used in the Version Message if the Task Controller version is 3 or higher.

Number of sections

Maximum number of sections that is supported. This value is used in the Version Message if the Task Controller version is 3 or higher.

Number of control channels

Maximum number of individual control channels that is supported. This value is used in the Version Message if the Task Controller version is 3 or higher.

Process Data Values

Use Value Command to set values

If you select this option the Interactive Task Controller uses the command Value Command to write new values to the process data dictionary of the implement in the device view. The successful execution of the command cannot be verified due the implement sends no response.

Use Set Value and Acknowledge Command to set values

If you select this option the Interactive Task Controller uses the command Set Value and Acknowledge Command to write new values to the process data dictionary of the implement in the device view. The response of the implement shows if the write access was successful or not.

This option is only enabled of the Task Controller version is 4.

Mode

Enable passive mode

The passive mode is enabled and the Interactive Task Controller monitors the bus traffic only.

If the two following settings are not used, the Interactive Task Controller with the address defined at Device Name is monitored and displayed.

Auto detect address of the Task Controller

The Interactive Task Controller that registers first on the bus with an "Address Claimed" message is monitored and displayed.

Auto detected Task Controller must have Function Instance

The Interactive Task Controller that has the same Function Instance as defined here is monitored and displayed.

View

Displayed items in Process Data Dictionary

Device Properties

Here you define whether the properties of a device are displayed in the Device view.

Device Process Data

Here you define which process data of a device are displayed in the Device view.

Network Management

Address

Here you set the address that is used by the Interactive Task Controller.

[Device Name...]

If you click this button a dialog is opened to configure the device name. This name is used by the Interactive Device Controller.

Settings

Use Task File (Settings matrix)

Use Task File

Select the file with the device description that is read by the Interactive Task Controller or in which a device description is written.

Depending on the activation of this option, the next option is named Load Task File or Reset task data.

Load Task File

Here you specify when the selected task file is load.

There are several options:

  • On configuration loading only (default)
  • On configuration loading and on each measurement start: The Interactive Task Controller is reset to its default state on each measurement start. Existing device descriptions are discarded and the selected task file is reload.

Reset task data

Here you specify when existing task data are reset.

There are several options:

  • On configuration loading only (default)
  • On configuration loading and on each measurement start: The Interactive Task Controller is reset to its default state on each measurement start. Existing device descriptions are discarded.

Store data to Task File

Here you specify when the device description of the Interactive Task Controller is stored to the selected task file.

There are several options:

  • On configuration saving only (default): The device description is stored during storage of the CANoe configuration.
  • On each measurement stop: The selected task file is overwritten with the device description of the Interactive Task Controller on each measurement stop.
  • Never: The selected task file is never overwritten with new device descriptions.

Object Pool Transfer

Available space for object pool in bytes

Here you set the size of the available memory for the object pool. This value influences the status field in the Request Object-pool Transfer Response message of the Interactive Task Controller.

Language

These settings can be queried from other devices with the Language command and is available since Task Controller version 3.

Get language from Primary Virtual Terminal

If you select this option, the language setting will be queried on measurement start by the primary Virtual Terminal. If the query fails, the user settings will be used.

User setting

If you select this option, you can change e.g. language, format, and unit settings.
