File Server Window

 

The File Server Window provides a ready-to-use file server instance without any additional CAPL programming. The window displays connected clients, opened files and read/write operations as well as a log file with detailed information about the communication.

Insert the File Server Window [▲ back]

To add a file server window to the Simulation Setup, open the context menu of the channel in the Simulation Setup with a right click on the bus. Then click on the context menu command Insert ISO11783 File Server. This adds a node to the Simulation Setup. With a double click on the node to open the File Server Window.

Layout of the File Server Window [▲ back]

The file server window is divided into two areas. On the left side you will find the three headings Configuration, Clients and Information, via which you can display the respective pages on the right. The selected element is highlighted in blue.

You will also find the Connection button with a plug icon at the bottom left. The symbol indicates whether the interaction layer is switched on or off. You can toggle the two states by clicking the button.

Configuration

Properties:

  • Version
    The version of the File Server (1-4).
  • Root directory
    The root directory containing all folders and files to be accessed by the server. The path is relative to the CANoe configuration.
  • Primary volume
    The primary volume is always the first connected removable media, i.e. the current directory of a newly connected client. The path is relative to the Root Directory.
  • Supports physical removable device
    Indicates if the file server supports removable devices.
  • Supports long filenames
    Indicates if the file server supports filenames up to 254 characters.
  • Empty root directory at measurement stop
    When checked the Root Directory will be deleted at measurement stop.

Device Name:

In this area you can configure a device name according to J1939 standard. The device name can either be taken from the database from an existing node, or configured manually. If a node is selected from the database, it must also be added in the Simulation Setup so that the window can link to the node.

Clients

This page is divided into three areas:

At the top there is a table that shows connected clients. You can customize the columns by a right click on one of the headers. For a client to be included in the list, it must have sent at least one message to the file server. The properties of the respective client reflect the view of the file server. I.e. only when the client has sent a request and the file server has sent the corresponding response, the property is changed.

In the center, additional properties are displayed as well as files that the client has accessed since the start of the measurement.

At the bottom is the output of the selected client in tabular form. You can also customize these columns with the context menu.

Information

General output of the File Server Interaction Layer and the File Server Window.

Operation Modes [▲ back]

The File Server Window supports the following operation modes:

  • Offline
    The File Server is not active but can be activated or deactivated during measurement.
  • Online
    The File Server is active and handles client requests.
  • Passive
    The File Server is in passive mode.

Passive Mode [▲ back]

In passive mode, the File Server Window observes the bus traffic and reflects the state of a real File Server in the network based on the messages it receives. A File Server Window in passive mode does not send any messages. After the start of measurement, the File Server Window expects a File Server status message from the real File Server and takes over the address for observation.

The passive mode is activated on the Configuration page in the Passive Mode area. All File Server settings will be invalidated and deactivated except for the version. Make sure that the version is the same as that of the File Server you are watching, otherwise some messages may not be interpreted correctly. If the File Server Window detects a version conflict, it issues a warning.

Note

The Window in passive mode does not know the names of the directories (such as the Primary Volume) at the beginning of the measurement. For this reason, the Primary Volume is displayed with the placeholder [Volume]. The Manufacturer Specific Directory can also only be derived from the received device names at the beginning. However, if a client executes a Get Current Directory request in these directories, the placeholders are replaced by the real paths.

Files in Passive Mode

Compared to the active mode, the File Server Window in the passive mode has no possibilities to access files. Therefore, the contents of the files cannot be displayed as soon as they are opened. Rather, the content is assembled from known information. This results in a few special features:

  • If a file is opened with the option Append data at end of file, the previously known content is removed, because this option does not give any information about the cursor position within the file.
  • If a file is opened and the file cursor is moved with Seek File and then written to or read from the file, the unknown bytes are initialized with 0x00. They are later replaced by the real data when the client reads or writes the corresponding bytes.
  • It is theoretically possible for a client to navigate and open files using only relative path specifications. However, if the client subsequently deletes such a file using an absolute path specification, the File Server Window cannot uniquely identify the file and consequently cannot mark it as deleted. The same applies to read and write accesses. In these cases, additional file is displayed.

Interaction with the File Server Interaction Layer [▲ back]

If the File Server is also to be influenced via a CAPL program, the File Server Window can be coupled with a simulation node that uses the File Serverl Interaction Layer (FS IL). Thus, for example, the file server can be controlled with the help of a test unit. If required, the operation can also be done via the user interface.

You can establish the coupling of the File Server with a simulation node via the following steps:

  1. Select the Configuration page on the left side of the window.
  2. In the Device Name area select the Use Node from database option.
  3. 3) Select the node from the Node list. (Node used by the corresponding simulation node).
    Note: Make sure that the selected database node is part of the Simulation Setup. Otherwise the File Server Window cannot connect to the selected node.

The File Server Window, which is coupled to a simulation node in this way, configures the corresponding simulation node: All parameters set on the Configuration page of the File Server Window are made known to the simulation node. This happens before each measurement start, and before the CAPL methods on preStart and on start are called. These settings can be overwritten for the simulation node using the FSIL_SetNodeProperty method, which is then called in on preStart or on start, for example.
