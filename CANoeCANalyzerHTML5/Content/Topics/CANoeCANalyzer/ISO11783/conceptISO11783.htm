ISO11783 for CANoe | Concept and Functions

 

Overview

CANoe.ISO11783 supports the basic concepts of CANoe and extends them with ISO11783 specific functions.

Analysis

Option ISO11783 supports and extends the basic analysis of CANoe.

The existing Trace Window extended such way that ISO11783 specific data can be interpreted and checked for protocol violations. Map Window and GNSS Monitor allow the numeric and graphical display of GPS data.

You can also use the Virtual Terminal Window and the Interactive Task Controller, which are able to analyze object pool data and process data without affecting network communication (passive mode).

Simulation and Stimulation

Option ISO11783 allows to simulate ISO11783 nodes (ECUs).

The ISO11783 Interaction Layer enables you to realize a complex simulation of an ISOBUS implementation including the communication to a Virtual Terminal, to a Task Controller, to a File Server and/or to a TIM Server/TIM Client with little programming effort.

Further specialized interaction layers allow you to simulate a Virtual Terminal, a Task Controller or a File Server.

You can also use the Virtual Terminal Window and the Interactive Task Controller, each of which provides you with a ready-to-use Virtual Terminal or Task Controller without any programming effort.

In addition, examples of a tractor ECU (TECU), a File Server and a TIM Sever/TIM Client are part of the option .ISO11783 and can be used for the remaining bus simulation.

GNSS receivers can be simulated with pre-built models, protocol files or CAPL. Depending on the requirements, the programming of CAPL source code can be completely dispensed with.

Test

The ISO11783 option extends CANoe, so that ISOBUS devices can be easily tested.

In addition to J1939-specific test functions and checks, there is also ISO11783-specific support in test units and test modules.

For testing elements, you can control other elements, the Virtual Terminal, or the Task Controller from within the test and check the reactions of the implement. You can export object pools and take screenshots of data masks and softkeys to compare them with reference data.

You can test a Virtual Terminal, Task Controller, TIM Server/Client or File Server (Client) with one or more simulated implements by controlling and monitoring the communication from within the test.

Data that is delivered to a simulated file server is stored on the Windows file system and can thus be analyzed.
