Segmented Transmission of a Device Description

 

Implements can transmit the device descriptions to a Task Controller or Data Logger in segments.

To allow this procedure in CANoe, flags can be added to the device description file (XML). These flags have to be a XML comment with syntax <!--canoe:split-->. If the device description file is used in a CAPL function of the Process Data API or in a database, the individual segments are transmitted and only after the last segmented the object pool will be activated.

Example

<?xml version="1.0" encoding="UTF-8"?>

<ISO11783_TaskData VersionMajor="4" VersionMinor="0" ManagementSoftwareManufacturer="Vector Informatik" ManagementSoftwareVersion="1.0" TaskControllerManufacturer="Vector Informatik" TaskControllerVersion="0.12.4" DataTransferOrigin="0">

    <DET A="DET1" B="1" C="1" D="ImplA Device" E="0" F="DVC1">
      <DOR A="5"/>
      <DOR A="6"/>
      <DOR A="7"/>
      <DOR A="8"/>
      <DOR A="9"/>
    </DET>
<!--canoe:split-->
    <DET A="DET2" B="2" C="4" D="Section1" E="1" F="DET1">
      <DOR A="10"/>
    </DET>
    <DET A="DET3" B="3" C="4" D="Section2" E="2" F="DET1">
      <DOR A="10"/>
    </DET>
    <DET A="DET4" B="4" C="4" D="Section3" E="3" F="DET1">
      <DOR A="10"/>
      <DOR A="13"/>
    </DET>
<!--canoe:split-->
