Configuration

 

On this page you can edit the settings of the Virtual Terminal.

Properties of the Virtual Terminal

VT version

Here you can select the VT version used by the Virtual Terminal.

Error handling

The Virtual Terminal has a mechanism to handle errors in the object pool (i.e. references to unknown objects). How the Virtual Terminal should react in error situations can be set with this list.

  • Delete Object Pool on Error: In case of error, the object pool is deleted from the Virtual Terminal and an error message is displayed. This is equivalent to the behavior of a real Virtual Terminal.
  • Deactivate Object Pool on Error: In case of an error, the object pool is deactivated, and the user can no longer operate the object pool. On the ISOBUS, it looks as if the object pool has been deleted. However, the object pool can be selected and is viewable. This setting can be used for later analysis.
  • Tolerant error checking: In case of errors, the object pool is deactivated (as in Deactivate Object Pool on Error). For minor errors (e.g. an unsupported color), the Virtual Terminal tries to display the mask and buttons as well as possible. You can use this setting to check the appearance of the objects even if there are some errors.

Error messages are displayed in the information view at the bottom of the Virtual Terminal Window.

Passive Mode

Enable passive mode

The passive mode is enabled and the Virtual Terminal only monitors the bus traffic.

If the two following settings are not used, the Virtual Terminal is monitored and displayed that has the same address as defined at Device Name.

Auto detect address of the VT

The Virtual Terminal that registers first on the bus with an Address Claimed message is monitored and displayed.

Auto detected VT must have Function Instance

The Virtual Terminal that has the same Function Instance as defined here is monitored and displayed.

ISOBUS Shortcut Button

Enable ISB Server

The ISB server support is switched on. The Virtual Terminal cyclically sends the message All Implements Stop Operations Switch State, which can be observed by ISB clients, and the ISB-Button is activated.

Hardware

Transmit pointing event

The command Pointing event is transmitted at a click in the data mask.

Report drag operation

The command Pointing event is cyclic transmitted while holding the left mouse button pressed.

Intermediate coordinates during a drag operation

The command Pointing event transmits the current coordinates while dragging. If the control field is not activated, only the start coordinates of the dragging process are transmitted.

Simultaneous activations of physical soft keys

If <Ctrl> is holding pressed, several soft keys can be activated at the same time.

Boot time

Maximum number of seconds from a power cycle to transmission of first VT status message

A value of 255 (which is the default for VT version 3 or less) means that this information is not available. This value is only used in the Get Hardware response.

Available memory

If the object pool size in a Get Memory message of an implement is bigger than this value then the Get Memory response of the Virtual Terminal contains the status value 1 (There is not enough memory available). Otherwise the status value is 0 (There can be enough memory).

Mask Settings

Colors

Number of colors which are supported by the Virtual Terminal

Data Mask size

Width and height of the square data mask in pixel

Soft Key Mask layout

Here you can specify the position of the soft keys relative to the data mask.

Soft Key Mask width

Width of a soft key in pixel

Soft Key Mask height

Height of a soft key in pixel

Number of physical / virtual Soft Keys

Number of physical and virtual Soft Keys per Soft Key Mask. Since VT version 4 the number of virtual Soft Keys is always 64 and cannot be modified.

Number of visible Working Sets

Sets the number of Working Sets visible at the same time. This setting is only available with Virtual Terminal version 4 or higher.

Number of User-Layout Data Masks

Sets the number of User-Layout data masks. This setting is only available with Virtual Terminal version 4 or higher.

Number of User-Layout Soft Keys

Sets the number of User-Layout soft keys. This setting is only available with Virtual Terminal version 4 or higher.

Supported Font Properties

Here you can specify the text font properties which are supported by the Virtual Terminal. These settings can be queried from other devices with the Get Text Font Data command.

Note

The property Proportional font rendering is only supported if the version of the Virtual Terminal is bigger than 2.

Language

These settings can be queried from other devices with the Language command.

Get language from Tractor ECU on startup

If this option is selected, the language setting will be queried when the tractor ECU measurement starts.

User setting

If this option is selected, e.g. the language settings can be changed by the user.

Only display languages supported by active Working Sets

If this option is selected, only the languages defined in the loaded object pools are displayed in the Language list. However, you can always enter a language manually in the Language input box.

Device Name

Use node from database

If this option is selected, you can select a node of the database to get the device name and the address.

If the selected node uses the Virtual Terminal Interaction Layer (VT IL), the Virtual Terminal can be operated both through the CAPL program of the node and through the interface of the Virtual Terminal Window.

Set Device Name and address manually

If this option is selected, you have to specify the device name and the address manually.

If the check box Dynamic is enabled, the Virtual Terminal checks before registering to the network, if the selected Function Instance (part of the device name) is already used by another Virtual Terminal before. In that case the Virtual Terminal changes its Function Instance to an unused value.

Enable Name Management

If this check box is selected, the Name Management is enabled. Other nodes can change the device name of the Virtual Terminal by sending a Name Management message during measurement.

Font & ScreenTips

Proprietary font type

Windows code page which is used if the proprietary font type (255) is selected.

Font

Text in the Virtual Terminal is rendered with the selected font (default: Courier New).

Enable ScreenTips

Enables support of tooltips.

Show object attributes

Shows not only the index and the object type but also the attributes of the object.

Show object frame

Shows the bounding rectangle of the object.

Version Management

Store object pool versions in this directory

In this directory the Virtual Terminal stores object pool versions. Path is relative relating to the folder of the CANoe configuration.

Remove all stored versions

Click this button to remove all stored object pools.
