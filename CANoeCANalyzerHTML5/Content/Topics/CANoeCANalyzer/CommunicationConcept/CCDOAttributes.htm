Predefined Attributes for Distibuted Objects

 

The following tables provide an overview of all predefined attributes for distributed objects. You cannot change the values of these attributes during the measurement. The symbol ✔ in the Interface column means that the attribute can only be set at interface level, which means that it cannot be overwritten by individual instances of distributed objects.

Note that an attribute ultimately always parameterizes one endpoint. If a value is set for the attribute on a superordinate object, this value is inherited by the end point, but can also be overwritten.

Access from C#

In C# you can get the current attribute value of a distributed object member through the static methods at the Vector.CANoe.Runtime.AttributeAccess class. To get the value, you have to provide the member instance, the string path or definition of the attribute and an out-variable for the value to be written in. For string-type attributes, you must additionally specify with a boolean value whether placeholders should be replaced (true) or not (false).

If an error occurs, the methods throw an Vector.CANoe.Runtime.AttributeValueException.

vCDL

namespace myNamespace
{
  //[Binding="Abstract"]
  interface ITF
  {
    [Binding="C#", CommunicationPattern="PublishSubscribe", AutoAnnounce="Never", AutoConnect=false]
    provided data int32 mData;
  }
  ITF DO;
  reverse<ITF> RevDO;
}

C#

public void AttributeDataMembers()
{
  // The attribute definition can be found in the according namespace at the attribute class
  // For string attributes, you must provide a Boolean value, whether you want placeholders to be replaced (true) or not (false) AttributeAccess.GetAttributeValue(DO.mData, _SystemAttributes.Binding.Definition, out string bindingValue, false);
  if(bindingValue=="C#")
    WriteLine("Binding Value: " + bindingValue);

  // Integers are cast automatically if possible
  AttributeAccess.GetAttributeValue(DO.mData, _SystemAttributes.CommunicationPattern.AttributeDefinitionPath, out int commpatValue);
  if(commpatValue == 1)
    WriteLine("Communication Pattern Value: " + commpatValue.ToString());

  // Enumeration values can be found at the according attribute class
  AttributeAccess.GetAttributeValue(DO.mData, "_SystemAttributes::AutoAnnounce", out UInt64 autoAnnounceValue);
  if(autoAnnounceValue == (UInt64)_SystemAttributes.AutoAnnounce.AutoAnnounce_Values.Never)
    WriteLine("Auto Announce Value: " + autoAnnounceValue.ToString());

  AttributeAccess.GetAttributeValue(DO.mData, "_SystemAttributes::AutoConnect", out bool autoConnectValue);
  if(autoConnectValue == false)
    WriteLine("Auto Connect Value: " + autoConnectValue.ToString());
}

Access from CAPL

In CAPL you can get and set attribute values with the functions getAttribute and setAttribute. For string attributes placeholders are always replaced. Errors are handled with return codes.

vCDL

version 1.4;
namespace SomeNamespace
{
  interface ITF
  {
    [Binding="CAPL", CommunicationPattern="PublishSubscribe", AutoAnnounce="Never", AutoConnect=false]
    provided data int32 mData;
  }
  ITF SomeDO;
}

CAPL

on start
{
  char bindingValue[10];
  long commpatValue;
  dword autoAnnounceValue;
  dword autoConnectValue;

  getAttribute(SomeDO.mData, _SystemAttributes::Binding, bindingValue);
  write("Binding Value: %s", bindingValue);

  getAttribute(SomeDO.mData, _SystemAttributes::CommunicationPattern, commpatValue);
  write("Communication Pattern Value: %d", commpatValue);

  getAttribute(SomeDO.mData, "_SystemAttributes::AutoAnnounce", autoAnnounceValue);
  write("Auto Announce Value: %d", autoAnnounceValue);

  getAttribute(SomeDO.mData, "_SystemAttributes::AutoConnect", autoConnectValue);
  write("Auto Connect Value: %d", autoConnectValue);
}
