Distributed Object Member

 

Members

Distributed objects can contain different kinds of members with some fixed direction. Like any member they can be accessed with the "." operator.

The names of distributes object members are case sensitive. In case of name conflicts with predefined members (e.g. distributed object member with name Path) an underscore will be appended. If the escaped name already exists, the escaping continues until the name is unique. This is done in such a way that a member always uses its real name if possible (i.e. if the members Path and Path_ exist, then Path will be mapped to Path__ (two underscores) and Path_ will stay the same).

The types of distributed object members are only used to provide certain methods and properties. The different kinds of members are described in the following sections.

vCDL

version 1.4;
namespace SomeNamespace
{
  interface IReferenced
  {
    provided data int32 mData;
  }

  [Binding="Abstract"]
  object SomeObject
  {
    internal data int32 SomeData;
    [CommunicationPattern="PublishSubscribe"]
    consumed event void SomeEvent;
    internal method int64 SomeMethod(int32 p1, inout int32 p2, out int32 p3);
    internal field string SomeField;
    reference<IReferenced> SomeReference;
  }

  [Binding="Abstract"]
  object OtherObject
  {
    [CommunicationPattern="PublishSubscribe", HasAnnounceAPI=true]
    provided event void SomeEvent;
  }
}

CAPL/C#/Python

// access the member “SomeMember” of the object “SomeObject”
SomeObject.SomeData;
SomeObject.SomeEvent;
SomeObject.SomeMethod;
SomeObject.SomeField;
SomeObject.SomeReference; // Not available in Python

Methods

The API for calls to methods is described on the Function Calls page. The available values at methods are described on the Value Access in Communication Concept page.

C#

int x = 2, y;

// synchronous call
SomeObject.SomeMethod.Call(1, ref x, out y);

// asynchronous call
SomeObject.SomeMethod.CallAsync(1, 2, delegate(long value, int p2, int p3)
{
Output.WriteLine("{0},{1},{2}", value, p2, p3);
});

CAPL

long x = 2, y;

// synchronous call
SomeObject.SomeMethod.Call(1, x, y);

// asynchronous call
SomeObject.SomeMethod.CallAsync(1, 2, delegate(callContext SomeObject.SomeMethod cco)
{
write("%d, %d, %d", cco.p1, cco.p2, cco.p3);
});

Python

x = vector.canoe.InOutParameter(2)
y = vector.canoe.OutParameter()

// synchronous call
SomeObject.SomeMethod.call(1, x, y);

// asynchronous call
SomeObject.SomeMethod.call_async(1, 2, lambda
return_value, p2, p3:
vector.canoe.write(f"{return_value}, {p1}, {p2}"))

Data, Event, and Field Members

Data, event and field members have values that are described on the Value Access in Communication Concept page.

C#

Output.WriteLine(“The value of {1} is {2}”, SomeObject.SomeData.Path, SomeObject.SomeData.Value);

CAPL

write(“The value of %s is %d”, SomeObject.SomeData.Path, $SomeObject.SomeData);

Python

vector.canoe.write(f" The value of {SomeObject.path} is {SomeObject.SomeData.copy()}")

Publish-Subscribe API

If the member has the communication pattern publish-subscribe there are additional methods available. Depending on their direction, these members can also have the attached values SubscriptionState or AnnouncementState (if announcements are used).

C#

SomeObject.SomeEvent.Subscribe();
Output.WriteLine(“SubscriptionState = {1}”, SomeObject.SomeEvent.SubscriptionState.Value);
OtherObject.SomeEvent.Announce();

Output.WriteLine(“AnnouncementState = {1}”, OtherObject.SomeEvent.AnnouncementState.Value);

CAPL

SomeObject.SomeEvent.Subscribe();
write(“SubscriptionState = %d”, $SomeObject.SomeEvent.SubscriptionState);
OtherObject.SomeEvent.Announce();
write(“AnnouncementState = %d”, $OtherObject.SomeEvent.AnnouncementState);

Event Trigger

A provided event can also be updated without setting a value by using the method Trigger. This method is only available for events of the void data type.

C#/CAPL

OtherObject.SomeEvent.Trigger();

Python

OtherObject.SomeEvent.trigger()

Field Getter, Setter, and Notification

Field members may contain a notification event, a Get method and a Set method. The notification event is referenced directly through the field name and has the same type and value as the field itself. The embedded getter and setter methods have the same API as normal distributed object methods. If there is a notification event, the Publish-Subscribe API is also available.

C#

SomeObject.SomeField.Set.Call(“Hello World”);
SomeObject.SomeField.Get.Call();
var val = SomeObject.SomeField.Value;

CAPL

char val[10];
SomeObject.SomeField.Set.Call(“Hello World”);
SomeObject.SomeField.Get.Call();
val = $SomeObject.SomeField;

Python

SomeObject.SomeField.set.call("Hello World")
SomeObject.SomeField.get.call()
val = SomeObject.SomeField.copy()

Reference Member

Reference members are like standalone reference objects described in Dynamic Objects and References. They have a target whose values can be accessed. The target can be invalidated. If the Target is invalid, the last known values can still be read, write access leads to an error.

C#

bool hasTarget = SomeObject.SomeReference.HasTarget;
SomeObject.SomeReference.mData.Value = 42;

CAPL

dword hasTarget = SomeObject.SomeReference.HasTarget;
$SomeObject.SomeReference.mData = 42;
