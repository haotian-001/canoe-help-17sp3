Programming with DO Interfaces

 

Each distributed object has an interface, which in turn can be composed of sub-interfaces. If you want to use the same code for several similar objects, then you should use the interface of these objects in the code. This is especially necessary if objects are created dynamically, i.e. they cannot be known to the program code at all.

First of all, this means that you do not access the static objects directly in the runtime environment, but use variables and parameters whose data type is the interface of the objects.

Example

// Example.vcdl
version 1.3;
namespace SomeNamespace
{
  interface SomeInterface
  {
    provided data int32 SomeMember;
  }
}

// Example.can
void SetSomeMember(distObjRef SomeInterface obj, long val)
{
  $obj.SomeMember = 42;
}

// Example.cs
void SetSomeMember(SomeNamespace.SomeInterface obj, Int32 val)
{
  obj.SomeMember.Value = 42;
}

In the handlers, you do not react to events on specific objects, but to events on all objects with a specific interface.

In CAPL you use the handlers on any_... for this purpose. They work similarly to the specific handlers, except that you must specify an interface member for them, and in the handler you access the concrete object on which the event was triggered with this.object. For value handlers you access the value with this.value, for method handlers you access the calling context with this.context.

Example

// Example.vcdl
version 1.3;
namespace SomeNamespace
{
  interface SomeInterface
  {
    consumed data int32 SomeMember;
    provided method void SomeMethod(double x);
  }
}

// Example.can
on any_value_change SomeInterface.SomeMember
{
  Write("SomeMember of %s changed to %d", this.object.Path, $this.value);
}
on any_fct_called SomeInterface.SomeMethod
{
  Write("%s called with %g", this.object.SomeMethod.Path, this.context.x);
}

In C# you use the attributes [OnAny...] for this. They work similarly to the specific attributes, except that you must specify the ID of an interface member for them and the handler gets as additional first parameter the concrete object where the event was triggered. This parameter must therefore be of the interface type.

Example

// Example.vcdl
version 1.3;
namespace SomeNamespace
{
  interface SomeInterface
  {
    consumed data int32 SomeMember;
    provided method void SomeMethod(double x);
  }
}

// Example.cs
[OnAnyChange(SomeNamespace.SomeInterface_Objects.MemberIDs.SomeMember)]
public void SomeMemberChanged(SomeNamespace.SomeInterface obj)
{
  Output.WriteLine($"SomeMember of {obj.Path} changed to {obj.SomeMember.Value}");
}
[OnAnyCalled(SomeNamespace.SomeInterface_Objects.MemberIDs.SomeMethod)]
public void SomeMethodCalled(SomeNamespace.SomeInterface obj,
SomeNamespace.SomeInterface_Types.SomeMethod.TransientCallContext cc)
{
  Output.WriteLine($"SomeMethod of {obj.Path} called with {cc.x}");
}

// Alternative to OnAnyCalled
[OnAnyCall(SomeNamespace.SomeInterface_Objects.MemberIDs.SomeMethod)]
public void SomeMethodCall(SomeNamespace.SomeInterface obj,
double x)
{
  Output.WriteLine($"SomeMethod of {obj.Path} called with {x}");
}

In Python, you use appropriate methods on the interface members for this, for example register_on_any_change_handler to register and unregister_on_any_change_handler to unregister.

Example

// Example.vcdl
version 1.3;
namespace SomeNamespace
{
  interface SomeInterface
  {
    consumed data int32 SomeMember;
    provided method void SomeMethod(double x);
  }
}

// example.py
  def start(self):
    SomeNamespace.SomeInterface_objects.SomeMember.register_on_any_change_handler(self.SomeMember_changed)
    SomeNamespace.SomeInterface_objects.SomeMember.register_on_any_called_handler(self.SomeMethod_called)

  def SomeMember_changed(self, obj):
    vector.canoe.write(f"SomeMember of {obj.path} changed to {obj.SomeMember.copy()}")

  def SomeMethod_called(self, obj, callContext):
    vector.canoe.write(f"SomeMethod of {obj.path} called with {callContext.x.copy()}")

The respective handlers are also called for events on embedded members if the embedded member has the corresponding interface.

Example

// Example.vcdl
version 1.3;
namespace SomeNamespace
{
  interface SomeInterface
  {
    internal data int32 SomeMember;
  }
  interface DerivedInterface : SomeInterface {}
  interface OuterInterface
  {
    embedded DerivedInterface EmbeddedMember;
  }
  OuterInterface SomeObj;
}

// Example.can
on any_value_change SomeInterface.SomeMember
{
  Write("SomeMember of %s changed to %d",
  this.object.Path, $this.value);
}
on key 't'
{
  // triggers the on_any_value_change handler
  $SomeObj.EmbeddedMember.SomeMember = 42;
}

// Example.cs
[OnAnyChange(SomeNamespace.SomeInterface_Objects.MemberIDs.SomeMember)]
public void SomeMemberChanged(SomeNamespace.SomeInterface obj)
{
  Output.WriteLine($"SomeMember of {obj.Path} changed to {obj.SomeMember.Value}");
}
[OnKey('t')]
public void Stimulate(char ch)
{
  // triggers SomeMemberChanged
  SomeNamespace.SomeObj.EmbeddedMember.SomeMember.Value = 42;
}

// example.py
  def start(self):
    SomeNamespace.SomeInterface_objects.SomeMember.register_on_any_change_handler(self.SomeMember_changed)

  def SomeMember_changed(self, obj):
    vector.canoe.write(f"SomeMember of {obj.path} changed to {obj.SomeMember.copy()}")

  @vector.canoe.on_key('t')
  def stimulate(self, ch):
    # triggers SomeMember_changed
    SomeNamespace.SomeObj.EmbeddedMember.SomeMember = 42
