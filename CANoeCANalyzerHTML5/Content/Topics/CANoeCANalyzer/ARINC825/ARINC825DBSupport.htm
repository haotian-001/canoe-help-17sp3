Database Support

 

CAN messages are described in a database (DBC format). The application data are then available as signals within <CANwin>. In addition, all network information can be represented symbolically.

The label is used as a unique key within the database. A new database can be created with CANdb++. The template CANaero_ARINC825.dbc must be used for this. Alternatively the Aerospace Message Editor may be used.

Network Attributes

The database contains a network object that is described using the following attributes. These attributes are created automatically.

  • "ARINC825" (specification prior to version 3)
  • "ARINC825V3" (specification according to version 3)

This attribute value makes sure that a DBC file is interpreted as an ARINC825 network description.

Message Attributes

The following attributes belong to a CAN message and may be changed by the user.

Name

Data Type

Values

Description

A825MsgType

Enum
  • "unspecific" = 0 (default)
  • "PeriodicHealth" = 1 (for Periodic Health Status Message)
  • "HighIntegrity" = 2 (for High Integrity Message)

With this attribute you may influence the interpretation of messages in the Trace Window.

A825Redundancy

Hex
  • 0x0 = default (no redundancy)
  • Redundancy coded in "0x<D><C><B><A>"
  • With D, C, B, A = {0, 1}

Note, that there is a correlation between this attribute and the attribute A825RCIBitSize.

  • A825RCIBitSize = 0 / A825Redundancy = "0x0"
  • A825RCIBitSize = 1 / A825Redundancy = "0x<B><A>"
  • A825RCIBitSize = 2 / A825Redundancy = "0x<D><C><B><A>"

Define the redundancy relation for this channel.
