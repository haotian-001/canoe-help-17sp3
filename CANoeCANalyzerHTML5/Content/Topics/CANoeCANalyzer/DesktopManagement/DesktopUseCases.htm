Use Cases

 

Use cases are presented here to demonstrate the utility of the desktop and window management concepts. In doing so, you must differentiate between the model producer and the end user.

  • A model producer constructs a test configuration. The producer groups the panels to be used according to topic/task areas. The producer also defines the selection of data windows and their configuration (contents and form).
  • An end user utilizes the configuration constructed by a model producer to test specific topic/task areas.

Examples of Configuring Desktops

  • Desktops permit sorting of windows by topic.
    Data Windows — like Trace, Statistics and Graphics windows — might be maintained on one desktop.
    Another desktop might provide the Configuration windows for the Measurement Setup and Simulation Setup.
  • Some users configure various bus systems, e.g. Powertrain Bus and Body Bus systems.
    The messages of each bus system are represented on separate desktops.
  • A gateway sends signals from one bus to another.
    There is a desktop for each bus system.
    These signals may be displayed in a Graphics window for comparison purposes.
    Therefore it makes sense to display this Graphics window on both desktops.
  • Some users utilize panels.
    Desktops are used to classify the panels as input panels and output panels.
    Other panels can be organized by the bus systems used.
  • The window management concept makes it possible to save opened windows structurally.
    windows which should be shown continuously are put in the Docked type.
    windows which should appear outside of the CANoe program window are put in the Floating type.
    Panels are put in the MDI type to integrate them into the program window.

Step-by-Step Procedure: Arranging Desktops for a Test Environment

The ribbon includes a selection of windows and panels contained in the configuration. When you select an object of your choice this object is opened on the active desktop.

The required windows/panels are selected by your own criteria. An individual window is configured by choosing the Configuration option in the context menu for the selected window.

It is assumed that you will be working with panels which permit input and output. The program window should be minimized after the measurement start so that only the panels on the desktop are visible. The Standard window type should be assigned for positioning outside of the program window. This ensures that the panels will remain in their positions even after the program window has been minimized. Select each panel and save it on the Windows Desktop.

If you wish to construct another desktop, a new desktop can be created from the Desktops context menu after pressing the right mouse button. This desktop automatically becomes the active desktop. Choose the desired output window from the ribbon. Then it is also possible to dock windows for example.

The window functions in the Layout ribbon tab support optimal arrangement of MDI windows on the desktop. Improved clarity is also achieved with the new window types Docked and Floating. Depending on your preferences, these may be used to influence the overall appearance of the desktop.

After the windows have been configured you switch to the panel desktop, then start the measurement and minimize the program window. The positioned panels are now located on your desktop, and testing can begin.

Notes on working with desktops

  • You want to move windows from one desktop to another?

Select the window, choose the Move to… command and select a target desktop from the list of available desktops.

  • You want to display panels on your desktops?

To do this, open Panel Configuration and integrate panels into CANoe with the Add… option. Afterwards open the panels of your choice in the active desktop from the Home ribbon tab. It should be noted that referenced panels (panels which are subordinate to the configured panels) are also shown in the Home ribbon tab.

  • You (as a model producer) want to configure various CANoe desktops by specific topics?

Desktop 1 contains the Trace window, and control and display elements for testing the functionality of a model. You create all windows/panels within the program window, since during the test the end user works both with panels and with the Trace window. Finally you dock the Trace window permanently in the program window.

Another desktop contains panels with control and display elements for another function to be tested. Place the panels of Desktop 2 outside of the program window. During the test the end user can minimize the program window to background in order to work exclusively with panels.

  • You want to change window type of a Write window from MDI to Docked?

To do this, press the right mouse button with the mouse pointer over the Write window title bar and select Docking view from the context menu that appears. The window is automatically docked to a position in the program window which corresponds to its previous position.

  • A Bus Statistics window should be positioned outside of the program window?

To do this, press the right mouse button with the mouse pointer over the Bus Statistics window's title bar and choose Docked window from the context menu that appears. Then drag the docked window outside of the program window. It should be noted that a window outside of the program window is minimized whenever the program window is minimized.
