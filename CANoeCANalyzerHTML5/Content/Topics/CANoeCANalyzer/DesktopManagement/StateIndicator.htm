Controlling via State Indicator and Ribbon

 

During the measurement, the status bar and the ribbon of CANoe displays buttons that enable you to quickly access various functions. These buttons, which are described in the following, are only shown when a corresponding object (e.g., a Signal Generator) has been configured in the associated window and is activated.

Status Bar

Click the button in the status bar to open a popup window. The current state of all elements is visualized in the popup window and you can control these elements.
A tooltip shows you state information.

Ribbon

A click on the arrow symbol opens a submenu to control the different elements. Before the start of the measurement you can here activate/deactivate elements.

Signal Generators and Signal Replay

You can start, stop, and pause Signal Generators during the measurement. The signal replay is only for display purposes and cannot be changed.

Screenshot

Description

  • Left column: Displays the current state (running/paused/stopped)
  • Center column: Displays the signals/variables in alphabetical order
  • Right column: Here, you can change the state using buttons.
  • Before the measurement you can activate/deactivate in a additional column Signal Generators via the Simulation ribbon tab.

Example in Screenshot:

  • The top two lines represent a variable replay. The status cannot be changed.
  • Signal WN_left_down is currently stopped. You can restart the replay.
  • Signal WN_left_up is currently paused. You can resume the replay.
  • Signal WN_right_down is currently running. You can stop or pause the replay.

The actions within this menu correspond to the actions within the Signal Generators and Signal Replay window, which you can open using the Simulation ribbon tab.

Automation Sequences

You can start, stop, pause, or record automation sequences (visual sequence, macro, .NET Snippet) during the measurement, depending on the type.

The sequence type is visualized with an icon (corresponds to those in the Automation Sequences window, which you can open using the Simulation ribbon tab).

Screenshot

Description

  • Left column: Displays the current state (running/stopped)
  • Center column: Displays the sequences in alphabetical order
  • Right column: Here, you can change the state using buttons.
  • Before the measurement you can activate/deactivate in a additional column automation sequences via the Simulation ribbon tab.

Example in Screenshot:

  • Macro Diagnostics is currently stopped. You can restart the replay.
  • Visual sequence DrivingCycle is currently running. You can stop the replay.
  • .NET-Snippet NETSnippet::Initialize is currently stopped. You can restart the replay.

Note

The symbol to edit an automation sequence is only available in the ribbon submenu.

The actions within this menu correspond to the actions within the Automation Sequences window, which you can open using the Simulation ribbon tab.

Video

The state indicator of the video recording is only shown if a recorder file was configured for at least one Video Window.

You can start or pause the video recording during the measurement.

Screenshot

Description

  • Left column: Displays the current state (active/stopped)
  • Center column: Display of the Video Windows with a configured recorder file in alphabetical order along with their state.
  • Right column: Here, you can change the state using buttons.

Example in Screenshot:

  • The recording of Video Window Camera 1 is currently active. You can pause the recording.
  • The recording of Video Window Camera 2 is currently paused. You can resume the recording.

The actions within this menu correspond to the actions within the Video Window, which you can open using the Analysis ribbon tab if a Video Window was configured.

Logging

The state indicator of the data recording is only shown if a Logging block has been integrated in the Measurement Setup and it is active.

You can start or pause the data recording during the measurement.

Screenshot

Description

  • Left column: Displays the current state (active/stopped)
  • Center column: Displays the Logging blocks in the Measurement Setup in alphabetical order along with their state.
  • Right column: Here, you can change the state using buttons.
  • Before the measurement you can
  • activate/deactivate in a additional column data logging
  • open the configuration dialog for triggers and logging file

    • via the Simulation ribbon tab

Example in Screenshot:

  • Logging block Log CAN1 is currently paused. You can resume the recording.
  • Logging block Log CAN2 is currently active. You can stop the recording.

Note

The symbols to configure the trigger conditions and the logging file are only available in the ribbon submenu.

The actions within this menu correspond to the actions that you can execute via the Logging block in the Measurement Setup.

MOST Stress

The state indicator for MOST Stress is always shown during the measurement if you have configured at least one MOST channel.

You can start and stop MOST Stress during the measurement. Whether that is possible in the particular MOST channel depends on the settings in the MOST Stress window.

Screenshot

Description

  • Left column: Displays the current state (started/stopped)
  • Center column: Displays the MOST channels and the Stress modes in alphabetical order.
  • Right column: Here, you can change the state using buttons.

Example in Screenshot:

  • MOST channel 1 MOST 1: Light Stress cannot be manually started or stopped due to the Stress mode.
  • MOST channel 2 MOST 2: Control Stress is currently stopped. You can restart the stress.
  • MOST channel 3 MOST 3: Control Stress is currently running. You can stop the Stress.

The actions within this menu correspond to the actions within the MOST Stress window, which you can open using the Simulation ribbon tab.
