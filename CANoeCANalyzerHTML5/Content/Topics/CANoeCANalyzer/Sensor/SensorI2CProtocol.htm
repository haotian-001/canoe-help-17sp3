I2C Protocol

 

The Inter-Integrated Circuit Interface (I2C) is a synchronous serial data network interface for data exchange between at least a master and one or several slaves. Multi-master configurations are permitted, but they are not supported by CANoe. The I2C interface is usually used for networking different parts of an electronic circuit.

Start Condition

To generate a start condition, the master switches the SDA from High to Low, while the SCL is set to High.

Bit Transmission

To transmit a bit, the clock line SCL must be set to High. In this phase, the level of the data line SDA indicates the value of the bit.

Stop Condition

To generate a stop condition, the master switches the SDA from Low to High, while the SCL is set to High.

Communication Protocol [â² back]

The I2C communication always takes place according to the following scheme:

  • The master generates a start condition on the bus.
  • The master sends the slave address and the R/W bit.
  • Any amount of data is transmitted byte-wise between the master and the addressed slave. These are acknowledged by the respective receiver via Acknowledge.
  • The master generates a stop condition on the bus.

Alternatively, the master can set another start condition instead of a stop condition, thus initiating a new data transmission immediately. This process is referred to as Repeated Start.

Addressing

A unique address is assigned to each slave on the bus through which the slave can be specifically addressed by the master. Originally, the address space was limited to 7 bits, but it has been increased to 10 bits after a subsequent extension of the I2C specification. It is still possible to operate slaves with 7-bit addresses and slaves with 10-bit addresses on a bus.

7-bit addressing

In the 7-bit addressing all address bits are transmitted in the first data byte after the start condition. The last bit is used to transmit the R/W bit.

10-bit addressing

Two data bytes are required for the transmission of a 10-bit address. In the first data byte, the master transmits the reserved bit sequence 11110. This is then followed by the two first bits of the address. In the second byte, the master transmits the remaining eight bits of the address.

R/W bit

This bit defines whether the master wants to receive data from the slave or whether it wants to send data to the slave.

  • To receive data (Read), the master sets the bit to 1.
  • To send data (Write), the master sets the bit to 0.

The value of the bit is valid for all data bytes that follow until a new transmission is started.

Acknowledge

The receiver (master or slave) acknowledges the reception of each data byte with a confirmation (Acknowledge). For this, the receiver switches the data line SDA to Low until the master has generated the next cycle pulse.

If this confirmation takes place, this also means that the receiver wants to receive a further byte. This is referred to as ACK. If the receiver does not want to receive further bytes, it sends no confirmation. This is referred to as NACK.

Clock Stretching [â² back]

If a slave is slower as the bus cycle defined by the master, the slave can slow down the master during data transmission using the clock stretching mechanism. The slave holds the SCL line at logical zero until it can receive or send the next bit.
