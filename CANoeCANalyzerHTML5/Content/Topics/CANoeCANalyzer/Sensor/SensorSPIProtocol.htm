SPI Protocol

 

SPI (Serial Peripheral Interface) is a synchronous serial data network interface for data exchange between a master and one or several slaves. The interface is usually used to exchange data between an ECU (master) and various sensors (e.g. temperature, pressure or acceleration sensors) and actuators (e.g. digital-to-analog converters).

In the star connection, all slaves are connected to the signal lines SCKL, MISO and MOSI in parallel. For each slave, the master has a separate chip select line which can be used to address the slave individually:

In the daisy chain mode, the individual slaves are connected one after the other. The master transmits the data through all connected slaves. For this, the MISO output of a slave is connected to the MOSI input of the next slave. The MISO output of the last slave is connected to the master again. Operating Modes [â² back]

Clock polarity (CPOL)

This information describes the idle level of the SCLK clock line.

  • CPOL 0: the clock line is idle low
  • CPOL 1: the clock line is idle high

Clock phase (CPHA)

This information describes at which edge of the clock signal the master and slave receive data from the bus:

  • CPHA 0: the data is transferred at the first edge
  • CPHA 1: the data is only transferred at the second edge

Thus, there are four different modes for the timing of the data exchange:

Mode 0 (CPOL 0, CPHA 0)

Mode 1 (CPOL 0, CPHA 1)

The clock signal is idle low, the data is transferred with the first (rising) edge.

The clock signal is idle low, the data is transferred with the second (falling) edge.

Mode 2 (CPOL 1, CPHA 0)

Mode 3 (CPOL 1, CPHA 1)

The clock signal is idle high, the data is transferred with the first (falling) edge.

The clock signal is idle high, the data is transferred with the second (rising) edge.

Communication [â² back]

The communication between the master and a slave usually starts with the activation of the chip select line of the slave by the master. After that, both participants output their data clock-synchronized with the SCLK signal. For the data exchange, both participants have shift registers which contain data for sending and receiving. The data from these registers is exchanged bitwise between master and slave with each clock. To terminate the communication, the master deactivates the chip select line of the slave. Such transfer is designated in CANoe as SPI frame.

There is no standardized data protocol. Therefore, the structure of the sent and received data is not further specified and greatly depends on the corresponding use case. The number of bits per transfer can vary. Depending on the slave, the number of transferred bytes can widely vary from a few bytes (e.g. A/D converter) up to several Mbytes (e.g. flash memory). The sequence of bits (LSB or MSB first) within the bytes is also not determined.
