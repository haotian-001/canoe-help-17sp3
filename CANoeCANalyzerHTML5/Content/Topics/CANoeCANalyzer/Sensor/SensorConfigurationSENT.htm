Configuration of the SENT Sensor Channel

Ribbon|Hardware Tab|Protocol configuration|SENT

The elements and the settings of the SENT sensor channel and all system variables available for a SENT channel are described in this section.

Overview of the SENT network elements

Element

Description/configuration


Network

 

SENT Settings

No configuration settings are required.


Channel

A channel describes the interconnection between an ECU and a SENT sensor.

All the settings that apply to the elements within a channel are configured on this level.

SENT Channel Settings

  • Assigned hardware channel
    Specifies which physical hardware channel (e.g. a VT2710 piggy channel) is assigned to this sensor channel. In case no channels have been configured, the sensor channel will be automatically deactivated at the start of the measurement.
  • Enable Short PWM Code (SPC) mode
    Activates the Short PMW Code (SPC) for the underlying sensors. A maximum of four sensors are supported.

Simulation Settings: Stimulation Data Source

  • Logging file used for sensor/ECU stimulation
    Here you can specify a previously recorded BLF logging file. This file can be used on this channel by simulated ECUs and sensors as data source.
    For example, the behavior of a real sensor can be logged with CANoe and later replayed via a simulated sensor


ECU

Each SENT channel contains an ECU to which the sensors are connected. This ECU can be physically available and connected to a computer or it can be simulated by CANoe.

SENT ECU Simulation Settings

  • Generated supply voltage
    Specifies the supply voltage generated by the ECU.


Sensor

Each SENT channel contains a sensor connected to the ECU. This sensor can be physically available and connected to a computer or it can be simulated by CANoe.

SENT Sensor Settings

  • Clock tick length
    Indicates the length of a single clock tick.
  • Number of low ticks
    Specifies the quantity of low ticks at the beginning of each nibble.
  • Pause mode
    Specifies the pause mode:
  • No pause: no pause will be made.
  • Fixed pause length: the pause length is fixed.
  • Fixed frame length: the pause length is dynamic. The pause length will always be selected in a way that the frames have a fixed length.
  • Number of pause ticks
    In case a fixed pause length has been configured as mode, the exact length can be specified here.
  • Min. trigger low time
    Specifies the minimum length of the low phase used to detect the trigger pulse.
  • Number of data nibbles
    Specifies how many nibble data must be sent in each frame (min. 1, max. 6).
  • CRC mode
    This setting allows you to specifiy if the recommended mode or the legacy mode must be used for the CRC calculation.
  • Serial messaging
    Specifies if this sensor sends short or long serial messages.
  • Max. startup time
    This setting specifies the maximum time a sensor takes for starting. This also determines when CANoe expects frames from the sensor at the earliest. If the time has expired and the sensor sends no frames, respective errors will be output.
  • Max. trigger low time
    Specifies the maximum length of the low phase used to detect the trigger pulse.

Simulation Settings

  • Sending start delay
    Specifies how long the sensor waits before starting the transmission, after that the supply voltage has been reached.
  • Supply voltage trigger level
    Specifies the voltage that the simulated sensor accepts as supply voltage.
  • Total trigger time
    Specifies the time between the start of the trigger pulse and the start of the sync pulse.
  • Replay sensor data from stimulation file
    If this setting has been activated, the simulated sensor replays data from a logging file. This file must exist in the BLF format and must have been configured in the higher-level SENT channel.
  • Stimulation data source
    With [Choose Sensor...] you open the Stimulation Data Source Browser dialog. Here you can select the sensor whose data should serve as simulation source.


Signal

CANoe assigns a system variable to each SENT sensor. This system variable allows to access the frames sent or received in this slot. Furthermore, it is also possible to directly access the single regions of the frames (e.g. data region or status region).

Example: definition of two signals in a SensorFrame

Signals provide a further option to access the bits within a frame. They can be defined via any of the areas of the frame by indicating the exact number of bits that compose the offset and the length of the signal. A conversion formula (defined via factor and offset) and a unit can also be specified. CANoe generates a further system variable for each signal. This system variable allows to access the corresponding part of the frame.

SENT Signal Settings

Signal Layout

  • Transmitted via
    Specifies in which part of the frame the signal is transmitted. The signal can also be assigned to a serial message. In this case, the message ID must also be specified (see below).
  • Serial message ID
    If the signal is transmitted to a serial message, the message ID can be specified here.
  • Signal length
    Specifies the quantity of bits or nibbles that compose the signal.
  • Signal offset
    Specifies the bit or nibble, within the region selected for the transmission, beginning from which the signal starts.
  • Nibble order
    If a SENT signal is defined via data nibbles, this can be interpreted in two ways:
  • Least to most significant nibble
    The signal is interpreted according to the nibble significance (e.g. from D1 to D3).
  • Most to least significant nibble
    The signal is interpreted opposite to the nibble significance (e.g. from D3 to D1).
  • Interpret as signal value
    The received raw values can be interpreted as unsigned or signed. E.g. the 8-bit value 0xFF generates -1 (signed) or 255 (unsigned).
    This setting allows you to specify which interpretation must be used.

Display Scaling

  • Factor
    This factor allows the raw values to be multiplied before they are displayed in the corresponding system variable in CANoe. When sending values (sensor simulation), the value of the system variables is divided by this factor, before it is sent as raw value.
    This results in the following formula for
  • received values:
    received value = raw value * factor + offset
  • sent values:
    raw value = (sent value – offset ) : factor

    • If no factor should be used, it can be set to 1.

  • Offset
    The offset is used in the same way as the factor, i.e. the received values are increased by the offset. On the other hand, the offset is subtracted from the sent values.
    This results in the following formula for
  • received values:
    received value = raw value * factor + offset
  • sent values:
    raw value = (sent value – offset ) : factor

    • If no offset should be used, it can be set to 0.

  • Unit
    Specifies the system variable unit displayed in CANoe.

Simulation Settings

  • Initial signal value (scaled)
    A simulated sensor sends this signal value until a new value is manually output. This can happen, for example, by setting the system variables or the corresponding CAPL calls.
    This value must be interpreted as scaled. The reason for this is that before it is sent, it has been converted with Offset and Factor (see above).
