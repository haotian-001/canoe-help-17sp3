Overview of the PSI5 System Variables [▲ back]

If you have configured a PSI5 network in the Sensor Configuration dialog, CANoe automatically creates the corresponding system variables.

    Member

    SENSOR::PSI5::ExampleChannel::ExampleECU

    Function

    The sync pulses of the ECU are reported in these variables. A simulated ECU generates TX Events (sent sync pulse), a simulated sensor generates RX Events (received sync pulse).

    The time stamp of this event corresponds to the beginning of the sync pulse. To be more precise, it corresponds to the time at which the ECU voltage has exceeded the sync pulse trigger threshold.

    Member

      Indicates the length of the sync pulse in nanoseconds.
    • CycleTime
      Indicates the time passed from the last sync pulse in nanoseconds.

Simulating a Sensor [▲ back]

CANoe provides the opportunity to simulate PSI5 sensors for the testing of ECUs. This allows you to simulate ECUs with predetermined sensor values.

Configuration of the Sensor Simulation [▲ back]

For the simulation of a sensor with CANoe, it is necessary to set the sensor mode to "simulated" in the Sensor Configuration dialog.

Starting Behavior of the Sensor Simulation [▲ back]

CANoe starts with the simulation of the sensor at the start of a measurement. First the sensor waits until the required supply voltage is applied by the ECU. The threshold can be defined in the Sensor Configuration dialog. As soon as the required voltage has been reached, this is reported by the PowerStatus system variable. Now the sensor undergoes the following phases:

  • In Phase 1 no data are sent by the sensor. The start time of the sensor is therefore simulated. The length of the start time can be set in the Sensor Configuration dialog.
  • In Phase 2 the meta header is sent first. If no meta headers should be sent, this function can be deactivated in the Sensor Configuration dialog.
    The initialization data are then sent according to the Sensor configuration. Also this option can be explicitly deactivated.
  • In Phase 3 the status value Sensor Ready is transmitted. How often or whether this status value is transmitted can be specified in the Sensor Configuration dialog.
  • Then the sensor starts to cyclically send the measured values.

Modifying the Starting Behavior [▲ back]

For certain test scenarios it can be necessary to modify parts of the starting sequence. To do so, the corresponding phases must be first deactivated in the Sensor Configuration dialog.

  • Deactivation of Sensor Ready: set the Sensor Ready messages value to 0 for the corresponding time slot.
  • Deactivation of the initialization data: set the Initialization mode to No initialization for the corresponding time slot.

Please note that the initial measured value of the sensor should be sent as last frame. It will then be automatically repeated by CANoe until the value is manually modified.

Behavior of the Sensor in Case of a Voltage Loss [▲ back]

If a supply voltage loss occurs during the measurement, the sensor will stop sending. This event is reported in CANoe by the PowerStatus system variable.

As soon as the required supply voltage is reached again, the sensor will start with its normal starting sequence (see above). Also this event is reported by the PowerStatus system variable.

Sending of Simulation Data [▲ back]

CANoe provides different options to affect the data sent by the simulated sensor. These options are described in the following sections.

The hardware channel repeats the last sent frame until new data are set by CANoe. This guarantees that the cycle times of the PSI5 protocol are observed.

  • Sending of signals via system variables
    CANoe creates a system variable for each signal that has been configured in the Sensor Configuration dialog. The signal values can be modified through, for example, a panel or a signal generator. The modification of the variables is transmitted to the hardware channel in which the new value will be initiated in the sent frame.
  • Sending of frames via system variables
    Similarly to sending signals via system variables, you can also send entire frames. To do this, modify the value of the frame simulation variables. This value is then transmitted to the hardware channel in which the new frame will be sent.
    The frame simulation variables provide access to the status and data regions of the frame. The parity or the CRC as well as the messaging bits are calculated automatically and therefore they cannot be set.
  • Sending of signals with CAPL
    The values of the sent signals can also be modified via CAPL. To do this, select the QueueValues method on the corresponding signal system variables. Alternatively, you can also use the CAPL function sensorQueueValues which expects the names of the signal system variables as first parameters. In both cases, an array of signal values is transmitted which will then wait in a queue.
  • Sending of frames with CAPL
    To send entire frames via CAPL use the QueuePsi5Frame or sensorQueuePsi5Frame function. A description of the frame is expected as individual parameter.
  • Sending of serial messages with CAPL
    The CAPL function QueueSerialMessage or sensorQueueSerialMessage allows you to prepare the sending of serial messages. The message ID (as parameter) and a flag are transmitted. The flag allows you to select the type of message:
  • Long ID: the ID is 8 bits long, the data are 12 bits long.
  • Short ID: the ID is 4 bits long, the data are 16 bits long.

    • The current data of the message are not transmitted to the CAPL function. These data are taken from the signal system variables assigned to the corresponding message IDs.

Logging and Replaying of the Sensor Communication [▲ back]

The logging function of CANoe allows you to log the communication of a real sensor. This log can be later replayed by a simulated sensor. To do so, proceed as follows:

  1. Load or create a CANoe configuration with a real PSI5 sensor.
  2. Activate the Logging Block in the Measurement Setup of your CANoe configuration.
  3. Select BLF as output format.
  4. Start the measurement in order to allow the communication of the sensor to be logged.
  5. Create a new CANoe configuration with a simulated PSI5 sensor.
  6. In the configuration of the PSI5 channel select the BLF logging file as data source.
  7. In the configuration of the simulated PSI5 sensor activate the replay from the logging file.
  8. Select the names of the sensor whose logging file you want to reproduce.

    9. The data are sent from the logging file through the simulated sensor at the start of the measurement.

Timing of Replayed Sensor Communication [▲ back]

The timing recorded in the logging file will be ignored when replaying it. Instead the current settings from the Sensor Configuration dialog will be used to determine the current timing (e.g. cycle time of the channel). Thus a previously recorded logging file can be reused in many different configurations.

Initialization Phase When Replaying Sensor Communication [▲ back]

If you configure a sensor to replay logged data its initialization phase changes as follows:

  • If the channel is in synchronous mode the configured start delay of the sensors will be ignored.
  • None of the configured initialization data will be sent.
  • None of the configured status values will be sent.

The idea behind this change in behavior is that the logging file is likely to contain a start delay, initialization data and status values. In order to avoid sending these twice, the configuration settings are ignored.

Simulating an ECU [▲ back]

CANoe provides the opportunity to simulate a PSI5 ECU for the analysis and testing of sensors. The data sent from the sensor can be displayed and evaluated in CANoe.

Configuration of the ECU Simulation [▲ back]

For the simulation of an ECU with CANoe, it is necessary to set the sensor mode to simulated in the Sensor Configuration dialog.

Behavior of a Simulated ECU [▲ back]

As soon as the measurement is started in CANoe, the simulation of the PSI5 ECU will begin. This allows the supply voltage for the connected sensors to be generated.

If the channel is operated in the synchronous mode, the simulated ECU generates the first sync pulse after that the start delay has elapsed. The sync pulse is then cyclically repeated until the measurement in CANoe is started. In the asynchronous mode no sync pulses are generated.

Analysis of the Communication between two Real Devices [▲ back]

The option .Sensor allows you to log the communication between a real ECU and real sensors and to analyze it in realtime. The mode of the ECU and those of the sensors must both be set to real in the Sensor Configuration dialog. The data received from the ECU and the sensors will then be reported in CANoe via the corresponding system variables.
