Option Sensor

 

Option Sensor offers a variety of applications concerning the analysis, simulation and testing of sensors and ECUs. Among others, it includes the following functions:

  • Logging and analysis of the communication between ECUs and sensors.
  • Simulation of sensors for testing the functions of your ECUs.
  • Simulation of ECUs for testing sensors.

Option Sensor is seamlessly integrated into the scope of functions of CANoe:

  • Visualize the sensor data in the numerous analysis windows of CANoe. Among others, the following windows are available: the Trace Window, the Graphics Window, the State Tracker and the Data Window.
  • The communication between ECUs and sensors can be logged through the logging functions of CANoe. The communication can be analyzed offline at a later time or played back in realtime.
  • The precise time stamps allow you to correlate sensor data with data of other bus systems, such as CAN or LIN, or I/O data from VT System, without problems.
  • Create simulation data for your simulated sensors or ECUs using Signal Generators, panels or through the flexible programming with CAPL.
  • Use the numerous functions of the Test Feature Set to easily create comprehensive tests for your ECUs and sensors and to execute them with CANoe.

Hardware

The operation of Option Sensor requires an appropriate network interface to which the sensors and ECUs can be connected. To this purpose, Vector offers the VT System VT2710 module which can be fitted with up to four sensor piggybacks. Each of these piggybacks represents a PSI5 or SENT channel. For SENT there is also the possibility to use a VN1640(A) network interface or an interface of the VN1500 family with a SENSORpiggy SENT for the physical connection.
