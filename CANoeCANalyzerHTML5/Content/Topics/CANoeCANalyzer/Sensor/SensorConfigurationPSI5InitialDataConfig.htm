Initialization Data Configuration

Ribbon|Hardware Tab|Protocol Configuration|PSI5|Initialization Data Content

In the Initialization Data Configuration dialog you can specify which initialization data are sent by the simulated sensor in this time slot.

General Settings

  • Data length
    This value specifies how many nibbles (4 bit) are sent as initialization data. This setting determines which settings are available in the following sections.
  • Repeat block
    Specifies how often each block (i.e. pair of ID and value) must be repeated. In the PSI5 specification this value is referred to as k times.

Contents

  • Protocol revision
    Specifies which protocol revision is reported in the initialization data.
  • Manufacturer
    Specifies the manufacturer of the sensor. The codes set by the PSI5 standard are also available for the selection. The setting user-defined allows you to use an own manufacturer code.
  • Sensor type
    Specifies the type of sensor. Detailed information on this value can be found in the PSI5 specification.
  • Sensor parameter
    Specifies the sensor parameters. Detailed information on this value can be found in the PSI5 specification.
  • Sensor code (sensor manufacturer)
    Specifies the sensor code of the sensor manufacturer. Detailed information on this value can be found in the PSI5 specification.
  • Sensor code (vehicle manufacturer)
    Specifies the sensor code of the vehicle manufacturer. Detailed information on this value can be found in the PSI5 specification.
  • Production date
    Specifies the production date of the sensor. Detailed information on this value can be found in the PSI5 specification.
  • Specific data
    Here you can specify further data in addition to those contained in the PSI5 standard.
