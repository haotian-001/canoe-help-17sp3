UART Protocol

 

The UART protocol (Universal Asynchronous Receiver Transmitter) is an asynchronous serial protocol for data transmission between two network nodes or more. It can be operated on diverse physical interfaces like RS232, RS485, RS422 or LVDS.

Depending on the used physical interface, different bus topologies and signal lines are used.

Serial Interface [â² back]

This simple interface can often be found in microcontrollers. It allows a full duplex communication between two network nodes. The data communication is usually done via two signal lines that link together the Rx and Tx pins of both nodes.

Since the data transmission is asynchronous, no additional clock line is necessary. Instead a receiver synchronizes itself via the start and stop bits of the frames.

In this interface, the logic levels are implemented by different tensions to the common ground. TTL/LVTTL levels (0 V for a logical 0, 5 V/3.3 V for a logical 1) are often used.

RS485 or RS422 Interface [â² back]

The RS485 and RS422 interfaces have been developed for the serial high-speed data transmission over long distances. They allow to operate up to 32 network nodes on one bus and are widely used in industrial applications. A differential twisted pair to which all nodes of the bus are connected serves as a signal line. This way, only one node on the bus at a time can be used for sending. A full duplex communication is not possible but can be enabled by using a second twisted pair.

The logic levels in this interface are generated by the voltage difference between both wires of the signal line. A logical 1 corresponds to a negative difference, a logical 0 to a positive difference.

No special protocol is required for the RS485 or RS422 interfaces. In practice however, protocols like UART, Modbus or Profibus are used in most cases.

LVDS Interface [â² back]

LVDS (Low Voltage Differential Signaling) is an interface standard for serial high-speed data transmission. The bus topology is limited to a point-to-point connection between two nodes. A differential twisted pair serves as a signal line. If you want to allow bidirectional operation, you can use a second twisted pair.

The logic levels in this interface are generated by the voltage difference between both wires of the signal line. A logical 1 corresponds to a positive difference, a logical 0 to a negative difference.

No special protocol is required for the LVDS interface. It can for instance be operated with the UART protocol.

Frame Structure [â² back]

The communication between the network nodes is frame-based. Every frame begins with a start bit (logic 0). Then seven to nine data bits follow. The frame is completed by an optional parity bit as well as by one or two stop bits (logic 1). A logic 1 applies in the pauses between two frames.
