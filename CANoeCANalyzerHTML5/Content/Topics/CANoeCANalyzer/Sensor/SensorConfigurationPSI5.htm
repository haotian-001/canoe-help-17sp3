Configuration of the PSI5 Sensor Channel

Ribbon|Hardware Tab|Protocol Configuration|PSI5

The elements and the settings of the PSI5 sensor channel are described in this section. All system variables available for a PSI5 channel are also described here.

Overview of the PSI5 Network Elements

Element

Description/Configuration


Network

PSI5 Settings

No configuration settings are required.


Channel

A channel describes the interconnection between an ECU and one or more PSI5 sensors.

All the settings that apply to the elements within a channel are configured on this level (e.g. communication mode or baudrate).

PSI5 Channel Settings

  • Assigned hardware channel
    Specifies which physical hardware channel (e.g. a VT2710 piggy channel) is assigned to this sensor channel. In case no channels have been configured, the sensor channel will be automatically deactivated at the start of the measurement.
  • Communication mode
    Specifies which communication mode must be used for this channel. Details relative to the different modes can be found in the PSI5 specification.
  • Bitrate
    Specifies the bitrate used by this channel. The standard values are 125 or 189 kbit/s. Different values can also be entered.
  • Cycle time
    Specifies the cycle time with which this channel works.

Simulation Settings: Common Sensor Settings

General Sensor Settings

  • Sending start delay
    Specifies how long a sensor simulated via CANoe waits before starting the transmission, after that the supply voltage has been reached. This setting applies for all simulated sensors of the channel.
  • Supply voltage trigger level
    Specifies the voltage that the simulated sensor should detect as supply voltage. Once this voltage has been reached or exceeded, the sensor will become active. This setting applies for all simulated sensors of the channel.
  • Sync voltage trigger level
    Specifies the voltage that the simulated sensor should detect as sync pulse.

Sensor Current Settings

  • Idle current
    Specifies the current that the simulated sensor allows to flow during its rest phase. This setting applies for all simulated sensors of the channel.
  • Send current
    Specifies the current that the simulated sensor allows to flow during transmission. This setting applies for all simulated sensors of the channel.

Simulation Settings: Simulation Data Source

  • Logging file used for sensor/ECU stimulation
    Here you can specify a previously recorded BLF logging file. This file can be used on this channel by simulated ECUs and sensors as data source.
    For example, the behavior of a real sensor can be logged with CANoe and later replayed via a simulated sensor


ECU

 Each PSI5 channel contains an ECU to which the sensors are connected. This ECU can be physically available and connected to a network interface or it can be simulated by CANoe.

PSI5 ECU Settings

Voltage Settings

  • Sensor supply voltage
    Specifies the supply voltage generated by the ECU.

  • Sync pulse sustain voltage
    Specifies the voltage during a sync pulse. This value is not absolute. It must be interpreted as a deviation from the supply voltage.

Timing Settings

  • Sync pulse rise time
    Specifies the duration of the rising edge of the sync pulse. A value of 0 means that the pulse rises as fast as allowed by the hardware.
  • Sync pulse sustain time
    Specifies the hold time of the sync pulse.
  • Sync pulse fall time
    Specifies the duration of the falling edge of the sync pulse. A value of 0 means that the pulse is reduced as fast as allowed by the the hardware.

Current Settings

  • Send current trigger level
    Specifies the threshold value from which a current must be detected as transmit current.

Simulation Settings

  • Sync pulse start delay
    Specifies how long an ECU simulated via CANoe waits before starting the transmission of the sync pulse, after that the measurement has been started.


Sensor

Each PSI5 channel contains one or more sensors connected to the ECU. The sensors can be physically available and connected to an network interface or they can be simulated by CANoe.

PSI5 Sensor Settings

  • Max. startup time
    This setting specifies the maximum time a sensor takes for starting. It also determines when CANoe expects frames from the sensor at the earliest. If the time has expired and the sensor sends no frames, respective errors will be output.

PSI5 Sensor Simulation Settings

  • Replay sensor data from a stimulation file
    If this setting has been activated, the simulated sensor replays data from a logging file. This file must exist in the BLF format and must have been configured in the higher-level PSI5 channel.
  • Stimulation data source
    With [Choose Sensor...] you open the Stimulation Data Source Browser dialog. Here you can select the sensor whose data should serve as simulation source.

Time slot

It is possible to assign one or more time slots to each sensor, into which the sensor transmits its data. The time slots of different sensors must not overlap.

PSI5 Time Slot Settings

General Settings

  • Slot start time
    Specifies the time at which the time slot starts after the end of a sync pulse .
  • Slot length
    Specifies the length of a time slot.
  • Frame start time
    Specifies the time at which the current frame start within the time slot. This value refers to the start of the time slot.
  • Status values
    With [Edit...] you can open the Start Value Interpretation Table dialog. Here you can modify the descriptions of the status values displayed in CANoe. The status values can differ according to the PSI5 substandard.
  • Initialization mode
    Specifies which initialization mode of the sensor is used in this time slot. This usually depends on the type of sensor (chassis, airbag, etc.). For a detailed description of the different initialization modes refer to the corresponding application-specific PSI5 specifications.

Frame Structure

  • Messaging bits
    Specifies how many bits the frame contains for the serial transmission.
  • Frame control bits
    Specifies how many bits the frame contains for the frame control functionality.
  • Status bits
    Specifies how many bits the frame contains for the status notification.
  • Size of data region B
    Specifies the size of the data region B in the frame.
  • Size of data region A
    Specifies the size of the data region A in the frame.
  • Error detection
    Specifies how many bits and which algorithm are used for the error detection (3 bits for CRC, 1 bit for parity).

Simulation Settings: Initialization Data

  • Sensor ready count
    This setting allows you to specify how often the sensor sends the status value Sensor Ready in this time slot. The message transmission takes place after sending the initialization data and before sending the first measured values.
  • Fill bits
    If the data region A is longer than 10 bits, the status values must be filled with the corresponding quantity of bits This setting allows you to specify which value should be accepted by the full bit.
  • Zero: it is always filled with 0.
  • One: it is always filled with 1.
  • Repeat the last bit: the last bit of the sent datum.
  • Initialization data content
    With [Edit...] you can open the Initialization Data Configuration dialog. Here you can specify which initialization data are sent by the simulated sensor in this time slot.


Signal

CANoe assigns a system variable to each PSI5 time slot. The system variable allows to access the frames sent or received in this slot. Furthermore, it is also possible to directly access the single regions for the frames (e.g. messaging bits, data region A or B).

Example: definition of three signals in a SensorFrame

Signals provide a further option to access the bits within a frame. They can be defined via any of the areas of the frame by indicating the exact number of bits that compose the offset and the length of the signal. A conversion formula (defined via factor and offset) and a unit can also be specified. CANoe generates a further system variable for each signal. This system variable allows to access the corresponding part of the frame.

PSI5 Signal Settings

Signal Layout

  • Transmitted via
    Specifies in which part of the frame the signal is transmitted. The signal can also be assigned to a serial message. In this case, the message ID must also be specified (see below).
  • Serial message ID
    If the signal is transmitted to a serial message, the message ID can be specified here.
  • Signal length
    Specifies the quantity of bits or nibbles that compose the signal.
  • Signal offset
    Specifies the bit, within the region selected for the transmission, beginning from which the signal starts.
  • Interpretation as signed value
    The received raw values can be interpreted as unsigned or signed. E.g. the 8-bit value 0xFF generates -1 (signed) or 255 (unsigned).
    This setting allows you to specify which interpretation must be used.

Display Scaling

  • Factor
    This factor allows the raw values to be multiplied before they are displayed in the corresponding system variable in CANoe. When sending values (sensor simulation), the value of the system variables is divided by this factor, before it is sent as raw value.
    This results in the following formula for
  • received values:
    received value = raw value * factor + offset
  • sent values:
    raw value = (sent value – offset ) : factor

    • If no factor should be used, it can be set to 1.

  • Offset
    The offset is used in the same way as the factor, i.e. the received values are increased by the offset. On the other hand, the offset is subtracted from the sent values.
    This results in the following formula for
  • received values:
    received value = raw value * factor + offset
  • sent values:
    raw value = (sent value – offset ) : factor

    • If no offset should be used, it can be set to 0.

  • Unit
    Specifies the system variable unit displayed in CANoe.

Simulation Settings

  • Initial signal value (scaled)
    A simulated sensor sends this signal value until a new value is manually output. This can happen, for example, by setting the system variables or the corresponding CAPL calls.
    This value must be interpreted as scaled. The reason for this is that before it is sent, it has been converted with Offset and Factor (see above).
