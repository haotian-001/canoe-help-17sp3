PSI5 Protocol

 

PSI5 (Peripheral Sensor Interface 5) is an open digital current interface for sensors. The interface is equipped with two lines supplied through the sensors. PSI5 is characterized by a high EMC safety and low radiated emissions. The communication is two-way: the data can also be sent from the ECU to the sensor.

Operating Modes [▲ back]

Depending on the topology and on the use case, PSI5 supports different operating modes. At the moment, CANoe supports the following modes:

  • Asynchronous
    The asynchronous mode is only possible with a point-to-point connection. The data exchange occurs only in one direction. A communication from the ECU to the sensor is not possible. In this mode the sensor is completely responsible for the timing. As soon as the required supply voltage is reached, the sensor will start with the transmission of data.

  • Synchronous
    The synchronous mode is used when several sensors are addressed through a bus. This is for example the case with a parallel or a daisy chain connection. This mode can also be used with a point-to-point connection.
    The sensors access the bus through the TDMA methode (Time Division Multiple Access). According to this method, each sensor has a fixed time slot (time range) within which it may send data. The synchronization is carried out by the ECU via a voltage modulation (sync pulse).

Sensor to ECU Communication [▲ back]

The data are transmitted via a current modulation. As the sensor is supplied also through the communication lines, its bias current remains constant. The sensor causes the current to increase only in case of communication. The current data are defined via a Manchester coding. A 0 corresponds to a rising current change edge, a 1 to a falling one. 125 kbit/s or 189 kbit/s are available as data rate.

Frame Structure [▲ back]

The data are transmitted in frames. After each frame it is necessary to observe a pause of at least 1 bit.

A frame always starts with two start bits which have a fixed value of 00.

Both start bits, the data region A and the CRC or parity bits are mandatory. The other frame components are optional. The sequence of the frame components is fixed and cannot be interchanged. First, the current data are always sent with LSB (least significant bit).

Error Detection [▲ back]

A 3-bit CRC or a 1-bit parity flag with event parity can be used for error detection. The 3-bit CRC should be used if more than 10 data bits are available. The checksum is calculated through the data region without considering the two start bits.

Data Interpretation [▲ back]

The data region A has a fixed data interpretation with three areas.

  • The area 1 (-480 to +480) contains the possible sensor values.
  • The area 3 (-512 to -481) contains block and data IDs and initialization values.
  • The area 2 (+481 to +511) contains status and error codes.

The data interpretation only applies to the data region A and not to other frame components, such as data region B or serial messaging channel. In case of more than 10 bits in the data region A the possible sensor value increases accordingly. The permanently defined values from the areas 2 and 3 will be maintained, but they only apply to the 10-bit of the MSB (most significant bit) of the data region A.

Serial Messages [▲ back]

PSI5 is equipped with a serial message channel in which data of more than 18 consecutive frames are sent. To do so, the messaging bits M0 and M1 of the frames are both used. Depending on the configuration, a serial message can contain an 8-bit ID and 12-bit data or a 4-bit ID and 16-bit data.

Initialization Data [▲ back]

As soon as the supply voltage is applied, a sensor will typically send initialization data to the ECU. The initialization data contain, among others, the type of sensor, a manufacturer code and the production date of the sensor. The PSI5 standard provides different options for the transmission of these data. The volume of data and the transmission method (in data areas of the frame, via serial messager) depend on the type of sensor.

ECU to Sensor Communication [▲ back]

At the moment, CANoe doesn't support the ECU to sensor communication. Detailed information about this topic can be found in the PSI5 specification (https://psi5.org/).
