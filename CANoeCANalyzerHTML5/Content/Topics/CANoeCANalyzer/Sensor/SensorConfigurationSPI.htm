Configuration of the SPI Sensor Channel

Ribbon|Hardware Tab|Protocol Configuration|SPI

The elements and the settings of the SPI sensor channel are described in this section. All system variables available for a SPI channel are also described here.

Overview of the SPI Network Elements

Element

Description/Configuration


Network

SPI Settings

No configuration settings are required.


Channel

A channel describes the interconnection between a master and one or more slaves.

All the settings that apply to the elements within a channel are configured on this level.

SPI channel settings

  • Assigned hardware channel
    Specifies which physical hardware channel (e.g. a VT2710 piggy channel) is assigned to this sensor channel. If no channels have been configured, the sensor channel will be automatically deactivated at the start of the measurement.
  • Voltage level high
    Determines the high voltage level on the SPI signal lines. Depending on the network interface used, this setting must be made elsewhere (e.g. in the CANoeVT System Configuration dialog).
  • Voltage level low
    Determines the low voltage level on the SPI signal lines. This value is permanently set to 0 V.


Master

Each SPI channel contains a master to which the slaves are connected. The master can be physically available and connected to a network interface or it can be simulated by CANoe.

SPI master simulation settings

  • Clock polarity in the idle state
    Determines which clock level is generated by the master when it does not communicate with the slave.

SPI master simulation settings (individual for each configured slave)

  • Clock frequency
    Determines the frequency of the clock signal generated by the simulated master for the corresponding slave.
  • Waiting time after chip select active
    After the simulated master has switched the chip select line to active, the master starts to transfer the clock signal and the MOSI bits. The waiting time between the activation of the chip select line and the transfer of the first bit can be individually modified for each configured slave using this setting.
  • Waiting time before chip select inactive
    After the simulated master has transferred the last bit, it switches the chip select line to inactive. The waiting time between the last transfer and the deactivation of the chip select line can be individually modified for each configured slave using this setting.


Slave

Each SPI channel contains one or several slaves which are connected to the master. They can be physically available and connected to a network interface or they can be simulated by CANoe.

Note

The chip select line assigned to a slave is configured implicitly. It is determined by the sequence of the slaves in the configuration. The first slave is assigned to the first chip select line, the second slave to the second line, etc. To modify this assignment, you can interchange the slaves in the configuration using drag and drop. You can also set slaves to Deactivated if you do not want to use the related chip select line.

SPI slave settings

General

  • Chip select mode
    Determines which level of the chip select line is interpreted by this slave as active.

Clock settings

  • Clock polarity (CPOL)
    Determines the value of the clock in the idle state.
  • Idle low
    In the idle state, the clock has the value 0 (low)
  • Idle high
    In the idle state, the clock has the value 1 (high)
  • Clock phase (CPHA)
    Determines whether the data is acquired at the first or second edge of the clock signal.

Frame

 

SPI frame settings

General

  • Frame length
    You can use this setting to specify the desired length of the frame in bits. It must contain at least 2 bits.


Signal
  • MOSI: Master Output, Slave Input
  • MISO: Master Input, Slave Output

SPI signal settings

General settings

  • Signal length
    Specifies the length of the signal. The length is specified in bits.
  • Signal offset
    Specifies where in the frame the signal starts. The offset, as well as the length, is specified in bits.
  • Sequence of bits
    Determines the sequence in which the bits of the signal are arranged.
  • LSB0: the least significant bit occupies the first position
  • MSB0: the most significant bit occupies the first position
  • Start value
    Here, you can assign a start value to the signal.
  • Interpretation with sign
  • Is a constant
    If you activate this option, the signal will be treated as a constant. Thus, this signal is, amongst other things, not available as stimulation value in CANoe.

Display scaling

  • Factor
    This factor allows the raw values to be multiplied before they are displayed in the corresponding system variable in CANoe. When sending values (sensor simulation), the value of the system variables is divided by this factor, before it is sent as raw value.
    This results in the following formula for
  • received values:
    received value = raw value * factor + offset
  • sent values:
    raw value = (sent value – offset ) : factor

    • If no factor should be used, it can be set to 1.

  • Offset
    The offset is used in the same way as the factor, i.e. the received values are increased by the offset. On the other hand, the offset is subtracted from the sent values.
    This results in the following formula for
  • received values:
    received value = raw value * factor + offset
  • sent values:
    raw value = (sent value – offset ) : factor

    • If no offset should be used, it can be set to 0.

  • Unit
    Specifies the system variable unit displayed in CANoe.
