Configuration

Ribbon|Hardware Tab|Protocol Configuration

The Option .Sensor can be configured in the Sensor Configuration dialog. This dialog allows you to create different types of channels in which sensors and ECUs can be configured. Depending on the use case, you can also select which devices are currently connected to the network interface and which ones shall be simulated through CANoe.

Adding a Sensor Channel

Via menu path Channels|Add Channel it is possible to add a new channel (PSI, SENT or SPI). Alternatively, via menu path Channels|Add Channel from Template the channel can also be selected from a library of pre-created user-specific configurations in the Add Channel from Template dialog.

Configuring a Sensor Channel

The configured channels together with the respective elements are displayed in the Sensor Channels column on the left side of the navigation tree.

Further functions are available in the context menu of the various elements, such as adding further sub-elements, deleting elements or copying and pasting elements.

Configuring Elements of the Sensor Channel

The settings of the selected elements are displayed on the right side.

  • Configuration of the PSI5 Sensor Channel
  • Configuration of the SENT Sensor Channel
  • Configuration of the SPI Sensor Channel
  • Configuration of the UART, RS485 and LVDS Sensor Channel
  • Configuration of the I2C Sensor Channel

The settings area on the right side consists of two sections. In the upper section there are general settings relevant for the real as well as simulated mode. In the lower section there are simulation settings which are relevant only if the sensor or the ECU has to be operated in the simulated mode. These settings are marked with the symbol .

Selecting the Operating Mode

The operating mode of the ECUs and sensors is displayed on the left side in the Mode column.
Depending on the ECU or sensor, the following modes are available:

  • Real: the ECU or sensor is a real device connected to the network interface.
  • Simulated: the ECU or sensor is simulated by CANoe.
  • Deactivated: the ECU or sensor is neither connected to the network interface nor simulated by CANoe.

These settings allows you to execute different functions:

  • Testing an ECU
    Connect the ECU to the network interface and set the operating mode of the ECU to real. The sensor operation mode is set to simulated. CANoe and your test cases allow you to have full control of the behavior of the sensor and of the data sent by the sensor. The simulated sensor also enables to evaluate the data sent by the ECU.
  • Testing a Sensor
    Connect the sensor to the network interface and set the operating mode of the sensor to real. The ECU simulation mode is set to simulated. CANoe and your test cases allow you to have full control of the behavior of the ECU and of the data sent by the ECU. The simulated ECU also enables to evaluate the data sent by the sensor.
  • Analyzing a Sensor Network
    Connect the ECU and the sensors to the network interface. Set both operating modes to "real". This allows the communication between both devices to be logged and analyzed into CANoe.

The deactivated mode can be appropriate if you want to switch often between two different configurations of a sensor. In this case, you can activate the desired configuration and deactivate the other.

Assigning the Hardware Channels

Before starting the measurement in CANoe, a physical hardware channel (e.g. on a VT2710 module) must be assigned to each sensor. The procedure to apply depends on the hardware used.

Step by Step Procedure for the VT2710 Module

  1. To configure the VT2710 for the usage with the Option .Sensor click on Hardware Configuration... from the menu of the Sensor Configuration dialog.
  2. If the VT System in CANoe hasn't been activated yet, the Options dialog opens. Activate the VT System and select the networks cards to be used. Quit the dialog with [OK].
  3. The VT System Configuration dialog opens. By clicking on Configuration|New... a search for the connected VT2710 will be started automatically. This should now be displayed in the list of the configured modules. If the module cannot be found, ensure that the VT System is correctly connected to the computer and that it is switched on. Also check if in the Options dialog the correct network card has been selected. If the module has been successfully configured, quit the dialog with [OK].
  4. The channels of the VT2710 module are now available for the Option .Sensor. If you select a sensor channel in the navigation tree, the assigned hardware channel can be selected in the settings area on the right side.

Exporting and Importing Configurations

You can export your sensor configurations into an XML file. This file can then be imported and re-used in other CANoe configurations. To do so, select menu path Configuration|Export... or Configuration|Import....

Alternatively, you can also export or import only the configuration of a certain element via the context menu of the elements of the navigation tree.
