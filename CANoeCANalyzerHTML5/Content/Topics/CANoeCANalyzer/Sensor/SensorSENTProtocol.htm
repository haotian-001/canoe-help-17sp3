SENT Protocol

 

SENT (Single Edge Nibble Transmission) is a digital voltage interface between a sensor and an ECU. The connection is one-way: the data are only transferred from the sensor to the ECU. SENT is intended to allow you to implement sensors in the most cost effective manner. This is possible thanks to a reduced pin number, a simple protocol and low timing requirements.

Signal Lines [â² back]

A SENT sensor is usually equipped with only three connections:

  • supply voltage (5V)
  • a signal line
  • GND

The sensor is connected to the corresponding pins on the ECU.

The high/low signal levels are clearly defined: the minimum voltage for the high level is 4.1 V; for the low level the voltage must be inferior to 0.5 V.

Frame Structure [â² back]

As soon as the supply voltage is applied to the sensor, the sensor will begin to cyclically transmit his data. The data are transmitted in nibbles (4-bit) whose value is codified using the length between two falling edges. The sent frames have always the same structure that must be known to the ECU.

A frame starts with the sync pulse (56 ticks) followed by the status and communication nibbles. These are then followed by 1 to 6 data nibbles (12-162 ticks) and by the CRC checksum (12-27 ticks). At the end of the frame it is possible to set an optional pause time of 12-768 ticks after which the next frame begins.

Nibble Structure [â² back]

Each nibble starts with a falling edge that must remain low for at least 5 ticks. The time of the next falling edge determines the current nibble value. The next falling edge must occur not earlier than after 12 ticks which corresponds to a nibble value of 0. Any additional delay of the falling edge of one tick causes the nibble value to increase by 1. A nibble must not be longer than 27 ticks which corresponds to the maximum value of 15.

Usage of the Pause [â² back]

Some sensors use the pause function at the end of a frame in order to create a fixed frame length. This enables to reach a constant cycle time in case of changing data by dynamically adapting the pause time. Other sensors don't use the pause function or they only have a fixed number of pause ticks. Whether the sensor uses the pause function or not must be known to the connected ECU before the transmission is started.

Serial Messages [â² back]

With bit 2 and 3 of the status nibble the sensor can send further data to the ECU in form of serial messages. These can contain additional information regarding the sensor, such as temperature, error codes or type information. The data can be static (e.g. type information) or dynamic (e.g. temperature). Two different data formats are available for these messages:

  • Short Serial Message

    • The transmission of a short serial message starts when the status bit 3 has the value 1. After that, the status bit 3 must have the value 0 for 15 times. The current data are transmitted in status bit 2. These consist of a message ID, a data byte and a 4-bit CRC checksum.

  • Enhanced Serial Message

    • This format is an extension of the format Short Serial Message available as an option. In this format the data are transmitted distributed over 18 frames instead of 16. The data are transmitted in status bits 2 and 3. In status bit 3 the start of a message is identified by the unique pattern 01111110. The first 1 in the pattern determines the start of the frame. The bits 7, 13 and 18 in status bit 3 must always be 0. The data of a message are interpreted differently according to the value of the configuration bits (bit 8 of status bit 3). In both cases the CRC checksum is 6 bits long.

    If the configuration bit is 0, the data are 12 bits long and the message ID is 8 bits long. In contrast, if the configuration bit is 1, the data can be up to 16 bits long and the message ID is only 4 bits long.
