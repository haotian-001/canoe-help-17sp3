Configuration of the I2C Sensor Channel

Ribbon|Hardware Tab|Protocol Configuration|SPI

The elements and the settings of the I2C sensor channel are described in this section. All system variables available for a I2C channel are also described here.

Overview of the I2C Network Elements

Element

Description/Configuration


Network

I2C Settings

No configuration settings are required.


Channel

A channel describes the interconnection between a master and one or more slaves.

I2C channel settings

  • Assigned hardware channel
    Specifies which physical hardware channel (e.g. a VT2710 channel) is assigned to this sensor channel. If no channels have been configured, the sensor channel will be automatically deactivated at the start of the measurement.
  • Bitrate
    Specifies the bitrate of the communication on this channel.
  • Generate low level events
    When this option is activated, additional system variable events will be generated. This is especially useful when analyzing the exact timing on the bus (e.g. clock stretching).
    Note: enabling this option may greatly increase the CANoe event load.
  • Generate raw frame events
    When this option is activated, CANoe will generate additional raw frame system variable events for every transmitted I2C frame.
  • Voltage level high
    Determines the high voltage level on the I2C signal lines. Depending on the network interface used, this setting must be made elsewhere (e.g. in the CANoeVT System Configuration dialog).
  • Voltage level low
    Determines the low voltage level on the I2C signal lines. This value is permanently set to 0 V.


Master

Each I2C channel contains a master to which the slaves are connected. The master can be physically available and connected to a network interface or it can be simulated by CANoe.

I2C master simulation settings

No configuration settings are required.


Slave

Each I2C channel contains one or several slaves which are connected to the master. They can be physically available and connected to a network interface or they can be simulated by CANoe.

I2C slave settings

General

  • Address mode
    Indicates if the slave uses a 7 bit or a 10 bit address.
  • Address
    Specifies the slave’s address.
  • Acknowledge general call address
    If you enable this option the simulated slave will acknowledge any request from the master to the general call address.

Frame

 

I2C frame settings

General

  • Length of write sequence
    This setting specifies the length of the frame's write sequence in bytes. If set to zero the frame will have no write sequence.
  • Length of read sequence
    This setting specifies the length of the frame's read sequence in bytes. If set to zero the frame will have no read sequence.


Signal

 

I2C signal settings

General settings

  • Signal length
    Specifies the length of the signal. The length is specified in bits.
  • Signal offset
    Specifies where in the frame the signal starts. The offset, as well as the length, is specified in bits.
  • Sequence of bits
    Determines the sequence in which the bits of the signal are arranged.
  • LSB0: the least significant bit occupies the first position
  • MSB0: the most significant bit occupies the first position
  • Start value
    Here, you can assign a start value to the signal.
  • Is a constant
    If you activate this option, the signal will be treated as a constant. Thus, this signal is, amongst other things, not available as stimulation value in CANoe.

Display scaling

  • Factor
    This factor allows the raw values to be multiplied before they are displayed in the corresponding system variable in CANoe. When sending values (sensor simulation), the value of the system variables is divided by this factor, before it is sent as raw value.
    This results in the following formula for
  • received values:
    received value = raw value * factor + offset
  • sent values:
    raw value = (sent value – offset ) : factor

    • If no factor should be used, it can be set to 1.

  • Offset
    The offset is used in the same way as the factor, i.e. the received values are increased by the offset. On the other hand, the offset is subtracted from the sent values.
    This results in the following formula for
  • received values:
    received value = raw value * factor + offset
  • sent values:
    raw value = (sent value – offset ) : factor

    • If no offset should be used, it can be set to 0.

  • Unit
    Specifies the system variable unit displayed in CANoe.
