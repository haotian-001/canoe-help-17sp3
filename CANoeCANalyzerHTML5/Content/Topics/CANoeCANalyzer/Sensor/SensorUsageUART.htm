Overview of the UART, RS485 and LVDS System Variables [▲ back]

If you have configured an UART, RS485 or LVDS network in the Sensor Configuration dialog, CANoe automatically creates the corresponding system variables. As all three interfaces communicate by using the UART protocol, the system variables in the namespaces are identical.

Communication with an UART Node [▲ back]

CANoe offers the possibility to communicate with another node per UART protocol via different interfaces (serial interface, RS485 and LVDS).

Sending UART Frames [▲ back]

There are two ways sending UART frames from CANoe:

  • Sending via system variable
    To do so, modify the value of a Frame_Stim variables, for example via CAPL or in a panel. This triggers the sending of a corresponding frame.
  • Sending via CAPL command
    To do this, call up one of the QueueSerialFrame/sensorQueueSerialFrame or QueueSerialFrames/sensorQueueSerialFrames functions from the namespace of the channel. You can send data to these functions so that they are then added to one or more frames of the send queue of the node.

Note

To avoid Serial Error Frames, RS485 should be operated in a bias network where Rx and Tx are pulled up/down to the correct levels. Suitable pull-up/pull-down resistances are 470 Ω on each line.
