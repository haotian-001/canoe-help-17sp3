Union Data Types

 

Union data types are declared using the union keyword followed by the data type name and a body declaration in curly braces. In the body, the union members are declared in any order, specifying a data type and a name for the member.

Example: Union definition with mixed data types.

union Location

{

  Position RelativePos;

  Position AbsolutePos;

  float Distance;

}

Additional attributes allow to explicitly identify a union discriminator in a surrounding struct data type and to specify discriminator value ranges for each union member. Analogous to struct members, union members can be augmented by all attributes defined for their data type.

Example: Union definition with explicit discriminator settings.

union Location
{
  Position RelativePos [discriminatorValues={0,3}];
  Position AbsolutePos [discriminatorValues={4,6}];
  float Distance       [discriminatorValues=7];
}

struct LocContainer
{
  int16 ValidMember;

  [discriminator=ValidMember]
  Location TheLocation;
}

If no explicit discriminator values are specified, then each member shall have a default discriminator value associated, which matches its index in the union declaration. If any member has an explicit discriminator value, then all members shall have an explicit value. The discriminator values can be expressed either as a single value or as a series of ranges, where every two consecutive values are the lower and upper limit of an inclusive range.

Each union definition implies a new data type, i.e. the data type name can be used like built-in primitive data types. Union definitions always must be named and cannot be inlined with a member, communication object or parameter declaration.

Union data types must be declared at namespace level, i.e. their declaration cannot be nested in other complex types, services or interfaces.

Supported Attributes (Union)

Union declarations support the following attributes:

Name

Type

Default

Semantics

byteOrder

String

"Intel"

Specifies whether values and length fields are serialized in Motorola or Intel byte order.

fixLayout

Bool

false

Specifies whether the data type has a fix binary layout. Requires an explicit discriminator and all members must have fix layout data type.

bitCount

UInt

Largest field's bitcount.

Size of the data type in bits. If not specified, it will be calculated from the union’s largest field. (only if fix layout is specified)

rawBitCount

UInt

Largest field's raw bitcount.

Size of the data type’s raw representation in bits. If not specified, it will be calculated from the union’s largest field. (only if fix layout is specified)

Union attributes may be placed before the union keyword only.

Supported Attributes (Union Member)

Union members support the following attributes:

Name

Type

Default

Semantics

discriminatorValues

[UInt]

Member index

Identifies explicit discriminator values which select the according member.

Union member attributes and attributes for inlined union member data types may be placed before the member data type or directly before the member delimiter.
