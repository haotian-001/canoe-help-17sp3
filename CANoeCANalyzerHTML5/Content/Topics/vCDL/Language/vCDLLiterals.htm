Literals

 

vCDL often requires you to initialize elements such as attributes or constants with certain values. These are either single literals or expressions of multiple literals. Each data type in vCDL has its own type of literal. Following table outlines the different literals:

Data Type

Example Literals

Additional Information

bool

true, false

Boolean literal

int

65, 0x4f, 0b1001

Integer literals ca be written either binary, decimal or hexadecimal.

float, double

4.5

Decimal float literal

.067e-10

Decimal float literal with exponent (base 10) in scientific notation

0xE.3FA2p+7

Hexadecimal float literal with exponent (base 2) in scientific notation

time

56s

Available units are s, ms, us, ns, ps, fs and as

enum

BLUE, off, LONGER_VALUE

Enumeration literals are always written without the enumeration type. The vCDL parser derives the referred enumeration from the literal’s context.

string

"Description\n"

Standard string: Special characters need to be escaped with a backslash, in particular the backslash itself.

@"\ Description \"

Verbatim string: There is no character escaping. A single backslash remains a single backslash.

array, list

{ "first", "second", "third" }

Full array/list literal: All elements must match the array’s/list’s data type. For arrays the number of elements must match the array’s length, for lists it must lie within the minimal and maximal length.

{ 2: "third", 0: "first"}

Sparse array/list literal: Not all elements need to be specified. The indices are 0-based and can be out of order but must be within the array’s or list’s (maximum) length.

struct

{ text: "Description", value: null }

Sparse struct literal: Not all members need to be initialized and those, who are initialized, can be in different order than within the struct declaration. Optional struct members can be set to null.

{ "Description", null }

Full struct literal: All members must be initialized in the same order as they have been declared. Optional struct members can be set to null.

union

{ AsString: "Description" }

Like the sparse struct literal, but only a single member must be initialized.

Of course, nested complex data types such as multidimensional arrays or structs using other structs as member can be initialized as well by simply nesting the corresponding literals.

Example: Nesting complex literals

struct InnerType
{
  [optional]
  string Name;
  double Value;
}

struct ComplexType
{
  bool      BasicMember;
  InnerType ComplexMember1;
  InnerType ComplexMember2;
}

const ComplexType ComplexObject =
{
  BasicMember:     true,
  ComplexMember1: { "AnyName", 4 },
  ComplexMember2: { null, 5.6 }
};

const array<array<int32, 4>, 4> NestedArray =
{
  1: {5, 8, -67, 4},
  2: {3, 0, -20, 7}, // last comma is optional
};

const array<array<int32, 4>, 4> NestedArray2 =
{
  {3, 11, -24, 1},
  {5,  8, -67, 4},
  {3,  0, -20, 7},
  {5, 15, -35, 8}, // last comma is optional
};

Note

Although both the struct declaration and the constant end with a curly bracket, there must be a delimiter after the constant while there must be no delimiter after the struct declaration. The main reason is that the curly bracket is part of the struct declaration itself, while for constants there may often be basic literals as right-hand-side without any closing curly bracket at the end which makes the delimiter necessary.
