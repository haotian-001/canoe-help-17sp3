Struct Data Types

 

Struct data types are declared using the struct keyword followed by the data type name and a body declaration in curly braces. In the body, the struct members are declared in order, specifying a data type and a name for the member.

Example: Struct definition with primitive data type members.

struct Position
{
  float x;
  float y;
  float z;
}

Struct members can be declared optional and may specify a bit offset relative to the end of the predecessor or beginning of the struct (attribute keywords optional and relOffset). In addition, primitive type members can use attributes from semantic type definitions (e.g. unit, min, max, …).

Example: Struct definition with complex data type member and additional attributes.

struct Sample
{
  Position Pos;
  float Timestamp [optional, unit="ns", relOffset=32];
}

Each struct definition implies a new data type, i.e. the data type name can be used like built-in primitive data types. Struct definitions always must be named and cannot be inlined with a member, communication object or parameter declaration.

Struct data types must be declared at namespace level, i.e. their declaration cannot be nested in other complex types, services or interfaces.

Supported Attributes (Struct)

Struct declarations support the following attributes:

Name

Type

Default

Semantics

byteOrder

String

"Intel"

Specifies whether values and length fields are serialized in Motorola or Intel byte order

fixLayout

Bool

false

Specifies whether the data type has a fix binary layout (requires all members to have a fix layout data type).

bitCount

UInt

Sum of fields’ bitcount.

Size of the data type in bits. If not specified, it will be calculated from the struct’s fields. (only if fix layout is specified)

rawBitCount

UInt

Sum of fields’ raw bitcount.

Size of the data type’s raw representation in bits. If not specified, it will be calculated from the struct’s fields. (only if fix layout is specified)

Struct attributes may be placed before the struct keyword only.

Supported Attributes (Struct Member)

Struct members support the following attributes:

Name

Type

Default

Semantics

optional

Bool

false

Marks a struct member as optional.

relOffset

UInt

0

Specifies an offset of a struct member relative to the end of its predecessor (or the beginning of the struct in case of the first member) in bits.

discriminator

Reference

None

Only supported on union data type members. References an integer data type struct member to be used as the union’s discriminator.

Struct member attributes and attributes for inlined struct member data types may be placed before the member data type or directly before the member delimiter.
