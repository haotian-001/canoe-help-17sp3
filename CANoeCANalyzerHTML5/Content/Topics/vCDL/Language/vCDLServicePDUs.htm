Service PDUs

 

Service PDUs are simply PDUs declared within a service interface. They reference a PDU type and thus specify a layout for data elements to be transmitted with the PDU. A service PDU is declared by the keyword pdu, followed by a PDU type reference and the PDU name. The statement is finished by a semicolon.

Example: Explicit PDU type declaration and service PDU.

pdutype TrackData
{
  string [maxBytes=32] Title;
  uint16 TrackNumber;
  float [unit="s", relOffset=16] Length;
}

interface IMediaSource
{
  pdu TrackData CurrentTrackData;
}

The PDU type declaration can be inlined with the PDU declaration. In that case the PDU type reference is omitted and the statement is finished by a body declaration identical to the PDU type declaration body.

Example: Service PDU with inlined PDU type declaration.

interface IMediaSource
{
  pdu CurrentTrackData
  {
    string [maxBytes=32] Title;
    uint16 TrackNumber;
    float [unit="s", relOffset=16] Length;
  }
}

Service PDUs must be declared at service interface level or at service level when the interface is inlined.

Supported Attributes

Service PDU declarations support the following attributes:

Name

Type

Default

Semantics

txValueRepr

String

"Default"

Whether transmitted signal values should be stored in Default, Impl or Raw representation. Can be overridden per endpoint.

Service PDU attributes may be placed before a PDU declaration. If the PDU type is not inlined, the attributes may as well be placed before the delimiter.
