Secure Onboard Communication

 

The Secure Onboard Communication shall provide authentication and integrity protection for sensitive data transmitted over the network. It is similar in nature to E2E protection and accordingly the language integration is comparable as well. There is an explicitly named profile definition, introduced by the keyword secoc followed by the name of the profile. Profile parameters are specified as attributes on the profile definition.

Example: Secure Onboard Communication profile definition.

[ authAlgorithm="CMAC/AES-128",
  authInfoTxLength=28,
  authenticationRetries=3,
  freshnessTimestampPeriodFactor=1,
  freshnessValueLength=64,
  freshnessValueTxLength=4,
  useFreshnessTimestamp=true]
secoc SecOCProfile3;

A defined profile can then be referenced from PDUs, Service PDUs and SOME/IP events and fields by attribute secOCProfile. Additional attributes allow individual settings per communication object instance. All these attributes can already be attached to the corresponding type definitions (e.g. a PDU type definition) and can be refined at the instance level.

Example: PDU refinement with Secure Onboard Communication parameters.

[ secOCDataId=17,
  secOCFreshnessValueId=72
  secOCProfile=SecOCProfile3];
refine EngineControl.EngineInfo

The parameters are designed along the AUTOSAR Secure Onboard Communication specification and valid values are depending on the selected profile.

Type

authInfoTxLength

UInt

24

Number of (most significant) bits to transmit from the authentication info.

UInt

0

Number of authentication verification attempts.

UInt

0

UInt

Whether to use timestamps as freshness values.

Supported authAlgorithm attribute values

The authAlgorithm attribute is enumerated, allowing the following values.
