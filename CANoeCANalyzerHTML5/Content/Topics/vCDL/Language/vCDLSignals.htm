Signals

 

A signal is declared by the keyword signal followed by the signal’s data type and name. The body of a signal declaration contains sender and receiver endpoints.

Example: Primitive type and complex type signal.

struct EngineInfoType
{
  float Temperature;
  float OilPressure;
  uint16 RPM;
}

signal uint16 Speed
{
  sender WheelFL, WheelFR, WheelBL, WheelBR;
  receiver Dashboard;
}

signal EngineInfoType EngineInfo
{
  sender Engine;
  receiver Dashboard, Navigation;
}

Signals can be attributed with PDU mappings, which implies that the signal is only sent via the mapped PDU and according endpoints. The signal attribute mappings contains a list of references to the PDUs to which the signal shall be mapped and the same attribute at signal endpoints lists the PDU endpoints which send or receive the signal. A signal must not be mapped to multiple members of the same PDU and if none of its endpoints is mapped to an endpoint of the same PDU, the mapping has no effect. Further it is necessary to map signal sender endpoints to PDU sender/provider endpoints only and signal receiver endpoints to PDU receiver/consumer endpoints only.

Example: Inline PDU definition with mapped signal.

pdu DoorInfo
{
  bool   IsClosed;
  bool   IsLocked;
  uint16 WindowPosition;

  sender   DoorFL, DoorFR, DoorBL, DoorBR;
  receiver BodyController;
}

[mappings=DoorInfo.WindowPosition]
signal uint16 DoorFL_WindowPosition
{
  sender   DoorFL         [mappings=DoorInfo.DoorFL];
  receiver BodyController [mappings=DoorInfo.BodyController];
}

Signals must be declared at namespace level, i.e. they cannot be nested in other complex types, interfaces or services.

Supported Attributes

Signal declarations support the following attributes:

Name

Type

Default

Semantics

txValueRepr

String

"Default"

Whether transmitted signal values should be stored in Default, Impl or Raw representation. Can be overridden per endpoint.

broadcast

Bool

false

Whether the signal is broadcast to all receiver endpoints when being sent.

mappings

{Reference}

Empty

Lists PDUs and/or service PDUs to which the signal shall be mapped.

Signals attributes may be placed before keyword signal only.
