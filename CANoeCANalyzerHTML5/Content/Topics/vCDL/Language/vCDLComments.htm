Comments

 

Comments are supported as line style and block comments similar to C++ and C#. A line comment starts with a double slash // and stops before the next newline character. Block comments start with the token /* and end with the next */ token. These comments are meant for authors of t he vCDL file only and are usually discarded by components that import this file. However, managed line comments starting with /// or //! and managed block comments starting with /** or /*! (both closing with */) are tried to be kept by the importer and can be used for documentation of the data model represented in the vCDL file.

Note

If you have used comments in vCDL files prior to version v1.2 and still want them to be visible after import into CANoe, you must update all of them to managed comments!

If the closing */ for block comments is missing, then the source text is malformed and importing it will show an error. Block comments cannot be nested, i.e. any text containing /* within a block comment shall not be considered yet another block comment start, but simple text within the outer block comment.

Comments are associated with specific statements by vicinity rules. Statements are either delimited by a semicolon or by a closing curly brace in case of a block statement. Each comment is associated with the current or following statement except if a comment directly follows a statement delimiter on the same line (ignoring whitespace), in which case it is associated with that statement.

Example: Multiple comment locations and their association with statements.

// comment 1.1
statement1 // comment 1.2
{
  // comment 2.1 managed
  statement2 /* comment 2.2 managed */; // comment 2.3
  /* comment 3.1 */ statement3;
} // comment 1.3 managed
