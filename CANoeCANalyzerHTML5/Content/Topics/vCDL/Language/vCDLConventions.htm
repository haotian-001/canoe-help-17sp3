Although we strongly recommend using the vCDL Editor, you can get almost the same syntax highlighting by importing the language definition file (vCDL_1.3.xml) into Notepad++. However, this gives you the highlighting only without any advanced features such as code completion.

The documentation of individual language elements lists supported attributes each with the required value data types. In case a specific attribute’s value data type is a collection type, the element type of the collection is noted in square brackets [] if ordered list semantics are implied and in curly braces {} if set semantics are implied.

The language is case sensitive regarding all keywords, attribute keys, binding namespaces, primitive data types and the encoding formula variable. Identifiers consist of underscores, letters and decimal digits, starting with an underscore or a letter. Verbatim identifiers are supported, enabling the use of language keywords as identifiers. In that case the identifier must be prefixed with the @ character. Identifiers are as well case sensitive, i.e. all references must exactly match the declaration names.

Whitespace, especially newline, is in general ignored (except for terminating line style comments) but any kind and amount is required for separating identifiers and keywords.
