PDU Types

 

The layout of a PDU can be defined as a named PDU type, which consist of the list of signals being transported by the PDU and their layout information. The syntax is identical to a struct declaration but the supported set of attributes is different. A PDU type declaration starts with keyword pdutype followed by the type name and the body definition, listing the signals in the PDU.

Example: Explicit PDU Type declaration.

pdutype HealthInfo
{
  bool IsAlive;
  uint16 Age;
  uint16 LastCalibration;
}

The PDU name cannot be used like a generic data type definition but only for the declaration of a PDU or a service PDU. If there is no need to reuse the definition, then the type definition can be inlined in a PDU or service PDU declaration.

Example: Inlined PDU Type declaration with layout attributes.

[bitCount=64]
pdu HealthInfo
{
  bool IsAlive;
  [relOffset=7]
  uint16 Age;
  uint16 LastCalibration;
}

PDU types must be declared at namespace level, i.e. they cannot be nested in other complex types, interfaces or services.

Supported Attributes (PDU Type)

The PDU type declaration supports the following attributes:

Name

Type

Default

Semantics

byteOrder

String

"Intel”

Specifies whether values are serialized in Motorola or Intel byte order.

fixLayout

Bool

false

Specifies whether the PDU type has a fix binary layout. Requires that all member types and signal groups have a fix layout as well.

bitCount

UInt

Sum of fields’ bitcount.

Size of the data type in bits. If not specified, it will be calculated from the PDU’s fields. (only if fix layout)

rawBitCount

UInt

Sum of fields’ raw bitcount.

Size of the data type’s raw representation in bits. If not specified, it will be calculated from the PDU’s fields. (only if fix layout)

PDU type attributes may be placed before the pdutype keyword only.

Supported Attributes (Signal Member)

PDU type signal members support the following attributes:

Name

Type

Default

Semantics

relOffset

UInt

0

Specifies an offset of a PDU member relative to the end of its predecessor (or the beginning of the PDU in case of the first member) in bits.

PDU type signal member attributes and attributes for inlined PDU type member data types may be placed before the signal member data type or directly before the delimiter.
