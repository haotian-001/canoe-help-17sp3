Namespaces

 

Namespaces allow grouping top-level declarations like interfaces, services, functions, signals, data types, constants and alias names. Similar to C++, names need to be unique within the local namespace only and references to other declarations need absolute or relative qualification (including none for local references). A namespace is declared with the keyword namespace followed by its name and a body in curly braces, containing declarations belonging to the namespace. Namespace declarations can be nested, thus spanning a tree of named elements which can be qualified starting from the outermost namespace through nested namespaces up to the element name itself.

Example: Nested namespace declaration with service interface.

namespace Multimedia
{
  namespace Audio
  {
    interface IAudioSource
    {
      field uint16 CurrentTrack { get; set; notify; }
    }
  }
}

Namespace declarations may be reopened with different content, i.e. it is not necessary to declare all elements of a specific namespace within one and the same block. The content of repeated namespaces is simply considered as a single collection of declarations, all located within that namespace.

Similar to programming languages references into a namespace can be facilitated by a using statement. These statements can appear globally or within a namespace and have priority over parent namespaces when resolving references. It does not make a difference whether a using statement is located before or after a reference in the same block – the using statement is always considered valid throughout the whole block.

Example: Using statement to avoid lengthy references.

namespace DataTypes
{
  struct Position
  {
    float x;
    float y;
    float z;
  }
}

[byteOrder="Motorola", fixLayout]
namespace Navigation
{
  using ::DataTypes;
  interface INavigator
  {
    field Position CurrentPosition { get; set; }
  }
}

Namespaces can only be nested in other namespaces, but never in other block statements like interface or service declarations.

Supported Attributes

Although namespace declarations themselves do not support any attributes, they may be augmented with specific attributes valid for declarations within the namespace. Doing so will provide a default value applied to all declarations within the namespace, which do not have an individual setting for the same attribute. The attributes must be placed before the namespace keyword.

Name

Type

Default

Semantics

byteOrder

String

"Intel”

Specifies whether values are serialized in Motorola or Intel byte order.

fixLayout

Bool

false

Specifies whether the data type has a fix binary layout.

encoding

String

"UTF-8”

Selects a character encoding for strings from a predefined set of encodings. Supported are UTF-8, UTF-16, ISO 8859-1, ISO 8859-2 and Windows 1252.
