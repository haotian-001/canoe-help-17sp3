Attribute Refinements

 

To be able to split the pure communication description from binding specific settings or for overriding attribute settings (e.g. per endpoint) the refine keyword allows to reference any named objects and members or endpoints of those. The statement itself does not carry further semantic but allows to redefine or add attributes to the referenced entity. In case a combination of endpoint and member is referenced then the refined attributes are only applied for that combination, thus allowing to specify individual attribute values per endpoint and member.

Example: Service adding attributes to members declared in the implemented service interface.

namespace Multimedia
{
  namespace Audio
  {
    interface IAudioSource
    {
      field uint16 NumberOfTracks { get; }
      field uint16 CurrentTrack { get; set; notify; }
      void NextTrack();
      void PrevTrack();
    }
  }

  [inParamsRepr="Raw", txValueRepr="Impl"]
  service CDPlayer : Audio::IAudioSource
  {
    [inParamsRepr="Impl"]
    consumer Console;

    [inParamsRepr="Default", getterID=1]
    refine NumberOfTracks;

    [inParamsRepr="Raw"]
    refine Console.NumberOfTracks;
  }
}

Refinements may appear globally, on namespace level or within complex data types, within communication objects or within distributes objects. The latter can only refine members within the communication object/distributes object. If there is a member within the communication object/distributes object with the same name as objects outside of the communication object/distributes object, the refinement will always refer to the member without generating any warning.

Any reference to communication object members or endpoints from within the communication object body can be explicitly qualified with keyword this to be able to distinguish the referenced member or endpoint from an independent object which could be a sibling to the communication object (e.g. a constant definition). If there is no such ambiguity, the qualification can be omitted.

Supported Attributes

Refinement statements support all attributes of the referenced object.
