Integration in CANoe

 

The integration of vCDL in CANoe is carried out in the same way as for other data sources. vCDL files can be selected and imported via the Data Sources Import Wizard. It is not necessary to select all dependencies – the roots of the dependency graphs are sufficient. It is recommended to create a hierarchy between source files, for example communication objects, data types and bindingsNetwork connection of communication objects, defines the communication stack and its parameters.. In particular, if the binding is defined in a completely separate way, it can be exchanged more easily: either by adapting an import statement for the binding or vice versa, if the binding source file serves as anchor for the import into CANoe and imports the definition of communication objects and data types implicitly. This separation is particularly convenient when, for example, an abstract binding is used at first and then it is necessary to switch to a concrete binding.

When importing data sources by means of implicit imports, it must be noted that conflicts may arise if the same file is imported several times via the dependency graphs. In this case, the files directly imported into CANoe must be imported as a group.

Once import has been successfully completed, the defined communication objects are available for further editing in the Model Editor and for the use in application models.
