Functional Scope

 

In the current release the complete functional scope of vCDL is not available.

Restrictions

  • Communication objects can not be configured with PDU bindingNetwork connection of communication objects, defines the communication stack and its parameters..
  • For editing vCDL files it is recommended to use the provided vCDL Editor, because only there correct syntax highlighting, autocompletion and error display are available at programming time.
  • No objects from an external data source (e.g. an AUTOSAR extract) can be referenced.

New in Version 1.1 (12.0 SP3)

  • E2E protection
  • PDU timings
  • Secure onboard communication
  • Signal groups

New in Version 1.2 (13.0)

  • Distributed objects (DOs)
  • Virtual networks
  • New distributed object interfaces
  • New enumeration attribute value and custom attributes
  • Changed comment syntax
  • New supported attributes for SOME/IP binding: mac (SOME/IP endpoint) and stringTermination (global)

New in Version 1.3 (14)

  • DO references and DO container
  • Initial values for distributes objects
  • Constants and literals also for complex data types
  • New possibilities for the definition of network communication

New in Version 1.4 (15)

  • Preprocessor for conditional compilation
  • Initial values at DO interfaces
  • More options for initializing DO containers

  • Attributes returnDelay and serverSimulatorMode for DO methods and DO fields
