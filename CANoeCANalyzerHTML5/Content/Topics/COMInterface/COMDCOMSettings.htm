Settings for Distributed COM

 

Notice!

Before editing DCOM settings, a system backup should be done in any case, because wrong settings can make Windows unusable!

In order to use COM in a distributed environment, the windows firewall must be deactivated and the DCOM settings must be configured on both the server computer (i.e., the computer where CANoe runs) and the client computer (i.e., the computer where the COM application runs).

Editing DCOM Settings

DCOM settings can be edited with the program dcomcnfg:

  1. From the Start menu, click Run and type dcomcnfg.EXE.

  2. Click Console root, expand Component Services, expand Computers, and then click My Computer. On the Action menu, click Properties.

  3. In the Default Properties page, the Enable Distributed COM on this computer check box must be activated, the Default Authentication Level must be set to Connect, and the Default Impersonation Level must be set to Identify.

  4. In the COM Security page, in the Access Permissions section, click Edit limits…

  5. Add the user ANONYMOUS and give him permissions for local access and remote access.

  6. Add the user INTERACTIVE and give him permissions for local access.

  7. Close the Access Permissions dialog.

  8. In the COM Security page, in the Launch and Activation Permissions section, click Edit limits…

  9. Add the user ANONYMOUS and give him all possible permissions.

  10. Add the user INTERACTIVE and give him permissions for local launch and local activation.

  11. Close the Access Permissions dialog and the My Computer Properties dialog.

The following steps only have to be performed on the Server Computer:

  1. In the Component Services dialog, expand DCOM Config and click Vector CANoe application.
    On the Action menu, click Properties.

  2. On the Identity page, select the Interactive User.

  3. On the Security page, in the Launch and Activation Permissions section, select Customize and click Edit…

  4. Add the user ANONYMOUS and give him all possible permissions.

  5. Add the user INTERACTIVE and give him permissions for local launch and local activation.

  6. Close the Launch Permissions dialog.

  7. On the Security page, in the Access Permissions section, select Customize and click Edit…

  8. Add the user ANONYMOUS and give him permissions for local access and remote access.

  9. Add the user INTERACTIVE and give him permissions for local access.

  10. Close all dialogs.

The following steps only have to be performed on the Client Computer:

  1. You must register the CANoe typelib on the client Computer with RegisterComponents.exe /tlb <FullPathOf>\CANoe.tlb.
    It is important to use the full path and not a relative path, otherwise you will still get errors.

  2. Copy the files CANoe.tlb and RegisterComponents.exe from CANoe PC (<full path to CANoe installation>\Exec32\COMdev\CANoe.tlb) to Client PC (use a permanent valid path e.g.: C:\CANoe\type_library, because the link is saved in the PC's registry during registration)

  3. Run RegisterComponents.exe with /tlb and full path of CANoe.tlb as parameter in a command shell with admin rights ("C:\CANoe\type_library\RegisterComponents.exe /tlb C:\CANoe\type_library\CANoe.tlb")

    For unregistering a CANoe.tlb, which got registered with this workflow, you should use the command C:\CANoe\type_library\RegisterComponents.exe /u /tlb C:\CANoe\type_library\CANoe.tlb.
