Computer Configuration

 

When CANoe is installed all necessary steps are taken to properly configure your computer.
Summarized below are the requirements for utilizing the COM Interface.

Registration of the COM Interface

The COM Interface is already registered when the application is installed.

Step by Step Procedure: Registering COM

  • If you move the installation directory, a new registration is necessary:
  1. Open the MS-DOS prompt
    (e.g. Windows Start menu: All Programs|<CANoe folder>|Tools).
  2. Change the directory to the Exec64 folder of the installation directory.
  3. Execute RegisterComponents.exe
    (command available since CANoe version 8.1).

Step by Step Procedure: Unregistering COM

  • If you wish to delete the registration of the COM Interface(s):
  1. Open the MS-DOS prompt
    (e.g. Windows Start menu: All Programs|<CANoe folder>|Tools).
  2. Change the directory to the Exec64 folder of the installation directory.
  3. Execute RegisterComponents.exe /u
    (command available since CANoe version 8.1).

.NET Interoperability

To control the CANoeCOM Interface with a .NET client you can use a particular assembly (Vector.CANoe.Interop.DLL).

Step by Step Procedure: Integrating an Interop DLL

Therefore you have to perform the following steps:

  1. Copy the Interop assembly to your client application.
  2. Reference the Interop assembly directly in your client project.

This procedure ensures that your client application always refers to the correct Interop DLL during compile time.

If you want to control different CANoe versions you should always reference the Interop DLL of the oldest CANoe version to make sure that your client application doesn't use any interfaces that are not yet available during runtime in the oldest version.

Script Runtime Environment

To permit control of CANoe with the help of scripts you need Microsoft Windows Script. This program package includes the following items in its current Version 5.6 (Level: April 2004):

  • Visual Basic® Script Edition (VBScript) Version 5.6
  • JScript® Version 5.6
  • Windows Script Components
  • Windows Script Host 5.6
  • Windows Script Runtime Version 5.6

For detailed information on Microsoft Windows Script go to: https://msdn.microsoft.com/en-us/library/ms950396.aspx

Note — If Several CANoe Versions are Installed

  • During runtime you have to make sure that only the CANoe version that is currently used by the client application is registered globally.
  • If multiple CANoe versions are installed, please unregister/uninstall all existing CANoe versions (see above); afterwards register exactly the one CANoe version that your are going to use. Thereby you can avoid runtime errors that occur because a COM object of an older CANoe version cannot be converted to an interface type of the globally registered Interop assembly (of a newer CANoe version).
  • If you want to control different CANoe versions you should always register the oldest CANoe version you want to use.
