CANSystemDemo – Logging Filter.vbs

 

This sample script shows how a filter can be used to export/convert signals and messages. The script can be used with the CANoe CANSystemDemo configuration.

After the source file loading process all signals and messages found in the source file are added to the filter. Then the filter is enabled. After that the source file will be converted to Filter.asc and exported to Filter.csv.

The source file is the logging file specified in the Export dialog of the 2nd Logging Block, which is typically the logging file specified in this Logging Block, if it was not changed yet. Because by default the 2nd Logging Block in the CANSystemDemo configuration is not configured to start automatically you first have to make sure that a logging file is created by:

  1. Starting the Logging Block manually in the Measurement Setup.
  2. Changing the trigger configuration of the Logging Block.

Access the COM Interface

See COM Samples Introduction for general information on accessing and using the COM Interface.

Check if measurement is not running and stop measurement

It has to be checked, whether the measurement is not running. Then the measurement has to be stopped.

The Measurement object represents all measurement functions. To check the Running property of the Measurement object, the Measurement object has to be created first.

The Measurement object is assigned to the Measurement variable

Set Measurement = App.Measurement

Check and stop measurement using the Stop method of the Measurement object

If App.Measurement.Running Then
  App.Measurement.Stop
End If

Export/conversion of signals and messages

The Online mode is represented in the OnlineSetup property of the Configuration object. It returns the MeasurementSetup object. The LoggingCollection object comprises all Logging Blocks belonging to the MeasurementSetup object.

The LoggingCollection object is used to assign the second Logging Block to the logging variable

Set logging = app.Configuration.OnlineSetup.LoggingCollection(2)

The Exporter property of the Logging object represents the Export dialog, which is used to save a logging file in a new format.

The Export dialog can be opened via a Logging Block context menu in the Measurement Setup

Set exporter = logging.Exporter

The Filter property of the Exporter object represents a Pass Filter for signals and messages.

With the following programming code the filter will be created:

Set expfilter = exporter.Filter

Before every conversion the filter must be configured and activated.

Before the export/conversion the logging file must be loaded at first using the Load method of the Exporter object.

exporter.Load

The source file is loaded and the content can now be used.

To add signals and messages to the export filter the method Add method of the Filter object is used. Only signals and messages defined by the filter are available for the export/conversion.

Here all signals and messages are defined for export/conversion.

For Each symbol In exporter.Symbols
  expfilter.Add(symbol.FullName)
Next

For Each message In exporter.Messages
  expfilter.Add(Message.FullName)
Next

Next the deactivated filter (default) has to be activated using Enabled property of the Filter object

expfilter.Enabled = True

The target file is defined using Destinations property of the Exporter object. The target file formats are *.asc and *.csv. The return value of the Destinations property is the Files object.

With exporter.Destinations
  Clear
  Add(path + "\Filter.asc")
End With

With exporter.Destinations
  Clear
  Add(path + "\Filter.csv")
End With

Note

The Destinations property collects all target files sequentially assigned by the Add method (collection) of the Files object.
The Clear method of the Files object shall guarantee, that the collection is empty at the beginning.

The export/conversion is done using the Save method of the Exporter object

exporter.Save True
