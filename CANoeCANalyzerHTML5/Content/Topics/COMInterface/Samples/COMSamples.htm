COM Samples

 

Sample Script / Program (CANoe)

Description

CANSystemDemo – CANoe Environment.vbs

This sample is a general introduction to the usage of the COM Interface with:

  • Accessing the COM Interface
  • Loading the configuration
  • Starting the measurement
  • Access to CAPL, environment variables, panels

CANSystemDemo – CANoe Signal.vbs

This sample demonstrates the access to signals on the bus.

CentralLockingSystem – CANoe_TestSetup.vbs

This sample shows the access to the Test Setup (TestSetup object).

CentralLockingSystem – RunFailedTestcasesAgain.vbs

This sample implements a more detailed example usage of the Test Setup.

CANSystemDemo – vSignalyzer.vbs

This sample shows how vSignalyzer can be used to analyze MDF files. vSignalyzer must be installed. A reduced demo version of vSignalyzer you get on the Vector homepage.

Export/Conversion of Logging Files (CANoe)

CANSystemDemo – Logging Filter.vbs

This sample script shows how a filter can be used to export/convert signals and messages.

CANSystemDemo – Logging Settings.vbs

This sample shows how to change the exporter settings.

CANSystemDemo – Logging Short.vbs

This sample shows how simple a logging file can be exported/converted.

CANSystemDemo – Logging Specials.vbs

This sample shows some special features of the Exporter object of the COM Interface.

Visual Basic Applications (CANoe)

Description

Easy – Panel Control.exe

This sample shows how an external application can control the CANoe panels:

  • Save panel positions
  • Create a panel group
  • Add a panel to a panel group
  • Remove a panel from the panel group
  • Get the number of panels assigned to a panel group
  • Set a panel visible/invisible

More Examples – Automation.exe

This Visual Basic application shows the following features of the COM Interface:

  • Access environment variables and environment variable groups
  • Disable/enable the CANoe Stop button
  • Access to the Write Window

More Examples – DisableStop.vbs

This Visual Basic application shows the following features of the COM Interface:

  • Access environment variables
  • Disable/enable the CANoe Stop button
  • Access to the Write Window

Description

CANMainDemo – CANalyzer Hello.vbs

  • Output text into the Write Window

CANMainDemo – CANalyzer Logging.vbs

This Visual Basic application shows the following features of the COM Interface:

CANMainDemo – vSignalyzer.vbs

This sample shows how vSignalyzer can be used to analyze MDF files. vSignalyzer must be installed. A reduced demo version of vSignalyzer you get on the Vector homepage.

More Examples – Sinus2Excel.vbs / Sinus.vbs
