More Examples – DisableStop

 

This Visual Basic script shows how to access environment variables, disable/enable the Stop button of CANoe to avoid breaking off uncompleted measurements as well as the access to the Write Window. The application can be used with the CANoe COM_Automation configuration (Demo_CAN_CN\MoreExamples\COM_Automation).

The following operations can be done using the COM Interface:

Access the COM Interface

See COM Samples Introduction for general information on accessing and using the COM Interface.

Load the COM Automation sample configuration

The sample configuration is loaded using the Open method of the Application object

App.Open (full)

Access to CAPL functions — Compile all nodes

See CANSystemDemo – CANoe Environment.vbs; the same procedure is used here.

Access environment variables

Access environment variables

To access environment variables the Environment object is used.
The environment variable TestCompleted is assigned to the created variable TestCompleted.

Set TestCompleted = App.Environment.GetVariable("TestCompleted")
Set TestVariable = App.Environment.GetVariable("TestVariable")

Change the value of an environment variable

The property Value property of the Environment object is used to change the value of an environment variable.

TestVariable.Value = 50

Event triggering after value change of an environment variable

The TestCompleted_OnChange event occurs when the value of the environment variable TestCompleted has changed.

Sub TestCompleted_OnChange(completed)

Enable/disable the CANoe Stop button

To enable/disable the CANoe Stop button the CommandEnabled property of the UI object is used.

App.UI.CommandEnabled("stop") = enable

Using the Write Window

Access to the Write Window

To access the Write Window the Write object is used.
The Write object is part of the user interface and accessed by the Write property of the UI object (user interface).
The following command creates a Write object and assigns the Write object to a variable.

Set Write = App.UI.Write

Clear the Write Window

Use the Clear method of the Write object to delete the content of the Write Window.

Write.Clear

Output text in the Write Window

Use the Output method of the Write object to output text in the Write Window.

gWrite.Output "Enabling <Stop> command from Visual Basic..."

Getting the full name of the loaded CANoe configuration

With the FullName property of the Configuration object it is possible to get the full name of the loaded CANoe configuration file

app.Configuration.FullName
