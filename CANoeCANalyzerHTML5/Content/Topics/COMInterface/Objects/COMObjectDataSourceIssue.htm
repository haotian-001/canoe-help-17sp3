Description

The DataSourceIssue object provides access to data of an issue that is associated with a data source.

Issues are emitted during an import of a data source to signal errors and or potential problems.

Example
Copy

The following VBScript snippet creates a source for a single source file, imports it and displays any arising issue. It assumes an AUTOSAR XML file "SystemDescription.arxml" to exist, relative to the current configuration's location.

set Application     = CreateObject("CANoe.Application")
set DataSourceSetup = Application.Configuration.CommunicationSetup.DataSourceSetup

' Clear away any existing source.
while DataSourceSetup.DataSources.Count > 0
  call DataSourceSetup.DataSources.Remove(1)
wend

set NewSource = DataSourceSetup.DataSources.AddSingleFileDataSource("SystemDescription.arxml")
call NewSource.Import()

if NewSource.ImportStatus = cImportStatus_Succeeded and NewSource.ImportIssues.Count = 0 then
  call MsgBox("Import successful without issues.", vbInformation or vbOkOnly)
else
  Msg = "The following " & CStr(NewSource.ImportIssues.Count) & " issues were raised by the import:"
  for each Issue in NewSource.ImportIssues
    Msg = Msg & vbCrLf & "* Description: '" & Issue.Description & "'; severity: " & CStr(Issue.Severity) & _
    "; emitting source ID: " & CStr(Issue.Emitter.Id)
  next
  call MsgBox(Msg, vbInformation or vbOkOnly)
end if

' Enable usage of symbolic names.
const cImportStatus_Failed    = 0
const cImportStatus_Succeeded = 1
const cImportStatus_None      = 2

To use symbolic names for the severity, the following definitions can be used:

const cSeverity_Informative = 0
const cSeverity_Warning     = 1
const cSeverity_Error       = 2
