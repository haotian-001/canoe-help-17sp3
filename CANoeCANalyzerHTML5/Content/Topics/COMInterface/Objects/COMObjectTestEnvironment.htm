ExecuteAll

This method starts all test modules within the test environment consecutively..

This means that initially the first test module is started. As soon as it finished (completely) the second test module is started, then the third.

This method can only be used with test environments that exclusively contain test modules.

Syntax: object.ExecuteAll

✔

—

Save

This method saves the test environment.

Syntax: object.Save([name, [promptUser]])

✔

name, promptUser

object

Type: TestEnvironment

name

(optional)

Type: STRING

Sets the (new) path for the test environment, if applicable.

If no path is specified, the test environment is saved under its current name.

If it is not saved yet, the user will be prompted for a name.

promptUser

(optional)

Type: BOOL

Indicates whether the user should intervene in error situations, see below.

If not specified, the default value is True; possible values:

TRUE

(default)

If an error occurs the user will be prompted.

FALSE

If an error occurs the user will not be prompted; the function call fails.

SaveAs

This method saves the test environment in older formats.

If you specify "0, 0" (or any invalid or pre 5.1 version) for the "major, minor" parameters, the test environment will be saved in the file format of the CANoe version you are currently running.

Syntax: object.SaveAs(path, major, minor, [promptUser])

✔

name,major, minor, promptUser

object

Type: TestEnvironment

name

Type: STRING

The path of the file in which the test environment will be saved.

major

Prefix of the version number: e.g. 5 with version 5.1.

This function is valid for all program versions since version 5.1 (e.g. 5.1, 5.2, 6.0).

minor

Suffix of the version number: e.g. 1 with version 5.1.

This function is valid for all program versions since version 5.1 (e.g. 5.1, 5.2, 6.0).

TRUE

(default)

FALSE

If an error occurs the user will not be prompted; the function call fails.

StopSequence

This method stops the consecutive execution of all test modules in the test environment.

This method can only be used with test environments that exclusively contain test modules.

It may only be called while a measurement and the sequential execution is running.

Syntax: object.StopSequence
