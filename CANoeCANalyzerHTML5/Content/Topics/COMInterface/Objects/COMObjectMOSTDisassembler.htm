MostDisassembler

Application » Networks » Network » Devices » Device » MostDisassembler

Description

The MostDisassembler object activates disassembling of MOST messages via XML catalogs, that are assigned to the CANoe configuration.

Disassembling of MOST messages via XML catalog requires considerable computing time. Therefor the methods listed above should be used sparingly with running realtime applications.

SymbolicParameterList1

The message handed over as a BYTE array is being disassembled.

This method has been optimized for CMS messages that are handed over to the client as a pure BYTE array.

This means that this array contains all (maximum: 17) bytes of the CMS message including FBlock, OpType and user data.

Syntax: object.SymbolicParameterList1( dataLength, dataArray [, maxParams])

(see example below)

maxParams

(optional)

It defines the maximum number of message parameters that will be read out of the message

Example

Example — SymbolicMessageIDComponents method

This example gets the symbolic values of the message ID (FBlock, Function, OpType) of the message 01.01.000.C from the XML catalog in your CANoe configuration.

A dialog will be opened, that shows the retrieved symbolic information of the message ID. In case the entire symbolic information is available, the example above should result in NetBlock.01.FBlockIDs.Status.

Dim fBlockName, functionName, opTypeName, result
result = MostDisassembler.SymbolicMessageIDComponents 1, 1, 0, 12, fBlockName, functionName, opTypeName
If result = 0 Then
  MsgBox fBlockName & ".01." & functionName & "." & opTypeName
ElseIf result = 1 Then
  MsgBox fBlockName & ".01." & functionName
ElseIf result = 2 Then
  MsgBox fBlockName & ".01"
ElseIf result = 3 Then
  MsgBox "No symbolic information available"
End If

Example — SymbolicParameterList1 method

This example disassembles the parameters of a CMS message.

The below used value dataArray is a byte array of length dataLength (max. 17 bytes) with the structure of a CMS message as it is used in the OutputCMS method (hint: dataLength = DLC+5).

Up to three lines in the form ParameterName : SymbolicValue (RawValue) will be displayed in the shell.

Dim result, i
result = MostDisassembler.SymbolicParameterList1 dataLength, dataArray, 3
For i = LBound(result) To UBound(result)
  Wscript.echo result(i)(0) & ": " & result(i)(1) & " (" & result(i)(2) & ")"
Next

Example — SymbolicParameterList2 method

This example disassembles the parameters of an AMS message.

The below used value dataArray is a byte array of length dataLength (max. 64 kbytes) and the values fBlockID, instanceID, functionID and opType are the message ID components as used in the OutputAMS method.

Up to three lines in the form ParameterName : SymbolicValue (RawValue) will be displayed in the shell.

Dim result, i
result = MostDisassembler.SymbolicParameterList2 fBlockID, instanceID, functionID, opType, dataLength, dataArray, 3
For i = LBound(result) To UBound(result)
  Wscript.echo result(i)(0) & ": " & result(i)(1) & " (" & result(i)(2) & ")"
Next

Example — ThisMessageIDComponents method

This example determines the components of a message ID (FBlock, Function, OpType) of a just received message.

A dialog will be opened, that displays the information hexadecimal in the form fBlockID.instanceID.functionID.opType.

This method is particularly useful for CMS messages.

Dim fBlockID, instanceID, functionID, opType
MostDisassembler.ThisMessageIDComponents fBlockID, instanceID, functionID, opType
MsgBox Hex(fBlockID) & "." & Hex(instanceID) & "." & Hex(functionID) & "." & Hex(opType)

Example — ThisSymbolicMessageIDComponents method

This example gets the symbolic values of the message ID (FBlock, Function, OpType) of a just received message from the XML catalog in your CANoe configuration.

A dialog will be opened, that shows the retrieved symbolic information of the message ID.

Dim fBlockName, functionName, opTypeName, result
result = MostDisassembler.ThisSymbolicMessageIDComponents fBlockName, functionName, opTypeName
If result = 0 Then
  MsgBox fBlockName & "." & functionName & "." & opTypeName
ElseIf result = 1 Then
  MsgBox fBlockName & "." & functionName
ElseIf result = 2 Then
  MsgBox fBlockName
ElseIf result = 3 Then
  MsgBox "No symbolic information available"
End If
