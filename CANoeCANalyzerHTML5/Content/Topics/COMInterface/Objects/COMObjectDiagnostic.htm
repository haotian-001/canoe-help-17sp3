Diagnostic

Application » Networks » Network » Devices » Device » Diagnostic

Description

The Diagnostic object represents the diagnostic properties of an ECU on the bus or the basic diagnostic functionality of a CANoe network. It is identified by the ECU qualifier that has been specified for the loaded diagnostic description (CDD/ODX).

For the Diagnostic object there may be a node on the bus or the functionality is implemented by a real ECU.

CreateRequest

This method creates a request object with given qualifier path.

It is not possible to create response objects since they can only be generated by a responding ECU!

Syntax: object.CreateRequest( primitivePath)

(see example below)

✔

primitivePath

primitivePath

Service qualifier or diagnostic service shortname as defined in the diagnostic description.

Note:
You may also use the qualifier path of the primitive, consisting of diagnostic call, diagnostic instance and diagnostic service, separated by a "/" (slash) for backward compatibility reasons.

Example:
Sessions/DefaultSession_start. However, Vector does not recommend this.

Type: DiagnosticRequest

CreateRequestFromStream

This method creates a request object with the given byte stream.

If no request for the given byte sequence can be found in the diagnostic description (CDD/ODX), it is not possible to access the parameters symbolically.

Syntax: object.CreateRequestFromStream( byteStream)

✔

byteStream

byteStream

Bytes to be sent to the ECU as a request.

Type: DiagnosticRequest

DiagStartTesterPresent

This method starts sending autonomous/cyclical Tester Present requests to the ECU.

The TesterPresent remains active only as long as the COM script is running.

When the COM script is finished, the diagnostic channel is closed and the Tester Present process is switched off.

Syntax: object.DiagStartTesterPresent

Example

Example — TesterPresentStatus property / DiagStartTesterPresent method / DiagStopTesterPresent method

This code snippet shows how to control the Tester Present; it can be used in the UDS Diagnostic System Configuration:

Option Explicit
dim Application, Diagnostic, hint

'Load and initialize CANoe
Set Application = CreateObject("CANoe.Application")

'Start the measurement
If Not Application.Measurement.Running Then
  Application.Measurement.Start
  wscript.sleep 500
End If

'Select the diagnostic device
Set Diagnostic = Application.Networks("CAN").Devices("Door").Diagnostic

'Start Tester Present, wait and check if started
Diagnostic.DiagStartTesterPresent
wscript.sleep 50
If Not Diagnostic.TesterPresentStatus then
  hint="Error: Cannot DiagStartTesterPresent!"
  msgbox hint
End If

'Stop Tester Present, wait and check if stopped
Diagnostic.DiagStopTesterPresent
wscript.sleep 50
If Diagnostic.TesterPresentStatus then
  hint="Error: Cannot DiagStopTesterPresent!"
  msgbox hint
End If

Example — CreateRequest method

Sample 1

This example sends a diagnostic request and evaluates the diagnostic response.

This example is based on the UDS Diagnostic System Configuration (CANoe only) that already contains a "Door" ECU.

Option Explicit
dim gApp, gMeasurement, diagDev, diagReq, diagResp, k, tempstr

'Load (if necessary) and initialize CANoe and the related objects,
'assuming that the correct configuration is loaded at CANoe start
Set gApp = CreateObject("CANoe.Application")
Set gMeasurement = gApp.Measurement

'Compile all nodes and start the measurement
If Not gMeasurement.Running Then
  gApp.CAPL.Compile
  gMeasurement.Start
'wait until measurement is started before continuing
  wscript.sleep 500
End If

'Select the diagnostic device, create and send the request
Set diagDev = gApp.Networks("CAN").Devices("Door").Diagnostic
set diagReq = diagDev.CreateRequest("DefaultSession_Start")
  'Wait until diagnostics structures have been initialized
  wscript.sleep 200
diagReq.Send

'Wait for a response or a timeout, then output response details in message box
while diagReq.Pending
  wscript.sleep 20
wend
if (diagReq.Responses.Count = 0) then
  msgbox "No Response received!"
else
  for k=1 to diagReq.Responses.Count
    set diagResp=diagReq.Responses(k)
    tempstr="Attributes of response "& k &": " & chr(13) & chr(13)
    if (diagResp.Positive) then tempstr=tempstr & "* Positive" else tempstr=tempstr & "* Negative" end if
    tempstr=tempstr & " Response with code: " & diagResp.ResponseCode & chr(13)
    tempstr=tempstr & "* Sender: " & diagResp.Sender
    msgbox tempstr
    set diagResp = Nothing
  next
end if
set diagDev = Nothing

Sample 2

This Visual Basic script shows how to access bytes of a diagnostic response:

Option Explicit
dim gApp, gMeasurement, diagDev, diagReq, diagResp, i, k, tempstr, stream, str

'Load (if necessary) and initialize CANoe and the related objects,
'assuming that the correct configuration is loaded at CANoe start
Set gApp         = CreateObject("CANoe.Application")
Set gMeasurement = gApp.Measurement

'Compile all nodes and start the measurement
If Not gMeasurement.Running Then
  gApp.CAPL.Compile
  gMeasurement.Start
  'wait until measurement is started before continuing
  wscript.sleep 500
End If

'Select the diagnostic device, create and send the request
Set diagDev = gApp.Networks("CAN").Devices("Door").Diagnostic
Set diagReq = diagDev.CreateRequest("DefaultSession_Start")
  'Wait until diagnostics structures have been initialized
  wscript.sleep 200
diagReq.Send

'Wait for a response or a timeout, then output response details in message box
while diagReq.Pending
  wscript.sleep 20
wend
if (diagReq.Responses.Count = 0) then
  msgbox "No Response received!"
else
  for k=1 to diagReq.Responses.Count
    set diagResp=diagReq.Responses(k)
    tempstr="Attributes of response "& k &": " & chr(13) & chr(13)
    if (diagResp.Positive) then tempstr=tempstr & "* Positive" else tempstr=tempstr & "* Negative" end if
    tempstr=tempstr & " Response with code: " & diagResp.ResponseCode & chr(13)
    tempstr=tempstr & "* Sender:" & chr(9) & diagResp.Sender
    stream = diagResp.Stream
    if isarray(stream) then
      str = cstr(stream)
      tempstr=tempstr & chr(13) & "* Stream:" & chr(9)
      for i = 1 to lenb(stream)
        tempstr = tempstr & right("0" & hex(ascb(midb(str,i,1))), 2) & " "
      next
    end if
    msgbox tempstr
    set diagResp = Nothing
  next
end if
set diagDev = Nothing
