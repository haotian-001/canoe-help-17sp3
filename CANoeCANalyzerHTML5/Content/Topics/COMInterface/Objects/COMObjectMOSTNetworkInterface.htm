Application » Networks » Network » Devices » Device » MostNetworkInterface

Description

The MostNetworkInterface object activates the configuration of the MOST network interface and the transmission of data on the control channel and the asynchronous channel.

AllBypass

Syntax: object.AllBypass [= allBypass]

Activates/deactivates the bypass of the connected network interface or determines whether the bypass of the connected network interface is active.

The bypass enables the network interface to stay "invisible" in the ring — this means that the ring behaves as if the network interface isn't connected to the ring.

If the bypass of a device is opened or closed in a running ring, the node positions are renumbered.

If the bypass is active no messages/packets can be sent.

A typical use case for a bypass is a pure monitoring unit that can be realized by combining it with the ControlSpy property or the AsynchronousSpy property.

The activation state of the bypass can be set in the initialization phase of the measurement or during a measurement.

If the measurement is not running the determination of the bypass mode returns the last (before measurement stop) valid value.

If no measurement has taken place, yet, the default of the loaded configuration will be returned.

(see example below)

AllocationTable

Syntax: object.AllocationTable

Determines the current allocation table.

The allocation table shows which synchronous channels are allocated and to which connection labels they are combined.

The allocation table is delivered as a byte array of length 60. The allocation table cannot be edited directly; an allocate/deallocate message has to be sent (see OutputCMS method).

If the measurement is not running the determination of the allocation table returns the last (before measurement stop) valid table. If no measurement has taken place, yet, an array will be returned, in which every element has the value 0x70. This special value means "not allocated".

The value of each allocated synchronous channel is the connection label of the assigned channel.

(read-only)

(see example below)

✔

The current allocation table.

AsynchronousSpy

Syntax: object.AsynchronousSpy[=asynchronousSpy]

Activates/Deactivates the asynchronous spy of the connected network interface or determines whether the asynchronous spy of the connected network interface is activated.

The asynchronous spy enables monitoring of all packets of the asynchronous channel.

A typical use case for the asynchronous spy is a pure monitoring unit that can be realized by combining it with an activated Bypass and a ControlSpy if applicable.

If the asynchronous spy is activated but the bypass is open, the spy only monitors packets that are not being received/sent by the node. Thereby there is no doubling of the packets as it happens with the ControlSpy.

The activation state of the asynchronous spy can be set in the initialization phase of the measurement or during a measurement.

If the measurement is not running the determination of the activation state of the asynchronous spy returns the last (before measurement stop) valid value. If no measurement has taken place, yet, the default of the loaded configuration will be returned.

AsynchronousSpyMode

Syntax: object.AsynchronousSpyMode[= mode]

If a VN2600 runs in Node mode with the control-spy additionally activated, one receives all messages via the spy and additionally all node messages via the node. Thus one receives all node messages twice: once via the node and once via the spy. For the asynchronous channel this is different. One only receives those packets via the spy that are not already received via the node.

Using this property one can switch between the standard asynchronous spy mode (the spy delivers only those packets, that are not delivered by the node) and a mode, that is equivalent to the control spy mode (the spy delivers all packets, no matter whether the node already delivered the respective packet or not).

This setting only affects the MostNetworkInterface object.

(see example below)

✔

the alternative packet address to be set; possible values:

object

Type: MostNetworkInterface

mode

The mode to use for the asynchronous spy; possible values:

0

Standard behavior
(only none-node packets are delivered by the asynchronous spy)

1

Control-spy-like behavior
(node packets are also delivered by the asynchronous spy)

Return Value:

The mode used for the asynchronous spy.

EventSources

Syntax: object.EventSources [=eventSourcesBitField]

The purpose of this property is to optimize the performance of the client-server communication. If a client implements one or more event handlers, the server object fires all events. Events, the client does not have a event handler implemented for, are intercepted by the respective runtime environment (e.g. VB, VBScript, .NET), but they still consume precious CPU time.

This property activates/deactivates the event sources in the MostNetworkInterface object. Each event source corresponds to one of the events fired by the MostNetworkInterface object. By default, all event sources are active, i.e. all events are fired. If a client only needs one specific event type, it can deactivate all other events in the MostNetworkInterface object using this property.

(see example below)

TimingMode

Syntax: object.TimingMode [= timingMode]

Sets or determines the timing mode of the connected network interface.

Exactly one timing master must exist for each ring. With more or less masters in the ring no communication is possible.

If the measurement is not running the determination of the timing mode returns the last (before measurement stop) valid value.

OutputCMS

Syntax: object.OutputCMS(rType, destinationAddress, DLC, dataArray)

Sends a CMS message on the control channel. The Control Message Service (CMS) describes the control messages as they are actually sent on the ring. A CMS message contains up to 17 data bytes.

With rType normal the first 5 bytes are reserved for FBlockID, InstanceID, FunctionID, OpType, TelID and TelLen.

With Groupcast and Broadcast addressing the last byte is reserved, too.

Messages can only be sent during measurement.

  • For rType normal messages the TelLen as given in data byte 4 will be overwritten with the value of the DLC before sending.
  • For rType unequal normal messages the value of the DLC is ignored.

The transmission acknowledgment is provided by the OnCMSTx event.

(see example below)

OnCriticalUnlock

Syntax: object_OnCriticalUnlock( timeHigh, time )

Occurs when the Fibre Optical Transmitter (FOT) of the connected network interface gets a change in state to critical unlock.

Critical unlock occurs if — for a certain amount of time — there was no stable lock continuously or accumulated.

Valid values are:

  • t_criticalunlock_typ: 70 ms
  • t_criticalunlock_max: 100 ms

Changes in state of the FOT are only reported during measurement.

Example: OnCMSRx event

The following example compares a received CMS message with a saved one and presents the comparison result, the time stamp and the status information of the message:

'result, time stamp and state are StaticControls
'mCMSMessage is a data structure, that contains the data of the comparison message. The message has rType 'normal'

Private Sub myMostNetworkInterface_OnCMSRx(ByVal timeHigh As Long, ByVal time As Long, ByVal rType As CANoe.eMostRType, ByVal sourceAddress As Long, ByVal destinationAddress As Long, ByVal DLC As Long, ByVal dataArray As Variant, ByVal state As Long)
'compare result
  If CompareCMS(rType, sourceAddress, destinationAddress, DLC, dataArray) Then
    result.Text = "OK"
  Else
    result.Text = "ERROR"
  End If
'timestamp
  timestamp.Text = compose_timestamp   'see timestamps in COM
'state
  state.Text = Hex(state)
End Sub

Private Function CompareCMS(ByVal rType As CANoe.eMostRType, ByVal sourceAddress As Long, ByVal destinationAddress As Long, ByVal DLC As Long, ByVal dataArray As Variant) As Boolean
'rType
  If rType <> cMostRType_Normal Then
    Return False
  End If
'source address
  If sourceAddress <> mCMSMessage.sourceAddress Then
    Return False
  End If
'destination address
  If destinationAddress <> mCMSMessage.destinationAddress Then
    Return False
  End If
'DLC
  If DLC <> mCMSMessage.DLC Then
    Return False
  End If
'data
  Dim i As Integer
  For i = 0 To DLC + 4   'DLC+4: Because of the five leading FCat format bytes of rType 'normal' messages
    If dataArray(i) <> mCMSMessage.hexdump(i) Then
      Return False
   End If
  Next
    Return True
End Function
