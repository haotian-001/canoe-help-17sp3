AudioInterface

Application » Networks » Network » Devices » Device » AudioInterface

Description

The AudioInterface object represents the possibilities to access and configure the MOST network interface regarding synchronous audio data.

You can access the properties and methods during a measurement only.

Mute

This property sets or determines the mute state of the analogue input/output of the connected network interface.

Independent from the volume setting no signal is put on the ring or sent to the analogue output if muting is active.

The state of the muting can only be changed during measurement.

If the measurement isn't running the determination of the mute state returns the last valid value before the measurement stop.

If no measurement has taken place so far, the value "0" (muting inactive) will be returned.

Volume

This property sets or determines the volume setting of the analogue input/output of the connected network interface.

Independent from the volume setting no signal is put on the ring or sent to the analogue output if muting is active.

The volume setting can only be changed during measurement.

If the measurement isn't running the determination of the volume setting returns the last valid value before the measurement stop.

If no measurement has taken place so far, the value 31 will be returned.

ConnectToLabel

This method programs the routing engine of the MOST network interface so that the audio data of the analogue input can be routed from the analogue input to the given Connection Label or from the connection label to the analogue output, respectively.

Each connection label consists of 1…8 synchronous channels.

For broadcasting CD-Audio a label with 4 synchronous channels is needed.

The routing engine can only be programmed during measurement.
The given connection label must exist in the MOST ring.

  • You can create new labels by using allocate messages.

  • You can delete existing labels by using deallocate messages.

The AllocationTable property gives a good overview of all connection labels of the ring.

Syntax: object.ConnectToLabel( lineInOut, connectionLabel )

DisconnectFromLabel

This method undoes the programming (done by the ConnectToLabel method) of the routing engine of the connected network interface.

It's recommended to undo the LineOut programming first; at least the output should be muted.

If you undo the LineIn programming first, without muting the output a nasty noise may occur.

The routing engine can only be programmed during measurement.
The given connection label must exist in the MOST ring.

  • You can create new labels by using allocate messages.

  • You can delete existing labels by using deallocate messages.

The AllocationTable property gives a good overview of all connection labels of the ring.

OnMute

This event occurs when the activation state of the muting of the analog input/output of the MOST network interface changes.

Changes of the muting activation state are returned during measurement only.

Timestamps with high values need special treatment.

Example

Example — Mute property

The following example activates the analogue output of the MOST network interface:

AudioInterface.Mute(cMostLineOut) = 0

Example — Volume property

The following example sets the analogue output of the MOST network interface to maximum volume:

AudioInterface.Volume(cMostLineOut) = 255

Example — ConnectToLabel method

The following example routes an audio signal — lying on Connection Label 0 in the MOST ring — to the analogue output of the MOST network interface:

AudioInterface.ConnectToLabel( cMostLineOut, 0 )

Example — DisconnectFromLabel method

The following example undoes the routing that connects connection label 0 with the analogue output of the MOST network interface:

AudioInterface.DisconnectFromLabel( cMostLineOut, 0 )

Example — OnMute event

The following example shows a procedure that is called on any change of the mute status:

Sub AudioTester_OnMute( timeHigh, time, lineInOut, mute )
  If lineInOut = cMostLineOut Then
    MuteCheckBoxLineOut.Checked = mute
  Elseif lineInOut = cMostLineIn Then
    MuteCheckBoxLineIn.Checked = mute
  End If
  TimeLo = time
  TimeHi = timeHigh
End Sub

Example — OnVolume event

The following example shows a procedure that will be called whenever the volume settings change:

Sub AudioTester_OnVolume( timeHigh, time, lineInOut, volume )
  If lineInOut = cMostLineOut Then
    VolumeTrackBarLineOut.Value = volume
  Elseif lineInOut = cMostLineIn Then
    VolumeTrackBarLineIn.Value = volume
  End If
  TimeLo = time
  TimeHi = timeHigh
End Sub
