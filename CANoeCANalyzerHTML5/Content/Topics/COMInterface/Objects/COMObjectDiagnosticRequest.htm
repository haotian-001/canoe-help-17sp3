DiagnosticRequest

Application » Networks » Network » Devices » Device » Diagnostic » DiagnosticRequest

Description

The DiagnosticRequest object represents the query of a diagnostic tester (client) to an ECU (server) in CANoe. It can be replied by a DiagnosticResponse object.

After its generation the Request object passes through the following states:

SuppressPositiveResponse

If supported by the diagnostic service, the Suppress positive response bit can be set in the request.

By doing so the ECU will return a negative response only in case of an error.

This functionality is available for UDS only (not for KWP) and has to be activated in the diagnostic description (CDD/ODX) at the service.

Send

This method causes the DiagnosticRequest object to be sent.

By calling this method, the Request object is put in a send queue; this means that it might not be visible on the bus, immediately.

After a successful call of the function the request is in state Pending.

SetComplexParameter

This method sets the value of the parameter of the request to the given value.

If the value is a string, the parameter will be set symbolically; if it is numeric the according raw value will be set.

This may lead to different results if there is a conversion formula defined for the parameter (see example on page SetParameter method).

The value can also be handed over as a byte sequence.

OnCompletion

This event is called as the last event after sending a request.

As an alternative the OnTimeout event can be called.

Syntax: object_OnCompletion()

✔

—

 

OnConfirmation

This event is called after a successful sending of the request.

If the request object can be sent in a single CAN message this event just means that such a frame has successfully been transmitted on the bus; which means that a bus participant has generated the Ack bit.

Only with longer requests and appropriate settings for the transport layer you can assume that the target ECU has received the request.

Syntax: object_OnConfirmation()

✔

—

 

OnResponse

This event is called after a receiving a response to a request.

The response can be positive or negative.

Syntax: object_OnResponse( response)

✔

OnTimeout

This event is called as the last event after sending a request if the request couldn't be sent (OnConfirmation event has not been called) or if the expected response failed to appear, respectively.

As an alternative the OnCompletion event can be called.

Example

Example — Pending property / Responses property / Send method

  • Sample 1

    This example sends a diagnostic request and evaluates the diagnostic response.

    This example is based on the UDS Diagnostic System Configuration (CANoe only) that already contains a "Door" ECU.

    Option Explicit
    dim gApp, gMeasurement, diagDev, diagReq, diagResp, k, tempstr

    'Load (if necessary) and initialize CANoe and the related objects,
    'assuming that the correct configuration is loaded at CANoe start
    Set gApp = CreateObject("CANoe.Application")
    Set gMeasurement = gApp.Measurement

    'Compile all nodes and start the measurement
    If Not gMeasurement.Running Then
      gApp.CAPL.Compile
      gMeasurement.Start
    'wait until measurement is started before continuing
      wscript.sleep 500
    End If

    'Select the diagnostic device, create and send the request
    Set diagDev = gApp.Networks("CAN").Devices("Door").Diagnostic
    set diagReq = diagDev.CreateRequest("DefaultSession_Start")
      'Wait until diagnostics structures have been initialized
      wscript.sleep 200
    diagReq.Send

    'Wait for a response or a timeout, then output response details in message box
    while diagReq.Pending
      wscript.sleep 20
    wend
    if (diagReq.Responses.Count = 0) then
      msgbox "No Response received!"
    else
      for k=1 to diagReq.Responses.Count
        set diagResp=diagReq.Responses(k)
        tempstr="Attributes of response "& k &": " & chr(13) & chr(13)
        if (diagResp.Positive) then tempstr=tempstr & "* Positive" else tempstr=tempstr & "* Negative" end if
        tempstr=tempstr & " Response with code: " & diagResp.ResponseCode & chr(13)
        tempstr=tempstr & "* Sender: " & diagResp.Sender
        msgbox tempstr
        set diagResp = Nothing
      next
    end if
    set diagDev = Nothing

  • Sample 2

    This Visual Basic script shows how to access bytes of a diagnostic response:

    Option Explicit
dim gApp, gMeasurement, diagDev, diagReq, diagResp, i, k, tempstr, stream, str

'Load (if necessary) and initialize CANoe and the related objects,
    'assuming that the correct configuration is loaded at CANoe start
Set gApp         = CreateObject("CANoe.Application")
Set gMeasurement = gApp.Measurement

'Compile all nodes and start the measurement
If Not gMeasurement.Running Then
  gApp.CAPL.Compile
  gMeasurement.Start
  'wait until measurement is started before continuing
  wscript.sleep 500
End If

'Select the diagnostic device, create and send the request
Set diagDev = gApp.Networks("CAN").Devices("Door").Diagnostic
Set diagReq = diagDev.CreateRequest("DefaultSession_Start")
  'Wait until diagnostics structures have been initialized
  wscript.sleep 200
diagReq.Send

'Wait for a response or a timeout, then output response details in message box
while diagReq.Pending
  wscript.sleep 20
wend
if (diagReq.Responses.Count = 0) then
  msgbox "No Response received!"
else
  for k=1 to diagReq.Responses.Count
    set diagResp=diagReq.Responses(k)
    tempstr="Attributes of response "& k &": " & chr(13) & chr(13)
    if (diagResp.Positive) then tempstr=tempstr & "* Positive" else tempstr=tempstr & "* Negative" end if
    tempstr=tempstr & " Response with code: " & diagResp.ResponseCode & chr(13)
    tempstr=tempstr & "* Sender:" & chr(9) & diagResp.Sender
    stream = diagResp.Stream
    if isarray(stream) then
      str = cstr(stream)
      tempstr=tempstr & chr(13) & "* Stream:" & chr(9)
      for i = 1 to lenb(stream)
        tempstr = tempstr & right("0" & hex(ascb(midb(str,i,1))), 2) & " "
      next
    end if
    msgbox tempstr
    set diagResp = Nothing
  next
end if
set diagDev = Nothing

Example — SetParameter method

The Voltage parameter is defined with the following conversion formula:

Vphysical = Raw value / 20.0

If this parameter is being set with

request.SetParameter( "Voltage", 12.2)

the raw value will be 12, the physical value is 0.6.

If you use the following formula instead

request.SetParameter( "Voltage", "12.2")

the raw value will be 244, the physical value is 12.2.

Note: You can easily convert the argument into a string by adding an empty string:

request.SetParameter( "Voltage", 12.2 & "")
