StopEx

StopEx repairs differences in the behavior of the deprecatedStop method (for the Measurement object) on deferred stops concerning simulated and real mode in CANoe.

Calling the StopEx method correlates to clicking the Stop button (Home ribbon tab).

The function must not be called if measurement has already ended.

Syntax: object.StopEx

(see example below)

OnExit

Occurs when the measurement is exited.

When trying to update any control/indicator on the VB .NET panel using callback functions such as OnInit and OnQuit following have to be considered:
The VB panel runs in an own thread.
So it is not allowed to update VB .NET panel elements from the CANoe thread. If the VB panel shall run in its own thread, then the update has to be done using the Delegate function.

Syntax: object_OnExit

(see example below)

Example

Example — AnimationDelay property

The following example sets the animation delay for the measurement in Offline mode:

Application.Measurement.AnimationDelay = 100

Example — MeasurementIndex property

The following example sets the measurement index for the next measurement to 15:

Application.Measurement.MeasurementIndex = 15

Example — Running property

The following example starts the measurement if necessary:

If Not App.Measurement.Running Then
  App.Measurement.Start
End If

See also CANSystemDemo – CANoe Environment.vbs (COM Sample)

Example — StopEx method

Sample code for delayed stop and full stop. The function must not be called if measurement has already ended.

Application.Measurement.StopEx()   'So called 'deferred stop' if something delays the stop
Wscript.Sleep 3000                 'Wait
If Application.Measurement.Running Then
  Application.Measurement.StopEx()   'Full stop
End If
