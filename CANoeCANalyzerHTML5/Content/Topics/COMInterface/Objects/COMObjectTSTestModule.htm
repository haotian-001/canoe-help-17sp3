Description

The TSTestModule object represents a test module in the CANoe test setup.

TestVariant

Returns the name of the current test variant or activates the test variant with the given name.

The property is available for XML test module only. The call fails if no variant of the given name exists in the test module.

Switching the variant is possible during a measurement as long as the specific test module isn't running.

Switching the variant may take some time depending on the size of the test module since the test module is imported again.

Syntax: object.TestVariant = variant

Pause

Instructs the test module to pause.

The test module pauses when the test function / control function / test case that has just been executed is terminated.

The execution dialog can be used to specify whether the test module pauses after test cases or test functions / control functions.

The OnPause event indicates the actual point at which the test module pauses.

The execution can be resumed from the point at which the test module paused by means of the Resume method.

Syntax: object.Pause

(see example below)

✔

—

—

Reload

This reloads the XML file with the test specification for XML test modules.

This method can also be called during a measurement in order to load a modified XML specification.

The call fails if the affected test module is being executed at that time.

This method is not implemented for CAPL test modules.

Syntax: object.Reload

✔

—

—

Resume

Instructs the test module to exit Pause mode and resume the execution from the point at which it was aborted.

If the test module received an abort command, but is still waiting for the current test case or test function / control function to be terminated, the execution continues regardless. In other words, the execution continues without the test module ever actually pausing. In this case, no OnPause event is called up.

Syntax: object.Resume

(see example below)

OnPause

OnPause is called when the test module execution has been aborted.

The event is first called when the test module actually pauses the execution. Depending on the setting, this occurs after either test cases or test functions/control functions. The event will not be called immediately when either the [Pause] button in the execution dialog is clicked or the Pause method is called up.

If the test module received an abort command, but is still waiting for the current test case or test function/control function to be terminated, the execution continues regardless: by means of the Resume method, for example. In other words, the execution continues without the test module ever actually pausing. In this case, the OnPause event is not called.

Syntax: object_OnPause

(see example below)

✔

Example — ExecuteRepeatedly/ExecutionMode/NumberOfExecutions property

In the following example, the first test module in the first test environment is set to be executed 10 times when started:

Dim App, TestModule
Set App        = CreateObject("CANoe.Application")
Set TestModule = App.Configuration.TestSetup.TestEnvironments.Item(1).TestModules(1)

TestModule.ExecuteRepeatedly  = True
TestModule.ExecutionMode      = 0
TestModule.NumberOfExecutions = 10

Example — PausingMode property

In the following example, debug pauses are activated after every test case and test pattern for the first test module in the first test environment:

Dim App, TestModule
Set App        = CreateObject("CANoe.Application")

TestModule.DebugMode   = 2
TestModule.PausingMode = 1

Example — Verdict property

This example starts the first available test configuration and displays the verdict when finished:

Dim CanoeApp, Configuration, Measurement, TestConfigurations, TestConfiguration
Dim verdicts, measuring, testing

Set CanoeApp           = CreateObject("CANoe.Application")
Set Configuration      = CanoeApp.Configuration
Set Measurement        = CanoeApp.Measurement
Set TestConfigurations = Configuration.TestConfigurations
Set TestConfiguration  = TestConfigurations.Item(1)

WScript.ConnectObject Measurement, "Measurement_"
WScript.ConnectObject TestConfiguration, "TestConfiguration_"

measuring = False
Call Measurement.Start()

' wait for measurement start
While Not measuring
  Wscript.Sleep 100
Wend

testing = False
Call TestConfiguration.Start()

' wait for testing start
While Not testing
  Wscript.Sleep 100
Wend

' here we are sure the test configuration has started
' ...

' wait for testing end
While testing
  Wscript.Sleep 100
Wend

verdicts = Array("n.a.", "passed", "failed")
WScript.Echo "Verdict of """ & CStr(TestConfiguration.Name) & """" _
& " is " & verdicts(TestConfiguration.Verdict)

Sub Measurement_OnStart()
  measuring = True
End Sub

Sub TestConfiguration_OnStart()
  testing = True
End Sub

Sub TestConfiguration_OnStop(reason)
  testing = False
End Sub

Example — Pause method, Resume method, OnPause event

In the example below, the execution of the test module is paused and then resumed at a later point:

Dim App, TestModule, TMPauseStarted

Set App        = CreateObject("CANoe.Application")
Set TestModule = App.Configuration.TestSetup.TestEnvironments.Item(1).Items(1)

' Start measurement and test module
' ...

' Send message to pause execution and wait for event
TestModule.Pause()
While (Not TMPauseStarted)
  WScript.Sleep(200)
Wend
' Resume execution
  TestModule.Resume()

' Now we can be sure that the test module is suspended
Sub TestModule_OnPause()
  TMPauseStarted = True
End Sub

Example — OnReportGenerated event

In the following example, a test module is started. After the test, the HTML test report is copied to another directory.

Dim App, Meas, TestModule, TMStarted, MeasStarted, FileSys, SourceFile
Dim GenFile, ReportGenerated, ReportGenSuccess, ReportSource
Dim TMStopReason

TMStarted        = False
TMStopped        = False
MeasStarted      = False
ReportGenerated  = False
ReportGenSuccess = False
ReportSource     = ""

Set FileSys    = CreateObject("Scripting.FilesystemObject")
Set App        = CreateObject("CANoe.Application")
Set TestModule = App.Configuration.TestSetup.TestEnvironments.Item(1).Items(1)
Set Meas       = App.Measurement

' Set up connection to event handlers
WScript.ConnectObject TestModule, "TestModule_"
WScript.ConnectObject Meas, "Meas_"

' Start Measurement
Meas.Start()
While (Not MeasStarted)
  WScript.Sleep(200)
Wend

' Start Testmodule
TestModule.Start()
While (Not TMStarted)
   WScript.Sleep(200)
Wend

' Wait until report is generated
While (Not ReportGenerated)
   WScript.Sleep(200)
Wend

TesterDecision = MsgBox("Report generated: " & ReportGenSuccess & ". Move XML report file?", vbYesNo)
If TesterDecision = vbNo Then
  WScript.Quit()
Else
  Set GenFile = FileSys.GetFile(GenFile)
  If (ReportGenSuccess) then
' If successful, move the report to other folder (folder must exist, or operation will fail)
    GenFile.Move("C:\TestReports\")
  End If
End If

' Event Handlers 

Sub TestModule_OnStart()
TMStarted = True
End Sub

Sub TestModule_OnReportGenerated(Success, SourceFullName, GeneratedFullName)
ReportGenerated  = True
ReportGenSuccess = Success
SourceFile       = SourceFullName
GenFile          = GeneratedFullName
End Sub

Sub Meas_OnStart()
MeasStarted = True
End Sub
