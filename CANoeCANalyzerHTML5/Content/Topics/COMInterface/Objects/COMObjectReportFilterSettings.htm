Description

The ReportFilterSettings object represents the filter settings of a test module's test report.

You can e.g. define, that only failed test steps are documented (output) in the test report.

MaxTestStepLevel

This property sets/returns the setting up to which level test steps shall be included in the test report.

This property is not available for XML test modules.

Syntax: object.MaxTestStepLevel [= value]

(see example below)

✔

Parameters

object

Type: ReportFilterSettings

value

Type: INT

Maximum level that a test step may have to be included in the test report.

  • A value of 0 means that only test steps of level 0 will be included.
  • A value of 1 means that only test steps of level 0 and level 1 will be included.

—

ReportedTestCases

This property sets/returns the setting which test case shall be included in the test report details section.

In failed mode, failed test cases functions are completely reported with all their test steps while the passed test cases are omitted from the details section.

Syntax: object.ReportedTestCases [=value]

(see example below)

✔

Parameters

object

Type: ReportFilterSettings

value

Type: ENUM

Setting which test case shall be included in the test report details section; possible values:

0|cTestCaseAll

All test cases will be included in the test report details.

1|cTestCaseFailed
  • Only failed test cases will be included in the test report details.
  • Passed test cases will be omitted from the report details.
    		•

    ReportedTestPatterns

    This property sets/returns the setting which test and control functions (including their test steps) shall be included in the test report.

    For .NET test modules this property is equivalent to the ReportedTestSteps property, both control the reporting of test steps and test/control functions. In failed only mode, failed test/control functions are completely reported with all their test steps.

    This property is only applicable for XML test modules.

    Syntax: object.ReportedTestPatterns [= value]

    (see example below)

    ✔

    Parameters

    object

    Type: ReportFilterSettings

    value

    Type: ENUM

    Setting which test and control functions (including their test steps) shall be included in the test report; possible values:

    0|cAllTestPatterns

    All test and control functions will be included in the test report.

    1|cFailedTestPatternsOnly
  • Only failed test and control functions will be included in the test report.
    		•

    2|cNoTestPatterns
  • No test and control functions will be included in the test report.
    		•

    —

    ReportedTestSteps

    This property sets/returns the setting which test steps shall be included in the test report.

    For .NET test modules this property is equivalent to the ReportedTestPatterns property, both control the reporting of test steps and test/control functions. In failed only mode, failed test/control functions are completely reported with all their test steps.

    The MaxTestStepLevel property also influences test step filtering.

    This property is not available for XML test modules.

    Syntax: object.ReportedTestSteps [= value]

    (see example below)

    ✔

    Example

    Example — MaxTestStepLevel property

    The following example sets the maximum test step level to be included to 3:

    Dim App, TestModule, FilterSettings

    Set App            = CreateObject("CANoe.Application")
    Set TestModule     = App.Configuration.TestSetup.TestEnvironments.Item(1).Items(1)
    Set FilterSettings = TestModule.Report.FilterSettings

    FilterSettings.MaxTestStepLevel = 3

    Example — ReportedTestCases property

    The following example sets that only failed test and control functions will be in the test report:

    Dim App, TestModule, FilterSettings

    Set App            = CreateObject("CANoe.Application")
    Set TestModule     = App.Configuration.TestSetup.TestEnvironments.Item(1).Items(1)
    Set FilterSettings = TestModule.Report.FilterSettings

    FilterSettings.ReportedTestCases = 1

    Example — ReportedTestPatterns property

    The following example sets that only failed test and control functions will be in the test report:

    Dim App, TestModule, FilterSettings

    Set App            = CreateObject("CANoe.Application")
    Set TestModule     = App.Configuration.TestSetup.TestEnvironments.Item(1).Items(1)
    Set FilterSettings = TestModule.Report.FilterSettings

    FilterSettings.ReportedTestPatterns = 1

    Example — ReportedTestSteps property

    The following example sets that no test steps will be in the test report:

    Dim App, TestModule, FilterSettings

    Set App            = CreateObject("CANoe.Application")
    Set TestModule     = App.Configuration.TestSetup.TestEnvironments.Item(1).Items(1)
    Set FilterSettings = TestModule.Report.FilterSettings

    FilterSettings.ReportedTestSteps = 2

    Example — IsActive/SetActive method

    The following example activates the filter for timeout resumes:

    Dim App, TestModule, FilterSettings

    Set App            = CreateObject("CANoe.Application")
    Set TestModule     = App.Configuration.TestSetup.TestEnvironments.Item(1).Items(1)
    Set FilterSettings = TestModule.Report.FilterSettings

    FilterSettings.SetActive 3, true
