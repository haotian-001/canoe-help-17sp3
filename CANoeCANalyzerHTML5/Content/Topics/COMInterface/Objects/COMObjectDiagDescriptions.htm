DiagDescriptions

Application » Configuration » GeneralSetup » DiagnosticsSetup » DiagDescriptions

Description

The DiagDescriptions object represents (and only contains) the diagnostic "master" descriptions configured in CANoe.

Use the AdditionalDescriptions property of the DiagDescription object to access additional descriptions.

Adding or removing additional diagnostic descriptions while the measurement is running or the diagnostics configuration dialog is open leads to an error.

Add

This method loads the diagnostic document from the given path to the specified network.

If the document is an ODX/PDX or MDX file, the identifier of one ECU defined in the ODX file set must be given.

It is only possible to add a description while the measurement is stopped.

If there already is another description loaded in the configuration that has the same ECU qualifier, loading fails.

Note that in a CANdela diagnostic description, only one ECU is defined, while ODX/PDX or MDX files may contain any number of ECU definitions.

Adding or removing additional diagnostic descriptions while the measurement is running or the diagnostics configuration dialog is open leads to an error.

Additional descriptions are added to the interpretation order in the same order as they are added to the description. Changing the order can be done using the Move method of the InterpretationOrder object.

Syntax: object.Add( network, filePath[, ecuIdentifier])

(see example below)

AddDiagAccess

This method loads the diagnostic document from the given path for the given access type.

If the document is an ODX/PDX or MDX file, the identifier of one ECU defined in the ODX file set must be given.

It is only possible to add a description while the measurement is stopped.

If there already is another description loaded in the configuration that has the same ECU qualifier, loading fails.

Note that in a CANdela diagnostic description, only one ECU is defined, while ODX/PDX or MDX files may contain any number of ECU definitions.

Syntax: object.AddDiagAccess( accessType, filePath[, ecuIdentifier])

✔

This parameter expects the short name from the ODX/PDX or MDX file. This short name must not contain spaces (typically you would replace spaces by underscores ("_") for all parameters).

Type: DiagDescription

Remove

This method removes the indicated diagnostic description from the configuration.

It is only possible to remove a description while the measurement is stopped.

Adding or removing additional diagnostic descriptions while the measurement is running or the diagnostics configuration dialog is open leads to an error.

Example

Example — General

  • Sample 1

    Print the number of diagnostic descriptions loaded in the configuration:

    Wscript.echo "Number of descriptions: " &
    Application.Configuration.GeneralSetup.DiagnosticsSetup.DiagDescriptions.Count

  • Sample 2:

    This example adds two additional descriptions to the first "master" description and defines a certain interpretation order for them.

    All diagnostic windows for the first additional description and the fault memory window for the second additional description are opened.

    set app               = CreateObject("CANoe.Application")
    set masterDescription = app.Configuration.GeneralSetup.DiagnosticsSetup.DiagDescriptions.Item(1)
    set addDesc1          = desc.AdditionalDescriptions.Add("CDD\UDS-ExampleEcu-4.0.2.cdd")
    set addDesc2          = desc.AdditionalDescriptions.Add("ODX\UDS-ExampleEcu-8.0.1.pdx", "Door")

    masterDescription.InterpretationOrder.Move 1, 2

    addDesc1.OpenWindows
    addDesc2.OpenWindows 4

Example — AddDiagAccess method

  • Sample 1

    Load a diagnostic description to K-Line, then set its port to a network interface:

    Err.clear

    Set description = Application.Configuration.GeneralSetup.DiagnosticsSetup.DiagDescriptions.AddDiagAccess( 0, "D:\Descriptions\ECU1.CDD")

    If Err.number = 0 Then
      Wscript.echo "Loading OK: " & description.Qualifier
      description.Port = "K-Line1"
    Else
      Wscript.echo "Loading failed!"
    End If

  • Sample 2:

    Load a diagnostic description to K-Line, then set its port to a COM interface:

    Err.clear

    Set description = Application.Configuration.GeneralSetup.DiagnosticsSetup.DiagDescriptions.AddDiagAccess( 0, "D:\Descriptions\ECU1.CDD")

    If Err.number = 0 Then
      Wscript.echo "Loading OK: " & description.Qualifier
      description.Port = "COM3"
    Else
      Wscript.echo "Loading failed!"
    End If
