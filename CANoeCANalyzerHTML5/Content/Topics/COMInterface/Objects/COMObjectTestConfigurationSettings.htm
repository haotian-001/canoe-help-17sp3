TestConfigurationSettings

Application » Configuration » TestConfigurations » TestConfiguration » TestConfigurationSettings

Description

The TestConfigurationSettings object represents the settings of a test configuration.

DebugMode

This property sets/returns the settings specifying whether and how the debugger is pausing automatically.

See DebugModePausingVerdict for specifying a verdict mask for tree elements that are causing a debug pause.

The debug mode; possible values:

0|cNoDebugPause

2|cDebugPauseAfterEachTestCaseOrPattern

DebugModePausingVerdict

This property sets/returns the flag setting specifying which verdicts will trigger the debug pause, if the debug pausing mode is set to use this specific verdict.

The flags can be used in any combination using a bitwise OR operation, except as noted.

See also DebugMode for specifying the pausing mode of the debugger.

Syntax: object.DebugModePausingVerdict [=value]

(see example below)

Type: ENUM

The debug mode; possible values:

0|cVerdictFlagDisabled

Disables the debug pause; cannot be used with other flags.

1|cVerdictFlagPass

The debugger will pause after each passed element.

2|cVerdictFlagFail

The debugger will pause after each failed element.

4|cVerdictFlagNone

The debugger will pause after each element without a verdict.

8|cVerdictFlagInconclusive

The debugger will pause after each element with an inconclusive verdict.

The debugger will pause after each element with an error in test system verdict.

VerdictImpact

This property sets/returns the setting that defines the behavior of a test configuration when the one of the verdicts error in test system, fail or inconclusive occurs.

Syntax: object.VerdictImpact [= value]

(see example below)

8.5

Parameters:

object

Type: TestConfigurationSettings

value

(optional)

Type: ENUM

A value that defines the behavior of a test configuration when the one of the verdicts error in test system, fail or inconclusive occurs; possible values:

0|cTestConfigurationVerdictImpactNone

test cases or test patterns with verdict error in test system, fail or inconclusive have no effect on the execution of a test configuration

1|cTestConfigurationVerdictImpactEndOnFail

test cases or test patterns with verdict fail lead to an immediate abortion of the test configuration

2|cTestConfigurationVerdictImpactEndOnErrorInTestSystem

test cases or test patterns with verdict error in test system lead to an immediate abortion of the test configuration

4|cTestConfigurationVerdictImpactEndOnInconclusive

test cases or test patterns with verdict inconclusive lead to an immediate abortion of the test configuration

Note on Combined Flags

Since the flags are set bitwise they can be combined with one another.

For example if fail and error in test system should be set you have to use number 3 (011 in binary).

Example

Example — DebugMode/DebugModePausingVerdict property

Copy

In the following example the debugger is set to pause after passed and failed elements:

Const cVerdictFlagDisabled = 0, cVerdictFlagPass = 1, cVerdictFlagFail = 2
Const cVerdictFlagNone = 4, cVerdictFlagInconclusive = 8, cVerdictFlagError = 10
Const cNoPause = 0, cPauseAfterEachElement = 1, cPauseAfterSpecificVerdict = 2
Dim CanoeApp, TestConfiguration, Settings

Set CanoeApp          = CreateObject("CANoe.Application")
Set TestConfiguration = CanoeApp.Configuration.TestConfigurations(1)
Set Settings          = TestConfiguration.Settings

' sets the debugger to pause after passed or failed elements
Settings.DebugMode               = cPauseAfterSpecificVerdict
Settings.DebugModePausingVerdict = cVerdictFlagPass + cVerdictFlagFail

Example — ExecutionMode/NumberOfExecutions property / SetExecutionTime method

This example sets the execution mode of the test configuration.

Dim CanoeApp, TestConfigurations, TestConfig, Settings

Set CanoeApp           = CreateObject("CANoe.Application")
Set TestConfigurations = CanoeApp.Configuration.TestConfigurations
Set TestConfig         = TestConfigurations.Item(1)
Set Settings           = TestConfig.Settings

' single execution
Settings.ExecutionMode = 3

' multiple execution
Settings.ExecutionMode      = 0
Settings.NumberOfExecutions = 5

' time limited multiple execution
Settings.ExecutionMode = 1
Settings.SetExecutionTime 0, 1, 30

' endless multiple execution
Settings.ExecutionMode = 2

Example — IgnoreBreakOnErrorInTestSystem property

In the following example the test configuration will execute the test cases or test sequences completely regardless occurrence of Error in test system verdict:

Dim CanoeApp, TestConfiguration, Settings

Set CanoeApp          = CreateObject("CANoe.Application")
Set TestConfiguration = CanoeApp.Configuration.TestConfigurations(1)
Set Settings          = TestConfiguration.Settings

Settings.IgnoreBreakOnErrorInTestSystem = true

Example — RandomizeExecutionMode property

In the following example, the execution order of the test configuration is set to randomize each cycle separately:

Dim CanoeApp, TestConfiguration, Settings

Set CanoeApp          = CreateObject("CANoe.Application")
Set TestConfiguration = CanoeApp.Configuration.TestConfigurations(1)
Set Settings          = TestConfiguration.Settings

' randomize the execution order for each cycle separately
Settings.RandomizeExecutionMode = cRandomizeEachCycle
