TimeSection

Application » Configuration » MeasurementSetup » LoggingCollection » Logging » Exporter » TimeSection
Application » Configuration » MeasurementSetup » OfflineSource » TimeSection

Description

The TimeSection object represents the time section that will be considered while replaying a file in offline mode or converting a file using the applications logging converter features.

Type

Type: ENUM

For details on the different settings have a look at the Offline Mode help page.

The type of the defined time section; possible values:

0|cEntire

The entire file is considered.

1|cStartFromTimeStamp

The part between the start time stamp and the end time stamp is considered.

The time stamps have to be passed in the format "HH:MM:SS.MSEC".

2|cIntervalBetweenTwoTimeStamps

The part between the start time stamp and the end time stamp is considered.

The time stamps have to be passed in the format "HH:MM:SS.MSEC".

3|cBreakAtTimeStamp

The part between the start time stamp and the end time stamp is considered.

The time stamps have to be passed in the format "HH:MM:SS.MSEC".

4|cBreakAtDateTime

The part between the start date/time and the end date/time is considered.

The date/time has to be passed in the format "YYYY-MM-DD HH:MM:SS".

The passed date/times refer to the measurement start information contained in the logging file, if available.

5|cStartFromDateTime

The part between the start date/time and the end date/time is considered.

The date/time has to be passed in the format "YYYY-MM-DD HH:MM:SS".

The passed date/times refer to the measurement start information contained in the logging file, if available.

6|cIntervalBetweenTwoDateTimes

The part between the start date/time and the end date/time is considered.

The date/time has to be passed in the format "YYYY-MM-DD HH:MM:SS".

The passed date/times refer to the measurement start information contained in the logging file, if available.

7|cStartFromTimeStampInSeconds

The part between the start time stamp and the end time stamp is considered.

The time stamps have to be passed in the format "SS.MSEC".

8|cIntervalBetweenTwoTimeStampsInSeconds

The part between the start time stamp and the end time stamp is considered.

The time stamps have to be passed in the format "SS.MSEC".

Example

Example — General

The following example configures a time section for the offline source and defines a start and end time stamp to be considered.

Set offlineSource               = Application.Configuration.OfflineSetup.Source
offlineSource.TimeSection.Type  = 2
offlineSource.TimeSection.Start = "01:12:15.123"
offlineSource.TimeSection.End   = "02:13:16.321"
Application.Measurement.Start
