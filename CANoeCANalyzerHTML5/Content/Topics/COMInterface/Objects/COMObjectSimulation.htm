Application Â» Simulation

Description

The Simulation object represents the measurement functions in the Simulation mode. With the help of the Simulation object you can control the system time from an external source during the measurement.

CANoe automatically goes into Slave mode at the measurement start if you access the Simulation object.

Note

In Slave mode, only an animation factor of 1.0 makes sense.

You may use the Animation property (see below) only to reset the animation factor to 1.0. For more control of the simulation mode consider using CanoeApp.Configuration.OnlineSetup.

IncrementTime

This method increases the simulation time during the measurement in simulation mode.

Whereas IncrementTime returns immediately after calling, the IncrementTimeAndWait function returns not until the simulation step is done in CANoe.

Note that the system time also depends on the animation factor.

If this factor is 0.1, for example, the system time will be increased by 10 ms if the time is increased by 1 ms.

IncrementTimeAndWait

This method increases the simulation time during the measurement in simulation mode.

Whereas IncrementTime returns immediately after calling, the IncrementTimeAndWait method returns not until the simulation step is done in CANoe.

The increment must be a multiple of the CANoe base time period, which is by default 10 ms.

This value can be changed in the CANoe settings (CAN.ini), section System, key AnalyzeTimeBasePeriod.

Do not use IncrementTimeAndWait in combination with OnIdle or OnIdleU (deprecated), because this will lead to a deadlock.
