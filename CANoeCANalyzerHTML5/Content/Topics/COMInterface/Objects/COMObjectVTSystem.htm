VTSystem

Application » Configuration » VTSystem

Description

The VTSystem object represents the configuration of VT System in CANoe.

AdaptToHardware

When calling this method CANoe will adapt the current VT System configuration to the actually connected modules.

For connected modules with no counterpart in the configuration new modules will be added to the configuration.
Configured modules for which no matching connected module can be found will be disabled by switching them to offline mode.
Also the order of the connected modules might be changed to match the order of the connected modules.

Syntax: object.AdaptToHardware

(see example below)

15

NewConfigurationFromHardware

This method discards the current VT System configuration.

After that a new configuration will be created, which matches the actually connected VT System modules.

If no modules are currently connected, a blank configuration will be created.

Example

The following example written in PowerShell demonstrates the usage of various methods of the VT System instance:

# Get the VT System configuration instance
$canoe = New-Object -ComObject CANoe.Application
$vts   = $canoe.Configuration.VTSystem
# Export and import a configuration
$vts.ExportConfiguration("D:\MyConfiguration.vtcfg")
$vts.ImportConfiguration("D:\MyConfiguration.vtcfg", 0) # Replace current config
$vts.ImportConfiguration("D:\MyConfiguration.vtcfg", 1) # Appends to the current config

# Create a new configuration based on the connected hardware
$vts.NewConfigurationFromHardware()
# Adapt the existing configuration to the connected hardware
$vts.AdaptToHardware()

# Import a module description into CANoe
$vts.ImportModuleDescription("D:\MyModuleDescription.xml")

# Switch all modules offline/online
$vts.SetAllModulesOffline()
$vts.SetAllModulesOnline()

# Access a module or channel by its name
$myModule  = $vts.GetModuleByName("M1_VT1004")
$myChannel Ja, = $vts.GetChannelByName("M1_Ch1")
