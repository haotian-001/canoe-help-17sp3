TestConfigurationReportFilterSettings

Application » Configuration » TestConfigurations » TestConfiguration » TestConfigurationReport » TestConfigurationReportFilterSettings

Description

The TestConfigurationReportFilterSettings object represents the filter settings of a test configuration's report.

You can e.g. define, that only failed test cases are documented (output) in the test configuration report.

MaxTestStepLevel

This property sets/returns the setting up to which level test steps shall be included in the test report.

This property is not available for XML test modules.

Syntax: object.MaxTestStepLevel [= value]

(see example below)

9.0

Parameters

object

Type: TestConfigurationReportFilterSettings

value

Type: INT

Maximum level that a test step may have to be included in the test report.

  • A value of 0 means that only test steps of level 0 will be included.
  • A value of 1 means that only test steps of level 0 and level 1 will be included.

—

ReportedTestCases

This property sets/returns the flag settings specifying which test case shall be included in the test report details section.

The flags can be specified in any combination, except as noted.

A combination of flags can be specified using a bitwise OR operation.

Syntax: object.ReportedTestCases [=value]

(see example below)

9.0

Parameters

object

Type: TestConfigurationReportFilterSettings

value

Type: ENUM

Setting which test case shall be included in the test report details section; possible values:

0|cNoTestCase

No test cases will be included in the test report.

Cannot be used in combination with other flags.

1|cTestCasePass

Test cases with the verdict pass will be included in the test report.

2|cTestCaseFail

Test cases with the verdict fail will be included in the test report.

4|cTestCaseNone

Test cases with the verdict none will be included in the test report.

8|cTestCaseInconclusive

Test cases with the verdict inconclusive will be included in the test report.

16|cTestCaseError
  • Test cases with the verdict error in test system will be included in the test report.

    0|cNoTestStepCommand

    Neither test steps nor commands will be included in the test report.

    Cannot be used with other flags.

    1|cTestStepCommandPass

    Test steps and commands with the result pass will be included in the test report.

    2|cTestStepCommandFail

    Test steps and commands with the result fail will be included in the test report.

    4|cTestStepCommandWithoutVerdict

    Test steps and commands without result will be included in the test report.

    8|cTestStepCommandInconclusive

    Test steps and commands with the result inconclusive will be included in the test report.

    16|cTestStepCommandError

    Test steps and commands with the result error in test system will be included in the test report.

    Example

    Example — MaxTestStepLevel property

    In the following example the maximum level of reported test steps is set to 1:

    Dim CanoeApp, TestConfiguration, FilterSettings

    Set CanoeApp          = CreateObject("CANoe.Application")
    Set TestConfiguration = CanoeApp.Configuration.TestConfigurations(1)
    Set FilterSettings    = TestConfiguration.Report.FilterSettings

    ReportFilterSettings.MaxTestStepLevel = 1

    Example — ReportedTestCases property

    In the following example the test cases having the verdict none will be excluded from the test report:

    Dim CanoeApp, TestConfiguration, FilterSettings

    Set CanoeApp          = CreateObject("CANoe.Application")
    Set TestConfiguration = CanoeApp.Configuration.TestConfigurations(1)
    Set FilterSettings    = TestConfiguration.Report.FilterSettings

    Const tcPass = 1, tcFail = 2, tcNone = 4, tcInconclusive = 8, tcError = 16

    FilterSettings.ReportedTestCases = tcPass Or tcFail Or tcInconclusive Or tcError

    Example — ReportedTestStepsCommands property

    In the following example only the test steps and commands having the result passed or failed will be included in the test report:

    Dim CanoeApp, TestConfiguration, FilterSettings

    Set CanoeApp          = CreateObject("CANoe.Application")

    Set TestConfiguration = CanoeApp.Configuration.TestConfigurations(1)
    Set FilterSettings    = TestConfiguration.Report.FilterSettings

    Const cmdPass = 1, cmdFail = 2, cmdNone = 4, cmdInconclusive = 8, cmdError = 16

    FilterSettings.ReportedTestStepsCommands = cmdPass Or cmdFail

    Example — IsActive/SetActive method

    In the following example the comments, details and variable definitions are disabled from the test report:

    Dim CanoeApp, TestConfiguration, FilterSettings

    Set CanoeApp          = CreateObject("CANoe.Application")
    Set TestConfiguration = CanoeApp.Configuration.TestConfigurations(1)
    Set FilterSettings    = TestConfiguration.Report.FilterSettings

    If FilterSettings.IsActive(1) Then FilterSettings.SetActive 1, false
    If FilterSettings.IsActive(2) Then FilterSettings.SetActive 2, false
