Description

The TestReport object represents the reporting settings of a test environment, of a test module or of a directory in the CANoe test setup.

The reporting of test environments and directories in the CANoe test setup is always activated.

This means that Enabled always is set to TRUE.

LastWrittenFullName

Provides the full path and filename of the last written test report file.

An empty string is returned when no such file is known.

If the CANoe Test Report Viewer output format is used, and the test module is still running at the time this property is evaluated, the temporary path of the report currently being written is returned. This path can be used to open the report even before it is completed. Call the property again after the test run is finished to retrieve the final file path.

GenerateReport

In the TestReport objects of test environments or test modules or Test Setup directories this function creates a test report summarizing all test reports of all contained test modules. This is only possible for test environments and directories that contain test modules only.

This method cannot be called if a required XML test report is unavailable.

Syntax: object.GenerateReport

(You should use the GenerateReportAsync method to create test reports.)

deprecated

—

—

GenerateReportAsync

In the TestReport objects of test environments, test modules, or Test Setup directories this function creates a test report summarizing all test reports of all contained test modules. It runs in a background thread not blocking the GUI.

After finishing the generation of the HTML report the OnReportGenerated event is sent.

This method cannot be called if a required XML test report is unavailable.

While a transformation is running asynchronously in the background (started by COM), the test module for the same report must not be started again manually in the GUI.

Syntax: object.GenerateReportAsync

(see example below)

OnReportGenerated

This event is called after an HTML test report has been generated (successfully or not) from an XML test report.

The test report paths are often given even if generation is unsuccessful.

They can be used e.g. to delete incomplete test reports.

Example

Example — FilterSettings property

The following example accesses the property to switch off a filter:

Dim App, TestModule, FilterSettings

Set App            = CreateObject("CANoe.Application")
Set TestModule     = App.Configuration.TestSetup.TestEnvironments.Item(1).Items(1)
Set FilterSettings = TestModule.Report.FilterSettings

FilterSettings.SetActive 0, false

Example — LastWrittenFullName property

set app      = CreateObject("CANoe.Application")
set testEnvs = app.Configuration.TestSetup.TestEnvironments
set tenv     = testenvs(1)
set folders  = tenv.Folders
set folder   = folders(1)
set tms      = folder.TestModules
set tm       = tms(1)

' last written test report for the test environment:
set tr = tenv.Report
MsgBox "Last report for the Test Environment " & tr.LastWrittenFullName

' last written test report for the test folder:
set tr = folder.Report
MsgBox "Last report for the Folder: " & tr.LastWrittenFullName

' last written test report for the test module:
set tr = tm.Report
MsgBox "Last report for the Test Module : " & tr.LastWrittenFullName

Example — GenerateReportAsync method

Dim Application, Report, Finished
Set Application = CreateObject("CANoe.Application")   ' access a CANoe application via the COM Interface
Set Report      = Application.Configuration.TestSetup.TestEnvironments.Item(1).Items.Item(1).Report

WScript.ConnectObject Report, "Report_"   ' Build up the connection to the event handlers

Finished = False
Report.GenerateReportAsync()
While (Not Finished)
  Wscript.Sleep 100
Wend

' Deallocate objects: every object shall be deallocated at the end of the script.
Set Application = Nothing
Set Report      = Nothing
Set Finished    = Nothing

Sub Report_OnReportGenerated(Success, SourceFullName, GeneratedFullName)
  Finished = True
End Sub

Example — OnReportGenerated event

In the following example, a test module is started. After the test, the HTML test report is copied to another directory.

Dim App, Meas, TestModule, TMStarted, MeasStarted, FileSys, SourceFile
Dim GenFile, ReportGenerated, ReportGenSuccess, ReportSource
Dim TMStopReason

TMStarted        = False
TMStopped        = False
MeasStarted      = False
ReportGenerated  = False
ReportGenSuccess = False
ReportSource     = ""

Set FileSys    = CreateObject("Scripting.FilesystemObject")
Set App        = CreateObject("CANoe.Application")
Set TestModule = App.Configuration.TestSetup.TestEnvironments.Item(1).Items(1)
Set Meas       = App.Measurement

' Set up connection to event handlers
WScript.ConnectObject TestModule, "TestModule_"
WScript.ConnectObject Meas, "Meas_"

' Start Measurement
Meas.Start()
While (Not MeasStarted)
  WScript.Sleep(200)
Wend

' Start Testmodule
TestModule.Start()
While (Not TMStarted)
   WScript.Sleep(200)
Wend

' Wait until report is generated
While (Not ReportGenerated)
   WScript.Sleep(200)
Wend

TesterDecision = MsgBox("Report generated: " & ReportGenSuccess & ". Move XML report file?", vbYesNo)
If TesterDecision = vbNo Then
  WScript.Quit()
Else
  Set GenFile = FileSys.GetFile(GenFile)
  If (ReportGenSuccess) then
' If successful, move the report to other folder (folder must exist, or operation will fail)
    GenFile.Move("C:\TestReports\")
  End If
End If

' Event Handlers
Sub TestModule_OnStart()
TMStarted = True
End Sub

Sub TestModule_OnReportGenerated(Success, SourceFullName, GeneratedFullName)
ReportGenerated  = True
ReportGenSuccess = Success
SourceFile       = SourceFullName
GenFile          = GeneratedFullName
End Sub

Sub Meas_OnStart()
MeasStarted = True
End Sub
