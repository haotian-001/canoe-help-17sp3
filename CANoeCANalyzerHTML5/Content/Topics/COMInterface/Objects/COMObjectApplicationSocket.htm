ApplicationSocket

Application » Networks » Network » Devices » Device » ApplicationSocket

Description

The ApplicationSocket object represents the MOST Network Management Layer of CANoe.

Properties

Name

Availability

(since version)

Property Value/Type

Property Value Element

BusRegistry

This property determines the current bus registry.

The bus registry is a central list in which all function blocks of the complete MOST ring are stored with their according node address and InstanceID.

The bus registry is only valid if the OnNetState event has the value configOk.

Outside a measurement the determination of the bus registry returns the last valid table (before measurement stop).

If no measurement has been started, so far, an empty array will be returned.

Syntax: object.BusRegistry

(read-only)

(see example below)

✔

Type: BYTE ARRAY

The bus registry as a byte array with dynamic length.

Return values:

A byte array of the following length: 4 * Number of entries, that contains the current Bus Registry.

Each entry is represented in the byte array as follows:

byte 1

The high order byte of the node address.

byte 2

The low order byte of the node address.

byte 3

The Function Block ID.

byte 4

The Instance ID.

—

FBlockList

This property returns the FBlock list as a byte arrays with dynamic length.

Outside a measurement the determination of the FBlock list returns the last valid table (before measurement stop).

If no measurement has been started, so far, an empty array will be returned.

Syntax: object.FBlockList

(read-only)

(see example below)

✔

Type: BYTE ARRAY

The FBlock list as a byte array with dynamic length.

Return values:

The FBlock list as a byte array with dynamic length (Length= 2 * Number of entries of the current FBlock list).

Each entry is represented in the byte array in the following way:

byte 1

The Function Block ID.

byte 2

The Instance ID.

OnBusRegistry

This event occurs when the Bus Registry has changed or has been queried.

The current Bus Registry will only be responded during measurement.

To assure that the current registry is available in every situation after measurement start, the Poll Configuration status after measurement start option should be activated in the hardware settings of all channels.

High time stamps require a special treatment.

Syntax: object_OnBusRegistry( timeHigh, time, dataArray )

(see example below)

✔

OnFBlockList

This event occurs if the local FBlock List has changed or has been queried.

The FBlock List is a local list, in which all function blocks of the simulated node of the connected network interface are stored with their according InstanceIDs.

The local FBlock List will only be responded during measurement.
High time stamps require a special treatment.

Syntax: object_OnFBlockList( timeHigh, time, dataArray )

Example

Example — BusRegistry property

The following example determines the current bus registry:

Dim myRegistry As Variant
myRegistry = myApplicationSocketInterface.BusRegistry

Example — FBlockList property

The following example determines the current FBlock list:

Dim myFBlockList As Variant
myFBlockList = myApplicationSocketInterface.FBlockList

Example — OnBusRegistry event

The following example displays the first entry of the Bus Registry in a MessageBox:

Sub myApplicationSocketInterface_OnBusRegistry( ByVal timeHigh As Long, ByVal time As Long, ByVal dataArray As Variant )
  Dim adr As Long
  Set adr = dataArray(0) * &H100
  adr = adr + dataArray(1)
  MsgBox "RxTxLog: 0x" & Hex(adr) & ", FBlockID: " & dataArray(2) & ", InstanceID: " & dataArray(3)
End Sub

Example — OnFBlockList event

The following example displays the first entry of the local FBlock List in a MessageBox:

Sub myApplicationSocketInterface_OnFBlockList( ByVal timeHigh As Long, ByVal time As Long, ByVal dataArray As Variant )
  MsgBox FBlockID: " & dataArray(2) & ", InstanceID: " & dataArray(3)
End Sub
