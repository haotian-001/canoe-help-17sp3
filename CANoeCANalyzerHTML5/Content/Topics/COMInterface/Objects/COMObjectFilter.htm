Description

Add

This method adds a message or a signal to the explorer's filter.

The Exporter object needs fully qualified names of all messages and signals that have to be taken into consideration during export or conversion:

  • Messages:
    <DatabaseName>::<MessageName>
  • Signals:
    <DatabaseName>::<MessageName>::<SignalName>
  • System variables:
    <Namespace>::<SystemVariable>
  • Environment variables:
    <DatabaseName>::<EnvironmentVariable>

When performing a message-oriented conversion only messages, system variables and environment variables are taken into consideration. Any (even specified) signals will be ignored.

When performing a signal-orientierted export messages, signals, system variables and environment variables are taken into consideration. All signals included in specified messages, system variables and environment variables will be exported as well.

Besides using their symbolic name messages can be declared by using their numeric ID, which either can be decimal or hexadecimal.

Note

  • With CANoe 10.0 all message and system variable types are supported.
  • Only messages or system variables which have explicitly been added will be considered during export or conversion.

Syntax: object.Add(fullname)

Example

Example — Enabled property

You have to activate the Filter object before you can use it:

Filter.Enabled = True   ' turn on filter

The following example assigns the Filter object to a variable:

Dim Filter
Set Filter = Application.Configuration.OnlineSetup.LoggingCollection(1).Exporter.Filter

Example — Add method

The following example adds the message Comfort::Console_1, the system variable PowerTrain::BrakeActive and the signal PowerTrain::ABSdata::CarSpeed to the filter. The signal is declared using the hexadecimal identifier (C9) of the message:

Set Exporter = Application.Configuration.OnlineSetup.LoggingCollection(1).Exporter
Set Filter   = Exporter.Filter
Filter.Add("Comfort::Console_1")
Filter.Add("PowerTrain::BrakeActive")
Filter.Add("PowerTrain::0xC9::CarSpeed")

When doing a signal-orientierted export all signals of the message Comfort::Console_1, the system variable PowerTrain::BrakeActive and the signal PowerTrain::ABSdata::CarSpeed will be taken into consideration.

When doing a message-orientierted conversion the signal Powertrain::ABSdata::CarSpeed will not be considered. The system variable PowerTrain::BrakeActive semantically has to be seen as a signal. Nevertheless in logging formats it is treated as a discrete event and therefore is taken into consideration during a message-oriented conversion.
