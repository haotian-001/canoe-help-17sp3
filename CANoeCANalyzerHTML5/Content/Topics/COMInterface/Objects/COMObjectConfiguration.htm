ExecutionEnvironment

This property sets or returns the variant (32-bit or 64-bit) the RT kernel should be used as execution environment.

If you change variant of the RT kernel, the current execution environment will be terminated and the selected one will be started. At the same time all modules will be loaded in the selected execution environment.

Please note that the 32-bit variant is still used for the execution in real-time mode (distributed mode or standalone mode) and this setting has no effect on it.

If the EnforceRtkTargetArchitecture switch in [SYSTEM] section in the CAN.ini is set to 1 (for 32-bit variant) or 2 (for 64-bit variant), the selection in this dialog is deactivated.

You can find further information on the Notes on 64-Bit/32-Bit help page.

HwConfigurationSelection

Configurations for Ethernet devices (XML files) can be configured in the Vector Hardware Configuration dialog and several sets of such configurations can be provided.

However at runtime only one set can be active and the active set must be stored in the configuration. This property allows to change the active set with changing the configuration, and moreover to loop over all provided sets by external scripts.

The getter variant retrieves the number of available sets.

Syntax: object.HwConfigurationSelection (=value)

Modified

This property returns information on whether the current configuration was modified since the time it was loaded or created, or sets this property.

This property determines whether the user is prompted to save when another configuration is loaded.

If the Modified property is set to false with a modified configuration, other configurations can be loaded without any user interface prompts.

This makes it possible, for instance, to automatically discard a modified configuration used during an automated test, in order to return to the original configuration.

However, the modified configuration is then lost!

(read-only)

Save

This method saves the configuration.

Syntax: object.Save([path], [promptUser])

3.0

path, promptUser

object

Type: Configuration

path

(optional)

Type: STRING

The complete filename.

The configuration will be saved under a different path/name.

CANoe will automatically open this newly saved configuration.

If no path is specified, the configuration is saved under its current name.

If it is not saved yet, the user will be prompted for a name.

promptUser

(optional)

Type: BOOL

Indicates whether the user should intervene in error situations, see below.

If not specified, the default value is True; possible values:

TRUE

(default)

The user is prompted for input in error situations.

FALSE

The user is not notified.

If False is input for the promptUser parameter, the function fails if an error situation occurs.

Possible error situations are:

  • The active configuration was created or changed and was not saved.

  • Saving fails for some reason (write protection, full disk).

—

SaveAs

The configuration will be saved under a different name with all necessary compatibility adjustments for the selected version.

CANoe will not automatically open this newly saved configuration.

To use this type, the configuration must have been saved in the current version format and must not contain any unsaved changes.

See also Save Configuration As.

Syntax: object.SaveAs(path, major, minor, [promptUser])

3.0

path,major, minor, promptUser

object

Type: Configuration

path

Type: STRING

The complete filename.

major

The major version number of the target version,

  • e.g. 13 for CANoe 13.0.

  • e.g. 15 for CANoe 15.

minor

The minor version number of the target version, e.g. 0 for CANoe 13.0

  • Up to and including version 8.5, this parameter represents the "real" minor version of the application (e.g. 5 for CANoe 8.5).

  • In all later versions its value must always be set to 0.

Example — CLibraries property

Set App   = CreateObject( "CANoe.Application")
Set Files = App.Configuration.CLibraries
Set File  = Files.Add( "c:\test.TXT")

i = 1
While i <= Files.Count
  Text = Files( i).FullName
  Text = Text & " is "
  If Files( i).Enabled Then
    Text = Text & "enabled."
  Else
    Text = Text & "disabled."
  End If
  MsgBox Text
  i = i + 1
Wend

Files.Remove( 1)

Set File  = Nothing
Set Files = Nothing
Set App   = Nothing

End If

Example — StandaloneMode property

  MsgBox "Error:" & Chr(13) & exportError
Else
  MsgBox "Ok."
End If

Set StandaloneMode = Nothing
Application.Quit

Example — StartValueList property

Set mCANoeApp = CreateObject("CANoe.Application")

' Open CANoe configuration
mCANoeApp.Open "MyConfiguration.cfg", True, True

Set ocResult = mCANoeApp.Configuration.OpenConfigurationResult

If (ocResult.Result > 0) Then
  MsgBox "Configuration could not be opened."
  Set ocResult = Nothing
  mCANoeApp.Quit
  Set mCANoeApp = Nothing
  MsgBox "Script done..."
End If

Set mStartValueList = mCANoeApp.Configuration.StartValueList
Dim mValidationMessage

' Load
If (mStartValueList.Load("File1.svc", mValidationMessage)) Then
  If (Not (Len(mValidationMessage & "") = 0)) Then
    MsgBox "Start values file successfully imported. Message: " & mValidationMessage
  Else
    MsgBox "Start values file successfully imported."
  End If
Else
  MsgBox "Start values file import failed. Message: " & mValidationMessage
End If

' Close CANoe
Set mStartValueList = Nothing
mCANoeApp.Quit
Set mCANoeApp = Nothing

MsgBox "Script done..."
