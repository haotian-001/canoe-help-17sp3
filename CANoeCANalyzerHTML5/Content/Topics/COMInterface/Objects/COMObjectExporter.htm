Exporter

Application » Configuration » MeasurementSetup » LoggingCollection » Logging » Exporter
Application » Configuration » StandaloneMode » StandaloneLogging » Exporter

Description

The Exporter object represents an export dialog, as it can be used in CANoe e.g. in a Logging Block in the Measurement Setup.

Destinations

Returns a Files object for the destination files of an exporter.

On the first access to the destination files of an exporter the Files object contains the destination file as it is defined in the according Export dialog.

If no destination file is set a destination file is derived from the source file.
Name and path of this destination file correspond to the name and path of the source file. The format is CSV.
If as many destination files as source files are given, an according destination file will be generated for each source file (n to n).

If the numbers of destination files and source files don't match, the number of destination files must be 1. All source files will be merged into the destination file (n to 1).

Syntax: object.Destinations

Sources

Returns a Files object for each source file of the exporter or offline source.

On the first access to the source files of the exporter the Files object contains the source file as defined in the Export dialog.

Typically this is the logging file as set in the Logging Block, as long as no other source file has been defined manually.

Load

Loads all source files of an exporter and determines the contained signals and messages; see CANSystemDemo – Logging Filter.vbs (COM Sample).

If several source files are set, all signals and messages of all source files are determined.

Save

Starts the export/conversion; see CANSystemDemo – Logging Filter.vbs (COM Sample).

Although the parameter noPromptUser is set to True, the function will fail if a failure situation occurs and the storage cannot be performed.

Possible failure situations are e.g. write-protection or disk full.

Syntax: object.Save([noPromptUser])

✔

Example

Example — Messages property

The following example determines all messages in the source files of an exporter and adds them to the exporter's filter:

Set Exporter = Application.Configuration.OnlineSetup.LoggingCollection(1).Exporter
Set Filter   = Exporter.Filter

Exporter.Load

For Each Message in Exporter.Messages
  Filter.Add(Message.FullName)
Next

Example — Symbols property

The following example detects all symbols in the source files of an exporter and adds them to the filter:

Set Exporter = Application.Configuration.OnlineSetup.LoggingCollection(1).Exporter
Set Filter   = Exporter.Filter

Exporter.Load
For Each Symbol in Exporter.Symbols
  Filter.Add(Symbol.FullName)
Next
