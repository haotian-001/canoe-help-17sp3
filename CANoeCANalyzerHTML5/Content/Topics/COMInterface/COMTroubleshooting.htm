Troubleshooting

 

COM Server Unavailable

You get the following error message if the COM Interface was not installed properly on your computer:

In this case repeat the registration process for the COM Interface.

Microsoft Windows PowerShell Error Notification

Error Notification: "Specified cast is not valid / Expression of type 'System.Reflection.Missing' cannot be used for parameter…"

When you use Microsoft Windows PowerShell to call functions of the COM Interface of CANoe, you may experience one of the following error notifications:

  • Exception calling <method> with <count> argument(s): Specified cast is not valid.

  • Expression of type System.Reflection.Missing cannot be used for parameter of type <type> of method <method>.

You get this error notifications when you call a function with optional parameters and do not specify a value for these optional parameters, e.g.:

$application = New-Object -COM CANoe.Application
$application.Open("test.cfg")

To call the function with the default values for optional parameters, specify "[Type]::Missing", e.g.:

$application = New-Object -COM CANoe.Application
$application.Open("test.cfg", [Type]::Missing, [Type]::Missing)

VB Demo Does Not Start

If the following error message appears when you start the VB demo application, your system is lacking the Visual Basic Runtime environment:

Execute the following file: Redist\VBRun60sp3.EXE
