Notes on Writing XML Files

 

When creating XML files, please heed the following notes.

Empty Spaces And Line Breaks

In many places in the XML file, character strings are expected that are enclosed in corresponding XML elements, for example:

<description>Description test</description>

Here a description test is enclosed by the element <description>. It should be noted that all characters after the > of the opening XML tag up to the </ of the closing XML tag belong to the description test, also leading empty spaces and line breaks. The line above is therefore not the same as the apparently only better-formatted.

<description>
  Description text
</description>

The corresponding breaks and empty spaces were copied here into the protocol file, which may lead to less-than-beautiful output. In case of internally-used specifications, e.g. numeric specifications for signal values, this can even lead to the failure of the test because parameters cannot be assigned the correct values.

Note

Line breaks cannot be used to layout a descriptive text in an HTML report with line breaks.

The line breaks will be lost.

Characters With Fixed Meanings in XML

Some characters have special meaning in XML and may therefore not occur in character strings. These are especially the characters <, > and &. For these, the corresponding XML substitute sequences must be entered (here &lt; and &gt; and &amp;).

Note

These substitute sequences are the same ones as in HTML.

Inserting HTML Code

HTML-Code itself has an XML-like structure, which eventually causes problems in the processing of XML files that contain HTML code in <extendedinfo> lines. Essentially three problems can occur:

  • Since the test module XML schema does not recognize HTML code, a validating editor will report errors.
  • HTML permits tags that are not closed, e.g. <img src="pic.jpg">. In XML there would have to be a closing tag </img>, which is permitted in HTML but is not required.
  • HTML permits the specification of attribute values without quotation marks, that is, e.g. <img src=pic.jpg>. In XML the value must always be placed in quotation marks.

Note

You can avoid these problems

  • by formulating the HTML code conform to XHTML or
  • by placing the HTML code in a CDATA section.
    If the HTML code contains the "<" (&lt;) character entity, you must use the CDATA section anyway.

Example 1: <img src="icon.jpg"/><p>Valid XHTML Code</p>

Example 2: <![CDATA[<img src=icon.jpg><p>HTML, not XML-conformant Code<p>]]>

Copy/Paste From Existing Protocol Files

Some information is only taken from the XML test module file and copied into the associated XML protocol. These structures generally have the same XML format, so that conversely, information blocks such as specifications about the test setup can be copied from existing XML protocol files into the XML test module file.

This way, it is possible to work more efficiently when writing XML test module files if it is possible to borrow from existing protocol files.
