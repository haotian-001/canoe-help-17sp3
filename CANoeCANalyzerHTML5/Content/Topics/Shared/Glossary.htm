A navigation and measurement system, actually NAVSTAR-GPS (Navigation Satellite Timing and Ranging Global Positioning System). An arrangement of US satellites that makes it possible for users with suitable receivers to determine their position on or above the surface of the earth. GPS currently consists of 27 active satellites. If signals are received from at least four or more GPS satellites, this makes it possible for Â»WGS coordinates to be determined for the receiver. The accuracy depends on the output level of the receiver, the frequencies used and the time available for reception. Realtime navigation and positioning are possible with accuracy in the range of centimeters.

In addition to the US system, there is also a Russian system, Â»GLONASS and a future Russian system called Â»GNSS.

The Russian counterpart to the American Â»NAVSTAR-GPS. It has been fully available since January 1996. Just like the American GPS system, every GLONASS satellite sends two signals in the GHz range (1.2 GHz and 1.6 GHz). In contrast to the NAVSTAR GPS system, however, each GLONASS satellite has its own frequency. These differences in frequency serve to identify the satellites. There are receivers that are capable of receiving and evaluating the signals of both systems.

Test Case

A test case is the semantic test entity. A test case returns a unique test result in form of a Â»verdict and appears in the Â»test report. A test case can be a direct component of the execution tree, such as an element within a Â»test group. In this instance, the test case is visible in CANoe at execution time and for example can be activated/deactivated or individually started for the execution. Alternatively, several test cases can be activated and started as a related entity, if they are combined using a Â»test sequence.

A test case can optionally have a preparation section and a completion section. All commands contained in these sections are always executed (at the start or end of the test case) if the text case is executed. These commands are executed regardless of a possible abort of the test case due to a failed test.

Test Sequence

Test sequences are only available with Â»test units.

A test sequence is an implementation entity that implements 1â¦n test cases. The construct of the test sequence enables the semantic entity of the Â»test case to be separated from the implementation entity that is visible in the execution tree. A test sequence is visible in the execution tree in CANoe and for example can be activated and deactivated there for the test execution. On the other hand, the test cases contained in a test sequence are not individually visible in the static execution tree in CANoe and therefore cannot be activated and deactivated. With the help of a test sequence it is thus possible to implement n related test cases that cannot be executed individually. In addition to the test cases, a test sequence can contain unlimited execution code between the test cases.

Analogous to a test case, a test sequence can also have a preparation and/or a completion. Their commands are always executed regardless of a possible abort response of the test sequence.

Test Step

A test step describes a part of test execution which is entered in the Â»test report at specific time points. The use of test steps allows Â»test cases to be reported in a finely structured and comprehensible manner. Test steps can contain a notes about the meaning of the test step for the entire test case:

  • without verdict
    The test step only specifies that an action was carried out. The execution of the test step passes no judgment with respect to the total result. Example: a message was sent.
  • pass
    In this test step, an expected result was achieved, for example a message arrived as expected.
  • inconclusive
    This test step couldn't be judged finally; e.g. due to an interaction in the Tester Confirmation dialog.
  • fail
    In this test step, a deviation from the expected behavior was detected, which will lead to a fail of the Â»test case.
  • error in test system
    In this step an error occurred in the test system.
  • warning
    In the test step no deviation from the expected behavior was detected, however the behavior detected could contribute to a problem later on, which will then lead to a fail of the Â»test case.
