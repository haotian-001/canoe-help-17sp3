Symbol Explorer

(CANoe) Ribbon: Environment Tab|Symbol Explorer
(CAPL Browser) Ribbon: Layout Tab|Symbols

You can assign symbols to the windows (CANoe) / Text Editor (CAPL Browser) / Panel Designer (Vector Tools Environment) via drag and drop from the Symbol Explorer.

The Symbol Explorer supports several options for accessing the same symbol. For example, you can select a symbol directly from the signal or message list or find the corresponding symbol via a specific node.

Selecting Symbols

Use the toolbar to change the display of the tree view in the Symbol Explorer. Depending on the selection you make, either the available network symbols , variables , participants , application layer objects (communication objects, distributed objects)), diagnostic classes or diagnostic services will be displayed. With you can select for the diagnostic classes and services if the names (longname) or qualifiers (shortname) should be displayed. Select the desired symbol in the tree view and drag and drop to assign it to the desired window.

Only those application layer objects are displayed that you switched to Simulated (CANoe) in the Communication Setup or that you measure.

With you can collapse the tree view.

In the CAPL Browser an symbol is qualified automatically after inserting in the Document View.

Search Options

You can search for a word or part of a word using the input field of the search . All existing texts in all columns are taken into account.

The tree view will be filtered by matches during entry. The filtered matches are highlighted in color. The tree view does not contain any entries if no match is found in it. You can delete the search entry using . The tree view will be shown completely.

The most recently used search terms can be selected again via a selection list.

The wildcards * and ? are supported. The texts will be searched for character strings.

  • * stands for an arbitrary number of characters.
    dow*Do finds e.g. WindowLeftDown and WindowRightDown
  • ? stands for exactly one character.
    dow????Do finds e.g. WindowLeftDown

Note

As long as the search in CANoe has not been completed, the Suspend Search command is not available, nor can you use the link to change the network of a database.

Suspend Search (Partially Canceling of Search)

When you search for an element in the Symbol Explorer, only the matches (turn on the image) will be shown in the tree view.

If you would like to know which element is located in LightSystem for example, you can use the context menu command Suspend Search or the link in the tooltip to partially cancel the search. That means all the sub-elements for LightSystem will now be displayed.

If you have partially canceled a search, the search term will no longer be highlighted. The element from which the search was canceled is marked with the symbol.
For arrays you can now reduce/expand the aggregated view again with a click on [Collapse Items]/[Expand Items].

With the context menu command Resume Search you can resume the partial search.

You can cancel and resume the partial search for several subtrees.

Navigation Path

Beneath the search area, the path to the currently selected element (Mirror_l2d on the image) is shown. This path is especially helpful for (vertically) very long lists, if the superordinate elements are not located in visible area.

Use the arrow (<Alt> + <Arrow to the left>) to navigate to a previously selected symbol, use the arrow (<Alt> + <Arrow to the right>) to navigate back to the symbol selected afterwards. A history of up to 50 selected symbols is supported.

By clicking on an element in the navigation path, you can navigate to this element in the tree view.

By clicking on the arrows in the navigation path, you can navigate to the sub-elements of the selected element in the tree view.
On the image Console_1, Console_2, NM_Console and TP_Console are sub-elements of Tx Frames.

All currently invisible elements of the path can be displayed by clicking on the double arrows.

The navigation path may not be completely displayed due to the width of the Symbol Explorer.

Aggregations

For simplified usage the following elements are merged to form aggregations:

  • Arrays
  • Networks

The following editors are available for selecting the elements to be included in an aggregation.

List Editor

The List Editor can be used to select one or more elements. For networks and special arrays for selecting a point in time multiple elements can be selected.

For selecting a single element, click with the mouse on the desired element.

For selecting multiple elements, proceed as follows:

  • Left mouse button + <Ctrl>: Selection of individual elements
  • Left mouse button + <Shift>: Selection of several successive elements

Unspecified Array Selection

For Some/IP there are arrays whose number of elements is unknown when elements are selected. For these arrays, you can only determine the index via a text field and not via multi-selection. The arrays may be multi-dimensional as well. In this case, you can specify individual dimensions separated by a comma.

Example

Dynamic Arrays

For communication objects and distributed objects there are arrays whose number of elements can be dynamic. For these arrays, the length of the array is displayed as file in the tree view. This allows you to drag this property onto a panel and edit it.

Additional Information for CANoe

If you have opened the Symbol Explorer once, it is displayed on every desktop of the configuration. After configuring the analysis windows you can auto-hide the Symbol Explorer via . Then the Symbol Explorer will be displayed as a page at the border of the main window.

Edit Symbol Values

When you open the Symbol Explorer, depending on the symbol type, you can open the Symbol Panel or the Application Panel for each displayed symbol via the Change Value… context menu command. There you can view and change values.
