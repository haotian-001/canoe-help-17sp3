Symbolic Signal Access

 

When using a project database individual signals can be accessed by entering their names. The compiler assumes the task of determining the bit offset and bit length as well as the task of converting the machine format (Intel <-> Motorola). The values are interpreted in raw format.

Operator phys

If a physical interpretation is to be used, the operator phys must be appended. The compiler then scales the value according to the conversion formula (factor, offset) indicated in the database. You can assign symbolic descriptions to the signal values using value tables in the CAN database.

Example Symbolic Signal Access

If you have defined the message EngineData with a signal Rpm in the database you would write:

message EngineData m;
char errSymbol[32];
...
m.Rpm      = 100;
m.Rpm.phys = 123.5;                   // Use of conversion formula
m.Rpm      = EngineData.Rpm::Error    // Use a symbolic value description
if(m.Rpm  == EngineData.Rpm::Error)   // Accessing the symbolic
write("EngineData.Rpm = Error");      //description of a signal value

In the first line the raw value 100 is assigned to the data bits which you have allocated to the signal Rpm in the database. In the second line the conversion formula assigned in the database is utilized to calculate the bit value. In the third line the symbolic description of a value is used. The symbolic value description will be converted into a raw value using the value table of the signal. The last line shows an access of the symbolic value description.

Note

Saving the signal values is always discrete. If a physical value is assigned, the closest discrete raw value is saved after scaling. If the signal is then read out, the result will not necessarily agree with the initial value. The deviation for floating point numbers is usually negligible.

Data selectors

With data selectors you can also access to single bytes of a signal. You can use as data selectors the integer CAPL data types (long, dword, int, byte,â¦) in the same way as for the access on the message bytes. As parameter the index of the first selected byte is transferred.

Example

message MsgWithLargeSignal msg;
msg.LargeSignal.byte(0) = 0x1;
msg.LargeSignal.byte(1) = 0x2;
msg.LargeSignal.word(2) = 0x3;

Note

Data selectors are available since version 8.0 SP4 and can not be combined with the selector .phys.
