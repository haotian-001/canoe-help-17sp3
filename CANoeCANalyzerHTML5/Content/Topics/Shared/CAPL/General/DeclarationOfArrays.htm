Declaration of Arrays

 

Declaration of arrays (arrays, vectors, matrices) is permitted in CAPL. They are used and initialized in a manner analogous to C language. A particular feature is that arrays of the type CHAR can be initialized by assigning a string.

Arrays are passed formally, as in C. However, as an expansion it is also possible to pass multidimensional arrays. The number of elements of a particular dimension can be queried with the function elCount. The compiler executes a value range check. If an index less than zero or greater than the number of elements is used, a runtime error is generated by runError.

Example 1

variables {

   int vector[5] = {1,2,3,4,5};

   int matrix[2][2] = {{11,12},{21,22}};

}

write (int v[]) {
   int i;
   for (i=0; i<elCount(v); i++) write("%03d",v[i]);
}

write (int v[][]) { // Overload function
int j;
   for (j=0; j<elCount(v); j++) write(v[j]);   // Size by elCount()
}

on start {
   write(vector);
   write(matrix);
}

Message arrays can be initialized with local (previously declared) variables or with database definitions. If you select the message * array type, the messages in the array may have different types.

Example 2

message ABSdata msgABSdata = {byte(0) = 0x11, byte(1) = 0x22, byte(2) = 0x33, byte(3) = 0x44, byte(4) = 0x55, byte(5) = 0x66, byte(6) = 0x77, byte(7) = 0x88};

message motbus::EngineData msgEngineData;

message motbus::GearBoxInfo msgGearBoxInfo = {byte(0) = 0x66};

message * msgArray[4] = {msgABSdata, motbus::EngineData, msgGearBoxInfo, {dlc = 8}};

Parts of fields of type byte or char can be interpreted and read out as single values using selectors. The selectors have the name of the type that they return; the offset in bytes inside the array must be specified in brackets.

Example 3

byte datagram[128];
datagram.dword(0) = 0xB869DB8D   // magic cookie
datagram.WORD(4)  = 0;           // major version
datagram.WORD(6)  = 2;           // minor version
