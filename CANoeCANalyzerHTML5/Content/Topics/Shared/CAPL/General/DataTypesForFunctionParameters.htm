Data Types for Function Parameters

 

You can use the following data types in function parameter declarations:

  • Integers (byte, word, dword, int, long, qword, int64)
    Example declaration: long 1
  • Floating point numbers (float, double)
    Example declaration: double d
  • Individual characters (char)
    Example declaration: char ch
  • Enums
    Example declaration: enum Colors c
  • Associative fields
    Example declaration: int m[float]. Associative fields are transferred as reference automatically.
  • Structures
    Example declaration: struct TCPMessage m. Structures are transferred as reference automatically.
  • References to integers, floating point numbers, individual characters and enums.
    Example declaration: word& w
  • Timers (timer, mstimer)
    Example declaration: timer t
  • Messages (message, pdu, pg, linFrame, frFrame, frPDU, mostMessage, mostAMSmessage, mostrawmessage, gmlanMessage, j1587Param); either an actual message or *.
    Example declaration: message 100 m, linFrame * m
  • Signals, only with *
    Example declaration: signal * s or serviceSignalString * s
  • Environment variables, with indication of type and *.
    Possible types: Int, Float, String, and Data.
    Example declaration: envvarInt * ev
  • System variables, with indication of type and *.
    Possible types: Data, Int, Long, Float, String, IntArray, and FloatArray.
    Example declaration: sysvarFloat * sv
  • Diagnostics requests, only with *
    Example declaration: diagRequest * dr
  • Diagnostics responses, only with *
    Example declaration: diagResponse * dr
  • FlexRay configuration
    Example declaration: frConfiguration frConf
  • Fields of the above data types, with the exception of timers, signals, environment and system variables, diagnostics requests and responses, references.
    Example declaration: int matrix[][]
  • Message definitions from the database, only with *.
    Please note that the parameter is not a concrete message; this would have the type message.
    Example declaration: dbMsg * messageDefinition
  • FlexRay frame or PDU definitions from the database, only with *.
    Please note that the parameter is not a concrete frame or PDU; these would have the type frFrame.or frpdu.
    Example declaration: dbFrFrame * frFrameDefinition, dbFrPdu * frPDUDefinition
  • Signal definitions from the database, only with *.
    Please note that the parameter is not a concrete signal; this would have the type signal.
    Example declaration: dbSig * signalDefinition
  • Node definitions from the database, only with *.
    Example declaration: dbNode * nodeDefinition
  • PDU definitions from the database, only with *.
    Please note that the parameter is not a concrete PDU; this would have the type pdu.
    Example declaration: dbPDU * pduDefinition
  • Data types for communication concept
  • Structures and arrays that are defined in the communication description
  • Enumeration types that are defined in the communication description
  • callContext
  • References to endpoints, e.g. serviceProviderRef, serviceConsumerRef, eventProviderRef, eventConsumerRef (see also Objects on Communication Objects CAPL Functions)
  • References to connections between endpoints, e.g. providedServiceRef, consumedServiceRef, providedEventRef, consumedEventRef, txSignalRef (see also Objects on Communication Objects CAPL Functions)
