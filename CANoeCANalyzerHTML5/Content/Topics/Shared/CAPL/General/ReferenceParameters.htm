Reference Parameters

 

As in C++, reference parameters in CAPL functions are declared using &. The parameter is not marked specifically when called.

Example

void add(long& x, long y)
{
  x += y;
}
void f()
{
  long x = 3;
  add(x, 4);
  write("%d", x); // output: 7
}

All numeric data types and enumeration types can be declared as reference parameters. Data types such as message types and fields are automatically transferred as reference parameters.

The variable given in the function must have the exact same data type as the parameter in the function declaration. An implicit conversion, e.g., from byte to long, is not possible here.

It must also be possible to assign a value to the parameter.

Example

void f(long & x)
{
  x = 3;
}
void g()
{
  long a;
  int b;
  long c[2];
  f(a);          // allowed
  // f(b);       // not allowed
  f(c[0]);       // allowed
  // f(a + 4);   // not allowed
}

During function execution, a temporary variable is used for the parameter. As soon as the function has been completed, the value of this variable is copied back to the transferred parameter (pass-by-copy-restore).

Reference parameters can also be used in functions exported by CAPL DLLs. To do this, the parameter must be declared in the function as a pointer to the actual data type. Subtract 128 from the letter representing the data type of the parameter in the CAPL export table. The function will then be assigned the address of the transferred variable as the value of the parameter.

Example (C++)

void sum(long i, long j, long* s)

{

  *s = i + j;

}

CAPL_DLL_INFO3 table[] = {
  {CDLL_VERSION_NAME, (CAPL_FARCALL)CDLL_VERSION, "", "", CAPL_DLL_CDECL, 0xabcd, CDLL_EXPORT },
  {"sum", (CAPL_FARCALL) sum, "CAPL_DLL", "Sum via reference parameter", 'V', 3, {'L', 'L', 'L' - 128}, "", {"i", "j", "s"} },
  {0, 0}
};
CAPLEXPORT CAPL_DLL_INFO3 far * caplDllTable3 = table;
