Working with Symbolic Identifiers

 

In addition to symbolic names for messages, signals and environment variables, you can also use symbolic names for signal and integer environment variable values. To do this, you would assign symbolic names to the appropriate signal or integer environment variables in the database editor.

Note

Please note the C naming conventions at working with symbolic identifiers.

In CAPL, you use these names instead of the values by prefacing the particular signal or environment variable name followed by two colons (::). Thus, let us assume that you have defined the values of the signal LightStatus of the message LightData as follows in the database LIGHT.DBC:

Value

Name

1

Red

2

Yellow

3

Green

Then in order to send a message LightData with the symbolic value Red (= 1) of the signal LightStatus on the bus you would write the following in CAPL:

message LightData msg;
msg.LightStatus = LightData.LightStatus::Red;
output(msg);

To access the symbolic value description use this:

on message LightData
{
 if(this.LightStatus == LightData.LightStatus::Red)
 write("LightData.LightStatus = Red");
}

Analogously, you can also work with symbolic values for environment variables of the type Integer, if you have defined them in the database.
