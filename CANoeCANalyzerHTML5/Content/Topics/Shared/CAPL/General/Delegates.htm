Delegates

 

A delegate describes a reference to a function.

Instead of passing the name of a CAPL function explicitly defined elsewhere in the program as a parameter to a predefined function (e.g. consumedMethodRef::CallAsync), you can also define a small function directly at the position of the parameter. This function has no name and is called a delegate (in other programming languages you will also find the terms lambda expression or anonymous function).

Definition

The definition of a delegate starts with the keyword delegate. Then the parameters and the function content follow in curly brackets as in a normal function declaration.

Example

MirrorAdjustment.consumerSide[CANoe,LeftMirror].Adjust.CallAsync(-50, 0,
    delegate (callContext MirrorAdjustment.Adjust cco1)
    {
    write("Call returned with result %d", cco1.Result);
    }
);

A delegate behaves like a CAPL function defined at the top level. The different is that the code can be written exactly where it is needed. On the other hand, the delegate cannot be called explicitly from an other places in the code. You can also call other CAPL functions in delegates.

In the delegate, you have access to global variables and the parameters of the delegate. Access to parameters and local variables of the function in which the delegate is defined (so called captures) is also possible from CANoe version 12.0 SP3, whereby the following rules apply:

  • The captures must not be declared, the access on the captures is made automatically.
  • Depending on the data type, the variables or parameters are passed to the delegate as if they were additional parameters of the delegate. This means that values of primitive types such as dword or float are copied, but a reference is used for complex types such as message or struct.
    This is the reason that changes on the values within the delegate are not visible to the outside for primitive types, but they are visible for complex types.
  • To ensure that the values are still present when the delegate is executed, it is not allowed
  • to access parameters whose data type is complex.
  • to access locale variables that are created with the stack keyword on the stack and have a complex data type.

Example

void MyFunction(int x, struct Point p)
{
  double d = 3;
  stack long l = 4;
  struct Point p2;
  stack struct Point p3;
  memcpy(p2, p);
  memcpy(p3, p);
  MirrorAdjustment.consumerSide[CANoe,LeftMirror].Adjust.CallAsync(-50, 0,

Note

  • Currently delegates can be passed to the following predefined functions:
  • consumedMethodRef::CallAsync
  • consumedMethodRef::CallAsync_Phys
  • Using delegates is not compatible with older versions of CANoe. Therefore, they may only be used in CAPL programs with CANoe from version 12.0 on.
