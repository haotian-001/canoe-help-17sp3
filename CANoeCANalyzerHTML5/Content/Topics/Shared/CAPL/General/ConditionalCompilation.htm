Conditional Compilation

 

Some lines within a function, an include section or a variable section can only be compiled if certain conditions are fulfilled. For this purpose, write #if <Condition> before the lines concerned and then #endif after them.

Example

on message *
{
#if MEASUREMENT_SETUP
  output(this);
#endif
}

A host of predefined conditions are available and can be linked together into an overall condition as desired using brackets and logic operators such as || and &&. You can also use numbers and relational operators such < and == in the condition.

Predefined Conditions

  • MEASUREMENT_SETUP

    true (1), if the CAPL program is part of the Measurement Setup of CANoe

  • SIMULATION

    true (1), if the CAPL program is not part of the Measurement Setup of CANoe

  • ANALYSIS

    true (1), if the CAPL program is part of the Measurement Setup of CANoe (in CANoe, ANALYSIS is the same as MEASUREMENT_SETUP, whereas in CANalyzer the values might be different)

  • TEST_NODE

    true, if the CAPL program is a CAPL test node or a test library

  • CANOE

    true, if the CAPL program is compiled for CANoe

  • CANOE_LT

    true, if the CAPL program is compiled for CANoe LT

  • CAPL_ON_BOARD

    true, if the CAPL program is compiled for the execution on a network interface

  • EXTENDED_REAL_TIME

    true, if the CAPL program is compiled for the execution in the Extended Real Time part

  • TOOL_MAJOR_VERSION

    main version of CANoe, e.g. 7 for CANoe 7.5 SP3

  • TOOL_MINOR_VERSION

    subversion of CANoe, e.g. 5 for CANoe 7.5 SP3

  • TOOL_SERVICE_PACK

    service pack of CANoe, e.g. 3 for CANoe 7.5 SP3

  • DEBUG

    true, if CAPL debugging is activated

  • X64

    true, if the CAPL program is located in the Simulation Setup or Test Setup and the 64-bit variant of the RT kernel (Linux or Windows64) is used (in this case e.g. the #pragma library instruction command has to reference a 64bit DLL).

  • LINUX

    true, if the CAPL program is part of the Simulation Setup or Test Setup of CANoe and the Linux variant of the RT kernel is used.

Example

#if (TOOL_MAJOR_VERSION == 7 && TOOL_MINOR_VERSION == 5 && TOOL_SERVICE_PACK < 2) || CANALYZER
#pragma message("This program needs at least CANoe 7.5 SP 3")
#endif

You can add additional sections with other conditions by using #elif <Condition> or #else. You can also nest conditions.

Example

#if TOOL_MAJOR_VERSION == 7
  #if TOOL_MINOR_VERSION == 5
    write("CANoe 7.5");
  #else
    write("Unknown CANoe 7.x");
  #endif
#elif TOOL_MAJOR_VERSION == 8
  write("Unknown CANoe 8.x");
#else
  write("Unknown CANoe version");
#endif

Beginning with CANoe version 12.0 SP3 you can define your own constants for conditions by using #define <Condition>.

Example

variables
{
  #define DEBUG_OUTPUTS
}

on message EngineState
{
#if DEBUG_OUTPUTS
  if (this.State == 3)
  {
    write(“State 3 reached”);
  }
#endif
}

Please note:

  • In contrast to C/C++, #define is no macro definition. It is not possible to assign a value to the constant or to use the constant outside #if/#elif.
  • A #define in a variables section is valid from its position on for the rest of the file. If it is located in an included file, it is valid for the included file as of the #include instruction.
  • A #define in an includes section is taken into account only in a main file; it will be ignored in an included file. It is also valid from its position on for the rest of the file as well as for included files after the #define.
  • A #define in other functions or handlers are only applicable for the function itself.
  • There is no #undef or a similar type to remove a constant.
  • You also cannot exclude function declarations from the compile. #if can only be used within functions.

Note

  • Conditional compiling is supported starting with CANoe 7.5.

  • CAPL programs which are to be translated with older program versions cannot use conditional compiling.
