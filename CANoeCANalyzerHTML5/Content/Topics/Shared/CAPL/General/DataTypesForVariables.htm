Data Types for Variables

 

A CAPL program consists of two parts:

  • Declaration of global variables
  • Declaration of user-defined functions and event procedures

The following data types for variables are available. They can already be initialized in the declaration.

Integers and floating point numbers are used as in other programming languages. They are compatible with one another, and arithmetic is performed with 32 bit resolution for integers and 80 bit resolution for floating point numbers.

Scalar Data Types
  • Integers in form of
    byte (unsigned, 1 Byte)
    word (unsigned, 2 Byte)
    dword (unsigned, 4 Byte)
    int (signed, 2 Byte)
    long (signed, 4 Byte)
    int64 (signed, 8 Byte)
    qword (unsigned, 8 Byte)
  • Individual character
    char (1 Byte)
  • Floating point numbers
    float (8 Byte)
    double (8 Byte)

  • CAN/LIN messages
    CAN messages to be transmitted on the bus by the CAPL program are declared with message.
  • message

  • Database definition

    • Definitions are no concrete objects with values, but simply descriptions. To pass the variable to a function, it must first refer to a database object. This can be done by an assignment (e.g., a test unit parameter) or a lookup function call.

    Example:

    void function1(dbMsg * msgDef)
    {
      ...
    }

    void function2()
    {
      dbMsg * msgDefinition; // variable of database definition type
      msgDefinition = ABSdata; // assign message definition to variable
      function1(msgDefinition); // commit variable to function1

      // lookup function call
      msgDefinition = lookupMessage("EngineData"); // pass message definition to variable
      function1(msgDefinition); // commit variable to function1
    }

  • Diagnostics messages
  • diagRequest

  • diagResponse

    • Note

    When editing CAPL files containing variables of type diagRequest or diagResponse, you have to note the following regarding the service qualifiers:

    • if CAPL keywords occur, you have to put the service qualifiers always in quotation marks:
      E.G. "START_SESSION::DEFAULT_SESSION::Start"
    • up to CANoe < 7.0 you have to put service qualifiers longer than 50 characters generally in quotation marks.
    • since CANoe 7.0 you have to put service qualifiers longer than 128 characters generally in quotation marks.

  • Timer
    Variables of the type timer or msTimer are used to create time events.
  • timer (s)

Data Types for the Communication Concept
  • Structures and arrays that are defined in the communication description
  • Enumeration types that are defined in the communication description
  • callContext
  • References to endpoints, e.g. serviceProviderRef, serviceConsumerRef, eventProviderRef, eventConsumerRef (see also Objects on Communication Objects CAPL Functions)
  • References to connections between endpoints, e.g. providedServiceRef, consumedServiceRef, providedEventRef, consumedEventRef, txSignalRef (see also Objects on Communication Objects CAPL Functions)
