CAPL Warnings

 

Code

Message

Description

1177

Handler for all messages not allowed in Simulation Setup, will be ignored

An on * handler is not allowed in the Simulation Setup.
It will be ignored.

2001

implicitly converting float index value to integer

An array index of the float type is implicitly converted into an integer.
Decimal places are cut off.

2002

Conversion from <x> to <y>, possible loss of data

It is implicitly converted from one data type into another, e.g. from long to int. A data loss can occur in the process.

2003

Specified channel not available!

The specified channel is not available.

2004

Message id out of range

The message ID lies outside the valid range.

2005

Condition is always true/false

The condition is always true or false.
This could indicate an error in the code.

2007

Code has no effect

The code does nothing.
This may be an error, e.g. '==' was used instead of '=' .

2008

Array is only partially initialized

An initialization was only specified for part of the array elements.

2010

Timer <x> has no corresponding 'on timer'

There in no on timer procedure for the timer.
This means there is no reaction to the timer procedure anywhere.

2011

Timer variable defined in function or handler

No timer variable was defined in the 'variables' section.

2012

Name of CANdb object will be truncated. Maximum length of the names of objects from databases is <x>

The name of a CANdb object is too long and will be cut off.

2013

Changes on 'this' have only local effect

If the object referred to with this has been changed in a handler, then the changes will not be passed on during the output(this). It only applies locally for the handler.

2014

Message <x> in database <y> has different bus system!

The message has a different bus system than the database.
Check the data type.

2015

Initializer <x> out of range

The variable is initialized with a value which is too large or too small. Check the data type.

2016

Initial value of signal <x> out of range

The initial value for the signal is too large/too small.
Check the value and the size of the signal.

2017

Constant is out of range

The constant has a value which is too large or too small.
Check the data type.

2018

Init string too long â is cut

The string for the initialization is too long.
Check the size of the char array.

2020

Node/Message/Envvar x is ambiguous in databases y and z. Assuming node 'y::x'

There are two possible objects of the same name in different databases. You qualify the object with the database name.

2023

Handler used for multiple message IDs, signal is possibly only valid in message <x>

The handler is called for several different messages; the signal may only be valid in one of them.

2024

Variable name already defined as database name

The variable name is already in use as a database name and covers it.

2026

Duplicate .phys, ignoring the second .phys

An expression of the type ".phys.phys" was found. The second ".phys" is superfluous.

2027

Condition/Expression has array type. Assuming first array element as condition/resulting expression

An array was specified instead of a scalar variable.
For compatibility reasons, the first array element is used; the expression is actually an error.

2029

Use of <NETWORK NAME>/<CHANNEL>/<BUS TYPE>, but node doesn't belong to a network

<NETWORK NAME>/<CHANNEL>/<BUS TYPE> was used, although the node does not belong to any network.

2030

Use of <NETWORK NAME>/<CHANNEL>/<BUS TYPE> in a gateway node

<NETWORK NAME>/<CHANNEL>/<BUS TYPE> was used in a gateway node.
The result is always the first network/the first bus system.

2032

Function is deprecated. See help for more information.

The function call was replaced by another function or method and should no longer be used.

2033

Use of function 'MainTest' seems to indicate that this node should be a test module.

There is a 'MainTest' function, however the node is not a test module.

2034

Enum "x" does not have continuous values, operation may result in an inconsistent state.

The value of the Enums are not consecutive, but instead there are gaps. Therefore, be careful when converting to/from integers.

2035

';' : empty controlled statement found; is this the intent?

An additional semi-colon was found. Code may be missing here.

2036

Optional argument has complex type

An object was transferred to a function with "..." parameters which has no simple data type. This normally an error; e.g. write or snprintf cannot handle objects of this kind correctly.

2037

Some signals don't fit in message <message> because of DLC==x; initial values from database therefore ignored.

Some signals lie outside the DLC of the message. The initial values from the database will be ignored. Check the DLC and the signal layout.

2038

Function can't handle different string encodings. If you use non-ASCII characters, save the source file in the local encoding.

The encoding of the computer is different than the encoding of the CAPL program.

As a result special characters that are output in the sprintf function or with writeEx may not be displayed correctly.

ASCII characters can always be displayed error-free.

The write function outputs the special characters correctly even in the case of different encodings.

If you store the CAPL programs with the local encoding, characters that are available on the computer can also be displayed correctly.

2040

Recursive #include, ignoring this include

An #include file includes itself, possibly via the detour of one or several other #include files

2041

Duplicate #include, ignoring this include

An #include file was integrated several times. It is only processed the first time.

2042

Output(this) in [*] is not recommended

Should not be used in a [*] handler, output(this), as this can output the event double if another handler is also present.

2043

PGN should be a multiple of 0x100

A PGN should be a multiple of 0x100.

2050

Constant out of range. Missing suffix "LL" for 64 bit constants?

A constant is too large/too small. Check the data type. For 64-bit constants, your must add "LL" onto the value.

2052

Database "x" assigned on multiple channels, using channel 1

The database is assigned to several channels. The first one is used. Qualify with the channel if necessary.

2053

Possibly overloaded Test Check callback functions 'X(dword)' and 'X(TestCheck)'. Callback function should have different names

Two TestCheck functions only differ in the parameter. It is better to use different names.

2054

Local variable hides global variable 'x'

The local variable coves a global variable. This can result in errors if you want to access the global variable, however use the local variable instead.

2055

Assignment in condition

An assignment was used in a condition. This is frequently an error, as '=' was used instead of '==' .

2056

Output(this) in CANoe Simulation/Test Setup is not recommended

Directly outputting a received message in the Simulation Setup again is dangerous and often also false.

2057

Not all control paths return a value

The function has a return value, however not all code paths lead to a return instruction. A return may be missing in a branch.

2058

Struct is only partially initialized

The structure is only partially initialized. Initial values are missing from some members.

2059

This program should be compiled with version <x> of <y>

A CAPL-DLL has a different version than was specified in the #pragma library-instruction.

2061

Accessing integer signal with more than 52 bits, possible loss of data

In $ instructions only integer signals with up to 52 bits can be used without data loss, as values are saved and disclosed internally as doubles.

2062

Multiple transmitter frame needs qualification with node

The frame must be qualified with a node.

2063

Multiple transmitter PDU needs qualification with node

The PDU must be qualified with a node.

2064

Symbol is ambiguous

The database symbol is ambiguous. Qualify it if necessary.

2065

Variable will only be initialized once

Variables in handles/functions are only initialized during the first call of the function and then retain their value over the entire function calls Use an assignment outside the variable declaration if necessary.

2135

invalid message data offset, dlc =

A data byte is accessed in a frame, which lies behind the length of the frame specified by the DLC.

2201

Signed/unsigned mismatch

Expressions with and without signs are compared. If a value is negative, it may be an error.

2202

Unary minus operator applied to unsigned type, result still unsigned

A minus operator was applied to an unsigned expression. The result continues to have no sign and is not negative.

2220

slot id out of range

The slot ID is too large/too small.

2221

cycle out of range

The cycle is too large/too small.

2222

base cycle out of range

The base cycle is too large/too small.

2223

cycle repetition out of range

The cycle repetition is too large/too small.

2224

System Variable <Name> is only for analysis purposes and should not be accessed in simulation / test programs

The specified system variable is marked as for analysis only. It should not be used in simulation and test nodes. Its current value can not be read out there. Therefore, use the variable only in CAPL programs in the Measurement Setup. If you have defined the variable yourself, you can change the relevant property of the variable in the System Variablen Configuration dialog or via the COM interface.
