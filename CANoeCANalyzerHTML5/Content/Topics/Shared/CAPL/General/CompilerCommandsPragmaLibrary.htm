Command #Pragma Library

 

You can use the #pragma library command to link a C/C++ DLL in CAPL. The command can only be used in the context of an includes section.

Syntax

#pragma library("file path")

#pragma library("file path", version)

#pragma library("file path", version, "message text")

  • The file path references either a Windows DLL or a module description file.
  • The file path can be specified either relative to the source file or absolute.
  • The version must be an integer.

When using relative paths the library will be searched relatively to the following folders:

  • The folder of the file that contains the #pragma library command
  • <User data folder>\Reusable\capl_includes
  • <Installation directory>\Exec32\capl_includes (32 bit installation)
  • <Installation directory>\Exec64\capl_includes (64 bit installation)

Semantics

Functions exported by the DLL can be used anywhere in the CAPL program, even in include files linked prior to the command.

If a version is set, the CAPL compiler checks whether the DLL exports this version number. If no version or another version number is exported a warning will be output: either the message text if defined or a message text exported by the DLL or a predefined text that indicates the version conflict.

The CAPL program generated must be compiled once in CANoe so that the DLL will appear in the list of used files.

Note

The #pragma library command is not compatible with older versions of CANoe.
Therefore, it can only be used in CAPL programs with CANoe version 6.1 or higher.
