Socket based Communication

 

Sockets can be used to build an end-to-end communication. An end is referred to as a Server Socket. It waits for incoming connections. The other end is referred to as the Client Socket. It initiates communication and builds a connection with the Server Socket.

The type of connection depends on the protocol used. Generally used for sockets are:

  • TCP (connection-oriented) and
  • UDP (connectionless)

The endpoints are defined by the IP address and port number.

CANoe supports socket-based communication in simulated nodes. The TCP/IP API is used for this.

Note

The use of sockets may differ from other socket implementations.
