Generate CAPL Code (Car2x Network Explorer)

 

Method

In the tree view, select those items for which you wish to generate CAPL code. To select multiple items, hold the <Ctrl> key and select the desired items with the mouse.

You can generate the CAPL code in the following ways:

  • Use drag and drop to move the items to the code editor of the CAPL Browser.
  • Press <Ctrl>+<C> to copy the selected items to the clipboard. In the code editor of the CAPL Browser, press <Ctrl>+<V>.
  • Open the context menu with the right mouse button and choose Copy to clipboard to copy the selected items to the clipboard. In the code editor of the CAPL Browser, choose Insert in the context menu.

You may also copy the generated CAPL code to any other code editor which supports inserting text from the clipboard or by drag and drop.

Generated CAPL Code

Main Instance

The CAPL data structures that are set up hierarchically for the ASN.1 description contain a main element (e.g. in the CAM message the main element is known as CAM). An instance must be created as a global or local variable for this element. You assign the name to this instance in the Variable name for main instance field.

Example

The name of the main instance is set to myCam and the protocolVersion element is selected. In CAPL generation, the following code is generated:

// variable you have created
CAM myCam;

// generated code
myCam.header.protocolVersion = 1;

Optional Elements

The definition of the data structures may contain optional elements, i.e. they do not need to be filled in the CAPL code. Optional elements are shown with a gray symbol in the tree view.

If you wish to use the optional elements, you must set the isValid flag in the CAPL code to 1 for these elements. Then the element is considered in encoding and is available in the transmitted message.

Example

For the optional element ptScheduleDelay, when the isValid flag is set, the following CAPL code is generated:

// variable you have created
CAM myCam;

// generated code with initialization of the isValid flag
myCam.cam.camParameters.specialVehicleContainer.publicTransportContainer.ptScheduleDelay.isValid = 1;
myCam.cam.camParameters.specialVehicleContainer.publicTransportContainer.ptScheduleDelay = -300;

Maintaining pre-initialized Values

When the CAPL structures are created, the fields are initialized so that the value range is maintained for each field. If this were not the case, ASN.1 errors would occur during transmission. If you want to make changes in the data structures (for example, in the OnPreTx callback), it is recommended to maintain the pre-initialized data structure beforehand and work on it. In this way, you avoid errors when sending.

Example

void OnPreTxCAM(LONG packet)
{
  struct CAM myCAM;

  C2xCompletePacket(packet);
  API_GetCAMParams(packet, myCAM);

 // Fill your myCAM structure
  // myCAM.cam.camParameters.basicContainer...

  API_SetCAMParams(packet, myCAM);

}

Generate CAPL Include Files

With the Car2x Network Explorer the CAPL include files (.CIN) can be generated that are needed to work with application messages. You can open the generation dialog in the navigation area via the context menu of Messages.
