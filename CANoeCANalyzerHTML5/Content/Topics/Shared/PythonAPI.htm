Python Programming

 

General

In CANoe you can use Python scripts for programming application models.

For security reasons, no interpreter is shipped with CANoe. Instead, the users or their organization are responsible for ensuring that a Python interpreter of suitable version is installed. A download is available from the official page, among others. CANoe only provides special API extensions and thus an Embedded Domain Specific Language.

You can also use Python outside CANoe to control CANoe via the integrated COM server. However, this help page refers to application models. You can use the vector.canoe module only for application models and not for controlling the COM server.

Note on Python Version

As of CANoe version 15 Python version 3.9.2 is supported.

Note on Compatibility When Using Native Libraries

When using native Python libraries (so-called C extensions) in the Python scripts, the architecture and platform must match the used or set RT server.

Note on Supported Modes

Distributed mode, standalone mode and Linux are supported.

A Python interpreter with version 3.9.x has to be installed manually on the machine that will be running the RT Server. Please note, that this is currently not possible on VN8900 devices. For Linux the Python interpreter must be installed relative to the /usr folder. In these cases CANoe synchronizes all Python files and Python packages that are located next to the configured Python models to the target system. Python packages are only considered if they contain a __init__.py in each subfolder, system site packages are not synchronized.

Note on security vulnerabilities

Python installations can contain known and unknown vulnerabilities that manifest themselves in certain application models.

Using Python

Using a Python Editor

You can edit Python scripts comfortably by using e.g. Visual Studio Code or PyCharm as development environment. To use Visual Studio Code for developing Python scripts in CANoe, open the External Programs page in the Options dialog and select Edit_Python_Source_With_VisualStudioCode.vbs from the Python file editor. You will find this file in the Exec64\Scripts subdirectory of your CANoe installation directory. Similarly, select Edit_Python_Source_With_PyCharm.vbs for PyCharm.

Note

If you specify an editor directly, the editor does not get the paths to the interpreter used by CANoe and to the API provided by CANoe, and the auto-completion may be faulty.

Event Procedures

To react to an event, you can provide methods with a special decorator. The methods are then called when the event occurs.

The following decorator is available in CANoe:

  • vector.canoe.on_key(char: str)

Example

Copy
The method is called when key 'a' is pressed:
@vector.canoe.on_key(‘a‘)
def key_a_pressed(self, char: str):
    vector.canoe.write(“key a pressed”)

In addition, timers based on simulation time can be created using the vector.canoe.Timer. These can be used like threading.Timer.

Programming Application Models

To program application models, specify a Python script. The script must contain exactly one class that has the vector.canoe.measurement_script decorator. The class must have a default constructor.

At runtime this class is instantiated by CANoe and its event procedures are registered automatically.

Test modules are not supported.

Access to Distributed Objects

For distributed objects, Python provides a generated API that you can use to access the distributed objects in the CANoe runtime environment. The API is generated below the Python package application_layer created for this purpose.

For this purpose, the data types in Python are mapped as follows:

  • Integer data types to Python data type int.
  • Float and double to float
  • String to str
  • Bool to bool
  • Bytes to bytearray
  • Enum are mapped via derivatives of the IntEnum data type.
  • Arrays and lists are mapped via classes that behave like native Python lists, but additionally check the minimum and maximum size. However, native lists may also be passed when setting values.
  • Structs are mapped via classes with one property per struct member.
  • Optional struct members can have None as value.
  • Unions are mapped via classes with one property per union member. At any time, only one member has a valid value. When setting a property, this changes automatically. The discriminator property can be used to query which member has a valid value.
  • The names of the Python data types for Enum, Array, List, Struct and Union data types correspond to the definition in CANoe. According to the namespaces, these are generated in the Python packages with the same names.
  • Dependent on the defined conversions the corresponding representations may be accessed by the properties impl_value, phys_value, raw_value and symb_value.

The distributed objects are mapped in Python as follows:

  • For distributed objects defined before the measurement, Python objects with the same name are created in the Python package matching the namespace.
  • The creation of dynamic distributed objects is currently not supported.
  • For data members, events, and methods, properties of the same name are created on the Python object.

The values can be read and written as described below:

  • Currently, the values of data members, events and fields can be read and written.
  • When reading a value without defined conversions, a copy of the value must be created using the copy method.
  • When writing a value without defined conversions, complex values must always be set as a whole.
  • When reading or writing a value with defined conversions, one of the properties impl_value, phys_value, raw_value and symb_value must be used.
  • Handlers can be registered directly at the member using the method register_on_change_handler or register_on_update_handler.

Example

Copy 
from application_layer import Mirrors
# Read the value of provided data member “CurrentPosition” of the object “RightMirror” in namespace “Mirrors”
pos = Mirrors.RightMirror.CurrentPosition.copy()

# change value and write it back
pos.x = 10
Mirrors.RightMirror.CurrentPosition = pos

# define a and register an on-change handler for consumed data member “CurrentPositionLeft” of the object “Controller” in Namespace “Mirrors”
def OnControllerCurrentPositionLeft():
  pos = Mirrors.Controller.CurrentPositionLeft.copy()
  vector.canoe.write(str(pos.x))

Mirrors.Controller.CurrentPositionLeft.register_on_change_handler(OnControllerCurrentPositionLeft)

Methods can be called and implemented as follows:

  • For provided methods, you can use the set_call_handler method to register a callback that contains the implementation of the method. The signature of the callback is essentially the same as the defined method. However, for primitive data types, implemented in Python as so-called immutables, and for optional parameters, instances of a generic data type are passed in the case of Out and InOut parameters. This data type allows both reading and changing the parameter via a value property, which has the original data type as its data type.
  • For consumed methods, you can use the call_async method to make an asynchronous call. The method call_async expects all In and all InOut parameters. As the last parameter, you must specify a callback that will be invoked when the response is received. If present, the callback receives the return value as the first parameter. Subsequently it receives all Out and InOut parameters.
  • For internal methods, you can use the call_sync method to make a synchronous call. The method call_sync expects all In, InOut and Out parameters.

Example

Copy 
from application_layer import Mirrors

@vector.canoe.measurement_script
class MirrorControlModel:
def initialize(self):
  # register implementation for provided method “Adjust” of object “LeftMirror” in Namespace “Mirrors”
  Mirrors.LeftMirror.Adjust.set_call_handler(self.Adjust_implementation)

def Adjust_implementation(self, deltaX:int, deltaY: int) -> int:
  return 0

def start(self):
  # call consumed method “Adjust” of object “MirrorAdjustment” in Namespace “Mirrors”
  Mirrors.MirrorAdjustment.Adjust.call_async(-50, 0, lambda result: vector.canoe.write(“result: “ + str(result)))
