J1939/ISO11783 Interaction Layer Network Management

 

Configure J1939 Device Name

With the Address Claimed message the device name (NAME) is sent from the address of the ECU. For a node that uses a J1939 or ISO11783 Interaction Layer (IL), the device name is defined with the node attributes with the NmJ1939 prefix in the database.

Activate Address Claim Support

The J1939 and ISO11783 Interaction Layers distinguish between two types of address claim support: basic address claim support and advanced address claim support. If no address claim support is activated, the interaction layer does not send an Address Claimed message by itself.

The default address claim support can be changed in CAPL or via database configuration:

  • To activate the basic address claim support when using the J1939 IL, assign the Address Claimed message (PGN EE00h) to the node as a transmit message in the database. You can do this by dragging the Address Claimed (AC) message in the CANdb++ Editor onto the node.
  • To change the address claim support in CAPL, call the associated <Prefix>SetNodeProperty function with the AddressClaimSupport parameter and one of the following values:
  • 0: no support
  • 1: basic support
  • 2: advanced support

Behavior During Address Claiming

The behavior of J1939 and ISO11783 ECUs with regard to network management differs only slightly. While J1939 devices start immediately with the Address Claimed message, ISO11783 devices first send a Request for Address Claimed message. After 250 ms, they then send the Address Claimed message with an address that is still free. The interaction layers behave according to the assigned protocol.

If an address conflict occurs during address claiming (i.e. there are two ECUs claiming the same address), the behavior of the interaction layer depends on the result of the address arbitration, on the selected address claim support and on the value of the NmJ1939AAC node attribute in the database. In general, the node whose J1939 device name has the higher priority wins the address arbitration.

With advanced address claim support, the CAPL callback <Prefix>OnAddressClaimConflict is called in case of an address conflict. In this calback the address claim behavior and the new address can be influenced and the address claimed message can be suppressed.

The change of the IL to the Stopped state can be monitored in the CAPL callback <Prefix>OnChangedState. In this case the callback is called with the value 3. If the node is to be started again with another address, you can achieve this with the function <Prefix>ControlStart.

The state transitions of the J1939/ISO11783 Interaction Layer during address claiming are described in more detail on the following page: States (J1939 IL).
