J1939 GB/T 27930 Interaction Layer

 

The Specification GB/T 27930 (2015) describes the communication between the charging station (Charger) and the battery management system (BMS) in the electric car via CAN.

The option .J1939 in combination with the option .SmartCharging provides a GB/T 27930 interaction layer (GBT27930 IL), with which you can simulate both participating of a GB/T 27930 network. These are:

  • Charging Station (Charger)
  • Battery Management System in e-vehicles (BMS)

To do this, assign the module description file GBT27930_IL.vmodule to a node in the Simulation Setup. After a configuration via CAPL commands the GBT27930 IL is ready and takes over the communication with the other GB/T-27930 participant.

Functions of the GB/T 27930 Interaction Layer (GBT27930 IL)

  • Configuration of the internal state machine.
  • Control of charging communication.
  • Control of the simulated node from a test node.

Setup GBT27930 IL [▲ back]

A node simulated with GBT27930 IL should be configured partly via the database and partly via CAPL. Via the database you must first activate the use of the GBT27930 IL and define the messages to be sent by the simulated node. The following steps are necessary:

Note: The database in the GB/T 27930 Simulation sample configuration contains already the Charger and the BMS node.

  1. Define a node in the Database (see Define a J1939 Node).
  2. Enter the value GBT27930_IL.vmodule in the node attribute NodeLayerModules.
  3. For the node, define the list of messages that the simulated node is to send by moving the corresponding messages into the Tx list.
  4. Define the address to be used by the simulated node.

In CAPL, you must configure additional parameters for the simulated node. First, you must define whether the simulated node is to be used as Charger (GBT27930IL_InitAsCharger) or should interact as BMS (GBT27930IL_InitAsBMS). You can also define how fast the simulated node should switch between internal states. (GBT27930IL_SetDelayForStateTransition).

Control of the Simulated Node [▲ back]

After starting the measurement, you have the following options to control the behavior of the simulated node:

  • First you start the charging communication by calling the function GBT27930IL_StartChargingSimulation. The simulated node starts to send corresponding messages. To fill these messages with meaningful content, you have the following options:
  • You can define the content of the message permanently by assigning values to individual signals or by using the following functions: GBT27930IL_SetSignal, GBT27930IL_SetSignalRaw, GBT27930IL_SetMsgRawData.
  • You can customize individual messages directly before sending by implementing the callback function: GBT27930IL_OnTxPrepare.
  • You can define rules according to which certain bytes and bits in any message are modified. You can do this using function GBT27930IL_ManipulateMessage.
  • There are some messages that are relevant to the protocol itself. These messages are set directly by the GBT27930 IL. These include the messages CTS, CRO, CRM and BRO.
  • Switching between the internal states occurs immediately after the corresponding message from the other side is received for the first time. You can delay the switching by calling the function GBT27930IL_SetDelayForStateTransition with the corresponding parameter.
  • You can then end the simulation of the charging process. You have the following options:
  • Rule-compliant final charging is initiated with function GBT27930IL_SuspendCharging. The simulated node passes through the Suspension of charging and End of charging phases.
  • Immediate termination of any communication is achieved with function GBT27930IL_StopChargingCommunication. The simulated node no longer sends messages.

Fault Injection [▲ back]

You can influence the behavior of the GBT27930 IL from test units or test modules in order to provoke errors. In a remaining bus simulation, you can generate communication errors to which the SUT (System Under Test) must react. The description of the corresponding CAPL functions can be found in the CAPL function overview FS - Fault Injection.

You can also activate, skip or repeat some states. To do this, use the GBT27930IL_SetCommunicationState function. The following call is sufficient, e.g. to activate the sending of the message BEM or CEM (Error Message): GBT27930IL_SetSetCommunicationState(50).
