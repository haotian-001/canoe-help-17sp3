Format of the Module Description

 

The module description file is an XML document in UTF-8 format. A formal description (XML schema) is located in the <Path to the sample configurations>\Programming\vModule\Schema\CANoe_vmodule_1.5.xsd. Relative file paths within the XML document are relative to the module description files.

Many XML elements of the document allow an XML attribute called arch, which includes a list of supported platforms. This allows the effectiveness of the XML element to be limited to one or more platforms. The attribute is optional. If the attribute is not specified, the XML element is valid for all platforms supported by the module.
Note: The elements of an XML list are separated by spaces.

The identifiers for the platforms are:

Identifier

Platform

Linux-x64

Linux with 64-bit computer architecture.

Windows-x64

Windows with 64-bit computer architecture.

Windows-x86

Windows with 32-bit computer architecture.

CaplOnBoard-ARM32

CAPL-on-Board with 32-bit ARM architecture.

WEC-ERT

Windows Embedded Compact architecture for Extended Real Time.

Many XML elements of the document allow the use of the version attributes minAppVersion and maxAppVersion. This can be used to create a restriction in relation to an CANoe version. If versioning is used, it is continuous and consistent. With the minAppVersion a certain minimum version can be required. The maxAppVersion may only be used if there is also an element without maxAppVersion; this is the only way to ensure the compatibility of an CANoe configuration. The use of versioning is generally optional. If versioning is not used, all CANoe versions can handle it.

The XML file consists of the root element Module, which contains the following XML elements:

XML Element

Description

Name

The name of the module.

Version

The version number of the module.

Application

The name of the application for which this module is suitable.

The value must be CANoe or CANalyzer.

This element may be specified multiple times if the module is suitable for several applications.

ModuleType

Type and interface implemented by this module.

The value must be CAPL-DLL, NodeLayer-DLL, or DOTNET-DLL.

This element may be specified multiple times if the module implements multiple interfaces.

SupportedArchitecture

Platforms for which this module is suitable.

The XML attribute arch contains the list of supported architectures.

The XML attributes minAppVersion/maxAppVersion contain an optional restriction regarding the CANoe version.

LibraryPath

File path for implementing the module, typically a library for Windows (*.dll) or Linux (*.so).

The XML attribute arch specifies for which platform(s) this XML element is valid.

The XML attributes minAppVersion/maxAppVersion contain an optional restriction regarding the CANoe version.

AdditionalFile

File path to additional files, for the implementation of the module.

If a simulation is executed on a remote system (e.g. RT Rack, VN8900, CANoe4SW Server Edition), these files are automatically transferred to the remote system.

This element may be specified multiple times.

The XML attribute arch specifies for which platform(s) this XML element is valid.

The XML attributes minAppVersion/maxAppVersion contain an optional restriction regarding the CANoe version.

CAPL-DLL

Additional data for modules of type CAPL-DLL.

The XML attribute arch specifies for which platform(s) this XML element is valid.

The XML attributes minAppVersion/maxAppVersion contain an optional restriction regarding the CANoe version.

NodeLayer-DLL

Additional data for modules of type NodeLayer-DLL.

The XML attribute arch specifies for which platform(s) this XML element is valid.

The XML attributes minAppVersion/maxAppVersion contain an optional restriction regarding the CANoe version.

SysvarPath

File path to additional files that define system variables.

The XML attribute arch specifies for which platform(s) this XML element is valid.

The XML attributes minAppVersion/maxAppVersion contain an optional restriction regarding the CANoe version.

Example of a module description file (*.vmodule)

<?xml version="1.0" encoding="UTF-8"?>
<Module xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:noNamespaceSchemaLocation="CANoe_vmodule_1.5.xsd">
  <Name>CANoeILNLVector</Name>
  <Version>3.17.70</Version>
  <Application>CANoe</Application>
  <ModuleType>NodeLayer-DLL</ModuleType>
  <ModuleType>CAPL-DLL</ModuleType>
  <SupportedArchitecture arch="Windows-x64 Windows-x86 WEC-ERT"/>
  <SupportedArchitecture arch="Linux-x64" minAppVersion="13.0.0"/>
  <LibraryPath arch="Linux-x64" minAppVersion="13.0.0">../Modules_Linux-x64/libCANoeILNLVector.so</LibraryPath>
  <LibraryPath arch="Windows-x64">../Exec64/CANoeILNLVector.dll</LibraryPath>
  <LibraryPath arch="Windows-x86">../Exec32/CANoeILNLVector.dll</LibraryPath>
  <LibraryPath arch="WEC-ERT">../Exec32/ERT/CANoeILNLVector.dll</LibraryPath>
</Module>
