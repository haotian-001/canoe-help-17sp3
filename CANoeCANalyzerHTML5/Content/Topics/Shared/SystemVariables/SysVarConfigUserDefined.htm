System Variables – User-Defined

Ribbon: Environment Tab|System Variables|User-Defined

User-Defined

System-Defined

Name Display

 

On this page you can add, edit and delete system variables and struct definitions. For this, all currently defined system variables are displayed in a tree view at the left side of the dialog and the struct definitions in a tree view at the right side of the dialog. When selecting a struct definition in the right tree, the corresponding system variables which are using this struct definition are highlighted in the left tree view.

You can edit namespaces, system variables, struct definitions and struct members directly in the table. For system variables, struct definitions and struct members it is also possible to edit several elements simultaneously. Details see below under information about editing elements.

You can move namespaces, system variables and struct definitions with drag and drop.

You can open the tree view of the struct definitions either via the splitter () or via in the toolbar.

Editing Elements

Editing Properties Individually

Available properties for namespaces, system variables, struct definitions and struct member can be directly edited in the table.

Editing Properties for Several Elements Simultaneously

  • Several elements of the same type (e.g. struct definitions) can be edited simultaneously. For this purpose, select the elements in the table and open the Edit dialog via the toolbar or the context menu.
  • Only those values that are available for all selected elements can be edited simultaneously.
    For example, the Min., Max. and initial value is not available for every data type and can therefore only be edited if all elements have a data type which provides these properties.
  • If a cell is empty in the Edit dialog, it means either no value was assigned or the values of the elements to be edited are different.
  • If a cell is filled out in the Edit dialog, this value will be the same for all elements to be edited.
  • In the Edit dialog, check boxes show a filled-up square when the values of the elements to be edited are different.
  • If you modify a value in the Edit dialog, this value will be applied to all elements to be edited.

Key Combinations

  • With <F2> you open the editor (e.g. text editor, list selection) in the selected cell (if the value can be edited).
    • Example:
  • With <Enter> you close the currently opened editor and apply the entered value.
  • With <Esc> you close the currently opened editor without accepting the current entry. The old value will be restored.
  • Navigate to the next cell using <Tab>. When the last cell of a row is focused, use <Tab> to navigate to the first cell of the next row.
    If the editor is opened in a cell, you close the editor with <Tab>, accept your entry and navigate to the next cell. The editor for editing the cell will not be opened in the new cell.
  • Open the list selection with <Alt>+<Down Arrow>. With <Up Arrow> and <Down Arrow> you navigate to the possible values in these lists. With <Enter> you apply the selected value.
    • Example:
  • With <Left Arrow> and <Right Arrow> you navigate within the cell to be edited to the left or right.
  • With <Space> you modify the value of a check box.

Referenced Files

You can reference variables in the configuration, i.e., these tags are saved in files but not in the configuration itself.

To do so, add new files using the [...] button in the Referenced Files area.

If system variables and struct definitions are stored in the same referenced file, they will be added to the respective tree views when referencing.

The button at the right edge of the list (or the <Del> key) can be used to remove a file again.

You can use the check boxes at the left edge to activate or deactivate a file. This can be useful, for example, when the system variables contained therein are to be defined only occasionally. This could be the case if they serve as placeholders for I/O ports that are available only with specific hardware. As soon as a file is added and activated, the variables contained therein are displayed in the tree view and are available for use.

Note

If you have changed a system variable that is defined in a referenced file, it will be written to the file when the configuration dialog is closed with [OK].

Sorting:

The elements are sorted alphabetically before saving. The following structuring takes place:

  1. Namespaces
  2. Structure definitions
  3. System variables

The selection of the referenced files is saved in the configuration and is available again when the configuration is reloaded.

If a referenced file is no longer available, the file is marked with an error icon , and a corresponding message is displayed in the display area.
