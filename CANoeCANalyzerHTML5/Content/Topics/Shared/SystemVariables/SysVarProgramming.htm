Programming System Variables

 

You can access system variables by means of a CAPL API in case of CAPL programs or by means of a COM API in case of external programs.

Example

The example below is derived from Visual Basic. In this example a namespace (ns) is defined in the system, a new system variable (var) is added to that namespace and a start value is set for the system variable.

value = 10

Set Namespaces = app.System.Namespaces
Set Namespace = Namespaces.Add("ns")
Set Variables = Namespace.Variables
Set Variable = Variables.Add("var", value)

The value of the variable can be changed with the following command:

Variable.Value = 20

In CAPL the respective program code (without trouble shooting) is the following:

SysDefineNamespace( "ns");
SysDefineVariableInt( "ns", "var", 20);

or

@sysvar::ns::var = 20;

In order to access (read) a system variable the following commands are used in Visual Basic:

Set Namespace = app.System.Namespaces("ns")
Set Variable = Namespace.Variables("var")
edit.Text = Variable.Value

in CAPL it is:

value = @sysvarInt::ns::var;

(The syntax @ can only be used for variables of type integer or float. For other data types the respective functions sysSetVariableâ¦ and sysGetVariableâ¦ have to be used.)

Example

The example shows the use of system variable types as a return value of functions and as a local variable.

In the use case, the variableTest test is performed for 3 different system variables by transferring an ID for the respective system variable. With the help of the ID and the user-defined function selectVariable (that has the system variable type Int as a return value), the system variable is determined.

The user-defined function doVariableTest() uses the system variable type Int for the local variable usedVariable.

Note

The CAPL code contains the following system variables of type Int32. You must define these in the System Variables Configuration dialog first:

  • SomeNamespace::Variable1
  • SomeNamespace::Variable2
  • SomeNamespace::Variable3
  • TestControl::VariableSelector

variables
{
  int variableIds[3] = { 1, 2, 3 };
}
  // returns the variable which the test shall use,
  // depending on a selector key
  sysvarInt * selectVariable(int variableId)
  {
    switch (variableId)
    {
      case 1:
        return sysvar::SomeNamespace::Variable1;
      case 2:
        return sysvar::SomeNamespace::Variable2;
      case 3:
        return sysvar::SomeNamespace::Variable3;
      default:
        return sysvar::SomeNamespace::Variable3;
    }
  }
  void doVariableTest()
  {
    sysvarInt * usedVariable;
    usedVariable = selectVariable(@TestControl::VariableSelector);

    @usedVariable = 42;
    if (testWaitForSignalMatch(usedVariable, 42, 100) != 1)
      testStepFail("Wait", "Variable %s didn't reach the specified value in time", usedVariable.name);
    else
      testStepPass("Wait", "Variable %s has reached the specified value in time", usedVariable.name);
  }
  testcase variableTest(int variableId)
  {
    char desc[50];
    snprintf(desc, elcount(desc), "Testing Variable%d", variableId);
    testCaseDescription(desc);

    // use the system variable "VariableSelector" just for demonstration
    // purposes (the id could also be given directly to âdoVariableTestâ)
    @TestControl::VariableSelector = variableId;
    testWaitForTimeout(10);

    doVariableTest();
  }

  void MainTest ()
  {
    int i;
    for (i = 0; i < elcount(variableIds); ++i)
    variableTest(variableIds[i]);
  }
