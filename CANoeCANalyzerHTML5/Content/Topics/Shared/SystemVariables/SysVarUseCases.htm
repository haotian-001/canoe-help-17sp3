Use Cases for System Variables

 

One application of system variables is the calculation of auxiliary variables in the Measurement Setup, which then can be displayed in the Graphics or Data Window. This, for example, allows for a simple analysis of the difference between two signal values:

on message Msg1
{
  // System variable Analysis::SignalDifference is defined in the
// CANoe System Variables dialog
  @sysvarInt::Analysis::SignalDifference = this.signal1 – this.signal2;
}

In contrast to environment variables, system variable changes which were made in a program node in the Measurement Setup are immediately transmitted and their time stamp is the current simulation time (in case of on message procedures this time stamp corresponds to the time stamp of the message). In COM this behavior can also be set up by changing the AnalysisOnly property.

System variables which were used for analysis can still be used by the real-time area of CANoe. However, a warning is displayed in the Write Window which says that the value changes in the analysis area won't be transmitted to the real-time area anymore.

In a CAPL program in the Measurement Setup it is possible to change a system variable and yet transfer the changed value to the simulation part by means of the sysSetAnalysisOnlyVariable CAPL function.

Note

In the Measurement Setup changes on system variables are only effective for the branch in which the change was done.

Application Fields

Typical application fields for system variables are for example:

  • Parameterization of node models before measurement (including large data sources such as Excel tables)
  • Parameterization of node models during measurement
  • Analysis/debugging of node models
  • Parameterization of test modules
  • Measurement and analysis of I/O signals
  • Calculation of analysis values, e.g. the difference between two signal values
  • For generic test code that is to function with different system variables, system variable types can be used as a local variable and as a return value of functions. The specific system variable is determined using a key/index that is a parameter of the test.
