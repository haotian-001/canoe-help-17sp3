Add/Edit Struct Definitions

Ribbon: Environment Tab|System Variables|User-Defined|Add/Edit Struct Definitions

The New Struct Definition/Edit Struct Definition dialog is the same for both application purposes. If struct definitions are being edited, previously entered values are automatically displayed in the relevant fields.

Entries can always be made in the following fields:

Field Name

Brief Description

Namespace

If you specify a namespace that doesn't exist yet, it is generated automatically.

Namespaces can also be nested. In programs they are separated by "::".

You can also change the namespace of a variable and thus move it to another namespace.

Name

The name must be unique within the namespace and must not contain "::".

Comment

You can enter a comment for each struct definition.

The comment is visible e.g. in the Symbol Explorer.

Storage

You can save a struct definition either in the configuration or in a separate file. To do so, select one of the already referenced files from the list or use the button with three dots on the right in order to add another file.

Saving the struct definition in the configuration has the advantage that the configuration can be forwarded more easily.

Element Length

When using fixed length, you have to set the length of string and data members explicitly. The length will be the maximum length of the member.

If you are writing data that is longer than the maximum length of the member, some data will be lost.

If you are using a struct of variable length, there is no restriction regarding the length of string and data members. You do not have to set the length explicitly. It will be calculated automatically.

Struct Members

In this area you can add, edit, delete or reposition the members of the struct definition. To do so, use the buttons on the right, rename members directly in the name field or select a data type by opening the list in the Data Type column.

Properties

After you have selected a struct member in Struct Members area, the associated properties and their values are displayed here. Entries can be made depending on the data type of the member in the following fields:

Field Name

Brief Description

Array Size

The array size, i.e., the number of values, must be specified for the integer array and float array data types. The array size cannot be changed dynamically, i.e., the variable always contains this number of values.

Initial Value

If you specify a value here, the member assumes this value after its creation. The member is also set to this initial value at the start of the measurement, if you have selected the corresponding option in the global settings. For arrays, you must specify as many values as the array size and separate the individual values with semicolons. For data-type variables, you specify the respective values of the bytes as two-digit hexadecimal numbers that you can separate with blank spaces (e.g., 01 2A FF for three bytes).

Minimum and Maximum

Minimum and Maximum are used optionally to define a valid value range for the struct member. CANoe does not check this range itself when a value is set, but it uses the range, e.g., for configuring display elements. For arrays, the limits for each individual array element apply. For strings and data-type variables, no limits are possible. For integer variables, the limits can also be specified by the value table.

Linear Encodings

Use linear encodings to assign a raw value to a physical value.

With [...] you open the Linear Encodings dialog.

Value Table

For integer and integer array-type struct members, you can define a value table. You can select an existing template from the list for this. The individual values of this table are then listed below this line. If you select the Edit command from this list, you can adapt an existing template or create a new template. Each member contains its own copy of the value table; changes to the table therefore always apply only to the respective Member.

Minimum/Maximum from the Value Table

This check box is only available if you have selected a value table. If this option is set, the Minimum and Maximum fields are set with values from the value table and can no longer be edited manually.

Length

For string type struct members, a length has to be specified. Values for such a string cannot be longer than this length. Also note that strings are null terminated, i.e. you can only set Length-1 characters.
