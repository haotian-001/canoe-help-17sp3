Add/Edit System Variables

Ribbon: Environment Tab|System Variables|User-Defined|Add/Edit System Variables

The New System Variable/Edit System Variable dialog is the same for both application purposes. If system variables are being edited, previously entered values are automatically displayed in the relevant fields.

Entries can always be made in the following fields:

Field Name

Brief Description

Namespace

If you specify a namespace that doesn't exist yet, it is generated automatically.

Namespaces can also be nested. In programs they are separated by "::".

You can also change the namespace of a variable and thus move it to another namespace.

Name

The name must be unique within the namespace and must not contain "::".

Comment

You can enter a comment for each variable.

The comment is visible e.g. in the Symbol Explorer.

Data type

Each variable has one fixed data type and can only accept values of this type.

The following data types are available:

Field Name

Brief Description

Integer (32 bit signed)

Signed integers, available for 32 bit.

Integer (32 bit unsigned)

Unsigned integers, available for 32 bit.

Integer (64 bit signed)

Signed integers, available for 64 bit.

Integer (64 bit unsigned)

Unsigned integers, available for 64 bit.

Double (64 bit)

Floating point numbers, which are stored internally as 64 bit IEEE double precision floating point numbers.

String

Character strings of unlimited length.

Integer array (32 bit signed)

Fixed number of signed integers, available for 32 bit.

Double array

Fixed number of float values.

Data

A number of bytes that are transferred as a block.

The specific number is variable and can differ with each change of the variable value; this is not an array of fixed size.

Struct Definition

An existing struct definition which is also displayed in the tree view of the struct definitions.

Storage

You can save a system variable either in the configuration or in a separate file. To do so, select one of the already referenced files from the list or use the button with three dots on the right in order to add another file.

Saving the variable in the configuration has the advantage that the configuration can be forwarded more easily.

Properties

Entries can made depending on the data type in the following fields:

Field Name

Brief Description

Only Used For Analysis

If the variable is used as an auxiliary variable in the Measurement Setup, you should select Only Used For Analysis.

Then, changes to the variable by program nodes in the Measurement Setup will not be forwarded to the real-time area, but will be immediately processed instead. As a result, the variable change receives the time stamp of the current simulation time. Otherwise, the change is processed only in the real-time area, which results in a certain delay.

If the setting is neither switched on nor off, the variable is defined automatically for use only for analysis purposes, as soon as it is changed (during runtime) by a program node in the Measurement Setup. System variables have this undefined status only if they were defined in earlier program versions before this option existed.

System variables which were used for analysis can still be used by the real-time area of CANoe. However, a warning is displayed in the Write Window which says that the value changes in the analysis area won't be transmitted to the real-time area anymore.

In Offline Mode the real-time area of CANoe is not used. Thus system variables can only be changed in the Measurement Setup, and to have these changes processed directly you have to define the variable as "for analysis only".

Unit

A unit can be specified for system variables of types integer, float, integer array, and float array.

Array Size

The array size, i.e., the number of values, must be specified for the integer array and float array data types. The array size cannot be changed dynamically, i.e., the variable always contains this number of values.

Initial Value

Specify a value here to assign an initial value to the system variable. If you have activated the corresponding option in the global settings, the system variable is also reset to this initial value at every measurement start.

Depending on the data type of the system variable, the initial value is specified as follows:

  • Data: Specify the values of the bytes as two-digit hexadecimal numbers that you can separate with blank spaces. Example: 01 2A FF for three bytes.

  • Integer Array: Specify the values of the array elements as Int32 values separated by semicolons. Example: 5;8;33 for an array of size 3.

  • Other data types: Specify a value within the value range of the respective data type.

Minimum and Maximum

Minimum and Maximum are used optionally to define a valid value range for the system variable. CANoe does not check this range itself when a value is set, but it uses the range, e.g., for configuring display elements. For arrays, the limits for each individual array element apply. For strings and data-type variables, no limits are possible. For integer variables, the limits can also be specified by the value table.

Value Table

For integer and integer array-type system variables, you can define a value table. You can select an existing template from the list or via the Edit Value Table dialog. The individual values of this table are then listed below this line.

Edit Value Table

Click on to open the Edit Value Table dialog.

Here, you can assign a description to each value and vice versa. The descriptions must be valid CAPL-/.NET identifiers, they must begin with a letter or underscore followed by letters, numbers or underscores only. You can edit the descriptions directly in the table and sort the table by value or description.

The changed values apply only to the system variable which you are currently editing. The changed values are not applied to the value table template.

If you want to apply the changed values to a new value table template, enter a new name into Name and then click on .

Click on to open the Value Table Templates dialog.

Minimum/Maximum from the Value Table

This check box is only available if you have selected a value table. If this option is set, the Minimum and Maximum fields are set with values from the value table and can no longer be edited manually.

Read-Only

Read-only system variables are write-protected and cannot be changed from CAPL or panels.

The default value is Writeable.

Multi CANoe Write Access

Here you select for the Multi CANoe operation the CANoe instance that can write the variable. The other CANoe instances receive automatically the value changes of this variable.

If the field is empty, it is a local system variable. Value changes are not given to the other CANoe instances.

Events on Value Assignment

If a value is assigned to a system variable an event is generated.

With this option you can set how many events will be generated if multiple values are assigned at the same point in time.

  • One Event with Last Value
    Only one event is generated with the last assigned value.
  • All Events
    For every assigned value an event is generated. All events get the same time stamp.
