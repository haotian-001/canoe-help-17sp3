System Variables

 

General

In CANoe system variables can be used for modeling internal parameters.

In contrast to environment variables, system variables describe neither external input/output signals of a node nor they are used for communication of signals between nodes in a real network
Instead, system variables are used for development and testing since they provide the opportunity to change and to analyze internal variables and parameters. They are also used to indicate signals in CANoe which are not defined in a database and which were not received via a network (e.g. I/O signals).

If, in previous versions of CANoe, an analysis was carried out by means of an environment variable, it often occurred that the time stamp of the difference value differed from the time stamp of the message. This results from the fact that environment variable changes in the analysis area of CANoe are first transmitted to the real-time area (CANoe only). The value then won't be received by the analysis area and transmitted to the Measurement Setup until it has changed in the real-time area (CANoe only).

Types of System Variables

System variables are typed. Each variable can only take values of one specific data type.
Following data types are possible:

  • Integer (32 bit signed, value range -2.147.483.648 to +2.147.483.647)
  • Integer (32 bit unsigned, value range 0 to +4.294.967.296)
  • Integer (64 bit signed, value range -9.223.372.036.854.775.808 to +9.223.372.036.854.775.807)
  • Integer (64 bit unsigned, value range 0 to +18.446.744.073.709.551.616)
  • Double (64 bit IEEE double)
  • String (variable length)
  • Data (variable number of bytes)
  • Integer array (fixed number of signed 32 bit values)
  • Double array (fixed number of 64 bit IEEE double values)
  • Struct (for both user-defined variables and variables created by the program)
  • Maps and Curves data types for XCP
  • Generic array (not for user-defined variables, but only for variables created by the program)
    You can use individual elements of a variable of type Struct or Generic Array as symbols in all places in which you can use variables itself, e.g. in stimulation windows, display windows and programs.

Name Spaces

  • System variables are defined within namespaces.
  • Each system variable belongs to one particular namespace and is uniquely defined therein (upper and lower case letters are distinguished).
  • Namespaces can also be nested. In programs they are separated by "::".

Initial Value

System variables can have an initial value that the variable assumes after it is created. The variable is also set to this initial value on measurement start, if you have selected the corresponding option in the global options dialog. If no initial value has been defined for a system variable, its value remains unchanged at the start of measurement.

Value Tables

System variables of type Integer can have value tables. Such a table assigns a symbolic description to specified values. This description â similar to value tables of signals â may be displayed e.g. in the Data Window of CANoe or may be used as a constant in CAPL.

Read and Write Access to System Variables

Some of the system variables defined by CANoe components can be read-only, e.g. variables which model received I/O signals. The values of these variables can neither be changed by means of panels nor by COM or CAPL programs.

Display and Analysis of System Variables

The values of system variables can be displayed in several CANoe program modules:

  • In the Trace Window, if you have defined the predefined filters for variables accordingly.
  • In the Graphics Window
  • In the Data Window
  • On panels which were created with the Panel Designer

System variable changes are written in logging files and recorded in macros.

Note

In the Measurement Setup changes on system variables are only effective for the branch in which the change was done.

Logging, Playing, Import and File Conversion

System variable changes are written both in frame logging files and signal logging files.

Please note the following when using frame logging files containing system variable changes:

  • If you play these frame logging files in offline mode (as import in the Trace Window or in File Conversion) the system variables must be defined in the configuration.
    If the system variable is not defined in the configuration, the system variable changes are ignored. As notification the system variable CANoe/CANalyzer::UnknownSysVar is updated. It contains the following information about the ignored system variable: the namespace and the name of the system variable change.

Please note the following when using signal logging files containing system variable changes:

  • If you import such a signal logging file in the Graphics Window, the system variable does not have to be defined in the configuration.

Hints for Using System Variables with Different CANoe Versions

The system variables have been widely enhanced in CANoe version 7.0.
In previous versions they could only be used for data exchange between COM and CAPL. You should take this into account if you are making configurations for older CANoe versions.

In CANalyzer /pro system variables are supported since version 7.2 SP3, in CANalyzer /exp since version 7.5.
