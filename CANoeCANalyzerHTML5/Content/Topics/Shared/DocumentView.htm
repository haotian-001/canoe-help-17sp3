Document View

 

Here you can see the opened CAPL programs distributed among the individual pages. The display of the individual CAPL program is divided into a Navigator and a Text Editor. The entire source code of the CAPL program is displayed in the Text Editor. Using the Navigator, you can quickly and easily navigate to the desired sections in the source code (e.g. an event handler).

The CAPL Browser monitors the symbols, i.e. database symbols, diagnostic parameters, system and environment variables are updated automatically if changes are made. Configuration data of the current CANoe configuration, such as networks, channels, nodes etc., is not updated automatically. They can be read in with the import CANoe/CANalyzer environment command from a running CANoe application.

In order for this environment information and data to be available to a CAPL program, CAPL programs should always be opened from CANoe, either by double-clicking a CAPL program node or via the Edit context menu.

In addition, you can open a new or existing program in the CAPL Browser via the File ribbon tab or the Quick Access Toolbar. When doing so, however, you must keep in mind that environment information and data on the CAPL program are not available.
If you have already opened a CAPL program with environment information and data, it is already available to you in the newly opened CAPL program as well.

Page Groups

The individual documents or files are displayed in separate pages. These pages can be arranged in groups vertically or horizontally.

Left: Single page; Right: Vertical group

You can create groups and assign individual pages to these groups using the Layout ribbon tab, the context menu of the page or by moving the page with the left mouse button.

Page Context Menu

Context Menu

Close
Closes the active file. If you have changed the displayed file and not yet saved the changes, you are asked whether the changes are to be saved.

Close All But This
Closes all files except the active one. If you have changed files and not yet saved the changes, you are asked whether the changes are to be saved for each individual program.

New Horizontal Group
Creates a new horizontal group. Individual pages and additional groups are displayed horizontally.
If a vertical group was created previously, this menu command is not available.

New Vertical Group
Creates a new vertical group. Individual pages and additional groups are displayed vertically.
If a horizontal group was created previously, this menu command is not available.

Move to Previous/Next Group
Shifts an individual page to the next/previous group.

Navigator

The Navigator displays the structure of the file as a tree view.
The following categories may be present:

  • Includes
  • Variables
  • Event handlers (event procedures)

All event handler available for the opened nodes are displayed in the Navigator. The display is adapted to the environment configured in CANoe. If a MOST bus is not used in the CANoe configuration, no MOST event handlers are displayed.

You can add or remove these filters through the Filter ribbon tab.

If the CANoe environment is changed while the CAPL Browser is running, the data in the CAPL Browser can be updated via in the ribbon.

The event handler are separated into the following categories:

  • System
    Contains event handlers for system events
  • Value Objects
    Contains event handlers for events on symbols
  • Bus Systems and Options
    Contains event handlers for bus- or option-specific events
  • User-defined functions
  • Test functions
  • Test cases
  • Test procedures

Using the Navigator, you can navigate directly to a point in the Text Editor by selecting an event handler, for example.
If you select an event handler in the Text Editor, on the other hand, it is highlighted in the Navigator.

The navigator can be opened and closed by clicking the Splitter function.

Groups

For a clear arrangement and structure of use cases you can add groups and sub-groups via the context menu for

  • Event handlers
  • User-defined functions
  • Test functions
  • Test cases
  • Test procedures

If you delete a group, the sub-entries of the group will be moved up one level, the entries will not be deleted. You can rename the groups via the context menu.

With drag and drop you can move existing functions, test cases, test procedures and event handlers to and from groups.

Updating the Navigator

When changes are made to the source code, the tree view in the Navigator is updated automatically after a brief delay. The tree view cannot be displayed if, for example, a faulty (and thus uncompilable) file is opened. In this case, you must clean the source code first so that the tree view can be displayed.

Text Editor

The source code of the opened file is displayed in the Text Editor and it can be edited there. You can declare variables and implement event handlers and functions there.

Individual sections, such as Includes, Variables or individual event procedures, for example, can be opened and closed via and . A preview indicates the content of the collapsed lines by size.

To the left of the line number set bookmarks () and breakpoints () are displayed.

Support While Editing Files

The Editor offers several options for inserting symbols, event handlers and predefined functions.

  • drag and drop from the CAPL Function Explorer
    Event handlers and predefined functions can be inserted into the Text Editor.
    If you insert a function, the name of the function will be displayed in the source code. When entering parameters support is provided by the autocomplete input assistance.
    If you insert an event handler, the entire body of the procedure is inserted.
  • Autocomplete input assistance
    The autocomplete input assistance makes it easier for you to enter source code by listing functions, symbolic messages and signals.
  • drag and drop from the Symbol Explorer / MOST Symbol Explorer
    Symbols can be inserted into the Text Editor via dragging and dropping from the Symbol Explorer / MOST Symbol Explorer.
  • Symbol Selection
    In the Symbol Selection dialog you can select signals, system variables, environmental variables, and diagnostic parameters and insert it into the Text Editor.
  • Bookmarks
    You can set bookmarks for faster locating of code lines. The bookmarks you set are displayed in the Bookmark Window and can be edited there.
  • Breakpoints
    The Breakpoint Window of the CAPL Browser displays the breakpoints that were set in the Debugger. The breakpoints can be enabled, disabled or deleted there.

Automatic Error Detection

For quick error detection before compilation, source code errors are indicated with a wavy red line in the Text Editor similar to that used for spelling errors in Word®, for example. In addition, a possible cause for the error is displayed as a tooltip.
